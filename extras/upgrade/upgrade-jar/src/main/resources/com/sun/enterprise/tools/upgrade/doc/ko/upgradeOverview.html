업그레이드 도구: 개요

Application Server 9.1 또는 GlassFish Prelude 설치에서 GlassFish Server Open Source Edition 3.1 설치로 구성 정보를 전송하려면 업그레이드 도구를 사용합니다. 전송되는 정보에는 배포된 응용 프로그램, 파일 영역, 보안 인증서, 기타 자원 및 서버 구성 설정 등에 대한 데이터가 포함됩니다.

이 도구는 지정된 소스 서버 설치 버전을 자동으로 감지합니다.

참고: 업그레이드 프로세스를 시작하기 전에 소스 서버(업그레이드할 소스 서버) 및 대상 서버(업그레이드할 대상 서버)를 모두 중지해야 합니다.

업그레이드 용어

다음은 업그레이드 프로세스와 관련된 중요 용어입니다.

  • 소스 도메인 디렉토리: 새 서버로 업그레이드할 이전 도메인입니다. 원본 도메인은 변경되지 않고 복사되며 복사본이 업그레이드됩니다.

  • 대상 도메인 루트 디렉토리: 업그레이드할 대상 설치의 도메인 디렉토리입니다. 어떤 디렉토리든지 이 루트 디렉토리가 될 수 있지만 소스 도메인이 SERVER_HOME/domains 디렉토리에서 다른 디렉토리로 이동되는 경우에는 타사 jar 파일을 소스 서버에서 대상 서버로 복사할 수 있습니다.

  • 마스터 비밀번호: 대상 서버의 마스터 비밀번호입니다. 비워 두면 기본값이 적용됩니다.
