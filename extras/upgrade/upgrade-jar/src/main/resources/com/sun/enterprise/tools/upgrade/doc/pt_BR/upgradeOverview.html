Upgrade Tool: visão geral

Use a Ferramenta de atualização para transferir informações de configuração de uma instalação do Application Server 9.1 para uma instalação do servidor GlassFish Open Source Edition 3.1. As informações transferidas constam de dados sobre aplicativos implantados, o reino do aplicativo, certificados de segurança e outras configurações de recursos e servidores.

A ferramenta detecta automaticamente a versão da instalação especificada do servidor de origem.

Observação: antes de iniciar o processo de atualização, certifique-se de que o servidor de origem (o servidor a partir do qual está realizando a atualização) e o servidor de destino (o servidor ao qual está realizando a atualização) estejam interrompidos.

Terminologia do processo de atualização

Encontram-se abaixo importantes termos relacionados ao processo de atualização:

  • Diretório de origem do domínio: domínio antigo que será atualizado com o novo servidor. O domínio original não é alterado. Ele é copiado e a cópia é atualizada.

  • Diretório raiz de destino dos domínios: diretório dos domínios de instalação para o qual você está atualizando. Pode ser qualquer diretório, mas se o domínio de origem estiver sendo movido de um SERVER_HOME/diretório de domínios para outro, os arquivos jar de terceiros podem ser copiados do servidor de origem para o servidor de destino.

  • Senha mestre: senha mestre do servidor de destino. Deixe em branco por padrão.
