Strumento di aggiornamento: panoramica

Utilizzare lo strumento di aggiornamento per trasferire le informazioni di configurazione dall'installazione di Application Server 9.1 o GlassFish Prelude a un'installazione di GlassFish Server Open Source Edition 3.1. Tra i dati trasferiti vi sono le informazioni relative alle applicazioni distribuite, al settore dei file, ai certificati di sicurezza e ad altre impostazioni della configurazione delle risorse e del server.

Lo strumento rileva automaticamente la versione dell'installazione del server di origine specificata.

Nota: prima di avviare il processo di aggiornamento, verificare che sia il server di origine (ossia il server dal quale si esegue l'aggiornamento) e il server di destinazione (il server nel quale si esegue l'aggiornamento) vengano arrestati.

Terminologia relativa all'aggiornamento

Vengono spiegati di seguito alcuni termini importanti per il processo di aggiornamento:

  • Directory del dominio di origine: il vecchio dominio che verrà aggiornato al nuovo server. Il dominio originale non viene modificato; ne viene eseguita una copia che viene quindi aggiornata.

  • Directory radice del dominio di destinazione: la directory del dominio è l'installazione nella quale si esegue l'aggiornamento. Può essere una directory qualsiasi, ma se il dominio di origine viene spostato da una directory SERVER_HOME/domains all'altra, i file jar di terze parti possono essere copiati dal server di origine a quello di destinazione.

  • Password principale: la password principale del server di destinazione. Lasciare vuoto per la password predefinita.
