Herramienta de actualización: descripción general

Utilice la herramienta de actualización para transferir la información de configuración de una instalación de Application Server 9.1. o GlassFish Prelude a una instalación de GlassFish Server Open Source Edition 3.1. La información que se transfiere incluye datos sobre las aplicaciones implementadas, el ámbito de archivos, los certificados de seguridad y otros recursos y opciones de configuración del servidor.

La herramienta detecta automáticamente la versión de la instalación de servidor de origen especificada.

Nota: antes de iniciar el proceso de actualización, asegúrese de que tanto el servidor de origen (el servidor que está actualizando) como el servidor de destino (el servidor al que está actualizando) están detenidos.

Terminología de actualización

A continuación se muestran algunos términos importantes relacionados con el proceso de actualización:

  • Directorio del dominio de origen: el dominio antiguo que se actualizará al nuevo servidor. El dominio original no cambia, sino que se copia y se actualiza la copia.

  • Directorio root de dominios de destino: directorio de dominios de instalación al que se va a actualizar. Puede ser cualquier directorio, pero si el dominio de origen se mueve de un directorio SERVER_HOME/domains a otro, se pueden copiar archivos jar de terceros del servidor de origen al servidor de destino.

  • Contraseña maestra: la contraseña maestra del servidor de destino. Déjela en blanco para usar la predeterminada.
