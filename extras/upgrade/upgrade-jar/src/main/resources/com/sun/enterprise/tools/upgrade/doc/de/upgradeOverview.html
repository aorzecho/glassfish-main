Das Upgrade-Tool: Ein Überblick

Mit dem Upgrade-Tool können Sie Konfigurationsinformationen von einer Application Server 9.1.- oder einer GlassFish Prelude-Installation auf eine Installation von GlassFish Server Open Source Edition 3.1 übertragen. Die übertragenen Informationen beinhalten Daten über bereitgestellte Anwendungen, den Dateibereich, Sicherheitszertifikate und andere Ressourcen- und Serverkonfigurationseinstellungen.

Das Tool erkennt automatisch die Version der angegebenen Quellserver-Installation.

Hinweis: Stellen Sie vor Beginn des Upgrade-Vorgangs sicher, dass sowohl der Quellserver (der Server, von dem aktualisiert wird) als auch der Zielserver (der Server, auf den aktualisiert wird) angehalten wurde.

Upgrade-Terminologie

Im Folgenden sind wichtige Begriffe im Zusammenhang mit dem Upgrade-Prozess aufgeführt:

  • Quelldomänenverzeichnis: die alte Domäne, die auf den neuen Server aktualisiert wird. Die ursprüngliche Domäne wird nicht geändert, sie wird kopiert und die Kopie wird aktualisiert.

  • Ziel-Domänen-Root-Verzeichnis: das Domänenverzeichnis der Installation, auf die Sie aktualisieren. Dies kann ein beliebiges Verzeichnis sein. Wird jedoch die Quelldomäne aus einem SERVER_HOME/domains-Verzeichnis in das andere verschoben, können JAR-Dateien von Drittanbietern vom Quellserver zum Zielserver kopiert werden.

  • Master-Passwort: das Master-Passwort des Zielservers. Wenn Sie das Feld leer lassen, wird ein Standardwert verwendet.
