Outil de mise à niveau : présentation

L’outil de mise à niveau permet de transférer des informations de configuration d’une installation Application Server 9.1 ou GlassFish Prelude vers un serveur GlassFish v 3.1 édition Open Source. Les informations transférées incluent des données sur les applications déployées, le domaine du fichier, les certificats de sécurité et d’autres ressources et paramètres de configuration du serveur.

L’outil détecte automatiquement la version de l’installation de serveur source spécifiée.

Remarque : avant de lancer le processus de mise à niveau, assurez-vous que le serveur source (le serveur à partir duquel vous effectuez la mise à niveau) et le serveur cible (le serveur vers lequel vous effectuez la mise à niveau) sont arrêtés.

Terminologie relative à la mise à niveau

Vous trouverez ci-dessous la liste des termes importants associés au processus de mise à niveau :

  • Répertoire du domaine source : l’ancien domaine, qui sera mis à niveau sur le nouveau serveur. Le domaine d’origine n’est pas modifié ; il est copié et la copie est mise à niveau.

  • Répertoire racine de domaines cible : le répertoire d’installation des domaines, dans lequel vous effectuez la mise à niveau. Il peut s’agir de tout répertoire, mais si le domaine source est déplacé d’un répertoire SERVER_HOME/domains vers un autre, des fichiers JAR tiers peuvent être copiés du serveur source vers le serveur cible.

  • Mot de passe principal : mot de passe principal du serveur cible. Laissez ce champ vide pour utiliser la valeur par défaut.
