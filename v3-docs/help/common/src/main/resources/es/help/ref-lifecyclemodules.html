Módulos de ciclo de vida

Utilice la página Módulos de ciclo de vida para configurar los módulos de ciclo de vida.

Un módulo de ciclo de vida realiza tareas cuando es activado por uno o más eventos del ciclo de vida del servidor. Los posibles eventos del servidor que pueden activarlo son: inicialización, arranque, disponible para atender peticiones y apagado. Los módulos de ciclo de vida no forman parte de la especificación de Java EE, sino que son una mejora de Servidor GlassFish.

Se proporciona la siguiente información para cada módulo de ciclo de vida.

Nombre

Nombre del módulo de ciclo de vida.

Activado o Estado

Una marca de verificación si la aplicación está habilitada o una X si está inhabilitada, en el caso de que solo exista la instancia de servidor predeterminada, server.

Si existen clústeres u otras instancias de servidores independientes, seleccione Habilitado en Todos los destinos para ver los destinos en los que está implementada la aplicación.

Orden de carga

Valor que especifica el orden relativo en el que el módulo de ciclo de vida debería cargarse.

La tabla Módulos de ciclos de vida también contiene las siguientes opciones.

Nuevo

Botón para crear un nuevo módulo de ciclo de vida.

Borrar

Botón para eliminar uno o más módulos de ciclo de vida seleccionados.

Habilitar

Botón para habilitar uno o más módulos de ciclo de vida seleccionados. Solo está presente si existe la instancia de servidor predeterminada, server.

Deshabilitar

Botón para deshabilitar uno o más módulos de ciclo de vida seleccionados. Solo está presente si existe la instancia de servidor predeterminada, server.
