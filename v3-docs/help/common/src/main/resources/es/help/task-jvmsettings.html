Definir la Configuración General de JVM

Java Virtual Machine (JVM) permite la ejecución de los códigos de bytes de Java en una máquina física. Al ajustar la configuración de JVM, mejora el rendimiento y se evitan errores de asignación de memoria.

Java Virtual Machine se incluye en la plataforma Java, Standard Edition (plataforma Java SE), que necesita Servidor GlassFish. La introducción de una configuración de JVM incorrecta hará que no funcione el servidor, por lo que tiene que ir con cuidado a la hora de cambiar esta configuración.

Nota:

Los términos "Java Virtual Machine" y "JVM" se refieren a una máquina virtual para la plataforma Java.

  1. In the navigation tree, expand the Configuration node.

  2. Select the configuration that you are changing.

  3. Select the JVM Settings node.

    The JVM General Settings tab page opens.

  4. En el árbol de navegación, seleccione el nodo Configuración.

  5. En el nodo Configuración, seleccione el nodo Configuración de JVM.

    Se abrirá la página Configuración General de JVM.

  6. En la página Configuración General de JVM, en el campo Directorio Raíz de Java, escriba el nombre del directorio de instalación del software Java SE.

    Servidor GlassFish está basado en la plataforma Java SE. Para comprobar que la versión del software de Java SE que ha especificado está soportada en esta versión, consulte Notas Técnicas sobre la Versión de Oracle GlassFish Server. Si especifica un nombre de directorio inexistente o el directorio de una versión no soportada del software Java SE, no se iniciará Servidor GlassFish.

  7. En el campo Opciones de Javac, introduzca las opciones de línea de comandos del compilador del lenguaje de programación Java.

    Servidor GlassFish ejecuta el compilador cuando se despliegan los componentes de Enterprise JavaBeans (EJB).

  8. Active la casilla de control Depuración Activada para activar la depuración con Java Platform Debugger Architecture (JPDA).

    Los desarrolladores de aplicaciones utilizan JPDA. Para obtener más información, consulte Guía de Desarrollo de la Aplicación de Oracle GlassFish Server.

  9. En el campo Opciones de Depuración, introduzca las opciones de JPDA que van a pasar a JVM.

    Este campo sólo será aplicable si se ha activado la depuración en el paso anterior.

  10. En el campo Opciones de Compilación RMI, introduzca las opciones de línea de comandos del compilador rmic.

    Servidor GlassFish ejecuta el compilador rmic cuando se despliegan componentes EJB.

  11. En el campo Preprocesador de Código de Byte, introduzca una lista de nombres de clase.

    Utilice comas para separar los nombres de clase. Cada clase debe implementar la interfaz com.sun.appserv.BytecodePreprocessor. Las clases se llaman en el orden especificado.

  12. En la sección Propiedades Adicionales, especifique las propiedades adicionales.

    Para agregar una propiedad, haga clic en el botón Agregar Propiedad. En la fila en blanco que aparece, escriba el nombre de propiedad en el campo Nombre y escriba el valor de propiedad en el campo Valor.

    Servidor GlassFish no define ninguna propiedad adicional para JVM.

  13. Haga clic en Guardar.

    Sugerencia:

    Para volver a la configuración por defecto, haga clic en el botón Cargar Valores por Defecto y, a continuación, en Guardar.

  14. Reinicie el servidor.
