Creación de un módulo de ciclo de vida

  1. En el árbol de navegación, seleccione el nodo Módulos de ciclo de vida.

    Se abrirá la página Módulos de ciclo de vida.

  2. En la página Módulos de ciclo de vida, haga clic en Nuevo.

    Se abrirá la página Nuevo módulo de ciclo de vida.

  3. En el campo Nombre, introduzca un nombre para módulo de ciclo de vida.
  4. En el campo Nombre de clase, escriba el nombre completo del archivo de clase del módulo de ciclo de vida.

    La clase debe implementar la interfaz com.sun.appserv.server.LifecycleListener.

  5. En el campo ruta de clase, introduzca la ruta de clase para el módulo de ciclo de vida.

    La ruta de clase especifica dónde está ubicado el módulo de ciclo de vida. El valor predeterminado es domain-dir/applications. Si el módulo ya se encuentra en este directorio (es decir, en la ruta de clase del servidor), este campo puede dejarse en blanco.

  6. En el campo Orden de carga, introduzca un valor que especifique el orden en que este módulo de ciclo de vida se cargará en el arranque.

    Los módulos con valores de orden de carga inferiores se cargarán antes. El valor puede estar entre 101 y el MAXINT del sistema operativo. Los valores de 0 a 100 están reservados.

  7. En el campo Descripción, introduzca una descripción del módulo de ciclo de vida.
  8. Seleccione la casilla de verificación Estado habilitado para habilitar el módulo de ciclo de vida.

    Esta opción está habilitada por defecto.

  9. Seleccione la casilla de verificación Arranque de instancia para impedir fallo en la carga para especificar que el servidor debe apagarse si se produce un error en la carga del módulo de ciclo de vida.

    Esta opción se deshabilita por defecto.

  10. Para agregar instancias de servidor y clústeres de destino en los que habilitar el módulo de ciclo de vida, selecciónelos en la lista Destinos disponibles y utilice el botón Agregar para moverlos a la lista Destinos seleccionados.

    La opción Destino aparece si existen clústeres o instancias independientes del servidor aparte de la instancia de servidor predeterminada (server). De manera predeterminada, el módulo de ciclo de vida está habilitado en la instancia de servidor predeterminada, denominada server. Utilice el botón Agregar todos para implementar en todos los destinos. Utilice los botones Eliminar y Eliminar todos para eliminar destinos de la lista Destinos seleccionados.

  11. Haga clic en Aceptar.
