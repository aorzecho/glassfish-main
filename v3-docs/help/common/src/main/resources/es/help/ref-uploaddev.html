Desplegar Aplicaciones o Módulos

Utilice la página Desplegar Aplicaciones o Módulos para desplegar una aplicación.

La página Desplegar Aplicaciones o Módulos contiene las opciones siguientes para todas las aplicaciones. Únicamente aparecen más opciones en caso de que especifique el campo Ubicación para la aplicación.

Ubicación

Ubicación del archivo para la aplicación que está desplegando.

Las opciones siguientes especifican desde dónde se puede acceder al archivo y si es un archivo o un directorio.

Archivo Empaquetado que Cargar en el Servidor

El archivo reside en la máquina cliente o se puede acceder a él desde la misma.

La máquina cliente es el host en el que está viendo la Consola de Administración por medio de un explorador.

Archivo empaquetado local o directorio accesible desde Servidor GlassFish

El archivo reside en la máquina del servidor o es una aplicación desempaquetada en un directorio separado.

La máquina del servidor es el host que está ejecutando el servidor de administración de dominio de Servidor GlassFish.

Tipo

Tipo de aplicación. Las opciones disponibles son:

  • Aplicación Web

  • Aplicación Empresarial

  • Cliente de Aplicaciones

  • Módulo del Conector

  • JAR EJB

  • Otro

Destinos

Clusters e instancias independientes en los que desplegar la aplicación o módulo. Mueva los destinos deseados a la columna Destinos Seleccionados mediante los botones Agregar y Agregar Todos. Mueva los destinos que no necesite a la columna Destinos Disponibles mediante los botones Eliminar y Eliminar Todos.

Esta opción sólo se muestra si se han creado clusters o instancias independientes en el dominio.

Las opciones adicionales para los distintos tipos de aplicación se describen en las siguientes secciones.

  • Opciones Adicionales para una Aplicación Web

  • Opciones Adicionales para una Aplicación Empresarial

  • Opciones Adicionales para un Cliente de Aplicaciones

  • Opciones Adicionales para un Módulo del Conector

  • Opciones Adicionales para un JAR EJB

Opciones Adicionales para una Aplicación Web

Si el tipo de aplicación corresponde a una aplicación web, aparecen las siguientes opciones.

Raíz de Contexto

Ruta de acceso a la aplicación. En la URL de la aplicación web, la raíz de contexto sigue inmediatamente al número de puerto (http://host:puerto/raíz-contexto/...). La raíz de contexto debe comenzar por una barra diagonal, por ejemplo, /hello.

Nombre de Aplicación

Nombre de la aplicación.

El nombre puede incluir un identificador de versión opcional, que va a continuación del nombre y está separado de él por dos puntos (:). El identificador de versión debe comenzar por una letra o número. Puede contener caracteres alfanuméricos más los caracteres de subrayado (_), guión (-) y punto (.). Para obtener más información sobre las versiones de módulo y de aplicación, consulte el apartado sobre las versiones de módulo y de aplicación en Oracle GlassFish Server Application Deployment Guide.

Servidores Virtuales

Servidores virtuales asociados a esta aplicación.

La opción Servidores Virtuales aparece si solo existe la instancia de servidor por defecto, server. Si existen clusters u otras instancias de servidor independientes, puede seleccionar los servidores virtuales después del despliegue. Vaya a la página Editar Aplicación, seleccione el separador Destino y seleccione Gestionar Servidores Virtuales para el destino que desee.

Estado

Si esta opción está seleccionada, entonces la aplicación está activada. Esta opción está seleccionada por defecto.

Disponibilidad

Si se activa la casilla de control Activado, se activa la alta disponibilidad para las sesiones web, la aplicación de puntos de control de bean de sesión con estado (SFSB) y posiblemente para la desactivación. Si se define en false (el valor por defecto), se desactiva el guardado de sesiones web y los puntos de control de SFSB para la aplicación, aplicación web o módulo EJB especificado. Si se define en true, la aplicación o módulo especificados se activa para alta disponibilidad. Defina esta opción como true sólo si se ha configurado y activado la alta disponibilidad en niveles superiores, como, por ejemplo, los niveles de servidor y contenedor.

Esta opción aparece si existen clusters o instancias del servidor independientes aparte de la instancia de servidor por defecto (server).

Precompilar JSP

Si está seleccionada esta opción, los archivos JavaServer Pages (JSP) se precompilarán. Si está desactivada esta opción, los archivos JSP se compilarán en tiempo de ejecución cuando se acceda a los mismos por primera vez. Esta opción está desactivada por defecto.

Ejecutar Verificador

Si esta opción está seleccionada, los descriptores de despliegue se verifican antes del despliegue. Si se produce un fallo en la verificación, no se realiza el despliegue. El verificador examinará la estructura y el contenido de los descriptores de despliegue. La verificación de aplicaciones de gran tamaño tiende a llevar mucho tiempo. Esta opción está desactivada por defecto.

Los paquetes de verificador se deben haber instalado desde la Herramienta de Actualización o se registrará una advertencia y se ignorará esta opción.

Forzar Nuevo Despliegue

Si esta opción está seleccionada, se volverá a desplegar la aplicación si está ya desplegada. Si esta opción no está seleccionada y se intenta desplegar una aplicación que ya se haya desplegado, se producirá un error. Esta opción está desactivada por defecto.

Mantener Estado

Esta opción controla si las sesiones web, instancias SFSB y temporizadores EJB creados de forma persistente se conservan entre los nuevos despliegues.

Esta opción está desactivada por defecto. Esta opción sólo se soporta en la instancia de servidor por defecto, denominada server. No se soporta para ningún otro destino y se ignorará.

Algunos cambios efectuados en una aplicación entre nuevos despliegues impiden que esta opción funcione correctamente. Por ejemplo, no cambie el juego de variables de instancia en la clase de bean SFSB.

Para las aplicaciones web, esta función solo es aplicable si, en el archivo glassfish-web-app.xml, el atributo persistence-type del elemento session-manager es file.

Para instancias de bean de sesión con estado, el tipo de persistencia sin alta disponibilidad se define en el servidor (con la opción Tipo de Persistencia SFSB) y se debe definir en file, que es el valor por defecto y recomendado.

Si una sesión web, instancia SFSB o temporizador EJB activo no se conserva o restaura, ninguno de ellos estará disponible cuando se haya completado el nuevo despliegue. Sin embargo, el nuevo despliegue proseguirá y se registrará una advertencia.

Para conservar datos de estado activo, Servidor GlassFish serializa los datos y los guarda en la memoria. Para restaurar los datos, el cargador de clases de la aplicación que se acaba de volver a desplegar deserializa los datos guardados anteriormente.

Conservar Recursos de Ámbito de Aplicación

Si se activa, conserva todos los recursos de ámbito restringido a la aplicación y los restaura durante el nuevo despliegue. Esta opción no está activada por defecto.

Bibliotecas

Lista de archivos JAR de la biblioteca separados por comas específicos de este módulo o aplicación. Las rutas de acceso pueden ser absolutas o relativas. Una ruta de acceso relativa es relativa a domain-dir/lib/applibs. Si la ruta de acceso es absoluta, ésta debe ser accesible para el servidor de administración de dominio (DAS), lo que implica que debe estar en domain-dir. Las bibliotecas están disponibles para la aplicación en el orden en que están especificadas.

Descripción

Descripción de la aplicación.

Compatibilidad

En concreto, los clientes de aplicaciones no deben tener acceso a los archivos JAR EJB o a otros archivos JAR del archivo EAR a menos que las referencias utilicen los mecanismos Java SE estándar (extensiones, por ejemplo) o el mecanismo biblioteca-directorio de Java EE. Al activar esta casilla se eliminan estas restricciones de Java EE 6.

Algunos cambios efectuados en una aplicación entre nuevos despliegues impiden que esta opción funcione correctamente. Por ejemplo, no cambie el juego de variables de instancia en la clase de bean SFSB.

Para las aplicaciones web, esta función solo es aplicable si, en el archivo glassfish-web-app.xml, el atributo persistence-type del elemento session-manager es file.

Para instancias de bean de sesión con estado, el tipo de persistencia sin alta disponibilidad se define en el servidor (con la opción Tipo de Persistencia SFSB) y se debe definir en file, que es el valor por defecto y recomendado.

Si una sesión web, instancia SFSB o temporizador EJB activo no se conserva o restaura, ninguno de ellos estará disponible cuando se haya completado el nuevo despliegue. Sin embargo, el nuevo despliegue proseguirá y se registrará una advertencia.

Para conservar datos de estado activo, Servidor GlassFish serializa los datos y los guarda en la memoria. Para restaurar los datos, el cargador de clases de la aplicación que se acaba de volver a desplegar deserializa los datos guardados anteriormente.

Bibliotecas

Lista de archivos JAR de la biblioteca separados por comas específicos de este módulo o aplicación. Especifica una ruta de acceso absoluta o relativa. Una ruta de acceso relativa es relativa a domain-dir/lib/applibs. Si la ruta de acceso es absoluta, ésta debe ser accesible para el servidor de administración de dominio (DAS), lo que implica que debe estar en domain-dir. Las bibliotecas están disponibles para la aplicación en el orden en que están especificadas.
