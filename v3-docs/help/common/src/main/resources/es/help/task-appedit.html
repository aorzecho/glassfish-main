Editar una Aplicación Desplegada

  1. En el árbol de navegación, seleccione el nodo Aplicaciones.

    Se abrirá la página Aplicaciones.

  2. En la tabla Aplicaciones Desplegadas, haga clic en el nombre de la aplicación que se va a editar.

    El nombre puede incluir un identificador de versión opcional, que va a continuación del nombre y está separado de él por dos puntos (:). Para obtener más información sobre las versiones de módulo y de aplicación, consulte el apartado sobre las versiones de módulo y de aplicación en Oracle GlassFish Server Application Deployment Guide.

    Se abrirá la página Editar Aplicación.

  3. Desactive la casilla de control Estado Activado para desactivar el estado de la aplicación y que no se pueda acceder a ella.

    Por defecto, la aplicación presenta el estado Activado una vez desplegada.

  4. Para una aplicación web, en el campo Raíz de Contexto, introduzca la ruta de acceso de la aplicación.

    En la URL de la aplicación web, la raíz de contexto sigue inmediatamente al número de puerto (http://host:puerto/raíz-contexto/...). Asegúrese de que la raíz de contexto empieza con una barra inclinada, por ejemplo, /hello.

  5. Desactive la casilla de control Disponibilidad Activada para desactivar la persistencia de sesiones y los puntos de control de SFSB para la aplicación.

    Si se activa la casilla de control Activado, se activa la alta disponibilidad para las sesiones web, la aplicación de puntos de control de bean de sesión con estado (SFSB) y posiblemente para la desactivación. Si se define en false (el valor por defecto), se desactiva el guardado de sesiones web y los puntos de control de SFSB para la aplicación, aplicación web o módulo EJB especificado. Si se define en true, la aplicación o módulo especificados se activa para alta disponibilidad. Defina esta opción como true sólo si se ha configurado y activado la alta disponibilidad en niveles superiores, como, por ejemplo, los niveles de servidor y contenedor.

    Esta opción aparece si existen clusters o instancias del servidor independientes aparte de la instancia de servidor por defecto (server).

  6. En la lista desplegable Servidores Virtuales, seleccione el servidor virtual asociado a esta aplicación.

    La opción Servidores Virtuales aparece si solo existe la instancia de servidor por defecto, server. Si existen clusters u otras instancias de servidor independientes, puede seleccionar los servidores virtuales después del despliegue. Vaya a la página Editar Aplicación, seleccione la ficha Destino y seleccione Gestionar Servidores Virtuales para el destino que desee.

  7. Active la casilla de control Java Web Start Activado para usar el software Java Web Start para desplegar la aplicación.

    Esta opción está activada o desactivada por defecto en función de su configuración en el despliegue original.

  8. En el campo Descripción, escriba una descripción de esta aplicación.

  9. Para ver la lista de los descriptores de módulos de la aplicación, haga clic en el separador Descripción. Para obtener más información, consulte Ver los Descriptores de una Aplicación

  10. Por defecto, el estado de una aplicación es Activado en todos los destinos. Para ver los destinos en los que se ha activado la aplicación, haga clic en el separador Destinos. Puede activar o desactivar aplicaciones en uno o en varios destinos. Consulte Ver los Destinos de Aplicaciones.

  11. Haga clic en Guardar.
