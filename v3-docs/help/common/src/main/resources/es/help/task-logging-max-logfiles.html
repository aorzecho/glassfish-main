Cambiar el Límite del Número de Archivos Log Rotados

Cuando Servidor GlassFish rota un archivo log, crea un nuevo archivo vacío denominado server.log y cambia el nombre del archivo antiguo con el formato server.log_date, donde date corresponde a la fecha y la hora de rotación del archivo.

Por defecto, Servidor GlassFish limita a 10 el número de archivos log rotados. Al llegar a este límite, el archivo log más antiguo se suprime cuando Servidor GlassFish vuelve a rotar un archivo log.

Si es necesario, puede cambiar el límite del número de archivos log rotados. Cualquier límite nuevo que se define se aplica a archivos log rotados para acceder al registro y al log del servidor.

  1. Defina la propiedad del sistema com.sun.enterprise.server.logging.max_history_files para que mantenga el número máximo de archivos log rotados.

    Para obtener información sobre cómo definir una propiedad del sistema, consulte Gestionar Propiedades del Sistema.

    El comportamiento de la propiedad del sistema com.sun.enterprise.server.logging.max_history_files es el siguiente:

    • Si la propiedad no está definida, Servidor GlassFish mantiene un máximo de 10 archivos log rotados.

    • Si la propiedad se define en un número no válido o nulo, Servidor GlassFish mantiene un máximo de 10 archivos log rotados.

    • Si la propiedad se define en 0, Servidor GlassFish no mantiene archivos log rotados.
