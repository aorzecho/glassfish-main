Para cambiar el límite del número de archivos de registro rotados

Cuando Servidor GlassFish rota un archivo de registro, crea un nuevo archivo denominado server.log y renombra el archivo antiguo con el formato server.log_ fecha, en el cual fecha corresponde al valor temporal de fecha y hora de rotación del archivo.

De forma predeterminada, Servidor GlassFish limita a 10 el número de archivos de registro rotados. Al llegar a este límite, el archivo de registro más antiguo se elimina cuando Servidor GlassFish vuelve a rotar un archivo de registro.

Si es necesario, puede cambiar el límite del número de archivos de registro rotados. Cualquier límite nuevo que se establece se aplica a archivos de registro rotados para acceder al registro y al registro del servidor.

  • Defina la propiedad del sistema com.sun.enterprise.server.logging.max_history_files para que mantenga el número máximo de archivos de registro.

    Para obtener información sobre cómo definir una propiedad del sistema, consulte Administración de propiedades del sistema.

    El comportamiento de la propiedad del sistema com.sun.enterprise.server.logging.max_history_files es el siguiente:

    • Si la propiedad no está definida, Servidor GlassFish mantiene un máximo de 10 archivos de registro rotados.

    • Si la propiedad se define en un número no válido o nulo, Servidor GlassFish mantiene un máximo de 10 archivos de registro rotados.

    • Si la propiedad se establece en 0, Servidor GlassFish no mantiene archivos de registro rotados.
