Editar Configuración de SSL para el Conector JMX

Antes de Empezar

Esta tarea es relevante únicamente en caso de que la seguridad esté activada para el conector JMX.

  1. En el árbol de navegación, amplíe el nodo Configuración.

  2. En el nodo Configuración, seleccione el nodo Servicio de Administración.

    Se abrirá la página Editar Conector JMX.

  3. Active la casilla de control Seguridad Activada para que la comunicación JMX esté cifrada.

    Esta opción está desactivada por defecto.

  4. Haga clic en el separador SSL.

    Se abrirá la página SSL.

  5. En la página SSL, active la casilla de control SSL3 Activado para activar SSL3.

    Esta opción está activada por defecto.

  6. Active la casilla de control TLS Activado para activar la seguridad de la capa de transporte (TLS).

    Esta opción está activada por defecto.

  7. Active la casilla de control Autenticación de Cliente Activada para solicitar a los clientes que se identifiquen en el servidor con cada solicitud.

    Esta opción está desactivada por defecto.

  8. En el campo Apodo de Certificado, introduzca el apodo del certificado del servidor en la base de datos de certificados o token PKCS#11.

    En el certificado, el formato del nombre es tokenname:nickname. Incluyendo el tokenname: parte del nombre de este atributo es opcional.

  9. En el campo Almacén de Claves, introduzca el nombre del archivo de almacén de claves (por ejemplo, keystore.jks).

  10. En el campo Algoritmo de Confianza, introduzca el nombre del algoritmo de gestión de confianza (por ejemplo, PKIX) que se utilizará para la validación de rutas de acceso certificación.

  11. En el campo Longitud de Certificado Máxima, introduzca el número máximo de certificados intermedios no autoemitidos que puede haber en una ruta de acceso de certificación.

    Este campo se utiliza solamente en caso de que el campo Algoritmo de Confianza esté configurado en PKIX. Un valor de 0 implica que la ruta de acceso únicamente puede contener un certificado. Un valor de -1 implica que la longitud de la ruta de acceso no está restringida (no hay un máximo). La configuración en un valor inferior a -1 inicia una excepción.

  12. En el campo Almacén de Confianza, introduzca el nombre de un archivo del almacén de confianza (por ejemplo, cacerts.jks).

  13. En el área Conjuntos de Cifrado, especifique los conjuntos de cifrado que se utilizarán.

    Si no agrega ningún conjunto de cifrado, se utilizarán todos los conjuntos de cifrado.

  14. Haga clic en Aceptar.
