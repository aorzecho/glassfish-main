Nuevo Módulo de Ciclo de Vida

Consulte la página Nuevo Módulo de Ciclo de Vida para definir un nuevo módulo de ciclo de vida.

La página Nuevo Módulo de Ciclo de Vida contiene las siguientes opciones.

Nombre

Nombre del módulo de ciclo de vida.

Nombre de Clase

Nombre completo del archivo de clase del módulo de ciclo de vida. La clase debe implementar la interfaz com.sun.appserv.server.LifecycleListener.

Classpath

Classpath para el módulo de ciclo de vida. La classpath especifica la ubicación del módulo de ciclo de vida. El valor por defecto es domain-dir/applications. Si el módulo ya se encuentra en este directorio (es decir, en la Classpath del servidor), este campo puede dejarse en blanco.

Orden de Carga

Orden en el que este módulo de ciclo de vida se cargará al inicio. Los módulos con valores de orden de carga enteros inferiores se cargarán antes. El valor puede estar entre 101 y el MAXINT del sistema operativo. Los valores de 0 a 100 están reservados.

Descripción

Descripción del módulo de ciclo de vida.

Estado

Si se selecciona esta opción, el módulo de ciclo de vida estará activado. Esta opción está activada por defecto.

Ante Fallo de Carga

Si se selecciona esta opción, el servidor se apagará si se produce un fallo en la carga del módulo de ciclo de vida. Esta opción está desactivada por defecto.

Destinos

Clusters e instancias independientes para el módulo de ciclo de vida. Mueva los destinos deseados a la columna Destinos Seleccionados mediante los botones Agregar y Agregar Todos. Mueva los destinos que no necesite a la columna Destinos Disponibles mediante los botones Eliminar y Eliminar Todos.

Esta opción sólo se muestra si se han creado clusters o instancias independientes en el dominio.
