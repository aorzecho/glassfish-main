Nuevo módulo de ciclo de vida

Consulte la página Nuevo módulo de ciclo de vida para definir un nuevo módulo de ciclo de vida.

La página Nuevo módulo de ciclo de vida contiene las siguientes opciones.

Nombre

Nombre del módulo de ciclo de vida.

Nombre de clase

Nombre completo del archivo de clase del módulo de ciclo de vida. La clase debe implementar la interfaz com.sun.appserv.server.LifecycleListener.

Ruta de clase

Ruta de clase para el módulo de ciclo de vida. La ruta de clase especifica la ubicación del módulo de ciclo de vida. El valor predeterminado es dir_dominio/applications. Si el módulo ya se encuentra en este directorio (es decir, en la ruta de clase del servidor), este campo puede dejarse en blanco.

Orden de carga

Orden en el que este módulo de ciclo de vida se cargará en el arranque. Los módulos con valores de orden de carga inferiores se cargarán antes. El valor puede estar entre 101 y el MAXINT del sistema operativo. Los valores de 0 a 100 están reservados.

Descripción

Descripción del módulo de ciclo de vida.

Estado

Si se selecciona esta opción, el módulo de ciclo de vida estará habilitado. Esta opción está habilitada por defecto.

Error en la carga

Si se selecciona esta opción, el servidor se apagará si se produce un error en la carga del módulo de ciclo de vida. Esta opción se deshabilita por defecto.

Destinos

Clústeres e instancias independientes para el módulo de ciclo de vida. Mueva los destinos seleccionados a la columna Destinos seleccionados mediante los botones Agregar y Agregar todos. Mueva los destinos que no necesite a la columna Destinos disponibles mediante los botones Eliminar y Eliminar todos.

Esta opción sólo se muestra si se han creado clústeres o instancias independientes en el dominio.
