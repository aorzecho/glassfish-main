Configuración de Registrador

Utilice la página Configuración de Registrador para configurar el registro para la instancia o cluster de Servidor GlassFish seleccionado.

La página Configuración de Registrador contiene las siguientes opciones.

Nombre de Configuración

Nombre de la configuración a la que se aplican los valores de esta página. Este campo es de sólo lectura.

Escribir en el log del sistema

Si esta opción está activada, la salida de registro se envía al recurso syslog además de al log del servidor. Esta función sólo está disponible en sistemas Solaris y Linux. Esta opción está desactivada por defecto.

Archivo Log

Nombre o ubicación alternativa para el archivo log del servidor. La ubicación por defecto es domain-dir/logs/server.log.

Manejador de Logs

Nombre de clase absoluto de un manejador de log personalizado. Un manejador de registro personalizado le permite enviar logs a un destino distinto a server.log o syslog. El manejador personalizado debe comprender la clase java.util.logging.Handler (una API compatible con JSR 047). Indique también la clase de manejador en la classpath Servidor GlassFish para que el manejador se instale durante el inicio del servidor.

Límite de Rotación de Archivo

Tamaño en bytes que el archivo log del servidor debe alcanzar antes de rotar. Cuando el log del servidor alcanza el tamaño especificado, Servidor GlassFish crea un nuevo archivo vacío llamado server.log y cambia el nombre del archivo antiguo server.log_date, donde date es la fecha y hora en la que se produjo la rotación del archivo. El valor por defecto es 2 megabytes. El valor mínimo para el límite es 500 kilobytes; si se especifica un valor inferior, el archivo rota cuando alcanza el tamaño de 500 KB. Para desactivar la rotación del archivo log, defina el valor en 0.

Límite Temporal de Rotación de Archivo

Minutos que transcurren antes de que el log del servidor tenga que rotar. El valor por defecto es 0, lo que significa que el archivo rota cuando alcanza el tamaño especificado en el campo Límite de Rotación de Archivo. Si especifica uno o más minutos, el límite de tiempo tiene prioridad sobre el límite de tamaño especificado en el campo Límite de Rotación de Archivo.

Frecuencia de Vaciado

Número máximo de mensajes que se escribirán desde la cola en el log del servidor a la vez. El valor por defecto es 1.
