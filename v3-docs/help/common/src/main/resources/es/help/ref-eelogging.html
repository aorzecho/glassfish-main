Configuración de registro

Utilice la página Configuración de registro para configurar el registro para la instancia o clúster de Servidor GlassFish seleccionado.

La página Configuración de registro contiene las siguientes opciones.

Configuration Name

El nombre de la configuración a la que aplicar los valores de esta página. Este campo es de sólo lectura.

Escribir en el registro del sistema

Si esta opción está seleccionada, la salida de registro se envía al recurso syslog además de al registro del servidor. Esta característica sólo está disponible en sistemas Solaris y Linux. Esta opción se deshabilita por defecto.

Archivo de registro

Nombre o ubicación alternativa para el archivo de registro del servidor. La ubicación predeterminada es glassfish/domains/domain1/logs/server.log.

Controlador de registro

Nombre de clase absoluto de un controlador de registro personalizado. Un controlador de registro personalizado le permite enviar registros a un destino diferente de server.log o syslog. El controlador personalizado debe comprender la clase java.util.logging.Handler (una API compatible con JSR 047). Indique también la clase de controlador en la ruta de clase Servidor GlassFish para que el controlador se instale durante el arranque del servidor.

Límite de rotación de archivo

Tamaño en bytes que el archivo de registro del servidor debe alcanzar antes de rotar. Cuando el registro del servidor alcanza el tamaño especificado, Servidor GlassFish crea un nuevo archivo vacío llamado server.log y cambia el nombre del archivo antiguo server.log_date, donde date es la fecha y hora en la que se produjo la rotación del archivo. El valor predeterminado es 2 megabytes. El valor mínimo para el límite es 500 kilobytes; si se especifica un valor inferior, el archivo rota cuando alcanza el tamaño de 500 KB. Para deshabilitar la rotación del archivo de registros, defina el valor en 0.

Límite temporal de rotación de archivo

Minutos que transcurren antes de que el archivo del servidor tenga que rotar. El valor predeterminado es 0, lo que significa que el archivo rota cuando alcanza el tamaño especificado en el campo Límite de rotación de archivo. Si especifica uno o más minutos, el límite de tiempo tiene prioridad sobre el límite de tamaño especificado en el campo Límite de rotación de archivo.

Vaciar Frecuencia

El número máximo de mensajes que se escribirán desde la cola en el registro del servidor a la vez. El valor predeterminado es 1.
