Seguridad

Utilice la página Seguridad para establecer las propiedades de seguridad para la instancia o clúster de Servidor GlassFish seleccionado.

La página Seguridad contiene las siguientes opciones.

Configuration Name

El nombre de la configuración a la que aplicar los valores de esta página. Este campo es de sólo lectura.

Administrador de seguridad

Si se selecciona esta opción, se activará el administrador de seguridad para el dominio. Esta opción se encuentra deshabilitada de forma predeterminada.

Si se activa esta opción, se agregará una opción de JVM, -Djava.security.manager , a los valores de configuración de JVM de Servidor GlassFish. Consulte Para configurar las opciones de JVM Debe reiniciar el servidor para poder hacer este cambio.

Asegúrese de que ha otorgado los permisos correctos para todas las aplicaciones. Puede deshabilitar el administrador de seguridad para mejorar el rendimiento.

Registro de auditoría

Si esta opción está activada, el servidor cargará y ejecutará todos los módulos de auditoría especificados en la configuración de Módulos de auditoría. Si la opción está desactivada, el servidor no accederá a los módulos de auditoría. Esta opción se encuentra deshabilitada de forma predeterminada.

Dominio predeterminado

El dominio activo (predeterminado) que el servidor utiliza para autenticación. Las aplicaciones utilizan este dominio a menos que su descriptor de implementación especifique un dominio distinto. Todos los dominios configurados aparecen en la lista. El valor predeterminado es file.

Principal predeterminado

Especifica el nombre de usuario predeterminado. El servidor utiliza este nombre de usuario cuando no se proporciona ningún otro principal. Si introduce un valor en este campo, tiene que introducir la contraseña correspondiente en el campo Contraseña principal predeterminada.

No se requiere un principal predeterminado para el funcionamiento normal de un servidor.

Contraseña principal predeterminada

Contraseña principal predeterminada especificada en el campo Principal predeterminado. No se requiere un principal predeterminado para el funcionamiento normal de un servidor.

JACC

Nombre de clase de un proveedor JACC configurado. El valor predeterminado es default.

Módulos de auditoría

Módulos del proveedor de auditoría que utilizará el subsistema de auditorías si se habilita el registro de auditorías. Por defecto el servidor utiliza un módulo de auditoría llamado default. Para obtener información sobre la creación de nuevos módulos de auditoría, consulte Cómo crear un módulo de auditoría.

Habilitada asignación de principal predeterminado a función

Si esta opción está seleccionada, la asignación de principal predeterminado a función se aplicará a las aplicaciones que no tengan una asignación específica de las aplicaciones.

Clases principales asignadas

Especifica una implementación personalizada de la clase java.security.Principal utilizada en la asignación predeterminada de principal a función, en caso de que la asignación predeterminada de principal a función esté habilitada.

Propiedades adicionales

Las propiedades de seguridad adicionales del servidor.

Las propiedades válidas dependen del tipo de dominio seleccionado en el campo Dominio predeterminado y suelen especificarse cuando edita un dominio. Para obtener más información, consulte los siguientes temas:

  • Propiedades específicas de la clase FileRealm

  • Propiedades específicas de la clase CertificateRealm

  • Propiedades específicas de la clase JDBCRealm

  • Propiedades específicas de la clase LDAPRealm

  • Propiedades específicas de la clase SolarisRealm

  • Propiedades específicas de la clase PamRealm
