Configuración General de JVM

Java Virtual Machine (JVM) permite la ejecución de los códigos de bytes de Java en una máquina física. Al ajustar la configuración de JVM, mejora el rendimiento y se evitan errores de asignación de memoria.

Java Virtual Machine se incluye en la plataforma Java, Standard Edition (plataforma Java SE), que necesita Servidor GlassFish. La introducción de una configuración de JVM incorrecta hará que no funcione el servidor, por lo que tiene que ir con cuidado a la hora de cambiar esta configuración.

Nota:

Los términos "Java Virtual Machine" y "JVM" se refieren a una máquina virtual para la plataforma Java.

La página Configuración General de JVM contiene las siguientes opciones.

Cargar Valores por Defecto

Botón para restaurar la configuración que tiene valores por defecto a sus valores por defecto. La configuración que no tiene valores por defecto no se modifica.

Nombre de Configuración

Nombre de la configuración a la que se aplican los valores de esta página. Este campo es de sólo lectura.

Directorio Raíz de Java

Nombre del directorio de instalación del software de Java SE, que necesita Servidor GlassFish para poder ejecutarse. Para comprobar que la versión del software de Java SE que ha especificado está soportada en esta versión, consulte Notas Técnicas sobre la Versión de Oracle GlassFish Server. Si introduce un nombre de directorio inexistente o un directorio para una versión no soportada del software de Java SE, Servidor GlassFish no se podrá iniciar.

Opciones de Javac

Opciones de línea de comandos para el compilador del lenguaje de programación de Java. Servidor GlassFish ejecuta el compilador cuando se despliegan los componentes de Enterprise JavaBeans (EJB).

Depurar

Activa la depuración con Java Platform Debugger Architecture (JPDA).

Los desarrolladores de aplicaciones utilizan JPDA. Para obtener más información, consulte Guía de Desarrollo de la Aplicación de Oracle GlassFish Server.

Opciones de Depuración

Especifica las opciones de JPDA enviadas a JVM al activar la depuración.

Opciones de Compilación RMI

Opciones de línea de comandos para el compilador rmic. Servidor GlassFish ejecuta el compilador rmic cuando se despliegan componentes EJB.

Preprocesador de Código de Byte

Nombres de clases para el preprocesamiento de código de byte. Utilice comas para separar los nombres de clase. Cada clase debe implementar la interfaz com.sun.appserv.BytecodePreprocessor. Las clases se llaman en el orden especificado.

Propiedades Adicionales

Propiedades adicionales para JVM. Servidor GlassFish no define ninguna propiedad adicional para JVM.
