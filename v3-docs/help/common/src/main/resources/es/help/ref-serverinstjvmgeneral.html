Configuración general JVM

La Máquina virtual Java (JVM) permiten la ejecución de los códigos de bytes de Java en una máquina física. Al ajustar los valores de configuración de JVM, mejora el rendimiento y se evitan errores de asignación de memoria.

La máquina virtual de Java se incluye en la plataforma Java, Standard Edition (plataforma Java SE), que necesita GlassFish Server. La introducción de valores incorrectos para la configuración de JVM hará que no funcione el servidor, por lo que tiene que ir con cuidado a la hora de cambiar estos valores.

Nota - Los términos “máquina virtual de Java” y “JVM” hacen referencia a una máquina virtual para la plataforma Java.

La página Configuración general de JVM contiene las siguientes opciones.

Cargar valores predeterminados

Botón para restablecer los valores predeterminados de todos los parámetros de configuración.

Nombre de configuración

El nombre de la configuración a la que se aplican los valores de esta página. Este campo es de sólo lectura.

Inicio de Java

Nombre del directorio de instalación del software de Java SE, que necesita GlassFish Server para poder ejecutarse. Para comprobar que la versión del software Java SE que se especifica sea compatible con esta versión, consulte Oracle GlassFish Server 3.1 Release Notes. Si introduce un nombre de directorio inexistente o un directorio para una versión no compatible del software de Java SE, GlassFish Server no se podrá iniciar.

Opciones de Javac

Opciones de línea de comandos para el compilador del lenguaje de programación de Java. GlassFish Server ejecuta el compilador cuando se implementan los componentes de Enterprise JavaBeans (EJB).

Depuración

Habilita la depuración con Java Platform Debugger Architecture (JPDA).

Los desarrolladores de aplicaciones utilizan JPDA. Para obtener más información, consulte Oracle GlassFish Server 3.1 Application Development Guide.

Opciones de depuración

Especifica las opciones de JPDA que enviará a JVM cuando se habilite la depuración.

Opciones de compilación RMI

Opciones de línea de comandos para el compilador rmic. GlassFish Server ejecuta el compilador rmic cuando se implementan componentes EJB.

Preprocesador de código byte

Nombres de clases para el preprocesamiento de código byte. Utilice comas para separar los nombres de clase. Cada clase debe implementar la interfaz com.sun.appserv.BytecodePreprocessor . Las clases se llaman en el orden especificado.

Propiedades adicionales

Propiedades adicionales para JVM. GlassFish Server no define ninguna propiedad adicional para JVM.
