Para configurar parámetros generales de registros

  1. En el árbol de navegación, amplíe el nodo Configuraciones.
  2. En el nodo Configuraciones, haga clic en la configuración de instancia o clúster del servidor para el que desee configurar la Configuración de registro.

    Se abrirá la página Configuración.

  3. En la página Configuración, haga clic en Configuración de registro.

    Se abre la página Configuración de registro del destino de configuración seleccionado.

  4. Seleccione la casilla de verificación Escribir en el registro del sistema habilitado para enviar la salida de registro al recurso syslog además de al registro del servidor.

    Esta característica sólo está disponible en sistemas Solaris y Linux. Esta opción se deshabilita por defecto.

  5. En el campo Archivo de registro, introduzca un nombre o ubicación alternativos para el archivo de registro del servidor.

    La ubicación predeterminada es ${com.sun.aas.instanceRoot}/logs/server.log.

  6. En el campo Controlador de registro, introduzca el nombre de clase absoluto de un controlador de registro personalizado.

    Un controlador de registro personalizado le permite enviar registros a un destino distinto a server.log o syslog. El controlador personalizado debe comprender la clase java.util.logging.ConsoleHandler (una API compatible con JSR 047). Indique también la clase de controlador en la ruta de clase Servidor GlassFish para que el controlador se instale durante el arranque del servidor.

  7. En el campo Límite de rotación del archivo, introduzca el tamaño en bytes que el archivo de registro del servidor debe alcanzar antes de rotar.

    Cuando el registro del servidor alcanza el tamaño especificado, crea un nuevo archivo vacío llamado server.log y cambia el nombre del archivo antiguo server.log_date, donde date es la fecha y hora en la que se produjo la rotación del archivo. El valor predeterminado es 2 megabytes. El valor mínimo para el límite es 500 kilobytes; si se especifica un valor inferior, el archivo rota cuando alcanza el tamaño de 500 KB. Para deshabilitar la rotación del archivo de registros, defina el valor en 0.

  8. En el campo Límite de tiempo para rotación del archivo, introduzca los minutos que transcurren antes de que el archivo del servidor tenga que rotar.

    El valor predeterminado es 0, lo que significa que el archivo rota cuando alcanza el tamaño especificado en el campo Límite de rotación de archivo.

    Nota - Si especifica uno o más minutos, el límite de tiempo tiene prioridad sobre el límite de tamaño especificado en el campo Límite de rotación de archivo.

  9. En el campo Frecuencia de vaciado, escriba el número máximo de mensajes que se deben escribir desde la cola en el registro del servidor a la vez.

    El valor predeterminado es 1.

  10. Haga clic en Guardar.

    Nota - Según la configuración para la que se activó la función de registro, puede ser necesario reiniciar el servidor.

    También puede ser necesario reiniciar el servidor si se cambia el nombre del archivo de registro por otro que no sea el valor predeterminado. Si es necesario reiniciar el servidor se mostrará el mensaje Reinicio necesario en la esquina superior derecha de la Consola de administración.
