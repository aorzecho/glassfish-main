Definir la Configuración de Registro General

  1. En el árbol de navegación, amplíe el nodo Configuraciones.

  2. En el nodo Configuraciones, haga clic en la configuración de instancia de servidor o cluster para el que desee definir la configuración de registrador.

    Se abrirá la página Configuración.

  3. En la página Configuración, haga clic en Configuración de Registrador.

    Se abrirá la página Configuración de Registrador del destino de configuración seleccionado.

  4. Active la casilla de control Escribir en el log del sistema activado para enviar la salida de registro a la utilidad syslog además de al log del servidor.

    Esta función sólo está disponible en sistemas Solaris y Linux. Esta opción está desactivada por defecto.

  5. En el campo Archivo Log, introduzca un nombre o ubicación alternativos para el archivo log del servidor.

    La ubicación por defecto es ${com.sun.aas.instanceRoot}/logs/server.log.

  6. En el campo Manejador de Logs, introduzca el nombre de clase absoluto de un manejador de logs personalizado.

    Un manejador de logs personalizado le permite enviar logs a un destino distinto a server.log o syslog. El manejador personalizado debe comprender la clase java.util.logging.ConsoleHandler (una API compatible con JSR 047). Indique también la clase de manejador en la classpath Servidor GlassFish para que el manejador se instale durante el inicio del servidor.

  7. En el campo Límite de Rotación de Archivo, introduzca el tamaño en bytes que el archivo log del servidor debe alcanzar antes de rotar.

    Cuando el log del servidor alcanza el tamaño especificado, Servidor GlassFish crea un nuevo archivo vacío llamado server.log y cambia el nombre del archivo antiguo server.log_date, donde date es la fecha y hora en la que se produjo la rotación del archivo. El valor por defecto es 2 megabytes. El valor mínimo para el límite es 500 kilobytes; si se especifica un valor inferior, el archivo rota cuando alcanza el tamaño de 500 KB. Para desactivar la rotación del archivo log, defina el valor en 0.

  8. En el campo Límite Temporal de Rotación de Archivo, introduzca los minutos que transcurren antes de que el log del servidor tenga que rotar.

    El valor por defecto es 0, lo que significa que el archivo rota cuando alcanza el tamaño especificado en el campo Límite de Rotación de Archivo.

    Nota:

    Si especifica uno o más minutos, el límite de tiempo tiene prioridad sobre el límite de tamaño especificado en el campo Límite de Rotación de Archivo.

  9. En el campo Frecuencia de Vaciado, escriba el número máximo de mensajes que se deben escribir desde la cola en el log del servidor a la vez.

    El valor por defecto es 1.

  10. Haga clic en Guardar.

    Nota:

    Según la configuración para la que se activó la función de registro, puede ser necesario reiniciar el servidor.

    También puede ser necesario reiniciar el servidor si se cambia el nombre del archivo log por otro que no sea el valor por defecto. Si es necesario reiniciar el servidor se mostrará el mensaje Es Necesario Reiniciar en la esquina superior derecha de la Consola de Administración.
