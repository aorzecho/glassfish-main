Edición de la configuración de seguridad de mensajes.

Para habilitar la seguridad de los mensajes para puntos finales de servicios web implementados en Servidor GlassFish, debe especificar un proveedor para usar por defecto en el servidor. Si activa un proveedor predeterminado para la seguridad de los mensajes, también tendrá que habilitar proveedores para ser usados por clientes de los servicios web implementados en Servidor GlassFish. Encontrará información sobre la habilitación de los proveedores usados por clientes en Habilitación de seguridad de mensajes para clientes de aplicaciones.

Para habilitar la seguridad de los mensajes para invocaciones de servicios web que se originan en puntos finales implementados, debe especificar un proveedor de cliente predeterminado. Si ha activado un proveedor de cliente predeterminado para Servidor GlassFish, debe asegurarse que los servicios invocados de los puntos finales implementados en Servidor GlassFish están configurados de forma compatible para la seguridad de la capa de mensajes.

  1. En el árbol de navegación, expanda el nodo Configuración.
  2. Expanda el nodo Seguridad.
  3. Seleccione el nodo Seguridad de mensajes

    Se abrirá la página Configuraciones de seguridad de mensajes.

  4. En la página Configuraciones de seguridad de mensajes, haga clic en el nombre de la Capa de seguridad de la configuración que está editando.

    De manera predeterminada, la única configuración de seguridad de mensajes configurada es SOAP.

    Se abrirá la página Editar configuración de seguridad de mensajes.

  5. En la página Editar configuración de seguridad de mensajes, en la lista desplegable Proveedores predeterminados, seleccione un proveedor de servidor.

    El proveedor de servidor predeterminado, también llamado proveedor predeterminado, identifica el proveedor de servidor invocado para una aplicación que no identifica un proveedor de servidor específico. Las opciones disponibles son ServerProvider, XWS_ServerProvider o ninguno. De manera predeterminada no hay ningún proveedor predeterminado, por tanto Servidor GlassFish no invocará un proveedor de seguridad de mensajes para el servidor. Generalmente, debe seleccionar ServerProvider como proveedor predeterminado.

  6. En la lista desplegable Proveedor de cliente predeterminado, seleccione un proveedor de cliente.

    El proveedor de cliente predeterminado identifica el proveedor de cliente invocado para una aplicación que no identifica un proveedor de cliente específico. Las opciones disponibles son ClientProvider, XWS_ClientProvider o ninguno. De manera predeterminada no hay ningún proveedor seleccionado, por tanto Servidor GlassFish no invocará un proveedor de seguridad de mensajes para clientes. Generalmente, no debe seleccionar ninguno como proveedor de clientes. Seleccione ClientProvider para habilitar un proveedor predeterminado y una directiva de protección de mensajes para aplicar a las llamadas a servicios web que se originan en los puntos finales de servicios web implementados en Servidor GlassFish.

  7. Haga clic en Guardar.
  8. Para obtener instrucciones sobre cómo modificar las directivas de protección de mensajes de cliente habilitado o proveedores de servidores, consulte Edición de una configuración del proveedor de seguridad de mensajes.
