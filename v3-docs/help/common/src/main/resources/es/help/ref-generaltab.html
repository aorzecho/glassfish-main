Información General

The Servidor GlassFish creates one application server instance, called server at the time of installation. You can delete the server instance and create a new instance with a different name if you prefer.

Each Servidor GlassFish instance has its own Java configuration, Java resources, application deployment areas, and server configuration settings. Changes to one application server instance have no effect on other application server instances. You can have one application server instance within one administrative domain.

For many users, one application server instance meets their needs. However, depending upon your environment, you might want to create one or more additional application server instances. For example, in a development environment you can use different application server instances to test different Servidor GlassFish configurations, or to compare and test different application deployments. Because you can easily add or delete an application server instance, you can use them to create temporary "sandbox" areas to experiment with while developing.

In addition, for each application server instance you can also create virtual servers. Within a single installed application server instance you can offer companies or individuals domain names, IP Addresses, and some administration capabilities. For the users, it is almost as if they have their own web server, without the hardware and basic server maintenance. These virtual servers do not span application server instances. For more information about virtual servers, see Configuración General de JVM.

In operational deployments, for many purposes you can use virtual servers instead of multiple application server instances. However, if virtual servers do not meet your needs, you can also use multiple application server instances.

An Servidor GlassFish instance is not started automatically. Once you start an instance, the instance runs until you stop it. When you stop an application server instance, it stops accepting new connections, then waits for all outstanding connections to complete. If your machine crashes or is taken offline, the server quits and any requests it was servicing may be lost.

Consulte la página Información General para comprobar la configuración de Servidor GlassFish y ver los datos de Java Virtual Machine.

La página Información General contiene la siguiente información.

Botón Parar

Haga clic en Parar para parar Servidor GlassFish.

Botón Reiniciar

Haga clic en Reiniciar para reiniciar Servidor GlassFish.

Botón Ver Archivos Log

Haga clic en el botón Ver Archivos Log para ver los archivos log para una instancia o cluster de Servidor GlassFish.

Botón Ver Log No Procesado

Haga clic en el botón Ver Log No Procesado para ver los archivos log no procesados para el servidor de administración (llamado server).

Botón Administración Segura

Haga clic en el botón Administración Segura para activar o desactivar la administración segura en la página Administración Segura.

Nombre

Nombre del current servidor.

Estado

Estado actual de la instancia de servidor. El servidor puede estar parado, iniciado o en ejecución.

Tiempo de Actividad

Número de horas y minutos que la instancia de servidor ha estado continuamente en ejecución. Es de sólo lectura.

HTTP Load Balancer

The name of the load balancer, if it is enabled for the server instance.

JVM

Si hace clic en Informe de JVM, se abrirá una ventana independiente que muestra informes sobre Java Virtual Machine, incluyendo un informe resumido, un informe sobre gestión de la memoria y recolección de basura, otro informe sobre carga de clase y un volcado de thread actual.

Directorio de Configuración

Directorio de la máquina host que contiene archivos de configuración, como domain.xml.

Versión Instalada

Identificador de versión del producto.

Administración Segura

Estado actual de la administración segura. Los valores posibles son Activada y No Activada.

Depurar

Identifica si la depuración está activada. Si la depuración está activada, también aparecerá el número de puerto.

Puerto(s) HTTP

Puertos configurados actualmente para solicitudes y respuestas HTTP.

Puerto(s) IIOP

Puertos configurados actualmente para solicitudes y respuestas IIOP.
