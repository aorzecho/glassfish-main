Proveedores JACC

Utilice la página Proveedores JACC para configurar los proveedores de Java Authorization Contract for Containers (JACC) para la instancia o clúster de Servidor GlassFish seleccionado.

Los proveedores de JACC los pueden utilizar módulos de autorización de terceros para aplicaciones que se ejecuten en Servidor GlassFish. La instalación predeterminada de Servidor GlassFish incluye dos proveedores de JACC denominados default y simple. Estos proveedores predeterminados implementan un motor de autorización simple, basado en archivos, que cumple la especificación JACC. También puede crear sus propios proveedores JACC personalizados. Consulte Creación de un proveedor JACC.

Para obtener información acerca de cómo escribir clases de implementación para un proveedor JACC, consulte la especificación JACC en http://jcp.org/en/jsr/detail?id=115.

Se proporciona la siguiente información para cada proveedor JACC.

Nombre

Nombre del proveedor JACC. Este nombre se utiliza para hacer referencia al elemento jacc-provider elemento en domain.xml.

Configuración de directivas

El nombre de clase completo de javax.security.jacc.PolicyConfigurationFactory.provider que implementa el javax.security.jacc.PolicyConfigurationFactory específico del proveedor.

Proveedor de directivas

El nombre de clase completo de javax.security.jacc.policy.provider que implementa java.security.Policy.

Los proveedores JACC predeterminados también admiten la siguiente propiedad.

repository

Directorio que contiene el archivo de política de JACC. Para el proveedor JACC de Servidor GlassFish default, el directorio predeterminado es ${com.sun.aas.instanceRoot}/generated/policy. Esta propiedad no está definida de manera predeterminada para el proveedor JACC de Servidor GlassFish simple.

La tabla Proveedores JACC también contiene las siguientes opciones.

Nuevo

Botón para crear un nuevo proveedor JACC.

Borrar

Botón para eliminar uno o más proveedores JACC seleccionados.
