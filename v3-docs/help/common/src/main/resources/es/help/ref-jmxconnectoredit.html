Editar conector JMX

Consulte la página Editar conector JMX para editar el conector JMX compatible con JSR 360.

La página Editar conector JMX contiene las opciones siguientes.

Configuration Name

El nombre de la configuración a la que aplicar los valores de esta página. Este campo es de sólo lectura.

Nombre

Nombre del conector JMX. Éste es un campo de sólo lectura.

Protocolo JMX

Nombre del protocolo que admite este conector JMX. Éste es un campo de sólo lectura.

JMXServiceURL

URL del servicio JMX. Éste es un campo de sólo lectura.

Seguridad

Si la casilla de verificación Habilitado aparece seleccionada, la comunicación JMX estará cifrada. Esta opción se encuentra deshabilitada de forma predeterminada.

Dirección

Dirección IP del servicio de asignación de nombres donde se encuentra registrado el código auxiliar del servidor del conector JMX. Éste no es el puerto del socket del servidor que posibilita la comunicación real JMX. Ésta es la dirección de la interfaz de red donde se ha iniciado el registro RMI. Si su sistema dispone de varias interfaces de red, modifique este valor para que sólo se seleccione una interfaz determinada. El valor predeterminado es 0.0.0.0.

Puerto

Número de puerto en el que el servicio de asignación de nombres (registro RMI) escucha las conexiones del cliente RMI. La única función de este servicio de asignación de nombres es descargar los códigos auxiliares de RMI. Si el puerto predeterminado está ocupado, se utilizará otro puerto que esté libre. Los valores válidos se encuentran entre 1 y 65535. En sistemas UNIX, la creación de sockets de escucha para puertos 1-1024 requiere privilegios de superusuario. El valor predeterminado es 8686.

Nombre de dominio

Nombre que representa el dominio administrativo especial. Este dominio administrará toda la autenticación. El valor predeterminado es admin-realm.

Propiedades adicionales

Propiedades adicionales del conector JMX. Hay varias propiedades especificadas de manera predeterminada.
