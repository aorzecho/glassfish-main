Crear un Módulo de Auditoría

La auditoría es el modo en que Servidor GlassFish registra ciertos eventos, como errores o violaciones de seguridad. Servidor GlassFish registra todos los eventos de autenticación en archivos log. Un log de acceso completo ofrece una pista secuencial de los eventos de acceso de Servidor GlassFish.

Servidor GlassFish ofrece un sencillo módulo de auditoría por defecto; para obtener más información, consulte Utilizar el Módulo de Auditoría por Defecto.

  1. En el árbol de navegación, amplíe el nodo Configuración.

  2. Select the instance to configure:

    • To configure a particular instance, expand the instance's config node. For example, the default instance, server, expand the server-config node.

    • To configure the default settings for all instances, expand the default-config node.

  3. En el nodo Configuración, amplíe el nodo Seguridad.

  4. En el nodo Seguridad, seleccione el nodo Módulos de Auditoría.

    Se abrirá la página Módulos de Auditoría.

  5. En la página Módulos de Auditoría, haga clic en Nuevo.

    Se abrirá la página Nuevo Módulo de Auditoría.

  6. En la página Nuevo Módulo de Auditoría, en el campo Nombre, escriba el nombre del módulo de auditoría.

  7. En el campo Nombre de Clase, escriba el nombre completo de la clase que implanta este módulo.

    El nombre de clase para el módulo de auditoría por defecto es com.sun.enterprise.security.Audit.

  8. En la sección Propiedades Adicionales, especifique las propiedades adicionales.

    Para agregar una propiedad, haga clic en el botón Agregar Propiedad. En la fila en blanco que aparece, escriba el nombre de propiedad en el campo Nombre y escriba el valor de propiedad en el campo Valor.

    Para ver una descripción de las propiedades adicionales disponibles para los módulos de auditoría, consulte Propiedades específicas de los módulos de auditoría.

  9. Haga clic en Aceptar.
