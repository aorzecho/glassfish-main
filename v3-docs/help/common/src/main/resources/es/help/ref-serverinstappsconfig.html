Configuración de Aplicaciones

Utilice la página Configuración de Aplicaciones para definir las propiedades que ayudan a garantizar que se detecten los cambios en aplicaciones desplegadas y que se vuelvan a cargar las clases modificadas.

La página Configuración de Aplicaciones contiene las siguientes opciones.

Cargar Valores por Defecto

Botón para restaurar la configuración que tiene valores por defecto a sus valores por defecto. La configuración que no tiene valores por defecto no se modifica.

Volver a Cargar

Si esta opción está activada, las clases modificadas se vuelven a cargar de forma dinámica. El servidor comprueba periódicamente los cambios realizados en el directorio de despliegue y vuelve a desplegar la aplicación de forma automática y dinámica con los cambios. Esto es útil en un entorno de desarrollo ya que permite poner a prueba los cambios realizados en el código de forma rápida. En cambio, en un entorno de producción, la nueva carga dinámica puede hacer que disminuya el rendimiento. Además, cuando ha finalizado la nueva carga, las sesiones del periodo de tránsito pierden su validez. El cliente debe reiniciar la sesión. Esta opción está activada por defecto.

Intervalo de Sondeo de Nueva Carga

Identifica la frecuencia con que el servidor debería comprobar los cambios de código en las aplicaciones y módulos desplegados. El valor por defecto es false.

Timeout de Sesión de Administrador

Identifica el número de minutos de inactividad antes de que la Consola de Administración sufra un timeout y se deba conectar de nuevo. El valor por defecto es 60.

Despliegue Automático

Si esta opción está activada, las aplicaciones del directorio de despliegue automático se despliegan automáticamente. Esta opción está activada por defecto.

Intervalo de Sondeo de Despliegue Automático

Especifica la frecuencia con que el servidor debería comprobar el directorio de despliegue automático para saber si hay cambios del módulo o de la aplicación. Si cambia el intervalo de sondeo no quedará afectado el tiempo que se tarda en desplegar una aplicación o un módulo. El valor por defecto es false.

Timeout para el Reintento de Despliegue Automático

Especifica el número de segundos que el tamaño de un archivo copiado parcialmente puede permanecer sin cambios antes de que se produzca un error. El valor por defecto es 4.

Si se copia un archivo lentamente en el directorio de despliegue automático, éste puede aparecer antes de que se copie el archivo completo. Como resultado, se produce un error al intentar desplegar la aplicación de forma automática. Si se produce un error al intentar desplegar una aplicación de forma automática por este motivo, Servidor GlassFish intentará desplegar de nuevo la aplicación. Si el tamaño del archivo copiado parcialmente permanece sin cambios durante el período de tiempo especificado, se producirá un error.

Directorio de Despliegue Automático

Identifica el directorio para supervisar el despliegue automático de las aplicaciones. El valor por defecto es domain-dir/autodeploy.

Validación de XML

Especifica el tipo de validación de XML que se llevará a cabo en los descriptores de despliegue estándar y de Servidor GlassFish. El tipo puede ser cualquiera de los siguientes:

Completo

Si se produce un fallo en la validación de XML, no se realizará el despliegue. Éste es el valor por defecto.

Análisis

Se informa acerca de errores en la validación de XML, pero se realiza el despliegue.

Ninguno

No se realiza ninguna validación de XML.

Verificador

Si se activa esta opción, el verificador se ejecuta antes del despliegue automático. Si se produce un fallo en la verificación, no se realiza el despliegue. El verificador examinará la estructura y contenido del archivo. La verificación de aplicaciones de gran tamaño tiende a llevar mucho tiempo. Esta opción está desactivada por defecto.

Precompilar

Si está activada esta opción, los archivos JavaServer Pages (JSP) se precompilarán. Si está desactivada esta opción, los archivos JSP se compilarán en tiempo de ejecución cuando se acceda a los mismos por primera vez. Esta opción está desactivada por defecto.

Propiedades Adicionales

Propiedades adicionales para las aplicaciones. Servidor GlassFish no define ninguna propiedad adicional para las aplicaciones.
