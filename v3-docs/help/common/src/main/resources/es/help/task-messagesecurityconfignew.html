Crear una Configuración de Seguridad de Mensajes

Normalmente no será necesario que cree una configuración de seguridad de mensajes. Servidor GlassFish incluye una configuración de seguridad de mensajes para la capa de autenticación SOAP. Los proveedores para esta configuración no están activados por defecto, pero puede activarlos editando la configuración de seguridad de mensajes SOAP. No obstante, puede crear una configuración para la capa de autenticación HttpServlet y especificar una o más configuraciones del proveedor para ella.

  1. En el árbol de navegación, amplíe el nodo Configuración.

  2. Amplíe el nodo Seguridad.

  3. Seleccione el nodo Seguridad de Mensajes.

    Se abrirá la página Configuraciones de Seguridad de Mensajes.

  4. En la página Configuraciones de Seguridad de Mensajes, haga clic en Nuevo.

    Se abrirá la página Nueva Configuración de Seguridad de Mensajes.

    Nota:

    No es posible crear o suprimir proveedores de seguridad de mensajes para la configuración por defecto server-config. Los únicos proveedores de seguridad de mensajes soportados con la configuración por defecto server-config son SOAP y HttpServlet. Dado que estos dos proveedores de mensajes se crean por defecto al instalar Servidor GlassFish, el botón Nuevo está atenuado en la página Configuraciones de Seguridad de Mensajes para la configuración por defecto server-config.

  5. En la página Nueva Configuración de Seguridad de Mensajes, en la lista desplegable Capa de Autenticación, seleccione la capa de mensajes para la que está creando la configuración.

    Debido a que por defecto hay una configuración SOAP disponible, la única opción es HttpServlet.

  6. En el campo ID de Proveedor, introduzca un identificador exclusivo de configuración del proveedor para esta configuración de seguridad de mensajes.

  7. Active la casilla de control Proveedor por Defecto para especificar que esta configuración de proveedor será el proveedor por defecto.

  8. En la lista desplegable Tipo de Proveedor, seleccione el tipo de proveedor.

    Las opciones disponibles son:

    client

    Especifica que el proveedor es el proveedor de autenticación de clientes.

    server

    Especifica que el proveedor es el proveedor de autenticación del servidor.

    client-server

    Especifica que el proveedor es tanto el proveedor de autenticación del servidor como de clientes.

  9. En el campo Nombre de Clase, introduzca el nombre de la clase de implantación de Java del proveedor.

    Los proveedores de autenticación del cliente deben implantar la interfaz com.sun.enterprise.security.jauth.ClientAuthModule. Los proveedores del servidor deben implementar la interfaz com.sun.enterprise.security.jauth.ServerAuthModule. Los proveedores de servidor y cliente deben implantar ambas interfaces.

  10. En la sección Propiedades Adicionales, especifique las propiedades adicionales.

    Para agregar una propiedad, haga clic en el botón Agregar Propiedad. En la fila en blanco que aparece, escriba el nombre de propiedad en el campo Nombre y escriba el valor de propiedad en el campo Valor.

    Servidor GlassFish no define ninguna propiedad adicional para la configuración de seguridad de mensajes.
