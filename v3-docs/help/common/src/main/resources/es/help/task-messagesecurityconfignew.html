Creación de la configuración de seguridad de mensajes

Normalmente no será necesario que cree una configuración de seguridad de mensajes. Servidor GlassFish incluye una configuración de seguridad de mensajes para la capa de autenticación SOAP. Los proveedores para esta configuración no están activados de manera predeterminada, pero puede activarlos editando la configuración de seguridad de mensajes SOAP. No obstante, puede crear una configuración para la capa de autenticación HttpServlet y especificar una o más configuraciones del proveedor para ella.

  1. En el árbol de navegación, expanda el nodo Configuración.
  2. Expanda el nodo Seguridad.
  3. Seleccione el nodo Seguridad de mensajes

    Se abrirá la página Configuraciones de seguridad de mensajes.

  4. En la página Configuraciones de seguridad de mensajes, haga clic en Nuevo.

    Se abrirá la página Nueva configuración de seguridad de mensajes.

    Nota - No es posible crear o suprimir proveedores de seguridad de mensajes para la configuración predeterminada server-config. Los únicos proveedores de seguridad de mensajes compatibles con la configuración predeterminada server-config son SOAP y HttpServlet. Dado que estos dos proveedores de mensajes se crean de forma predeterminada al instalar Servidor GlassFish, el botón Nuevo está atenuado en la página Configuraciones de seguridad de mensajes para la configuración predeterminada server-config.

  5. En la página Nueva configuración de seguridad de mensajes, en la lista desplegable Capa de autenticación, seleccione la capa del mensajes para la que está creando la configuración.

    Debido a que de manera predeterminada hay una configuración SOAP disponible, la única opción es HttpServlet.

  6. En el campo Id. de proveedor, introduzca un identificador exclusivo de configuración del proveedor para esta configuración de seguridad de mensajes.
  7. Seleccione la casilla de verificación Proveedor predeterminado para especificar que esta configuración de proveedor será el proveedor predeterminado.
  8. En la lista desplegable Tipo de proveedor, seleccione el tipo de proveedor.

    Las opciones disponibles son:

    client

    Especifica que el proveedor es el proveedor de autenticación de clientes.

    server

    Especifica que el proveedor es el proveedor de autenticación del servidor.

    client-server

    Especifica que el proveedor es tanto el proveedor de autenticación del servidor como del cliente.

  9. En el campo Nombre de clase, introduzca el nombre de la clase de implementación de Java del proveedor.

    Los proveedores de autenticación de clientes deben implementar la com.sun.enterprise.security.jauth.ClientAuthModule interfaz. Los proveedores de servidor deben implementar la com.sun.enterprise.security.jauth.ServerAuthModule interfaz. Los proveedores de servidor y cliente deben implementar ambas interfaces.

  10. (Opcional) En la sección Propiedades adicionales, especifique las propiedades adicionales.

    Para añadir una propiedad, haga clic en el botón Agregar propiedad. En la fila vacía escriba el nombre de la propiedad en el campo Nombre y el valor de la propiedad en el campo Valor.

    Servidor GlassFish no define ninguna propiedad adicional para la configuración de seguridad de mensajes.
