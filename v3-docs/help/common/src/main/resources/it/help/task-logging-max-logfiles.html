Cambiare il numero limite dei file di log sottoposti a rotazione

Quando in GlassFish Server un file di log viene sottoposto a rotazione, si crea un nuovo file vuoto denominato server.log e il file precedente viene rinominato come server.log_ date, dove date indica la data e ora di rotazione del file.

Per impostazione predefinita, il numero di file di log sottoposti a rotazione è limitato a 10. Quando viene raggiunto tale limite, il file di log meno recente viene sovrascritto alla successiva rotazione.

Se necessario è possibile modificare il numero limite di file sottoposti a rotazione. Il nuovo limite sarà valido per la rotazione di file di log relativi sia alla registrazione degli accessi sia al log del server.

  • Impostare la proprietà com.sun.enterprise.server.logging.max_history_files sul numero massimo di file di log sottoposti a rotazione da conservare.

    Per informazioni su come impostare una proprietà di sistema vedere Gestire le proprietà di sistema.

    Di seguito è indicato il funzionamento della proprietà di sistema com.sun.enterprise.server.logging.max_history_files:

    • Se la proprietà non è impostata, vengono conservati non oltre 10 file di log sottoposti a rotazione.

    • Se la proprietà è impostata su un numero non valido o su un valore nullo, vengono conservati non oltre 10 file di log sottoposti a rotazione.

    • Se la proprietà è impostata su 0, non viene conservato alcun file di log sottoposto a rotazione.
