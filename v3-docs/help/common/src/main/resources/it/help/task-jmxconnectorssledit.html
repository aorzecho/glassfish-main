Modificare le impostazioni SSL per il connettore JMX

Prima di iniziare

Questo task risulta utile solo se per il connettore JMX è stata abilitata la sicurezza.

  1. Nell'albero di navigazione espandere il nodo Configurazione.

  2. Nel nodo Configurazione, selezionare il nodo Servizio di amministrazione.

    Viene visualizzata la pagina Modifica connettore JMX.

  3. Selezionare la casella di controllo Sicurezza abilitata per impostare la cifratura delle comunicazioni JMX.

    Questa opzione è disabilitata per impostazione predefinita.

  4. Fare clic sulla scheda SSL.

    Viene visualizzata la pagina SSL.

  5. Nella pagina SSL selezionare la casella di controllo SSL3 abilitato per abilitare SSL3.

    Questa opzione è abilitata per impostazione predefinita.

  6. Selezionare la casella di controllo TLS abilitato per abilitare TLS (Transport Layer Security).

    Questa opzione è abilitata per impostazione predefinita.

  7. Selezionare la casella di controllo Autenticazione client abilitata per richiedere l'identificazione dei client a ogni richiesta del server.

    Questa opzione è disabilitata per impostazione predefinita.

  8. Nel campo Pseudonimo certificato, immettere lo pseudonimo del certificato del server nel database dei certificati o nel token PKCS#11.

    Nel certificato, il formato del nome è tokenname:nickname. L'inclusione della porzione tokenname: del nome in questo attributo è facoltativa.

  9. Nel campo Keystore, immettere il nome del file keystore (ad esempio keystore.jks).

  10. Nel campo Algoritmo di protezione, immettere il nome dell'algoritmo di gestione attendibile (ad esempio PKIX) da utilizzare per la convalida del percorso di certificazione.

  11. Nel campo Lunghezza massima certificato, immettere il numero massimo di certificati intermedi non autoemessi che possono essere presenti in un percorso di certificazione.

    Il campo viene utilizzato solo se il campo Algoritmo di protezione è impostato su PKIX. Il valore 0 indica che il percorso può contenere un solo certificato. Il valore -1 indica che la lunghezza del percorso non è vincolata (non esiste un valore massimo). Se si imposta un valore inferiore a -1, viene generata un'eccezione.

  12. Nel campo Truststore immettere il nome del file (ad esempio cacerts.jks).

  13. Nell'area Suite di cifratura specificare i pacchetti di cifratura da utilizzare.

    Se non si aggiunge alcun pacchetto di crittografia, verranno utilizzati tutti quelli disponibili.

  14. Fare clic su OK.
