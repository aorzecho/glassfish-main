Modificare le impostazioni SSL del connettore JMX

Prima di cominciare

Questa attività risulta utile solo se per il connettore JMX è stata abilitata la sicurezza.

  1. Nella struttura ad albero, espandere il nodo Configuration.
  2. Nel nodo Configuration, selezionare il nodo Admin Service.

    Viene visualizzata la pagina Edit JMX Connector.

  3. Selezionare la casella di controllo Security Enabled per impostare la crittografia delle comunicazioni JMX.

    Questa opzione è disabilitata per impostazione predefinita.

  4. Fare clic sulla scheda SSL.

    Viene visualizzata la pagina SSL.

  5. Nella pagina SSL selezionare la casella di controllo SSL3 Enabled per abilitare SSL3.

    Questa opzione è abilitata per impostazione predefinita.

  6. Selezionare la casella di controllo TLS Enabled per abilitare TLS (Transport Layer Security).

    Questa opzione è abilitata per impostazione predefinita.

  7. Selezionare la casella di controllo Client Authentication Enabled per richiedere l'identificazione dei client a ogni richiesta del server.

    Questa opzione è disabilitata per impostazione predefinita.

  8. Nel campo Certificate Nickname, immettere il nickname del certificato del server nel database dei certificati o nel token PKCS#11.

    Nel certificato, il formato del nome è tokenname: nickname. L'inclusione della parte tokenname: in questo attributo è facoltativa.

  9. Nel campo Key Store, immettere il nome del file keystore (ad esempio keystore.jks).
  10. Nel campo Trust Algorithm, immettere il nome dell'algoritmo di gestione attendibile (ad esempio PKIX) da utilizzare per la convalida del percorso di certificazione.
  11. Nel campo Max Certificate Length, immettere il numero massimo di certificati intermedi non autoemessi che possono essere presenti in un percorso di certificazione.

    Il campo viene utilizzato solo se il campo Trust Algorithm è impostato su PKIX. Il valore 0 indica che il percorso può contenere un solo certificato. Il valore -1 indica che la lunghezza del percorso non è vincolata (non esiste un valore massimo). Se si imposta un valore inferiore a -1, viene generata un'eccezione.

  12. Nel campo Trust Store immettere il nome del file (ad esempio cacerts.jks).
  13. Nell'area Cipher Suites specificare i pacchetti di crittografia da utilizzare.

    Se non si aggiunge alcun pacchetto di crittografia, verranno utilizzati tutti quelli disponibili.

  14. Fare clic su OK.
