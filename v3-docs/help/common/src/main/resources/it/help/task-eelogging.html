Configurare le impostazioni di log generali

  1. Nell'albero di navigazione espandere il nodo Configurazioni.

  2. Nel nodo Configurazioni, fare clic sulla configurazione dell'istanza server o del cluster per cui si desidera configurare le impostazioni del programma di log.

    Viene visualizzata la pagina Configurazione.

  3. Nella pagina Configurazione, fare clic su Impostazioni logger.

    Viene visualizzata la pagina Impostazioni logger per la destinazione della configurazione selezionata.

  4. Selezionare la casella di controllo Scrittura nel log di sistema abilitata per inviare l'output di log alla risorsa syslog in aggiunta al log del server.

    Questa funzione è disponibile soltanto nei sistemi Solaris e Linux. Questa opzione è disabilitata per impostazione predefinita.

  5. Nel campo File di log digitare una posizione o un nome alternativo per il file di log del server.

    La posizione predefinita è ${com.sun.aas.instanceRoot}/logs/server.log.

  6. Nel campo Handler di log digitare il nome classe assoluto di un gestore del log personalizzato.

    Un gestore del log personalizzato consente di inviare log a una destinazione diversa da server.log o syslog. L'handler personalizzato deve estendere la classe java.util.logging.ConsoleHandler (un'API conforme allo standard JSR 047). Inserire la classe handler nel classpath di GlassFish Server, in modo che l'handler venga installato durante l'avvio del server.

  7. Nel campo Limite di rotazione file digitare la dimensione in byte che il file di log del server deve raggiungere per essere sottoposto a rotazione.

    Quando il log del server raggiunge la dimensione specificata, in GlassFish Server viene creato un nuovo file vuoto denominato server.log e il file precedente viene rinominato server.log_date, dove date indica data e ora di rotazione del file. Il valore predefinito è 2 MB. Il valore limite minimo è 500 KB; se si specifica un valore inferiore, il file verrà sottoposto a rotazione una volta raggiunti i 500 KB. Per disattivare la rotazione del file di log impostare il valore su 0.

  8. Nel campo Limite di tempo rotazione file, digitare il numero di minuti trascorso il quale si desidera eseguire la rotazione del log del server.

    Il valore predefinito è 0. Indica che il file viene sottoposto a rotazione quando raggiunge la dimensione specificata nel campo Limite di rotazione file.

    Nota:

    Se si specificano uno o più minuti, il limite di tempo ottiene la precedenza sul limite di dimensione specificato nel campo Limite di rotazione file.

  9. Nel campo Frequenza flush digitare il numero massimo di messaggi scrivibili contemporaneamente dalla coda al log del server.

    Il valore predefinito è 1.

  10. Fare clic su Salva.

    Nota:

    A seconda della configurazione per cui è stato abilitato il log, potrebbe essere necessario riavviare il server.

    Il riavvio del server potrebbe inoltre essere necessario in caso di modifica del nome del file di log rispetto all'impostazione predefinita. Se è necessario riavviare il server, verrà visualizzato un messaggio Riavvio necessario nell'angolo superiore destro di Console di amministrazione.
