Configurare impostazioni generali per il log

  1. Nella struttura ad albero, espandere il nodo Configurations.
  2. Nel nodo Configurations, fare clic sulla configurazione dell'istanza server o del cluster per cui si desidera configurare le impostazioni del programma di log.

    Viene visualizzata la pagina Configuration.

  3. Nella pagina Configuration, fare clic su Logger Settings.

    Viene visualizzata la pagina Logger Settings per la destinazione della configurazione selezionata.

  4. Selezionare la casella di controllo Write to System Log Enabled per inviare l'output di log alla risorsa syslog in aggiunta al log del server.

    Questa caratteristica è disponibile soltanto nei sistemi Solaris e Linux. Questa opzione è disabilitata per impostazione predefinita.

  5. Nel campo Log File digitare una posizione o un nome alternativo per il file di log del server.

    La posizione predefinita è ${com.sun.aas.instanceRoot}/logs/server.log .

  6. Nel campo Log Handler digitare il nome classe assoluto di un gestore del log personalizzato.

    Un gestore del log personalizzato consente di inviare log a una destinazione diversa da server.log o syslog. Tale gestore deve estendere la classe java.util.logging.ConsoleHandler (un'API conforme allo standard JSR 047). Inserire la classe handler nel classpath di GlassFish Server, in modo che l'handler venga installato durante l'avvio del server.

  7. Nel campo File Rotation Limit digitare la dimensione in byte che il file di log del server deve raggiungere per essere sottoposto a rotazione.

    Quando il log del server raggiunge la dimensione specificata, in GlassFish Server viene creato un nuovo file vuoto denominato server.log e il file precedente viene rinominato server.log_date, dove date corrisponde alla data e all'ora di rotazione del file. Il valore predefinito è 2 MB. Il valore limite minimo è 500 KB; se si specifica un valore inferiore, il file verrà sottoposto a rotazione una volta raggiunti i 500 KB. Per disattivare la rotazione del file di log impostare il valore su 0.

  8. Nel campo File Rotation Time Limit, digitare il numero di minuti trascorso il quale si desidera eseguire la rotazione del log del server.

    Il valore predefinito è 0. Indica che il file viene sottoposto a rotazione quando raggiunge la dimensione specificata nel campo File Rotation Limit.

    Nota - Se si specificano uno o più minuti, il limite di tempo ottiene la precedenza sul limite di dimensione specificato nel campo File Rotation Limit.

  9. Nel campo Flush Frequency digitare il numero massimo di messaggi scrivibili contemporaneamente dalla coda al log del server.

    Il valore predefinito è 1.

  10. Fare clic su Save.

    Nota - A seconda della configurazione per cui è stato abilitato il log, potrebbe essere necessario riavviare il server.

    Il riavvio del server potrebbe inoltre essere necessario in caso di cambiamento del nome del file di log rispetto all'impostazione predefinita. Se è necessario riavviare il server, verrà visualizzato un messaggio Restart required nell'angolo superiore destro di Administration Console.
