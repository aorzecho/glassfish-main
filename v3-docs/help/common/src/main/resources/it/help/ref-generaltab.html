Informazioni generali

The GlassFish Server creates one application server instance, called server at the time of installation. You can delete the server instance and create a new instance with a different name if you prefer.

Each GlassFish Server instance has its own Java configuration, Java resources, application deployment areas, and server configuration settings. Changes to one application server instance have no effect on other application server instances. You can have one application server instance within one administrative domain.

For many users, one application server instance meets their needs. However, depending upon your environment, you might want to create one or more additional application server instances. For example, in a development environment you can use different application server instances to test different GlassFish Server configurations, or to compare and test different application deployments. Because you can easily add or delete an application server instance, you can use them to create temporary "sandbox" areas to experiment with while developing.

In addition, for each application server instance you can also create virtual servers. Within a single installed application server instance you can offer companies or individuals domain names, IP Addresses, and some administration capabilities. For the users, it is almost as if they have their own web server, without the hardware and basic server maintenance. These virtual servers do not span application server instances. For more information about virtual servers, see Impostazioni generali JVM.

In operational deployments, for many purposes you can use virtual servers instead of multiple application server instances. However, if virtual servers do not meet your needs, you can also use multiple application server instances.

An GlassFish Server instance is not started automatically. Once you start an instance, the instance runs until you stop it. When you stop an application server instance, it stops accepting new connections, then waits for all outstanding connections to complete. If your machine crashes or is taken offline, the server quits and any requests it was servicing may be lost.

La pagina Informazioni generali contiene le informazioni riportate di seguito.

Pulsante Arresta

Fare clic sul pulsante Arresta per arrestare GlassFish Server.

Pulsante Riavvia

Fare clic sul pulsante Riavvia per riavviare GlassFish Server.

Pulsante Visualizza file di log

Fare clic sul pulsante Visualizza file di log per visualizzare i file di log per un'istanza o un cluster GlassFish Server.

Pulsante Visualizza log raw

Fare clic sul pulsante Visualizza log raw per visualizzare i file di log raw per il server di amministrazione (denominato server).

Pulsante Amministrazione sicura

Fare clic sul pulsante Amministrazione sicura per abilitare o disabilitare l'amministrazione sicura nella pagina Amministrazione sicura.

Nome

Nome del current server.

Stato

Stato corrente dell'istanza server. Il server può avere lo stato Arrestato, Avviato o In esecuzione

Tempo di attività

Il numero di ore e minuti in cui l'istanza server è stata in esecuzione in modo continuativo. Questo campo è di sola lettura.

HTTP Load Balancer

The name of the load balancer, if it is enabled for the server instance.

JVM

Se si fa clic su Report JVM, viene visualizzata una finestra separata in cui sono disponibili i report relativi alla Java Virtual Machine, tra cui un report di riepilogo, un report di gestione memoria e garbage collection, un report sul caricamento delle classi e un dump del thread corrente.

Directory di configurazione

Directory del computer host contenente i file di configurazione, ad esempio domain.xml.

Versione installata

Identificativo della versione di prodotto.

Amministrazione sicura

Stato corrente dell'amministrazione sicura. I possibili valori sono Abilitato e Non abilitato.

Debug

Indica se il debug è abilitato. Se il debug è abilitato, viene visualizzato anche il numero di porta.

Porta/e HTTP

Porte attualmente configurate per le richieste e le risposte HTTP.

Porta/e IIOP

Porte attualmente configurate per le richieste e le risposte IIOP.
