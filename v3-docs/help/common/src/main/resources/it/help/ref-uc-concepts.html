Nozioni fondamentali su Strumento di aggiornamento

Strumento di aggiornamento consente di gestire le installazioni dei componenti software di GlassFish Server inclusi in una singola directory del file system nel sistema in uso.

La versione Console di amministrazione di Strumento di aggiornamento consente di effettuare le operazioni riportate di seguito.

  • Installare i componenti aggiuntivi.

  • Visualizzare gli aggiornamenti per i componenti installati.

  • Visualizzare i componenti installati.

Per aggiornare o rimuovere i componenti installati, è necessario arrestare il dominio GlassFish Server, quindi eseguire uno dei comandi riportati di seguito.

  • as-install-parent/bin/updatetool, che consente di avviare la utility grafica Strumento di aggiornamento.

  • as-install-parent/bin/pkg, una versione della riga di comando dello Strumento di aggiornamento.

Nelle sezioni seguenti vengono descritti i concetti chiave di Strumento di aggiornamento.

Componente

Per componente si intende l'unità installabile gestita da Strumento di aggiornamento. Un componente è composto da file installati come unità di sistema in un'immagine di installazione.

Immagine di installazione

Per immagine di installazione si intende la directory contenente i componenti software, incluso il contenuto della directory. L'immagine di installazione viene aggiunta a Strumento di aggiornamento in seguito allo scaricamento e all'installazione del software GlassFish Server. Per abilitare l'aggiunta e l'aggiornamento dei componenti software di GlassFish Server, ogni immagine di installazione include informazioni sull'installazione dei componenti.

Repository di rete

Il repository di rete contiene gli elementi riportati di seguito.

  • Componenti che possono essere aggiunti a un'immagine di installazione

  • Aggiornamenti ai componenti esistenti di un'immagine di installazione

A uno o più repository di rete è associata un'immagine di installazione. Ogni repository di rete è rappresentato da un nome e un URL associato. Il nome di repository è univoco per ogni immagine di installazione e rappresenta sia l'organizzazione o il progetto che gestisce il repository sia un ambito di contenuti definito correttamente. Per ulteriori informazioni sui repository di rete preconfigurati per GlassFish Server, vedere Preconfigured Repositories for GlassFish Server in Oracle GlassFish Server Administration Guide.

Componente aggiuntivo

Per componente add-on si intende un componente dell'immagine di installazione non ancora installato.
