New Lifecycle Module

Nella pagina New Lifecycle Module è possibile definire un nuovo modulo del ciclo di vita.

La pagina New Lifecycle Module contiene le seguenti opzioni.

Name

Nome del modulo del ciclo di vita.

Class Name

Nome completo del file di classe del modulo del ciclo di vita. La classe deve implementare l'interfaccia com.sun.appserv.server.LifecycleListener.

Classpath

Classpath per il modulo del ciclo di vita. Il classpath specifica la posizione del modulo del ciclo di vita. Il valore predefinito è domain-dir/applications. Se il modulo si trova già in questa directory (ossia nel classpath del server) il campo può essere lasciato vuoto.

Load Order

Ordine di caricamento del modulo del ciclo di vita all'avvio. I moduli con i numeri di ordine di caricamento più bassi verranno caricati per primi. Il valore può essere compreso tra 101 e il valore MAXINT del sistema operativo. I valori compresi tra 1 e 100 sono riservati.

Description

Descrizione del modulo del ciclo di vita.

Status

Se l'opzione è selezionata, il modulo del ciclo di vita è abilitato. Questa opzione è abilitata per impostazione predefinita.

On Load Failure

Se l'opzione è selezionata, il server viene arrestato se il modulo del ciclo di vita non viene caricato correttamente. Questa opzione è disabilitata per impostazione predefinita.

Targets

Cluster e istanze standalone per il modulo del ciclo di vita. Per spostare le destinazioni desiderate nella colonna Selected Targets, utilizzare i pulsanti Add e Add All. Per spostare le destinazioni non necessarie nella colonna Available Targets, utilizzare i pulsanti Remove e Remove All.

Questa opzione viene visualizzata solo se nel dominio sono stati creati cluster o istanze standalone.
