Nuovo modulo ciclo di vita

Nella pagina Nuovo modulo ciclo di vita è possibile definire un nuovo modulo del ciclo di vita.

La pagina Nuovo modulo ciclo di vita contiene le opzioni riportate di seguito.

Nome

Nome del modulo del ciclo di vita.

Nome classe

Nome completamente qualificato del file di classe del modulo del ciclo di vita. La classe deve implementare l'interfaccia com.sun.appserv.server.LifecycleListener.

Classpath

Classpath per il modulo del ciclo di vita. Il classpath specifica la posizione del modulo del ciclo di vita. Il valore predefinito è domain-dir/applications. Se il modulo si trova già in questa directory, ovvero nel classpath del server, il campo può essere lasciato vuoto.

Ordine di caricamento

Ordine in cui viene caricato questo modulo del ciclo di vita all'avvio. I moduli con valori interi più bassi vengono caricati per primi. Il valore può essere compreso tra 101 e il valore MAXINT del sistema operativo. I valori compresi tra 1 e 100 sono riservati.

Descrizione

Descrizione del modulo del ciclo di vita.

Stato

Se questa opzione è selezionata, il modulo del ciclo di vita è abilitato. Questa opzione è abilitata per impostazione predefinita.

In caso di errore di caricamento

Se questa opzione è selezionata, il server viene chiuso se il caricamento del modulo del ciclo di vita non riesce. Questa opzione è disabilitata per impostazione predefinita.

Destinazioni

Cluster e istanze standalone per il modulo del ciclo di vita. Per spostare le destinazioni desiderate nella colonna Destinazioni selezionate, utilizzare i pulsanti Aggiungi e Aggiungi tutto. Per spostare le destinazioni non necessarie nella colonna Destinazioni disponibili, utilizzare i pulsanti Rimuovi e Rimuovi tutto.

Questa opzione viene visualizzata solo se nel dominio sono stati creati cluster o istanze standalone.
