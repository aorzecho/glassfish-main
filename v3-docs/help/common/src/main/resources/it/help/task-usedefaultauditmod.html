Utilizzare il modulo di audit predefinito

Il modulo di audit default registra le richieste di autenticazione e autorizzazione nel file di log del server. Per informazioni sulla modifica della posizione del file di log, vedere Configurare le impostazioni di log generali.

Le voci del log di autenticazione includono le seguenti informazioni:

  • Nomi degli utenti che hanno tentato l'autenticazione

  • Realm che ha elaborato la richiesta di accesso

  • Componente URI o EJB del modulo Web richiesto

  • Esito positivo o negativo della richiesta

Indipendentemente dal fatto che il log di revisione sia abilitato o meno, vengono registrati tutti gli eventi di autenticazione non concessa.

Le voci del log di autorizzazione includono le seguenti informazioni:

  • Nomi degli utenti autenticati, se presenti

  • Componente URI o EJB Web richiesto

  • Esito positivo o negativo delle richieste

  1. Nell'albero di navigazione espandere il nodo Configurazione.

  2. Select the instance to configure:

    • To configure a particular instance, expand the instance's config node. For example, the default instance, server, expand the server-config node.

    • To configure the default settings for all instances, expand the default-config node.

  3. Nel nodo Configurazione, selezionare il nodo Sicurezza.

    Viene visualizzata la pagina Sicurezza.

  4. Nell'elenco a discesa Moduli di audit, selezionare il modulo di revisione default.

  5. Fare clic su Salva.
