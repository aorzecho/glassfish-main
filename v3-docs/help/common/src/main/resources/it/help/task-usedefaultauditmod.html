Utilizzare il modulo di revisione attivo

Il modulo di revisione default registra le richieste di autenticazione e autorizzazione nel file di log del server. Per informazioni sulla modifica della posizione del file di log, vedere Configurare impostazioni generali per il log.

Le voci del log di autenticazione includono le seguenti informazioni:

  • Nomi degli utenti che hanno tentato l'autenticazione

  • Settore che ha elaborato la richiesta di accesso

  • Componente URI o EJB del modulo Web richiesto

  • Esito positivo o negativo della richiesta

Indipendentemente dal fatto che il log di revisione sia abilitato o meno, vengono registrati tutti gli eventi di autenticazione non concessa.

Le voci del log di autorizzazione includono le seguenti informazioni:

  • Nomi degli utenti autenticati, se presenti

  • Componente URI o EJB Web richiesto

  • Esito positivo o negativo delle richieste

  1. Nella struttura ad albero, espandere il nodo Configuration.
  2. Nel nodo Configuration, selezionare il nodo Security.

    Viene visualizzata la pagina Security.

  3. Nell'elenco a discesa Audit Modules, selezionare il modulo di revisione default.
  4. Fare clic su Save.
