Distribuisci applicazioni o moduli

Nella pagina Distribuisci applicazioni o moduli è possibile distribuire un'applicazione.

La pagina Distribuisci applicazioni o moduli contiene le opzioni riportate di seguito per tutte le applicazioni. Le opzioni aggiuntive vengono visualizzate solo dopo aver specificato un valore nel campo Posizione per l'applicazione.

Posizione

Posizione dell'archivio per l'applicazione che si sta distribuendo.

Le opzioni riportate di seguito consentono di specificare la posizione da cui è accessibile l'archivio e se si tratta di un file o di una directory.

File incluso in un package da caricare sul server

L'archivio si trova in un file che risiede sul computer client o è accessibile da tale computer.

Il computer client è l'host sul quale viene visualizzata la Console di amministrazione tramite un browser.

Directory o file locale incluso in un package accessibile da GlassFish Server

L'archivio è un file che risiede sul computer server o è un'applicazione decompressa in una directory esplosa.

Il computer server è l'host sul quale è in esecuzione il server di amministrazione del dominio di GlassFish Server.

Tipo

Tipo di applicazione. Sono disponibili le opzioni riportate di seguito.

  • Applicazione Web

  • Applicazione enterprise

  • Client applicazione

  • Modulo connettore

  • Contenitore EJB

  • Altro

Destinazioni

Cluster e istanze standalone su cui distribuire l'applicazione o il modulo. Per spostare le destinazioni desiderate nella colonna Destinazioni selezionate, utilizzare i pulsanti Aggiungi e Aggiungi tutto. Per spostare le destinazioni non necessarie nella colonna Destinazioni disponibili, utilizzare i pulsanti Rimuovi e Rimuovi tutto.

Questa opzione viene visualizzata solo se nel dominio sono stati creati cluster o istanze standalone.

Nelle sezioni seguenti vengono descritte opzioni aggiuntive per i vari tipi di applicazione.

  • Opzioni aggiuntive per un'applicazione Web

  • Opzioni aggiuntive per un'applicazione enterprise

  • Opzioni aggiuntive per un client dell'applicazione

  • Opzioni aggiuntive per un modulo connettore

  • Opzioni aggiuntive per un contenitore EJB

Opzioni aggiuntive per un'applicazione Web

Se il tipo di applicazione è Applicazione Web, vengono visualizzate le opzioni riportate di seguito.

Radice contesto

Percorso dell'applicazione. Nell'URL dell'applicazione Web la radice contesto si trova subito dopo il numero di porta (http://host:porta/radice-contesto/...). La radice contesto deve iniziare con una barra, ad esempio /hello.

Nome applicazione

Nome dell'applicazione.

Il nome può includere un identificativo di versione opzionale, che segue il nome ed è separato da questo da due punti (:). L'identificativo di versione deve iniziare con una lettera o un numero. Può contenere caratteri alfanumerici più caratteri di sottolineatura (_), trattini (-) e punti (.). Per ulteriori informazioni sulle versioni di moduli e applicazioni, vedere Module and Application Versions in Oracle GlassFish Server Application Deployment Guide.

Server virtuali

Server virtuali associati all'applicazione corrente.

L'opzione Server virtuali viene visualizzata se esiste solo l'istanza server predefinita server. Se esistono cluster o altre istanze server standalone, è possibile selezionare i server virtuali dopo la distribuzione. Andare alla pagina Modifica applicazione, selezionare la scheda Destinazione, quindi selezionare Gestisci server virtuali per la destinazione desiderata.

Stato

Se questa opzione è selezionata, l'applicazione è abilitata. Questa opzione è selezionata per impostazione predefinita.

Disponibilità

Se la casella di controllo Abilitato è selezionata, l'alta disponibilità è abilitata per le sessioni Web e per la definizione di checkpoint e la passivazione potenziale dei bean di sessione con conservazione dello stato (SFSB). Se questa opzione è impostata su false (valore predefinito), il salvataggio delle sessioni Web e la definizione di checkpoint SFSB vengono disabilitati per l'applicazione, l'applicazione Web o il modulo EJB specificato. Se questa opzione è impostata su true, l'applicazione o il modulo specificato viene abilitato per l'alta disponibilità. Impostare questa opzione su true solo se l'alta disponibilità è configurata e abilitata ai livelli superiori, ad esempio a livello di server e di contenitore.

Questa opzione viene visualizzata se esistono cluster o istanze server standalone oltre all'istanza server predefinita (server).

Precompila JSP

Se questa opzione è selezionata, i file JSP (JavaServer Pages) vengono precompilati. Se questa opzione è disabilitata, i file JSP vengono compilati in fase di esecuzione al primo accesso. Questa opzione è disabilitata per impostazione predefinita.

Esegui verificatore

Se questa opzione è selezionata, i descrittori di distribuzione vengono verificati prima della distribuzione. Se la verifica non riesce, la distribuzione non viene eseguita. Il verificatore esamina la struttura e il contenuto dei descrittori di distribuzione. La verifica di applicazioni di grandi dimensioni richiede spesso tempi lunghi. Questa opzione è disabilitata per impostazione predefinita.

I package del verificatore devono essere installati tramite la utility Strumento di aggiornamento; in caso contrario, viene registrata un'avvertenza e l'opzione viene ignorata.

Imponi ridistribuzione

Se questa opzione è selezionata, l'applicazione viene ridistribuita se è già stata distribuita. Se questa opzione non è selezionata, un tentativo di distribuire un'applicazione già distribuita causa un errore. Questa opzione è disabilitata per impostazione predefinita.

Mantieni stato

Questa opzione controlla se le sessioni Web, le istanze SFSB e i timer EJB creati in modo persistente vengono mantenuti tra le ridistribuzioni.

Questa opzione è disabilitata per impostazione predefinita. È supportata solo nell'istanza server predefinita, denominata server. Non è supportata e viene ignorata per qualsiasi altra destinazione.

Alcune modifiche apportate a un'applicazione tra le ridistribuzioni impediscono il corretto funzionamento di questa funzione. Ad esempio, non modificare il set di variabili di istanza nella classe di bean SFSB.

Per le applicazioni Web, questa funzione è applicabile solo se nel file glassfish-web-app.xml l'attributo persistence-type dell'elemento session-manager è file.

Per le istanze di bean di sessione con conservazione dello stato, il tipo di persistenza senza alta disponibilità viene impostato nel server (opzione Tipo di persistenza SFSB) e deve essere impostato su file, che è il valore predefinito e consigliato.

Se un elemento attivo, quale una sessione Web, un'istanza SFSB o un timer EJB, non viene preservato o ripristinato, nessuno di tali elementi sarà disponibile una volta completata la ridistribuzione. La ridistribuzione viene tuttavia proseguita e viene registrata un'avvertenza.

Per preservare i dati con stato attivo, GlassFish Server li serializza e li salva nella memoria. Per ripristinare i dati, il classloader dell'applicazione appena ridistribuita deserializza i dati salvati in precedenza.

Conserva risorse con ambito applicazione

Se l'opzione è selezionata, le risorse con ambito a livello di applicazione vengono conservate e ripristinate durante la ridistribuzione. Questa opzione non è selezionata per impostazione predefinita.

Librerie

Lista di valori separati da virgole di file JAR della libreria specifici del modulo o dell'applicazione. I percorsi possono essere assoluti o relativi. Un percorso relativo fa riferimento a domain-dir/lib/applibs. Se il percorso è assoluto, deve essere accessibile al server di amministrazione del dominio (DAS), ovvero deve trovarsi in domain-dir. Le librerie vengono rese disponibili all'applicazione nell'ordine in cui vengono specificate.

Se si seleziona questa casella, queste restrizioni Java EE 6 vengono rimosse.

I package del verificatore devono essere installati tramite la utility Strumento di aggiornamento; in caso contrario, viene registrata un'avvertenza e l'opzione viene ignorata.

Imponi ridistribuzione

Se questa opzione è selezionata, l'applicazione viene ridistribuita se è già stata distribuita. Se questa opzione non è selezionata, un tentativo di distribuire un'applicazione già distribuita causa un errore. Questa opzione è disabilitata per impostazione predefinita.

Mantieni stato

Questa opzione controlla se le sessioni Web, le istanze SFSB e i timer EJB creati in modo persistente vengono mantenuti tra le ridistribuzioni.

Questa opzione è disabilitata per impostazione predefinita. È supportata solo nell'istanza server predefinita, denominata server. Non è supportata e viene ignorata per qualsiasi altra destinazione.

Alcune modifiche apportate a un'applicazione tra le ridistribuzioni impediscono il corretto funzionamento di questa funzione. Ad esempio, non modificare il set di variabili di istanza nella classe di bean SFSB.

Per le applicazioni Web, questa funzione è applicabile solo se nel file glassfish-web-app.xml l'attributo persistence-type dell'elemento session-manager è file.

Per le istanze di bean di sessione con conservazione dello stato, il tipo di persistenza senza alta disponibilità viene impostato nel server (opzione Tipo di persistenza SFSB) e deve essere impostato su file, che è il valore predefinito e consigliato.

Se un elemento attivo, quale una sessione Web, un'istanza SFSB o un timer EJB, non viene preservato o ripristinato, nessuno di tali elementi sarà disponibile una volta completata la ridistribuzione. La ridistribuzione viene tuttavia proseguita e viene registrata un'avvertenza.

Per preservare i dati con stato attivo, GlassFish Server li serializza e li salva nella memoria. Per ripristinare i dati, il classloader dell'applicazione appena ridistribuita deserializza i dati salvati in precedenza.

Librerie

Lista di valori separati da virgole di file JAR della libreria specifici del modulo o dell'applicazione. Specifica un percorso assoluto o relativo. Un percorso relativo fa riferimento a domain-dir/lib/applibs. Se il percorso è assoluto, deve essere accessibile al server di amministrazione del dominio (DAS), ovvero deve trovarsi in domain-dir. Le librerie vengono rese disponibili all'applicazione nell'ordine in cui vengono specificate.
