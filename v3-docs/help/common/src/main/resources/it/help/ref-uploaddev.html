Deploy Applications or Modules

Nella pagina Deploy Applications or Modules è possibile distribuire un'applicazione.

La pagina Deploy Applications or Modules contiene le seguenti opzioni per tutte le applicazioni. Le opzioni aggiuntive vengono visualizzate dopo che viene specificato un valore nel campo Location per l'applicazione.

Location

La posizione dell'archivio per l'applicazione da distribuire.

Le seguenti opzioni consentono di specificare la posizione da cui è accessibile l'archivio e se si tratta di un file o di una directory.

Packaged File to Be Uploaded to the Server

L'archivio si trova in un file del computer client o è accessibile da tale computer.

Il computer client è l'host in cui viene visualizzata Administration Console tramite un browser.

Local Packaged File or Directory That Is Accessible From the GlassFish Server

L'archivio è un file che si trova in un computer server o è un'applicazione decompressa in una directory esplosa.

Il computer server è l'host in cui è in esecuzione il server di amministrazione del dominio di GlassFish Server.

Type

Tipo di applicazione. Sono disponibili le seguenti opzioni:

  • Web Application

  • Enterprise Application

  • Application Client

  • Connector Module

  • EJB Jar

  • Other

Targets

Cluster e istanze standalone in cui distribuire l'applicazione o il modulo. Per spostare le destinazioni desiderate nella colonna Selected Targets, utilizzare i pulsanti Add e Add All. Per spostare le destinazioni non necessarie nella colonna Available Targets, utilizzare i pulsanti Remove e Remove All.

Questa opzione viene visualizzata solo se nel dominio sono stati creati cluster o istanze standalone.

Nelle sezioni seguenti vengono descritte opzioni aggiuntive per i vari tipi di applicazione.

  • Opzioni aggiuntive per applicazioni Web

  • Opzioni aggiuntive per applicazioni Enterprise

  • Opzioni aggiuntive per un client dell'applicazione

  • Opzioni aggiuntive per un modulo connettore

  • Opzioni aggiuntive per un jar EJB

Opzioni aggiuntive per applicazioni Web

Se l'applicazione è di tipo Web Application, vengono visualizzate le seguenti opzioni.

Context Root

Percorso dell'applicazione. Nell'URL dell'applicazione Web, la root di contesto si trova subito dopo il numero della porta (http:// host:port/context-root/... ). La root di contesto deve iniziare con una barra, ad esempio, /hello.

Application Name

Nome dell'applicazione.

Il nome può includere un identificatore di versione opzionale, che segue il nome ed è separato da questo da due punti (:). L'identificatore di versione deve iniziare con una lettera o un numero. Può contenere caratteri alfanumerici più trattini di sottolineatura (_), trattini (-) e punti (.). Per ulteriori informazioni sulle versioni di moduli e applicazioni, vedere Module and Application Versions in Oracle GlassFish Server 3.1 Application Deployment Guide.

Virtual Servers

Server virtuali associati all'applicazione.

L'opzione Virtual Servers viene visualizzata se esiste solo l'istanza server predefinita server. Se esistono cluster o altre istanze server standalone, è possibile selezionare i server virtuali dopo la distribuzione. Andare alla pagina Edit Application, selezionare la scheda Target, quindi selezionare Manage Virtual Servers per la destinazione desiderata.

Status

Se l'opzione è selezionata, l'applicazione è abilitata. Questa opzione è selezionata per impostazione predefinita.

Availability

Se è selezionata la casella di controllo Enabled, è abilitata l'alta disponibilità per le sessioni Web e per la definizione di checkpoint e potenzialmente la passivazione dei bean di sessione con stato (SFSB). Se si imposta false (valore predefinito), il salvataggio delle sessioni Web e la definizione di checkpoint SFSB vengono disabilitati per l'applicazione, l'applicazione Web o il modulo EJB specificato. Se si imposta true, l'applicazione o il modulo specificato viene abilitato per l'alta disponibilità. Impostare l'opzione su true solo se l'alta disponibilità è configurata e abilitata ai livelli superiori, ad esempio al livello del server e del contenitore.

Questa opzione viene visualizzata se esistono cluster o istanze server standalone oltre all'istanza server predefinita (server).

Precompile JSPs

Se questa opzione è selezionata, i file JSP (JavaServer Pages ) vengono precompilati. Se l'opzione è disabilitata, i file JSP vengono compilati in fase di runtime al primo accesso. Questa opzione è disabilitata per impostazione predefinita.

Run Verifier

Se questa opzione è selezionata, prima della distribuzione vengono verificati i descrittori di distribuzione. Se la verifica non riesce, la distribuzione non viene eseguita. Il verificatore esamina la struttura e il contenuto dei descrittori di distribuzione. La verifica di applicazioni di grandi dimensioni richiede spesso tempi lunghi. Questa opzione è disabilitata per impostazione predefinita.

I pacchetti di verifica devono essere installati tramite l'utility Update Tool; in caso contrario verrà registrato un avviso e l'opzione verrà ignorata.

Force Redeploy

Se questa opzione è selezionata e l'applicazione è già stata distribuita, viene ridistribuita. Se l'opzione non è selezionata, un tentativo di distribuire un'applicazione già distribuita causa un errore. Questa opzione è disabilitata per impostazione predefinita.

Keep State

Questa opzione determina se le sessioni Web, le istanze SFSB e i timer EJB creati in modo persistente vengono conservati tra le ridistribuzioni.

Questa opzione è disabilitata per impostazione predefinita. È supportata solo nell'istanza server predefinita, denominata server. Non è supportata e viene ignorata per qualsiasi altra destinazione.

Alcune modifiche apportate a un'applicazione tra le ridistribuzioni impediscono a questa funzione di operare correttamente. Ad esempio, non modificare l'insieme di variabili delle istanze nella classe bean SFSB.

Per le applicazioni Web, questa funzione è applicabile solo se nel file glassfish-web-app.xml l'attributo persistence-type dell'elemento session-manager è file.

Per le istanze di bean di sessione con stato, il tipo di persistenza senza alta disponibilità viene impostato nel server (opzione SFSB Persistence Type) e deve essere impostato su file, che è il valore predefinito e consigliato.

Se un elemento attivo quale una sessione Web, un'istanza SFSB o un timer EJB non viene preservato o ripristinato, nessuno di tali elementi sarà disponibile una volta completata la ridistribuzione. La ridistribuzione viene tuttavia proseguita e viene registrato un avviso.

Per preservare i dati con stato attivo, GlassFish Server li serializza e li salva nella memoria. Per ripristinare i dati, il classloader dell'applicazione appena ridistribuita deserializza i dati precedentemente salvati.

Preserve Application Scoped Resources

Se l'opzione è selezionata, le risorse con ambito a livello di applicazione vengono preservate e ripristinate durante la ridistribuzione. Questa opzione non è selezionata per impostazione predefinita.

Libraries

Elenco di file JAR della libreria separato da virgole e specifico per il modulo o l'applicazione. I percorsi possono essere assoluti o relativi. Un percorso relativo fa riferimento a domain-dir/lib/applibs. Se il percorso è completo, deve essere accessibile al DAS (Domain Administration Server, server di amministrazione dominio) ossia deve trovarsi in domain-dir. Le librerie sono disponibili per l'applicazione nell'ordine in cui sono specificate.

Description

Descrizione dell'applicazione.

Se si seleziona questa casella, tali restrizioni Java EE 6 vengono rimosse.

Per le applicazioni Web, questa funzione è applicabile solo se nel file glassfish-web-app.xml l'attributo persistence-type dell'elemento session-manager è file.

Per le istanze di bean di sessione con stato, il tipo di persistenza senza alta disponibilità viene impostato nel server (opzione SFSB Persistence Type) e deve essere impostato su file, che è il valore predefinito e consigliato.

Se un elemento attivo quale una sessione Web, un'istanza SFSB o un timer EJB non viene preservato o ripristinato, nessuno di tali elementi sarà disponibile una volta completata la ridistribuzione. La ridistribuzione viene tuttavia proseguita e viene registrato un avviso.

Per preservare i dati con stato attivo, GlassFish Server li serializza e li salva nella memoria. Per ripristinare i dati, il classloader dell'applicazione appena ridistribuita deserializza i dati precedentemente salvati.

Libraries

Elenco di file JAR della libreria separato da virgole e specifico per il modulo o l'applicazione. Specifica un percorso assoluto o relativo. Un percorso relativo fa riferimento a domain-dir/lib/applibs. Se il percorso è completo, deve essere accessibile al DAS (Domain Administration Server, server di amministrazione dominio) ossia deve trovarsi in domain-dir. Le librerie sono disponibili per l'applicazione nell'ordine in cui sono specificate.
