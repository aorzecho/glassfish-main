Distribuire un'applicazione enterprise

  1. Nell'albero di navigazione selezionare il nodo Applicazioni.

    Viene visualizzata la pagina Applicazioni.

  2. Nella tabella Applicazioni distribuite, fare clic sul pulsante Distribuisci.

    Viene visualizzata la pagina Distribuisci applicazioni o moduli.

  3. Specificare la posizione dell'applicazione da distribuire.

    • Se l'applicazione è un file di archivio che si trova nel computer client o è accessibile da tale computer, selezionare l'opzione File incluso in un package da caricare sul server.

      Il computer client è l'host in cui viene visualizzata Console di amministrazione tramite un browser.

      Fare clic su Sfoglia per individuare il file oppure digitarne il percorso completo.

    • Se l'applicazione è un file di archivio che si trova nel computer server o è un'applicazione decompressa da una directory esplosa, selezionare l'opzione Directory o file locale incluso in un package accessibile da GlassFish Server.

      Il computer server è l'host in cui è in esecuzione il server di amministrazione del dominio (DAS, Domain Administration Server).

      Fare clic su Sfoglia file per cercare un file o su Sfoglia cartelle per cercare una directory. In alternativa, digitare il percorso completo del file o della directory. La distribuzione da una directory esplosa deve essere eseguita da sviluppatori esperti e non è consigliata per gli ambienti di produzione.

  4. Dall'elenco a discesa Tipo, selezionare Applicazione enterprise.

    Se è stata selezionata l'opzione File incluso in un package da caricare sul server o Directory o file locale incluso in un package accessibile da GlassFish Server, il tipo di applicazione viene rilevato automaticamente. Se è stata selezionata l'opzione Directory o file locale incluso in un package accessibile da GlassFish Server, è necessario specificare il tipo manualmente.

  5. Nel campo Nome applicazione, lasciare il nome predefinito, che è il nome della directory o il prefisso del nome file, oppure digitare un altro nome.

    Se si è scelto di caricare un file viene visualizzato il nome predefinito. Il nome dell'applicazione deve essere univoco.

    Il nome può includere un identificativo di versione opzionale, che segue il nome ed è separato da questo da due punti (:). L'identificativo di versione deve iniziare con una lettera o un numero. Può contenere caratteri alfanumerici più trattini di sottolineatura (_), trattini (-) e punti (.). Per ulteriori informazioni sulle versioni di moduli e applicazioni, vedere Module and Application Versions in Oracle GlassFish Server Application Deployment Guide.

  6. Dall'elenco a discesa Server virtuale, selezionare il server virtuale da associare a questa applicazione.

    L'opzione Server virtuali viene visualizzata se esiste solo l'istanza server predefinita server. Se esistono cluster o altre istanze server standalone, è possibile selezionare i server virtuali dopo la distribuzione. Andare alla pagina Modifica applicazione, selezionare la scheda Destinazione, quindi selezionare Gestisci server virtuali per la destinazione desiderata.

  7. Selezionare la casella di controllo Stato abilitato per abilitare l'applicazione.

    Questa opzione è selezionata per impostazione predefinita. Se l'opzione è disabilitata, l'applicazione non è disponibile.

  8. Selezionare la casella di controllo Disponibilità abilitata per abilitare la persistenza delle sessioni e la definizione di checkpoint SFSB per l'applicazione.

    Se è selezionata la casella di controllo Abilitato, è abilitata l'alta disponibilità per le sessioni Web e per la definizione di checkpoint e potenzialmente la passivazione dei bean di sessione con conservazione dello stato (SFSB). Se si imposta false (valore predefinito), il salvataggio di tutte le sessioni Web e la definizione di checkpoint SFSB vengono disabilitati per l'applicazione Web specificata. Se si imposta true, l'applicazione specificata viene abilitata per l'alta disponibilità. Impostare l'opzione su true solo se l'alta disponibilità è configurata e abilitata ai livelli superiori, ad esempio al livello del server e del contenitore.

    Questa opzione viene visualizzata se esistono cluster o istanze server standalone oltre all'istanza server predefinita (server).

  9. Selezionare la casella di controllo Java Web Start per abilitare l'accesso a Java Web Start per un client dell'applicazione.

    Questa opzione è abilitata per impostazione predefinita.

  10. Selezionare la casella di controllo Precompila JSP per precompilare i file JSP (JavaServer Pages).

    Se l'opzione è disabilitata, i file JSP vengono compilati in fase di esecuzione al primo accesso. Questa opzione è disabilitata per impostazione predefinita.

  11. Selezionare la casella di controllo Esegui verificatore per verificare i descrittori di distribuzione prima della distribuzione.

    Se l'opzione è selezionata ed è impossibile effettuare la verifica, la distribuzione non viene eseguita. Il verificatore esamina la struttura e il contenuto dei descrittori di distribuzione. La verifica di applicazioni di grandi dimensioni richiede spesso tempi lunghi. Questa opzione è disabilitata per impostazione predefinita.

    I package del verificatore devono essere installati tramite la utility Strumento di aggiornamento; in caso contrario verrà registrata un'avvertenza e l'opzione verrà ignorata.

  12. Selezionare la casella di controllo Compatibilità per utilizzare i requisiti di visibilità JAR di GlassFish Server v2 anziché i più rigidi requisiti Java EE 6 implementati nelle versioni v3 di GlassFish Server, compresa la 3.1.2.

    Questa opzione è disabilitata per impostazione predefinita.

    La specifica della piattaforma Java EE 6 impone requisiti più rigidi rispetto a Java EE 5, per cui i file JAR possono essere visibili per diversi moduli in un file EAR. In particolare, i client di applicazioni non devono avere accesso ai file JAR EJB o ad altri file JAR nel file EAR a meno che per i riferimenti non sia utilizzato il meccanismo Java SE standard (ad esempio, estensioni) o il meccanismo libreria-directory di Java EE. Se si seleziona questa casella, tali restrizioni Java EE 6 vengono rimosse.

  13. Selezionare la casella di controllo Imponi ridistribuzione per forzare la ridistribuzione se l'applicazione è già stata distribuita.

    Se l'opzione non è selezionata, un tentativo di distribuire un'applicazione già distribuita causa un errore. Questa opzione è disabilitata per impostazione predefinita.

  14. Selezionare la casella di controllo Mantieni stato per conservare le sessioni Web, le istanze SFSB e i timer EJB creati in modo persistente tra le ridistribuzioni.

    Questa opzione è disabilitata per impostazione predefinita. È supportata solo nell'istanza server predefinita, denominata server. Non è supportata e viene ignorata per qualsiasi altra destinazione.

    Alcune modifiche apportate a un'applicazione tra le ridistribuzioni impediscono a questa funzione di operare correttamente. Ad esempio, non modificare l'insieme di variabili delle istanze nella classe bean SFSB.

    Per le applicazioni Web, questa funzione è applicabile solo se nel file glassfish-web-app.xml l'attributo persistence-type dell'elemento session-manager è file.

    Per le istanze di bean di sessione con conservazione dello stato, il tipo di persistenza senza alta disponibilità viene impostato nel server (opzione Tipo di persistenza SFSB) e deve essere impostato su file, che è il valore predefinito e consigliato.

    Se un elemento attivo quale una sessione Web, un'istanza SFSB o un timer EJB non viene preservato o ripristinato, nessuno di tali elementi sarà disponibile una volta completata la ridistribuzione. La ridistribuzione viene tuttavia proseguita e viene registrata un'avvertenza.

    Per preservare i dati con stato attivo, GlassFish Server li serializza e li salva nella memoria. Per ripristinare i dati, il classloader dell'applicazione appena ridistribuita deserializza i dati precedentemente salvati.

  15. Selezionare la casella di controllo Conserva risorse con ambito applicazione per preservare le risorse con ambito a livello di applicazione e ripristinarle durante la ridistribuzione.

    Questa opzione è disabilitata per impostazione predefinita.

  16. Nel campo Libraries, digitare una lista separata da virgole di file JAR della libreria specifici di questa applicazione.

    I percorsi possono essere assoluti o relativi. Un percorso relativo fa riferimento a domain-dir/lib/applibs. Se il percorso è assoluto, deve essere accessibile al server di amministrazione del dominio (Domain Administration Server, DAS) ossia deve trovarsi in domain-dir. Le librerie sono disponibili per l'applicazione nell'ordine in cui sono specificate.

  17. Nel campo Descrizione, digitare una descrizione per l'applicazione.

  18. Per aggiungere istanze server e cluster di destinazione in cui distribuire il modulo o l'applicazione, selezionarli dalla lista Destinazioni disponibili e utilizzare il pulsante Aggiungi per spostarli nella lista Destinazioni selezionate.

    L'opzione Destinazione viene visualizzata se esistono cluster o istanze server standalone oltre all'istanza server predefinita (server). Per impostazione predefinita, la distribuzione del modulo o dell'applicazione viene eseguita nell'istanza server predefinita, denominata server. Utilizzare il pulsante Aggiungi tutto per eseguire la distribuzione in tutte le destinazioni. Utilizzare i pulsanti Rimuovi e Rimuovi tutto per rimuovere destinazioni dalla lista Destinazioni selezionate.

  19. Fare clic su OK.
