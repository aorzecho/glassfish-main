Visualizzare informazioni su endpoint per servizi Web

  1. Nella struttura ad albero, selezionare il nodo Applications.

    Viene visualizzata la pagina Applications.

  2. Nella pagina Applications, fare clic sul nome del servizio Web di cui si visualizzano le informazioni relative all'endpoint.

    Il nome può includere un identificatore di versione opzionale, che segue il nome ed è separato da questo da due punti (:). Per ulteriori informazioni sulle versioni di moduli e applicazioni, vedere Module and Application Versions in Oracle GlassFish Server 3.1 Application Deployment Guide.

    Viene visualizzata la pagina Edit Application.

  3. Nella tabella Modules and Components, fare clic sul collegamento View Endpoint per il sottocomponente.

    Viene visualizzata la pagina Web Service Endpoint Information.

  4. Se il collegamento Tester è visibile, fare clic su di esso per visualizzare una pagina che consente di fornire parametri per testare metodi di servizio Web.

    Il collegamento Tester è visibile solo se il Deployment Type dell'applicazione è diverso da RI (per esempio, se è 109). La pagina Tester viene aperta in una nuova scheda o finestra del browser.

  5. Fare clic sul collegamento WSDL per visualizzare il file WSDL (Web Services Definition Language) per l'endpoint.

    Il file WSDL viene aperto in una nuova scheda o finestra del browser.

    Quando si fa clic sul collegamento WSDL, alcuni browser Web tentano di interpretare l'origine WSDL visualizzando quindi una pagina vuota. Per visualizzare l'origine WSDL con tali browser, utilizzare il comando del browser per la visualizzazione dell'origine nella pagina vuota.

  6. Fare clic sul collegamento Application Name o sul pulsante Back per tornare alla pagina Edit Application.
