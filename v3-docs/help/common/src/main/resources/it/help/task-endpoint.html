Visualizzare le informazioni sull'endpoint per un servizio Web

  1. Nell'albero di navigazione selezionare il nodo Applicazioni.

    Viene visualizzata la pagina Applicazioni.

  2. Nella pagina Applicazioni, fare clic sul nome del servizio Web di cui si visualizzano le informazioni relative all'endpoint.

    Il nome può includere un identificativo di versione opzionale, che segue il nome ed è separato da questo da due punti (:). Per ulteriori informazioni sulle versioni di moduli e applicazioni, vedere Module and Application Versions in Oracle GlassFish Server Application Deployment Guide.

    Viene visualizzata la pagina Modifica applicazione.

  3. Nella tabella Moduli e componenti, fare clic sul collegamento Visualizza endpoint per il componente secondario.

    Viene visualizzata la pagina Informazioni sull'endpoint del servizio Web.

  4. Se il collegamento Tester è visibile, fare clic su di esso per visualizzare una pagina che consente di fornire parametri per testare metodi di servizio Web.

    Il collegamento Tester è visibile solo se il tipo di distribuzione dell'applicazione è diverso da RI (per esempio, se è 109). La pagina Tester viene aperta in una nuova scheda o finestra del browser.

  5. Fare clic sul collegamento WSDL per visualizzare il file WSDL (Web Services Definition Language) per l'endpoint.

    Il file WSDL viene aperto in una nuova scheda o finestra del browser.

    Quando si fa clic sul collegamento WSDL, alcuni browser Web tentano di interpretare l'origine WSDL visualizzando quindi una pagina vuota. Per visualizzare l'origine WSDL con tali browser, utilizzare il comando del browser per la visualizzazione dell'origine nella pagina vuota.

  6. Fare clic sul collegamento Nome applicazione o sul pulsante Indietro per tornare alla pagina Modifica applicazione.
