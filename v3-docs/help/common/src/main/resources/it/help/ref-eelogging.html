Logger Settings

Nella pagina Logger Settings è possibile configurare la registrazione del log per l'istanza o il cluster GlassFish Server selezionato.

La pagina Logger Settings contiene le seguenti opzioni.

Configuration Name

Nome della configurazione a cui si applicano le impostazioni visualizzate nella pagina. Campo di sola lettura.

Write to System Log

Se l'opzione è selezionata, l'output di log viene inviato alla risorsa syslog oltre al log del server. Questa caratteristica è disponibile soltanto nei sistemi Solaris e Linux. Questa opzione è disabilitata per impostazione predefinita.

Log File

Nome alternativo o posizione del file di log del server. La posizione predefinita è glassfish/domains/domain1/logs/server.log.

Log Handler

Nome classe assoluto di un gestore del log personalizzato. Un gestore del log personalizzato consente di inviare log a una destinazione diversa da server.log o syslog. L'handler di sistema deve estendere la classe java.util.logging.Handler (un'API conforme allo standard JSR 047). Inserire la classe handler nel classpath di GlassFish Server, in modo che l'handler venga installato durante l'avvio del server.

File Rotation Limit

Dimensione che il file di log del server deve raggiungere per essere sottoposto a rotazione. Quando il log del server raggiunge la dimensione specificata, in GlassFish Server viene creato un nuovo file vuoto denominato server.log e il file precedente viene rinominato server.log_date, dove date corrisponde alla data e all'ora di rotazione del file. Il valore predefinito è 2 MB. Il valore limite minimo è 500 KB; se si specifica un valore inferiore, il file verrà sottoposto a rotazione una volta raggiunti i 500 KB. Per disattivare la rotazione del file di log impostare il valore su 0.

File Rotation Time Limit

Numero di minuti al termine dei quali si desidera eseguire la rotazione del log del server. Il valore predefinito è 0. Indica che il file viene sottoposto a rotazione quando raggiunge la dimensione specificata nel campo File Rotation Limit. Se si specificano uno o più minuti, il limite di tempo ottiene la precedenza sul limite di dimensione specificato nel campo File Rotation Limit.

Flush Frequency

Numero massimo di messaggi scrivibili contemporaneamente dalla coda al log del server. Il valore predefinito è 1.
