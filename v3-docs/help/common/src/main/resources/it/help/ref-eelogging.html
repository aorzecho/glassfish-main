Impostazioni logger

Nella pagina Impostazioni logger è possibile configurare il log per l'istanza o il cluster GlassFish Server selezionato.

La pagina Impostazioni logger contiene le opzioni riportate di seguito.

Nome configurazione

Nome della configurazione a cui vengono applicate le impostazioni definite in questa pagina. Questo campo è di sola lettura.

Scrivi nel log di sistema

Se questa opzione è selezionata, l'output di log viene inviato alla risorsa syslog oltre al log del server. Questa funzione è disponibile solo nei sistemi Solaris e Linux. Questa opzione è disabilitata per impostazione predefinita.

File di log

Nome o posizione alternativa del file di log del server. La posizione predefinita è domain-dir/logs/server.log.

Handler di log

Nome classe assoluto di un handler di log personalizzato. Un handler di log personalizzato consente di inviare log a una destinazione diversa da server.log o syslog. L'handler personalizzato deve estendere la classe java.util.logging.Handler (un'API conforme allo standard JSR 047). Inserire la classe handler nel classpath di GlassFish Server in modo che l'handler venga installato durante l'avvio del server.

Limite di rotazione file

Dimensione in byte che il file di log del server deve raggiungere prima di essere ruotato. Quando il log del server raggiunge la dimensione specificata, GlassFish Server crea un nuovo file vuoto denominato server.log e rinomina il file precedente server.log_data, dove data corrisponde alla data e all'ora di rotazione del file. Il valore predefinito è 2 MB. Il valore limite minimo è 500 KB; se si specifica un valore inferiore, il file viene ruotato quando raggiunge 500 KB. Per disattivare la rotazione del file di log, impostare il valore su 0.

Limite di tempo rotazione file

Numero di minuti al termine dei quali il server deve essere ruotato. Il valore predefinito è 0. Indica che il file viene ruotato quando raggiunge la dimensione specificata nel campo Limite di rotazione file. Se si specificano uno o più minuti, il limite di tempo ha la precedenza sul limite di dimensione specificato nel campo Limite di rotazione file.

Frequenza flush

Numero massimo di messaggi che possono essere scritti contemporaneamente dalla coda al log del server. Il valore predefinito è 1.
