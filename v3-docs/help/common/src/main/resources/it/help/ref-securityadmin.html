Amministrazione sicura

Nella pagina Amministrazione sicura è possibile abilitare o disabilitare l'amministrazione sicura.

Quando l'amministrazione sicura è abilitata:

  • DAS e le istanze nel dominio utilizzano i certificati SSL per cifrare i messaggi che inviato tra loro;

  • DAS accetta i messaggi di amministrazione dai client di amministrazione remoti, ad esempio la utility asadmin e dagli IDE.

La pagina Amministrazione sicura contiene le informazioni riportate di seguito.

Abilita amministrazione sicura

Pulsante che consente di abilitare l'amministrazione sicura. Questo pulsante viene visualizzato solo se l'amministrazione sicura non è abilitata.

Nota:

Se un utente amministrativo nel dominio dispone di una password vuota, i tentativi di abilitare l'amministrazione sicura non riescono.
Disabilita amministrazione sicura

Pulsante che consente di disabilitare l'amministrazione sicura. Questo pulsante viene visualizzato solo se l'amministrazione sicura è abilitata.

Alias di amministrazione

Alias che fa riferimento al certificato SSL/TLS su DAS. Questo alias viene utilizzato da DAS per identificarsi con le istanze. Il valore predefinito è s1as. Questa opzione viene visualizzata solo se l'amministrazione sicura non è abilitata.

Alias istanze

Alias che fa riferimento al certificato SSL/TLS sulle istanze. Questo alias viene utilizzato dalle istanze per identificarsi con DAS. Il valore predefinito è glassfish-instance. Questa opzione viene visualizzata solo se l'amministrazione sicura non è abilitata.
