Sicurezza

Nella pagina Sicurezza è possibile impostare le proprietà di sicurezza per l'istanza o il cluster GlassFish Server selezionato.

La pagina Sicurezza contiene le opzioni riportate di seguito.

Nome configurazione

Nome della configurazione a cui vengono applicate le impostazioni definite in questa pagina. Questo campo è di sola lettura.

Security Manager

Se questa opzione è selezionata, viene abilitato il Security Manager per il dominio. Questa opzione è disabilitata per impostazione predefinita.

Quando questa opzione è abilitata, l'opzione JVM -Djava.security.manager viene aggiunta alle impostazioni JVM di GlassFish Server. Vedere Configurare le opzioni JVM. Per abilitare questa modifica, è necessario riavviare il server.

Assicurarsi di disporre delle autorizzazioni corrette per tutte le applicazioni. È possibile disattivare il Security Manager per migliorare le prestazioni.

Log di audit

Se questa opzione è abilitata, il server carica ed esegue tutti i moduli di audit specificati nell'impostazione Moduli di audit. Se l'opzione è disabilitata, il server non accede ai moduli di audit. Questa opzione è disabilitata per impostazione predefinita.

Realm predefinito

Realm (predefinito) attivo utilizzato dal server per l'autenticazione. Le applicazioni utilizzano questo realm a meno che il descrittore di distribuzione non specifica un realm differente. Tutti i realm configurati vengono visualizzati nella lista. Il valore predefinito è file.

Principal predefinito

Specifica il nome utente predefinito. Il server utilizza questo nome utente se non viene fornito un altro principal. Se si digita un valore in questo campo, digitare la password corrispondente nel campo Password principal predefinito.

Un principal predefinito non è richiesto per il normale funzionamento del server.

Password principal predefinito

Password del principal predefinito specificata nel campo Principal predefinito. Un principal predefinito non è richiesto per il normale funzionamento del server.

JACC

Nome classe di un provider JACC configurato. Il valore predefinito è default.

Moduli di audit

Moduli del provider di audit che verranno utilizzati dal sottosistema di audit se è abilitato il log di audit. Per impostazione predefinita, il server utilizza un modulo di audit denominato default. Per informazioni sulla creazione di nuovi moduli di audit, vedere Creare un modulo di audit.

Mapping principal/ruolo predefinito abilitato

Se questa opzione è selezionata, il mapping principal/ruolo predefinito viene applicato a tutte le applicazioni per le quali non è definito un mapping specifico.

Classe principal mappata

Specifica un'implementazione personalizzata della classe java.security.Principal utilizzata nel mapping principal/ruolo predefinito se è abilitato il mapping principal/ruolo predefinito.

Proprietà aggiuntive

Proprietà di sicurezza aggiuntive per il server.

Le proprietà valide dipendono dal tipo di realm selezionato nel campo Realm predefinito e vengono in genere specificate quando si modifica un realm. Per informazioni dettagliate, vedere i seguenti argomenti:

  • Proprietà specifiche della classe FileRealm

  • Proprietà specifiche della classe CertificateRealm

  • Proprietà specifiche della classe JDBCRealm

  • Proprietà specifiche della classe LDAPRealm

  • Proprietà specifiche della classe SolarisRealm

  • Proprietà specifiche della classe PamRealm
