Security

Nella pagina Security è possibile impostare le proprietà di sicurezza per l'istanza o il cluster GlassFish Server selezionato.

La pagina Security contiene le seguenti opzioni.

Configuration Name

Nome della configurazione a cui si applicano le impostazioni visualizzate nella pagina. Campo di sola lettura.

Security Manager

Se l'opzione è selezionata viene abilitato il gestore della sicurezza per il dominio. Questa opzione è disabilitata per impostazione predefinita.

Quando questa opzione è abilitata, l'opzione JVM -Djava.security.manager viene aggiunta alle impostazioni JVM di GlassFish Server. Vedere Configurare le opzioni JVM. Per rendere effettiva questa modifica è necessario riavviare il server.

Verificare di aver impostato le autorizzazioni necessarie per tutte le applicazioni. È possibile disattivare il gestore della sicurezza per migliorare le prestazioni.

Audit Logging

Se questa opzione è abilitata, il server carica ed esegue tutti i moduli di revisione specificati nell'impostazione Audit Modules. Se l'opzione è disabilitata, il server non accede ai moduli di revisione. Questa opzione è disabilitata per impostazione predefinita.

Default Realm

Settore (predefinito) attivo che il server utilizza per l'autenticazione. Le applicazioni utilizzeranno tale settore, salvo se il descrittore di distribuzione specifica un settore diverso. Tutti i settori configurati sono visibili nell'elenco. Il valore predefinito è file.

Default Principal

Specifica il nome utente predefinito. Se non è specificata nessun'altra identità, il server utilizza questo nome utente. Se si digita un valore in questo campo, è necessario digitare la password corrispondente nel campo Default Principal Password.

Un'identità predefinita non è obbligatoria per il normale funzionamento del server.

Default Principal Password

Password dell'identità predefinita specificata nel campo Default Principal. Un'identità predefinita non è obbligatoria per il normale funzionamento del server.

JACC

Nome della classe di un provider JACC configurato. Il valore predefinito è default.

Audit Modules

Moduli del provider di revisione che verranno utilizzati dal sottosistema di revisione se viene abilitato il log di revisione. Per impostazione predefinita il server utilizza un modulo di revisione denominato default. Per informazioni sulla creazione di nuovi moduli di revisione, vedere Creare un modulo di revisione.

Default Principal To Role Mapping Enabled

Se l'opzione è selezionata, il mapping identità-ruolo predefinito viene applicato a tutte le applicazioni per le quali non è definito un mapping specifico.

Mapped Principal Class

Specifica un'implementazione personalizzata della classe java.security.Principal utilizzata nel mapping identità-ruolo predefinito, se è abilitato il mapping identità-ruolo predefinito.

Additional Properties

Proprietà di sicurezza aggiuntive per il server.

Le proprietà valide dipendono dal tipo di settore selezionato nel campo Default Realm e vengono in genere specificate quando si modifica un settore. Per informazioni dettagliate, vedere i seguenti argomenti:

  • Proprietà specifiche della classe FileRealm

  • Proprietà specifiche della classe CertificateRealm

  • Proprietà specifiche della classe JDBCRealm

  • Proprietà specifiche della classe LDAPRealm

  • Proprietà specifiche della classe SolarisRealm

  • Proprietà specifiche della classe PamRealm
