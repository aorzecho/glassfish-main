JVM General Settings

La macchina virtuale Java (JVM) consente l'esecuzione di bytecode Java in un computer fisico. È possibile regolare le impostazioni JVM per ottenere prestazioni migliori ed eliminare gli errori di allocazione della memoria.

La macchina virtuale Java è inclusa in Java Platform, Standard Edition (Java SE), obbligatoria per GlassFish Server. Con impostazioni JVM errate il server non funziona; è pertanto necessario prestare particolare attenzione in caso di modifica di tali impostazioni.

Nota - I termini "macchina virtuale Java" e "JVM" indicano una macchina virtuale per la piattaforma Java.

La pagina JVM General Settings contiene le seguenti opzioni.

Load Defaults

Pulsante che consente di ripristinare i valori predefiniti di tutte le impostazioni.

Configuration Name

Nome della configurazione a cui si applicano le impostazioni visualizzate nella pagina. Campo di sola lettura.

Java Home

Nome della directory di installazione del software Java SE, necessario per l'esecuzione di GlassFish Server. Per verificare che la versione del software Java SE specificata sia supportata nell'attuale release, vedere Oracle GlassFish Server 3.1 Release Notes. Se si immette il nome di una directory inesistente o il nome della directory di una versione di Java SE non supportata, GlassFish Server non viene avviato.

Javac Options

Opzioni della riga di comando per il compilatore del linguaggio di programmazione. In GlassFish Server il compilatore viene eseguito al momento della distribuzione dei componenti EJB (Enterprise JavaBeans).

Debug

Abilita il debug con JPDA (Java Platform Debugger Architecture).

JPDA è utilizzato dagli sviluppatori di applicazioni. Per ulteriori informazioni, vedere Oracle GlassFish Server 3.1 Application Development Guide.

Debug Options

Specifica le opzioni JPDA passate a JVM quando viene abilitato il debug.

RMI Compile Options

Opzioni della riga di comando per il compilatore rmic. In GlassFish Server il compilatore rmic viene eseguito al momento della distribuzione dei componenti EJB.

Bytecode Preprocessor

Nomi di classi per l'elaborazione preliminare del bytecode. Utilizzare le virgole per separare i nomi delle classi. Ogni classe deve implementare l'interfaccia com.sun.appserv.BytecodePreprocessor. Le classi vengono chiamate nell'ordine specificato.

Additional Properties

Proprietà aggiuntive per JVM. In GlassFish Server non è definita alcuna proprietà aggiuntiva per JVM.
