Applicazioni

Nella pagina Applications viene visualizzato l'elenco delle applicazioni distribuite in GlassFish Server. È possibile visualizzare e gestire le applicazioni già distribuite e distribuirne di nuove.

Per ogni applicazione vengono fornite le seguenti informazioni.

Name

Nome dell'applicazione.

Il nome può includere un identificatore di versione opzionale, che segue il nome ed è separato da questo da due punti (:). Per ulteriori informazioni sulle versioni di moduli e applicazioni, vedere Module and Application Versions in Oracle GlassFish Server 3.1 Application Deployment Guide.

Enabled or Status

Il segno di spunta indica che l'applicazione è abilitata, mentre la X indica che è disabilitata, se esiste solo l'istanza server predefinita server.

Se esistono cluster o altre istanze server standalone, selezionare Enabled on Targets per visualizzare le destinazioni in cui l'applicazione è distribuita.

Engines

Tipi di contenitori utilizzati nell'applicazione. Di seguito sono indicati i tipi di contenitori disponibili:

  • web

  • webservices

  • ejb

  • connector

  • appclient

  • weld (contenitore per Contexts e Dependency Injection per le applicazioni Java EE Platform)

Action

Crea un collegamento alle azioni che è possibile eseguire con il componente dopo la sua distribuzione: Redeploy e Reload per tutti i componenti, Launch per applicazioni Web.

La tabella Deployed Applications contiene inoltre le seguenti opzioni.

Deploy

Pulsante che consente di distribuire un'applicazione.

Undeploy

Pulsante che consente di annullare la distribuzione di una o più applicazioni selezionate.

Enable

Pulsante che consente di abilitare una o più applicazioni selezionate, se esiste solo l'istanza server predefinita server.

Disable

Pulsante che consente di disabilitare una o più applicazioni selezionate, se esiste solo l'istanza server predefinita server.

Filter

Elenco a discesa che filtra applicazioni in base al motore.
