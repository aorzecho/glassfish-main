Applicazioni

Nella pagina Applicazioni viene visualizzata una lista delle applicazioni distribuite in GlassFish Server. È possibile visualizzare e gestire le applicazioni già distribuite e distribuirne altre.

Per ogni applicazione vengono fornite le informazioni riportate di seguito.

Nome

Nome dell'applicazione.

Il nome può includere un identificativo di versione opzionale, che segue il nome ed è separato da questo da due punti (:). Per ulteriori informazioni sulle versioni di moduli e applicazioni, vedere Module and Application Versions in Oracle GlassFish Server Application Deployment Guide.

Abilitato o Stato

Il segno di spunta indica che l'applicazione è abilitata, mentre la X indica che è disabilitata, se esiste solo l'istanza server predefinita server.

Se esistono cluster o altre istanze server standalone, selezionare Abilitato su destinazioni per visualizzare le destinazioni sulle quali l'applicazione viene distribuita.

Motori

Tipi di contenitori utilizzati dall'applicazione. Di seguito sono indicati i tipi di contenitori disponibili:

  • web

  • webservices

  • ejb

  • connector

  • appclient

  • weld (contenitore per Contexts and Dependency Injection per le applicazioni della piattaforma Java EE)

Azione

Collegamenti alle azioni che è possibile eseguire sul componente dopo la distribuzione: Ridistribuisci e ricarica per tutti i componenti, Avvia per le applicazioni Web e i client applicazione e Scarica stub client per i client applicazione.

La tabella Applicazioni distribuite contiene inoltre le opzioni riportate di seguito.

Distribuisci

Pulsante che consente di distribuire un'applicazione.

Annulla distribuzione

Pulsante che consente di annullare la distribuzione di una o più applicazioni selezionate.

Abilita

Pulsante che consente di abilitare una o più applicazioni selezionate, se esiste solo l'istanza server predefinita server.

Disabilita

Pulsante che consente di disabilitare una o più applicazioni selezionate, se esiste solo l'istanza server predefinita server.

Filtro

Elenco a discesa che filtra le applicazioni in base al motore.
