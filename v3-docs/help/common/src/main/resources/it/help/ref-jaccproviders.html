JACC Providers

Nella pagina JACC Providers è possibile configurare provider JACC (Java Authorization Contract for Containers) per l'istanza o il cluster GlassFish Server selezionato.

I provider JACC possono essere utilizzati da moduli di autorizzazione di terze parti per applicazioni in esecuzione su GlassFish Server. Nell'installazione predefinita di GlassFish Server sono inclusi due provider JACC, denominati default e simple . Tali provider predefiniti implementano un motore di autorizzazione semplice, basato su file e conforme alla specifica JACC. È inoltre possibile creare provider JACC personalizzati. Vedere Creare un provider JACC.

Per informazioni su come scrivere le classi di implementazione per un provider JACC, vedere la specifica JACC all'indirizzo http://jcp.org/en/jsr/detail?id=115.

Per ogni provider JACC vengono fornite le seguenti informazioni.

Name

Nome del provider JACC. Questo nome viene utilizzato per fare riferimento all'elemento jacc-provider nel file domain.xml.

Policy Configuration

Nome di classe completo per javax.security.jacc.PolicyConfigurationFactory.provider che implementa la classe javax.security.jacc.PolicyConfigurationFactory specifica del provider.

Policy Provider

Nome di classe completo per javax.security.jacc.policy.provider che implementa java.security.Policy.

I provider JACC predefiniti supportano inoltre la seguente proprietà.

repository

Directory contenente il file dei criteri JACC. Per il provider JACC default di GlassFish Server la directory predefinita è ${com.sun.aas.instanceRoot}/generated/policy . Per il provider JACC simple di GlassFish Server questa proprietà non è definita per impostazione predefinita.

La tabella JACC Providers contiene le seguenti opzioni.

New

Pulsante che consente di creare un nuovo provider JACC.

Delete

Pulsante che consente di eliminare uno o più provider JACC selezionati.
