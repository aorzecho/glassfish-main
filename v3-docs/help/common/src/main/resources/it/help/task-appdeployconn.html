Distribuire un modulo connettore

  1. Nella struttura ad albero, selezionare il nodo Applications.

    Viene visualizzata la pagina Applications.

  2. Nella tabella Deployed Applications, fare clic sul pulsante Deploy.

    Viene visualizzata la pagina Deploy Applications or Modules.

  3. Specificare la posizione dell'applicazione da distribuire.
    • Se l'applicazione è un file di archivio che si trova nel computer client o è accessibile da tale computer, selezionare l'opzione Packaged File to Be Uploaded to the Server.

      Il computer client è l'host in cui viene visualizzata Administration Console tramite un browser.

      Fare clic su Browse per individuare il file oppure digitarne il percorso completo.

    • Se l'applicazione è un file di archivio che si trova nel computer server o è un'applicazione decompressa da una directory esplosa, selezionare l'opzione Local Packaged File or Directory That Is Accessible From the GlassFish Server.

      Il computer server è l'host in cui è in esecuzione il server di amministrazione del dominio (DAS, Domain Administration Server).

      Fare clic su Browse Files per cercare un file o su Browse Folders per cercare una directory. In alternativa, digitare il percorso completo del file o della directory. La distribuzione da una directory esplosa deve essere eseguita da sviluppatori esperti e non è consigliata per gli ambienti di produzione.

  4. Dall'elenco a discesa Type, selezionare Connector Module.

    Se è stata selezionata l'opzione Packaged File to Be Uploaded to the Server o Local Packaged File That Is Accessible From the GlassFish Server, il tipo di applicazione viene rilevato automaticamente. Se è stata selezionata l'opzione Directory That Is Accessible From the GlassFish Server, è necessario specificare il tipo manualmente.

  5. Nel campo Application Name, lasciare il nome predefinito, che è il nome della directory o il prefisso del nome file, oppure digitare un altro nome.

    Se si è scelto di caricare un file viene visualizzato il nome predefinito. Il nome dell'applicazione deve essere univoco.

    Il nome può includere un identificatore di versione opzionale, che segue il nome ed è separato da questo da due punti (:). L'identificatore di versione deve iniziare con una lettera o un numero. Può contenere caratteri alfanumerici più trattini di sottolineatura (_), trattini (-) e punti (.). Per ulteriori informazioni sulle versioni di moduli e applicazioni, vedere Module and Application Versions in Oracle GlassFish Server 3.1 Application Deployment Guide.

  6. Selezionare la casella di controllo Status Enabled per abilitare l'applicazione.

    Questa opzione è selezionata per impostazione predefinita. Se l'opzione è deselezionata, l'applicazione non è disponibile.

  7. Selezionare la casella di controllo Run Verifier per verificare i descrittori di distribuzione prima della distribuzione.

    Se l'opzione è selezionata ed è impossibile effettuare la verifica, la distribuzione non viene eseguita. Il verificatore esamina la struttura e il contenuto dei descrittori di distribuzione. La verifica di applicazioni di grandi dimensioni richiede spesso tempi lunghi. Questa opzione è disabilitata per impostazione predefinita.

    I pacchetti di verifica devono essere installati tramite l'utility Update Tool; in caso contrario verrà registrato un avviso e l'opzione verrà ignorata.

  8. Selezionare la casella di controllo Force Redeploy per forzare la ridistribuzione se l'applicazione è già stata distribuita.

    Se l'opzione non è selezionata, un tentativo di distribuire un'applicazione già distribuita causa un errore. Questa opzione è disabilitata per impostazione predefinita.

  9. Selezionare la casella di controllo Preserve Application Scoped Resources per preservare le risorse con ambito a livello di applicazione e ripristinarle durante la ridistribuzione.

    Questa opzione è disabilitata per impostazione predefinita.

  10. Nel campo Description, digitare una descrizione per l'applicazione.
  11. Per aggiungere istanze server e cluster di destinazione in cui distribuire il modulo o l'applicazione, selezionarli dall'elenco Available Targets e utilizzare il pulsante Add per spostarli nell'elenco Selected Targets.

    L'opzione Target viene visualizzata se esistono cluster o istanze server standalone oltre all'istanza server predefinita (server). Per impostazione predefinita, la distribuzione del modulo o dell'applicazione viene eseguita nell'istanza server predefinita, denominata server. Utilizzare il pulsante Add All per eseguire la distribuzione in tutte le destinazioni. Utilizzare i pulsanti Remove e Remove All per rimuovere destinazioni dall'elenco Selected Targets.

  12. Fare clic su OK.
