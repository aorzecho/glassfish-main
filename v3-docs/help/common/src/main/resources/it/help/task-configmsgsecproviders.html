Configurare le funzioni di GlassFish Server per l'utilizzo dai parte dei provider di sicurezza dei messaggi

  1. Se si utilizza un token nome utente, configurare un database utente (se necessario).

    Se si utilizza un token nome utente/password, in primo luogo configurare un realm e un database utente appropriato per il realm.

    La configurazione di un database utente è illustrata in Modificare un realm.

  2. Se necessario, gestire i certificati e le chiavi private.

Passi successivi

Una volta configurate le risorse di GlassFish Server per l'utilizzo da parte dei provider di sicurezza dei messaggi, è possibile abilitare i provider installati con GlassFish Server, come descritto in Modificare una configurazione di sicurezza dei messaggi.
