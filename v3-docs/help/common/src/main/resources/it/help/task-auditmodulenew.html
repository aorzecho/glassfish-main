Creare un modulo di audit

Mediante la revisione, in GlassFish Server vengono registrati eventi significativi, quali errori o violazioni della sicurezza. GlassFish Server registra tutti gli eventi di autenticazione nei file di log. Un log di accesso completo fornisce una traccia sequenziale degli eventi di accesso di GlassFish Server.

In GlassFish Server è disponibile un modulo di audit predefinito; per ulteriori informazioni, vedere Utilizzare il modulo di audit predefinito.

  1. Nell'albero di navigazione espandere il nodo Configurazione.

  2. Select the instance to configure:

    • To configure a particular instance, expand the instance's config node. For example, the default instance, server, expand the server-config node.

    • To configure the default settings for all instances, expand the default-config node.

  3. Nel nodo Configurazione, espandere il nodo Sicurezza.

  4. Nel nodo Sicurezza, selezionare il nodo Moduli di audit.

    Viene visualizzata la pagina Moduli di audit.

  5. Nella pagina Moduli di audit, fare clic su Nuovo.

    Viene visualizzata la pagina Nuovo modulo di audit.

  6. Nel campo Nome della pagina Nuovo modulo di audit, digitare il nome del modulo di revisione.

  7. Nel campo Nome classe, digitare il nome completo della classe che implementa il modulo.

    Il nome della classe per il modulo di revisione predefinito è com.sun.enterprise.security.Audit.

  8. Nella sezione Proprietà aggiuntive, specificare le proprietà aggiuntive.

    Per aggiungere una proprietà, fare clic sul pulsante Aggiungi proprietà. Nella riga vuota visualizzata, digitare il nome della proprietà nel campo Nome e il valore della proprietà nel campo Valore.

    Per una descrizione delle proprietà aggiuntive disponibili per i moduli di audit, vedere Proprietà specifiche dei moduli di audit.

  9. Fare clic su OK.
