Creare un modulo di revisione

Mediante la revisione, in GlassFish Server vengono registrati eventi significativi, quali errori o violazioni della sicurezza. GlassFish Server registra tutti gli eventi di autenticazione nei file di log. Un log di accesso completo fornisce una traccia sequenziale degli eventi di accesso di GlassFish Server.

In GlassFish Server è disponibile un modulo di revisione predefinito; per ulteriori informazioni, vedere Utilizzare il modulo di revisione attivo.

  1. Nella struttura ad albero, espandere il nodo Configuration.
  2. Nel nodo Configuration, espandere il nodo Security.
  3. Nel nodo Security, selezionare il nodo Audit Modules.

    Viene visualizzata la pagina Audit Modules.

  4. Nella pagina Audit Modules, fare clic su New.

    Viene visualizzata la pagina New Audit Module.

  5. Nel campo Name della pagina New Audit Module, digitare il nome del modulo di revisione.
  6. Nel campo Class Name, digitare il nome completo della classe che implementa il modulo.

    Il nome della classe per il modulo di revisione predefinito è com.sun.enterprise.security.Audit.

  7. (Opzionale) Nella sezione Additional Properties, specificare le proprietà aggiuntive.

    Per aggiungere una proprietà, fare clic sul pulsante Add Property. Nella riga vuota visualizzata, digitare il nome della proprietà nel campo Name e il valore della proprietà nel campo Value.

    Per la descrizione delle proprietà aggiuntive disponibili per i moduli di revisione, vedere Proprietà specifiche dei moduli di revisione.

  8. Fare clic su OK.
