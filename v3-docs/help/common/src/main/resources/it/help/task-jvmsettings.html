Configurare le impostazioni generali di JVM

La macchina virtuale Java (JVM) consente l'esecuzione di bytecode Java in un computer fisico. È possibile regolare le impostazioni JVM per ottenere prestazioni migliori ed eliminare gli errori di allocazione della memoria.

La macchina virtuale Java è inclusa in Java Platform, Standard Edition (Java SE), obbligatoria per GlassFish Server. Con impostazioni JVM errate il server non funziona; è pertanto necessario prestare particolare attenzione in caso di modifica di tali impostazioni.

Nota - I termini "macchina virtuale Java" e "JVM" indicano una macchina virtuale per la piattaforma Java.

  1. Nella struttura ad albero, selezionare il nodo Configuration.
  2. Nel nodo Configuration, selezionare il nodo JVM Settings.

    Viene visualizzata la pagina JVM General Settings.

  3. Nel campo Java Home della pagina JVM General Settings, digitare il nome della directory di installazione del software Java SE.

    GlassFish Server si basa sulla piattaforma Java SE. Per verificare che la versione del software Java SE specificata sia supportata in questa release, vedere Oracle GlassFish Server 3.1-3.1.1 Release Notes . Se si immette il nome di una directory inesistente o della directory di una versione di Java SE non supportata, GlassFish Server non viene avviato.

  4. Nel campo Java Options digitare le opzioni della riga di comando per il compilatore del linguaggio di programmazione Java.

    In GlassFish Server il compilatore viene eseguito al momento della distribuzione dei componenti EJB (Enterprise JavaBeans).

  5. Selezionare la casella di controllo Debug Enabled per abilitare il debug con JPDA (Java Platform Debugger Architecture).

    JPDA è utilizzato dagli sviluppatori di applicazioni. Per ulteriori informazioni, vedere Oracle GlassFish Server 3.1 Application Development Guide.

  6. Nel campo Debug Options digitare le opzioni JPDA da passare a JVM.

    Questo campo è valido soltanto se nel passaggio precedente è stato abilitato il debug.

  7. Nel campo RMI Compile Options digitare le opzioni della riga di comando per il compilatore rmic.

    In GlassFish Server il compilatore rmic viene eseguito al momento della distribuzione dei componenti EJB.

  8. Nel campo Bytecode Preprocessor digitare un elenco di nomi classe.

    Utilizzare le virgole per separare i nomi delle classi. Ogni classe deve implementare l'interfaccia com.sun.appserv.BytecodePreprocessor. Le classi vengono chiamate nell'ordine specificato.

  9. (Opzionale) Nella sezione Additional Properties, specificare le proprietà aggiuntive.

    Per aggiungere una proprietà, fare clic sul pulsante Add Property. Nella riga vuota visualizzata, digitare il nome della proprietà nel campo Name e il valore della proprietà nel campo Value.

    In GlassFish Server non è definita alcuna proprietà aggiuntiva per JVM.

  10. Fare clic su Save.

    Suggerimento - Per tornare alle impostazioni predefinite, fare clic sul pulsante Load Defaults, quindi su Save.

  11. Riavviare il server.
