Configurare le impostazioni generali JVM

La Java Virtual Machine (JVM) consente l'esecuzione di bytecode Java byte in un computer fisico. È possibile regolare le impostazioni JVM per ottenere prestazioni migliori ed evitare errori di allocazione della memoria.

La Java Virtual Machine è inclusa in Java Platform, Standard Edition (Java SE), obbligatoria per GlassFish Server. Con impostazioni JVM errate il server non funziona; è pertanto necessario prestare particolare attenzione in caso di modifica di tali impostazioni.

Nota:

I termini "Java Virtual Machine" e "JVM" indicano una virtual machine per la piattaforma Java.

  1. In the navigation tree, expand the Configuration node.

  2. Select the configuration that you are changing.

  3. Select the JVM Settings node.

    The JVM General Settings tab page opens.

  4. Nell'albero di navigazione selezionare il nodo Configurazione.

  5. Nel nodo Configurazione, selezionare il nodo Impostazioni JVM.

    Viene visualizzata la pagina Impostazioni generali JVM.

  6. Nel campo Java Home della pagina Impostazioni generali JVM, digitare il nome della directory di installazione del software Java SE.

    GlassFish Server si basa sulla piattaforma Java SE. Per verificare che la versione del software Java SE specificata sia supportata in questa release, vedere Oracle GlassFish Server Release Notes;. Se si immette il nome di una directory inesistente o della directory di una versione di Java SE non supportata, GlassFish Server non viene avviato.

  7. Nel campo Opzioni Java digitare le opzioni della riga di comando per il compilatore del linguaggio di programmazione Java.

    In GlassFish Server il compilatore viene eseguito al momento della distribuzione dei componenti EJB (Enterprise JavaBeans).

  8. Selezionare la casella di controllo Debug abilitato per abilitare il debug con JPDA (Java Platform Debugger Architecture).

    JPDA è utilizzato dagli sviluppatori di applicazioni. Per ulteriori informazioni, vedere Oracle GlassFish Server Application Development Guide.

  9. Nel campo Opzioni di debug digitare le opzioni JPDA da passare a JVM.

    Questo campo è valido soltanto se nel passo precedente è stato abilitato il debug.

  10. Nel campo Opzioni di compilazione RMI digitare le opzioni della riga di comando per il compilatore rmic.

    In GlassFish Server il compilatore rmic viene eseguito al momento della distribuzione dei componenti EJB.

  11. Nel campo Preprocessore bytecode digitare una lista di nomi classe.

    Utilizzare le virgole per separare i nomi delle classi. Ogni classe deve implementare l'interfaccia com.sun.appserv.BytecodePreprocessor. Le classi vengono chiamate nell'ordine specificato.

  12. Nella sezione Proprietà aggiuntive, specificare le proprietà aggiuntive.

    Per aggiungere una proprietà, fare clic sul pulsante Aggiungi proprietà. Nella riga vuota visualizzata, digitare il nome della proprietà nel campo Nome e il valore della proprietà nel campo Valore.

    In GlassFish Server non è definita alcuna proprietà aggiuntiva per JVM.

  13. Fare clic su Salva.

    Suggerimento:

    Per tornare alle impostazioni predefinite, fare clic sul pulsante Carica impostazioni predefinite, quindi su Salva.

  14. Riavviare il server.
