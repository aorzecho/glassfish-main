Filtrare i record dei log visualizzati

Prima di iniziare

Verificare che i file di log visualizzati si riferiscano al server di amministrazione del dominio (DAS, Domain Administration Server), a un'istanza di GlassFish Server in cluster o a un'istanza standalone. Per ulteriori informazioni, vedere i seguenti argomenti:

  • Visualizzare file di log per DAS

  • To View Log Files for a Clustered GlassFish Server Instance

  • To View Log Files for a Standalone GlassFish Server Instance

  1. Nell'area Criteri di ricerca, filtrare i record in base a contenuto, periodo di tempo e livello di log.

    1. Per visualizzare solo i record contenenti un testo specifico, digitare il testo nel campo Ricerca di testo.

      La ricerca distingue tra maiuscole e minuscole.

    2. Per selezionare il periodo di tempo relativo ai record di log da visualizzare, utilizzare l'opzione Indicatore orario.

      • Per visualizzare i messaggi più recenti, selezionare Più recente (impostazione predefinita).

      • Per visualizzare i messaggi relativi a un determinato periodo di tempo, selezionare Intervallo specifico e digitare valori di data e ora nei campi Da e A visualizzati.

        Il valore di data deve essere espresso nel seguente formato:

        MM/DD/YY

        MM è un numero intero compreso nell'intervallo da 1 a 12 corrispondente al mese, ad esempio 1 per gennaio o 10 per ottobre. DD è un numero intero compreso nell'intervallo da 1 a 31 corrispondente al giorno del mese. YY sono le ultime due cifre dell'anno, ad esempio 11 per l'anno 2011.

        Ad esempio, per indicare la data 1 febbraio 2011, digitare 2/1/11 nel campo Data.

        Il valore Ora deve essere espresso nel seguente formato:

        hh:mm:ss.SSS

        hh è un numero intero compreso nell'intervallo da 0 a 23 indicante l'ora. Per le ore del pomeriggio, utilizzare numeri interi compresi nell'intervallo da 12 a 23, ad esempio 14 per indicare le 2 del pomeriggio. mm è un numero intero compreso nell'intervallo da 0 a 59 corrispondente al numero di minuti dopo l'ora. ss è un numero intero compreso nell'intervallo da 0 a 59 corrispondente al numero di secondi. SSS è un numero intero a tre cifre, se necessario con zeri iniziali, compreso nell'intervallo da 000 a 999 corrispondente al numero di millisecondi.

        Ad esempio, per indicare le 5:10 del pomeriggio, digitare 17:10:00.000 nel campo Ora.

        Se il valore incluso in Da è successivo al valore incluso in A, viene visualizzato un messaggio di errore.

    3. Filtrare i messaggi in base al livello di log.

      Per assicurarsi che i messaggi da visualizzare siano presenti nel log del server, innanzitutto impostare i livelli di log appropriati nella pagina Livelli di log dei moduli. Per istruzioni, vedere Configurare i livelli di log.

      Se si sceglie di filtrare messaggi di log in base al livello di log, vengono visualizzati solo i messaggi che soddisfano i criteri di filtro specificati. Il filtraggio non ha tuttavia effetto sui messaggi che sono registrati nel log del server.

      1. Dall'elenco a discesa Livello di log, selezionare il livello di log.

      2. Per visualizzare i messaggi corrispondenti solo al livello selezionato, selezionare l'opzione Non includere messaggi di livello SEVERE maggiore.

        Per impostazione predefinita, vengono visualizzati tutti i messaggi del log del server corrispondenti al livello di log scelto e ai livelli di maggior gravità.

  2. Nell'area Ricerca avanzata, filtrare i record in base al namespace del programma di log e selezionare la quantità di informazioni da visualizzare.

    1. Fare clic sul collegamento Ricerca avanzata.

      Alla finestra Visualizzatore log viene aggiunta l'area Ricerca avanzata.

    2. Per il filtraggio in base al modulo, nell'elenco a discesa Logger, selezionare uno o più namespace.

      Per selezionare più namespace, fare clic su di essi tenendo premuto il tasto Maiusc o Ctrl.

      Se si seleziona uno spazio di nomi namespace a un livello elevato vengono selezionati tutti gli spazi di nomi di livello sottostante. Ad esempio, se si seleziona javax.enterprise.system, vengono selezionati anche i programmi di log per tutti i moduli in tale spazio di nomi, come javax.enterprise.system.core, javax.enterprise.system.tools.admin e così via.

    3. Per visualizzare i messaggi dei programmi di log per un'applicazione particolare, nel campo Logger personalizzato digitare il nome di ogni programma di log su una riga separata.

      Se l'applicazione è costituita da diversi moduli, è possibile visualizzarne alcuni o tutti. Ad esempio, un'applicazione potrebbe disporre dei seguenti programmi di log:

      com.example.myapp.module1
com.example.myapp.module2
com.example.myapp.module3

      Per visualizzare i messaggi di tutti i moduli dell'applicazione, digitare com.mycompany.myapp . Per visualizzare solo i messaggi di module2, digitare com.example.myapp.module2.

      Quando si specifica uno o più programmi di log personalizzati, i messaggi dei moduli GlassFish Server vengono visualizzati solo se vengono specificati esplicitamente nell'area Logger.

    4. Nell'elenco a discesa Visualizza, selezionare il numero di messaggi da visualizzare.

      Il numero predefinito di messaggi visualizzati è 40. Altre opzioni sono 100, 250 e 1000.

    5. Per visualizzare gli stack trace, deselezionare l'opzione Limita messaggi troppo lunghi.

      Per impostazione predefinita, gli stack trace non vengono visualizzate nel visualizzatore. Per visualizzarle, fare clic sul collegamento (details) per un messaggio.

    6. Per nascondere l'area Ricerca avanzata, fare clic sul collegamento Ricerca di base.

  3. Per specificare l'istanza e il file di log associato da visualizzare, utilizzare l'area Modifica ricerca.

    • Nell'elenco a discesa Istanza, selezionare l'istanza.

      L'impostazione predefinita è l'istanza per la quale è stata inizialmente selezionata la visualizzazione dei file di log. Se è stata selezionata la visualizzazione dei file di log per il server di amministrazione del dominio (DAS, Domain Administration Server), l'impostazione predefinita è server, che denota il DAS.

    • Nell'elenco a discesa File di log, selezionare il file di log.

      Il valore predefinito è server.log. Altre opzioni dipendono dalle impostazioni di log per la destinazione selezionata.

  4. Nell'area Risultati ricerca, personalizzare la visualizzazione dei messaggi del log.

    L'area Risultati ricerca visualizza le 40 voci più recenti del log del server, con le impostazioni specificate nelle pagine Impostazioni log e Livelli di log.

    • Per ordinare i messaggi in modo diverso, fare clic sulle frecce presenti nelle colonne.

    • Per visualizzare la versione formattata di un messaggio, fare clic sul collegamento (details) per il messaggio.

      Viene visualizzata una finestra Dettagli voce log contenente la versione formattata del messaggio.

    • Per visualizzare voci precedenti o successive del file di log, fare clic su Record prima di e Record dopo nella parte superiore della tabella.
