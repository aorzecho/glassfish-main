Archivia utenti

Nella pagina Archivia utenti è possibile gestire gli utenti del realm file. Gli utenti e i gruppi nel realm file vengono elencati nel file di chiave, la cui posizione è specificata dalla proprietà file.

Un utente nel realm file può appartenere a un gruppo Java, una categoria di utenti con caratteristiche comuni. Ad esempio, gli utenti di un'applicazione di e-commerce potrebbero far parte del gruppo CUSTOMER, ma quelli che spendono di più verrebbero inclusi nel gruppo PREFERRED. La suddivisione degli utenti in categorie di gruppi semplifica il controllo dell'accesso di numerosi utenti.

Inizialmente, dopo l'installazione di GlassFish Server, l'unico utente è l'amministratore immesso durante l'installazione. Per impostazione predefinita, tale utente appartiene al gruppo asadmin, nel realm admin-realm, che dispone dei diritti per la modifica di GlassFish Server. Qualsiasi utente assegnato a questo gruppo disporrà dei privilegi di amministratore, ovvero avrà accesso allo strumento asadmin e alla Console di amministrazione.

Nella pagina Archivia utenti viene visualizzata una lista di utenti per il realm selezionato. Per ogni utente vengono fornite le informazioni riportate di seguito.

ID utente

Nome utente.

Lista gruppi

Gruppi di appartenenza dell'utente.

La tabella Archivia utenti contiene le opzioni riportate di seguito.

Nuovo

Pulsante che consente di creare un nuovo utente.

Elimina

Pulsante che consente di eliminare uno o più utenti selezionati.
