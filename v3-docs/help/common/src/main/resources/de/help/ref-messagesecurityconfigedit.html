Meldungssicherheitskonfiguration bearbeiten

Verwenden Sie die Seite "Meldungssicherheitskonfiguration bearbeiten", um Anbieter für Meldungssicherheit zu aktivieren.

Zum Aktivieren der Meldungssicherheit für in GlassFish-Server bereitgestellte Webdienstendpunkte muss ein Anbieter als Standardanbieter für die Serverseite angegeben werden. Wenn Sie einen Standardanbieter für die Meldungssicherheit aktivieren, müssen ebenfalls Anbieter für die Clients der in GlassFish-Server bereitgestellten Webdienste aktiviert werden. Informationen zum Aktivieren der Anbieter, die von Clients verwendet werden, werden in So aktivieren Sie die Meldungssicherheit für Anwendungs-Clients diskutiert.

Zum Aktivieren der Meldungssicherheit für Webdienstaufrufe von bereitgestellten Endpunkten muss ein standardmäßiger Clientanbieter angegeben werden. Wenn ein standardmäßiger Clientanbieter für GlassFish-Server aktiviert wurde, muss sichergestellt werden, dass die von in GlassFish-Server bereitgestellten Endpunkten aufgerufenen Dienste für Sicherheit auf Meldungsebene konfiguriert sind.

Die Seite "Neue Meldungssicherheitskonfiguration" enthält die folgenden Optionen.

Konfigurationsname

Der Name der Konfiguration, für die die Einstellungen auf dieser Seite gelten. Dieses Feld ist schreibgeschützt.

Authentifizierungs-Layer

Die Nachrichtenlayer, in der Authentifizierung durchgeführt wird. Dies ist ein schreibgeschütztes Feld. Der Wert ist entweder SOAP (für die Standard-Meldungssicherheitskonfiguration) oder HttpServlet.

Standardanbieter

Der standardmäßige Server-seitige Anbieter, falls vorhanden. Der standardmäßige Serveranbieter, auch Standardanbieter genannt, gibt den Serveranbieter an, der für Anwendungen aufgerufen wird, für die kein spezifischer Serveranbieter festgelegt ist. Zur Auswahl stehen Serveranbieter, XWS_ServerProvider oder keine. Standardmäßig wird kein Anbieter ausgewählt, so dass GlassFish-Server Server-seitig keinen Meldungssicherheitsanbieter aufruft. Wählen Sie im Allgemeinen ServerProvider als Standardanbieter aus.

Standard-Clientanbieter

Der Standard-Clientanbieter, falls vorhanden. Der standardmäßige Clientanbieter gibt den Clientanbieter an, der für Anwendungen aufgerufen wird, für die kein spezifischer Clientanbieter festgelegt ist. Zur Auswahl stehen ClientProvider, XWS_ClientProvider oder keine. Standardmäßig ist kein Anbieter ausgewählt, so dass GlassFish-Server Client-seitig keinen Meldungssicherheitsanbieter aufruft. Wählen Sie im Allgemeinen für den Standard-Clientanbieter "keine" aus. Wählen Sie ClientProvider, um einen Standardanbieter und eine Meldungssicherheitsrichtilinie zu aktivieren, die für Webdienstaufrufe von in GlassFish-Server bereitgestellten Webdienstendpunkten angewendet werden.

Verwandte asadmin-Befehle

Verwenden Sie den Befehl set(1), um eine Meldungssicherheitskonfiguration zu ändern.

  • Zum Festlegen des standardmäßigen Serveranbieters:

    asadmin set --user admin-user --port admin-port 
server-config.security-service.message-security-config.SOAP.
default_provider=ServerProvider

  • Zum Festlegen des standardmäßigen Clientanbieters:

    asadmin set --user admin-user --port admin-port 
server-config.security-service.message-security-config.SOAP.
default_client_provider=ClientProvider
