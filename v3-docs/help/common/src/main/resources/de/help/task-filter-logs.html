So filtern Sie angezeigte Protokolldatensätze

Bevor Sie beginnen

Stellen Sie sicher, dass Sie die Protokolldateien für den Domain Administration Server (DAS), eine GlassFish-Server-Clusterinstanz oder eine eigenständige Instanz anzeigen. Weitere Informationen finden Sie in den folgenden Abschnitten:

  • So zeigen Sie Protokolldateien für den DAS an

  • So zeigen Sie Protokolldateien für eine GlassFish-Server-Cluster-Instanz an

  • So zeigen Sie Protokolldateien für eine eigenständige GlassFish-Server-Instanz an

  1. Filtern Sie im Bereich "Suchkriterien" die Datensätze nach Inhalt, Zeitraum und Protokollebene.
    1. Um nur Datensätze anzuzeigen, die einen bestimmten Text enthalten, geben Sie im Feld "Textsuche" den betreffenden Text ein.

      Bei der Suche wird zwischen Groß- und Kleinschreibung unterschieden.

    2. Um auszuwählen, wie lange Protokolldatensätze angezeigt werden, verwenden Sie die Option "Zeitstempel".
      • Um die aktuellsten Meldungen anzuzeigen, wählen Sie "Aktuell" (Standardeinstellung).
      • Um Meldungen aus einem bestimmten Zeitraum anzuzeigen, wählen Sie "Bestimmter Zeitraum", und geben Sie einen Datums- und Uhrzeitwert in die Felder "Von" und "An" ein.

        Der Wert "Datum" muss folgendes Format aufweisen:

        MM/DD/YY
        MM

        Eine Ganzzahl im Bereich von 1 bis 12 für den Monat, beispielsweise 1 für Januar oder 10 für Oktober.

        DD

        Eine Ganzzahl im Bereich von 1 bis 31 für den Tag des Monats.

        YY

        Die letzten beiden Zahlen des Jahres, beispielsweise 11 für das Jahr 2011.

        Um beispielsweise das Datum 1. Februar 2011 anzugeben, geben Sie im Feld "Datum" 2/1/11 ein.

        Der Wert "Zeit" muss folgendes Format aufweisen:

        hh:mm:ss.SSS
        hh

        Eine Ganzzahl im Bereich von 0 bis 23 stellt die Stunden dar. Für Zeiten am Nachmittag verwenden Sie Ganzzahlen im Bereich von 12 bis 23, beispielsweise 14 für 2 Uhr nachmittags.

        mm

        Eine Ganzzahl im Bereich von 0 bis 59 stellt die Minuten dar.

        ss

        Eine Ganzzahl im Bereich von 0 bis 59 stellt die Sekunden dar.

        SSS

        Eine dreistellige Ganzzahl (wenn nötig mit führenden Nullen) im Bereich von 000 bis 999 stellt die Millisekunden dar.

        Um beispielsweise die Zeit 5:10 Uhr vormittags anzugeben, geben Sie im Feld "Zeit" 17:10:00.000 ein.

        Wenn der Wert im Feld "Von" zeitlich hinter dem Wert in "Bis" liegt, wird eine Fehlermeldung angezeigt.

    3. Filtermeldungen nach Protokollebene.

      Um sicherzustellen, dass die Nachrichten, die Sie anzeigen möchten, im Serverprotokoll angezeigt werden, definieren Sie zunächst auf der Seite "Modul-Protokollebenen" die entsprechenden Protokollebenen.

      Wenn Sie sich für eine Filterung von Protokollmeldungen basierend auf der Protokollebene entschieden haben, werden nur Meldungen angezeigt, die den angegebenen Filterkriterien entsprechen. Diese Filterung wirkt sich jedoch nicht darauf aus, welche Meldungen im Serverprotokoll aufgezeichnet werden.

      1. Wählen Sie aus der Dropdown-Liste "Protokollebene" die Protokollebene aus.
      2. Um Meldungen ausschließlich auf der gewählten Ebene anzuzeigen, aktivieren Sie die Option "Keine weiteren als schwerwiegend eingestuften Meldungen aufnehmen".

        Standardmäßig werden alle im Serverprotokoll enthaltenen Meldungen angezeigt, die der gewählten Protokollebene oder einer Ebene mit höherem Schweregrad entsprechen.

  2. Im Bereich "Erweiterte Suche" filtern Sie Datensätze entsprechend dem Namespace der Protokollfunktion und wählen den Informationsumfang aus, der angezeigt wird.
    1. Klicken Sie auf den Link "Erweiterte Suche".

      Der Bereich "Erweiterte Suche" wird in das Fenster "Protokoll-Viewer" eingefügt.

    2. Um nach Modul zu filtern, wählen Sie aus der Dropdown-Liste "Protokollfunktion" einen oder mehrere Namespaces aus.

      Klicken Sie bei gedrückter Umschalt- oder Strg-Taste, um mehrere Namespaces auszuwählen.

      Die Auswahl eines Namespaces einer höheren Ebene führt zur Auswahl aller darunter angeordneten Namespaces. Wenn Sie beispielsweise javax.enterprise.system auswählen, werden auch die Protokollfunktionen aller Module unterhalb dieses Namespace ausgewählt, beispielsweise javax.enterprise.system.core, javax.enterprise.system.tools.admin usw.

    3. Um Meldungen von Protokollfunktionen für eine bestimmte Anwendung anzuzeigen, geben Sie im Feld "Benutzerdefinierte Protokollfunktion" den Namen jeder Protokollfunktion auf jeweils einer separaten Zeile ein.

      Wenn die Anwendung über mehrere Module verfügt, können Sie einige oder alle dieser Module anzeigen. Beispielsweise kann eine Anwendung über folgende Protokollfunktionen verfügen:

      • com.example.myapp.module1

      • com.example.myapp.module2

      • com.example.myapp.module3

      Um Meldungen aller Module in der Anwendung anzuzeigen, geben Sie com.mycompany.myapp ein. Um nur Meldungen von module2 anzuzeigen, geben Sie com.example.myapp.module2 ein.

      Wenn Sie eine oder mehrere benutzerdefinierte Protokollfunktionen angeben, werden Meldungen von GlassFish-Server-Modulen nur dann angezeigt, wenn Sie sie im Bereich "Protokollfunktion" explizit aufführen.

    4. Wählen Sie aus der Dropdown-Liste "Anzeigen" die Meldungen aus, die angezeigt werden sollen.

      Als Standardanzeige für Nachrichten wird 40 angezeigt. Darüber hinaus stehen 100, 250 und 1000 zur Auswahl.

    5. Um Daten zur Stapelablaufverfolgung anzuzeigen, deaktivieren Sie die Option "Sehr lange Meldungen begrenzen".

      Standardmäßig werden in der Anzeige keine Daten zur Stapelablaufverfolgung angezeigt. Um Daten zur Stapelablaufverfolgung anzuzeigen, klicken Sie auf den Link (Details) für eine Meldung.

    6. Um den Bereich "Erweiterte Suche" auszublenden, klicken Sie auf den Link "Einfache Suche".
  3. Um die Instanz und die zugehörige Protokolldatei anzugeben, die Sie anzeigen möchten, verwenden Sie den Bereich "Suche modifizieren".
    • Wählen Sie aus der Dropdown-Liste "Instanz" die Instanz aus.

      Der Standardwert ist die Instanz, für die Sie anfangs festgelegt hatten, dass Protokolldateien angezeigt werden sollen. Wenn Sie festgelegt hatten, dass Protokolldateien für den Domain Administration Server (DAS) angezeigt werden sollen, lautet der Standardwert, der auf den DAS verweist, server.

    • Wählen Sie aus der Dropdown-Liste "Protokolldatei" die Protokolldatei aus.

      Der Standardname lautet server.log. Andere Optionen hängen von den Protokolleinstellungen für das gewählte Ziel ab.

  4. Passen Sie im Bereich "Suchergebnisse" die Anzeige von Protokollmeldungen an.

    Im Bereich "Suchergebnisse" werden entsprechend den Einstellungen auf den Seiten "Protokollierungseinstellungen" und "Protokollebenen" die letzten 40 Einträge des Serverprotokolls angezeigt.

    • Um die Meldungen unterschiedlich zu sortieren, klicken Sie auf die Pfeile in den Spalten.
    • Um eine formatierte Version einer Meldung anzuzeigen, klicken Sie auf den Link (Details) für eine Meldung.

      Es wird ein Fenster "Protokolleintragsdetail" angezeigt, das eine formatierte Version der Meldung enthält.

    • Um vorhergehende oder nachfolgende Einträge in der Protokolldatei anzuzeigen, klicken Sie auf die Schaltflächen "Einträge vor" bzw. "Einträge nach", die sich am oberen Rand der Tabelle befinden.
