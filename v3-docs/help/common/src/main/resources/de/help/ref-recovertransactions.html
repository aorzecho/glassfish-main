Transaktionen wiederherstellen

Auf der Seite "Wiederherstellungstransaktionen" können Sie Transaktionen für eine GlassFish Server-Instanz wiederherstellen.

Wenn die Instanz ausgeführt wird, enthält die Seite "Recovery von Transaktionen" nur den Text Recovery is performed by the running server itself (Das Recovery wird vom Server selbst durchgeführt) und die Schaltfläche "OK".

Wenn die Instanz nicht ausgeführt wird, enthält die Seite "Recovery von Transaktionen" die folgenden Optionen.

Instanzname

Der Name, der der Instanz bei der Erstellung zugewiesen wurde. Dieses Feld ist schreibgeschützt.

Zielserver

Eine weitere Instanz, die die Transaktionen in der aktuellen Instanz wiederherstellen kann.

Transaktionslogverzeichnis

Das Verzeichnis, in dem die Transaktionslogs gespeichert werden, wenn die Logs in einem Shared File-System gespeichert werden. Diese Einstellung gilt nicht für Transaktionslogs, die in einer Datenbank gespeichert sind. Standardmäßig handelt es sich um /logs/tx unter dem Domainverzeichnis.

Transaktionslogs werden im Unterverzeichnis tx des Verzeichnisses gespeichert, das durch die Einstellung "Verzeichnis des Transaktionslogs" des Transaktionsservice für die Clusterkonfiguration angegeben ist. Der Standardwert für "Verzeichnis des Transaktionsprotokolls" ist das Verzeichnis, das auf der Seite "Domainattribute" im Feld "Log-Root" angegeben ist, das sich im Knoten GlassFish Server in der Registerkarte "Erweitert" befindet.

Entsprechende asadmin-Unterbefehle
