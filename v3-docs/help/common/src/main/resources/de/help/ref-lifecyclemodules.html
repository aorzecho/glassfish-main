Lebenszyklusmodule

Auf der Seite "Lebenszyklusmodule" können Sie Lebenszyklusmodule konfigurieren.

Ein Lebenszyklusmodul führt Aufgaben aus, wenn es durch mindestens ein Ereignis im Serverlebenszyklus ausgelöst wird. Mögliche Server-Auslösungsereignisse sind: Initialisierung, Start, Service-bereite Anfragen und Herunterfahren. Lebenszyklusmodule sind nicht Bestandteil der Java EE-Spezifikation, stellen jedoch eine Erweiterung von GlassFish-Server dar.

Folgende Informationen werden für jedes Lebenszyklusmodul zur Verfügung gestellt.

Name

Der Name des Lebenszyklusmoduls.

Aktiviert oder Status

Ein Häkchen, wenn die Anwendung aktiviert ist. Ein X, wenn die Anwendung deaktiviert und nur die standardmäßige Serverinstanz server vorhanden ist.

Wenn Cluster oder eigenständige Serverinstanzen vorhanden sind, aktivieren Sie "Enabled on All Targets" (Aktiviert auf allen Zielen), um die Ziele anzuzeigen, auf denen die Anwendung bereitgestellt wird.

Ladereihenfolge

Ein Wert, der die relative Reihenfolge festlegt, in der das Lebenszyklusmodul geladen werden soll.

Die Tabelle "Lebenszyklusmodule" enthält darüber hinaus die folgenden Optionen.

Neu

Schaltfläche zum Erstellen eines neuen Lebenszyklusmoduls.

Löschen

Schaltfläche zum Löschen eines oder mehrerer Lebenszyklusmodule.

Aktivieren

Schaltfläche zum Aktivieren eines oder mehrerer Lebenszyklusmodule. Ist verfügbar, wenn nur die standardmäßige Serverinstanz server vorhanden ist.

Deaktivieren

Schaltfläche zum Deaktivieren eines oder mehrerer Lebenszyklusmodule. Ist verfügbar, wenn nur die standardmäßige Serverinstanz server vorhanden ist.
