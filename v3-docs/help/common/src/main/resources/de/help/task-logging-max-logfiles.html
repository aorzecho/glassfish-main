So ändern Sie die maximale Anzahl der rotierenden Protokolldateien

Wenn GlassFish-Server eine Protokolldatei rotiert, wird eine neue, leere Datei mit dem Namen server.log erstellt und die alte Datei server.log_ date umbenannt, wobei date für das Datum und die Uhrzeit des Rotierens der Datei steht.

Standardmäßig wird durch GlassFish-Server die Anzahl der rotierten Dateien auf 10 begrenzt. Wenn die maximale Anzahl erreicht ist, wird die älteste Protokolldatei gelöscht, wenn GlassFish-Server eine nächste Protokolldatei rotiert.

Gegebenenfalls können Sie die maximale Anzahl der rotierten Protokolldateien ändern. Eine neu festgelegte Begrenzung gilt für rotierte Protokolldateien für sowohl Zugriffs- als auch Serverprotokollierung.

  • Definieren Sie für die Systemeigenschaft com.sun.enterprise.server.logging.max_history_files die maximale Anzahl der zu speichernden rotierten Protokolldateien.

    Weitere Informationen zum Definieren einer Systemeigenschaft finden Sie unter So verwalten Sie Systemeigenschaften.

    Die Systemeigenschaft com.sun.enterprise.server.logging.max_history_files verhält sich wie folgt:

    • Wenn die Eigenschaft nicht festgelegt ist, speichert GlassFish-Server maximal 10 rotierte Protokolldateien.

    • Wenn für die Eigenschaft ein ungültiger Wert oder Null festgelegt ist, speichert GlassFish-Server maximal 10 rotierte Protokolldateien.

    • Wenn die Eigenschaft auf 0 gesetzt ist, speichert GlassFish-Server keine rotierten Protokolldateien.
