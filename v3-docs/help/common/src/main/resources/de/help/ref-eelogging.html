Einstellungen Protokollfunktion

Verwenden Sie die Seite "Einstellungen Protokollfunktion", um die Protokollierung für die ausgewählte GlassFish-Server-Instanz oder den Cluster zu konfigurieren.

Die Seite Einstellungen Protokollfunktion enthält die folgenden Optionen.

Konfigurationsname

Der Name der Konfiguration, für die die Einstellungen auf dieser Seite gelten. Dieses Feld ist schreibgeschützt.

In Systemprotokoll schreiben

Wenn diese Option ausgewählt wird, wird der Protokolloutput zusätzlich zum Serverprotokoll an das Dienstprogramm syslog gesendet. Diese Funktion ist ausschließlich unter den Betriebssystemen Solaris und Linux verfügbar. Diese Option ist standardmäßig deaktiviert.

Protokolldatei

Ein alternativer Name oder Ort für die Server-Protokolldatei. Der Standardort ist glassfish/domains/domain1/logs/server.log.

Protokoll-Handler

Der absolute Klassenname eines benutzerdefinierten Protokoll-Handlers. Ein benutzerdefinierter Protokoll-Handler ermöglicht es Ihnen, Protokolle an einen Empfänger zu senden, der von server.log oder syslog verschieden ist. Der benutzerdefinierte Handler muss die Klasse java.util.logging.Handler erweitern (eine JSR 047-kompatible API). Platzieren Sie die Handler-Klasse im Klassenpfad von GlassFish-Server, damit der Handler beim Serversetup installiert wird.

Begrenzung für Dateirotation

Die Größe in Bytes, die die Server-Protokolldatei erreichen muss, bevor sie rotiert wird. Wenn das Serverprotokoll die angegebene Größe erreicht, wird eine neue, leere Datei mit Namen server.log erstellt. Die alte Datei wird in server.log_Datum umbenannt, wobei Datum das Datum und die Uhrzeit der Rotation angibt. Der Standardwert beträgt 2 MB. Der Mindestwert für diese Einstellung beträgt 500 KB; wenn Sie einen niedrigeren Wert einstellen, findet die Dateirotation statt, wenn die Protokolldatei eine Größe von 500 KB erreicht. Zur Deaktivierung der Protokolldateirotation setzen Sie den Wert auf 0.

Zeitlimit für Dateirotation

Die Anzahl der Minuten, nach denen das Serverprotokoll rotiert werden muss. Der Standardwert lautet 0, d. h.: Die Dateirotation wird auf Grundlage der im Feld "Begrenzung für Dateirotation" angegebenen Dateigröße durchgeführt. Wenn Sie eine oder mehrere Minuten festlegen, hat das Zeitlimit Vorrang vor dem Größenlimit, das im Feld Dateirotationslimit festgelegt wurde.

Leerungshäufigkeit

Die maximale Anzahl von Meldungen, die gleichzeitig aus der Warteschlange in das Serverprotokoll geschrieben werden können. Der Standardwert beträgt 1.
