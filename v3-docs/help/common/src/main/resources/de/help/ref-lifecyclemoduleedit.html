Lebenszyklusmodul bearbeiten

Verwenden Sie die Seite Lebenszyklusmodul bearbeiten, um die Einstellungen für ein Lebenszyklusmodul zu ändern.

Die Seite "Lebenszyklusmodul bearbeiten" enthält die folgenden Optionen.

Name

Der Name des Lebenszyklusmoduls. Der Name ist ein schreibgeschütztes Feld. Sie können den Namen nur angeben, wenn Sie ein neues Lebenszyklusmodul erstellen.

Class Name (Klassenname)

Der vollständig qualifizierte Name der Klassendatei des Lebenszyklusmoduls. Die Klasse muss die Schnittstelle com.sun.appserv.server.LifecycleListener implementieren.

Classpath (Klassenpfad)

Der Klassenpfad für das Lebenszyklusmodul. Der Klassenpfad legt fest, wo sich das Lebenszyklusmodul befindet. Der Standardwert ist domain-dir/applications . Wenn das Modul sich bereits in diesem Verzeichnis befindet (d.h. in dem Server-Klassenpfad), kann dieses Feld leer gelassen werden.

Ladereihenfolge

Die Reihenfolge, in der dieses Lebenszyklusmodul beim Start geladen werden soll. Module mit kleineren ganzzahligen Werten in der Ladereihenfolge werden früher geladen. Die Werte können von 101 bis zu MAXINT des Betriebssystems reichen. Werte von 1 bis 100 sind reserviert.

Beschreibung

Eine Beschreibung des Lebenszyklusmoduls.

Status

Wenn diese Option ausgewählt wird, ist das Lebenszyklusmodul aktiviert. Diese Option ist standardmäßig aktiviert.

Fehler beim Laden

Wenn diese Option ausgewählt ist, wird der Server heruntergefahren, wenn das Laden des Lebenszyklusmoduls fehlschlägt. Die Option ist standardmäßig deaktiviert.
