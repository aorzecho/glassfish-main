Providerkonfigurationen

Nutzen Sie die Seite "Providerkonfigurationen", um die Meldungssicherheitsprovider-Konfigurationen für die SOAP oder HttpServlet-Authentifizierungslayer zu konfigurieren.

Standardmäßig werden vier Provider für die SOAP Authentifizierungs-Layer konfiguriert.

Für jede Providerkonfiguration werden die folgenden Informationen zur Verfügung gestellt.

Provider-ID

Der Name der Providerkonfiguration.

Providertyp

Der Typ des Providers kann einer der folgenden sein:

Client

Legt fest, dass der Provider der Client-Authentifizierungsprovider ist.

Server

Legt fest, dass der Provider der Server-Authentifizierungsprovider ist.

Client-Server

Legt fest, dass der Provider sowohl ein Client- als auch ein Server-Authentifizierungsprovider ist.

Standardprovider

Falls die Einstellung wahr lautet, ist der Provider der Standardprovider.

Klassenname

Der Name der Java Implementierungsklasse des Providers.

Die Tabelle "Providerkonfigurationen" enthält darüber hinaus die folgenden Optionen.

Neu

Schaltfläche zum Erstellen einer neuen Providerkonfiguration.

Löschen

Schaltfläche zum Löschen einer oder mehrerer ausgewählter Providerkonfigurationen.

Verwandte Aufgaben

  • So werden GlassFish Server-Funktionalitäten zur Verwendung durch Nachrichtensicherheitsprovider konfiguriert

  • So wird eine Nachrichtensicherheitskonfiguration erstellt

  • So wird eine Nachrichtensicherheitskonfiguration bearbeitet

  • So wird eine Nachrichtensicherheitskonfiguration gelöscht

  • So wird eine Konfiguration eines Nachrichtensicherheitsprovider erstellt

  • So wird eine Konfiguration eines Nachrichtensicherheitsprovider bearbeitet

  • So wird eine Konfiguration eines Nachrichtensicherheitsproviders gelöscht

  • So wird die Nachrichtensicherheit für Anwendungsclients aktiviert
