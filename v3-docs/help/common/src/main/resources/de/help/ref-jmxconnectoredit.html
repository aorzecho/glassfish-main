Bearbeiten von JMX-Connector

Verwenden Sie die Seite "JMX-Connector bearbeiten", um den mit JSR 360-konformen JMX-Connector zu bearbeiten.

Die Seite "JMX-Connector bearbeiten" enthält die folgenden Optionen.

Name der Konfiguration

Der Name der Konfiguration, für die die Einstellungen auf dieser Seite angewendet werden. Dieses Feld ist schreibgeschützt.

Name

Der Name des JMX-Connectors. Dies ist ein schreibgeschütztes Feld.

JMX-Protokoll

Der Name des Protokolls, das dieser JMX-Connector unterstützt. Dies ist ein schreibgeschütztes Feld.

JMXServiceURL

Die JMX Service URL. Dies ist ein schreibgeschütztes Feld.

Security

Bei aktiviertem Kontrollkästchen Aktiviert ist die JMX-Datenübertragung verschlüsselt. Diese Option ist standardmäßig deaktiviert.

Address (Adresse)

Die IP-Adresse des Naming Service, in dem der Stub des JMX-Connector-Servers registriert ist. Dies ist nicht der Port des Server-Sockets, der die eigentliche JMX-Datenübertragung übernimmt. Dies ist die Adresse der Netzwerk-Schnittstelle, in der die RMI-Registry gestartet wird. Falls Ihr System mehrere Netzwerk-Schnittstellen besitzt, ändern Sie diesen Wert so, dass nur eine bestimmte Schnittstelle ausgewählt wird. Der Standardwert ist 0.0.0.0.

Port

Die Nummer des Ports, auf dem der Naming Service (RMI-Registry) auf RMI-Clientverbindungen horcht. Die einzige Verwendung dieses Naming Service besteht darin, die RMI-Stubs herunterzuladen. Falls der Standard-Port besetzt ist, wird ein freier Port verwendet. Gültige Werte sind 1 - 65535. Bei UNIX-Systemen erfordert das Erstellen von Sockets, die auf den Ports 1 - 1024 horchen, Superuser-Berechtigungen. Der Standardwert beträgt 8686.

Realm-Name

Der Name, der die spezielle Administrations-Realm darstellt. Alle Authentifizierungsvorgänge werden von dieser Realm abgewickelt. Der Standardwert ist admin-realm .

Weitere Eigenschaften für den JMX Connector. Verschiedene Eigenschaften werden durch Standardwerte konfiguriert.
