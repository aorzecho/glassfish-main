Aktualisierungstool - Schlüsselkonzepte

Das Aktualisierungstool verwaltet die Installationen der GlassFish-Server-Softwarekomponenten, die in einem einzelnen Dateisystemverzeichnis auf Ihrem System enthalten sind.

Die Administration Console-Version von Aktualisierungstool bietet die folgenden Möglichkeiten:

  • Add-On-Komponenten installieren

  • Aktualisierungen für installierte Komponenten anzeigen

  • Installierte Komponenten anzeigen

Um installierte Komponenten aktualisieren oder entfernen zu können, müssen Sie die GlassFish-Server-Domain anhalten und dann einen der folgenden Befehle verwenden:

  • as-install-parent/bin/updatetool (zum Starten des grafischen Aktualisierungstools)

  • as-install-parent/bin/pkg (Befehlszeilenversion des Aktualisierungstools)

Die Schlüsselkonzepte für das Aktualisierungstool werden in den folgenden Abschnitten beschrieben.

Komponente

Eine Komponente ist die installierbare Einheit, die vom Aktualisierungstool verwaltet wird. Eine Komponente besteht aus Dateien, die als Einheit auf Ihrem System innerhalb eines Installations-Image installiert sind.

Installations-Image

Ein Installations-Image ist das Verzeichnis, das die Softwarekomponenten enthält, einschließlich des Inhalts des Verzeichnisses. Das Installations-Image wird dem Aktualisierungstool als Ergebnis des Herunterladens und Installierens der GlassFish-Server-Software hinzugefügt. Um das Hinzufügen und Aktualisieren von GlassFish-Server-Softwarekomponenten zu ermöglichen, enthält jedes Installations-Image Informationen zur Installation von Komponenten.

Netzwerk-Repository

Ein Netzwerk-Repository enthält folgende Elemente:

  • Komponenten, die einem Installations-Image hinzugefügt werden können

  • Updates von vorhandenen Komponenten eines Installations-Image

Ein Installations-Image ist mit einem oder mehreren Netzwerk-Repositorys verbunden. Jedes Netzwerk-Repository wird durch einen Namen und eine entsprechende URL dargestellt. Ein Repository-Name ist eindeutig für jedes Installations-Image und steht sowohl für die Organisation oder das Projekt, die das Repository verwalten, als auch für einen klar definierten Inhaltsbereich. Weitere Informationen zu den für GlassFish-Server vorkonfigurierten Netzwerk-Repositorys finden Sie unter Preconfigured Repositories for GlassFish Server in Oracle GlassFish Server 3.1 Administration Guide .

Add-On-Komponente

Eine Add-On-Komponente ist eine Komponente des Installations-Image, die noch nicht installiert ist.
