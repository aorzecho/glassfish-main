Pour modifier la limite du nombre de fichiers journaux à rotation

Lorsque GlassFish Server fait pivoter un fichier journal, il crée un fichier vide nommé server.log et renomme l’ancien fichier server.log_ date, où date indique la date et l'heure de rotation du fichier.

Par défaut, GlassFish Server limite le nombre de fichiers journaux à rotation à 10. Lorsque cette limite est atteinte, le fichier journal le plus ancien est supprimé la prochaine fois que GlassFish Server fait pivoter un fichier journal.

Si nécessaire, vous pouvez modifier la limite du nombre de fichiers journaux à rotation. Toute nouvelle limite définie s'applique aux fichiers journaux à rotation pour la journalisation des accès et le journal du serveur.

  • Définissez la propriété système com.sun.enterprise.server.logging.max_history_files sur le nombre maximum de fichiers journaux à rotation à conserver.

    Pour plus d'informations sur la définition d'une propriété système, reportez-vous à Pour gérer les propriétés système.

    Le comportement de la propriété système com.sun.enterprise.server.logging.max_history_files est le suivant :

    • Si la propriété n'est pas définie, GlassFish Server conserve jusqu'à 10 fichiers journaux à rotation.

    • Si la propriété est définie sur un nombre incorrect ou nul, GlassFish Server conserve jusqu'à 10 fichiers journaux à rotation.

    • Si la propriété est définie sur 0, GlassFish Server ne conserve aucun fichier journal à rotation.
