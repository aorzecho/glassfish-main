Filtrer les enregistrements de journal affichés

Avant de commencer

Vérifiez que vous avez affiché les fichiers journaux du serveur d'administration de domaine (DAS), d'une instance GlassFish Server clusterisée ou d'une instance autonome. Pour plus d'informations, consultez les rubriques suivantes :

  • Afficher les fichiers journaux du serveur DAS

  • Afficher les fichiers journaux d'une instance GlassFish Server clusterisée

  • Afficher les fichiers journaux d'une instance GlassFish Server autonome

  1. Dans la zone Critères de recherche, filtrez les enregistrements par contenu, période et niveau de journalisation.

    1. Pour afficher uniquement les enregistrements contenant un texte spécifique, saisissez le texte dans le champ Recherche de texte.

      La recherche fait la distinction entre les majuscules et les minuscules.

    2. Afin de sélectionner la période pour laquelle les enregistrements de journal sont affichés, utilisez l'option Horodatage.

      • Pour afficher les messages les plus récents, sélectionnez Le plus récent (valeur par défaut).

      • Pour afficher uniquement les messages d'une période donnée, sélectionnez Plage spécifique, puis entrez une date et une heure dans les champs De et A qui apparaissent.

        La valeur Date doit avoir le format suivant :

        MM/DD/YY

        MM est un nombre entier compris entre 1 et 12 qui désigne le numéro du mois (par exemple, 1 pour le mois de janvier ou 10 pour le mois d'octobre). DD est un nombre entier compris entre 1 e  31 qui désigne le jour du mois. YY sont les deux derniers chiffres de l'année (par exemple, 11 pour l'année 2011).

        Par exemple, pour spécifier la date du 1er février 2011, saisissez 2/1/11 dans le champ Date.

        La valeur Heure doit avoir le format suivant :

        hh:mm:ss.SSS

        hh est un nombre entier compris entre 0 et 23 qui indique l'heure. Pour les heures de l'après-midi, utilisez des nombres entiers compris entre 12 et 23 (par exemple, 14 pour 2 PM). mm est un nombre entier compris entre 0 et 59 qui indique le nombre de minutes après l'heure. ss est un nombre entier compris entre 0 et 59 qui indique le nombre de secondes. SSS est un nombre entier à trois chiffres, précédé de zéros si nécessaire, compris entre 000 et 999, qui indique le nombre de millisecondes.

        Par exemple, pour indiquer l'heure 5.10 PM, saisissez 17:10:00.000 dans le champ Heure.

        Si la valeur de début est postérieure à la valeur de fin, un message d'erreur apparaît.

    3. Filtrez les messages par niveau de journalisation.

      Pour être sûr que les messages à afficher apparaissent dans le journal du serveur, commencez par définir les niveaux de journalisation appropriés sur la page Niveaux de journalisation de module. Pour plus d'informations, reportez-vous à Configurer les niveaux de journalisation.

      Si vous choisissez de filtrer les messages du journal en fonction du niveau de journalisation, seuls les messages correspondant aux critères de filtre spécifiés s'affichent. Toutefois, ce filtrage n'affecte pas les messages consignés dans le journal du serveur.

      1. Sélectionnez un élément de liste déroulante Niveau de journalisation.

      2. Pour afficher uniquement les messages du niveau choisi, sélectionnez l'option Ne pas inclure des messages plus graves.

        Par défaut, tous les messages figurant dans le journal du serveur à partir du niveau de journalisation choisi sont affichés.

  2. Dans la zone Recherche avancée, filtrez les enregistrements par espace de noms du journaliseur et sélectionnez la quantité d'informations à afficher.

    1. Cliquez sur le lien Recherche avancée.

      La zone Recherche avancée est ajoutée à la fenêtre du visualiseur de journal.

    2. Pour filtrer par module, sélectionnez au moins un espace de noms dans la liste déroulante Journaliseur.

      Appuyez sur les touches Maj ou Ctrl pendant que vous cliquez pour sélectionner plusieurs espaces de noms.

      Si vous sélectionnez un espace de noms à un niveau supérieur, tous les espaces de noms situés en dessous seront également sélectionnés. Par exemple, si vous sélectionnez javax.enterprise.system, seront également sélectionnés les journaliseurs de tous les modules situés sous cet espace de noms, tels que javax.enterprise.system.core, javax.enterprise.system.tools.admin , etc.

    3. Pour afficher les messages provenant de journaliseurs d'une application particulière, saisissez le nom de chaque journaliseur sur une ligne distincte dans le champ Journaliseur personnalisé.

      Si l'application comprend plusieurs modules, vous pouvez les afficher tous ou seulement une partie. Par exemple, une application peut disposer des journaliseurs suivants :

      com.example.myapp.module1
com.example.myapp.module2
com.example.myapp.module3

      Pour afficher les messages de tous les modules de l'application, tapez com.mycompany.myapp. Pour afficher les messages du module2 uniquement, entrez com.example.myapp.module2.

      Si vous indiquez des journaliseurs personnalisés, les messages des modules GlassFish Server n'apparaissent que si vous les avez explicitement spécifiés dans la zone Journaliseur.

    4. Dans la liste déroulante Afficher, sélectionnez le nombre de messages à afficher.

      Le nombre de messages affichés par défaut est de 40. Les autres possibilités sont 100, 250 et 1 000.

    5. Pour afficher les traces de pile, désélectionnez l'option Tronquer les messages trop longs.

      Par défaut, les traces de pile n'apparaissent pas dans le visualiseur. Pour les afficher, cliquez sur le lien (détails) d'un message.

    6. Pour masquer la zone Recherche avancée, cliquez sur le lien Recherche de base.

  3. Pour indiquer l'instance et le fichier journal associé à afficher, utilisez la zone Modifier la recherche.

    • Dans la liste déroulante Instance, sélectionnez l'instance.

      La valeur par défaut est l'instance sélectionnée initialement pour afficher les fichiers journaux. Si vous avez choisi d'afficher des fichiers journaux du serveur d'administration de domaine (DAS), la valeur par défaut est server, ce qui désigne le serveur DAS.

    • Dans la liste déroulante Fichier journal, sélectionnez le fichier journal.

      La valeur par défaut est server.log. Les autres possibilités dépendent des paramètres du journal pour la cible sélectionnée.

  4. Dans la zone Résultats de la recherche, vous pouvez personnaliser l'affichage des messages.

    La zone Résultats de la recherche affiche les 40 dernières entrées du journal du serveur, en fonction des paramètres définis sur les pages Paramètres de journalisation et Niveaux de journalisation.

    • Pour trier les messages différemment, cliquez sur les flèches situées dans les colonnes.

    • Pour afficher la version formatée d'un message, cliquez sur le lien (détails) du message.

      La fenêtre Détails de l'entrée de journal apparaît. Elle contient une version formatée du message.

    • Pour afficher les entrées antérieures ou postérieures du fichier journal, cliquez sur Enregistrements avant et Enregistrements après en haut de la table.
