Récupérer les transactions

La page Récupérer les transactions vous permet de récupérer les transactions effectuées pour une instance GlassFish Server.

Si l'instance est en cours d'exécution, la page Récupérer les transactions contient uniquement le texte La récupération est effectuée par le serveur en cours d’exécution lui-même et le bouton OK.

Si l'instance n'est pas en cours d'exécution, la page Récupérer les transactions contient les options suivantes.

Nom d’instance

Nom assigné à l'instance lors de sa création. Ce champ est en lecture seule.

Serveur de destination

Autre instance pouvant récupérer les transactions sur l'instance active.

Répertoire du journal des transactions

Répertoire dans lequel les journaux de transactions sont stockés. La valeur par défaut est /logs/tx sous le répertoire du domaine.

Les journaux des transaction sont conservés dans le sous-répertoire tx du répertoire spécifié par le paramètre Emplacement du journal des transactions du Service de transaction pour la configuration du cluster. La valeur Emplacement du journal des transactions par défaut est le répertoire spécifié dans le champ Racine journal de la page Attributs de domaine, qui se trouve dans l’onglet Avancé du nœud GlassFish Server.
