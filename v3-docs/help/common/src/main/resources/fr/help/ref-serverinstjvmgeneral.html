Paramètres généraux JVM

La machine virtuelle Java (JVM) permet d'exécuter du code binaire Java sur un ordinateur physique. Vous pouvez régler les paramètres JVM pour améliorer les performances et empêcher les erreurs d'allocation de mémoire.

La machine virtuelle Java est incluse dans le logiciel Java Platform Standard Edition (plate-forme Java SE), nécessaire à l'exécution de GlassFish Server. La saisie de paramètres JVM incorrects empêchant l'exécution du serveur, vous devez être prudent lors de la modification de ces paramètres.

Remarque:

Les termes "Java Virtual Machine" et "JVM" désignent une machine virtuelle pour la plate-forme Java.

La page Paramètres généraux JVM contient les options suivantes :

Charger les valeurs par défaut

Bouton de restauration des valeurs par défaut des paramètres qui ont des valeurs par défaut. Les paramètres qui n'ont pas de valeurs par défaut ne sont pas modifiés.

Nom de configuration

Nom de la configuration à laquelle s'appliquent les paramètres de cette page. Ce champ est en lecture seule.

Répertoire d'origine Java

Nom du répertoire d'installation du logiciel Java SE, nécessaire à l'exécution de GlassFish Server. Pour vérifier que la version du logiciel Java SE spécifié est prise en charge par cette version, consultez le document Oracle GlassFish Server Release Notes. Si vous saisissez un nom de répertoire inexistant ou le répertoire d'une version non prise en charge du logiciel Java SE, GlassFish Server ne démarre pas.

Options Javac

Options de ligne de commande pour le compilateur de langage de programmation Java. GlassFish Server exécute le compilateur lorsque des composants EJB (Enterprise Java Beans) sont déployés.

Déboguer

Active le débogage avec JPDA (architecture de débogueur de plate-forme Java).

L'architecture JPDA est utilisée par les développeurs d'applications. Pour plus d'informations, reportez-vous au manuel Oracle GlassFish Server Application Development Guide.

Options de débogage

Indique les options JPDA transmises à la JVM lorsque le débogage est activé.

Options de compilation RMI

Options de ligne de commande pour le compilateur rmic. GlassFish Server exécute le compilateur rmic lorsque des composants EJB sont déployés.

Préprocesseur du code exécutable

Noms des classes de prétraitement du code exécutable. Séparez les noms de classe par des virgules. Chaque classe doit implémenter l'interface com.sun.appserv.BytecodePreprocessor. Les classes sont appelées par ordre d'énumération.

Autres propriétés

Autres propriétés de la JVM. GlassFish Server ne définit aucune autre propriété pour la JVM.
