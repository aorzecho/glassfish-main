Paramètres généraux JVM

La machine virtuelle Java (JVM) permet d’exécuter du code binaire Java sur une machine physique. Vous pouvez régler les paramètres de JVM pour améliorer les performances et empêcher des erreurs d’allocation de mémoire de se produire.

La machine virtuelle Java est incluse dans le logiciel Java Platform Standard Edition (plate-formeJavaSE), nécessaire à l’exécution de GlassFish Server. La saisie de paramètres JVM incorrects empêchant l’exécution du serveur, vous devez être prudent(e) lors de la modification de ces paramètres.

Remarque - Les termes « Machine Java Virtual » et « JVM » désignent une machine virtuelle pour plate-forme Java.

La page Paramètres généraux JVM contient les options suivantes :

Restaurer les valeurs par défaut

Bouton de restauration des valeurs par défaut des paramètres qui ont des valeurs par défaut. Les paramètres qui n'ont pas de valeurs par défaut ne sont pas modifiés.

Nom de la configuration

Nom de la configuration à laquelle les paramètres de cette page s'appliquent. Ce champ est en lecture seule.

Accueil Java

Nom du répertoire d’installation du logiciel Java SE, nécessaire à l’exécution d’GlassFish Server. Pour vérifier que la version du logiciel Java SE spécifié est prise en charge par cette version, consultez les Oracle GlassFish Server 3.1-3.1.1 Release Notes. Si vous saisissez un nom de répertoire inexistant ou le répertoire d’une version non prise en charge du logiciel Java SE, GlassFish Server ne démarrera pas.

Options Javac

Options de ligne de commande pour le compilateur de langage de programmation Java. GlassFish Server exécute le compilateur lorsque des composants Enterprise JavaBeans (EJB) sont déployés.

Débogage

Active le débogage avec JPDA (Architecture de débogueur de plate-forme Java).

L’architecture JPDA est utilisée par les développeurs d’applications. Pour plus d'informations, reportez-vous au Oracle GlassFish Server 3.1 Application Development Guide.

Options de débogage

Indique les options JPDA transmises à la JVM quand le débogage est activé.

Options de compilation RMI

Options de ligne de commande pour le compilateur rmic. GlassFish Server exécute le compilateur rmic lorsque des composants EJB sont déployés.

Préprocesseur du bytecode

Noms des classes de prétraitement du bytecode. Séparez les noms de classe par des virgules. Chaque classe doit implémenter l’interface com.sun.appserv.BytecodePreprocessor . Les classes sont appelées par ordre d’énumération.

Autres propriétés

Autres propriétés de la JVM. Le GlassFish Server ne définit aucune autre propriété pour la JVM.
