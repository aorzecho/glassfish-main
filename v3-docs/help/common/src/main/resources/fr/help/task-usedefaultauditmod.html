Utiliser le module d’audit par défaut

Le module d’audit default consigne les requêtes d’authentification et d’autorisation dans le fichier journal du serveur. Pour savoir comment changer l’emplacement du fichier journal, voir Configurer les paramètres généraux de journalisation.

Les entrées relatives à l’authentification comprennent les informations suivantes :

  • Noms des utilisateurs qui tentent de s’authentifier

  • Domaine ayant traité la requête d’accès

  • Module Web URI ou composant EJB requis

  • Réussite ou échec de la requête

Que la fonction de journalisation d’audit soit activée ou non, le GlassFish Server consigne tous les événements d’authentification refusée.

Les entrées relatives aux autorisations comprennent les informations suivantes :

  • Noms des utilisateurs authentifiés, le cas échéant

  • Module Web URI ou composant EJB requis

  • Réussite ou échec des requêtes

  1. Dans l’arborescence de navigation, développez le nœud Configuration.
  2. Sous le nœud Configuration, sélectionnez le nœud Sécurité.

    La page Sécurité s’ouvre.

  3. Dans la liste déroulante Modules d’audit, sélectionnez le module d’audit nommé valeur par défaut.
  4. Cliquez sur Enregistrer.
