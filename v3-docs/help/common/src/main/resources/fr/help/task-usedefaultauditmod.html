Utiliser le module d'audit par défaut

Le module d'audit default consigne les demandes d'authentification et d'autorisation dans le fichier journal du serveur. Pour plus d'informations sur la modification de l'emplacement du fichier journal, reportez-vous à Configurer les paramètres généraux de journalisation.

Les entrées de journal relatives à l'authentification comprennent les informations suivantes :

  • Noms des utilisateurs qui tentent de s'authentifier

  • Domaine ayant traité la demande d'accès

  • Module Web URI ou composant EJB requis

  • Succès ou échec de la demande

Que la journalisation d'audit soit activée ou non, GlassFish Server consigne tous les événements d'authentification refusée.

Les entrées de journal relatives aux autorisations comprennent les informations suivantes :

  • Noms des utilisateurs authentifiés, le cas échéant

  • Module Web URI ou composant EJB requis

  • Succès ou échec de la demande

  1. Dans l'arborescence de navigation, développez le noeud Configuration.

  2. Select the instance to configure:

    • To configure a particular instance, expand the instance's config node. For example, the default instance, server, expand the server-config node.

    • To configure the default settings for all instances, expand the default-config node.

  3. Sous le noeud Configuration, sélectionnez le noeud Sécurité.

    La page Sécurité apparaît.

  4. Dans la liste déroulante Modules d'audit, sélectionnez le module d'audit nommé default.

  5. Cliquez sur Enregistrer.
