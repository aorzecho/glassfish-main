Administration sécurisée

Utilisez la page Administration sécurisée pour activer ou désactiver l'administration sécurisée.

Lorsque l'administration sécurisée est activée :

  • Le serveur DAS et les instances du domaine utilisent des certificats SSL pour crypter les messages qu'ils échangent.

  • Le serveur DAS accepte les messages d'administration des clients admin distants tels que l'utilitaire asadmin et les IDE.

La page Administration sécurisée contient les informations suivantes.

Activer l'administration sécurisée

Bouton d'activation de l'administration sécurisée. Ce bouton apparaît uniquement lorsque l'administration sécurisée est désactivée.

Remarque:

Si l'administrateur du domaine possède un mot de passe vide, les tentatives d'activation de l'administration sécurisée échouent.
Désactiver l'administration sécurisée

Bouton de désactivation de l'administration sécurisée. Ce bouton apparaît uniquement lorsque l'administration sécurisée est activée.

Alias d'administration

Alias qui fait référence au certificat SSL/TLS sur le serveur DAS. Cet alias est utilisé par le serveur DAS en guise d'authentification vis-à-vis des instances. La valeur par défaut est s1as. Cette option apparaît uniquement lorsque l'administration sécurisée est désactivée.

Alias d'instance

Alias qui fait référence au certificat SSL/TLS sur les instances. Cet alias est utilisé par les instances en guise d'authentification vis-à-vis du serveur DAS. La valeur par défaut est glassfish-instance. Cette option apparaît uniquement lorsque l'administration sécurisée est désactivée.
