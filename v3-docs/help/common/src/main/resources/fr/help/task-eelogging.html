Configurer les paramètres généraux de journalisation

  1. Dans l'arborescence de navigation, développez le noeud Configurations.

  2. Sous le noeud Configurations, cliquez sur la configuration d'instance de serveur ou de cluster pour laquelle vous souhaitez définir les paramètres du journaliseur.

    La page Configuration apparaît.

  3. Sur la page Configuration, cliquez sur Paramètres du journaliseur.

    La page Paramètres du journaliseur de la cible de configuration sélectionnée apparaît.

  4. Cochez la case Ecrire dans le journal système pour envoyer le résultat de la journalisation à l'utilitaire syslog en plus du journal du serveur.

    Cette fonctionnalité est disponible sur les systèmes Solaris et Linux uniquement. Cette option est désactivée par défaut.

  5. Dans le champ Fichier journal, saisissez un autre nom ou emplacement pour le fichier journal du serveur.

    L'emplacement par défaut est ${com.sun.aas.instanceRoot}/logs/server.log.

  6. Dans le champ Gestionnaire de journaux, saisissez le nom de classe absolu d'un gestionnaire de journal personnalisé.

    Un gestionnaire de journal personnalisé permet d'envoyer des journaux vers une destination autre que server.log ou syslog. Celui-ci doit venir en extension de la classe java.util.logging.ConsoleHandler (API compatible JSR 047). Ajoutez la classe du gestionnaire au classpath de GlassFish Server de façon à installer le gestionnaire au démarrage du serveur.

  7. Dans le champ Limite de rotation du fichier, saisissez la taille en octets que le fichier journal du serveur doit atteindre avant sa rotation.

    Lorsque le journal du serveur atteint la taille indiquée, GlassFish Server crée un fichier vide nommé server.log et renomme l'ancien fichier server.log_date, où date indique la date et l'heure de rotation du fichier. La valeur par défaut est 2 Mo. La valeur minimale est 500 ko. Si vous spécifiez une valeur inférieure, le système effectuera la rotation à 500 ko. Pour désactiver la rotation du fichier journal, spécifiez la valeur 0.

  8. Dans le champ Temps limite de rotation du fichier, saisissez le nombre de minutes au bout duquel la rotation du journal du serveur doit être effectuée.

    La valeur par défaut est 0 ; cela signifie que le système effectuera la rotation lorsque le fichier aura atteint la taille spécifiée dans le champ Limite de rotation des fichiers.

    Remarque:

    Si vous spécifiez une ou plusieurs minutes, le temps limite remplace la taille limite indiquée par le champ Limite de rotation des fichiers.

  9. Dans le champ Fréquence de vidage, saisissez le nombre maximal de messages à écrire simultanément de la file d'attente vers le journal du serveur.

    La valeur par défaut est 1.

  10. Cliquez sur Enregistrer.

    Remarque:

    Selon la configuration pour laquelle la journalisation est activée, vous devrez peut-être redémarrer le serveur.

    Un redémarrage du serveur peut également être nécessaire si le nom du fichier journal est modifié par rapport à la valeur par défaut. Un message Redémarrage requis apparaît dans le coin supérieur droit de la Console d'administration s'il est nécessaire de redémarrer le serveur.
