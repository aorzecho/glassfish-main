Service de surveillance

La page Service de surveillance permet de configurer les options de surveillance des composants ou services des serveurs individuels.

Vous devez déployer une application, un composant ou un service avant d'en configurer la surveillance.

La page Service de surveillance contient les options suivantes :

Nom de configuration

Nom de la configuration à laquelle s'appliquent les paramètres de cette page. Ce champ est en lecture seule.

Service de surveillance

Si cette case est cochée, la surveillance est activée pour GlassFish Server. Cette option est activée par défaut.

MBeans de surveillance

Si cette case est cochée, tous les MBeans nécessaires à la surveillance sont déployés sur GlassFish Server. Cette option est activée par défaut.

Si la case n'est pas cochée, vous ne pouvez pas afficher les données de surveillance dans la Console d'administration , même si la case Service de surveillance est cochée. Au lieu de cela, vous devez utiliser la ligne de commande pour afficher les données de surveillance. Si vous désactivez la case à cocher, l'empreinte de mémoire de GlassFish Server est réduite.

DTrace

Si la case DTrace est cochée et si vous avez activé la surveillance DTrace de Oracle Enterprise Manager, vous pouvez utiliser l'outil DTrace Solaris. Cette option est désactivée par défaut.

Pour chaque composant disponible, les informations suivantes sont fournies.

Composant

Nom du composant de GlassFish Server.

Niveau de surveillance

Niveau de surveillance défini pour le composant. Pour chaque composant, les niveaux disponibles sont LOW, HIGH et OFF. Le niveau par défaut est OFF. Dans cette version, il n'existe aucune différence entre les options LOW et HIGH.

La table Paramètres de niveau composant contient également les options suivantes :

Modifier le niveau

Bouton permettant de modifier le niveau de surveillance des composants sélectionnés.

Niveau

Liste déroulante dans laquelle vous pouvez sélectionner un niveau de surveillance à appliquer à tous les composants sélectionnés.
