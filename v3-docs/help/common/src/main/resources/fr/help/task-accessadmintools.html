Autoriser l’accès aux outils d’administration

Seuls les utilisateurs du groupe asadmin ont accès à la Console d'administration et à l’utilitaire de ligne de commande asadmin. Pour autoriser l’accès à ces outils d’administration, ajoutez les utilisateurs au groupe asadmin dans le domaine admin-realm.

  1. Sous le nœud Configurations, développez la configuration de l'instance de serveur ou de cluster pour lequel vous souhaitez configurer les privilèges d'accès d'administration.
  2. Développez le nœud Sécurité.
  3. Développez le nœud Domaines.
  4. Sélectionnez le nœud admin-realm.

    La page Modification du domaine s’ouvre.

  5. Sur la page Modification du domaine, cliquez sur le bouton Gérer les utilisateurs.

    La page Administrateurs s’ouvre.

    Après l’installation, le nom et le mot de passe administrateur entrés lors de l’installation figurent dans un fichier nommé admin-keyfile. Par défaut, cet utilisateur appartient au groupe asadmin, qui confère le droit de modifier le GlassFish Server. N’assignez des utilisateurs à ce groupe que si vous voulez leur accorder des privilèges d’administrateur pour l'instance ou le cluster de &ProductName.

    Si vous ajoutez des utilisateurs au domaine admin-realm, mais que vous les assignez à un autre groupe que asadmin, les informations utilisateur seront toujours enregistrées dans le fichier nommé admin-keyfile, mais l’utilisateur n’aura pas accès aux outils d’administration ni aux applications du domaine file.

  6. Sur la page Administrateurs, cliquez sur Nouveau pour ajouter un utilisateur au domaine admin-realm.

    La page Nouvel utilisateur du domaine fichier s’ouvre.

  7. Dans le champ ID utilisateur, saisissez le nouveau nom d’utilisateur.
  8. Dans le champ Liste des groupes, saisissez les noms des groupes auxquels l’utilisateur appartient.

    Pour autoriser un utilisateur à modifier le GlassFish Server, ajoutez le groupe asadmin à la Liste des groupes.

  9. Dans les champs Nouveau mot de passe et Confirmer le mot de passe, saisissez le mot de passe du nouvel utilisateur.
  10. Cliquez sur OK pour ajouter cet utilisateur au domaine admin-realm ou sur Annuler pour quitter sans sauvegarder.
