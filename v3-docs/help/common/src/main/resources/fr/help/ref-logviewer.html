Visualiseur de journaux

La page Visualiseur de journaux permet d'afficher les fichiers journaux des clusters et instances GlassFish Server sélectionnés, et de personnaliser l'affichage desdits journaux.

La page Visualiseur de journaux se compose de quatre zones :

  • Critères de recherche

  • Recherche avancée

  • Modifier la recherche

  • Résultats de la recherche

La zone Critères de recherche contient les options suivantes :

Recherche avancée

Un clic sur ce lien ouvre une zone qui vous permet d'affiner davantage le visualiseur du journal.

Horodatage

Le bouton d'option Le plus récent affiche les messages les plus récents. Le bouton d'option Plage spécifique ouvre les champs De et A qui permettent d'afficher les messages d'une certaine période uniquement.

Pour la valeur Heure, la syntaxe doit avoir le format suivant (SSS correspond aux millisecondes) :

hh:mm:ss.SSS

Exemple :

17:10:00.000

Si la valeur de début est postérieure à la valeur de fin, un message d'erreur apparaît.

Niveau de journalisation

Niveau de journalisation sélectionné pour le filtrage. Par défaut, tous les messages figurant dans le journal du serveur à partir du niveau de journalisation choisi sont affichés. Pour afficher uniquement les messages du niveau choisi, cochez la case Ne pas inclure des messages plus graves.

Pour vous assurer que les messages à afficher apparaissent dans le journal du serveur, commencez par définir les niveaux de journalisation appropriés sur la page Niveaux de journalisation de module. Reportez-vous à Configurer les niveaux de journalisation.

Si vous choisissez de filtrer les messages du journal en fonction du niveau de journalisation, seuls les messages correspondant aux critères de filtre spécifiés apparaissent. Toutefois, ce filtrage n'affecte pas les messages consignés dans le journal du serveur.

La zone Recherche avancée contient les options suivantes :

Journaliseur

Espaces de noms du journaliseur à filtrer par module. Appuyez sur les touches Maj ou Ctrl pendant que vous cliquez pour sélectionner plusieurs espaces de noms.

Si vous sélectionnez un espace de noms à un niveau supérieur, tous les espaces de noms situés en dessous seront également sélectionnés. Par exemple, si vous sélectionnez javax.enterprise.system, seront également sélectionnés les journaliseurs de tous les modules situés sous cet espace de noms : javax.enterprise.system.core, javax.enterprise.system.tools.admin , etc.

Journaliseur personnalisé

Journaliseurs propres à une application particulière, un par ligne.

Si l'application comprend plusieurs modules, vous pouvez les afficher tous ou seulement une partie. Prenons l'exemple d'une application comprenant les journaliseurs suivants :

com.mycompany.myapp.module1
com.mycompany.myapp.module2
com.mycompany.myapp.module3

Pour afficher les messages de tous les modules de l'application, tapez com.mycompany.myapp. Pour afficher les messages du module2 uniquement, entrez com.mycompany.myapp.module2.

Si vous indiquez des journaliseurs personnalisés, les messages des modules GlassFish Server n'apparaissent que si vous les avez explicitement spécifiés dans la zone Journaliseur.

Affichage

Nombre de messages à afficher simultanément. Le nombre de messages affichés par défaut est de 40. Les autres possibilités sont 100, 250 et 1 000.

Pour afficher les traces de pile, désélectionnez la case Tronquer les messages trop longs. Par défaut, les traces de pile n'apparaissent pas dans le visualiseur. Pour les afficher, cliquez sur le lien (détails) du message.

La zone Modifier la recherche permet d'indiquer l'instance de serveur ou le cluster, ainsi que le nom d'un fichier journal associé à afficher.

Instance

Instance de serveur ou cluster dont les enregistrements de journal seront affichés. La valeur par défaut est le serveur. Les autres possibilités dépendent des clusters et des instances pouvant être définis et activés pour le domaine.

Fichier journal

Nom du fichier journal à afficher. La valeur par défaut est server.log. Les autres possibilités dépendent des paramètres du journal pour la cible sélectionnée.

La table Résultats du visualiseur de journaux affiche les messages du journal du serveur. Utilisez les boutons Enregistrements avant et Enregistrements après situés en haut de la table pour choisir la plage d'enregistrements à afficher.

Pour chaque message, la table Résultats du visualiseur de journaux contient les informations suivantes :

Numéro d'enregistrement

Numéro du message dans le journal du serveur.

Niveau de journalisation

Niveau de journalisation du message.

Message

Texte bref du message du journal. Cliquez sur le lien (détails) pour afficher la version formatée du message.

Journaliseur

Module de journalisation qui a généré le message du journal.

Horodatage

Date et heure d'émission du message du journal.

Paires nom-valeur

Nom du thread et ID du message du journal.
