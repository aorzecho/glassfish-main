Nouveau module de cycle de vie

La page Nouveau module de cycle de vie permet de définir un nouveau module de cycle de vie.

Elle contient les options suivantes :

Nom

Nom du module de cycle de vie.

Nom de classe

Nom qualifié complet du fichier de classe du module de cycle de vie. La classe doit implémenter l'interface com.sun.appserv.server.LifecycleListener.

Classpath

Classpath du module de cycle de vie. Le classpath indique l'emplacement du module de cycle de vie. La valeur par défaut est domain-dir/applications. Si le module est déjà dans ce répertoire (c'est-à-dire dans le classpath du serveur), ce champ peut être laissé vide.

Ordre de chargement

Ordre dans lequel ce module de cycle de vie doit être chargé au démarrage. Les modules aux valeurs entières les plus faibles dans l'ordre de chargement sont chargés en premier. Ce chiffre peut aller de 101 à la valeur MAXINT du système d'exploitation. Les valeurs de 1 à 100 sont réservées.

Description

Description du module de cycle de vie.

Statut

Si cette option est sélectionnée, le module de cycle de vie est activé. Cette option est activée par défaut.

Echec au chargement

Si cette option est sélectionnée, le serveur sera arrêté en cas d'échec de chargement du module de cycle de vie. Cette option est désactivée par défaut.

Cibles

Clusters et instances autonomes pour le module de cycle de vie. Déplacez les cibles souhaitées dans la colonne Cibles sélectionnées à l'aide des boutons Ajouter et Tout ajouter. Déplacez toutes les cibles inutiles dans la colonne Cibles disponibles à l'aide des boutons Enlever et Tout enlever.

Cette option apparaît uniquement si des clusters ou des instances autonomes ont été créés dans le domaine.
