Afficher les informations relatives aux adresses d'un service Web

  1. Dans l'arborescence de navigation, sélectionnez le noeud Applications.

    La page Applications apparaît.

  2. Sur le page Applications, cliquez sur le nom du service Web dont vous affichez les informations d'adresse.

    Le nom peut être suivi d'un identificateur de version facultatif dont il est séparé par un signe deux-points (:). Pour plus d'informations sur les versions des modules et des applications, reportez-vous à "Module and Application Versions" dans le manuel Oracle GlassFish Server Application Deployment Guide.

    La page Modifier une application apparaît.

  3. Dans la table Modules et composants, cliquez sur le lien Afficher l'adresse du sous-composant.

    La page Informations relatives à une adresse de service Web apparaît.

  4. Si le lien Testeur est visible, cliquez dessus pour afficher une page qui permet d'indiquer des paramètres visant à tester les méthodes du service Web.

    Le lien Testeur apparaît uniquement si le type de déploiement de l'application n'est pas RI (par exemple, si c'est 109). La page Testeur apparaît dans un nouvel onglet ou une nouvelle fenêtre du navigateur.

  5. Cliquez sur le lien WSDL pour afficher le fichier WSDL (Web Services Definition Language) associé à l'adresse.

    Le fichier WSDL s'ouvre dans un nouvel onglet ou une nouvelle fenêtre du navigateur.

    Notez que certains navigateurs Web tentent d'interpréter la source WSDL lorsque vous cliquez sur le lien WSDL, affichant ainsi une page vierge. Pour de tels navigateurs, utilisez la commande d'affichage de la source du navigateur sur la page vierge pour afficher la source WSDL.

  6. Cliquez sur le lien Nom de l'application ou sur le bouton Précédent pour revenir à la page Modifier une application.
