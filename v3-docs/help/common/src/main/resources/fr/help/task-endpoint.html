Affichage des informations relatives aux extrémités des services Web

  1. Dans l’arborescence de navigation, sélectionnez le nœud Applications.

    La page Applications s’affiche.

  2. Sur le page Applications, cliquez sur le nom du service Web dont vous affichez les informations d’extrémité.

    Le nom peut être suivi d'un identificateur de version facultatif dont il est séparé par un signe deux-points (:). Pour plus d'informations sur le module et les versions d'application, reportez-vous à la section Module and Application Versions du Oracle GlassFish Server 3.1 Application Deployment Guide.

    La page Modification de l’application s’affiche.

  3. Dans le tableau Modules et composants, cliquez sur le lien Afficher le point d’extrémité pour le sous-composant.

    La page Informations relatives aux extrémités des services Web s’ouvre.

  4. Si le lien Testeur est visible, cliquez sur ce lien pour afficher une page qui vous permet de fournir des paramètres pour tester les méthodes du service Web.

    Le lien Testeur s’affiche uniquement si le type de déploiement de l’application n'est pas RI (par exemple, s’il est 109). La page Testeur s'affiche dans un nouvel onglet ou une nouvelle fenêtre de navigateur.

  5. Cliquez sur le lien WSDL pour afficher le fichier Web Services Definition Language (WSDL) pour le point d’extrémité.

    Le fichier WSDL s’ouvre dans un nouvel onglet ou une nouvelle fenêtre du navigateur.

    Notez que certains navigateurs Web tentent d'interpréter la source WSDL lorsque vous cliquez sur le lien WSDL, affichant ainsi une page vierge. Pour de tels navigateurs, utilisez la commande d'affichage de la source du navigateur sur la page vierge pour afficher la source WSDL.

  6. Cliquez sur le lien Nom de l’application ou sur le bouton Précédent pour revenir à la page Modification d’une application.
