Paramètres du journaliseur

La page Paramètres du journaliseur permet de configurer la journalisation pour l'instance ou le cluster GlassFish Server.

Elle contient les options suivantes :

Nom de configuration

Nom de la configuration à laquelle s'appliquent les paramètres de cette page. Ce champ est en lecture seule.

Ecrire dans le journal système

Si cette option est sélectionnée, le résultat de la journalisation est envoyé à l'utilitaire syslog en plus du journal du serveur. Cette fonctionnalité est disponible sur les systèmes Solaris et Linux uniquement. Cette option est désactivée par défaut.

Fichier journal

Autre nom ou emplacement pour le fichier du journal du serveur. L'emplacement par défaut est domain-dir/logs/server.log.

Gestionnaire du journal

Nom de classe absolu d'un gestionnaire de journal personnalisé. Un gestionnaire de journal personnalisé permet d'envoyer des journaux vers une destination autre que server.log ou syslog. Il doit venir en extension de la classe java.util.logging.Handler (API compatible JSR 047). Ajoutez la classe du gestionnaire au classpath de GlassFish Server de façon à installer le gestionnaire au démarrage du serveur.

Limite de rotation des fichiers

Taille en octets que le fichier journal du serveur doit atteindre avant la rotation. Lorsque le journal du serveur atteint la taille indiquée, GlassFish Server crée un fichier vide nommé server.log et renomme l'ancien fichier server.log_date, où date indique la date et l'heure de rotation du fichier. La valeur par défaut est 2 Mo. La valeur minimale est 500 ko. Si vous spécifiez une valeur inférieure, le système effectuera la rotation à 500 ko. Pour désactiver la rotation du fichier journal, spécifiez la valeur 0.

Temps limite de rotation des fichiers

Nombre de minutes au bout duquel la rotation du journal du serveur doit être effectuée. La valeur par défaut est 0 ; cela signifie que le système effectuera la rotation lorsque le fichier aura atteint la taille spécifiée dans le champ Limite de rotation des fichiers. Si vous spécifiez une ou plusieurs minutes, le temps limite remplace la taille limite indiquée par le champ Limite de rotation des fichiers.

Fréquence de vidage

Nombre maximal de messages à écrire simultanément depuis la file d'attente sur le journal du serveur. La valeur par défaut est 1.
