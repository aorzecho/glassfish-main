Applications

La page Applications présente la liste des applications déployées sur le GlassFish Server. Vous pouvez afficher et gérer les applications qui sont déjà déployées et vous pouvez déployer d’autres applications.

Pour chaque application, les informations suivantes sont fournies :

Nom

Nom de l’application.

Le nom peut être suivi d'un identificateur de version facultatif dont il est séparé par un signe deux-points (:). Pour plus d'informations sur le module et les versions d'application, reportez-vous à la section Module and Application Versions du Oracle GlassFish Server 3.1 Application Deployment Guide.

Activé ou état

Coche si l’application est activée ou X si l’application est désactivée, si seule l'instance de serveur par défaut server existe.

Si des clusters ou d'autres instances de serveur autonomes existent, sélectionnez Activé sur les cibles pour afficher les cibles sur lesquelles l'application est déployée.

Moteurs

Types de conteneurs utilisés par l’application. Voici les différents types de conteneurs :

  • web

  • webservices

  • ejb

  • connector

  • appclient

  • weld (le conteneur Contextes et Injection de dépendance des applications de la plate-forme Java EE)

Action

Liens vers les actions que vous pouvez exécuter sur le composant après son déploiement : Redéployer et Recharger pour tous les composants et Lancer pour les applications Web.

Le tableau Applications déployées contient également les options suivantes.

Déployer

Bouton permettant de déployer une application.

Annuler le déploiement

Bouton permettant d’annuler le déploiement d’une ou de plusieurs applications sélectionnées.

Activer

Bouton permettant d'activer une ou plusieurs applications sélectionnées, si seule l'instance de serveur par défaut, server, existe.

Désactiver

Bouton permettant de désactiver une ou plusieurs applications sélectionnées, si seule l'instance de serveur par défaut, server, existe.

Filtre

Liste déroulante qui filtre les applications par moteur.
