Fournisseurs JACC

Utilisez la page Fournisseurs JACC pour configurer les fournisseurs JACC (Java Authorization Contract for Containers) pour le cluster ou l'instance GlassFish Server sélectionné.

Les fournisseurs JACC peuvent être utilisés par des modules d'autorisation tiers pour des applications s'exécutant sur GlassFish Server. L'installation GlassFish Server par défaut inclut deux fournisseurs JACC nommés default et simple . Ces fournisseurs par défaut mettent en œuvre un moteur d'autorisation simple, basé sur des fichiers, et conforme à la spécification JACC. Vous pouvez également créer vos propres fournisseurs JACC personnalisés Reportez-vous à la section Pour créer un fournisseur JACC.

Pour en savoir plus sur l’écriture de classes d’implémentation pour un fournisseur JACC, consultez la spécification JACC sur http://jcp.org/en/jsr/detail?id=115.

Pour chaque fournisseur JACC, les informations suivantes sont fournies.

Nom

Nom du fournisseur JACC. Ce nom est utilisé pour référencer l'élément jacc-provider dans domaine.xml.

Configuration de stratégies

Nom complet du javax.security.jacc.PolicyConfigurationFactory.provider mettant en œuvre le javax.security.jacc.PolicyConfigurationFactory spécifique au fournisseur.

Fournisseur de stratégies

Nom complet du javax.security.jacc.policy.provider mettant en œuvre le java.security.Policy.

Les fournisseurs JACC par défaut prennent également en charge la propriété suivante.

Référentiel

Répertoire contenant le fichier de stratégie JACC. Pour le fournisseur JACC GlassFish Server default , le répertoire par défaut est ${com.sun.aas.instanceRoot}/generated/policy. Cette propriété n'est pas définie par défaut pour le fournisseur JACC GlassFish Server simple.

Le tableau Fournisseurs JACC contient également les options suivantes.

Nouveau

Bouton permettant de créer un fournisseur JACC.

Supprimer

Bouton permettant de supprimer un ou plusieurs fournisseurs JACC sélectionnés.
