Informations relatives à une adresse de service Web

La page Informations relatives à une adresse de service Web permet d'afficher les informations sur les adresses des services Web.

Elle contient les informations suivantes :

Nom de l'application

Nom de l'application déployée. Cliquez sur ce lien ou sur le bouton Précédent pour revenir à la page Modifier une application.

Le nom peut être suivi d'un identificateur de version facultatif dont il est séparé par un signe deux-points (:). Pour plus d'informations sur les versions des modules et des applications, reportez-vous à "Module and Application Versions" dans le manuel Oracle GlassFish Server Application Deployment Guide.

Testeur

Le lien Testeur apparaît uniquement si le type de déploiement de l'application n'est pas RI (par exemple, si c'est 109). Cliquez sur ce lien afin d'afficher une page qui permet de fournir des paramètres pour tester les méthodes du service Web.

WSDL

Langage WSDL (Web Services Definition Language) pour l'adresse. Cliquez sur ce lien pour afficher le fichier WSDL dans un nouvel onglet ou dans une nouvelle fenêtre du navigateur.

Notez que certains navigateurs Web tentent d'interpréter la source WSDL lorsque vous cliquez sur le lien WSDL, affichant ainsi une page vierge. Pour de tels navigateurs, utilisez la commande d'affichage de la source du navigateur sur la page vierge pour afficher la source WSDL.

Nom de l'adresse.

Nom du service Web.

Nom du port

Nom du port de service Web.

Type de déploiement

Type de déploiement du service Web (par exemple, RI ou 109).

Type d'implémentation

Type d'implémentation du service Web (par exemple, SERVLET ou EJB).

Nom de classe d'implémentation

Nom de classe d'implémentation du service Web.

Espace de noms

Description

Description de l'adresse, le cas échéant.
