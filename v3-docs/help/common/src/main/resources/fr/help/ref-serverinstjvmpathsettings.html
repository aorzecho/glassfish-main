Paramètres du chemin JVM

Utilisez la page Paramètres du chemin JVM pour configurer le chemin d’accès à la bibliothèque native. Le chemin de la bibliothèque native est une concaténation des chemins suivants :

  • Préfixe du chemin d’accès à la bibliothèque native

  • Chemin d’accès du serveur à ses bibliothèques natives (c’est-à-dire, as-install/lib)

  • Répertoires de bibliothèque native 64 bits, le cas échéant

  • Valeur de la variable java.library.path pour la JVM utilisée pour démarrer GlassFish Server (ceci diffère selon les systèmes d’exploitation)

  • Tous les chemins spécifiés sur la page Paramètres du profileur JVM, si le profileur est activé

  • Suffixe du chemin d’accès à la bibliothèque native

D’autres paramètres sont affichés comme des valeurs en lecture seule pour les domaines qui ont été mis à niveau à partir d’une version précédente de GlassFish Server. Ils ne sont pas pris en charge dans la version v3.

La page Paramètres du chemin JVM contient les options suivantes.

Nom de configuration

Nom de la configuration à laquelle les paramètres de cette page s'appliquent. Ce champ est en lecture seule.

Classpath de l’environnement

Si la case à cocher Ignorer est activée, la variable d’environnement CLASSPATH est ignorée. La variable d’environnement CLASSPATH est utile pour les didacticiels de base relatifs à la programmation, mais déconseillée pour les environnements d’entreprise. Ce champ est en lecture seule.

Classpath du système

Chemin des classes sur votre système. Ce champ est en lecture seule.

Préfixe du chemin de classe

Chemins complets des fichiers JAR à ajouter au début du classpath du serveur, un par ligne. Ce champ est en lecture seule.

Suffixe du chemin de classe

Chemins complets des fichiers JAR à ajouter à la fin du classpath du serveur, un par ligne. Ce champ est en lecture seule.

Préfixe du chemin de bibliothèque native

Chemins complets des fichiers JAR à ajouter au préfixe du chemin de bibliothèque native, un par ligne.

Suffixe du chemin de bibliothèque native

Chemins complets des fichiers JAR à ajouter au chemin de bibliothèque native, un par ligne.
