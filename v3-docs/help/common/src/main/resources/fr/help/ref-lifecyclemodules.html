Modules de cycle de vie

Utilisez la page Modules de cycle de vie pour configurer les modules de cycle de vie.

Un module de cycle de vie accomplit certaines tâches lorsqu’il est déclenché par un ou plusieurs événements dans le cycle de vie du serveur. Les événements de serveur qui peuvent être déclencheurs sont : l’initialisation, le démarrage, les requêtes prêt pour la maintenance et l’arrêt. Les modules de cycle de vie ne font pas partie de la spécification Java EE, mais optimisent le fonctionnement du GlassFish Server.

Pour chaque module de cycle de vie, les informations suivantes sont fournies.

Nom

Nom du module de cycle de vie.

Activé ou État

Coche si l’application est activée ou X si l’application est désactivée, si seule l'instance de serveur par défaut server existe.

Si des clusters ou d'autres instances de serveur autonomes existent, sélectionnez Activé sur toutes les cibles pour afficher les cibles sur lesquelles l'application est déployée.

Ordre de chargement

Valeur indiquant l’ordre relatif dans lequel le module de cycle de vie doit être chargé.

Le tableau Modules de cycle de vie contient également les options suivantes.

Nouveau

Bouton permettant de créer un nouveau module de cycle de vie.

Supprimer

Bouton permettant de supprimer un ou plusieurs modules de cycle de vie sélectionnés.

Activer

Bouton permettant d’activer un ou plusieurs modules de cycle de vie sélectionnés. Présent si seule l'instance de serveur par défaut, server, existe.

Désactiver

Bouton permettant de désactiver un ou plusieurs modules de cycle de vie sélectionnés. Présent si seule l'instance de serveur par défaut, server, existe.
