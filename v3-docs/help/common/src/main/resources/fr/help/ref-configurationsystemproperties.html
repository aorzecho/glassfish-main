Propriétés système

La page Propriétés système affiche la liste des propriétés système Java définies pour la configuration nommée sélectionnée. Les propriétés système Java sont transmises au lanceur d'application Java par l'intermédiaire de l'option -D du lanceur d'application Java au démarrage de GlassFish Server.

Ces propriétés définissent les valeurs par défaut des paramètres de port pour toutes les instances GlassFish Server faisant référence à la configuration.

Pour plus d'informations sur les paramètres de port prédéfinis dans une , reportez-vous à la section Propriétés système prédéfinies .

Pour chaque propriété, les informations suivantes sont affichées :

Nom de la variable d’instance

Nom de la propriété système.

Valeur par défaut

Valeur définie pour la propriété dans la configuration nommée à laquelle l'instance fait référence. Ce champ est en lecture seule.

Valeurs de l’instance

Lien vers la page Valeurs de l'instance pour la propriété.

Le tableau Propriétés supplémentaires contient également les options suivantes.

Ajouter une propriété

Bouton permettant d'ajouter une propriété. Le fait de cliquer sur ce bouton permet d'ajouter une ligne au tableau Propriétés supplémentaires.

Supprimer des propriétés

Bouton permettant de supprimer une ou plusieurs propriétés sélectionnées. Toute propriété supprimée revient à sa valeur par défaut ou, si aucune valeur par défaut n'est définie, n'est pas définie.

La page Propriétés système contient également les options suivantes :

Nom de configuration

Nom de la configuration à laquelle les paramètres de cette page s'appliquent. Ce champ est en lecture seule.

Reconfiguration dynamique

Si cette option est activée, les modifications apportées à la configuration sont appliquées aux instances faisant référence à la configuration sans nécessiter le redémarrage du serveur.

Clusters et/ou instances utilisant cette configuration

Liste des clusters et instances faisant référence à la configuration avec une indication de si chaque cluster ou instance est en cours d'exécution :

  • Si le cluster ou l'instance est en cours d'exécution, une coche (✓) s'affiche.

  • Si le cluster ou l'instance est arrêté, un point d'exclamation (!) s'affiche.
