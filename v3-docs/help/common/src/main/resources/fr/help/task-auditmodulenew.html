Créer un module d'audit

Les modules d'audit contrôlent la manière dont le GlassFish Server consigne les événements importants, comme les erreurs et les brèches de sécurité. GlassFish Server consigne tous les événements d'authentification dans des fichiers journaux. Un journal d'accès complet fournit un suivi chronologique des événements d'accès dans GlassFish Server.

GlassFish Server fournit un simple module d'audit par défaut. Pour plus d'informations, reportez-vous à Utiliser le module d'audit par défaut.

  1. Dans l'arborescence de navigation, développez le noeud Configuration.

  2. Select the instance to configure:

    • To configure a particular instance, expand the instance's config node. For example, the default instance, server, expand the server-config node.

    • To configure the default settings for all instances, expand the default-config node.

  3. Sous le noeud Configuration, développez le noeud Sécurité.

  4. Sous le noeud Sécurité, sélectionnez le noeud Modules d'audit.

    La page Modules d'audit apparaît.

  5. Sur la page Modules d'audit, cliquez sur Nouveau.

    La page Nouveau module d'audit apparaît.

  6. Sur la page Nouveau module d'audit, dans le champ Nom, saisissez le nom du module d'audit.

  7. Dans le champ Nom de la classe, saisissez le nom complet de la classe qui implémente ce module.

    Le nom de classe du module d'audit par défaut est com.sun.enterprise.security.Audit.

  8. Dans la section Autres propriétés, spécifiez les propriétés supplémentaires.

    Pour ajouter une propriété, cliquez sur le bouton Ajouter une propriété. Sur la ligne vide qui apparaît, saisissez le nom de la propriété dans le champ Nom et la valeur de la propriété dans le champ Valeur.

    Pour une description de la propriété supplémentaire disponible dans les modules d'audit, reportez-vous à Propriétés propres aux modules d'audit.

  9. Cliquez sur OK.
