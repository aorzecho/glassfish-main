Modifier une application déployée

  1. Dans l'arborescence de navigation, sélectionnez le noeud Applications.

    La page Applications apparaît.

  2. Dans la table Applications déployées, cliquez sur le nom de l'application à modifier.

    Le nom peut être suivi d'un identificateur de version facultatif dont il est séparé par un signe deux-points (:). Pour plus d'informations sur les versions des modules et des applications, reportez-vous à "Module and Application Versions" dans le manuel Oracle GlassFish Server Application Deployment Guide.

    La page Modifier une application apparaît.

  3. Désélectionnez la case Statut activé pour désactiver le statut de l'application et la rendre indisponible.

    Par défaut, le statut d'une application est Activé dès son déploiement.

  4. Pour une application Web, dans le champ Racine du contexte, saisissez le chemin de l'application.

    Dans l'URL de l'application Web, la racine de contexte suit immédiatement le numéro de port (http://host:port/context-root/...). Assurez-vous que la racine du contexte commence par une barre oblique (par exemple, /hello).

  5. Désélectionnez la case Disponibilité activée afin de désactiver la persistance de session et la réalisation de points de contrôle SFSB pour l'application.

    Si la case Activé est cochée, la haute disponibilité est activée pour la réalisation des points de contrôle et, éventuellement, la mise en mode passif des sessions Web et des beans Session avec conservation de statut (SFSB). Si la valeur est False (valeur par défaut), tous les enregistrements de session Web et la réalisation des points de contrôle SFSB sont désactivés pour l'application, l'application Web ou le module EJB spécifié. Si cette valeur est True, l'application ou le module spécifié est activé pour la haute disponibilité. Définissez cette option sur True seulement si la haute disponibilité est configurée et activée à des niveaux supérieurs, tels que les niveaux du serveur et du conteneur.

    Cette option apparaît si des clusters ou des instances de serveur autonomes existent en plus de l'instance de serveur par défaut (server).

  6. Dans la liste déroulante Serveurs virtuels, sélectionnez le serveur virtuel à associer à cette application.

    L'option Serveurs virtuels apparaît si seule l'instance de serveur par défaut, server, existe. S'il existe des clusters ou d'autres instances de serveur autonomes, vous pouvez sélectionner des serveurs virtuels après le déploiement. Ouvrez la page Modifier une application, sélectionnez l'onglet Cible, puis Gérer les serveurs virtuels pour la cible de votre choix.

  7. Cochez la case Java Web Start activé pour déployer l'application à l'aide du logiciel Java Web Start.

    Cette option est activée ou désactivée par défaut en fonction de son paramètre lors du déploiement initial de l'application.

  8. Dans le champ Description, entrez une brève description de cette application.

  9. Pour afficher la liste des descripteurs de module de l'application, cliquez sur l'onglet Description. Pour plus d'informations, reportez-vous à Afficher les descripteurs d'une application.

  10. Par défaut, le statut d'une application est Activé sur toutes les cibles. Pour afficher les cibles sur lesquelles l'application est activée, cliquez sur l'onglet Cibles. Vous pouvez activer ou désactiver les applications sur une ou plusieurs cibles. Reportez-vous à Afficher les cibles de l'application.

  11. Cliquez sur Enregistrer.
