Modification d’une application déployée

  1. Dans l’arborescence de navigation, sélectionnez le nœud Applications.

    La page Applications s’affiche.

  2. Dans le tableau Applications déployées, cliquez sur le nom de l’application à modifier.

    Le nom peut être suivi d'un identificateur de version facultatif dont il est séparé par un signe deux-points (:). Pour plus d'informations sur le module et les versions d'application, reportez-vous à la section Module and Application Versions du Oracle GlassFish Server 3.1 Application Deployment Guide.

    La page Modification de l’application s’affiche.

  3. Décochez la case État activé pour désactiver l’état de l’application afin de la rendre indisponible.

    Par défaut, l’état d’une application est Activé dès son déploiement.

  4. Pour une application Web, dans le champ Racine contexte, saisissez le chemin de l’application.

    Dans l’URL de l’application Web, la racine contexte suit immédiatement le numéro de port (http://host: port/context-root/...). Assurez-vous que la racine contexte commence par une barre oblique, comme par exemple /hello.

  5. Désélectionnez la case Disponibilité activée pour désactiver la persistance de session et la réalisation de points de contrôle SFSB pour l'application.

    Si la case Activé est cochée, la haute disponibilité est activée pour la réalisation des points de contrôle et, éventuellement, la mise en mode passif des sessions web et des beans de session avec état (SFSB). Si elle est définie sur false (valeur par défaut), tous les enregistrements de session web et la réalisation des points de contrôle SFSB sont désactivés pour l'application, l'application Web ou le module EJB spécifié. Si cette valeur est définie sur true, l'application ou le module spécifié est activé pour la haute disponibilité. Définissez cette option sur true seulement si la haute disponibilité est configurée et activée à des niveaux supérieurs, tels que les niveaux du serveur et du conteneur.

    Cette option s'affiche si des clusters ou des instances de serveur autonomes existent en plus de l'instance de serveur par défaut (server).

  6. Dans la liste déroulante Serveurs virtuels, sélectionnez le serveur virtuel à associer à cette application.

    L'option Serveurs virtuels s'affiche si seule l'instance de serveur par défaut, server, existe. Si des clusters ou d'autres instances de serveur autonomes existent, vous pouvez sélectionner des serveurs virtuels après le déploiement. Ouvrez la page Modifier l'application, sélectionnez l'onglet Cible, puis Gestion des serveurs virtuels pour la cible de votre choix.

  7. Cochez la case Java Web Start Enabled (Java Web Start activé) pour utiliser le logiciel Java Web Start afin de déployer l’application.

    Cette option est activée ou désactivée par défaut en fonction de son paramètre lorsque l'application a été initialement déployée.

  8. Dans le champ Description, entrez une brève description de cette application.
  9. Pour afficher la liste des descripteurs de module pour l’application, cliquez sur l’onglet Description. Pour plus de détails, reportez-vous à la section Affichage des descripteurs d'une application.
  10. Par défaut, l’état d’une application est Activé sur toutes les cibles. Pour afficher les cibles sur lesquelles l’application a été activée, cliquez sur l’onglet Cibles. Vous pouvez activer ou désactiver les applications sur une ou plusieurs cibles. Reportez-vous à la section Afficher les cibles de l'application.
  11. Cliquez sur Enregistrer.
