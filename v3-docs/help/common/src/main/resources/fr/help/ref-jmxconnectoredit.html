Modification d’un connecteur JMX

Utilisez la page Modifier le connecteur JMX pour modifier le connecteur JMX conforme au JSR 360.

La page Modifier le connecteur JMX contient les options suivantes.

Nom de la configuration

Nom de la configuration à laquelle les paramètres de cette page s'appliquent. Ce champ est en lecture seule.

Nom

Nom du connecteur JMX. Ce champ est en lecture seule.

Protocole JMX

Nom du protocole que ce connecteur JMX prend en charge. Ce champ est en lecture seule.

JMXServiceURL

URL du service JMX. Ce champ est en lecture seule.

Sécurité

Si la case Activé est cochée, la communication JMX est chiffrée. Cette option est désactivée par défaut.

Adresse

Adresse IP du service de nommage où le stub de serveurs de connecteurs JMX est enregistré. Ce n’est pas le port du socket de serveur qui effectue la véritable communication JMX. C’est l’adresse de l’interface réseau depuis laquelle le registre RMI est lancé. Si votre système est doté de plusieurs interfaces réseau, modifiez cette valeur pour qu’une seule interface soit sélectionnée. La valeur par défaut est 0.0.0.0.

Port

Numéro du port sur lequel le service de nommage (registre RMI) écoute les connexions client RMI. La seule utilisation de ce service de nommage consiste à télécharger les stubs RMI. Si le port par défaut est occupé, on utilise un port libre. Les valeurs légales sont de 1 à 65535. Sous les systèmes UNIX, la création de sockets qui écoutent sur les ports 1 à 1024 nécessite des privilèges de superutilisateur. La valeur par défaut est 8686.

Nom de domaine

Nom qui représente le domaine administratif spécial. Toutes les authentifications sont gérées par ce domaine. La valeur par défaut est admin-realm .

Autres propriétés

Autres propriétés du connecteur JMX. Plusieurs propriétés sont indiquées par défaut.

Tâche connexe

  • Pour modifier le connecteur JMX

  • Pour modifier les paramètres SSL du connecteur JMX

Commandes asadmin connexes

Aucune.
