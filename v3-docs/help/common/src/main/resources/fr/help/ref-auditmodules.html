Modules d’audit

Utilisez la page Modules d’audit pour configurer les modules d’audit.

Les modules d’audit contrôlent la manière dont le GlassFish Server consigne les événements importants, comme les erreurs et les brèches de sécurité. Le GlassFish Server consigne tous les événements d’authentification dans des fichiers journaux. Un journal d’accès fournit un suivi séquentiel des événements d’accès du GlassFish Server.

Pour chaque module d’audit, les informations suivantes sont fournies.

Nom

Nom du module d’audit.

Nom de classe

Nom complet de la classe qui implémente ce module. Le nom de classe du module d’audit par défaut est com.sun.enterprise.security.Audit.

Le tableau Modules contient également les options suivantes.

Nouveau

Bouton permettant de créer un module d’audit.

Supprimer

Bouton permettant de supprimer un ou plusieurs modules d’audit sélectionnés.

Commandes asadmin connexes

  • create-audit-module(1)

  • list-audit-modules(1)

  • delete-audit-module(1)
