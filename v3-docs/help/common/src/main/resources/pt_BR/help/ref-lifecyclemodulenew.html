Novo Módulo do Ciclo de Vida

Use a página Novo Módulo do Ciclo de Vida para definir um novo módulo do ciclo de vida.

A página Novo Módulo do Ciclo de Vida contém estas opções.

Nome

O nome do módulo do ciclo de vida.

Nome da Classe

O nome totalmente qualificado do arquivo de classes do módulo do ciclo de vida. A classe deve implementar a interface com.sun.appserv.server.LifecycleListener.

Caminho da classe

O caminho da classe do módulo do ciclo de vida. O caminho da classe especifica onde o módulo do ciclo de vida está localizado. O valor default é domain-dir/applications. Se o módulo já estiver neste diretório (isto é, no caminho da classe do servidor), esse campo poderá ser deixado em branco.

Ordem de Carregamento

A ordem na qual este módulo do ciclo de vida deve ser carregado na inicialização. Os módulos com valores inteiros menores de ordem de carregamento são carregados antes. O valor pode variar de 101 até o MAXINT do sistema operacional. Valores de 1 a 100 são reservados.

Descrição

Uma descrição do módulo do ciclo de vida.

Status

Se esta opção for selecionada, o módulo do ciclo de vida será ativado. Por default, a opção está ativada.

Em caso de falha do carregamento

Se esta opção estiver selecionada, o servidor será submetido a shutdown se o módulo do ciclo de vida falhar ao ser carregado. Por default, essa opção está desativada.

Destinos

Clusters e instâncias standalone do módulo do ciclo de vida. Mova os destinos desejados para a coluna Destinos selecionados utilizando os botões Adicionar e Adicionar tudo. Mova os destinos desnecessários para a coluna Destinos disponíveis utilizando os botões Remover e Remover tudo.

Esta opção é exibida somente se os clusters ou instâncias standalone tiverem sido criados no domínio.
