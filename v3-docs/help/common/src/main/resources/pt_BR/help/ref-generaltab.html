Informação Geral

The GlassFish Server creates one application server instance, called server at the time of installation. You can delete the server instance and create a new instance with a different name if you prefer.

Each GlassFish Server instance has its own Java configuration, Java resources, application deployment areas, and server configuration settings. Changes to one application server instance have no effect on other application server instances. You can have one application server instance within one administrative domain.

For many users, one application server instance meets their needs. However, depending upon your environment, you might want to create one or more additional application server instances. For example, in a development environment you can use different application server instances to test different GlassFish Server configurations, or to compare and test different application deployments. Because you can easily add or delete an application server instance, you can use them to create temporary "sandbox" areas to experiment with while developing.

In addition, for each application server instance you can also create virtual servers. Within a single installed application server instance you can offer companies or individuals domain names, IP Addresses, and some administration capabilities. For the users, it is almost as if they have their own web server, without the hardware and basic server maintenance. These virtual servers do not span application server instances. For more information about virtual servers, see Definições Gerais de JVM.

In operational deployments, for many purposes you can use virtual servers instead of multiple application server instances. However, if virtual servers do not meet your needs, you can also use multiple application server instances.

An GlassFish Server instance is not started automatically. Once you start an instance, the instance runs until you stop it. When you stop an application server instance, it stops accepting new connections, then waits for all outstanding connections to complete. If your machine crashes or is taken offline, the server quits and any requests it was servicing may be lost.

Use a página Informações Gerais para verificar as definições do GlassFish Server e para exibir os dados do Java Virtual Machine.

A página Informações Gerais contém as seguintes informações.

Botão Parar

Clique no botão Parar para interromper o GlassFish Server.

Botão Reiniciar

Clique no botão Reiniciar para iniciar novamente o GlassFish Server.

Botão Exibir arquivos de log.

Clique no botão Visualizar Arquivos de Log para visualizar arquivos de log para um GlassFish Server instância ou cluster.

Visualizar Botão de Log Bruto

Clique no botão Visualizar Log Bruto para visualizar arquivos de log bruto para o Admin Server (denominadoservidor).

Botão de Administração Segura

Clique no botão de Administração Segura para ativar ou desativar a administração segura na página Administração Segura.

Nome

O nome do current servidor.

Status

O status atual da instância do servidor. O servidor pode estar interrompido, iniciado ou em execução.

Tempo de funcionamento

O número de horas e minutos que a instância do servidor está em execução continuamente. Esse é somente para leitura.

HTTP Load Balancer

The name of the load balancer, if it is enabled for the server instance.

JVM

Se você clicar em JVM Report, será aberta uma janela separada que exibe relatórios sobre o Java Virtual Machine, incluindo um relatório de resumo, um relatório de gerenciamento de memória e de coleta de erros,um relatório de carregamento de classe e um dump de threads atuais.

Diretório de Configuração

O diretório localizado no computador host, que contém arquivos de configuração; por exemplo, domain.xml.

Versão Instalada

O identificador de versão do produto.

Administração Segura

O status atual da administração segura. Valores possíveis são Ativado ou Não-Ativado

Depuração

Identifica se a depuração está ativada. Se a depuração estiver ativada, o número de porta também será exibido.

Porta(s) HTTP

As portas configuradas atualmente para solicitações e respostas HTTP.

Porta(s) IIOP

As portas configuradas atualmente para solicitações e respostas IIOP.
