Definições de Caminho de JVM

Use a página de definições do Caminho JVM para configurar o caminho da biblioteca nativa. O caminho da biblioteca nativa é uma concatenação dos seguintes caminhos:

  • O prefixo do caminho da biblioteca nativa

  • O caminho do servidor para suas bibliotecas nativas (ou seja, as-install/lib)

  • Os diretórios da biblioteca nativa de 64-bit, se for aplicável

  • O valor da variável java.library.path para a JVM, usada para iniciar o GlassFish Server (isso é diferente entre os sistemas operacionais)

  • Qualquer caminho especificado na Definições de Caminho de JVM página, se o gerador de perfis estiver ativado

  • O sufixo do caminho da biblioteca nativa

Outras definições são mostradas como valores somente para leitura para domínios que tiveram upgrade de uma versão anterior do GlassFish Server. Elas não são suportadas na versão v3.

A página Definições do Caminho JVM contém as opções a seguir.

Nome da Configuração

O nome da configuração à qual as definições nesta página se aplicam. Este campo é somente para leitura.

Caminho de Classe de Ambiente

Se a caixa de seleção Ignorar estiver marcada, a variável de ambiente CAMINHO DE CLASSE será ignorada. A variável de ambiente CAMINHO DE CLASSE é conveniente para tutoriais básicos de programação, mas não é recomendada para ambientes empresariais. Este é um campo somente para leitura.

Caminho de Classe do Sistema

O caminho para as classes no seu sistema. Este é um campo somente leitura.

Prefixo de Caminho de Classe

Caminhos completos para os arquivos JAR a serem colocados no início do caminho de classe do servidor, um por linha. Este é um campo somente leitura.

Sufixo de Caminho de Classe

Caminhos completos para os arquivos JAR a serem acrescentados ao caminho de classe do servidor, um por linha. Este é um campo somente leitura.

Prefixo de Caminho da Biblioteca Nativa

Caminhos completos para os arquivos JAR a serem colocados no início do caminho da biblioteca nativa, um por linha.

Sufixo de Caminho da Biblioteca Nativa

Caminhos completos para os arquivos JAR a serem acrescentados ao caminho da biblioteca nativa, um por linha.
