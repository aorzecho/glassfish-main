Para Usar o Módulo de Auditoria Default

O módulo de auditoria default registra as solicitações de autenticação e de autorização no arquivo de log do servidor. Para obter informações sobre a alteração da localização do arquivo de log, consulte Para Configurar Definições de Log Gerais.

As entradas de log de autenticação incluem estas informações:

  • Nomes do usuários que tentaram se autenticar

  • O realm que processou a solicitação de acesso

  • Componente URI ou EJB do módulo Web solicitado

  • Êxito ou falha da solicitação

Independentemente de o modo de log de auditoria estar ou não ativado, o GlassFish Server registra em log todos os eventos de autenticação recusados.

As entradas de log de autorização incluem estas informações:

  • Nomes de usuários autenticados, se houver

  • Componente URI ou EJB da Web solicitada

  • Êxito ou falha das solicitações

  1. Na árvore de navegação, expanda o nó Configuração.

  2. Select the instance to configure:

    • To configure a particular instance, expand the instance's config node. For example, the default instance, server, expand the server-config node.

    • To configure the default settings for all instances, expand the default-config node.

  3. Sob o nó Configuração, selecione o nó Segurança.

    A página Segurança é exibida.

  4. Na lista drop-down Módulos de Auditoria, selecione o módulo de auditoria denominado default.

  5. Clique em Salvar.
