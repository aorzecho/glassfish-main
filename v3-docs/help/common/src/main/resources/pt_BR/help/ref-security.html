Segurança

Utilize a página Segurança para definir as propriedades de segurança do cluster ou instância selecionada do &ProductName.

A página Segurança contém estas opções.

Nome da Configuração

O nome da configuração à qual as definições nesta página se aplicam. Este campo é somente para leitura.

Gerenciador de Segurança

Se esta opção for selecionada, o gerenciador de segurança do domínio será ativado. Por default, essa opção está desativada.

Quando essa opção for ativada, a opção JVM, -Djava.security.manager, será adicionada à definição JVM do GlassFish Server. Consulte Para Configurar as Oções da JVM. É preciso reiniciar o servidor para ativar a alteração.

Verifique se você tem as permissões corretas para todas as aplicações. Para melhorar o desempenho, desative o gerenciador de segurança.

Log de Auditoria

Se esta opção for ativada, o servidor carregará e executará todos os módulos de auditoria especificados na definição Módulos de Auditoria. Se a opção for desativada, o servidor não acessará os módulos de auditoria. Por default, essa opção está desativada.

Realm Default

O realm (default) ativo que o servidor usa para autenticação. As aplicações usam esse realm, a menos que o descritor de implantação especifique outro realm. Todos os realms configurados são exibidos na lista. O valor default é arquivo.

Principal Default

Especifica o nome de usuário default. O servidor usa esse nome de usuário quando nenhum outro principal é fornecido. Se você digitar um valor nesse campo, digite a senha correspondente no campo Senha do Principal Default.

Um principal default não é exigido para operação normal do servidor.

Senha do Principal Default

A senha do principal default especificada no campo Principal Default. Um principal default não é exigido para operação normal do servidor.

JACC

O nome da classe de um provedor de JACC configurado. O valor default é default.

Módulos de Auditoria

Os módulos do provedor de auditoria que serão usados pelo subsistema de auditoria se o log de auditoria estiver ativado. Por default, o servidor usa um módulo de auditoria chamado default. Para obter informações sobre como criar novos módulos de auditoria, consulte Para Criar um Módulo de Auditoria.

Mapeamento Principal Default para Atribuição Ativado

Se esta opção estiver selecionada, o mapeamento principal-para-atribuição será usado com as aplicações que não tiverem um mapeamento específico para aplicação.

Classe de Principal Mapeada

Especifica uma implementação personalizada da classe java.security.Principal usada no mapeamento principal-to-role default, se o mapeamento principal-to-role default estiver ativado.
