Definições Gerais de JVM

O Java Virtual Machine (JVM) possibilita que códigos de bytes de Java seja executados em uma máquina física. O ajuste das definições de JVM melhora o desempenho e evita erros de alocação de memória.

O Java Virtual Machine está incluído na Java Platform, Standard Edition (plataforma Java SE), que é exigida pelo GlassFish Server. Definições de JVM incorretas impedirão que o servidor seja executado, portanto, tenha cuidado ao alterar essas configurações.

Observação:

Os termos "Java Virtual Machine" e "JVM" significam uma Máquina Virtual para a plataforma Java.

A página Definições Gerais de JVM contém as seguintes opções.

Carregar Defaults

Botão para restaurar com os valores default as definições que apresentam valores default. As definições que não apresentam valores default não são alteradas.

Nome da Configuração

O nome da configuração à qual as definições nesta página se aplicam. Este campo é somente para leitura.

Java Home

O nome do diretório de instalação do software Java SE, que é exigido ára que &ProductName seja executado. Para verificar se a versão do software Java SE especificado é suportada nesta release, consulte o Notas de Release do Oracle GlassFish Server. Se você especificar um nome de diretório inexistente ou se a versão do diretório não for compatível com o software Java SE, o GlassFish Server não será iniciado.

Opções Javac

As opções de linha de comando do compilador da linguagem de programação Java. O GlassFish Server executa o compilador quando componentes Enterprise JavaBeans (EJB) são implantados.

Depuração

Ativa a depuração com o Java Platform Debugger Architecture (JPDA).

O JPDA é utilizado pelos desenvolvedores de aplicações. Para obter mais informações, consulte o Oracle GlassFish Server Application Development Guide.

Opções de Depuração

Especifica as opções do JPDA passadas à JVM quando a depuração está ativada.

Opções de Compilação RMI

As opções de linha de comando do compilador rmic. O GlassFish Server executa o compilador rmic quando os componentes EJB são implantados.

Pré-processador de Código de Bytes

Nomes de classes para pré-processamento de códigos de bytes. Use vírgulas para separar os nomes de classes. Cada classe deve implementar a interface com.sun.appserv.BytecodePreprocessor. As classes são chamadas na ordem especificada.

Propriedades Adicionais

Propriedades adicionais da JVM. O GlassFish Server não define propriedades adicionais para a JVM.
