Para Alterar o Limite de Número de Arquivos de Log Girados

Quando o GlassFish Server gira um arquivo de log, ele cria um arquivo novo e vazio denominado server.log e renomeia o arquivo antigo server.log_date, em que date representa a data e o horário em que o arquivo foi girado.

Por default, o GlassFish Server limita o número de arquivos do log rotacionados para 10. Quando esse limite for alcançado, o arquivo do log mais antigo será deletado da próxima vez em que o GlassFish Server girar um arquivo de log.

Se necessário, é possível alterar o número limite de arquivos do log rotacionados. Os novos limites definidos se aplicam aos arquivos de log rotacionados para o log de acesso e o log do servidor.

  1. Defina a propriedade de sistema com.sun.enterprise.server.logging.max_history_files para o número máximo permitido para a manutenção de arquivos de log girados.

    Para obter informações sobre como definir uma propriedade de sistema, consulte Para Gerenciar Propriedades do Sistema.

    O comportamento da propriedade de sistema the com.sun.enterprise.server.logging.max_history_files é como se segue:

    • Se a propriedade não estiver definida, o GlassFish Server mantém um número máximo de 10 arquivos do log rotacionados.

    • Se a propriedade estiver definida como um número inválido ou nulo, o GlassFish Server mantém um número máximo de 10 arquivos do log rotacionados.

    • Se a propriedade estiver definida como 0, o GlassFish Server não mantém arquivos do log rotacionados.
