Para alterar o número limite de arquivos do log rotacionados

Quando o Servidor GlassFish gira um arquivo do log, ele cria um arquivo novo e vazio denominado server.log e renomeia o arquivo antigo server.log_ date, onde date é a data e a hora em que o arquivo foi rotacionado.

Por padrão, o Servidor GlassFish limita o número de arquivos do log rotacionados para 10. Quando esse limite for alcançado, o arquivo do log mais antigo será excluído da próxima vez em que o Servidor GlassFish girar um arquivo de registro.

Se necessário, é possível alterar o número limite de arquivos do log rotacionados. Os novos limites definidos se aplicam aos arquivos do log rotacionados do registro de acesso e do servidor.

  • Defina a propriedade do sistema com.sun.enterprise.server.logging.max_history_files para o número máximo de arquivos do log a serem mantidos.

    Para obter informações sobre como definir uma propriedade do sistema, consulte Para gerenciar propriedades do sistema.

    O comportamento da propriedade do sistema com.sun.enterprise.server.logging.max_history_files é apresentado a seguir:

    • Se a propriedade não estiver definida, o Servidor GlassFish mantém um número máximo de 10 arquivos do log rotacionados.

    • Se a propriedade estiver definida como um número inválido ou nulo, o Servidor GlassFish mantém um número máximo de 10 arquivos do log rotacionados.

    • Se a propriedade estiver definida como 0, o Servidor GlassFish não mantém arquivos do log rotacionados.
