Configuração de Aplicativos

Use a página Configuração de Aplicativos para definir as propriedades que ajudarão a garantir que as alterações em aplicativos implantados serão detectadas e que as classes modificadas serão recarregadas.

A página Configuração de Aplicativos contém as seguintes opções.

Carregar padrões

Botão para restaurar com os valores padrão as configurações que apresentam valores padrão. As configurações que não apresentam valores padrão não são alteradas.

Recarregar

Se esta opção estiver ativada, as classes modificadas serão recarregadas dinamicamente. O servidor verifica periodicamente se há alterações no diretório de implantação e reimplanta o aplicativo, automática e dinamicamente, com as alterações. Isso é útil em um ambiente de desenvolvimento, pois permite testar rapidamente as alterações de códigos. Em um ambiente de produção, no entanto, o recarregamento dinâmico pode prejudicar o desempenho. Além disso, sempre que ocorre um recarregamento, as sessões nesse tempo de transição se tornam inválidas. O cliente deve reiniciar a sessão. Por padrão, a opção está ativa.

Intervalo de Consulta de Recarregamento

Identifica com que frequência o servidor deve verificar alterações de código nos aplicativos e módulos implantados. O valor padrão é 2.

Tempo-limite de Sessão Admin

Identifica o número de minutos de inatividade que antecede o tempo-limite do Console de Administração, você precisa fazer login novamente. O valor padrão é 60.

Implantação Automática

Se esta opção for ativada, os aplicativos no diretório de implantação automática serão implantados dessa forma. Por padrão, a opção está ativa.

Intervalo de Consulta de Implantação Automática

Especifica com que frequência o servidor deve verificar o diretório de implantação automática quanto a alterações em aplicativos ou módulos. A alteração do intervalo de consulta não afeta o tempo destinado à implantação de um aplicativo ou módulo. O valor padrão é 2.

Tempo-limite de Novas Tentativas de Implantação Automática

Especifica o número de segundos que um arquivo parcialmente copiado pode permanecer inalterado em termos de tamanho antes que ocorra um erro. O valor padrão é 4.

Se um arquivo for copiado lentamente no diretório de implantação automática, esse arquivo poderá ser exibido antes que todo o arquivo seja copiado. Como resultado, a tentativa de implantá-lo automaticamente falhará. Se uma tentativa de implantação automática de um aplicativo falhar por esse motivo, o Servidor GlassFish tentará implantar o arquivo novamente. Se o tamanho do arquivo parcialmente copiado permanecer inalterado durante o período especificado, ocorrerá um erro.

Diretório de Implantação Automática

Identifica o diretório para monitorar a implantação automática de aplicativos. O valor padrão é domain-dir/autodeploy.

Validação do XML

Especifica o tipo de validação do XML a ser realizada nos descritores de implantação padrão e do Servidor GlassFish. O tipo de ser qualquer um dos seguintes:

Completo

Se a validação do XML falhar, a implantação falhará. Esse é o valor padrão.

Analisando

A implantação ocorreu, apesar dos erros de validação do XML relatados.

Nenhum

Não foi realizada nenhuma validação do XML.

Verificador

Se esta opção estiver ativada, o verificador será executado antes da implantação automática. Se a verificação falhar, a implantação não será realizada. O verificador examina a estrutura e o conteúdo do arquivo. A verificação de aplicativos grandes geralmente consome muito tempo. A opção está desativada por padrão.

Pré-compilar

Se esta opção for ativada, os arquivos JavaServer Pages ( JSP) serão pré-compilados. Se esta opção for desativada, os arquivos JSP serão compilados no tempo de execução, quando forem acessados pela primeira vez. A opção está desativada por padrão.

Propriedades adicionais

Propriedades adicionais dos aplicativos. O Servidor GlassFish não define nenhuma propriedade adicional para os aplicativos.
