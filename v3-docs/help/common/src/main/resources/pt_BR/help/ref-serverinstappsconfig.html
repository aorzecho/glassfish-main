Configuração das Aplicações

Use a página Configuração de Aplicações para definir as propriedades que ajudarão a garantir que as alterações em aplicações implantadas serão detectadas e que as classes modificadas serão recarregadas.

A página Configuração de Aplicações contém as opções a seguir.

Carregar Defaults

Botão para restaurar com os valores default as definições que apresentam valores default. As definições que não apresentam valores default não são alteradas.

Recarregar

Se esta opção estiver ativada, as classes modificadas serão recarregadas dinamicamente. O servidor verifica periodicamente se há alterações no diretório de implantação e reimplanta a aplicação, automática e dinamicamente, com as alterações. Isso é útil em um ambiente de desenvolvimento, pois permite testar rapidamente as alterações de códigos. Em um ambiente de produção, no entanto, o recarregamento dinâmico pode prejudicar o desempenho. Além disso, sempre que ocorre um recarregamento, as sessões nesse tempo de transição se tornam inválidas. O cliente deve reiniciar a sessão. Por default, a opção está ativada.

Intervalo de Sondagem para Recarregamento

Identifica com que frequência o servidor deve verificar alterações de código nos módulos e aplicações implantados. O valor default é 2.

Time-out para Sessão Admin

Identifica o número de minutos de inatividade que antecede o time-out do Console de Administração, você precisa fazer log-in novamente. O valor default é 60.

Implantação Automática

Se esta opção estiver ativada, as aplicações no diretório de implantação automática serão implantadas dessa forma. Por default, a opção está ativada.

Intervalo de Sondagem para Implantação Automática

Especifica com que frequência o servidor deve verificar o diretório de implantação automática quanto a alterações em aplicações ou módulos. A alteração do intervalo de sondagem não afeta o tempo destinado à implantação de uma aplicação ou módulo. O valor default é 2.

Time-out para Novas Tentativas de Implantação Automática

Especifica o número de segundos que um arquivo parcialmente copiado pode permanecer inalterado em termos de tamanho antes que ocorra um erro. O valor default é 4.

Se um arquivo for copiado lentamente no diretório de implantação automática, esse arquivo poderá ser exibido antes que todo o arquivo seja copiado. Como resultado, a tentativa de implantá-lo automaticamente falhará. Se uma tentativa de implantação automática de uma aplicação falhar por esse motivo, o GlassFish Server tentará implantar a aplicação novamente. Se o tamanho do arquivo parcialmente copiado permanecer inalterado durante o período especificado, ocorrerá um erro.

Diretório de Implantação Automática

Identifica o diretório para monitorar a implantação automática de aplicações. O valor default é domain-dir/autodeploy.

Validação do XML

Especifica o tipo de validação do XML a ser realizada nos descritores de implantação padrão e do GlassFish Server. O tipo de ser qualquer um dos seguintes:

Completo

Se a validação do XML falhar, a implantação falhará. Esse é o valor default.

Fazendo Parsing

A implantação ocorreu, apesar dos erros de validação do XML relatados.

Nenhum

Não foi realizada nenhuma validação do XML.

Verificador

Se esta opção estiver ativada, o verificador será executado antes da implantação automática. Se a verificação falhar, a implantação não será realizada. O verificador examina a estrutura e o conteúdo do arquivo. A verificação de aplicações grandes geralmente consome muito tempo. Por default, essa opção está desativada.

Pré-compilar

Se esta opção estiver ativada, arquivos do JavaServer Pages (JSP) serão pré-compilados. Se esta opção for desativada, os arquivos JSP serão compilados durante o runtime, quando forem acessados pela primeira vez. Por default, essa opção está desativada.

Propriedades Adicionais

Propriedades adicionais das aplicações. O GlassFish Server não define nenhuma propriedade adicional para as aplicações.
