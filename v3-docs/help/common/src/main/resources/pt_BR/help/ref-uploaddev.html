Implantar Aplicações ou Módulos

Use a página Implantar Aplicações ou Módulos para implantar uma aplicação.

A página Implantar Aplicações ou Módulos contém as opções a seguir para todas as aplicações. Opções adicionais aparecem apenas após você especificar o campo Local da Aplicação.

Local

O local do arquivo compactado da aplicação que está sendo implantada.

As seguintes opções especificam de onde o arquivo compactado é acessado e se é um arquivo ou um diretório.

Arquivo Encapsulado para Upload no Servidor

O arquivo compactado reside ou é acessado no computador cliente.

O computador cliente é o host em que você exibe o Console de Administração por meio de um browser.

Arquivo Encapsulado ou Diretório Local Acessível no GlassFish Server

O arquivo compactado é um arquivo que reside no computador servidor ou é uma aplicação não encapsulada, em um diretório explodido.

O computador servidor é o host que executa o servidor de administração de domínios do GlassFish Server .

Tipo

O tipo da aplicação. As opções disponíveis são:

  • Aplicação Web

  • Aplicação Enterprise

  • Cliente de Aplicação

  • Módulo do conector

  • EJB Jar

  • Outro

Destinos

Clusters e instâncias standalone nos quais a aplicação ou o módulo será implantado. Mova os destinos desejados para a coluna Destinos selecionados utilizando os botões Adicionar e Adicionar tudo. Mova os destinos desnecessários para a coluna Destinos disponíveis utilizando os botões Remover e Remover tudo.

Esta opção é exibida somente se os clusters ou instâncias standalone tiverem sido criados no domínio.

As opções adicionais dos diversos tipos de aplicações são descritas nas seções a seguir.

  • Opções Adicionais para uma Aplicação Web

  • Opções Adicionais para uma Aplicação Enterprise

  • Opções adicionais para um cliente de aplicação

  • Opções adicionais para um módulo conector

  • Opções adicionais para um EJB Jar.

Opções Adicionais para uma Aplicação Web

Se o tipo da aplicação for Aplicação Web, as opções aseguir serão exibidas.

Raiz de Contexto

O caminho da aplicação. No URL da aplicação web, a raiz do contexto segue imediatamente o número da porta (http://host:porta/context-root/...). A raiz do contexto deve começar com uma barra à direita, por exemplo, /hello.

Nome da Aplicação

O nome da aplicação.

O nome pode incluir um identificador de versão opcional, que segue o nome e é separado do nome por dois-pontos (:). O identificador de versão deve começar com uma letra ou número. ELa pode conter caracteres alfanuméricos e sublinhado (_), traço (-), e ponto (.). Para obter mais informações sobre versões modulares ou da aplicação, consulte Versões Modulares e da Aplicação em Oracle GlassFish Server Application Deployment Guide.

Servidores Virtuais

Os servidores virtuais associados à aplicação.

A opção Servidores Virtuais aparece somente se a instância default do servidor, server, existir. Se os clusters ou outras instâncias de servidores standalone existirem, é possível selecionar os servidores virtuais depois da implantação. Vá à página Editar Aplicação, selecione a guia Destino e selecione Gerenciar Servidores Virtuais para o destino desejado.

Status

Se esta opção estiver selecionada, a aplicação será ativada. Por default, essa opção está selecionada.

Disponibilidade de

Se a caixa de seleção Ativado estiver selecionada, a alta disponibilidade será ativada para sessões Web e para checkpoint e, potencialmente, a passivação do bean da sessão stateful (SFSB). Se esta opção estiver definida como falsa (default), todo os salvamentos de sessão Web e checkpoints SFSB serão desativados para a aplicação, a aplicação Web ou o módulo EJB especificado. Se esta opção estiver definida como verdadeira, a aplicação ou o módulo especificado será ativado para alta disponibilidade. Defina essa opção como verdadeira apenas se a alta disponibilidade estiver configurada e ativada em níveis mais altos, como os níveis do servidor e contêiner.

Esta opção aparece se clusters ou instâncias de servidores standalone, além da instância default do servidor (server), existirem.

Precompilar JSPs

Se essa opção for selecionada, arquivos JavaServer Pages (JSP) são précompilados. Se esta opção for desativada, os arquivos JSP serão compilados durante o runtime, quando forem acessados pela primeira vez. Por default, essa opção está desativada.

Executar Verificador

Se esta opção for selecionada, os descritores de implantação são verificados antes da implantação. Se a verificação falhar, a implantação não será realizada. O verificador examina a estrutura e o conteúdo dos descritores de implantação. A verificação de aplicações grandes geralmente consome muito tempo. Por default, essa opção está desativada.

Os pacotes verificadores devem ser instalados pela Ferramenta de Atualização, ou uma advertência será registrada e esta opção será ignorada.

Impor Reimplantação

Se esta opção estiver selecionada, a aplicação será reimplantada, caso já tenha sido implantada. Se esta opção não tiver sido selecionada, a tentativa de implantar uma aplicação já implantada resultará em erro. Por default, essa opção está desativada.

Manter Estado

Essa opção controla se as sessões Web, as instâncias SFSB e os timers EJB criados persistentemente são retidos entre as reimplantações.

Por default, essa opção está desativada. Esta opção só é suportada na instância do servidor default, denominada servidor. Não é suportada e ignorada em nenhum outro destino.

Algumas alterações em uma aplicação entre as reimplantações impedem que este recurso funcione apropriadamente. Por exemplo, não altere o conjunto de variáveis da instância na classe do bean SFSB.

Para aplicações Web, este recurso somente é aplicável se no arquivo glassfish-web-app.xml o atributo persistence-type do elemento session-manager for file.

Em instâncias do bean de sessão stateful, o tipo de persistência sem alta disponibilidade está definido no servidor (a opção Tipo de persistência SFSB) e deve estar definido como file, que é o valor default e recomendado.

Se qualquer sessão Web ativa, instância SFSB ou timer EJB não for preservado(a) ou restaurado(a), nenhum deles estará disponível quando a reimplantação tiver sido concluída. Entretanto, a reimplantação continuará e uma advertência será registrada em log.

Para preservar dados do estado ativo, o GlassFish Server serializa os dados e os salva na memória. Para restaurar os dados, o carregador de classe da aplicação recém-reimplantada desserializa todos os dadosalvos anteriormente.

Preservar Recursos no Escopo da Aplicação

Se esta opção estiver marcada, os recursos no escopo da aplicação serão preservados e restaurados durante a reimplantação. Por default, esta opção não está marcada.

Bibliotecas

Uma lista separada por vírgulas dos arquivos JAR da biblioteca, específicos deste módulo ou aplicação. Os caminhos podem ser absolutos ou relativos. Um caminho relativo é relativo a domain-dir/lib/applibs. Se o caminho for absoluto, ele deverá estar acessível ao servidor de administração de domínios (DAS), o que significa que ele deve estar em domain-dir. As bibliotecas são disponibilizadas para a aplicação na ordem em que elas são especificadas.

Descrição

Uma descrição da aplicação.

Compatibilidade

Se essa opção estiver marcada, os requisitos de visibilidade JAR do GlassFish Server v2 serão usados para aplicações em lugar dos estritos requisitos Java EE 6 implantados nas versões do GlassFish Server v3, incluindo 3.1.2. Por default, esta opção não está marcada.

Se esta opção for marcada, estas restrições do Java EE 6 são removidas.

Bibliotecas

Uma lista, separada por vírgulas, de arquivos JAR de biblioteca específicos dessa aplicação. Especifica um caminho absoluto ou relativo. Um caminho relativo é relativo a domain-dir/lib/applibs. Uma lista separada por vírgulas dos arquivos JAR da biblioteca, específicos deste módulo ou aplicação. Especifica um caminho absoluto ou relativo. Um caminho relativo é relativo a domain-dir/lib/applibs. Se o caminho for absoluto, ele deverá estar acessível ao servidor de administração de domínios (DAS), o que significa que ele deve estar em domain-dir. As bibliotecas são disponibilizadas para a aplicação na ordem em que elas são especificadas.
