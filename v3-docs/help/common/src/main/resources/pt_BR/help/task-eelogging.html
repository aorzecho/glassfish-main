Para definir as configurações gerais de log

  1. Na árvore de navegação, expanda o nó Configurações.
  2. No nó Configurações, clique na configuração de cluster ou instância de servidor na qual deseja configurar as configurações do agente de log.

    A página Configuração é exibida.

  3. Na página Configuração, clique em Configurações do agente de log.

    A página Configurações de log da configuração selecionada se abre.

  4. Marque a caixa de seleção Gravar no Log do Sistema Ativado para enviar a saída de log para o recurso syslog, além do log do servidor.

    Esse recurso está disponível apenas nos sistemas Solaris e Linux. Por padrão, a opção está desativada.

  5. No campo Arquivo de Log, digite um nome alternativo ou local para o arquivo de log do servidor.

    O local padrão é ${com.sun.aas.instanceRoot}/logs/server.log .

  6. No campo Manipulador de Logs, digite o nome da classe absoluta de um manipulador de logs personalizado.

    Um manipulador de logs personalizado permite que você envie logs para um destino diferente de server.log ou syslog O manipulador personalizado deve estender a classe java.util.logging.ConsoleHandler (uma API compatível com JSR 047). Coloque a classe do manipulador no caminho da classe do Servidor GlassFish, para que o manipulador seja instalado durante a inicialização do servidor.

  7. No campo Limite de Rotação de Arquivo, digite o tamanho em bytes que o arquivo de log do servidor deve alcançar antes de ser rotacionado.

    Quando o log do servidor alcança o tamanho especificado, o Servidor GlassFish cria um arquivo novo, vazio, chamado server.log e renomeia o arquivo antigo para server.log_date, onde date é a data e a hora em que o arquivo foi rotacionado. O valor padrão é 2 megabytes. O valor mínimo para o limite é de 500 quilobytes; se você especificar um valor menor, o arquivo será rotacionado quando alcançar 500 KB. Para desativar a rotação de arquivos de log, defina o valor para 0.

  8. No campo Limite de Tempo de Rotação de Arquivo, digite o número de minutos após o qual o log do servidor deve ser rotacionado.

    O valor padrão é 0, o que significa que o arquivo é rotacionado quando ele alcança o tamanho especificado no campo Limite de Rotação de Arquivo.

    Observação - Se você especificar um ou mais minutos, o limite de tempo tem precedência sobre o limite de tamanho especificado pelo campo Limite de Rotação de Arquivo.

  9. No campo Frequência de Descarga, digite o número máximo de mensagens a serem escritas de cada vez a partir da fila para o registro do servidor.

    O valor padrão é 1.

  10. Clique em Salvar.

    Observação - Dependendo da configuração para a qual o log foi ativado, pode ser necessário reiniciar o servidor.

    Também pode ser necessário reiniciar o servidor se o nome do arquivo de log estiver alterado na configuração padrão. Será exibida a mensagem Reinicialização necessária no canto superior direito do Console de Administração caso seja necessário reiniciar o servidor.
