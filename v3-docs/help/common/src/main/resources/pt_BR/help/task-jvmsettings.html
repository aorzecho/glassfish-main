Para Configurar as Definições Gerais da JVM

O Java Virtual Machine (JVM) possibilita que códigos de bytes de Java seja executados em uma máquina física. O ajuste das definições de JVM melhora o desempenho e evita erros de alocação de memória.

O Java Virtual Machine está incluído na Java Platform, Standard Edition (plataforma Java SE), que é exigida pelo GlassFish Server. Definições de JVM incorretas impedirão que o servidor seja executado, portanto, tenha cuidado ao alterar essas definições.

Observação:

Os termos "Java Virtual Machine" e "JVM" significam uma Máquina Virtual para a plataforma Java.

  1. In the navigation tree, expand the Configuration node.

  2. Select the configuration that you are changing.

  3. Select the JVM Settings node.

    The JVM General Settings tab page opens.

  4. Na árvore de navegação, selecione o nó Configuração.

  5. Sob o nó Configuração, selecione o nó Configurações de JVM.

    A página Definições Gerais de JVM é exibida.

  6. Na página Definições Gerais de JVM, no campo Java Home, digite o nome do diretório de instalação do software Java SE.

    O GlassFish Server depende da plataforma Java SE. Para verificar se a versão do software Java SE especificado é suportada nesta release, consulte o Notas de Release do Oracle GlassFish Server. Se você especificar um nome de diretório inexistente ou se a versão do diretório não for compatível com o software Java SE, o GlassFish Server não será iniciado.

  7. No campo Opções Javac, digite as opções de linha de comando para o compilador de linguagem de programação Java.

    O GlassFish Server executa o compilador quando componentes Enterprise JavaBeans (EJB) são implantados.

  8. Marque a caixa de seleção Depuração Ativada para ativar a depuração com o Java Platform Debugger Architecture (JPDA).

    O JPDA é utilizado pelos desenvolvedores de aplicações. Para obter mais informações, consulte o Oracle GlassFish Server Application Development Guide.

  9. No campo Opções de Depuração, digite as opções do JPDA a serem transmitidas para a JVM.

    Esse campo só será aplicável se a depuração tiver sido ativada na etapa precedente.

  10. No campo Opções de Compilação RMI, digite as opções de linha de comando para o compilador rmic.

    O GlassFish Server executa o compilador rmic quando os componentes EJB são implantados.

  11. No campo Pré-processador de Código de Byte, digite uma lista de nomes de classes.

    Use vírgulas para separar os nomes de classes. Cada classe deve implementar a interface com.sun.appserv.BytecodePreprocessor. As classes são chamadas na ordem especificada.

  12. Na seção Propriedades Adicionais, especifique essas propriedades.

    Para adicionar uma propriedade, clique no botão Adicionar Propriedade. Na linha em branco que é exibida, digite o nome da propriedade no campo Nome e digite o valor da propriedade no campo Valor.

    O GlassFish Server não define propriedades adicionais para a JVM.

  13. Clique em Salvar.

    Dica:

    Para voltar às definições default, clique no botão Carregar Defaults e depois em Salvar.

  14. Reinicie o servidor.
