Para implantar um módulo conector

  1. Na árvore de navegação, selecione o nó Aplicativos.

    A página Aplicativos é exibida.

  2. Na tabela Aplicativos Implantados, clique no botão Implantar.

    A página Implementar Aplicativos ou Módulos é exibida.

  3. Especifique o local do aplicativo a ser implementado.
    • Se o aplicativo for um arquivo que reside na máquina cliente ou é acessível a partir dela, selecione a opção Arquivo Empacotado a Ser Carregado no Servidor.

      O computador cliente é o host em que você visualiza o Console de Administração por meio de um navegador.

      Clique em Procurar para localizar o arquivo ou digite o caminho completo para o arquivo.

    • Se o aplicativo for um arquivo que reside na máquina servidor, ou se for um aplicativo não empacotado de um diretório explodido, selecione a opção Arquivo Empacotado Local ou Diretório que é Acessível a parir do Servidor GlassFish.

      O computador servidor é o host que executa o servidor de administração de domínio (DAS).

      Clique em Procurar Arquivos para localizar um arquivo ou clique em Procurar Pastas para localizar um diretório. Como alternativa, digite o caminho completo para o arquivo ou diretório. A implantação a partir de um diretório avançado destina-se a desenvolvedores avançados e não é recomendada para ambientes de produção.

  4. Na lista suspensa Tipo, selecione Módulo Conector.

    Se você tiver especificado um Arquivo Empacotador a Ser Carregado no Servidor ou um Arquivo Empacotado Local Acessado do Servidor GlassFish, o tipo do aplicativo será detectado automaticamente. Se você tiver especificado um Diretório Acessado do Servidor GlassFish, você deve especificar o tipo manualmente.

  5. No campo Nome do Aplicativo, preserve o nome padrão, que é o nome do diretório ou o prefixo do nome do arquivo, ou digite outro nome.

    O nome padrão será exibido se você optar por carregar um arquivo. O nome do aplicativo deve ser exclusivo.

    O nome pode incluir um identificador de versão opcional, que segue o nome e é separado do nome por dois pontos (:). O identificador de versão deve começar com uma letra ou número. Contém caracteres alfanuméricos e caracteres de traço baixo (_), traço (-) e ponto final (.). Para obter mais informações sobre versões de aplicativos e módulos, consulte Module and Application Versions no Oracle GlassFish Server 3.1 Application Deployment Guide .

  6. Marque a caixa de seleção Status Ativado para habilitar o aplicativo.

    Por padrão, essa opção está selecionada. Esta opção está desativada, o aplicativo não está disponível.

  7. Marque a caixa de seleção Executar verificador para verificar o descritor de implantação antes da implantação.

    Se essa opção for selecionada e a verificação falhar, a implantação não será realizada. O verificador examina a estrutura e o conteúdo dos descritores de implantação. A verificação de aplicativos grandes geralmente consome muito tempo. A opção está desativada por padrão.

    Os pacotes verificadores devem ser instalados a partir da Ferramenta de atualização, ou um aviso é registrado e esta opção é ignorada.

  8. Marque a caixa de seleção Forçar reimplantação para forçar a reimplantação caso o aplicativo já esteja implantado.

    Se essa opção não for selecionada, a tentativa de reimplantar o aplicativo que já esteja implantado resultará em erro. A opção está desativada por padrão.

  9. Marque a caixa de seleção Preservar recursos no escopo do aplicativo para preservar os recursos no escopo do aplicativo e restaurá-los durante a reimplantação.

    A opção está desativada por padrão.

  10. No campo Descrição, digite uma descrição para o aplicativo.
  11. Para adicionar instâncias de servidor e clusters de destino nos quais implantar o módulo ou aplicativo, selecione-os na lista Destinos disponíveis e utilize o botão Adicionar para movê-los para a lista Destinos selecionados.

    A opção Destino aparece se clusters ou instâncias de servidores independentes, além da instância padrão do servidor (server), existirem. Por padrão, o módulo ou aplicativo é implantado na instância padrão do servidor, denominado server. Utilize a botão Adicionar tudo para implantar em todos os destinos. Utilize os botões Remover e Remover tudo para remover os destinos da lista Destinos selecionados.

  12. Clique em OK.
