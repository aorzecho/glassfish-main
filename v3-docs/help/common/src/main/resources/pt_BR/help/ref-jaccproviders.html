Provedores de JACC

Utilize a página Provedores de JACC para configurar os provedores do Java Authorization Contract for Containers (JACC) do cluster ou instância selecionada do GlassFish Server.

Os provedores de JACC podem ser utilizados por módulos de autorização de terceiros em aplicativos em execução no GlassFish Server. A instalação padrão do GlassFish Server inclui dois provedores de JACC, denominados default e simple . Estes provedores padrão implementam um mecanismo simples de autorização com base em arquivo que cumpre com a especificação do JACC. Você também pode criar seus próprios provedores de JACC. Consulte Para criar um provedor de JACC.

Para obter informações sobre como gravar as classes de implementação do provedor de JACC, consulte a especificação JACC em http://jcp.org/en/jsr/detail?id=115.

Para cada provedor de JACC, são fornecidas as seguintes informações.

Nome

O nome do provedor JACC. Este nome é utilizado para fazer referência ao elemento jacc-provider no domain.xml.

Configuração da Diretiva

Nome de classe totalmente qualificado do javax.security.jacc.PolicyConfigurationFactory.provider que implementa o javax.security.jacc.PolicyConfigurationFactory específico do provedor.

Provedor de Diretivas

Nome de classe totalmente qualificado do javax.security.jacc.policy.provider que implementa o java.security.Policy.

Os provedores de JACC padrão também oferecem suporte a seguinte propriedade.

repositório

O diretório que contém o arquivo da diretiva do JACC. Para o provedor de JACC default do GlassFish Server, o diretório padrão é ${com.sun.aas.instanceRoot}/generated/policy . Esta propriedade não está definida por padrão para o provedor de JACC simple do GlassFish Server.

A tabela Provedores JACC também contém as seguintes opções.

Novo

Botão para criar um novo provedor JACC.

Excluir

Botão para excluir um ou mais provedores JACC selecionados.
