Provedores de JACC

Use a página de Provedores de JACC (Java Authorization Contract for Containers) para a instância ou cluster do GlassFish Server selecionado.

Provedores de JACC podem ser utilizados por módulos de autorização de terceiros para aplicações que estão sendo executadas em GlassFish Server. A instalação default do &ProductName inclui dois provedores de JACC, denominados default e simple. Estes provedores default implementam um mecanismo simples de autorização com base em arquivo que cumpre com a especificação do JACC. Você também pode criar seus próprios provedores de JACC. Consulte Para Criar um Provedor de JACC.

Para obter mais informações sobre como escrever as classes de implementação para um provedor de JACC, consulte a especificação do JACC em http://jcp.org/en/jsr/detail?id=115.

Para cada provedor de JACC, são fornecidas as seguintes informações.

Nome

O nome do provedor de JACC. Este nome é utilizado para fazer referência ao elemento jacc-provider no domain.xml.

Configuração da Política

Nome de classe totalmente qualificado do javax.security.jacc.PolicyConfigurationFactory.provider que implementa o javax.security.jacc.PolicyConfigurationFactory específico do provedor.

Provedor de Políticas

Nome de classe totalmente qualificado do javax.security.jacc.policy.provider que implementa o java.security.Policy.

Os provedores de JACC default também oferecem suporte a seguinte propriedade.

repositório

O diretório que contém o arquivo da política do JACC. Para o provedor de JACC do GlassFish Server default, o diretório default é ${com.sun.aas.instanceRoot}/generated/policy. Esta propriedade não está definida por default para o provedor de JACC do GlassFish Server simple.

A tabela Provedores de JACC também contém as seguintes opções.

Novo

Botão para criar um novo provedor de JACC.

Deletar

Botão para deletar um ou mais provedores de JACC selecionados.
