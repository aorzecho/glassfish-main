Para Criar um Módulo de Auditoria

A auditoria é como o GlassFish Server grava eventos significativos, como erros ou vulnerabilidades de segurança. O GlassFish Server registra todos os eventos de autenticação em arquivos de log. Um log de acesso completo fornece uma pista sequencial dos eventos de acesso do GlassFish Server.

O GlassFish Server oferece um módulo de auditoria simples default; para obter mais informações, consulte Para Usar o Módulo de Auditoria Default .

  1. Na árvore de navegação, expanda o nó Configuração.

  2. Select the instance to configure:

    • To configure a particular instance, expand the instance's config node. For example, the default instance, server, expand the server-config node.

    • To configure the default settings for all instances, expand the default-config node.

  3. Sob o nó Configuração, expanda o nó Segurança.

  4. Sob o nó Segurança, selecione o nó Módulos de Auditoria.

    A página Módulos de Auditoria é exibida.

  5. Na página Módulos de Auditoria, clique em Novo.

    A página Novo Módulo de Auditoria é exibida.

  6. Na página Novo Módulo de Auditoria, no campo Nome, digite o nome do módulo de auditoria.

  7. No campo Nome da Classe, digite o nome totalmente qualificado da classe que implementa esse módulo.

    O nome da classe do módulo de auditoria default é com.sun.enterprise.security.Audit .

  8. Na seção Propriedades Adicionais, especifique essas propriedades.

    Para adicionar uma propriedade, clique no botão Adicionar Propriedade. Na linha em branco que é exibida, digite o nome da propriedade no campo Nome e digite o valor da propriedade no campo Valor.

    Para obter uma descrição de uma propriedade adicional que está disponível para módulos de auditoria, consulte Propriedades Específicas para Auditar Módulos.

  9. Clique em OK.
