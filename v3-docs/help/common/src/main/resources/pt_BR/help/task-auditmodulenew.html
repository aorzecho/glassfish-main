Para criar um módulo de auditoria

A auditoria é como o Servidor GlassFish grava eventos significativos, como erros ou vulnerabilidades de segurança. O Servidor GlassFish registra todos os eventos de autenticação em arquivos de log. Um log de acesso completo fornece uma pista sequencial dos eventos de acesso do Servidor GlassFish.

O Servidor GlassFish fornece um módulo de auditoria padrão, que é nem simples. Para obter mais informações, consulte Para usar o módulo de auditoria padrão .

  1. Na árvore de navegação, expanda o nó Configuração.
  2. Sob o nó Configuração, expanda o nó Segurança.
  3. Sob o nó Segurança, selecione o nó Módulos de Auditoria.

    A página Módulos de Auditoria é exibida.

  4. Na página Módulos de Auditoria, clique em Novo.

    A página Novo Módulo de Auditoria é exibida.

  5. Na página Novo Módulo de Auditoria, no campo Nome, digite o nome do módulo de auditoria.
  6. No campo Nome da Classe, digite o nome totalmente qualificado da classe que implementa esse módulo.

    O nome da classe do módulo de auditoria padrão é com.sun.enterprise.security.Audit .

  7. (Opcional) Na seção Propriedades Adicionais, especifique essas propriedades.

    Para adicionar uma propriedade, clique no botão Adicionar Propriedade. Na linha em branco que é exibida, digite o nome da propriedade no campo Nome e digite o valor da propriedade no campo Valor.

    Para uma descrição da propriedade adicional que está disponível para módulos de auditoria, consulte Propriedades Específicas de Módulos de Auditoria.

  8. Clique em OK.
