Aplicações

A página Aplicações exibe uma lista das aplicações que estão implantadas no GlassFish Server. É possível visualizar e gerenciar as aplicações implantadas, e você pode implantar mais aplicações.

Para cada aplicação, as informações a seguir são mostradas.

Nome

O nome da aplicação.

O nome pode incluir um identificador de versão opcional, que segue o nome e é separado do nome por dois-pontos (:). Para obter mais informações sobre versões modulares ou da aplicação, consulte Versões Modulares e da Aplicação em Oracle GlassFish Server Application Deployment Guide.

Ativado ou status

Uma marca de seleção se a aplicação estiver ativada ou um X se a aplicação estiver desativada, se somente a instância default do servidor ,server, existir.

Se existirem clusters ou outras instâncias de servidores standalone, selecione Ativado em Destinos para visualizar os destinos nos quais a aplicação está implantada.

Mecanismos

Os tipos de contêineres usados pela aplicação. Os tipos de contêineres podem ser:

  • web

  • webservices

  • ejb

  • connector

  • appclient

  • weld (o contêiner para Contextos e Injeção de dependência para aplicações da Plataforma Java EE)

Ação

Links para as ações que você pode realizar no componente depois de implantá-lo: Reimplantar e Recarregar para todos os componentes, Acionar para aplicações da web e clientes de aplicações e Fazer download de stubs para clientes da aplicação;

A tabela Aplicações Implantadas também contém as opções a seguir.

Implantar

Botão para implantar uma aplicação.

Cancelar Implantação

Botão para cancelar a implantação de uma ou mais aplicações selecionadas.

Ativar

Botão para ativar um ou mais aplicações selecionadas, somente se a instância default do servidor, server, existir.

Desativar

Botão para desativar um ou mais aplicações selecionadas, somente se a instância default do servidor, server, existir.

Filtro

Lista drop-down que filtra as aplicações por mecanismo.
