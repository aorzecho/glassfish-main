Aplicativos

A página Aplicativos exibe uma lista dos aplicativos que são implantados no Servidor GlassFish. É possível visualizar e gerenciar os aplicativos implantados, e você pode implantar mais aplicativos.

Para cada aplicativo, as seguintes informações são fornecidas.

Nome

O nome do aplicativo.

O nome pode incluir um identificador de versão opcional, que segue o nome e é separado do nome por dois pontos (:). Para obter mais informações sobre versões de aplicativos e módulos, consulte Module and Application Versions no Oracle GlassFish Server 3.1 Application Deployment Guide .

Ativado ou status

Uma marca de seleção se o aplicativo estiver ativado ou um X se o aplicativo estiver desativado, somente se a instância padrão do servidor ,server, existir.

Se os clusters ou outras instâncias de servidores independentes existirem, selecione Ativado em Destinos para visualizar os destinos nos quais o aplicativo está implantado.

Mecanismos

Os tipos de recipientes usados pelo aplicativo. Os tipos de recipientes podem ser:

  • web

  • webservices

  • ejb

  • connector

  • appclient

  • weld (o container para Contextos e Injeção de dependência para aplicativos da plataforma Java EE)

Ação

Links para as ações que você pode executar no componente após a implantação: Reimplantar e Recarregar para todos os componentes e Iniciar para aplicativos Web.

A tabela Aplicativos implementados também contém as seguintes opções.

Implementar

Botão para implementar um aplicativo.

Cancelar implementação

Botão para cancelar a implementação de um ou mais aplicativos selecionados.

Ativar

Botão para ativar um ou mais aplicativos selecionados, somente se a instância padrão do servidor, server, existir.

Desativar

Botão para desativar um ou mais aplicativos selecionados, somente se a instância padrão do servidor, server, existir.

Filtro

Lista suspensa que filtra os aplicativos por mecanismo.
