Para Definir a Configuração de Aplicações

Use a página Configuração de Aplicações para definir as propriedades que ajudarão a garantir que as alterações em aplicações implantadas serão detectadas e que as classes modificadas serão recarregadas.

  1. Na árvore de navegação, selecione o nó Domínio.

    A página Atributos do Domínio é exibida.

  2. Selecione a guia Configuração de Aplicações.

    A página Configuração de Aplicações é exibida.

  3. Marque a caixa de seleção Recarregamento Ativado para ativar o recarregamento dinâmico de aplicações.

    Quando o recarregamento dinâmico está ativado, você não precisa reimplantar aplicações ou módulos caso altere o respectivo código ou os descritores de implantação. Tudo que você tem a fazer é copiar as JavaServer Pages (JSP) ou arquivos de classe no diretório de implantação da aplicação ou módulo. O servidor verifica periodicamente se há alterações no diretório de implantação e reimplanta a aplicação, automática e dinamicamente, com as alterações. Isso é útil em um ambiente de desenvolvimento, pois permite testar rapidamente as alterações de códigos. Em um ambiente de produção, no entanto, o recarregamento dinâmico pode prejudicar o desempenho. Além disso, sempre que ocorre um recarregamento, as sessões nesse tempo de transição se tornam inválidas. O cliente deve reiniciar a sessão.

    Por default, a opção está ativada.

  4. No campo Intervalo de Sondagem para Recarregamento, digite a duração, em segundos, do intervalo em que aplicações e módulos deverão ser verificados, para detecção de alterações de códigos, e recarregados dinamicamente.

    O valor default é 2.

  5. No campo Time-out para Sessão Admin, digite o número de minutos de inatividade, após o qual a sessão do Console de Administração será encerrada.

    O valor default é 60.

  6. Marque a caixa de seleção Implantação Automática Ativada para ativar a implantação automática de aplicações.

    Se você copiar um arquivo de aplicação ou módulo (JAR, WAR, RAR ou EAR) para um diretório especial e depois ativar a implantação automática, a aplicação ou o módulo será automaticamente implantado pelo GlassFish Server. Por default, a opção está ativada.

  7. No campo Intervalo de Sondagem para Implantação Automática, digite a duração, em segundos, do intervalo em que aplicações e módulos deverão ser verificados, para detecção de alterações de códigos, e recarregados dinamicamente.

    O valor default é 2. A alteração do intervalo de sondagem não afeta o tempo destinado à implantação de uma aplicação ou módulo.

  8. No campo Time-out para Novas Tentativas de Implantação Automática, digite o número de segundos em que o tamanho de um arquivo parcialmente copiado pode permanecer inalterado antes que ocorra um erro.

    O valor default é 4.

  9. O campo Diretório de Implantação Automática, digite o caminho completo do diretório para monitorar a implantação automática de aplicações.

    O valor default é domain-dir/autodeploy.

  10. Na lista drop-down XML, selecione o tipo de validação XML a ser realizada nos descritores de implantação padrão e do GlassFish Server.

    O tipo de ser qualquer um dos seguintes:

    completo

    Se a validação do XML falhar, a implantação falhará. Esse é o valor default.

    fazendo parsing

    A implantação ocorreu, apesar dos erros de validação do XML relatados.

    nenhum

    Não foi realizada nenhuma validação do XML.

  11. Marque a caixa de seleção Verificador Ativado para verificar os arquivos de descritor de implantação.

    O verificador examina a estrutura e o conteúdo do arquivo antes da implantação automática. Se a verificação falhar, a implantação não será realizada. A verificação de aplicações grandes geralmente consome muito tempo. Por default, essa opção está desativada.

  12. Marque a caixa de seleção Pré-compilação Ativada para pré-compilar arquivos JSP.

    Se esta opção for desativada, os arquivos JSP serão compilados durante o runtime, quando forem acessados pela primeira vez. Por default, essa opção está desativada.

  13. Na seção Propriedades adicionais, especifique essas propriedades.

    O GlassFish Server não define nenhuma propriedade adicional para as aplicações.

    • Para adicionar uma propriedade, clique no botão Adicionar propriedade. Na linha em branco que é exibida, digite o nome da propriedade no campo Nome e digite o valor da propriedade no campo Valor.

    • Para modificar uma propriedade, edite o campo Valor da propriedade.

    • Para deletar uma propriedade, marque a caixa de seleção à esquerda do campo Nome da Propriedade a ser deletada. Clique no botão Deletar Propriedades.

  14. Clique em Salvar.

    Dica:

    Para voltar às definições default, clique no botão Carregar Defaults e depois em Salvar.

  15. Reinicie o servidor.
