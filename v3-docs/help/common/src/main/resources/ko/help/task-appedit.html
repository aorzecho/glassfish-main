배포된 응용 프로그램을 편집하는 방법

  1. 탐색 트리에서 응용 프로그램 노드를 선택합니다.

    응용 프로그램 페이지가 열립니다.

  2. 배포된 응용 프로그램 테이블에서 편집할 응용 프로그램의 이름을 누릅니다.

    이름에는 버전 ID(옵션)가 포함될 수 있으며, 이 ID는 이름 다음에 오고 콜론(:)으로 이름과 구분됩니다. 모듈 및 응용 프로그램 버전에 대한 자세한 내용은 Oracle GlassFish Server 3.1 Application Deployment Guide의 Module and Application Versions를 참조하십시오.

    응용 프로그램 편집 페이지가 열립니다.

  3. 응용 프로그램 상태를 비활성화하여 사용할 수 없게 만들려면 상태 사용 가능 확인란을 선택 해제합니다.

    기본적으로 응용 프로그램이 배포되는 즉시 상태는 사용 가능이 됩니다.

  4. 웹 응용 프로그램의 경우 컨텍스트 루트 필드에 응용 프로그램의 경로를 입력합니다.

    웹 응용 프로그램의 URL에서 컨텍스트 루트 바로 앞에는 포트 번호(http://host:port /context-root/...)가 있습니다. 컨텍스트 루트는 슬래시로 시작해야 합니다(예: /hello).

  5. 응용 프로그램에 대한 세션 지속성 및 SFSB 체크 포인트 지정을 비활성화하려면 가용성 사용 확인란을 선택 해제합니다.

    사용 가능 확인란을 선택한 경우 고가용성이 웹 세션 및 SFSB(상태 유지 세션 빈) 체크 포인트 지정과 잠재적인 패시베이션에 대해 활성화됩니다. false(기본값)로 설정된 경우 지정한 응용 프로그램, 웹 응용 프로그램 또는 EJB 모듈에 대해 모든 웹 세션 저장과 SFSB 체크 포인트 지정이 비활성화됩니다. true로 설정된 경우 지정된 응용 프로그램 또는 모듈이 고가용성에 대해 활성화됩니다. 서버 및 컨테이너 수준과 같은 보다 높은 수준에서 고가용성이 구성 및 활성화되어 있는 경우에만 이 옵션을 true로 설정합니다.

    이 옵션은 기본 서버 인스턴스(server) 외에 클러스터나 독립 실행형 서버 인스턴스가 있는 경우에 표시됩니다.

  6. 가상 서버 드롭다운 목록에서 이 응용 프로그램과 연관될 가상 서버를 선택합니다.

    가상 서버 옵션은 기본 서버 인스턴스(server)만 있는 경우에 표시됩니다. 클러스터 또는 다른 독립 실행형 서버 인스턴스가 있는 경우에는 배포 후에 가상 서버를 선택할 수 있습니다. 응용 프로그램 편집 페이지로 이동하고 대상 탭을 선택한 다음 원하는 대상에 대해 가상 서버 관리를 선택합니다.

  7. Java Web Start 소프트웨어를 사용하여 응용 프로그램을 배포하려면 Java Web Start 사용 가능 확인란을 선택합니다.

    이 옵션은 기본적으로 처음 배포 시 설정에 따라 활성화되거나 비활성화됩니다.

  8. 설명 필드에 이 응용 프로그램의 설명을 입력합니다.
  9. 응용 프로그램에 대한 모듈 설명자 목록을 보려면 설명 탭을 누릅니다. 자세한 내용은 응용 프로그램의 설명자를 보는 방법을 참조하십시오.
  10. 기본적으로 응용 프로그램의 상태는 모든 대상에서 사용 가능입니다. 응용 프로그램이 활성화되어 있는 대상을 보려면 대상 탭을 누릅니다. 하나 이상의 대상에서 응용 프로그램을 활성화하거나 비활성화할 수 있습니다. 응용 프로그램 대상을 보는 방법을 참조하십시오.
  11. 저장을 누릅니다.
