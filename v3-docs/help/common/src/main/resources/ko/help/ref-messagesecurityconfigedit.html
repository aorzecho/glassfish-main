메시지 보안 구성 편집

[메시지 보안 구성 편집] 페이지에서는 메시지 보안에 대한 제공자를 사용으로 설정할 수 있습니다.

GlassFish Server에 배치된 웹 서비스 끝점에 대한 메시지 보안을 사용으로 설정하려면 서버측에서 기본적으로 사용되는 제공자를 지정해야 합니다. 메시지 보안을 위한 기본 제공자를 사용으로 설정한 경우 GlassFish Server에 배치된 웹 서비스의 클라이언트에서 사용할 제공자도 사용으로 설정해야 합니다. 클라이언트에서 사용하는 제공자를 사용으로 설정하는 방법은 응용 프로그램 클라이언트에 대해 메시지 보안을 사용으로 설정하는 방법을(를) 참조하십시오.

배치된 끝점에서 발생한 웹 서비스 호출에 대해 메시지 보안을 사용으로 설정하려면 기본 클라이언트 제공자를 지정해야 합니다. GlassFish Server에 대한 기본 클라이언트 제공자를 사용으로 설정한 경우 GlassFish Server에 배치된 끝점에서 호출한 모든 서비스가 메시지 계층 보안과 호환 가능하게 구성되도록 해야 합니다.

[메시지 보안 구성 편집] 페이지에는 다음 옵션이 포함되어 있습니다.

구성 이름

이 페이지의 설정이 적용되는 구성의 이름입니다. 이 필드는 읽기 전용입니다.

인증 계층

인증이 수행되는 메시지 계층입니다. 읽기 전용 필드입니다. 값은 SOAP(기본 메시지 보안 구성의 경우) 또는 HttpServlet입니다.

기본 제공자

있는 경우 기본 서버-클라이언트 제공자입니다. 기본 제공자라고도 하는 기본 서버 제공자는 특정 서버 제공자를 식별하지 않는 응용 프로그램에 대해 호출된 서버 제공자를 식별합니다. ServerProvider, XWS_ServerProvider 또는 없음을 선택할 수 있습니다. 기본적으로 제공자가 선택되어 있지 않으며, GlassFish Server에서는 서버측의 메시지 보안 제공자를 호출하지 않습니다. 일반적으로 ServerProvider를 기본 제공자로 선택합니다.

기본 클라이언트 제공자

있는 경우 기본 클라이언트 제공자입니다. 기본 클라이언트 제공자는 특정 클라이언트 제공자를 식별하지 않는 응용 프로그램에 호출된 클라이언트 제공자를 식별합니다. ClientProvider, XWS_ClientProvider 또는 없음을 선택할 수 있습니다. 기본적으로 제공자가 선택되어 있지 않으며, GlassFish Server에서는 클라이언트측의 메시지 보안 제공자를 호출하지 않습니다. 일반적으로 기본 클라이언트 제공자에 대해 없음을 선택합니다. ClientProvider를 선택하면 기본 제공자 및 메시지 보호 정책을 사용으로 설정하여 GlassFish Server에 배치된 웹 서비스 끝점에서 발생한 웹 서비스 호출에 적용할 수 있습니다.

관련 작업

  • GlassFish Server 기능을 메시지 보안 제공자가 사용할 수 있도록 구성하는 방법

  • 메시지 보안 구성을 생성하는 방법

  • 메시지 보안 구성을 편집하는 방법

  • 메시지 보안 구성을 삭제하는 방법

  • 메시지 보안 제공자 구성을 생성하는 방법

  • 메시지 보안 제공자 구성을 편집하는 방법

  • 메시지 보안 제공자 구성을 삭제하는 방법

  • 응용 프로그램 클라이언트에 대해 메시지 보안을 사용으로 설정하는 방법

관련 asadmin 명령

set(1) 명령을 사용하여 메시지 보안 구성을 수정할 수 있습니다.

  • 기본 서버 제공자를 지정하려면 다음 작업을 수행합니다.

    asadmin set --user admin-user --port admin-port 
server-config.security-service.message-security-config.SOAP.
default_provider=ServerProvider

  • 기본 클라이언트 제공자를 지정하려면 다음 작업을 수행합니다.

    asadmin set --user admin-user --port admin-port 
server-config.security-service.message-security-config.SOAP.
default_client_provider=ClientProvider
