모니터링 서비스를 구성하는 방법

  1. 구성 노드에서 모니터링을 구성할 서버 인스턴스나 클러스터 구성을 확장합니다.
  2. 모니터링 노드를 확장합니다.

    모니터링 서비스 페이지가 열립니다.

  3. 모니터링 서비스 사용 확인란을 선택하여 GlassFish Server에 대한 모니터링을 활성화하거나 확인란을 선택 취소하여 모니터링을 비활성화합니다.

    이 옵션은 기본적으로 활성화됩니다.

  4. MBean 모니터링 사용 확인란을 선택하여 모니터링에 필요한 모든 MBean을 배포하거나 확인란을 선택 취소하여 이 MBean의 배포를 취소합니다.

    이 옵션은 기본적으로 활성화됩니다.

    이 확인란이 선택되어 있지 않으면 모니터링 서비스 확인란이 선택되어 있어도 관리 콘솔의 모니터링 데이터를 볼 수 없습니다. 모니터링 데이터를 보려면 명령줄을 사용해야 합니다. 확인란 선택을 취소하면 GlassFish Server의 메모리 범위가 줄어듭니다.

  5. DTrace 사용 확인란을 선택하여 Solaris DTrace 기능을 사용하도록 설정합니다.

    Solaris DTrace 기능을 사용하려면 Oracle Enterprise Manager DTrace Monitoring도 있어야 합니다. 이 옵션은 기본적으로 비활성화됩니다.

  6. 테이블에서 다음 작업 중 하나를 수행합니다.
    • 모니터링 수준 열에서 구성 요소 열의 하나 이상의 구성 요소에 대한 드롭다운 목록에서 LOW 또는 HIGH를 선택하여 해당 구성 요소에 대한 모니터링을 활성화합니다.
    • 하나 이상의 구성 요소 옆에 있는 확인란을 선택한 다음 테이블 맨 위의 수준 드롭다운 목록에서 LOW 또는 HIGH를 선택하고 수준 변경 버튼을 눌러 해당 구성 요소 모두에 대해 모니터링 수준을 이 값으로 설정합니다.

    이 릴리스에서는 낮음 및 높음 옵션에서 제공되는 모니터링 수준 간에 차이가 없습니다. 하나 이상의 구성 요소에 대한 모니터링을 비활성화하려면 OFF를 선택합니다. 기본값은 해제입니다.

    응용 프로그램, 구성 요소 또는 서비스에 대한 모니터링을 구성하려면 먼저 배포해야 합니다.

  7. 저장을 누릅니다.
