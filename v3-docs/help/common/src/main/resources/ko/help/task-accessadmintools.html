관리 툴에 대한 액세스 권한 부여

asadmin 그룹의 사용자만이 관리 콘솔 및 asadmin 명령행 유틸리티에 액세스할 수 있습니다. 이 관리 툴에 대한 사용자 액세스를 허용하려면 사용자를 admin-realm의 asadmin 그룹에 추가합니다.

  1. [구성] 노드에서 관리 권한을 구성할 서버 인스턴스나 클러스터 구성을 확장합니다.

  2. [보안] 노드를 확장합니다.

  3. [영역] 노드를 확장합니다.

  4. admin-realm 노드를 선택합니다.

    [영역 편집] 페이지가 열립니다.

  5. [영역 편집] 페이지에서 [사용자 관리] 단추를 누릅니다.

    [관리 사용자] 페이지가 열립니다.

    설치 후 처음에는 설치 시 입력한 관리자 이름 및 비밀번호가 admin-keyfile 파일에 나열됩니다. 기본적으로 이 사용자는 GlassFish Server을(를) 수정할 수 있는 권한을 부여하는 asadmin 그룹에 속합니다. 사용자에게 선택한 GlassFish Server 인스턴스나 클러스터에 대한 관리자 권한을 부여하려는 경우에만 해당 사용자를 이 그룹에 지정합니다.

    admin-realm 영역에 사용자를 추가하였지만 사용자를 asadmin 이외의 그룹에 지정한 경우에는 사용자 정보가 admin-keyfile 파일에 그대로 유지되지만 사용자에는 관리 툴 또는 file 영역에 있는 응용 프로그램에 대한 액세스가 부여되지 않습니다.

  6. [관리 사용자] 페이지에서 [새로 만들기]를 눌러 새 사용자를 admin-realm 영역에 추가합니다.

    [새 파일 영역 사용자] 페이지가 열립니다.

  7. 사용자 ID 필드에 새 사용자 이름을 입력합니다.

  8. [그룹 목록] 필드에 사용자가 속한 그룹의 이름을 입력합니다.

    사용자에게 GlassFish Server을(를) 수정할 수 있는 권한을 부여하려면 그룹 목록에 asadmin 그룹을 포함시킵니다.

  9. [새 비밀번호] 및 [새 비밀번호 확인] 필드에 새 사용자의 비밀번호를 입력합니다.

  10. 확인을 눌러 이 사용자를 admin-realm 영역에 추가하거나 취소를 눌러 저장하지 않고 종료합니다.

관련 항목

  • 관리 영역에서 사용자 비밀번호를 변경하는 방법
