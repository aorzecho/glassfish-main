보안

보안 페이지를 사용하여 선택한 GlassFish Server 인스턴스나 클러스터에 대한 보안 등록 정보를 설정합니다.

보안 페이지에는 다음 옵션이 포함되어 있습니다.

구성 이름

이 페이지의 설정을 적용할 구성의 이름입니다. 이 필드는 읽기 전용입니다.

보안 관리자

이 옵션을 선택하면 도메인의 보안 관리자가 활성화됩니다. 이 옵션은 기본적으로 비활성화됩니다.

이 옵션을 활성화하면 JVM 옵션인 -Djava.security.manager가 GlassFish Server의 JVM 설정에 추가됩니다. JVM 옵션을 구성하는 방법을 참조하십시오. 이 변경 사항을 활성화하려면 서버를 다시 시작해야 합니다.

모든 응용 프로그램에 올바른 권한을 부여했는지 확인하십시오. 성능 향상을 위해 보안 관리자를 비활성화할 수 있습니다.

감사 로깅

이 옵션을 활성화하면 감사 모듈 목록에서 지정한 모든 감사 모듈을 서버에서 로드하여 실행합니다. 이 옵션을 비활성화하면 서버가 감사 모듈에 액세스하지 않습니다. 이 옵션은 기본적으로 비활성화됩니다.

기본 영역

서버가 인증에 사용하는 활성(기본) 영역입니다. 배포 설명자가 다른 영역을 지정하지 않는 한 응용 프로그램은 이 영역을 사용합니다. 구성된 모든 영역이 목록에 표시됩니다. 기본값은 file입니다.

기본 사용자

기본 사용자 이름을 지정합니다. 다른 기본값을 제공하지 않을 경우 서버는 이 사용자 이름을 사용합니다. 이 필드에 값을 입력하는 경우에는 기본 사용자 암호 필드에 해당 암호를 입력합니다.

정상적인 서버 작동에 대해서는 기본 기본값이 필요하지 않습니다.

기본 사용자 암호

기본 사용자 필드에 지정한 기본 사용자의 암호입니다. 정상적인 서버 작동에 대해서는 기본 기본값이 필요하지 않습니다.

JACC

구성된 JACC 공급자의 클래스 이름입니다. 기본값은 default입니다.

Audit Modules

감사 로깅이 활성화된 경우 감사 하위 시스템이 사용할 감사 공급자 모듈입니다. 기본적으로 서버는 이름이 default인 감사 모듈을 사용합니다. 새 감사 모듈 만들기에 대한 자세한 내용은 감사 모듈을 만드는 방법을 참조하십시오.

역할 매핑에 대한 기본 사용자 사용 가능

이 옵션을 선택하면 역할 매핑에 대한 기본 사용자가 응용 프로그램 관련 매핑이 없는 응용 프로그램에 적용됩니다.

매핑된 기본 클래스

기본과 역할 사이의 기본 매핑이 활성화된 경우 기본과 원칙 사이의 매핑에 사용되는 java.security.Principal 클래스의 사용자 정의 구현을 지정합니다.

추가 등록 정보

서버의 추가 보안 등록 정보입니다.

유효한 등록 정보는 기본 영역 필드에서 선택한 영역의 유형에 따라 달라지며 일반적으로 영역을 편집할 때 지정됩니다. 자세한 내용은 다음 항목을 참조하십시오.

  • FileRealm 클래스와 관련된 등록 정보

  • CertificateRealm 클래스와 관련된 등록 정보

  • JDBCRealm 클래스와 관련된 등록 정보

  • LDAPRealm 클래스와 관련된 등록 정보

  • SolarisRealm 클래스와 관련된 등록 정보

  • PamRealm 클래스와 관련된 등록 정보
