표시된 로그 레코드를 필터링하는 방법

시작하기 전에

DAS(도메인 관리 서버)에 대한 로그 파일, 클러스터링된 GlassFish Server 인스턴스 또는 독립 실행형 인스턴스에 대한 로그 파일이 표시되었는지 확인합니다. 자세한 내용은 다음 항목을 참조하십시오.

  • DAS에 대한 로그 파일을 보는 방법

  • 클러스터링된 GlassFish Server 인스턴스에 대한 로그 파일을 보는 방법

  • 독립 실행형 GlassFish Server 인스턴스에 대한 로그 파일을 보는 방법

  1. 검색 기준 영역에서 내용, 기간 또는 로그 수준별로 레코드를 필터링합니다.
    1. 특정 텍스트가 포함된 레코드만 표시하려면 텍스트 검색 필드에 텍스트를 입력합니다.

      검색은 대소문자를 구분합니다.

    2. 로그 레코드가 표시되는 기간을 선택하려면 타임스탬프 옵션을 사용합니다.
      • 가장 최근 메시지를 보려면 가장 최근(기본값)을 선택합니다.
      • 특정 기간의 메시지만 보려면 특정 범위를 선택하고 표시되는 시작 및 끝 필드에 날짜와 시간 값을 입력합니다.

        날짜 값 형식은 다음과 같아야 합니다.

        MM/DD/YY
        MM

        월 수에 대한 1–12 범위의 정수입니다(예: 1월인 경우 1, 10월인 경우 10).

        DD

        한 달 중 날짜에 대한 1–31 범위의 정수입니다.

        YY

        연도의 마지막 두 자리 숫자입니다(예: 2011년인 경우 11).

        예를 들어, 2011년 2월 1일을 지정하려면 날짜 필드에 2/1/11을 입력합니다.

        시간 값 형식은 다음과 같아야 합니다.

        hh:mm:ss.SSS
        hh

        시간을 나타내는 0–23 범위의 정수입니다. 오후 시간에는 12–23 범위의 정수를 사용합니다(예: 오후 2시를 나타내려면 14 사용).

        mm

        분 숫자에 대한 0–59 범위의 정수입니다.

        ss

        초에 대한 0–59 범위의 정수입니다.

        SSS

        밀리초 숫자에 대한 000–999 범위의 세 자리 정수로, 필요한 경우 앞에 0을 표시합니다.

        예를 들어, 오후 5시 10분을 지정하려면 시간 필드에 17:10:00.000을 입력합니다.

        시작 값이 끝 값보다 이후일 경우 오류 메시지가 표시됩니다.

    3. 로그 수준별로 메시지를 필터링합니다.

      확인할 메시지를 서버 로그에 표시하려면 먼저 모듈 로그 수준 페이지에서 적절한 로그 수준을 설정합니다.

      로그 수준을 기준으로 로그 메시지를 필터링하도록 선택한 경우 지정한 필터 기준에 맞는 메시지만 표시됩니다. 그러나 이 필터링은 서버 로그에 어떤 메시지를 기록할지에는 영향을 주지 않습니다.

      1. 로그 수준 드롭다운 목록에서 로그 수준을 선택합니다.
      2. 선택한 수준의 메시지만 표시하려면 더 심각한 메시지는 포함되지 않습니다. 옵션을 선택합니다.

        기본적으로 선택한 로그 수준 및 더 심각한 수준에 있는 서버 로그의 메시지는 모두 표시되지 않습니다.

  2. 고급 검색 영역에서 로거 이름 공간을 기준으로 레코드를 필터링하고 표시되는 정보 양을 선택합니다.
    1. 고급 검색 링크를 누릅니다.

      고급 검색 영역이 로그 뷰어 창에 추가됩니다.

    2. 모듈별로 필터링하려면 로거 드롭다운 목록에서 이름 공간을 한 개 이상 선택합니다.

      Shift 키나 Ctrl 키를 사용하여 이름 공간을 여러 개 선택할 수 있습니다.

      더 높은 수준의 이름 공간을 선택하면 그 아래에 있는 모든 이름 공간이 선택됩니다. 예를 들어, javax.enterprise.system을 선택하면 해당 이름 공간 아래의 모든 모듈(예: javax.enterprise.system.core , javax.enterprise.system.tools.admin 등)에 대한 로거도 선택됩니다.

    3. 특정 응용 프로그램에 대한 로거의 메시지를 보려면 사용자 정의 로거 필드의 별도 줄에 각 로거 이름을 입력합니다.

      응용 프로그램에 모듈이 여러 개 있는 경우 그 중 일부 또는 전부를 볼 수 있습니다. 예를 들면, 응용 프로그램에 다음과 같은 로거가 있을 수 있습니다.

      • com.example.myapp.module1

      • com.example.myapp.module2

      • com.example.myapp.module3

      응용 프로그램에 있는 모든 모듈의 메시지를 확인하려면 com.mycompany.myapp를 입력합니다. module2의 메시지만 보려면 com.example.myapp.module2를 입력합니다.

      사용자 정의 로거를 하나 이상 지정한 경우 해당 로거를 로거 영역에 명시적으로 지정해야만 GlassFish Server 모듈의 메시지가 표시됩니다.

    4. 표시 드롭다운 목록에서 표시할 메시지 수를 선택합니다.

      표시되는 기본 메시지 수는 40입니다. 기타 선택 항목은 100, 250 및 1000입니다.

    5. 스택 추적을 보려면 과도하게 긴 메시지 제한 옵션을 선택 취소합니다.

      기본적으로 스택 추적은 뷰어에 표시되지 않습니다. 스택 추적을 보려면 메시지의 (세부 사항) 링크를 누릅니다.

    6. 고급 검색 영역을 숨기려면 기본 검색 링크를 누릅니다.
  3. 보려는 인스턴스 및 연관된 로그 파일을 지정하려면 검색 수정 영역을 사용합니다.
    • 인스턴스 드롭다운 목록에서 인스턴스를 선택합니다.

      기본값은 로그 파일을 보기 위해 처음에 선택한 인스턴스입니다. DAS(도메인 관리 서버)에 대한 로그 파일을 선택하여 보는 경우 기본값은 server이며, 이 서버는 DAS를 의미합니다.

    • 로그 파일 드롭다운 목록에서 로그 파일을 선택합니다.

      기본값은 server.log입니다. 다른 선택 항목은 선택한 대상의 로그 설정에 따라 다릅니다.

  4. 검색 결과 영역에서 로그 메시지 표시를 사용자 정의합니다.

    검색 결과 영역에는 로깅 설정 및 로그 수준 페이지의 설정으로 서버 로그에 있는 40개의 최근 항목이 표시됩니다.

    • 메시지를 다르게 정렬하려면 열에 있는 화살표를 누릅니다.
    • 형식이 지정된 메시지 버전을 보려면 메시지의 (세부 사항) 링크를 누릅니다.

      형식이 지정된 메시지 버전이 들어 있는 로그 항목 세부 정보 창이 열립니다.

    • 로그 파일에서 이전 또는 최신 항목을 보려면 테이블 맨 위에 있는 이전 레코드 및 이후 레코드를 누릅니다.
