JACC 공급자

JACC 공급자 페이지를 사용하여 선택한 GlassFish 서버 인스턴스나 클러스터에 대한 JACC(Java Authorization Contract for Containers) 공급자를 구성합니다.

JACC 공급자는 GlassFish 서버에서 실행 중인 응용 프로그램의 타사 인증 모듈에서 사용할 수 있습니다. 기본 GlassFish 서버 설치에 default와 simple이라는 두 개의 JACC 공급자가 포함됩니다. 이러한 기본 공급자는 JACC 사양을 준수하는 간단한 파일 기반 인증 엔진을 구현합니다. 또한 고유한 사용자 정의 JACC 공급자를 만들 수 있습니다. JACC 공급자를 만드는 방법을 참조하십시오.

JACC 공급자에 대한 구현 클래스를 작성하는 방법에 대한 자세한 내용은 http://jcp.org/en/jsr/detail?id=115

각 JACC 공급자에 대해 제공되는 정보는 다음과 같습니다.

이름

JACC 공급자의 이름입니다. 이 이름은 domain.xml의 jacc-provider 요소를 참조하는 데 사용됩니다.

정책 구성

공급자별 javax.security.jacc.PolicyConfigurationFactory를 구현하는 javax.security.jacc.PolicyConfigurationFactory.provider에 대한 정규화된 클래스 이름입니다.

정책 공급자

java.security.Policy를 구현하는 javax.security.jacc.policy.provider에 대한 정규화된 클래스 이름입니다.

기본 JACC 공급자는 또한 다음 등록 정보를 지원합니다.

저장소

JACC 정책 파일이 들어 있는 디렉토리입니다. 기본 GlassFish 서버 JACC 공급자의 기본 디렉토리는 ${com.sun.aas.instanceRoot}/generated/policy입니다. 이 등록 정보는 simpleGlassFish 서버 JACC 공급자에 대해 기본적으로 정의되어 있지 않습니다.

JACC 공급자 테이블에는 다음 옵션도 포함되어 있습니다.
