라이프사이클 모듈을 편집하는 방법

  1. 탐색 트리에서 라이프사이클 모듈 노드를 선택합니다.

    라이프사이클 모듈 페이지가 열립니다.

  2. 라이프사이클 모듈 페이지에서 편집 중인 라이프사이클 모듈의 이름을 누릅니다.

    라이프사이클 모듈 편집 페이지가 열립니다.

  3. 클래스 이름 필드에 라이프사이클 모듈의 클래스 파일에 대한 정규화된 이름을 입력합니다.

    클래스는 com.sun.appserv.server.LifecycleListener 인터페이스를 구현해야 합니다.

  4. 클래스 경로 필드에 라이프사이클 모듈의 클래스 경로를 입력합니다.

    라이프사이클 모듈이 있는 위치를 지정하는 클래스 경로입니다. 기본값은 domain-dir/applications입니다. 모듈이 이미 이 디렉토리(즉, 서버 클래스 경로)에 있으면 이 필드를 비워둘 수 있습니다.

  5. 로드 순서 필드에 시작할 때 이 라이프사이클 모듈을 로드할 순서를 지정하는 값을 입력합니다.

    정수 로드 순서 값이 작은 모듈이 먼저 로드됩니다. 값의 범위는 101부터 운영 체제의 MAXINT까지일 수 있습니다. 1부터 100까지의 값은 예약되어 있습니다.

  6. 설명 필드에 라이프사이클 모듈의 설명을 입력합니다.
  7. 라이프사이클 모듈을 사용하려면 상태 사용 가능 확인란을 선택합니다.

    이 옵션은 기본적으로 활성화됩니다.

  8. 라이프사이클 모듈이 로드에 실패할 경우 서버를 종료하도록 지정하려면 로드 실패 시 인스턴스 시작 방지 확인란을 선택합니다.

    이 옵션은 기본적으로 비활성화됩니다.

  9. 대상 탭은 기본 서버 인스턴스(server) 외에 클러스터 또는 독립 실행형 서버 인스턴스가 있는 경우에 표시됩니다. 대상 탭을 선택하여 라이프사이클 모듈이 활성화된 대상을 확인합니다. 한 대상이나 여러 대상에서 라이프사이클 모듈을 활성화하거나 비활성화하는 방법에 대한 지침은 응용 프로그램 대상을 보는 방법을 참조하십시오.
  10. 저장을 누릅니다.

참조

  • 라이프사이클 모듈을 삭제하는 방법
