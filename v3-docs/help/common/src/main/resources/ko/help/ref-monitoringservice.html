모니터링 서비스

모니터링 서비스 페이지에서는 개별 서버 구성 요소 또는 서비스의 모니터링 옵션을 구성할 수 있습니다.

응용 프로그램, 구성 요소 또는 서비스에 대한 모니터링을 구성하려면 먼저 배치해야 합니다.

모니터링 서비스 페이지에는 다음 옵션이 포함되어 있습니다.

구성 이름

이 페이지의 설정이 적용되는 구성의 이름입니다. 이 필드는 읽기 전용입니다.

모니터링 서비스

모니터링 서비스 사용 체크 박스가 선택되어 있으면 GlassFish Server에 대한 모니터링이 사용으로 설정됩니다. 이 옵션은 기본적으로 사용으로 설정되어 있습니다.

MBean 모니터링

MBean 모니터링 사용 체크 박스가 선택되어 있으면 모니터링에 필요한 모든 MBean이 GlassFish Server에 배치됩니다. 이 옵션은 기본적으로 사용으로 설정되어 있습니다.

이 체크 박스가 선택되어 있지 않으면 모니터링 서비스 체크 박스가 선택되어 있어도 관리 콘솔의 모니터링 데이터를 볼 수 없습니다. 모니터링 데이터를 보려면 명령행을 사용해야 합니다. 체크 박스 선택을 해제하면 GlassFish Server의 메모리 범위가 줄어듭니다.

DTrace

DTrace 사용 체크 박스가 선택되어 있고 Oracle Enterprise Manager DTrace Monitoring이 있는 경우 Solaris DTrace 기능을 사용할 수 있습니다. 이 옵션은 기본적으로 사용 안함으로 설정되어 있습니다.

사용 가능한 각 구성 요소에 대해 제공되는 정보는 다음과 같습니다.

구성 요소

GlassFish Server 구성 요소의 이름입니다.

모니터링 레벨

구성 요소에 대해 설정된 모니터링의 레벨입니다. 각 구성 요소에 대해 사용 가능한 레벨은 LOW, HIGH 및 OFF입니다. 기본 레벨은 끄기입니다. 이 릴리스에서는 LOW 및 HIGH 옵션에서 제공되는 모니터링 레벨 간에 차이가 없습니다.

구성 요소 레벨 설정 테이블에는 다음 옵션도 포함되어 있습니다.

레벨 변경

선택한 하나 이상의 구성 요소에 대해 모니터링 레벨을 변경하는 단추입니다.

레벨

모든 선택된 구성 요소에 적용되는 모니터링 레벨을 선택할 수 있는 드롭다운 목록입니다.

관련 작업

  • 모니터링 서비스를 구성하는 방법

  • 응용 프로그램 모니터링 데이터를 보는 방법

  • 서버 모니터링 데이터를 보는 방법

  • 리소스 모니터링 데이터를 보는 방법

관련 asadmin 명령

  • enable-monitoring(1)

  • disable-monitoring(1)

  • monitor(1)
