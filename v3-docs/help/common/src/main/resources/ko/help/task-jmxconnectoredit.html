JMX 커넥터를 편집하는 방법

관리 서비스를 사용하여 JSR-160 호환 원격 JMX 커넥터를 구성합니다.

JSR-160 호환 원격 JMX 커넥터는 도메인 관리를 위해 JConsole 등의 JMX 클라이언트와 JMX 서비스 간의 통신을 처리합니다.

  1. 탐색 트리에서 [구성] 노드를 확장합니다.

  2. [구성] 노드에서 [관리 서비스] 노드를 선택합니다.

    [JMX 커넥터 편집] 페이지가 열립니다.

  3. JMX 통신을 암호화하려면 [보안 사용] 체크 박스를 선택합니다.

    이 옵션은 기본적으로 사용 안함으로 설정되어 있습니다.

  4. [주소] 필드에 JMX 커넥터 서버 스텁이 등록된 이름 지정 서비스의 IP 주소를 입력합니다.

    이는 실제 JMX 통신을 수행하는 서버 소켓의 포트가 아닙니다. RMI 레지스트리가 시작되는 네트워크 인터페이스의 주소입니다. 시스템에 여러 개의 네트워크 인터페이스가 있는 경우 특정 인터페이스만 선택되도록 이 값을 수정합니다. 기본값은 0.0.0.0입니다.

  5. [포트] 필드에 이름 지정 서비스(RMI 레지스트리)가 RMI 클라이언트 접속을 수신하는 포트 번호를 입력합니다.

    이 이름 지정 서비스는 RMI 스텁을 다운로드하는 데만 사용합니다. 기본 포트가 사용 중인 경우 빈 포트가 사용됩니다. 유효한 값은 1 - 65535입니다. UNIX 시스템에서 1 - 1024 포트를 수신하는 소켓을 생성하려면 수퍼유저 권한이 필요합니다. 기본값은 8686입니다.

  6. [영역 이름] 필드에 특별 관리 영역을 나타내는 이름을 입력합니다.

    모든 인증이 이 영역에 의해 처리됩니다. 기본값은 admin-realm입니다.

  7. [추가 속성] 섹션에서 추가 속성을 지정합니다.

    GlassFish Server에서 기본적으로 몇 개의 속성을 지정합니다.

    • 속성을 추가하려면 [속성 추가] 단추를 누릅니다. 빈 행이 표시되면 [이름] 필드에 속성 이름을 입력하고 [값] 필드에 속성값을 입력합니다.

    • 속성을 수정하려면 속성의 [값] 필드를 편집합니다.

    • 속성을 삭제하려면 삭제할 속성의 [이름] 필드 왼쪽에 있는 체크 박스를 선택한 다음 [속성 삭제] 단추를 누릅니다.

  8. [저장]을 누릅니다.

  9. [보안 사용] 체크 박스를 선택한 경우 JMX 커넥터 SSL 설정을 편집하는 방법의 단계를 따릅니다.
