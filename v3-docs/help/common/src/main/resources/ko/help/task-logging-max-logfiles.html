회전된 로그 파일의 수 제한을 변경하는 방법

GlassFish Server에서 로그 파일을 회전할 때 server.log라는 빈 파일을 새로 만들고 이전 파일의 이름을 server.log_ date로 바꿉니다. 여기서 date는 파일이 회전된 날짜 및 시간입니다.

기본적으로 GlassFish Server에서는 회전된 로그 파일 수를 10으로 제한합니다. 이 제한에 도달하면 GlassFish Server에서 다음에 로그 파일을 회전할 때 가장 오래된 로그 파일이 삭제됩니다.

필요한 경우 회전된 로그 파일의 수 제한을 변경할 수 있습니다. 설정하는 새 제한은 액세스 로깅 및 서버 로그에 대한 회전된 로그 파일에 적용됩니다.

  • com.sun.enterprise.server.logging.max_history_files 시스템 등록 정보를 유지할 최대 회전된 로그 파일 수로 설정합니다.

    시스템 등록 정보를 설정하는 방법에 대한 내용은 시스템 등록 정보를 관리하는 방법를 참조하십시오.

    com.sun.enterprise.server.logging.max_history_files 시스템 등록 정보의 동작은 다음과 같습니다.

    • 등록 정보가 설정되지 않은 경우 GlassFish Server은(는) 최대 10개의 회전된 로그 파일을 유지합니다.

    • 등록 정보가 잘못된 수 또는 null로 설정된 경우 GlassFish Server은(는) 최대 10개의 회전된 로그 파일을 유지합니다.

    • 등록 정보가 0으로 설정된 경우 GlassFish Server은(는) 회전된 로그 파일을 유지하지 않습니다.

참조

  • 일반 로깅 설정을 구성하는 방법

  • 로그 수준을 구성하는 방법

  • DAS에 대한 로그 파일을 보는 방법

  • 모든 로그 파일의 ZIP 아카이브를 만드는 방법

  • 시스템 등록 정보를 관리하는 방법
