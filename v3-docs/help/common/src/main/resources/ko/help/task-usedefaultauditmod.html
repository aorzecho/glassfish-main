기본 감사 모듈을 사용하는 방법

default 감사 모듈은 인증 및 권한 부여 요청을 서버 로그 파일에 기록합니다. 로그 파일 위치 변경에 대한 자세한 내용은 일반 로깅 설정을 구성하는 방법을(를) 참조하십시오.

인증 로그 항목에는 다음 정보가 포함됩니다.

  • 인증을 시도한 사용자의 이름

  • 액세스 요청을 처리한 영역

  • 요청된 웹 모듈 URI 또는 EJB 구성 요소

  • 요청의 성공 또는 실패 여부

감사 로깅의 사용 여부에 관계없이 GlassFish Server은(는) 거부된 모든 인증 이벤트를 기록합니다.

권한 부여 로그 항목에는 다음 정보가 포함됩니다.

  • 인증된 사용자의 이름(있는 경우)

  • 요청된 웹 URI 또는 EJB 구성 요소

  • 요청의 성공 또는 실패 여부

  1. 탐색 트리에서 [구성] 노드를 확장합니다.

  2. Select the instance to configure:

    • To configure a particular instance, expand the instance's config node. For example, the default instance, server, expand the server-config node.

    • To configure the default settings for all instances, expand the default-config node.

  3. [구성] 노드에서 [보안] 노드를 선택합니다.

    [보안] 페이지가 열립니다.

  4. 감사 모듈 드롭다운 목록에서 default라는 감사 모듈을 선택합니다.

  5. [저장]을 누릅니다.

관련 항목

  • 감사 모듈을 생성하는 방법

  • 감사 모듈을 편집하는 방법

  • 감사 모듈을 삭제하는 방법

  • 감사 로깅을 사용 또는 사용 안함으로 설정하는 방법

  • 활성 감사 모듈을 설정하는 방법
