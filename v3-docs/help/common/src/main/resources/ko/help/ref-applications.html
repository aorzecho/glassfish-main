응용 프로그램

응용 프로그램 페이지에는 &ProductName에 배포된 응용 프로그램의 목록이 표시됩니다. 여기에서 이미 배포된 응용 프로그램을 보고 관리할 뿐만 아니라 더 많은 응용 프로그램을 배포할 수 있습니다.

각 응용 프로그램에 대해 제공되는 정보는 다음과 같습니다.

이름

응용프로그램 이름입니다.

이름에는 버전 ID(옵션)가 포함될 수 있으며, 이 ID는 이름 다음에 오고 콜론(:)으로 이름과 구분됩니다. 모듈 및 응용 프로그램 버전에 대한 자세한 내용은 Oracle GlassFish Server 3.1 Application Deployment Guide의 Module and Application Versions를 참조하십시오.

사용 가능 또는 상태

기본 서버 인스턴스(server)만 있는 경우 응용 프로그램이 활성화되면 체크 표시가 나타나고 비활성화되면 X가 표시됩니다.

클러스터 또는 다른 독립 실행형 서버 인스턴스가 있는 경우 응용 프로그램이 배포되는 대상을 보려면 대상에서 사용 가능을 선택합니다.

엔진

응용 프로그램에서 사용되는 컨테이너의 유형입니다. 컨테이너 유형에는 다음과 같은 것이 있습니다.

  • web

  • webservices

  • ejb

  • connector

  • appclient

  • weld(Java EE 플랫폼 응용 프로그램에 대한 컨텍스트 및 종속성 삽입용 컨테이너)

작업

구성 요소를 배포한 후 해당 구성 요소에 대해 수행할 수 있는 작업에 대한 링크입니다. 모든 구성 요소에 대해서는 재배포와 재로드가 표시되고 웹 응용 프로그램에 대해서는 시작이 표시됩니다.

배포된 응용 프로그램 테이블에는 다음 옵션도 포함되어 있습니다.

배포

응용 프로그램을 배포하는 버튼입니다.

배포 해제

하나 이상의 선택된 응용 프로그램을 배포 해제하는 버튼입니다.

활성화

기본 서버 인스턴스(server)만 있는 경우 하나 이상의 선택한 응용 프로그램을 활성화하는 버튼입니다.

비활성화

기본 서버 인스턴스(server)만 있는 경우 하나 이상의 선택한 응용 프로그램을 비활성화하는 버튼입니다.

필터

엔진별로 응용 프로그램을 필터링하는 드롭다운 목록입니다.

관련 작업

  • 배포된 응용 프로그램의 목록을 보는 방법

  • 웹 응용 프로그램을 배포하는 방법

  • 엔터프라이즈 응용 프로그램을 배포하는 방법

  • 응용 프로그램 클라이언트를 배포하는 방법

  • 커넥터 모듈을 배포하는 방법

  • EJB Jar를 배포하는 방법

  • 배포된 응용 프로그램을 편집하는 방법

  • 응용 프로그램의 설명자를 보는 방법

  • 응용 프로그램 대상을 보는 방법

  • 웹 서비스에 대한 종단점 정보 보기

  • 응용 프로그램을 시작하는 방법

  • 응용 프로그램 클라이언트를 시작하는 방법

  • 배포된 응용 프로그램을 재배포하는 방법

  • 배포된 응용 프로그램을 다시 로드하는 방법
