응용 프로그램

응용 프로그램 페이지에는 &ProductName에 배치된 응용 프로그램의 목록이 표시됩니다. 여기에서 이미 배치된 응용 프로그램을 보고 관리할 뿐만 아니라 더 많은 응용 프로그램을 배치할 수 있습니다.

각 응용 프로그램에 대해 제공되는 정보는 다음과 같습니다.

이름

응용 프로그램 이름입니다.

이름에는 버전 식별자(옵션)가 포함될 수 있으며, 이 식별자는 이름 다음에 오고 콜론(:)으로 이름과 구분됩니다. 모듈 및 응용 프로그램 버전에 대한 자세한 내용은 Oracle GlassFish Server Application Deployment Guide의 Module and Application Versions를 참조하십시오.

사용 또는 상태

기본 서버 인스턴스(server)만 있는 경우 응용 프로그램이 사용으로 설정되면 체크 표시가 나타나고 사용 안함으로 설정되면 X가 표시됩니다.

클러스터 또는 다른 독립형 서버 인스턴스가 있는 경우 응용 프로그램이 배치되는 대상을 보려면 대상에서 사용을 선택합니다.

엔진

응용 프로그램에서 사용되는 컨테이너의 유형입니다. 컨테이너 유형에는 다음과 같은 것이 있습니다.

  • web

  • webservices

  • ejb

  • connector

  • appclient

  • weld(Java EE 플랫폼 응용 프로그램에 대한 Contexts and Dependency Injection 컨테이너)

작업

배치 후 구성 요소에 대해 수행할 수 있는 작업에 대한 링크로, 재배치 및 재로드(모든 구성 요소의 경우), 실행(웹 응용 프로그램 및 응용 프로그램 클라이언트의 경우), 클라이언트 스텁 다운로드(응용 프로그램 클라이언트의 경우)가 있습니다.

배치된 응용 프로그램 테이블에는 다음 옵션도 포함되어 있습니다.

배치

응용 프로그램을 배치하는 단추입니다.

배치 해제

선택한 하나 이상의 응용 프로그램을 배치 해제하는 단추입니다.

사용

기본 서버 인스턴스(server)만 있는 경우 선택한 하나 이상의 응용 프로그램을 사용으로 설정하는 단추입니다.

사용 안함

기본 서버 인스턴스(server)만 있는 경우 선택한 하나 이상의 응용 프로그램을 사용 안함으로 설정하는 단추입니다.

필터

엔진별로 응용 프로그램을 필터링하는 드롭다운 목록입니다.

관련 작업

  • 배치된 응용 프로그램 목록을 보는 방법

  • 웹 응용 프로그램을 배치하는 방법

  • 엔터프라이즈 응용 프로그램을 배치하는 방법

  • 응용 프로그램 클라이언트를 배치하는 방법

  • 커넥터 모듈을 배치하는 방법

  • EJB Jar를 배치하는 방법

  • 배치된 응용 프로그램을 편집하는 방법

  • 응용 프로그램 기술자를 보는 방법

  • 응용 프로그램 대상을 보는 방법

  • 웹 서비스 끝점 정보를 보는 방법

  • 응용 프로그램을 실행하는 방법

  • 응용 프로그램 클라이언트를 실행하는 방법

  • 응용 프로그램 클라이언트 스텁을 다운로드하는 방법

  • 배치된 응용 프로그램을 재배치하는 방법

  • 배치된 응용 프로그램을 재로드하는 방법

  • 응용 프로그램 범위 리소스를 보는 방법

  • 응용 프로그램 범위 리소스를 수정하는 방법
