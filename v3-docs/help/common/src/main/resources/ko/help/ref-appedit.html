응용 프로그램 편집

[응용 프로그램 편집] 페이지에서는 기존 응용 프로그램을 편집할 수 있습니다.

[응용 프로그램 편집] 페이지에는 응용 프로그램 유형에 따라 다음 옵션 중 일부 또는 전부가 포함되어 있습니다.

이름

편집 중인 응용 프로그램의 이름을 표시하는 읽기 전용 필드입니다.

이름에는 버전 식별자(옵션)가 포함될 수 있으며, 이 식별자는 이름 다음에 오고 콜론(:)으로 이름과 구분됩니다. 모듈 및 응용 프로그램 버전에 대한 자세한 내용은 Oracle GlassFish Server Application Deployment Guide의 Module and Application Versions를 참조하십시오.

상태

사용 체크 박스가 선택되어 있으면 응용 프로그램을 사용할 수 있습니다. 이 옵션은 기본적으로 선택됩니다.

컨텍스트 루트

웹 응용 프로그램의 경우 응용 프로그램 경로를 지정합니다. 웹 응용 프로그램의 URL에서 포트 번호 바로 뒤에 컨텍스트 루트가 옵니다(http://host:port/context-root/...). 컨텍스트 루트는 슬래시로 시작해야 합니다(예: /hello).

가용성

[사용] 체크 박스를 선택한 경우 고가용성이 웹 세션 및 SFSB(상태 유지 세션 Bean) 체크포인트 지정과 잠재적인 비활성화에 대해 사용으로 설정됩니다. false(기본값)로 설정된 경우 지정한 응용 프로그램, 웹 응용 프로그램 또는 EJB 모듈에 대해 모든 웹 세션 저장과 SFSB 체크포인트 지정이 사용 안함으로 설정됩니다 true로 설정된 경우 지정된 응용 프로그램 또는 모듈이 고가용성에 대해 사용으로 설정됩니다. 서버 및 컨테이너 레벨과 같은 더 높은 레벨에서 고가용성이 구성 및 사용으로 설정되어 있는 경우에만 이 옵션을 true로 설정합니다.

이 옵션은 기본 서버 인스턴스(server) 외에 클러스터나 독립형 서버 인스턴스가 있는 경우에 표시됩니다.

가상 서버

이 응용 프로그램과 연관된 가상 서버입니다.

가상 서버 옵션은 기본 서버 인스턴스(server)만 있는 경우에 표시됩니다. 클러스터 또는 다른 독립형 서버 인스턴스가 있는 경우에는 배치 후에 가상 서버를 선택할 수 있습니다. [응용 프로그램 편집] 페이지로 이동하고 [대상] 탭을 선택한 다음 원하는 대상에 대해 [가상 서버 관리]를 선택합니다.

Java Web Start

(일부 응용 프로그램 유형의 경우) 사용 체크 박스가 선택되어 있으면 응용 프로그램에서 Java Web Start 소프트웨어를 사용합니다. Java Web Start를 사용하면 전용 JVM(Java Virtual Machine)에서 실행되는 Java 응용 프로그램을 브라우저와는 상관 없이 배치할 수 있습니다.

모듈 및 구성 요소

응용 프로그램의 모듈과 관련 엔진의 이름을 표시하는 테이블입니다. 웹 서비스 끝점에서 끝점 보기를 선택하여 웹 서비스 끝점 정보 페이지를 표시할 수 있습니다. 응용 프로그램 클라이언트의 경우, 실행을 선택하여 응용 프로그램 클라이언트 실행 페이지를 표시하거나, 클라이언트 스텁 다운로드를 선택하여 클라이언트 스텁을 다운로드할 수 있습니다.
