응용 프로그램 또는 모듈 재배포

응용 프로그램 또는 모듈 재배포 페이지에서는 이전에 배포한 응용 프로그램을 재배포할 수 있습니다.

응용 프로그램 또는 모듈 재배포 페이지에는 다음 옵션이 포함되어 있습니다.

위치

재배포 중인 응용 프로그램의 새 아카이브 위치입니다.

다음 옵션에서는 아카이브에서 액세스할 수 있는 위치 및 아카이브가 파일 또는 디렉토리인지 여부를 지정합니다.

서버에 업로드할 패키지화된 파일

아카이브는 클라이언트 시스템에 상주하거나 클라이언트 시스템에서 액세스할 수 있는 파일에 있습니다.

클라이언트 시스템은 브라우저를 통해 관리 콘솔을 보는 호스트입니다.

GlassFish Server에서 액세스할 수 있는 패키지화된 로컬 파일 또는 디렉토리

아카이브는 서버 시스템에 상주하는 파일이거나 확장된 디렉토리의 압축 해제된 응용 프로그램입니다.

서버 시스템은 GlassFish Server 도메인 관리 서버를 실행하는 호스트입니다.

응용 프로그램 이름

재배포 중인 응용 프로그램의 이름을 표시하는 읽기 전용 필드입니다.

이름에는 버전 ID(옵션)가 포함될 수 있으며, 이 ID는 이름 다음에 오고 콜론(:)으로 이름과 구분됩니다. 모듈 및 응용 프로그램 버전에 대한 자세한 내용은 Oracle GlassFish Server 3.1 Application Deployment Guide의 Module and Application Versions를 참조하십시오.

JSP 사전 컴파일

이 옵션을 선택하면 JSP(JavaServer Pages) 파일이 사전 컴파일됩니다. 이 옵션을 사용하지 않도록 설정된 경우 JSP 파일은 런타임에 처음으로 액세스될 때 컴파일됩니다. 이 옵션은 기본적으로 비활성화됩니다.

검증자 실행

이 옵션을 선택하면 배포 전에 배포 설명자가 검증됩니다. 검증에 실패하면 배포가 수행되지 않습니다. 검증자는 배포 설명자의 구조와 내용을 검사합니다. 큰 응용 프로그램의 검증에는 많은 시간이 걸릴 수 있습니다. 이 옵션은 기본적으로 비활성화됩니다.

업데이트 도구에서 검증자 패키지를 설치해야 합니다. 그렇지 않으면 경고가 기록되고 이 옵션이 무시됩니다.

Java Web Start

이 옵션을 선택하면 응용 프로그램 클라이언트 모듈에 대해 Java Web Start 액세스가 허용됩니다. 이 옵션은 기본적으로 비활성화됩니다.

상태 유지

이 옵션은 웹 세션, SFSB 인스턴스 및 지속적으로 만들어진 EJB 타이머가 재배포 간에 유지되는지를 제어합니다.

이 옵션은 기본적으로 비활성화됩니다. 이 옵션은 server라는 기본 서버 인스턴스에서만 지원됩니다. 다른 대상에 대해서는 지원되지 않고 무시됩니다.

재배포 간에 응용 프로그램을 몇 가지 변경하면 이 기능이 제대로 작동하지 않습니다. 예를 들면, SFSB bean 클래스에서 인스턴스 변수 집합을 변경하지 마십시오.

웹 응용 프로그램에서 이 기능은 glassfish-web-app.xml 파일에 있는 session-manager 요소의 persistence-type 속성이 file인 경우에만 적용할 수 있습니다.

상태 유지 세션 빈 인스턴스에서 고가용성이 없는 지속성 유형이 서버에서 설정되므로(SFSB 지속성 유형 옵션) 기본값이자 권장 값인 file로 설정해야 합니다.

활성 웹 세션, SFSB 인스턴스 또는 EJB 타이머를 보존 또는 복원하지 못할 경우 재배포가 완료될 때 이들을 모두 사용할 수 없습니다. 그러나 재배포가 계속되고 경고가 기록됩니다.

활성 상태 데이터를 보존하기 위해 GlassFish Server은(는) 데이터를 일련화하여 메모리에 저장합니다. 데이터를 복원하기 위해 새로 재배포한 응용 프로그램의 클래스 로더에서는 이전에 저장된 데이터를 일련화 해제합니다.

응용 프로그램 범위 자원 보존

선택하면 응용 프로그램 범위 자원을 보존하고 재배포 중에 그러한 자원을 복원합니다. 이 옵션은 기본적으로 선택되어 있지 않습니다.
