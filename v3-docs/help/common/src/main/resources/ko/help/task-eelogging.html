일반 로깅 설정을 구성하는 방법

  1. 탐색 트리에서 구성 노드를 확장합니다.
  2. 구성 노드에서 로깅 설정을 구성할 서버 인스턴스나 클러스터 구성을 누릅니다.

    구성 페이지가 열립니다.

  3. 구성 페이지에서 로깅 설정을 누릅니다.

    선택한 구성 대상에 대한 로깅 설정 페이지가 열립니다.

  4. 서버 로그 이외에 syslog 기능으로 로깅 출력을 보내려면 시스템 로그에 쓰기 사용 가능 확인란을 선택합니다.

    이 기능은 Solaris 및 Linux 시스템에서만 사용할 수 있습니다. 이 옵션은 기본적으로 비활성화됩니다.

  5. 로그 파일 필드에서 서버 로그 파일의 대체 이름 또는 위치를 입력합니다.

    기본 위치는 ${com.sun.aas.instanceRoot}/logs/server.log 입니다.

  6. 로그 처리기 필드에 사용자 정의 로그 처리기의 절대 클래스 이름을 입력합니다.

    사용자 정의 로그 처리기를 사용하면 로그를 server.log 또는 syslog 이외의 대상에 보낼 수 있습니다. 사용자 정의 처리기에서 java.util.logging.ConsoleHandler(JSR 047 호환 API)를 확장해야 합니다. GlassFish Server 클래스 경로에 처리기 클래스를 넣어 서버 시작 시 처리기가 설치되도록 합니다.

  7. 파일 회전 제한 필드에 서버 로그 파일이 회전되기 전에 도달해야 하는 크기(바이트)를 입력합니다.

    서버 로그가 지정한 크기에 도달하면 GlassFish Server에서는 server.log라는 이름의 새로운 빈 파일을 만들고 이전 파일의 이름을 server.log_date로 변경합니다. 여기서 date는 파일이 회전된 날짜 및 시간입니다. 기본값은 2MB입니다. 최소 제한 값은 500KB입니다. 이보다 더 낮은 값을 지정하면 파일 크기가 500KB에 도달할 때 파일이 회전합니다. 로그 파일 회전을 해제하려면 값을 0으로 설정합니다.

  8. 파일 회전 시간 제한 필드에서 서버 로그를 회전해야 하는 시간(분)을 입력합니다.

    기본값은 0이며 이는 파일 크기가 파일 회전 제한 필드에 지정된 크기에 도달할 때 파일이 회전한다는 것을 의미합니다.

    주 - 1분 이상을 지정하면 파일 회전 제한 필드에 지정된 크기 제한보다 시간 제한이 더 높은 우선 순위를 가집니다.

  9. 플러시 빈도 필드에서 대기열에서 서버 로그에 한 번에 기록되는 메시지의 최대 수를 입력합니다.

    기본값은 1입니다.

  10. 저장을 누릅니다.

    주 - 로깅이 활성화된 구성에 따라 서버를 다시 시작해야 할 수도 있습니다.

    로그 파일 이름이 기본 설정에서 변경된 경우에도 서버를 다시 시작해야 합니다. 서버를 다시 시작해야 하는 경우 관리 콘솔의 오른쪽 맨 위에 다시 시작해야 함 메시지가 표시됩니다.

참조

  • 로그 수준을 구성하는 방법

  • 회전된 로그 파일의 수 제한을 변경하는 방법

  • DAS에 대한 로그 파일을 보는 방법

  • 모든 로그 파일의 ZIP 아카이브를 만드는 방법
