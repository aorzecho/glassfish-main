라이프사이클 모듈

라이프사이클 모듈 페이지에서는 라이프사이클 모듈을 구성할 수 있습니다.

라이프사이클 모듈은 서버의 라이프사이클에서 한 개 이상의 이벤트로 실행되면 작업을 수행합니다. 발생할 수 있는 트리거 서버 이벤트에는 초기화, 시작, 요청 서비스 준비 및 종료 등이 있습니다. 라이프사이클 모듈은 Java EE 사양의 일부가 아니지만 GlassFish Server의 개선 사항입니다.

각 라이프사이클 모듈에 대해 제공되는 정보는 다음과 같습니다.

이름

라이프사이클 모듈의 이름입니다.

사용 가능 또는 상태

기본 서버 인스턴스(server)만 있는 경우 응용 프로그램이 활성화되면 체크 표시가 나타나고 비활성화되면 X가 표시됩니다.

클러스터 또는 다른 독립 실행형 서버 인스턴스가 있는 경우 응용 프로그램이 배포되는 대상을 보려면 모든 대상에서 사용 가능을 선택합니다.

로드 순서

라이프사이클 모듈을 로드해야 할 상대적 순서를 지정하는 값입니다.

라이프사이클 모듈 테이블에는 다음 옵션도 포함되어 있습니다.

새로 만들기

새 라이프사이클 모듈을 만드는 버튼입니다.

삭제

하나 이상의 선택된 라이프사이클 모듈을 삭제하는 버튼입니다.

활성화

하나 이상의 선택된 라이프사이클 모듈을 사용하도록 설정하는 버튼입니다. 기본 서버 인스턴스(server)만 있는 경우에 표시됩니다.

비활성화

하나 이상의 선택된 라이프사이클 모듈을 사용할 수 없도록 설정하는 버튼입니다. 기본 서버 인스턴스(server)만 있는 경우에 표시됩니다.

관련 작업

  • 라이프사이클 모듈을 만드는 방법

  • 라이프사이클 모듈을 편집하는 방법

  • 라이프사이클 모듈을 삭제하는 방법

관련 asadmin 명령

  • create-lifecycle-module(1)

  • list-lifecycle-modules(1)

  • delete-lifecycle-module(1)
