アプリケーションまたはモジュールの配備

アプリケーションを配備するには、「アプリケーションまたはモジュールを配備」ページを使用します。

「アプリケーションまたはモジュールを配備」ページには、すべてのアプリケーションで次のオプションが表示されます。追加のオプションは、アプリケーションの「位置」フィールドを指定したあとに表示されます。

位置

配備するアプリケーションのアーカイブの場所。

次のオプションは、アーカイブにアクセス可能な場所とアーカイブがファイルであるかディレクトリであるかを指定します。

サーバーにアップロードされるパッケージファイル

このアーカイブは、クライアントマシンにあるファイル、またはクライアントマシンからアクセス可能なファイルです。

クライアントマシンとは、ブラウザを介して 管理コンソール を表示しているホストです。

GlassFish サーバー からアクセス可能なローカルのパッケージファイルまたはディレクトリ

このアーカイブは、サーバーマシンにあるファイル、または展開されたディレクトリにあるパッケージ化されていないアプリケーションです。

サーバーマシンは、GlassFish サーバー のドメイン管理サーバーを実行しているホストです。

タイプ

アプリケーションのタイプ。次のタイプを指定できます。

  • Web アプリケーション

  • エンタープライズアプリケーション

  • アプリケーションクライアント

  • コネクタモジュール

  • EJB JAR

  • その他

ターゲット

アプリケーションまたはモジュールの配備先であるクラスタとスタンドアロンインスタンス。必要なターゲットを「選択したターゲット」列に移動するには、「追加」ボタンまたは「すべて追加」ボタンを使用します。不要なターゲットを「利用可能なターゲット」列に移動するには、「削除」ボタンまたは「すべて削除」ボタンを使用します。

このオプションは、ドメイン内にクラスタまたはスタンドアロンインスタンスが作成されている場合にのみ表示されます。

各アプリケーションのタイプの追加オプションについては、次の節で説明します。

  • 「Web アプリケーションの追加オプション」

  • 「エンタープライズアプリケーションの追加オプション」

  • 「アプリケーションクライアントの追加オプション」

  • 「コネクタモジュールの追加オプション」

  • 「EJB JAR の追加オプション」

Web アプリケーションの追加オプション

アプリケーションのタイプが Web アプリケーションの場合、次のオプションが表示されます。

コンテキストルート

アプリケーションのパス。Web アプリケーションの URL では、コンテキストルートはポート番号の直後に続きます (http://host:port /context-root/...)。たとえば /hello のように、コンテキストルートがスラッシュで始まるようにしてください。

アプリケーション名

アプリケーションの名前。

名前には、オプションのバージョン ID を含めることができます。このバージョン ID は名前のあとに続き、コロン (:) で名前と区切ります。バージョン ID は、文字または数値で始まる必要があります。バージョン ID には、英数字とアンダースコア (_)、ダッシュ (-)、およびピリオド (.) を含めることができます。モジュールとアプリケーションのバージョンについては、『Oracle GlassFish Server 3.1 Application Deployment Guide』の「Module and Application Versions」を参照してください。

Virtual Servers

このアプリケーションと関連付けられている仮想サーバー。

「仮想サーバー」オプションは、デフォルトサーバーインスタンス server だけが存在する場合に表示されます。クラスタまたはその他のスタンドアロンサーバーインスタンスが存在する場合は、配備後、仮想サーバーを選択できます。「アプリケーションを編集」ページに移動して「ターゲット」タブを選択し、目的のターゲットに「仮想サーバーの管理」を選択してください。

ステータス

このオプションを選択すると、アプリケーションが有効になります。これは標準でオンになっています。

可用性

「有効」チェックボックスを選択すると、Web セッション、ステートフルセッション Bean (SFSB) チェックポイント設定、パッシベーションなどに備えて高可用性が有効になります。false (デフォルト) に設定すると、すべての Web セッションの保存と SFSB のチェックポイント設定が、指定されたアプリケーション、Web アプリケーション、または EJB モジュールで無効になります。true に設定すると、指定されたアプリケーションまたはモジュールで高可用性が有効になります。このオプションは、高可用性がサーバーレベルやコンテナレベルなどのより高いレベルで設定および有効化されている場合のみ、true に設定します。

このオプションは、デフォルトのサーバーインスタンス (server) のほかにクラスタまたはスタンドアロンサーバーインスタンスが存在する場合に表示されます。

JSP をプリコンパイル

このオプションを選択すると、JavaServer Pages (JSP) ファイルがプリコンパイルされます。このオプションが無効な場合、JSP ファイルは初めてアクセスされた実行時にコンパイルされます。このオプションは、デフォルトでは無効になっています。

ベリファイアを実行

このオプションを選択すると、配備の前に配備記述子が検証されます。検証が失敗すると、配備は実行されません。ベリファイアは、配備記述子の構造とコンテンツを調べます。大きなアプリケーションの検証は時間がかかる可能性があります。このオプションは、デフォルトでは無効になっています。

更新ツールからベリファイアパッケージがインストールされている必要があり、このパッケージが存在しないと警告が記録され、オプションは無視されます。

強制的に再配備

このオプションを選択すると、アプリケーションがすでに配備されている場合はそれが配備され直されます。このオプションを選択しないと、すでに配備されているアプリケーションを配備しようとすると、エラーになります。このオプションは、デフォルトでは無効になっています。

状態の維持

このオプションは、Web セッション、SFSB インスタンス、および持続的に作成された EJB タイマーを、次の再配備でも保持するかどうかを制御します。

このオプションは、デフォルトでは無効になっています。このオプションがサポートされるのは、server という名前が付いたデフォルトのサーバーインスタンス上だけです。ほかのターゲットではサポートされず、無視されます。

再配備までの間に行われるアプリケーションの変更によっては、この機能が正常に動作しなくなることがあります。たとえば、SFSB Bean クラスでインスタンス変数セットを変更することは避けてください。

Web アプリケーションの場合、この機能を適用できるのは glassfish-web-app.xml ファイルで session-manager 要素の persistence-type 属性が file のときだけです。

ステートフルセッション Bean インスタンスの場合、高可用性を提供しない持続性タイプはサーバー (「SFSB 持続性のタイプ」オプション) 内で設定し、file (デフォルトで、推奨値) に設定する必要があります。

アクティブな Web セッション、SFSB インスタンス、または EJB タイマーが保持または復元されないと、再配備が完了したときにこれらすべてを使用できません。ただし、再配備を続けると警告がログに記録されます。

アクティブな状態データを保持するために、GlassFish サーバー はデータを直列化し、メモリーに保存します。データを復元するために、新たに配備されたアプリケーションのクラスローダーは以前に保存されたデータを直列化復元します。

アプリケーションを対象範囲とするリソースの維持

このオプションを選択すると、アプリケーションスコープのリソースが保持され、再配備中に復元されます。デフォルトでは、このオプションは選択されません。

ライブラリ

このモジュールまたはアプリケーションに固有のライブラリ JAR ファイルのコンマ区切りのリスト。絶対パスと相対パスのどちらでも指定できます。相対パスは、domain-dir/lib/applibs からの相対で指定します。絶対パスの場合は、ドメイン管理サーバー (DAS) がパスにアクセスできる必要があります。つまり、domain-dir 以下にある必要があります。ライブラリは、指定した順にアプリケーションで利用可能になります。

Java EE 5 では EAR ファイル内のさまざまなモジュールから JAR ファイルを参照できましたが、Java EE 6 プラットフォーム仕様は Java EE 5 よりも厳しい要件を課します。特に、アプリケーションクライアントは、標準 Java SE メカニズム (拡張子など) または Java EE library-directory メカニズムを使用した参照が行われない限り、EJB JAR ファイルまたは EAR ファイル内のほかの JAR ファイルにアクセスできません。このボックスを選択すると、これらの Java EE 6 制限が排除されます。

「有効」チェックボックスを選択すると、Web セッション、ステートフルセッション Bean (SFSB) チェックポイント設定、パッシベーションなどに備えて高可用性が有効になります。false (デフォルト) に設定すると、すべての Web セッションの保存と SFSB のチェックポイント設定が、指定されたアプリケーション、Web アプリケーション、または EJB モジュールで無効になります。true に設定すると、指定されたアプリケーションまたはモジュールで高可用性が有効になります。このオプションは、高可用性がサーバーレベルやコンテナレベルなどのより高いレベルで設定および有効化されている場合のみ、true に設定します。

このオプションは、デフォルトのサーバーインスタンス (server) のほかにクラスタまたはスタンドアロンサーバーインスタンスが存在する場合に表示されます。

ベリファイアを実行

このオプションを選択すると、配備の前に配備記述子が検証されます。検証が失敗すると、配備は実行されません。ベリファイアは、配備記述子の構造とコンテンツを調べます。大きなアプリケーションの検証は時間がかかる可能性があります。このオプションは、デフォルトでは無効になっています。

更新ツールからベリファイアパッケージがインストールされている必要があり、このパッケージが存在しないと警告が記録され、オプションは無視されます。

互換性

このオプションを選択すると、GlassFish サーバー v3 リリース (3.1 など) で適用される厳格な Java EE 6 要件の代わりに、GlassFish サーバー v2 の JAR 可視性要件がアプリケーションに使用されます。デフォルトでは、このオプションは選択されません。

Java EE 5 では EAR ファイル内のさまざまなモジュールから JAR ファイルを参照できましたが、Java EE 6 プラットフォーム仕様は Java EE 5 よりも厳しい要件を課します。特に、アプリケーションクライアントは、標準 Java SE メカニズム (拡張子など) または Java EE library-directory メカニズムを使用した参照が行われない限り、EJB JAR ファイルまたは EAR ファイル内のほかの JAR ファイルにアクセスできません。このボックスを選択すると、これらの Java EE 6 制限が排除されます。

強制的に再配備

このオプションを選択すると、アプリケーションがすでに配備されている場合はそれが配備され直されます。このオプションを選択しないと、すでに配備されているアプリケーションを配備しようとすると、エラーになります。このオプションは、デフォルトでは無効になっています。

状態の維持

このオプションは、Web セッション、SFSB インスタンス、および持続的に作成された EJB タイマーを、次の再配備でも保持するかどうかを制御します。

このオプションは、デフォルトでは無効になっています。このオプションがサポートされるのは、server という名前が付いたデフォルトのサーバーインスタンス上だけです。ほかのターゲットではサポートされず、無視されます。

再配備までの間に行われるアプリケーションの変更によっては、この機能が正常に動作しなくなることがあります。たとえば、SFSB Bean クラスでインスタンス変数セットを変更することは避けてください。

Web アプリケーションの場合、この機能を適用できるのは glassfish-web-app.xml ファイルで session-manager 要素の persistence-type 属性が file のときだけです。

ステートフルセッション Bean インスタンスの場合、高可用性を提供しない持続性タイプはサーバー (「SFSB 持続性のタイプ」オプション) 内で設定し、file (デフォルトで、推奨値) に設定する必要があります。

アクティブな Web セッション、SFSB インスタンス、または EJB タイマーが保持または復元されないと、再配備が完了したときにこれらすべてを使用できません。ただし、再配備を続けると警告がログに記録されます。

アクティブな状態データを保持するために、GlassFish サーバー はデータを直列化し、メモリーに保存します。データを復元するために、新たに配備されたアプリケーションのクラスローダーは以前に保存されたデータを直列化復元します。

アプリケーションを対象範囲とするリソースの維持

このオプションを選択すると、アプリケーションスコープのリソースが保持され、再配備中に復元されます。デフォルトでは、このオプションは選択されません。

ライブラリ

このモジュールまたはアプリケーションに固有のライブラリ JAR ファイルのコンマ区切りのリスト。絶対パスまたは相対パスで指定します。相対パスは、domain-dir/lib/applibs からの相対で指定します。絶対パスの場合は、ドメイン管理サーバー (DAS) がパスにアクセスできる必要があります。つまり、domain-dir 以下にある必要があります。ライブラリは、指定した順にアプリケーションで利用可能になります。
