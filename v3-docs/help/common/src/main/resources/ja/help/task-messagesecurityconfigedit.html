メッセージ・セキュリティ構成を編集する

GlassFish ServerにデプロイされたWebサービス・エンド・ポイントのメッセージ・セキュリティを有効にするには、サーバー側でデフォルトで使用されるプロバイダを指定する必要があります。メッセージ・セキュリティのデフォルトのプロバイダを有効にする場合、GlassFish ServerにデプロイされたWebサービスのクライアントが使用するプロバイダも有効にする必要があります。クライアントが使用するプロバイダを有効にする方法については、アプリケーション・クライアントのメッセージ・セキュリティを有効にするに記載されています。

デプロイ済エンド・ポイントからのWebサービス呼出しに対してメッセージ・セキュリティを有効にするには、デフォルト・クライアント・プロバイダを指定する必要があります。GlassFish Serverのデフォルト・クライアント・プロバイダを有効にした場合、GlassFish Serverにデプロイされたエンド・ポイントから呼び出されるすべてのサービスが、メッセージ・レイヤーセキュリティと互換性を持つように構成されていることを確認する必要があります。

  1. ナビゲーション・ツリーで、「構成」ノードを展開します。

  2. 「セキュリティ」ノードを展開します。

  3. 「メッセージ・セキュリティ」ノードを選択します。

    「メッセージ・セキュリティ構成」ページが開きます。

  4. 「メッセージ・セキュリティ構成」ページで、編集する構成の認証レイヤー名をクリックします。

    デフォルトでは、構成済のメッセージ・セキュリティ構成はSOAPのみです。

    「メッセージ・セキュリティ構成の編集」ページが開きます。

  5. 「メッセージ・セキュリティ構成の編集」ページで、「デフォルト・プロバイダ」ドロップダウン・リストからサーバー側プロバイダを選択します。

    デフォルト・サーバー・プロバイダ(デフォルト・プロバイダ)は、特定のサーバー・プロバイダを識別しないアプリケーションに対して呼び出されるサーバー・プロバイダです。ServerProviderまたはXWS_ServerProviderを指定するか、何も指定しません。デフォルトでは、プロバイダは選択されていません。したがって、GlassFish Serverはサーバー側でメッセージ・セキュリティ・プロバイダを呼び出しません。通常は、ServerProviderをデフォルト・プロバイダとして選択してください。

  6. 「デフォルト・クライアント・プロバイダ」ドロップダウン・リストから、クライアント・プロバイダを選択します。

    デフォルト・クライアント・プロバイダは、特定のクライアント・プロバイダを識別しないアプリケーションに対して呼び出されるクライアント・プロバイダです。ClientProviderまたはXWS_ClientProviderを指定するか、何も指定しません。デフォルトでは、プロバイダは選択されていません。したがって、GlassFish Serverはクライアント側でメッセージ・セキュリティ・プロバイダを呼び出しません。通常は、デフォルト・クライアント・プロバイダには何も選択しないでください。デフォルト・プロバイダとメッセージ保護ポリシーを有効にして、GlassFish ServerにデプロイされたWebサービス・エンド・ポイントから呼び出されるWebサービスに適用する場合、ClientProviderを選択します。

  7. 「保存」をクリックします。

  8. 有効にしたクライアント・プロバイダまたはサーバー・プロバイダのメッセージ保護ポリシーを変更する方法は、メッセージ・セキュリティ・プロバイダ構成を編集するを参照してください。
