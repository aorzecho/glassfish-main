メッセージセキュリティー設定を編集する

GlassFish サーバー に配備された Web サービスエンドポイントのメッセージセキュリティーを有効にするには、サーバー側でデフォルトで使用されるプロバイダを指定する必要があります。メッセージセキュリティーのデフォルトのプロバイダを有効にする場合、GlassFish サーバー に配備された Web サービスのクライアントが使用するプロバイダも有効にする必要があります。クライアントが使用するプロバイダを有効にする方法については、「アプリケーションクライアントのメッセージセキュリティーを有効にする」を参照してください。

配備済みエンドポイントからの Web サービス呼び出しに対してメッセージセキュリティーを有効にするには、デフォルトクライアントプロバイダを指定する必要があります。GlassFish サーバー のデフォルトクライアントプロバイダを有効にした場合、GlassFish サーバー に配備されたエンドポイントから呼び出されるすべてのサービスが、メッセージ層セキュリティーと互換性を持つように設定されていることを確認する必要があります。

  1. ナビゲーションツリーで、「設定」ノードを展開します。
  2. 「セキュリティー」ノードを開きます。
  3. 「メッセージセキュリティー」ノードを選択します。

    「メッセージセキュリティー設定」ページが開きます。

  4. 「メッセージセキュリティー設定」ページで、編集する設定の認証層名をクリックします。

    デフォルトでは、設定済みのメッセージセキュリティー設定は SOAP だけです。

    「メッセージセキュリティ設定の編集」ページが開きます。

  5. 「メッセージセキュリティ設定の編集」ページで、「デフォルトプロバイダ」ドロップダウンリストからサーバー側プロバイダを選択します。

    デフォルトサーバープロバイダ (デフォルトプロバイダ) は、特定のサーバープロバイダを識別しないアプリケーションに対して呼び出されるサーバープロバイダです。ServerProvider または XWS_ServerProvider を指定するか、何も指定しません。デフォルトでは、プロバイダは選択されていません。したがって、GlassFish サーバー はサーバー側でメッセージセキュリティープロバイダを呼び出しません。通常は、ServerProvider をデフォルトプロバイダとして選択してください。

  6. 「デフォルトクライアントプロバイダ」ドロップダウンリストから、クライアントプロバイダを選択します。

    デフォルトクライアントプロバイダは、特定のクライアントプロバイダを識別しないアプリケーションに対して呼び出されるクライアントプロバイダです。ClientProvider または XWS_ClientProvider を指定するか、何も指定しません。デフォルトでは、プロバイダは選択されていません。したがって、GlassFish サーバー はクライアント側でメッセージセキュリティープロバイダを呼び出しません。通常は、デフォルトクライアントプロバイダには何も選択しないでください。デフォルトプロバイダとメッセージ保護ポリシーを有効にして、GlassFish サーバー に配備された Web サービスエンドポイントから呼び出される Web サービスに適用する場合、ClientProvider を選択します。

  7. 「保存」をクリックします。
  8. 有効にしたクライアントプロバイダまたはサーバープロバイダのメッセージ保護ポリシーを変更する方法については、「メッセージセキュリティープロバイダ設定を編集する」を参照してください。
