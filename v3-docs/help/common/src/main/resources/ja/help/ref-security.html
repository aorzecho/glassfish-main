セキュリティー

「セキュリティー」ページを使用し、選択した GlassFish Server インスタンスまたはクラスタのセキュリティープロパティーを設定します。

「セキュリティー」ページには、次のオプションが含まれます。

設定名

このページの設定を適用する設定の名前。 このフィールドは読み取り専用です。

セキュリティーマネージャー

このオプションを選択すると、ドメインのセキュリティーマネージャーが有効になります。このオプションは、デフォルトでは無効になっています。

このオプションを有効にすると、JVM オプションの -Djava.security.manager が GlassFish Server の JVM 設定に追加されます。「JVM オプションを設定する」を参照してください。この変更を有効にするには、サーバーを再起動する必要があります。

すべてのアプリケーションについて正しいアクセス権を与えていることを確認してください。セキュリティーマネージャーを無効にすると、パフォーマンスを向上させることができます

監査ログ

このオプションを有効にすると、サーバーは「監査モジュール」設定で指定されているすべての監査モジュールをロードし、実行します。このオプションを無効にすると、サーバーは監査モジュールにアクセスしません。このオプションは、デフォルトでは無効になっています。

デフォルトレルム

サーバーが認証用に使用する有効な (デフォルト) レルム。アプリケーションでは、配備記述子によって別のレルムが指定されないかぎり、このレルムが使用されます。設定されているすべてのレルムがこのリストに表示されます。デフォルト値は file です。

デフォルトの主体

デフォルトのユーザー名を指定します。ほかの主体が指定されていないとき、サーバーはこのユーザー名を使用します。このフィールドに値を入力する場合は、「デフォルト主体のパスワード」フィールド内の、対応するパスワードの入力が必要になります。

通常のサーバーの操作にデフォルト主体は必要ありません。

デフォルト主体のパスワード

「デフォルト主体」フィールドで指定したデフォルト主体のパスワード。通常のサーバーの操作にデフォルト主体は必要ありません。

JACC

設定されている JACC プロバイダのクラス名。デフォルト値は default です。

Audit Modules

監査ログが有効な場合に、監査サブシステムが使用する監査プロバイダモジュール。デフォルトで、サーバーは default という名前の監査モジュールを使用します。新しい監査モジュールの作成方法については、「監査モジュールを作成する」を参照してください。

デフォルトのロールマッピングの主体を有効

このオプションを選択すると、デフォルトのロールマッピングの主体がアプリケーション固有のマッピングを持たないアプリケーションに適用されます。

マッピングされた主体のクラス

デフォルトのロールマッピングの主体が有効な場合に、デフォルトのロールマッピングの主体で使用される java.security.Principal クラスのカスタム実装を指定します。

追加プロパティー

サーバーの追加のセキュリティープロパティー。

有効なプロパティーは、「デフォルトレルム」フィールドで選択したレルムのタイプによって決まり、通常はレルムの編集時に指定します。詳細については、次のトピックを参照してください。

  • 「FileRealm クラスに固有のプロパティー」

  • 「CertificateRealm クラスに固有のプロパティー」

  • 「JDBCRealm クラスに固有のプロパティー」

  • 「LDAPRealm クラスに固有のプロパティー」

  • 「SolarisRealm クラスに固有のプロパティー」

  • 「PamRealm クラスに固有のプロパティー」
