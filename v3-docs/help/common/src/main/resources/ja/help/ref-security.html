セキュリティ

「セキュリティ」ページを使用し、選択したGlassFish Serverインスタンスまたはクラスタのセキュリティ・プロパティを設定します。

「セキュリティ」ページには、次のオプションが含まれます。

構成名

このページの設定が適用される構成の名前。このフィールドは読取り専用です。

セキュリティ・マネージャ

このオプションを選択すると、ドメインのセキュリティ・マネージャが有効になります。このオプションは、デフォルトでは無効になっています。

このオプションを有効にすると、JVMオプションの-Djava.security.managerがGlassFish ServerのJVM設定に追加されます。JVMオプションを構成するを参照してください。この変更を有効にするには、サーバーを再起動する必要があります。

すべてのアプリケーションについて正しい権限を与えていることを確認してください。セキュリティ・マネージャを無効にすると、パフォーマンスを向上させることができます

監査ロギング

このオプションを有効にすると、サーバーは「監査モジュール」設定で指定されているすべての監査モジュールをロードし、実行します。このオプションを無効にすると、サーバーは監査モジュールにアクセスしません。このオプションは、デフォルトでは無効になっています。

デフォルト・レルム

サーバーが認証用に使用する有効な(デフォルト)レルム。アプリケーションでは、デプロイメント記述子によって別のレルムが指定されないかぎり、このレルムが使用されます。構成されているすべてのレルムがこのリストに表示されます。デフォルト値はfileです。

デフォルトのプリンシパル

デフォルトのユーザー名を指定します。他のプリンシパルが指定されていないとき、サーバーはこのユーザー名を使用します。このフィールドに値を入力する場合は、「デフォルト・プリンシパルのパスワード」フィールド内の、対応するパスワードの入力が必要になります。

通常のサーバーの操作にデフォルト・プリンシパルは必要ありません。

デフォルト・プリンシパルのパスワード

「デフォルト・プリンシパル」フィールドで指定したデフォルト・プリンシパルのパスワード。通常のサーバーの操作にデフォルト・プリンシパルは必要ありません。

JACC

構成されているJACCプロバイダのクラス名。デフォルト値はdefaultです。

監査モジュール

監査ロギングが有効な場合に、監査サブシステムが使用する監査プロバイダ・モジュール。デフォルトで、サーバーはdefaultという名前の監査モジュールを使用します。新しい監査モジュールの作成方法については、監査モジュールを作成するを参照してください。

デフォルトのロール・マッピングのプリンシパルを有効

このオプションを選択すると、デフォルトのロール・マッピングのプリンシパルがアプリケーション固有のマッピングを持たないアプリケーションに適用されます。

マッピングされたプリンシパルのクラス

デフォルトのロール・マッピングのプリンシパルが有効な場合に、デフォルトのロール・マッピングのプリンシパルで使用されるjava.security.Principalクラスのカスタム実装を指定します。

追加プロパティ

サーバーの追加のセキュリティ・プロパティ。

有効なプロパティは、「デフォルト・レルム」フィールドで選択したレルムのタイプによって決まり、通常はレルムの編集時に指定します。詳細は、次のトピックを参照してください。

  • FileRealmクラスに固有のプロパティ

  • CertificateRealmクラスに固有のプロパティ

  • JDBCRealmクラスに固有のプロパティ

  • LDAPRealmクラスに固有のプロパティ

  • SolarisRealmクラスに固有のプロパティ

  • PamRealmクラスに固有のプロパティ
