配備済みアプリケーションを編集する

  1. ナビゲーションツリーで、「アプリケーション」ノードを選択します。

    「アプリケーション」ページが開きます。

  2. 「配備されているアプリケーション」の表から、編集するアプリケーションの名前をクリックします。

    名前には、オプションのバージョン ID を含めることができます。このバージョン ID は名前のあとに続き、コロン (:) で名前と区切ります。モジュールとアプリケーションのバージョンについては、『Oracle GlassFish Server 3.1 Application Deployment Guide』の「Module and Application Versions」を参照してください。

    「アプリケーションを編集」ページが開きます。

  3. 「状態を有効」チェックボックスをオフにしてアプリケーションの状態を無効にし、使用できないようにします。

    デフォルトでは、アプリケーションの状態は配備後すぐに「有効」になります。

  4. Web アプリケーションの場合、「コンテキストルート」フィールドにアプリケーションのパスを入力します。

    Web アプリケーションの URL では、コンテキストルートはポート番号の直後に続きます (http://host:port /context-root/...)。たとえば /hello のように、コンテキストルートがスラッシュで始まるようにしてください。

  5. アプリケーションのセッション持続と SFSB チェックポイントを無効にするには、「可用性の有効化」チェックボックスの選択を解除します。

    「有効」チェックボックスを選択すると、Web セッション、ステートフルセッション Bean (SFSB) チェックポイント設定、パッシベーションなどに備えて高可用性が有効になります。false (デフォルト) に設定すると、すべての Web セッションの保存と SFSB のチェックポイント設定が、指定されたアプリケーション、Web アプリケーション、または EJB モジュールで無効になります。true に設定すると、指定されたアプリケーションまたはモジュールで高可用性が有効になります。このオプションは、高可用性がサーバーレベルやコンテナレベルなどのより高いレベルで設定および有効化されている場合のみ、true に設定します。

    このオプションは、デフォルトのサーバーインスタンス (server) のほかにクラスタまたはスタンドアロンサーバーインスタンスが存在する場合に表示されます。

  6. 「仮想サーバー」ドロップダウンリストから、このアプリケーションに関連付ける仮想サーバーを選択します。

    「仮想サーバー」オプションは、デフォルトサーバーインスタンス server だけが存在する場合に表示されます。クラスタまたはその他のスタンドアロンサーバーインスタンスが存在する場合は、配備後、仮想サーバーを選択できます。「アプリケーションを編集」ページに移動して「ターゲット」タブを選択し、目的のターゲットに「仮想サーバーの管理」を選択してください。

  7. 「Java Web Start を有効」チェックボックスをオンにし、Java Web Start ソフトウェアを使用してアプリケーションを配備します。

    このオプションは、当初配備される際にその設定に基づいて、デフォルトで有効または無効に設定されます。

  8. 「説明」フィールドに、このアプリケーションの説明を入力します。
  9. アプリケーションのモジュール記述子のリストを表示するには、「説明」タブをクリックします。詳細は、「アプリケーションの記述子を表示する」を参照してください。
  10. デフォルトでは、アプリケーションの「状態」はすべてのターゲットで「有効」です。アプリケーションが有効化されているターゲットを表示するには、「ターゲット」タブをクリックします。1 つまたは複数のターゲットでアプリケーションを有効または無効にできます。「アプリケーションターゲットを表示する」を参照してください。
  11. 「保存」をクリックします。
