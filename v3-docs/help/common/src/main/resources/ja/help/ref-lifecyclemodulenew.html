新しいライフサイクル・モジュール

「新しいライフサイクル・モジュール」ページを使用して、新しいライフサイクル・モジュールを定義します。

「新しいライフサイクル・モジュール」ページには、次のオプションが含まれます。

名前

ライフサイクル・モジュールの名前。

クラス名

ライフサイクル・モジュールのクラス・ファイルの完全修飾名。クラスはcom.sun.appserv.server.LifecycleListenerインタフェースを実装する必要があります。

クラスパス

ライフサイクル・モジュールのクラスパス。クラスパスは、ライフサイクル・モジュールが配置されている場所を指定します。デフォルト値は次のとおりです: domain-dir/applications。モジュールがすでにこのディレクトリ(つまり、サーバーのクラスパス)にある場合、このフィールドは空白のままでかまいません。

ロード順序

起動時にライフサイクル・モジュールがロードされる順序。ロード順の整数値が小さいモジュールほど、先にロードされます。指定できる値は、101からオペレーティング・システムのMAXINTまでです。1から100の値は予約されています。

説明

ライフサイクル・モジュールの説明。

ステータス

このオプションを選択すると、ライフサイクル・モジュールが有効になります。このオプションはデフォルトで有効になっています。

ロード時の障害

このオプションを選択すると、ライフサイクル・モジュールのロードに失敗したときに、サーバーがシャットダウンされます。このオプションは、デフォルトでは無効になっています。

ターゲット

ライフサイクル・モジュールのクラスタとスタンドアロン・インスタンス。必要なターゲットを「選択したターゲット」列に移動するには、「追加」ボタンまたは「すべて追加」ボタンを使用します。不要なターゲットを「使用可能なターゲット」列に移動するには、「削除」ボタンまたは「すべて削除」ボタンを使用します。

このオプションは、ドメイン内にクラスタまたはスタンドアロン・インスタンスが作成されている場合にのみ表示されます。
