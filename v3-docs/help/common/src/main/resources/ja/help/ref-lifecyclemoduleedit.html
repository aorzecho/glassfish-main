ライフサイクルモジュールを編集

「ライフサイクルモジュールを編集」ページを使用して、ライフサイクルモジュールの設定を変更します。

「ライフサイクルモジュールを編集」ページには、次のオプションが含まれます。

名前

ライフサイクルモジュールの名前。名前は読み取り専用フィールドです。名前を指定できるのは、新しいライフサイクルモジュールの作成時のみです。

クラス名

ライフサイクルモジュールのクラスファイルの完全修飾名。クラスは com.sun.appserv.server.LifecycleListener インタフェースを実装する必要があります。

クラスパス

ライフサイクルモジュールのクラスパス。クラスパスは、ライフサイクルモジュールが配置されている場所を指定します。デフォルト値は domain-dir/applications です。モジュールがすでにこのディレクトリ (つまり、サーバーのクラスパス) にある場合、このフィールドは空白のままでかまいません。

読み込み順序

起動時にライフサイクルモジュールが読み込まれる順序。読み込み順の整数値が小さいモジュールほど、先に読み込まれます。指定できる値は、101 からオペレーティングシステムの MAXINT までです。1 ～ 100 の値は予約されています。

説明

ライフサイクルモジュールの説明。

ステータス

このオプションを選択すると、ライフサイクルモジュールが有効になります。このオプションはデフォルトで有効になっています。

読み込み時の障害

このオプションを選択すると、ライフサイクルモジュールの読み込みに失敗したときに、サーバーがシャットダウンされます。このオプションは、デフォルトでは無効になっています。
