監視サービスを設定する

  1. 「設定」ノードの下で、監視を設定するサーバーインスタンスまたはクラスタの設定を展開します。
  2. 「監視」ノードを展開します。

    「監視サービス」ページが開きます。

  3. 「監視サービス」の「有効」チェックボックスをオンにして GlassFish サーバー の監視を有効にするか、チェックボックスのチェックマークを外して監視を無効にします。

    このオプションはデフォルトで有効になっています。

  4. 「監視 MBean」の「有効」チェックボックスをオンにして監視に必要なすべての MBean を配備するか、チェックボックスのチェックマークを外してこれららの MBean の配備を取り消します。

    このオプションはデフォルトで有効になっています。

    チェックボックスがオンでない場合、「監視サービス」チェックボックスがオンになっている場合でも、管理コンソール で監視データを表示することはできません。この場合は、コマンド行を使用して監視データを表示する必要があります。チェックボックスのチェックマークを外すと、GlassFish サーバー のメモリーフットプリントが減少します。

  5. 「DTrace」の「有効」チェックボックスをオンにして、Solaris の動的トレース機能を有効にします。

    Solaris の動的トレース機能を使用するには、Oracle Enterprise Manager DTrace Monitoring も必要です。 このオプションは、デフォルトでは無効になっています。

  6. 表で次のいずれかの操作を実行します。
    • 「コンポーネント」列の 1 つまたは複数のコンポーネントに対して、「監視レベル」列のドロップダウンリストから「LOW」または「HIGH」を選択し、コンポーネントの監視を有効にします。
    • 1 つまたは複数のコンポーネントの横にあるチェックボックスを選択したあと、表の上部にある「レベル」ドロップダウンから「LOW」または「HIGH」を選択し、「レベルの変更」ボタンをクリックして、選択したコンポーネントの監視レベルを変更します。

    現在のリリースでは、「LOW」オプションと「HIGH」オプションによる監視レベルに差はありません。1 つまたは複数のコンポーネントの監視を無効にするには、「OFF」を選択します。デフォルト値は「OFF」です。

    監視を設定する前に、監視対象のアプリケーション、コンポーネント、またはサービスを配備する必要があります。

  7. 「保存」をクリックします。
