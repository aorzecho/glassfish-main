ロガーの設定

「ロガーの設定」ページを使用し、選択した GlassFish Server インスタンスまたはクラスタのログを設定します。

「ロガーの設定」ページには、次のオプションが含まれます。

設定名

このページの設定を適用する設定の名前。 このフィールドは読み取り専用です。

システムログに書き込み

このオプションを選択すると、ログ出力がサーバーログに加え、syslog 機能にも送信されます。この機能は Solaris および Linux システムのみで使用できます。このオプションは、デフォルトでは無効になっています。

ログファイル

サーバーログファイルの別名または場所。デフォルトの場所は glassfish/domains/domain1/logs/server.log です。

「ログハンドラ」

カスタムログハンドラの絶対クラス名。カスタムログハンドラを使用すると、ログを server.log または syslog 以外の宛先に送信できます。カスタムハンドラを使用する場合は、クラス java.util.logging.Handler (JSR 047 準拠の API) を拡張する必要があります。ハンドラクラスを GlassFish Server のクラスパスに置き、サーバーの起動時にハンドラがインストールされるようにします。

「ファイルローテーション制限」

サーバーログファイルがローテーションされるサイズ (バイト単位)。サーバーログが指定したサイズに達すると、GlassFish Server は server.log という名前の新しい空のファイルを作成し、古いファイルの名前を server.log_date に変更します。date は、ファイルがローテーションされた日時を表します。デフォルト値は 2M バイトです。制限の最小値は 500K バイトです。それより小さな値を指定すると、ファイルは 500K バイトに達した時点でローテーションされます。ログファイルのローテーションをオフにするには、この値を 0 に設定します。

「ファイルローテーションの制限時間」

サーバーログがローテーションされるまでの時間 (分単位)。デフォルト値は 0 です。これは、「ファイルローテーション制限」フィールドで指定したサイズに達したときに、ファイルがローテーションされることを意味しています。1 分以上の値を指定すると、「ファイルローテーション制限」フィールドで指定したサイズ制限よりも、時間による制限が優先されます。

更新の頻度

キューからサーバーログに一度に書き込むメッセージの最大数。デフォルト値は 1 です。
