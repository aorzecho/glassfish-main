ローテーションされるログファイル数の制限を変更する

ログファイルをローテーションするとき、GlassFish サーバー は server.log という名前で新しい空のファイルを作成し、古いファイルの名前を server.log_ date に変更します。date はファイルがローテーションされた日付と時刻になります。

デフォルトでは、ローテーションされるログファイル数は 10 個までに制限されます。この制限に達すると、次回のログファイルのローテーション時にもっとも古いログファイルが削除されます。

必要に応じて、ローテーションされるログファイル数の制限を変更することができます。設定した新しい制限は、アクセスログとサーバーログの両方のログファイルのローテーションに適用されます。

  • com.sun.enterprise.server.logging.max_history_files システムプロパティーに、ローテーションされるログファイルを保持する最大数を設定します。

    システムプロパティーを設定する方法については、「システムプロパティーを管理する」を参照してください。

    com.sun.enterprise.server.logging.max_history_files システムプロパティーの動作は次のとおりです。

    • このプロパティーが設定されていない場合、GlassFish サーバー は、ローテーションされるログファイルを最大で 10 個まで保持します。

    • このプロパティーに無効な数値または null が設定されている場合、GlassFish サーバー は、ローテーションされるログファイルを最大で 10 個まで保持します。

    • このプロパティーに 0 が設定されている場合、GlassFish サーバー は、ローテーションされるログファイルを保持しません。
