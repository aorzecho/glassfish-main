系统属性

“系统属性”页面显示在选定的命名配置中定义的 Java 系统属性列表。启动 GlassFish Server 时，Java 系统属性通过 Java 应用程序启动器的 -D 选项传递到 Java 应用程序启动器。

这些属性定义了引用该配置的所有 GlassFish Server 实例的端口设置默认值。

有关配置中的预定义端口设置的信息，请参见预定义系统属性 。

对于每个属性，都将显示以下信息：

实例变量名称

系统属性名称。

默认值

为实例引用的命名配置中的属性所设置的值。此字段为只读。

实例值

指向该属性的实例值页面的链接。

“其他属性”表还包含以下选项。

添加属性

用于添加属性的按钮。单击此按钮可向“其他属性”表中添加一行。

删除属性

用于删除一个或多个选定属性的按钮。删除的任何属性会恢复其默认值，或者如果没有设置默认值，则未定义。

“系统属性”页面还包含以下选项：

配置名称

要应用该页面上设置的配置名称。此字段为只读。

动态重新配置

如果启用此选项，对配置所做的更改将应用到引用该配置的实例，而无需重新启动这些实例。

使用此配置的群集和/或实例

引用此配置的群集和实例的列表，同时指示每个群集或实例是否正在运行：

  • 如果群集或实例正在运行，将显示一个复选标记 (✓)。

  • 如果群集或实例已停止，将显示一个感叹号 (!)。
