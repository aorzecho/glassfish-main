恢复事务处理

使用“恢复事务处理”页可恢复 GlassFish Server 实例的事务处理。

如果实例正在运行，“恢复事务处理”页仅包含文本恢复由运行的服务器本身来执行以及“确定”按钮。

如果实例未在运行，“恢复事务处理”页包含以下选项：

实例名称

创建实例时为其指定的名称。此字段为只读。

目标服务器

可恢复当前实例的事务处理的另一个实例。

事务处理日志目录

存储事务处理日志 (如果日志存储在共享文件系统中) 的目录。此设置不适用于数据库中存储的事务处理日志。默认目录是域目录下的 /logs/tx。

事务处理日志保存在“事务处理日志位置”设置 (位于集群配置的“事务处理服务”中) 指定的目录的 tx 子目录中。“事务处理日志位置”的默认值为“域属性”页 (位于 GlassFish Server 节点的“高级”选项卡下) 的“日志根目录”字段指定的目录。

等效的 asadmin 子命令

recover-transactions(1)
