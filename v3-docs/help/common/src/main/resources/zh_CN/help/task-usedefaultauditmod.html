使用默认审计模块

default 审计模块将验证和授权请求记录到服务器日志文件中。有关更改日志文件位置的信息，请参见配置常规日志记录设置。

验证日志条目包括以下信息：

  • 已尝试进行验证的用户的名称

  • 处理过访问请求的领域

  • 请求的 Web 模块 URI 或 EJB 组件

  • 请求成功或失败

无论是否启用审计日志记录，GlassFish Server 都将记录所有被拒绝的验证事件。

授权日志条目包括以下信息：

  • 经过验证的用户（如果有）的名称

  • 请求的 Web URI 或 EJB 组件

  • 请求成功或失败

  1. 在导航树中，展开“配置”节点。
  2. 在“配置”节点下，选择“安全性”节点。

    将打开“安全性”页面。

  3. 从“审计模块”下拉式列表中，选择名为 default 的审计模块。
  4. 单击“保存”。
