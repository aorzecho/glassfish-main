生命周期模块

使用“生命周期模块”页可以配置生命周期模块。

在服务器的生命周期中，如果一个或多个事件触发了生命周期模块，此生命周期模块将执行一些任务。可能的触发服务器事件为：初始化、启动、准备为请求提供服务和关机。生命周期模块并不在 Java EE 规范的定义范围内，而是 GlassFish Server 的增强功能。

对于每个生命周期模块，将提供以下信息：

名称

生命周期模块名称。

已启用或状态

只要存在存在默认服务器实例 server，则当应用程序已启用时，此为复选标记；应用程序已禁用时，则为 X。

如果存在集群或其他独立服务器实例，选择“已在所有目标上启用”可查看已部署的应用程序的目标。

加载顺序

指定应加载生命周期模块相对顺序的值。

“生命周期模块”表还包含以下选项：

新建

用于创建新的生命周期模块的按钮。

删除

用于删除一个或多个所选生命周期模块的按钮。

启用

用于启用一个或多个所选生命周期模块的按钮。只存在默认服务器实例 server 时才会显示。

禁用

用于禁用一个或多个所选生命周期模块的按钮。只存在默认服务器实例 server 时才会显示。
