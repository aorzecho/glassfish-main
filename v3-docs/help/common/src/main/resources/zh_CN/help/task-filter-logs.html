过滤显示的日志记录

开始之前

请确保查看的是域管理服务器 (domain administration server, DAS)、群集 GlassFish Server 实例或独立实例的日志文件。有关更多信息，请参见以下主题：

  • 查看 DAS 的日志文件

  • 查看群集 GlassFish Server 实例的日志文件

  • 查看独立 GlassFish Server 实例的日志文件

  1. 在“搜索条件”区域中，按内容、时间段和日志级别过滤记录。
    1. 要只显示包含特定文本的记录，请在“文本搜索”字段中键入该文本。

      该搜索区分大小写。

    2. 要选择显示日志记录的时间段，请使用“时间戳”选项。
      • 要查看最近的消息，请选择“最近”（默认值）。
      • 如果只查看特定时间段内的消息，请选择“特定范围”并在显示的“从”字段和“到”字段中键入日期和时间值。

        “日期”值必须为以下格式：

        MM/DD/YY
        MM

        范围在 1–12 之间的整数，表示月份。例如，1 表示 1 月，10 表示 10 月。

        DD

        范围在 1–31 之间的整数，表示每月中的日期。

        YY

        年的最后两位数字，例如，11 代表 2011 年。

        例如，要指定日期 2011 年 2 月 1 日，请在“日期”字段键入 2/1/11。

        “时间”值必须为以下格式：

        hh:mm:ss.SSS
        hh

        范围在 0–23 之间的整数，表示小时。对于下午的时间，使用范围在 12–23 之间的整数，例如，14 表示下午 2 点。

        MM

        范围在 0–59 之间的整数，表示小时中的分钟数。

        ss

        范围在 0–59 之间的整数，表示秒数。

        SSS

        如有需要，可使用范围在 000–999 之间的三位数整数（带前导零）表示毫秒数。

        例如，要指定时间下午 5:10，请在“时间”字段中键入 17:10:00.000。

        如果“从”字段中的时间值迟于“到”字段中的时间值，将显示错误消息。

    3. 按日志级别过滤消息。

      为确保要查看的消息都显示在服务器日志中，请先在“模块日志级别”页面上设置适当的日志级别。

      如果选择按日志级别过滤日志消息，将只显示符合指定过滤条件的消息。不过，这种过滤不影响将哪些消息记录到服务器日志中。

      1. 在“日志级别”下拉式列表中，选择日志级别。
      2. 要只显示选定级别的消息，请选择“请勿包括更严重的消息”选项。

        默认情况下，将显示服务器日志中选定日志级别和更严重级别的所有消息。

  2. 在“高级搜索”区域中，按日志程序名称空间过滤记录，并选择要显示的信息量。
    1. 单击“高级搜索”链接。

      “高级搜索”区域将添加到“日志查看器”窗口。

    2. 要按模块过滤，请从“日志程序”下拉式列表中选择一个或多个名称空间。

      通过按住 Shift 键单击或按住 Ctrl 键单击选择多个名称空间。

      选择较高级别的名称空间也就选择了该名称空间下的所有名称空间。例如，选择 javax.enterprise.system 会同时选择该名称空间下所有模块的日志程序，例如 javax.enterprise.system.core、javax.enterprise.system.tools.admin 等等。

    3. 要从日志程序中查看某个特定应用程序的消息，请在“自定义日志程序”字段中，分行键入每个日志程序的名称。

      如果应用程序具有多个模块，您可以查看任何模块，也可以查看所有模块。例如，某个应用程序可能具有以下日志程序：

      • com.example.myapp.module1

      • com.example.myapp.module2

      • com.example.myapp.module3

      要查看应用程序中所有模块的消息，请键入 com.mycompany.myapp。要只查看 module2 的消息，请键入 com.example.myapp.module2。

      指定一个或多个自定义日志程序时，仅当在“日志程序”区域中显式指定了 GlassFish Server 模块的消息，才会显示这些消息。

    4. 从“显示”下拉式列表中，选择要显示的消息量。

      所显示消息的默认数目为 40。其他选项为 100、250 和 1000。

    5. 要查看堆栈追踪，请取消选中“限制过长的消息”选项。

      默认情况下，堆栈追踪不出现在查看器中。要查看堆栈追踪，请单击消息的（详细资料）链接。

    6. 要隐藏“高级搜索”区域，请单击“基本搜索”链接。
  3. 要指定您要查看的实例和相关联的日志文件，请使用“修改搜索”区域。
    • 从“实例”下拉式列表中，选择实例。

      默认值是您最初选择查看日志文件的实例。如果选择查看域管理服务器 (domain administration server, DAS) 的日志文件，则默认值是 server（表示 DAS）。

    • 在“日志文件”下拉式列表中，选择日志文件。

      默认名称为 server.log。其他选项取决于选定目标的日志设置。

  4. 在“搜索结果”区域中，自定义日志消息的显示。

    “搜索结果”区域将按照“日志设置”页面和“日志级别”页面中的设置，显示最近的 40 条服务器日志条目。

    • 要按不同方式对消息进行排序，请单击列中的箭头。
    • 要以格式化版本查看消息，请单击此消息的（详细资料）链接。

      此时将打开“日志条目详细资料”窗口，其中包含此消息的格式化版本。

    • 要查看日志文件中较早或较新的条目，请单击位于表顶部的“记录早于”和“记录晚于”。
