编辑生命周期模块

  1. 在导航树中，选择“生命周期模块”节点。

    此时将打开“生命周期模块”页。

  2. 在“生命周期模块”页上，单击要编辑的生命周期模块的名称。

    此时将打开“编辑生命周期模块”页。

  3. 在“类名称”字段中，键入生命周期模块类文件的全限定名称。

    此类必须实现 com.sun.appserv.server.LifecycleListener 接口。

  4. 在“类路径”字段中，键入生命周期模块的类路径。

    类路径指定生命周期模块的位置。默认值为 domain-dir/applications。如果模块已经在此目录中 (即，在服务器类路径中)，则此字段可以留为空白。

  5. 在“加载顺序”字段中，键入指定启动时加载此生命周期模块顺序的值。

    加载顺序值为较小整数的模块会较早加载。值的范围在 101 和操作系统的 MAXINT 之间。1 和 100 之间的值为保留值。

  6. 在“说明”字段中，键入生命周期模块的说明。

  7. 选择“启用状态”复选框以启用生命周期模块。

    默认情况下启用此选项。

  8. 选中“加载失败阻止实例启动时”复选框以指定在生命周期模块无法加载时关闭服务器。

    默认情况下禁用此选项。

  9. 如果存在除默认服务器实例 (server) 以外的集群或独立服务器实例，则会显示“目标”选项卡。选择“目标”选项卡可查看已启用生命周期模块的目标。有关如何在一个或多个目标上启用或禁用生命周期模块的说明，请参阅查看应用程序目标。

  10. 单击“保存”。
