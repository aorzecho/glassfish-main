创建生命周期模块

  1. 在导航树中，选择“生命周期模块”节点。

    此时将打开“生命周期模块”页。

  2. 在“生命周期模块”页上，单击“新建”。

    此时将打开“新建生命周期模块”页。

  3. 在“名称”字段中，键入生命周期模块的名称。

  4. 在“类名称”字段中，键入生命周期模块类文件的全限定名称。

    此类必须实现 com.sun.appserv.server.LifecycleListener 接口。

  5. 在“类路径”字段中，键入生命周期模块的类路径。

    类路径指定生命周期模块的位置。默认值为 domain-dir/applications。如果模块已经在此目录中 (即，在服务器类路径中)，则此字段可以留为空白。

  6. 在“加载顺序”字段中，键入指定启动时加载此生命周期模块顺序的值。

    加载顺序值为较小整数的模块会较早加载。值的范围在 101 和操作系统的 MAXINT 之间。1 和 100 之间的值为保留值。

  7. 在“说明”字段中，键入生命周期模块的说明。

  8. 选择“启用状态”复选框以启用生命周期模块。

    默认情况下启用此选项。

  9. 选中“加载失败阻止实例启动时”复选框以指定在生命周期模块无法加载时关闭服务器。

    默认情况下禁用此选项。

  10. 要添加将在其上启用生命周期模块的目标服务器实例和集群，请从“可用目标”列表中选择它们，然后使用“添加”按钮将它们移动到“所选目标”列表。

    如果存在除默认服务器实例 (server) 以外的集群或独立服务器实例，则会显示“目标”选项。默认情况下，在名为 server 的默认服务器实例上启用了生命周期模块。使用“全部添加”按钮可部署到所有目标。使用“删除”和“全部删除”按钮可将目标从“所选目标”列表中删除。

  11. 单击“确定”。
