编辑已部署的应用程序

  1. 在导航树中，选择“应用程序”节点。

    此时将打开“应用程序”页面。

  2. 从“已部署的应用程序”表中，单击要编辑的应用程序的名称。

    名称可包含可选的版本标识符，它跟在名称后面，并与名称以冒号(:) 分隔。有关模块和应用程序版本的更多信息，请参见《Oracle GlassFish Server 3.1 Application Deployment Guide》中的"Module and Application Versions"。

    此时将打开“编辑应用程序”页面。

  3. 取消选择“启用状态”复选框以禁用应用程序状态，使其变为不可用。

    默认情况下，只要一部署应用程序，其状态便为“启用”。

  4. 对于 Web 应用程序，请在“上下文根目录”字段中，键入应用程序的路径。

    在 Web 应用程序的 URL 中，上下文根目录要紧跟着端口号 (http://host:port /context-root/...)。确保上下文根路径以正斜杠开头，例如 /hello。

  5. 取消选中“已启用可用性”复选框可为应用程序禁用会话持久性和 SFSB 检查点操作。

    如果选中“已启用”复选框，则为 Web 会话和有状态会话 Bean (stateful session bean, SFSB) 检查点操作和潜在钝化启用高可用性。如果设置为 false（默认值），则对指定应用程序、Web 应用程序或 EJB 模块禁用所有 Web 会话保存和 SFSB 检查点操作。如果设置为 true，则为指定的应用程序或模块启用高可用性。仅当在更高级别上（比如服务器级别和容器级别）配置和启用高可用性时，此选项才设置为 true。

    如果除默认服务器实例 (server) 以外还存在群集或独立服务器实例，则会显示该选项。

  6. 从“虚拟服务器”下拉式列表中，选择要与此应用程序关联的虚拟服务器。

    只要存在存在默认服务器实例 server，则出现“虚拟服务器”选项。如果存在群集或其他独立服务器实例，则可以在部署后选择虚拟服务器。转至“编辑应用程序”页面，选择“目标”选项卡，然后为所需目标选择“管理虚拟服务器”。

  7. 选中“启用 Java Web Start”复选框，以使用 Java Web Start 软件来部署应用程序。

    默认情况下，根据最初部署时的设置禁用或启用该选项。

  8. 在“说明”字段中，键入此应用程序的说明。
  9. 要查看应用程序的模块描述符列表，请单击“描述”选项卡。有关更多详细信息，请参见查看应用程序描述符
  10. 默认情况下，应用程序在所有目标上的“状态”都为“已启用”。要查看启用了应用程序的目标，请单击“目标”选项卡。可以在一个或多个目标上启用或禁用应用程序。请参见查看应用程序目标。
  11. 单击“保存”。
