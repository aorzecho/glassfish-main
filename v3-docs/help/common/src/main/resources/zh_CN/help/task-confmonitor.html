配置监视服务

  1. 在“配置”节点下，展开您要为其配置监视的服务器实例或群集配置。
  2. 展开“监视”节点。

    此时将打开“监视服务”页面。

  3. 选中“启用监视服务”复选框可为 GlassFish Server 启用监视，取消选中该复选框则可禁用监视。

    默认情况下启用此选项。

  4. 选中“启用监视 MBean”复选框可部署监视所需的所有 MBean，取消选中该复选框则可取消部署这些 MBean。

    默认情况下启用此选项。

    如果未选中该复选框，则无法在管理控制台中查看监视数据，即使选中“监视服务”复选框也是如此。此时，您必须改用命令行来查看监视数据。如果取消选中该复选框，则会减少 GlassFish Server 的内存占用。

  5. 选中“启用 DTrace”复选框可以启用 Solaris DTrace 工具。

    要使用 Solaris DTrace 工具，还必须有 Oracle Enterprise Manager DTrace Monitoring。默认情况下禁用此选项。

  6. 在表中，执行以下操作之一：
    • 在“监视级别”列中，从下拉式列表为“组件”列中的一个或多个组件选择“低”或“高”，为这些组件启用监视。
    • 选中一个或多个组件旁边的复选框，然后从此表顶部的“级别”下拉式列表选择“低”或“高”，并单击“更改级别”按钮，以将所有这些组件的监视级别设置为此值。

    在此发行版中，“低”和“高”选项所提供的监视级别之间没有差别。要为一个或多个组件禁用监视，请选择“关闭”。默认值为“关闭”。

    您必须首先部署应用程序、组件或服务，然后才能为它们配置监视。

  7. 单击“保存”。
