删除消息安全性提供者配置

开始之前

无法创建或删除默认 server-config 配置的消息安全性提供者。只有 SOAP 和 HttpServlet 这两个消息安全性提供者支持默认的 server-config。因为默认情况下安装 GlassFish Server 时已经创建了这两个消息提供者，所以在默认 server-config 的“消息安全性配置”页面上不显示“删除”按钮。

  1. 在导航树中，展开“配置”节点。
  2. 展开“安全性”节点。
  3. 选择“消息安全性”节点。

    此时将打开“消息安全性配置”页面。

  4. 在“消息安全性配置”页面上，选择 "SOAP"。

    此时将打开“编辑消息安全性配置”页面。

  5. 在“编辑消息安全性配置”页面上，单击“提供者”选项卡。

    此时将打开“编辑提供者配置”页面。

  6. 在“编辑提供者配置”页面上，选中要删除的提供者配置左侧的复选框。
  7. 单击“删除”。
