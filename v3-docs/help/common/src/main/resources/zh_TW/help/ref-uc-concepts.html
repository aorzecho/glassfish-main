更新工具重要概念

更新工具管理 GlassFish Server 軟體元件的安裝作業，在系統的單一檔案系統目錄下包含這些元件。

更新工具的管理主控台版本能讓您進行以下作業：

  • 安裝附加元件

  • 檢視已安裝元件的更新

  • 檢視已安裝的元件

若要更新或移除已安裝的元件，您必須停止 GlassFish Server 網域，然後使用下列其中一個命令：

  • as-install-parent/bin/updatetool，這會啟動更新工具圖形化公用程式

  • as-install-parent/bin/pkg，這是命令行版本的更新工具

下列章節說明更新工具的重要概念。

元件

元件是更新工具所管理的可安裝單元。元件由安裝影像內安裝為系統單元的檔案所組成。

安裝影像

安裝影像是包含軟體元件的目錄，包括目錄的內容。下載及安裝 GlassFish Server 軟體之後，即會將安裝影像增加至更新工具。若要啟用 GlassFish Server 軟體元件的新增和更新功能，每個安裝影像必須包含有關這些元件的安裝資訊。

網路儲存庫

網路儲存庫包含下列項目：

  • 可增加至安裝影像的元件

  • 對安裝影像中現有元件的更新

安裝影像會與一或多個網路儲存庫相關。每個網路儲存庫會以名稱和相關的 URL 表示。每個安裝影像皆具有唯一的儲存庫名稱，此名稱代表管理儲存庫的組織或專案，以及明確的內容範圍。如需瞭解更多預先為 GlassFish Server 組態的網路儲存庫相關資訊，請參閱 Oracle GlassFish Server Administration Guide 中的 Preconfigured Repositories for GlassFish Server。

附加元件

附加元件是尚未安裝的安裝影像元件。
