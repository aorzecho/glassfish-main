新建 JACC 提供者

使用「新建 JACC 提供者」頁面可建立與 JSR-115/196 相容的新 Java Authorization Contract for Containers (JACC) 提供者，其可由協力廠商授權模組用於 GlassFish Server 中執行的應用程式。會將 JACC 提供者建立為網域之 domain.xml 檔案中 security-service 元素內的 jacc-provider 元素。

預設 GlassFish Server 安裝包括名為 default 與 simple 的兩個 JACC 提供者。您建立的任何新 JACC 提供者均不是這兩個預設的提供者。預設 GlassFish Server JACC 提供者會實作符合 JACC 規格的簡單檔案型授權引擎。

所有 GlassFish Server JACC 提供者必須使用 Java Authentication and Authorization Service (JAAS) API。JAAS 可以讓服務對使用者進行認證，並強制對使用者進行存取控制。JAAS 會實作 Java 技術版本的標準可外掛式驗證模組 (PAM) 架構。

JSR 196 可讓您開發不同層的外掛程式。您可以定義能變更組態新認證機制方式的外掛程式，例如 AuthConfigProvider 與 AuthConfigFactory。您也可以定義新的認證機制，例如 ServerAuthModule 與 ClientAuthModule。

「新建 JACC 提供者」 頁面包含下列選項。

組態名稱

此頁面上的設定套用的組態名稱。本欄位是唯讀的。

名稱

JACC 提供者的名稱。此名稱用於參照 domain.xml 中的 jacc-provider 元素。

策略組態

javax.security.jacc.PolicyConfigurationFactory.provider 的完全合格類別名稱，會實作提供者特定的 javax.security.jacc.PolicyConfigurationFactory。

策略提供者

javax.security.jacc.policy.provider 的完全合格類別名稱，會實作 java.security.Policy。

其他特性

JACC 提供者的其他特性。如需適用於 JACC 提供者之其他特性的說明，請參閱JACC 提供者特定的特性。
