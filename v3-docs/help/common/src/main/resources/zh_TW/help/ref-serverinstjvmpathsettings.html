JVM 路徑設定

使用 [JVM 路徑設定] 頁面可配置本機庫路徑。本機程式庫路徑串連了下列路徑：

  • 本機庫路徑前綴

  • 伺服器的本機庫路徑 (亦即 as-install/lib)

  • 64 位元本機庫目錄 (如果適用)

  • JVM 用來啟動 GlassFish Server 的 java.library.path 變數值 (此值會視作業系統而有所不同)

  • JVM 效能評測器設定頁面上指定的任何路徑 (如果已啟用效能評測器)

  • 本機庫路徑後綴

在已從較舊發行版本的 GlassFish Server 升級的網域中，其他設定會顯示為唯讀的值。v3 發行版本不支援這些設定。

[JVM 路徑設定] 頁面包含下列選項。

配置名稱

此頁面上的設定要套用到的配置名稱。此為唯讀欄位。

環境類別路徑

若選取 [忽略] 核取方塊，則會忽略 CLASSPATH 環境變數。CLASSPATH 環境變數對於程式設計基本指導非常方便，但是不建議用於企業環境。此為唯讀欄位。

系統類別路徑

系統上的類別的路徑。此為唯讀欄位。

類別路徑前綴

前置於伺服器的類別路徑之 JAR 檔案完整路徑，一行一個。此為唯讀欄位。

類別路徑後綴

附加於伺服器的類別路徑之 JAR 檔案完整路徑，一行一個。此為唯讀欄位。

本機程式庫路徑前綴

前置於本機程式庫路徑之 JAR 檔案完整路徑，一行一個。

本機程式庫路徑後綴

附加於本機程式庫路徑之 JAR 檔案完整路徑，一行一個。
