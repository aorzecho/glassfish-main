變更自動重建的記錄檔數目限制

當 GlassFish Server 自動重建記錄檔時，會建立名為 server.log 的全新空白檔案，並將舊檔案重新命名為 server.log_date，其中 date 是檔案自動重建的日期和時間。

依照預設，GlassFish Server 會將自動重建的記錄檔數目限制為 10。 達到此限制時，最舊的記錄檔會在 GlassFish Server 下次自動重建記錄檔時加以刪除。

如有必要，您可以變更自動重建記錄檔的數目限制。您設定的任何新限制都會套用到存取記錄和伺服器記錄自動重建記錄檔。

  • 將 com.sun.enterprise.server.logging.max_history_files 系統特性設為要保留的動重建記錄檔最大數目。

    如需有關如何設定系統特性的更多資訊，請參閱管理系統特性。

    com.sun.enterprise.server.logging.max_history_files 系統特性的運作方式如下：

    • 如果未設定特性，GlassFish Server 會保留最多 10 個自動重建的記錄檔。

    • 如果特性設為無效數目或空值，GlassFish Server 會保留最多 10 個自動重建的記錄檔。

    • 如果特性設為 0，GlassFish Server 不會保留重建的記錄檔。
