應用程式組態

使用 「應用程式組態」 頁面可設定特性，確保會偵測到對已建置應用程式所做的變更，並會重新載入修改的類別。

「應用程式組態」 頁面包含下列選項。

載入預設值

用於將具有預設值的設定復原成其預設值的按鈕。沒有預設值的設定保持不變。

重新載入

若啟用此選項，則會動態重新載入修改的類別。伺服器會定期檢查建置目錄中的變更，然後自動且動態地重新載入已有變更的應用程式。動態重新載入在開發環境中非常有用，因為它能快速測試程式碼變更。但在生產環境中，動態重新載入可能會使效能降低。另外，每當重新載入完成時，轉換時間內的階段作業都會變得無效。用戶端必須重新啟動階段作業。此選項預設為啟用。

重新載入輪詢間隔

決定伺服器檢查已建置應用程式和模組中的程式碼有無變更的頻率。預設值為 2。

管理階段作業逾時

決定管理主控台逾時而必須重新登入之前，需等候的未使用分鐘數。預設值為 60。

自動建置

若啟用此選項，自動建置目錄中的應用程式將會自動建置。此選項預設為啟用。

自動建置輪詢間隔

指定伺服器檢查自動建置目錄中有無應用程式或模組變更的頻率。變更輪詢間隔不會影響建置應用程式或模組所用的時間。預設值為 2。

自動建置重試逾時

指定發生錯誤之前，部分複製的檔案可維持大小不變的秒數。預設值為 4。

若檔案緩慢地複製到自動建置目錄，則在複製整個檔案之前可能會出現該檔案。因此，嘗試自動建置應用程式會失敗。若嘗試自動建置應用程式因為此原因而失敗，GlassFish Server 會重新嘗試建置該應用程式。若部分複製的檔案在指定的時段內大小維持不變，即會出現錯誤。

自動建置目錄

指定自動建置應用程式功能所要監視的目錄。預設值為 domain-dir/autodeploy。

XML 驗證

指定要針對標準和 GlassFish Server 建置描述元執行的 XML 驗證類型。類型可能為下列其中之一：

完整

如果 XML 驗證失敗，建置也將失敗。此為預設值。

剖析

回報 XML 驗證錯誤，但建置仍會執行。

無

不執行 XML 驗證。

檢驗器

若啟用此選項，則會在自動建置之前執行檢驗器。若檢驗失敗，則不會執行建置。檢驗器將檢查檔案的結構和內容。檢驗大型應用程式通常非常費時。此選項預設為停用。

預編譯

若啟用此選項，則會預編譯 JavaServer Pages (JSP) 檔案。若停用此選項，則首次存取 JSP 檔案時會於執行階段編譯這些檔案。此選項預設為停用。

其他特性

應用程式的其他特性。GlassFish Server 未定義應用程式的任何其他特性。
