監視服務

使用 「監視服務」 頁面可組態個別伺服器元件或服務的監視選項。

您必須先建置應用程式、元件或服務，才能為其組態監視。

「監視服務」 頁面包含下列選項：

組態名稱

此頁面上的設定套用的組態名稱。本欄位是唯讀的。

監視服務

如果選取了「監視服務」 的「已啟用」核取方塊，會啟用針對 GlassFish Server 的監視功能。此選項預設為啟用。

監視 MBean

如果選取了「監視 MBean」的「已啟用」 核取方塊，會將監視需要的所有 MBeans 建置到 GlassFish Server 上。此選項預設為啟用。

如果沒有選取此核取方塊，即使已選取 「監視服務」 核取方塊，您也無法在管理主控台上檢視監視資料。您必須使用命令行來檢視監視資料。如果您取消選取此核取方塊，GlassFish Server 的記憶體使用量會減少。

DTrace

如果選取了 DTrace 的「已啟用」核取方塊，且您具有 Oracle Enterprise Manager DTrace Monitoring，您就可以使用 Solaris DTrace 功能。此選項預設為停用。

對於每個可用的模組，均提供以下資訊。

元件

GlassFish Server 元件的名稱。

監視層級

針對元件所設的監視層級。每個元件的可用層級為 「低」、「高」 和 「關閉」。預設層級為 「關閉」。在此發行版本中，「低」 與 「高」 選項所提供的監視層級並無不同。

元件層級程式設定表也包含下列選項。

變更層級

變更一或多個選取元件之監視層級的按鈕。

層級

下拉式清單，您可以從中選取要套用到所有選取元件的監視層級。
