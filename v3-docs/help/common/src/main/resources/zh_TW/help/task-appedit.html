編輯已部署的應用程式

  1. 在瀏覽樹狀結構中，選取 [應用程式] 節點。

    [應用程式] 頁面會隨即開啟。

  2. 從 [已部署的應用程式] 表格中，按一下您要編輯的應用程式名稱。

    名稱中可以包含一個選擇性的版本識別碼，後面接著名稱，兩者間用冒號 (:) 隔開。 如需有關模組與應用程式版本的更多資訊，請參閱「Oracle GlassFish Server 3.1 Application Deployment Guide」中的「Module and Application Versions」。

    [編輯應用程式] 頁面會隨即開啟。

  3. 取消選取 [啟用狀態] 核取方塊，以停用應用程式狀態，使其無法使用。

    應用程式在部署後的狀態預設為 [已啟用]。

  4. 若為 Web 應用程式，請在 [環境根目錄] 欄位中，鍵入該應用程式的路徑。

    在 Web 應用程式的 URL 中，環境根目錄緊跟在連接埠號碼之後 (http://host: port/context-root/...)。請確保環境根目錄的開頭是正斜線，例如 /hello。

  5. 取消選取 [可用性已啟用] 核取方塊，停用應用程式的階段作業持續性與 SFSB 檢查點。

    若選取 [已啟用] 核取方塊，則會對 Web 階段作業、有狀態的階段作業 Bean (SFSB) 檢查點及潛在的鈍化啟用高可用性。如果設定為 false (預設值)，將對指定的應用程式、Web 應用程式或 EJB 模組停用所有 Web 階段作業儲存及 SFSB 檢查點。如果設定為 true，將啟用指定的應用程式或模組以獲得高可用性。僅在較高層級 (例如伺服器和容器層級) 配置和啟用高可用性時才可將此選項設定為 true。

    如果存在預設伺服器實例 (server) 以外的叢集或獨立伺服器實例，便會顯示此選項。

  6. 從 [虛擬伺服器] 下拉式清單中，選取要與此應用程式關聯的虛擬伺服器。

    僅存在預設伺服器實例 server 時，才會顯示 [虛擬伺服器] 選項。如果存在叢集或其他獨立伺服器實例，您可以在部署之後選取虛擬伺服器。請移至 [編輯應用程式] 頁面，選取 [目標] 標籤，然後選取所需目標的 [管理 VirtualServer]。

  7. 選取 [啟用 Java Web Start] 核取方塊，以使用 Java Web Start 軟體來部署應用程式。

    此選項根據原始部署時的設定預設為啟用或停用。

  8. 在 [說明] 欄位中，鍵入此應用程式的說明。
  9. 若要檢視應用程式的模組描述元清單，請按一下 [說明] 標籤。如需更多詳細資訊，請參閱檢視應用程式的描述元
  10. 依預設，所有目標上的應用程式 [狀態] 均為 [已啟用]。若要檢視啟用了應用程式的目標，請按一下 [目標] 標籤。您可以在一個或多個目標上啟用或停用應用程式。請參閱檢視應用程式目標。
  11. 按一下 [儲存]。
