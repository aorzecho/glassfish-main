JACC 提供者

使用「JACC 提供者」頁面可組態所選 GlassFish Server 實例或叢集的 Java Authorization Contract for Containers (JACC) 提供者。

JACC 提供者可由協力廠商授權模組用於 GlassFish Server 中執行的應用程式。預設 GlassFish Server 安裝包括名為 default 與 simple 的兩個 JACC 提供者。這些預設提供者會實作符合 JACC 規格的簡單檔案型授權引擎。您也可以建立自己的自訂 JACC 提供者。請參閱建立 JACC 提供者。

如需了解如何撰寫 JACC 提供者的實作類別，請參閱 JACC 規格，網址為 http://jcp.org/en/jsr/detail?id=115。

對於每個 JACC 提供者，均提供以下資訊：

名稱

JACC 提供者的名稱。此名稱用於參照 domain.xml 中的 jacc-provider 元素。

策略組態

javax.security.jacc.PolicyConfigurationFactory.provider 的完全合格類別名稱，會實作提供者特定的 javax.security.jacc.PolicyConfigurationFactory。

策略提供者

javax.security.jacc.policy.provider 的完全合格類別名稱，會實作 java.security.Policy。

預設 JACC 提供者也支援下列特性。

儲存庫

包含 JACC 策略檔案的目錄。對於 default GlassFish Server JACC 提供者，預設目錄為 ${com.sun.aas.instanceRoot}/generated/policy。依預設，沒有為 simple GlassFish Server JACC 提供者定義此特性。

JACC 提供者表也包含下列選項。

新建

建立新 JACC 提供者的按鈕。

刪除

刪除一或多個選取的 JACC 提供者的按鈕。
