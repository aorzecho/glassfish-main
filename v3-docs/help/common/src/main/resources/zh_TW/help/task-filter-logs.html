篩選顯示的記錄

開始前

確定您正在檢視網域管理伺服器 (DAS)、叢集 GlassFish Server 實例或獨立實例的記錄檔。如需更多資訊，請參閱以下主題：

  • 檢視 DAS 的記錄檔

  • 檢視叢集 GlassFish Server 實例的記錄檔

  • 檢視獨立 GlassFish Server 實例的記錄檔

  1. 在 「搜尋條件」 區域中，依據內容、時段及記錄層級來篩選記錄。

    1. 若要僅顯示包含特定文字的記錄，請在 「文字搜尋」 欄位中，鍵入該文字。

      搜尋會區分大小寫。

    2. 若要選取顯示記錄的時段，請使用 「時間戳記」 選項。

      • 若要檢視最近訊息，請選取 「最近」 (預設值)。

      • 若要僅檢視特定時段內的訊息，請選取 「指定範圍」，然後在所顯示的 「從」 和 「到」 欄位中鍵入日期和時間值。

        「日期」 值必須採用以下格式：

        MM/DD/YY

        MM 是範圍 1-12 代表月份的整數，例如 1 表示一月，而 10 表示十月。DD 是範圍 1-31 代表當月某天的整數。YY 是年份的後兩個數字，例如 11 代表 2011 年。

        例如，若要指定的日期為 2011 年 2 月 1 日，請在「日期」欄位中鍵入 2/1/11。

        「時間」 值必須採用以下格式：

        hh:mm:ss.SSS

        hh 是範圍 0-23 代表小時的整數。對於下午的時間，請使用範圍 12-23 的整數，例如 14 代表下午 2 點。mm 是範圍 0-59 的整數，代表該小時已過去的分鐘數。ss 是範圍 0-59 的整數，代表秒數。SSS 是範圍 000-999 的三位整數 (必要時可加入前導零)，代表毫秒數。

        例如，若要指定時間為下午 5 點 10 分，請在 「時間」 欄位中鍵入 17:10:00.000。

        如果 「從」 欄位中的時間值遲於 「到」 欄位中的時間值，將顯示錯誤訊息。

    3. 依據記錄層級篩選訊息。

      為確保要檢視的訊息顯示在伺服器記錄中，請先在「模組記錄層級」頁面上設定適當的記錄層級。如需說明，請參閱組態記錄層級。

      如果選擇依據記錄層級篩選記錄訊息，則只會顯示符合指定篩選條件的訊息。不過，此篩選不影響那些記錄到伺服器記錄中的訊息。

      1. 從 「記錄層級」 下拉式清單中選取記錄層級。

      2. 若要僅顯示所選層級的訊息，請選取 「請勿包含更多嚴重訊息」 選項。

        依預設，會顯示伺服器記錄中所選擇記錄層級與更嚴重層級的所有訊息。

  2. 在 「進階搜尋」 區域中，依據記錄程式名稱空間篩選記錄，然後選取要顯示的資訊量。

    1. 按一下 「進階搜尋」 連結。

      會將 「進階搜尋」 區域增加至 「記錄檢視器」 視窗。

    2. 若要依據模組進行篩選，請從 「記錄程式」 下拉式清單中選取一或多個名稱空間。

      如果要選擇多個名稱空間，請在按住 Shift 鍵或 Ctrl 鍵的同時以滑鼠點選。

      選取較高層級的名稱空間時，也會連帶選取下層的所有名稱空間。例如，選取 javax.enterprise.system 的同時也就選取了該名稱空間下所有模組的記錄程式，例如 javax.enterprise.system.core、javax.enterprise.system.tools.admin 等。

    3. 若要檢視特定應用程式之記錄程式的訊息，請在 「自訂記錄程式」 欄位中，分行鍵入每個記錄程式的名稱。

      如果應用程式具有多個模組，您可以檢視任一模組，也可以檢視所有模組。例如，應用程式可能會有下列記錄程式：

      com.example.myapp.module1
com.example.myapp.module2
com.example.myapp.module3

      若要檢視應用程式中所有模組的訊息，請鍵入 com.mycompany.myapp。若要僅檢視 module2 的訊息，請鍵入 com.example.myapp.module2。

      如果指定了一個或多個自訂記錄程式，則僅當在 「記錄程式」 區域中明確指定 GlassFish Server 模組時，才會顯示這些模組的訊息。

    4. 從 「顯示」 下拉式清單中，選取要顯示的訊息數目。

      顯示的預設訊息數目為 40。其他選項包括 100、250 和 1000。

    5. 若要檢視堆疊追蹤，請取消選取 「限制過長訊息」 選項。

      依預設，堆疊追蹤不會出現在檢視器中。若要檢視堆疊追蹤，請按一下訊息的 (詳細資訊) 連結。

    6. 若要隱藏 「進階搜尋」 區域，請按一下 「基本搜尋」 連結。

  3. 若要指定希望檢視的實例及相關聯的記錄檔，請使用 「修改搜尋」 區域。

    • 從 「實例」 下拉式清單中選取實例。

      預設值為您最初選取要檢視記錄檔的實例。如果您最初選取要檢視網域管理伺服器 (DAS) 的記錄檔，則預設值為代表 DAS 的伺服器。

    • 從 「記錄檔」 下拉式清單中選取記錄檔。

      預設為 server.log。其他選項取決於所選取目標的記錄設定。

  4. 在 「搜尋結果」 區域中，自訂記錄訊息的顯示。

    「搜尋結果」 區域會透過 「記錄設定」 與 「記錄層級」 頁面中的設定，顯示伺服器記錄中最新的 40 個項目。

    • 若要以不同方式對訊息排序，請按一下欄中的箭頭。

    • 若要檢視訊息的格式化版本，請按一下訊息的 (詳細資訊) 連結。

      隨即會開啟包含訊息格式化版本的 「記錄項目詳細資訊」 視窗。

    • 若要檢視記錄檔中較舊或較新的項目，請按一下表格頂端的 「之前的記錄」 與 「之後的記錄」。
