編輯 JMX 連接器

使用 「管理服務」 可組態與 JSR-160 相容的遠端 JMX 連接器。

與 JSR-160 相容的遠端 JMX 連接器會處理 JMX 用戶端 (如 JConsole) 和 JMX 服務之間的通訊，以進行網域管理作業。

  1. 在瀏覽樹狀結構中，展開 「組態」 節點。

  2. 在 「組態」 節點下，選取 「管理服務」 節點。

    此時將會開啟 「編輯 JMX 連接器」 頁面。

  3. 選取「安全性」的「已啟用」核取方塊，以加密 JMX 通訊。

    此選項預設為停用。

  4. 在 「位址」 欄位中，鍵入 JMX 連接器伺服器 Stub 註冊所在命名服務的 IP 位址。

    這不是負責實際 JMX 通訊之伺服器通訊端的連接埠。這是 RMI 登錄啟動所在之網路介面的位址。如果您的系統有多個網路介面，請修改此值，以便只選取唯一特定介面。預設值為 0.0.0.0。

  5. 在 「連接埠」 欄位中，鍵入命名服務 (RMI 登錄) 偵聽 RMI 用戶端連線使用的連接埠號碼。

    此命名服務的唯一用途是下載 RMI Stub。如果預設連接埠被佔用，則會使用可用的連接埠。合法值為 1 - 65535。在 UNIX 系統上，建立在連接埠 1 - 1024 上偵聽的通訊端需要超級使用者權限。預設值為 8686。

  6. 在 「範圍名稱」 欄位中，鍵入表示特殊管理範圍的名稱。

    所有認證均由此範圍處理。預設值為 admin-realm。

  7. 在 「其他特性」 區段中，指定其他特性。

    GlassFish Server 預設會指定數個特性。

    • 若要增加特性，請按一下 「增加特性」 按鈕。在出現的空白列中，於 「名稱」 欄位中鍵入特性名稱，然後在 「值」 欄位中鍵入特性值。

    • 若要修改特性，請編輯特性的 「值」 欄位。

    • 若要刪除特性，請選取您要刪除之特性 「名稱」 欄位左側的核取方塊，然後按一下 「刪除特性」 按鈕。

  8. 按一下 「儲存」。

  9. 如果您選取「安全性」的「已啟用」核取方塊，請依照編輯 JMX 連接器的 SSL 設定中的步驟進行。
