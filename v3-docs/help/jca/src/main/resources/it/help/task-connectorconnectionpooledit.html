Modificare un pool di connessione del connettore

  1. Nella struttura ad albero, espandere il nodo Resources.
  2. Espandere il nodo Connectors.
  3. Nel nodo Connectors, selezionare il nodo Connector Connection Pools.

    Viene visualizzata la pagina Connector Connection Pools.

  4. Nella pagina Connector Connection Pools, fare clic sul nome del pool di connessione del connettore che verrà modificato.

    Viene visualizzata la pagina Edit Connector Connection Pool.

  5. Dall'elenco a discesa Resource Adapter, selezionare un altro adattatore di risorsa.
  6. Dall'elenco a discesa Connection Definition, selezionare il nome della classe per la definizione di connessione.

    Le opzioni nell'elenco dipendono dall'adattatore di risorsa selezionato. La definizione della connessione identifica un elemento connection-definition dell'adattatore di risorsa nel file ra.xml.

  7. Selezionare la casella di controllo Ping Enabled per eseguire il ping del pool di connessione durante il processo di creazione o riconfigurazione al fine di identificare e segnalare valori non corretti per i relativi attributi.

    Questa opzione è disabilitata per impostazione predefinita.

  8. Nel campo Description, digitare una descrizione del pool di connessione del connettore.
  9. Nel campo Initial and Minimum Pool Size, specificare il numero minimo di connessioni per il pool di connessione del connettore.

    Il valore predefinito è 8.

  10. Nel campo Maximum Pool Size, specificare il numero massimo di connessioni per il pool di connessione del connettore.

    Il valore predefinito è 32.

  11. Nel campo Pool Resize Quantity, specificare il numero di connessioni da rimuovere allo scadere del timeout di inattività del pool.

    Il valore predefinito è 2.

  12. Nel campo Idle Timeout, specificare il numero massimo di secondi durante i quali una connessione può rimanere inattiva nel pool.

    Il valore predefinito è 300.

  13. Nel campo Max Wait Time, specificare il tempo di attesa per la richiesta di connessione di un'applicazione, prima che si verifichi il timeout della connessione.

    Il valore predefinito è 60000.

  14. Selezionare la casella di controllo Connection Validation Required per abilitare la convalida della connessione.

    Se la casella di controllo Required è selezionata, le connessioni vengono convalidate prima di essere fornite all'applicazione. Se la convalida non viene eseguita, la risorsa viene eliminata. Viene creata e restituita una risorsa nuova. Questa opzione è disabilitata per impostazione predefinita.

  15. Se la casella di controllo On Any Failure è selezionata, in GlassFish Server tutte le connessioni del pool vengono chiuse e quindi ristabilite in caso di errore di una singola connessione.

    Se non si seleziona la casella di controllo, le singole connessioni vengono ristabilite solo quando vengono utilizzate.

  16. Dall'elenco a discesa Transaction Support, selezionare il tipo di supporto di transazione per il pool di connessione.

    Il supporto delle transazioni scelto sovrascrive l'attributo relativo al supporto delle transazioni nell'adattatore di risorsa associato a questo pool di connessione compatibilmente con gli elementi derivati. In altre parole, può supportare un livello di transazione inferiore a quello specificato nell'adattatore di risorsa, oppure un livello di transazione uguale a quello specificato nell'adattatore di risorsa, ma non un livello superiore.

    Nell'elenco a discesa sono disponibili le seguenti opzioni:

    • NoTransaction: indica che l'adattatore di risorsa non supporta le transazioni locali o JTA del gestore delle risorse e non implementa le interfacce XAResource o LocalTransaction.
    • LocalTransaction: indica che l'adattatore di risorsa supporta le transazioni locali tramite l'implementazione dell'interfaccia LocalTransaction.

      Le transazioni locali sono gestite internamente a un gestore delle risorse e non coinvolgono gestori di transazioni esterni.

    • XATransaction: indica che l'adattatore di risorsa supporta le transazioni locali e JTA del gestore delle risorse tramite l'implementazione delle interfacce LocalTransaction e XAResource.

      Le transazioni XA sono controllate e coordinate da un gestore delle transazioni esterno a un gestore delle risorse.

  17. (Opzionale) Fare clic sul pulsante Flush per scaricare tutte le connessioni dal pool.

    Se si fa clic sul pulsante Flush vengono eliminate le connessioni esistenti, ricreate le connessioni stabilite per il pool e ripristinate le dimensioni minime e iniziali del pool.

  18. (Opzionale) Fare clic sul pulsante Ping per verificare l'utilizzabilità del pool di connessione.
  19. Fare clic su Save.

    Suggerimento - Per tornare alle impostazioni predefinite, fare clic sul pulsante Load Defaults, quindi su Save.
