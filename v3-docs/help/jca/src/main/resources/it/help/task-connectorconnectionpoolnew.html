Creare un pool di connessione del connettore

Prima di cominciare

Prima di creare il pool, distribuire il modulo connettore (adattatore di risorsa) associato al pool. I valori specificati per il nuovo pool dipendono dal modulo connettore che viene distribuito.

  1. Nella struttura ad albero, espandere il nodo Resources.
  2. Espandere il nodo Connectors.
  3. Nel nodo Connectors, selezionare il nodo Connector Connection Pools.

    Viene visualizzata la pagina Connector Connection Pools.

  4. Nella pagina Connector Connection Pools, fare clic su New.

    Viene visualizzata la pagina New Connector Connection Pool (Step 1 of 2).

  5. Nel campo Pool Name, digitare un nome univoco per il pool.

    Specificare questo nome durante la creazione di una risorsa connettore.

  6. Dall'elenco a discesa Resource Adapter, selezionare un'opzione dall'elenco degli adattatori di risorsa distribuiti (moduli connettori).
  7. Dall'elenco a discesa Connection Definition, selezionare la definizione di connessione per il pool di connessione del connettore.

    Le opzioni nell'elenco dipendono dall'adattatore di risorsa selezionato. La definizione della connessione identifica un elemento connection-definition dell'adattatore di risorsa nel file ra.xml.

  8. Fare clic su Next.

    Viene visualizzata la pagina New Connector Connection Pool (Step 2 of 2).

  9. Selezionare la casella di controllo Ping Enabled per eseguire il ping del pool di connessione durante il processo di creazione o riconfigurazione al fine di identificare e segnalare valori non corretti per i relativi attributi.

    Questa opzione è disabilitata per impostazione predefinita.

  10. Nel campo Description, digitare una descrizione del pool di connessione.
  11. Nel campo Initial and Minimum Pool Size, specificare il numero minimo di connessioni per il pool.

    Il valore predefinito è 8.

  12. Nel campo Maximum Pool Size, specificare il numero massimo di connessioni per il pool.

    Il valore predefinito è 32.

  13. Nel campo Pool Resize Quantity, specificare il numero di connessioni da rimuovere allo scadere del timeout di inattività del pool.

    Il valore predefinito è 2.

  14. Nel campo Idle Timeout, specificare il numero massimo di secondi durante i quali una connessione può rimanere inattiva nel pool.

    Il valore predefinito è 300.

  15. Nel campo Max Wait Time, specificare il tempo di attesa per la richiesta di connessione di un'applicazione, prima che si verifichi il timeout della connessione.

    Il valore predefinito è 60000.

  16. Selezionare la casella di controllo Connection Validation Required per abilitare la convalida della connessione.

    Se la casella di controllo Required è selezionata, le connessioni vengono convalidate prima di essere fornite all'applicazione. Se la convalida non viene eseguita, la risorsa viene eliminata. Viene creata e restituita una risorsa nuova. Questa opzione è disabilitata per impostazione predefinita.

  17. Se la casella di controllo On Any Failure Close All Connections è selezionata, in GlassFish Server tutte le connessioni del pool vengono chiuse e quindi ristabilite in caso di errore di una singola connessione.

    Se non si seleziona la casella di controllo, le singole connessioni vengono ristabilite solo quando vengono utilizzate. Questa opzione è disabilitata per impostazione predefinita.

  18. Dall'elenco a discesa Transaction Support, selezionare il tipo di supporto di transazione per il pool di connessione.

    Il supporto delle transazioni scelto sovrascrive l'attributo relativo al supporto delle transazioni nell'adattatore di risorsa associato a questo pool di connessione compatibilmente con gli elementi derivati. In altre parole, può supportare un livello di transazione inferiore a quello specificato nell'adattatore di risorsa, oppure un livello di transazione uguale a quello specificato nell'adattatore di risorsa, ma non un livello superiore.

    Nell'elenco a discesa sono disponibili le seguenti opzioni:

    • NoTransaction: indica che l'adattatore di risorsa non supporta le transazioni locali o JTA del gestore delle risorse e non implementa le interfacce XAResource o LocalTransaction.
    • LocalTransaction: indica che l'adattatore di risorsa supporta le transazioni locali tramite l'implementazione dell'interfaccia LocalTransaction.

      Le transazioni locali sono gestite internamente a un gestore delle risorse e non coinvolgono gestori di transazioni esterni.

    • XATransaction: indica che l'adattatore di risorsa supporta le transazioni locali e JTA del gestore delle risorse tramite l'implementazione delle interfacce LocalTransaction e XAResource.

      Le transazioni XA sono controllate e coordinate da un gestore delle transazioni esterno a un gestore delle risorse.

  19. (Opzionale) Nella sezione Additional Properties, specificare le proprietà aggiuntive per il pool di connessione del connettore.

    Le proprietà disponibili dipendono dall'adattatore di risorsa relativo al pool di connessione del connettore.

    Per aggiungere una proprietà, fare clic sul pulsante Add Property. Nella riga vuota visualizzata, digitare il nome della proprietà nel campo Name e il valore della proprietà nel campo Value.

    Se si specifica un nome ma non un valore per una proprietà, questa viene rimossa dalla tabella al termine della creazione del pool di connessione. Se anziché impostare il valore di una proprietà in questa fase si preferisce conservare la proprietà nella tabella in modo da impostare il valore in un secondo momento, digitare due parentesi vuote nel campo Value:

    ()
  20. Fare clic su OK.
