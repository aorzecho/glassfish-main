Modifica del pool di connessione del connettore

Utilizzare la pagina Edit Connector Connection Pool per modificare le impostazioni di pool di connessione del connettore.

La pagina Edit Connector Connection Pool contiene le opzioni seguenti.

Load Defaults

Pulsante che consente di ripristinare i valori predefiniti delle impostazioni che dispongono di tali valori. Le impostazioni prive di valori predefiniti restano invariate.

Flush

Pulsante che consente di reinizializzare le connessioni nel pool di connessione. Se si fa clic su questo pulsante vengono eliminate le connessioni esistenti, ricreate le connessioni stabilite per il pool e ripristinate le dimensioni minime e iniziali del pool.

Ping

Pulsante che consente di verificare se il pool di connessione è utilizzabile.

Pool Name

Nome univoco che identifica il pool di connessione del connettore. Il campo è di sola lettura. È possibile specificare il nome del pool soltanto quando si crea un pool di connessione del connettore.

Resource Adapter

Adattatore di risorsa associato al pool di connessione del connettore.

Connection Definition

Definizione della connessione per il pool di connessione del connettore. Le opzioni nell'elenco dipendono dall'adattatore di risorsa selezionato. La definizione della connessione identifica un elemento connection-definition dell'adattatore di risorsa nel file ra.xml.

Ping

Se questa opzione è abilitata, durante il processo di creazione o riconfigurazione viene eseguito il ping del pool di connessione per identificare e segnalare eventuali valori non corretti per i relativi attributi. Questa opzione è disabilitata per impostazione predefinita.

Description

Descrizione del pool di connessione del connettore.

Initial and Minimum Pool Size

Numero minimo di connessioni per il pool di connessione del connettore. Il valore predefinito è 8.

Maximum Pool Size

Numero massimo di connessioni per il pool di connessione del connettore. Il valore predefinito è 32.

Pool Resize Quantity

Numero di connessioni da rimuovere allo scadere del timeout di inattività del pool. Il valore predefinito è 2.

Idle Timeout

Numero massimo di secondi durante i quali una connessione può rimanere inattiva nel pool. Il valore predefinito è 300.

Max Wait Time

Tempo di attesa per una richiesta di connessione da parte dell'applicazione prima che si verifichi di un timeout. Il valore predefinito è 60000.

Connection Validation

Se la casella di controllo Required è selezionata, le connessioni vengono convalidate prima di essere fornite all'applicazione. Se la convalida non viene eseguita, la risorsa viene eliminata. Viene creata e restituita una risorsa nuova. Questa opzione è disabilitata per impostazione predefinita.

On Any Failure

Se la casella di controllo Close All Connections è selezionata, in GlassFish Server tutte le connessioni del pool vengono chiuse e quindi ristabilite in caso di errore di una singola connessione. Questa opzione è disabilitata per impostazione predefinita.

Transaction Support

Il supporto delle transazioni scelto sovrascrive l'attributo relativo al supporto delle transazioni nell'adattatore di risorsa associato a questo pool di connessione compatibilmente con gli elementi derivati. In altre parole, può supportare un livello di transazione inferiore a quello specificato nell'adattatore di risorsa, oppure un livello di transazione uguale a quello specificato nell'adattatore di risorsa, ma non un livello superiore.

Il valore può essere uno qualsiasi dei seguenti:

XATransaction

L'adattatore di risorsa supporta le transazioni locali e JTA del gestore delle risorse tramite l'implementazione delle interfacce LocalTransaction e XAResource. Le transazioni XA sono controllate e coordinate da un gestore delle transazioni esterno a un gestore delle risorse.

LocalTransaction

L'adattatore di risorsa supporta le transazioni locali tramite l'implementazione dell'interfaccia LocalTransaction. Le transazioni locali sono gestite internamente a un gestore delle risorse e non coinvolgono gestori di transazioni esterni.

NoTransaction

L'adattatore di risorsa non supporta le transazioni locali o JTA del gestore delle risorse e non implementa le interfacce XAResource o LocalTransaction.
