Work Security Maps

Nella pagina Work Security Maps è possibile configurare le mappe di sicurezza operativa per adattatori di risorsa.

Una mappa di sicurezza operativa per un connettore esegue il mapping dell'ID chiamante dell'operazione inviata da identità EIS o gruppi utenti EIS di un adattatore di risorsa su un'identità o gruppo utenti idoneo presente nel dominio di protezione di GlassFish Server. A un adattatore di risorsa è possibile associare una o più mappe di sicurezza operativa. La configurazione delle mappe di sicurezza operativa del connettore supporta l'utilizzo dell'asterisco come carattere speciale (*) per indicare tutti gli utenti o i gruppi di utenti.

Una mappa di sicurezza operativa è diversa da una mappa di sicurezza per un pool di connessione del connettore, che esegue il mapping dell'identità ricevuta durante l'autenticazione di servlet o EJB sulle credenziali accettate dal sistema EIS. Per informazioni sulle mappe di sicurezza per i pool di connessione del connettore, vedere Connector Connection Pool Security Maps.

Per sistemi EIS (Enterprise Information Software) si intendono tutti i sistemi che consentono la gestione dei dati di un'organizzazione. Possono essere rappresentati da mainframe, sistemi di messaggistica, sistemi di database o applicazioni.

Per ogni mappa di sicurezza operativa vengono fornite le seguenti informazioni.

Name

Nome della mappa di sicurezza operativa.

La tabella Work Security Maps contiene inoltre le seguenti opzioni.

New

Pulsante che consente di creare una nuova mappa di sicurezza operativa.

Delete

Pulsante che consente di eliminare una o più mappe di sicurezza operativa selezionate.
