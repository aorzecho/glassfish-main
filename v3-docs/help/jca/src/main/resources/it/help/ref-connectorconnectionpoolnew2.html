Nuovo connection pool connettore (passo 2 di 2)

Nella pagina Nuovo connection pool connettore (passo 2 di 2) è possibile creare un nuovo connection pool del connettore.

La pagina Nuovo connection pool connettore (passo 2 di 2) contiene le opzioni riportate di seguito.

Ping

Se questa opzione è abilitata, viene eseguito il ping del connection pool durante la creazione o la riconfigurazione in modo da identificare eventuali valori errati degli attributi e segnalarli. Questa opzione è disabilitata per impostazione predefinita.

Descrizione

Descrizione del connection pool del connettore.

Dimensione pool iniziale e minima

Numero minimo di connessioni nel connection pool del connettore. Il valore predefinito è 8.

Dimensione massima pool

Numero massimo di connessioni nel connection pool del connettore. Il valore predefinito è 32.

Quantità di ridimensionamento pool

Numero di connessioni da rimuovere allo scadere del timeout di inattività del pool. Il valore predefinito è 2.

Timeout inattività

Tempo massimo, in secondi, durante il quale una connessione può rimanere inattiva nel pool. Il valore predefinito è 300.

Tempo massimo di attesa

Quantità di tempo che l'applicazione che richiede la connessione attende prima che si verifichi il timeout della connessione. Il valore predefinito è 60000.

Convalida connessione

Se la casella di controllo Obbligatorio è selezionata, le connessioni vengono convalidate prima di essere fornite all'applicazione. Se la convalida di una risorsa non riesce, la risorsa viene eliminata e viene creata e restituita una nuova risorsa. Questa opzione è disabilitata per impostazione predefinita.

In caso di qualsiasi errore

Se la casella di controllo Chiudi tutte le connessioni è selezionata, GlassFish Server chiude tutte le connessioni nel pool e le ristabilisce se una singola connessione non riesce. Questa opzione è disabilitata per impostazione predefinita.

Supporto transazioni

Il supporto delle transazioni scelto sostituisce l'attributo relativo al supporto delle transazioni nel Resource Adapter associato a questo connection pool in modo compatibile verso il basso. In altre parole, può supportare un livello di transazione inferiore o uguale a quello specificato nel Resource Adapter, ma non un livello superiore.

Il valore può essere uno dei seguenti:

XATransaction

Il Resource Adapter supporta le transazioni locali e JTA del Resource Manager mediante l'implementazione delle interfacce LocalTransaction e XAResource. Le transazioni XA sono controllate e coordinate da un Transaction Manager esterno a un Resource Manager.

LocalTransaction

Il Resource Adapter supporta le transazioni locali mediante l'implementazione dell'interfaccia LocalTransaction. Le transazioni locali sono gestite internamente a un Resource Manager e non coinvolgono alcun Transaction Manager esterno.

NoTransaction

Il Resource Adapter non supporta le transazioni locali o JTA del Resource Manager e non implementa le interfacce XAResource o LocalTransaction.

Proprietà aggiuntive

Proprietà aggiuntive per il connection pool del connettore. Le proprietà disponibili dipendono dal Resource Adapter relativo al connection pool del connettore. Le coppie nome-valore specificate in questa tabella possono essere utilizzate per sostituire i valori predefiniti per le proprietà specificate dal fornitore del Resource Adapter.

Se si specifica un nome ma non un valore per una proprietà, questa viene rimossa dalla tabella al termine della creazione del connection pool. Se anziché impostare il valore di una proprietà in questa fase si preferisce conservare la proprietà nella tabella in modo da impostare il valore in un secondo momento, digitare un set di parentesi vuote nel campo Valore:
