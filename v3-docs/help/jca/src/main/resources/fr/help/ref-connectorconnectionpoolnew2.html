Nouveau pool de connexions de connecteur (étape 2 sur 2)

Utilisez la page Nouveau pool de connexions de connecteur (étape 2 sur 2) pour créer un nouveau pool de connexions de connecteur.

La page Nouveau pool de connexions de connecteur (étape 2 sur 2) contient les options suivantes.

Ping

Lorsque cette option est activée, un ping est réalisé vers le pool au cours de la création ou de la reconfiguration afin d’identifier et de signaler les valeurs incorrectes de ses attributs, le cas échéant. Cette option est désactivée par défaut.

Description

Une description du pool de connexions du connecteur.

Taille initiale et minimale du pool

Le nombre minimum de connexions dans le pool de connexions de connecteur. La valeur par défaut est 8.

Taille maximale du pool

Le nombre maximum de connexions dans le pool de connexions de connecteur. La valeur par défaut est 32.

Valeur de redimensionnement du pool

Le nombre de connexions à supprimer lorsque le délai d’inactivité du pool expire. La valeur par défaut est 2.

Délai d’inactivité

Temps maximum en secondes durant lequel une connexion demeure inactive dans le pool. La valeur par défaut est 300.

Attente max.

Durée pendant laquelle l’application demandant une connexion doit patienter avant d’être notifiée de l’expiration de la connexion. La valeur par défaut est 60 000.

Validation des connexions

Si la case Requis est sélectionnée, les connexions sont validées avant d’être données à l’application. Si la validation d’une ressource échoue, elle est détruite et une nouvelle ressource est créée et renvoyée. Cette option est désactivée par défaut.

Sur tout échec

Si la case Fermer toutes les connexions est cochée, GlassFish Server ferme toutes les connexions du pool et les rétablit si une seule connexion échoue. Cette option est désactivée par défaut.

Prise en charge des transactions

La prise en charge des transactions choisie remplace l’attribut de prise en charge des transactions dans l’adaptateur de ressources associé à ce pool de connexions de façon à garantir la compatibilité descendante. En d’autres mots, il peut prendre en charge un niveau de transaction inférieur à celui spécifié dans l’adaptateur de ressources, ou le même niveau de transaction que celui spécifié dans l’adaptateur de ressources, mais il ne peut pas spécifier un niveau supérieur.

La valeur peut être l’une des suivantes :

XATransaction

L’adaptateur de ressources prend en charge les transactions JTA et les transactions locales du gestionnaire de ressources en implémentant les interfaces LocalTransaction et XAResource. Les transactions XA sont contrôlées et déterminées par un gestionnaire de transactions externe à un gestionnaire de ressources.

LocalTransaction

L’adaptateur de ressources prend en charge les transactions locales en implémentant l’interface LocalTransaction. Les transactions locales sont gérées dans un gestionnaire de ressources et n’impliquent aucun gestionnaire de transactions externe.

NoTransaction

L’adaptateur de ressources ne prend pas en charge les transactions JTA ou les transactions locales du gestionnaire de ressources et n’implémente pas les interfaces XAResource ou LocalTransaction.

Autres propriétés

Propriétés supplémentaires pour le pool de connexions de connecteur. Les propriétés disponibles dépendent de l’adaptateur de ressources pour le pool de connexions du connecteur. Les paires Nom-Valeur spécifiées dans ce tableau peuvent être utilisées pour remplacer les valeurs par défaut pour les propriétés définies par le fournisseur de l’adaptateur de ressources.

Si vous indiquez un nom mais pas une valeur pour une propriété, il est supprimé du tableau lorsque vous terminez la création du pool de connexions. Si vous ne souhaitez pas définir immédiatement la valeur d’une propriété mais souhaitez conserver cette propriété dans la table afin de définir sa valeur ultérieurement, entrez des parenthèses vides dans le champ Valeur :
