Modifier un pool de connexions de connecteur

  1. Dans l'arborescence de navigation, développez le noeud Ressources.

  2. Développez le noeud Connecteurs.

  3. Sous le noeud Connecteurs, sélectionnez le noeud Pools de connexions du connecteur.

    La page Pools de connexions du connecteur apparaît.

  4. Sur la page Pools de connexions du connecteur, cliquez sur le nom du pool de connexions de connecteur à modifier.

    La page Modifier le pool de connexions du connecteur apparaît.

  5. Dans la liste déroulante Adaptateur de ressources, sélectionnez un autre adaptateur de ressource.

  6. Dans la liste déroulante Définition de connexion, sélectionnez le nom de classe de la définition de connexion.

    Les options disponibles dans la liste dépendent de l'adaptateur de ressource sélectionné. La définition de connexion identifie l'élément connection-definition d'un adaptateur de ressource dans le fichier ra.xml.

  7. Sélectionnez la case Activé du champ Ping pour envoyer la commande ping au pool de connexions lors de sa création ou de sa reconfiguration, afin d'identifier et de signaler toute erreur de valeur d'attribut.

    Cette option est désactivée par défaut.

  8. Dans le champ Description, saisissez la description du pool de connexions de connecteur.

  9. Dans le champ Taille initiale et minimale du pool, spécifiez le nombre minimal de connexions dans le pool de connexions de connecteur.

    La valeur par défaut est 8.

  10. Dans le champ Taille maximale du pool, spécifiez le nombre maximal de connexions dans le pool de connexions du connecteur.

    La valeur par défaut est 32.

  11. Dans le champ Valeur de redimensionnement du pool, spécifiez le nombre de connexions à enlever lorsque le délai d'inactivité du pool est dépassé.

    La valeur par défaut est 2.

  12. Dans le champ Délai d'inactivité, spécifiez la durée maximale, en secondes, pendant laquelle une connexion peut rester inactive dans le pool.

    La valeur par défaut est 300.

  13. Dans le champ Attente max., spécifiez le délai d'attente au-delà duquel il y a temporisation de la connexion pour l'application demandant une connexion.

    La valeur par défaut est 60 000.

  14. Cochez la case Requis du champ Validation des connexions pour activer la validation des connexions.

    Si la case Requis est cochée, les connexions sont validées avant d'être affectées à l'application. Si la validation d'une ressource échoue, elle est détruite et une autre ressource est créée puis renvoyée. Cette option est désactivée par défaut.

  15. Sélectionnez la case Echec sur tout afin que GlassFish Server ferme toutes les connexions du pool et les rétablisse en cas d'échec de l'une d'entre elles.

    Si vous ne sélectionnez pas cette case, les connexions individuelles ne seront rétablies que lorsqu'elles seront utilisées.

  16. Dans la liste déroulante Prise en charge des transactions, sélectionnez le type de prise en charge des transactions pour le pool de connexions.

    La prise en charge des transactions choisie remplace l'attribut de prise en charge des transactions dans l'adaptateur de ressource associé au pool de connexions de façon à garantir la compatibilité descendante. En d'autres termes, la prise en charge d'un niveau de transaction inférieur ou égal à celui spécifié dans l'adaptateur de ressource est assurée, mais pas la prise en charge d'un niveau supérieur.

    Les options suivantes sont disponibles dans la liste déroulante :

    • NoTransaction : indique que l'adaptateur de ressource ne prend pas en charge les transactions JTA ou les transactions locales du gestionnaire de ressources et n'implémente pas les interfaces XAResource ou LocalTransaction.

    • LocalTransaction : indique que l'adaptateur de ressource prend en charge les transactions locales en implémentant l'interface LocalTransaction.

      Les transactions locales sont gérées dans un gestionnaire de ressources et n'impliquent aucun gestionnaire de transactions externe.

    • XATransaction : indique que l'adaptateur de ressource prend en charge les transactions JTA et les transactions locales du gestionnaire de ressources en implémentant les interfaces LocalTransaction et XAResource.

      Les transactions XA sont contrôlées et déterminées par un gestionnaire de transactions externe à un gestionnaire de ressources.

  17. Cliquez sur le bouton Vider pour vider toutes les connexions du pool.

    Cliquez sur le bouton Vider revient à détruire toutes les connexions existantes, à recréer les connexions établies pour le pool et à restaurer le pool à sa taille initiale et minimale.

  18. Cliquez sur le bouton Ping pour vérifier que le pool de connexions est utilisable.

  19. Cliquez sur Enregistrer.

    Conseil:

    Pour revenir aux paramètres par défaut, cliquez sur le bouton Charger les valeurs par défaut, puis sur Enregistrer.
