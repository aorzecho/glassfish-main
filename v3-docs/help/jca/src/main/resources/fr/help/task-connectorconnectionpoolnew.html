Créer un pool de connexions de connecteur

Avant de commencer

Avant de créer le pool, déployez le module connecteur (adaptateur de ressources) associé. Les valeurs qui sont spécifiées pour le nouveau pool dépendent du module de connecteur déployé.

  1. Dans l’arborescence de navigation, développez le nœud Ressources.
  2. Développez le nœud Connecteurs.
  3. Sous le nœud Connecteurs, sélectionnez le nœud Pools de connexions du connecteur.

    La page Pools de connexions du connecteur s’ouvre.

  4. Cliquez sur le bouton Nouveau de la page Pools de connexions du connecteur.

    La page Nouveau pool de connexions de connecteur (étape 1 sur 2) s’ouvre.

  5. Dans le champ Nom du pool, saisissez un nom unique pour le pool.

    Saisissez ce nom lors de la création d’une ressource de connecteur.

  6. Dans la liste déroulante Adaptateur de ressources, faites une sélection dans la liste des adaptateurs de ressources déployés (modules connecteurs).
  7. Dans la liste déroulante Définition de connexion, sélectionnez la définition de connexion pour le pool de connexions du connecteur.

    Les choix dans la liste dépendent de l’adaptateur de ressource que vous avez sélectionné. La définition de connexion identifie l’élément connection-definition d’un adaptateur de ressource dans le fichier ra.xml.

  8. Cliquez sur Suivant.

    La page Nouveau pool de connexions de connecteur (étape 2 sur 2) s’ouvre.

  9. Sélectionnez la case Activé de Ping pour envoyer une requête de ping au pool de connexions lors de la création ou de la reconfiguration pour identifier et avertir de toute valeur erronée pour ses attributs.

    Cette option est désactivée par défaut.

  10. Dans le champ Description, saisissez une description du pool de connexions.
  11. Dans le champ Taille initiale et minimale du pool, spécifiez le nombre minimum de connexions dans le pool.

    La valeur par défaut est 8.

  12. Dans le champ Taille maximale du pool, spécifiez le nombre maximum de connexions dans le pool.

    La valeur par défaut est 32.

  13. Dans le champ Valeur de redimensionnement du pool, spécifiez le nombre de connexions à supprimer lorsque le délai d’inactivité du pool expire.

    La valeur par défaut est 2.

  14. Dans le champ Délai d’inactivité, spécifiez le temps maximum (en secondes) pendant lequel une connexion peut rester inactive dans le pool.

    La valeur par défaut est 300.

  15. Dans le champ Attente max., spécifiez le temps pendant lequel l’application demandant une connexion patientera avant d’obtenir une expiration de la connexion.

    La valeur par défaut est 60 000.

  16. Cochez la case Validation des connexions requise pour activer la validation des connexions.

    Si la case Requis est sélectionnée, les connexions sont validées avant d’être données à l’application. Si la validation d’une ressource échoue, elle est détruite et une nouvelle ressource est créée et renvoyée. Cette option est désactivée par défaut.

  17. Si la case Échec sur tout Fermer toutes les connexions est cochée, GlassFish Server interrompt toutes les connexions du pool et les rétablit si une seule connexion échoue.

    Si vous ne sélectionnez pas cette case, les connexions individuelles ne seront rétablies que lorsqu’elles seront utilisées. Cette option est désactivée par défaut.

  18. Dans la liste déroulante Prise en charge des transactions, sélectionnez le type de prise en charge de transactions pour le pool de connexions.

    La prise en charge des transactions choisie remplace l’attribut de prise en charge des transactions dans l’adaptateur de ressources associé à ce pool de connexions de façon à garantir la compatibilité descendante. En d’autres mots, il peut prendre en charge un niveau de transaction inférieur à celui spécifié dans l’adaptateur de ressources, ou le même niveau de transaction que celui spécifié dans l’adaptateur de ressources, mais il ne peut pas spécifier un niveau supérieur.

    Les options suivantes sont disponibles dans la liste déroulante :

    • NoTransaction : indique que l’adaptateur de ressources ne prend pas en charge les transactions JTA ou les transactions locales du gestionnaire de ressources et n’implémente pas les interfaces XAResource ou LocalTransaction.
    • LocalTransaction : indique que l’adaptateur de ressources prend en charge des transactions locales en implémentant l’interface LocalTransaction.

      Les transactions locales sont gérées dans un gestionnaire de ressources et n’impliquent aucun gestionnaire de transactions externe.

    • XATransaction : indique que l’adaptateur de ressources prend en charge les transactions JTA et les transactions locales du gestionnaire de ressources en implémentant les interfaces LocalTransaction et XAResource.

      Les transactions XA sont contrôlées et déterminées par un gestionnaire de transactions externe à un gestionnaire de ressources.

  19. (Facultatif) Dans la section Autres propriétés, spécifiez les autres propriétés pour le pool de connexions du connecteur.

    Les propriétés disponibles dépendent de l’adaptateur de ressources pour le pool de connexions du connecteur.

    Pour ajouter une propriété, cliquez sur le bouton Ajouter propriété. Sur la ligne vide qui apparaît, saisissez le nom de la propriété dans le champ Nom et la valeur de la propriété dans le champ Valeur.

    Si vous indiquez un nom mais pas une valeur pour une propriété, il est supprimé du tableau lorsque vous terminez la création du pool de connexions. Si vous ne souhaitez pas définir immédiatement la valeur d’une propriété mais souhaitez conserver cette propriété dans la table afin de définir sa valeur ultérieurement, entrez des parenthèses vides dans le champ Valeur :

    ()
  20. Cliquez sur OK.
