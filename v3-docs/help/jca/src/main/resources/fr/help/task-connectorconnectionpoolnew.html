Créer un pool de connexions de connecteur

Avant de commencer

Avant de créer le pool, déployez le module connecteur (adaptateur de ressource) associé. Les valeurs qui sont spécifiées pour le nouveau pool dépendent du module connecteur déployé.

  1. Dans l'arborescence de navigation, développez le noeud Ressources.

  2. Développez le noeud Connecteurs.

  3. Sous le noeud Connecteurs, sélectionnez le noeud Pools de connexions du connecteur.

    La page Pools de connexions du connecteur apparaît.

  4. Sur la page Pools de connexions du connecteur, cliquez sur Nouveau.

    La page Nouveau pool de connexions de connecteur (étape 1 sur 2) apparaît.

  5. Dans le champ Nom du pool, attribuez un nom unique au pool.

    Saisissez ce nom lors de la création d'une ressource de connecteur.

  6. Dans la liste déroulante Adaptateur de ressources, sélectionnez les adaptateurs de ressource déployés (modules connecteur) souhaités.

  7. Dans la liste déroulante Définition de connexion, sélectionnez la définition de connexion pour le pool de connexions de connecteur.

    Les options disponibles dans la liste dépendent de l'adaptateur de ressource sélectionné. La définition de connexion identifie l'élément connection-definition d'un adaptateur de ressource dans le fichier ra.xml.

  8. Cliquez sur Suivant.

    La page Nouveau pool de connexions de connecteur (étape 2 sur 2) apparaît.

  9. Sélectionnez la case Activé du champ Ping pour envoyer la commande ping au pool de connexions lors de sa création ou de sa reconfiguration, afin d'identifier et de signaler toute erreur de valeur d'attribut.

    Cette option est désactivée par défaut.

  10. Dans le champ Description, saisissez la description du pool de connexions.

  11. Dans le champ Taille initiale et minimale du pool, spécifiez le nombre minimal de connexions dans le pool.

    La valeur par défaut est 8.

  12. Dans le champ Taille maximale du pool, spécifiez le nombre maximal de connexions dans le pool.

    La valeur par défaut est 32.

  13. Dans le champ Valeur de redimensionnement du pool, spécifiez le nombre de connexions à enlever lorsque le délai d'inactivité du pool est dépassé.

    La valeur par défaut est 2.

  14. Dans le champ Délai d'inactivité, spécifiez la durée maximale, en secondes, pendant laquelle une connexion peut rester inactive dans le pool.

    La valeur par défaut est 300.

  15. Dans le champ Attente max., spécifiez le délai d'attente au-delà duquel il y a temporisation de la connexion pour l'application demandant une connexion.

    La valeur par défaut est 60 000.

  16. Cochez la case Requis du champ Validation des connexions pour activer la validation des connexions.

    Si la case Requis est cochée, les connexions sont validées avant d'être affectées à l'application. Si la validation d'une ressource échoue, elle est détruite et une autre ressource est créée puis renvoyée. Cette option est désactivée par défaut.

  17. Si la case Fermer toutes les connexions du champ Echec sur tout est cochée, GlassFish Server ferme toutes les connexions du pool et les rétablit en cas d'échec de l'une d'entre elles.

    Si vous ne sélectionnez pas cette case, les connexions individuelles ne seront rétablies que lorsqu'elles seront utilisées. Cette option est désactivée par défaut.

  18. Dans la liste déroulante Prise en charge des transactions, sélectionnez le type de prise en charge des transactions pour le pool de connexions.

    La prise en charge des transactions choisie remplace l'attribut de prise en charge des transactions dans l'adaptateur de ressource associé au pool de connexions de façon à garantir la compatibilité descendante. En d'autres termes, la prise en charge d'un niveau de transaction inférieur ou égal à celui spécifié dans l'adaptateur de ressource est assurée, mais pas la prise en charge d'un niveau supérieur.

    Les options suivantes sont disponibles dans la liste déroulante :

    • NoTransaction : indique que l'adaptateur de ressource ne prend pas en charge les transactions JTA ou les transactions locales du gestionnaire de ressources et n'implémente pas les interfaces XAResource ou LocalTransaction.

    • LocalTransaction : indique que l'adaptateur de ressource prend en charge les transactions locales en implémentant l'interface LocalTransaction.

      Les transactions locales sont gérées dans un gestionnaire de ressources et n'impliquent aucun gestionnaire de transactions externe.

    • XATransaction : indique que l'adaptateur de ressource prend en charge les transactions JTA et les transactions locales du gestionnaire de ressources en implémentant les interfaces LocalTransaction et XAResource.

      Les transactions XA sont contrôlées et déterminées par un gestionnaire de transactions externe à un gestionnaire de ressources.

  19. Dans la section Autres propriétés, spécifiez les propriétés supplémentaires du pool de connexions de connecteur.

    Les propriétés disponibles dépendent de l'adaptateur de ressource du pool de connexions de connecteur.

    Pour ajouter une propriété, cliquez sur le bouton Ajouter une propriété. Sur la ligne vide qui apparaît, saisissez le nom de la propriété dans le champ Nom et la valeur de la propriété dans le champ Valeur.

    Si vous indiquez un nom mais pas de valeur pour une propriété, celle-ci est enlevée de la table à l'issue de la création du pool de connexions. Si vous ne souhaitez pas définir immédiatement la valeur d'une propriété mais souhaitez conserver cette propriété dans la table afin de définir sa valeur ultérieurement, entrez des parenthèses vides dans le champ Valeur :

    ()

  20. Cliquez sur OK.
