Modifier le pool de connexions du connecteur

La page Modifier le pool de connexions du connecteur permet de modifier les paramètres d'un pool de connexions de connecteur.

Elle contient les options suivantes.

Charger les valeurs par défaut

Bouton de restauration des valeurs par défaut des paramètres qui ont des valeurs par défaut. Les paramètres qui n'ont pas de valeurs par défaut ne sont pas modifiés.

Vider

Bouton permettant de réinitialiser les connexions dans le pool de connexions. Cliquer sur ce bouton revient à détruire toutes les connexions existantes, à recréer les connexions établies pour le pool et à restaurer le pool à sa taille initiale et minimale.

Ping

Bouton permettant de vérifier que le pool de connexions est utilisable.

Nom du pool

Nom unique identifiant le pool de connexions de connecteur. Ce champ est en lecture seule. Vous ne pouvez spécifier le nom JNDI qu'à la création d'un pool de connexions de connecteur.

Adaptateur de ressources

Adaptateur de ressource associé au pool de connexions de connecteur.

Définition de connexion

Définition de connexion pour le pool de connexions de connecteur. Les options disponibles dans la liste dépendent de l'adaptateur de ressource sélectionné. La définition de connexion identifie l'élément connection-definition d'un adaptateur de ressource dans le fichier ra.xml.

Ping

Lorsque cette option est activée, la commande ping est envoyée au pool de connexions lors de sa création ou de sa reconfiguration, afin d'identifier et de signaler toute erreur de valeur d'attribut. Cette option est désactivée par défaut.

Description

Description du pool de connexions de connecteur.

Taille initiale et minimale du pool

Nombre minimal de connexions dans le pool de connexions de connecteur. La valeur par défaut est 8.

Taille maximale du pool

Nombre maximal de connexions dans le pool de connexions de connecteur. La valeur par défaut est 32.

Valeur de redimensionnement du pool

Nombre de connexions à enlever lorsque le délai d'inactivité du pool est dépassé. La valeur par défaut est 2.

Délai d'inactivité

Durée maximale, en secondes, pendant laquelle une connexion peut rester inactive dans le pool. La valeur par défaut est 300.

Attente max.

Délai d'attente d'une application demandant une connexion au-delà duquel il y a temporisation de la connexion. La valeur par défaut est 60 000.

Validation de la connexion

Si la case Requis est cochée, les connexions sont validées avant d'être affectées à l'application. Si la validation d'une ressource échoue, elle est détruite et une autre ressource est créée puis renvoyée. Cette option est désactivée par défaut.

Echec sur tout

Si la case Fermer toutes les connexions est cochée, GlassFish Server ferme toutes les connexions du pool et les rétablit en cas d'échec de l'une d'entre elles. Cette option est désactivée par défaut.

Prise en charge des transactions

La prise en charge des transactions choisie remplace l'attribut de prise en charge des transactions dans l'adaptateur de ressource associé au pool de connexions de façon à garantir la compatibilité descendante. En d'autres termes, la prise en charge d'un niveau de transaction inférieur ou égal à celui spécifié dans l'adaptateur de ressource est assurée, mais pas la prise en charge d'un niveau supérieur.

Les valeurs possibles sont les suivantes :

XATransaction

L'adaptateur de ressource prend en charge les transactions JTA et les transactions locales du gestionnaire de ressources en implémentant les interfaces LocalTransaction et XAResource. Les transactions XA sont contrôlées et déterminées par un gestionnaire de transactions externe à un gestionnaire de ressources.

LocalTransaction

L'adaptateur de ressource prend en charge les transactions locales en implémentant l'interface LocalTransaction. Les transactions locales sont gérées dans un gestionnaire de ressources et n'impliquent aucun gestionnaire de transactions externe.

NoTransaction

L'adaptateur de ressource ne prend pas en charge les transactions JTA ou les transactions locales du gestionnaire de ressources et n'implémente pas les interfaces XAResource ou LocalTransaction.
