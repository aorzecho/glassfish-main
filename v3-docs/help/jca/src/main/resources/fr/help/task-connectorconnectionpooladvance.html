Modifier les attributs avancés d'un pool de connexions de connecteur

  1. Dans l'arborescence de navigation, développez le noeud Ressources.

  2. Développez le noeud Connecteurs.

  3. Sous le noeud Connecteurs, sélectionnez le noeud Pools de connexions du connecteur.

    La page Pools de connexions du connecteur apparaît.

  4. Sur la page Pools de connexions du connecteur, cliquez sur le nom du pool de connexions de connecteur à modifier.

    La page Modifier le pool de connexions du connecteur apparaît.

  5. Sur la page Modifier le pool de connexions du connecteur, cliquez sur l'onglet Avancé.

    La page Modifier les attributs avancés d'un pool de connexions de connecteur apparaît.

  6. Dans le champ Valider une fois au plus, saisissez le délai, en secondes, au bout duquel une connexion est validée une fois au plus.

    La définition de cet attribut sur une valeur appropriée réduit le nombre de demandes de validation émanant d'une connexion. La valeur par défaut est 0, ce qui signifie que l'attribut n'est pas activé.

  7. Dans le champ Temporisation de perte, saisissez la durée, en secondes, nécessaire au suivi des pertes de connexion dans un pool de connexions.

    La valeur 0 indique que le suivi des pertes de connexion est désactivé. La valeur par défaut est 0.

  8. Sélectionnez la case Récupération de perte pour que les connexions présentant des pertes soient restaurées dans le pool une fois le suivi de pertes de connexion terminé.

    Cette option est désactivée par défaut.

  9. Dans le champ Tentatives de création, saisissez le nombre de tentatives effectuées en cas d'échec de création d'une connexion.

    La valeur 0 indique qu'aucune tentative ne sera effectuée pour recréer une connexion. La valeur par défaut est 0.

  10. Dans le champ Intervalle entre les tentatives, saisissez l'intervalle, en secondes, entre deux tentatives de création d'une connexion.

    Cet attribut n'est utilisé que si la valeur de l'option Tentatives de création est supérieure à 0. La valeur par défaut est de 10 secondes.

  11. Cochez la case Groupement pour activer le groupement des connexions du pool.

    Cette option est activée par défaut.

  12. Cochez la case Association lazy pour que les connexions puissent être associées en différé lorsque vous effectuez une opération sur celles-ci.

    Les connexions sont dissociées lorsque la transaction est terminée et qu'une méthode de composant prend fin, ce qui facilite la réutilisation des connexions physiques. Cette option est désactivée par défaut.

  13. Cochez la case Participation des connexions lazy à la transaction pour engager une ressource dans une transaction uniquement lorsqu'elle est utilisée dans une méthode.

    Cette option est désactivée par défaut.

  14. Sélectionnez la case Associer à un thread pour associer une connexion à un thread.

    Lorsque ce dernier nécessite une connexion, il peut réutiliser celle qui lui est déjà associée, limitant ainsi le temps système lié à l'obtention d'une connexion à partir du pool. Cette option est désactivée par défaut.

  15. Sélectionnez la case Faire correspondre les connexions pour activer l'association de connexions pour le pool.

    Désactivez cette option si vous savez que les connexions du pool seront toujours homogènes et qu'il est donc inutile pour l'adaptateur de ressource d'associer une connexion empruntée au pool. Cette option est activée par défaut.

  16. Dans le champ Utilisation maximale des connexions, saisissez le nombre de réutilisations possibles d'une connexion par le pool.

    Une fois qu'une connexion a été réutilisée le nombre de fois spécifié, elle est fermée. La valeur 0 indique que cet attribut n'est pas activé. La valeur par défaut est 0.

  17. Cliquez sur Enregistrer.

    Conseil:

    Pour revenir aux paramètres par défaut, cliquez sur le bouton Charger les valeurs par défaut, puis sur Enregistrer.
