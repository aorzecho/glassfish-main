Modifier les attributs avancés d’un pool de connexions de connecteur

  1. Dans l’arborescence de navigation, développez le nœud Ressources.
  2. Développez le nœud Connecteurs.
  3. Sous le nœud Connecteurs, sélectionnez le nœud Pools de connexions du connecteur.

    La page Pools de connexions du connecteur s’ouvre.

  4. Sur la page Pools de connexions du connecteur, cliquez le nom du pool de connexions du connecteur que vous modifiez.

    La page Modification du pool de connexions du connecteur s’ouvre.

  5. Sur la page Modification du pool de connexions du connecteur, cliquez sur l’onglet Avancé.

    La page Modifier les attributs avancés d'un pool de connexions de connecteur s’affiche.

  6. Dans le champ Valider une fois au plus, saisissez le délai, en secondes, au bout duquel une connexion est validée une fois au plus.

    La définition de cet attribut sur une valeur appropriée réduit le nombre de demandes de validation émanant d'une connexion. La valeur par défaut est 0, ce qui signifie que l'attribut n'est pas activé.

  7. Dans le champ Délai de fuite, saisissez la durée, en secondes, du suivi des fuites de connexion dans un pool de connexions.

    Une valeur de 0 indique que le suivi des fuites de connexion est désactivé. La valeur par défaut est 0.

  8. Sélectionnez la case Récupération de fuite pour activer les connections avec fuites à restaurer vers le pool une fois le suivi de fuites de connexions terminé.

    Cette option est désactivée par défaut.

  9. Dans le champ Tentatives de création, saisissez le nombre de tentatives de création en cas d’échec lors de la création d’une connexion.

    Une valeur de 0 indique qu’aucune tentative ne sera effectuée pour créer une nouvelle connexion. La valeur par défaut est 0.

  10. Dans le champ Intervalle de tentative, saisissez l’intervalle, en secondes, entre deux tentatives de création d’une connexion.

    Cet attribut n’est utilisé que si la valeur de l’option Nouvelles tentatives de création est supérieure à 0. La valeur par défaut est 10 secondes.

  11. Sélectionnez la case Groupement pour activer le groupement des connexions pour le pool.

    Cette option est activée par défaut.

  12. Sélectionnez la case Association paresseuse pour que les connexions puissent être associées de façon paresseuse lorsque vous effectuez une opération sur celles-ci.

    Les connexions sont dissociées lorsque la transaction est terminée et qu’une méthode de composant se termine, ce qui permet la réutilisation des connexions physiques. Cette option est désactivée par défaut.

  13. Sélectionnez la case Inscription des connexions paresseuses pour inscrire une ressource dans la transaction uniquement lorsqu’elle est réellement utilisée dans une méthode.

    Cette option est désactivée par défaut.

  14. Sélectionnez la case Associer à un thread pour associer une connexion avec un thread.

    Lorsque le même thread a besoin d’une connexion, il peut réutiliser la connexion qui est déjà associée, par conséquent en évitant la surcharge d’obtention d’une connexion depuis le pool. Cette option est désactivée par défaut.

  15. Sélectionnez la case Faire correspondre les connexions pour activer l’association de connexions pour le pool.

    Désactivez cette option si vous savez que les connexions dans le pool seront toujours homogènes et, par conséquent, si une connexion prise du pool n’a pas besoin d’être associée par l’adaptateur de ressources. Cette option est activée par défaut.

  16. Dans le champ Utilisation maximale des connexions, saisissez le nombre de fois qu’une connexion doit être réutilisée par le pool.

    Une fois qu’une connexion a été réutilisée le nombre de fois spécifié, elle est fermée. Une valeur de 0 indique que cet attribut n’est pas activé. La valeur par défaut est 0.

  17. Cliquez sur Enregistrer.

    Astuce - Pour revenir aux paramètres par défaut, cliquez sur le bouton Charger valeurs par défaut, puis sur Enregistrer.
