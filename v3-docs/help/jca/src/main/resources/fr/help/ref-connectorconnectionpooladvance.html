Modifier les attributs avancés d'un pool de connexions de connecteur

La page Modifier les attributs avancés d'un pool de connexions de connecteur permet de modifier les paramètres avancés d'un pool de connexions de connecteur.

Elle contient les options suivantes.

Charger les valeurs par défaut

Bouton de restauration des valeurs par défaut des paramètres qui ont des valeurs par défaut. Les paramètres qui n'ont pas de valeurs par défaut ne sont pas modifiés.

Valider une fois au plus

Indique l'intervalle, exprimé en secondes, entre des demandes successives de validation d'une connexion, une fois au plus. La valeur par défaut est 0, ce qui signifie que l'attribut n'est pas activé.

Temporisation de perte

Durée, en secondes, nécessaire au suivi des pertes de connexion dans un pool de connexions. La valeur 0 indique que le suivi des pertes de connexion est désactivé. La valeur par défaut est 0.

Récupération de perte

Si cette option est sélectionnée, les connexions présentant des pertes seront restaurées dans le pool une fois le suivi des pertes de connexion terminé. Cette option est désactivée par défaut.

Tentatives de création

Nombre de tentatives effectuées en cas d'échec de création d'une connexion. La valeur 0 indique qu'aucune tentative ne sera effectuée pour recréer une connexion. La valeur par défaut est 0.

Intervalle entre les tentatives

Intervalle, en secondes, entre deux tentatives de création d'une connexion. Cet attribut n'est utilisé que si la valeur de l'option Tentatives de création est supérieure à 0. La valeur par défaut est de 10 secondes.

Groupement

Active le groupement des connexions pour le pool. Cette option est activée par défaut.

Association lazy

Si cette option est sélectionnée, les connexions sont associées en différé lorsque vous effectuez une opération sur celles-ci. Les connexions sont dissociées lorsque la transaction est terminée et qu'une méthode de composant prend fin, ce qui facilite la réutilisation des connexions physiques. Cette option est désactivée par défaut.

Participation des connexions lazy à la transaction

Si cette option est sélectionnée, une ressource est engagée dans une transaction uniquement lorsqu'elle est utilisée dans une méthode. Cette option est désactivée par défaut.

Associer à un thread

Si cette option est sélectionnée, une connexion est associée à un thread. Lorsque ce dernier nécessite une connexion, il peut réutiliser celle qui lui est déjà associée, limitant ainsi le temps système lié à l'obtention d'une connexion à partir du pool. Cette option est désactivée par défaut.

Faire correspondre les connexions

Si cette option est sélectionnée, la correspondance de connexions est activée pour le pool. Cette option peut être désactivée si vous savez que les connexions du pool seront toujours homogènes et qu'il est donc inutile pour l'adaptateur de ressource d'associer une connexion empruntée au pool. Cette option est activée par défaut.

Utilisation des correspondances de connexions

Nombre de réutilisations possibles d'une connexion par le pool. Une fois qu'une connexion a été réutilisée le nombre de fois spécifié, elle est fermée. La valeur 0 indique que cet attribut n'est pas activé. La valeur par défaut est 0.
