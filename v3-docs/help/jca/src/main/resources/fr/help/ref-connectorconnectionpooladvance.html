Modifier les attributs avancés d’un pool de connexions de connecteur

Utilisez la page Modifier les attributs avancés d’un pool de connexions de connecteur pour modifier les paramètres d’un pool de connexions du connecteur.

La page Modifier les attributs avancés du pool de connexions de connecteur présente les options suivantes :

Restaurer les valeurs par défaut

Bouton de restauration des valeurs par défaut des paramètres qui ont des valeurs par défaut. Les paramètres qui n'ont pas de valeurs par défaut ne sont pas modifiés.

Valider au maximum une fois

Indique l'intervalle, exprimé en secondes, entre des demandes successives de validation d'une connexion, une fois au plus. La valeur par défaut est 0, ce qui signifie que l'attribut n'est pas activé.

Délai d’expiration de la fuite

Période (en secondes) nécessaire pour effectuer un suivi des fuites de connexion dans un pool de connexions. Une valeur de 0 indique que le suivi des fuites de connexion est désactivé. La valeur par défaut est 0.

Récupération de la fuite

Si cette option est sélectionnée, les connexions qui présentent des fuites seront restaurées dans le pool une fois le suivi des fuites de connexion terminé. Cette option est désactivée par défaut.

Nouvelles tentatives de création

Nombre de tentatives effectuées en cas d’échec de création d’une connexion. Une valeur de 0 indique qu’aucune tentative ne sera effectuée pour créer une nouvelle connexion. La valeur par défaut est 0.

Fréquence des tentatives

L’intervalle, en secondes, entre les deux tentatives pour créer une connexion. Cet attribut n’est utilisé que si la valeur de l’option Nouvelles tentatives de création est supérieure à 0. La valeur par défaut est 10 secondes.

Groupement

Permet le groupement des connexions pour le pool. Cette option est activée par défaut.

Association différée

Si cette option est sélectionnée, les connexions sont associées normalement lorsque vous effectuerez une opération sur celles-ci. Les connexions sont dissociées lorsque la transaction est terminée et qu’une méthode de composant se termine, ce qui permet la réutilisation des connexions physiques. Cette option est désactivée par défaut.

Enrôlement différé de la connexion

Si cette option est sélectionnée, une ressource est répertoriée dans la transaction uniquement lorsqu’elle est effectivement utilisée dans une méthode. Cette option est désactivée par défaut.

Associer au thread

Si cette option est sélectionnée, une connexion est associée à un thread. Lorsque le même thread a besoin d’une connexion, il peut réutiliser la connexion qui est déjà associée, par conséquent en évitant la surcharge d’obtention d’une connexion depuis le pool. Cette option est désactivée par défaut.

Associer des connexions

Si cette option est sélectionnée, la correspondance de connexions est activée pour le pool. Cette option peut être désactivée si vous savez que les connexions dans le pool seront toujours homogènes et par conséquent, si une connexion prise dans le pool n’a pas besoin que l’adaptateur de ressource lui trouve une correspondance. Cette option est activée par défaut.

Utilisation des correspondances de connexions

Le nombre de réutilisation possible d’une connexion par le pool. Une fois qu’une connexion a été réutilisée le nombre de fois spécifié, elle est fermée. Une valeur de 0 indique que cet attribut n’est pas activé. La valeur par défaut est 0.
