作業セキュリティーマップ

リソースアダプタの作業セキュリティーマップを設定するには、「作業セキュリティーマップ」ページを使用します。

コネクタの作業セキュリティーマップは、リソースアダプタの EIS 主体または EIS グループが提出した作業の呼び出し側 ID を、GlassFish サーバー セキュリティードメインの適切な主体またはユーザーグループにマッピングします。1 つまたは複数の作業セキュリティーマップを、1 つのリソースアダプタに関連付けることができます。コネクタの作業セキュリティーマップの設定では、ワイルドカード文字のアスタリスク (*) を使用して、すべてのユーザーまたはすべてのユーザーグループを示すことができます。

作業セキュリティーマップは、コネクタ接続プールのセキュリティーマップとは異なります。コネクタ接続プールのセキュリティーマップは、サーブレットまたは EJB の認証中に受け取った主体を、EIS で承認された証明書にマッピングします。コネクタ接続プールのセキュリティーマップについては、「コネクタ接続プールのセキュリティーマップ」を参照してください。

EIS (Enterprise Information System) は、組織のデータを保持するシステムです。メインフレーム、メッセージングシステム、データベースシステム、またはアプリケーションがこれに使用できます。

作業セキュリティーマップごとに、次の情報が表示されます。

名前

作業セキュリティーマップの名前。

「作業セキュリティーマップ」ページには、次のオプションも表示されます。

新規

新しい作業セキュリティーマップを作成するボタン。

削除

選択した 1 つまたは複数の作業セキュリティーマップを削除するボタン。

  • 「作業セキュリティーマップを削除する」
