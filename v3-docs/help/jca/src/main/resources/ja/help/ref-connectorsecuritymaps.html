コネクタ接続プールのセキュリティーマップ

コネクタ接続プールのセキュリティーマップを管理するには、「コネクタ接続プールのセキュリティーマップ」ページを使用します。

コネクタ接続プールのセキュリティーマップを使用すると、Java EE ユーザーおよびグループと EIS ユーザーおよびグループ間のマッピングを作成できます。セキュリティーマップは、サーブレットまたは EJB の認証中に受け取った主体を、EIS で承認された証明書にマッピングします。複数の GlassFish サーバー 主体を、同じバックエンド主体にマッピングできます。

接続プールのセキュリティーマップは、コネクタの作業セキュリティーマップとは異なります。コネクタの作業セキュリティーマップは、受け取った作業インスタンスに関連付けられた主体を、GlassFish サーバー ドメインのセキュリティーレルムの主体にマッピングします。コネクタの作業セキュリティーマップについては、「作業セキュリティーマップ」を参照してください。

セキュリティーマップごとに、次の情報が表示されます。

名前

セキュリティーマップの名前。

「セキュリティーマップ」の表には、次のオプションも表示されます。

新規

新しいセキュリティーマップを作成するボタン。

削除

選択した 1 つまたは複数のセキュリティーマップを削除するボタン。
