コネクタ接続プールのセキュリティ・マップ

コネクタ接続プールのセキュリティ・マップを管理するには、「コネクタ接続プールのセキュリティ・マップ」ページを使用します。

コネクタ接続プールのセキュリティ・マップを使用すると、Java EEユーザーおよびグループとEISユーザーおよびグループ間のマッピングを作成できます。セキュリティ・マップは、サーブレットまたはEJBの認証中に受け取ったプリンシパルを、EISで受諾された資格証明にマッピングします。複数のGlassFish Serverプリンシパルを、同じバックエンド・プリンシパルにマッピングできます。

接続プールのセキュリティ・マップは、コネクタ作業セキュリティ・マップとは異なります。コネクタ作業セキュリティ・マップは、受け取った作業インスタンスに関連付けられたプリンシパルを、GlassFish Serverドメインのセキュリティ・レルムのプリンシパルにマッピングします。コネクタ作業セキュリティ・マップについては、作業セキュリティ・マップを参照してください。

セキュリティ・マップごとに、次の情報が表示されます。

名前

セキュリティ・マップの名前

「セキュリティ・マップ」の表には、次のオプションも表示されます。

新規

新しいセキュリティ・マップを作成するボタン。

削除

選択した1つまたは複数のセキュリティ・マップを削除するボタン。
