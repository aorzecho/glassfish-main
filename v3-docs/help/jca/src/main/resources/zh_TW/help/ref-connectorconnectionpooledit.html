編輯連接器連線池

使用 [編輯連接器連線池] 頁面可修改連接器連線池的設定。

[編輯連接器連線池] 頁面包含下列選項。

載入預設值

用於將具有預設值的設定復原成其預設值的按鈕。沒有預設值的設定保持不變。

清除

在連線池中重新初始化連線的按鈕。按下此按鈕可銷毀任何現有的連線、重新建立已針對池建立的連線，並將池復原為其初始與最小的池大小。

Ping

驗證連線池是否可用的按鈕。

池名稱

可識別連接器連線池的唯一名稱。此為唯讀欄位。您僅能在建立新的連接器連線池時指定池名稱。

資源配接卡

與連接器連線池關聯的資源配接卡。

連線定義

連接器連線池的連線定義。清單中提供的選項會視您所選取的資源配接卡而定。連線定義會識別資源配接卡在 ra.xml 檔案中的 connection-definition 元素。

Ping

若啟用了此選項，將會在建立或重新配置期間 Ping 連線池，以識別並警告任何錯誤的屬性值。此選項預設為停用。

說明

連接器連線池的說明。

池的初始大小和最小大小

連接器連線池中的最小連線數目。預設值為 8。

池的最大大小

連接器連線池中的最大連線數目。預設值為 32。

池設定大小數量

池的閒置逾時過期時要移除的連線數目。預設值為 2。

閒置逾時

連線可以在儲存區中閒置的最大時間 (以秒表示)。預設值為 300。

最大等待時間

請求連線的應用程式在達到連線逾時之前等待的時間。預設值為 60000。

連線驗證

若選取了 [需要連線驗證] 核取方塊，指定連線給應用程式之前會先驗證。若資源的驗證失敗，則會銷毀該資源，然後建立並傳回新的資源。此選項預設為停用。

一旦失敗

若選取了 [關閉所有連線] 核取方塊，GlassFish Server 將會在任何一個連線失敗時，關閉池中的所有連線並重新建立連線。此選項預設為停用。

作業事件支援

選擇的作業事件支援以向下相容方式置換與此連線池關聯的資源介面中的作業事件支援屬性。也就是說，它可以支援比資源配接卡中指定的作業事件層級低或與其相同的作業事件層級，但它不能指定更高的層級。

值可能為下列其中之一：

XATransaction

資源配接卡會透過實作 LocalTransaction 和 XAResource 介面來支援資源管理員的本機和 JTA 作業事件。XA 作業事件由作業事件管理員在資源管理員外部進行控制和協調。

LocalTransaction

資源配接卡會透過實作 LocalTransaction 介面來支援本機作業事件。本機作業事件的管理在資源管理員內部進行，不涉及任何外部作業事件管理員。

NoTransaction

資源配接卡不支援資源管理員的本機或 JTA 作業事件，且不會實作 XAResource 或 LocalTransaction 介面。
