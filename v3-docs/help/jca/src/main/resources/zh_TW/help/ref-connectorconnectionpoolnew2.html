新建連接器連線集區 (步驟 2，共 2 步)

使用 「新建連接器連線集區 (步驟 2，共 2 步)」 頁面可建立新的連接器連線集區。

「新建連接器連線集區 (步驟 2，共 2 步)」 頁面包含下列選項。

Ping

若啟用了此選項，將會在建立或重新組態期間 Ping 連線集區，以識別並警告任何錯誤的屬性值。此選項預設為停用。

說明

連接器連線集區的說明。

集區的初始大小和最小大小

連接器連線集區中的最小連線數目。預設值為 8。

集區的最大大小

連接器連線集區中的最大連線數目。預設值為 32。

集區設定大小數量

集區的閒置逾時過期時要移除的連線數目。預設值為 2。

閒置逾時

連線可以在集區中閒置的最大時間 (以秒表示)。預設值為 300。

最大等待時間

請求連線的應用程式在達到連線逾時之前等待的時間。預設值為 60000。

連線驗證

若選取了 「需要連線驗證」 核取方塊，指定連線給應用程式之前會先驗證。若資源的驗證失敗，則會銷毀該資源，然後建立並傳回新的資源。此選項預設為停用。

一旦失敗

若選取了 「關閉所有連線」 核取方塊，GlassFish Server 將會在任何一個連線失敗時，關閉集區中的所有連線並重新建立連線。此選項預設為停用。

作業事件支援

選擇的作業事件支援以向下相容方式置換與此連線集區關聯的資源介面中的作業事件支援屬性。也就是說，它可以支援比資源配接卡中指定的作業事件層級低或與其相同的作業事件層級，但它不能指定更高的層級。

值可能為下列其中之一：

XATransaction

資源配接卡會透過實作 LocalTransaction 和 XAResource 介面來支援資源管理員的本機和 JTA 作業事件。XA 作業事件由作業事件管理員在資源管理員外部進行控制和協調。

LocalTransaction

資源配接卡會透過實作 LocalTransaction 介面來支援本機作業事件。本機作業事件的管理在資源管理員內部進行，不涉及任何外部作業事件管理員。

NoTransaction

資源配接卡不支援資源管理員的本機或 JTA 作業事件，且不會實作 XAResource 或 LocalTransaction 介面。

其他特性

連接器連線集區的其他特性。可用的特性會視連接器連線集區的資源配接卡而定。此表中指定的「名稱/值」組，可用於置換資源配接卡供應商定義的特性預設值。

若您為特性指定了名稱但沒有指定值，連線集區建立完成時，會從表中移除該特性。若您不想立即為特性設定值，但希望將該特性保留在表中，待稍後為其設定值，請在 「值」 欄位中鍵入一組空的圓括號：

()
