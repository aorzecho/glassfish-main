Konnektor-Verbindungspool bearbeiten

Auf der Seite "Konnektor-Verbindungspool bearbeiten" können Sie die Einstellungen für einen Konnektor-Verbindungspool ändern.

Die Seite "Konnektor-Verbindungspool bearbeiten" enthält die folgenden Optionen.

Standardeinstellungen laden

Schaltfläche zum Wiederherstellen der Standardwerte für alle Einstellungen.

Leeren

Schaltfläche zum erneuten Initialisieren der Verbindungen im Verbindungspool. Wenn Sie auf diese Schaltfläche klicken, werden alle bestehenden Verbindungen gelöscht, für den Pool eingerichtete Verbindungen werden neu erstellt, und die anfängliche, minimale Poolgröße wird wiederhergestellt.

Ping

Schaltfläche, mit der die Verwendbarkeit des Pools überprüft werden kann.

Pool Name (Poolname)

Ein eindeutiger Name, der den Konnektor-Verbindungspool identifiziert. Dies ist ein schreibgeschütztes Feld. Sie können den Poolnamen nur festlegen, wenn Sie einen neuen Konnektor-Verbindungspool erstellen.

Ressourcenadapter

Der mit dem Konnektor-Verbindungspool verknüpfte Ressourcenadapter.

Verbindungsdefinition

Die Verbindungsdefinition für den Konnektor-Verbindungspool. Die Auswahloptionen in der Liste richten sich nach dem gewählten Ressourcenadapter. Die Verbindungsdefinition gibt das Element connection-definition eines Ressourcenadapters in der Datei ra.xml an.

Ping

Wenn diese Option aktiviert ist, wird bei der Erstellung oder Neukonfiguration eines Verbindungspools automatisch ein Ping-Signal an den Pool gesendet, um falsche Attributwerte zu ermitteln und eine diesbezügliche Warnung auszugeben. Diese Option ist standardmäßig deaktiviert.

Description (Beschreibung)

Eine Beschreibung des Konnektor-Verbindungspools.

Initial and Minimum Pool Size (Ursprüngliche und minimale Poolgröße)

Die minimale Anzahl an Verbindungen im Konnektor-Verbindungspool. Der Standardwert ist 8.

Maximum Pool Size (Maximale Poolgröße)

Die maximale Anzahl an Verbindungen im Konnektor-Verbindungspool. Der Standardwert lautet 32.

Pool Resize Quantity (Anzahl der Größenänderungen des Pools)

Die Anzahl an Verbindungen, die nach Ablauf der Pool-Leerlaufzeit entfernt werden sollen. Der Standardwert ist 2.

Idle Timeout (Leerlaufzeitlimit)

Die maximale Zeit in Sekunden, die sich eine Verbindung im Pool im Leerlauf befinden kann. Der Standardwert beträgt 300.

Max Wait Time (Maximale Wartezeit)

Eine Anwendung, die eine Verbindung angefordert hat, wartet die hier angegbene Zeitspanne ab, bevor eine Verbindungszeitüberschreitung ausgegeben wird. Der Standardwert lautet 60000.

Connection Validation (Verbindungsüberprüfung)

Wenn das Kontrollkästchen "Erforderlich" aktiviert ist, werden Verbindungen vor der Übergabe an die Anwendung überprüft. Schlägt die Überprüfung einer Ressource fehl, so wird die Ressource zerstört, und eine neue Ressource wird erstellt und zurückgegeben. Diese Option ist standardmäßig deaktiviert.

On Any Failure (Bei sämtlichen Fehlern)

Wenn das Kontrollkästchen "Sämtliche Verbindungen trennen" aktiviert ist, schließt GlassFish-Server bei einem einzigen Verbindungsabbruch alle Verbindungen im Pool und stellt sie wieder her. Diese Option ist standardmäßig deaktiviert.

Transaktionsunterstützung

Die gewählte Transaktionsunterstützung überschreibt das Transaktionsunterstützungsattribut des mit diesem Verbindungspool verknüpften Ressourcenadapters in einer Weise, mit der Abwärtskompatibilität gewährleistet wird. Anders ausgedrückt: Es kann dieselbe oder eine niedrigere Transaktionsebene unterstützt werden als die im Ressourcenadapter angegebene, jedoch keine höhere Transaktionsebene.

Folgende Werte sind möglich:

XATransaction

Der Ressourcenadapter unterstützt lokale und JTA-Transaktionen des Ressourcenmanangers durch Implementierung der Schnittstellen LocalTransaction und XAResource. XA-Transaktionen werden durch einen externen Transaktions-Manager gesteuert und koordiniert.

LocalTransaction

Der Ressourcenadapter unterstützt lokale Transaktionen durch Implementierung der Schnittstelle LocalTransaction. Lokale Transaktionen werden intern durch einen Ressourcen-Manager verwaltet und beziehen keinen externen Transaktions-Manager ein.

NoTransaction

Der Ressourcenadapter unterstützt keine lokalen oder JTA-Transaktionen des Ressourcenmanagers und implementiert die Schnittstellen XAResource und LocalTransaction nicht.
