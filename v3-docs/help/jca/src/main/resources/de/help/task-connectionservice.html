So konfigurieren Sie den Verbindungsdienst

  1. Erweitern Sie in der Navigationsstruktur den Konfigurationsknoten.
  2. Wählen Sie unter dem Konfigurationsknoten den Knoten "Verbindungsdienst" aus.

    Die Seite "Verbindungsdienst" wird geöffnet.

  3. Geben Sie im Feld "Zeitüberschreitung beim Herunterfahren" die maximale Anzahl Sekunden ein, die während des Herunterfahrens von GlassFish-Server, auf die Ausführung der ResourceAdapter.stop-Methode einer Verbindungsmodulinstanz gewartet wird.

    Ressourcenadapter, deren Herunterfahren länger dauert, werden ignoriert, und das Herunterfahren von GlassFish-Server wird fortgesetzt. Der Standardwert beträgt 30 Sekunden.

  4. Wählen Sie in der Dropdown-Liste "Richtlinie zum Laden von Konnektorklassen" die zu verwendende Richtlinie aus.
    • Wählen Sie derived, um anzugeben, dass die Ressourcenadapter entsprechend den Verweisen einer Anwendung auf beliebige Ressourcen des Ressourcenadapters bereitgestellt werden.

      Diese Richtlinie ist die Standardeinstellung.

    • Wählen Sie global, um anzugeben, dass alle Ressourcenadapter für alle Anwendungen sichtbar sind.

    Diese Richtlinie gilt nicht für den vorinstallierten JMS-Systemressourcenadapter, jmsra, der immer für alle Anwendungen sichtbar ist.

  5. Klicken Sie auf "Speichern".

    Tipp - Um zu den Standardeinstellungen zurückzukehren, klicken Sie auf die Schaltfläche "Standardwerte laden" und anschließend auf "Speichern".
