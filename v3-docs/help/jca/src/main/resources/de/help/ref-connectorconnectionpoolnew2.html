Neuer Connector Connection Pool (Schritt 2 von 2)

Sie verwenden die Seite "Neuer Connector Connection Pool (Schritt 2 von 2)" zum Erstellen eines neuen Connector Connection Pools.

Die Seite "Neuer Connector Connection Pool (Schritt 2 von 2)" enthält die folgenden Optionen.

Ping

Wenn diese Option aktiviert ist, wird bei der Erstellung oder Neukonfiguration eines Connection Pools automatisch ein Ping-Signal an den Pool gesendet, um falsche Attributwerte zu ermitteln und eine diesbezügliche Warnung auszugeben. Diese Option ist standardmäßig deaktiviert.

Description (Beschreibung)

Eine Beschreibung des Connector Connection Pools.

Initial and Minimum Pool Size (Ursprüngliche und minimale Poolgröße)

Die Mindestanzahl Verbindungen im Connector Connection Pool. Der Standardwert ist 8.

Maximum Pool Size (Maximale Poolgröße)

Die maximale Anzahl an Verbindungen im Connector Connection Pool. Der Standardwert lautet 32.

Pool Resize Quantity (Anzahl der Größenänderungen des Pools)

Die Anzahl an Verbindungen, die nach Ablauf der Pool-Leerlaufzeit entfernt werden sollen. Der Standardwert ist 2.

Leerlauftimeout

Die maximale Zeit in Sekunden, die sich eine Verbindung im Pool im Leerlauf befinden kann. Der Standardwert beträgt 300.

Max. Wartezeit

Die Zeitspanne, die eine Anwendung nach dem Anfordern einer Verbindung abwartet, bevor ein Verbindungstimeout ausgegeben wird. Der Standardwert lautet 60000.

Connection Validation (Verbindungsüberprüfung)

Wenn das Kontrollkästchen "Erforderlich" aktiviert ist, werden Verbindungen vor der Übergabe an die Anwendung validiert. Schlägt die Überprüfung einer Ressource fehl, so wird die Ressource zerstört, und eine neue Ressource wird erstellt und zurückgegeben. Diese Option ist standardmäßig deaktiviert.

On Any Failure (Bei sämtlichen Fehlern)

Wenn das Kontrollkästchen "Sämtliche Verbindungen trennen" aktiviert ist, schließt GlassFish Server bei einem einzigen Verbindungsabbruch alle Verbindungen im Pool und stellt sie wieder her. Diese Option ist standardmäßig deaktiviert.

Transaktionsunterstützung

Die gewählte Transaktionsunterstützung überschreibt das Transaktionsunterstützungsattribut des mit diesem Connection Pool verknüpften Ressourcenadapters in einer Weise, mit der Abwärtskompatibilität gewährleistet wird. Anders ausgedrückt: Es kann dieselbe oder eine niedrigere Transaktionsebene unterstützt werden als die im Ressourcenadapter angegebene, jedoch keine höhere Transaktionsebene.

Folgende Werte sind möglich:

XATransaction

Der Ressourcenadapter unterstützt lokale und JTA-Transaktionen des Ressourcenmanangers durch Implementierung der LocalTransaction- und XAResource-Schnittstellen. XA-Transaktionen werden durch einen externen Transaktions-Manager gesteuert und koordiniert.

LocalTransaction

Der Ressourcenadapter unterstützt lokale Transaktionen durch Implementierung der LocalTransaction-Schnittstelle. Lokale Transaktionen werden intern durch einen Ressourcen-Manager verwaltet und beziehen keinen externen Transaktions-Manager ein.

NoTransaction

Der Ressourcenadapter unterstützt lokale oder JTA-Transaktionen des Ressourcenmanagers nicht und implementiert die XAResource- oder LocalTransaction-Schnittstellen nicht.

Weitere Eigenschaften

Weitere Eigenschaften für den Connector Connection Pool. Die verfügbaren Eigenschaften richten sich nach dem Ressourcenadapter, der von dem Connector Connection Pool verwendet wird. Mit den in dieser Tabelle angegebenen Name/Werte-Paaren können die vom Ressourcenadapterprovider definierten Standardwerte für die Eigenschaften überschrieben werden.

Wenn Sie für eine Eigenschaft einen Namen, aber keinen Wert angeben, wird die Eigenschaft nach Abschluss der Erstellung des Connection Pools aus der Tabelle entfernt. Falls Sie zum jetzigen Zeitpunkt keinen Wert für eine Eigenschaft festlegen möchten, die Eigenschaft jedoch in der Tabelle verbleiben soll, damit Sie den Wert später angeben können, geben Sie ein leeres Klammernpaar in das Wertefeld ein:
