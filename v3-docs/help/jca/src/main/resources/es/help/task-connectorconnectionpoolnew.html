Para crear un conjunto de conexiones de conector

Antes de empezar

Antes de crear el conjunto, implemente el módulo del conector (adaptador de recursos) asociado al conjunto. Los valores que se especifican para el nuevo conjunto dependen del módulo de conector que se implemente.

  1. En el árbol de navegación, amplíe el nodo Recursos.
  2. Expanda el nodo Conectores.
  3. En el nodo Configuración, seleccione el nodo Conjuntos de conexiones de conectores.

    Se abrirá la página Conjuntos de conexiones de conectores.

  4. En la página Conjuntos de conexiones del conector, haga clic en Nuevo.

    Se abrirá la página Nuevo conjunto de conexiones de conectores (paso 1 de 2).

  5. En el campo Nombre de conjunto, asigne un nombre exclusivo al conjunto.

    Especifique este nombre cuando cree un recurso de conector.

  6. En la lista desplegable Adaptador de recursos, haga una selección en la lista de adaptadores de recursos implementados (módulos de conector).
  7. En la lista desplegable Definición de conexión, seleccione la definición de conexión para el conjunto de conexiones del conector.

    Las opciones de la lista dependen del adaptador de recursos seleccionado. La definición de conexión identifica un elemento connection-definition del adaptador de recursos en el archivo ra.xml.

  8. Haga clic en Siguiente.

    Se abrirá la página Nuevo conjunto de conexiones de conectores (paso 2 de 2).

  9. Seleccione la casilla de verificación Ping habilitado para realizar un ping en el conjunto de conexiones durante la creación o reconfiguración para identificar los valores erróneos de sus atributos y avisar de ellos.

    Esta opción se deshabilita por defecto.

  10. En el campo Descripción, introduzca una descripción del conjunto de conexiones.
  11. En el campo Tamaño de conjunto inicial y mínimo, introduzca el número mínimo de conexiones del conjunto.

    El valor predeterminado es 8.

  12. En el campo Tamaño de conjunto máximo, introduzca el número máximo de conexiones de este conjunto.

    El valor predeterminado es 32.

  13. En el campo Cantidad de redimensionamiento del conjunto, especifique el número de conexiones que se eliminarán cuando se upere el tiempo límite de espera de inactividad del conjunto.

    El valor predeterminado es 2.

  14. En el campo Tiempo de espera de inactividad, indique el tiempo máximo que una conexión puede permanecer inactiva en el conjunto, en segundos.

    El valor predeterminado es 300.

  15. En el campo Tiempo de espera máximo, indique la cantidad de tiempo que la aplicación que solicita una conexión puede esperar antes de superar el límite de espera para establecer la conexión.

    El valor predeterminado es 60000.

  16. Marque la casilla de verificación Validación de conexión necesaria para habilitar la validación de la conexión.

    Si la casilla de verificación Necesaria está seleccionada, las conexiones se validarán antes de proporcionárselas a la aplicación. Si falla la validación de un recurso, éste se destruye, para a continuación crear y devolver un recurso nuevo. Esta opción se deshabilita por defecto.

  17. Seleccione la casilla de verificación Cerrar todas las conexiones en caso de fallo, de manera que GlassFish Server cerrará todas las conexiones del conjunto y las restablecerá con que tan sólo se produzca el fallo de una conexión.

    Si no selecciona esta casilla de verificación, las conexiones individuales se restablecerán sólo cuando vaya a utilizarlas. Esta opción se deshabilita por defecto.

  18. Utilice la lista Compatibilidad de transacción para seleccionar el tipo de compatibilidad de transacción para el conjunto de conexiones.

    La compatibilidad de transacción seleccionada sobrescribe el atributo de compatibilidad de transacción en el adaptador de recursos asociado con el conjunto de conexiones de una forma descendente. Dicho de otro modo, se puede admitir un nivel de transacción inferior o idéntico al que se especifica en el adaptador de recursos, pero no se puede indicar un nivel superior.

    Las opciones siguientes están disponibles en la lista desplegable:

    • NoTransaction: indica que el adaptador de recursos no admite transacciones JTA o transacciones locales del administrador de recursos y no implementa las interfaces XAResource o LocalTransaction.
    • LocalTransaction: indica que el adaptador de recursos admite transacciones locales mediante la implementación de la interfaz LocalTransaction.

      Las transacciones locales se administran de manera interna en un administrador de recursos y no involucran a administradores de transacciones externos.

    • XATransaction: indica que el adaptador de recursos admite transacciones JTA y transacciones locales del administrador de recursos mediante la implementación de las interfaces LocalTransaction y XAResource.

      Las transacciones XA se controlan y coordinan a través de un administrador de transacciones ajeno a un administrador de recursos.

  19. (Opcional) En la sección Propiedades adicionales, especifique propiedades adicionales del conjunto de conexiones de conectores.

    Las propiedades disponibles dependen del adaptador de recursos para el conjunto de conexiones de conectores.

    Para añadir una propiedad, haga clic en el botón Añadir propiedad. En la fila en blanco que aparece, escriba el nombre de propiedad en el campo Nombre y escriba el valor de propiedad en el campo Valor.

    Si especifica un nombre, pero no un valor para una propiedad, éste se elimina de la tabla al acabar de crear el conjunto de conexiones. Si no desea definir un valor para una propiedad ahora, pero desea que la propiedad permanezca en la tabla para que el valor se pueda definir más adelante, escriba un conjunto de paréntesis vacío en el campo Valor: 

    ()
  20. Haga clic en Aceptar.
