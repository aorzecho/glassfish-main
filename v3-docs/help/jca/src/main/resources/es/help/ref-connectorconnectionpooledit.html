Editar conjunto de conexiones del conector

Utilice la página Editar conjunto de conexiones de conectores para modificar los parámetros de configuración de un conjunto de conexiones de conectores.

La página Editar conjunto de conexiones de conectores contiene las siguientes opciones.

Cargar valores predeterminados

Botón para restablecer los valores predeterminados de todos los parámetros de configuración.

Vaciar

Botón para reinicializar las conexiones del conjunto de conexiones. Al hacer clic en este botón se destruyen todas las conexiones existentes, se recrean las conexiones establecidas para el conjunto y se restablece el tamaño de conjunto inicial y mínimo del conjunto.

Ping

Botón para verificar que el conjunto de conexiones se puede utilizar.

Nombre de conjunto

Un nombre exclusivo que identifica el conjunto de conexiones de conectores. Éste es un campo de solo lectura. El nombre del conjunto se puede especificar únicamente al crear un nuevo conjunto de conexiones del conector.

Adaptador de recursos

El adaptador de recursos asociado con el conjunto de conexiones de conectores.

Definición de conexión

La definición de conexión para el conjunto de conexiones de conectores. Las opciones de la lista dependen del adaptador de recursos seleccionado. La definición de conexión identifica un elemento connection-definition del adaptador de recursos en el archivo ra.xml.

Ping

Si esta opción está habilitada, se realiza un ping en el conjunto de conexiones durante la creación o la reconfiguración para identificar cualquier valor incorrecto para sus atributos y avisar de él. Esta opción se deshabilita por defecto.

Descripción

Una descripción del conjunto de conexiones de conectores.

Tamaño inicial y mínimo del conjunto

El número mínimo de conexiones del conjunto de conexiones de conectores. El valor predeterminado es 8.

Tamaño máximo del conjunto

El número máximo de conexiones del conjunto de conexiones de conectores. El valor predeterminado es 32.

Cantidad de cambio de tamaño del conjunto

El número de conexiones que se eliminarán cuando se agote el tiempo de espera de inactividad del conjunto. El valor predeterminado es 2.

Tiempo de espera inactivo

El tiempo máximo en segundos que una conexión puede permanecer inactiva en el conjunto. El valor predeterminado es 300.

Tiempo de espera máximo

La cantidad de tiempo que la aplicación que solicita una conexión puede esperar para obtener la conexión. El valor predeterminado es 60000.

Validación de la conexión

Si la casilla de verificación Necesaria está seleccionada, las conexiones se validarán antes de proporcionárselas a la aplicación. Si falla la validación de un recurso, éste se destruye, para a continuación crear y devolver un recurso nuevo. Esta opción se deshabilita por defecto.

Con cualquier fallo

Si la casilla de verificación Cerrar todas las conexiones está seleccionada, GlassFish Server cerrará todas las conexiones del conjunto y las restablecerá si falla una conexión individual. Esta opción se deshabilita por defecto.

Compatibilidad de transacción

La compatibilidad de transacción seleccionada sobrescribe el atributo de compatibilidad de transacción en el adaptador de recursos asociado con el conjunto de conexiones de una forma descendente. Dicho de otro modo, se puede admitir un nivel de transacción inferior o idéntico al que se especifica en el adaptador de recursos, pero no se puede indicar un nivel superior.

El valor puede ser cualquiera de los siguientes:

XATransaction

El adaptador de recursos admite transacciones JTA y transacciones locales del administrador de recursos mediante la implementación de las interfaces LocalTransaction y XAResource. Las transacciones XA se controlan y coordinan a través de un administrador de transacciones ajeno a un administrador de recursos.

LocalTransaction

El adaptador de recursos admite transacciones locales mediante la implementación de la interfaz LocalTransaction. Las transacciones locales se administran de manera interna en un administrador de recursos y no involucran a administradores de transacciones externos.

NoTransaction

El adaptador de recursos no admite transacciones JTA o transacciones locales del administrador de recursos y no implementa las interfaces XAResource o LocalTransaction.
