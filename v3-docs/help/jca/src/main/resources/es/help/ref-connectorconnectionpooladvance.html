Editar atributos avanzados del conjunto de conexiones de conector

Utilice la página Editar atributos avanzados del conjunto de conexiones de conector para modificar las opciones avanzadas de un conjunto de conexiones del conector.

La página Editar atributos avanzados del conjunto de conexiones de conector presenta las opciones siguientes:

Cargar valores predeterminados

Botón para restablecer las opciones que tienen valores por defecto a estos. Las opciones que no tienen valores por defecto no se modifican.

Validar como máximo una vez

Especifica el intervalo de tiempo, en segundos, entre solicitudes consecutivas para validar una conexión como máximo una vez. El valor predeterminado es 0, lo que significa que el atributo no está habilitado.

Tiempo de espera de pérdida

La duración de tiempo, en segundos, para realizar un seguimiento de las pérdidas de conexión en un conjunto de conexiones. El valor 0 indica que el seguimiento de pérdidas de conexión está deshabilitado. El valor predeterminado es 0.

Reclamación de pérdidas

Si esta opción está seleccionada, las conexiones perdidas se restablecerán en el conjunto tras completar un seguimiento de las pérdidas de conexión. Esta opción se deshabilita por defecto.

Intentos de reintento de creación

El número de intentos que se harán si se produce un fallo al crear una nueva conexión. El valor de 0 significa que no se hará ningún intento para crear la conexión de nuevo. El valor predeterminado es 0.

Intervalo de reintento

El intervalo, en segundos, entre dos intentos para crear una conexión. Este atributo se utiliza sólo si el valor de intentos de reintento de creación es superior a 0. El valor predeterminado es 10 segundos.

Crear conjuntos

Permite crear conjuntos de conexiones en el conjunto. Esta opción está habilitada por defecto.

Asociación no activa

Si se selecciona esta opción, las conexiones se asociarán levemente cuando se realice una operación en ellas. Las conexiones se disocian cuando la transacción se completa y un método de componentes finaliza, lo que ayuda a reutilizar las conexiones físicas. Esta opción se deshabilita por defecto.

Reclutamiento de conexiones no activas

Si se selecciona esta opción, sólo se incluye un recurso en la transacción cuando realmente se utiliza en un método. Esta opción se deshabilita por defecto.

Asociar con subproceso

Si esta opción está seleccionada, la conexión está asociada a un subproceso. Cuando el mismo subproceso necesita otra conexión, puede volver a utilizar la conexión que ya está asociada con ese subproceso, sin necesidad de incurrir en la sobrecarga de coger otra conexión del conjunto. Esta opción se deshabilita por defecto.

Conexiones coincidentes

Si selecciona esta opción, la correspondencia de conexiones está habilitada en el conjunto. Esta opción se puede deshabilitar si sabe que las conexiones del conjunto siempre serán homogéneas y, por tanto, para una conexión recogida del conjunto no se necesita establecer correspondencia mediante el adaptador de recursos. Esta opción se habilita por defecto.

Uso de la correspondencia de conexiones

El número de veces que el conjunto debe reutilizar una conexión. La conexión se cerrará una vez reutilizada durante un número específico de veces. El valor 0 indica que el atributo no está habilitado. El valor predeterminado es 0.
