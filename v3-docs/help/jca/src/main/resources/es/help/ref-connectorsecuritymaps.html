Asignaciones de seguridad del conjunto de conexiones del conector

Utilice la página Asignaciones de seguridad de conjuntos de conexiones de conector para administrar las asignaciones de seguridad relativas a conjuntos de conexiones del conector.

Las asignaciones de seguridad para conjuntos de conexiones del conector permiten crear una asignación entre los usuarios y grupos de usuarios de Java EE y los de EIS. Un mapa de seguridad asigna un elemento principal recibido durante el servlet o la autenticación EJB a las credenciales aceptadas por el EIS. Es posible asignar varios elementos principales de Servidor GlassFish al mismo elemento principal backend.

Un mapa de seguridad de un conjunto de conexiones es diferente a un mapa de seguridad de tareas de conectores, ya que éste asigna un elemento principal asociado con una instancia de tareas entrante a un elemento principal de un dominio de seguridad del dominio de Servidor GlassFish. Consulte Mapas de seguridad de tareas para obtener información acerca de los mapas de seguridad de tareas de conectores.

Para cada mapa de seguridad, se proporciona la siguiente información.

Nombre

El nombre del mapa de seguridad.

La tabla Mapas de seguridad también contiene las siguientes opciones.

Nuevo

Botón para crear un mapa de seguridad nuevo.

Borrar

Botón para eliminar uno o más mapas de seguridad seleccionados.
