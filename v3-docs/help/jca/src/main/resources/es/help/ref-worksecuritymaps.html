Mapas de seguridad de tareas

Utilice la página Mapas de seguridad de tareas para configurar los mapas de seguridad de tareas para los adaptadores de recursos.

Un mapa de seguridad de tareas de conectores asigna la identidad de origen de la llamada de las tareas presentadas por un adaptador de recursos a un elemento principal o a un grupo de usuarios adecuado en el dominio de seguridad de Servidor GlassFish. Uno o más mapas de seguridad de tareas pueden asociarse con un adaptador de recursos. La configuración de los mapas de seguridad de tareas de conectores admite el uso del asterisco comodín (*) para referirse a todos los usuarios o grupos de usuarios.

Un mapa de seguridad de tareas es diferente de un mapa de seguridad para un conjunto de conexiones de conectores, el cual asigna el elemento principal recibido durante el servlet o la autenticación EJB a las credenciales aceptadas por el EIS. Consulte Asignaciones de seguridad del conjunto de conexiones del conector para obtener más información sobre las asignaciones de seguridad relativas a grupos de conexiones del conector.

El sistema de información empresarial (EIS) es cualquier sistema que guarde los datos de una organización. Puede ser un mainframe, un sistema de mensajería, un sistema de base de datos o una aplicación.

Para cada mapa de seguridad de tareas, se proporciona la siguiente información.

Nombre

El nombre del mapa de seguridad de tareas.

La tabla Mapas de seguridad de tareas también contiene las siguientes opciones.

Nuevo

Botón para crear un mapa de seguridad de tareas nuevo.

Borrar

Botón para eliminar uno o más mapas de seguridad de tareas seleccionados.
