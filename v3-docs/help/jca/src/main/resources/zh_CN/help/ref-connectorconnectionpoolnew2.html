新建连接器连接池 (第 2 步，共 2 步)

使用“新建连接器连接池 (第 2 步，共 2 步)”页可以创建新的连接器连接池。

“新建连接器连接池 (第 2 步，共 2 步)”页包含以下选项：

试通

如果启用此选项，则在创建或重新配置过程中，将对连接池进行试通操作，以识别其属性的任何错误值并发出警告。默认情况下禁用此选项。

说明

连接器连接池的说明。

初始和最小池大小

连接器连接池中连接的最小数目。默认值为 8。

最大池大小

连接器连接池中连接的最大数目。默认值为 32。

池调整大小数量

池空闲超时到期时要删除的连接的数目。默认值为 2。

空闲超时

连接在池中保持空闲的最长时间 (秒)。默认值为 300。

最长等待时间

在达到连接超时之前，请求连接的应用程序所等待的时间。默认值为 60000。

连接验证

如果选中“必需的”复选框，则先验证连接，然后才将其提供给应用程序。如果资源验证失败，则销毁此资源，然后创建并返回一个新的资源。默认情况下禁用此选项。

一旦失败

如果选中“关闭所有连接”复选框，则 GlassFish Server 将在某个连接失败时关闭池中的所有连接并重新建立这些连接。默认情况下禁用此选项。

事务处理支持

选择的事务处理支持将以向下兼容方式覆盖与此连接池关联的资源适配器中的事务处理支持属性。也就是说，它支持的事务处理级别可以比资源适配器中指定的事务处理级别低或与其相同，但不能指定更高的级别。

值可为以下任意一种：

XATransaction

资源适配器通过实现 LocalTransaction 和 XAResource 接口来支持资源管理器本地事务处理和 JTA 事务处理。XA 事务处理由事务处理管理器在资源管理器外部进行控制和调整。

LocalTransaction

资源适配器通过实现 LocalTransaction 接口来支持本地事务处理。本地事务处理的管理在资源管理器内部进行，不涉及任何外部事务处理管理器。

NoTransaction

资源适配器不支持资源管理器本地事务处理或 JTA 事务处理，也不实现 XAResource 或 LocalTransaction 接口。

其他属性

连接器连接池的其他属性。可用属性取决于连接器连接池的资源适配器。可以使用在此表中指定的名称/值对，覆盖由资源适配器提供方定义的属性默认值。

如果您为属性指定了名称而没有指定值，则在创建完连接池时，该属性将从表中删除。如果您现在不希望设置某个属性的值，但希望将该属性保留在表中以便在以后设置值，请在“值”字段中键入一对空括号：

()
