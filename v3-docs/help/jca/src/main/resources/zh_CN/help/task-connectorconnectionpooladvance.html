编辑连接器连接池高级属性

  1. 在导航树中，展开“资源”节点。

  2. 展开“连接器”节点。

  3. 在“连接器”节点下，选择“连接器连接池”节点。

    此时将打开“连接器连接池”页。

  4. 在“连接器连接池”页上，单击要编辑的连接器连接池的名称。

    此时将打开“编辑连接器连接池”页。

  5. 在“编辑连接器连接池”页上，单击“高级”选项卡。

    此时将打开“编辑连接器连接池高级属性”页。

  6. 在“最多验证一次”字段中，键入以秒为单位的一段时间，经过这段时间后，最多验证连接一次。

    将该属性设置为适当的值可以最大程度减少连接的验证请求数。默认值为 0，表示未启用该属性。

  7. 在“泄漏超时”字段中，键入用于在连接池中跟踪连接泄漏的时间长度 (秒)。

    如果值为 0，则表示禁用连接泄漏跟踪。默认值为 0。

  8. 选中“泄漏回收”复选框，在完成泄漏连接跟踪后将泄漏的连接还原到池中。

    默认情况下禁用此选项。

  9. 在“创建重试尝试次数”字段中，键入创建新的连接失败后进行尝试的次数。

    如果值为 0，则表示不会尝试再次创建连接。默认值为 0。

  10. 在“重试时间间隔”字段中，键入两次尝试创建连接之间的时间间隔 (秒)。

    仅当“创建重试尝试次数”的值大于 0 时才使用此属性。默认值为 10 秒。

  11. 选中“共享”复选框可以启用池的连接共享。

    默认情况下启用此选项。

  12. 选中“延迟关联”复选框，以便在对连接执行操作时，延迟关联连接。

    当完成事务处理并且结束组件方法时，对连接取消关联，这有助于重新使用物理连接。默认情况下禁用此选项。

  13. 选中“延迟连接登记”复选框，仅当在方法中实际使用资源时才将资源添加至事务处理。

    默认情况下禁用此选项。

  14. 选中“与线程关联”复选框，以将连接与线程关联。

    如果同一个线程需要连接，则该线程可以重新使用已与其关联的连接，因此不会导致从池中获取连接的开销。默认情况下禁用此选项。

  15. 选中“匹配连接”复选框，为池启用连接匹配。

    如果您知道池中的连接将始终同构，因此从池中选择的连接不需要由资源适配器进行匹配，请禁用此选项。默认情况下启用此选项。

  16. 在“匹配连接使用次数”字段中，键入池可以重新使用连接的次数。

    连接在重复使用指定的次数后将被关闭。如果值为 0，则表示不启用此属性。默认值为 0。

  17. 单击“保存”。

    提示:

    要返回到默认设置，请单击“加载默认值”按钮，然后单击“保存”。
