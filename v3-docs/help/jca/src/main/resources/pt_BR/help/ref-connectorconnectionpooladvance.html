Editar Atributos Avançados do Pool de Conexões do Conector

Use a página Editar Pool de Conexões do Conector para modificar as definições avançadas do pool de conexões do conector.

A página Editar Atributos Avançados do Pool de Conexões do Conector contém as opções a seguir.

Carregar Defaults

Botão para restaurar com os valores default as definições que apresentam valores default. As definições que não apresentam valores default não são alteradas.

Validar no máximo uma vez

Especifica o intervalo de tempo, em segundos, entre sucessivas solicitações para validar uma conexão no máximo uma vez. O valor default é 0, o que significa que o atributo não está ativado.

Time-out para perda

Período em segundos para rastrear as perdas de conexão em um pool de conexões. Um valor de 0 indica que o rastreamento do vazamento da conexão está desativado. O valor default é 0.

Reclamação de perda

Se esta opção está selecionada, as conexões com vazamento serão restauradas no pool após a conclusão do rastreamento de vazamento da conexão. Por default, essa opção está desativada.

Novas tentativas de criação

O número de tentativas a serem feitas se houver uma falha na criação de uma nova conexão. O valor de 0 indica que nenhuma tentativa será feita para criar a conexão novamente. O valor default é 0.

Intervalo de nova tentativa

O intervalo de tempo, em segundos, entre duas tentativas de criar uma conexão. Esse atributo só será usado se o valor Novas tentativas de criação for maior que 0. O valor default é 10 segundos.

Colocar no Pool

Permite colocar no pool as conexões para o pool. Por default, a opção está ativada.

Associação lenta

Se esta opção está selecionada, as conexões serão associadas lentamente quando uma operação for executada nelas. As conexões serão desassociadas quando a transação for concluída e um método de componente for finalizado, o que ajuda a reutilizar as conexões físicas. Por default, essa opção está desativada.

Inscrição de conexão lenta

Se esta opção está selecionada, um recurso para a transação será inscrito somente quando a transação for realmente usada em um método. Por default, essa opção está desativada.

Associar com Thread

Se esta opção está selecionada, uma conexão é associada a um thread. Quando o mesmo thread for necessário a uma conexão, a conexão já associada ao thread poderá ser reutilizada, não incorrendo, portanto, na sobrecarga para obtenção de outra conexão do pool. Por default, essa opção está desativada.

Conexões correspondentes

Se esta opção está selecionada, a correspondência de conexão será ativada para o pool. Essa opção poderá ser desativada se você souber que as conexões no pool serão sempre homogêneas e, portanto, uma conexão selecionada no pool não precisará coincidir com o adaptador de recurso. Por default, a opção está ativada.

Corresponder o uso da conexão

O número de vezes que uma conexão deve ser reutilizada pelo pool. Depois que uma conexão é reutilizada por um número especificado de vezes, ela é fechada. O valor de 0 indica que o atributo está desativado. O valor default é 0.
