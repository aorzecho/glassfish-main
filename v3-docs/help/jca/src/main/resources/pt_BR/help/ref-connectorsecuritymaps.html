Mapas de segurança do grupo de conexões do conector

Use a página Mapas de segurança do conjunto de conexões do conector para definir os mapas de segurança do conjunto de conexões do conector.

Os mapas de segurança dos conjuntos de conexões do conector permitem a criação de um mapeamento entre usuários e grupos de Java EE e usuários e grupos de EIS. Um mapa de segurança mapeia o principal recuperado durante a autenticação do servlet ou EJB, para as credenciais aceitas pelo EIS. É possível mapear múltiplos principais do &ProductName: para o mesmo principal back-end.

Um mapa de segurança para um conjunto de conexões é diferente de um mapa de segurança do conector de trabalho, no sentido que ele mapeia um principal associado com uma instância de trabalho de entrada para um principal no domínio de segurança do Servidor GlassFish. Consulte Mapas de segurança de trabalho para obter informações sobre os mapas de segurança do conector do trabalho.

Para cada mapa de segurança, as seguintes informações são fornecidas.

Nome

O nome do mapa de segurança.

A tabela Mapas de segurança também contém as seguintes opções.

Novo

Botão para criar um novo mapa de segurança.

Excluir

Botão para excluir um ou mais mapas de segurança selecionados.
