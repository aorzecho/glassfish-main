Editar Pool de Conexões do Conector

Use a página Editar Pool de Conexões do Conector para modificar as definições do pool de conexões do conector.

A página Editar Pool de Conexões do Conector contém as opções a seguir.

Carregar Defaults

Botão para restaurar com os valores default as definições que apresentam valores default. As definições que não apresentam valores default não são alteradas.

Descarregar

Botão para reinicializar as conexões no pool de conexões. Clicar nesse botão destrói quaisquer conexões existentes, recria as conexões estabelecidas para o pool e restaura o pool para seu tamanho inicial e mínimo.

Ping

Botão utilizado para verificar se o pool de conexões é utilizável.

Nome do conjunto

Um nome único que identifica o pool de conexões do conector. Esse é um campo somente leitura. Você só pode especificar o nome do pool ao criar um novo pool de conexões do conector.

Adaptador de recurso

O adaptador de recursos associado ao pool de conexões do conector.

Definição da conexão

A definição da conexão para o pool de conexões do conector. As escolhas na lista dependem do adaptador de recurso selecionado. A definição da conexão identifica um elemento conexão-definição do adaptador de recurso no arquivo ra.xml.

Ping

Se essa opção for selecionada, será efetuado o ping automático no pool de conexões durante a criação ou reconfiguração, para identificar e avisar sobre valores de atributos com erros. Por default, essa opção está desativada.

Descrição

Uma descrição do pool de conexões do conector.

Tamanho inicial e mínimo do conjunto

O número mínimo de conexões no pool de conexões do conector. O valor default é 8.

Tamanho máximo do conjunto

O número máximo de conexões no pool de conexões do conector. O valor default é 32.

Quantidade de Redimensionamento do Pool

O número de conexões a serem removidas quando o time-out para inatividade do pool expirar. O valor default é 2.

Time-out para inatividade

A quantidade máxima de tempo, em segundos, que uma conexão pode permanecer inativa no pool. O valor default é 300.

Tempo máx. de espera

O tempo que a aplicação solicitante de uma conexão aguardará até obter um time-out. O valor default é 60000.

Validação da conexão

Se a caixa de seleção Obrigatório estiver marcada, as conexões serão validadas antes de serem enviadas à aplicação. Se uma validação de recurso falhar, ela será destruída e um novo recurso será criado e retornado. Por default, essa opção está desativada.

Em qualquer falha

Se a caixa de seleção Fechar todas as conexões estiver marcada, o GlassFish Server fechará todas as conexões no pool e as restabelecerá se uma única conexão falhar. Por default, essa opção está desativada.

Suporte de transação

O suporte de transação escolhido substitui o atributo do suporte de transação no adaptador de recursos associado a esse pool de conexões de uma forma compatível com descendência. Em outras palavras, pode dar suporte a um nível de transação mais baixo do que o especificado no adaptador de recurso, ou ao mesmo nível de transação especificado no adaptador de recurso, mas não pode especificar um nível mais alto.

O valor pode ser um dos seguintes:

Transação XA

O adaptador de recursos suporta transações locais e de JTA do gerenciador de recursos implementando interfaces de LocalTransaction e XAResource. As transações XA são controladas e coordenadas por um gerenciador de transações externo ao gerenciador de recursos.

LocalTransaction

O adaptador do recurso suporta transações locais implementando a interface do LocalTransaction. As transações locais são gerenciadas internamente a um gerenciador de recursos e não envolve nenhum gerenciador de transações.

NoTransaction

O adaptador de recursos não suporta transações locais ou de JTA do gerenciador de recursos e não implementa as interfaces XAResource ou LocalTransaction.
