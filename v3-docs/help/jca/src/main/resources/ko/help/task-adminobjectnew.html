관리 대상 객체 자원을 만드는 방법

  1. 탐색 트리에서 자원 노드를 확장합니다.
  2. 커넥터 노드를 확장합니다.
  3. 커넥터 노드에서 관리 객체 자원 노드를 선택합니다.

    관리 객체 자원 페이지가 열립니다.

  4. 관리 객체 자원 페이지에서 새로 만들기 버튼을 누릅니다.

    새 관리 객체 자원 페이지가 열립니다.

  5. JNDI 이름 필드에 관리 객체 자원을 식별하는 고유한 이름을 입력합니다.
  6. 자원 어댑터 드롭다운 목록에서 관리 객체 자원에 대한 자원 어댑터를 선택합니다.
  7. 자원 유형 드롭다운 목록에서 관리 객체 자원에 대한 Java 유형을 선택합니다.
  8. 클래스 이름 드롭다운 목록에서 자원 유형과 연결된 구현 클래스 이름을 선택합니다.
  9. 설명 필드에 관리 객체 자원에 대한 설명을 입력합니다.
  10. 관리 객체 자원을 활성화하려면 상태 사용 가능 확인란을 선택합니다.

    이 옵션은 기본적으로 활성화됩니다.

  11. (옵션) 추가 등록 정보 표에서 관리 객체 자원에 대한 추가 등록 정보를 지정합니다.

    사용 가능한 등록 정보는 커넥터 연결 풀의 자원 어댑터에 따라 다릅니다.

    등록 정보를 추가하려면 등록 정보 추가 버튼을 누릅니다. 빈 행이 표시되면 이름 필드에 등록 정보 이름을 입력하고 값 필드에 등록 정보 값을 입력합니다.

  12. 사용 가능한 대상 섹션에서 관리 객체가 있게 될 도메인, 클러스터 또는 서버 인스턴스를 선택하고 추가를 누릅니다.

    선택된 필드에 나열된 도메인, 클러스터 또는 서버 인스턴스 중 하나를 제거하려면 필드에서 원하는 대상을 선택하고 제거를 누릅니다.

  13. 확인을 누릅니다.
