새 관리 객체 자원

새 관리 객체 자원 페이지에서는 새 관리 객체 자원을 만들 수 있습니다.

새 관리 객체 자원 페이지에는 다음 옵션이 포함되어 있습니다.

JNDI 이름

관리 객체 자원을 식별하는 고유한 이름입니다.

자원 어댑터

관리 객체 자원에 대한 자원 어댑터입니다.

자원 유형

관리 객체 자원의 Java 유형입니다(예: javax.jms.Topic).

클래스 이름

자원 유형과 연결된 구현 클래스 이름입니다.

설명

관리 객체 자원에 대한 설명입니다.

상태

이 옵션을 선택하면 관리 객체 자원이 활성화됩니다. 이 옵션은 기본적으로 활성화됩니다.

추가 등록 정보

관리 객체 자원에 대한 추가 등록 정보입니다. 사용 가능한 등록 정보는 커넥터 연결 풀의 자원 어댑터에 따라 다릅니다. 이 표에 지정된 이름-값 쌍을 사용하여 자원 어댑터 공급업체에서 정의한 등록 정보의 기본값을 대체할 수 있습니다.

대상

관리 객체 자원에 대한 클러스터 및 독립 실행형 인스턴스입니다. 추가 및 모두 추가 버튼을 사용하여 원하는 대상을 선택한 대상 열로 이동합니다. 제거 및 모두 제거 버튼을 사용하여 필요하지 않은 대상을 사용 가능한 대상 열로 이동합니다.

이 옵션은 클러스터나 독립 실행형 인스턴스가 도메인에 만들어진 경우에만 표시됩니다.

관련 작업

  • 관리 대상 객체 자원을 만드는 방법

  • 관리 대상 객체 자원을 편집하는 방법

  • 관리 대상 객체 자원을 삭제하는 방법

관련 asadmin 명령

  • create-admin-object(1)

  • list-admin-objects(1)

  • delete-admin-object(1)
