配置 ORB

  1. 在树组件中，展开“配置”节点。
  2. 选择要配置的实例：
    • 要配置特定的实例，请选择该实例的配置节点。例如，对于默认实例 server，请选择 server-config 节点。
    • 要为以后使用 default-config 副本的实例配置默认设置，请选择 default-config 节点。
  3. 选择 "ORB" 节点。

    此时将打开 "ORB" 页面。

  4. 从“线程池 ID”下拉式列表中选择 ORB 使用的线程池。

    ORB 使用线程池响应来自企业 Bean 的远程客户机和其他客户机（使用 RMI-IIOP 进行通信）的请求。ORB 的默认线程池为 thread-pool-1，它使用适合 RMI/IIOP 请求的默认值。

    有关更多信息，请参见线程池。

  5. 从“最大消息片段大小”下拉式列表中，选择 IIOP 消息的最大片段大小。

    大于此大小的消息将被分段。默认值为 1024 字节。

  6. 在“连接总数”字段中，设置所有 IIOP 侦听器的最大外来连接数。

    默认值为 1024。

  7. 选中“需要进行 IIOP 客户机验证”复选框以指定需要进行 IIOP 客户机验证。

    如果启用 IIOP 客户机验证，则服务器将拒绝未经验证的请求并在发送给客户机的可交互使用的对象引用 (Interoperable Object Reference, IOR) 中插入需要验证位。默认情况下禁用此选项。

  8. （可选的）在“其他属性”区域中，指定其他属性。

    GlassFish Server 不定义 ORB 的任何其他属性。

    • 要添加属性，请单击“添加属性”按钮。在显示的空白行中，在“名称”字段中键入属性名称，在“值”字段中键入属性值。
    • 要修改属性，请编辑该属性的“值”字段。
    • 要删除属性，请选中位于您要删除属性的“名称”字段左边的复选框，然后单击“删除属性”按钮。
  9. 单击“保存”。

    提示 - 要返回到默认设置，请单击“装入默认值”按钮，然后单击“保存”。

  10. 重新启动 GlassFish Server。
