ORB

Nella pagina ORB è possibile configurare Object Request Broker.

ORB include l'infrastruttura necessaria per identificare e individuare gli oggetti, gestire le connessioni, fornire dati e richiedere comunicazioni. I client remoti dei bean enterprise (moduli EJB) comunicano con GlassFish Server mediante ORB.

La pagina ORB contiene le seguenti opzioni.

Load Defaults

Pulsante che consente di ripristinare i valori predefiniti di tutte le impostazioni.

Configuration Name

Nome della configurazione a cui si applicano le impostazioni visualizzate nella pagina. Campo di sola lettura.

Thread Pool ID

Pool del thread utilizzato da ORB. ORB utilizza i pool del thread per rispondere alle richieste di client remoti di bean enterprise e di altri client che comunicano mediante RMI-IIOP. Il pool del thread predefinito per ORB è thread-pool-1, che utilizza impostazioni predefinite appropriate per RMI/IIOP.

Max Message Fragment Size

Dimensioni massime del frammento per i messaggi IIOP. Se i messaggi hanno dimensioni superiori a quella specificata, vengono frammentati. Il valore predefinito è pari a 1024 byte.

Total Connections

Numero massimo di connessioni in entrata per tutti i listener IIOP. Il valore predefinito è pari a 1024.

IIOP Client Authentication

Se la casella di controllo Enabled è selezionata, il server rifiuta le richieste non autenticate e inserisce un bit richiesto per l'autenticazione nei riferimenti IOR (Interoperable Object References) inviati ai client. L'autenticazione client IIOP può essere abilitata o disabilitata. Questa opzione è disabilitata per impostazione predefinita.

Additional Properties

Proprietà aggiuntive per ORB. In GlassFish Server non è definita alcuna proprietà aggiuntiva per ORB.
