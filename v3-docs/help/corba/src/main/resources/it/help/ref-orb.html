ORB

Nella pagina ORB è possibile configurare Object Request Broker.

ORB offre l'infrastruttura necessaria per identificare e individuare gli oggetti, gestire le connessioni, distribuire i dati e richiedere comunicazioni. I client remoti di bean enterprise (moduli EJB) comunicano con GlassFish Server mediante ORB.

La pagina ORB contiene le opzioni riportate di seguito.

Carica impostazioni predefinite

Pulsante che consente di ripristinare i valori predefiniti delle impostazioni che dispongono di tali valori. Le impostazioni prive di valori predefiniti restano invariate.

Nome configurazione

Nome della configurazione a cui vengono applicate le impostazioni definite in questa pagina. Questo campo è di sola lettura.

ID pool di thread

Pool di thread usato da ORB. ORB utilizza i pool di thread per rispondere alle richieste di client remoti di bean enterprise e di altri client che comunicano mediante RMI-IIOP. Il pool di thread predefinito per l'ORB è thread-pool-1, che utilizza impostazioni predefinite appropriate per le richieste RMI/IIOP.

Dimensione massima frammento messaggio

Dimensione massima del frammento per i messaggi IIOP. Se i messaggi hanno dimensioni superiori a quella specificata, vengono frammentati. Il valore predefinito è 1024 byte.

Connessioni totali

Numero massimo di connessioni in entrata per tutti i listener IIOP. Il valore predefinito è 1024.

Autenticazione client IIOP

Se la casella di controllo Abilitato è selezionata, il server rifiuta le richieste non autenticate e inserisce un bit richiesto per l'autenticazione nei riferimenti IOR (Interoperable Object References) inviati ai client. L'autenticazione client IIOP può essere abilitata o disabilitata. Questa opzione è disabilitata per impostazione predefinita.

Proprietà aggiuntive

Proprietà aggiuntive per ORB. In GlassFish Server non è definita alcuna proprietà aggiuntiva per ORB.
