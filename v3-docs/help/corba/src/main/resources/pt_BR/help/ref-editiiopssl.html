SSL

Utilize a página SSL para modificar as configurações SSL para uma escuta IIOP.

A página SSL contém estas opções.

Configuration Name

O nome da configuração para a qual as definições nesta página se aplicam. Este campo é de somente leitura.

SSL3

Se essa caixa de seleção estiver marcada, o protocolo SSL3 é ativado para a escuta de IIOP. A opção está ativada por padrão.

TLS

Se esta caixa de seleção estiver marcada, o protocolo TLS é ativado para a escuta de IIOP. A opção está ativada por padrão.

Autenticação do cliente

Se a caixa de seleção estiver marcada, os clientes deverão identificar-se para o servidor a cada solicitação. A opção está desativada por padrão.

Apelido do certificado

O apelido do certificado do servidor no banco de dados de certificados ou o símbolo PKCS#11. No certificado, o formato do nome é tokenname:nickname. Incluir a parte do nome do símbolo: nesse atributo é opcional.

Armazenamento de chave

O nome do arquivo de armazenamento de chave (por exemplo, keystore.jks).

Algoritmo confiável

O nome do algoritmo de gerenciamento confiável (por exemplo, PKIX) a ser utilizado para validação do caminho de certificação.

Comprimento máx. do certificado

O número máximo de certificados intermediários não auto-emitidos que podem existir em um caminho de certificação. Esse campo é utilizado apenas se o campo Algoritmo confiável estiver definido como PKIX. O valor 0 indica que o caminho pode conter somente um certificado. O valor -1 indica que o comprimento do caminho não é restringido (não existe máximo). Definir um valor menor do que -1 causa uma exceção a ser lançada.

Armazenamento confiável

O nome do arquivo truststore (por exemplo, cacerts.jks).

Conjuntos codificados

Uma área onde é possível adicionar ou remover conjuntos codificados. Se você não adicionar quaisquer conjuntos codificados, todos os conjuntos codificados serão utilizados.
