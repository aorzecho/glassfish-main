ORB

Utilize a página ORB para configurar o Object Request Broker.

O ORB oferece a infraestrutura para identificar e localizar objetos, processar o gerenciamento de conexões, entregar dados e solicitar comunicações. Clientes remotos de enterprise beans (módulos EJB) se comunicam com o GlassFish Server utilizando o ORB.

A página ORB contém as opções a seguir.

Carregar Defaults

Botão para restaurar com os valores default as definições que apresentam valores default. As definições que não apresentam valores default não são alteradas.

Nome da configuração

O nome da configuração à qual as definições nesta página se aplicam. Este campo é somente para leitura.

ID do Pool de Threads

O pool de threads utilizado pelo ORB. O ORB utiliza pools de threads para responder a solicitações de clientes remotos de enterprise beans e outros clientes que se comunicam utilizando RMI-IIOP. O pool de thread default do ORB é thread-pool-1, que usa defaults adequados para solicitações de RMI/IIOP.

Tamanho máx. do fragmento de mensagem

O tamanho máximo di fragmento de mensagem para mensagens IIOP. As mensagens com tamanho maior do que o máximo permitido são fragmentadas. O valor default é 1024 bytes.

Total de conexões

O número máximo de conexões de entrada para todas as listeners IIOP.↓ O valor default é 1024.

Autenticação do cliente IIOP

Se a caixa de seleção Ativado estiver marcada, o servidor rejeita solicitações não autenticadas e insere um bit autenticação-necessária nas Referências de Objeto Interoperáveis (IORs) enviadas para clientes. A autenticação do cliente IIOP pode ser ativada ou desativada. Por default, essa opção está desativada.

Propriedades Adicionais

Propriedades adicionais do ORB. O GlassFish Server não define propriedades adicionais para o ORB.
