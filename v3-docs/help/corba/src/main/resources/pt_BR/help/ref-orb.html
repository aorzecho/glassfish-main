ORB

Utilize a página ORB para configurar o Object Request Broker.

O ORB fornece a infraestrutura para identificar e localizar objetos, processar o gerenciamento de conexões, entregar dados e solicitar comunicações. Clientes remotos de beans empresariais (módulos EJB) se comunicam com o Servidor GlassFish utilizando o ORB.

A página ORB contém as opções a seguir.

Carregar Padrões

Botão para restaurar os valores padrão de todas as configurações.

Nome da configuração

Nome da configuração ao qual as configurações desta página serão enviadas. Este campo é de somente leitura.

ID do conjunto de segmentos

O grupo de segmentos utilizado pelo ORB. O ORB utiliza grupos de segmentos para responder a solicitações de clientes remotos de beans empresariais e outros clientes que se comunicam utilizando RMI-IIOP. O grupo de segmentos padrão do ORB é thread-pool-1, que utiliza padrões apropriados para solicitações RMI/IIOP.

Tamanho máx. do fragmento de mensagem

O tamanho máximo di fragmento de mensagem para mensagens IIOP. As mensagens com tamanho maior do que o máximo permitido são fragmentadas. O valor padrão é 1024 bytes.

Total de conexões

O número máximo de conexões de entrada para todas as escutas de IIOP. O valor padrão é 1024.

Autenticação do cliente IIOP

Se a caixa de seleção Ativado estiver marcada, o servidor rejeita solicitações não autenticadas e insere um bit autenticação-necessária nas Referências de Objeto Interoperáveis (IORs) enviadas para clientes. A autenticação do cliente IIOP pode ser ativada ou desativada. Por padrão, a opção está desativada.

Propriedades adicionais

Propriedades adicionais do ORB. O Servidor GlassFish não define propriedades adicionais para o ORB.
