ORB

La page ORB permet de configurer le programme Object Request Broker.

Le broker ORB fournit l'interface requise pour identifier et localiser des objets, traiter la gestion des connexions, fournir des données et demander des communications. Les clients distants d'Enterprise Beans (modules EJB) communiquent avec GlassFish Server à l'aide du programme ORB.

La page ORB contient les options suivantes.

Charger les valeurs par défaut

Bouton de restauration des valeurs par défaut des paramètres qui ont des valeurs par défaut. Les paramètres qui n'ont pas de valeurs par défaut ne sont pas modifiés.

Nom de configuration

Nom de la configuration à laquelle s'appliquent les paramètres de cette page. Ce champ est en lecture seule.

ID du pool de threads

Pool de threads utilisé par l'ORB. L'ORB utilise des pools de threads pour répondre aux demandes des clients distants d'Enterprise Beans et d'autres clients qui communiquent via le protocole RMI-IIOP. Le pool de threads par défaut de l'ORB est thread-pool-1 qui utilise les valeurs par défaut appropriées aux demandes RMI/IIOP.

Taille max. du fragment de message

Taille maximale du fragment des messages IIOP. Les messages dépassant cette taille sont fragmentés. La valeur par défaut est 1 024 octets.

Nombre total de connexions

Nombre maximal de connexions entrantes pour l'ensemble des processus d'écoute IIOP. La valeur par défaut est 1 024.

Authentification client IIOP

Si la case Activé est cochée, le serveur rejette les demandes non authentifiées et insère un bit de demande d'authentification dans les références IOR (Interoperable Object Reference) envoyées aux clients. L'authentification client IIOP peut être activée ou désactivée. Cette option est désactivée par défaut.

Autres propriétés

Propriétés supplémentaires de l'ORB. GlassFish Server ne définit aucune autre propriété pour l'ORB.
