ORB

Utilisez la page ORB pour configurer l’ORB (Object Request Broker).

Le courtier ORB fournit l’interface requise pour identifier et localiser des objets, traiter la gestion des connexions, fournir des données et demander des communications. Les clients distants de beans entreprise (modules EJB) communiquent avec GlassFish Server à l’aide du programme ORB.

La page ORB contient les options suivantes :

Restaurer les valeurs par défaut

Bouton de restauration des valeurs par défaut des paramètres qui ont des valeurs par défaut. Les paramètres qui n'ont pas de valeurs par défaut ne sont pas modifiés.

Nom de la configuration

Nom de la configuration à laquelle les paramètres de cette page s'appliquent. Ce champ est en lecture seule.

ID de pool de threads

Pool de threads utilisé par l’ORB. L’ORB utilise des pools de threads pour répondre aux requêtes des clients distants de beans entreprise et d’autres clients qui communiquent par RMI-IIOP. Le pool de threads par défaut de l’ORB est thread-pool-1 qui utilise les valeurs par défaut appropriées aux requêtes RMI/IIOP.

Taille max. du fragment de message

Taille maximale du fragment des messages IIOP. Les messages dépassant cette taille sont fragmentés. La valeur par défaut est 1024 octets.

Nombre total de connexions

Nombre maximum de connexions entrantes pour l’ensemble des listeners IIOP. La valeur par défaut est 1024.

Authentification client IIOP

Si la case Activé est sélectionnée, le serveur refuse les requêtes non authentifiées et insère un bit de demande d’authentification dans les IOR (Interoperable Object Reference) envoyées aux clients. L’authentification client IIOP peut être activée ou désactivée. Cette option est désactivée par défaut.

Autres propriétés

Autres propriétés de l’ORB. GlassFish Server ne définit aucune autre propriété pour l’ORB.
