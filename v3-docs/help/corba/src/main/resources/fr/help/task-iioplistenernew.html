Pour créer un listener IIOP

  1. Dans l’arborescence, développez le nœud Configuration.
  2. Sélectionnez l’instance à configurer :
    • Pour configurer une instance en particulier, sélectionnez le nœud de configuration de l’instance. Par exemple, pour l’instance par défaut, server, sélectionnez le nœud server-config.
    • Pour configurer les paramètres par défaut des futures instances utilisant une copie de default-config, sélectionnez le nœud default-config.
  3. Sous le nœud Configuration, développez le nœud ORB.
  4. Sélectionnez Listeners IIOP.

    La page Modules d’écoute IIOP s’affiche.

  5. Cliquez sur Nouveau.
  6. Dans le champ Nom, entrez un nom permettant d’identifier le listener.
  7. Dans le champ Adresse réseau, entrez l’adresse réseau du serveur.

    Vous pouvez entrer une adresse IP ou un nom d’hôte DNS pouvant être résolu.

  8. Dans le champ Port du listener, entrez le numéro de port d’écoute que le listener doit utiliser.
  9. Cochez la case Activé pour activer les connexions entrantes IIOP de l’ORB à GlassFish Server.

    Cette option est activée par défaut.

  10. Cochez la case Sécurité permise pour activer la SSL pour le listener IIOP.

    Cette option est désactivée par défaut.

  11. (Facultatif) Dans la section Autres propriétés, spécifiez d’autres propriétés.

    GlassFish Server ne définit aucune autre propriété pour les listeners IIOP.

    Pour ajouter une propriété, cliquez sur le bouton Ajouter propriété. Sur la ligne vide qui apparaît, saisissez le nom de la propriété dans le champ Nom et la valeur de la propriété dans le champ Valeur.

  12. Cliquez sur OK.
  13. Si vous avez coché la case Sécurité permise, suivez la procédure fournie à la rubrique Pour modifier les paramètres SSL d'un listener IIOP sécurisé.
