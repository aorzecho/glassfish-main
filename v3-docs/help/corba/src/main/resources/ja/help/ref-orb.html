ORB

Object Request Brokerを構成するには、「ORB」ページを使用します。

ORBは、オブジェクトの識別と検索、接続管理、およびデータとリクエストの配信に必要なインフラストラクチャを提供します。エンタープライズBean(EJBモジュール)のリモート・クライアントは、ORBを使用してGlassFish Serverと通信します。

「ORB」ページには、次のオプションが含まれます。

デフォルトをロード

デフォルト値を持つ設定をデフォルト値に戻すボタン。デフォルト値を持たない設定は変更されません。

構成名

このページの設定が適用される構成の名前。このフィールドは読取り専用です。

スレッド・プール・アイデンティティ

ORBが使用するスレッド・プール。ORBはスレッド・プールを使用して、エンタープライズBeanのリモート・クライアントおよびRMI-IIOPを使用して通信する他のクライアントからのリクエストに応答します。ORBのデフォルトのスレッド・プールはthread-pool-1です。このスレッド・プールは、RMI/IIOPリクエストに適切なデフォルト設定を使用します。

最大メッセージ・フラグメント・サイズ

IIOPメッセージの最大フラグメント・サイズ。このサイズより大きいメッセージは分割されます。デフォルト値は1024バイトです。

接続総数

すべてのIIOPリスナーの入力接続の最大数。デフォルト値は1024です。

IIOPクライアント認証

「有効」チェックボックスを選択すると、サーバーは認証されていないリクエストを拒否し、クライアントに送信されたInteroperable Object Reference (IOR)に認証要求のビットを挿入します。IIOPクライアント認証は有効または無効にできます。このオプションは、デフォルトでは無効になっています。

追加プロパティ

ORBの追加プロパティ。GlassFish ServerはORBの追加プロパティを定義しません。
