ORB

Utilice la página ORB para configurar el agente intermediarios de solicitud de objetos.

El ORB proporciona la infraestructura necesaria para identificar y localizar objetos, administrar las conexiones, enviar datos y solicitar comunicación. Los clientes remotos de beans empresariales (módulos EJB) se comunican con el Servidor GlassFish mediante el ORB.

La página ORB contiene las siguientes opciones.

Cargar valores predeterminados

Botón para restablecer las opciones que tienen valores por defecto a estos. Las opciones que no tienen valores por defecto no se modifican.

Configuration Name

El nombre de la configuración a la que aplicar los valores de esta página. Este campo es de sólo lectura.

Id. del conjunto de subprocesos

El conjunto de subprocesos que utiliza el ORB. ORB utiliza conjuntos de subprocesos para responder a solicitudes de clientes remotos de beans empresariales y otros clientes que se comunican por medio de RMI-IIOP. El conjunto de subprocesos predeterminado para ORB es thread-pool-1, que utiliza valores predeterminados apropiados para las solicitudes de RMI/IIOP.

Tamaño máx. de fragmentos de mensaje

El tamaño máximo para los fragmentos de mensajes IIOP. Los mensajes más grandes que este tamaño quedarán fragmentados. El valor predeterminado es de 1024 bytes.

Total de conexiones

El número máximo de conexiones entrantes para las escuchas IIOP. El valor predeterminado es 1024.

Autenticación de cliente IIOP

Si la casilla de verificación Habilitado está seleccionada, el servidor rechaza las solicitudes no autenticadas e inserta un bit de autenticación requerida en el las referencias de objetos remotos interoperables (IOR) enviadas a los clientes. La autenticación de cliente IIOP se puede activar o desactivar. Esta opción se deshabilita por defecto.

Propiedades adicionales

Propiedades adicionales de ORB Servidor GlassFish no define ninguna propiedad adicional para el ORB.
