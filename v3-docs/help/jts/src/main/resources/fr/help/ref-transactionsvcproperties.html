Propriétés spécifiques au service de transaction

Les propriétés suivantes sont disponibles pour la configuration du service de transaction.

oracle-xa-recovery-workaround

Si le paramètre est défini sur true, la solution de contournement Oracle XA Resource est utilisée dans la récupération de transaction. La valeur par défaut est true.

disable-distributed-transaction-logging

Si l’attribut true est sélectionné, la journalisation des transactions est désactivée, ce qui peut améliorer les performances. Si la case Au redémarrage est sélectionnée, cette propriété est ignorée. La valeur par défaut est false.

xaresource-txn-timeout

Modifie le délai d’attente XAResource . Dans certains cas, le délai d’attente XAResource par défaut peut entraîner l’abandon des transactions, il est donc souhaitable de le modifier. La valeur par défaut est spécifique à la XAResource utilisée.

pending-txn-cleanup-interval

Spécifie l’intervalle, en secondes, auquel un thread asynchrone vérifie les transactions en attente et les termine. Si cette propriété n’est pas spécifiée, il n’y a pas de valeur par défaut. Si cette propriété est présente mais n’a pas de valeur, la valeur par défaut est 60.

use-last-agent-optimization

Si l’attribut true est sélectionné, l’optimisation du dernier agent est activée, ce qui améliore la capacité de traitement des transactions. Si une ressource non-XA est utilisée avec des ressources XA dans la même transaction, la ressource non-XA est le dernier agent. La valeur par défaut est true.

delegated-recovery

Si l’attribut true est sélectionné, la récupération déléguée à l’échelle du cluster est activée. La valeur par défaut est ’false’ (faux).

wait-time-before-recovery-insec

Spécifie le temps d’attente, en secondes, après lequel une instance lance la récupération pour une instance bloquée.

db-logging-resource

Spécifie le nom JNDI de la ressource JDBC pour la base de données dans laquelle les transactions sont journalisées. Pour de plus amples informations, reportez-vous au Chapitre 14, Using the Transaction Service du Oracle GlassFish Server 3.1 Application Development Guide. Il n’y a pas de valeur par défaut.

xa-servername

Spécifie le nom d’hôte utilisé par le service de transaction pour identifier les transactions gérées par le GlassFish Server installé. Cela peut parfois être utile pour la récupération de transactions dans le fichier journal créé sur un hôte différent qui exécute le GlassFish Server.
