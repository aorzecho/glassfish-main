Définition de l’intervalle entre points clés

Les opérations de point clé permettent de compresser les fichiers journaux de transactions. L’intervalle entre points clés est le nombre de transactions effectuées entre les opérations de point clé dans le journal. Les opérations de point clé permettent de réduire la taille des fichiers journaux de transaction. Plus le nombre d’intervalles entre points clés est élevé et plus la taille des fichiers journaux est importante, affectant potentiellement les performances. Un nombre plus petit d’intervalles entre points clés (par exemple 256) entraîne des fichiers journaux plus petits, mais diminue légèrement les performances en raison du plus grand nombre d’opérations de point clé.

  1. Dans l’arborescence de navigation, développez le nœud Configuration.
  2. Sélectionnez l’instance à configurer :
    • Pour configurer une instance en particulier, sélectionnez le nœud de configuration de l’instance. Par exemple, pour l’instance par défaut, server, sélectionnez le nœud server-config.
    • Pour configurer les réglages par défaut pour toutes les instances, sélectionnez le nœud default-config.
  3. Sélectionnez le nœud Service de transaction.

    La page Service de transaction s’affiche.

  4. Dans le champ Intervalle de point clé, saisissez le nombre de transactions entre les opérations de points clés.

    La valeur par défaut est 65 536.

  5. Cliquez sur Enregistrer.
