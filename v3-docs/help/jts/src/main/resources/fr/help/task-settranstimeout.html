Définir une valeur de temporisation des transactions

Par défaut, le serveur n'applique aucune temporisation aux transactions. Le serveur attend indéfiniment l'exécution de la transaction. Si vous avez défini une valeur de temporisation pour les transactions et qu'une transaction n'est pas terminée dans le délai configuré, celle-ci est annulée par GlassFish Server.

  1. Dans l'arborescence de navigation, développez le noeud Configuration.

  2. Sélectionnez l'instance à configurer :

    • Pour configurer une instance spécifique, sélectionnez le noeud de configuration de l'instance. Par exemple, pour l'instance par défaut, server, sélectionnez le noeud server-config.

    • Pour configurer les paramètres par défaut pour toutes les instances, sélectionnez le noeud default-config.

  3. Sélectionnez le noeud Service de transaction.

    La page Service de transaction apparaît.

  4. Dans le champ Temporisation de transaction, saisissez le nombre de secondes devant s'écouler avant la temporisation de la transaction.

    La valeur par défaut est 0. Cette valeur désactive la temporisation des transactions.

  5. Cliquez sur Enregistrer.

  6. Redémarrez GlassFish Server.
