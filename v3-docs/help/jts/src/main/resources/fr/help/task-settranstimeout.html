Définition du délai d’attente d’une transaction

Par défaut, le serveur n’applique aucun délai d’expiration aux transactions. Le serveur attend indéfiniment la réalisation de la transaction. Si vous avez défini un délai d’expiration pour les transactions et qu’une transaction n’est pas terminée dans le délai configuré, celle-ci est annulée par GlassFish Server.

  1. Dans l’arborescence de navigation, développez le nœud Configuration.
  2. Sélectionnez l’instance à configurer :
    • Pour configurer une instance en particulier, sélectionnez le nœud de configuration de l’instance. Par exemple, pour l’instance par défaut, server, sélectionnez le nœud server-config.
    • Pour configurer les réglages par défaut pour toutes les instances, sélectionnez le nœud default-config.
  3. Sélectionnez le nœud Service de transaction.

    La page Service de transaction s’affiche.

  4. Dans le champ Délai d’attente de transaction, saisissez le nombre de secondes devant s’écouler avant expiration de la transaction.

    La valeur par défaut est 0. ce qui désactive le délai d’attente des transactions.

  5. Cliquez sur Enregistrer.
  6. Redémarrez GlassFish Server.
