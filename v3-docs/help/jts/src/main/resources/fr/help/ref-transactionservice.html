Service de transaction

La page Service de transaction vous permet de configurer la récupération de transaction, les délais d’attente et la journalisation.

La page Service de transaction contient les options suivantes :

Charger les valeurs par défaut

Bouton permettant de restaurer les valeurs par défaut de tous les paramètres.

Nom de configuration

Le nom de la configuration à laquelle les paramètres de cette page s'appliquent. Ce champ est en lecture seule.

Au redémarrage

Si la case Activé est cochée, le GlassFish Server essaie de récupérer les transactions incomplètes au redémarrage du serveur. Cette option est désactivée par défaut.

Délai d’attente de transaction

Durée d’attente, en secondes, du serveur avant d’annuler une transaction qui ne s’est pas terminée. La valeur par défaut est zéro qui signifie que le serveur attend la fin d’une transaction pendant une durée indéterminée.

Délai d’attente d’essai

Durée, en secondes, pendant laquelle le GlassFish Server essaie de se connecter à un serveur inaccessible. La valeur par défaut est de 600 (10 minutes).

Emplacement du journal des transactions

Répertoire où sont conservés les journaux du serveur. Les journaux de transaction sont conservés dans le sous-répertoire tx du répertoire spécifié par ce champ. La valeur par défaut est le répertoire spécifié dans le champ Racine journal de la page Attributs de domaine, qui se trouve dans l’onglet Avancé du nœud GlassFish Server.

Décision heuristique

Indique si les transactions qui impliquent des serveurs inaccessibles doivent être validées ou annulées. La valeur par défaut est Annulation. Valider les transactions indéterminées risque de compromettre l’intégrité des données dans votre application.

Intervalle de point clé

Nombre de transactions effectuées entre des opérations de point clé qui compriment le fichier journal de transactions. La valeur par défaut est 65 536.

Autres propriétés du service de transaction.
