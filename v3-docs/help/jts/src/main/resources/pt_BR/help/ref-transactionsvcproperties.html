Propriedades específicas do serviço de transação

As propriedades a seguir estão disponíveis para configuração do serviço de transação.

oracle-xa-recovery-workaround

Se definido como verdadeiro, a solução alternativa Oracle XA Resource será utilizada na recuperação da transação. O valor padrão é verdadeiro.

disable-distributed-transaction-logging

Se definido como verdadeiro, desativa o log de transação, o que pode melhorar o desempenho. Se a caixa de seleção Habilitado ao reiniciar estiver marcada, essa propriedade é ignorada. O valor padrão é falso.

xaresource-txn-timeout

Altera o XAResource XAResource. Em alguns casos, o XAResource pode anular a transação, portanto, é aconselhável alterá-lo. O valor padrão é específico do XAResource utilizado.

pending-txn-cleanup-interval

Especifica o intervalo, em segundos, em que um segmento assíncrono verifica se há transações pendentes e as conclui. Se essa propriedade não for especificada, não haverá nenhum padrão a ser aplicado. Se a propriedade estiver presente, mas não incluir nenhum valor, o valor padrão será 60.

use-last-agent-optimization

Se definido como verdadeiro, ativa a otimização do último agente, o que melhora o resultado das transações. Se um recurso não-XA for utilizado com recursos XA na mesma transação, o recurso não-XA será o último agente. O valor padrão é verdadeiro.

delegated-recovery

Se definido como verdadeiro, a recuperação delegada em todo o cluster será ativada. O valor padrão é falso.

wait-time-before-recovery-insec

Especifica o tempo de espera, em segundos, após o qual uma instância iniciará a recuperação de uma instância inativa.

db-logging-resource

Especifica o nome JNDI do recurso do JDBC para o banco de dados em que as transações estão registradas em log. Para obter mais informações, consulte o Capítulo 15, Using the Transaction Service, no Oracle GlassFish Server 3.1 Application Development Guide. Não há valor padrão.

xa-servername

Especifica o nome de host utilizado pelo serviço de transação para identificar as transações gerenciadas pelo Servidor GlassFish instalado. Isso pode, às vezes, ser útil para a recuperação de transações do arquivo de log criado em outro host com Servidor GlassFish em execução.
