Cómo establecer el intervalo de puntos clave

Las operaciones de puntos clave comprimen el archivo de registro de transacciones. El intervalo de puntos clave es el número de transacciones entre las operaciones de puntos clave en el registro. Las operaciones de puntos clave pueden reducir el tamaño de los archivos de registro de transacciones. Un número elevado de intervalos de puntos clave puede tener como resultado archivos de registro de transacciones mayores pero menos operaciones de puntos clave y potencialmente un rendimiento mejor. Un intervalo de puntos clave más pequeño (por ejemplo, 256) originará archivos de registro más pequeños, pero se verá afectado el rendimiento debido a la mayor frecuencia de operaciones de puntos clave.

  1. En el árbol de navegación, amplíe el nodo Configuración.
  2. Seleccione la instancia que va a configurar:
    • Para configurar una instancia determinada, seleccione el nodo de configuración de la instancia. Por ejemplo para la instancia predeterminada server, seleccione el nodo server-config.
    • Para configurar los ajustes predeterminados para todas las instancias, seleccione el nodo default-config.
  3. Seleccione el nodo Servicio de transacciones.

    Se abrirá la página Servicio de transacciones.

  4. En el campo del intervalo de puntos clave, escriba el número de transacciones entre las operaciones de puntos clave.

    El valor predeterminado es 65536.

  5. Haga clic en Guardar.
