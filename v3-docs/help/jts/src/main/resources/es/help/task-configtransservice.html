Cómo configurar el servicio de transacciones

  1. En el árbol de navegación, amplíe el nodo Configuración.
  2. Seleccione la instancia que va a configurar:
    • Para configurar una instancia determinada, seleccione el nodo de configuración de la instancia. Por ejemplo para la instancia predeterminada server, seleccione el nodo server-config.
    • Para configurar los ajustes predeterminados para todas las instancias, seleccione el nodo default-config.
  3. Seleccione el nodo Servicio de transacciones.

    Se abrirá la página Servicio de transacciones.

  4. En la página Servicio de transacciones, seleccione la casilla de verificación Habilitado al reiniciar para habilitar la recuperación de transacciones incompletas cuando se reinicie el servidor.

    Esta opción se deshabilita por defecto.

  5. En el campo Tiempo de espera de transacción, escriba el número de segundos que el servidor esperará antes deshacer una transacción que no se ha completado.

    El valor 0 deshabilita los tiempos de espera de transacciones. El valor predeterminado es cero, lo que supone que el servidor espera indefinidamente para que se complete la transacción.

  6. En el campo Tiempo de espera de reintento, introduzca el número de segundos que Servidor GlassFish intenta conectarse a un servidor inaccesible.

    El valor predeterminado es 600 (10 minutos).

  7. En el campo Ubicación del registro de transacción, escriba el directorio donde se guardan los registros del servidor.

    Los registros de transacciones se guardan en el subdirectorio tx del directorio especificado en el campo. El valor predeterminado es el directorio especificado en el campo Raíz de registro de la página Atributos de dominio, que está ubicada en la ficha Avanzado del nodo Servidor GlassFish.

  8. En la lista desplegable Decisión heurística, seleccione la política que debe seguirse si el resultado de una transacción distribuida no se puede determinar en caso de que otros participantes no estén disponibles.

    A menos que exista una buena razón para seleccionar Confirmar, deje la opción Decisión heurística en Anulación (el modo predeterminado). Enviar transacciones indeterminadas puede poner en peligro la integridad de los datos de su aplicación.

  9. En el campo del intervalo de puntos clave, escriba el número de transacciones entre las operaciones de puntos clave.

    Las operaciones de puntos clave comprimen el archivo de registro de transacciones. El valor predeterminado es 65536.

  10. (Opcional) En la sección Propiedades adicionales, especifique las propiedades adicionales.

    Para obtener una descripción de las propiedades disponibles, consulte Propiedades específicas del servicio de transacción.

    • Para añadir una propiedad, haga clic en el botón Agregar propiedad. En la fila vacía escriba el nombre de la propiedad en el campo Nombre y el valor de la propiedad en el campo Valor.
    • Para modificar una propiedad, edite el campo Valor de la propiedad.
    • Para eliminar una propiedad, marque la casilla de verificación situada a la izquierda del campo Nombre de la propiedad que quiera eliminar y, a continuación, haga clic en el botón Eliminar propiedades.
  11. Haga clic en Guardar.

    Consejo - Para volver a los valores de configuración predeterminados, haga clic en el botón Cargar predeterminados y, a continuación, en Guardar.

  12. Reinicie Servidor GlassFish.
