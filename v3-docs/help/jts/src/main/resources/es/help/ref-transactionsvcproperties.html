Propiedades específicas del servicio de transacción

Las siguientes propiedades adicionales están disponibles para configurar el servicio de transacción.

oracle-xa-recovery-workaround

Si se establece en true, se utiliza la solución alternativa del recurso Oracle XA en la recuperación de transacción. El valor predeterminado es verdadero.

disable-distributed-transaction-logging

Si se establece como true, se deshabilita el registro de transacción, que puede mejorar el rendimiento. Si la casilla de verificación Habilitado al reiniciar está seleccionada, esta propiedad se ignora. El valor predeterminado es falso.

xaresource-txn-timeout

Cambia el tiempo de espera de XAResource . En algunos casos, el tiempo de espera predeterminado de XAResource puede provocar que se anulen las transacciones, por ello se recomienda cambiarlo. El valor predeterminado es específico para el XAResource empleado.

pending-txn-cleanup-interval

Especifica el intervalo en segundos en el que un subproceso asincrónico comprueba las transacciones pendientes y las completa. Si no se especifica esta propiedad, no existe ningún valor predeterminado. Si esta propiedad está presente pero no tiene ningún valor, el valor predeterminado es 60.

use-last-agent-optimization

Si se establece como true, se habilita el último agente de optimización, que mejora el rendimiento de las transacciones. Si se utiliza uno de los recursos no-XA junto con recursos XA en la misma transacción, el recurso no-XA es el último agente. El valor predeterminado es verdadero.

delegated-recovery

Si se establece como true, se habilita la recuperación delegada de ancho de clúster. El valor predeterminado es falso.

wait-time-before-recovery-insec

Especifica el tiempo de espera, en segundos, después del cual una instancia inicia la recuperación de una instancia inactiva.

db-logging-resource

Especifica el nombre JNDI del recurso JDBC para la base de datos en la que se registran las transacciones. Para obtener más información, consulte el Capítulo 14, Using the Transaction Service de Oracle GlassFish Server 3.1 Application Development Guide. No hay ningún valor predeterminado.

xa-servername

Especifica el nombre de host que utiliza el servicio de transacción para identificar las transacciones administradas por Servidor GlassFish instalado. A veces esto puede ser útil para recuperar transacciones del archivo de registro que se había creado en un host diferente que ejecute Servidor GlassFish.
