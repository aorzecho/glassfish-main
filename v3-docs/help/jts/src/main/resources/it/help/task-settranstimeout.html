Impostare un valore di timeout per le transazioni

Per impostazione predefinita nel server non è previsto alcun timeout per le transazioni. Ciò significa che non è definito alcun limite di attesa del server per il completamento delle transazioni. Se si imposta un valore di timeout per le transazioni e una transazione non viene completata nell'intervallo di tempo configurato, viene eseguito il rollback della transazione.

  1. Nell'albero di navigazione espandere il nodo Configurazione.

  2. Selezionare l'istanza da configurare:

    • Per configurare un'istanza specifica, selezionare il nodo di configurazione dell'istanza. Ad esempio, per l'istanza predefinita server, selezionare il nodo server-config.

    • Per configurare le impostazioni predefinite per tutte le istanze, selezionare il nodo default-config.

  3. Selezionare il nodo Servizio transazioni.

    Viene visualizzata la pagina Servizio transazioni.

  4. Nel campo Timeout transazione, digitare il numero di secondi trascorso il quale si verifica il timeout di una transazione.

    Il valore predefinito è 0. Con questo valore, il timeout delle transazioni viene disabilitato.

  5. Fare clic su Salva.

  6. Riavviare GlassFish Server.
