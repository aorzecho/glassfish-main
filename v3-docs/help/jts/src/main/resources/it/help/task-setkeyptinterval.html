Impostare l'intervallo keypoint

Le operazioni keypoint determinano la compressione del file di log delle transazioni. L'intervallo keypoint rappresenta il numero di transazioni tra un'operazione keypoint e l'altra nel log. Le operazioni keypoint possono ridurre le dimensioni dei file di log delle transazioni. Con un valore di intervallo keypoint più elevato si ottengono file di log delle transazioni di dimensioni maggiori con un numero minore di operazioni keypoint e prestazioni potenzialmente migliori. Con un valore di intervallo keypoint più basso, ad esempio 256, si ottengono file di log di dimensioni minori con prestazioni di livello leggermente inferiore a causa della maggiore frequenza di operazioni keypoint.

  1. Nella struttura ad albero, espandere il nodo Configuration.
  2. Selezionare l'istanza da configurare:
    • Per configurare una particolare istanza, selezionare il nodo config dell'istanza. Ad esempio, per l'istanza predefinita server, selezionare il nodo server-config.
    • Per configurare le impostazioni predefinite per tutte le istanze, selezionare il nodo default-config.
  3. Selezionare il nodo Transaction Service.

    Viene visualizzata la pagina Transaction Service.

  4. Nel campo Keypoint Interval, digitare il numero di transazioni tra un'operazione keypoint e l'altra.

    Il valore predefinito è 65.536.

  5. Fare clic su Save.
