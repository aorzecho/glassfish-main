Impostare l'intervallo dei punti chiave

Le operazioni punti chiave determinano la compressione del file di log delle transazioni. L'intervallo punti chiave rappresenta il numero di transazioni tra un'operazione punto chiave e l'altra nel log. Le operazioni punti chiave possono ridurre le dimensioni dei file di log delle transazioni. Con un valore di intervallo punti chiave più elevato si ottengono file di log delle transazioni di dimensioni maggiori con un numero minore di operazioni punti chiave e prestazioni potenzialmente migliori. Con un valore di intervallo punti chiave più basso, ad esempio 256, si ottengono file di log di dimensioni minori con prestazioni di livello leggermente inferiore a causa della maggiore frequenza di operazioni punti chiave.

  1. Nell'albero di navigazione espandere il nodo Configurazione.

  2. Selezionare l'istanza da configurare:

    • Per configurare un'istanza specifica, selezionare il nodo di configurazione dell'istanza. Ad esempio, per l'istanza predefinita server, selezionare il nodo server-config.

    • Per configurare le impostazioni predefinite per tutte le istanze, selezionare il nodo default-config.

  3. Selezionare il nodo Servizio transazioni.

    Viene visualizzata la pagina Servizio transazioni.

  4. Nel campo Intervallo punti chiave, digitare il numero di transazioni tra un'operazione punto chiave e l'altra.

    Il valore predefinito è 65.536.

  5. Fare clic su Salva.
