Configurare Transaction Service

  1. Nella struttura ad albero, espandere il nodo Configuration.
  2. Selezionare l'istanza da configurare:
    • Per configurare una particolare istanza, selezionare il nodo config dell'istanza. Ad esempio, per l'istanza predefinita server, selezionare il nodo server-config.
    • Per configurare le impostazioni predefinite per tutte le istanze, selezionare il nodo default-config.
  3. Selezionare il nodo Transaction Service.

    Viene visualizzata la pagina Transaction Service.

  4. Nella pagina Transaction Service, selezionare la casella di controllo On Restart Enabled per abilitare il ripristino delle transazioni incomplete al riavvio del server.

    Questa opzione è disabilitata per impostazione predefinita.

  5. Nel campo Transaction Timeout, immettere il numero di secondi di attesa trascorsi i quali il server esegue il rollback di una transazione incompleta.

    Con un valore pari a 0, il timeout delle transazioni viene disabilitato. Il valore predefinito è 0, che indica un tempo di attesa illimitato del server per il completamento di una transazione.

  6. Nel campo Retry Timeout, immettere il numero di secondi durante i quali GlassFish Server tenterà di eseguire la connessione a un server irraggiungibile.

    Il valore predefinito è 600 (10 minuti).

  7. Nel campo Transaction Log Location, digitare la directory di memorizzazione dei log del server.

    I log delle transazioni vengono memorizzati nella sottodirectory tx della directory specificata in questo campo. Il valore predefinito è la directory specificata nel campo Log Root disponibile nella pagina Domain Attributes all'interno della scheda Advanced del nodo GlassFish Server.

  8. Dall'elenco a discesa Heuristic Decision, selezionare il criterio a cui attenersi nel caso in cui non sia possibile determinare il risultato di una transazione distribuita a causa dell'irraggiungibilità di altri partecipanti.

    È consigliabile lasciare Heuristic Decision impostato su Rollback (impostazione predefinita) a meno che non sia effettivamente utile selezionare Commit. L'esecuzione del commit di un numero indeterminato di transazioni potrebbe avere ripercussioni sull'integrità dei dati dell'applicazione.

  9. Nel campo Keypoint Interval, digitare il numero di transazioni tra un'operazione keypoint e l'altra.

    Le operazioni keypoint determinano la compressione del file di log delle transazioni. Il valore predefinito è 65.536.

  10. (Opzionale) Nella sezione Additional Properties, specificare le proprietà aggiuntive.

    Per una descrizione delle proprietà disponibili, vedere Proprietà specifiche di Transaction Service.

    • Per aggiungere una proprietà, fare clic sul pulsante Add Property. Nella riga vuota visualizzata, digitare il nome della proprietà nel campo Name e il valore della proprietà nel campo Value.
    • Per modificare una proprietà, è necessario cambiare il valore del campo Value della proprietà.
    • Per eliminare una proprietà, selezionare la casella di controllo a sinistra del campo Name relativo alla proprietà da eliminare, quindi fare clic sul pulsante Delete Properties.
  11. Fare clic su Save.

    Suggerimento - Per tornare alle impostazioni predefinite, fare clic sul pulsante Load Defaults, quindi su Save.

  12. Riavviare GlassFish Server.
