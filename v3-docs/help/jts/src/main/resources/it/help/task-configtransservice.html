Configurare il servizio transazioni

  1. Nell'albero di navigazione espandere il nodo Configurazione.

  2. Selezionare l'istanza da configurare:

    • Per configurare un'istanza specifica, selezionare il nodo di configurazione dell'istanza. Ad esempio, per l'istanza predefinita server, selezionare il nodo server-config.

    • Per configurare le impostazioni predefinite per tutte le istanze, selezionare il nodo default-config.

  3. Selezionare il nodo Servizio transazioni.

    Viene visualizzata la pagina Servizio transazioni.

  4. Nella pagina Servizio transazioni, selezionare la casella di controllo Al riavvio abilitato per abilitare il recupero delle transazioni incomplete al riavvio del server.

    Questa opzione è disabilitata per impostazione predefinita.

  5. Nel campo Timeout transazione, immettere il numero di secondi di attesa trascorsi i quali il server esegue il rollback di una transazione incompleta.

    Con un valore pari a 0, il timeout delle transazioni viene disabilitato. Il valore predefinito è 0, che indica un tempo di attesa illimitato del server per il completamento di una transazione.

  6. Nel campo Timeout tentativi, immettere il numero di secondi durante i quali GlassFish Server tenterà di eseguire la connessione a un server irraggiungibile.

    Il valore predefinito è 600 (10 minuti).

  7. Nel campo Posizione log delle transazioni, digitare la directory di memorizzazione dei log del server.

    I log delle transazioni vengono memorizzati nella sottodirectory tx della directory specificata in questo campo. Il valore predefinito è la directory specificata nel campo Radice log disponibile nella pagina Attributi dominio all'interno della scheda Avanzate del nodo GlassFish Server.

  8. Dall'elenco a discesa Decisione euristica, selezionare il criterio a cui attenersi nel caso in cui non sia possibile determinare il risultato di una transazione distribuita a causa dell'irraggiungibilità di altri partecipanti.

    È consigliabile lasciare Decisione euristica impostato su Rollback (impostazione predefinita) a meno che non sia effettivamente utile selezionare Commit. L'esecuzione del commit di un numero indeterminato di transazioni potrebbe avere ripercussioni sull'integrità dei dati dell'applicazione.

  9. Nel campo Intervallo punti chiave, digitare il numero di transazioni tra un'operazione punto chiave e l'altra.

    Le operazioni punti chiave determinano la compressione del file di log delle transazioni. Il valore predefinito è 65.536.

  10. Nella sezione Proprietà aggiuntive, specificare le proprietà aggiuntive.

    Per una descrizione delle proprietà disponibili, vedere Proprietà specifiche del servizio transazioni.

    • Per aggiungere una proprietà, fare clic sul pulsante Aggiungi proprietà. Nella riga vuota visualizzata, digitare il nome della proprietà nel campo Nome e il valore della proprietà nel campo Valore.

    • Per modificare una proprietà, è necessario modificare il valore del campo Valore della proprietà.

    • Per eliminare una proprietà, selezionare la casella di controllo a sinistra del campo Nome relativo alla proprietà da eliminare, quindi fare clic sul pulsante Elimina proprietà.

  11. Fare clic su Salva.

    Suggerimento:

    Per tornare alle impostazioni predefinite, fare clic sul pulsante Carica impostazioni predefinite, quindi su Salva.

  12. Riavviare GlassFish Server.
