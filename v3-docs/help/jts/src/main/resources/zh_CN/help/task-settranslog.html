设置事务日志的位置

为了保持被调用资源的数据完整性，同时为了能够从故障中恢复，事务日志将记录有关每个事务的信息。事务日志保存在“事务日志位置”字段指定的目录的 tx 子目录中。这些日志不是人可以阅读的。

  1. 在导航树中，展开“配置”节点。
  2. 选择要配置的实例：
    • 要配置特定的实例，请选择该实例的配置节点。例如，对于默认实例 server，请选择 server-config 节点。
    • 要配置所有实例的默认设置，请选择 default-config 节点。
  3. 选择“事务服务”节点。

    此时将打开“事务服务”页面。

  4. 在“事务日志位置”字段中，键入事务日志的位置。

    创建 tx 子目录，事务日志将保存在该目录下。

  5. 单击“保存”。
