配置事务服务

  1. 在导航树中，展开“配置”节点。
  2. 选择要配置的实例：
    • 要配置特定的实例，请选择该实例的配置节点。例如，对于默认实例 server，请选择 server-config 节点。
    • 要配置所有实例的默认设置，请选择 default-config 节点。
  3. 选择“事务服务”节点。

    此时将打开“事务服务”页面。

  4. 在“事务服务”页面上，选中“启用重新启动时”复选框，以便在服务器重新启动时恢复未完成的事务。

    默认情况下禁用此选项。

  5. 在“事务超时”字段中，键入回滚未完成事务之前，服务器等待的秒数。

    如果值为 0，则禁用事务超时。默认值为 0，表示服务器无限期地等待事务完成。

  6. 在“重试超时”字段中，键入 GlassFish Server 尝试连接到不可访问服务器的秒数。

    默认值为 600（10 分钟）。

  7. 在“事务日志位置”字段中，键入保存服务器日志的目录。

    事务日志保存在此字段所指定目录的 tx 子目录中。默认值为“域属性”页面（位于 "GlassFish Server" 节点的“高级”选项卡下）的“日志根”字段指定的目录。

  8. 从“启发式决策”下拉式列表中，选择因其他参与者不可访问而无法确定分布式事务的结果时要遵循的策略。

    除非有充分理由需要选择“提交”，否则请将“启发式决策”设置保留为“回滚”（默认值）。提交不确定的事务会破坏应用程序的数据完整性。

  9. 在“关键点间隔”字段中，键入关键点操作之间的事务数量。

    关键点操作可以压缩事务日志文件。默认值为 65,536。

  10. （可选的）在“其他属性”部分中，指定其他属性。

    有关可用属性的说明，请参见特定于事务服务的属性。

    • 要添加属性，请单击“添加属性”按钮。在显示的空白行中，在“名称”字段中键入属性名称，在“值”字段中键入属性值。
    • 要修改属性，请编辑该属性的“值”字段。
    • 要删除属性，请选中位于您要删除属性的“名称”字段左边的复选框，然后单击“删除属性”按钮。
  11. 单击“保存”。

    提示 - 要返回到默认设置，请单击“装入默认值”按钮，然后单击“保存”。

  12. 重新启动 GlassFish Server。
