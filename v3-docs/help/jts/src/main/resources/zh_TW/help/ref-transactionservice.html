作業事件服務

使用 [作業事件服務] 頁面可配置作業事件回復、逾時與記錄。

[作業事件服務] 頁面包含下列選項。

載入預設值

用於將具有預設值的設定復原成其預設值的按鈕。沒有預設值的設定保持不變。

配置名稱

此頁面上的設定要套用到的配置名稱。此為唯讀欄位。

在重新啟動時

若選取 [已啟用] 核取方塊，當伺服器重新啟動時，GlassFish Server 會嘗試回復未完成的作業事件。此選項預設為停用。

作業事件逾時

伺服器在回復未完成之作業事件前要等待的秒數。預設值為 0，表示伺服器會一直等待作業事件完成。

重試逾時

GlassFish Server 嘗試連線至無法連線的伺服器的秒數。預設值為 600 (10 分鐘)。

作業事件記錄位置

保存伺服器記錄的目錄。作業事件記錄會保存在此欄位指定之目錄的 tx 子目錄中。預設值為 [網域屬性] 頁面上的 [記錄根目錄] 欄位所指定的目錄，頁面位於 [GlassFish Server] 節點的 [進階] 標籤之下。

啟發式決策

決定要確定還是回復與無法連接伺服器有關的作業事件。預設值為 [回復]。確定不確定的作業事件會破壞應用程式的資料完整性。

關鍵點間隔

壓縮作業事件記錄檔之關鍵點作業間的作業事件數。預設值為 65,536。
