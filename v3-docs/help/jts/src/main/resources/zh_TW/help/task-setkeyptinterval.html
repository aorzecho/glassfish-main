設定關鍵點間隔

關鍵點作業會壓縮作業事件記錄檔。關鍵點間隔是指記錄上關鍵點作業之間的作業事件數目。關鍵點作業可以減小作業事件記錄檔的大小。關鍵點間隔數越大，作業事件記錄檔也越大，但關鍵點作業較少，效能可能更佳。關鍵點間隔越小 (例如，256)，記錄檔也越小，而同時由於關鍵點作業較為頻繁，效能會略微降低。

  1. 在瀏覽樹狀結構中，展開 「組態」 節點。

  2. 選取要組態的實例：

    • 若要組態特定的實例，請選取該實例的組態節點。例如，對於預設實例 server，請選取 「server-config」 節點。

    • 若要組態所有實例的預設設定，請選取 「default-config」 節點。

  3. 選取 「作業事件服務」 節點。

    「作業事件服務」 頁面會隨即開啟。

  4. 在 「關鍵點間隔」 欄位中，鍵入關鍵點作業之間的作業事件數目。

    預設值為 65,536。

  5. 按一下 「儲存」。
