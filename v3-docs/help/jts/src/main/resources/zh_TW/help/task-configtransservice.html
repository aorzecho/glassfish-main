配置作業事件服務

  1. 在瀏覽樹狀結構中，展開 [配置] 節點。
  2. 選取要配置的實例：
    • 若要配置特定的實例，請選取該實例的配置節點。例如，對於預設實例 server，請選取 [server-config] 節點。
    • 若要配置所有實例的預設設定，請選取 [default-config] 節點。
  3. 選取 [作業事件服務] 節點。

    [作業事件服務] 頁面會隨即開啟。

  4. 在 [作業事件服務] 頁面上，選取 [啟用在重新啟動時] 核取方塊，以在伺服器重新啟動時，回復未完成的作業事件。

    此選項預設為停用。

  5. 在 [作業事件逾時] 欄位中，鍵入伺服器在回復未完成的作業事件前等候的秒數。

    0 值會停用作業事件逾時。預設值為 0，表示伺服器會一直等待作業事件完成。

  6. 在 [重試逾時] 欄位中，鍵入 GlassFish Server 嘗試連線至無法連線之伺服器的秒數。

    預設值為 600 (10 分鐘)。

  7. 在 [作業事件記錄位置] 欄位中，鍵入保存伺服器記錄的目錄。

    作業事件記錄會保存在此欄位指定之目錄的 tx 子目錄中。預設值為 [網域屬性] 頁面上的 [記錄根目錄] 欄位所指定的目錄，頁面位於 [GlassFish Server] 節點的 [進階] 標籤之下。

  8. 從 [啟發式決策] 下拉式清單中，選取由於無法連線其他參加者而無法確定分散式作業事件的結果時，所要遵循的策略。

    除非有很好的理由選取 [確定]，否則請將 [啟發式決策] 保留設定為 [回復] (預設值)。確定不確定的作業事件會破壞應用程式的資料完整性。

  9. 在 [關鍵點間隔] 欄位中，鍵入關鍵點作業之間的作業事件數目。

    關鍵點作業會壓縮作業事件記錄檔。預設值為 65,536。

  10. (可選擇) 在 [其他特性] 區段中，指定其他特性。

    如需可用特性的說明，請參閱作業事件服務特有的特性。

    • 若要增加特性，請按一下 [增加特性] 按鈕。在出現的空白列中，於 [名稱] 欄位中鍵入特性名稱，然後在 [值] 欄位中鍵入特性值。
    • 若要修改特性，請編輯特性的 [值] 欄位。
    • 若要刪除特性，請選取您要刪除之特性 [名稱] 欄位左側的核取方塊，然後按一下 [刪除特性] 按鈕。
  11. 按一下 [儲存]。

    提示 - 若要回到預設設定，請按一下 [載入預設值] 按鈕，然後按一下 [儲存]。

  12. 重新啟動 GlassFish Server。
