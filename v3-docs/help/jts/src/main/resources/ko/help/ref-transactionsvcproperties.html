트랜잭션 서비스와 관련된 등록 정보

트랜잭션 서비스 구성의 경우 다음과 같은 추가 등록 정보를 사용할 수 있습니다.

oracle-xa-recovery-workaround

true로 설정하면 트랜잭션 복구에 Oracle XA 자원 해결 방법이 사용됩니다. 기본값은 true입니다.

disable-distributed-transaction-logging

true로 설정하면 트랜잭션 로깅을 비활성화하여 성능을 향상시킬 수 있습니다. 다시 시작 시 사용 가능 확인란이 선택되어 있으면 이 등록 정보는 무시됩니다. 기본값은 false입니다.

xaresource-txn-timeout

XAResource 시간 초과를 변경합니다. 경우에 따라 XAResource 기본 시간 초과 값 때문에 트랜잭션이 중단될 수 있으므로 값을 변경하는 것이 좋습니다. 기본값은 사용되는 XAResource에 따라 결정됩니다.

pending-txn-cleanup-interval

비동기 스레드에서 보류 중인 트랜잭션을 확인하고 완료하는 간격(초)을 지정합니다. 이 등록 정보를 지정하지 않을 경우 사용되는 기본값은 없습니다. 이 등록 정보가 있지만 값이 없는 경우의 기본값은 60입이다.

use-last-agent-optimization

true로 설정하면 마지막 에이전트 최적화를 활성화하여 트랜잭션 처리량을 향상시킵니다. 같은 트랜잭션에서 XA 자원과 함께 XA가 아닌 자원을 사용한 경우에는 XA가 아닌 자원이 마지막 에이전트입니다. 기본값은 true입니다.

delegated-recovery

true로 설정하면 클러스터 전체의 위임 복구가 활성화됩니다. 기본값은 false입니다.

wait-time-before-recovery-insec

사용 불능 상태인 인스턴스의 복구를 시작할 때까지 대기하는 시간(초)을 지정합니다.

db-logging-resource

트랜잭션을 로깅하는 데이터베이스의 JDBC 자원에 대해 JNDI 이름을 지정합니다. 자세한 내용은 Oracle GlassFish Server 3.1 Application Development Guide의 14 장, Using the Transaction Service를 참조하십시오. 기본값은 없습니다.

xa-servername

설치된 GlassFish Server에서 관리하는 트랜잭션을 식별할 때 트랜잭션 서비스에서 사용하는 호스트 이름을 지정합니다. 이는 GlassFish Server를 실행하는 다른 호스트에서 만든 로그 파일로부터 트랜잭션을 복구할 경우에 유용할 수 있습니다.
