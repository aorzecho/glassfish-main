完成 DCOM 節點的更新

此作業是編輯節點DCOM 節點的接續說明。

  1. 若要強制在網域管理伺服器 (DAS) 的組態中建立節點 (即使驗證節點參數失敗)，請啟用 「強制」 選項。

  2. 如果您需要 GlassFish Server 在更新節點前先測試透過 DCOM 的節點主機連線，請執行下列步驟：

    1. 啟用「驗證 DCOM」選項。

    2. 在「遠端測試目錄」欄位中，鍵入遠端主機上 GlassFish Server 應該在其中建立檔案以測試連線的資料夾名稱。

  3. 在「Windows 網域」欄位中，鍵入包含「Windows 使用者名稱」選項所指定之使用者的 Windows 網域名稱。

    如果您將節點類型變更為 DCOM，並讓此欄位保留空白，則會將「Windows 網域」選項設定為正在執行管理主控台的主機名稱。

  4. 在「Windows 使用者名稱」欄位中，鍵入要在此節點主機上執行透過 DCOM 連線至主機之程序的使用者名稱。

    如果您將節點類型從 CONFIG 變更為 DCOM，並讓此欄位保留空白，則會將「Windows 使用者名稱」選項設定為正在執行 DAS 程序的使用者。

  5. 指定登入此節點主機時驗證 Windows 使用者的方式。

    注意:

    GlassFish Server 也支援透過密碼檔案中的 AS_ADMIN_WINDOWSPASSWORD 項目進行密碼驗證。密碼檔案指定於 asadmin(1) 公用程式的 --paswordfile 選項中。

    1. 若要指定透過儲存在 DAS 組態中的密碼來進行認證，請執行以下步驟：

      警告:

      此密碼在 DAS 組態中是以純文字儲存。如需增強安全性，請使用密碼別名。

      1. 從「Windows 使用者驗證」下拉式清單中，選取「密碼」。

      2. 在「Windows 使用者密碼」欄位中，鍵入 SSH 使用者在登入此節點主機時將使用的密碼。

      3. 在「確認密碼」欄位中，再次鍵入密碼。

    2. 若要指定透過密碼別名來進行認證，請執行以下步驟：

      1. 從「Windows 使用者驗證」下拉式清單中，選取「密碼別名」。

      2. 從 「密碼別名」 下拉式清單中選取現有的別名。

  6. 按一下 「儲存」。
