建立節點

節點代表其上安裝 GlassFish Server 軟體的主機。GlassFish Server 實例所在的每部主機都必須存在對應的節點。

  1. 在瀏覽樹狀結構中，選取 [節點] 節點。

    此時將開啟 [節點] 頁面。

  2. 在 [節點] 頁面上，按一下 [新建]。

    此時將開啟 [新建節點] 頁面。

  3. 在 [名稱] 欄位中，鍵入您選擇的節點名稱。
  4. 從 [類型] 下拉式清單中，選取要建立的節點類型。
    • 若要建立可支援透過安全 shell (SSH) 進行通訊的節點，請選取 [SSH]。
    • 若要建立不支援遠端通訊的節點，請選取 [CONFIG]。
  5. 在 [節點主機] 欄位中，鍵入節點所代表的主機名稱。

    此欄位是否為必要欄位，取決於節點類型：

    • 如果節點類型為 CONFIG，可能會省略主機名稱。可在建立位於節點上的實例時決定主機名稱。

    • 如果節點類型為 SSH，則必須指定主機名稱。否則，將發生錯誤。

  6. 在 [節點目錄] 欄位中，鍵入要包含在節點上所建立的 GlassFish Server 實例之目錄的路徑。

    如果節點類型為 CONFIG，請讓此欄位保留空白。可在建立位於節點上的實例時決定節點目錄。

    如果您鍵入相對路徑，則該路徑會相對於 as-install 目錄，其中 as-install 是主機上 GlassFish Server 軟體的基底安裝目錄。

  7. 在 [安裝目錄] 欄位中，鍵入主機上 GlassFish Server 軟體的基底安裝目錄之父系目錄的完整路徑，例如 /export/glassfish3/。

    如果節點類型為 CONFIG，請讓此欄位保留空白。可在建立位於節點上的實例時決定安裝目錄。

  8. 若要強制在 Domain Administration Server (DAS) 的配置中建立節點 (即使驗證節點參數失敗)，請啟用 [強制] 選項。
  9. 在 [SSH 連接埠] 欄位中，鍵入用於與此節點主機連線之 SSH 的連接埠。
  10. 在 [SSH 使用者名稱] 欄位中，鍵入要在此節點之主機上，執行透過 SSH 連線至主機之程序的使用者名稱。
  11. 指定在登入此節點之主機時如何認證 SSH 使用者。
    • 若要指定透過 SSH 私密金鑰檔案來進行認證，請執行以下步驟：
      1. 從 [SSH 使用者認證] 下拉式清單中選取 [金鑰檔案]。
      2. 在 [金鑰檔案] 欄位中，鍵入 [SSH 使用者名稱] 選項所指定使用者之 SSH 私密金鑰檔案的絕對路徑。

        備註 - GlassFish Server 也支援透過密碼檔案中的 AS_ADMIN_SSHPASSWORD 項目進行密碼認證。密碼檔案是在 asadmin(1M) 公用程式的 --paswordfile 選項中指定。

    • 若要指定透過儲存在 DAS 配置中的密碼來進行認證，請執行以下步驟：

      注意 - 此密碼在 DAS 配置中是以純文字儲存。如需增強安全性，請使用金鑰檔案或密碼別名

      1. 從 [SSH 使用者認證] 下拉式清單中選取 [密碼]。
      2. 在 [SSH 密碼] 欄位中，鍵入 SSH 使用者將在登入此節點之主機時使用的密碼。
      3. 在 [確認 SSH 密碼] 欄位中，再次鍵入密碼。
    • 若要指定透過密碼別名來進行認證，請執行以下步驟：
      1. 從 [SSH 使用者認證] 下拉式清單中選取 [密碼別名]。
      2. 從 [密碼別名] 下拉式清單中選取現有的別名。
  12. 按一下 [儲存]。
