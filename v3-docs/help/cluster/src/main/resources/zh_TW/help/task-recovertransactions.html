回復叢集 GlassFish Server 實例的作業事件

  1. 在瀏覽樹狀結構中，展開 「叢集」 節點。

  2. 在展開的 「叢集」 節點下，選取該叢集的節點。

    此時將開啟叢集的 「一般資訊」 頁面。

  3. 在 「一般資訊」 頁面上，按一下 「實例」 標籤。

    此時將開啟該叢集的 「叢集伺服器實例」 頁面。

  4. 選取需要回復其作業事件的實例。

    此時將開啟實例的 「一般資訊」 頁面。

  5. 在 「一般資訊」 頁面上，按一下 「回復作業事件」。

    「回復作業事件」 頁面會隨即開啟。

    此頁面的內容取決於實例是否正在執行中。

    • 如果實例在執行中，「回復作業事件」 頁面只包含文字「回復是由正在執行的伺服器自行執行」與 「確定」 按鈕。

    • 如果實例不在執行中，「回復作業事件」 頁面會包含其他選項。

  6. 從 「目標伺服器」 下拉式清單中，選取相同叢集內能夠回復目前實例之作業事件的其他實例。

  7. 在 「作業事件記錄目錄」 欄位中，鍵入儲存作業事件記錄的目錄。

    預設是位於網域目錄底下的 /logs/tx。作業事件記錄保存在 tx 中，它是叢集組態之作業事件服務 「作業事件記錄位置」 設定所指定目錄的子目錄。「作業事件記錄位置」 預設值為 「網域屬性」 頁面上的 「記錄根目錄」 欄位所指定的目錄，該頁面位於 「GlassFish Server」 節點的 「進階」 標籤之下。

  8. 按一下 「確定」。
