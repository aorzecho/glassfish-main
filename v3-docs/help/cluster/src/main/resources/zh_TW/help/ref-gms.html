編輯群組管理服務

使用 [群組管理服務] 頁面可變更 GMS 設定。

群組管理服務 (GMS) 是一種進程服務，可提供叢集監視、叢集成員身份和群組通訊服務。GMS 建立在群組成員身份探索和運作狀態監視協定的可配置堆疊之上。GMS 會執行下列功能：

  • 從叢集增加或移除一或多個成員實例，或者一或多個成員實例疑似故障或確認故障時，會通知已在 GlassFish Server 實例中註冊的模組。

  • 能夠在一組程序之間傳送和接收訊息。

[群組管理服務] 頁面包含下列選項。

載入預設值

用於將具有預設值的設定復原成其預設值的按鈕。沒有預設值的設定保持不變。

配置名稱

此頁面上的設定要套用到的配置名稱。此為唯讀欄位。

遺失的活動訊號數上限

懷疑伺服器實例發生故障之前的監視嘗試次數。預設值為 3。

活動訊號頻率

監視嘗試之間的時間間隔。預設值為 2000 毫秒。

群組探索逾時

GMS 等候探索其他叢集成員的時間量。預設值為 5000 毫秒。

故障驗證等待時間

將疑似故障標示為已驗證之前等候的時間量。預設值為 1500 毫秒。

其他特性

群組管理服務的其他特性表格。

GlassFish Server 未定義任何其他 GMS (群組管理服務) 特性。

對於每個特性，都會顯示以下資訊：

名稱

特性的名稱。

值

為所選取實例設定的特性值。

說明

提供有關特性之更多資訊的文字說明。

其他特性表也包含下列選項。

增加特性

增加特性的按鈕。按一下此按鈕即可在其他特性表中增加一列。

刪除特性

刪除一或多個所選取特性的按鈕。刪除的任何特性都會復原成其預設值或未定義 (如果未設定任何預設值)。
