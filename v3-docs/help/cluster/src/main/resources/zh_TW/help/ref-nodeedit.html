編輯節點

使用 「編輯節點」 頁面可更新 GlassFish Server 節點的組態資料。您也可以使用此頁面，將節點類型從 CONFIG 變更為 DCOM 或 SSH，以啟用節點的遠端通訊功能。

如果您將節點類型從 CONFIG 變更為 DCOM，則若下列任一欄位為空白，便會套用預設值：

  • Windows 網域

  • Windows 使用者

如果您將節點類型從 CONFIG 變更為 SSH，則下列任一欄位保留空白時，便會套用預設值：

  • SSH 連接埠

  • 使用者名稱

  • 金鑰檔案

「編輯節點」頁面所包含的選項取決於節點類型。

所有節點類型的選項

下列是所有節點類型的可用選項：

名稱

建立節點時指定給節點的名稱。本欄位是唯讀的。

類型

下拉式清單，可指定下列其中一種節點類型：

組態

節點不支援遠端通訊。如果您選取「CONFIG」，則不會顯示 DCOM 或 SSH 組態的任何選項。如果目前的節點類型是 SSH，而您選取「CONFIG」，則儲存變更時，會從節點組態中移除 DCOM 或 SSH 組態資料。

DCOM

節點支援透過分散式元件物件模型 (DCOM) 遠端協定的通訊。DCOM 協定只適用於 Windows 系統。

SSH

節點支援透過安全 shell (SSH) 進行通訊。

節點主機

更新節點之後，節點所代表的主機名稱。此欄位是否為必要欄位，取決於節點類型：

  • 如果節點類型為 CONFIG，可能會省略主機名稱。可在建立位於節點上的實例時決定主機名稱。

  • 如果節點類型為 DCOM 或 SSH，則必須指定主機名稱。否則，將發生錯誤。

節點目錄

要包含在節點上所建立的 GlassFish Server 實例之目錄的路徑。如果指定相對路徑，則該路徑會相對於 as-install 目錄，其中 as-install 是主機上 GlassFish Server 軟體的基底安裝目錄。

如果將此欄位保留爲空，將按照如下方式更新節點的組態：

  • 如果節點類型為 CONFIG，則不會為網域管理伺服器 (DAS) 組態中的節點指定任何實例目錄。可在建立位於節點上的實例時決定節點目錄。

  • 如果節點類型是 DCOM，則預設值為 as-install\nodes，其中 as-install 是主機上 GlassFish Server 軟體的基底安裝目錄。

  • 如果節點類型是 SSH，則節點目錄為 as-install/nodes，其中 as-install 是主機上 GlassFish Server 軟體的基底安裝目錄。

安裝目錄

主機上 GlassFish Server 軟體基底安裝目錄之父系目錄的完整路徑 (例如 /export/glassfish3/)。

如果將此欄位保留爲空，將按照如下方式更新節點的組態：

  • 如果節點類型為 CONFIG，則不會為 DAS 組態中的節點指定任何安裝目錄。可在建立位於節點上的實例時決定安裝目錄。

  • 如果節點類型為 DCOM 或 SSH，則 DAS 安裝目錄為 GlassFish Server 軟體之預設基底安裝目錄的父系目錄。只有當 GlassFish Server 安裝在所有主機上的相同位置中時，此設定才有用。

DCOM 節點的其他選項

只有在「類型」欄位設定為 DCOM 時，才能使用下列選項：

強制

如果選取 「已啟用」 選項，則即使驗證節點參數失敗，也會在 DAS 組態中建立節點。若要驗證節點的參數，DAS 必須能夠透過 DCOM 連絡節點的主機。

驗證 DCOM

如果選取「已啟用」選項，GlassFish Server 會先測試透過 DCOM 的節點主機連線，然後再更新節點。測試時，GlassFish Server 會將名稱為 delete_me.bat 的程序檔複製至主機。

如果您選取此選項，則會啟用「遠端測試目錄」欄位以讓您提供 GlassFish Server 複製程序檔的目標目錄。

遠端測試目錄

遠端主機上 GlassFish Server 應該在其中建立 delete_me.bat 檔案的資料夾。「Windows 使用者名稱」選項所指定的使用者必須具有此資料夾的寫入存取權。否則，將發生錯誤。預設值為 C:\。

Windows 網域

包含「Windows 使用者名稱」選項所指定之使用者的 Windows 網域名稱。預設值為執行管理主控台所在的主機名稱。

Windows 使用者名稱

在此節點之主機上的使用者，會執行透過 DCOM 連線至主機的程序。預設值為執行 DAS 程序的使用者。如果「節點主機」選項設定為 localhost-domain，便會忽略「Windows 使用者名稱」選項。

Windows 使用者驗證

指定登入此節點主機時如何驗證 Windows 使用者的下拉式清單。可能的認證方法如下：

密碼

Windows 使用者是透過儲存在 DAS 組態中的密碼來驗證。

警告:

此密碼在 DAS 組態中是以純文字儲存。如需增強安全性，請使用金鑰檔案或密碼別名。

如果您選取此選項，即會啟用「Windows 使用者密碼」欄位和「確認密碼」欄位，讓您可以提供密碼。

密碼別名

Windows 使用者會透過密碼別名來驗證。如果選取此選項，請從 「密碼別名」 下拉式清單中選取密碼別名。

Windows 使用者密碼

Windows 使用者登入此節點主機時將使用的密碼。

確認密碼

確認 Windows 使用者登入此節點主機時將使用的密碼。此欄位的內容必須與「Windows 使用者密碼」欄位的內容完全相符。

密碼別名

為此網域定義之密碼別名的下拉式清單。

SSH 節點的其他選項

只有在「類型」欄位設定為 SSH 時，才能使用下列選項：

強制

如果選取 「已啟用」 選項，則即使驗證節點參數失敗，也會在網域管理伺服器 (DAS) 組態中更新節點。若要驗證節點的參數，DAS 必須能夠透過 SSH 連絡節點的主機。

SSH 連接埠

用於與此節點主機連線之 SSH 的連接埠。

如果您將節點類型從 CONFIG 變更為 SSH，並讓此欄位保留空白，「SSH 連接埠」 選項便會設定為 22。

如果「節點主機」選項設定為 localhost-domain，便會忽略「SSH 連接埠」選項。

SSH 使用者名稱

在此節點之主機上的使用者，會執行透過 SSH 連線至主機的程序。

如果您將節點類型從 CONFIG 變更為 SSH，並讓此欄位保留空白，「使用者名稱」 選項便會設定為正在執行 DAS 程序的使用者。

如果「節點主機」選項設定為 localhost-domain，便會忽略「SSH 使用者」選項。

SSH 使用者認證

可指定在登入此節點的主機時如何認證 SSH 使用者的下拉式清單。可能的認證方法如下：

金鑰檔案

SSH 使用者是透過 SSH 私密金鑰檔案來認證。如果選取此選項，請在 「金鑰檔案」 欄位中指定該檔案的絕對路徑。

密碼

SSH 使用者是透過儲存在 DAS 組態中的密碼來認證。

警告:

此密碼在 DAS 組態中是以純文字儲存。如需增強安全性，請使用金鑰檔案或密碼別名。

如果您選取此選項，即會啟用「SSH 使用者密碼」欄位和「確認密碼」欄位，讓您可以提供密碼。

密碼別名

SSH 使用者是透過密碼別名來認證。如果選取此選項，請從 「密碼別名」 下拉式清單中選取密碼別名。

設定 SSH

如果選取「已啟用」選項，則 GlassFish Server 會在節點主機上設定 SSH 金鑰。GlassFish Server 設定 SSH 金鑰的方式是複製 DAS 主機中的現有金鑰檔案，或視需要於複製檔案前先產生檔案。只有在「SSH 使用者認證」選項設定為「金鑰檔案」時，才能使用此選項。

產生金鑰

如果選取「已啟用」選項，則 GlassFish Server 會先產生 SSH 金鑰檔案，再將檔案複製至節點主機。

SSH 使用者密碼

SSH 使用者將在登入此節點之主機時使用的密碼。

確認密碼

確認 SSH 使用者將在登入此節點之主機時使用的密碼。此欄位的內容必須與「SSH 使用者密碼」欄位的內容完全相符。

密碼別名

為此網域定義之密碼別名的下拉式清單。

公開金鑰檔案

「SSH 使用者名稱」選項所指定之使用者 SSH 公開金鑰檔案的絕對路徑。公開金鑰檔案的內容會附加至使用者在節點主機上的 .ssh/authorized_keys 檔案。如果主機上沒有 .ssh/authorized_keys 檔案，則 GlassFish Server 會建立此檔案。

DAS 必須可以存取金鑰檔案的路徑，且 DAS 必須可以讀取金鑰檔案。

預設值為使用者之 .ssh 目錄中的金鑰檔案。若有多個金鑰檔案，GlassFish Server 會使用下列喜好設定順序：

  1. id_rsa

  2. id_dsa

  3. identity

金鑰檔案

SSH 私密金鑰檔案的絕對路徑，該檔案屬於 「使用者名稱」 選項指定的使用者。這個檔案用於向節點主機上的 sshd 常駐程式認證。

注意:

GlassFish Server 也支援透過密碼檔案中的 AS_ADMIN_SSHPASSWORD 項目進行密碼認證。密碼檔案指定於 asadmin(1) 公用程式的 --paswordfile 選項中。

如果 SSH 私密金鑰檔案受到複雜密碼保護，則密碼檔案必須包含 AS_ADMIN_SSHKEYPASSPHRASE 項目。

DAS 必須可以存取金鑰檔案的路徑，且 DAS 必須可以讀取金鑰檔案。

預設值取決於您是否將節點類型從 CONFIG 變更為 SSH，以對節點啟用 SSH 通訊：

如果您將節點類型從 CONFIG 變更為 SSH，並讓此欄位保留空白，「金鑰檔案」 選項便會設定為使用者之 .ssh 目錄中的金鑰檔案。若有多個金鑰檔案，GlassFish Server 會使用下列喜好設定順序：

  1. id_rsa

  2. id_dsa

  3. identity
