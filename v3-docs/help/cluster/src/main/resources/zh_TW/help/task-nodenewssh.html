完成 SSH 節點的建立

此作業是建立節點 SSH 節點的接續說明。

  1. 若要將 GlassFish Server 軟體從 DAS 主機複製至節點主機，請啟用「安裝 GlassFish Server」選項。

  2. 若要強制在網域管理伺服器 (DAS) 的組態中建立節點 (即使驗證節點參數失敗)，請啟用 「強制」 選項。

  3. 在 「SSH 連接埠」 欄位中，鍵入用於與此節點主機連線之 SSH 的連接埠。

  4. 在 「SSH 使用者名稱」 欄位中，鍵入要在此節點之主機上，執行透過 SSH 連線至主機之程序的使用者名稱。

  5. 指定在登入此節點之主機時如何認證 SSH 使用者。

    • 若要指定透過 SSH 私密金鑰檔案來進行認證，請執行以下步驟：

      1. 從 「SSH 使用者認證」 下拉式清單中選取 「金鑰檔案」。

      2. 如果您需要 GlassFish Server 在節點主機上設定 SSH 金鑰，請啟用「設定 SSH」選項。

        GlassFish Server 設定 SSH 金鑰的方式是複製 DAS 主機中的現有金鑰檔案，或視需要於複製檔案前先產生檔案。

        「產生金鑰」欄位和「公開金鑰檔案」欄位會增加至此視窗。

      3. 如果您需要 GlassFish Server 先產生 SSH 金鑰檔案，然後再將檔案複製至節點主機，請啟用「產生金鑰」選項。

      4. 在「SSH 使用者密碼」欄位中，鍵入 SSH 使用者在登入此節點主機時將使用的密碼。

      5. 在「公開金鑰檔案」欄位中，鍵入「SSH 使用者名稱」選項所指定使用者之 SSH 公開金鑰檔案的絕對路徑。

      6. 在 「金鑰檔案」 欄位中，鍵入 「SSH 使用者名稱」 選項所指定使用者之 SSH 私密金鑰檔案的絕對路徑。

        注意:

        GlassFish Server 也支援透過密碼檔案中的 AS_ADMIN_SSHPASSWORD 項目進行密碼認證。密碼檔案指定於 asadmin(1) 公用程式的 --paswordfile 選項中。

    • 若要指定透過儲存在 DAS 組態中的密碼來進行認證，請執行以下步驟：

      警告:

      此密碼在 DAS 組態中是以純文字儲存。如需增強安全性，請使用金鑰檔案或密碼別名

      1. 從 「SSH 使用者認證」 下拉式清單中選取 「密碼」。

      2. 在「SSH 使用者密碼」欄位中，鍵入 SSH 使用者在登入此節點主機時將使用的密碼。

      3. 在「確認密碼」欄位中，再次鍵入密碼。

    • 若要指定透過密碼別名來進行認證，請執行以下步驟：

      1. 從 「SSH 使用者認證」 下拉式清單中選取 「密碼別名」。

      2. 從 「密碼別名」 下拉式清單中選取現有的別名。

  6. 按一下 「儲存」。
