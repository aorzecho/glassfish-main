新節點

使用 [新建節點] 頁面可建立 GlassFish Server 節點。節點代表其上安裝 GlassFish Server 軟體的主機。GlassFish Server 實例所在的每部主機都必須存在對應的節點。

[新建節點] 頁面包含下列選項：

名稱

節點的名稱。

名稱必須符合下列需求：

  • 名稱只能包含 ASCII 字元。

  • 名稱必須以字母、數字或底線為開頭。

  • 名稱只能包含下列字元：

    • 小寫字母

    • 大寫字母

    • 數字

    • 連字符

    • 小數點號

    • 底線

  • 名稱在網域中必須是唯一的，且不得為另一個節點、叢集、已命名配置或 GlassFish Server 實例的名稱。

  • 名稱不得為 domain、server 或 GlassFish Server 所保留的任何其他關鍵字。

類型

下拉式清單，可指定下列其中一種節點類型：

SSH

節點支援透過安全 shell (SSH) 進行通訊。

CONFIG

節點不支援遠端通訊。如果已選取 CONFIG，則不會顯示 SSH 配置的任何選項，並會清除 [安裝目錄] 欄位。

節點主機

節點代表的主機名稱。此欄位是否為必要欄位，取決於節點類型：

  • 如果節點類型為 CONFIG，可能會省略主機名稱。可在建立位於節點上的實例時決定主機名稱。

  • 如果節點類型為 SSH，則必須指定主機名稱。否則，將發生錯誤。

節點目錄

要包含在節點上所建立的 GlassFish Server 實例之目錄的路徑。如果指定相對路徑，則該路徑會相對於 as-install 目錄。

預設值取決於節點類型：

  • 如果節點類型為 CONFIG，則不會為 Domain Administration Server (DAS) 配置中的節點指定任何實例目錄。可在建立位於節點上的實例時決定節點目錄。

  • 如果節點類型為 SSH，則預設值為 as-install/nodes，其中 as-install 是主機上 GlassFish Server 軟體的基底安裝目錄。

安裝目錄

主機上 GlassFish Server 軟體的基底安裝目錄之父系目錄的完整路徑，例如 /export/glassfish3/。

預設值取決於節點類型：

  • 如果節點類型為 CONFIG，則不會為 DAS 配置中的節點指定任何安裝目錄。可在建立位於節點上的實例時決定安裝目錄。

  • 如果節點類型為 SSH，則對於 DAS，預設值為 GlassFish Server 軟體預設基底安裝目錄的父系目錄。只有當 GlassFish Server 安裝在所有主機上的相同位置中，此預設值才有用。

強制

如果選取 [已啟用] 選項，則即使驗證節點參數失敗，也會在 DAS 配置中建立節點。若要驗證節點的參數，DAS 必須能夠透過 SSH 連絡節點的主機。

SSH 連接埠

用於與此節點主機連線之 SSH 的連接埠。預設值為 22。如果 [節點主機] 選項設定為 localhost-domain，便會忽略 [SSH 連接埠] 選項。

SSH 使用者名稱

在此節點之主機上的使用者，會執行透過 SSH 連線至主機的程序。預設值是執行 DAS 程序的使用者。若要確保 DAS 可以讀取此使用者的 SSH 私密金鑰檔案，請指定執行 DAS 程序的使用者。如果 [節點主機] 選項設定為 localhost-domain，便會忽略 [使用者名稱] 選項。

SSH 使用者認證

可指定在登入此節點的主機時如何認證 SSH 使用者的下拉式清單。可能的認證方法如下：

金鑰檔案

SSH 使用者是透過 SSH 私密金鑰檔案來認證。如果選取此選項，請在 [金鑰檔案] 欄位中指定該檔案的絕對路徑。

密碼

SSH 使用者是透過儲存在 DAS 配置中的密碼來認證。

注意 - 此密碼在 DAS 配置中是以純文字儲存。如需增強安全性，請使用金鑰檔案或密碼別名。

如果選取此選項，即會啟動 [SSH 密碼] 欄位與 [確認 SSH 密碼] 欄位，讓您提供密碼。

密碼別名

SSH 使用者是透過密碼別名來認證。如果選取此選項，請從 [密碼別名] 下拉式清單中選取密碼別名。

SSH 密碼

SSH 使用者將在登入此節點之主機時使用的密碼。

確認 SSH 密碼

確認 SSH 使用者將在登入此節點之主機時使用的密碼。此欄位的內容必須與 [SSH 密碼] 欄位的內容完全相符。

密碼別名

為此網域定義之密碼別名的下拉式清單。

金鑰檔案

SSH 私密金鑰檔案的絕對路徑，該檔案屬於 [使用者名稱] 選項指定的使用者。這個檔案用於向節點主機上的 sshd 常駐程式認證。

備註 - GlassFish Server 也支援透過密碼檔案中的 AS_ADMIN_SSHPASSWORD 項目進行密碼認證。密碼檔案是在 asadmin(1M) 公用程式的 --paswordfile 選項中指定。

如果 SSH 私密金鑰檔案受複雜密碼保護，則密碼檔案必須包含 AS_ADMIN_SSHKEYPASSPHRASE 項目。

DAS 必須可以存取金鑰檔案的路徑，且 DAS 必須可以讀取金鑰檔案。

預設值為使用者之 .ssh 目錄中的金鑰檔案。若有多個金鑰檔案，GlassFish Server 會使用下列喜好設定順序：

  1. id_rsa

  2. id_dsa

  3. identity
