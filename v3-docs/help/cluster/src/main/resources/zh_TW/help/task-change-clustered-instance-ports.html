修改叢集 GlassFish Server 實例的連接埠號碼

叢集 GlassFish Server 實例的連接埠號碼，最初是在實例繼承自其父系叢集的配置中設定。例如，如果出現兩個叢集實例位於相同主機的情況，則必須修改這些連接埠號碼。在此情況下，必須修改其中一個實例的連接埠號碼，因為兩個實例會共用相同的配置。

實例的連接埠號碼會儲存為 Java 系統特性。GlassFish Server 啟動時，這些特性會透過 Java 應用程式啟動程式的 -D 選項，傳遞至 Java 應用程式啟動程式。

  1. 在瀏覽樹狀結構中，展開 [叢集] 節點。
  2. 在展開的 [叢集] 節點下，選取該叢集的節點。

    此時將開啟叢集的 [一般資訊] 頁面。

  3. 在 [一般資訊] 頁面上，按一下 [實例] 標籤。

    此時將開啟該叢集的 [叢集伺服器實例] 頁面。

  4. 選取要修改其連接埠號碼的實例。

    此時將開啟實例的 [一般資訊] 頁面。

  5. 在 [一般資訊] 頁面上，按一下 [特性] 標籤。

    此時將開啟 [實例系統特性] 頁面。

    此頁面會顯示實例之下列系統特性目前的值：

    • 實例配置中預先定義的連接埠號碼

    • 為實例定義的自訂特性 (如果有)

  6. 對於您要修改的每個現有特性，請在該特性的 [置換值] 欄位中鍵入新值。
  7. 為所需的任何其他特性設定值。
    1. 在 [其他特性] 表格中，按一下 [增加特性]。

      隨即會在其他特性表中增加一列。

    2. 在 [實例變數名稱] 欄位中，鍵入特性名稱。
    3. 在 [置換值] 欄位中，鍵入值。
  8. 按一下 [儲存]。
