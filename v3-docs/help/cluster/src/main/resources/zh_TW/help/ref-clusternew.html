新建叢集

使用 [新建叢集] 頁面可建立叢集。

[新建叢集] 頁面包含下列選項。

叢集名稱

叢集的名稱。

名稱必須符合下列需求：

  • 名稱只能包含 ASCII 字元。

  • 名稱必須以字母、數字或底線為開頭。

  • 名稱只能包含下列字元：

    • 小寫字母

    • 大寫字母

    • 數字

    • 連字符

    • 小數點號

    • 底線

  • 名稱在網域中必須是唯一的，且不得為另一個叢集、已命名配置、GlassFish Server 實例或節點的名稱。

  • 名稱不得為 domain、server 或 GlassFish Server 所保留的任何其他關鍵字。

配置

叢集可以使用之現有已命名配置的下拉式清單。叢集需要已命名配置，該配置會定義增加至叢集之所有實例的配置。您必須從清單中選取一個配置。建立叢集時，叢集將使用所選取的配置。

複製選取的配置

如果選取此選項，就會在建立叢集時複製選取的配置。為配置副本指定的名稱是 cluster-name-config，其中 cluster-name 是在 [叢集名稱] 欄位中鍵入的叢集名稱。

如果已從 [配置]下拉式清單中選取 default-config，則必須選取此選項。default-config 配置僅可複製，無法參照。

參照選取的配置

如果選取此選項，叢集將會使用指定的現有已命名配置。

如果已從 [配置] 下拉式清單中選取 default-config，則不能選取此選項。default-config 配置無法參照，僅可複製。

要建立的伺服器實例

即將在建立叢集時建立的 GlassFish Server 實例清單。對於每個實例，均提供以下資訊：

實例名稱

實例的名稱。

權數

一個整數，代表實例的負載平衡權數。

負載平衡權數可決定送至叢集供實例處理之所有請求的比例。

例如，在有兩個實例的叢集中，您可能需要讓其中一個實例處理每四個請求中的一個請求，而讓另一個實例處理每四個請求中的三個請求。在這種情況下，可將處理每四個請求中之一個請求的實例權數設定為 1，將另一個實例權數設定為 3。如果您要使用百分比，請將這兩個實例的權數分別設定為 25 與 75。

預設權數為 100。

節點

可存在實例之現有節點的下拉式清單。您必須從清單中選取一個節點。建立叢集時，實例將位於選取的節點上。

要建立的伺服器實例表也包含下列選項。

新建

建立實例的按鈕。按一下此按鈕即可在要建立的伺服器實例表中增加一列。

刪除

刪除實例的按鈕。
