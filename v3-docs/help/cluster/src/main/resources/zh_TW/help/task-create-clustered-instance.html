將 GlassFish Server 實例增加至叢集

GlassFish Server 實例構成了應用程式建置的基礎。叢集實例是叢集的成員。每個實例都屬於單一網域，並且具有自己的目錄結構、組態和已建置的應用程式。每個實例也包括 Java Platform Enterprise Edition (Java EE) Web 容器和 Enterprise JavaBeans 技術 (EJB 技術) 容器。每個新實例都必須包含節點的參照，節點用於定義實例將位於的主機。

注意:

若要在類型為 CONFIG 且代表遠端主機的節點上建立實例，您必須使用 create-local-instance(1) 子命令。您無法使用管理主控台來達成此目的。
開始前

請確認下列項目存在：

  • 您要在其中增加實例的叢集。如需說明，請參閱建立叢集。

  • 實例所在的節點。如需說明，請參閱建立節點。

  1. 在瀏覽樹狀結構中，展開 「叢集」 節點。

  2. 在展開的 「叢集」 節點下，選取該叢集的節點。

    此時將開啟叢集的 「一般資訊」 頁面。

  3. 在 「一般資訊」 頁面上，按一下 「實例」 標籤。

    此時將開啟該叢集的 「叢集伺服器實例」 頁面。

  4. 在 「叢集伺服器實例」 頁面中，按一下 「新建」。

    此時將開啟 「新建叢集伺服器實例」 頁面。

  5. 在 「名稱」 欄位中，鍵入實例的名稱。

    名稱必須符合下列需求：

    • 名稱只能包含 ASCII 字元。

    • 名稱必須以字母、數字或底線為開頭。

    • 名稱只能包含下列字元：

      • 小寫字母

      • 大寫字母

      • 數字

      • 連字符

      • 小數點號

      • 底線

    • 名稱在網域中必須是唯一的，且不得為另一個 GlassFish Server 實例、叢集、已命名組態或節點的名稱。

    • 名稱不得為 domain、server 或 GlassFish Server 所保留的任何其他關鍵字。

  6. 從 「節點」 下拉式清單中，選取實例將位於的節點。

  7. 按一下 「確定」。
