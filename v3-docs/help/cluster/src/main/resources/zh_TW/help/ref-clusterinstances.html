叢集伺服器實例

使用 [叢集伺服器實例] 頁面可建立和管理目前叢集中的 GlassFish Server 實例。

[叢集名稱] 欄位是唯讀欄位，會顯示目前叢集的名稱。

[叢集伺服器實例] 頁面會顯示隸屬於目前叢集的實例清單。對於每個實例，均提供以下資訊。

名稱

建立實例時指定給實例的名稱。按一下該名稱，隨即會開啟實例的 [一般資訊] 頁面。

負載平衡權數

一個整數，代表實例的負載平衡權數。

負載平衡權數可決定送至叢集供實例處理之所有請求的比例。

例如，在有兩個實例的叢集中，您可能需要讓其中一個實例處理每四個請求中的一個請求，而讓另一個實例處理每四個請求中的三個請求。在這種情況下，可將處理每四個請求中之一個請求的實例權數設定為 1，將另一個實例權數設定為 3。如果您要使用百分比，請將這兩個實例的權數分別設定為 25 與 75。

預設權數為 100。

配置

實例參照的配置。按一下配置，隨即會開啟實例的 [配置] 頁面。

節點

實例所在之節點的名稱。按一下該名稱，隨即會開啟節點的編輯節點頁面。

狀態

指示實例是在執行中 (需要重新啟動才能更新配置變更) 還是不在執行中。本欄位是唯讀的。

伺服器實例表也包含下列選項。

新建

將實例增加至叢集的按鈕。

刪除

從叢集中移除一或多個所選取實例的按鈕。

注意 - 如果透過主代理程式使用 Java Message Service (JMS) 叢集，請勿刪除與主代理程式相關聯的實例。如果必須刪除此實例，請使用 change-master-broker(1) 子指令，將主代理程式指定給不同的實例。

啟動

啟動叢集中一或多個所選取實例的按鈕。

停止

停止叢集中一或多個所選取實例的按鈕。
