編輯節點

編輯節點會更新 GlassFish Server 節點的配置資料。編輯節點也可讓您將節點類型從 CONFIG 變更為 SSH，以對節點啟用 SSH 通訊。

如果您將節點類型從 CONFIG 變更為 SSH，則下列任一欄位保留空白時，便會套用預設值：

  • SSH 連接埠

  • SSH 使用者名稱

  • 金鑰檔案

  1. 在瀏覽樹狀結構中，展開 [節點] 節點。
  2. 從節點清單選取您要編輯的節點。

    此時將開啟 [編輯節點] 頁面。

  3. 從 [類型] 下拉式清單中，選取更新的節點類型。
    • 若要讓節點支援透過安全 shell (SSH) 進行通訊，請選取 [SSH]。
    • 若要移除節點中的遠端配置支援，請選取 [CONFIG]。

      如果您選取 [CONFIG]，則不會顯示 SSH 配置的任何選項。如果目前的節點類型是 SSH，而您選取 CONFIG，則儲存變更時，會從節點配置中移除 SSH 配置資料。

  4. 在 [節點主機] 欄位中，鍵入更新的節點要代表的主機名稱。

    此欄位是否為必要欄位，取決於節點類型：

    • 如果節點類型為 CONFIG，可能會省略主機名稱。可在建立位於節點上的實例時決定主機名稱。

    • 如果節點類型為 SSH，則必須指定主機名稱。否則，將發生錯誤。

  5. 在 [節點目錄] 欄位中，鍵入要包含在節點上所建立的 GlassFish Server 實例之目錄的路徑。

    如果您鍵入相對路徑，則該路徑會相對於 as-install 目錄，其中 as-install 是主機上 GlassFish Server 軟體的基底安裝目錄。

  6. 在 [安裝目錄] 欄位中，鍵入主機上 GlassFish Server 軟體的基底安裝目錄之父系目錄的完整路徑，例如 /export/glassfish3/。
  7. 若要強制在 Domain Administration Server (DAS) 的配置中建立節點 (即使驗證節點參數失敗)，請啟用 [強制] 選項。
  8. 在 [SSH 連接埠] 欄位中，鍵入用於與此節點主機連線之 SSH 的連接埠。

    如果您將節點類型從 CONFIG 變更為 SSH，並讓此欄位保留空白，[SSH 連接埠] 選項便會設定為 22。

  9. 在 [SSH 使用者名稱] 欄位中，鍵入要在此節點之主機上，執行透過 SSH 連線至主機之程序的使用者名稱。

    如果您將節點類型從 CONFIG 變更為 SSH，並讓此欄位保留空白，[SSH 使用者名稱] 選項便會設定為正在執行 DAS 程序的使用者。

  10. 指定在登入此節點之主機時如何認證 SSH 使用者。
    • 若要指定透過 SSH 私密金鑰檔案來進行認證，請執行以下步驟：
      1. 從 [SSH 使用者認證] 下拉式清單中選取 [金鑰檔案]。
      2. 在 [金鑰檔案] 欄位中，鍵入 [SSH 使用者名稱] 選項所指定使用者之 SSH 私密金鑰檔案的絕對路徑。

        備註 - GlassFish Server 也支援透過密碼檔案中的 AS_ADMIN_SSHPASSWORD 項目進行密碼認證。密碼檔案是在 asadmin(1M) 公用程式的 --paswordfile 選項中指定。

        如果您將節點類型從 CONFIG 變更為 SSH，並讓此欄位保留空白，[金鑰檔案] 選項便會設定為使用者之 .ssh 目錄中的金鑰檔案。若有多個金鑰檔案，GlassFish Server 會使用下列喜好設定順序：

        • id_rsa

        • id_dsa

        • identity

    • 若要指定透過儲存在 DAS 配置中的密碼來進行認證，請執行以下步驟：

      注意 - 此密碼在 DAS 配置中是以純文字儲存。如需增強安全性，請使用金鑰檔案或密碼別名。

      1. 從 [SSH 使用者認證] 下拉式清單中選取 [密碼]。
      2. 在 [SSH 密碼] 欄位中，鍵入 SSH 使用者將在登入此節點之主機時使用的密碼。
      3. 在 [確認 SSH 密碼] 欄位中，再次鍵入密碼。
    • 若要指定透過密碼別名來進行認證，請執行以下步驟：
      1. 從 [SSH 使用者認證] 下拉式清單中選取 [密碼別名]。
      2. 從 [密碼別名] 下拉式清單中選取現有的別名。
  11. 按一下 [儲存]。
