一般資訊

「一般資訊」 頁面包含下列選項。

啟動叢集

啟動叢集的按鈕。如果叢集中的所有實例都在執行中，便會關閉此按鈕。

停止叢集

停止叢集的按鈕。如果叢集中的所有實例都已停止，便會關閉此按鈕。

遷移 EJB 計時器

此按鈕可將 Enterprise JavaBeans (EJB) 技術計時器從已停止的 GlassFish Server 實例遷移到叢集中正在執行的實例。按一下此按鈕，隨即會開啟遷移 EJB 計時器頁面。如果叢集中的所有實例都在執行中，便會關閉此按鈕。

自動重建記錄

為叢集自動重建記錄檔的按鈕。自動重建記錄檔時，會使用時間戳記名稱 (格式為 server.log_date-and-time) 來重新命名檔案，並且為新的記錄訊息建立空白記錄檔。變更會以動態方式套用。不需要重新啟動伺服器。

叢集名稱

建立叢集時指定給叢集的名稱。本欄位是唯讀的。

JVM

如果按一下 「JVM 報告」，會開啟另外一個視窗並顯示 Java 虛擬機器的相關報告，包括摘要報告、記憶體管理和資源回收報告、類別載入報告以及目前執行緒傾印。

組態

建立叢集時為叢集選取的組態。按一下該組態，即會開啟叢集的 「組態」 頁面。本欄位是唯讀的。

GMS

如果已啟用此選項，就會在叢集與網域管理伺服器 (DAS) 的每個 GlassFish Server 實例中啟動群組管理服務 (GMS)。 DAS 會參與每個已啟用 GMS 的叢集。GMS 會提供叢集監視、叢集成員身份及群組通訊服務。此選項預設為啟用。

多重播送連接埠

GMS 偵聽群組事件之通訊埠的連接埠號碼。此選項必須指定有效的連接埠號碼，範圍介於 2048-32000 之間。預設值為自動產生的值 (介於此範圍內)。

多重播送位址

GMS 偵聽群組事件的位址。此選項必須指定多重播送位址，範圍介於 224.0.0.0 至 239.255.255.255 之間。 預設值是 228.9.XX.YY，其中 XX 與 YY 是自動產生的獨立值 (介於 0 至 255 之間)。

連結介面位址

GMS 所連結之網路介面的網際網路協定 (IP) 位址。此選項必須指定本機網路介面的 IP 位址。預設值為所有公用網路介面位址。

在多址機器上，此選項會組態用於 GMS 的網路介面。多址機器擁有兩個或兩個以上的網路介面。

若要指定叢集中所有 GlassFish Server 實例的有效位址，請使用系統特性來為每個實例個別設定位址。

例如，使用叢集系統特性頁面可建立系統特性 GMS-BIND-INTERFACE-ADDRESS-cluster-name。接著，將此頁面中的「連結位址」選項設定為 ${GMS-BIND-INTERFACE-ADDRESS-cluster-name}，來指定系統特性。最後，針對叢集中的每個實例，將 GMS-BIND-INTERFACE-ADDRESS-cluster-name 系統特性設定為實例機器上所需的網路介面位址。

狀態

指示叢集中正在執行與未在執行的實例數目。本欄位是唯讀的。
