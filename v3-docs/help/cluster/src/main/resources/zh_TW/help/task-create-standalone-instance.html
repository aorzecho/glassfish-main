建立獨立 GlassFish Server 實例

GlassFish Server 實例構成了應用程式建置的基礎。獨立實例不是叢集的成員。每個實例都屬於單一網域，並且具有自己的目錄結構、組態和已建置的應用程式。每個實例也包括 Java Platform Enterprise Edition (Java EE) Web 容器和 Enterprise JavaBeans 技術 (EJB 技術) 容器。每個新實例都必須包含節點的參照，節點用於定義實例將位於的主機。

注意:

若要在類型為 CONFIG 且代表遠端主機的節點上建立實例，您必須使用 create-local-instance(1) 子命令。您無法使用管理主控台來達成此目的。
開始前

確定實例所在的節點已存在。如需說明，請參閱建立節點。

  1. 在瀏覽樹狀結構中，選取 「獨立實例」 節點。

    此時將開啟 「獨立伺服器實例」 頁面。

  2. 在 「獨立伺服器實例」 頁面中，按一下 「新建」。

    此時將開啟 「新建獨立伺服器實例」 頁面。

  3. 在 「名稱」 欄位中，鍵入實例的名稱。

    名稱必須符合下列需求：

    • 名稱只能包含 ASCII 字元。

    • 名稱必須以字母、數字或底線為開頭。

    • 名稱只能包含下列字元：

      • 小寫字母

      • 大寫字母

      • 數字

      • 連字符

      • 小數點號

      • 底線

    • 名稱在網域中必須是唯一的，且不得為另一個 GlassFish Server 實例、叢集、已命名組態或節點的名稱。

    • 名稱不得為 domain、server 或 GlassFish Server 所保留的任何其他關鍵字。

  4. 從 「節點」 下拉式清單中，選取實例將位於的節點。

  5. 從 「組態」 下拉式清單中，選取實例將使用的已命名組態。

    注意:

    如果您選取 default-config，就必須在下個步驟中將組態指定為不共用。default-config 組態無法共用。

  6. 指定是否要共用實例將使用的組態。

    • 如果不共用組態，請保留選取 「複製選取的組態」 選項。

      建立實例時，會複製選取的組態。為組態副本指定的名稱是 instance-name-config，其中 instance-name 是您要建立的實例名稱。

    • 如果共用組態，請選取 「參照選取的組態」 選項。

      實例將使用指定的現有已命名組態。

  7. 按一下 「確定」。
