遷移 EJB 計時器

如果 GlassFish Server 實例停止，則該實例上安裝的 Enterprise JavaBeans (EJB) 技術計時器也會停止執行。如果實例異常停止或意外地停止，則為了讓這些計時器在實例停止時繼續執行，您必須將計時器遷移至叢集內正在執行的伺服器實例。

  1. 在瀏覽樹狀結構中，展開 「叢集」 節點。

  2. 在瀏覽樹狀結構的 「叢集」 節點下，選取叢集。

    此時將開啟叢集的 「一般資訊」 頁面。

  3. 在 「一般資訊」 頁面上，按一下 「遷移 EJB 計時器」。

    此時將開啟 「遷移 EJB 計時器」 頁面。

    注意:

    如果叢集內的所有實例都在執行，則會關閉 「遷移 EJB 計時器」 按鈕。

    為了協助您遷移計時器，「遷移 EJB 計時器」頁面包含 EJB 計時器表格與下列資訊。

    伺服器名稱

    叢集中的 GlassFish Server 實例清單。

    計時器計數

    每個實例上的永久性計時器數目。

  4. 在 「遷移 EJB 計時器」 頁面上，選取計時器的遷移來源實例和目標實例。

    1. 從 「來源」 下拉式清單中，選取要遷移的計時器所在的已停止的實例。

    2. 從 「目標」 下拉式清單中，選取要將計時器遷移至的正在執行的實例。

    3. 按一下 「確定」。

  5. 停止並重新啟動您遷移 EJB 計時器的目標實例。

    如果來源實例正在執行或目標實例未執行，則管理主控台將顯示錯誤訊息。
