為叢集配置群組管理服務設定

群組管理服務 (GMS) 是一種進程服務，可提供叢集監視、叢集成員身份和群組通訊服務。GMS 會執行下列功能：

  • 從叢集增加或移除一或多個成員實例，或者一或多個成員實例疑似故障或確認故障時，會通知已在 GlassFish Server 實例中註冊的模組。

  • 能夠在一組程序之間傳送和接收訊息。

  1. 在瀏覽樹狀結構中，展開 [叢集] 節點。
  2. 選取要為其配置群組管理服務 (GMS) 設定的叢集。

    此時將開啟叢集的 [一般資訊] 頁面。

  3. 選取 [GMS 已啟用] 核取方塊，為叢集啟用 GMS。

    此選項預設為選取。若要停用叢集的 GMS，請取消核取 [GMS 已啟用] 核取方塊。

    如果啟用，就會在叢集與 Domain Administration Server (DAS) 的每個伺服器實例中啟動群組管理服務。DAS 會參與每個已啟用 GMS 的叢集。

  4. 在 [多重播送連接埠] 欄位中，輸入群組管理服務用來偵聽群組事件的連接埠。

    指定標準 UDP 連接埠號碼，範圍介於 2048-32000 之間。預設值是自動產生的值 (介於此範圍內)。

  5. 在 [多重播送位址] 欄位中，輸入群組管理服務用來偵聽群組事件的位址。

    指定類別 D IP 多重播送位址。類別 D IP 位址的範圍是 224.0.0.0 到 239.255.255.255 (含)。位址 224.0.0.0 為保留位址，不可使用。預設值是 228.9.XX.YY，其中 XX 與 YY 是自動產生的獨立值 (介於 0 至 255 之間)。

  6. 在 [連結介面位址] 欄位中，輸入群組管理服務的網路介面位址 (如果有多個網路介面)。

    如果叢集內託管伺服器實例的任一機器上有多個網路介面，則此設定可讓您配置將哪個網路介面用於 GMS。此值必須是本機網路介面 IP 位址。依預設，會使用所有公用網路介面位址。

    若要指定叢集中所有 GlassFish Server 實例的有效位址，請使用系統特性來為每個實例個別設定位址。

    例如，在 DAS 上建立系統特性 GMS-BIND-INTERFACE-ADDRESS- cluster-name。然後將這個 [連結介面位址] 選項設為 ${GMS-BIND-INTERFACE-ADDRESS- cluster-name}，來指定系統特性。最後，針對叢集中的每個實例，將 GMS-BIND-INTERFACE-ADDRESS- cluster-name 系統特性設定為實例的機器上所需的網路介面位址。

  7. (可選擇) 若要指定其他特性，請選取 [特性] 標籤和 [叢集特性] 子標籤。

    GlassFish Server 未定義 GMS 的任何其他特性。

    • 若要增加特性，請按一下 [增加特性] 按鈕。在出現的空白列中，於 [名稱] 欄位中鍵入特性名稱，然後在 [值] 欄位中鍵入特性值。
    • 若要修改特性，請編輯特性的 [值] 欄位。
    • 若要刪除特性，請選取您要刪除之特性 [名稱] 欄位左側的核取方塊，然後按一下 [刪除特性] 按鈕。
  8. 按一下 [儲存]。
