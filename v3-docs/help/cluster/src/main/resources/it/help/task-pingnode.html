Eseguire un test di raggiungibilità del nodo

Per creare o avviare istanze di GlassFish Server su un nodo che rappresenta un host remoto, DAS deve poter contattare il nodo. Prima di tentare la creazione o l'avvio di istanze su tale nodo, verificare che DAS possa contattare il nodo.

I nodi su cui eseguire il test devono essere di tipo DCOM o SSH. Non è possibile eseguire il test su un nodo di tipo CONFIG poiché i nodi CONFIG non sono abilitati per le comunicazioni remote.

Prima di iniziare

Assicurarsi che sull'host che il nodo rappresenta sia configurato il protocollo appropriato per il tipo di nodo. Il nodo viene abilitato per le comunicazioni remote su questo protocollo.

  • Se il nodo è di tipo DCOM, deve essere configurato il protocollo DCOM (Distributed Component Object Model).

  • Se il nodo è di tipo SSH, deve essere configurato il protocollo SSH (Secure Shell) protocol must be configured.

    1. Nell'albero di navigazione selezionare il nodo Nodi.

      Viene visualizzata la pagina Nodi. Nella pagina sono elencati tutti i nodi del dominio corrente.

    2. Nella tabella di nodi, seguire il link Ping per il nodo su cui si sta eseguendo il test.

      Alla pagina Nodi viene aggiunto un messaggio che indica se DAS è riuscito a contattare il nodo.
