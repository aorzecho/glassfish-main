Aggiungere un'istanza di GlassFish Server a un cluster

Le istanze di GlassFish Server formano la base della distribuzione dell'applicazione. L'istanza in cluster fa parte del cluster. Ogni istanza appartiene a un dominio singolo e presenta la propria struttura di directory, la propria configurazione e le proprie applicazioni distribuite. Ogni istanza include inoltre il contenitore Web Java EE (Enterprise Edition) Platform e il contenitore di tecnologia EJB (Enterprise JavaBeans). Ogni nuova istanza deve contenere un riferimento a un nodo che definisce l'host in cui collocare l'istanza.

Nota - Per creare un'istanza in un nodo di tipo CONFIG che rappresenta un host remoto, è necessario utilizzare il comando secondario create-local-instance(1). Per questa operazione non è possibile utilizzare Administration Console.

Prima di cominciare

Verificare che siano disponibili i seguenti elementi:

  • The cluster to which you are adding the instance

  • The node on which the instance is to reside

  1. Nella struttura ad albero, espandere il nodo Clusters.
  2. Dopo aver espanso il nodo Clusters, selezionare il nodo per tale cluster.

    Viene visualizzata la pagina General Information per il cluster.

  3. Nella pagina General Information, fare clic sulla scheda Instances.

    Viene visualizzata la pagina Clustered Server Instances per il cluster.

  4. Nella pagina Clustered Server Instances, fare clic su New.

    Viene visualizzata la pagina New Clustered Server Instance.

  5. Nel campo Name digitare il nome per l'istanza.

    Il nome deve soddisfare i seguenti requisiti:

    • Il nome deve iniziare con una lettera, un numero o un carattere di sottolineatura.

    • Il nome può contenere solo i seguenti caratteri:

      • Lettere minuscole

      • Lettere maiuscole

      • Numeri

      • Trattino

      • Punto

      • Carattere di sottolineatura

    • Il nome deve essere univoco nel dominio e non deve essere il nome di un'altra istanza di GlassFish Server, di un cluster, di una configurazione con nome o di un nodo.

    • Il nome non deve essere domain, server né nessun'altra parola chiave riservata per GlassFish Server.

  6. Nell'elenco a discesa Node selezionare il nodo in cui collocare l'istanza.
  7. Fare clic su OK.
