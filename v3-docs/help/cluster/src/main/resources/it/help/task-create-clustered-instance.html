Aggiungere un'istanza di GlassFish Server a un cluster

Le istanze di GlassFish Server formano la base della distribuzione dell'applicazione. L'istanza in cluster fa parte del cluster. Ogni istanza appartiene a un dominio singolo e presenta la propria struttura di directory, la propria configurazione e le proprie applicazioni distribuite. Ogni istanza include inoltre il contenitore Web Java Platform, Enterprise Edition (Java EE) e il contenitore di tecnologia EJB (Enterprise JavaBeans). Ogni nuova istanza deve contenere un riferimento a un nodo che definisce l'host in cui risiederà l'istanza.

Nota:

Per creare un'istanza in un nodo di tipo CONFIG che rappresenta un host remoto, è necessario utilizzare il comando secondario create-local-instance(1). Per questa operazione non è possibile utilizzare Console di amministrazione.
Prima di iniziare

Verificare che siano disponibili i seguenti elementi:

  • Il cluster in cui si sta aggiungendo l'istanza. Per istruzioni, vedere Creare un cluster.

  • Il nodo in cui risiederà l'istanza. Per istruzioni, vedere Creare un nodo.

  1. Nell'albero di navigazione espandere il nodo Cluster.

  2. Dopo aver espanso il nodo Cluster, selezionare il nodo per tale cluster.

    Viene visualizzata la pagina Informazioni generali per il cluster.

  3. Nella pagina Informazioni generali, fare clic sulla scheda Istanze.

    Viene visualizzata la pagina Istanze server in cluster per il cluster.

  4. Nella pagina Istanze server in cluster, fare clic su Nuovo.

    Viene visualizzata la pagina Nuova istanza server in cluster.

  5. Nel campo Nome digitare il nome per l'istanza.

    Il nome deve soddisfare i seguenti requisiti:

    • Il nome può contenere solo caratteri ASCII.

    • Il nome deve iniziare con una lettera, un numero o un carattere di sottolineatura.

    • Il nome può contenere solo i seguenti caratteri:

      • Lettere minuscole

      • Lettere maiuscole

      • Numeri

      • Trattino

      • Punto

      • Carattere di sottolineatura

    • Il nome deve essere univoco nel dominio e non deve essere il nome di un'altra istanza di GlassFish Server, di un cluster, di una configurazione denominata o di un nodo.

    • Il nome non deve essere domain, server né nessun'altra parola chiave riservata per GlassFish Server.

  6. Nell'elenco a discesa Nodo selezionare il nodo in cui risiederà l'istanza.

  7. Fare clic su OK.
