Proprietà cluster

Nella pagina Proprietà cluster viene visualizzata una lista di proprietà impostate per i cluster selezionati. Queste proprietà aggiungono informazioni di configurazione opzionali sul cluster.

Il campo Nome cluster è un campo di sola lettura in cui viene visualizzato il nome del cluster selezionato.

Per ogni proprietà vengono visualizzate le seguenti informazioni:

Nome

Nome della proprietà.

Valore

Il valore della proprietà è impostato per l'istanza selezionata.

Descrizione

Descrizione testuale in cui sono riportate ulteriori informazioni sulla proprietà.

La tabella Proprietà aggiuntive contiene inoltre le seguenti opzioni.

Aggiungi proprietà

Pulsante che consente di aggiungere una proprietà. Se si fa clic su questo pulsante, viene aggiunta una riga alla tabella Proprietà aggiuntive.

Elimina proprietà

Pulsante che consente di eliminare una o più proprietà selezionate. Quando una proprietà viene eliminata, viene ripristinato il rispettivo valore predefinito oppure, se non è specificato il valore predefinito, il valore resta non definito.

GlassFish Server definisce le seguenti proprietà del cluster:

GMS_LISTENER_PORT

Numero della porta su cui il cluster ascolta i messaggi provenienti dal servizio GMS (Group Management Service).

L'impostazione predefinita è ${GMS_LISTENER_PORT-cluster-name}. Nella maggior parte delle situazioni l'impostazione predefinita è sufficiente.

Tuttavia, se GMS non utilizza il multicast per trasmettere i messaggi, questa proprietà deve specificare un numero di porta valido per tutte le istanze GlassFish Server presenti nel cluster. Per utilizzare l'impostazione predefinita per soddisfare questo requisito, utilizzare una proprietà di sistema per impostare il numero di porta a livello individuale per ciascuna istanza.

Ad esempio, utilizzare la pagina Proprietà di sistema del cluster per creare la proprietà di sistema GMS_LISTENER_PORT-cluster-name. Quindi, per ciascuna istanza nel cluster, impostare la proprietà di sistema GMS_LISTENER_PORT-cluster-name sul numero della porta su cui l'istanza ascolta i messaggi da GMS. L'impostazione predefinita della proprietà GMS_LISTENER_PORT per il cluster fa riferimento a questa proprietà di sistema.

GMS_LOOPBACK

Specifica se consentire a un'istanza di ricevere da se stessa messaggi a livello di applicazione trasmessi dall'istanza al cluster.

I possibili valori sono i seguenti:

false

L'istanza non può ricevere messaggi provenienti da se stessa (opzione predefinita).

true

L'istanza può ricevere messaggi da se stessa. Utilizzare questa impostazione per verificare un'istanza quando è l'unica istanza in un cluster.

GMS_MULTICAST_TIME_TO_LIVE

Numero massimo di ripetizioni o trasmissioni che possono verificarsi per i seguenti tipi di eventi prima che un messaggio multicast venga ignorato:

  • Individuazione gruppo

  • Heartbeat membro

  • Modifiche appartenenza

Per soddisfare la configurazione della rete in cui vengono distribuite le istanze DAS e le istanze in cluster, impostare il minimo valore possibile. Per determinare il minimo valore possibile per il sistema in uso, utilizzare il comando secondario validate-multicast(1).

Se si imposta un valore pari a 0, i messaggi multicast non lasciano mai l'host dal quale vengono trasmessi.

Se si imposta un valore pari a 1, la trasmissione di messaggi tra host della stessa sottorete connessi tramite switch o router potrebbe non riuscire.

Il valore predefinito è 4 e assicura che i messaggi vengano trasmessi correttamente a tutti i membri del cluster delle reti in cui gli host sono collegati tramite switch o router.

GMS_DISCOVERY_URI_LIST

Le posizioni delle istanze di GlassFish Server in cui GMS invierà gli eventi di gruppo. Questa proprietà è obbligatoria solo se GMS non utilizza il multicast per la trasmissione dei messaggi.

I valori validi per questa proprietà sono riportati di seguito.

  • Una lista di URI (Uniform Resource Identifier) separati da virgole. Ciascun URI deve individuare un'istanza di GlassFish Server. Questo formato è obbligatorio se più istanze di GlassFish Server sono in esecuzione sullo stesso host.

    Il formato di ciascun URI nella lista è il seguente:

    scheme://host-name-or -IP-address:port

    • scheme è lo schema dell'URI, ossia tcp.

    • host-name-or -IP-address è il nome host o l'indirizzo IP dell'host su cui è in esecuzione l'istanza.

    • porta è il numero della porta su cui l'istanza ascolta i messaggi da GMS. La proprietà di sistema GMS_LISTENER_PORT-clustername deve essere impostata per l'istanza. Per informazioni su come impostare questa proprietà di sistema per un'istanza, vedere Locating GlassFish Server Instances When Multicast Is Disabled, in Oracle GlassFish Server High Availability Administration Guide.

  • Una lista di indirizzi IP o nomi host separati da virgole. La lista può contenere un misto di indirizzi IP e nomi host. Questo formato può essere utilizzato solo se un'istanza in cluster è in esecuzione su ciascun host. Il valore della proprietà GMS_LISTENER_PORT deve essere univoco per ciascun cluster in un dominio.

  • La parola chiave generate. Questo formato può essere utilizzato solo se un'istanza in un cluster è in esecuzione su ciascun host. Più istanze sullo stesso host non possono essere membri dello stesso cluster. Il valore della proprietà GMS_LISTENER_PORT deve essere univoco per ciascun cluster in un dominio.
