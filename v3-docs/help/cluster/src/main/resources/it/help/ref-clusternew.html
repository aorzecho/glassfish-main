Nuovo Cluster

Nella pagina Nuovo cluster è possibile creare un cluster.

La pagina Nuovo cluster contiene le seguenti opzioni.

Nome cluster

Nome del cluster.

Il nome deve soddisfare i seguenti requisiti:

  • Il nome può contenere solo caratteri ASCII.

  • Il nome deve iniziare con una lettera, un numero o un carattere di sottolineatura.

  • Il nome può contenere solo i seguenti caratteri:

    • Lettere minuscole

    • Lettere maiuscole

    • Numeri

    • Trattino

    • Punto

    • Carattere di sottolineatura

  • Il nome deve essere univoco nel dominio e non deve essere il nome di un altro cluster, di una configurazione denominata, di un'istanza di GlassFish Server o di un nodo.

  • Il nome non deve essere domain, server né nessun'altra parola chiave riservata per GlassFish Server.

Configurazione

Elenco a discesa di configurazioni denominate esistenti utilizzabili dal cluster. Un cluster richiede una configurazione denominata per la definizione della configurazione di tutte le istanze che vengono aggiunte al cluster. È necessario selezionare una configurazione dalla lista. Il cluster utilizzerà la configurazione selezionata al momento della creazione del cluster.

Crea una copia della configurazione selezionata

Se l'opzione è selezionata, la configurazione selezionata viene copiata al momento della creazione del cluster. Alla copia della configurazione viene assegnato il nome cluster-name-config, dove cluster-name è il nome del cluster come digitato nel campo Nome cluster.

È necessario selezionare tale opzione se nell'elenco a discesa Configurazione è selezionato default-config. La configurazione default-config può essere copiata ma non utilizzata come riferimento.

Crea riferimento alla configurazione selezionata

Se l'opzione è selezionata, il cluster utilizza la configurazione con nome esistente specificata.

L'opzione non deve essere selezionata se nell'elenco a discesa Configurazione è selezionato default-config. La configurazione default-config non può essere utilizzata come riferimento, ma solo copiata.

Istanze server da creare

Lista di istanze di &ProductName da creare al momento della creazione del cluster. Per ogni istanza vengono fornite le seguenti informazioni.

Nome istanza

Nome dell'istanza.

Fattore ponderale

Numero intero che rappresenta il fattore ponderale per il bilanciamento del carico dell'istanza.

Il fattore ponderale per il bilanciamento del carico determina la proporzione di tutte le richieste inviate al cluster che l'istanza dovrebbe elaborare.

Ad esempio, in un cluster a due istanze può essere necessaria un'istanza per elaborare una di quattro richieste e un'altra istanza per elaborarne tre di quattro. In questa situazione, impostare su 1 il fattore ponderale dell'istanza che elabora una delle quattro richieste e su 3 il fattore ponderale dell'altra istanza. Se si preferisce utilizzare valori percentuali, impostare i fattori ponderali delle istanze rispettivamente su 25 e 75.

Il fattore ponderale predefinito è 100.

Nodo

Elenco a discesa di nodi esistenti in cui può risiedere l'istanza. È necessario selezionare un nodo dalla lista. L'istanza risiederà nel nodo selezionato al momento della creazione del cluster.

La tabella Istanze server da creare contiene inoltre le seguenti opzioni.

Nuovo

Pulsante che consente di creare un'istanza. Se si fa clic su questo pulsante, viene aggiunta una riga alla tabella Istanze server da creare.

Elimina

Pulsante che consente di eliminare un'istanza.
