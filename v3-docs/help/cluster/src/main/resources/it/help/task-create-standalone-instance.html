Creare un'istanza di GlassFish Server standalone

Le istanze di GlassFish Server formano la base della distribuzione dell'applicazione. Un'istanza standalone non fa parte di un cluster. Ogni istanza appartiene a un dominio singolo e presenta la propria struttura di directory, la propria configurazione e le proprie applicazioni distribuite. Ogni istanza include inoltre il contenitore Web Java Platform, Enterprise Edition (Java EE) e il contenitore di tecnologia EJB (Enterprise JavaBeans). Ogni nuova istanza deve contenere un riferimento a un nodo che definisce l'host in cui risiederà l'istanza.

Nota:

Per creare un'istanza in un nodo di tipo CONFIG che rappresenta un host remoto, è necessario utilizzare il comando secondario create-local-instance(1). Per questa operazione non è possibile utilizzare Console di amministrazione.
Prima di iniziare

Verificare che il nodo in cui deve risiedere l'istanza esista. Per istruzioni, vedere Creare un nodo.

  1. Nella struttura ad albero di navigazione selezionare il nodo Istanze standalone.

    Viene visualizzata la pagina Istanze server standalone.

  2. Nella pagina Istanze server standalone, fare clic su Nuovo.

    Viene visualizzata la pagina Nuova istanza server standalone.

  3. Nel campo Nome digitare il nome per l'istanza.

    Il nome deve soddisfare i seguenti requisiti:

    • Il nome può contenere solo caratteri ASCII.

    • Il nome deve iniziare con una lettera, un numero o un carattere di sottolineatura.

    • Il nome può contenere solo i seguenti caratteri:

      • Lettere minuscole

      • Lettere maiuscole

      • Numeri

      • Trattino

      • Punto

      • Carattere di sottolineatura

    • Il nome deve essere univoco nel dominio e non deve essere il nome di un'altra istanza di GlassFish Server, di un cluster, di una configurazione denominata o di un nodo.

    • Il nome non deve essere domain, server né nessun'altra parola chiave riservata per GlassFish Server.

  4. Nell'elenco a discesa Nodo selezionare il nodo in cui risiederà l'istanza.

  5. Nell'elenco a discesa Configurazione selezionare la configurazione con nome che verrà utilizzata dall'istanza.

    Nota:

    Se si seleziona default-config, è necessario specificare nel passo successivo che la configurazione non è condivisa. La configurazione default-config non può essere condivisa.

  6. Specificare se si desidera condividere la configurazione utilizzata dall'istanza.

    • Se la configurazione non è condivisa, lasciare selezionata l'opzione Crea una copia della configurazione selezionata.

      La configurazione selezionata viene copiata al momento della creazione dell'istanza. Alla copia della configurazione viene assegnato il nome instance-name-config, dove instance-name è il nome dell'istanza da creare.

    • Se la configurazione è condivisa, selezionare l'opzione Crea riferimento alla configurazione selezionata.

      L'istanza utilizzerà la configurazione con nome esistente specificata.

  7. Fare clic su OK.
