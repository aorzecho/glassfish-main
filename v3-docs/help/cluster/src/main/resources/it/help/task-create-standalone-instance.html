Creare un'istanza standalone di GlassFish Server

Le istanze di GlassFish Server formano la base della distribuzione dell'applicazione. Un'istanza standalone non fa parte di un cluster. Ogni istanza appartiene a un dominio singolo e presenta la propria struttura di directory, la propria configurazione e le proprie applicazioni distribuite. Ogni istanza include inoltre il contenitore Web Java EE (Enterprise Edition) Platform e il contenitore di tecnologia EJB (Enterprise JavaBeans). Ogni nuova istanza deve contenere un riferimento a un nodo che definisce l'host in cui collocare l'istanza.

Nota - Per creare un'istanza in un nodo di tipo CONFIG che rappresenta un host remoto, è necessario utilizzare il comando secondario create-local-instance(1). Per questa operazione non è possibile utilizzare Administration Console.

Prima di cominciare

Verificare che il nodo in cui collocare l'istanza esista.

  1. Nella struttura ad albero selezionare il nodo Standalone Instances.

    Viene visualizzata la pagina Standalone Server Instances.

  2. Nella pagina Standalone Server Instances, fare clic su New.

    Viene visualizzata la pagina New Standalone Server Instance.

  3. Nel campo Name digitare il nome per l'istanza.

    Il nome deve soddisfare i seguenti requisiti:

    • Il nome può contenere solo caratteri ASCII.

    • Il nome deve iniziare con una lettera, un numero o un carattere di sottolineatura.

    • Il nome può contenere solo i seguenti caratteri:

      • Lettere minuscole

      • Lettere maiuscole

      • Numeri

      • Trattino

      • Punto

      • Carattere di sottolineatura

    • Il nome deve essere univoco nel dominio e non deve essere il nome di un'altra istanza di GlassFish Server, di un cluster, di una configurazione con nome o di un nodo.

    • Il nome non deve essere domain, server né nessun'altra parola chiave riservata per GlassFish Server.

  4. Nell'elenco a discesa Node selezionare il nodo in cui collocare l'istanza.
  5. Nell'elenco a discesa Configuration selezionare la configurazione con nome che verrà utilizzata dall'istanza.

    Nota - Se si seleziona default-config, è necessario specificare nel passaggio successivo che la configurazione non è condivisa. La configurazione default-config non può essere condivisa.

  6. Specificare se si desidera condividere la configurazione utilizzata dall'istanza.
    • Se la configurazione non è condivisa, lasciare selezionata l'opzione Make a Copy of the Selected Configuration.

      La configurazione selezionata viene copiata al momento della creazione dell'istanza. Alla copia della configurazione viene assegnato il nome instance-name-config, dove instance-name è il nome dell'istanza da creare.

    • Se la configurazione è condivisa, selezionare l'opzione Reference the Selected Configuration.

      L'istanza utilizzerà la configurazione con nome esistente specificata.

  7. Fare clic su OK.
