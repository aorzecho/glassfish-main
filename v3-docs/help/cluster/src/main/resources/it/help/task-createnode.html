Creare un nodo

Un nodo rappresenta un host in cui è installato il software GlassFish Server. È necessario un nodo per ogni host in cui risiedono le istanze di GlassFish Server.

  1. Nell'albero di navigazione selezionare il nodo Nodi.

    Viene visualizzata la pagina Nodi.

  2. Nella pagina Nodi fare clic su Nuovo.

    Viene visualizzata la pagina Nuovo nodo.

  3. Nel campo Nome digitare il nome scelto per il nodo.

  4. Nell'elenco a discesa Tipo selezionare il tipo di nodo da creare.

    • Per abilitare il supporto delle comunicazioni su protocollo remoto DCOM (Distributed Component Object Model) da parte del nodo, selezionare DCOM.

    • Per creare un nodo per il supporto della comunicazione SSH (Secure Shell), selezionare SSH.

    • Per creare un nodo che non supporta comunicazioni remote, selezionare CONFIG.

  5. Nel campo Host nodo immettere il nome dell'host rappresentato dal nodo.

    L'obbligatorietà di tale campo dipende dal tipo del nodo:

    • Se il nodo è di tipo CONFIG, è possibile omettere il nome dell'host. Tale nome potrà essere determinato al momento della creazione delle istanze che risiedono nel nodo.

    • Se il nodo è di tipo DCOM o SSH, è necessario specificare il nome dell'host. Altrimenti si verifica un errore.

  6. Nel campo Directory nodo digitare il percorso nella directory destinata a contenere le istanze GlassFish Server che vengono create nel nodo.

    Se il nodo è di tipo CONFIG, lasciare vuoto il campo. La directory del nodo potrà essere determinata al momento della creazione delle istanze che risiedono nel nodo.

    Se si digita un percorso relativo, il percorso è relativo alla directory as-install, dove as-install è la directory di installazione di base del software GlassFish Server sull'host.

  7. Nel campo Directory di installazione, digitare il percorso completo dell'elementopadre della directory di installazione di base del software GlassFish Server nell'host, ad esempio /export/glassfish3.

    Se il nodo è di tipo CONFIG, lasciare vuoto il campo. La directory di installazione potrà essere determinata al momento della creazione delle istanze che risiedono nel nodo.

  8. Terminare la creazione del nodo come richiesto in base al tipo di nodo.

    1. Se il nodo è di tipo CONFIG, fare clic su Salva.

    2. Se il nodo è di tipo DCOM, seguire le istruzioni in Terminare la creazione di un nodo DCOM.

    3. Se il nodo è di tipo SSH, seguire le istruzioni inTerminare la creazione di un nodo SSH .
