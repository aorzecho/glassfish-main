Creare un nodo

Un nodo rappresenta un host in cui è installato il software GlassFish Server. È necessario un nodo per ogni host in cui si trovano istanze di GlassFish Server.

  1. Nella struttura ad albero, selezionare il nodo Nodes.

    Viene visualizzata la pagina Nodes.

  2. Nella pagina Nodes fare clic su New.

    Viene visualizzata la pagina New Node.

  3. Nel campo Name digitare il nome scelto per il nodo.
  4. Nell'elenco a discesa Type selezionare il tipo di nodo da creare.
    • Per creare un nodo per il supporto della comunicazione SSH (Secure Shell), selezionare SSH.
    • Per creare un nodo che non supporta comunicazioni remote, selezionare CONFIG.
  5. Nel campo Node Host immettere il nome dell'host rappresentato dal nodo.

    L'obbligatorietà di tale campo dipende dal tipo del nodo:

    • Se il nodo è di tipo CONFIG, è possibile omettere il nome dell'host. Tale nome potrà essere determinato al momento della creazione delle istanze che risiedono nel nodo.

    • Se il nodo è di tipo SSH, è necessario specificare il nome dell'host. Altrimenti si verifica un errore.

  6. Nel campo Node Directory digitare il percorso nella directory destinata a contenere le istanze GlassFish Server che vengono create nel nodo.

    Se il nodo è di tipo CONFIG, lasciare vuoto il campo. La directory del nodo potrà essere determinata al momento della creazione delle istanze che risiedono nel nodo.

    Se si digita un percorso relativo, il percorso si riferisce alla directory as-install, dove as-install è la directory base di installazione del software GlassFish Server nell'host.

  7. Nel campo Installation Directory, digitare il percorso completo del livello superiore della directory base di installazione del software GlassFish Server nell'host, ad esempio /export/glassfish3/.

    Se il nodo è di tipo CONFIG, lasciare vuoto il campo. La directory di installazione potrà essere determinata al momento della creazione delle istanze che risiedono nel nodo.

  8. Per forzare la creazione del nodo nella configurazione del server di amministrazione del dominio (DAS, Domain Administration Server) anche in caso di errore di convalida dei parametri del nodo, abilitare l'opzione Force.
  9. Nel campo SSH Port digitare la porta da utilizzare per le connessioni SSH all'host del nodo.
  10. Nel campo SSH User Name digitare il nome dell'utente nell'host del nodo che deve eseguire il processo per la connessione all'host tramite SSH.
  11. Specificare la modalità di autenticazione dell'utente SSH per il login all'host del nodo.
    • Per specificare l'autenticazione tramite un file di chiave privata SSH, attenersi alla procedura seguente:
      1. Nell'elenco a discesa SSH User Authentication selezionare Key File.
      2. Nel campo Key File digitare il percorso assoluto del file della chiave privata SSH per l'utente specificato nell'opzione SSH User Name.

        Nota - GlassFish Server supporta anche l'autenticazione della password tramite la voce AS_ADMIN_SSHPASSWORD nel file delle password. Il file delle password è specificato nell'opzione --paswordfile dell'utility asadmin(1M).

    • Per specificare l'autenticazione tramite una password memorizzata nella configurazione del DAS, attenersi alla procedura seguente:

      Avvertenza - La password viene memorizzata come testo non crittografato nella configurazione del DAS. Per una maggiore sicurezza, utilizzare un file di chiave o un alias di password.

      1. Nell'elenco a discesa SSH User Authentication selezionare Password.
      2. Nel campo SSH Password digitare la password che l'utente SSH utilizzerà per il login all'host del nodo.
      3. Nel campo Confirm SSH Password ridigitare la nuova password.
    • Per specificare l'autenticazione tramite un alias di password, attenersi alla procedura seguente:
      1. Nell'elenco a discesa SSH User Authentication selezionare Password Alias.
      2. Nell'elenco a discesa Password Alias selezionare un alias esistente.
  12. Fare clic su Save.
