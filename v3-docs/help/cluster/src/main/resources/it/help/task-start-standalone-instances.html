Avviare le istanze standalone di GlassFish Server

Prima di iniziare

Verificare che siano soddisfatti i seguenti prerequisiti:

  • Tutti i nodi in cui risiedono le istanze sono abilitati per la comunicazione remota.

  • L'utente del server di amministrazione del dominio (DAS, Domain Administration Server) può utilizzare il protocollo remoto DCOM (Distributed Component Object Model) o SSH (Secure Shell) per eseguire il login nell'host per qualsiasi nodo in cui risiedono le istanze del cluster.

Se uno o più prerequisiti non sono soddisfatti, avviare ogni istanza singolarmente dalla riga di comando utilizzando il comando secondario start-local-instance(1).

  1. Nella struttura ad albero di navigazione selezionare il nodo Istanze standalone.

    Viene visualizzata la pagina Istanze server standalone.

  2. Per ogni istanza da avviare, selezionare la casella di controllo corrispondente nella tabella Istanze server.

  3. Fare clic su Avvia.

    Viene richiesto di confermare l'avvio delle istanze.

  4. Confermare l'avvio delle istanze.

    Lo stato di ogni istanza viene aggiornato nella tabella Istanze server quando l'istanza viene avviata.
