Eseguire la migrazione dei timer EJB

Se si arresta un'istanza di GlassFish Server, viene interrotta l'esecuzione dei timer di tecnologia EJB (Enterprise JavaBeans) installati nell'istanza. Se l'istanza è stata arrestata in modo anomalo o inatteso, per abilitare l'esecuzione dei timer quando l'istanza viene interrotta, è necessario eseguirne la migrazione in una istanza del server in esecuzione nel cluster.

  1. Nell'albero di navigazione espandere il nodo Cluster.

  2. Nel nodo Cluster della struttura ad albero di navigazione, selezionare il cluster.

    Viene visualizzata la pagina Informazioni generali per il cluster.

  3. Nella pagina Informazioni generali fare clic su Esegui migrazione timer EJB.

    Viene visualizzata la pagina Esegui migrazione timer EJB.

    Nota:

    Se tutte le istanze nel cluster sono in esecuzione, il pulsante Esegui migrazione timer EJB è disattivato.

    Per assistere l'utente nella migrazione dei timer, la pagina Esegui migrazione timer EJB contiene una tabella di timer EJB con le informazioni riportate di seguito.

    Nome server

    Lista delle istanze di GlassFish Server nel cluster.

    Conteggio timer

    Numero di timer persistenti in ciascuna istanza.

  4. Nella pagina Esegui migrazione timer EJB selezionare l'istanza dalla quale e alla quale eseguire la migrazione dei timer.

    1. Nell'elenco a discesa Origine selezionare l'istanza arrestata da cui eseguire la migrazione dei timer.

    2. Nell'elenco a discesa Destinazione selezionare l'istanza in esecuzione in cui eseguire la migrazione dei timer.

    3. Fare clic su OK.

  5. Arrestare e riavviare l'istanza alla quale sono stati migrati i timer EJB.

    Se l'istanza di origine è in esecuzione o l'istanza di destinazione non è in esecuzione, nell'Console di amministrazione viene visualizzato un messaggio di errore.
