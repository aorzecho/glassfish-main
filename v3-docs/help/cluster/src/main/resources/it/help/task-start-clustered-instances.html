Avviare le istanze di GlassFish Server in cluster

Prima di iniziare

Verificare che siano soddisfatti i seguenti prerequisiti:

  • Tutti i nodi in cui risiedono le istanze sono abilitati per la comunicazione remota.

  • L'utente del server di amministrazione del dominio (DAS, Domain Administration Server) può utilizzare SSH (Secure Shell) per eseguire il login sull'host per qualsiasi nodo in cui risiedono le istanze.

Se uno o più prerequisiti non sono soddisfatti, avviare ogni istanza singolarmente dalla riga di comando utilizzando il comando secondario start-local-instance(1).

  1. Nell'albero di navigazione espandere il nodo Cluster.

  2. Dopo aver espanso il nodo Cluster, selezionare il nodo per tale cluster.

    Viene visualizzata la pagina Informazioni generali per il cluster.

  3. Nella pagina Informazioni generali, fare clic sulla scheda Istanze.

    Viene visualizzata la pagina Istanze server in cluster per il cluster.

  4. Per ogni istanza da avviare, selezionare la casella di controllo corrispondente nella tabella Istanze server.

  5. Fare clic su Avvia.

    Viene richiesto di confermare l'avvio delle istanze.

  6. Confermare l'avvio delle istanze.

    Lo stato di ogni istanza viene aggiornato nella tabella Istanze server quando l'istanza viene avviata.
