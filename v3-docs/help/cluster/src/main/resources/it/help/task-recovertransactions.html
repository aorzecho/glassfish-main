Recuperare le transazioni per un'istanza di GlassFish Server in cluster

  1. Nell'albero di navigazione espandere il nodo Cluster.

  2. Dopo aver espanso il nodo Cluster, selezionare il nodo per tale cluster.

    Viene visualizzata la pagina Informazioni generali per il cluster.

  3. Nella pagina Informazioni generali, fare clic sulla scheda Istanze.

    Viene visualizzata la pagina Istanze server in cluster per il cluster.

  4. Selezionare l'istanza per la quale è necessario il recupero delle transazioni

    Viene visualizzata la pagina Informazioni generali per l'istanza.

  5. Nella pagina Informazioni generali fare clic su Recupera transazioni.

    Viene visualizzata la pagina Recupera transazioni.

    Il contenuto della pagina varia a seconda che l'istanza sia in esecuzione o meno.

    • Se l'istanza è in esecuzione, la pagina Recupera transazioni contiene solo il testo Il recupero viene eseguito dallo stesso server in esecuzione e il pulsante OK.

    • Se l'istanza non è in esecuzione, la pagina Recupera transazioni contiene opzioni aggiuntive.

  6. Nell'elenco a discesa Destinazione selezionare un'altra istanza nello stesso cluster in grado di ripristinare le transazioni nell'istanza server corrente.

  7. Nel campo Directory log delle transazioni digitare la directory in cui sono memorizzati i log di transazione.

    L'impostazione predefinita è /logs/tx nella directory di dominio. I log delle transazioni sono memorizzati nella sottodirectory tx della directory specificata nell'impostazione Posizione log delle transazioni del servizio transazioni relativo alla configurazione cluster. Il valore predefinito Posizione log delle transazioni è la directory specificata nel campo Radice log disponibile nella pagina Attributi dominio all'interno della scheda Avanzate del nodo GlassFish Server.

  8. Fare clic su OK.
