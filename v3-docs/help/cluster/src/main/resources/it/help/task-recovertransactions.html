Ripristinare transazioni per un'istanza di GlassFish Server in cluster

  1. Nella struttura ad albero, espandere il nodo Clusters.
  2. Dopo aver espanso il nodo Clusters, selezionare il nodo per tale cluster.

    Viene visualizzata la pagina General Information per il cluster.

  3. Nella pagina General Information, fare clic sulla scheda Instances.

    Viene visualizzata la pagina Clustered Server Instances per il cluster.

  4. Selezionare l'istanza per la quale è necessario il ripristino delle transazioni

    Viene visualizzata la pagina General Information per l'istanza.

  5. Nella pagina General Information fare clic su Recover Transactions.

    Viene visualizzata la pagina Recover Transactions.

    Il contenuto della pagina varia a seconda che l'istanza sia in esecuzione o meno.

    • Se l'istanza è in esecuzione, la pagina Recover Transactions contiene solo il testo in cui viene indicato che il ripristino viene eseguito dal server in esecuzione e il pulsante OK.

    • Se l'istanza non è in esecuzione, la pagina Recover Transactions contiene opzioni aggiuntive.

  6. Nell'elenco a discesa Destination selezionare un'altra istanza nello stesso cluster in grado di ripristinare le transazioni nell'istanza server corrente.
  7. Nel campo Transaction Log Directory digitare la directory in cui sono memorizzati i log di transazione.

    L'impostazione predefinita è /logs/tx nella directory di dominio. I log delle transazioni sono memorizzati nella sottodirectory tx della directory specificata nell'impostazione Transaction Log Location del servizio transazioni relativo alla configurazione cluster. Il valore predefinito Transaction Log Location è la directory specificata nel campo Log Root disponibile nella pagina Domain Attributes all'interno della scheda Advanced del nodo GlassFish Server.

  8. Fare clic su OK.
