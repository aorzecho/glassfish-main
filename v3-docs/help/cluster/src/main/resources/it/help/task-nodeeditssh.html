Terminare la creazione di un nodo SSH

Questo task è la continuazione per un nodo SSH di Modificare un nodo .

  1. Per forzare l'aggiornamento del nodo nella configurazione del server di amministrazione del dominio (DAS, Domain Administration Server) anche in caso di errore di convalida dei parametri del nodo, abilitare l'opzione Imponi.

  2. Nel campo Porta SSH digitare la porta da utilizzare per le connessioni SSH all'host del nodo.

    Se si modifica il tipo di nodo da CONFIG a SSH e si lascia il campo vuoto, l'opzione Porta SSH viene impostata su 22.

  3. Nel campo Nome utente SSH digitare il nome dell'utente nell'host del nodo che deve eseguire il processo per la connessione all'host tramite SSH.

    Se si modifica il tipo di nodo da CONFIG a SSH e si lascia il campo vuoto, l'opzione Nome utente SSH viene impostata sull'utente che esegue il processo DAS.

  4. Specificare la modalità di autenticazione dell'utente SSH per il login all'host del nodo.

    • Per specificare l'autenticazione tramite un file di chiave privata SSH, attenersi alla procedura riportata di seguito.

      1. Nell'elenco a discesa Autenticazione utente SSH selezionare File di chiave.

      2. Se si desidera che GlassFish Server imposti una chiave SSH sull'host del nodo, abilitare l'opzione Imposta SSH.

        GlassFish Server imposta la chiave SSH copiando file di chiave esistenti dall'host di DAS o, se necessario, generando i file prima di copiarli.

        I campi Genera chiave e File di chiave pubblica vengono aggiunti alla finestra.

      3. Se si desidera che GlassFish Server generi i file di chiave SSH prima di copiare i file nell'host del nodo, abilitare l'opzione Genera chiave.

      4. Nel campo Password dell'utente SSH digitare la password che l'utente SSH utilizzerà per il login all'host del nodo.

      5. Nel campo File di chiave pubblica digitare il percorso assoluto del file di chiave pubblica SSH per l'utente specificato nell'opzione Nome utente SSH.

      6. Nel campo File di chiave digitare il percorso assoluto del file di chiave privata SSH per l'utente specificato nell'opzione Nome utente SSH.

        Nota:

        GlassFish Server supporta inoltre l'autenticazione della password mediante la voce AS_ADMIN_SSHPASSWORD nel file delle password. Il file delle password è specificato nell'opzione --passwordfile della utility asadmin(1).

        Se si modifica il tipo di nodo in SSH e si lascia il campo vuoto, l'opzione File di chiave viene impostata su un file di chiave nella directory .ssh dell'utente. Se vengono rilevati più file di chiave, GlassFish Server utilizza l'ordine di preferenza riportato di seguito.

        id_rsa
id_dsa
identity

    • Per specificare l'autenticazione tramite una password memorizzata nella configurazione del DAS, attenersi alla procedura riportata di seguito.

      Attenzione:

      La password viene memorizzata come testo in chiaro nella configurazione del DAS. Per una maggiore sicurezza, utilizzare un file di chiave o un alias password.

      1. Nell'elenco a discesa Autenticazione utente SSH selezionare Password.

      2. Nel campo Password dell'utente SSH digitare la password che l'utente SSH utilizzerà per il login all'host del nodo.

      3. Nel campo Conferma password digitare nuovamente la password.

    • Per specificare l'autenticazione tramite un alias password, attenersi alla procedura riportata di seguito.

      1. Nell'elenco a discesa Autenticazione utente SSH selezionare Alias password.

      2. Nell'elenco a discesa Alias password selezionare un alias esistente.

  5. Fare clic su Salva.
