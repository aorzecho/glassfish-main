Terminare l'aggiornamento di un nodo DCOM

Questo task è la continuazione per un nodo DCOM di Modificare un nodo.

  1. Per forzare la creazione del nodo nella configurazione del server di amministrazione del dominio (DAS, Domain Administration Server) anche in caso di errore di convalida dei parametri del nodo, abilitare l'opzione Imponi.

  2. Se si desidera che GlassFish Server esegua il test della connessione su DCOM all'host del nodo prima di creare il nodo, effettuare le operazioni riportate di seguito.

    1. Abilitare e convalidare l'opzione DCOM.

    2. Nel campo Directory di test remota, digitare il nome della cartella sll'host remoto in cui GlassFish Server deve creare un file per eseguire il test della connessione.

  3. Nel campo Dominio Windows Domain digitare il nome del dominio Windows che contiene l'utente specificato dall'opzione Nome utente Windows .

    Se si modifica il tipo di nodo in DCOM e si lascia vuoto questo campo, l'opzione Dominio Windows viene impostata sul nome dell'host su cui Console di amministrazione è in esecuzione.

  4. Nel campo Nome utente Windows digitare il nome dell'utente nell'host del nodo che deve eseguire il processo per la connessione all'host tramite DCOM.

    Se si modifica il tipo di nodo da CONFIG a DCOM e si lascia il campo vuoto, l'opzione Nome utente Windows viene impostata sull'utente che sta eseguendo il processo DAS.

  5. Specificare la modalità di autenticazione dell'utente Windows per il login all'host del nodo.

    Nota:

    GlassFish Server supporta inoltre l'autenticazione della password mediante la voce AS_ADMIN_WINDOWSPASSWORD nel file delle password. Il file delle password è specificato nell'opzione --passwordfile della utility asadmin(1).

    1. Per specificare l'autenticazione tramite una password memorizzata nella configurazione del DAS, attenersi alla procedura riportata di seguito.

      Attenzione:

      La password viene memorizzata come testo in chiaro nella configurazione del DAS. Per aumentare la sicurezza, utilizzare un alias password.

      1. Nell'elenco a discesa Autenticazione utente Windows selezionare Password.

      2. Nel campo Password dell'utente Windows digitare la password che l'utente SSH utilizzerà per il login all'host del nodo.

      3. Nel campo Conferma password digitare nuovamente la password.

    2. Per specificare l'autenticazione tramite un alias password, attenersi alla procedura riportata di seguito.

      1. Nell'elenco a discesa Autenticazione utente Windows selezionare Alias password.

      2. Nell'elenco a discesa Alias password selezionare un alias esistente.

  6. Fare clic su Salva.
