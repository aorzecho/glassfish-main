Configurare le impostazioni di GMS per un cluster

Il servizio GMS (Group Management Service) è un servizio in-process che offre monitoraggio del cluster, appartenenza del cluster e servizi di comunicazione di gruppo. Esegue le funzioni seguenti:

  • Notifica ai moduli registrati in un'istanza di GlassFish Server l'aggiunta o la rimozione di una o più istanze membro da un cluster oppure un errore sospetto o confermato di tali istanze.

  • Consente di inviare e ricevere messaggi tra processi di un gruppo.

  1. Nell'albero di navigazione espandere il nodo Cluster.

  2. Selezionare il cluster per il quale configurare le impostazioni del servizio GMS.

    Viene visualizzata la pagina Informazioni generali per il cluster.

  3. Selezionare la casella di controllo GMS abilitato per abilitare il servizio GMS per il cluster.

    Questa opzione è selezionata per impostazione predefinita. Per disabilitare il servizio GMS per il cluster, rimuovere il segno di spunta dalla casella di controllo GMS abilitato.

    Il servizio GMS, se abilitato, è avviato in ogni istanza del server nel cluster e nel server di amministrazione del dominio (DAS, Domain Administration Server). Il DAS partecipa a ciascun cluster con GMS abilitato.

  4. Nel campo Porta multicast immettere la porta utilizzata dal servizio GMS per restare in attesa degli eventi di gruppo.

    Specificare un numero di porta UDP standard nell'intervallo compreso tra 2048 e 32000. Il valore predefinito è un valore generato automaticamente in questo intervallo.

  5. Nel campo Porta multicast immettere l'indirizzo utilizzato dal servizio Group Management Service per restare in attesa degli eventi di gruppo.

    Specificare un indirizzo multicast IP classe D. Gli indirizzi IP di classe D rientrano nell'intervallo compreso tra 224.0.0.0 e 239.255.255.255 inclusi. L'indirizzo 224.0.0.0 è riservato e non deve essere utilizzato. L'impostazione predefinita è 228.9.XX. YY, dove XX e YY rappresentano valori indipendenti generati automaticamente inclusi tra 0 e 255.

  6. Nel campo Associa indirizzo di interfaccia immettere l'indirizzo dell'interfaccia di rete per il servizio GMS se sono disponibili più interfacce di rete.

    L'impostazione consente di configurare l'interfaccia di rete da utilizzare per il servizio GMS se ne è presente più di una su uno dei computer host delle istanze del server nel cluster. Il valore deve essere un indirizzo IP di interfaccia di rete locale. Per impostazione predefinita vengono utilizzati tutti gli indirizzi di interfaccia di rete pubblici.

    Per specificare un indirizzo valido per tutte le istanze di GlassFish Server nel cluster, utilizzare una proprietà di sistema per impostare l'indirizzo singolarmente per ciascuna istanza.

    Ad esempio, creare la proprietà di sistema GMS-BIND-INTERFACE-ADDRESS-cluster-name in DAS. Quindi impostare l'opzione Associa indirizzo di interfaccia su ${GMS-BIND-INTERFACE-ADDRESS-cluster-name} per specificare la proprietà di sistema. Infine, per ogni istanza nel cluster, impostare la proprietà di sistema GMS-BIND-INTERFACE-ADDRESS-cluster-name per l'indirizzo dell'interfaccia di rete necessario nel computer dell'istanza.

  7. Per specificare proprietà aggiuntive, selezionare la scheda Proprietà e la scheda secondaria Proprietà cluster.

    • Per aggiungere una proprietà, fare clic sul pulsante Aggiungi proprietà. Nella riga vuota visualizzata, digitare il nome della proprietà nel campo Nome e il valore della proprietà nel campo Valore.

    • Per modificare una proprietà, è necessario modificare il valore del campo Valore della proprietà.

    • Per eliminare una proprietà, selezionare la casella di controllo a sinistra del campo Nome relativo alla proprietà da eliminare, quindi fare clic sul pulsante Elimina proprietà.

  8. Fare clic su Salva.
