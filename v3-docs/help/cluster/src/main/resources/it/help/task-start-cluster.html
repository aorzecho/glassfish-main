Avviare un cluster

Se si avvia un cluster, vengono avviate tutte le istanze di GlassFish Server nel cluster che non sono già in esecuzione.

Prima di iniziare

Verificare che siano soddisfatti i seguenti prerequisiti:

  • Tutti i nodi in cui risiedono istanze del cluster sono abilitati per la connessione remota.

  • L'utente del server di amministrazione del dominio (DAS, Domain Administration Server) può utilizzare il protocollo remoto DCOM (Distributed Component Object Model) o SSH (Secure Shell) per eseguire il login nell'host per qualsiasi nodo in cui risiedono le istanze del cluster.

Se uno o più prerequisiti non vengono soddisfatti, avviare il cluster avviando ciascuna istanza singolarmente dalla riga di comando tramite il comando secondario start-local-instance(1).

  1. Nell'albero di navigazione selezionare il nodo Cluster.

    Viene visualizzata la pagina Cluster.

  2. Nella pagina Cluster selezionare la casella di controllo in corrispondenza del nome del cluster.

  3. Fare clic su Avvia cluster.

    Viene richiesto di confermare l'avvio del cluster.

  4. Confermare l'avvio del cluster.

    Il cluster viene avviato. Lo stato di ogni istanza del cluster è aggiornato nella pagina Cluster quando l'istanza viene avviata.
