Avviare un cluster

Se si avvia un cluster, vengono avviate tutte le istanze di GlassFish Server nel cluster che non sono già in esecuzione.

Prima di cominciare

Verificare che siano soddisfatti i seguenti prerequisiti:

  • Tutti i nodi in cui sono presenti istanze del cluster sono abilitati per la connessione remota.

  • L'utente del server di amministrazione del dominio (DAS, Domain Administration Server) può utilizzare SSH (Secure Shell) per eseguire il login nell'host per qualsiasi nodo in cui si trovano istanze del cluster.

Se uno o più prerequisiti non sono soddisfatti, avviare il cluster avviando ciascuna istanza singolarmente dalla riga di comando tramite il comando secondario start-local-instance(1).

  1. Nella struttura ad albero, selezionare il nodo Clusters.

    Viene visualizzata la pagina Clusters.

  2. Nella pagina Clusters selezionare la casella di controllo in corrispondenza del nome del cluster.
  3. Fare clic su Start Cluster.

    Viene richiesto di confermare l'avvio del cluster.

  4. Confermare l'avvio del cluster.

    Il cluster viene avviato. Lo stato di ogni istanza del cluster è aggiornato nella pagina Clusters quando l'istanza viene avviata.
