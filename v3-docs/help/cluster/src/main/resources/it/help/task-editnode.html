Modificare un nodo

La modifica di un nodo causa l'aggiornamento dei dati di configurazione del nodo GlassFish Server. La modifica di un nodo consente inoltre di modificare il tipo di nodo da CONFIG a DCOM o SSH per abilitare le comunicazioni remote per il nodo.

Se si modifica il tipo di nodo da CONFIG a DCOM e si lascia vuoto uno dei campi indicati di seguito, vengono applicati i valori predefiniti.

  • Dominio Windows

  • Utente Windows

Se si modifica il tipo di nodo da CONFIG in SSH e si lascia vuoto uno dei campi indicati di seguito, vengono applicati i valori predefiniti.

  • Porta SSH

  • Nome utente SSH

  • File di chiave

  1. Nell'albero di navigazione espandere il nodo Nodi.

  2. Nella lista di nodi, selezionare il nodo da modificare.

    Viene visualizzata la pagina Modifica nodo.

  3. Nell'elenco a discesa Tipo selezionare il tipo di nodo da creare.

    • Per abilitare il supporto delle comunicazioni su protocollo remoto DCOM (Distributed Component Object Model) da parte del nodo, selezionare DCOM.

    • Per abilitare il nodo per il supporto della comunicazione SSH (Secure Shell), selezionare SSH.

    • Per rimuovere il supporto della configurazione remota dal nodo, selezionare CONFIG.

      Se si seleziona CONFIG, non vengono visualizzate opzioni per la configurazione SSH o DCOM. Se il tipo corrente del nodo è DCOM o SSH e si seleziona CONFIG, i dati di configurazione DCOM o SSH vengono rimossi dalla configurazione del nodo quando si salvano le modifiche.

  4. Nel campo Host nodo digitare il nome dell'host che il nodo aggiornato deve rappresentare.

    L'obbligatorietà di tale campo dipende dal tipo del nodo:

    • Se il nodo è di tipo CONFIG, è possibile omettere il nome dell'host. Tale nome potrà essere determinato al momento della creazione delle istanze che risiedono nel nodo.

    • Se il nodo è di tipo DCOM o SSH, è necessario specificare il nome dell'host. Altrimenti si verifica un errore.

  5. Nel campo Directory nodo digitare il percorso nella directory destinata a contenere le istanze GlassFish Server che vengono create nel nodo.

    Se si digita un percorso relativo, il percorso è relativo alla directory as-install, dove as-install è la directory di installazione di base del software GlassFish Server sull'host.

  6. Nel campo Directory di installazione, digitare il percorso completo dell'elementopadre della directory di installazione di base del software GlassFish Server nell'host, ad esempio /export/glassfish3.

  7. Terminare l'aggiornamento del nodo come richiesto in base al tipo di nodo.

    1. Se il nodo è di tipo CONFIG, fare clic su Salva.

    2. Se il nodo è di tipo DCOM, seguire le istruzioni in Terminare l'aggiornamento di un nodo DCOM.

    3. Se il nodo è di tipo SSH, seguire le istruzioni inTerminare la creazione di un nodo SSH .
