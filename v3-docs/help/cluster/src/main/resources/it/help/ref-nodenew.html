Nuovo nodo

Nella pagina Nuovo nodo è possibile creare un nodo di GlassFish Server. Un nodo rappresenta un host in cui è installato il software GlassFish Server. È necessario un nodo per ogni host in cui risiedono le istanze di GlassFish Server.

Le opzioni contenute nella pagina Nuovo nodo dipendono dal tipo di nodo.

Opzioni per tutti i tipi di nodi

Le opzioni riportate di seguito sono disponibili per tutti i tipi di nodo.

Nome

Nome del nodo.

Il nome deve soddisfare i seguenti requisiti:

  • Il nome può contenere solo caratteri ASCII.

  • Il nome deve iniziare con una lettera, un numero o un carattere di sottolineatura.

  • Il nome può contenere solo i seguenti caratteri:

    • Lettere minuscole

    • Lettere maiuscole

    • Numeri

    • Trattino

    • Punto

    • Carattere di sottolineatura

  • Il nome deve essere univoco nel dominio e non deve essere il nome di un altro cluster, di una configurazione denominata, di un'istanza di GlassFish Server o di un nodo.

  • Il nome non deve essere domain, server né nessun'altra parola chiave riservata per GlassFish Server.

Tipo

Elenco a discesa che consente di specificare il tipo di nodo. Di seguito sono indicati i tipi possibili:

CONFIG

Il nodo non supporta la comunicazione remota. Se è selezionato CONFIG, non è visualizzata alcuna opzione per la configurazione SSH o per la configurazione DCOM e il campo Directory di installazione è vuoto.

DCOM

Il nodo supporta comunicazioni sul protocollo remoto DCOM (Distributed Component Object Model). Il protocollo DCOM è disponibile solo sui sistemi Windows.

SSH

Il nodo supporta la comunicazione su SSH (Secure Shell).

Host nodo

Nome dell'host che il nodo rappresenta. L'obbligatorietà di tale campo dipende dal tipo del nodo:

  • Se il nodo è di tipo CONFIG, è possibile omettere il nome dell'host. Tale nome potrà essere determinato al momento della creazione delle istanze che risiedono nel nodo.

  • Se il nodo è di tipo DCOM o SSH, è necessario specificare il nome dell'host. Altrimenti si verifica un errore.

Directory nodo

Percorso della directory destinata a contenere le istanze di GlassFish Server create nel nodo. Se viene specificato un percorso relativo, il percorso è relativo alla directory as-install.

Il valore predefinito dipende dal tipo del nodo:

  • Se il nodo è di tipo CONFIG, per il nodo non viene specificata alcuna directory per le istanze nella configurazione del server di amministrazione del dominio (DAS, Domain Administration Server). La directory del nodo potrà essere determinata al momento della creazione delle istanze che risiedono nel nodo.

  • Se il tipo di nodo è DCOM, l'impostazione predefinita è as-install\nodes, dove as-install è la directory di installazione di base del software GlassFish Server sull'host.

  • Se il tipo di nodo è SSH, l'impostazione predefinita è as-install/nodes, dove as-install è la directory di installazione di base del software GlassFish Server sull'host.

Directory di installazione

Percorso completo dell'elementopadre della directory di installazione di base del software GlassFish Server nell'host, ad esempio /export/glassfish3.

Il valore predefinito dipende dal tipo del nodo:

  • Se il nodo è di tipo CONFIG, per il nodo non viene specificata alcuna directory di installazione nella configurazione del DAS. La directory di installazione potrà essere determinata al momento della creazione delle istanze che risiedono nel nodo.

  • Se il nodo è DCOM o SSH, l'impostazione predefinita è il livello superiore della directory di installazione di base predefinita del software GlassFish Server per DAS. Questa impostazione predefinita è utile solo se GlassFish Server è installato nello stesso percorso in tutti gli host.

Opzioni aggiuntive per i nodi DCOM

Le opzioni riportate di seguito sono disponibili solo se il campo Tipo è impostato su DCOM.

Installa GlassFish Server

Se si seleziona l'opzione Abilitato, DAS eseguirà la copia del software GlassFish Server dall'host di DAS all'host del nodo. Per copiare il software GlassFish Server nell'host, DAS deve poter contattare l'host tramite DCOM.

Imponi

Se viene selezionata l'opzione Abilitato, il nodo viene creato nella configurazione DAS anche se la convalida dei parametri del nodo non riesce. Per convalidare i parametri di un nodo, DAS deve poter contattare l'host del nodo tramite DCOM.

Convalida DCOM

Se si seleziona l'opzione Abilitato, GlassFish Server esegue il test della connessione su DCOM sull'host del nodo prima di creare il nodo. Come parte del test, GlassFish Server copia uno script denominato delete_me.bat sull'host.

Se si seleziona questa opzione, il campo Directory di test remota viene attivato per consentire di specificare la directory in cui GlassFish Server deve copiare lo script.

Directory di test remota

La cartella nell'host remoto in cui GlassFish Server deve creare il file delete_me.bat. L'utente specificato dall'opzione Nome utente Windows deve disporre di accesso in scrittura a questa cartella. In caso contrario si verifica un errore. L'impostazione predefinita è C:\.

Dominio Windows

Nome del dominio Windows che contiene l'utente specificato dall'opzione Nome utente Windows. L'impostazione predefinita è il nome dell'host su cui Console di amministrazione è in esecuzione.

Nome utente Windows

L'utente sull'host del nodo che eseguirà il processo per la connessione all'host tramite DCOM. L'impostazione predefinita è l'utente che esegue il processo DAS. Se l'opzione Host nodo è impostata su localhost-domain, l'opzione Nome utente Windows viene ignorata.

Autenticazione utente Windows

Elenco a discesa che consente di specificare la modalità di autenticazione dell'utente Windows per il login all'host del nodo. I metodi possibili per l'autenticazione sono riportati di seguito.

Password

L'utente Windows viene autenticato tramite una password memorizzata nella configurazione di DAS.

Attenzione:

La password viene memorizzata come testo in chiaro nella configurazione del DAS. Per una maggiore sicurezza, utilizzare un file di chiave o un alias password.

Se si seleziona questa opzione, vengono attivati i campi Password dell'utente Windows e Conferma password per consentire l'immissione della password.

Alias password

L'utente Windows viene autenticato tramite un alias password. Se si seleziona questa opzione, selezionare l'alias password dall'elenco a discesa Alias password.

Password dell'utente Windows

Password che l'utente Windows utilizzerà per il login all'host del nodo.

Conferma password

Conferma della password che l'utente Windows utilizzerà per il login all'host del nodo. Il contenuto del campo deve corrispondere esattamente al contenuto del campo Password dell'utente Windows.

Alias password

Elenco a discesa di alias password definiti per il dominio.

Opzioni aggiuntive per i nodi SSH

Le opzioni riportate di seguito sono disponibili solo se il campo Tipo è impostato su SSH:

Installa GlassFish Server

Se si seleziona l'opzione Abilitato, DAS copierà il software GlassFish Server dall'host di DAS all'host del nodo. Per copiare il software GlassFish Server nell'host, DAS deve poter contattare l'host tramite SSH.

Imponi

Se viene selezionata l'opzione Abilitato, il nodo viene creato nella configurazione DAS anche se la convalida dei parametri del nodo non riesce. Per convalidare i parametri di un nodo, il DAS deve poter contattare l'host del nodo tramite SSH.

Porta SSH

Porta da utilizzare per le connessione SSH all'host del nodo. Il valore predefinito è 22. Se l'opzione Host nodo è impostata su localhost-domain, l'opzione Porta SSH viene ignorata.

Nome utente SSH

Utente nell'host del nodo che deve eseguire il processo per la connessione all'host tramite SSH. Il valore predefinito è l'utente che esegue il processo DAS. Per garantire che il DAS possa leggere il file della chiave privata SSH di questo utente, specificare l'utente che esegue il processo DAS. Se l'opzione Host nodo è impostata su localhost-domain, l'opzione Nome utente viene ignorata.

Autenticazione utente SSH

Elenco a discesa che consente di specificare la modalità di autenticazione dell'utente SSH per il login all'host del nodo. I metodi possibili per l'autenticazione sono i seguenti:

File di chiave

L'utente SSH viene autenticato tramite un file di chiave privata SSH. Se si seleziona questa opzione, specificare il percorso assoluto del file nel campo File di chiave.

Password

L'utente SSH viene autenticato tramite una password memorizzata nella configurazione del DAS.

Attenzione:

La password viene memorizzata come testo in chiaro nella configurazione del DAS. Per una maggiore sicurezza, utilizzare un file di chiave o un alias password.

Se si seleziona questa opzione, vengono attivati i campi Password dell'utente SSH e Conferma password per consentire l'immissione della password.

Alias password

L'utente SSH viene autenticato tramite un alias password. Se si seleziona questa opzione, selezionare l'alias password dall'elenco a discesa Alias password.

Imposta SSH

Se si seleziona l'opzione Abilitato, GlassFish Server imposta una chiave SSH sull'host del nodo. GlassFish Server imposta la chiave SSH copiando file di chiave esistenti dall'host di DAS o, se necessario, generando i file prima di copiarli. Questa opzione è disponibile solo se l'opzione Autenticazione utente SSH è impostata su File di chiave.

Genera chiave

Se si seleziona l'opzione Abilitato, GlassFish Server genera i file di chiave SSH prima di copiare i file nell'host locale.

Password dell'utente SSH

Password che l'utente SSH utilizzerà per il login all'host del nodo.

Conferma password

Conferma della password che l'utente SSH utilizzerà per il login all'host del nodo. Il contenuto del campo deve corrispondere esattamente al contenuto del campo Password dell'utente SSH.

Alias password

Elenco a discesa di alias password definiti per il dominio.

File di chiave pubblica

Percorso assoluto del file di chiave pubblica SSH per l'utente specificato dall'opzione Nome utente SSH. Il contenuto del file di chiave pubblica viene aggiunto al file .ssh/authorized_keys dell'utente sull'host del nodo. Se il file .ssh/authorized_keys non esiste sull'host, GlassFish Server lo crea.

Il percorso del file della chiave deve essere raggiungibile dal DAS e il file stesso deve essere leggibile per il DAS.

Il valore predefinito è un file di chiave nella directory .ssh dell'utente. Se vengono rilevati più file di chiave, GlassFish Server utilizza l'ordine di preferenza riportato di seguito.

  1. id_rsa

  2. id_dsa

  3. identità

File di chiave

Percorso assoluto del file di chiave privata SSH per l'utente specificato dall'opzione Nome utente SSH. Questo file viene usato per l'autenticazione al daemon sshd nell'host del nodo.

Nota:

GlassFish Server supporta inoltre l'autenticazione della password mediante la voce AS_ADMIN_SSHPASSWORD nel file delle password. Il file delle password è specificato nell'opzione --passwordfile della utility asadmin(1).

Se il file della chiave privata SSH è protetto da una passphrase, il file delle password deve contenere la voce AS_ADMIN_SSHKEYPASSPHRASE.

Il percorso del file della chiave deve essere raggiungibile dal DAS e il file stesso deve essere leggibile per il DAS.

Il valore predefinito è un file di chiave nella directory .ssh dell'utente. Se vengono rilevati più file di chiave, GlassFish Server utilizza l'ordine di preferenza riportato di seguito.

  1. id_rsa

  2. id_dsa

  3. identity
