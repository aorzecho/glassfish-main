Istanze server in cluster

Nella pagina Istanze server in cluster è possibile creare e gestire istanze di GlassFish Server nel cluster corrente.

Il campo Nome cluster è un campo di sola lettura in cui viene visualizzato il nome del cluster corrente.

Nella pagina Istanze server in cluster viene visualizzata una lista di istanze appartenenti al cluster corrente. Per ogni istanza vengono fornite le seguenti informazioni.

Nome

Nome assegnato all'istanza quando l'istanza è stata creata. Se si fa clic sul nome, viene visualizzata la pagina Informazioni generali per l'istanza.

Fattore ponderale LB

Numero intero che rappresenta il fattore ponderale per il bilanciamento del carico dell'istanza.

Il fattore ponderale per il bilanciamento del carico determina la proporzione di tutte le richieste inviate al cluster che l'istanza dovrebbe elaborare.

Ad esempio, in un cluster a due istanze può essere necessaria un'istanza per elaborare una di quattro richieste e un'altra istanza per elaborarne tre di quattro. In questa situazione, impostare su 1 il fattore ponderale dell'istanza che elabora una delle quattro richieste e su 3 il fattore ponderale dell'altra istanza. Se si preferisce utilizzare valori percentuali, impostare i fattori ponderali delle istanze rispettivamente su 25 e 75.

Il fattore ponderale predefinito è 100.

Configurazione

Configurazione a cui l'istanza fa riferimento. Se si fa clic sulla configurazione, viene visualizzata la pagina Configurations per l'istanza.

Nodo

Nome del nodo nel quale si trova l'istanza. Se si fa clic sul nome, viene visualizzata la pagina Modifica nodo per il nodo.

Stato

Indica se l'istanza è in esecuzione, se richiede il riavvio per l'aggiornamento con modifiche alla configurazione o se non è in esecuzione. Campo di sola lettura.

La tabella Istanze server contiene inoltre le seguenti opzioni.

Nuovo

Pulsante che consente di aggiungere un'istanza al cluster.

Elimina

Pulsante che consente di rimuovere una o più istanze selezionate dal cluster.

Attenzione:

Se si utilizza un cluster JMS (Java Message Service) con un broker principale, non eliminare l'istanza associata al broker principale. Se è necessario eliminare l'istanza, utilizzare il comando secondario change-master-broker(1) per assegnare il broker principale a un'altra istanza.
Avvia

Pulsante che consente di avviare una o più istanze selezionate nel cluster.

Arresta

Pulsante che consente di arrestare una o più istanze selezionate nel cluster.
