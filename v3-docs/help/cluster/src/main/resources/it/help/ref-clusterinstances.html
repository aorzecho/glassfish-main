Istanze server in cluster

Nella pagina Clustered Server Instances è possibile creare e gestire istanze di GlassFish Server nel cluster corrente.

Il campo Cluster Name è un campo di sola lettura in cui viene visualizzato il nome del cluster corrente.

Nella pagina Clustered Server Instances viene visualizzato un elenco di istanze appartenenti al cluster corrente. Per ogni istanza vengono fornite le seguenti informazioni.

Name

Nome assegnato all'istanza quando l'istanza è stata creata. Se si fa clic sul nome, viene visualizzata la pagina General Information per l'istanza.

LB Weight

Numero intero che rappresenta il peso per il bilanciamento del carico dell'istanza.

Il peso per il bilanciamento del carico determina la proporzione di tutte le richieste inviate al cluster che l'istanza dovrebbe elaborare.

Ad esempio, in un cluster a due istanze può essere necessaria un'istanza per elaborare una di quattro richieste e un'altra istanza per elaborarne tre di quattro. In questa situazione, impostare su 1 il peso dell'istanza che elabora una delle quattro richieste e su 3 il peso dell'altra istanza. Se si preferisce utilizzare valori percentuali, impostare i pesi delle istanze rispettivamente su 25 e 75.

Il peso predefinito è 100.

Configuration

Configurazione a cui l'istanza fa riferimento. Se si fa clic sulla configurazione, viene visualizzata la pagina Configurations per l'istanza.

Node

Nome del nodo nel quale si trova l'istanza. Se si fa clic sul nome, viene visualizzata la pagina Edit Node per il nodo.

Status

Indica se l'istanza è in esecuzione, se richiede il riavvio per l'aggiornamento con modifiche alla configurazione o se non è in esecuzione. Campo di sola lettura.

La tabella Server Instances contiene inoltre le seguenti opzioni.

New

Pulsante che consente di aggiungere un'istanza al cluster.

Delete

Pulsante che consente di rimuovere una o più istanze selezionate dal cluster.

Avvertenza - Se si utilizza un cluster JMS (Java Message Service) con un broker principale, non eliminare l'istanza associata al broker principale. Se è necessario eliminare tale istanza, utilizzare il comando secondario change-master-broker(1) per assegnare il broker principale a un'altra istanza.

Start

Pulsante che consente di avviare una o più istanze selezionate nel cluster.

Stop

Pulsante che consente di arrestare una o più istanze selezionate nel cluster.
