Para crear un nodo

Un nodo representa un host en el que está instalado el software GlassFish Server. Debe existir un nodo en cada host en el que residan instancias de GlassFish Server.

  1. En el árbol de navegación, seleccione el nodo Nodos.

    Se abrirá la página Nodos.

  2. En la página Nodos, haga clic en Nuevo.

    Se abrirá la página Nuevo nodo.

  3. En el campo Nombre, escriba el nombre que desee para el nodo.
  4. En la lista desplegable Tipo, seleccione el tipo de nodo que vaya a crear.
    • Para crear un nodo que admita la comunicación sobre Secure Shell (SSH), seleccione SSH.
    • Para crear un nodo que no admita la comunicación remota, seleccione CONFIG.
  5. En el campo Host de nodo, escriba el nombre del host que representa el nodo.

    Este campo es obligatorio o no en función del tipo de nodo:

    • Si el tipo de nodo es CONFIG., el nombre del host puede omitirse. El nombre del host se puede determinar al crear las instancias que residen en el nodo.

    • Si el tipo de nodo es SSH, se debe especificar el nombre del host. De lo contrario, se produce un error.

  6. En el campo Directorio de nodos, indique la ruta al directorio que contendrá las instancias de GlassFish Server que se crean en el nodo.

    Si el tipo de nodo es CONFIG., deje este campo en blanco. El directorio del nodo se puede determinar al crear las instancias que residen en el nodo.

    Si especifica una ruta relativa, la ruta es relativa al directorio as-install, donde as-install es el directorio base de instalación del software GlassFish Server en el host.

  7. En el campo Directorio de instalación, especifique la ruta completa al directorio principal del directorio base de instalación del software GlassFish Server en el host, por ejemplo /export/glassfish3/.

    Si el tipo de nodo es CONFIG., deje este campo en blanco. El directorio de instalación se puede determinar al crear las instancias que residen en el nodo.

  8. Para forzar al nodo a que se cree en la configuración del servidor de administración de dominios (DAS) aunque falle la validación de los parámetros del nodo, habilite la opción Forzar.
  9. En el campo Puerto SSH, escriba el puerto que se utilizará para las conexiones SSH con el host de este nodo.
  10. En el campo Nombre de usuario, escriba el nombre del usuario del host de este nodo que ejecutará el proceso para conectar al host mediante SSH.
  11. Especifique cómo se autenticará el usuario de SSH al iniciar sesión en el host de este nodo.
    • Para especificar la autenticación a través de un archivo de claves privadas SSH, efectúe los pasos siguientes:
      1. Seleccione Archivo de claves en la lista desplegable Autenticación de usuario de SSH.
      2. En el campo Archivo de claves, escriba la ruta absoluta al archivo de claves privadas SSH para el usuario especificado en la opción Nombre de usuario.

        Nota - GlassFish Server también admite la autenticación con contraseña a través de la entrada AS_ADMIN_SSHPASSWORD en el archivo de contraseñas. El archivo de contraseñas se configura en la opción --paswordfile de la utilidad asadmin(1M).

    • Para especificar autenticación mediante una contraseña guardada en la configuración de DAS, efectúe los pasos siguientes:

      Precaución - Esta contraseña se almacena en forma de texto en la configuración de DAS. Para mayor seguridad, utilice un archivo de claves o un alias de contraseña.

      1. Seleccione Contraseña en la lista desplegable Autenticación de usuario de SSH.
      2. En el campo Contraseña SSH, escriba la contraseña que el usuario de SSH utilizará al conectarse al host de este nodo.
      3. En el campo Confirmar contraseña SSH, vuelva a escribir la contraseña.
    • Para especificar autenticación mediante un alias de contraseña, efectúe los pasos siguientes:
      1. Seleccione Alias de contraseña en la lista desplegable Autenticación de usuario de SSH.
      2. En la lista desplegable Alias de contraseña, seleccione un alias existente.
  12. Haga clic en Guardar.
