Para iniciar un clúster

Al iniciar un clúster se inician todas las instancias de Servidor GlassFish del clúster que no estén aún en ejecución.

Antes de empezar

Asegúrese de que se cumplan los siguientes requisitos previos:

  • Todos los nodos en los que residen las instancias del clúster están habilitados para la comunicación remota.

  • El usuario del servidor de administración de dominios (DAS) puede usar Secure Shell (SSH) para iniciar sesión en el host para cualquier nodo en el que residan las instancias del clúster.

Si no se cumplen algunos de estos requisitos previos, inicie el clúster iniciando cada instancia de forma individual desde la línea de comandos mediante el subcomando start-local-instance(1).

  1. En el árbol de navegación, seleccione el nodo Clústeres.

    Se abrirá la página Clústeres.

  2. En la página Clústeres, seleccione la casilla de verificación adyacente al nombre del clúster.
  3. Haga clic en Iniciar clúster.

    Se le pedirá que confirme que desea iniciar el clúster.

  4. Confirme que quiere iniciar el clúster.

    Se iniciará el clúster. El estado de cada instancia del clúster se actualiza en la página Clústeres cuando se inicia la instancia.
