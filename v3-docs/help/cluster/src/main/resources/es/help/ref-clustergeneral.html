Información general

La página Información general contiene las opciones siguientes.

Iniciar clúster

Botón para iniciar el clúster. Si todas las instancias del clúster están en ejecución, este botón está desactivado.

Detener clúster

Botón para detener el clúster. Si todas las instancias en el clúster se detienen, este botón se desactiva.

Migrar temporizadores EJB

Botón para migrar temporizadores de tecnología Enterprise JavaBeans (EJB) desde una instancia de Servidor GlassFish detenida a una instancia en ejecución en el clúster. Al pulsar este botón se abre la página Migrar temporizadores EJB. Si todas las instancias del clúster están en ejecución, este botón está desactivado.

Girar registro

Botón para girar el archivo de registro para el clúster. Cuando se gira el archivo de registro, se cambia el nombre del archivo por un nombre con marca de tiempo con el formato server.log_ fecha_y_hora y se crea un archivo de registro vacío para los nuevos mensajes de registro. Los cambios se aplican de forma dinámica. No es necesario reiniciar el servidor.

Nombre de clúster

El nombre que se asignó al clúster en el momento de crearlo. Este campo es de sólo lectura.

JVM

Si hace clic sobre el Informe de JVM, se abrirá una ventana independiente que muestra informes en la máquina virtual de Java, incluyendo un informe resumido, un informe sobre administración de la memoria y recolección de elementos no utilizados, otro informe sobre carga de clases y un volcado de subprocesos actuales.

Configuración

La configuración que se seleccionó para el clúster en el momento de crearlo. Al hacer clic en la configuración se abre la página Configuración para el clúster. Este campo es de sólo lectura.

GMS

Si esta opción está habilitada, se inician los Servicios de administración de grupos(GMS) en cada instancia de Servidor GlassFish del clúster y en el servidor de administración de dominios (DAS). El DAS participa en cada clúster para el que se han habilitado los GMS. Los GMS proporcionan supervisión del clúster, pertenencia al clúster y servicios de comunicación para los grupos. Esta opción se habilita por defecto.

Puerto de multidifusión

Número del puerto de comunicación en el que los GMS escuchan los eventos de grupos. Esta opción debe especificar un número de puerto válido entre 2048 y 32000. El valor predeterminado se genera automáticamente dentro de este rango.

Dirección de multidifusión

Dirección en la que los GMS escuchan los eventos de grupos. Esta opción debe especificar una dirección de multidifusión entre 224.0.0.0 y 239.255.255.255. El valor predeterminado es 228.9.XX.YY, donde XX e YY son valores independientes generados automáticamente entre 0 y 255.

Enlazar dirección de interfaz

Dirección IP (Internet Protocol) de la interfaz de red a la que se vinculan los GMS. Esta opción debe especificar la dirección IP de una interfaz de red local. Los valores predeterminados son direcciones de interfaz de red públicas.

En un equipo multiorigen, esta opción configura la interfaz de red usada para los GMS. Un equipo multiorigen posee dos o más interfaces de red.

Para especificar una dirección que sea válida para todas las instancias de Servidor GlassFish del clúster, utilice una propiedad del sistema para establecer la dirección de forma individual para cada instancia.

Por ejemplo, utilice la página Propiedades del sistema de clúster para crear la propiedad del sistema GMS-BIND-INTERFACE-ADDRESS-nombre_clúster. A continuación, configure la opción de enlace de dirección de esta página como ${GMS-BIND-INTERFACE-ADDRESS-nombre_clúster} para especificar la propiedad del sistema. Por último, para cada instancia del clúster, configure la propiedad del sistema GMS-BIND-INTERFACE-ADDRESS-nombre_clúster con la dirección de interfaz de red necesaria del equipo de la instancia.

Estado

Indicación del número de instancias del clúster que se ejecutan y que no se ejecutan. Este campo es de sólo lectura.
