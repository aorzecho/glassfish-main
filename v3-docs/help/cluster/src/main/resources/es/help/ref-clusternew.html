Clúster nuevo

Utilice la página Clúster nuevo para crear un clúster.

La página Clúster nuevo contiene las opciones siguientes.

Nombre de clúster

El nombre del clúster.

El nombre debe cumplir los siguientes requisitos:

  • El nombre debe empezar por una letra, un número o un carácter de subrayado.

  • El nombre sólo puede contener los siguientes caracteres:

    • Letras en minúscula

    • Letras en mayúscula

    • Números

    • Guión

    • Punto

    • Subrayado

  • El nombre debe ser exclusivo en el dominio y no debe ser el nombre de otro clúster, una configuración con nombre, una instancia de GlassFish Server ni otro nodo.

  • El nombre non debe ser domain, server ni otra palabra clave reservada para GlassFish Server.

Configuración

Lista desplegable de las configuraciones con nombre existentes que puede utilizar el clúster. Un clúster requiere una configuración con nombre que define la configuración de todas las instancias que se añaden al clúster. Debe seleccionarse una configuración en la lista. El clúster utilizará la configuración seleccionada cuando se cree.

Realizar una copia de la configuración seleccionada

Si se selecciona esta opción, se copia la configuración seleccionada cuando se crea el clúster. Se asigna el nombre nombre_clúster-config a la copia de la configuración, donde nombre_clúster se reemplaza con el nombre del clúster del campo Nombre de clúster.

Si se selecciona default-config en la lista desplegable Configuración, debe elegir esta opción. La configuración default-config sólo se puede copiar, no tomar como referencia.

Hacer referencia a la configuración seleccionada

Si se selecciona esta opción, el clúster utilizará la configuración con nombre especificada.

Si se selecciona default-config en la lista desplegable Configuración, no debe seleccionar esta opción. La configuración default-config no puede tomarse como referencia, sino sólo copiarse.

Instancias del servidor que se crearán

Lista de instancias de GlassFish Server que se crearán cuando se cree el clúster. Para cada instancia, se proporciona la información siguiente:

Nombre de instancia

Nombre de la instancia.

Ponderación

Entero que representa la ponderación de equilibrio de carga de la instancia.

La ponderación de equilibrio de carga determina la proporción de todas las solicitudes con respecto al clúster que debe procesar la instancia.

Por ejemplo, en un clúster de dos instancias, puede necesitar que una instancia procese una solicitud de cuatro, y que otra procese las otras tres. En este caso, configure la ponderación de la instancia que debe procesar una de las cuatro solicitudes como 1 y la ponderación de la otra instancia como 3. Si prefiere usar porcentajes, defina las ponderaciones de las instancias como 25 y 75, respectivamente.

La ponderación predeterminada es 100.

Nodo

Lista desplegable de los nodos existentes en los que puede residir la instancia. Debe seleccionarse un nodo en la lista. La instancia residirá en el nodo seleccionado cuando se cree el clúster.

La tabla Instancias del servidor que se crearán también contiene las opciones siguientes.

Nuevo

Botón para crear una instancia. Al hacer clic en este botón se añade una fila a la tabla Instancias del servidor que se crearán.

Borrar

Botón para eliminar una instancia.
