Propiedades del clúster

La página Propiedades del clúster muestra una lista de las propiedades que se definen para los clústeres seleccionados. Estas propiedades agregan información adicional sobre el clúster.

El campo Nombre de clúster es un campo de sólo lectura que muestra el nombre del clúster seleccionado.

Para cada propiedad se muestra la información siguiente:

Nombre

Nombre de la propiedad.

Valor

Valor de la propiedad que se define para la instancia seleccionada.

Descripción

Descripción textual que ofrece más información sobre la propiedad.

La tabla Propiedades adicionales también contiene las opciones siguientes.

Agregar propiedad

Botón para agregar una propiedad. Al hacer clic en este botón se añade una fila a la tabla Propiedades adicionales.

Eliminar propiedades

Botón para eliminar una o varias propiedades seleccionadas. Cualquier propiedad que se elimine vuelve a su valor predeterminado o, si no se ha definido ningún valor predeterminado, se queda sin definir.

Servidor GlassFish define las siguientes propiedades del clúster:

GMS_LISTENER_PORT

Número del puerto en el que el clúster escucha los mensajes de Servicios de administración de grupos (GMS). El valor predeterminado será suficiente en la mayoría de las situaciones.

GMS_LOOPBACK

Especifica si una instancia recibirá de ella misma mensajes de nivel de aplicación que emita para el clúster.

Los valores posibles son los siguientes:

false

Es posible que la instancia no reciba mensajes de ella misma (configuración predeterminada).

true

La instancia puede recibir mensajes de ella misma. Utilice esta configuración para probar una instancia cuando sea la única de un clúster.

GMS_MULTICAST_TIME_TO_LIVE

Número máximo de iteraciones o transmisiones que puede experimentar un mensaje de multidifusión para los siguientes tipos de eventos antes de descartar el mensaje:

  • Detección de grupos

  • Latidos de miembros

  • Cambios de miembros

Para coincidir con la configuración de la red en la que se implementan el DAS y las instancias agrupadas en clúster, configure esta opción con el valor más bajo posible. Para determinar el valor mínimo posible para el sistema, utilice el subcomando validate-multicast(1).

Un valor de 0 garantiza que los mensajes de multidifusión nunca dejarán el host desde el cual se emiten.

Un valor de 1 puede impedir la emisión de mensajes entre hosts de la misma subred que están conectados mediante un conmutador o un enrutador.

El valor predeterminado es 4, que garantiza que los mensajes se emitan correctamente a todos los miembros del clúster de las redes en las que se han conectado los hosts mediante conmutadores o enrutadores.
