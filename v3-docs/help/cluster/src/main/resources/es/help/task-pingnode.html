Probar si un Nodo es Accesible

Para crear o iniciar instancias de GlassFish Server en un nodo que representa un host remoto, el DAS debe poder contactar con el nodo. Antes de intentar crear o iniciar instancias en ese nodo, pruebe si el DAS puede contactar con el nodo.

El tipo de nodo que va a probar debe ser DCOM o SSH. No puede probar un nodo del tipo CONFIG porque un nodo del tipo CONFIG no está activado para la comunicación remota.

Antes de Empezar

Asegúrese de que esté configurado el protocolo adecuado para el tipo de nodo en el host que el nodo representa. El nodo está activado para la comunicación remota mediante este protocolo

  • Si el tipo de nodo es DCOM, debe configurarse el protocolo Distributed Component Object Model (DCOM).

  • Si el tipo de nodo es SSH, debe configurarse el protocolo Secure Shell (SSH).

    1. En el árbol de navegación, seleccione el nodo Nodos.

      Se abre la página Nodos. En esta página se enumeran todos los nodos del dominio actual.

    2. En la tabla de nodos, siga el enlace Ping para el nodo que va a probar.

      Se agrega a la página Nodos un mensaje que indica si el DAS podría ponerse en contacto con el nodo.
