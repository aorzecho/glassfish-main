Configurar Opciones del Servicio de Gestión de Grupos para un Cluster

El Servicio de Gestión de Grupos (GMS) es un servicio en curso que proporciona supervisión del cluster, pertenencia al cluster y servicios de comunicación de grupos. Los GMS realizan las funciones siguientes:

  • Notifican los módulos registrados en una instancia de GlassFish Server cuando se agregan o eliminan una o varias instancias en un cluster, o cuando se sospecha o se confirma que han fallado.

  • Permiten enviar y recibir mensajes entre un grupo de procesos.

  1. En el árbol de navegación, amplíe el nodo Clusters.

  2. Seleccione el cluster para el que va a configurar el Servicio de Gestión de Grupos (GMS).

    Se abre la página Información General para el cluster.

  3. Seleccione la casilla de verificación de activación de GMS para activar los GMS para el cluster.

    Esta opción está seleccionada por defecto. Para desactivar los GMS para el cluster, quite la marca de selección de la casilla de verificación de activación de GMS.

    Si está activado, el Servicio de Gestión de Grupos se inicia en cada instancia del servidor en el cluster y en el Servidor de Administración de Dominio (DAS). El DAS participa en cada cluster que tenga los GMS activados.

  4. En el campo Puerto de Multidifusión, especifique el puerto que utiliza el Servicio de Gestión de Grupos para escuchar los eventos del grupo.

    Especifique un número de puerto UDP estándar entre 2048 y 32000. El valor por defecto se genera automáticamente dentro de este rango.

  5. En el campo Dirección de Multidifusión, especifique la dirección que utiliza el Servicio de Gestión de Grupos para escuchar los eventos del grupo.

    Especifique una dirección de multidifusión IP de clase D. Las direcciones IP de clase D están en el rango 224.0.0.0 a 239.255.255.255, inclusive. La dirección 224.0.0.0 está reservada y no debe utilizarse. El valor por defecto es 228.9.XX. YY, donde XX e YY son valores independientes generados automáticamente entre 0 y 255.

  6. En el campo Dirección de Interfaz de Enlace, indique la dirección de la interfaz de red para el Servicio de Gestión de Grupos si hay más de una interfaz de red.

    Esta opción permite configurar la interfaz de red que se utilizará para los GMS si hay más de una en cualquiera de los equipos en los que se alojan las instancias de servidor del cluster. Este valor debe ser una dirección IP de una interfaz de red local. Por defecto, se utilizan todas las direcciones de interfaz de red públicas.

    Para especificar una dirección que sea válida para todas las instancias de GlassFish Server del cluster, utilice una propiedad del sistema para establecer la dirección de forma individual para cada instancia.

    Por ejemplo, cree la propiedad del sistema GMS-BIND-INTERFACE-ADDRESS-nombre_cluster en el DAS. A continuación, configure la opción Dirección de Interfaz de Enlace como ${GMS-BIND-INTERFACE-ADDRESS-nombre_cluster} para especificar la propiedad del sistema. Por último, para cada instancia del cluster, defina la propiedad del sistema GMS-BIND-INTERFACE-ADDRESS-cluster-name en la dirección de interfaz de red necesaria de la máquina de la instancia.

  7. Para especificar propiedades adicionales, seleccione el separador Propiedades y el subseparador Propiedades del Cluster.

    • Para agregar una propiedad, haga clic en el botón Agregar Propiedad. En la fila en blanco que aparece, escriba el nombre de propiedad en el campo Nombre y escriba el valor de propiedad en el campo Valor.

    • Para modificar una propiedad, edite su campo Valor.

    • Para suprimir una propiedad, seleccione la casilla de verificación a la izquierda del campo Nombre de la propiedad que está suprimiendo y, a continuación, haga clic en el botón Suprimir Propiedades.

  8. Haga clic en Guardar.
