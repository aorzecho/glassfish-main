Para configurar los Servicios de administración de grupos para un clúster

Los Servicios de administración de grupos (GMS) son servicios en curso que proporcionan supervisión del clúster, pertenencia al clúster y servicios de comunicación de grupos. Los GMS realizan las funciones siguientes:

  • Notifican los módulos registrados en una instancia de GlassFish Server cuando se añaden o eliminan una o varias instancias en un clúster, o cuando se sospecha o se confirma que han fallado.

  • Permiten enviar y recibir mensajes entre un grupo de procesos.

  1. En el árbol de navegación, expanda el nodo Clústeres.
  2. Seleccione el clúster para el que va a configurar los Servicios de administración de grupos (GMS).

    Se abrirá la página Información general para el clúster.

  3. Seleccione la casilla Habilitado de GMS para activar los GMS para el clúster.

    Esta opción está seleccionada de forma predeterminada. Para inhabilitar los GMS para el clúster, quiete la marca de selección de la casilla Habilitado de GMS.

    Si están habilitados, los Servicios de administración de grupos se inician en cada instancia del servidor en el clúster y en el servidor de administración de dominios (DAS). El DAS participa en cada clúster que tenga los GMS habilitados.

  4. En el campo Puerto de multidifusión, especifique el puerto que utilizan los Servicios de administración de grupos para escuchar los eventos del grupo.

    Especifique un número de puerto UDP estándar entre 2048 y 32000. El valor predeterminado se genera automáticamente dentro de este rango.

  5. En el campo Dirección de multidifusión, especifique la dirección que utilizan los Servicios de administración de grupos para escuchar los eventos del grupo.

    Especifique una dirección de multidifusión IP de clase D. Las direcciones IP de clase D están en el intervalo 224.0.0.0 a 239.255.255.255, inclusive. La dirección 224.0.0.0 está reservada y no debe utilizarse. El valor predeterminado es 228.9.XX. YY, donde XX e YY son valores independientes generados automáticamente entre 0 y 255.

  6. En el campo Enlazar dirección de interfaz, indique la dirección de la interfaz de red para los Servicios de administración de grupos si hay más de una interfaz de red.

    Esta opción permite configurar la interfaz de red que se utilizará para los GMS si hay más de una en cualquiera de los equipos en los que se alojan las instancias de servidor del clúster. Este valor debe ser una dirección IP de una interfaz de red local. De modo predeterminado, se utilizan todas las direcciones de interfaz de red públicas.

    Para especificar una dirección que sea válida para todas las instancias de GlassFish Server del clúster, utilice una propiedad del sistema para establecer la dirección de forma individual para cada instancia.

    Por ejemplo, cree la propiedad del sistema GMS-BIND-INTERFACE-ADDRESS-nombre_clúster en el DAS. A continuación, configure la opción Enlazar dirección de interfaz como ${GMS-BIND-INTERFACE-ADDRESS-nombre_clúster} para especificar la propiedad del sistema. Por último, para cada instancia del clúster, configure la propiedad del sistema GMS-BIND-INTERFACE-ADDRESS-nombre_clúster con la dirección de interfaz de red necesaria del equipo de la instancia.

  7. (Opcional) Para especificar propiedades adicionales, seleccione la ficha Propiedades y la subficha Propiedades del clúster.

    GlassFish Server no define ninguna propiedad adicional para los GMS.

    • Para añadir una propiedad, haga clic en el botón Añadir propiedad. En la fila vacía escriba el nombre de la propiedad en el campo Nombre y el valor de la propiedad en el campo Valor.
    • Para modificar una propiedad, edite su campo Valor.
    • Para eliminar una propiedad, marque la casilla de verificación situada a la izquierda del campo Nombre de la propiedad que quiera eliminar y, a continuación, haga clic en el botón Eliminar propiedades.
  8. Haga clic en Guardar.
