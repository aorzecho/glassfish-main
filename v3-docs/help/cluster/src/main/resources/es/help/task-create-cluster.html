Para crear un clúster

Un clúster es un grupo de instancias de GlassFish Server con nombre que comparten las mismas aplicaciones, los mismos recursos y la información de configuración.

Un clúster proporciona una gran disponibilidad a través la escalabilidad, el equilibrio de carga y la protección contra fallos. Cuando falla una instancia o un host de un clúster, GlassFish Server detecta el error, redirige las solicitudes de la instancia fallida a otras instancias del clúster y recupera el estado de la sesión del usuario. Dado que los mismos recursos y aplicaciones están en todas las instancias del clúster, una instancia puede conmutar por error a cualquier otra instancia del clúster.

GlassFish Server permite administrar todas las instancias de un clúster como una sola unidad desde un único host, independientemente de si residen en el mismo host o no.

Antes de empezar

Si tiene previsto añadir instancias de GlassFish Server al clúster que está creando, asegúrese de que los nodos en los que vayan a residir las instancias ya existan.

  1. En el árbol de navegación, seleccione el nodo Clústeres.

    Se abrirá la página Clústeres.

  2. En la página Clústeres, haga clic en Nuevo.

    Se abrirá la página Clúster nuevo.

  3. En el campo Clúster nuevo, escriba un nombre para el clúster.

    El nombre debe cumplir los siguientes requisitos:

    • El nombre debe empezar por una letra, un número o un carácter de subrayado.

    • El nombre sólo puede contener los siguientes caracteres:

      • Letras en minúscula

      • Letras en mayúscula

      • Números

      • Guión

      • Punto

      • Subrayado

    • El nombre debe ser exclusivo en el dominio y no debe ser el nombre de otro clúster, una configuración con nombre, una instancia de GlassFish Server ni otro nodo.

    • El nombre non debe ser domain, server ni otra palabra clave reservada para GlassFish Server.

  4. En la lista desplegable Configuración, seleccione la configuración con nombre que utilizará el clúster.

    Nota - Si selecciona default-config, debe especificar en el paso siguiente que la configuración no es compartida. La configuración de default-config no se puede compartir.

  5. Indique si la configuración que utilizará el clúster es compartida.
    • Si la configuración no es compartida, deje seleccionada la opción Realizar una copia de la configuración seleccionada.

      La configuración seleccionada se copia cuando se crea el clúster. Se asigna el nombre nombre_clúster-config a la copia de la configuración, donde nombre_clúster es el nombre del clúster que se está creando.

    • Si la configuración es compartida, seleccione la opción Hacer referencia a la configuración seleccionada.

      El clúster utilizará la configuración con nombre especificada.

  6. (Opcional) Agregue instancias de GlassFish Server al clúster.

    Nota - Para crear una instancia en un nodo de tipo CONFIG que representa un host remoto deberá utilizar el subcomando create-local-instance(1). No se puede utilizar el Consola de administración para este fin.

    También puede agregar instancias después de crear clúster.

    Agregue cada una de las instancias del modo siguiente:

    1. En el área Instancias del servidor que se crearán, haga clic en Agregar.

      Se agregará una fila a la tabla de instancias del clúster.

    2. En el campo Nombre de instancia, escriba un nombre para la instancia.
    3. En el campo Ponderación, escriba un número entero que representará la ponderación del equilibrio de carga de la instancia.

      La ponderación de equilibrio de carga determina la proporción de todas las solicitudes con respecto al clúster que debe procesar la instancia.

      Por ejemplo, en un clúster de dos instancias, puede necesitar que una instancia procese una solicitud de cuatro, y que otra procese las otras tres. En este caso, configure la ponderación de la instancia que debe procesar una de las cuatro solicitudes como 1 y la ponderación de la otra instancia como 3. Si prefiere usar porcentajes, defina las ponderaciones de las instancias como 25 y 75, respectivamente.

      La ponderación predeterminada es 100.

    4. En la lista desplegable Nodo, seleccione el nodo en el que residirá la instancia.
  7. Haga clic en Aceptar.

    Volverá a la página Clústeres. El clúster se añadirá a la tabla de clústeres de la página.
