Crear un Cluster

Un cluster es una recopilación de instancias de GlassFish Server con nombre que comparten las mismas aplicaciones, recursos e información de configuración.

Un cluster proporciona una gran disponibilidad a través de la escalabilidad, el equilibrio de carga y la protección contra fallos. Cuando falla una instancia o un host de un cluster, GlassFish Server detecta el error, redirige las solicitudes de la instancia fallida a otras instancias del cluster y recupera el estado de la sesión del usuario. Dado que los mismos recursos y aplicaciones están en todas las instancias del cluster, una instancia puede generar un failover en cualquier otra instancia del cluster.

GlassFish Server permite administrar todas las instancias de un cluster como una sola unidad desde un único host, independientemente de si residen en el mismo host o no.

Antes de Empezar

Si tiene previsto agregar instancias de GlassFish Server al cluster que está creando, asegúrese de que los nodos en los que vayan a residir las instancias ya existan.

  1. En el árbol de navegación, seleccione el nodo Clusters.

    Se abre la página Clusters.

  2. En la página Clusters, haga clic en Nuevo.

    Se abre la página Cluster Nuevo.

  3. En el campo Nombre de Cluster, escriba un nombre para el cluster.

    El nombre debe cumplir los siguientes requisitos:

    • El nombre sólo puede contener caracteres ASCII.

    • El nombre debe empezar por una letra, un número o un carácter de subrayado.

    • El nombre sólo puede contener los siguientes caracteres:

      • Letras en minúscula

      • Letras en mayúscula

      • Números

      • Guión

      • Punto

      • Subrayado

    • El nombre debe ser exclusivo en el dominio y no debe ser el nombre de otro cluster, una configuración con nombre, una instancia de GlassFish Server ni otro nodo.

    • El nombre no debe ser domain, server ni otra palabra clave reservada por GlassFish Server.

  4. En la lista desplegable Configuración, seleccione la configuración con nombre que utilizará el cluster.

    Nota:

    Si selecciona default-config, debe especificar en el paso siguiente que la configuración no es compartida. La configuración de default-config no se puede compartir.

  5. Indique si la configuración que utilizará el cluster es compartida.

    • Si la configuración no es compartida, deje seleccionada la opción Realizar una Copia de la Configuración Seleccionada.

      La configuración seleccionada se copia cuando se crea el cluster. Se asigna el nombre nombre_cluster-config a la copia de la configuración, donde nombre_cluster es el nombre del cluster que se está creando.

    • Si la configuración es compartida, seleccione la opción Hacer Referencia a la Configuración Seleccionada.

      El cluster utilizará la configuración con nombre especificada existente.

  6. Agregar Instancias de GlassFish Server al Cluster.

    Nota:

    Para crear una instancia en un nodo de tipo CONFIG que representa un host remoto, debe utilizar el subcomando create-local-instance(1). No se puede utilizar el Consola de Administración para este fin.

    También puede agregar instancias después de crear el cluster, tal y como se describe en Agregar una Instancia de GlassFish Server a un Cluster.

    Agregue cada una de las instancias del modo siguiente:

    1. En el área Instancias del Servidor que se Van a Crear, haga clic en Agregar.

      Se agregará una fila a la tabla de instancias del cluster.

    2. En el campo Nombre de Instancia, escriba un nombre para la instancia.

    3. En el campo Peso, escriba un número entero que representa el peso del equilibrio de carga de la instancia.

      El peso del equilibrio de carga determina la proporción de todas las solicitudes con respecto al cluster que debe procesar la instancia.

      Por ejemplo, en un cluster de dos instancias, puede necesitar que una instancia procese una solicitud de cuatro, y que otra procese las otras tres. En este caso, configure el peso de la instancia que debe procesar una de las cuatro solicitudes como 1 y el peso de la otra instancia como 3. Si prefiere usar porcentajes, defina los pesos de las instancias como 25 y 75, respectivamente.

      El peso por defecto es 100.

    4. En la lista desplegable Nodo, seleccione el nodo en el que residirá la instancia.

  7. Haga clic en Aceptar.

    Volverá a la página Clusters. El cluster se agrega a la tabla de clusters de la página.
