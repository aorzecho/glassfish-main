Finalizar la Actualización de un Nodo DCOM

Esta tarea es la continuación para un nodo DCOM de Editar un Nodo.

  1. Para forzar la creación del nodo en la configuración del Servidor de Administración de Dominio (DAS) aunque falle la validación de los parámetros del nodo, active la opción Forzar.

  2. Si necesita que GlassFish Server pruebe la conexión mediante DCOM con el host del nodo antes de actualizar el nodo, lleve a cabo estos pasos:

    1. Active la opción Validar DCOM.

    2. En el campo Directorio de Prueba Remoto, escriba el nombre de la carpeta del host remoto en la que GlassFish Server deba crear un archivo para probar la conexión.

  3. En el campo Dominio de Windows, escriba el nombre del domino de Windows que contenga el usuario que la opción Nombre de Usuario de Windows especifica.

    Si cambia el tipo de nodo por DCOM y deja este campo en blanco, la opción Dominio de Windows se establece en el nombre del host en el que se ejecuta Consola de Administración.

  4. En el campo Nombre de Usuario de Windows, escriba el nombre del usuario del host de este nodo que ejecutará el proceso para conectar al host mediante DCOM.

    Si cambia el tipo de nodo de CONFIG a DCOM y deja este campo en blanco, la opción Nombre de Usuario de Windows se configura con el usuario que ejecuta el proceso DAS.

  5. Especifique cómo se autenticará el usuario de Windows al iniciar sesión en el host de este nodo.

    Nota:

    GlassFish Server también admite la autenticación con contraseña a través de la entrada AS_ADMIN_WINDOWSPASSWORD en el archivo de contraseñas. El archivo de contraseñas se especifica en la opción -paswordfile de la utilidad asadmin(1).

    1. Para especificar la autenticación mediante una contraseña guardada en la configuración de DAS, efectúe los pasos siguientes:

      Atención:

      Esta contraseña se almacena en forma de texto no cifrado en la configuración de DAS. Para mejorar la seguridad, utilice un alias de contraseña.

      1. Seleccione Contraseña en la lista desplegable Autenticación de Usuario de Windows.

      2. En el campo Contraseña de Usuario de Windows, escriba la contraseña que el usuario de SSH utilizará al iniciar sesión en el host de este nodo.

      3. En el campo Confirmar Contraseña, vuelva a escribir la contraseña.

    2. Para especificar la autenticación mediante un alias de contraseña, efectúe los pasos siguientes:

      1. Seleccione Alias de Contraseña en la lista desplegable Autenticación de Usuario de Windows.

      2. En la lista desplegable Alias de Contraseña, seleccione un alias existente.

  6. Haga clic en Guardar.
