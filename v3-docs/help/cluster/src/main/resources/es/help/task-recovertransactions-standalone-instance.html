Para recuperar transacciones para una instancia de GlassFish Server independiente

  1. En el árbol de navegación, seleccione el nodo Instancias independientes.

    Se abrirá la página Instancias de servidor independientes.

  2. Seleccione la instancia en la que se deben recuperar transacciones.

    Se abrirá la página Información general para la instancia.

  3. En la página Información General, haga clic en Recuperar transacciones.

    Se abrirá la página Recuperar transacciones.

    El contenido de la página depende de si la instancia se está ejecutando.

    • Si la instancia se está ejecutando, la página Recuperar transacciones solo contiene el texto Recovery is performed by the running server itself (La recuperación la realiza el propio servidor en ejecución) y el botón Aceptar.

    • Si la instancia no se está ejecutando, la página Recuperar transacciones contiene opciones adicionales.

  4. En la lista desplegable Servidor de destino, seleccione otra instancia que pueda recuperar transacciones en la instancia actual.

    Para una instancia independiente, seleccione la instancia denominada server, que representa el servidor de administración de dominio (DAS).

  5. En el campo Directorio de registro de transacciones, escriba el directorio donde se guardarán los registros de transacciones.

    El valor predeterminado es /logs/tx en el directorio de dominio. Los registros de transacciones se guardan en el subdirectorio tx del directorio especificado en el campo Ubicación del registro de transacción del servicio de transacción para la configuración del clúster. El valor predeterminado de Ubicación del registro de transacción es el directorio especificado en el campo Raíz de registro de la página Atributos de dominio, que está ubicada en la ficha Avanzado del nodo GlassFish Server.

  6. Haga clic en Aceptar.
