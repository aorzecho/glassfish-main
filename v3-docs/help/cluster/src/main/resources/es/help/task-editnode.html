Para editar un nodo

Al editar un nodo se actualizan los datos de configuración de un nodo de GlassFish Server. La edición de un nodo también permite modificar el tipo de nodo de CONFIG a SSH para habilitar la comunicación SSH para el nodo.

Si cambia el tipo de nodo de CONFIG a SSH , se aplicarán los valores predeterminados si se deja vacío cualquiera de los campos siguientes:

  • Puerto SSH

  • Nombre de usuario SSH

  • Archivo de claves

  1. En el árbol de navegación, expanda el nodo Nodos.
  2. En la lista de nodos, seleccione el nodo que vaya a editar.

    Se abrirá la página Editar nodo.

  3. En la lista desplegable Tipo, seleccione el tipo de nodo actualizado.
    • Para permitir que el nodo admita la comunicación sobre Secure Shell (SSH), seleccione SSH.
    • Para que el nodo deje de admitir la configuración remota, seleccione CONFIG.

      Si selecciona CONFIG, no se mostrará ninguna opción para la configuración de SSH. Si el tipo actual del nodo es SSH y selecciona CONFIG, se eliminarán los datos de configuración SSH de la configuración del nodo cuando guarde los cambios.

  4. En el campo Host de nodo, indique el nombre del host que representará el nodo actualizado.

    Este campo es obligatorio o no en función del tipo de nodo:

    • Si el tipo de nodo es CONFIG., el nombre del host puede omitirse. El nombre del host se puede determinar al crear las instancias que residen en el nodo.

    • Si el tipo de nodo es SSH, se debe especificar el nombre del host. De lo contrario, se produce un error.

  5. En el campo Directorio de nodos, indique la ruta al directorio que contendrá las instancias de GlassFish Server que se crean en el nodo.

    Si especifica una ruta relativa, la ruta es relativa al directorio as-install, donde as-install es el directorio base de instalación del software GlassFish Server en el host.

  6. En el campo Directorio de instalación, especifique la ruta completa al directorio principal del directorio base de instalación del software GlassFish Server en el host, por ejemplo /export/glassfish3/.
  7. Para forzar al nodo a que se cree en la configuración del servidor de administración de dominios (DAS) aunque falle la validación de los parámetros del nodo, habilite la opción Forzar.
  8. En el campo Puerto SSH, escriba el puerto que se utilizará para las conexiones SSH con el host de este nodo.

    Si cambia el tipo de nodo de CONFIG a SSH y deja este campo vacío, la opción Puerto SSH se establece en 22.

  9. En el campo Nombre de usuario, escriba el nombre del usuario del host de este nodo que ejecutará el proceso para conectar al host mediante SSH.

    Si cambia el tipo de nodo de CONFIG a SSH y deja este campo vacío, la opción Nombre de usuario SSH se configura con el usuario que ejecuta el proceso DAS.

  10. Especifique cómo se autenticará el usuario de SSH al iniciar sesión en el host de este nodo.
    • Para especificar la autenticación a través de un archivo de claves privadas SSH, efectúe los pasos siguientes:
      1. Seleccione Archivo de claves en la lista desplegable Autenticación de usuario de SSH.
      2. En el campo Archivo de claves, escriba la ruta absoluta al archivo de claves privadas SSH para el usuario especificado en la opción Nombre de usuario.

        Nota - GlassFish Server también admite la autenticación con contraseña a través de la entrada AS_ADMIN_SSHPASSWORD en el archivo de contraseñas. El archivo de contraseñas se configura en la opción --paswordfile de la utilidad asadmin(1M).

        Si cambia el tipo de nodo de CONFIG a SSH y deja este campo vacío, la opción Archivo de claves se configura a un archivo de claves situado en el directorio .ssh del usuario. Si se encuentran varios archivos de claves, GlassFish Server utiliza el siguiente orden de preferencia:

        • id_rsa

        • id_dsa

        • identity

    • Para especificar autenticación mediante una contraseña guardada en la configuración de DAS, efectúe los pasos siguientes:

      Precaución - Esta contraseña se almacena en forma de texto en la configuración de DAS. Para mayor seguridad, utilice un archivo de claves o un alias de contraseña.

      1. Seleccione Contraseña en la lista desplegable Autenticación de usuario de SSH.
      2. En el campo Contraseña SSH, escriba la contraseña que el usuario de SSH utilizará al conectarse al host de este nodo.
      3. En el campo Confirmar contraseña SSH, vuelva a escribir la contraseña.
    • Para especificar autenticación mediante un alias de contraseña, efectúe los pasos siguientes:
      1. Seleccione Alias de contraseña en la lista desplegable Autenticación de usuario de SSH.
      2. En la lista desplegable Alias de contraseña, seleccione un alias existente.
  11. Haga clic en Guardar.
