Migrar Temporizadores EJB

Si se para una instancia de GlassFish Server, los temporizadores EJB (Enterprise JavaBeans) instalados en dicha instancia también se pararán. Si la instancia se ha parado de forma anormal o inesperada, puede activar los temporizadores. Para ello, ejecútelos mientras la instancia esté parada y mígrelos a una instancia de servidor en ejecución del cluster.

  1. En el árbol de navegación, amplíe el nodo Clusters.

  2. En el nodo Clusters del árbol de navegación, seleccione el cluster.

    Se abre la página Información General para el cluster.

  3. En la página Información General, haga clic en Migrar Temporizadores EJB.

    Se abre la página Migrar Temporizadores EJB.

    Nota:

    Si todas las instancias del cluster se están ejecutando, el botón Migrar Temporizadores EJB estará desactivado.

    Para ayudarle en la migración de temporizadores, la página Migrar Temporizadores EJB contiene una tabla de temporizadores EJB con la información siguiente.

    Nombre de Servidor

    Lista de instancias de GlassFish Server en el cluster.

    Recuento de Temporizadores

    Número de temporizadores persistentes en cada instancia.

  4. En la página Migrar Temporizadores EJB, seleccione la instancia a la que se migrarán los temporizadores y desde la cual se migrarán.

    1. En la lista desplegable Origen, seleccione la instancia parada desde la cual se migrarán los temporizadores.

    2. En la lista desplegable Destino, seleccione la instancia en ejecución a la que se migrarán los temporizadores.

    3. Haga clic en Aceptar.

  5. Pare y reinicie la instancia a la cual se han migrado los temporizadores EJB.

    Si la instancia de origen se está ejecutando o la instancia de destino no se está ejecutando, la Consola de Administración mostrará un mensaje de error.
