Instancias de servidor agrupadas

Utilice la página Instancias de servidor agrupadas para crear y administrar instancias de Servidor GlassFish en el clúster actual.

El campo Nombre de clúster es un campo de sólo lectura que muestra el nombre del clúster actual.

La página Instancias de servidor agrupadas muestra una lista de las instancias que forman parte del clúster actual. Para cada instancia se proporciona la información siguiente.

Nombre

El nombre asignado a la instancia durante su creación. Al hacer clic en el nombre se abre la página Información general para la instancia.

Peso de LB

Entero que representa la ponderación de equilibrio de carga de la instancia.

La ponderación de equilibrio de carga determina la proporción de todas las solicitudes con respecto al clúster que debe procesar la instancia.

Por ejemplo, en un clúster de dos instancias, puede necesitar que una instancia procese una solicitud de cuatro, y que otra procese las otras tres. En este caso, configure la ponderación de la instancia que debe procesar una de las cuatro solicitudes como 1 y la ponderación de la otra instancia como 3. Si prefiere usar porcentajes, defina las ponderaciones de las instancias como 25 y 75, respectivamente.

La ponderación predeterminada es 100.

Configuración

la configuración a la que hace referencia la instancia. Al hacer clic en la configuración se abre la página Configuraciones para la instancia.

Nodo

Nombre del nodo en el que reside la instancia. Al hacer clic en el nombre se abre la página Editar nodo del nodo.

Estado

Una indicación de si la instancia se está ejecutando, si es necesario reiniciar para que se actualice con cambios de configuración, o si no se está ejecutando. Este campo es de sólo lectura.

La tabla Instancias del servidor también contiene las opciones siguientes.

Nuevo

Botón para agregar una instancia al clúster.

Borrar

Botón para eliminar una o varias instancias seleccionadas del clúster.

Precaución - Si utiliza un clúster Java Message Service (JMS) con un agente maestro, no suprima la instancia asociada al agente maestro. Si esta instancia se debe suprimir, utilice el subcomando change-master-broker(1) para asignar el agente maestro a una instancia diferente.

Inicio

Botón para iniciar una o varias instancias seleccionadas en el clúster.

Detener

Botón para detener una o varias instancias seleccionadas en el clúster.
