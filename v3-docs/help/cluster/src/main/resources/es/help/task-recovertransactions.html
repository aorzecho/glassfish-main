Para recuperar transacciones para una instancia de Servidor GlassFish agrupada en clúster

  1. En el árbol de navegación, expanda el nodo Clústeres.
  2. En el nodo Clústeres expandido, seleccione el nodo para el clúster.

    Se abrirá la página Información general para el clúster.

  3. En la página Información general, haga clic en la ficha Instancias.

    Se abrirá la página Instancias de servidor agrupadas para el clúster.

  4. Seleccione la instancia en la que se deben recuperar transacciones.

    Se abrirá la página Información general para la instancia.

  5. En la página Información General, haga clic en Recuperar transacciones.

    Se abrirá la página Recuperar transacciones.

    El contenido de la página depende de si la instancia se está ejecutando.

    • Si la instancia se está ejecutando, la página Recuperar transacciones solo contiene el texto Recovery is performed by the running server itself (La recuperación la realiza el propio servidor en ejecución) y el botón Aceptar.

    • Si la instancia no se está ejecutando, la página Recuperar transacciones contiene opciones adicionales.

  6. En la lista desplegable Servidor de destino, seleccione otra instancia del mismo clúster que pueda recuperar transacciones en la instancia actual.
  7. En el campo Directorio de registro de transacciones, escriba el directorio donde se guardarán los registros de transacciones.

    El valor predeterminado es /logs/tx en el directorio de dominio. Los registros de transacciones se guardan en el subdirectorio tx del directorio especificado en el campo Ubicación del registro de transacción del servicio de transacción para la configuración del clúster. El valor predeterminado de Ubicación del registro de transacción es el directorio especificado en el campo Raíz de registro de la página Atributos de dominio, que está ubicada en la ficha Avanzado del nodo Servidor GlassFish.

  8. Haga clic en Aceptar.
