Recuperar Transacciones para una Instancia de GlassFish Server en Cluster

  1. En el árbol de navegación, amplíe el nodo Clusters.

  2. En el nodo Clusters ampliado, seleccione el nodo para el cluster.

    Se abre la página Información General para el cluster.

  3. En la página Información General, haga clic en el separador Instancias.

    Se abre la página Instancias de Servidor en Cluster para el cluster.

  4. Seleccione la instancia para la que se deben recuperar transacciones.

    Se abre la página Información General para la instancia.

  5. En la página Información General, haga clic en Recuperar Transacciones.

    Se abre la página Recuperar Transacciones.

    El contenido de la página depende de si la instancia se está ejecutando.

    • Si la instancia se está ejecutando, la página Recuperar Transacciones sólo contiene el texto La recuperación la realiza el propio servidor en ejecución ) y el botón Aceptar.

    • Si la instancia no se está ejecutando, la página Recuperar Transacciones contiene opciones adicionales.

  6. En la lista desplegable Servidor de Destino, seleccione otra instancia del mismo cluster que pueda recuperar transacciones en la instancia actual.

  7. En el campo Directorio de Log de Transacciones, escriba el directorio donde se guardarán los logs de transacciones.

    El valor por defecto es /logs/tx en el directorio de dominio. Los logs de transacciones se guardan en el subdirectorio tx del directorio especificado por la configuración de Ubicación de Log de Transacciones del Servicio de Transacción para la configuración del cluster. El valor por defecto de Ubicación de Log de Transacciones es el directorio especificado en el campo Raíz de Log de la página Atributos de Dominio, que está ubicada en el separador Avanzado del nodo GlassFish Server.

  8. Haga clic en Aceptar.
