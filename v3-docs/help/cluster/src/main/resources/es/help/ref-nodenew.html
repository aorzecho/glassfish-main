Nuevo Nodo

Utilice la página Nuevo Nodo para crear un nodo de GlassFish Server. Un nodo representa un host en el que está instalado el software GlassFish Server. Debe existir un nodo para cada host en el que residan instancias de GlassFish Server.

Las opciones que aparezcan en la página Nuevo Nodo dependerán del tipo de nodo.

Opciones para Todos los Tipos de Nodos

Las opciones siguientes están disponibles para todos los tipos de nodos:

Nombre

Nombre del nodo.

El nombre debe cumplir los siguientes requisitos:

  • El nombre sólo puede contener caracteres ASCII.

  • El nombre debe empezar por una letra, un número o un carácter de subrayado.

  • El nombre sólo puede contener los siguientes caracteres:

    • Letras en minúscula

    • Letras en mayúscula

    • Números

    • Guión

    • Punto

    • Carácter de subrayado

  • El nombre debe ser exclusivo en el dominio y no debe ser el nombre de otro nodo, un cluster, una configuración con nombre ni una instancia de GlassFish Server.

  • El nombre no debe ser domain, server ni otra palabra clave reservada por GlassFish Server.

Tipo

Lista desplegable para especificar el tipo de nodo, que puede ser:

CONFIG

El nodo no soporta la comunicación remota. Si está seleccionada la opción CONFIG, no se muestran opciones de configuración de SSH o DCOM y se borra el contenido del campo Directorio de Instalación.

DCOM

El nodo soporta la comunicación mediante el protocolo remoto Distributed Component Object Model (DCOM). El protocolo DCOM está disponible sólo en los sistemas Windows.

SSH

El nodo soporta comunicación sobre shell seguro (SSH).

Host de Nodo

Nombre del host que el nodo representa. Este campo es obligatorio o no en función del tipo de nodo:

  • Si el tipo de nodo es CONFIG., el nombre del host puede omitirse. El nombre del host se puede determinar al crear las instancias que residen en el nodo.

  • Si el tipo de nodo es DCOM o SSH, se debe especificar el nombre del host. De lo contrario, se produce un error.

Directorio de Nodos

Ruta de acceso al directorio que va a contener las instancias de GlassFish Server que se crean en el nodo. Si se especifica una ruta de acceso relativa, la ruta de acceso es relativa al directorio as-install.

El valor por defecto depende del tipo de nodo:

  • Si el tipo de nodo es CONFIG, no se especifica ningún directorio de instancias para el nodo en la configuración del servidor de administración de dominio (DAS). El directorio del nodo se puede determinar al crear las instancias que residen en el nodo.

  • Si el tipo de nodo es DCOM, el valor por defecto es as-install\nodes, donde as-install es el directorio de instalación base del software GlassFish Server en el host.

  • Si el tipo de nodo es SSH, el valor por defecto es as-install/nodes, donde as-install es el directorio de instalación base del software GlassFish Server en el host.

Directorio de Instalación

Ruta de acceso completa al principal del directorio de instalación base del software GlassFish Server en el host, por ejemplo /export/glassfish3.

El valor por defecto depende del tipo de nodo:

  • Si el tipo de nodo es CONFIG., no se especifica ningún directorio de instalación para el nodo en la configuración de DAS. El directorio de instalación se puede determinar al crear las instancias que residen en el nodo.

  • Si el tipo de nodo es DCOM o SSH, el valor por defecto es el principal del directorio de instalación base por defecto del software GlassFish Server para DAS. Este valor por defecto sólo es útil si se ha instalado GlassFish Server en la misma ubicación en todos los hosts.

Opciones Adicionales para los Nodos DCOM

Las opciones siguientes están disponibles sólo si el campo Tipo se define en DCOM:

Instalar GlassFish Server

Si se selecciona la opción Activado, DAS copiará el software GlassFish Server del host de DAS en el host del nodo. Para copiar el software GlassFish Server en el host, DAS debe poder contactar con el host mediante DCOM.

Forzar

Si se selecciona la opción Activado, se crea el nodo en la configuración de DAS aunque falle la validación de los parámetros del nodo. Para validar los parámetros de un nodo, DAS debe poder establecer contacto con el host del nodo a través de DCOM.

Validar DCOM

Si se selecciona la opción Activado, GlassFish Server prueba la conexión mediante DCOM con el host del nodo antes de crear el nodo. Como parte de la prueba, GlassFish Server copia un script llamado delete_me.bat en el host.

Si selecciona esta opción, se activa el campo Directorio de Prueba Remoto para permitirle proporcionar el directorio en el que GlassFish Server debe copiar el script.

Directorio de Prueba Remoto

Carpeta del host remoto en la que GlassFish Server debe crear el archivo delete_me.bat. El usuario que la opción Nombre de Usuario de Windows especifica debe tener acceso de escritura a esta carpeta. De lo contrario, se produce un error. El valor por defecto es C:\.

Dominio de Windows

Nombre del dominio de Windows que contiene el usuario que especifica la opción de Nombre de Usuario de Windows. El valor por defecto es el nombre del host en el que se está ejecutando la Consola de Administración.

Nombre de Usuario de Windows

Usuario del host de este nodo que va a ejecutar el proceso para conectarse al host a través de DCOM. El valor por defecto es el usuario que está ejecutando el proceso DAS. Si la opción Host de Nodo se define en localhost-domain, se ignora la opción Nombre de Usuario de Windows.

Autenticación de Usuario de Windows

Lista desplegable para especificar cómo se autentica el usuario de Windows al conectarse al host de este nodo. Los métodos de autenticación posibles son los siguientes:

Contraseña

El usuario de Windows se autentica mediante una contraseña que se almacena en la configuración de DAS.

Atención:

Esta contraseña se almacena en forma de texto no cifrado en la configuración de DAS. Para mayor seguridad, utilice un archivo de claves o un alias de contraseña.

Si selecciona esta opción, los campos Contraseña de Usuario de Windows y Confirmar Contraseña se activan para que pueda indicar la contraseña.

Alias de Contraseña

El usuario de Windows se autentica mediante un alias de contraseña. Si selecciona esta opción, seleccione el alias de contraseña en la lista desplegable Alias de Contraseña.

Contraseña de Usuario de Windows

Contraseña que el usuario de Windows utilizará al conectarse al host de este nodo.

Confirmar Contraseña

Confirmación de la contraseña que el usuario de Windows utilizará al conectarse al host de este nodo. El contenido de este campo debe coincidir exactamente con el del campo Contraseña de Usuario de Windows.

Alias de Contraseña

Lista desplegable de alias de contraseña definidos para este dominio.

Opciones Adicionales para los Nodos SSH

Las opciones siguientes están disponibles sólo si el campo Tipo se define en SSH:

Instalar GlassFish Server

Si se selecciona la opción Activado, DAS copiará el software GlassFish Server del host de DAS en el host del nodo. Para copiar el software GlassFish Server en el host, DAS debe poder contactar con el host mediante SSH.

Forzar

Si se selecciona la opción Activado, se crea el nodo en la configuración de DAS aunque falle la validación de los parámetros del nodo. Para validar los parámetros de un nodo, DAS debe poder establecer contacto con el host del nodo a través de SSH.

Puerto SSH

Puerto que se usará para las conexiones SSH con el host de este nodo. El valor por defecto es 22. Si la opción Host de Nodo se define en localhost-domain, se ignora la opción Puerto SSH.

Nombre de Usuario SSH

Usuario del host de este nodo que va a ejecutar el proceso para conectarse al host a través de SSH. El valor por defecto es el usuario que está ejecutando el proceso DAS. Para asegurarse de que DAS pueda leer el archivo de claves privadas SSH de este usuario, especifique el usuario que está ejecutando el proceso DAS. Si la opción Host de Nodo se define en localhost-domain, se ignora la opción Nombre de Usuario.

Autenticación de Usuario de SSH

Lista desplegable para especificar cómo se autentica el usuario de SSH alconectarse al host de este nodo. Los métodos de autenticación posibles son los siguientes:

Archivo de Claves

El usuario de SSH se autentica a través de un archivo de claves privadas SSH. Si selecciona esta opción, especifique la ruta de acceso absoluta al archivo en el campo Archivo de Claves.

Contraseña

El usuario de SSH se autentica mediante una contraseña que se almacena en la configuración de DAS.

Atención:

Esta contraseña se almacena en forma de texto no cifrado en la configuración de DAS. Para mayor seguridad, utilice un archivo de claves o un alias de contraseña.

Si selecciona esta opción, los campos Contraseña de Usuario de SSH y Confirmar Contraseña se activan para que pueda indicar la contraseña.

Alias de Contraseña

El usuario de SSH se autentica mediante un alias de contraseña. Si selecciona esta opción, seleccione el alias de contraseña en la lista desplegable Alias de Contraseña.

Configurar SSH

Si se selecciona la opción Activado, GlassFish Server configura una clave SSH en el host del nodo. GlassFish Server configura la clave SSH copiando archivos de claves existentes desde el host de DAS o, en caso necesario, generando los archivos antes de copiarlos. Esta opción está disponible sólo si la opción Autenticación de Usuario de SSH se define en Archivo de Claves.

Generar Clave

Si se selecciona la opción Activado, GlassFish Server genera los archivos de claves SSH antes de copiar los archivos en el host del nodo.

Contraseña de Usuario de SSH

La contraseña que el usuario de SSH utilizará al conectarse al host de este nodo.

Confirmar Contraseña

Confirmación de la contraseña que el usuario de SSH utilizará al conectarse al host de este nodo. El contenido de este campo debe coincidir exactamente con el del campo Contraseña de Usuario de SSH.

Alias de Contraseña

Lista desplegable de alias de contraseña definidos para este dominio.

Archivo de Claves Públicas

Ruta de acceso absoluta al archivo de claves públicas SSH para el usuario que especifique la opción Nombre de Usuario de SSH. El contenido del archivo de claves públicas se agrega al archivo .ssh/authorized_keys del usuario en el host del nodo. Si el archivo .ssh/authorized_keys no existe en el host, GlassFish Server lo crea.

La ruta de acceso al archivo de claves debe ser accesible para DAS y el archivo de claves debe ser legible para el mismo.

El valor por defecto es un archivo de claves en el directorio .ssh del usuario. Si se encuentran varios archivos de claves, GlassFish Server utiliza el siguiente orden de preferencia:

  1. id_rsa

  2. id_dsa

  3. identity

Archivo de Claves

Ruta de acceso absoluta al archivo de claves privadas SSH para el usuario que especifique la opción Nombre de Usuario de SSH. Este archivo se utiliza para la autenticación en el daemon sshd en el host del nodo.

Nota:

GlassFish Server también soporta la autenticación con contraseña a través de la entrada AS_ADMIN_SSHPASSWORD en el archivo de contraseñas. El archivo de contraseñas se especifica en la opción -paswordfile de la utilidad asadmin(1).

Si el archivo de claves privadas SSH se protege con una contraseña, el archivo de contraseñas debe contener la entrada AS_ADMIN_SSHKEYPASSPHRASE.

La ruta de acceso al archivo de claves debe ser accesible para DAS y el archivo de claves debe ser legible para el mismo.

El valor por defecto es un archivo de claves en el directorio .ssh del usuario. Si se encuentran varios archivos de claves, GlassFish Server utiliza el siguiente orden de preferencia:

  1. id_rsa

  2. id_dsa

  3. identity
