Nuevo nodo

Utilice la página Nuevo nodo para crear un nodo de GlassFish Server. Un nodo representa un host en el que está instalado el software GlassFish Server. Debe existir un nodo en cada host en el que residan instancias de GlassFish Server.

La página Nuevo nodo contiene las opciones siguientes:

Nombre

El nombre del nodo.

El nombre debe cumplir los siguientes requisitos:

  • El nombre debe empezar por una letra, un número o un carácter de subrayado.

  • El nombre sólo puede contener los siguientes caracteres:

    • Letras en minúscula

    • Letras en mayúscula

    • Números

    • Guión

    • Punto

    • Subrayado

  • El nombre debe ser exclusivo en el dominio y no debe ser el nombre de otro nodo, un clúster, una configuración con nombre ni una instancia de GlassFish Server.

  • El nombre non debe ser domain, server ni otra palabra clave reservada para GlassFish Server.

Tipo

Lista desplegable para especificar el tipo de nodo, que puede ser:

SSH

El modo admite comunicación sobre Secure Shell (SSH).

CONFIG

El nodo no admite comunicación remota. Si está seleccionada la opción CONFIG, no se muestran opciones de configuración de SSH y se borra el campo Directorio de instalación.

Host de nodo

El nombre del host que el nodo representa. Este campo es obligatorio o no en función del tipo de nodo:

  • Si el tipo de nodo es CONFIG., el nombre del host puede omitirse. El nombre del host se puede determinar cuando se crean las instancias que residen en el nodo.

  • Si el tipo de nodo es SSH, se debe especificar el nombre del host. De lo contrario, se produce un error.

Directorio de nodos

Ruta al directorio que va a contener las instancias de GlassFish Server que se crean en el nodo. Si se especifica una ruta relativa, la ruta es relativa al directorio as-install.

El valor predeterminado depende del tipo de nodo:

  • Si el tipo de nodo es CONFIG, no se especifica directorio de instancias para el nodo en la configuración del servidor de administración de dominios (DAS). El directorio del nodo se puede determinar cuando se crean las instancias que residen en el nodo.

  • Si el tipo de nodo es SSH, el valor predeterminado es as-install/nodes, donde as-install es el directorio base de instalación del software GlassFish Server en el host.

Directorio de instalación

La ruta completa al directorio superior del directorio base de instalación del software GlassFish Server en el host, por ejemplo /export/glassfish3.

El valor predeterminado depende del tipo de nodo:

  • Si el tipo de nodo es CONFIG., no se especifica directorio de instalación para el nodo en la configuración del DAS. El directorio de instalación se puede determinar al crear las instancias que residen en el nodo.

  • Si el tipo de nodo es SSH, el valor predeterminado es el directorio superior del directorio de instalación predeterminado del software GlassFish Server para el DAS. Este valor predeterminado sólo es útil si se ha instalado GlassFish Server en la misma ubicación en todos los hosts.

Forzar

Si se selecciona la opción Habilitado, se crea el nodo en la configuración de DAS aunque falle la validación de los parámetros del nodo. Para validar los parámetros de un nodo, el DAS debe poder establecer contacto con el host del nodo a través de SSH.

Puerto SSH

Puerto que se usará para las conexiones SSH con el host de este nodo. El valor predeterminado es 22. Si la opción Host de nodo se configura como localhost-dominio, se hace caso omiso de la opción Puerto SSH.

Nombre de usuario SSH

El usuario del host de este nodo que va a ejecutar el proceso para conectarse al host a través de SSH. El valor predeterminado es el usuario que está ejecutando el proceso DAS. Para asegurarse de que el DAS pueda leer el archivo de clave privada de este usuario SSH, especifique el usuario que está ejecutando el proceso DAS. Si la opción Host de nodo se configura como localhost-dominio, se hace caso omiso de la opción Nombre de usuario.

Autenticación de usuario de SSH

Lista desplegable para especificar cómo se autentica el usuario de SSH al iniciar sesión en el host de este nodo. Los métodos de autenticación posibles son los siguientes:

Archivo clave

El usuario de SSH se autentica a través de un archivo de claves privadas SSH. Si selecciona esta opción, especifique la ruta de acceso absoluta al archivo en el campo Archivo de claves.

Contraseña

El usuario de SSH se autentica mediante una contraseña que se almacena en la configuración de DAS.

Precaución - Esta contraseña se almacena en forma de texto en la configuración de DAS. Para mayor seguridad, utilice un archivo de claves o un alias de contraseña.

Si selecciona esta opción, los campos Contraseña SSH y Confirmar contraseña SSH se activan para que pueda indicar la contraseña.

Alias de contraseña

El usuario de SSH se autentica mediante un alias de contraseña. Si selecciona esta opción, seleccione el alias de contraseña en la lista desplegable Alias de contraseña.

Contraseña SSH

La contraseña que el usuario de SSH utilizará al iniciar sesión en el host de este nodo.

Confirmar contraseña SSH

Confirmación de la contraseña que el usuario de SSH utilizará al iniciar sesión en el host de este nodo. El contenido de este campo debe coincidir exactamente con el del campo Contraseña SSH.

Alias de contraseña

Lista desplegable de alias de contraseña definidos para este dominio.

Archivo clave

Ruta absoluta al archivo de claves privadas SSH para el usuario que especifique la opción Nombre de usuario. Este archivo se utiliza para la autenticación en el daemon sshd en el host del nodo.

Nota - GlassFish Server también admite la autenticación con contraseña a través de la entrada AS_ADMIN_SSHPASSWORD en el archivo de contraseñas. El archivo de contraseñas se configura en la opción --paswordfile de la utilidad asadmin(1M).

Si el archivo de claves privadas SSH se protege con una contraseña, el archivo de contraseñas debe contener la entrada AS_ADMIN_SSHKEYPASSPHRASE.

La ruta al archivo de claves debe ser accesible al DAS y el archivo de claves debe ser legible para el DAS.

El valor predeterminado es un archivo de claves en el directorio .ssh del usuario. Si se encuentran varios archivos de claves, GlassFish Server utiliza el siguiente orden de preferencia:

  1. id_rsa

  2. id_dsa

  3. identity
