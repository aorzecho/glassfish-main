So migrieren Sie EJB-Zeitgeber

Wenn eine GlassFish-Server-Instanz angehalten wird, werden ebenfalls die auf dieser Instanz installierten EJB-Zeitgeber (Enterprise JavaBeans) angehalten. Wenn die Instanz anormal oder unerwartet angehalten wird, können Sie bei angehaltener Instanz diese Zeitgeber aktivieren, indem Sie die Zeitgeber auf eine ausgeführte Serverinstanz im Cluster migrieren.

  1. Erweitern Sie in der Navigationsstruktur den Knoten "Cluster".
  2. Wählen Sie in der Navigationsstruktur im Knoten "Cluster" den Cluster aus.

    Die Seite "Allgemeine Informationen" für den Cluster wird geöffnet.

  3. Klicken Sie auf der Seite "Allgemeine Informationen" auf "EJB-Zeitgeber migrieren".

    Die Seite "EJB-Zeitgeber migrieren" wird geöffnet.

    Hinweis - Wenn alle Instanzen im Cluster ausgeführt werden, ist die Schaltfläche "EJB-Zeitgeber migrieren" deaktiviert.

  4. Wählen Sie auf der Seite "EJB-Zeitgeber migrieren" die Instanz, von der und auf die die Zeitgeber migriert werden sollen.
    1. Wählen Sie in der Dropdown-Liste "Quelle" die angehaltene Instanz, von der aus Sie die Zeitgeber migrieren möchten.
    2. (Optional) Wählen Sie in der Dropdown-Liste "Ziel" die ausgeführte Instanz, auf die Sie die Zeitgeber migrieren möchten.
    3. Klicken Sie auf "OK".
  5. Halten Sie die Instanz, auf die Sie die EJB-Zeitgeber migriert haben, an und starten Sie sie erneut.

    Wenn die Quellinstanz ausgeführt oder die Zielinstanz nicht ausgeführt wird, zeigt die Administration Console eine Fehlermeldung an.
