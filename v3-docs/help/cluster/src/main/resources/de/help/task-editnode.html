So bearbeiten Sie einen Knoten

Durch das Bearbeiten eines Knotens werden Konfigurationsdaten eines GlassFish-Server-Knotens aktualisiert. Außerdem wird durch das Bearbeiten eines Knotens der Knotentyp von KONFIGURATION in SSH geändert, um die Kommunikation über SSH für den Knoten zu aktivieren.

Nach dem Ändern des Knotentyps von KONFIGURATION in SSH werden die Standardwerte angewendet, wenn eines der folgenden Felder leer ist:

  • SSH-Anschluss

  • SSH-Benutzername

  • Schlüsseldatei

  1. Erweitern Sie in der Navigationsstruktur den Knoten "Knoten".
  2. Wählen Sie in der Liste der Knoten den zu bearbeitenden Knoten.

    Die Seite "Knoten bearbeiten" wird geöffnet.

  3. Wählen Sie aus der Dropdown-Liste "Typ" den aktualisierten Knotentyp.
    • Wählen Sie SSH, um eine Kommunikation über Secure Shell (SSH) für den Knoten zu unterstützen.
    • Wählen Sie KONFIGURATION, um keine Remote-Konfiguration für den Knoten zu unterstützen.

      Wenn Sie KONFIGURATION wählen, werden keine Optionen für die SSH-Konfiguration angezeigt. Wenn der aktuelle Knotentyp SSH ist und Sie KONFIGURATION auswählen, werden beim Speichern der Änderungen die SSH-Konfigurationsdaten aus der Knotenkonfiguration entfernt.

  4. Geben Sie im Feld "Knoten-Host" den Namen des Hosts ein, den der aktualisierte Knoten darstellen soll.

    Ob dieses Feld obligatorisch ist, hängt vom Knotentyp ab:

    • Wenn der Knotentyp KONFIGURATION ist, kann der Name des Hosts ausgelassen werden. Der Name des Hosts kann festgelegt werden, wenn die Instanzen erstellt werden, die sich auf dem Knoten befinden.

    • Wenn der Knotentyp SSH ist, muss der Name des Hosts angegeben werden. Anderenfalls tritt ein Fehler auf.

  5. Geben Sie im Feld "Knotenverzeichnis" den Pfad zu dem Verzeichnis ein, das die GlassFish-Server-Instanzen enthalten soll, die auf dem Knoten erstellt werden.

    Wenn Sie einen relativen Pfad angeben, ist der Pfad relativ zum Verzeichnis as-install. Dabei steht as-install für das Basisinstallationsverzeichnis der GlassFish-Server-Software auf dem Host.

  6. Geben Sie im Feld "Installationsverzeichnis" den vollständigen Pfad zum übergeordneten Verzeichnis des Basisinstallationsverzeichnis der GlassFish-Server-Software auf dem Host ein. Beispiel: /export/glassfish3/.
  7. Damit der Knoten in der Konfiguration des Domänenadministrationsservers (DAS, Domain Administration Server) erstellt wird, auch wenn die Validierung der Knotenparameter fehlschlägt, aktivieren Sie die Option "Erzwingen".
  8. Geben Sie im Feld "SSH-Anschluss" den Port ein, der für SSH-Verbindungen zum Host dieses Knotens verwendet werden soll.

    Wenn Sie den Knotentyp von KONFIGURATION in SSH ändern und dieses Feld leer lassen, wird die Option "SSH-Anschluss" auf 22 gesetzt.

  9. Geben Sie im Feld "SSH-Benutzername" den Namen des Benutzers auf dem Host dieses Knotens an, der die Verbindung mit dem Host über SSH herstellen soll.

    Wenn Sie den Knotentyp von KONFIGURATION in SSH ändern und dieses Feld leer lassen, wird für die Option "SSH-Benutzername" der Benutzer verwendet, der den DAS-Prozess ausführt.

  10. Geben Sie an, wie der SSH-Benutzer authentifiziert wird, wenn er sich beim Host dieses Knotens anmeldet.
    • Um die Authentifizierung durch eine Datei für den privaten SSH-Schlüssel anzugeben, gehen Sie wie folgt vor:
      1. Wählen Sie aus der Dropdown-Liste "SSH-Benutzerauthentifizierung" die Option "Schlüsseldatei".
      2. Geben Sie im Feld "Schlüsseldatei" den absoluten Pfad zur Datei für den privaten SSH-Schlüssel des Benutzers ein, der durch die Option "SSH-Benutzername" angegeben ist.

        Hinweis - GlassFish-Server unterstützt ebenfalls eine Passwort-Authentifizierung über den Eintrag AS_ADMIN_SSHPASSWORD in der Passwortdatei. Die Passwort-Datei wird in der Option --paswordfile des Dienstprogramms asadmin(1M) angegeben.

        Wenn Sie den Knotentyp von KONFIGURATION in SSH ändern und dieses Feld leer lassen, wird die Option "Schlüsseldatei" auf eine Schlüsseldatei im Verzeichnis .ssh des Benutzers gesetzt. Wenn mehrere Schlüsseldateien gefunden werden, verwendet GlassFish-Server die folgende Reihenfolge:

        • id_rsa

        • id_dsa

        • identity

    • Um die Authentifizierung durch ein Passwort anzugeben, die in der DAS-Konfiguration gespeichert wird, gehen Sie wie folgt vor:

      Achtung - Dieses Passwort wird als Klartext in der DAS-Konfiguration gespeichert. Verwenden Sie eine Schlüsseldatei oder eine Passwort-Alias, um die Sicherheit zu erhöhen.

      1. Wählen Sie aus der Dropdown-Liste "SSH-Benutzerauthentifizierung" die Option "Passwort".
      2. Geben Sie im Feld "SSH-Passwort" das Passwort ein, das der SSH-Benutzer verwendet, wenn er sich beim Host dieses Knotens anmeldet.
      3. Geben Sie in das Feld "SSH-Passwort bestätigen" erneut das Passwort ein.
    • Um die Authentifizierung durch ein Passwort-Alias anzugeben, gehen Sie wie folgt vor:
      1. Wählen Sie aus der Dropdown-Liste "SSH-Benutzerauthentifizierung" die Option "Passwort-Alias".
      2. Wählen Sie aus der Dropdown-Liste "Passwort-Alias" ein vorhandenes Alias.
  11. Klicken Sie auf "Speichern".
