So stellen Sie Vorgänge für eine eigenständige GlassFish-Server-Instanz wieder her

  1. Wählen Sie in der Navigationsstruktur den Knoten "Eigenständige Instanzen".

    Die Seite "Eigenständige Serverinstanz" wird geöffnet.

  2. Wählen Sie die Instanz, für die Wiederherstellung von Vorgängen erforderlich ist.

    Die Seite "Allgemeine Informationen" für die Instanz wird geöffnet.

  3. Klicken Sie auf der Seite "Allgemeine Informationen" auf "Wiederherstellungsvorgänge".

    Die Seite "Wiederherstellungsvorgänge" wird angezeigt.

    Der Inhalt der Seite hängt davon ab, ob die Instanz ausgeführt wird.

    • Wenn die Instanz ausgeführt wird, enthält die Seite "Wiederherstellungsvorgänge" nur den Text Recovery is performed by the running server itself (Die Wiederherstellung wird vom Server selbst durchgeführt) und die Schaltfläche "OK".

    • Wenn die Instanz nicht ausgeführt wird, enthält die Seite "Wiederherstellungsvorgänge" zusätzliche Optionen.

  4. Wählen Sie in der Dropdown-Liste "Zielserver" eine andere Instanz, die die Vorgänge in der aktuellen Instanz wiederherstellen kann.

    Wählen Sie bei einer eigenständigen Instanz die Instanz namens server, die den Domänenadministrationsserver (DAS) repräsentiert.

  5. Geben Sie im Feld "Transaktionsprotokollverzeichnis" das Verzeichnis ein, in dem Transaktionsprotokolle gespeichert sind.

    Standardmäßig handelt es sich um /logs/tx im Domänenverzeichnis. Transaktionsprotokolle werden im Unterverzeichnis tx des Verzeichnisses gespeichert, das durch die Einstellung "Verzeichnis des Transaktionsprotokolls" des Transaktionsdienstes für die Clusterkonfiguration angegeben ist. Der Standardwert für "Verzeichnis des Transaktionsprotokolls" ist das Verzeichnis, das auf der Seite "Domänenattribute" im Feld "Protokollstamm" angegeben ist, das sich im Knoten GlassFish-Server auf der Registerkarte "Erweitert" befindet.

  6. Klicken Sie auf "OK".
