Cluster-Serverinstanzen

Auf der Seite "Cluster-Serverinstanzen" können Sie GlassFish-Server-Instanzen im aktuellen Cluster erstellen und verwalten.

Das Feld "Clustername" ist ein schreibgeschütztes Feld, in dem der Name des aktuellen Clusters angezeigt wird.

Auf der Seite "Cluster-Serverinstanzen" werden die Instanzen aufgelistet, die zum aktuellen Cluster gehören. Für jede Instanz werden die folgenden Informationen zur Verfügung gestellt.

Name

Der Name, der der Instanz bei der Erstellung zugewiesen wurde. Wenn Sie auf den Namen klicken, wird die Seite Allgemeine Informationen für die Instanz geöffnet.

Gewichtung des Lastenausgleichs

Eine Ganzzahl, die die Gewichtung des Lastenausgleichs der Instanz darstellt.

Die Gewichtung des Lastenausgleichs bestimmt den Anteil aller Anforderungen an den Cluster, die von der Instanz verarbeitet werden.

Beispielsweise muss bei einem Cluster mit zwei Instanzen eine Instanz eine von vier Anforderungen und die andere Instanz drei von vier Anforderungen verarbeiten. Definieren Sie in diesem Fall für die Gewichtung der Instanz, die eine von vier Anforderungen verarbeiten soll, den Wert 1 und für die Gewichtung der anderen Instanz den Wert 3. Wenn Sie stattdessen Prozentsätze bevorzugen, legen Sie für die Gewichtung der Instanzen jeweils den Prozentwert 25 und 75 fest.

Die Standardgewichtung ist 100.

Konfiguration

Die Konfiguration, auf die die Instanz verweist. Durch Klicken auf die Konfiguration wird die Seite Konfigurationen für die Instanz geöffnet.

Knoten

Der Name des Knotens, auf dem sich die Instanz befindet. Wenn Sie auf den Namen klicken, wird die Seite "Knoten bearbeiten" für den Knoten geöffnet (siehe Knoten bearbeiten).

Status

Gibt an, ob die Instanz ausgeführt wird, einen Neustart benötigt, um mit Konfigurationsänderungen aktualisiert zu werden, oder nicht ausgeführt wird. Dieses Feld ist schreibgeschützt.

Die Tabelle "Serverinstanzen" enthält darüber hinaus die folgenden Optionen.

Neu

Schaltfläche zum Hinzufügen einer Instanz zum Cluster.

Löschen

Schaltfläche zum Entfernen von mindestens einer ausgewählten Instanz aus dem Cluster.

Achtung - Löschen Sie nicht die Instanz, die mit dem Master Broker verknüpft ist, wenn Sie einen Java Message Service (JMS)-Cluster mit einem Master Broker verwenden. Wenn diese Instanz gelöscht werden muss, verwenden Sie den Unterbefehl change-master-broker(1), um den Master Broker einer anderen Instanz zuzuweisen.

Starten

Schaltfläche zum Starten von mindestens einer ausgewählten Instanz im Cluster.

Anhalten

Schaltfläche zum Anhalten von mindestens einer ausgewählten Instanz im Cluster.

Verwandte Aufgaben

  • So fügen Sie eine GlassFish-Server-Instanz zu einem Cluster hinzu

  • So löschen Sie GlassFish-Server-Cluster-Instanzen

  • So starten Sie GlassFish-Server-Cluster-Instanzen

  • So halten Sie GlassFish-Server-Cluster-Instanzen an
