Wiederherstellungsvorgänge

Auf der Seite "Wiederherstellungsvorgänge" können Sie Vorgänge für eine GlassFish-Server-Instanz wiederherstellen.

Wenn die Instanz ausgeführt wird, enthält die Seite "Wiederherstellungsvorgänge" nur den Text Recovery is performed by the running server itself (Die Wiederherstellung wird vom Server selbst durchgeführt) und die Schaltfläche "OK".

Wenn die Instanz nicht ausgeführt wird, enthält die Seite "Wiederherstellungsvorgänge" die folgenden Optionen.

Instanzname

Der Name, der der Instanz bei der Erstellung zugewiesen wurde. Dieses Feld ist schreibgeschützt.

Zielserver

Eine weitere Instanz, die die Vorgänge in der aktuellen Instanz wiederherstellen kann.

Transaktionsprotokollverzeichnis

Das Verzeichnis, in dem Transaktionsprotokolle gespeichert sind. Standardmäßig handelt es sich um /logs/tx im Domänenverzeichnis.

Transaktionsprotokolle werden im Unterverzeichnis tx des Verzeichnisses gespeichert, das durch die Einstellung "Verzeichnis des Transaktionsprotokolls" des Transaktionsdienstes für die Clusterkonfiguration angegeben ist. Der Standardwert für "Verzeichnis des Transaktionsprotokolls" ist das Verzeichnis, das auf der Seite "Domänenattribute" im Feld "Protokollstamm" angegeben ist, das sich im Knoten GlassFish-Server auf der Registerkarte "Erweitert" befindet.

Verwandte Aufgaben

  • So stellen Sie Vorgänge für eine GlassFish-Server-Cluster-Instanz wieder her

  • So stellen Sie Vorgänge für eine eigenständige GlassFish-Server-Instanz wieder her

Entsprechende asadmin-Unterbefehle

  • recover-transactions(1)
