Informações Gerais

A página Informações gerais contém as seguintes opções.

Iniciar

Botão para iniciar a instância. Se a instância estiver em execução, este botão está desativado.

Interromper

Botão para interromper a instância. Se a instância estiver interrompida, este botão está desativado.

Reiniciar

Botão para reiniciar a instância. Se a instância estiver interrompida, este botão está desativado.

Exibir arquivos de log

Botão para exibir os arquivos de log da instância.

Efetuar rotação de log

Botão para efetuar rotação do arquivo log da instância. Quando o arquivo de log for rotacionado, ele será renomeado com um nome do timestamp no formato server.log_date-and-time e um arquivo de log vazio será criado para novas mensagens de log. As alterações são aplicadas dinamicamente. Não é necessário reiniciar o servidor.

Recuperar transações

Botão para recuperar transações da instância.

Nome

Nome designado à instância quando a instância foi criada. Este campo é somente para leitura.

JVM

Se você clicar em JVM Report, será aberta uma janela separada que exibe relatórios sobre o Java Virtual Machine, incluindo um relatório de resumo, um relatório de gerenciamento de memória e de coleta de erros,um relatório de carregamento de classe e um dump de threads atuais.

Status

Indicação de que a instância está ou não em execução. Este campo é somente para leitura.

Nó

O nome do nó no qual a instância reside. Clicar no nome abre a página Editar Nó para o nó.

Configuração

Configuração nomeada que a instância faz referência. Ao clicar na configuração, a página Configurações da instância se abre.

Depuração

Um indicativo de que a depuração do JPDA (Java Platform Debugger Architecture) http://java.sun.com/javase/technologies/core/toolsapis/jpda/) está ativada para a instância. Este campo é somente para leitura.

Se uma instância é iniciada a partir do Console de Administração, a depuração JPDA não está ativada para a instância. Para iniciar uma instância com a depuração do JPDA ativada, use o subcomando start-instance(1).

HTTP Ports (Portas HTTP)

Lista separada por vírgulas dos números das portas na qual a instância faz listening dos seguintes tipos de solicitações:

  • Solicitações de administração

  • Solicitações HTTP

  • Solicitações HTTPS

Este campo é somente para leitura.

IIOP Ports (Portas IIOP)

Lista separada por vírgulas dos números das portas que a instância utiliza nos seguintes tipos de conexões:

  • Conexão IIOP segura

  • Conexões IIOP seguras com autenticação do cliente

  • Conexões IIOP

Este campo é somente para leitura.
