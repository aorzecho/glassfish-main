Para criar uma instância independente do Servidor GlassFish

As instâncias do Servidor GlassFish formam a base da implantação de um aplicativo. A instância independente não é um membro de um cluster. Cada instância pertence a um único domínio e tem sua própria estrutura de diretório, configuração e aplicativos implantados. Cada instância também inclui o contêiner Web da plataforma Java, Enterprise Edition (Java EE) e o contêiner da tecnologia Enterprise JavaBeans (tecnologia EJB). Cada instância nova deve conter uma referência a um nó que defina o host no qual a instância residirá.

Observação - Para criar uma instância em um nó de tipo CONFIG que represente um host remoto, você deve utilizar o subcomando create-local-instance(1). É possível utilizar o Console de Administração para este propósito.

Antes de começar

Certifique-se de que o nó no qual a instância residirá existe.

  1. Na árvore de navegação, selecione o nó Instâncias independentes.

    A página Instância de servidor independente é exibida.

  2. Na página Instâncias do servidor independente, clique em Novo.

    A página Nova instância de servidor independente é exibida.

  3. No campo Nome, digite um nome para a instância.

    O nome deve cumprir os seguintes requisitos:

    • O nome deve começar com uma letra, um número ou um sublinhado.

    • O nome pode conter somente os seguintes caracteres:

      • Letras minúsculas

      • Letras maiúsculas

      • Números

      • Hífen

      • Período

      • Sublinhado

    • O nome deve ser único no domínio e não deve ser o nome de outra instância, um cluster, uma configuração nomeada ou um nó do Servidor GlassFish.

    • O nome não deve ser domain, server, nem nenhuma outra palavra-chave que esteja reservada pelo Servidor GlassFish.

  4. Na lista suspensa Nó, selecione o nó no qual a instância residirá.
  5. Na lista suspensa Configuração, selecione a configuração nomeada que a instância utilizará.

    Observação - Se selecionar default-config, deve especificar na etapa seguinte que a configuração não é compartilhada. A configuração default-config não pode ser compartilhada.

  6. Especifique se a configuração que a instância utilizará é compartilhada.
    • Se a configuração não for compartilhada, deixe marcada a opção Fazer uma cópia da configuração selecionada.

      A configuração selecionada será copiada quando a instância for criado. À cópia da configuração é atribuído o nome nome-da-instância-config, em que nome-da-instância corresponde ao nome da instância que está sendo criada.

    • Se a configuração for compartilhada, selecione a opção Referenciar a configuração selecionada.

      A instância utilizará a configuração nomeada existente especificada.

  7. Clique em OK.
