Para Editar um Nó

Ao editar um nó, os dados de configuração de um nó do GlassFish Server são atualizados. A edição de um nó também permite que você altere o tipo do nó de CONFIG para DCOM ou SSH para ativar a comunicação remota para o nó.

Se você alterar o tipo de nó de CONFIG para DCOM, os valores default serão aplicados se qualquer um dos campos a seguir ficarem vazios:

  • Domínio do Windows

  • Usuário do Windows

Se o tipo de nó for alterado de CONFIG para SSH , os valores default são aplicados se nenhum dos campos a seguir forem deixados em branco:

  • Porta SSH

  • Nome de usuário SSH

  • Arquivo de Chave

  1. Na árvore de navegação, expanda o nó Nós.

  2. Na lista de nós, selecione o nó que está sendo editado.

    A página Editar nó é exibida.

  3. Na lista drop-down Tipo, selecione o tipo do nó atualizado.

    • Para ativar o nó de modo a suportar a comunicação por meio do protocolo remoto Distributed Component Object Model (DCOM), selecione DCOM.

    • Para permitir que o nó ofereça suporte à comunicação através de shell seguro (SSH), selecione SSH.

    • Para remover o suporte à configuração remota do nó, selecione CONFIG.

      Se você selecionar CONFIG, nenhuma opção para configuração do SSH ou DCOM será exibida. Se o tipo atual de nó for DCOM ou SSH e você selecionar CONFIG, os dados de configuração do DCOM ou do SSH serão removidos da configuração do nó quando você salvar as alterações.

  4. No campo Host do nó, digite o nome do host que o nó atualizado representará.

    Se este campo é obrigatório ou não, depende do tipo de nó:

    • Se o tipo de nó for CONFIG, o nome do host pode ser omitido. O nome do host pode ser determinado quando as instâncias que residem no nó forem criadas.

    • Se o tipo de nó for DCOM ou SSH, o nome do host deverá ser especificado. Caso contrário, ocorre um erro.

  5. No campo Diretório do nó, digite o caminho para o diretório que contém as instâncias GlassFish Server que são criadas no nó.

    Se você digitar um caminho relativo, o caminho será relativo ao diretório as-install em que as-install é o diretório de instalação base do software GlassFish Server no host.

  6. No campo Diretório de Instalação, digite o caminho completo para o pai do diretório de instalação base do software GlassFish Server no host, por exemplo: /export/glassfish3.

  7. Finalize a atualização do nó conforme exigido pelo tipo do nó.

    1. Se o tipo de nó for CONFIG, clique em Salvar.

    2. Se o tipo de nó for DCOM, siga as instruções contidas em Para Finalizar a Atualização de um Nó DCOM.

    3. Se o tipo de nó for SSH, siga as instruções contidas em Para Finalizar a Atualização de um Nó SSH.
