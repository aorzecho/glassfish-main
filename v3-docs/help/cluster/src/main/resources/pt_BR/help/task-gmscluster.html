Para configurar as definições do serviço de gerenciamento de grupos de um cluster

O serviço de gerenciamento de grupos (GMS) é um serviço em processo que oferece monitoramento de cluster, associação de cluster e serviços de comunicação de grupos. O GMS realiza as seguintes funções:

  • Notifica os módulos registrados em uma instância do Servidor GlassFish quando uma ou mais instâncias do membro são adicionadas ou removidas de um cluster, ou quando tais instâncias sejam suspeitas de ter falhas ou realmente tenham falhas.

  • Oferece a capacidade de enviar e receber mensagens entre um grupo de processos.

  1. Na árvore de navegação, expanda o nó Clusters.
  2. Selecione o cluster para o qual estiver configurando as definições do serviço de gerenciamento de grupos (GMS).

    A página Informações gerais do cluster se abre.

  3. Marque a caixa de seleção GMS ativado para ativar o GMS do cluster.

    Por padrão, essa opção está selecionada. Para desativar o GMS do cluster, desmarque a caixa de seleção GMS ativado.

    Se ativado, o serviço de gerenciamento de grupos (GMS) é iniciado em cada instância do servidor do cluster e do servidor de administração do domínio (DAS). O DAS participa de cada cluster no qual o GMS estiver ativado.

  4. No campo Porta multicast, insira a porta que o serviço de gerenciamento de grupos utiliza para escutar os eventos dos grupos.

    Especifique um número de porta UDP padrão no intervalo de 2048 a 32000. O padrão é um valor gerado automaticamente dentro deste intervalo.

  5. No campo Endereço multicast, insira o endereço que o serviço de gerenciamento de grupos utiliza para escutar os eventos dos grupos.

    Especifique um endereço IP multicast de classe D. Os endereços IP de classe D estão no intervalo de 224.0.0.0 a 239.255.255.255. O endereço 224.0.0.0 está reservado e não deve ser utilizado. O padrão é 228.9.XX. YY, em que XX e YY são valores independentes gerados automaticamente entre 0 e 255.

  6. No campo Endereço da interface de vinculação, insira o endereço da interface de rede do serviço de gerenciamento de grupos se houver mais de uma interface de rede.

    Esta definição permite configurar qual interface de rede será utilizada no GMS se houver mais de uma em qualquer uma das máquinas que hospedam as instâncias do servidor no cluster. Este valor deve ser um endereço IP de interface de rede local. Por padrão, são utilizados todos os endereços de interface de rede.

    Para especificar um endereço que seja válido para todas as instâncias do Servidor GlassFish do cluster, utilize uma propriedade do sistema para definir o endereço de forma individual de cada instância.

    Crie, por exemplo, a propriedade de sistema GMS-BIND-INTERFACE-ADDRESS-nome-do-cluster no DAS. A seguir, defina a opção Endereço de interface de vinculação ${GMS-BIND-INTERFACE-ADDRESS-nome-do-cluster} para especificar a propriedade do sistema. Finalmente, defina em cada instância do cluster a propriedade do sistema GMS-BIND-INTERFACE-ADDRESS-nome-do-cluster como o endereço da interface de rede na máquina da instância.

  7. (Opcional) Para especificar as propriedades adicionais, selecione a guia Propriedades e a subguia Propriedades do cluster.

    O Servidor GlassFish não define propriedades adicionais para o GMS.

    • Para adicionar uma propriedade, clique no botão Adicionar propriedade. Na linha em branco que é exibida, digite o nome da propriedade no campo Nome e digite o valor da propriedade no campo Valor.
    • Para modificar uma propriedade, edite o campo Valor da propriedade.
    • Para excluir uma propriedade, marque a caixa de seleção à esquerda do campo Nome da propriedade a ser excluída. Clique no botão Excluir propriedades.
  8. Clique em Salvar.
