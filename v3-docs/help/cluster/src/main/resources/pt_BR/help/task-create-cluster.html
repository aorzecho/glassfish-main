Para criar um cluster

Um cluster é um conjunto nomeado de instâncias do Servidor GlassFish que compartilham os mesmos aplicativos, recursos e informações de configuração.

Um cluster proporciona alta disponibilidade através da escalabilidade, balanceamento de carga e proteção a falhas. Quando uma instância ou um host de um cluster falha, o Servidor GlassFish detecta a falha, redireciona as solicitações da instância falha a outras instâncias do cluster e recupera o estado da sessão do usuário. Como os mesmos aplicativos e recursos estão em todas as instâncias do cluster, uma instância pode falhar em qualquer outra instância do cluster.

O Servidor GlassFish permite administrar todas as instâncias de um cluster como uma única unidade de um único host, independentemente se as instâncias residem no mesmo host ou em hosts diferentes.

Antes de começar

Se tiver a intenção de adicionar instâncias do Servidor GlassFish ao cluster que estiver criando, certifique-se de que os nós nos quais as instâncias devem residir já existem.

  1. Na árvore de navegação, selecione o nó Clusters.

    A página Clusters é exibida.

  2. Na página Clusters, clique em Novo.

    A página Novo cluster é exibida.

  3. No campo Nome do cluster, digite um nome para o cluster.

    O nome deve cumprir os seguintes requisitos:

    • O nome deve começar com uma letra, um número ou um sublinhado.

    • O nome pode conter somente os seguintes caracteres:

      • Letras minúsculas

      • Letras maiúsculas

      • Números

      • Hífen

      • Período

      • Sublinhado

    • O nome deve ser único no domínio e não deve ser o nome de outro cluster, uma configuração nomeada, uma instância do Servidor GlassFish ou um nó.

    • O nome não deve ser domain, server, nem nenhuma outra palavra-chave que esteja reservada pelo Servidor GlassFish.

  4. Na lista suspensa Configuração, selecione a configuração nomeada que o cluster utilizará.

    Observação - Se selecionar default-config, deve especificar na etapa seguinte que a configuração não é compartilhada. A configuração default-config não pode ser compartilhada.

  5. Especifique se a configuração que o cluster utilizará é compartilhada.
    • Se a configuração não for compartilhada, deixe marcada a opção Fazer uma cópia da configuração selecionada.

      A configuração selecionada será copiada quando o cluster for criado. À cópia da configuração é atribuído o nome nome-do-cluster-config, em que nome-do-cluster corresponde ao nome do cluster que está sendo criado.

    • Se a configuração for compartilhada, selecione a opção Referenciar a configuração selecionada.

      O cluster utilizará a configuração nomeada existente especificada.

  6. (Opcional) Adicione as instâncias do Servidor GlassFish ao cluster.

    Observação - Para criar uma instância em um nó de tipo CONFIG que represente um host remoto, você deve utilizar o subcomando create-local-instance(1). É possível utilizar o Console de Administração para este propósito.

    Também é possível adicionar instâncias depois que o cluster for criado.

    Adicione cada instância da seguinte forma:

    1. Na área Instâncias do servidor a serem criadas, clique em Adicionar.

      Uma linha é adicionada à tabela de instâncias do cluster.

    2. No campo Nome da instância, digite um nome para a instância.
    3. No campo Peso, digite um inteiro que represente o valor de balanceamento de carga da instância.

      O peso do balanceamento de carga determina a proporção de todas as solicitações do cluster que a instância deve processar.

      Por exemplo, em um cluster com duas instâncias, é possível processar uma das quatro solicitações em uma instância e processar três das quatro solicitações na outra instância. Nesse caso, defina como 1 o peso da instância que processa uma das quatro solicitações e como 3 o peso da outra instância. Se preferir utilizar porcentagem, defina os pesos das instâncias como 25 e 75, respectivamente.

      O peso padrão é 100.

    4. Na lista suspensa Nó, selecione o nó no qual a instância deve residir.
  7. Clique em OK.

    Você volta à página Clusters. O cluster é adicionado à tabela de clusters na página.
