Informações Gerais

A página Informações gerais contém as seguintes opções.

Iniciar cluster

Botão para iniciar o cluster. Se todas as instâncias do cluster estiverem em execução, este botão está desativado.

Interromper cluster

Botão para interromper o cluster. Se todas as instâncias do cluster estiverem interrompidas, este botão está desativado.

Migrar Timers EJB

Botão para migrar os timers da tecnologia Enterprise JavaBeans (EJB) de uma instância interrompida do GlassFish Server para uma instância em execução no cluster. Clicar neste botão abre a página Migrar Timers de EJB. Se todas as instâncias do cluster estiverem em execução, este botão está desativado.

Efetuar rotação de log

Botão para efetuar rotação de arquivo log no cluster. Quando o arquivo de log for rotacionado, ele será renomeado com um nome do timestamp no formato server.log_date-and-time e um arquivo de log vazio será criado para novas mensagens de log. As alterações são aplicadas dinamicamente. Não é necessário reiniciar o servidor.

Nome do cluster

Nome designado ao cluster quando o cluster é criado. Este campo é somente para leitura.

JVM

Se você clicar em JVM Report, será aberta uma janela separada que exibe relatórios sobre o Java Virtual Machine, incluindo um relatório de resumo, um relatório de gerenciamento de memória e de coleta de erros,um relatório de carregamento de classe e um dump de threads atuais.

Configuração

Nome selecionado para o cluster quando o cluster foi criado. Ao clicar na configuração, a página Configurações do cluster se abre. Este campo é somente para leitura.

GMS

Se esta opção estiver ativada, o serviço de gerenciamento de grupos (GMS) será iniciado em cada instância do GlassFish Server do cluster e do servidor de administração de domínios (DAS). O DAS participa de cada cluster no qual o GMS estiver ativado. O GMS oferece monitoramento de cluster, associação de cluster e serviços de comunicação do grupo. Por default, a opção está ativada.

Porta multicast

Número da porta de comunicação na qual o GMS faz listening dos eventos do grupo. Esta opção deverá especificar um número válido de porta na faixa entre 2048-32000. O default é um valor gerado automaticamente dentro dessa faixa.

Endereço do multicast

Endereço no qual o GMS faz listening dos eventos do grupo. Esta opção deve especificar um endereço de multicast na faixa de 224.0.0.0 a 239.255.255.255. O default é 228.9.XX.YY, em que XX e YY são valores independentes gerados automaticamente entre 0 e 255.

Endereço da interface de vinculação

Endereço do protocolo de Internet (IP) da interface de rede ao qual o GMS se vincula. Esta opção deve especificar o endereço IP de uma interface de rede local. O default corresponde a todos os endereços públicos de interface de rede.

Em uma máquina com vários endereços (multihome), esta opção configura a interface de rede usada para o GMS. A máquina com vários endereços (multihome) possui duas ou mais interfaces de rede.

Para especificar um endereço que seja válido para todas as instâncias do GlassFish Server do cluster, utilize uma propriedade do sistema para definir o endereço de forma individual de cada instância.

Por exemplo, use a página Propriedades do Sistema do Cluster para criar a propriedade do sistema GMS-BIND-INTERFACE-ADDRESS-cluster-name. Defina a opção Endereço de Bind nesta página para ${GMS-BIND-INTERFACE-ADDRESS-cluster-name} para especificar a propriedade do sistema. Por fim, para cada instância no cluster, defina a propriedade do sistema GMS-BIND-INTERFACE-ADDRESS-cluster-name para o endereço da interface solicitada na máquina da instância.

Status

Indicação do número de instâncias do cluster que estão e não estão em execução. Este campo é somente para leitura.
