Informações gerais

A página Informações gerais contém as seguintes opções.

Iniciar cluster

Botão para iniciar o cluster. Se todas as instâncias do cluster estiverem em execução, este botão está desativado.

Interromper cluster

Botão para interromper o cluster. Se todas as instâncias do cluster estiverem interrompidas, este botão está desativado.

Migrar temporizadores EJB

Botão para migrar os temporizadores da tecnologia Enterprise JavaBeans (EJB) de uma instância interrompida do Servidor GlassFish para uma instância em execução no cluster. Ao clicar neste botão, a página Migrar temporizadores EJB se abre. Se todas as instâncias do cluster estiverem em execução, este botão está desativado.

Efetuar rotação de log

Botão para efetuar rotação de arquivo log no cluster. Quando o arquivo de log sofre rotação, o arquivo é renomeado com um nome com carimbo de data e hora no formato server.log_ data-e-hora, e um arquivo de log vazio é criado para novas mensagens de log. As alterações são aplicadas dinamicamente. Não é necessário reiniciar o servidor.

Nome do cluster

Nome atribuído ao cluster quando o cluster é criado. Este campo é somente leitura.

JVM

Se você clicar em JVM Report, será aberta uma janela separada que exibe relatórios sobre o Java Virtual Machine, incluindo um relatório de resumo, um relatório de gerenciamento de memória e de coleta de erros,um relatório de carregamento de classe e um despejo de segmentos atual.

Configuração

Nome selecionado para o cluster quando o cluster foi criado. Ao clicar na configuração, a página Configurações do cluster se abre. Este campo é somente leitura.

GMS

Se esta opção estiver ativada, o serviço de gerenciamento de grupos (GMS) será iniciado em cada instância do Servidor GlassFish do cluster e do servidor de administração do domínio (DAS). O DAS participa de cada cluster no qual o GMS estiver ativado. O GMS oferece monitoramento de cluster, associação de cluster e serviços de comunicação do grupo. Por padrão, a opção está ativa.

Porta multicast

Número da porta de comunicação na qual o GMS escuta os eventos do grupo. Esta opção deve especificar um número de porta válido no intervalo de 2048 a 32000. O padrão é um valor gerado automaticamente dentro deste intervalo.

Endereço do multicast

Endereço no qual o GMS escuta os eventos do grupo. Esta opção deve especificar um endereço de multicast no intervalo de 224.0.0.0 a 239.255.255.255. O padrão é 228.9.XX.YY, em que XX e YY são valores independentes gerados automaticamente entre 0 e 255.

Endereço da interface de vinculação

Endereço do protocolo de Internet (IP) da interface de rede ao qual o GMS se vincula. Esta opção deve especificar o endereço IP de uma interface de rede local. O padrão corresponde a todos os endereços públicos de interface de rede.

Em uma máquina com vários endereços (multihome), esta opção configura a interface de rede usada para o GMS. A máquina com vários endereços (multihome) possui duas ou mais interfaces de rede.

Para especificar um endereço que seja válido para todas as instâncias do Servidor GlassFish do cluster, utilize uma propriedade do sistema para definir o endereço de forma individual de cada instância.

Por exemplo, utilize a página Propriedades do sistema do cluster para criar a propriedade do sistema GMS-BIND-INTERFACE-ADDRESS-nome-do-cluster. A seguir, defina a opção Endereço de vinculação nesta página como ${GMS-BIND-INTERFACE-ADDRESS-nome-do-cluster} para especificar a propriedade do sistema. Finalmente, defina em cada instância do cluster a propriedade do sistema GMS-BIND-INTERFACE-ADDRESS-nome-do-cluster como o endereço da interface de rede na máquina da instância.

Status

Indicação do número de instâncias do cluster que estão e não estão em execução. Este campo é somente leitura.
