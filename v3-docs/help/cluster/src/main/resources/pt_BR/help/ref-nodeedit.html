Editar nó

Utilize a página Editar nó para atualizar os dados de configuração de um nó do Servidor GlassFish. Também é possível utilizar esta página para alterar o tipo de um nó de CONFIG para SSH para ativar a comunicação SSH do nó.

Se o tipo de nó for alterado de CONFIG para SSH , os valores padrão são aplicados se nenhum dos campos a seguir forem deixados em branco:

  • Porta SSH

  • Nome de usuário:

  • Arquivo principal

A página Editar nó contém as seguintes opções:

Nome

Nome atribuído ao nó quando o nó foi criado. Este campo é somente leitura.

Tipo

Lista suspensa para especificar o tipo do nó, que são os seguintes:

SSH

O nó oferece suporte a comunicações com shell seguro (SSH).

CONFIG

O nó não oferece suporte à comunicação remoto. Se CONFIG estiver selecionado, nenhuma opção é exibida para a configuração de SSH. Se o tipo atual do nó for SSH e você selecionar CONFIG, os dados de configuração de SSH são removidos da configuração do nó quando as alterações forem salvas.

Host do nó

Nome do host que o nó representará depois que o nó for atualizado. Se este campo é obrigatório ou não, depende do tipo de nó:

  • Se o tipo de nó for CONFIG, o nome do host pode ser omitido. O nome do host pode ser determinado quando as instâncias que residem no nó forem criadas.

  • Se o tipo de nó for SSH, o nome do host deve ser especificado. Caso contrário, ocorre um erro.

Diretório do nó

O caminho para o diretório que contém as instâncias Servidor GlassFish que são criadas no nó. Se um caminho relativo for especificado, o caminho é relativo ao diretório as-install, no qual as-install corresponde ao diretório de instalação de base do software Servidor GlassFish no host.

Diretório de instalação

O caminho inteiro para o pai do diretório de instalação de base do software Servidor GlassFish no host, por exemplo, /export/glassfish3/.

Forçar

Se a opção Ativo estiver selecionada, o nó é atualizado na configuração do servidor de administração do domínio (DAS) mesmo se a validação dos parâmetros do nó falhar. Para validar os parâmetros de um nó, o DAS deve ser capaz de contactar o host do nó por meio de SSH.

Porta SSH

Porta para utilizar nas conexões SSH a este host do nó.

Se o tipo de nó for alterado de CONFIG para SSH e este campo for deixado vazio, a opção Porta SSH é definida como 22.

Se a opção Host do nó estiver definida como localhost-domínio, a opção Porta SSH é ignorada.

Nome de usuário SSH

Usuário deste host do nó que deve executar o processo para conexão com o host através de SSH.

Se o tipo de nó for alterado de CONFIG para SSH e este campo for deixado vazio, a opção Nome de usuário é definida como o usuário que estiver em execução no processo DAS.

Se a opção Host do nó estiver definida como localhost-domínio, a opção Usuário SSH é ignorada.

Autenticação de usuário SSH

Lista suspensa para especificar como o usuário SSH é autenticado ao efetuar login neste host do nó. Os possíveis métodos de autenticação são os seguintes:

Arquivo principal

O usuário SSH é autenticado através de um arquivo de chave privada SSH. Se selecionar esta opção, especifique um caminho absoluto para o arquivo no campo Arquivo principal.

Senha

O usuário SSH é autenticado através de uma senha que é armazenada na configuração do DAS.

Cuidado - Esta senha é armazenada em forma de texto na configuração do DAS. Para maior segurança, utilize um arquivo de chave ou um alias de senha.

Se selecionar esta opção, os campos Senha SSH e Confirmar senha SSH são ativados para permitir que você forneça a senha.

Alias de senha

O usuário SSH é autenticado através de um alias de senha. Se selecionar esta opção, selecione o alias de senha na lista suspensa Alias de senha.

Senha SSH

A senha que o usuário SSH utilizará ao efetuar login neste host do nó.

Confirmar senha SSH

Confirmação da senha que o usuário SSH utilizará ao efetuar login neste host do nó. O conteúdo deste campo deve corresponder exatamente ao conteúdo do campo Senha SSH.

Alias de senha

Lista suspensa de aliases de senha definidos para este domínio.

Arquivo principal

Caminho absoluto para o arquivo da chave privada de SSH para o usuário que a opção Nome de usuário especifica. Este arquivo é utilizado para autenticação do daemon sshd no host do nó.

Observação - O Servidor GlassFish também oferece suporte à autenticação com senha através da entrada AS_ADMIN_SSHPASSWORD no arquivo de senha. O arquivo de senha é especificado na opção --paswordfile do utilitário asadmin(1M).

Se o arquivo de chave privada de SSH estiver protegido por uma frase-senha, o arquivo de senha deve conter a entrada AS_ADMIN_SSHKEYPASSPHRASE.

O caminho para o arquivo da chave deve ser acessível pelo DAS e o arquivo de chave poderá ser lido pelo DAS.

O padrão dependerá de se o tipo do nó foi alterado de CONFIG para SSH para ativar a comunicação SSH do nó:

Se o tipo de nó for alterado de CONFIG para SSH e este campo for deixado vazio, a opção Arquivo principal é definida como um arquivo de chave no diretório .ssh do usuário. Se forem encontrados vários arquivos de chave, o Servidor GlassFish usa a seguinte ordem de preferência:

  1. id_rsa

  2. id_dsa

  3. identity
