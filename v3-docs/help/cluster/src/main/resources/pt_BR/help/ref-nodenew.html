Novo nó

Utilize a página Novo nó para criar um nó do Servidor GlassFish. Um nó representa um host no qual o software Servidor GlassFish está instalado. Um nó deve estar presente em cada host na qual as instâncias Servidor GlassFish residem.

A página Novo nó contém as opções seguinte:

Nome

O nome do nó.

O nome deve cumprir os seguintes requisitos:

  • O nome deve começar com uma letra, um número ou um sublinhado.

  • O nome pode conter somente os seguintes caracteres:

    • Letras minúsculas

    • Letras maiúsculas

    • Números

    • Hífen

    • Período

    • Sublinhado

  • O nome deve ser único no domínio e não deve ser o nome de outro nó, um cluster, uma configuração nomeada ou uma instância do Servidor GlassFish.

  • O nome não deve ser domain, server, nem nenhuma outra palavra-chave que esteja reservada pelo Servidor GlassFish.

Tipo

Lista suspensa para especificar o tipo do nó, que são os seguintes:

SSH

O nó oferece suporte a comunicações com shell seguro (SSH).

CONFIG

O nó não oferece suporte à comunicação remoto. Se CONFIG estiver selecionado, nenhuma opção é exibida para a configuração de SSH e o campo Diretório de instalação é apagado.

Host do nó

O nome do host que o nó representa. Se este campo é obrigatório ou não, depende do tipo de nó:

  • Se o tipo de nó for CONFIG, o nome do host pode ser omitido. O nome do host pode ser determinado quando as instâncias que residem no nó forem criadas.

  • Se o tipo de nó for SSH, o nome do host deve ser especificado. Caso contrário, ocorre um erro.

Diretório do nó

O caminho para o diretório que contém as instâncias Servidor GlassFish que são criadas no nó. Se um caminho relativo for especificado, o caminho é relativo ao diretório as-install.

O padrão depende do tipo do nó:

  • Se o tipo do nó for CONFIG, nenhum diretório das instâncias é especificado para o nó na configuração do servidor de administração do domínio (DAS). O diretório do nó pode ser determinado quando as instâncias que residem no nó forem criadas.

  • Se o tipo do nó for SSH, o padrão é as-install/nodes, onde as-install é o diretório base de instalação do software Servidor GlassFish no host.

Diretório de instalação

O caminho inteiro para o pai do diretório de instalação de base do software Servidor GlassFish no host, por exemplo, /export/glassfish3.

O padrão depende do tipo do nó:

  • Se o tipo do nó for CONFIG, nenhum diretório de instalação é especificado para o nó na configuração do DAS. O diretório de instalação pode ser determinado quando as instâncias que residem no nó forem criadas.

  • Se o tipo do nó for SSH, o padrão é o pai do diretório base de instalação padrão do software Servidor GlassFish para o DAS. Esse padrão é útil apenas se Servidor GlassFish estiver instalado no mesmo local em todos os hosts.

Forçar

Se a opção Ativo estiver selecionada, o nó é criado na configuração DAS mesmo se a validação dos parâmetros do nó falhar. Para validar os parâmetros de um nó, o DAS deve ser capaz de contactar o host do nó por meio de SSH.

Porta SSH

Porta para utilizar nas conexões SSH a este host do nó. O padrão é 22. Se a opção Host do nó estiver definida como localhost-domínio, a opção Porta SSH é ignorada.

Nome de usuário SSH

Usuário deste host do nó que deve executar o processo para conexão com o host através de SSH. O padrão é o usuário que está executando o processo DAS. Para garantir que o DAS possa ler o arquivo de chave privada de SSH desse usuário, especifique o usuário que está executando o processo DAS. Se a opção Host do nó estiver definida como localhost-domínio, a opção Nome de usuário é ignorada.

Autenticação de usuário SSH

Lista suspensa para especificar como o usuário SSH é autenticado ao efetuar login neste host do nó. Os possíveis métodos de autenticação são os seguintes:

Arquivo principal

O usuário SSH é autenticado através de um arquivo de chave privada SSH. Se selecionar esta opção, especifique um caminho absoluto para o arquivo no campo Arquivo principal.

Senha

O usuário SSH é autenticado através de uma senha que é armazenada na configuração do DAS.

Cuidado - Esta senha é armazenada em forma de texto na configuração do DAS. Para maior segurança, utilize um arquivo de chave ou um alias de senha.

Se selecionar esta opção, os campos Senha SSH e Confirmar senha SSH são ativados para permitir que você forneça a senha.

Alias de senha

O usuário SSH é autenticado através de um alias de senha. Se selecionar esta opção, selecione o alias de senha na lista suspensa Alias de senha.

Senha SSH

A senha que o usuário SSH utilizará ao efetuar login neste host do nó.

Confirmar senha SSH

Confirmação da senha que o usuário SSH utilizará ao efetuar login neste host do nó. O conteúdo deste campo deve corresponder exatamente ao conteúdo do campo Senha SSH.

Alias de senha

Lista suspensa de aliases de senha definidos para este domínio.

Arquivo principal

Caminho absoluto para o arquivo da chave privada de SSH para o usuário que a opção Nome de usuário especifica. Este arquivo é utilizado para autenticação do daemon sshd no host do nó.

Observação - O Servidor GlassFish também oferece suporte à autenticação com senha através da entrada AS_ADMIN_SSHPASSWORD no arquivo de senha. O arquivo de senha é especificado na opção --paswordfile do utilitário asadmin(1M).

Se o arquivo de chave privada de SSH estiver protegido por uma frase-senha, o arquivo de senha deve conter a entrada AS_ADMIN_SSHKEYPASSPHRASE.

O caminho para o arquivo da chave deve ser acessível pelo DAS e o arquivo de chave poderá ser lido pelo DAS.

O padrão é um arquivo de chave no diretório .ssh do usuário. Se forem encontrados vários arquivos de chave, o Servidor GlassFish usa a seguinte ordem de preferência:

  1. id_rsa

  2. id_dsa

  3. identity
