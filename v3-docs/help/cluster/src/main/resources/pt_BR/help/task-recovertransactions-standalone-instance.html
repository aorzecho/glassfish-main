Para Recuperar Transações de uma Instância Standalone do GlassFish Server

  1. Na árvore de navegação, selecione o nó Instâncias standalone.

    A página Instância de servidor standalone é exibida.

  2. Selecione a instância para a qual é necessária a recuperação de transações.

    A página Informações gerais da instância se abre.

  3. Na página Informações gerais, clique em Recuperar transações.

    A página Recuperar transações se abre.

    O conteúdo da página depende se a instância está ou não em execução.

    • Se a instância estiver em execução, a página Recuperar transações contém somente o texto A recuperação é realizada pelo próprio servidor em execução e o botão OK.

    • Se a instância não estiver em execução, a página Recuperar transações contém opções adicionais.

  4. Na lista drop-down Servidor de destino, selecione outra instância que pode recuperar as transações na instância atual.

    Selecione, para uma instância standalone, a instância denominada server, que representa o servidor de administração de domínios (DAS).

  5. No campo Diretório do log de transação, digite o diretório em que os logs de transação são armazenados.

    O default é /logs/tx no diretório de domínio. Os logs de transação são mantidos no subdiretório tx do diretório especificado pela definição do Local do Log de Transação do Serviço de transação na configuração do cluster. O valor default do Local do log de transação é o diretório especificado pelo campo Raiz de log da página Atributos do Domínio, que está localizado na guia Avançado do nó do GlassFish Server.

  6. Clique em OK.
