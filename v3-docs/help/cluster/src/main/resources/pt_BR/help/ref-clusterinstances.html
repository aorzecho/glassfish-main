Instâncias do servidor em clusters

Utilize a página Instâncias do servidor em clusters para criar e gerenciar as instâncias do Servidor GlassFish no cluster atual.

O campo Nome do cluster é um campo de somente leitura que exibe o nome do cluster atual.

A página Instâncias do servidor em clusters exibe uma lista de instâncias que são membros do cluster atual. Para cada instância, são fornecidas as seguintes informações:

Nome

Nome atribuído à instância quando a instância foi criada. Ao clicar no nome da instância, a página Informações gerais se abre.

Peso do balanceador de cargas

Inteiro que representa o valor de balanceamento de carga da instância.

O peso do balanceamento de carga determina a proporção de todas as solicitações do cluster que a instância deve processar.

Por exemplo, em um cluster com duas instâncias, é possível processar uma das quatro solicitações em uma instância e processar três das quatro solicitações na outra instância. Nesse caso, defina como 1 o peso da instância que processa uma das quatro solicitações e como 3 o peso da outra instância. Se preferir utilizar porcentagem, defina os pesos das instâncias como 25 e 75, respectivamente.

O peso padrão é 100.

Configuração

Configuração nomeada que a instância faz referência. Ao clicar na configuração, a página Configurações da instância se abre.

Nó

O nome do nó no qual a instância reside. Ao clicar no nome do nó, a página Editar nó se abre.

Status

Indica se a instância está em execução, se requer reinicialização para que seja atualizada com as alterações na configuração ou se não está em execução. Este campo é somente leitura.

A tabela Instâncias do servidor também contém as seguintes opções.

Novo

Botão para adicionar uma instância ao cluster.

Excluir

Botão para remover uma ou mais instâncias selecionadas do cluster.

Cuidado - Se estiver utilizando um cluster Java Message Service (JMS) com um broker mestre, não exclua a instância que está associada ao broker mestre. Se esta instância deve ser excluída, utilize o subcomando change-master-broker(1) para atribuir o broker mestre a outra instância.

Iniciar

Botão para iniciar uma ou mais instâncias selecionadas no cluster.

Interromper

Botão para interromper uma ou mais instâncias selecionadas no cluster.
