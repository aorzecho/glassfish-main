Novo cluster

Utilize a página Novo cluster para criar um cluster.

A página Novo cluster contém as seguintes opções.

Nome do cluster

O nome do cluster.

O nome deve cumprir os seguintes requisitos:

  • O nome pode conter somente caracteres ASCII.

  • O nome deve começar com uma letra, um número ou um sublinhado.

  • O nome pode conter somente os seguintes caracteres:

    • Letras minúsculas

    • Letras maiúsculas

    • Números

    • Hífen

    • Período

    • Sublinhado

  • O nome deve ser único no domínio e não deve ser o nome de outro cluster, uma configuração nomeada, uma instância do Servidor GlassFish ou um nó.

  • O nome não deve ser domain, server, nem nenhuma outra palavra-chave que esteja reservada pelo Servidor GlassFish.

Configuração

Lista suspensa das configurações nomeadas existentes que o cluster pode utilizar. Um cluster precisa de uma configuração nomeada que defina a configuração de todas as instâncias adicionadas ao cluster. É necessário selecionar uma configuração na lista. O cluster utilizará a configuração selecionada quando o cluster for criado.

Criar cópia da configuração selecionada

Se esta opção for selecionada, a configuração selecionada será copiada quando o cluster for criado. À cópia da configuração é atribuído o nome nome-do-cluster-config, em que nome-do-cluster corresponde ao nome do cluster conforme digitado no campo Nome do cluster.

Se default-config estiver selecionado na lista suspensa Configuração, é necessário selecionar esta opção. A configuração default-config pode ser apenas copiada, não referenciada.

Referenciar a configuração selecionada

Se esta opção for selecionada, o cluster utilizará a configuração nomeada existente especificada.

Se default-config estiver selecionado na lista suspensa Configuração, esta opção não deve ser selecionada. A configuração default-config não pode ser referenciada, somente copiada.

Instâncias do servidor a serem criadas

Lista das instâncias do Servidor GlassFish que serão criadas quando o cluster for criado. Para cada instância, as informações a seguir são fornecidas:

Nome da instância

O nome da instância.

Peso

Inteiro que representa o valor de balanceamento de carga da instância.

O peso do balanceamento de carga determina a proporção de todas as solicitações do cluster que a instância deve processar.

Por exemplo, em um cluster com duas instâncias, é possível processar uma das quatro solicitações em uma instância e processar três das quatro solicitações na outra instância. Nesse caso, defina como 1 o peso da instância que processa uma das quatro solicitações e como 3 o peso da outra instância. Se preferir utilizar porcentagem, defina os pesos das instâncias como 25 e 75, respectivamente.

O peso padrão é 100.

Nó

Lista suspensa dos nós existentes nos quais a instância pode estar. É necessário selecionar um nó na lista. O instância residirá no nó selecionado quando o cluster for criado.

A tabela Instâncias do servidor a serem criadas também contém as seguintes opções.

Novo

Botão para criar uma instância. Se clicar neste botão, uma linha é adicionada à tabela Instâncias do servidor a serem criadas.

Excluir

Botão para excluir uma instância.
