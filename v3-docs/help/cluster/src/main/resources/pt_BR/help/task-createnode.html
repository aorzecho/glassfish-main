Para criar um nó

Um nó representa um host no qual o software Servidor GlassFish está instalado. Um nó deve estar presente em cada host na qual as instâncias Servidor GlassFish residem.

  1. Na árvore de navegação, selecione o nó Nós.

    A página Nós é exibida.

  2. Na página Nós, clique em Novo.

    A página Novo nó é exibida.

  3. No campo Nome, digite o nome desejado para o nó.
  4. Na lista suspensa Tipo, selecione o tipo do nó a ser criado.
    • Para criar um nó que ofereça suporte à comunicação através de shell seguro (SSH), selecione SSH.
    • Para criar um nó que não ofereça suporte à comunicação remota, selecione CONFIG.
  5. No campo Host do nó, digite o nome do host que o nó representa.

    Se este campo é obrigatório ou não, depende do tipo de nó:

    • Se o tipo de nó for CONFIG, o nome do host pode ser omitido. O nome do host pode ser determinado quando as instâncias que residem no nó forem criadas.

    • Se o tipo de nó for SSH, o nome do host deve ser especificado. Caso contrário, ocorre um erro.

  6. No campo Diretório do nó, digite o caminho para o diretório que contém as instâncias Servidor GlassFish que são criadas no nó.

    Se o tipo do nó for CONFIG, deixe este campo vazio. O diretório do nó pode ser determinado quando as instâncias que residem no nó forem criadas.

    Se for digitado um caminho relativo, o caminho é relativo ao diretório as-install, no qual as-install corresponde ao diretório de instalação de base do software Servidor GlassFish no host.

  7. No campo Diretório de instalação, digite o caminho completo para o pai do diretório de instalação de base do software Servidor GlassFish no host, por exemplo, /export/glassfish3/.

    Se o tipo do nó for CONFIG, deixe este campo vazio. O diretório de instalação pode ser determinado quando as instâncias que residem no nó forem criadas.

  8. Para forçar a criação do nó na configuração do servidor de administração do domínio (DAS) mesmo se a validação dos parâmetros do nó falhar, ative a opção Forçar.
  9. No campo Porta SSH, digite a porta para utilizar as conexões SSH com este host do nó.
  10. No campo Nome do usuário SSH, digite o nome do usuário deste host do nó que deve executar o processo para conexão com o host através de SSH.
  11. Especifique como o usuário SSH será autenticado ao efetuar login no host do nó.
    • Para especificar a autenticação através de um arquivo de chave privada SSH, realize estas etapas:
      1. Na lista suspensa Autenticação de usuário SSH, selecione Arquivo principal.
      2. No campo Arquivo principal, digite o caminho absoluto para o arquivo da chave privada de SSH para o usuário que a opção Nome de usuário SSH especifica.

        Observação - O Servidor GlassFish também oferece suporte à autenticação com senha através da entrada AS_ADMIN_SSHPASSWORD no arquivo de senha. O arquivo de senha é especificado na opção --paswordfile do utilitário asadmin(1M).

    • Para especificar a autenticação através de uma senha armazenada na configuração do DAS, realize estas etapas:

      Cuidado - Esta senha é armazenada em forma de texto na configuração do DAS. Para maior segurança, utilize um arquivo de chave ou um alias de senha.

      1. Na lista suspensa Autenticação de usuário SSH, selecione Senha.
      2. No campo Senha SSH, digite a senha que o usuário SSH utilizará ao efetuar login neste host do nó.
      3. No campo Confirmar senha SSH, digite novamente a senha.
    • Para especificar a autenticação através de um alias de senha, realize estas etapas:
      1. Na lista suspensa Autenticação de usuário SSH, selecione Alias de senha.
      2. Na lista suspensa Alias de senha, selecione o alias existente.
  12. Clique em Salvar.
