Para Finalizar a Atualização de um Nó SSH

Esta tarefa é a continuação de um nó SSH de Para Editar um Nó.

  1. Para forçar a atualização do nó na configuração do servidor de administração de domínios (DAS) mesmo que haja falha na validação dos parâmetros do nó, ative a opção Impor.

  2. No campo Porta SSH, digite a porta para utilizar as conexões SSH com este host do nó.

    Se o tipo de nó for alterado de CONFIG para SSH e este campo for deixado vazio, a opção Porta SSH é definida como 22.

  3. No campo Nome do usuário SSH, digite o nome do usuário deste host do nó que deve executar o processo para conexão com o host através de SSH.

    Se o tipo de nó for alterado de CONFIG para SSH e este campo for deixado vazio, a opção Nome de usuário SSH é definida como o usuário que estiver em execução no processo DAS.

  4. Especifique como o usuário SSH será autenticado ao efetuar log-in no host do nó.

    • Para especificar a autenticação através de um arquivo de chave privada SSH, realize estas etapas:

      1. Na lista drop-down Autenticação de Usuário SSH, selecione Arquivo Principal.

      2. Se você quiser que o GlassFish Server configure uma chave SSH no host do nó, ative a Opção Configurar SSH.

        O GlassFish Server configura a chave de SSH copiando arquivos-chave existentes do host DAS ou, se necessário, gerando os arquivos antes de copiá-los.

        O campo Chave Gerada e o campo Arquivo de Chaves Públicas são adicionados à janela.

      3. Se você quiser que o GlassFish Server gere os arquivos de chave SSH antes de copiá-los para o host do nó, ative a Opção Gerar Chave.

      4. No campo Senha do Usuário SSH, digite a senha que o usuário do SSH usará quando efetuar log-in no host deste nó.

      5. No campo Arquivo de Chaves Públicas, digite o caminho absoluto para o arquivo de chaves públicas SSH relativo ao usuário especificado pela opção Nome do Usuário SSH.

      6. No campo Arquivo de Chave, digite o caminho absoluto para o arquivo da chave privada de SSH para o usuário que a opção Nome de Usuário SSH especifica.

        Observação:

        GlassFish Server também suporta autenticação por senha por meio da entrada AS_ADMIN_SSHPASSWORD no arquivo de senhas. O arquivo de senhas é especificado na opção -paswordfiledo recurso asadmin(1).

        Se você alteraro tipo do nó para SSH e deixar este campo em branco, a opção Arquivo de Chaves será definida para um arquivo de chaves no diretório .ssh do usuário. Se forem encontrados vários arquivos de chave, o GlassFish Server usa a seguinte ordem de preferência:

        id_rsa
id_dsa
identity

    • Para especificar a autenticação através de uma senha armazenada na configuração do DAS, realize estas etapas:

      Cuidado:

      Esta senha é armazenada em forma de texto na configuração do DAS. Para maior segurança, utilize um arquivo de chave ou um alias de senha.

      1. Na lista drop-down Autenticação de usuário SSH, selecione Senha.

      2. No campo Senha do Usuário SSH, digite a senha que o usuário do SSH usará quando efetuar log-in no host deste nó.

      3. No campo Confirmar Senha, digite a senha novamente.

    • Para especificar a autenticação através de um alias de senha, realize estas etapas:

      1. Na lista drop-down Autenticação de usuário SSH, selecione Alias de senha.

      2. Na lista drop-down Alias de senha, selecione o alias existente.

  5. Clique em Salvar.
