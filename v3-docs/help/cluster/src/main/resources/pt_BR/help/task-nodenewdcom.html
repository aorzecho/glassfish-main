Para Finalizar a Criação de um Nó do DCOM

Esta tarefa é a continuação de um nó DCOM de Para Criar um Nó.

  1. Para copiar o software GlassFish Server do host DAS para o host do nó, ative a opão Instalar GlassFish Server.

  2. Para forçar a criação do nó na configuração do servidor de administração de domínios (DAS) mesmo se a validação dos parâmetros do nó falhar, ative a opção Forçar.

  3. Se você quiser que o GlassFish Server teste a conexão por meio do DCOM com o host do nó antes de criar o nó, execute as etapas a seguir:

    1. Ative a opção Validar DCOM.

    2. No campo Diretório de Teste Remoto, digite o nome da pasta no host remoto na qual o GlassFish Server deverá criar um arquivo para testar a conexão.

  4. No campo Domínio do Windows, digite o nome do domínio Windows que contém o usuário especificado pela opção Nome do Usuário do Windows.

  5. No campo Nome do Usuário do Windows, digite o nome do usuário no host deste nó que deverá executar o processo de conexão com o host por meio do DCOM.

  6. Especifique como o usuário do Windows será autenticado ao efetuar log-in no host deste nó.

    Observação:

    O GlassFish Server também suporta autenticação de senha por meio da entrada AS_ADMIN_WINDOWSPASSWORD no arquivo de senha. O arquivo de senhas é especificado na opção -paswordfiledo recurso asadmin(1).

    1. Para especificar a autenticação através de uma senha armazenada na configuração do DAS, realize estas etapas:

      Cuidado:

      Esta senha é armazenada em forma de texto na configuração do DAS. Para aumentar a segurança, use um apelido de senha

      1. Na lista drop-down Autenticação do Usuário Windows, selecione Senha.

      2. No campo Senha do Usuário do Windows, digite a senha que o usuário do SSH usará ao efetuar log-in no host deste nó.

      3. No campo Confirmar Senha, digite a senha novamente.

    2. Para especificar a autenticação através de um alias de senha, realize estas etapas:

      1. Na lista drop-down Autenticação do Usuário do Windows, selecione Apelido da Senha.

      2. Na lista drop-down Alias de senha, selecione o alias existente.

  7. Clique em Salvar.
