Para migrar temporizadores EJB

Se uma instância do Servidor GlassFish for interrompida, os temporizadores da tecnologia Enterprise JavaBeans (EJB) instalados em tal instância também interrompem a execução. Se a instância foi interrompida de forma anormal ou inesperada, para permitir que estes temporizadores funcionem enquanto a instância estiver interrompida, é necessário migrar os temporizadores para uma instância do servidor em execução no cluster.

  1. Na árvore de navegação, expanda o nó Clusters.
  2. No nó Clusters da árvore de navegação, selecione o cluster.

    A página Informações gerais do cluster se abre.

  3. Na página Informações gerais, clique Migrar temporizadores EJB.

    A página Migrar temporizadores EJB é exibida.

    Observação - Se todas as instâncias do cluster estiverem em execução, o botão Migrar temporizadores EJB está desativado.

  4. Na página Migrar temporizadores EJB, selecione a instância da qual e para a qual migrar os temporizadores.
    1. Na lista suspensa Origem, selecione a instância interrompida da qual migrar os temporizadores.
    2. (Opcional) Na lista suspensa Destino, selecione a instância em execução para a qual migrar os temporizadores.
    3. Clique em OK.
  5. Interrompa e reinicie a instância para a qual migrou os temporizadores EJB.

    Se a instância de origem estiver em execução ou se a instância de destino não estiver em execução, o Console de Administração exibe uma mensagem de erro.
