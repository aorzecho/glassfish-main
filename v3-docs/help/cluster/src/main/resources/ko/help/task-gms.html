그룹 관리 서비스를 구성하는 방법

그룹 관리 서비스(Group Management Service, GMS)는 클러스터 모니터링, 클러스터 멤버쉽 및 그룹 통신 서비스를 제공하는 in-process 서비스입니다. GMS는 다음 기능을 수행합니다.

  • 하나 이상의 멤버 인스턴스가 클러스터에 추가되거나 클러스터에서 제거되거나 장애가 발생한 것으로 의심되거나 확인되면 GlassFish Server 인스턴스의 등록된 모듈에 알립니다.

  • 프로세스 그룹 간에 메시지를 주고 받을 수 있는 기능을 제공합니다.

  1. 탐색 트리에서 [구성] 노드를 확장합니다.

  2. 변경할 클러스터에 대한 구성을 선택합니다.

  3. 구성 노드 아래에서 그룹 관리 서비스 노드를 선택합니다.

    그룹 관리 서비스 페이지가 열립니다.

  4. 누락된 최대 하트비트 수 필드에 GMS가 장애 발생이 의심된다고 확인하기 전에 수행되는 최대 모니터링 시도 횟수를 입력합니다.

    기본값은 3입니다.

  5. 하트비트 빈도 필드에 모니터링 시도 간의 시간을 입력합니다.

    기본값은 2000밀리초입니다.

  6. [그룹 검색 시간 초과] 필드에 GMS가 다른 클러스터 멤버를 검색할 때까지 대기하는 시간을 입력합니다.

    기본값은 5000밀리초입니다.

  7. 장애 발생 확인 대기 시간 필드에 의심되는 장애 발생이 확인됨으로 표시된 후의 시간을 입력합니다.

    기본값은 1500밀리초입니다.

  8. 기본값을 복원하려면 [기본값 로드] 단추를 누릅니다.

  9. [추가 속성] 섹션에서 추가 속성을 지정합니다.

    • 속성을 추가하려면 [속성 추가] 단추를 누릅니다. 빈 행이 표시되면 [이름] 필드에 속성 이름을 입력하고 [값] 필드에 속성값을 입력합니다.

    • 속성을 수정하려면 속성의 [값] 필드를 편집합니다.

    • 속성을 삭제하려면 삭제할 속성의 [이름] 필드 왼쪽에 있는 체크 박스를 선택한 다음 [속성 삭제] 단추를 누릅니다.

  10. [저장]을 누릅니다.
