노드를 만드는 방법

노드는 GlassFish 서버 소프트웨어가 설치되어 있는 호스트를 나타냅니다. 노드는 GlassFish 서버 인스턴스가 있는 모든 호스트에 대해 있어야 합니다.

  1. 탐색 트리에서 노드 노드를 선택합니다.

    노드 페이지가 열립니다.

  2. 노드 페이지에서 새로 만들기를 누릅니다.

    새 노드 페이지가 열립니다.

  3. 이름 필드에 원하는 노드 이름을 입력합니다.
  4. 유형 드롭다운 목록에서 만들 노드 유형을 선택합니다.
    • 보안 쉘(secure shell, SSH)을 통한 통신을 지원하는 노드를 만들려면 SSH를 선택합니다.
    • 원격 통신을 지원하지 않는 노드를 만들려면 CONFIG를 선택합니다.
  5. 노드 호스트 필드에 노드가 나타내는 호스트의 이름을 입력합니다.

    이 필드가 필수인지의 여부는 노드 유형에 따라 결정됩니다.

    • 노드 유형이 CONFIG이면 호스트 이름을 생략할 수 있습니다. 호스트 이름은 노드에 있는 인스턴스를 만들 때 지정할 수 있습니다.

    • 노드 유형이 SSH이면 호스트 이름을 지정해야 합니다. 그렇지 않으면 오류가 발생합니다.

  6. 노드 디렉토리 필드에 노드에 만들어져 있는 GlassFish 서버 인스턴스를 포함할 디렉토리의 경로를 입력합니다.

    노드 유형이 CONFIG이면 이 필드를 비워 둡니다. 노드 디렉토리는 노드에 있는 인스턴스를 만들 때 지정할 수 있습니다.

    상대 경로를 입력하는 경우 경로가 as-install 디렉토리를 기준으로 합니다. 여기서 as-install은(는) 호스트의 GlassFish 서버 소프트웨어의 기본 설치 디렉토리입니다.

  7. 설치 디렉토리 필드에 호스트에 있는 GlassFish 서버 소프트웨어의 상위 기본 설치 디렉토리의 전체 경로를 입력합니다(예: /export/glassfish3/).

    노드 유형이 CONFIG이면 이 필드를 비워 둡니다. 설치 디렉토리는 노드에 있는 인스턴스를 만들 때 지정할 수 있습니다.

  8. 노드의 매개 변수 검증이 실패하더라도 DAS(도메인 관리 서버)의 구성에 노드가 만들어지도록 강제 적용하려면 강제 적용 옵션을 활성화합니다.
  9. SSH 포트 필드에 이 노드의 호스트에 대한 SSH 연결에 사용할 포트를 입력합니다.
  10. SSH 사용자 이름 필드에 SSH를 통해 호스트에 연결하기 위한 프로세스를 실행할 이 노드의 호스트 사용자 이름을 입력합니다.
  11. 이 노드의 호스트에 로그인할 때 SSH 사용자를 인증하는 방식을 지정합니다.
    • SSH 개인 키 파일을 통해 인증을 지정하려면 다음 단계를 수행합니다.
      1. SSH 사용자 인증 드롭다운 목록에서 키 파일을 선택합니다.
      2. 키 파일 필드에 SSH 사용자 이름 옵션에서 지정하는 사용자에 대한 SSH 개인 키 파일의 절대 경로를 입력합니다.

        주 - 또한 GlassFish 서버는 암호 파일의 AS_ADMIN_SSHPASSWORD 항목을 통한 암호 인증을 지원합니다. 암호 파일은 asadmin(1M) 유틸리티의 --paswordfile 옵션에 지정되어 있습니다.

    • DAS 구성에 저장된 암호를 통해 인증을 지정하려면 다음 단계를 수행합니다.

      주의 - 이 암호는 DAS 구성에 일반 텍스트로 저장됩니다. 보안 강화를 위해 키 파일이나 암호 별칭을 사용합니다.

      1. SSH 사용자 인증 드롭다운 목록에서 암호를 선택합니다.
      2. SSH 암호 필드에 SSH 사용자가 이 노드의 호스트에 로그인할 때 사용하는 암호를 입력합니다.
      3. SSH 암호 확인 필드에 암호를 다시 입력합니다.
    • 암호 별칭을 통해 인증을 지정하려면 다음 단계를 수행합니다.
      1. SSH 사용자 인증 드롭다운 목록에서 암호 별칭을 선택합니다.
      2. 암호 별칭 드롭다운 목록에서 기존 별칭을 선택합니다.
  12. 저장을 누릅니다.
