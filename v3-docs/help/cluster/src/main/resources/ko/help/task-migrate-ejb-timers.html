EJB 타이머를 마이그레이션하는 방법

GlassFish Server 인스턴스가 정지된 경우 해당 인스턴스에 설치된 EJB(Enterprise JavaBeans) 기술 타이머도 실행이 정지됩니다. 인스턴스가 비정상적으로 또는 예기치 않게 정지된 경우 인스턴스가 정지된 동안 이러한 타이머를 사용으로 설정하려면 타이머를 클러스터의 실행되고 있는 서버 인스턴스로 마이그레이션해야 합니다.

  1. 탐색 트리에서 [클러스터] 노드를 확장합니다.

  2. 탐색 트리의 클러스터 노드 아래에서 클러스터를 선택합니다.

    클러스터에 대한 [일반 정보] 페이지가 열립니다.

  3. [일반 정보] 페이지에서 EJB 타이머 마이그레이션을 누릅니다.

    EJB 타이머 마이그레이션 페이지가 열립니다.

    주:

    클러스터의 모든 인스턴스가 실행되고 있는 경우 EJB 타이머 마이그레이션 단추가 비활성화됩니다

    타이머 마이그레이션을 지원하기 위해 EJB 타이머 마이그레이션 페이지는 다음 정보를 포함하는 EJB 타이머 테이블로 구성됩니다.

    서버 이름

    클러스터에 있는 GlassFish Server 인스턴스의 목록입니다.

    타이머 수

    각 인스턴스에 있는 영구 타이머의 수입니다.

  4. EJB 타이머 마이그레이션 페이지에서 타이머를 마이그레이션할 소스 인스턴스 및 대상 인스턴스를 선택합니다.

    1. 소스 드롭다운 목록에서 타이머를 마이그레이션할 정지된 인스턴스를 선택합니다.

    2. 대상 드롭다운 목록에서 타이머를 마이그레이션할 실행되고 있는 인스턴스를 선택합니다.

    3. [확인]을 누릅니다.

  5. EJB 타이머를 마이그레이션한 인스턴스를 정지하고 재시작합니다.

    소스 인스턴스가 실행되고 있거나 대상 인스턴스가 실행되고 있지 않으면 관리 콘솔에 오류 메시지가 표시됩니다.
