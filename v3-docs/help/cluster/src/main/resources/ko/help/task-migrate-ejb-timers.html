EJB 타이머를 마이그레이션하는 방법

GlassFish 서버 인스턴스가 중지된 경우 해당 인스턴스에 설치된 EJB(Enterprise JavaBeans) 기술 타이머도 실행이 중지됩니다. 인스턴스가 비정상적으로 또는 예기치 않게 중지된 경우 인스턴스가 중지된 동안 이러한 타이머를 활성화하려면 타이머를 클러스터의 실행되고 있는 서버 인스턴스로 마이그레이션해야 합니다.

  1. 탐색 트리에서 클러스터 노드를 확장합니다.
  2. 탐색 트리의 클러스터 노드 아래에서 클러스터를 선택합니다.

    클러스터에 대한 일반 정보 페이지가 열립니다.

  3. 일반 정보 페이지에서 EJB 타이머 마이그레이션을 누릅니다.

    EJB 타이머 마이그레이션 페이지가 열립니다.

    주 - 클러스터의 모든 인스턴스가 실행되고 있는 경우 EJB 타이머 마이그레이션 버튼이 비활성화됩니다.

  4. EJB 타이머 마이그레이션 페이지에서 타이머를 마이그레이션할 소스 인스턴스 및 대상 인스턴스를 선택합니다.
    1. 소스 드롭다운 목록에서 타이머를 마이그레이션할 중지된 인스턴스를 선택합니다.
    2. (옵션) 대상 드롭다운 목록에서 타이머를 마이그레이션할 실행되고 있는 인스턴스를 선택합니다.
    3. 확인을 누릅니다.
  5. EJB 타이머를 마이그레이션한 인스턴스를 중지하고 다시 시작합니다.

    소스 인스턴스가 실행되고 있거나 대상 인스턴스가 실행되고 있지 않으면 관리 콘솔에 오류 메시지가 표시됩니다.
