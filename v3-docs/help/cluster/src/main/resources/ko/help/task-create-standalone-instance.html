독립 실행형 GlassFish Server 인스턴스를 만들려면

GlassFish Server 인스턴스는 응용 프로그램 배포의 기본을 구성합니다. 독립 실행형 인스턴스는 클러스터의 구성원이 아닙니다. 각 인스턴스는 단일 도메인에 속하며 자체 디렉토리 구조, 구성 및 배포된 응용 프로그램을 가집니다. 또한 각 인스턴스에는 Java EE(Java Platform, Enterprise Edition) 웹 컨테이너 및 EJB(Enterprise JavaBeans) 기술 컨테이너가 포함됩니다. 모든 새 인스턴스에는 인스턴스가 있게 될 호스트를 정의하는 노드에 대한 참조가 포함되어야 합니다.

주 - 원격 호스트를 나타내는 CONFIG 유형의 노드에 인스턴스를 만들려면 create-local-instance(1) 하위 명령을 사용해야 합니다. 이 용도로는 관리 콘솔을 사용할 수 없습니다.

시작하기 전에

인스턴스가 상주하게 될 노드가 있는지 확인합니다.

  1. 탐색 트리에서 독립 실행형 인스턴스 노드를 선택합니다.

    독립 실행형 서버 인스턴스 페이지가 열립니다.

  2. 독립 실행형 서버 인스턴스 페이지에서 새로 만들기를 누릅니다.

    새 독립 실행형 서버 인스턴스 페이지가 열립니다.

  3. 이름 필드에 인스턴스의 이름을 입력합니다.

    이름은 다음 요구 사항을 충족해야 합니다.

    • 이름에는 ASCII 문자만 포함될 수 있습니다

    • 이름은 문자, 숫자 또는 밑줄로 시작해야 합니다.

    • 이름에는 다음 문자만 포함할 수 있습니다.

      • 소문자

      • 대문자

      • 숫자

      • 하이픈

      • 마침표

      • 밑줄

    • 이름은 도메인에서 고유해야 하며 다른 GlassFish Server 인스턴스, 클러스터, 명명된 구성 또는 노드의 이름이어서는 안 됩니다.

    • 이름은 domain, server 또는 GlassFish Server에서 예약된 다른 모든 키워드여서는 안 됩니다.

  4. 노드 드롭다운 목록에서 인스턴스가 상주하게 될 노드를 선택합니다.
  5. 구성 드롭다운 목록에서 인스턴스가 사용할 명명된 구성을 선택합니다.

    주 - default-config를 선택하는 경우 다음 단계에서 구성이 공유되지 않도록 지정해야 합니다. default-config 구성은 공유할 수 없습니다.

  6. 인스턴스가 사용할 구성이 공유되는지를 지정합니다.
    • 구성이 공유되지 않는 경우 선택한 구성의 복사본을 만듭니다. 옵션을 선택한 채로 둡니다.

      인스턴스를 만들 때 선택한 구성이 복사됩니다. 구성의 복사본에는 instance-name-config라는 이름이 지정됩니다. 여기서 instance-name은 만들려는 인스턴스의 이름입니다.

    • 구성이 공유되는 경우 선택한 구성을 참조합니다. 옵션을 선택합니다.

      인스턴스가 지정된 기존 명명된 구성을 사용합니다.

  7. 확인을 누릅니다.
