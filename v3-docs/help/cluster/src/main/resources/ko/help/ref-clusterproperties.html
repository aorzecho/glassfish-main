클러스터 등록 정보

클러스터 등록 정보 페이지에는 선택된 클러스터에 대해 설정된 등록 정보 목록이 표시됩니다. 이러한 등록 정보는 클러스터에 대한 선택적 구성 정보를 추가합니다.

클러스터 이름 필드는 선택한 클러스터의 이름을 표시하는 읽기 전용 필드입니다.

각 등록 정보에 대해 표시되는 정보는 다음과 같습니다.

이름

등록 정보의 이름입니다.

값

선택된 인스턴스에 대해 설정된 등록 정보의 값입니다.

설명

등록 정보에 대한 자세한 정보를 제공하는 텍스트 설명입니다.

또한 추가 등록 정보 테이블에는 다음 옵션이 포함되어 있습니다.

등록 정보 추가

등록 정보를 추가하는 버튼입니다. 이 버튼을 누르면 추가 등록 정보 테이블에 행이 추가됩니다.

등록 정보 삭제

하나 이상의 선택된 등록 정보를 삭제하는 버튼입니다. 삭제된 등록 정보는 해당 기본값으로 되돌려지거나 기본값이 설정되어 있지 않은 경우 정의되지 않습니다.

GlassFish 서버은(는) 다음 클러스터 등록 정보를 정의합니다.

GMS_LISTENER_PORT

클러스터가 그룹 관리 서비스(Group Management Service, GMS)의 메시지를 수신하는 포트의 포트 번호입니다. 기본값은 대부분의 경우에 충분해야 합니다.

GMS_LOOPBACK

인스턴스가 해당 인스턴스가 클러스터에 브로드캐스트하는 자체 응용 프로그램 수준 메시지를 수신할 수 있는지를 지정합니다.

가능한 값은 다음과 같습니다.

false

인스턴스가 자체 메시지를 수신할 수 없습니다(기본값).

true

인스턴스가 자체 메시지를 수신할 수 있습니다. 인스턴스가 클러스터의 유일한 인스턴스일 경우 인스턴스 테스트에 이 설정을 사용합니다.

GMS_MULTICAST_TIME_TO_LIVE

메시지가 무시되기 전에 다음 이벤트 유형에 대한 멀티캐스트 메시지에 발생할 수 있는 최대 반복 또는 전송 횟수입니다.

  • 그룹 검색

  • 구성원 하트비트

  • 구성원 자격 변경

DAS 및 클러스터링된 인스턴스가 배포된 네트워크의 구성을 일치시키려면 이 값을 가능한 한 낮게 설정합니다. 시스템에 대한 가능한 최저값을 확인하려면 validate-multicast(1) 하위 명령을 사용합니다.

0 값은 멀티캐스트 메시지가 해당 멀티캐스트 메시지가 브로드캐스트되는 호스트를 절대 떠나지 않도록 합니다.

1 값은 스위치 또는 라우터로 연결된 동일한 서브넷에 있는 호스트 간에 메시지가 브로드캐스트되지 않도록 막을 수 있습니다.

기본값 4는 호스트가 스위치 또는 라우터로 연결된 네트워크에 있는 모든 클러스터 구성원에 메시지가 성공적으로 브로드캐스트되도록 합니다.
