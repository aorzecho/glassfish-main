트랜잭션 복구

GlassFish 서버 인스턴스에 대한 트랜잭션을 복구하려면 트랜잭션 복구 페이지를 사용합니다.

인스턴스가 실행 중이면 트랜잭션 복구 페이지에 복구는 실행 중인 서버 자체에서 수행합니다.라는 텍스트와 확인 버튼만 표시됩니다.

인스턴스가 실행 중이 아니면 트랜잭션 복구 페이지에 다음 옵션이 표시됩니다.

인스턴스 이름

인스턴스가 만들어질 때 인스턴스에 지정된 이름입니다. 이 필드는 읽기 전용입니다.

대상 서버

현재 인스턴스에서 트랜잭션을 복구할 수 있는 다른 인스턴스입니다.

트랜잭션 로그 디렉토리

트랜잭션 로그를 저장할 디렉토리입니다. 기본값은 도메인 디렉토리 아래의 /logs/tx입니다.

트랜잭션 로그는 클러스터 구성에 대한 트랜잭션 서비스의 트랜잭션 로그 위치 설정에 지정된 디렉토리의 tx 하위 디렉토리에 저장됩니다. 기본 트랜잭션 로그 위치 값은 GlassFish 서버 노드의 고급 탭 아래에 있는 도메인 속성 페이지의 로그 루트 필드에 지정된 디렉토리입니다.

관련 작업

  • 클러스터링된 GlassFish 서버 인스턴스에 대한 트랜잭션을 복구하려면

  • 독립 실행형 GlassFish 서버 인스턴스에 대한 트랜잭션을 복구하는 방법

동일한 asadmin 하위 명령

  • recover-transactions(1)
