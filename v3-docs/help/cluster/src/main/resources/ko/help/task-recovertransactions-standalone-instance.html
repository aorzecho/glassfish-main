독립형 GlassFish Server 인스턴스에 대한 트랜잭션을 복구하는 방법

  1. 탐색 트리에서 [독립형 인스턴스] 노드를 선택합니다.

    [독립형 서버 인스턴스] 페이지가 열립니다.

  2. 트랜잭션 복구가 필요한 인스턴스를 선택합니다.

    인스턴스에 대한 [일반 정보] 페이지가 열립니다.

  3. [일반 정보] 페이지에서 [트랜잭션 복구]를 누릅니다.

    [트랜잭션 복구] 페이지가 열립니다.

    페이지 내용은 인스턴스가 실행되고 있는지 여부에 따라 결정됩니다.

    • 인스턴스가 실행 중이면 [트랜잭션 복구] 페이지에 복구는 실행 중인 서버 자체에서 수행합니다.라는 텍스트와 [확인] 단추만 표시됩니다.

    • 인스턴스가 실행 중이 아니면 [트랜잭션 복구] 페이지에 추가 옵션이 표시됩니다.

  4. 대상 서버 드롭다운 목록에서 현재 인스턴스의 트랜잭션을 복구할 수 있는 다른 인스턴스를 선택합니다.

    독립형 인스턴스의 경우 server라고 명명된 인스턴스를 선택합니다. 이 서버는 DAS(도메인 관리 서버)를 나타냅니다.

  5. [트랜잭션 로그 디렉토리] 필드에 트랜잭션 로그를 저장할 디렉토리를 입력합니다.

    기본값은 도메인 디렉토리 아래의 /logs/tx입니다. 트랜잭션 로그는 클러스터 구성에 대한 트랜잭션 서비스의 트랜잭션 로그 위치 설정에 지정된 디렉토리의 tx 하위 디렉토리에 저장됩니다. 기본 트랜잭션 로그 위치 값은 GlassFish Server 노드의 [고급] 탭 아래에 있는 [도메인 속성] 페이지의 [로그 루트] 필드에 지정된 디렉토리입니다.

  6. [확인]을 누릅니다.
