독립형 인스턴스

독립형 GlassFish Server 인스턴스를 생성하려면 새 독립형 서버 인스턴스 페이지를 사용합니다.

새 독립형 서버 인스턴스 페이지에는 다음 옵션이 포함되어 있습니다.

인스턴스 이름

생성하려는 인스턴스의 이름입니다.

이름은 다음 요구 사항을 충족해야 합니다.

  • 이름에는 ASCII 문자만 포함될 수 있습니다

  • 이름은 문자, 숫자 또는 밑줄로 시작해야 합니다.

  • 이름에는 다음 문자만 포함할 수 있습니다.

    • 소문자

    • 대문자

    • 숫자

    • 하이픈

    • 마침표

    • 밑줄

  • 이름은 도메인에서 고유해야 하며 다른 GlassFish Server 인스턴스, 클러스터, 명명된 구성 또는 노드의 이름이어서는 안됩니다.

  • 이름은 domain, server 또는 GlassFish Server에서 예약된 다른 모든 키워드여서는 안됩니다.

노드

인스턴스가 있을 수 있는 호스트를 정의하는 기존 노드의 드롭다운 목록입니다. 목록에서 하나의 노드를 선택해야 합니다. DAS(도메인 관리 서버)가 실행 중인 호스트에 인스턴스를 생성할 경우 미리 정의된 노드 localhost-domain을 선택합니다.

구성

인스턴스에서 사용할 수 있는 기존에 명명된 구성의 드롭다운 목록입니다. 목록에서 하나의 구성을 선택해야 합니다. 인스턴스는 인스턴스를 생성할 때 선택한 구성을 사용합니다.

선택한 구성의 복사본 만들기

이 옵션을 선택한 경우 인스턴스를 생성할 때 선택한 구성이 복사됩니다. 구성 복사본에는 instance-name-config라는 이름이 지정됩니다. 여기서 instance-name은 [인스턴스 이름] 필드에 입력한 인스턴스의 이름입니다.

구성 드롭다운 목록에서 default-config를 선택한 경우 이 옵션을 선택해야 합니다. default-config 구성은 복사만 할 수 있고 참조할 수는 없습니다.

선택한 구성 참조

이 옵션을 선택한 경우 인스턴스가 지정된 기존 명명된 구성을 사용합니다.

구성 드롭다운 목록에서 default-config를 선택한 경우 이 옵션을 선택하면 안됩니다. default-config 구성은 참조할 수는 없고 복사만 할 수 있습니다.

관련 작업

독립형 GlassFish Server 인스턴스를 생성하는 방법

동일한 asadmin 하위 명령

create-instance(1)
