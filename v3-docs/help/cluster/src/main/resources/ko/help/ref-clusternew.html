새 클러스터

클러스터를 만들려면 새 클러스터 페이지를 사용합니다.

새 클러스터 페이지에는 다음 옵션이 포함되어 있습니다.

클러스터 이름

클러스터의 이름입니다.

이름은 다음 요구 사항을 충족해야 합니다.

  • 이름에는 ASCII 문자만 포함될 수 있습니다

  • 이름은 문자, 숫자 또는 밑줄로 시작해야 합니다.

  • 이름에는 다음 문자만 포함할 수 있습니다.

    • 소문자

    • 대문자

    • 숫자

    • 하이픈

    • 마침표

    • 밑줄

  • 이름은 도메인에서 고유해야 하며 다른 클러스터, 명명된 구성, GlassFish Server 인스턴스 또는 노드의 이름이어서는 안 됩니다.

  • 이름은 domain, server 또는 GlassFish Server에서 예약된 다른 모든 키워드여서는 안 됩니다.

구성

클러스터에서 사용할 수 있는 기존에 명명된 구성의 드롭다운 목록입니다. 클러스터는 클러스터에 추가된 모든 인스턴스의 구성을 정의하는 명명된 구성을 필요로 합니다. 목록에서 하나의 구성을 선택해야 합니다. 클러스터는 클러스터를 만들 때 선택한 구성을 사용합니다.

선택한 구성의 복사본을 만듭니다.

이 옵션을 선택한 경우 클러스터를 만들 때 선택한 구성이 복사됩니다. 구성의 복사본에는 cluster-name-config라는 이름이 지정됩니다. 여기서 cluster-name은 클러스터 이름 필드에 입력한 클러스터의 이름입니다.

구성 드롭다운 목록에서 default-config를 선택한 경우 이 옵션을 선택해야 합니다. default-config 구성은 복사만 할 수 있고 참조할 수는 없습니다.

선택한 구성을 참조합니다.

이 옵션을 선택한 경우 클러스터가 지정된 기존 명명된 구성을 사용합니다.

구성 드롭다운 목록에서 default-config를 선택한 경우 이 옵션을 선택하면 안 됩니다. default-config 구성은 참조할 수는 없고 복사만 할 수 있습니다.

만들 서버 인스턴스

클러스터를 만들 때 만들 GlassFish Server 인스턴스 목록입니다. 각 인스턴스에 대해 제공되는 정보는 다음과 같습니다.

인스턴스 이름

인스턴스의 이름입니다.

가중치

인스턴스의 로드 균형 조정 가중치를 나타내는 정수입니다.

로드 균형 조정 가중치는 인스턴스가 처리해야 하는 클러스터에 대한 모든 요청의 비율을 결정합니다.

예를 들어 2개의 인스턴스 클러스터에서 한 인스턴스가 네 개의 요청 중에서 하나를 처리하고 다른 인스턴스가 네 개의 요청 중에서 세 개를 처리하도록 하고자 할 수 있습니다. 이 경우 네 개의 요청 중에서 하나를 처리할 인스턴스의 가중치를 1로 설정하고 다른 인스턴스의 가중치를 3으로 설정합니다. 백분율을 사용하려는 경우 인스턴스의 가중치를 각각 25와 75로 설정합니다.

기본 가중치는 100입니다.

노드

인스턴스가 있을 수 있는 기존 노드의 드롭다운 목록입니다. 목록에서 하나의 노드를 선택해야 합니다. 클러스터를 만들 때 인스턴스가 선택한 노드에 있게 됩니다.

만들 서버 인스턴스 테이블에는 다음 옵션도 포함되어 있습니다.

새로 만들기

인스턴스를 만드는 버튼입니다. 이 버튼을 누르면 만들 서버 인스턴스 테이블에 행이 추가됩니다.

삭제

인스턴스를 삭제하는 버튼입니다.
