일반 정보

[일반 정보] 페이지에는 다음 옵션이 포함되어 있습니다.

시작

인스턴스를 시작하는 단추입니다. 인스턴스가 실행되고 있는 경우 이 단추가 비활성화됩니다

정지

인스턴스를 정지하는 단추입니다. 인스턴스가 정지된 경우 이 단추가 비활성화됩니다

재시작

인스턴스를 재시작하는 단추입니다. 인스턴스가 정지된 경우 이 단추가 비활성화됩니다

로그 파일 보기

인스턴스에 대한 로그 파일을 보는 단추입니다.

로그 교체

인스턴스에 대한 로그 파일을 교체하는 단추입니다. 로그 파일이 교체되면 파일이 server.log_date-and-time 형식의 타임스탬프 이름이 포함된 이름으로 바뀌고 새 로그 메시지에 대한 빈 로그 파일이 생성됩니다. 변경 사항이 동적으로 적용됩니다. 서버를 재시작할 필요는 없습니다.

트랜잭션 복구

인스턴스에 대한 트랜잭션을 복구하는 단추입니다.

이름

인스턴스가 생성될 때 인스턴스에 지정된 이름입니다. 이 필드는 읽기 전용입니다.

JVM

JVM 보고서를 누르면 별도의 창이 열리고 요약 보고서, 메모리 관리 및 불필요한 정보 수집 보고서, 클래스 로딩 보고서 및 현재 스레드 덤프를 포함한 Java Virtual Machine에 대한 보고서가 표시됩니다.

상태

인스턴스가 실행되고 있는지 또는 실행되고 있지 않은지를 나타냅니다. 이 필드는 읽기 전용입니다.

노드

인스턴스가 있는 노드의 이름입니다. 이름을 누르면 해당 노드에 대한 노드 편집 페이지가 열립니다.

구성

인스턴스가 참조하는 구성입니다. 구성을 누르면 인스턴스에 대한 구성 페이지가 열립니다.

디버그

JPDA(Java Platform Debugger Architecture)(http://java.sun.com/javase/technologies/core/toolsapis/jpda/) 디버깅을 인스턴스에 사용할 수 있는지 여부를 나타냅니다. 이 필드는 읽기 전용입니다.

인스턴스를 관리 콘솔에서 시작한 경우 인스턴스에 대해 JPDA 디버깅이 사용으로 설정되지 않습니다. JPDA 디버깅을 사용으로 설정한 상태에서 인스턴스를 시작하려면 start-instance(1) 하위 명령을 사용합니다.

HTTP 포트

인스턴스가 다음 요청 유형을 수신하는 포트의 콤마로 구분된 포트 번호 목록입니다.

  • 관리 요청

  • HTTP 요청

  • HTTPS 요청

이 필드는 읽기 전용입니다.

IIOP 포트

인스턴스가 다음 접속 유형을 사용하는 포트의 콤마로 구분된 포트 번호 목록입니다.

  • 보안 IIOP 접속

  • 보안 IIOP 접속(클라이언트 인증 포함)

  • IIOP 접속

이 필드는 읽기 전용입니다.

관련 작업

  • 독립형 GlassFish Server 인스턴스를 시작하는 방법

  • 독립형 GlassFish Server 인스턴스를 정지하는 방법

  • 독립형 GlassFish Server 인스턴스에 대한 로그 파일을 보는 방법

  • 독립형 GlassFish Server 인스턴스에 대한 트랜잭션을 복구하는 방법

동일한 asadmin 하위 명령

  • generate-jvm-report(1)

  • start-instance(1)

  • stop-instance(1)

  • rotate-log(1)
