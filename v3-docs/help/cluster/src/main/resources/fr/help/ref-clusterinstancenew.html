Nouvelle instance de serveur clusterisée

La page Nouvelle instance de serveur clusterisée vous permet d'ajouter une instance de GlassFish Server à un cluster existant.

Elle contient les options suivantes.

Nom du cluster

Nom qui a été assigné au cluster lorsque le cluster a été créé. Ce champ est en lecture seule.

Nom d’instance

Nom de l'instance en cours de création.

Le nom doit respecter les exigences suivantes :

  • Le nom doit commencer par une lettre, un chiffre ou un caractère de soulignement.

  • Le nom ne peut contenir que les caractères suivants :

    • Lettres minuscules

    • Lettres majuscules

    • Chiffres

    • Trait d'union

    • Point

    • Caractère de soulignement

  • Le nom doit être unique dans le domaine et ne doit pas être le nom d'une autre instance de GlassFish Server, d'un cluster, d'une configuration nommée ou d'un nœud.

  • Le nom ne doit pas être domain, server ou tout autre mot-clé réservé par GlassFish Server.

Nœud

Liste déroulante des nœuds existants définissant les hôtes sur lesquels l'instance peut résider. Un nœud doit être sélectionné dans la liste. Si l'instance doit être créée sur l'hôte hébergeant le serveur d'administration de domaine (DAS), utilisez le nœud prédéfini localhost-domain.
