Instances de serveur clusterisées

La page Instances de serveur clusterisées permet de créer et gérer les instances GlassFish Server dans le cluster actif.

Le champ Nom du cluster est un champ en lecture seule qui affiche le nom du cluster actif.

La page Instances de serveur clusterisées affiche une liste des instances appartenant au cluster actif. Pour chaque instance, les informations suivantes sont fournies.

Nom

Nom assigné à l'instance lors de sa création. Le fait de cliquer sur le nom ouvre la page General Information de l'instance.

Pondération LB

Entier qui représente la pondération de l'équilibrage de charge de l'instance.

La pondération de l'équilibrage de charge détermine la proportion de toutes les requêtes pour le cluster que l'instance doit traiter.

Par exemple, dans un cluster comportant deux instances, vous pouvez exiger une instance pour le traitement de l'une des quatre requêtes, et une autre instance pour le traitement des trois requêtes restantes. Dans ce cas, définissez la pondération de l'instance qui doit traiter une requête sur quatre sur 1 et celle de l'autre sur 3. Si vous préférez utiliser des pourcentages, définissez la pondération des instances sur 25 et 75, respectivement.

La pondération par défaut est de 100.

Configuration

Configuration à laquelle l'instance fait référence. Le fait de cliquer sur la configuration ouvre la page Configurations de l'instance.

Nœud

Nom du nœud sur lequel se trouve l'instance. Le fait de cliquer sur le nom ouvre la page Modifier le nud pour le nœud.

Statut

Mention indiquant si l'instance est en cours d'exécution, doit être redémarrée pour être mise à jour avec les modifications de la configuration ou n'est pas en cours d'exécution. Ce champ est en lecture seule.

Le tableau Instances serveur contient également les options suivantes.

Nouveau

Bouton permettant d'ajouter une instance au cluster.

Supprimer

Bouton permettant de supprimer une ou plusieurs instances sélectionnées du cluster.

Attention - Si vous utilisez un cluster Java Message Service (JMS) avec un courtier principal, ne supprimez pas l'instance associée au courtier principal. Si cette instance doit être supprimée, utilisez la sous-commande change-master-broker(1) pour affecter le courtier principal à une autre instance.

Démarrage

Bouton permettant de démarrer une ou plusieurs instances sélectionnées dans le cluster.

Arrêter

Bouton permettant d'arrêter une ou plusieurs instances sélectionnées dans le cluster.
