Ajouter une instance de GlassFish Server à un cluster

Les instances de GlassFish Server forment la base d’un déploiement d’application. Une instance clusterisée est un membre d'un cluster. Chacune d’elles appartient à un domaine unique et possède sa propre structure de répertoire, configuration et ses propres applications déployées. Chaque instance comprend également le conteneur Web Java Platform, Enterprise Edition (Java EE) et le conteneur Enterprise JavaBeans (technologie EJB). Chaque nouvelle instance doit contenir une référence à un nœud qui définit l'hôte sur lequel l'instance va résider.

Remarque - Pour créer une instance sur un nœud de type CONFIG représentant un hôte distant, vous devez utiliser la sous-commande create-local-instance(1). Vous ne pouvez pas utiliser la Console d'administration à cette fin.

Avant de commencer

Assurez-vous que les éléments suivants existent :

  • The cluster to which you are adding the instance

  • The node on which the instance is to reside

  1. Dans l’arborescence de navigation, développez le nœud Clusters.
  2. Sous le nœud Clusters développé, sélectionnez le nœud pour le cluster.

    La page Informations générales du cluster s'ouvre.

  3. Dans la page Informations générales, cliquez sur l’onglet Instances.

    La page Instances de serveur clusterisées du cluster s'ouvre.

  4. Sur la page Instances de serveur clusterisées, cliquez sur Nouveau.

    La page Nouvelle instance de serveur clusterisée s'ouvre.

  5. Dans le champ Nom, entrez un nom d’instance.

    Le nom doit respecter les exigences suivantes :

    • Le nom ne peut contenir que des caractères ASCII.

    • Le nom doit commencer par une lettre, un chiffre ou un caractère de soulignement.

    • Le nom ne peut contenir que les caractères suivants :

      • Lettres minuscules

      • Lettres majuscules

      • Chiffres

      • Trait d'union

      • Point

      • Caractère de soulignement

    • Le nom doit être unique dans le domaine et ne doit pas être le nom d'une autre instance de GlassFish Server, d'un cluster, d'une configuration nommée ou d'un nœud.

    • Le nom ne doit pas être domain, server ou tout autre mot-clé réservé par GlassFish Server.

  6. À partir de la liste déroulante Nœuds, sélectionnez le nœud sur lequel l'instance va résider.
  7. Cliquez sur OK.
