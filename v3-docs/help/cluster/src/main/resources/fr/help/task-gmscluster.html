Configurer les paramètres du service de gestion de groupes pour un cluster

Le service de gestion de groupes (GMS) est un service inclus dans un processus offrant des services de surveillance de clusters, d'appartenance au cluster et de communication de groupe. Il exécute les fonctions suivantes :

  • Il signale les modules inscrits dans une instance GlassFish Server lorsqu'une ou plusieurs instances membres sont ajoutées à un cluster et enlevées d'un cluster, ou lorsque leur échec est suspecté ou a été confirmé.

  • Il permet d'envoyer et recevoir des messages dans un groupe de processus.

  1. Dans l'arborescence de navigation, développez le noeud Clusters.

  2. Sélectionnez le cluster pour lequel vous souhaitez configurer les paramètres du service de gestion de groupes (GMS).

    La page Informations générales du cluster apparaît.

  3. Sélectionnez la case Activé du champ GMS pour activer le service GMS pour le cluster.

    Cette option est sélectionnée par défaut. Pour désactiver le service GMS pour le cluster, désélectionnez la case Activé du champ GMS.

    Si cette option est activée, le service de gestion de groupes est démarré dans chaque instance de serveur du cluster et dans le serveur d'administration de domaine (DAS). Le serveur DAS se trouve dans chaque cluster dans lequel le service GMS est activé.

  4. Dans le champ Port de multidiffusion, indiquez le port utilisé par le service de gestion de groupes pour l'écoute des événements de groupe.

    Indiquez un numéro de port UDP standard compris entre 2048 et 32000. La valeur par défaut est générée automatiquement dans cette plage.

  5. Dans le champ Adresse de multidiffusion, indiquez l'adresse utilisée par le service de gestion de groupes pour l'écoute des événements de groupe.

    Spécifiez une adresse IP de multidiffusion de classe D. Les adresses IP de classe D sont comprises dans la plage 224.0.0.0 à 239.255.255.255 (inclus). L'adresse 224.0.0.0 est réservée et ne doit pas être utilisée. La valeur par défaut est 228.9.XX. YY, XX et YY étant des valeurs indépendantes générées automatiquement, comprises entre 0 et 255.

  6. Dans le champ Adresse d'interface de liaison, indiquez l'adresse de l'interface réseau pour le service de gestion de groupes, s'il en existe plusieurs.

    Ce paramètre permet de configurer les interfaces réseau utilisées pour le service GMS, s'il en existe plusieurs, sur les ordinateurs hébergeant les instances de serveur dans le cluster. Cette valeur doit être une adresse IP d'interface réseau locale. Par défaut, toutes les adresses d'interface réseau publique sont utilisées.

    Afin de spécifier une adresse valide pour toutes les instances GlassFish Server du cluster, utilisez une propriété système pour définir l'adresse de chaque instance séparément.

    Par exemple, créez la propriété système GMS-BIND-INTERFACE-ADDRESS-cluster-name sur le serveur DAS. Définissez ensuite l'option Adresse d'interface de liaison sur ${GMS-BIND-INTERFACE-ADDRESS-cluster-name} pour spécifier la propriété système. Enfin, pour chaque instance du cluster, définissez la propriété système GMS-BIND-INTERFACE-ADDRESS-cluster-name sur l'adresse d'interface réseau requise sur l'ordinateur de l'instance.

  7. Pour spécifier d'autres propriétés, sélectionnez l'onglet Propriétés et le sous-onglet Propriétés de cluster.

    • Pour ajouter une propriété, cliquez sur le bouton Ajouter une propriété. Sur la ligne vide qui apparaît, saisissez le nom de la propriété dans le champ Nom et la valeur de la propriété dans le champ Valeur.

    • Pour modifier une propriété, modifiez le champ Valeur de cette propriété.

    • Pour supprimer une propriété, sélectionnez la case située à gauche du champ Nom de la propriété à supprimer, puis cliquez sur le bouton Supprimer des propriétés.

  8. Cliquez sur Enregistrer.
