Récupérer les transactions d'une instance de GlassFish Server autonome

  1. Dans l’arborescence de navigation, sélectionnez le nœud Instances autonomes.

    La page Instances de serveur autonomes s’ouvre.

  2. Sélectionnez l'instance pour laquelle une récupération de transaction est requise.

    La page Informations générales de l'instance s'ouvre.

  3. Sur la page Informations générales, cliquez sur Récupérer les transactions.

    La page Récupérer les transactions s'ouvre.

    Le contenu de cette page varie selon que l'instance est en cours d'exécution.

    • Si l'instance est en cours d'exécution, la page Récupérer les transactions contient uniquement le texte La récupération est effectuée par le serveur en cours d’exécution lui-même et le bouton OK.

    • Si l'instance n'est pas en cours d'exécution, la page Récupérer les transactions contient des options supplémentaires.

  4. Dans la liste déroulante Serveur de destination, sélectionnez une autre instance pouvant récupérer les transactions sur l'instance active.

    Pour une instance autonome, sélectionnez l'instance nommée server représentant le serveur d'administration de domaine (DAS).

  5. Dans le Champ Répertoire du journal des transactions, entrez le répertoire dans lequel les journaux des transactions sont stockés.

    La valeur par défaut est /logs/tx sous le répertoire du domaine. Les journaux des transaction sont conservés dans le sous-répertoire tx du répertoire spécifié par le paramètre Emplacement du journal des transactions du Service de transaction pour la configuration du cluster. La valeur Emplacement du journal des transactions par défaut est le répertoire spécifié dans le champ Racine journal de la page Attributs de domaine, qui se trouve dans l’onglet Avancé du nœud GlassFish Server.

  6. Cliquez sur OK.
