Nouveau cluster

La page Nouveau cluster vous permet de créer un cluster.

Elle contient les options suivantes :

Nom du cluster

Nom du cluster.

Le nom doit respecter les exigences suivantes :

  • Le nom ne peut contenir que des caractères ASCII.

  • Le nom doit commencer par une lettre, un chiffre ou un caractère de soulignement.

  • Le nom ne peut contenir que les caractères suivants :

    • Lettres minuscules

    • Lettres majuscules

    • Chiffres

    • Trait d'union

    • Point

    • Caractère de soulignement

  • Le nom du nœud doit être unique dans le domaine et ne doit pas être le nom d'un autre cluster, d'une configuration nommée, d'une instance de GlassFish Server ou d'un nœud.

  • Le nom ne doit pas être domain, server ou tout autre mot-clé réservé par GlassFish Server.

Configuration

Liste déroulante des configurations nommées existantes pouvant être utilisées par le cluster. Un cluster nécessite une configuration nommée qui définit la configuration de toutes les instances ajoutées au cluster. Une configuration doit être sélectionnée dans la liste. Le cluster doit utiliser la configuration sélectionnée lors de sa création.

Créer une copie de la configuration sélectionnée

Si cette option est sélectionnée, la configuration sélectionnée est copiée lors de la création du cluster. Le nom cluster-name-config est affecté à la copie de la configuration, cluster-name étant le nom du cluster tel que vous l'avez saisi dans le champ Nom du cluster.

Si default-config est sélectionné dans la liste déroulante Configuration, vous devez sélectionner cette option. La configuration default-config peut être copiée, mais ne peut pas être utilisée comme référence.

Utiliser la configuration sélectionnée comme référence

Si cette option est sélectionnée, le cluster va utiliser la configuration nommée existante indiquée.

Si default-config est sélectionné dans la liste déroulante Configuration, vous ne devez pas sélectionner cette option. La configuration default-config peut être copiée, mais ne peut pas être utilisée comme référence.

Instances de serveur à créer

Une liste des instances GlassFish Server qui seront créées lors de la création du cluster. Pour chaque instance, les informations suivantes sont fournies :

Nom d’instance

Nom de l'instance.

Pondération

Entier qui représente la pondération de l'équilibrage de charge de l'instance.

La pondération de l'équilibrage de charge détermine la proportion de toutes les requêtes pour le cluster que l'instance doit traiter.

Par exemple, dans un cluster comportant deux instances, vous pouvez exiger une instance pour le traitement de l'une des quatre requêtes, et une autre instance pour le traitement des trois requêtes restantes. Dans ce cas, définissez la pondération de l'instance qui doit traiter une requête sur quatre sur 1 et celle de l'autre sur 3. Si vous préférez utiliser des pourcentages, définissez la pondération des instances sur 25 et 75, respectivement.

La pondération par défaut est de 100.

Nœud

Liste déroulante des nœuds existant sur lesquels l'instance peut résider. Un nœud doit être sélectionné dans la liste. L'instance résidera sur le nœud sélectionné lors de la création du cluster.

Le tableau Instances de serveur à créer contient également les options suivantes.

Nouveau

Bouton permettant de créer une instance. Le fait de cliquer sur ce bouton permet d'ajouter une ligne au tableau Instances de serveur à créer.

Supprimer

Bouton permettant de supprimer une instance.
