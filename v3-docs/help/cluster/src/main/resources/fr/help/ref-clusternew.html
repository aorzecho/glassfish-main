Nouveau cluster

La page Nouveau cluster permet de créer un cluster.

Elle contient les options suivantes :

Nom du cluster

Nom du cluster.

Le nom doit respecter les exigences suivantes :

  • Il ne peut contenir que des caractères ASCII.

  • Il doit commencer par une lettre, un chiffre ou un trait de soulignement.

  • Il ne peut contenir que les caractères suivants :

    • Lettres minuscules

    • Lettres majuscules

    • Chiffres

    • Trait d'union

    • Point

    • Trait de soulignement

  • Le nom du noeud doit être unique dans le domaine ; il ne doit pas appartenir à un autre cluster, à une configuration nommée, à une instance de GlassFish Server ou à un noeud.

  • Il ne doit pas être domain, server ou tout autre mot-clé réservé par GlassFish Server.

Configuration

Liste déroulante des configurations nommées existantes pouvant être utilisées par le cluster. Un cluster nécessite une configuration nommée qui définit la configuration de toutes les instances qu'il contient. Vous devez sélectionner une configuration dans la liste. Le cluster utilisera la configuration sélectionnée lors de sa création.

Créer une copie de la configuration sélectionnée

Si cette option est sélectionnée, la configuration choisie est copiée lors de la création du cluster. Le nom cluster-name-config est affecté à la copie de la configuration, cluster-name étant le nom du cluster tel que vous l'avez saisi dans le champ Nom du cluster.

Si default-config est sélectionné dans la liste déroulante Configuration, vous devez choisir cette option. La configuration default-config peut être copiée, mais pas utilisée comme référence.

Utiliser la configuration sélectionnée comme référence

Si cette option est sélectionnée, le cluster utilisera la configuration nommée existante indiquée.

Si default-config est sélectionné dans la liste déroulante Configuration, vous ne devez pas choisir cette option. La configuration default-config peut être copiée, mais pas utilisée comme référence.

Instances de serveur à créer

Liste des instances GlassFish Server qui seront créées en même temps que le cluster. Pour chaque instance, les informations suivantes sont fournies :

Nom d'instance

Nom de l'instance.

Pondération

Entier qui représente la pondération de l'équilibrage de charge de l'instance.

La pondération de l'équilibrage de charge détermine la proportion de toutes les demandes destinées au cluster que l'instance doit traiter.

Par exemple, dans un cluster comportant deux instances, vous pouvez avoir besoin d'une instance pour le traitement de l'une des quatre demandes, et une autre instance pour le traitement des trois demandes restantes. Dans ce cas, définissez la pondération de l'instance qui doit traiter une demande sur quatre sur 1 et celle de l'autre instance sur 3. Si vous préférez utiliser des pourcentages, définissez la pondération des instances sur 25 et 75, respectivement.

La pondération par défaut est de 100.

Noeud

Liste déroulante des noeuds existants sur lesquels l'instance peut résider. Vous devez sélectionner un noeud dans la liste. L'instance résidera sur le noeud sélectionné lors de la création du cluster.

La table Instances de serveur à créer contient également les options suivantes :

Nouveau

Bouton permettant de créer une instance. Le fait de cliquer sur ce bouton permet d'ajouter une ligne à la table Instances de serveur à créer.

Supprimer

Bouton permettant de supprimer une instance.
