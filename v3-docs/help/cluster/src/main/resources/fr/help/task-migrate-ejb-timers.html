Migrer des horloges EJB

Si une instance de GlassFish Server est arrêtée, les services d'horloge EJB (Enterprise JavaBeans) qui y sont installés s'arrêtent également. Si l'instance a été arrêtée de manière anormale ou inattendue, pour permettre à ces horloges de s'exécuter pendant que l'instance est stoppée, vous devez migrer les horloges vers une instance de serveur en cours d'exécution dans le cluster.

  1. Dans l’arborescence de navigation, développez le nœud Clusters.
  2. Dans l'arborescence de navigation, sous le nœud Clusters, sélectionnez le cluster.

    La page Informations générales du cluster s'ouvre.

  3. Cliquez sur le bouton Migrer les horloges EJB de la page Informations générales.

    La page Migrer les services d'horloge EJB s'ouvre.

    Remarque - Si toutes les instances du cluster sont en cours d'exécution, le bouton Migrer les services d'horloge EJB est désactivé.

  4. Sur la page Migrer les services d'horloge EJB, sélectionnez l'instance à partir de laquelle et vers laquelle vous souhaitez migrer les services d'horloge.
    1. Dans la liste déroulante Source, sélectionnez l’instance arrêtée à partir de laquelle vous souhaitez migrer les services d'horloge.
    2. (Facultatif) Dans la liste déroulante Destination, sélectionnez l’instance en cours d'exécution vers laquelle vous souhaitez migrer les services d'horloge.
    3. Cliquez sur OK.
  5. Arrêtez et redémarrez l'instance vers laquelle vous avez migré les services d'horloge EJB.

    Si l’instance source est en cours d'exécution ou si l’instance de destination n’est pas en cours d'exécution, Console d'administration affiche un message d’erreur.
