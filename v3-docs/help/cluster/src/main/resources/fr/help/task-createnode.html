Créer un nœud

Un noeud représente un hôte sur lequel le logiciel GlassFish Server est installé. Un nœud doit être présent sur chaque hôte sur lequel résident des instances GlassFish Server.

  1. Dans l’arborescence de navigation, sélectionnez le nœud Nœuds.

    La page Nœuds s'ouvre.

  2. Cliquez sur le bouton Nouveau de la page Nœuds.

    La page Nouveau nœud s’ouvre.

  3. Dans le champ Nom, entrez le nom choisi pour le nœud.
  4. Dans la liste déroulante Type, sélectionnez le type du nœud à créer.
    • Pour créer un nœud prenant en charge la communication SSH (shell sécurisé), sélectionnez SSH.
    • Pour créer un nœud qui ne prend pas en charge la communication à distance, sélectionnez CONFIG.
  5. Dans le champ Hôte du nœud, saisissez le nom de l'hôte que le nœud représente.

    Le caractère obligatoire de ce champ dépend du type du nœud :

    • Si le type du nœud est CONFIG, le nom de l'hôte peut être omis. Le nom de l'hôte peut être déterminé lors de la création d'instances résidant sur le nœud.

    • Si le type du nœud est SSH, le nom de l'hôte doit être spécifié. Sinon, une erreur se produit.

  6. Dans le champ Répertoire du nœud, entrez le chemin d'accès au répertoire qui contiendra les instances de GlassFish Server créées sur le nœud.

    Si le type du nœud est CONFIG, laissez ce champ vide. Le répertoire du nœud peut être déterminé lors de la création d'instances résidant sur le nœud.

    Si un chemin d'accès relatif est spécifié, il est relatif au répertoire as-install, as-install correspondant au répertoire d'installation de base du logiciel GlassFish Server sur l'hôte.

  7. Dans le champ Répertoire d'installation, entrez le chemin d'accès complet au parent du répertoire d'installation de base du logiciel GlassFish Server sur l'hôte, par exemple, /export/glassfish3/.

    Si le type du nœud est CONFIG, laissez ce champ vide. Le répertoire d'installation peut être déterminé lors de la création d'instances résidant sur le nœud.

  8. Pour forcer le nœud à être créé dans la configuration du serveur d'administration de domaine (DAS) même si la validation des paramètres du nœud échoue, activez l'option Forcer.
  9. Dans le champ Port SSH, entrez le port à utiliser pour les connexions SSH à ce l'hôte de ce nœud.
  10. Dans le champ Nom d'utilisateur SSH, entrez le nom de l'utilisateur sur l'hôte de ce nœud qui exécutera le processus de connexion à l'hôte via SSH.
  11. Spécifiez la manière dont l'utilisateur SSH est authentifié lors de la connexion à l'hôte de ce nœud.
    • Pour indiquer que l'authentification se fait par le biais d'un fichier de clés privées SSH, effectuez les étapes suivantes :
      1. Dans la liste déroulante Authentification de l'utilisateur SSH, sélectionnez Fichier de clés.
      2. Dans le champ Fichier de clés, entrez le chemin absolu du fichier de clés privées SSH pour l'utilisateur spécifié par l'option Nom d'utilisateur SSH.

        Remarque - GlassFish Server prend également en charge l'authentification par mots de passe via l'entrée AS_ADMIN_SSHPASSWORD dans le fichier de mots de passe. Le fichier de mots de passe est spécifié dans l'option --paswordfile de l'utilitaire asadmin(1M).

    • Pour indiquer que l'authentification se fait par le biais d'un mot de passe stocké dans la configuration DAS, effectuez les étapes suivantes :

      Attention - Ce mot de passe est stocké sous forme de texte en clair dans la configuration DAS. Pour une sécurité accrue, utilisez un fichier de clés ou un alias de mot de passe.

      1. Dans la liste déroulante Authentification de l'utilisateur SSH, sélectionnez Mot de passe.
      2. Dans le champ Mot de passe SSH, entrez le mot de passe que l'utilisateur SSH va utiliser lors de la connexion à l'hôte de ce nœud.
      3. Dans le champ Confirmer le mot de passe SSH, saisissez à nouveau le mot de passe.
    • Pour indiquer que l'authentification se fait par le biais d'un alias de mot de passe, effectuez les étapes suivantes :
      1. Dans la liste déroulante Authentification de l'utilisateur SSH, sélectionnez Alias de mot de passe.
      2. Dans la liste déroulante Alias de mot de passe, sélectionnez un alias existant.
  12. Cliquez sur Enregistrer.
