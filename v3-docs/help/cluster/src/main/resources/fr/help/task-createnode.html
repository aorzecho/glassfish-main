Créer un noeud

Un noeud représente un hôte sur lequel le logiciel GlassFish Server est installé. Un noeud doit exister au niveau de chaque hôte sur lequel résident des instances GlassFish Server.

  1. Dans l'arborescence de navigation, sélectionnez le noeud Noeuds.

    La page Noeuds apparaît.

  2. Sur la page Noeuds, cliquez sur Nouveau.

    La page Nouveau noeud apparaît.

  3. Dans le champ Nom, attribuez un nom au noeud.

  4. Dans la liste déroulante Type, sélectionnez le type du noeud à créer.

    • Pour activer la prise en charge par le noeud de la communication via le protocole distant DCOM (Distributed Component Object Model), sélectionnez DCOM.

    • Pour créer un noeud prenant en charge la communication SSH (shell sécurisé), sélectionnez SSH.

    • Pour créer un noeud qui ne prend pas en charge la communication à distance, sélectionnez CONFIG.

  5. Dans le champ Hôte du noeud, saisissez le nom de l'hôte que le noeud représente.

    Le caractère obligatoire de ce champ dépend du type du noeud :

    • Si le type du noeud est CONFIG, le nom de l'hôte peut être omis. Le nom de l'hôte peut être déterminé lors de la création des instances résidant sur le noeud.

    • Si le type du noeud est DCOM ou SSH, le nom de l'hôte doit être spécifié. Sinon, une erreur se produit.

  6. Dans le champ Répertoire du noeud, entrez le chemin du répertoire qui contiendra les instances de GlassFish Server créées sur le noeud.

    Si le type du noeud est CONFIG, laissez ce champ vide. Le répertoire du noeud peut être déterminé lors de la création des instances résidant sur le noeud.

    Si vous indiquez un chemin relatif, ce dernier est relatif au répertoire as-install où as-install est le répertoire d'installation de base du logiciel GlassFish Server sur l'hôte.

  7. Dans le champ Répertoire d'installation, entrez le chemin complet du parent du répertoire d'installation de base du logiciel GlassFish Server sur l'hôte, par exemple, /export/glassfish3.

    Si le type du noeud est CONFIG, laissez ce champ vide. Le répertoire d'installation peut être déterminé lors de la création des instances résidant sur le noeud.

  8. Terminez la création du noeud selon la méthode requise pour le type de noeud.

    1. Si le type de noeud est CONFIG, cliquez sur Enregistrer.

    2. Si le type de noeud est DCOM, suivez les instructions indiquées dans Terminer la création d'un noeud DCOM.

    3. Si le type de noeud est SSH, suivez les instructions indiquées dans Terminer la création d'un noeud SSH.
