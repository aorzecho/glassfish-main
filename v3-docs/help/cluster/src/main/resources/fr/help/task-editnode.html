Modifier un nœud

La modification d'un nœud met à jour les données de configuration d'un nœud GlassFish Server. Elle permet également de modifier le type du nœud de CONFIG à SSH afin d'activer la communication SSH pour le nœud.

Si vous modifiez le type du nœud de CONFIG à SSH, les valeurs par défaut sont appliquées si l'un des champs suivants n'est pas renseigné  :

  • Port SSH

  • Nom d'utilisateur SSH

  • Fichier de clés

  1. Dans l’arborescence de navigation, développez le nœud Nœuds.
  2. À partir de la liste des nœuds, sélectionnez le nœud à modifier.

    La page Modifier le nœud s’ouvre.

  3. Dans la liste déroulante Type, sélectionnez le type du nœud mis à jour.
    • Pour activer le nœud afin d'assurer la prise en charge de la communication SSH (shell sécurisé), sélectionnez SSH.
    • Pour supprimer la prise en charge de la configuration à distance à partir du nœud, sélectionnez CONFIG.

      Si vous sélectionnez CONFIG, aucune option de configuration SSH n'est affichée. Si le type actuel du nœud est SSH et que vous sélectionnez CONFIG, les données de configuration SSH sont supprimées de la configuration du nœud lorsque vous enregistrez vos modifications.

  4. Dans le champ Hôte du nœud, entrez le nom de l'hôte que le nœud mis à jour représentera.

    Le caractère obligatoire de ce champ dépend du type du nœud :

    • Si le type du nœud est CONFIG, le nom de l'hôte peut être omis. Le nom de l'hôte peut être déterminé lors de la création d'instances résidant sur le nœud.

    • Si le type du nœud est SSH, le nom de l'hôte doit être spécifié. Sinon, une erreur se produit.

  5. Dans le champ Répertoire du nœud, entrez le chemin d'accès au répertoire qui contiendra les instances de GlassFish Server créées sur le nœud.

    Si un chemin d'accès relatif est spécifié, il est relatif au répertoire as-install, as-install correspondant au répertoire d'installation de base du logiciel GlassFish Server sur l'hôte.

  6. Dans le champ Répertoire d'installation, entrez le chemin d'accès complet au parent du répertoire d'installation de base du logiciel GlassFish Server sur l'hôte, par exemple, /export/glassfish3/.
  7. Pour forcer le nœud à être créé dans la configuration du serveur d'administration de domaine (DAS) même si la validation des paramètres du nœud échoue, activez l'option Forcer.
  8. Dans le champ Port SSH, entrez le port à utiliser pour les connexions SSH à ce l'hôte de ce nœud.

    Si vous modifiez le type du nœud de CONFIG à SSH et laissez ce champ vide, l'option Port SSH est définie sur 22.

  9. Dans le champ Nom d'utilisateur SSH, entrez le nom de l'utilisateur sur l'hôte de ce nœud qui exécutera le processus de connexion à l'hôte via SSH.

    Si vous modifiez le type du nœud de CONFIG à SSH et laissez ce champ vide, l'option Nom d'utilisateur SSH est définie sur l'utilisateur qui exécute le processus DAS.

  10. Indiquez la manière dont l'utilisateur SSH est authentifié lors de la connexion à l'hôte de ce nœud.
    • Pour indiquer que l'authentification se fait par le biais d'un fichier de clés privées SSH, effectuez les étapes suivantes :
      1. Dans la liste déroulante Authentification de l'utilisateur SSH, sélectionnez Fichier de clés.
      2. Dans le champ Fichier de clés, entrez le chemin absolu du fichier de clés privées SSH pour l'utilisateur spécifié par l'option Nom d'utilisateur SSH.

        Remarque - GlassFish Server prend également en charge l'authentification par mots de passe via l'entrée AS_ADMIN_SSHPASSWORD dans le fichier de mots de passe. Le fichier de mots de passe est spécifié dans l'option --paswordfile de l'utilitaire asadmin(1M).

        Si vous modifiez le type du nœud de CONFIG à SSH et laissez ce champ vide, l'option Fichier de clés est définie sur un fichier de clés dans le répertoire .ssh de l'utilisateur. Si plusieurs fichiers de clés sont disponibles, GlassFish Server utilise l'ordre de préférence suivant :

        • id_rsa

        • id_dsa

        • identity

    • Pour indiquer que l'authentification se fait par le biais d'un mot de passe stocké dans la configuration DAS, effectuez les étapes suivantes :

      Attention - Ce mot de passe est stocké sous forme de texte en clair dans la configuration DAS. Pour une sécurité accrue, utilisez un fichier de clés ou un alias de mot de passe.

      1. Dans la liste déroulante Authentification de l'utilisateur SSH, sélectionnez Mot de passe.
      2. Dans le champ Mot de passe SSH, entrez le mot de passe que l'utilisateur SSH va utiliser lors de la connexion à l'hôte de ce nœud.
      3. Dans le champ Confirmer le mot de passe SSH, saisissez à nouveau le mot de passe.
    • Pour indiquer que l'authentification se fait par le biais d'un alias de mot de passe, effectuez les étapes suivantes :
      1. Dans la liste déroulante Authentification de l'utilisateur SSH, sélectionnez Alias de mot de passe.
      2. Dans la liste déroulante Alias de mot de passe, sélectionnez un alias existant.
  11. Cliquez sur Enregistrer.
