Récupérer les transactions

La page Récupérer les transactions permet de récupérer les transactions effectuées pour une instance GlassFish Server.

Si l'instance est en cours d'exécution, la page Récupérer les transactions contient uniquement le texte La récupération est effectuée par le serveur en cours d'exécution lui-même et le bouton OK.

Si l'instance n'est pas en cours d'exécution, la page Récupérer les transactions contient les options suivantes :

Nom d'instance

Nom affecté à l'instance lors de sa création. Ce champ est en lecture seule.

Serveur de destination

Autre instance pouvant récupérer les transactions sur l'instance en cours.

Répertoire des journaux des transactions

Répertoire dans lequel les journaux de transactions sont stockés. La valeur par défaut est /logs/tx sous le répertoire du domaine.

Les journaux des transactions sont conservés dans le sous-répertoire tx du répertoire spécifié par le paramètre Emplacement des journaux des transactions du service de transaction pour la configuration du cluster. La valeur par défaut du paramètre Emplacement des journaux des transactions est le répertoire spécifié dans le champ Racine journal de la page Attributs de domaine, qui se trouve dans l'onglet Avancé du noeud GlassFish Server.
