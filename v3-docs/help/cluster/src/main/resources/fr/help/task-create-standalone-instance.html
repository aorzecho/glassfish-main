Création d'une instance de GlassFish Server autonome

Les instances de GlassFish Server forment la base d'un déploiement d'application. Une instance autonome n'est pas un membre d'un cluster. Chacune d'elles appartient à un domaine unique et possède une structure de répertoire, une configuration et des applications déployées qui lui sont propres. Chaque instance comprend également le conteneur Web Java Platform, Enterprise Edition (Java EE) et le conteneur Enterprise JavaBeans (technologie EJB). Chaque nouvelle instance doit contenir une référence à un noeud qui définit l'hôte sur lequel elle résidera.

Remarque:

Pour créer une instance sur un noeud de type CONFIG représentant un hôte distant, vous devez utiliser la sous-commande create-local-instance(1). Vous ne pouvez pas utiliser la Console d'administration à cet effet.
Avant de commencer

Assurez-vous que le noeud sur lequel l'instance résidera existe. Pour plus d'informations, reportez-vous à Créer un noeud.

  1. Dans l'arborescence de navigation, sélectionnez le noeud Instances autonomes.

    La page Instances de serveur autonomes apparaît.

  2. Sur la page Instances de serveur autonomes, cliquez sur Nouveau.

    La page Nouvelle instance de serveur autonome apparaît.

  3. Dans le champ Nom, attribuez un nom à l'instance.

    Le nom doit respecter les exigences suivantes :

    • Il ne peut contenir que des caractères ASCII.

    • Il doit commencer par une lettre, un chiffre ou un trait de soulignement.

    • Il ne peut contenir que les caractères suivants :

      • Lettres minuscules

      • Lettres majuscules

      • Chiffres

      • Trait d'union

      • Point

      • Trait de soulignement

    • Le nom du noeud doit être unique dans le domaine ; il ne doit pas appartenir à une autre instance de GlassFish Server, à une configuration nommée, à un cluster ou à un noeud.

    • Il ne doit pas être domain, server ou tout autre mot-clé réservé par GlassFish Server.

  4. Dans la liste déroulante Noeuds, sélectionnez le noeud sur lequel l'instance résidera.

  5. Dans la liste déroulante Configuration, sélectionnez la configuration nommée que l'instance doit utiliser.

    Remarque:

    Si vous sélectionnez default-config, vous devez spécifier, à l'étape suivante, que la configuration n'est pas partagée. La configuration default-config ne peut pas être partagée.

  6. Indiquez si la configuration que l'instance va utiliser est partagée.

    • Si la configuration n'est pas partagée, laissez l'option Créer une copie de la configuration sélectionnée cochée.

      La configuration sélectionnée sera ainsi copiée lors de la création de l'instance. Le nom instance-name-config est affecté à la copie de la configuration, instance-name correspondant au nom de l'instance que vous créez.

    • Si la configuration est partagée, sélectionnez l'option Utiliser la configuration sélectionnée comme référence.

      L'instance utilisera la configuration nommée existante indiquée.

  7. Cliquez sur OK.
