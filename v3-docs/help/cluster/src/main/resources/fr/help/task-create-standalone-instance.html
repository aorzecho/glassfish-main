Créer une instance de GlassFish Server autonome

Les instances de GlassFish Server forment la base d’un déploiement d’application. Une instance autonome n'est pas un membre d'un cluster. Chacune d’elles appartient à un domaine unique et possède sa propre structure de répertoire, configuration et ses propres applications déployées. Chaque instance comprend également le conteneur Web Java Platform, Enterprise Edition (Java EE) et le conteneur Enterprise JavaBeans (technologie EJB). Chaque nouvelle instance doit contenir une référence à un nœud qui définit l'hôte sur lequel l'instance va résider.

Remarque - Pour créer une instance sur un nœud de type CONFIG représentant un hôte distant, vous devez utiliser la sous-commande create-local-instance(1). Vous ne pouvez pas utiliser la Console d'administration à cette fin.

Avant de commencer

Assurez-vous que The node on which the instance is to reside existe.

  1. Dans l’arborescence de navigation, sélectionnez le nœud Instances autonomes.

    La page Instances de serveur autonomes s’ouvre.

  2. Sur la page Instances de serveur autonomes, cliquez sur Nouveau.

    La page Nouvelle instance de serveur autonome s'ouvre.

  3. Dans le champ Nom, entrez un nom d’instance.

    Le nom doit respecter les exigences suivantes :

    • Le nom ne peut contenir que des caractères ASCII.

    • Le nom doit commencer par une lettre, un chiffre ou un caractère de soulignement.

    • Le nom ne peut contenir que les caractères suivants :

      • Lettres minuscules

      • Lettres majuscules

      • Chiffres

      • Trait d'union

      • Point

      • Caractère de soulignement

    • Le nom doit être unique dans le domaine et ne doit pas être le nom d'une autre instance de GlassFish Server, d'un cluster, d'une configuration nommée ou d'un nœud.

    • Le nom ne doit pas être domain, server ou tout autre mot-clé réservé par GlassFish Server.

  4. À partir de la liste déroulante Nœuds, sélectionnez le nœud sur lequel l'instance va résider.
  5. Dans la liste déroulante Configuration, sélectionnez la configuration nommée que l'instance doit utiliser.

    Remarque - Si vous sélectionnez default-config, vous devez spécifier dans l'étape suivante que la configuration n'est pas partagée. La configuration default-config ne peut pas être partagée.

  6. Indiquez si la configuration que l'instance va utiliser est partagée.
    • Si la configuration n'est pas partagée, laissez l'option Créer une copie de la configuration sélectionnée cochée.

      La configuration sélectionnée est copiée lors de la création de l'instance. Le nom instance-name-config est affecté à la copie de la configuration, instance-name correspondant au nom de l'instance que vous créez.

    • Si la configuration est partagée, sélectionnez l'option Utiliser la configuration sélectionnée comme référence.

      L'instance utilise la configuration nommée existante indiquée.

  7. Cliquez sur OK.
