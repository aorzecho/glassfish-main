Démarrer les instances de GlassFish Server autonomes

Avant de commencer

Assurez-vous que les conditions suivantes sont remplies :

  • Tous les nœuds sur lesquels résident les instances sont activés pour la communication à distance.

  • L'utilisateur du serveur d'administration de domaine (DAS) peut utiliser le protocole SSH (shell sécurisé) pour se connecter à l'hôte pour n'importe quel nœud dans lequel résident les instances.

Si l'une de ces conditions n'est pas remplie, démarrez chaque instance individuellement à partir de la ligne de commande à l'aide de la sous-commande start-local-instance(1).

  1. Dans l’arborescence de navigation, sélectionnez le nœud Instances autonomes.

    La page Instances de serveur autonomes s’ouvre.

  2. Pour chaque instance démarrée, cochez la case en regard de l'instance dans le tableau Instances de serveur.
  3. Cliquez sur Démarrer.

    Vous êtes invité à confirmer que vous souhaitez démarrer les instances.

  4. Confirmez le démarrage des instances.

    Le statut de chaque instance est mis à jour dans le tableau Instances de serveur au démarrage de l'instance.
