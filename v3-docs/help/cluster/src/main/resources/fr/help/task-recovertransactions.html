Récupérer des transactions d'une instance de GlassFish Server incluse dans un cluster

  1. Dans l'arborescence de navigation, développez le noeud Clusters.

  2. Sous le noeud Clusters développé, sélectionnez le noeud correspondant au cluster.

    La page Informations générales du cluster apparaît.

  3. Dans la page Informations générales, cliquez sur l'onglet Instances.

    La page Instances de serveur clusterisées du cluster apparaît.

  4. Sélectionnez l'instance pour laquelle une récupération des transactions est requise.

    La page Informations générales de l'instance apparaît.

  5. Sur la page Informations générales, cliquez sur Récupérer les transactions.

    La page Récupérer les transactions apparaît.

    Le contenu de cette page varie selon l'état d'exécution de l'instance.

    • Si l'instance est en cours d'exécution, la page Récupérer les transactions contient uniquement le texte La récupération est effectuée par le serveur en cours d'exécution lui-même et le bouton OK.

    • Si l'instance n'est pas en cours d'exécution, la page Récupérer les transactions contient des options supplémentaires.

  6. Dans la liste déroulante Serveur de destination, sélectionnez une autre instance dans le même cluster pouvant récupérer les transactions sur l'instance en cours.

  7. Dans le champ Répertoire des journaux des transactions, indiquez le répertoire dans lequel les journaux des transactions sont stockés.

    La valeur par défaut est /logs/tx sous le répertoire du domaine. Les journaux des transactions sont conservés dans le sous-répertoire tx du répertoire spécifié par le paramètre Emplacement des journaux des transactions du service de transaction pour la configuration du cluster. La valeur par défaut du paramètre Emplacement des journaux des transactions est le répertoire spécifié dans le champ Racine journal de la page Attributs de domaine, qui se trouve dans l'onglet Avancé du noeud GlassFish Server.

  8. Cliquez sur OK.
