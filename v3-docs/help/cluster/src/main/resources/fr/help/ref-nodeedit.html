Modifier le nœud

La page Modifier le nœud permet de mettre à jour les données de configuration d'un nœud GlassFish Server. Elle permet également de modifier le type d'un nœud de CONFIG à SSH pour activer la communication SSH pour ce nœud.

Si vous modifiez le type du nœud de CONFIG à SSH, les valeurs par défaut sont appliquées si l'un des champs suivants n'est pas renseigné  :

  • Port SSH

  • Nom d’utilisateur

  • Fichier de clés

La page Modifier le nœud contient les options suivantes :

Nom

Nom assigné au nœud lors de sa création. Ce champ est en lecture seule.

Type

Liste déroulante permettant de spécifier le type de nœud parmi les suivants :

SSH

Le nœud prend en charge la communication via le protocole SSH (shell sécurisé).

CONFIG

Le nœud ne prend pas en charge la communication à distance. Si CONFIG est sélectionné, aucune option de configuration SSH n'est affichée. Si le type actuel du nœud est SSH et que vous sélectionnez CONFIG, les données de configuration SSH sont supprimées de la configuration du nœud lorsque vous enregistrez vos modifications.

Hôte du nœud

Nom de l'hôte sur lequel le nœud sera représenté après sa mise à jour. Le caractère obligatoire de ce champ dépend du type du nœud :

  • Si le type du nœud est CONFIG, le nom de l'hôte peut être omis. Le nom de l'hôte peut être déterminé lors de la création d'instances résidant sur le nœud.

  • Si le type du nœud est SSH, le nom de l'hôte doit être spécifié. Sinon, une erreur se produit.

Répertoire du nœud

Chemin d'accès complet au répertoire devant contenir les instances de GlassFish Server créées sur le nœud. Si un chemin d'accès relatif est spécifié, il est relatif au répertoire as-install, as-install correspondant au répertoire d'installation de base du logiciel GlassFish Server sur l'hôte.

Si ce champ est vide, la configuration du nœud est mise à jour comme suit :

  • Si le type du nœud est CONFIG, aucun répertoire d'instances n'est spécifié pour le nœud dans la configuration du serveur d'administration de domaine (DAS). Le répertoire du nœud peut être déterminé lors de la création d'instances résidant sur le nœud.

  • Si le type du nœud est SSH, le répertoire du nœud est as-install/nodes, où as-install correspond au répertoire d'installation de base du logiciel GlassFish Server sur l'hôte.

Répertoire d’installation

Chemin d'accès complet au parent du répertoire d'installation de base du logiciel GlassFish Server sur l'hôte, par exemple, /export/glassfish3/.

Si ce champ est vide, la configuration du nœud est mise à jour comme suit :

  • Si le type du nœud est CONFIG, aucun répertoire d'installation n'est spécifié pour le nœud dans la configuration DAS. Le répertoire d'installation peut être déterminé lors de la création d'instances résidant sur le nœud.

  • Si le type du nœud est SSH, le répertoire d'installation est le parent du répertoire d'installation de base par défaut du logiciel GlassFish Server pour le serveur d'administration de domaine (DAS). Cette configuration n'est utile que si vous installez GlassFish Server dans le même emplacement sur tous les hôtes.

Force

Si l'option Activé est sélectionnée, le nœud est mis à jour dans la configuration du serveur d'administration de domaine (DAS) même si la validation des paramètres du nœud échoue. Pour valider les paramètres d'un noeud, le serveur DAS doit être capable de contacter l'hôte du noeud via SSH.

Port SSH

Port à utiliser pour les connexions SSH à l'hôte de ce nœud.

Si vous modifiez le type du nœud de CONFIG à SSH et laissez ce champ vide, l'option Port SSH est définie sur 22.

Si l'option Hôte du nœud est définie sur localhost-domain, l'option Port SSH est ignorée.

Nom d'utilisateur SSH

Utilisateur sur l'hôte de ce nœud qui va exécuter le processus de connexion à l'hôte via SSH.

Si vous modifiez le type du nœud de CONFIG à SSH et laissez ce champ vide, l'option Nom d'utilisateur est définie sur l'utilisateur qui exécute le processus DAS.

Si l'option Hôte du nœud est définie sur localhost-domain, l'option Utilisateur SSH est ignorée.

Authentification de l'utilisateur SSH

Liste déroulante pour indiquer la manière dont l'utilisateur SSH est authentifié lors de la connexion à l'hôte de ce nœud. Les méthodes possibles pour l'authentification sont les suivantes :

Fichier clé

L'utilisateur SSH est authentifié par l'intermédiaire d'un fichier de clés privées SSH. Si vous sélectionnez cette option, spécifiez le chemin d'accès absolu au fichier dans le champ Fichier de clés.

Mot de passe

L'utilisateur SSH est authentifié par le biais d'un mot de passe stocké dans la configuration DAS.

Attention - Ce mot de passe est stocké sous forme de texte en clair dans la configuration DAS. Pour une sécurité accrue, utilisez un fichier de clés ou un alias de mot de passe.

Si vous sélectionnez cette option, les champs Mot de passe SSH et Confirmer le mot de passe SSH sont activés pour vous permettre d'indiquer le mot de passe.

Alias de mot de passe

L'utilisateur SSH est authentifié par le biais d'un alias de mot de passe. Si vous sélectionnez cette option, choisissez l'alias de mot de passe dans la liste déroulante correspondante.

Mot de passe SSH

Mot de passe que l'utilisateur SSH va utiliser lors de la connexion à l'hôte de ce nœud.

Confirmer le mot de passe SSH

Confirmation du mot de passe que l'utilisateur SSH va utiliser lors de la connexion à l'hôte de ce nœud. Le contenu de ce champ doit correspondre exactement au contenu du champ Mot de passe SSH.

Alias de mot de passe

Liste déroulante d'alias de mot de passe définis pour ce domaine.

Fichier clé

Chemin d'accès absolu au fichier de clés privées SSH spécifié par l'option Nom d'utilisateur. Ce fichier est utilisé pour l'authentification du démon sshd sur l'hôte du nœud.

Remarque - GlassFish Server prend également en charge l'authentification par mots de passe via l'entrée AS_ADMIN_SSHPASSWORD dans le fichier de mots de passe. Le fichier de mots de passe est spécifié dans l'option --paswordfile de l'utilitaire asadmin(1M).

Si le fichier de clés privées SSH est protégé par une phrase de passe, le fichier de mots de passe doit contenir l'entrée AS_ADMIN_SSHKEYPASSPHRASE.

Le chemin d'accès au fichier de clés doit être accessible par le serveur DAS et le fichier de clés doit être lisible par le serveur DAS.

La valeur par défaut varie selon que vous modifiez le type de nœud de CONFIG à SSH pour activer la communication SSH pour le nœud :

Si vous modifiez le type du nœud de CONFIG à SSH et laissez ce champ vide, l'option Fichier de clés est définie sur un fichier de clés dans le répertoire .ssh de l'utilisateur. Si plusieurs fichiers de clés sont disponibles, GlassFish Server utilise l'ordre de préférence suivant :

  1. id_rsa

  2. id_dsa

  3. identity
