Démarrer un cluster

Le démarrage d'un cluster démarre toutes les instances de GlassFish Server du cluster qui ne sont pas déjà en cours d'exécution.

Avant de commencer

Assurez-vous que les conditions suivantes sont remplies :

  • Tous les nœuds où résident les instances dans le cluster sont activés pour la communication à distance.

  • L'utilisateur du serveur d'administration de domaine (DAS) peut utiliser le protocole SSH (shell sécurisé) pour se connecter à l'hôte pour n'importe quel nœud dans lequel résident les instances dans le cluster.

Si l'une de ces conditions n'est pas remplie, démarrez le cluster en démarrant chaque instance séparément à partir de la ligne de commande à l'aide de la sous-commande start-local-instance(1).

  1. Dans l’arborescence de navigation, sélectionnez le nœud Clusters.

    La page Clusters s’ouvre.

  2. Sur la page Clusters, cochez la case située en regard du nom du cluster.
  3. Cliquez sur Démarrer le cluster.

    Vous êtes invité à confirmer que vous souhaitez démarrer le cluster.

  4. Confirmez le démarrage du cluster.

    Le cluster est démarré. Le statut de chaque instance du cluster est mis à jour sur la page Clusters au démarrage de l'instance.
