Tester l'accessibilité d'un noeud

Pour créer ou démarrer des instances de GlassFish Server sur un noeud représentant un hôte distant, le serveur DAS doit être capable de contacter le noeud. Avant de créer ou de démarrer des instances sur un noeud de ce type, vérifiez que le serveur DAS peut contacter le noeud.

Le type de noeud que vous testez doit être DCOM ou SSH. Vous ne pouvez pas tester un noeud de type CONFIG car les noeuds de ce type ne sont pas activés pour la communication à distance.

Avant de commencer

Assurez-vous que le protocole approprié pour le type de noeud est configuré sur l'hôte représenté par le noeud. Le noeud est activé pour la communication à distance via ce protocole.

  • Si le type du noeud est DCOM, le protocole DCOM (Distributed Component Object Model) doit être configuré.

  • Si le type du noeud est SSH, le protocole SSH (shell sécurisé) doit être configuré.

    1. Dans l'arborescence de navigation, sélectionnez le noeud Noeuds.

      La page Noeuds apparaît. Cette page répertorie tous les noeuds du domaine en cours.

    2. Dans la table des noeuds, cliquez sur le lien Ping du noeud à tester.

      Un message indiquant si le serveur DAS a pu contacter le noeud est ajouté à la page Noeuds.
