Créer un cluster

Un cluster est une collection nommée d’instances de GlassFish Server qui partagent les mêmes applications, ressources et données de configuration.

Un cluster fournit une haute disponibilité grâce à son évolutivité, à l'équilibrage de charge et à une protection contre les pannes. Lorsqu'une instance ou un hôte d'un cluster est en panne, GlassFish Server détecte la défaillance, redirige les requêtes de l'instance en échec vers d'autres instances du cluster et rétablit l'état de la session utilisateur. Dans la mesure où les mêmes applications et ressources se trouvent sur toutes les instances du cluster, une instance peut basculer vers n'importe quelle autre instance du cluster.

GlassFish Server vous permet d'administrer toutes les instances d'un cluster comme une unité unique à partir d'un hôte unique, que les instances résident sur le même hôte ou sur des hôtes différents.

Avant de commencer

Si vous avez l'intention d'ajouter des instances GlassFish Server au cluster que vous avez créé, assurez-vous que les nœuds sur lesquels les instances résideront existent déjà.

  1. Dans l’arborescence de navigation, sélectionnez le nœud Clusters.

    La page Clusters s’ouvre.

  2. Cliquez sur le bouton Nouveau de la page Clusters.

    La page Nouveau cluster s’ouvre.

  3. Dans le champ Nom du cluster, attribuez un nom au cluster.

    Le nom doit respecter les exigences suivantes :

    • Le nom ne peut contenir que des caractères ASCII.

    • Le nom doit commencer par une lettre, un chiffre ou un caractère de soulignement.

    • Le nom ne peut contenir que les caractères suivants :

      • Lettres minuscules

      • Lettres majuscules

      • Chiffres

      • Trait d'union

      • Point

      • Caractère de soulignement

    • Le nom du nœud doit être unique dans le domaine et ne doit pas être le nom d'un autre cluster, d'une configuration nommée, d'une instance de GlassFish Server ou d'un nœud.

    • Le nom ne doit pas être domain, server ou tout autre mot-clé réservé par GlassFish Server.

  4. Dans la liste déroulante Configuration, sélectionnez la configuration nommée que le cluster doit utiliser.

    Remarque - Si vous sélectionnez default-config, vous devez spécifier dans l'étape suivante que la configuration n'est pas partagée. La configuration default-config ne peut pas être partagée.

  5. Indiquez si la configuration que le cluster utilisera est partagée.
    • Si la configuration n'est pas partagée, laissez l'option Créer une copie de la configuration sélectionnée cochée.

      La configuration sélectionnée est copiée lors de la création du cluster. Le nom cluster-name-config est affecté à la copie de la configuration, cluster-name étant le nom du cluster que vous créez.

    • Si la configuration est partagée, sélectionnez l'option Utiliser la configuration sélectionnée comme référence.

      Le cluster utilisera la configuration nommée existante indiquée.

  6. (Facultatif) Ajoutez les instances GlassFish Server au cluster.

    Remarque - Pour créer une instance sur un nœud de type CONFIG représentant un hôte distant, vous devez utiliser la sous-commande create-local-instance(1). Vous ne pouvez pas utiliser la Console d'administration à cette fin.

    Vous pouvez également ajouter des instances une fois le cluster créé.

    Ajoutez chaque instance comme suit :

    1. Dans la zone Instances de serveur à créer, cliquez sur Ajouter.

      Une ligne est ajoutée à la table des instances du cluster.

    2. Dans le champ Nom d’instance, entrez un nom d’instance.
    3. Dans le champ Pondération, entrez un entier qui représente la pondération de l'équilibrage de charge de l’instance.

      La pondération de l'équilibrage de charge détermine la proportion de toutes les requêtes pour le cluster que l'instance doit traiter.

      Par exemple, dans un cluster comportant deux instances, vous pouvez exiger une instance pour le traitement de l'une des quatre requêtes, et une autre instance pour le traitement des trois requêtes restantes. Dans ce cas, définissez la pondération de l'instance qui doit traiter une requête sur quatre sur 1 et celle de l'autre sur 3. Si vous préférez utiliser des pourcentages, définissez la pondération des instances sur 25 et 75, respectivement.

      La pondération par défaut est de 100.

    4. À partir de la liste déroulante Nœud, sélectionnez le nœud dans lequel l'instance doit résider.
  7. Cliquez sur OK.

    Vous êtes renvoyé à la page Clusters. Le cluster est ajouté au tableau des clusters sur la page.
