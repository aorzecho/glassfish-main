Démarrer des instances de GlassFish Server incluses dans un cluster

Avant de commencer

Assurez-vous que les prérequis suivants sont respectés :

  • Tous les noeuds sur lesquels résident les instances sont activés pour la communication à distance.

  • L'utilisateur du serveur d'administration de domaine (DAS) peut utiliser le protocole SSH (shell sécurisé) pour se connecter à l'hôte pour n'importe quel noeud dans lequel résident les instances.

Si l'un de ces prérequis n'est pas respecté, démarrez chaque instance séparément à partir de la ligne de commande à l'aide de la sous-commande start-local-instance(1).

  1. Dans l'arborescence de navigation, développez le noeud Clusters.

  2. Sous le noeud Clusters développé, sélectionnez le noeud correspondant au cluster.

    La page Informations générales du cluster apparaît.

  3. Dans la page Informations générales, cliquez sur l'onglet Instances.

    La page Instances de serveur clusterisées du cluster apparaît.

  4. Pour chaque instance à démarrer, cochez la case en regard de l'instance dans la table Instances de serveur.

  5. Cliquez sur Démarrer.

    Vous êtes invité à confirmer que vous souhaitez démarrer les instances.

  6. Confirmez le démarrage des instances.

    Le statut de chaque instance est mis à jour dans la table Instances de serveur au démarrage de l'instance.
