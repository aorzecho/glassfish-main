新建独立服务器实例

使用“新建独立服务器实例”页面可创建独立的 GlassFish Server 实例。

“新建独立服务器实例”包含以下选项。

实例名称

要创建的实例的名称。

名称必须满足以下要求：

  • 名称开头必须是字母、数字或下划线。

  • 名称只能包含以下字符：

    • 小写字母

    • 大写字母

    • 数字

    • 连字符

    • 句点

    • 下划线

  • 名称在域中必须是唯一的，不得是另一个 GlassFish Server 实例、群集、命名配置或节点的名称。

  • 名称不得是域、服务器或 GlassFish Server 保留的任何其他关键字。

节点

现有节点的下拉式列表，定义实例可驻留的主机。必须从列表中选择一个节点。如果要在运行域管理服务器 (domain administration server, DAS) 的主机上创建实例，请选择预定义节点 localhost-domain。

配置

实例可使用的现有命名配置的下拉式列表。必须从该列表中选择一个配置。创建实例时，实例将使用选定的配置。

复制选定的配置

如果选择此选项，则在创建实例时将复制选定的配置。会为该配置的副本指定一个名称 instance-name-config，其中 instance-name 是在“实例名称”字段中键入的实例名称。

如果从“配置”下拉式列表中选择了 default-config，则必须选择此选项。default-config 配置只能被复制而不能被引用。

引用选定的配置

如果选择此选项，实例将使用指定的现有命名配置。

如果从“配置”下拉式列表中选择了 default-config，则不能选择此选项。default-config 配置只能被复制而不能被引用。
