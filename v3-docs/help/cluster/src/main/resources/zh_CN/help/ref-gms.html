编辑组管理服务

使用“组管理服务”页面可更改 GMS 设置。

组管理服务 (Group Management Service, GMS) 是一种进程内服务，可提供群集监视、群集成员和组通信服务。GMS 构建于可配置的组成员搜索和运行状况监视协议堆栈之上。GMS 执行以下功能：

  • 在群集中添加或删除一个或多个成员实例，或者怀疑或确认一个或多个成员实例已失败时，通知 GlassFish Server 实例中的已注册模块。

  • 提供在一组进程之间发送和接收消息的功能。

“组管理服务”页面包含以下选项。

装入默认值

使具有默认值的设置恢复其默认值的按钮。不具有默认值的设置不会发生更改。

配置名称

要应用该页面上设置的配置名称。此字段为只读。

最大丢失心跳数

监视尝试次数，尝试此次数之后，将怀疑服务器实例出现故障。默认值为 3。

心跳频率

两次监视尝试之间的时间段。默认值为 2000 毫秒。

组搜索超时

GMS 等待搜索其他群集成员的时间。默认值为 5000 毫秒。

故障确认等待时间

一段时间，在此时间之后，将把可疑故障标记为已确认。默认值为 1500 毫秒。

其他属性

“组管理服务”的其他属性表。

GlassFish Server 不定义任何其他 GMS 属性。

对于每个属性，都将显示以下信息：

名称

属性名称。

值

为选定的实例设置的属性的值。

说明

提供有关属性的详细信息的文本说明。

“其他属性”表还包含以下选项。

添加属性

用于添加属性的按钮。单击此按钮可向“其他属性”表中添加一行。

删除属性

用于删除一个或多个选定属性的按钮。删除的任何属性会恢复其默认值，或者如果没有设置默认值，则未定义。
