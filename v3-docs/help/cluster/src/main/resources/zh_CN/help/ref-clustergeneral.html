常规信息

“常规信息”页面包含以下选项。

启动群集

用于启动群集的按钮。如果群集中的所有实例都在运行，则会取消激活此按钮。

停止群集

用于停止群集的按钮。如果群集中的所有实例都已停止，则会取消激活此按钮。

迁移 EJB 计时器

用于将 Enterprise JavaBeans (EJB) 技术计时器从已停止的 GlassFish Server 实例迁移到群集中正在运行的实例的按钮。单击此按钮可打开迁移 EJB 计时器页面。如果群集中的所有实例都在运行，则会取消激活此按钮。

轮转日志

用于轮转群集的日志文件的按钮。轮转日志文件时，会以 server.log_ date-and-time 格式用时间戳名称重命名该文件，并为新的日志消息创建一个空的日志文件。更改会动态应用。因此，无需重新启动服务器。

群集名称

创建群集时为其指定的名称。此字段为只读。

JVM

如果单击“JVM 报告”，则会打开一个单独的窗口，此窗口显示有关 Java 虚拟机的报告，包括摘要报告、内存管理和垃圾收集报告、类装入报告和当前线程转储。

配置

创建群集时为其选择的配置。单击配置将打开群集的配置页面。此字段为只读。

GMS

如果启用此选项，将在群集以及域管理服务器 (domain administration server, DAS) 的每个 GlassFish Server 实例中启动组管理服务 (Group Management Service, GMS)。DAS 会参与已为其启用了 GMS 的每个群集。GMS 提供群集监视服务、群集成员服务和组通信服务。默认情况下启用此选项。

多播端口

GMS 在其上侦听组事件的通信端口的端口号。此选项必须指定一个在 2048–32000 范围内的有效端口号。默认值是在此范围内自动生成的一个值。。

多播地址

GMS 侦听组事件的地址。此选项必须指定一个在 224.0.0.0 至 239.255.255.255 范围内的多播地址。默认值为 228.9.XX.YY，其中 XX 和 YY 是自动生成的 0 到 255 之间的独立值。

绑定接口地址

GMS 绑定到的网络接口的 Internet 协议 (Internet Protocol, IP) 地址。此选项必须指定本地网络接口的 IP 地址。默认值是所有公共网络接口地址。

在多宿主计算机上，此选项配置用于 GMS 的网络接口。多宿主计算机可处理两个或多个网络接口。

要指定一个对群集中所有 GlassFish Server 实例都有效的地址，请使用系统属性为每个实例分别设置地址。

例如，使用群集系统属性页面创建 GMS-BIND-INTERFACE-ADDRESS-cluster-name 系统属性。然后，将此页面中的“绑定地址”选项设置为 ${GMS-BIND-INTERFACE-ADDRESS- cluster-name} 以指定系统属性。最后，对于群集中的每个实例，将 GMS-BIND-INTERFACE-ADDRESS- cluster-name 系统属性设置为实例计算机上所需的网络接口地址。

状态

标明群集中运行以及未运行的实例数量。此字段为只读。
