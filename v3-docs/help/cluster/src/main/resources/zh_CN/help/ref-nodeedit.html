编辑节点

使用“编辑节点”页面可更新 GlassFish Server 节点的配置数据。还可以使用此页面将节点的类型从 CONFIG 更改为 SSH，以便为节点启用 SSH 通信。

如果将节点的类型从 CONFIG 更改为 SSH，则在以下任意字段留空时将应用默认值：

  • SSH 端口

  • 用户名

  • 密钥文件

“编辑节点”页面包含以下选项：

名称

创建节点时为其指定的名称。此字段为只读。

类型

用于指定节点类型的下拉式列表，节点类型可以为以下类型之一：

SSH

节点支持通过安全 shell (secure shell, SSH) 进行通信。

CONFIG

节点不支持远程通信。如果选择 CONFIG，则不会显示 SSH 配置的任何选项。如果在节点的当前类型是 SSH 时选择 CONFIG，则在保存更改时，将从节点的配置中删除 SSH 配置数据。

节点主机

节点更新后所代表的主机的名称。该字段是否是必需字段取决于节点类型：

  • 如果节点类型为 CONFIG，可以省略主机名。可在创建驻留在节点上的实例时确定该主机名。

  • 如果节点类型为 SSH，必须指定主机名。否则，会出现错误。

节点目录

用于包含节点上创建的 GlassFish Server 实例的目录路径。如果指定相对路径，则该路径相对于 as-install 目录，其中 as-install 是主机上 GlassFish Server 软件的基本安装目录。

如果该字段留空，则按如下所示更新节点的配置：

  • 如果节点类型为 CONFIG，则在域管理服务器 (domain administration server, DAS) 配置中不会为节点指定任何实例目录。可在创建驻留在节点上的实例时确定节点目录。

  • 如果节点类型为 SSH，则节点目录为 as-install/nodes，其中 as-install 是主机上 GlassFish Server 软件的基本安装目录。

安装目录

主机上 GlassFish Server 软件的基本安装目录的父目录的完整路径，例如，/export/glassfish3/。

如果该字段留空，则按如下所示更新节点的配置：

  • 如果节点类型为 CONFIG，则在 DAS 配置中不会为节点指定任何安装目录。可在创建驻留在节点上的实例时确定安装目录。

  • 如果节点类型为 SSH，则安装目录是 DAS 中 GlassFish Server 软件的默认基本安装目录的父目录。仅当 GlassFish Server 安装在所有主机上的同一个位置时，此设置才有用。

强制

如果选择“已启用”选项，则即使节点参数验证失败，也会在域管理服务器 (domain administration server, DAS) 配置中更新节点。要验证节点的参数，DAS 必须能通过 SSH 联系节点的主机。

SSH 端口

此节点的主机进行 SSH 连接使用的端口。

如果将节点的类型从 CONFIG 更改为 SSH 并将此字段留空，“SSH 端口”选项将被设置为 22。

如果将“节点主机”选项设置为 localhost-domain，将忽略“SSH 端口”选项。

SSH 用户名

该节点主机上要运行通过 SSH 连接到主机的进程的用户。

如果将节点的类型从 CONFIG 更改为 SSH 并将此字段留空，“用户名”选项将被设置为运行 DAS 进程的用户。

如果将“节点主机”选项设置为 localhost-domain，将忽略“SSH 用户”选项。

SSH 用户验证

用于指定 SSH 用户登录到该节点主机时如何对其进行验证的下拉式列表。可能的验证方法如下：

密钥文件

SSH 用户通过 SSH 私钥文件进行验证。如果选择该选项，请在“密钥文件”字段中指定该文件的绝对路径。

密码

SSH 用户通过存储在 DAS 配置中的密码进行验证。

注意 - 此密码以明文形式存储在 DAS 配置中。为了增强安全性，请使用密钥文件或密码别名。

如果选择该选项，将激活“SSH 密码”字段和“确认 SSH 密码”字段，以便您提供密码。

密码别名

SSH 用户通过密码别名进行验证。如果选择该选项，请从“密码别名”下拉式列表中选择相应密码别名。

SSH 密码

SSH 用户登录到该节点主机时要使用的密码。

确认 SSH 密码

确认 SSH 用户登录到该节点主机时要使用的密码。该字段的内容必须与“SSH 密码”字段的内容完全一致。

密码别名

为该域定义的密码别名的下拉式列表。

密钥文件

“用户名”选项指定的用户的 SSH 专用密钥文件的绝对路径。此文件用于向节点的主机上的 sshd 守护进程进行验证。

注 - GlassFish Server 还支持通过密码文件中的 AS_ADMIN_SSHPASSWORD 条目进行密码验证。密码文件在 asadmin(1M) 实用程序的 --paswordfile 选项中指定。

如果 SSH 专用密钥文件受口令短语保护，则密码文件必须包含 AS_ADMIN_SSHKEYPASSPHRASE 条目。

密钥文件的路径必须是 DAS 可以访问的路径，密钥文件必须是 DAS 可读的文件。

默认值取决于是否将节点的类型从 CONFIG 更改为 SSH 以便为节点启用 SSH 通信：

如果将节点类型从 CONFIG 更改为 SSH 并将此字段留空，“密钥文件”选项将设置为用户的 .ssh 目录中的密钥文件。如果找到多个密钥文件，GlassFish Server 遵循以下先后顺序：

  1. id_rsa

  2. id_dsa

  3. identity
