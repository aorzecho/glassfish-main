为群集配置组管理服务设置

组管理服务 (Group Management Service, GMS) 是一种进程内服务，可提供群集监视、群集成员和组通信服务。GMS 执行以下功能：

  • 在群集中添加或删除一个或多个成员实例，或者怀疑或确认一个或多个成员实例已失败时，通知 GlassFish Server 实例中的已注册模块。

  • 提供在一组进程之间发送和接收消息的功能。

  1. 在导航树中，展开“群集”节点。
  2. 选择要为其配置组管理服务 (Group Management Service, GMS) 设置的群集。

    此时将打开群集的“常规信息”页面。

  3. 选中“已启用 GMS”复选框为群集启用 GMS。

    默认情况下，此选项已选中。要为群集禁用 GMS，则将复选标记从“已启用 GMS”复选框中删除。

    如果启用，则会在群集的每个服务器实例和域管理服务器 (Domain Administration Server, DAS) 中启动组管理服务。DAS 会参与到启用 GMS 的每个群集中。

  4. 在“多播端口”字段中，输入组管理服务用于侦听组事件的端口。

    指定 2048–32000 范围内的标准 UDP 端口号。默认值是在此范围内自动生成的一个值。。

  5. 在“多播地址”字段中，输入组管理服务用于侦听组事件的地址。

    指定 D 类 IP 多播地址。D 类 IP 地址在 224.0.0.0 和 239.255.255.255 的范围内（包括两者）。地址 224.0.0.0 被保留，不应使用。默认值为 228.9.XX.YY，其中 XX 和 YY 是自动生成的 0 到 255 之间的独立值。

  6. 如果有多个网络接口，在“绑定接口地址”字段中输入组管理服务的网络接口地址。

    如果在承载群集中服务器实例的任何计算机上有多个网络接口，此设置使您能够配置用于 GMS 的网络接口。该值必须是本地网络接口 IP 地址。默认情况下会使用所有公用网络接口地址。

    要指定一个对群集中所有 GlassFish Server 实例都有效的地址，请使用系统属性为每个实例分别设置地址。

    例如，在 DAS 上创建系统属性 GMS-BIND-INTERFACE-ADDRESS- cluster-name。然后将此“绑定接口地址选项”选项设置为 ${GMS-BIND-INTERFACE-ADDRESS- cluster-name} 以指定该系统属性。最后，对于群集中的每个实例，将 GMS-BIND-INTERFACE-ADDRESS- cluster-name 系统属性设置为实例计算机上所需的网络接口地址。

  7. （可选的）要指定其他属性，请选择“属性”选项卡和“群集属性”子选项卡。

    GlassFish Server 不定义任何其他 GMS 属性。

    • 要添加属性，请单击“添加属性”按钮。在显示的空白行中，在“名称”字段中键入属性名称，在“值”字段中键入属性值。
    • 要修改属性，请编辑该属性的“值”字段。
    • 要删除属性，请选中位于您要删除属性的“名称”字段左边的复选框，然后单击“删除属性”按钮。
  8. 单击“保存”。
