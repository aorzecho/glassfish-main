创建节点

节点代表装有 GlassFish Server 软件的主机。对于 GlassFish Server 实例驻留的每个主机，必须存在一个对应的节点。

  1. 在导航树中，选择“节点”节点。

    此时将打开“节点”页。

  2. 在“节点”页上，单击“新建”。

    此时将打开“新建节点”页。

  3. 在“名称”字段中，键入为该节点选择的名称。

  4. 从“类型”下拉列表中，选择要创建的节点的类型。

    • 要允许节点支持通过分布式组件对象模型 (DCOM) 远程协议进行通信，请选择“DCOM”。

    • 要创建支持通过安全 shell (SSH) 进行通信的节点，请选择“SSH”。

    • 要创建不支持远程通信的节点，请选择“CONFIG”。

  5. 在“节点主机”字段中，键入节点代表的主机的名称。

    该字段是否是必需字段取决于节点类型：

    • 如果节点类型为 CONFIG，可以省略主机名。可在创建驻留在节点上的实例时确定该主机名。

    • 如果节点类型为 DCOM 或 SSH，必须指定主机名。否则，会出现错误。

  6. 在“节点目录”字段中，键入要包含节点上创建的 GlassFish Server 实例的目录路径。

    如果节点类型为 CONFIG，请将该字段留空。可在创建驻留在节点上的实例时确定节点目录。

    如果键入相对路径，则该路径将相对于 as-install 目录，其中 as-install 为 GlassFish Server 软件在主机上的基安装目录。

  7. 在“安装目录”字段中，键入主机上 GlassFish Server 软件基安装目录的父目录完整路径，例如 /export/glassfish3。

    如果节点类型为 CONFIG，请将该字段留空。可在创建驻留在节点上的实例时确定安装目录。

  8. 按节点类型的要求完成创建节点。

    1. 如果节点的类型为 CONFIG，请单击“保存”。

    2. 如果节点的类型为 DCOM，请按照完成创建 DCOM 节点中的说明操作。

    3. 如果节点的类型为 SSH，请按照完成创建 SSH 节点中的说明操作。
