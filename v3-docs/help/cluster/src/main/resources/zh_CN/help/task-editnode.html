编辑节点

编辑节点会更新 GlassFish Server 节点的配置数据。通过编辑节点，还可以将节点类型从 CONFIG 更改为 SSH，以便为节点启用 SSH 通信。

如果将节点的类型从 CONFIG 更改为 SSH，则在以下任意字段留空时将应用默认值：

  • SSH 端口

  • SSH 用户名

  • 密钥文件

  1. 在导航树中，展开“节点”节点。
  2. 从节点列表中，选择要编辑的节点。

    此时将打开“编辑节点”页面。

  3. 从“类型”下拉式列表中，选择已更新节点的类型。
    • 要允许节点支持通过安全 shell (secure shell, SSH) 进行通信，请选择 "SSH"。
    • 要从节点中删除对远程配置的支持，请选择 "CONFIG"。

      如果选择 "CONFIG"，则不会显示 SSH 配置的选项。如果在节点的当前类型是 SSH 时选择 CONFIG，则在保存更改时，将从节点的配置中删除 SSH 配置数据。

  4. 在“节点主机”字段中，键入已更新节点要代表的主机的名称。

    该字段是否是必需字段取决于节点类型：

    • 如果节点类型为 CONFIG，可以省略主机名。可在创建驻留在节点上的实例时确定该主机名。

    • 如果节点类型为 SSH，必须指定主机名。否则，会出现错误。

  5. 在“节点目录”字段中，键入要包含节点上创建的 GlassFish Server 实例的目录路径。

    如果键入一个相对路径，则该路径相对于 as-install 目录，其中 as-install 是主机上 GlassFish Server 软件的基本安装目录。

  6. 在“安装目录”字段中，键入主机上 GlassFish Server 软件基本安装目录父项的完整路径，例如 /export/glassfish3/。
  7. 要强制将节点创建在域管理服务器 (domain administration server, DAS) 的配置中（即使节点的参数验证失败），请启用“强制”选项。
  8. 在“SSH 端口”字段中，键入该端口，以用于与该节点主机的 SSH 连接。

    如果将节点的类型从 CONFIG 更改为 SSH 并将此字段留空，“SSH 端口”选项将被设置为 22。

  9. 在“SSH 用户名”字段中，键入该节点主机上要运行通过 SSH 连接到主机的进程的用户名称。

    如果将节点类型从 CONFIG 更改为 SSH 并将此字段留空，“SSH 用户名”选项将设置为运行 DAS 进程的用户。

  10. 指定 SSH 用户登录到该节点主机时如何对其进行验证。
    • 要指定通过 SSH 私钥文件进行验证，请执行以下步骤：
      1. 从“SSH 用户验证”下拉式列表中，选择“密钥文件”。
      2. 在“密钥文件”字段中，为“SSH 用户名”选项指定的用户键入 SSH 私钥文件的绝对路径。

        注 - GlassFish Server 还支持通过密码文件中的 AS_ADMIN_SSHPASSWORD 条目进行密码验证。密码文件在 asadmin(1M) 实用程序的 --paswordfile 选项中指定。

        如果将节点类型从 CONFIG 更改为 SSH 并将此字段留空，“密钥文件”选项将设置为用户的 .ssh 目录中的密钥文件。如果找到多个密钥文件，GlassFish Server 遵循以下先后顺序：

        • id_rsa

        • id_dsa

        • identity

    • 要指定通过存储在 DAS 配置中的密码进行验证，请执行以下步骤：

      注意 - 此密码以明文形式存储在 DAS 配置中。为了增强安全性，请使用密钥文件或密码别名。

      1. 从“SSH 用户验证”下拉式列表中，选择“密码”。
      2. 在“SSH 密码”字段中，键入 SSH 用户登录到该节点主机时要使用的密码。
      3. 在“确认 SSH 密码”字段中，再次键入该密码。
    • 要指定通过密码别名进行验证，请执行以下步骤：
      1. 从“SSH 用户验证”下拉式列表中，选择“密码别名”。
      2. 从“密码别名”下拉式列表中，选择一个现有别名。
  11. 单击“保存”。
