完成更新 DCOM 节点

此任务是 编辑节点 的 DCOM 节点的延续。

  1. 要强制将节点创建在域管理服务器 (DAS) 的配置中 (即使节点的参数验证失败)，请启用“强制”选项。

  2. 如果需要 GlassFish Server 在更新节点之前测试通过 DCOM 与节点主机的连接，请执行以下步骤：

    1. 启用“验证 DCOM”选项。

    2. 在“远程测试目录”字段中，键入远程主机上 GlassFish Server 应在其中创建用于测试连接的文件的文件夹名称。

  3. 在“Windows 域”字段中，键入包含“Windows 用户名”选项指定的用户的 Windows 域的名称。

    如果将节点类型更改为 DCOM 并将此字段留空，则会将“Windows 域”选项设置为运行管理控制台的主机的名称。

  4. 在“Windows 用户名”字段中，键入此节点的主机上要运行通过 DCOM 连接到主机的进程的用户名称。

    如果将节点类型从 CONFIG 更改为 DCOM 并将此字段留空，则会将“Windows 用户名”选项设置为运行 DAS 进程的用户。

  5. 指定在 Windows 用户登录到此节点的主机时如何对其进行验证。

    注:

    GlassFish Server 还支持通过口令文件中的 AS_ADMIN_WINDOWSPASSWORD 条目进行口令验证。口令文件在 asadmin(1) 实用程序的 --paswordfile 选项中指定。

    1. 要指定通过存储在 DAS 配置中的口令进行验证，请执行以下步骤：

      注意:

      此口令以明文形式存储在 DAS 配置中。为了增强安全性，请使用口令别名。

      1. 从“Windows 用户验证”下拉列表中，选择“口令”。

      2. 在“Windows 用户口令”字段中，键入 Windows 用户登录到此节点的主机时要使用的口令。

      3. 在“确认口令”字段中，再次键入该口令。

    2. 要指定通过口令别名进行验证，请执行以下步骤：

      1. 从“Windows 用户验证”下拉列表中，选择“口令别名”。

      2. 从“口令别名”下拉列表中，选择一个现有别名。

  6. 单击“保存”。
