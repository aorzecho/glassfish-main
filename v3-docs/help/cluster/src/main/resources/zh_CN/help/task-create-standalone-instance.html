创建独立 GlassFish Server 实例

GlassFish Server 实例构成了应用程序部署的基础。独立实例不是集群的成员。每个实例均属于一个域，并有自己的目录结构、配置和已部署的应用程序。每个实例还包括 Java Platform, Enterprise Edition (Java EE) Web 容器和 Enterprise JavaBeans 技术 (EJB 技术) 容器。每个新实例必须包含对节点 (该节点定义实例将驻留的主机) 的引用。

注:

要在表示远程主机的类型为 CONFIG 的节点上创建实例，必须使用 create-local-instance(1) 子命令。不能使用管理控制台来实现此目的。
开始之前

确保实例要驻留的节点存在。有关说明，请参阅创建节点。

  1. 在导航树中，选择“独立实例”节点。

    将打开“独立服务器实例”页。

  2. 在“独立服务器实例”页上，单击“新建”。

    此时将打开“新建独立服务器实例”页。

  3. 在“名称”字段中，键入实例的名称。

    名称必须满足以下要求：

    • 名称中仅可以包含 ASCII 字符。

    • 名称开头必须是字母、数字或下划线。

    • 名称只能包含以下字符：

      • 小写字母

      • 大写字母

      • 数字

      • 连字符

      • 句点

      • 下划线

    • 名称在域中必须是唯一的，不得是另一个 GlassFish Server 实例、集群、命名配置或节点的名称。

    • 名称不得是 domain、server 或 GlassFish Server 保留的任何其他关键字。

  4. 从“节点”下拉列表中，选择实例将驻留的节点。

  5. 从“配置”下拉列表中，选择实例将使用的命名配置。

    注:

    如果选择 default-config，则必须在下一步中指定不共享配置。不能共享 default-config 配置。

  6. 指定是否共享实例将使用的配置。

    • 如果不共享配置，请保留“复制所选配置”选项的选中状态。

      创建实例时会复制所选配置。会为该配置的副本指定一个名称 instance-name-config，其中 instance-name 是要创建的实例的名称。

    • 如果共享配置，请选择“引用所选配置”选项。

      该实例将使用指定的现有命名配置。

  7. 单击“确定”。
