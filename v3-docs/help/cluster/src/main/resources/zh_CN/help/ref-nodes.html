节点

“节点”页包含域中节点的列表。对于每个节点，都将显示以下信息：

名称

创建节点时为其指定的名称。单击该名称将打开节点的“编辑节点”页。

节点主机

节点代表的主机的名称。

类型

节点的类型，可以为以下类型之一：

CONFIG

节点不支持远程通信。

DCOM

节点支持通过分布式组件对象模型 (DCOM) 远程协议进行通信。DCOM 协议仅在 Windows 系统上可用。

SSH

节点支持通过安全 shell (SSH) 进行通信。

实例

驻留在节点上的实例的列表，其中标明了每个实例是否正在运行：

  • 如果实例正在运行，将显示一个复选标记 (✓)。

  • 如果实例已停止，将显示一个感叹号 (!)。

单击某个实例的名称可打开该实例的“一般信息”页。

操作

仅限类型为 DCOM 和 SSH 的节点：节点的试通操作的链接。单击试通链接将测试是否可以通过节点支持的通信协议到达该节点。

无法对 CONFIG 节点执行任何操作。

“节点”表还包含以下选项：

新建

用于创建节点的按钮。

删除

用于删除一个或多个所选节点的按钮。

删除和卸载

用于删除一个或多个所选节点并从每个已删除节点代表的主机卸载 GlassFish Server 软件的按钮。
