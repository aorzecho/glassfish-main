修改群集 GlassFish Server 实例的端口号

群集 GlassFish Server 实例的端口号最初是在实例从其父群集继承的配置中设置的。例如，如果两个群集实例驻留在同一主机上，则必须修改这些端口号。在这种情况下，必须修改一个实例的端口号，因为这两个实例共享同一配置。

实例的端口号存储为 Java 系统属性。启动 GlassFish Server 时，这些属性会通过 Java 应用程序启动器的 -D 选项传递给 Java 应用程序启动器。

  1. 在导航树中，展开“群集”节点。
  2. 在展开的“群集”节点下，选择某个群集的节点。

    此时将打开群集的“常规信息”页面。

  3. 在“常规信息”页面中，单击“实例”选项卡。

    此时将打开群集的“群集服务器实例”页面。

  4. 选择要修改端口号的实例。

    此时将打开实例的“常规信息”页面。

  5. 在“常规信息”页面中，单击“属性”选项卡。

    此时将打开“实例系统属性”页面。

    此页面会显示实例的以下系统属性的当前值：

    • 实例的配置中预定义的端口号

    • 为实例定义的自定义属性（如果有）

  6. 对于要修改的每个现有属性，在该属性的“覆盖值”字段中键入新的值。
  7. 对于其他属性，可根据需要设置相应的值。
    1. 在“其他属性”表中，单击“添加属性”。

      会向“其他属性”表中添加一行。

    2. 在“实例变量名称”字段中，键入属性名称。
    3. 在“覆盖值”字段中，键入值。
  8. 单击“保存”。
