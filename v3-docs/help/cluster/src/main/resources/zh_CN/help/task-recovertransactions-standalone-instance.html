恢复独立 GlassFish Server 实例的事务

  1. 在导航树中，选择“独立实例”节点。

    将打开“独立服务器实例”页面。

  2. 选择需要恢复事务的实例。

    此时将打开实例的“常规信息”页面。

  3. 在“常规信息”页面中，单击“恢复事务”。

    此时将打开“恢复事务”页面。

    该页面上的具体内容取决于实例是否在运行。

    • 如果实例正在运行，“恢复事务”页面仅包含文本恢复由运行的服务器本身来执行以及“确定”按钮。

    • 如果实例未在运行，“恢复事务”页面将包含其他选项。

  4. 从“目标服务器”下拉式列表中，选择可以恢复当前实例上事务的另一个实例。

    对于独立实例，请选择名为 server 的实例，该实例表示域管理服务器 (domain administration server, DAS)。

  5. 在“事务日志目录”字段中，键入存储事务日志的目录。

    默认目录是域目录下的 /logs/tx。事务日志保存在“事务日志位置”设置（位于群集配置的“事务服务”中）指定的目录的 tx 子目录中。“事务日志位置”的默认值为“域属性”页面（位于 GlassFish Server 节点的“高级”选项卡下）的“日志根目录”字段指定的目录。

  6. 单击“确定”。
