移植 EJB 计时器

如果 GlassFish Server 实例停止，该实例上安装的企业 Java Bean (EJB) 技术计时器也会停止运行。如果实例异常停止或意外停止，要使这些计时器能够在实例停止时运行，必须将这些计时器移植到集群中正在运行的服务器实例。

  1. 在导航树中，展开“集群”节点。

  2. 在导航树中的“集群”节点下，选择集群。

    此时将打开集群的“一般信息”页。

  3. 在“一般信息”页中，单击“移植 EJB 计时器”。

    此时将打开“移植 EJB 计时器”页。

    注:

    如果集群中的所有实例都在运行，则“移植 EJB 计时器”按钮处于取消激活状态。

    为帮助您移植计时器，“移植 EJB 计时器”页包含一个提供以下信息的 EJB 计时器表：

    服务器名称

    集群中 GlassFish Server 实例的列表。

    计时器计数

    每个实例上持久性计时器的数量。

  4. 在“移植 EJB 计时器”页上，选择要移植的计时器所在的实例和要将计时器移植到的实例。

    1. 从“源”下拉列表中，选择要移植的计时器所在的已停止实例。

    2. 从“目标”下拉列表中，选择要将计时器移植到的正在运行的实例。

    3. 单击“确定”。

  5. 停止并重新启动 EJB 计时器移植到的实例。

    如果源实例正在运行或目标实例未运行，则管理控制台将显示一条错误消息。
