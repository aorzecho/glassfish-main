迁移 EJB 计时器

如果 GlassFish Server 实例停止，该实例上安装的 Enterprise JavaBeans (EJB) 技术计时器也会停止运行。如果实例异常停止或意外停止，要使这些计时器能够在实例停止时运行，必须将这些计时器迁移到群集中正在运行的服务器实例。

  1. 在导航树中，展开“群集”节点。
  2. 在导航树中的“群集”节点下，选择群集。

    此时将打开群集的“常规信息”页面。

  3. 在“常规信息”页面中，单击“迁移 EJB 计时器”。

    此时将打开“迁移 EJB 计时器”页面。

    注 - 如果群集中的所有实例都在运行，则“迁移 EJB 计时器”按钮处于取消激活状态。

  4. 在“迁移 EJB 计时器”页面上，选择要迁移的计时器所在的实例和要将计时器迁移到的实例。
    1. 从“源”下拉式列表中，选择要迁移的计时器所在的已停止实例。
    2. （可选的）从“目标”下拉式列表中，选择要将计时器迁移到的正在运行的实例。
    3. 单击“确定”。
  5. 停止并重新启动 EJB 计时器迁移到的实例。

    如果源实例正在运行或目标实例未运行，则管理控制台将显示一条错误消息。
