群集属性

“群集属性”页面显示为所选群集设置的属性的列表。这些属性添加了有关群集的可选配置信息。

“群集名称”字段是只读字段，其中显示所选群集的名称。

对于每个属性，都将显示以下信息：

名称

属性名称。

值

为选定的实例设置的属性的值。

说明

提供有关属性的详细信息的文本说明。

“其他属性”表还包含以下选项。

添加属性

用于添加属性的按钮。单击此按钮可向“其他属性”表中添加一行。

删除属性

用于删除一个或多个选定属性的按钮。删除的任何属性会恢复其默认值，或者如果没有设置默认值，则未定义。

GlassFish Server 定义了以下群集属性：

GMS_LISTENER_PORT

群集用于侦听来自组管理服务 (Group Management Service, GMS) 消息的端口的端口号。默认值在大多数情况下可满足要求。

GMS_LOOPBACK

指定实例是否可以接收其广播到群集的应用级消息。

可能的值如下：

false

实例不能接收自己发送的消息（默认）。

true

实例可以接收自己发送的消息。当实例是群集中的唯一实例时使用此设置对该实例进行测试。

GMS_MULTICAST_TIME_TO_LIVE

以下类型事件的多播消息在被丢弃之前可以尝试的最大迭代数或传输数：

  • 组搜索

  • 成员心跳

  • 成员关系更改

要匹配在其上部署 DAS 和群集实例的网络配置，可将此值设置的尽可能小些。要确定系统的最小可能值，可使用 validate-multicast(1) 子命令。

值 0 确保多播消息从不离开广播它们的主机。

值 1 会阻止在同一子网上通过交换机或路由器连接的主机之间广播消息。

默认值为 4，将确保消息成功广播到通过交换机或路由器连接主机的网络中的所有群集成员。
