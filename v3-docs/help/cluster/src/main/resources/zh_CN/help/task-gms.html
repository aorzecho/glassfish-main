配置组管理服务

组管理服务 (Group Management Service, GMS) 是一种进程内服务，可提供群集监视、群集成员和组通信服务。GMS 执行以下功能：

  • 在群集中添加或删除一个或多个成员实例，或者怀疑或确认一个或多个成员实例已失败时，通知 GlassFish Server 实例中的已注册模块。

  • 提供在一组进程之间发送和接收消息的功能。

  1. 在导航树中，展开“配置”节点。
  2. 选择要更改的群集配置。
  3. 在配置的节点下，选择“组管理服务”节点。

    此时将打开“组管理服务”页面。

  4. 在“最大丢失心跳数”字段中，输入 GMS 确认出现可疑故障之前尝试监视的最多次数。

    默认值为 3。

  5. 在“心跳频率”字段中，输入两次监视尝试之间的时间段。

    默认值为 2000 毫秒。

  6. 在“组搜索超时”字段中，输入 GMS 等待搜索其他群集成员的时间。

    默认值为 5000 毫秒。

  7. 在“故障确认等待时间”字段，输入将可疑故障标记为已确认需等待的时间。

    默认值为 1500 毫秒。

  8. 要恢复默认值，请单击“装入默认值”按钮。
  9. （可选的）在“其他属性”部分中，指定其他属性。

    GlassFish Server 不定义任何其他 GMS 属性。

    • 要添加属性，请单击“添加属性”按钮。在显示的空白行中，在“名称”字段中键入属性名称，在“值”字段中键入属性值。
    • 要修改属性，请编辑该属性的“值”字段。
    • 要删除属性，请选中位于您要删除属性的“名称”字段左边的复选框，然后单击“删除属性”按钮。
  10. 单击“保存”。
