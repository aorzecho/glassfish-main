将 GlassFish Server 实例添加到群集

GlassFish Server 实例构成了应用程序部署的基础。群集实例是群集的成员。每个实例均属于一个域，并有自己的目录结构、配置和已部署的应用程序。每个实例也都包括 Java Platform, Enterprise Edition (Java EE) Web 容器和 Enterprise JavaBeans 技术（EJB 技术）容器。每个新实例必须包含对节点（该节点定义实例将驻留的主机）的引用。

注 - 要在表示远程主机的类型为 CONFIG 的节点上创建实例，必须使用 create-local-instance(1) 子命令。不能使用管理控制台来实现此目的。

开始之前

确保以下项存在：

  • 要将实例添加到的群集

  • 实例要驻留的节点

  1. 在导航树中，展开“群集”节点。
  2. 在展开的“群集”节点下，选择某个群集的节点。

    此时将打开群集的“常规信息”页面。

  3. 在“常规信息”页面中，单击“实例”选项卡。

    此时将打开群集的“群集服务器实例”页面。

  4. 在“群集服务器实例”页面上，单击“新建”。

    此时将打开“新建群集服务器实例”页面。

  5. 在“名称”字段中，键入实例的名称。

    名称必须满足以下要求：

    • 名称仅可以包含 ASCII 字符。

    • 名称开头必须是字母、数字或下划线。

    • 名称只能包含以下字符：

      • 小写字母

      • 大写字母

      • 数字

      • 连字符

      • 句点

      • 下划线

    • 名称在域中必须是唯一的，不得是另一个 GlassFish Server 实例、群集、命名配置或节点的名称。

    • 名称不得是域、服务器或 GlassFish Server 保留的任何其他关键字。

  6. 从“节点”下拉式列表中，选择实例将驻留的节点。
  7. 单击“确定”。
