向集群中添加 GlassFish Server 实例

GlassFish Server 实例构成了应用程序部署的基础。集群实例是集群的成员。每个实例均属于一个域，并有自己的目录结构、配置和已部署的应用程序。每个实例还包括 Java Platform, Enterprise Edition (Java EE) Web 容器和 Enterprise JavaBeans 技术 (EJB 技术) 容器。每个新实例必须包含对节点 (该节点定义实例将驻留的主机) 的引用。

注:

要在表示远程主机的类型为 CONFIG 的节点上创建实例，必须使用 create-local-instance(1) 子命令。不能使用管理控制台来实现此目的。
开始之前

确保以下项存在：

  • 要将实例添加到的集群。有关说明，请参阅创建集群。

  • 实例要驻留的节点。有关说明，请参阅创建节点。

  1. 在导航树中，展开“集群”节点。

  2. 在展开的“集群”节点下，选择某个集群的节点。

    此时将打开集群的“一般信息”页。

  3. 在“一般信息”页中，单击“实例”选项卡。

    此时将打开集群的“集群服务器实例”页。

  4. 在“集群服务器实例”页上，单击“新建”。

    此时将打开“新建集群服务器实例”页。

  5. 在“名称”字段中，键入实例的名称。

    名称必须满足以下要求：

    • 名称中仅可以包含 ASCII 字符。

    • 名称开头必须是字母、数字或下划线。

    • 名称只能包含以下字符：

      • 小写字母

      • 大写字母

      • 数字

      • 连字符

      • 句点

      • 下划线

    • 名称在域中必须是唯一的，不得是另一个 GlassFish Server 实例、集群、命名配置或节点的名称。

    • 名称不得是 domain、server 或 GlassFish Server 保留的任何其他关键字。

  6. 从“节点”下拉列表中，选择实例将驻留的节点。

  7. 单击“确定”。
