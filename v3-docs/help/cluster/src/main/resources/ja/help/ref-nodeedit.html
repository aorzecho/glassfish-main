ノードの編集

「ノードの編集」ページを使用して、GlassFish Serverノードの構成データを更新します。このページを使用して、ノードのタイプをCONFIGからDCOMまたはSSHに変更してノードのリモート通信を有効にすることもできます。

ノードのタイプをCONFIGからDCOMに変更し、次のいずれかのフィールドを空のままにした場合、デフォルト値が適用されます。

  • Windowsドメイン

  • Windowsユーザー

ノードのタイプをCONFIGからSSHに変更し、次のいずれかのフィールドを空のままにした場合、デフォルト値が適用されます。

  • SSHポート

  • ユーザー名

  • 鍵ファイル

「ノードの編集」ページに含まれるオプションは、ノードのタイプによって異なります。

すべてのタイプのノードに対するオプション

次のオプションは、すべてのタイプのノードに使用できます。

名前

ノードが作成されたときにノードに割り当てられた名前。このフィールドは読取り専用です。

タイプ

ノードのタイプを指定するドロップダウン・リスト。ノードのタイプは次のいずれかです。

CONFIG

このノードは、リモート通信をサポートしません。「CONFIG」を選択した場合、DCOMまたはSSH構成のオプションは表示されません。ノードの現在のタイプがSSHである場合にCONFIGを選択すると、変更を保存するときにDCOMまたはSSH構成データがノードの構成から削除されます。

DCOM

このノードは、Distributed Component Object Model(DCOM)リモート・プロトコル経由の通信をサポートします。DCOMプロトコルは、Windowsシステムでのみ使用できます。

SSH

このノードは、Secure Shell(SSH)経由の通信をサポートします。

ノード・ホスト

ノードの更新後にノードが表すホストの名前。このフィールドは、特定のノードのタイプの場合に必須フィールドとなります。

  • ノードのタイプがCONFIGの場合は、ホスト名は省略可能です。ホスト名は、ノード上に存在するインスタンスが作成される際に特定されます。

  • ノードのタイプがDCOMまたはSSHの場合は、ホスト名を指定する必要があります。それ以外の場合はエラーが発生します。

ノード・ディレクトリ

ノード上で作成されたGlassFish Serverインスタンスを含めるディレクトリのパス。相対パスを指定する場合、パスは次に対して相対的になります: as-installディレクトリ。ここで、 as-installは、ホスト上のGlassFish Serverソフトウェアのベース・インストール・ディレクトリです。

このフィールドを空のままにした場合、ノードの構成は次のように更新されます。

  • ノードのタイプがCONFIGの場合、ドメイン管理サーバー(DAS)構成では、ノードにインスタンスのディレクトリは指定されません。ノード・ディレクトリは、ノード上に存在するインスタンスが作成される際に特定されます。

  • ノードのタイプがDCOMの場合、デフォルトは as-install\nodesであり、ここで、 as-installは、ホスト上のGlassFish Serverソフトウェアのベース・インストール・ディレクトリです。

  • ノードのタイプがSSHの場合、ノード・ディレクトリは次のとおりです: as-install/nodesであり、ここで、 as-installは、ホスト上のGlassFish Serverソフトウェアのベース・インストール・ディレクトリです。

インストール・ディレクトリ

ホスト上のGlassFish Serverソフトウェアのベース・インストール・ディレクトリの親のフル・パス(例: /export/glassfish3/)。

このフィールドを空のままにした場合、ノードの構成は次のように更新されます。

  • ノードのタイプがCONFIGの場合、DAS構成では、ノードにインストール・ディレクトリは指定されません。インストール・ディレクトリは、ノード上に存在するインスタンスが作成される際に特定されます。

  • ノードのタイプがDCOMまたはSSHの場合、インストール・ディレクトリは、DASのGlassFish Serverソフトウェアのデフォルト・ベース・インストール・ディレクトリの親です。この設定を使用できるのは、GlassFish Serverがすべてのホストで同じ場所にインストールされる場合のみです。

DCOMノードの追加オプション

次のオプションは、「タイプ」フィールドが「DCOM」に設定されている場合にのみ使用できます。

強制

「有効」オプションを選択すると、ノードのパラメータの検証に失敗した場合でも、DAS構成にノードが作成されます。ノードのパラメータを検証するには、DASがDCOM経由でノードのホストにアクセスできる必要があります。

DCOMの検証

「有効」オプションを選択すると、GlassFish Serverによって、ノードの更新前にノード・ホストに対するDCOM経由の接続がテストされます。テストの一環として、GlassFish Serverによって、delete_me.batという名前のスクリプトがホストにコピーされます。

このオプションを選択すると、「リモート・テスト・ディレクトリ」フィールドがアクティブになり、GlassFish Serverがスクリプトをコピーするディレクトリを指定できるようになります。

リモート・テスト・ディレクトリ

GlassFish Serverがdelete_me.batファイルを作成する必要のあるリモート・ホスト上のフォルダ。「Windowsユーザー名」オプションで指定するユーザーには、このフォルダに対する書込みアクセス権が必要です。それ以外の場合はエラーが発生します。デフォルトはC:\です。

Windowsドメイン

「Windowsユーザー名」オプションで指定したユーザーが含まれているWindowsドメインの名前。デフォルトは、管理コンソールが実行されているホストの名前です。

Windowsユーザー名

DCOM経由でホストに接続するためのプロセスを実行する、このノードのホストのユーザー。デフォルトはDASプロセスを実行するユーザーです。「ノード・ホスト」オプションがlocalhost-domainに設定されている場合、「Windowsユーザー名」オプションは無視されます。

Windowsユーザー認証

このノードのホストへのログイン時にWindowsユーザーを認証する方法を指定するドロップダウン・リスト。指定できる認証方法は次のとおりです。

パスワード

Windowsユーザーは、DAS構成に格納されているパスワードによって認証されます。

注意:

このパスワードは、DAS構成にクリア・テキストで格納されています。セキュリティ強化のため、鍵ファイルまたはパスワード・エイリアスを使用してください。

このオプションを選択すると、「Windowsユーザー・パスワード」フィールドと「パスワードの確認」フィールドがアクティブになり、パスワードを入力できるようになります。

パスワード・エイリアス

Windowsユーザーは、パスワード・エイリアスによって認証されます。このオプションを選択する場合は、「パスワード・エイリアス」ドロップダウン・リストからパスワード・エイリアスを選択します。

Windowsユーザー・パスワード

このノードのホストへのログイン時にWindowsユーザーが使用するパスワード。

パスワードの確認

このノードのホストへのログイン時にWindowsユーザーが使用するパスワードの確認。このフィールドの内容は、「Windowsユーザー・パスワード」フィールドの内容と完全に一致している必要があります。

パスワード・エイリアス

このドメインに対して定義されているパスワード・エイリアスのドロップダウン・リスト。

SSHノードの追加オプション

次のオプションは、「タイプ」フィールドが「SSH」に設定されている場合にのみ使用できます。

強制

「有効」オプションを選択すると、ノードのパラメータの検証に失敗した場合でも、ドメイン管理サーバー(DAS)構成でノードが更新されます。ノードのパラメータを検証するには、DASがSSH経由でノードのホストにアクセスできる必要があります。

SSHポート

このノードのホストに対するSSH接続に使用するポート。

ノードのタイプをCONFIGからSSHに変更し、このフィールドを空のままにした場合、「SSHポート」オプションは22に設定されます。

「ノード・ホスト」オプションがlocalhost-domainに設定されている場合、「SSHポート」オプションは無視されます。

SSHユーザー名

SSH経由でホストに接続するためのプロセスを実行する、このノードのホストのユーザー。

ノードのタイプをCONFIGからSSHに変更し、このフィールドを空のままにした場合、「ユーザー名」オプションはDASプロセスを実行しているユーザーに設定されます。

「ノード・ホスト」オプションがlocalhost-domainに設定されている場合、SSHユーザー・オプションは無視されます。

SSHユーザー認証

このノードのホストへのログイン時にSSHユーザーを認証する方法を指定するドロップダウン・リスト。指定できる認証方法は次のとおりです。

鍵ファイル

SSHユーザーは、SSH秘密鍵ファイルによって認証されます。このオプションを選択する場合は、ファイルの絶対パスを「鍵ファイル」フィールドに指定します。

パスワード

SSHユーザーは、DAS構成に格納されているパスワードによって認証されます。

注意:

このパスワードは、DAS構成にクリア・テキストで格納されています。セキュリティ強化のため、鍵ファイルまたはパスワード・エイリアスを使用してください。

このオプションを選択すると、「SSHユーザー・パスワード」フィールドと「パスワードの確認」フィールドがアクティブになり、パスワードを入力できるようになります。

パスワード・エイリアス

SSHユーザーは、パスワード・エイリアスによって認証されます。このオプションを選択する場合は、「パスワード・エイリアス」ドロップダウン・リストからパスワード・エイリアスを選択します。

SSHの設定

「有効」オプションを選択すると、GlassFish Serverによって、ノード・ホストにSSH鍵が設定されます。GlassFish Serverでは、既存の鍵ファイルをDASホストからコピーするか、または必要な場合はファイルのコピー前にファイルを生成することによって、SSH鍵が設定されます。このオプションは、「SSHユーザー認証」オプションが「鍵ファイル」に設定されている場合にのみ使用可能です。

鍵の生成

「有効」オプションを選択すると、GlassFish Serverによって、ファイルをノード・ホストにコピーする前にSSH鍵ファイルが生成されます。

SSHユーザー・パスワード

このノードのホストへのログイン時にSSHユーザーが使用するパスワード。

パスワードの確認

このノードのホストへのログイン時にSSHユーザーが使用するパスワードの確認。このフィールドの内容は、「SSHユーザー・パスワード」フィールドの内容と完全に一致している必要があります。

パスワード・エイリアス

このドメインに対して定義されているパスワード・エイリアスのドロップダウン・リスト。

公開鍵ファイル

「SSHユーザー名」オプションで指定するユーザーのSSH公開鍵ファイルの絶対パス。公開鍵ファイルの内容は、ノード・ホスト上のユーザーの.ssh/authorized_keysファイルに追加されます。.ssh/authorized_keysファイルがホストに存在していない場合は、GlassFish Serverによってファイルが作成されます。

鍵ファイルは、DASから到達可能なパスにあり、DASが読取り可能である必要があります。

デフォルトは、ユーザーの.sshディレクトリにある鍵ファイルです。複数の鍵ファイルが検出された場合は、GlassFish Serverが次の優先順位に従います。

  1. id_rsa

  2. id_dsa

  3. アイデンティティ

鍵ファイル

「ユーザー名」オプションで指定するユーザーのSSH秘密鍵ファイルの絶対パス。このファイルはノードのホスト上のsshdデーモンに対する認証に使用されます。

注意:

GlassFish Serverでは、パスワード・ファイルのAS_ADMIN_SSHPASSWORDエントリを使用したパスワード認証もサポートされます。パスワード・ファイルは、asadmin(1)ユーティリティの--paswordfileオプションで指定します。

SSH秘密鍵ファイルがパス・フレーズで保護されている場合は、パスワード・ファイルにAS_ADMIN_SSHKEYPASSPHRASEエントリを含める必要があります。

鍵ファイルは、DASから到達可能なパスにあり、DASが読取り可能である必要があります。

デフォルトは、ノードのタイプをCONFIGからSSHに変更してノードのSSH通信を有効にしているかどうかによって異なります。

ノードのタイプをCONFIGからSSHに変更し、このフィールドを空のままにした場合、「鍵ファイル」オプションはユーザーの.sshディレクトリにある鍵ファイルに設定されます。複数の鍵ファイルが検出された場合は、GlassFish Serverが次の優先順位に従います。

  1. id_rsa

  2. id_dsa

  3. アイデンティティ
