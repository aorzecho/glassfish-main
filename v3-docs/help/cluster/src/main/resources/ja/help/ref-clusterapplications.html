アプリケーション

「アプリケーション」ページを使用して、現在のクラスタで次のタスクを実行します。

  • クラスタにすでに配備されているアプリケーションの表示と管理

  • クラスタへの追加のアプリケーションの配備

「クラスタ名」フィールドは、現在のクラスタの名前が表示される読み取り専用のフィールドです。

「アプリケーション」ページには、現在のクラスタに配備されているアプリケーションの一覧が表示されます。アプリケーションごとに、次の情報が表示されます。

名前

アプリケーションの名前。

名前には、オプションのバージョン ID を含めることができます。このバージョン ID は名前のあとに続き、コロン (:) で名前と区切ります。アプリケーションとモジュールのバージョンについては、『Oracle GlassFish Server 3.1 Application Deployment Guide』の「Module and Application Versions」を参照してください。

使用可能

アプリケーションが有効かどうかが表示されます。

  • アプリケーションが有効な場合は、チェックマーク (✓) が表示されます。

  • アプリケーションが無効な場合は、十字記号 (✗) が表示されます。

LB 有効

アプリケーションで負荷分散が有効になっているかどうかが表示されます。

  • アプリケーションで負荷分散が有効になっている場合は、チェックマーク (✓) が表示されます。

  • アプリケーションで負荷分散が無効になっている場合は、十字記号 (✗) が表示されます。

エンジン

アプリケーションが使用するコンテナのタイプ。コンテナのタイプは次のいずれかになります。

  • Web

  • webservices

  • ejb

  • connector

  • appclient

  • weld — Java EE プラットフォームアプリケーションの Contexts and Dependency Injection のコンテナ

「配備されているアプリケーション」の表には、次のオプションも表示されます。

配備する

アプリケーションを配備するボタン。

配備取消し

選択した 1 つまたは複数のアプリケーションの配備を取り消しするボタン。

アプリケーションの配備取消しでは、すべてのターゲットおよびドメインからアプリケーションを削除します。アプリケーションの配備が取り消された場合、そのアプリケーションは GlassFish サーバー に配備されているすべてのアプリケーションが表示される「アプリケーション」ページに表示されなくなります。

現在のターゲットからのみアプリケーションを削除するには、このページの「削除」ボタンを使用します。

削除

選択した 1 つまたは複数のアプリケーションを削除するボタン。

アプリケーションの削除では、現在のターゲットからのみアプリケーションを削除します。ターゲットからアプリケーションを削除しても、そのアプリケーションは GlassFish サーバー に配備されているすべてのアプリケーションが表示される「アプリケーション」ページに引き続き表示されます。配備が行われたほかのターゲットのアプリケーションも配備されたままとなります。

すべてのターゲットおよびドメインからアプリケーションを削除するには、このページの「配備取消し」ボタンを使用します。

追加の操作

選択した 1 つまたは複数のアプリケーションに対して追加の操作を実行するドロップダウンリスト。

有効

選択した 1 つまたは複数のアプリケーションを有効にします。

無効

選択した 1 つまたは複数のアプリケーションを無効にします。

ロードバランサ有効

選択した 1 つまたは複数のアプリケーションで負荷分散を有効にします。

ロードバランサ無効

選択した 1 つまたは複数のアプリケーションで負荷分散を無効にします。
