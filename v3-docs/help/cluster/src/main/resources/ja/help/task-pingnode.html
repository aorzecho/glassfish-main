ノードがアクセス可能かどうかをテストする

リモート・ホストを表すノードのGlassFish Serverインスタンスを作成または起動するには、DASがノードにアクセスできる必要があります。このようなノードでインスタンスを作成または起動する前に、事前にDASがノードにアクセスできることをテストします。

テストするノードのタイプは、「DCOM」または「SSH」である必要があります。「CONFIG」タイプのノードは、リモート通信に使用できないためテストできません。

開始する前の準備

ノードが存在するホストで、ノードのタイプに適切なプロトコルが構成されていることを確認します。ノードは、このプロトコルを介したリモート通信に対して有効になります。

  • ノードのタイプが「DCOM」の場合は、分散コンポーネント・オブジェクト・モデル(DCOM)プロトコルを構成する必要があります。

  • ノードのタイプが「SSH」の場合は、Secure Shell(SSH)プロトコルを構成する必要があります。

    1. ナビゲーション・ツリーで、「ノード」ノードを選択します。

      「ノード」ページが開きます。このページに、現在のドメイン内にあるすべてのノードが一覧表示されます。

    2. ノードの表で、テストするノードの「Ping」リンクに従います。

      DASがノードにアクセスできるかどうかを示すメッセージが「ノード」ページに追加されます。
