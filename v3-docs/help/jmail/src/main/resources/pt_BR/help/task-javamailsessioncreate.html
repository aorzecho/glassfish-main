Para criar uma sessão JavaMail

  1. Na árvore de navegação, expanda o nó Recursos.
  2. No nó Recursos, selecione Sessões JavaMail.

    Abre-se a página Sessões JavaMail.

  3. Na tabela Sessões, clique em Novo.

    Abre-se a página Nova sessão JavaMail.

  4. No campo Nome JNDI, digite um nome exclusivo para o recurso de sessão de correio.

    Utilize o prefixo de subcontexto de nomeação mail/ para recursos JavaMail. Por exemplo: mail/MySession. O nome deve conter somente caracteres alfanuméricos, sublinhado, traço ou ponto.

  5. No campo Host de correio, digite o nome do host do servidor de correio padrão.

    Os métodos de conexão dos objetos Armazenamento e Transporte usam esse valor se não é fornecida uma propriedade de host específica do protocolo. O nome deve ser resolúvel para um nome de host real.

  6. No campo Usuário padrão, digite o nome do usuário a ser fornecido quando conectar-se a um servidor de correio.

    Os métodos de conexão dos objetos Armazenamento e Transporte usam esse valor se não é fornecida uma propriedade de nome de usuário específica do protocolo. O nome de usuário deve conter somente caracteres alfanuméricos, sublinhado, traço ou ponto.

  7. No campo Endereço de remetente padrão, digite o endereço de e-mail do usuário padrão.

    O formato é nome de usuário@hostdomínio.

  8. No campo Descrição, digite uma descrição para a sessão de correio.
  9. Marque a caixa de seleção Status ativo para ativar a sessão de correio.

    O valor padrão é Ativo.

  10. No campo Protocolo do armazenamento, defina o serviço de protocolo do armazenamento a ser utilizado.

    O serviço de protocolo do armazenamento conecta-se a um servidor de correio, recupera mensagens e as salva em uma ou mais pastas. Os valores permitidos são imap, pop3, imaps e pop3s. O valor padrão é imap.

  11. No campo Classe de protocolo do armazenamento, especifique a classe de implementação do provedor de serviço para armazenamento.

    Os valores permitidos são:

    • com.sun.mail.imap.IMAPStore

    • com.sun.mail.pop3.POP3Store

    • com.sun.mail.imap.IMAPSSLStore

    • com.sun.mail.pop3.POP3SSLStore

    O valor padrão é com.sun.mail.imap.IMAPStore.

  12. No campo Protocolo do transporte, especifique o método de comunicação de transporte.

    O serviço de protocolo de transporte envia mensagens. Os valores permitidos são smtp e smtps. Por padrão, o Protocolo do transporte é smtp.

  13. No campo Classe de protocolo do transporte, especifique a classe de implementação do provedor de serviço para transporte.

    O valor padrão é com.sun.mail.smtp.SMTPTransport.

  14. Marque a caixa de seleção Depuração para ativar a saída de depuração extra, incluindo um rastreamento de protocolo, para a sessão de correio.

    Se o nível de log JavaMail estiver definido como BOM ou melhor, a saída de depuração é gerada e incluída no arquivo de log do sistema. Para obter informações sobre a configuração do nível de log, consulte Para configurar níveis de log. Por padrão, a opção está desativada.

  15. (Opcional) Na tabela Propriedades adicionais, especifique propriedades adicionais requeridas por aplicativos, como uma propriedade de nome de usuário ou de host específica do protocolo.

    Para obter uma lista das propriedades disponíveis, consulte a documentação da JavaMail API em . Propriedades adicionais devem começar com mail- e devem usar hífens, não pontos, como separadores. Por exemplo, mail-debug é correto, mas mail.debug não. O back-end do Servidor GlassFish converte os hífens nos pontos esperados pela JavaMail API.

    Para adicionar uma propriedade, clique no botão Adicionar propriedade. Na linha em branco que é exibida, digite o nome da propriedade no campo Nome e digite o valor da propriedade no campo Valor.

  16. Na lista de destinos disponíveis, selecione o destino ou destinos dos quais configurar a sessão de e-mail e clique em Adicionar.

    Os destinos podem ser armazenados em clusters ou em instâncias independentes do servidor. Se um destino não for selecionado, a sessão é configurado com a instância padrão do servidor server.

  17. Clique em OK.
