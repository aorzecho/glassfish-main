Editar sessão JavaMail

Utilize a página Editar sessão JavaMail para editar uma sessão JavaMail.

A página Editar sessão JavaMail contém as opções a seguir.

Carregar padrões

Botão para restaurar com os valores padrão as configurações que apresentam valores padrão. As configurações que não apresentam valores padrão não são alteradas.

Nome JNDI

Um nome exclusivo que identifica a sessão JavaMail. O nome JNDI é um campo somente leitura. Você somente pode especificar o nome JNDI quando cria uma nova sessão JavaMail.

Host de correio

O nome do host do servidor de correio padrão. Os métodos de conexão dos objetos Armazenamento e Transporte usam esse valor se não é fornecida uma propriedade de host específica do protocolo. O nome deve ser resolúvel para um nome de host real.

Usuário padrão

O nome de usuário a ser fornecido ao conectar-se a um servidor de correio. Os métodos de conexão dos objetos Armazenamento e Transporte usam esse valor se não é fornecida uma propriedade de nome de usuário específica do protocolo. O nome de usuário deve conter somente caracteres alfanuméricos, sublinhado, traço ou ponto.

Endereço padrão do remetente

O endereço de e-mail do usuário padrão. O formato é nome de usuário@hostdomínio.

Descrição

Uma descrição dessa sessão de correio.

Status

O status da sessão de correio. A sessão de correio pode ser ativada ou desativada. O valor padrão é Ativo.

Protocolo de armazenamento

O serviço de protocolo de armazenamento, que se conecta a um servidor de correio, recuperar mensagens e as salva em uma ou mais pastas. Os valores permitidos são imap, pop3, imaps e pop3s. O valor padrão é imap.

Classe do protocolo de armazenamento

A classe de implementação do provedor do serviço para armazenamento. Os valores permitidos são:

  • com.sun.mail.imap.IMAPStore

  • com.sun.mail.pop3.POP3Store

  • com.sun.mail.imap.IMAPSSLStore

  • com.sun.mail.pop3.POP3SSLStore

O valor padrão é com.sun.mail.imap.IMAPStore.

Protocolo de transporte

O serviço de protocolo de transporte, que envia mensagens. Os valores permitidos são smtp e smtps. O valor padrão é smtp.

Classe do protocolo de transporte

A classe de implementação do provedor do serviço de transporte. Os valores permitidos são:

  • com.sun.mail.smtp.SMTPTransport

  • com.sun.mail.smtp.SMTPSSLTransport

O valor padrão é com.sun.mail.smtp.SMTPTransport.

Depuração

Se essa opção for selecionada, a depuração para esse recurso é ativada. Se o nível de log JavaMail estiver definido como BOM ou melhor, a saída de depuração é gerada e incluída no arquivo de log do sistema. Para obter informações sobre a configuração do nível de log, consulte Para configurar níveis de log.

Propriedades adicionais

Propriedades adicionais para a sessão JavaMail. Para obter uma lista das propriedades disponíveis, consulte a documentação da JavaMail API em . Propriedades adicionais devem começar com mail- e devem usar hífens, não pontos, como separadores. Por exemplo, mail-debug é correto, mas mail.debug não. O back-end do Servidor GlassFish converte os hífens nos pontos esperados pela JavaMail API.
