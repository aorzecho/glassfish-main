Editar Sessão de JavaMail

Utilize a página Editar sessão JavaMail para editar uma sessão JavaMail.

A página Editar sessão JavaMail contém as opções a seguir.

Carregar Defaults

Botão para restaurar com os valores default as definições que apresentam valores default. As definições que não apresentam valores default não são alteradas.

Nome JNDI

Um nome exclusivo que identifica a sessão JavaMail. O nome JNDI é um campo somente para leitura. Você somente pode especificar o nome JNDI quando cria uma nova sessão JavaMail.

Host de correio

O nome do host do servidor de correio default. Os métodos de conexão dos objetos Armazenamento e Transporte usam esse valor se não é fornecida uma propriedade de host específica do protocolo. O nome deve ser resolúvel para um nome de host real.

Usuário Default

O nome de usuário a ser fornecido ao conectar-se a um servidor de correio. Os métodos de conexão dos objetos Armazenamento e Transporte usam esse valor se não é fornecida uma propriedade de nome de usuário específica do protocolo. O nome de usuário deve conter somente caracteres alfanuméricos, sublinhado, traço ou ponto.

Endereço Default do Remetente

O endereço de e-mail do usuário default. O formato é nome de usuário@host.domínio.

Descrição

Uma descrição dessa sessão de correio.

Status

O status da sessão de correio. A sessão de correio pode ser ativada ou desativada. O valor default é Ativo.

Protocolo de armazenamento

O serviço de protocolo de armazenamento, que se conecta a um servidor de correio, recuperar mensagens e as salva em uma ou mais pastas. Os valores permitidos são imap, pop3, imaps e pop3s. O valor default é imap.

Classe do protocolo de armazenamento

A classe de implementação do provedor do serviço para armazenamento. Os valores permitidos são:

  • com.sun.mail.imap.IMAPStore

  • com.sun.mail.pop3.POP3Store

  • com.sun.mail.imap.IMAPSSLStore

  • com.sun.mail.pop3.POP3SSLStore

O valor default é com.sun.mail.imap.IMAPStore.

Protocolo de transporte

O serviço de protocolo de transporte, que envia mensagens. Os valores permitidos são smtp e smtps. O valor default é smtp.

Classe do protocolo de transporte

A classe de implementação do provedor do serviço de transporte. Os valores permitidos são:

  • com.sun.mail.smtp.SMTPTransport

  • com.sun.mail.smtp.SMTPSSLTransport

O valor default é com.sun.mail.smtp.SMTPTransport.

Depuração

Se essa opção for selecionada, a depuração para esse recurso é ativada. Se o nível de log JavaMail estiver definido como BOM ou melhor, a saída de depuração é gerada e incluída no arquivo de log do sistema. Para obter informações sobre a definição do nível de log, consulte Para configurar níveis de log.

Propriedades adicionais

Propriedades adicionais para a sessão JavaMail. Para ver uma lista das propriedades disponíveis, consulte a documentação da API JavaMail (http://java.sun.com/products/javamail/javadocs/index.html). Propriedades adicionais devem começar com mail- e devem usar hífens, não pontos, como separadores. Por exemplo, mail-debug é correto, mas mail.debug não. O back-end do GlassFish Server converte os hífens nos pontos esperados pela JavaMail API.
