Para editar uma sessão JavaMail

  1. Na árvore de navegação, expanda o nó Recursos.
  2. No nó Recursos, selecione Sessões JavaMail.

    A página Sessões JavaMail é aberta.

  3. Na página Sessões JavaMail, clique no nome da sessão a ser editada.

    A página Editar sessões JavaMail é aberta.

  4. No campo Host de correio, especifique o nome do host do servidor de correio padrão.

    Os métodos de conexão dos objetos Armazenamento e Transporte usam esse valor se não é fornecida uma propriedade de host específica do protocolo. O nome deve ser resolúvel para um nome de host real.

  5. No campo Usuário padrão, digite outro nome de usuário a ser fornecido quando conectar-se a um servidor de correio.

    Os métodos de conexão dos objetos Armazenamento e Transporte usam esse valor se não é fornecida uma propriedade de nome de usuário específica do protocolo. O nome de usuário deve conter somente caracteres alfanuméricos, sublinhado, traço ou ponto.

  6. No campo Endereço de remetente padrão, digite o endereço de e-mail do usuário padrão.

    O formato é nome de usuário@hostdomínio.

  7. No campo Descrição, altere a descrição da sessão de correio.
  8. Marque a caixa de seleção Status ativo para ativar a sessão de correio.

    O valor padrão é Ativo.

  9. No campo Protocolo do armazenamento, defina o método de comunicação do objeto Armazenamento a ser usado.

    Por padrão, o Protocolo do armazenamento é imap.

  10. No campo Classe de protocolo do armazenamento, especifique a classe do método de comunicação do Armazenamento que implementa o protocolo do Armazenamento desejado.

    Por padrão, a Classe de protocolo do armazenamento é com.sun.mail.imap.IMAPStore.

  11. No campo Protocolo do transporte, especifique o método de comunicação de transporte.

    Por padrão, o Protocolo do transporte é smtp.

  12. No campo Classe de protocolo do transporte, especifique o método de comunicação para a classe de Transporte.

    Por padrão, a Classe de protocolo do transporte é com.sun.mail.smtp.SMTPTransport.

  13. Marque a caixa de seleção Depuração para ativar a saída de depuração extra, incluindo um rastreamento de protocolo, para a sessão de correio.

    Se o nível de log JavaMail estiver definido como BOM ou melhor, a saída de depuração é gerada e incluída no arquivo de log do sistema. Para obter informações sobre a configuração do nível de log, consulte Para configurar níveis de log.

  14. (Opcional) Na tabela Propriedades adicionais, especifique propriedades adicionais requeridas por aplicativos, como uma propriedade de nome de usuário ou de host específica do protocolo.

    Para obter uma lista das propriedades disponíveis, consulte a documentação da JavaMail API em . Propriedades adicionais devem começar com mail- e devem usar hífens, não pontos, como separadores. Por exemplo, mail-debug é correto, mas mail.debug não. O back-end do Servidor GlassFish converte os hífens nos pontos esperados pela JavaMail API.

    • Para adicionar uma propriedade, clique no botão Adicionar propriedade. Na linha em branco que é exibida, digite o nome da propriedade no campo Nome e digite o valor da propriedade no campo Valor.
    • Para modificar uma propriedade, edite o campo Valor da propriedade.
    • Para excluir uma propriedade, marque a caixa de seleção à esquerda do campo Nome da propriedade a ser excluída. Clique no botão Excluir propriedades.
  15. Clique em Salvar.

    Dica - Para restaurar as configurações padrão, clique no botão Carregar padrões e depois em Salvar.
