編輯 JavaMail 階段作業

使用 「編輯 JavaMail 階段作業」 頁面可編輯 JavaMail 階段作業。

「編輯 JavaMail 階段作業」 頁面包含下列選項。

載入預設值

用於將具有預設值的設定復原成其預設值的按鈕。沒有預設值的設定保持不變。

JNDI 名稱

可識別 JavaMail 階段作業的唯一名稱。「JNDI 名稱」 是唯讀欄位。您僅能在建立新的 JavaMail 階段作業時指定 JNDI 名稱。

郵件主機

預設郵件伺服器的主機名稱。如果未提供協定特定的主機特性，Store 和 Transport 物件的連線方法將使用該值。名稱必須可以解析為實際的主機名稱。

預設使用者

連線至郵件伺服器時要提供的使用者名稱。如果未提供協定特定的使用者名稱特性，Store 和 Transport 物件的連線方法使用該值。使用者名稱僅能包含字母數字、底線、破折號或點字元。

預設寄件者地址

預設使用者的電子郵件地址。格式為 username@host.domain。

說明

此郵件階段作業的說明。

狀態

郵件階段作業的狀態。您可啟用或停用郵件階段作業。預設值為 「啟用」。

儲存協定

儲存協定服務可連線至郵件伺服器、擷取郵件訊息，並將郵件儲存於一或多個資料夾。允許的值為 imap、pop3、imaps 和 pop3s。預設值為 imap。

儲存協定類別

儲存的服務提供者實作類別。允許的值包括 ：

  • com.sun.mail.imap.IMAPStore

  • com.sun.mail.pop3.POP3Store

  • com.sun.mail.imap.IMAPSSLStore

  • com.sun.mail.pop3.POP3SSLStore

預設值為 com.sun.mail.imap.IMAPStore。

傳輸協定

傳輸協定服務可傳送訊息。允許的值為 smtp 和 smtps。預設值為 smtp。

傳輸協定類別

傳輸的服務提供者實作類別。允許的值包括 ：

  • com.sun.mail.smtp.SMTPTransport

  • com.sun.mail.smtp.SMTPSSLTransport

預設值為 com.sun.mail.smtp.SMTPTransport。

除錯

若選取了此選項，會為此資源啟用除錯功能。如果將 JavaMail 記錄層級設為詳細 或更詳細，則系統會產生除錯輸出，並且該輸出會包含在系統記錄檔中。請參閱 To Configure Log Levels，以取得有關設定記錄層級的資訊。

其他特性

JavaMail 階段作業的其他特性。如需可用特性的清單，請參閱 JavaMail API 文件 (http://java.sun.com/products/javamail/javadocs/index.html)。其他特性必須以 mail- 開頭，且必須使用連字符 (而非點字元) 做為分隔符。例如 mail-debug 是正確的，而 mail.debug 則不正確。GlassFish Server 後端會將連字符轉換為 JavaMail API 預期的點字元。
