Modifier une session JavaMail

  1. Dans l'arborescence de navigation, développez le noeud Ressources.

  2. Sous le noeud Ressources, sélectionnez Sessions JavaMail.

    La page Sessions JavaMail apparaît.

  3. Sur la page Sessions JavaMail, cliquez sur le nom de session à modifier.

    La page Modifier une session JavaMail apparaît.

  4. Dans le champ Hôte de messagerie, spécifiez le nom d'hôte du serveur de messagerie par défaut.

    Les méthodes de connexion des objets Store et Transport utilisent cette valeur si aucune propriété d'hôte propre à un protocole n'est indiquée. Le nom doit pouvoir être résolu en un nom d'hôte réel.

  5. Dans le champ Utilisateur par défaut, entrez un autre nom utilisateur à indiquer lors de la connexion à un serveur de messagerie.

    Les méthodes de connexion des objets Store et Transport utilisent cette valeur si aucune propriété de nom utilisateur propre à un protocole n'est indiquée. Le nom utilisateur ne peut contenir que des caractères alphanumériques, des traits de soulignement, des tirets ou des points.

  6. Dans le champ Adresse de l'expéditeur par défaut, entrez l'adresse électronique de l'utilisateur par défaut.

    Le format est le suivant : nomutilisateur@hôte.domaine.

  7. Dans le champ Description, modifiez la description de la session de messagerie.

  8. Cochez la case Activé du champ Statut pour activer la session de messagerie.

    La valeur par défaut est Activé.

  9. Dans le champ Protocole de stockage, indiquez la méthode de communication des objets Store à utiliser.

    Par défaut, le protocole de stockage est imap.

  10. Dans le champ Classe de protocole de stockage, spécifiez la classe de la méthode de communication de stockage implémentant le protocole de stockage requis.

    Par défaut, la classe de protocole de stockage est com.sun.mail.imap.IMAPStore.

  11. Dans le champ Protocole de transport, spécifiez la méthode de communication de transport.

    Par défaut, le protocole de transport est smtp.

  12. Dans le champ Classe de protocole de transport, spécifiez la méthode de communication de la classe de transport.

    Par défaut, la classe de protocole de transport est com.sun.mail.smtp.SMTPTransport .

  13. Cochez la case Déboguer pour activer une sortie de débogage supplémentaire, notamment une trace de protocole, pour cette session de messagerie.

    Si le niveau de journalisation JavaMail a la valeur FINE ou plus fin, la sortie de débogage est créée et incluse dans le fichier journal du système. Pour plus d'informations sur la configuration du niveau de journalisation, reportez-vous à Configurer les niveaux de journalisation.

  14. Dans la table Autres propriétés, spécifiez les propriétés supplémentaires requises par les applications, notamment une propriété d'hôte ou de nom utilisateur propre à un protocole.

    Pour obtenir la liste des propriétés disponibles, reportez-vous à la documentation de l'API JavaMail (http://java.sun.com/products/javamail/javadocs/index.html). Les propriétés supplémentaires doivent commencer par mail- et des traits d'union doivent être utilisés comme séparateurs à la place des points. Par exemple, mail-debug est correct mais mail.debug ne l'est pas. L'arrière-plan de GlassFish Server convertit les traits d'union en points, attendus par l'API JavaMail.

    • Pour ajouter une propriété, cliquez sur le bouton Ajouter une propriété. Sur la ligne vide qui apparaît, saisissez le nom de la propriété dans le champ Nom et la valeur de la propriété dans le champ Valeur.

    • Pour modifier une propriété, modifiez le champ Valeur de cette propriété.

    • Pour supprimer une propriété, sélectionnez la case située à gauche du champ Nom de la propriété à supprimer, puis cliquez sur le bouton Supprimer des propriétés.

  15. Cliquez sur Enregistrer.

    Conseil:

    Pour revenir aux paramètres par défaut, cliquez sur le bouton Charger les valeurs par défaut, puis sur Enregistrer.
