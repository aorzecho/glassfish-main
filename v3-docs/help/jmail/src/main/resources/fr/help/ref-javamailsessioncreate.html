Nouvelle session JavaMail

Utilisez la page Nouvelle session JavaMail pour créer une ressource de session JavaMail.

La page Nouvelle session JavaMail contient les options suivantes :

Nom JNDI

Nom unique identifiant la session de courrier. Utilisez le préfixe de désignation de sous-contexte mail/ pour les ressources JavaMail. Par exemple : mail/MySession. Le nom ne peut contenir que des caractères alphanumériques, des caractères de soulignement, des tirets ou des points.

Hôte de messagerie

Nom d’hôte du serveur de messagerie par défaut. Les méthodes de connexion des objets Stockage et Transport utilisent cette valeur si aucune propriété d’hôte spécifique à un protocole n’est fournie. Le nom doit refléter un nom d’hôte réel.

Utilisateur par défaut

Nom d’utilisateur à saisir lors de la connexion à un serveur de messagerie. Les méthodes de connexion des objets Stockage et Transport utilisent cette valeur si aucune propriété de nom d’utilisateur spécifique à un protocole n’est fournie. Le nom d’utilisateur ne peut contenir que des caractères alphanumériques, des traits de soulignement, des tirets ou des points.

Adresse de l'expéditeur par défaut

Adresse e-mail de l’utilisateur par défaut Le format est nom d’utilisateur@hôte.domaine.

Description

Description de cette session de messagerie.

Statut

État de la session de messagerie. La session de messagerie peut être activée ou désactivée. La valeur par défaut est Activée.

Protocole de stockage

Le service de protocole de stockage, qui se connecte à un serveur de courrier, récupère les messages puis les enregistre dans un ou plusieurs dossiers. Les valeurs autorisées sont imap, pop3, imaps, et pop3s. La valeur par défaut est imap.

Classe de protocole de stockage

Classe d’implémentation du fournisseur de services de stockage. Les valeurs autorisées sont :

  • com.sun.mail.imap.IMAPStore

  • com.sun.mail.pop3.POP3Store

  • com.sun.mail.imap.IMAPSSLStore

  • com.sun.mail.pop3.POP3SSLStore

La valeur par défaut est com.sun.mail.imap.IMAPStore.

Protocole de transport

Protocole de transport, qui envoie des messages. Les valeurs autorisées sont smtp et smtps. La valeur par défaut est smtp.

Classe de protocole de transport

Classe d’implémentation du fournisseur de services de transport. Les valeurs autorisées sont :

  • com.sun.mail.smtp.SMTPTransport

  • com.sun.mail.smtp.SMTPSSLTransport

La valeur par défaut est com.sun.mail.smtp.SMTPTransport.

Débogage

Si cette option est sélectionnée, le débogage de cette ressource est activé. Si le niveau de journal JavaMail est défini sur FIN ou plus fin, la sortie de débogage est créée et incluse dans le fichier journal du système. Pour plus d’informations sur la configuration du niveau de journal, reportez-vous à la section Configurer les niveaux de journal.

Autres propriétés

Propriétés supplémentaires de la session JavaMail. Pour une liste des propriétés disponibles, consultez la JavaMail API documentation. Les propriétés supplémentaires doivent commencer par mail- et des traits d’union doivent être utilisés comme séparateurs à la place de points. Par exemple, mail-debug est correct mais mail.debug ne l’est pas. L’arrière-plan de GlassFish Server convertit les traits d’union en points attendus par l’API JavaMail.

Cibles

Clusters et instances autonomes de la session JavaMail. Déplacez les cibles voulues dans la colonne Cibles sélectionnées à l'aide des boutons Ajouter et Tout Ajouter. Déplacez toutes les cibles inutiles dans la colonne Cibles disponibles à l'aide des boutons Supprimer et Tout Supprimer.

Cette option s'affiche uniquement si les clusters ou les instances autonomes ont été créés dans le domaine.
