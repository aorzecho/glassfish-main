Pour créer une session JavaMail

  1. Dans l’arborescence de navigation, développez le nœud Ressources.
  2. Sous le nœud Ressources, sélectionnez Sessions JavaMail.

    La page Sessions JavaMail s’affiche.

  3. Dans le tableau Sessions, cliquez sur Nouveau.

    La page Nouvelle session JavaMail s’affiche.

  4. Dans le champ Nom JNDI, entrez un nom unique de ressource de session de courrier.

    Utilisez le préfixe de désignation de sous-contexte mail/ pour les ressources JavaMail. Par exemple : mail/MySession. Le nom ne peut contenir que des caractères alphanumériques, des caractères de soulignement, des tirets ou des points.

  5. Dans le champ Hôte de messagerie, entrez le nom d’hôte du serveur de messagerie par défaut.

    Les méthodes de connexion des objets Stockage et Transport utilisent cette valeur si aucune propriété d’hôte spécifique à un protocole n’est fournie. Le nom doit refléter un nom d’hôte réel.

  6. Dans le champ Utilisateur par défaut, entrez le nom d’utilisateur à fournir lors de la connexion à un serveur de messagerie.

    Les méthodes de connexion des objets Stockage et Transport utilisent cette valeur si aucune propriété de nom d’utilisateur spécifique à un protocole n’est fournie. Le nom d’utilisateur ne peut contenir que des caractères alphanumériques, des traits de soulignement, des tirets ou des points.

  7. Dans le champ Adresse de l'expéditeur par défaut, entrez l'adresse e-mail de l'utilisateur par défaut.

    Le format est nom d’utilisateur@hôte.domaine.

  8. Dans le champ Description, entrez une description de la session de messagerie.
  9. Cochez la case État activé pour activer la session de messagerie.

    La valeur par défaut est Activée.

  10. Dans le champ Protocole de stockage, indiquez le service de protocole de stockage à utiliser.

    Le service de protocole de stockage se connecte à un serveur de courrier, récupère les messages puis les enregistre dans un ou plusieurs dossiers. Les valeurs autorisées sont imap, pop3, imaps, et pop3s. La valeur par défaut est imap.

  11. Dans le champ Classe de protocole de stockage, spécifiez la classe d’implémentation du fournisseur de services de stockage.

    Les valeurs autorisées sont :

    • com.sun.mail.imap.IMAPStore

    • com.sun.mail.pop3.POP3Store

    • com.sun.mail.imap.IMAPSSLStore

    • com.sun.mail.pop3.POP3SSLStore

    La valeur par défaut est com.sun.mail.imap.IMAPStore.

  12. Dans le champ Protocole de transport, spécifiez la méthode de communication de transport.

    Le service de protocole de transport envoie les messages. Les valeurs autorisées sont smtp et smtps. Par défaut, le protocole de transport est smtp.

  13. Dans le champ Classe de protocole de transport, spécifiez la classe d’implémentation du fournisseur de services de transport.

    La valeur par défaut est com.sun.mail.smtp.SMTPTransport.

  14. Cochez la case Déboguer pour activer une sortie de débogage supplémentaire, notamment un suivi de protocole, pour cette session de messagerie.

    Si le niveau de journal JavaMail est défini sur FIN ou plus fin, la sortie de débogage est créée et incluse dans le fichier journal du système. Pour plus d’informations sur la configuration du niveau de journal, reportez-vous à la section Configurer les niveaux de journal. Cette option est désactivée par défaut.

  15. (Facultatif) Dans le tableau Autres propriétés, spécifiez les autres propriétés requises par les applications, notamment une propriété d’hôte ou de nom d’utilisateur spécifique à un protocole.

    Pour une liste des propriétés disponibles, consultez la JavaMail API documentation. Les propriétés supplémentaires doivent commencer par mail- et des traits d’union doivent être utilisés comme séparateurs à la place de points. Par exemple, mail-debug est correct mais mail.debug ne l’est pas. L’arrière-plan de GlassFish Server convertit les traits d’union en points attendus par l’API JavaMail.

    Pour ajouter une propriété, cliquez sur le bouton Ajouter propriété. Sur la ligne vide qui apparaît, saisissez le nom de la propriété dans le champ Nom et la valeur de la propriété dans le champ Valeur.

  16. Dans la liste des cibles disponibles, sélectionnez la ou les cibles souhaitées pour la configuration de la session de messagerie et cliquez sur Ajouter.

    Les cibles peuvent être des clusters ou des instances de serveur autonomes. Si vous ne sélectionnez aucune cible, la session est configurée en fonction de l'instance de serveur par défaut server.

  17. Cliquez sur OK.
