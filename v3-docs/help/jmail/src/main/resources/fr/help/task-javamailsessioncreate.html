Créer une session JavaMail

  1. Dans l'arborescence de navigation, développez le noeud Ressources.

  2. Sous le noeud Ressources, sélectionnez Sessions JavaMail.

    La page Sessions JavaMail apparaît.

  3. Dans la table Sessions, cliquez sur Nouveau.

    La page Nouvelle session JavaMail apparaît.

  4. Dans le champ Nom JNDI, attribuez un nom unique à la ressource de session de messagerie.

    Utilisez le préfixe de dénomination de sous-contexte mail/ pour les ressources JavaMail. Par exemple : mail/MySession. Le nom ne peut contenir que des caractères alphanumériques, des traits de soulignement, des tirets ou des points.

  5. Dans le champ Hôte de messagerie, entrez le nom d'hôte du serveur de messagerie par défaut.

    Les méthodes de connexion des objets Store et Transport utilisent cette valeur si aucune propriété d'hôte propre à un protocole n'est indiquée. Le nom doit pouvoir être résolu en un nom d'hôte réel.

  6. Dans le champ Utilisateur par défaut, entrez le nom utilisateur à indiquer lors de la connexion à un serveur de messagerie.

    Les méthodes de connexion des objets Store et Transport utilisent cette valeur si aucune propriété de nom utilisateur propre à un protocole n'est indiquée. Le nom utilisateur ne peut contenir que des caractères alphanumériques, des traits de soulignement, des tirets ou des points.

  7. Dans le champ Adresse de l'expéditeur par défaut, entrez l'adresse électronique de l'utilisateur par défaut.

    Le format est le suivant : nomutilisateur@hôte.domaine.

  8. Dans le champ Description, entrez la description de la session de messagerie.

  9. Cochez la case Activé du champ Statut pour activer la session de messagerie.

    La valeur par défaut est Activé.

  10. Dans le champ Protocole de stockage, indiquez le service de protocole de stockage à utiliser.

    Le service de protocole de stockage se connecte à un serveur de messagerie, extrait les messages puis les enregistre dans des dossiers. Les valeurs autorisées sont les suivantes : imap, pop3, imaps et pop3s. La valeur par défaut est imap.

  11. Dans le champ Classe de protocole de stockage, spécifiez la classe d'implémentation du fournisseur de services de stockage.

    Les valeurs autorisées sont les suivantes :

    • com.sun.mail.imap.IMAPStore

    • com.sun.mail.pop3.POP3Store

    • com.sun.mail.imap.IMAPSSLStore

    • com.sun.mail.pop3.POP3SSLStore

    La valeur par défaut est com.sun.mail.imap.IMAPStore.

  12. Dans le champ Protocole de transport, spécifiez la méthode de communication de transport.

    Le service de protocole de transport envoie les messages. Les valeurs autorisées sont les suivantes : smtp et smtps. Par défaut, le protocole de transport est smtp.

  13. Dans le champ Classe de protocole de transport, spécifiez la classe d'implémentation du fournisseur de services de transport.

    La valeur par défaut est com.sun.mail.smtp.SMTPTransport.

  14. Cochez la case Déboguer pour activer une sortie de débogage supplémentaire, notamment une trace de protocole, pour cette session de messagerie.

    Si le niveau de journalisation JavaMail a la valeur FINE ou plus fin, la sortie de débogage est créée et incluse dans le fichier journal du système. Pour plus d'informations sur la configuration du niveau de journalisation, reportez-vous à Configurer les niveaux de journalisation. Cette option est désactivée par défaut.

  15. Dans la table Autres propriétés, spécifiez les propriétés supplémentaires requises par les applications, notamment une propriété d'hôte ou de nom utilisateur propre à un protocole.

    Pour obtenir la liste des propriétés disponibles, reportez-vous à la documentation de l'API JavaMail (http://java.sun.com/products/javamail/javadocs/index.htmll). Les propriétés supplémentaires doivent commencer par mail- et des traits d'union doivent être utilisés comme séparateurs à la place des points. Par exemple, mail-debug est correct mais mail.debug ne l'est pas. L'arrière-plan de GlassFish Server convertit les traits d'union en points, attendus par l'API JavaMail.

    Pour ajouter une propriété, cliquez sur le bouton Ajouter une propriété. Sur la ligne vide qui apparaît, saisissez le nom de la propriété dans le champ Nom et la valeur de la propriété dans le champ Valeur.

  16. Dans la liste des cibles disponibles, sélectionnez les cibles souhaitées pour la configuration de la session de messagerie, puis cliquez sur Ajouter.

    Les cibles peuvent être des clusters ou des instances de serveur autonomes. Si vous ne sélectionnez aucune cible, la session est configurée en fonction de l'instance de serveur par défaut, server.

  17. Cliquez sur OK.
