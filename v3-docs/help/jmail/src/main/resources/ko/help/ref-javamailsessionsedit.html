JavaMail 세션 편집

JavaMail 세션 편집 페이지에서는 JavaMail 세션을 편집할 수 있습니다.

JavaMail 세션 편집 페이지에는 다음 옵션이 포함되어 있습니다.

기본값 로드

기본값이 있는 설정을 해당 기본값으로 복원하는 버튼입니다. 기본값이 없는 설정은 변경되지 않습니다.

JNDI 이름

JavaMail 세션을 식별하는 고유한 이름입니다. JNDI 이름은 읽기 전용 필드입니다. 새 JavaMail 세션을 만들 때만 JNDI 이름을 지정할 수 있습니다.

메일 호스트

기본 메일 서버의 호스트 이름입니다. 프로토콜 관련 호스트 등록 정보를 제공하지 않으면 저장소 및 전송 객체의 연결 메소드에서 이 값을 사용합니다. 이름을 실제 호스트 이름으로 확인할 수 있어야 합니다.

기본 사용자

메일 서버에 연결할 때 입력할 사용자 이름입니다. 프로토콜 관련 아이디 등록 정보를 제공하지 않으면 저장소 및 전송 객체의 연결 메소드에서 이 값을 사용합니다. 사용자 이름에는 영숫자, 밑줄, 대시 또는 점 문자만 사용해야 합니다.

기본 보낸 사람 주소

기본 사용자의 전자 메일 주소입니다. 형식은 username@host.domain입니다.

설명

이 메일 세션에 대한 설명입니다.

상태

메일 세션의 상태입니다. 메일 세션은 사용 가능하거나 사용 가능하지 않을 수 있습니다. 기본값은 사용 가능입니다.

저장소 프로토콜

메일 서버에 연결하여 메시지를 검색하고 하나 이상의 폴더에 메시지를 저장하는 저장소 프로토콜 서비스입니다. 허용되는 값은 imap, pop3, imaps·및 pop3s입니다. 기본값은 imap입니다.

저장소 프로토콜 클래스

저장소에 대한 서비스 공급자 구현 클래스입니다. 허용되는 값은 다음과 같습니다.

  • com.sun.mail.imap.IMAPStore

  • com.sun.mail.pop3.POP3Store

  • com.sun.mail.imap.IMAPSSLStore

  • com.sun.mail.pop3.POP3SSLStore

기본값은 com.sun.mail.imap.IMAPStore입니다.

전송 프로토콜

메시지를 보내는 전송 프로토콜 서비스입니다. 허용되는 값은 smtp 및 smtps입니다. 기본값은 smtp입니다.

전송 프로토콜 클래스

전송에 대한 서비스 공급자 구현 클래스입니다. 허용되는 값은 다음과 같습니다.

  • com.sun.mail.smtp.SMTPTransport

  • com.sun.mail.smtp.SMTPSSLTransport

기본값은 com.sun.mail.smtp.SMTPTransport입니다.

디버그

이 옵션을 선택하면 이 자원에 대한 디버깅이 활성화됩니다. JavaMail 로그 수준을 FINE 이상으로 설정한 경우 디버깅 출력이 생성되어 시스템 로그 파일에 포함됩니다. 로그 수준 설정에 대한 자세한 내용은 로그 수준을 구성하려면을 참조하십시오.

추가 등록 정보

JavaMail 세션에 대한 추가 등록 정보입니다. 사용 가능한 등록 정보 목록은 JavaMail API 설명서를 참조하십시오. 추가 등록 정보는 mail-로 시작해야 하며 마침표가 아닌 하이픈을 구분자로 사용해야 합니다. 예를 들어 mail-debug는 올바르며 mail.debug는 잘못되었습니다. GlassFish Server 백엔드는 하이픈을 JavaMail API에서 요구하는 마침표로 변환합니다.
