JavaMail 세션을 편집하는 방법

  1. 탐색 트리에서 자원 노드를 확장합니다.
  2. 자원 노드 아래에서 JavaMail 세션을 선택합니다.

    JavaMail 세션 페이지가 열립니다.

  3. JavaMail 세션 페이지에서 편집할 세션의 이름을 누릅니다.

    JavaMail 세션 편집 페이지가 열립니다.

  4. 메일 호스트 필드에서 기본 메일 서버의 호스트 이름을 지정합니다.

    프로토콜 관련 호스트 등록 정보를 제공하지 않으면 저장소 및 전송 객체의 연결 메소드에서 이 값을 사용합니다. 이름을 실제 호스트 이름으로 확인할 수 있어야 합니다.

  5. 기본 사용자 필드에 메일 서버에 연결할 때 제공할 다른 사용자 이름을 입력합니다.

    프로토콜 관련 아이디 등록 정보를 제공하지 않으면 저장소 및 전송 객체의 연결 메소드에서 이 값을 사용합니다. 사용자 이름에는 영숫자, 밑줄, 대시 또는 점 문자만 사용해야 합니다.

  6. 기본 보낸 사람 주소 필드에 기본 사용자의 전자 메일 주소를 입력합니다.

    형식은 username@host.domain입니다.

  7. 설명 필드에서 메일 세션에 대한 설명을 변경합니다.
  8. 메일 세션을 활성화하려면 상태 사용 가능 확인란을 선택합니다.

    기본값은 사용 가능입니다.

  9. 저장소 프로토콜 필드에서 사용할 저장소 객체 통신 방법을 정의합니다.

    기본적으로 저장소 프로토콜은 imap로 지정됩니다.

  10. 저장소 프로토콜 클래스 필드에서 원하는 저장소 프로토콜을 구현하는 저장소 통신 방법 클래스를 지정합니다.

    기본적으로 저장소 프로토콜 클래스는 com.sun.mail.imap.IMAPStore로 지정됩니다.

  11. 전송 프로토콜 필드에서 전송 통신 방법을 지정합니다.

    기본적으로 전송 프로토콜은 smtp로 지정됩니다.

  12. 전송 프로토콜 클래스 필드에서 전송 클래스에 대한 통신 방법을 지정합니다.

    기본적으로 전송 프로토콜 클래스는 com.sun.mail.smtp.SMTPTransport 입니다.

  13. 이 메일 세션에 대해 프로토콜 추적을 비롯한 추가 디버깅 출력을 활성화하려면 디버그 확인란을 선택하십시오.

    JavaMail 로그 수준을 FINE 이상으로 설정한 경우 디버깅 출력이 생성되어 시스템 로그 파일에 포함됩니다. 로그 수준 설정에 대한 자세한 내용은 To Configure Log Levels을 참조하십시오.

  14. (옵션) 추가 등록 정보 표에서 응용 프로그램에 필요한 추가 등록 정보(예: 프로토콜별 호스트 또는 사용자 이름 등록 정보)를 지정합니다.

    사용 가능한 등록 정보 목록은 JavaMail API 설명서를 참조하십시오. 추가 등록 정보는 mail-로 시작해야 하며 마침표가 아닌 하이픈을 구분자로 사용해야 합니다. 예를 들어 mail-debug는 올바르며 mail.debug는 잘못되었습니다. GlassFish 서버 백엔드는 하이픈을 JavaMail API에서 요구하는 마침표로 변환합니다.

    • 등록 정보를 추가하려면 등록 정보 추가 버튼을 누릅니다. 빈 행이 표시되면 이름 필드에 등록 정보 이름을 입력하고 값 필드에 등록 정보 값을 입력합니다.
    • 등록 정보를 수정하려면 등록 정보의 값 필드를 편집합니다.
    • 등록 정보를 삭제하려면 삭제할 등록 정보의 이름 필드 왼쪽에 있는 확인란을 선택한 다음 등록 정보 삭제 버튼을 누릅니다.
  15. 저장을 누릅니다.

    참고 - 기본 설정으로 돌아가려면 기본값 로드 버튼을 누른 다음 저장을 누릅니다.
