JavaMail-Session bearbeiten

Auf der Seite "JavaMail-Session bearbeiten" können Sie die Einstellungen einer JavaMail-Session ändern.

Die Seite "JavaMail-Session bearbeiten" enthält die folgenden Optionen.

Standardeinstellungen laden

Schaltfläche zum Wiederherstellen der Standardwerte für Einstellungen, für die es Standardwerte gibt. Einstellungen, die nicht über Standardwerte verfügen, können nicht geändert werden.

JNDI Name (JNDI-Name)

Ein eindeutiger Name, der die JavaMail-Session identifiziert. Der JNDI-Name ist ein schreibgeschütztes Feld. Sie können den JNDI-Namen nur festlegen, wenn Sie eine neue JavaMail-Session erstellen.

Mail-Host

Der Hostname des Standard-Mailservers. Die Verbindungsmethoden der Speicher- und Transportobjekte verwenden diesen Wert, wenn keine protokollspezifische Hosteigenschaft angegeben ist. Der Name muss in einen tatsächlichen Hostnamen auflösbar sein.

Standardbenutzer

Der Benutzername, der bei der Verbindungsherstellung mit einem Mailserver angegeben werden muss. Die Verbindungsmethode der Speicher- und Transportobjekte verwendet diesen Wert, wenn keine protokollspezifische Benutzernameneigenschaft angegeben ist. Der Benutzername darf nur alphanumerische Zeichen, Unterstriche, Gedankenstriche oder Punkte enthalten.

Standardabsenderadresse

Die E-Mail-Adresse des Standardbenutzers. Das Format ist username@host.domain.

Beschreibung

Eine Beschreibung der Mailsession.

Status

Der Status der Mailsession. Die Mailsession kann aktiviert oder deaktiviert sein. Der Standardwert ist "Aktiviert".

Speicherprotokoll

Der Speicherprotokollservice, der eine Verbindung zu einem Mailserver herstellt, Nachrichten abruft und Nachrichten in einem oder mehreren Ordnern speichert. Zulässige Werte sind imap, pop3, imapsund pop3s. Der Standardwert ist imap.

Speicherprotokollklasse

Die Serviceprovider-Implementierungsklasse für die Speicherung. Zulässige Werte sind:

  • com.sun.mail.imap.IMAPStore

  • com.sun.mail.pop3.POP3Store

  • com.sun.mail.imap.IMAPSSLStore

  • com.sun.mail.pop3.POP3SSLStore

Der Standardwert ist com.sun.mail.imap.IMAPStore.

Transportprotokoll

Der Transportprotokollservice, der Nachrichten sendet. Zulässige Werte sind smtp und smtps. Der Standardwert ist smtp.

Transportprotokollklasse

Die Serviceprovider-Implementierungsklasse für den Transport. Zulässige Werte sind:

  • com.sun.mail.smtp.SMTPTransport

  • com.sun.mail.smtp.SMTPSSLTransport

Der Standardwert ist com.sun.mail.smtp.SMTPTransport.

Debug (Fehlersuche)

Bei aktivierter Option ist das Debugging für diese Ressource aktiviert. Wenn die JavaMail-Logebene auf FINE (Genau) oder höher festgelegt ist, wird eine Debugging-Ausgabe generiert und in die Systemlogdatei geschrieben. Weitere Informationen zur Festlegung der Logebene finden Sie unter So konfigurieren Sie Logebenen.

Weitere Eigenschaften

Zusätzliche Eigenschaften für die JavaMail-Session. Eine Liste der verfügbaren Eigenschaften finden Sie in der JavaMail-API-Dokumentation (http://java.sun.com/products/javamail/javadocs/index.html). Zusätzliche Eigenschaften müssen mit mail- beginnen und Bindestriche anstelle von Punkten als Trennzeichen verwenden. Richtig ist z.B. mail-debug, während mail.debug falsch ist. Das GlassFish Server-Backend wandelt die Bindestriche in die von der JavaMail-API erwarteten Punkte um.

  • So wird eine JavaMail-Session bearbeitet

  • So wird eine JavaMail-Session gelöscht
