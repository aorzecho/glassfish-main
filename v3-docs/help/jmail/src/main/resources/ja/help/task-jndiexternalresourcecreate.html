外部JNDIリソースを作成する

外部JNDIリソースは、LDAPサーバーなどの外部JNDIリポジトリにアクセスします。

  1. ナビゲーション・ツリーで、「リソース」ノードを展開します。

  2. 「リソース」ノードで、「JNDI」ノードを展開します。

  3. 「JNDI」ノードで、「外部リソース」を選択します。

    「外部リソース」ページが開きます。

  4. 「外部リソース」ページで、「新規」をクリックします。

    「新しい外部リソース」ページが開きます。

  5. 「JNDI名」フィールドに、一意の名前を入力します。

    この名前はJNDIネーミング・サービスに登録されます。カード式の目録によって図書館の蔵書の分類と保管場所の管理を行う場合と同様の方法で、分散コンピューティング環境に配置されたコンポーネントの編成と場所の管理がJNDI名によって行われます。そのため、JNDI名はリソースへの重要なアクセス方法となります。

  6. 「リソース・タイプ」フィールドで、次のいずれかの操作を行います。

    • ドロップダウン・リストから、タイプ定義を選択します。

    • 2番目のラジオ・ボタンを選択し、完全修飾タイプ定義をxxx.xxxの形式で入力します。

  7. 「ファクトリ・クラス」フィールドに、作成する外部リソースのJNDIファクトリ・クラス外部リポジトリを入力します。

    「ファクトリ・クラス」は、com.sun.jndi.ldapなどの、ファクトリ・クラスのユーザー指定の名前です。このクラスはjavax.naming.spi.InitialContextFactoryインタフェースを実装します。

  8. 「JNDIルックアップ」フィールドに、外部リポジトリで検索するJNDI値を入力します。

    たとえば、外部リポジトリに接続してBeanクラスをテストする外部リソースを作成する場合、「JNDIルックアップ」はcn=testmybeanとなります。

  9. 「説明」フィールドに、リソースの簡単な説明を入力します。

    この説明は文字列値で、最大250文字を入力することができます。

  10. 「ステータス」の「有効」チェックボックスを選択し、リソースを有効にします。

    デフォルトでは、リソースは有効で、作成後すぐに使用できます。

  11. 「追加プロパティ」セクションで、外部リソースの追加プロパティを指定します。

    GlassFish Serverは外部リソースの追加プロパティを定義しません。

    プロパティを追加するには、「プロパティを追加」ボタンをクリックします。表示される空白行で、「名前」フィールドにプロパティ名を入力し、「値」フィールドにプロパティ値を入力します。

  12. 「OK」をクリックします。
