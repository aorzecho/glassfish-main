Modificare una sessione JavaMail

  1. Nell'albero di navigazione espandere il nodo Risorse.

  2. Nel nodo Risorse, selezionare Sessioni JavaMail.

    Viene visualizzata la pagina Sessioni JavaMail.

  3. Nella pagina Sessioni JavaMail fare clic sul nome della sessione da modificare.

    Viene visualizzata la pagina Modifica sessione JavaMail.

  4. Nel campo Host posta specificare il nome host del server di posta elettronica predefinito.

    Se non viene fornita una proprietà host specifica per un protocollo, i metodi di connessione degli oggetti Store e Transport utilizzano questo valore. Il nome deve essere risolvibile in un nome host reale.

  5. Nel campo Utente predefinito digitare un altro nome utente da fornire per la connessione a un server di posta elettronica.

    Se non viene fornita una proprietà nome utente specifica per un protocollo, i metodi di connessione degli oggetti Store e Transport utilizzano questo valore. Il nome utente può contenere solo caratteri alfanumerici, caratteri di sottolineatura, trattini e punti.

  6. Nel campo Indirizzo mittente predefinito, digitare l'indirizzo di posta elettronica dell'utente predefinito.

    Il formato è nome utente@host.dominio.

  7. Nel campo Descrizione modificare la descrizione per la sessione di posta elettronica.

  8. Selezionare la casella di controllo Stato abilitato per abilitare la sessione di posta elettronica.

    Il valore predefinito è Abilitato.

  9. Nel campo Protocollo di memorizzazione definire il metodo di comunicazione oggetto Store da utilizzare.

    Per impostazione predefinita il protocollo di memorizzazione è imap.

  10. Nel campo Classe protocollo di memorizzazione specificare la classe del metodo di comunicazione Store che implementa il protocollo Store desiderato.

    Per impostazione predefinita la classe protocollo di memorizzazione è com.sun.mail.imap.IMAPStore.

  11. Nel campo Protocollo di trasporto, specificare il metodo di comunicazione di trasporto.

    Per impostazione predefinita, il protocollo di trasporto è smtp.

  12. Nel campo Classe protocollo di trasporto specificare il metodo di comunicazione per la classe Transport.

    Per impostazione predefinita la classe protocollo di trasporto è com.sun.mail.smtp.SMTPTransport.

  13. Selezionare la casella di controllo Debug per abilitare output di debugging aggiuntivo, che include una traccia di protocollo per la sessione di posta elettronica corrente.

    Se il livello del log JavaMail è impostato su FINE o su un valore superiore, l'output di debugging viene generato e incluso nel file di log di sistema. Per informazioni sull'impostazione del livello del log, vedere To Configure Log Levels.

  14. Nella tabella Proprietà aggiuntive, specificare le proprietà aggiuntive necessarie per le applicazioni, ad esempio una proprietà host o nome utente specifica del protocollo.

    Per una lista delle proprietà disponibili, vedere la documentazione dell'API JavaMail (http://java.sun.com/products/javamail/javadocs/index.html). Le proprietà aggiuntive devono iniziare con mail- e come separatori devono utilizzare trattini, e non punti. Ad esempio, mail-debug è una proprietà corretta, mentre mail.debug non lo è. Il back-end di GlassFish Server converte i trattini nei punti previsti per l'API JavaMail.

    • Per aggiungere una proprietà, fare clic sul pulsante Aggiungi proprietà. Nella riga vuota visualizzata, digitare il nome della proprietà nel campo Nome e il valore della proprietà nel campo Valore.

    • Per modificare una proprietà, è necessario modificare il valore del campo Valore della proprietà.

    • Per eliminare una proprietà, selezionare la casella di controllo a sinistra del campo Nome relativo alla proprietà da eliminare, quindi fare clic sul pulsante Elimina proprietà.

  15. Fare clic su Salva.

    Suggerimento:

    Per tornare alle impostazioni predefinite, fare clic sul pulsante Carica impostazioni predefinite, quindi su Salva.
