New JavaMail Session

Nella pagina New JavaMail Session è possibile creare una nuova risorsa di sessione JavaMail.

La pagina New JavaMail Session contiene le seguenti opzioni.

JNDI Name

Nome univoco che identifica la sessione di posta elettronica. Utilizzare il prefisso di sottocontesto di denominazione mail/ per risorse JavaMail, ad esempio mail/MySession. Il nome può contenere solo caratteri alfanumerici, caratteri di sottolineatura, trattini e punti.

Mail Host

Nome host del server di posta elettronica predefinito. Se non viene fornita una proprietà host specifica per un protocollo, i metodi di connessione degli oggetti Store e Transport utilizzano questo valore. Il nome deve essere risolvibile in un nome host reale.

Default User

Nome utente da specificare al momento della connessione a un server di posta elettronica. Se non viene fornita una proprietà nome utente specifica per un protocollo, i metodi di connessione degli oggetti Store e Transport utilizzano questo valore. Il nome utente può contenere solo caratteri alfanumerici, caratteri di sottolineatura, trattini e punti.

Default Sender Address

Indirizzo di posta elettronica dell'utente predefinito. Il formato è username@host. domain.

Description

Descrizione della sessione di posta elettronica.

Status

Status della sessione di posta elettronica. La sessione di posta elettronica può essere abilitata o disabilitata. Il valore predefinito è Enabled.

Store Protocol

Servizio del protocollo di memorizzazione con cui viene eseguita la connessione a un server di posta elettronica, il recupero dei messaggi e il salvataggio dei messaggi in una o più cartelle. I valori consentiti sono imap, pop3, imaps e pop3s. Il valore predefinito è imap.

Store Protocol Class

Classe di implementazione del provider di servizi per la memorizzazione. I valori consentiti sono:

  • com.sun.mail.imap.IMAPStore

  • com.sun.mail.pop3.POP3Store

  • com.sun.mail.imap.IMAPSSLStore

  • com.sun.mail.pop3.POP3SSLStore

Il valore predefinito è com.sun.mail.imap.IMAPStore.

Transport Protocol

Servizio del protocollo di trasporto, che invia messaggi. I valori consentiti sono smtp e smtps. Il valore predefinito è smtp.

Transport Protocol Class

Classe di implementazione del provider di servizi per il trasporto. I valori consentiti sono:

  • com.sun.mail.smtp.SMTPTransport

  • com.sun.mail.smtp.SMTPSSLTransport

Il valore predefinito è com.sun.mail.smtp.SMTPTransport.

Debug

Se l'opzione è selezionata, il debugging per la risorsa è abilitato. Se il livello del log JavaMail è impostato su FINE o su un valore superiore, l'output di debugging viene generato e incluso nel file del log di sistema. Per informazioni sull'impostazione del livello del log, vedere To Configure Log Levels.

Additional Properties

Proprietà aggiuntive per la sessione JavaMail. Per un elenco di proprietà disponibili, vedere la documentazione dell'API JavaMail. Le proprietà aggiuntive devono iniziare con mail- e come separatori devono utilizzare trattini, e non punti. Ad esempio, mail-debug è una proprietà corretta, mentre mail.debug non lo è. Il back-end di GlassFish Server converte i trattini nei punti previsti per l'API JavaMail.

Targets

Cluster e istanze standalone per la sessione JavaMail. Per spostare le destinazioni desiderate nella colonna Selected Targets, utilizzare i pulsanti Add e Add All. Per spostare le destinazioni non necessarie nella colonna Available Targets, utilizzare i pulsanti Remove e Remove All.

Questa opzione viene visualizzata solo se nel dominio sono stati creati cluster o istanze standalone.
