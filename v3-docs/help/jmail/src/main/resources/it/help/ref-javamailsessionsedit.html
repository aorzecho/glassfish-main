Modifica sessione JavaMail

Nella pagina Modifica sessione JavaMail è possibile modificare una sessione JavaMail.

La pagina Modifica sessione JavaMail contiene le opzioni riportate di seguito.

Carica impostazioni predefinite

Pulsante che consente di ripristinare i valori predefiniti delle impostazioni che dispongono di tali valori. Le impostazioni prive di valori predefiniti restano invariate.

Nome JNDI

Nome univoco che identifica la sessione JavaMail. Il campo Nome JNDI è di sola lettura. È possibile specificare il nome JNDI solo quando si crea una nuova sessione JavaMail.

Host posta

Nome host del server di posta predefinito. Se non viene fornita una proprietà host specifica del protocollo, i metodi di connessione degli oggetti area di memorizzazione e trasporto utilizzano questo valore. Il nome deve essere risolvibile in un nome host effettivo.

Utente predefinito

Nome utente da fornire durante la connessione a un server di posta. Se non viene fornita una proprietà nome utente specifica del protocollo, i metodi di connessione degli oggetti area di memorizzazione e trasporto utilizzano questo valore. Il nome utente può contenere solo caratteri alfanumerici, caratteri di sottolineatura, trattini o punti.

Indirizzo mittente predefinito

Indirizzo di posta elettronica dell'utente predefinito. Il formato è nomeutente@host.dominio.

Descrizione

Descrizione della sessione di posta corrente.

Stato

Stato della sessione di posta. La sessione di posta può essere abilitata o disabilitata. Il valore predefinito è Abilitato.

Protocollo di memorizzazione

Servizio del protocollo di memorizzazione che consente la connessione a un server di posta, il recupero dei messaggi e il salvataggio dei messaggi in una o più cartelle. I valori consentiti sono imap, pop3, imaps e pop3s. Il valore predefinito è imap.

Classe protocollo di memorizzazione

Classe di implementazione del provider di servizi per la memorizzazione. I valori consentiti sono:

  • com.sun.mail.imap.IMAPStore

  • com.sun.mail.pop3.POP3Store

  • com.sun.mail.imap.IMAPSSLStore

  • com.sun.mail.pop3.POP3SSLStore

Il valore predefinito è com.sun.mail.imap.IMAPStore.

Protocollo di trasporto

Servizio del protocollo di trasporto, che invia messaggi. I valori consentiti sono smtp e smtps. Il valore predefinito è smtp.

Classe protocollo di trasporto

Classe di implementazione del provider di servizi per il trasporto. I valori consentiti sono:

  • com.sun.mail.smtp.SMTPTransport

  • com.sun.mail.smtp.SMTPSSLTransport

Il valore predefinito è com.sun.mail.smtp.SMTPTransport.

Debug

Se questa opzione è selezionata, il debug per la risorsa corrente è abilitato. Se il livello del log JavaMail è impostato su FINE o FINER, l'output di debug viene generato e incluso nel file di log di sistema. Per informazioni sull'impostazione dei livelli del log, vedere To Configure Log Levels.

Proprietà aggiuntive

Proprietà aggiuntive per la sessione JavaMail. Per una lista delle proprietà disponibili, consultare la documentazione dell'API JavaMail (http://java.sun.com/products/javamail/javadocs/index.html). Le proprietà aggiuntive devono iniziare con mail- e come separatori devono utilizzare trattini e non punti. Ad esempio, mail-debug è una proprietà corretta, mentre mail.debug non lo è. Il processo backend di GlassFish Server converte i trattini nei punti previsti dall'API JavaMail.
