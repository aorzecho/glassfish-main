Crear una Sesión de JavaMail

  1. En el árbol de navegación, amplíe el nodo Recursos.

  2. En el nodo Recursos, seleccione Sesiones JavaMail.

    Se abrirá la página Sesiones JavaMail.

  3. En la tabla Sesiones, haga clic en Nuevo.

    Se abrirá la página Nueva Sesión de JavaMail.

  4. En el campo Nombre JNDI, introduzca un nombre único para el recurso de sesión de correo.

    Utilice el prefijo de subcontexto de nombres mail/ para recursos de JavaMail. Por ejemplo: mail/MySession. El nombre debe contener únicamente caracteres alfanuméricos, de subrayado, guiones o puntos.

  5. En el campo Host de Correo, escriba el nombre de host del servidor de correo por defecto.

    Los métodos de conexión de los objetos Almacenamiento y Transporte utilizan este valor si no se suministra una propiedad de host específica de protocolo. El nombre debe poder resolverse en un nombre de host real.

  6. En el campo Usuario por Defecto escriba el nombre de usuario que se debe proporcionar cuando se conecta a un servidor de correo electrónico.

    Los métodos de conexión de los objetos Almacenamiento y Transporte utilizan este valor si no se suministra una propiedad de nombre de usuario específica del protocolo. El nombre de usuario debe contener únicamente caracteres alfanuméricos, de subrayado, guiones o puntos.

  7. En el campo Dirección de Remitente por Defecto, escriba la dirección de correo electrónico del usuario por defecto.

    El formato es username@host.domain.

  8. En el campo Descripción, escriba una descripción de esta sesión de correo.

  9. Seleccione la casilla de verificación de activación de Estado para activar la sesión de correo.

    El valor por defecto es Activado.

  10. En el campo Protocolo de Almacenamiento, defina el servicio de protocolo de almacenamiento que se va a utilizar.

    El servicio de protocolo de almacenamiento se conecta a un servidor de correo, recupera mensajes y los guarda en una o varias carpetas. Los valores permitidos son imap, pop3, imaps y pop3s. El valor por defecto es imap.

  11. En el campo Clase de Protocolo de Almacenamiento, especifique la clase de implementación del proveedor de servicios para el almacenamiento.

    Los valores permitidos son:

    • com.sun.mail.imap.IMAPStore

    • com.sun.mail.pop3.POP3Store

    • com.sun.mail.imap.IMAPSSLStore

    • com.sun.mail.pop3.POP3SSLStore

    El valor por defecto es com.sun.mail.imap.IMAPStore.

  12. En el campo Protocolo de Transporte, especifique el método de comunicación de transporte.

    El servicio de protocolo de transporte envía mensajes. Los valores permitidos son smtp y smtps. Por defecto, el Protocolo de Transporte es smtp.

  13. En el campo Clase de Protocolo de Transporte, especifique la clase de implementación del proveedor de servicios para el transporte.

    El valor por defecto es com.sun.mail.smtp.SMTPTransport.

  14. Seleccione la casilla de verificación Depurar para activar una salida de depuración adicional para esta sesión de correo, incluido un rastreo de protocolo.

    Si el nivel de log de JavaMail se ha definido como FINE o FINER, la salida de depuración se genera y se incluye en el archivo log del sistema. Vea Configurar Niveles de Log para obtener información sobre la configuración del nivel de log. Esta opción se encuentra desactivada por defecto.

  15. En la tabla Propiedades Adicionales, especifique otras propiedades requeridas por las aplicaciones, como una propiedad de host o nombre de usuario específico del protocolo.

    Para obtener una lista de las propiedades disponibles, consulte la documentación de la API de JavaMail (http://java.sun.com/products/javamail/javadocs/index.html). Las propiedades adicionales deben comenzar por mail- y como separadores se deben utilizar guiones en lugar de puntos. Por ejemplo, mail-debug es correcta, pero mail.debug no lo es. El back end de GlassFish Server convierte los guiones en los puntos que espera la API de JavaMail.

    Para agregar una propiedad, haga clic en el botón Agregar Propiedad. En la fila vacía, escriba el nombre de la propiedad en el campo Nombre y el valor de la propiedad en el campo Valor.

  16. En la lista de destinos disponibles, seleccione el destino o los destinos en los que desea configurar la sesión de correo. A continuación, haga clic en Agregar.

    Los destinos pueden ser clusters o instancias de servidor independientes. Si no selecciona un destino, la sesión se configura en la instancia de servidor por defecto server.

  17. Haga clic en Aceptar.
