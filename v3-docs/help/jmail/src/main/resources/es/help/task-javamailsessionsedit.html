Editar una Sesión de JavaMail

  1. En el árbol de navegación, amplíe el nodo Recursos.

  2. En el nodo Recursos, seleccione Sesiones JavaMail.

    Se abrirá la página Sesiones JavaMail.

  3. En la página Sesiones JavaMail, haga clic en el nombre de la sesión que esté editando.

    Se abrirá la página Editar Sesión JavaMail.

  4. En el campo Host de Correo, especifique el nombre de host del servidor de correo por defecto.

    Los métodos de conexión de los objetos Almacenamiento y Transporte utilizan este valor si no se suministra una propiedad de host específica de protocolo. El nombre debe poder resolverse en un nombre de host real.

  5. En el campo Usuario por Defecto, escriba otro nombre de usuario que se debe proporcionar al conectarse a un servidor de correo electrónico.

    Los métodos de conexión de los objetos Almacenamiento y Transporte utilizan este valor si no se suministra una propiedad de nombre de usuario específica del protocolo. El nombre de usuario debe contener únicamente caracteres alfanuméricos, de subrayado, guiones o puntos.

  6. En el campo Dirección de Remitente por Defecto, escriba la dirección de correo electrónico del usuario por defecto.

    El formato es username@host.domain.

  7. En el campo Descripción, cambie la descripción de la sesión de correo.

  8. Seleccione la casilla de verificación de activación de Estado para activar la sesión de correo.

    El valor por defecto es Activado.

  9. En el campo Protocolo de Almacenamiento, defina el método de comunicación de objetos de almacenamiento que se va a utilizar.

    Por defecto, el Protocolo de Almacenamiento es imap.

  10. En el campo Clase de Protocolo de Almacenamiento, especifique la clase del método de comunicación de almacenamiento que implementa el protocolo de almacenamiento deseado.

    Por defecto, la Clase de Protocolo de Almacenamiento es com.sun.mail.imap.IMAPStore.

  11. En el campo Protocolo de Transporte, especifique el método de comunicación de transporte.

    Por defecto, el Protocolo de Transporte es smtp.

  12. En el campo Clase de Protocolo de Transporte, especifique el método de comunicación para la clase de transporte.

    Por defecto, la Clase de Protocolo de Transporte es com.sun.mail.smtp.SMTPTransport .

  13. Seleccione la casilla de verificación Depurar para activar una salida de depuración adicional para esta sesión de correo, incluido un rastreo de protocolo.

    Si el nivel de log de JavaMail se ha definido como FINE o FINER, la salida de depuración se genera y se incluye en el archivo log del sistema. Vea Configurar Niveles de Log para obtener información sobre la definición del nivel de log.

  14. En la tabla Propiedades Adicionales, especifique otras propiedades requeridas por las aplicaciones, como una propiedad de host o nombre de usuario específico del protocolo.

    Para obtener una lista de las propiedades disponibles, consulte la documentación de la API de JavaMail (http://java.sun.com/products/javamail/javadocs/index.html). Las propiedades adicionales deben comenzar por mail- y como separadores se deben utilizar guiones en lugar de puntos. Por ejemplo, mail-debug es correcta, pero mail.debug no lo es. El back end de GlassFish Server convierte los guiones en los puntos que espera la API de JavaMail.

    • Para agregar una propiedad, haga clic en el botón Agregar Propiedad. En la fila en blanco que aparece, escriba el nombre de propiedad en el campo Nombre y escriba el valor de propiedad en el campo Valor.

    • Para modificar una propiedad, edite su campo Valor.

    • Para suprimir una propiedad, seleccione la casilla de verificación a la izquierda del campo Nombre de la propiedad que está suprimiendo y, a continuación, haga clic en el botón Suprimir Propiedades.

  15. Haga clic en Guardar.

    Sugerencia:

    Para volver a la configuración por defecto, haga clic en el botón Cargar Valores por Defecto y, a continuación, en Guardar.
