Para editar una sesión JavaMail

  1. En el árbol de navegación, amplíe el nodo Recursos.
  2. En el nodo Recursos, seleccione Sesiones JavaMail.

    Se abrirá la página Sesiones JavaMail.

  3. En la página Sesiones JavaMail, haga clic en el nombre de la sesión que esté editando.

    Se abrirá la página Editar sesiones JavaMail.

  4. En el campo Host de correo, especifique el nombre de host del servidor de correo predeterminado .

    Los métodos de conexión de los objetos Almacenamiento y Transporte utilizan este valor si no se suministra una propiedad de host específica de protocolo. El nombre debe ser soluble en un nombre de host real.

  5. En el campo Usuario predeterminado, escriba otro nombre de usuario que se debe proporcionar al conectarse a un servidor de correo electrónico.

    Los métodos de conexión de los objetos Almacenamiento y Transporte utilizan este valor si no se suministra una propiedad de nombre de usuario específica del protocolo. El nombre de usuario debe contener únicamente caracteres alfanuméricos, de subrayado, guiones y puntos.

  6. En el campo Dirección de remitente predeterminada, escriba la dirección de correo electrónico del usuario predeterminado.

    El formato es nombreusuario@host.dominio.

  7. En el campo Descripción, cambie la descripción de la sesión de correo electrónico.
  8. Seleccione la casilla de verificación Estado habilitado para habilitar la sesión de correo electrónico.

    El valor predeterminado es Habilitado.

  9. En el campo Protocolo de almacenamiento, defina el método de comunicación de objetos de almacenamiento que se va a utilizar.

    Por defecto, el Protocolo de almacenamiento es imap.

  10. En el campo Clase de protocolo de almacenamiento, especifique la clase del método de comunicación de almacenamiento que implementa el protocolo de almacenamiento deseado.

    Por defecto, la Clase de protocolo de almacenamiento es com.sun.mail.imap.IMAPStore.

  11. En el campo Protocolo de transporte, especifique el método de comunicación de transporte.

    Por defecto, el Protocolo de transporte es smtp.

  12. En el campo Clase de protocolo de transporte, especifique el método de comunicación para la clase de transporte.

    Por defecto, la Clase de protocolo de transporte es com.sun.mail.smtp.SMTPTransport .

  13. Seleccione la casilla de verificación Depurar para habilitar para esta sesión de correo electrónico una salida de depuración adicional, incluido un seguimiento de protocolo.

    Si el nivel de registro de JavaMail se ha definido como FINE o FINER, la salida de depuración se genera y se incluye en el archivo de registros del sistema. Vea Configurar niveles de registro para obtener información sobre la definición del nivel de registro.

  14. (Opcional) En la tabla Propiedades adicionales, especifique otras propiedades requeridas por las aplicaciones, como una propiedad de host o nombre de usuario específico del protocolo.

    Para obtener una lista de las propiedades disponibles, consulte la documentación de la API de JavaMail. Las propiedades adicionales deben comenzar por mail- y como separadores se deben utilizar guiones en lugar de puntos. Por ejemplo, mail-debug es correcta, pero mail.debug no lo es. El back end de GlassFish Server convierte los guiones en los puntos que espera la API de JavaMail.

    • Para añadir una propiedad, haga clic en el botón Añadir propiedad. En la fila vacía escriba el nombre de la propiedad en el campo Nombre y el valor de la propiedad en el campo Valor.
    • Para modificar una propiedad, edite su campo Valor.
    • Para eliminar una propiedad, marque la casilla de verificación situada a la izquierda del campo Nombre de la propiedad que quiera eliminar y, a continuación, haga clic en el botón Eliminar propiedades.
  15. Haga clic en Guardar.

    Consejo - Para volver a los valores de configuración predeterminados, haga clic en el botón Cargar predeterminados y, a continuación, en Guardar.
