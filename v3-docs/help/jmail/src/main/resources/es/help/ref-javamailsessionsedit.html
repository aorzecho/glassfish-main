Editar sesiones JavaMail

Utilice la página Editar sesiones JavaMail para editar una sesión JavaMail.

La página Editar sesiones JavaMail contiene las siguientes opciones.

Cargar valores predeterminados

Botón para restablecer las opciones que tienen valores por defecto a estos. Las opciones que no tienen valores por defecto no se modifican.

Nombre JNDI

Un nombre exclusivo que identifica las sesiones JavaMail. El nombre JNDI es un campo de sólo lectura. Únicamente puede especificar el nombre JNDI al crear una nueva sesión JavaMail.

Host de correo

El host del servidor de correo predeterminado. Los métodos de conexión de los objetos Almacenamiento y Transporte utilizan este valor si no se suministra una propiedad de host específica de protocolo. El nombre debe ser soluble en un nombre de host real.

Usuario predeterminado

El nombre de usuario que se debe proporcionar cuando se conecta a un servidor de correo. Los métodos de conexión de los objetos Almacenamiento y Transporte utilizan este valor si no se suministra una propiedad de nombre de usuario específica del protocolo. El nombre de usuario debe contener únicamente caracteres alfanuméricos, de subrayado, guiones y puntos.

Dirección de remitente predeterminada

La dirección de correo electrónico del usuario predeterminado. El formato es nombreusuario@host.dominio.

Descripción

Una descripción de esta sesión de correo electrónico.

Estado

El estado de la sesión de correo electrónico. La sesión de correo electrónico puede habilitarse o deshabilitarse. El valor predeterminado es Habilitado.

Protocolo de almacenamiento

El servicio de protocolo de almacenamiento, que se conecta con un servidor de correo electrónico, recupera mensajes y los guarda en una o varias carpetas. Los valores permitidos son imap, pop3, imaps y pop3s. El valor predeterminado es imap.

Clase de protocolo de almacenamiento

La clase de implementación del proveedor de servicios para el almacenamiento. Los valores permitidos son:

  • com.sun.mail.imap.IMAPStore

  • com.sun.mail.pop3.POP3Store

  • com.sun.mail.imap.IMAPSSLStore

  • com.sun.mail.pop3.POP3SSLStore

El valor predeterminado es com.sun.mail.imap.IMAPStore.

Protocolo de transporte

El servicio de protocolo de transporte, que envía mensajes. Los valores permitidos son smtp y smtps. El valor predeterminado es smtp.

Clase de protocolo de transporte

La clase de implementación del proveedor de servicios para el transporte. Los valores permitidos son:

  • com.sun.mail.smtp.SMTPTransport

  • com.sun.mail.smtp.SMTPSSLTransport

El valor predeterminado es com.sun.mail.smtp.SMTPTransport.

Depurar

Si esta opción está seleccionada, la depuración para este recurso está habilitada. Si el nivel de registro de JavaMail se ha definido como FINE o FINER, la salida de depuración se genera y se incluye en el archivo de registros del sistema. Vea Configurar niveles de registro para obtener información sobre la definición del nivel de registro.

Propiedades adicionales

Propiedades adicionales de las sesiones JavaMail. Para obtener una lista de las propiedades disponibles, consulte la documentación de la API de JavaMail. Las propiedades adicionales deben comenzar por mail- y como separadores se deben utilizar guiones en lugar de puntos. Por ejemplo, mail-debug es correcta, pero mail.debug no lo es. El back end de Servidor GlassFish convierte los guiones en los puntos que espera la API de JavaMail.
