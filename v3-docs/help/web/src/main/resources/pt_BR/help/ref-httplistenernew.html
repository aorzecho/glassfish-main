Novo Listener HTTP

Use a página Novo Listener HTTP para criar um novo listener HTTP.

Ao criar um listener HTTP, a guia SSL é disponibilizada para você também editar.

A página Novo Listener HTTP contém as seguintes opções.

Nome da Configuração

O nome da configuração à qual as definições nesta página se aplicam. Este campo é somente para leitura.

Nome

O nome único de um listener. O nome de um listener HTTP não pode começar com um número.

Porta

O número da porta do listener. Os valores válidos são 1 a 65.535. Em sistemas UNIX, criar soquetes que fazem listening nas portas 1 - 1024 exige privilégios de superusuário.

Status

O status do listener HTTP. O listener pode ser ativado ou desativado. Por default, a opção está ativada. Se o interpretador estiver desativado, qualquer tentativa de conectar ao listener resulta em uma exceção de soquete (java.net.ConnectException).

Nas versões do GlassFish Server anteriores a 9.1, um listener cujo atributo ativado estava definido como falso retornava um código de resposta 404, para quaisquer solicitações enviadas ao mesmo. Para obter esse comportamento na versão atual do GlassFish Server, defina o status do listener como ativado e cada estado de servidor virtual associado como DESLIGADO. Um servidor virtual lista suas listeners associadas na sua lista drop-down Listeners de rede.

Segurança

Se esta opção estiver selecionada, a segurança é ativada para o listener HTTP. Por default, essa opção está desativada.

Interpretador JK

Se esta opção estiver selecionada, o listener suportará o protocolo mod-jk do Apache. O protocolo mod-jk é um protocolo proprietário de communication/network entre o httpd do Apache (servidor da web) e uma instância de cack-end do GlassFish Server. Se o listener suportar este protocolo, httpd deverá suportar todo o conteúdo estático, enquanto qualquer solicitação de recurso dinâmico (ou seja, Servlets e JSPs) são roteados para o listener HTTP ativado do mod-jk do &Productname;, usando o protocolo mod-jk. Por default, essa opção está desativada.

Endereço

O endereço IP do listener de rede. O endereço pode estar na notação de par de pontos ou IPv6. Pode ser qualquer (para INADDR_ANY) para fazer listening em todos os endereços IP. Ele pode ser um nome de host.

Servidor Virtual Default

O servidor virtual a ser associado com este listener HTTP. Use a página Servidores Virtuais para defini-los.

Pool de Threads

O pool do thread associado com o listener HTTP. Normalmente, você seleciona um dos dois pool de threads que estão configurados quando o GlassFish Server é instalado.

Nome do servidor

O nome do host a ser utilizado nas URLs que o servidor envia ao cliente. Esse nome será o nome alias se o servidor usar um alias. Se o servidor não usar alias, deixe o campo em branco.

Esse valor somente afeta os URLs gerados automaticamente pelo servidor; ele não afeta os URLs para diretórios e arquivos armazenados no servidor. Caso seu servidor use um codinome, o nome do servidor deveria ser o nome do codinome. Se uma vírgula e um número de porta forem acrescentados, essa porta será usada nos URLs que o servidor envia para o cliente.
