SSL

Utilize a página SSL para modificar as configurações SSL para um protocolo de rede.

As configurações nessa página têm significado apenas se a segurança estiver ativada na página Editar Protocolo.

A página SSL contém estas opções.

Nome da configuração

Nome da configuração ao qual as configurações desta página serão enviadas. Este campo é de somente leitura.

SSL3

Se a caixa de seleção estiver selecionada, o protocolo SSL3 está disponível para o protocolo. Por padrão, a opção está ativada.

TLS

Se a caixa de seleção estiver selecionada, o protocolo TLS está disponível para o protocolo. Por padrão, a opção está ativada.

Autenticação do Cliente

Se a caixa de seleção estiver marcada, os clientes deverão identificar-se para o servidor a cada solicitação. Por padrão, a opção está desativada.

Apelido do certificado

O apelido do certificado do servidor no banco de dados de certificados ou o símbolo PKCS#11. No certificado, o formato do nome é tokenname:nickname. Incluir a parte do nome do símbolo: neste atributo é opcional.

Armazenamento de chave

O nome do arquivo de armazenamento de chave (por exemplo, keystore.jks).

Algoritmo confiável

O nome do algoritmo de gerenciamento confiável (por exemplo, PKIX) a ser usado para validação do caminho de certificação.

Comprimento máx. certificado

O número máximo de certificados intermediários não auto-emitidos que podem existir em um caminho de certificação. Esse campo é usado apenas se o campo Algoritmo confiável estiver definido como PKIX. O valor 0 indica que o caminho pode conter somente um certificado. O valor -1 indica que o comprimento do caminho não é restringido (não existe máximo). Definir um valor menor do que -1 causa uma exceção a ser lançada.

Armazenamento confiável

O nome do arquivo truststore (por exemplo, cacerts.jks).

Conjuntos codificados

Uma área onde você pode adicionar ou remover conjuntos codificados. Se você não adicionar quaisquer conjuntos codificados, todos os conjuntos codificados serão usados.
