Para configurar o serviço HTTP

  1. Na árvore de navegação, expanda o nó Configuração.
  2. No nó Configuração, selecione o nó Serviço HTTP.

    A página Serviço HTTP é exibida.

  3. Selecione a caixa de seleção SSO ativado, para ativar o logon único por padrão para todos os aplicativos da Web em todos os servidores virtuais neste servidor que estão configurados para o mesmo domínio.

    Você pode sobrepor esta configuração para um servidor virtual individual. Por padrão, a opção está desativada.

  4. Marque a caixa de seleção Log de Acesso Ativado para ativar o log de acesso para todos os subelementos de servidor virtual que não especificarem essa propriedade.

    Se a caixa de seleção Log de Acesso Ativado for desmarcada, o log de acesso será desativado para todos os subelementos de servidor virtual que não especificarem essa propriedade. Por padrão, a opção está desativada.

  5. Marque a caixa de seleção Rotação Ativada para ativar a circulação do arquivo de registro.

    Por padrão, a opção está ativada.

  6. Na lista suspensa Política de Rotação, selecione uma política.

    A única política de rotação disponível é tempo.

  7. No campo Intervalo de Rotação, digite o número de minutos entre as rotações do log de acesso.

    Esse campo só será válido se a Política de Rotação for tempo. O valor padrão é 1.440 minutos (24 horas).

  8. No campo Sufixo de Rotação, digite o valor de sequência a ser adicionado ao final do nome do arquivo de log, após a rotação.

    O valor padrão é dd-MM-yyyy.

    Verifique se o sufixo de rotação contém valores suficientes para criar um sufixo exclusivo para cada rotação de log. Por exemplo, se o log de acesso for circulado mais de uma vez ao dia, inclua a hora do dia no sufixo de rotação.

    Um sufixo exclusivo é necessário para assegurar que o arquivo de log de acesso seja circulado sempre que o intervalo de rotação for cumprido. Se o sufixo de rotação do novo arquivo de log de acesso for igual ao sufixo de rotação do arquivo de log existente, o arquivo não será circulado.

  9. No campo Contagem máxima de arquivos, digite o número máximo de arquivos de registro de acesso rotacionado a serem mantidos.

    Um valor negativo indica que não há limite para a contagem de arquivos. O valor padrão é -1.

  10. No campo Tamanho do buffer, digite o tamanho em bytes, do buffer de registros de acesso.

    Um valor menor que ou igual a 0 desativa o armazenamento em buffer de registros de acesso. O valor padrão é 32.768.

  11. No campo Intervalo de gravação, digite o número de segundos que antecede a gravação do registro em disco.

    O log de acesso é gravado quando o buffer está cheio ou quando o intervalo expira. Se o valor for menor que ou igual a 0, o buffer será sempre gravado, mesmo que não esteja cheio. Isso significa que, a cada vez que o servidor é acessado, a mensagem de log é armazenada diretamente no arquivo. O valor padrão é 300.

  12. No campo Formato, digite o valor de sequência para especificar as seguintes propriedades do registro de acesso:

    • As informações a serem capturadas no log de acesso

    • A ordem em que as informações serão capturadas

    O valor padrão é o seguinte:

    %client.name% %auth-user-name% %datetime% %request% %status% %response.length%

    Esses valores estão disponíveis para inclusão na sequência de formato:

    %attribute.name%

    Registra o valor (ao chamar seu método toString) do atributo de solicitação com o determinado nome

    %auth-user-name%

    Nome do usuário autorizado

    %client.name%

    Nome do host cliente

    %client.dns%

    DNS Cliente

    %cookie.name%

    O valor do cookie solicitado com o nome determinado

    %cookie.value%

    O valor do primeiro cookie encontrado na solicitação

    %datetime%

    Data do sistema

    %request%

    Linha de solicitação de HTTP completo

    %status%

    Status

    %response.length%

    Tamanho do conteúdo da resposta

    %header.referer%

    Cabeçalho do referenciador

    %header.user-agent%

    Cabeçalho de agente do usuário

    %http-method%

    Método HTTP

    %http-uri%

    URI HTTP

    %query-str%

    Sequência de consulta HTTP

    %http-version%

    Versão do protocolo HTTP

    %header.accept%

    Cabeçalho da aceitação

    %header.date%

    Cabeçalho de data

    %header.if-mod-since%

    Cabeçalho Se modificado desde

    %header.auth%

    Cabeçalho da autorização

    %header.any%

    Um valor de cabeçalho HTTP válido em RFC 2616 (qualquer também é um valor de cabeçalho válido; está especificado como variável, aqui)

    %session.name%

    O valor do atributo da sessão com o nome nome, NULL-SESSION-ATTRIBUTE-nome se o atributo nomeado não existe na sessão ou NULL-SESSION se nenhuma sessão existe

    %time-taken%

    Tempo (em milissegundos) que levou para efetuar o serviço de cada solicitação

    %vs.id%

    ID do servidor virtual

  13. (Opcional) Na seção Propriedades Adicionais, especifique essas propriedades.

    Para obter uma descrição das propriedades disponíveis, consulte Propriedades específicas do serviço HTTP.

    • Para adicionar uma propriedade, clique no botão Adicionar Propriedade. Na linha em brando que é exibida, digite o nome da propriedade no campo Nome e digite o valor da propriedade no campo Valor.
    • Para modificar uma propriedade, edite o campo Valor da propriedade.
    • Para excluir uma propriedade, marque a caixa de seleção à esquerda do campo Nome da propriedade a ser excluída. Clique no botão Excluir Propriedades.
  14. Clique em Salvar.

    Dica - Para restaurar as configurações padrão, clique no botão Carregar Padrões e depois em Salvar.
