Editar servidor virtual

Use a página Editar Servidor Virtual para modificar as configurações de um servidor virtual.

A página Novo servidor virtual contém as seguintes opções.

Carregar padrões

Botão para restaurar os valores padrão de todas as configurações.

Nome da configuração

Nome da configuração ao qual as configurações desta página serão enviadas. Este campo é de somente leitura.

Id

Identificador visível internamente do servidor virtual. Não é mostrado aos clientes HTTP. Os nomes de host apresentados aos clientes HTTP devem ser especificados no campo Hosts.

Hosts

O nome ou os nomes de host do computador em que o servidor está em execução. Use nomes de host reais ou virtuais, registrados no servidor de DNS da sua rede (e, no sistema UNIX, no arquivo /etc/hosts ). O valor padrão é a propriedade do sistema ${com.sun.aas.hostName}.

Estado

O estado desejado para o servidor. O valor pode ser um dos seguintes:

Ligado

O servidor virtual está ativo. Esse é o valor padrão.

Desligado

O servidor virtual está inativo. As tentativas de acessar o servidor retornam o código de erro 404 (recurso indisponível).

Desativado

O servidor virtual está desativado. As tentativas de acessar o servidor retornam o código de erro 403 (execução da solicitação recusada).

SSO

Especifique se o comportamento de logon único é herdado do serviço HTTP e se está ativado ou desativado. Por padrão, é herdado do serviço HTTP. Se esta opção está ativada, o logon único está ativado para aplicativos da Web neste servidor virtual que estão configurados para o mesmo domínio. Se esta opção não está ativada, o logon único é desativado para este servidor virtual, e os usuários precisam autenticar separadamente em cada aplicativo no servidor virtual.

Escutas de rede

A escuta ou escutas de rede associados a esse servidor.

Módulo da Web padrão

O módulo da Web implantado (se existente), designado para responder todas as solicitações que não puderem ser mapeadas por outros módulos da Web implantados no servidor virtual. Se um Módulo da Web Padrão não for especificado, o módulo da Web que tiver uma raiz de contexto vazia será utilizado. Se não houver nenhum módulo da Web com raiz de contexto vazia, um módulo da Web padrão do sistema será criado e utilizado.

Arquivo de log

O nome do caminho do arquivo em que as mensagens de log do servidor virtual serão exibidas. Por padrão, as mensagens de log aparecerão em domain-dir/logs/server.log ..

Docroot

O caminho absoluto do diretório de documento raiz do servidor. O valor padrão é domain-dir/docroot.

Log de Acesso

Especifica se o registro de acesso é herdado do serviço HTTP e se está ativado ou desativado. Por padrão, é herdado do serviço HTTP.

Diretório

O caminho absoluto de diretório para os logs de acesso do servidor. O padrão é domain-dir/logs/access.

Propriedades Adicionais

Propriedades adicionais do servidor virtual.

Para obter informações sobre as propriedades disponíveis, consulte Propriedades específicas de servidores virtuais.
