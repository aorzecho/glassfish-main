Propriedades específicas do serviço HTTP

As propriedades adicionais a seguir estão disponíveis para o serviço HTTP. Se você especifica essas propriedades, elas se aplicam a todos os Escutas de rede.

ssl-cache-entriese

O número se sessões SSL a serem colocadas no cache. O valor padrão é 10.000.

ssl3-session-timeout

O intervalo no qual as sessões SSL3 são colocadas no cache. O valor padrão é 86.400.

ssl-session-timeout

O intervalo no qual as sessões SSL2 são colocadas no cache. O valor padrão é 100.

proxyHandler

O nome da classe totalmente qualificada de uma implementação personalizada da classe abstrata com.sun.appserv.ProxyHandler que permite uma instância back-end do Servidor GlassFish, para recuperar informações sobre a solicitação original do cliente que foi interceptada por um proxy de servidor SSL de terminação (por exemplo, um balanceador de carga). Uma implementação desta classe abstrata inspeciona um determinado cabeçalho de solicitação do cliente, através da qual o proxy do servidor comunica as informações sobre a solicitação original do cliente para a instância do Enterprise Server, e retorna aquela informação para seu chamador.

A implementação padrão lê o endereço IP do cliente de um cabeçalho de solicitação HTTP denominado Proxy-ip, o keysize do SSL de um cabeçalho de solicitação denominado Proxy-keysize, e o certificado do cliente SSL de um cabeçalho de solicitação HTTP denominado Proxy-auth-cert. O valor Proxy-auth-cert precisa conter a cadeia do certificado do cliente codifica com em BASE-64, sem os limites BEGIN CERTIFICATE e END CERTIFICATE , e como \n substituído por % d% a.

Essa configuração é utilizada somente se a caixa de seleção Auth Pass Through para a página HTTP do protocolo do escuta de rede está definida como verdadeira.

connectionTimeout

Especifica o número de segundos aguardados pelos escutas de rede, após aceitar a conexão, pela apresentação de linha URI de solicitação. O valor padrão é 30.
