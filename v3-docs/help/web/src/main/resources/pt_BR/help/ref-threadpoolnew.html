Novo Pool de Thread

Use a página Novo Pool de Threads para criar um Pool de Threads.

A página Novo Pool de Threads contém estas opções.

Nome da Configuração

O nome da configuração à qual as definições nesta página se aplicam. Este campo é somente para leitura.

Nome

O nome do pool de threads.

Nome da classe

O nome totalmente qualificado da classe que implementa o pool de threads. O valor default é com.sun.grizzly.http.StatsThreadPool.

Tamanho máx. da fila

O número máximo de mensagens que podem ser colocadas na fila até que os threads estejam disponíveis para processá-las. Um valor de -1 indica que não existe limite para o tamanho da linha. O valor default é 4096.

Tamanho máx. do Pool de Threads

O número máximo de threads no pool de threads. O valor default é 5. Esse é o valor recomendado para pool de threads de listener de rede.

Tamanho mín. do Pool de Threads

O número mínimo de threads no pool de threads. O valor default é 2.

Time-out para Thread Inativo

O tempo máximo, em segundos, durante o qual um thread pode permanecer inativo no pool. Após esse período, o thread será removido do pool. O valor default é 900.
