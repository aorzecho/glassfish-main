Para Criar um Servidor Virtual

  1. Na árvore de navegação, expanda o nó Configuração.

  2. Select the configuration that you are changing.

  3. No nó Configuração, selecione o nó Servidores Virtuais.

    A página Servidores Virtuais é aberta.

  4. Na página Servidores Virtuais, clique em Novo.

    É aberta a página Novo Servidor Virtual.

  5. No campo de identificação, digite um valor que identifique o servidor virtual internamente.

    Esse valor não é exposto aos clientes HTTP. Os nomes de host apresentados aos clientes HTTP devem ser especificados no campo Hosts.

  6. No campo Hosts, digite os nomes de host ou endereços IP da máquina na qual o servidor está sendo executado.

    Use nomes de host reais ou virtuais que são registrados com o servidor DNS para a sua rede (e, em um sistema UNIX, no seu arquivo/etc/hosts).

    O valor default está no valor da propriedade do sistema ${com.sun.aas.hostName}.

  7. Selecione a opção Estado para o novo servidor virtual.

    O servidor virtual que pode estar ativado (ou seja, ativo) ou desativado. O valor default é ativado.

  8. Selecione uma opção SSO para o novo servidor virtual.

    Por default, um servidor virtual herda a definição de sign-on único (SSO) do serviço HTTP. Selecione Ativado ou Desativado para substituir a definição do serviço HTTP ao ativar ou desativar o sign-on único para este servidor virtual.

    O sign-on único em aplicações no GlassFish Server tem suporte de servlets e arquivos JSP. Esse recurso permite o compartilhamento dessa informação pelas várias aplicações que exigem as mesmas informações de sign-on de usuário, em vez de exigir que o usuário faça um sign-on separado em cada aplicação.

  9. Na lista drop-down Listeners de rede, selecione um ou mais listeners de rede ou deixe o campo em branco.

  10. Na lista drop-down Módulo da Web Default, selecione o módulo que deve responder a todas as solicitações que não podem ser mapeadas para outros módulos da Web implantados no servidor virtual.

    Se não for especificado um módulo da Web default, será usado o módulo da Web que tenha uma raiz de contexto vazia. Se não houver nenhum módulo da Web com raiz de contexto vazia, um módulo da Web default do sistema será criado e utilizado.

  11. No campo Arquivo de Log, digite o nome do caminho do arquivo no qual serão exibidas as mensagens de log desse servidor virtual.

    Por default, as mensagens de log serão enviadas ao log de servidor default, domain-dir/logs/server.log.

  12. No campo Docroot, digite o caminho absoluto para o diretório do documento raiz do servidor.

    Se o campo for deixado em branco, domain-dir/docroot será usado.

  13. Para ativar ou desativar o log de acesso, selecione a opção Log de Acesso.

    Você pode, alternativamente, especificar se o log de acesso é herdado do serviço HTTP. Por default, é herdado do serviço HTTP.

  14. No campo Diretório, digite o caminho absoluto do diretório para os logs de acesso do servidor.

    O valor default é domain-dir/logs/access.

  15. Na seção Propriedades Adicionais, especifique propriedades adicionais do servidor virtual.

    Para obter informações sobre propriedades disponíveis, consulte Propriedades Específicas a Servidores Virtuais.

    Para adicionar uma propriedade, clique no botão Adicionar propriedade. Na linha em branco que é exibida, digite o nome da propriedade no campo Nome e digite o valor da propriedade no campo Valor.

  16. Clique em OK.
