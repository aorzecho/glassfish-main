Para criar um servidor virtual

  1. Na árvore de navegação, expanda o nó Configuração.
  2. No nó Configuração, selecione o nó Servidores Virtuais.

    A página Servidores Virtuais é aberta.

  3. Na página Servidores Virtuais, clique em Novo.

    É aberta a página Novo Servidor Virtual.

  4. No campo de identificação, digite um valor que identifique o servidor virtual internamente.

    Esse valor não é exposto aos clientes HTTP. Os nomes de host apresentados aos clientes HTTP devem ser especificados no campo Hosts.

  5. No campo Hosts, digite os nomes de host ou endereços IP da máquina na qual o servidor está sendo executado.

    Use nomes de host reais ou virtuais, registrados no servidor de DNS da sua rede (e, no sistema UNIX, no arquivo /etc/hosts ).

    O valor padrão é a propriedade do sistema ${com.sun.aas.hostName}.

  6. Selecione a opção Estado para o novo servidor virtual.

    O servidor virtual que pode estar ativado (ou seja, ativo) ou desativado. O valor padrão é ativado.

  7. Selecione uma opção SSO para o novo servidor virtual.

    Por padrão, um servidor virtual herda a configuração de logon único (SSO) do serviço HTTP. Selecione Ativado ou Desativado para sobrepor a configuração do serviço HTTP ao ativar ou desativar o logon único para este servidor virtual.

    O logon único em aplicativos no Servidor GlassFish tem suporte de servlets e arquivos JSP. Esse recurso permite o compartilhamento dessa informação pelos vários aplicativos que exigem as mesmas informações de logon de usuário, em vez de exigir que o usuário faça logon separado em cada aplicativo.

  8. Na lista suspensa Escutas de rede, selecione um ou mais escutas de rede ou deixe o campo em branco.
  9. Na lista suspensa Módulo da Web Padrão, selecione o módulo que deve responder a todas as solicitações que não podem ser mapeadas para outros módulos da Web usados para o servidor virtual.

    Se não for especificado um módulo da Web padrão, será usado o módulo da Web que tenha uma raiz de contexto vazia. Se não houver nenhum módulo da Web com raiz de contexto vazia, um módulo da Web padrão do sistema será criado e utilizado.

  10. No campo Arquivo de Log, digite o nome do caminho do arquivo no qual serão exibidas as mensagens de log desse servidor virtual.

    Por padrão, as mensagens de log serão enviadas para o log do servidor padrão, domain-dir/logs/server.log .

  11. No campo Docroot, digite o caminho absoluto para o diretório do documento raiz do servidor.

    Se o campo for deixado em branco, será usado, domain-dir/docroot.

  12. Para ativar ou desativar o log de acesso, selecione a opção Log de Acesso.

    Você pode, alternativamente, especificar se o log de acesso é herdado do serviço HTTP. Por padrão, é herdado do serviço HTTP.

  13. No campo Diretório, digite o caminho absoluto do diretório para os logs de acesso do servidor.

    O valor padrão é domain-dir/logs/access .

  14. (Opcional) Na seção Propriedades Adicionais, especifique propriedades adicionais do servidor virtual.

    Para obter informações sobre as propriedades disponíveis, consulte Propriedades específicas de servidores virtuais.

    Para adicionar uma propriedade, clique no botão Adicionar Propriedade. Na linha em brando que é exibida, digite o nome da propriedade no campo Nome e digite o valor da propriedade no campo Valor.

  15. Clique em OK.
