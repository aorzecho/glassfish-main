Para Editar um Servidor Virtual

  1. Na árvore de navegação, expanda o nó Configuração.

  2. Select the configuration that you are changing.

  3. No nó Configuração, selecione o nó Servidores Virtuais.

    A página Servidores Virtuais é aberta.

  4. NA página Servidores Virtuais, clique no nome do servidor virtual a ser editado.

    É aberta a página Editar Servidor Virtual.

  5. No campo Hosts, digite os nomes de host ou endereços IP da máquina na qual o servidor está sendo executado.

    Use nomes de host reais ou virtuais que são registrados com o servidor DNS para a sua rede (e, em um sistema UNIX, no seu arquivo/etc/hosts). O valor default está no valor da propriedade do sistema ${com.sun.aas.hostName}.

  6. Selecione uma opção de Estado para o servidor virtual.

    O valor pode ser um dos seguintes:

    Ligado

    O servidor virtual está ativo. Esse é o valor default.

    Desligado

    O servidor virtual está inativo. As tentativas de acessar o servidor retornam o código de erro 404 (recurso indisponível).

    Desativado

    O servidor virtual está desativado. As tentativas de acessar o servidor retornam o código de erro 403 (execução da solicitação recusada).

  7. Selecione uma opção SSO para o novo servidor virtual.

    Por default, um servidor virtual herda a definição de sign-on único (SSO) do serviço HTTP. Selecione Ativado ou Desativado para substituir a definição do serviço HTTP ao ativar ou desativar o sign-on único para este servidor virtual.

    O sign-on único em aplicações no GlassFish Server tem suporte de servlets e arquivos JSP. Esse recurso permite o compartilhamento dessa informação pelas várias aplicações que exigem as mesmas informações de sign-on de usuário, em vez de exigir que o usuário faça um sign-on separado em cada aplicação.

  8. Na lista drop-down Listeners de rede, selecione um ou mais listeners de rede ou deixe o campo em branco.

  9. Na lista drop-down Módulo da Web Default, selecione o módulo que deve responder a todas as solicitações que não podem ser mapeadas para outros módulos Web implantados no servidor virtual.

    Se não for especificado um módulo da Web default, será usado o módulo da Web que tenha uma raiz de contexto vazia. Se não houver nenhum módulo da Web com raiz de contexto vazia, um módulo da Web default do sistema será criado e utilizado.

  10. No campo Arquivo de Log, digite o nome do caminho do arquivo no qual serão exibidas as mensagens de log desse servidor virtual.

    Por default, as mensagens de log serão enviadas ao log de servidor default, domain-dir/logs/server.log.

  11. No campo Docroot, digite o caminho absoluto para o diretório do documento raiz do servidor.

    O valor default é domain-dir/docroot.

  12. Para ativar ou desativar o log de acesso, selecione a opção Log de Acesso.

    Por default, um servidor virtual herda a definição de log de acesso do serviço HTTP. Selecione Ativado ou Desativado para substituir a definição do serviço HTTP ao ativar ou desativar o log de acesso para este servidor virtual.

  13. No campo Diretório, digite o caminho absoluto do diretório para os logs de acesso do servidor.

    O valor default é domain-dir/logs/access.

  14. Na seção Propriedades Adicionais, especifique propriedades adicionais do servidor virtual.

    Para obter informações sobre propriedades disponíveis, consulte Propriedades Específicas a Servidores Virtuais.

    • Para adicionar uma propriedade, clique no botão Adicionar propriedade. Na linha em branco que é exibida, digite o nome da propriedade no campo Nome e digite o valor da propriedade no campo Valor.

    • Para modificar uma propriedade, edite o campo Valor da propriedade.

    • Para deletar uma propriedade, marque a caixa de seleção à esquerda do campo Nome da Propriedade a ser deletada. Clique no botão Deletar Propriedades.

  15. Clique em Salvar.

    Dica:

    Para voltar às definições default, clique no botão Carregar Defaults e depois em Salvar.
