Editar um servidor virtual

  1. Na árvore de navegação, expanda o nó Configuração.
  2. No nó Configuração, selecione o nó Servidores Virtuais.

    A página Servidores Virtuais é aberta.

  3. NA página Servidores Virtuais, clique no nome do servidor virtual a ser editado.

    É aberta a página Editar Servidor Virtual.

  4. No campo Hosts, digite os nomes de host ou endereços IP da máquina na qual o servidor está sendo executado.

    Use nomes de host reais ou virtuais, registrados no servidor de DNS da sua rede (e, no sistema UNIX, no arquivo /etc/hosts ). O valor padrão é a propriedade do sistema ${com.sun.aas.hostName}.

  5. Selecione uma opção de Estado para o servidor virtual.

    O valor pode ser um dos seguintes:

    Ligado

    O servidor virtual está ativo. Esse é o valor padrão.

    Desligado

    O servidor virtual está inativo. As tentativas de acessar o servidor retornam o código de erro 404 (recurso indisponível).

    Desativado

    O servidor virtual está desativado. As tentativas de acessar o servidor retornam o código de erro 403 (execução da solicitação recusada).

  6. Selecione uma opção SSO para o novo servidor virtual.

    Por padrão, um servidor virtual herda a configuração de logon único (SSO) do serviço HTTP. Selecione Ativado ou Desativado para sobrepor a configuração do serviço HTTP ao ativar ou desativar o logon único para este servidor virtual.

    O logon único em aplicativos no Servidor GlassFish tem suporte de servlets e arquivos JSP. Esse recurso permite o compartilhamento dessa informação pelos vários aplicativos que exigem as mesmas informações de logon de usuário, em vez de exigir que o usuário faça logon separado em cada aplicativo.

  7. Na lista suspensa Escutas de rede, selecione um ou mais escutas de rede ou deixe o campo em branco.
  8. Na lista suspensa Módulo da Web Padrão, selecione o módulo que deve responder a todas as solicitações que não podem ser mapeadas para outros módulos da Web usados para o servidor virtual.

    Se não for especificado um módulo da Web padrão, será usado o módulo da Web que tenha uma raiz de contexto vazia. Se não houver nenhum módulo da Web com raiz de contexto vazia, um módulo da Web padrão do sistema será criado e utilizado.

  9. No campo Arquivo de Log, digite o nome do caminho do arquivo no qual serão exibidas as mensagens de log desse servidor virtual.

    Por padrão, as mensagens de log serão enviadas para o log do servidor padrão, domain-dir/logs/server.log .

  10. No campo Docroot, digite o caminho absoluto para o diretório do documento raiz do servidor.

    O valor padrão é domain-dir/docroot.

  11. Para ativar ou desativar o log de acesso, selecione a opção Log de Acesso.

    Por padrão, um servidor virtual herda a configuração de registro de acesso do serviço HTTP. Selecione Ativado ou Desativado para sobrepor a configuração do serviço HTTP ao ativar ou desativar o registro de acesso para este servidor virtual.

  12. No campo Diretório, digite o caminho absoluto do diretório para os logs de acesso do servidor.

    O valor padrão é domain-dir/logs/access .

  13. (Opcional) Na seção Propriedades Adicionais, especifique propriedades adicionais do servidor virtual.

    Para obter informações sobre as propriedades disponíveis, consulte Propriedades específicas de servidores virtuais.

    • Para adicionar uma propriedade, clique no botão Adicionar Propriedade. Na linha em brando que é exibida, digite o nome da propriedade no campo Nome e digite o valor da propriedade no campo Valor.
    • Para modificar uma propriedade, edite o campo Valor da propriedade.
    • Para excluir uma propriedade, marque a caixa de seleção à esquerda do campo Nome da propriedade a ser excluída. Clique no botão Excluir Propriedades.
  14. Clique em Salvar.

    Dica - Para restaurar as configurações padrão, clique no botão Carregar Padrões e depois em Salvar.
