Para Criar um Pool de Threads

O pool de threads são usados por listeners de rede e pelo ORB. Normalmente, o listener de rede usa um dos dois pool de threads que são criados quando o GlassFish Server é instalado, e o ORB usa o outro. Você pode editar qualquer um destes pool de threads default para especificar definições apropriadas para seus aplicações ou sistemas de hardware, ou pode criar e usar um pool de threads com essas definições.

  1. Na árvore de navegação, expanda o nó Configuração.

  2. No nó Configuração, selecione o nó Pool de Threads.

    É aberta a página Pool de Threads.

  3. Na página Pool de Threads, clique em Novo.

    É aberta a página Novo Pool de Threads.

  4. No campo Nome, digite um nome para o pool de threads.

  5. No campo Nome da Classe, digite o nome totalmente qualificado da classe que implementa o pool de threads.

    O valor default é com.sun.grizzly.http.StatsThreadPool.

  6. No campo Tamanho máximo da fila, digite o número máximo de mensagens que podem ser colocadas na fila até que os threads estejam disponíveis para processá-las.

    Um valor de -1 indica que não existe limite para o tamanho da linha. O valor default é 4096.

  7. No campo Tamanho Máximo do Pool de Threads, digite a quantidade máxima de threads que pode haver no pool de threads.

    O valor default é 5. Esse é o valor recomendado para pool de threads de listener de rede.

  8. No campo Tamanho Mínimo do Pool de Threads, digite a quantidade mínima de threads que pode haver no pool de threads.

    Esses pools são criados quando o pool de threads é instanciado. O valor default é 2.

  9. No campo Time-out para Threads Inativos, digite o tempo máximo, em segundos, durante o qual um thread pode permanecer ocioso no pool.

    Após esse período, o thread será removido do pool. O valor default é 900.

  10. Clique em OK.
