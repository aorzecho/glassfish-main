Para criar um protocolo

  1. Na árvore de navegação, expanda o nó Configuração.
  2. No nó Configuração, expanda o nó Configuração da Rede.
  3. No nó Config da rede, selecione o nó Protocolos.

    É aberta a página Protocolos.

  4. Na página Protocolos, clique em Novo.

    É aberta a página Novo Protocolo.

  5. No campo Nome, digite um nome único para o protocolo.
  6. Marque a caixa de seleção Segurança Ativada para habilitar a segurança do protocolo.

    Por padrão, a opção está desativada. Caso você ative a segurança, poderá usar a guia SSL da página Editar Protocolo para modificar as configurações de segurança.

  7. Marque a caixa de seleção Status Ativado para habilitar o cache do arquivo.

    Por padrão, a opção está ativada.

  8. No campo Tempo Máximo, digite o tempo máximo, em segundos, de uma entrada de cache válida.

    Esse parâmetro controla a quantidade de informações armazenadas em cache que será utilizada após o armazenamento de um arquivo no cache. Uma entrada mais antiga do que a duração máxima será substituída pela nova entrada do mesmo arquivo. Se o conteúdo não for alterado com frequência, aumente esse valor para melhorar o desempenho.

    A duração máxima mais eficiente depende da regularidade com que os arquivos existentes são modificados. Por exemplo, se os arquivos forem modificados quatro vezes ao dia, em intervalos regulares, considere a possibilidade de definir a duração máxima em 21600 segundos (6 horas). Ou então, defina a duração máxima como sendo o tempo mais longo desejado para a exibição da versão anterior de um arquivo após a modificação.

    O valor padrão é 30.

  9. No campo Tamanho máximo do cache, digite o tamanho total máximo (em bytes) do cache do arquivo em disco.

    O valor padrão é 10485760.

  10. No campo Contagem Máxima de Arquivos, digite o número máximo de arquivos que pode ser armazenado no cache do arquivo.

    Se o valor for muito alto, o servidor armazenará em cache os arquivos pouco necessários, o que consome memória. Se o valor for muito baixo, os benefícios do armazenamento em cache não será obtidos. Tente diferentes valores desse atributo para encontrar a solução mais eficiente para aplicativos específicos.

    O valor padrão é 1024.

  11. No campo Nome do Servidor, digite o nome do host a ser usado nas URLs que o servidor envia para o cliente.

    Esse nome será o nome alias se o servidor usar um alias. Se o servidor não usar alias, deixe o campo em branco.

    Esse valor somente afeta os URLs gerados automaticamente pelo servidor; ele não afeta os URLs para diretórios e arquivos armazenados no servidor. Caso seu servidor use um codinome, o nome do servidor deveria ser o nome do codinome. Se uma vírgula e um número de porta estão anexados, aquela porta é usada nos URLs que o servidor envia para o cliente.

  12. Na lista suspensa Servidor Virtual Padrão, selecione o servidor virtual a ser associado com este protocolo.

    Use a página Servidores Virtuais para defini-los.

  13. No campo Porta de Redirecionamento, digite o valor da porta a ser usada para redirecionar uma solicitação para outra porta.

    O Servidor GlassFish redirecionará automaticamente a solicitação se estas duas condições existirem:

    • A escuta de rede que referencia esse protocolo oferece suporte a solicitações não-SSL.

    • Uma solicitação é recebida, e a correspondente restrição de segurança exige transporte SSL.

    Se uma porta de redirecionamento não estiver especificada, o Servidor GlassFish usará o número de porta especificado na solicitação original. Por padrão, nenhuma porta é especificada.

  14. No campo Máximo de conexões, digite o número máximo de solicitações que podem ser colocadas na fila até que a conexão seja fechada pelo servidor.

    O subsistema de manutenção de conexões consulta periodicamente as conexões ociosas. O valor padrão é 256.

    Defina esta propriedade como 1 para desativar o keep-alive HTTP/1.0m assim como o keep-alive e a linha HTTP/1.1. Um valor de 0 significa que as solicitações são sempre rejeitadas. Um valor de -1 define nenhum limite para o número de conexões keep-alive.

  15. No campo Tempo limite, digite o máximo de tempo em segundos que uma conexão pode ser considerada inativa e mantida no estado keep-alive.

    Um valor de 0 ou menos significa que as conexões keep-alive são mantidas abertas indefinidamente. O valor padrão é 30 segundos.

  16. Selecione a caixa de seleção Tempo limite de upload ativada para que a conexão para um servlet que lê lentamente os bytes seja fechada após o limite de Tempo limite de conexão para upload é alcançado.

    Se essa opção for desativada, as conexões de servlet não expiram. Por padrão, a opção está desativada.

  17. No campo Tempo limite de conexão para upload, digite o tempo limite para uploads em milissegundos.

    Esse campo somente é aplicável se a caixa de seleção Tempo limite de upload desativada estiver selecionada. O valor padrão é 300.000 milissegundos.

  18. No campo Tempo limite da solicitação, digite o número de segundos antes que uma solicitação expire.

    Se a solicitação não é processada antes que o valor do tempo limite seja alcançado, a solicitação é ignorada. O valor padrão é 30 segundos.

  19. No campo Tamanho do buffer de envio, digite o tamanho em bytes do buffer de envio.

    O valor padrão é 8.192 bytes.

  20. No campo Comprimento do cabeçalho do buffer, digite o tamanho em bytes do buffer usado pelos segmentos de processamento de solicitação para ler os dados da solicitação.

    O valor padrão é 8.192 bytes.

  21. No campo Tamanho máximo de postagem, digite o tamanho máximo em bytes das ações de postagem.

    O valor padrão é 2.097.152 bytes.

  22. No campo Codificação URI, digite o nome do conjunto de caracteres usado para decodificar os URIs de solicitação recebidos.

    O valor precisa ser um nome válido de um conjunto de caracteres IANA. O valor padrão é UTF-8.

  23. No campo versão, digite a versão do protocolo HTTP usado.

    O valor padrão é HTTP/1.1.

  24. Na lista suspensa Compressão, selecione o uso da compressão GZIP HTTP/1.1 para economizar largura de banda do servidor.

    As opções disponíveis são:

    ligado

    Comprime os dados.

    desligado

    Desativa a compressão.

    forçar

    Força a compressão dos dados em todos os casos.

    O valor padrão desligado.

  25. No campo Tipos Mime com compressão, digite uma lista separada por vírgulas dos tipos MIME para os quais a compressão HTTP é usada.

    O valor padrão é text/html,text/xml,text/plain.

  26. No campo Tamanho mínimo da compressão, digite o tamanho mínimo de um arquivo quando a compressão é aplicada.

    Esse valor deve ser definido se a compressão estiver definida como ligado ou forçar. O valor padrão é 2.048 bytes.

  27. No campo Agentes de usuário sem compressão, digite uma lista separada por vírgulas de expressões regulares que coincidem com agentes do usuário ou clientes HTTP, para os quais a compressão não deve ser usada.

    Por padrão este valor é uma sequência vazia.

  28. No campo Agente de usuário restringido, digite uma lista de agentes do usuário restringido no qual a compressão HTTP é aplicada.

    Se nenhum agente do usuário é especificado, a compressão HTTP é aplicada em todos os agentes do usuário. Por padrão, nenhum agente do usuário é especificado.

  29. No campo Tipo de resposta padrão, digite uma sequência que especifica o tipo de resposta padrão.

    O formato é uma sequência delimitada por dois pontos, consistindo no tipo de conteúdo, codificação, idioma e conjunto de caracteres. O valor padrão é text/html; charset=iso-8859-1.

  30. No campo Tipo de resposta forçada, digite uma sequência que especifica o tipo de solicitação usado, caso nenhum mapeamento MIME esteja disponível que coincida com a extensão do arquivo.

    O formato é uma sequência delimitada por dois pontos, consistindo no tipo de conteúdo, codificação, idioma e conjunto de caracteres. O valor padrão é text/html; charset=iso-8859-1.

  31. No campo Adaptador, digite o nome da classe do adaptador de recursos estáticos.

    O valor padrão é com.sun.grizzly.tcp.StaticResourcesAdapter.

  32. Selecione a caixa de seleção Suporte Comet ativado para ativar o suporte Comet para a escuta de rede que referencia este protocolo.

    Por padrão, esta opção é desativada. Se o seu servlet ou página JSP usa a tecnologia Comet, assegure-se de que esteja inicializada quando o Servidor GlassFish é iniciado, ao adicionar o elemento load-on-startup em seu arquivo web.xml. Por exemplo:

    <servlet>
<servlet-name>CheckIn</servlet-name>
<servlet-class>CheckInServlet</servlet-class>
<load-on-startup>0</load-on-startup>
</servlet>
  33. Selecione a caixa de seleção Pesquisa DNS ativada para ativar a pesquisa DNS.

    Por padrão, a opção está desativada.

  34. Selecione a caixa de seleção Suporte RCM ativado para ativar o Gerenciamento de Configuração de Recursos (RCM).

    Por padrão, a opção está desativada.

  35. Selecione a caixa de seleção Rastreamento ativado para ativar a operação TRACE.

    Se esta opção for desativada, o Servidor GlassFish é menos suscetível ao ataques de scripts de site cruzado. Por padrão, a opção está ativada.

  36. Selecione a caixa de seleção Auth Pass Through ativada para indicar que a escuta de rede que usa este protocolo recebe tráfego de um proxy de servidor SSL com terminação.

    Por padrão, a opção está desativada.

  37. Selecione a caixa de seleção Amostragem ativada para ativar a amostragem de resposta HTTP.

    Por padrão, a opção está ativada.

  38. Selecione a caixa de seleção XPowered By ativado para indicar ao Servidor GlassFish para usar os cabeçalhos X-Powered-By das especificações Java Servlet 3.0 e do Java Server Pages 2.0.

    Por padrão, a opção está ativada.
