Para criar uma escuta de rede

Antes de começar

Quando você cria uma escuta de rede, pode criar simultaneamente um protocolo que será usado pelo escuta ou usa um protocolo criado primeiro. Normalmente, há um relacionamento um-a-um entre a escuta de rede e seu protocolo associado, mas isso não é necessário.

  1. Na árvore de navegação, expanda o nó Configuração.
  2. No nó Configuração, expanda o nó Configuração da Rede.
  3. No nó Config da rede, selecione o nó Escutas de rede.

    É exibida a página Escutas de rede.

  4. Na página Escutas de rede, clique em Novo.

    É exibida a página Nova Escuta de rede.

  5. No campo Nome, digite um nome para o escuta de rede.

    Um nome de escuta de rede não pode começar com um número.

  6. No campo Protocolo, especifique o protocolo a ser associado à escuta de rede.
    • Selecione Criar novo protocolo, para criar um novo protocolo com configurações padrão. Também, na lista suspensa Servidor Virtual, selecione o servidor virtual a ser associado ao protocolo.

      O nome padrão do protocolo é o nome da escuta de rede com -protocol anexado, mas é possível editar este valor.

    • Selecione Utilizar um protocolo existente e, em seguida, selecione um protocolo na lista suspensa.

      Caso você selecione esta opção, normalmente já teria criado o protocolo para esse interpretador.

    Você somente pode especificar um protocolo ao criar uma nova escuta de rede. Você não pode alterar em outro momento o protocolo associado.

  7. Marque a caixa de seleção Status Ativado para habilitar a escuta de rede.

    Por padrão, a opção está ativada. Se o interpretador estiver desativado, qualquer tentativa de conectar ao listener resulta em uma exceção de soquete (java.net.ConnectException).

    Nas versões do Servidor GlassFish anteriores a 9.1, um listener cujo atributo ativado estava definido como falso retornava um código de resposta 404, para quaisquer solicitações enviadas ao mesmo. Para obter este comportamento na versão atual do Servidor GlassFish, defina o status do interpretador como ativado e defina cada estado de servidor virtual associado como DESATIVADO. Um servidor virtual lista seus interpretadores associados na sua lista suspensa Escutas de rede.

  8. Selecione a caixa de seleção Segurança ativada para ativar a segurança do protocolo usado pela escuta de rede.

    Por padrão, a opção está desativada.

  9. Selecione a caixa de seleção Interpretador JK ativado para especificar que o interpretador suporta o protocolo mod-jk do Apache.

    O protocolo mod-jk é um protocolo proprietário de comunicação/rede entre o httpd (servidor da Web) da Apache e uma instância back-end do Servidor GlassFish. Se o interpretador suporta este protocolo, o httpd deveria manusear todo o conteúdo estático, enquanto quaisquer solicitações por recursos dinâmicos (ou seja, Servlets e JSPs) são roteadas para o interpretador mod-jk ativado para a rede do Servidor GlassFish, usando o protocolo mod-jk. Por padrão, a opção está desativada.

  10. No campo Porta, digite o número da porta pela qual a escuta de rede vai ouvir.

    Os valores válidos são 1 a 65.535. Em sistemas UNIX, a criação de soquetes que escutam nas portas 1 a 1024 requer privilégios de superusuário.

  11. No campo Endereço, digite o endereço IP no qual a escuta de rede vai ouvir.

    O endereço pode estar na notação de par de pontos ou IPv6. Ele pode ser qualquer (para INADDR_ANY) para escutar todos s endereços de IP. Ele pode ser um nome de host.

  12. Na lista suspensa Conjunto de segmentos, selecione o conjunto de segmentos a ser associado à escuta de rede.

    Normalmente, você seleciona um dos dois conjunto de segmentos que estão configurados quando o Servidor GlassFish é instalado.

  13. Na lista suspensa Transporte, selecione o tipo de transporte a ser usado pela escuta de rede.

    Por padrão, a escolha é tcp.

  14. Clique em OK.
Passos Seguintes

Se você ativou a segurança para a escuta de rede, poderá usar a guia SSL para editar as configurações SSL do protocolo utilizado pela escuta de rede.
