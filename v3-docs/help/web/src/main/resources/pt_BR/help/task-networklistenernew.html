Para Criar um Listener de Rede

Antes de Começar

Quando você cria um listener de rede, pode criar simultaneamente um protocolo que será usado pelo listener ou usar um protocolo criado primeiro. Normalmente, há um relacionamento um-a-um entre o listener de rede e seu protocolo associado, mas isso não é necessário.

  1. Na árvore de navegação, expanda o nó Configuração.

  2. No nó Configuração, expanda o nó Configuração da Rede.

  3. No nó Config da rede, selecione o nó Listeners de rede.

    É exibida a página Listeners de Rede.

  4. Na página Listeners de rede, clique em Novo.

    É exibida a página Novo Listener de Rede.

  5. No campo Nome, digite um nome para o listener de rede.

    Um nome de listener de rede não pode começar com um número.

  6. No campo Protocolo, especifique o protocolo a ser associado ao listener de rede.

    • Selecione Criar novo protocolo, para criar um novo protocolo com definições default. Também, na lista drop-down Servidor Virtual, selecione o servidor virtual a ser associado ao protocolo.

      O nome default do protocolo é o nome do listener de rede com -protocol acrescentado, mas é possível editar esse valor.

    • Selecione Utilizar um protocolo existente e, em seguida, selecione um protocolo na lista drop-down.

      Caso você selecione esta opção, normalmente já teria criado o protocolo para esse interpretador.

    Você somente pode especificar um protocolo ao criar um novo listener de rede. Você não pode alterar em outro momento o protocolo associado.

  7. Marque a caixa de seleção Status Ativado para ativar o listener de rede.

    Por default, a opção está ativada. Se o interpretador estiver desativado, qualquer tentativa de conectar ao listener resulta em uma exceção de soquete (java.net.ConnectException).

    Nas versões do GlassFish Server anteriores a 9.1, um listener cujo atributo ativado estava definido como falso retornava um código de resposta 404, para quaisquer solicitações enviadas ao mesmo. Para obter esse comportamento na versão atual do GlassFish Server, defina o status do listener como ativado e cada estado de servidor virtual associado como DESLIGADO. Um servidor virtual lista suas listeners associadas na sua lista drop-down Listeners de rede.

  8. Marque a caixa de seleção Segurança Ativada para ativar a segurança do protocolo usado pelo listener de rede.

    Por default, a opção está desativada.

  9. Marque a caixa de seleção Interpretador JK ativado para especificar que o interpretador suporta o protocolo mod-jk do Apache.

    O protocolo mod-jk é um protocolo proprietário de communication/network entre o httpd do Apache (servidor da web) e uma instância de back-end do GlassFish Server. Se o listener oferecer suporte a esse protocolo, o httpd deverá tratar todo o conteúdo estático, enquanto as solicitações por recursos dinâmicos (ou seja, Servlets e JSPs) serão roteadas para o interpretador mod-jk ativado para a rede do GlassFish Server, utilizando o protocolo mod-jk. Por default, a opção está desativada.

  10. No campo Porta, digite o número da porta do listener de rede.

    Os valores válidos são 1 a 65.535. Em sistemas UNIX, criar soquetes que fazem listening nas portas 1 - 1024 exige privilégios de superusuário.

  11. No campo Endereço, digite o endereço IP do listener de rede.

    O endereço pode estar na notação de par de pontos ou IPv6. Pode ser qualquer (para INADDR_ANY) para fazer listening em todos os endereços IP. Ele pode ser um nome de host.

  12. Na lista drop-down Pool de Threads, selecione o pool de threads a ser associado ao listener de rede.

    Normalmente, você seleciona um dos dois pool de threads que estão configurados quando o GlassFish Server é instalado.

  13. Na lista drop-down Transporte, selecione o tipo de transporte a ser usado pelo listener de rede.

    Por default, a escolha é tcp.

  14. Clique em OK.

Próximas Etapas

Se você ativou a segurança para o listener de rede, poderá usar a guia SSL para editar as definições SSL do protocolo utilizado pelo listener de rede.
