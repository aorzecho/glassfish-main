Editar transporte

Use a página Editar transporte para modificar as configurações de um transporte existente.

A página Editar transporte contém as seguintes opções.

Carregar padrões

Botão para restaurar os valores padrão de todas as configurações.

Nome da configuração

Nome da configuração ao qual as configurações desta página serão enviadas. Este campo é de somente leitura.

Nome

O nome do transporte. O nome é um campo de somente leitura. Você pode apenas especificar um nome quando cria um novo transporte.

Nome da classe

O nome da classe da implementação do transporte. O valor padrão é com.sun.grizzly.TCPSelectorHandler.

Manipulador de chave de seleção

O nome da classe da implementação do manipulador de chave de seleção. Um manipulador de chave de seleção é uma classe abstrata que gerencia o ciclo de vida das chaves de seleção. Se você não especifica um valor, a implementação padrão é usada.

Tipo do buffer de bytes

O tipo de ByteBuffer a ser usado. As escolhas disponíveis são HEAP e DIRECT. O valor padrão é HEAP.

Segmentos do aceitador

O número de processadores na máquina. Um valor de –1 indica que o Grizzly irá calcular por si mesmo o número de segmentos do aceitador com base no número de processadores na máquina. O valor padrão é 1.

Para definir o número de segmentos de processamento da solicitação, defina o valor Tamanho máximo do conjunto de segmentos usado por uma escuta de rede que usa este transporte.

Contagem máxima de conexões

O número máximo de conexões pendentes em uma escuta de rede que usa este transporte. O valor padrão é 4096.

Tamanho do buffer

O tamanho em bytes do buffer a ser fornecido para fluxos de entrada criados por uma escuta de rede que usa esse transporte. O valor padrão é 8.192.

Tempo limite de chave ociosa

O número de segundos após o qual uma chave ociosa será cancelada e o canal será fechado. O valor padrão é 30.

Tempo limite de leitura

O número em milissegundos que o Servidor GlassFish aguarda durante a análise do cabeçalho e corpo da operação de leitura. O valor padrão é 30.000.

Tempo limite do seletor do conjunto

O número em milissegundos que um seletor NIO será bloqueado enquanto aguarda por eventos (solicitações de usuários). O valor padrão é 1000.

Tempo limite de gravação

O número de milissegundos que o &ProductName. aguarda antes de considerar que o cliente remoto desconectou ao gravar a resposta. O valor padrão é 30.000.

Exibir configuração

Se essa opção é selecionada, a configuração interna do Grizzly é descarregada nos registros do servidor. Essa opção pode fornecer informações úteis para a depuração. Por padrão, a opção está desativada.

Snoop

Se esta opção é selecionada, as informações solicitadas/respondidas são descarregada nos registros do servidor. Essa opção pode fornecer informações úteis para a depuração, mas irá reduzir significativamente o desempenho. Por padrão, a opção está desativada.

Nenhum atraso TCP

Se esta opção for selecionada, TCP_NODELAY (também denominado como algoritmo de Nagle) é ativado. Por padrão, a opção está ativada.
