Editar Transporte

Use a página Editar transporte para modificar as definições de um transporte existente.

A página Editar transporte contém as seguintes opções.

Carregar Defaults

Botão para restaurar com os valores default as definições que apresentam valores default. As definições que não apresentam valores default não são alteradas.

Nome da Configuração

O nome da configuração à qual as definições nesta página se aplicam. Este campo é somente para leitura.

Nome

O nome do transporte. O nome é um campo somente para leitura. Você pode apenas especificar um nome quando cria um novo transporte.

Nome da classe

O nome da classe da implementação do transporte. O valor default é com.sun.grizzly.TCPSelectorHandler.

Handler de chave de seleção

O nome da classe do handler de chave de seleção. Um handler de chave de seleção é uma classe abstrata que gerencia o ciclo de vida das chaves de seleção. Se você não especifica um valor, a implementação default é usada.

Tipo do buffer de bytes

O tipo de ByteBuffer a ser usado. As opções disponíveis são HEAP e DIRECT. O valor default é HEAP.

Threads do Aceitador

O número de processadores na máquina. Um valor de -1 indica que Grizzly irá calcular o número de threads de quem aceita, com base no número de processadores na máquina. O valor default é 1.

Para definir o número de threads de processamento da solicitação, defina o valor Tamanho Máximo do Pool de Threads usado por um listener de rede que usa este transporte.

Contagem máxima de conexões

O número máximo de conexões pendentes em um listener de rede que usa este transporte. O valor default é 4096.

Tamanho do buffer

O tamanho em bytes do buffer a ser fornecido para fluxos de entrada criados por um listener de rede que utiliza esse transporte. O valor default é 8.192.

Time-out para chave ociosa

O número de segundos após o qual uma chave ociosa será cancelada e o canal será fechado. O valor default é 30.

Time-out para leitura

O número em milissegundos que o GlassFish Server aguarda durante o parsing do cabeçalho e corpo da operação de leitura. O valor default é 30.000.

Time-out para Sondagem do Seletor

O número em milissegundos que um seletor NIO será bloqueado enquanto aguarda por eventos (solicitações de usuários). O valor default é 1000.

Time-out para gravação

O número de milissegundos que o &ProductName. aguarda antes de considerar que o cliente remoto desconectou ao gravar a resposta. O valor default é 30.000.

Exibir configuração

Se esta opção é selecionada, a configuração interna do Grizzly é descarregada nos registros do servidor. Essa opção pode fornecer informações úteis para a depuração. Por default, essa opção está desativada.

Snoop

Se esta opção for selecionada, a informação request/response será submetida a dump no log do servidor. Essa opção pode fornecer informações úteis para a depuração, mas irá reduzir significativamente o desempenho. Por default, essa opção está desativada.

Nenhum atraso TCP

Se essa opção for selecionada, TCP_NODELAY (também chamado de Nagle) será ativado. Por default, a opção está ativada.
