Para criar um transporte

  1. Na árvore de navegação, expanda o nó Configuração.
  2. No nó Configuração, expanda o nó Configuração da Rede.
  3. No nó Config da rede, selecione o nó Transportes.

    É aberta a página Transportes.

  4. Na página Transportes, clique em Novo.

    É aberta a página Novo Transporte.

  5. No campo Nome, digite um nome único para o transporte.
  6. No campo Nome da classe, digite o nome da implementação do transporte

    O valor padrão é com.sun.grizzly.TCPSelectorHandler.

  7. No campo Manipulador de chave de seleção, digite o nome da classe de um manipulador de chave de seleção.

    Um manipulador de chave de seleção é uma classe abstrata que gerencia o ciclo de vida das chaves de seleção. Se você não especifica um valor, a implementação padrão é usada.

  8. Na lista suspensa Tipo de buffer, selecione o tipo de ByteBuffer a ser usado.

    As escolhas disponíveis são HEAP e DIRECT. O valor padrão é HEAP.

  9. No campo Segmentos do aceitador, digite o número de processadores na máquina.

    Um valor de –1 indica que o Grizzly irá calcular por si mesmo o número de segmentos do aceitador com base no número de processadores na máquina. O valor padrão é 1.

    Para definir o número de segmentos de processamento da solicitação, defina o valor Tamanho máximo do conjunto de segmentos usado por uma escuta de rede que usa este transporte.

  10. No campo Contagem máxima de conexões, digite o número máximo de conexões pendentes em uma escuta de rede que usa este transporte.

    O valor padrão é 4096.

  11. No campo Tamanho do buffer, digite o tamanho em bytes do buffer a ser fornecido para fluxos de entrada criados por uma escuta de rede que usa este transporte.

    O valor padrão é 8.192.

  12. No campo Tempo limite de chave ociosa, digite o número de segundos após o qual uma chave ociosa pode ser cancelada e o canal fechado.

    O valor padrão é 30.

  13. No campo Tempo limite de leitura, digite o número em milissegundos que o Servidor GlassFish aguarda durante a análise do cabeçalho e corpo da operação de leitura.

    O valor padrão é 30.000.

  14. No campo Tempo limite de amostragem, digite o número em milissegundos que um seletor NIO será bloqueado enquanto aguarda por eventos (solicitações de usuários).

    O valor padrão é 1000.

  15. No campo Tempo limite de gravação, digite o número de milissegundos que o Servidor GlassFish aguarda antes de considerar que o cliente remoto desconectou ao gravar a resposta.

    O valor padrão é 30.000.

  16. Selecione a caixa de seleção Exibir configurações ativado para descarregar as configurações internas do Grizzly nos registros do servidor.

    Essa opção pode fornecer informações úteis para a depuração. Por padrão, a opção está desativada.

  17. Selecione a caixa de seleção Snoop ativado para descarregar as informações de solicitações/respostas no registro do servidor.

    Essa opção pode fornecer informações úteis para a depuração, mas irá reduzir significativamente o desempenho. Por padrão, a opção está desativada.

  18. Selecione a caixa de diálogo Sem atraso do TCP, para ativar o TCP_NODELAY (também denominado como algoritmo de Nagel).

    Por padrão, a opção está ativada.

  19. Clique em OK.
