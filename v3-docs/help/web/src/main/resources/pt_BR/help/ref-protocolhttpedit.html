HTTP

Use a página HTTP para modificar as configurações HTTP para um protocolo de rede.

A página HTTP contém as seguintes opções:

Carregar padrões

Botão para restaurar os valores padrão de todas as configurações.

Nome da configuração

Nome da configuração ao qual as configurações desta página serão enviadas. Este campo é de somente leitura.

Nome do protocolo

O nome do protocolo. O campo Nome do protocolo é de somente leitura. Você só pode especificar o nome quando cria um novo protocolo.

Nome do servidor

O nome do host a ser utilizado nas URLs que o servidor envia ao cliente. Esse nome será o nome alias se o servidor usar um alias. Se o servidor não usar alias, deixe o campo em branco.

Servidor virtual padrão

O servidor virtual a ser associado ao protocolo. Utilize a página Servidores virtuais para defini-los.

Porta de redirecionamento

Valor de porta que redireciona uma solicitação para outra porta. O Servidor GlassFish redirecionará automaticamente a solicitação se estas duas condições existirem:

  • O protocolo suporta solicitações não-SSL.

  • Uma solicitação é recebida, e a correspondente restrição de segurança exige transporte SSL.

Se uma porta de redirecionamento não estiver especificada, o Servidor GlassFish usará o número de porta especificado na solicitação original.

Máx. de conexões

O número máximo de conexões persistentes a serem mantidas no modo Manutenção de conexão. O subsistema de manutenção de conexões consulta periodicamente as conexões ociosas. O valor padrão é 256.

Tempo limite

A quantidade máxima de tempo, em segundos, que uma conexão pode ser considerada inativa e mantida no estado keep-alive. Um valor de 0 ou menos significa que as conexões keep-alive são mantidas abertas indefinidamente. O valor padrão é 30 segundos.

Tempo limite de upload

Se essa opção é selecionada, a conexão para um servlet que lê lentamente os bytes seja fechada após o limite de Conexão para upload é alcançado. Se essa opção for desativada, as conexões de servlet não expiram. Por padrão, a opção está desativada.

Tempo limite da conexão do upload

O tempo limite para uploads em milissegundos. Esse campo somente é aplicável se a caixa de seleção Tempo limite de upload ativada estiver selecionada. O valor padrão é 300.000 milissegundos.

Tempo limite da solicitação

O número de segundos antes que a solicitação expira. Se a solicitação não é processada antes que o valor do tempo limite seja alcançado, a solicitação é ignorada. O valor padrão é 30 segundos.

Tamanho do buffer de envio

O tamanho máximo em bytes do buffer de envio. O valor padrão é 8.192 bytes.

Comprimento do cabeçalho do buffer

O tamanho em bytes do buffer utilizado pelos segmentos de processamento da solicitação para ler os dados da solicitação. O valor padrão é 8.192 bytes.

Tamanho máximo da postagem

O tamanho máximo em bytes de ações de postagem. O valor padrão é 2.097.152 bytes.

Codificação do URI

O nome do conjunto de caracteres utilizado para decodificar os URIs de solicitação recebidos. O valor precisa ser um nome válido de um conjunto de caracteres IANA. O valor padrão é UTF-8.

Versão

A versão do protocolo HTTP utilizado. O valor padrão é HTTP/1.1.

Compressão

Especifica o uso da compressão GZIP HTTP/1.1 para economizar largura de banda do servidor. As opções disponíveis são:

ligado

Comprime os dados.

desligado

Desativa a compressão.

forçar

Força a compressão dos dados em todos os casos.

O valor padrão desligado.

Tipos Mime compactáveis

Uma lista separada por vírgulas dos tipos MIME para os quais a compressão HTTP é usada. O valor padrão é text/html,text/xml,text/plain.

Tamanho mínimo da compressão

O tamanho mínimo de um arquivo quando a compressão é aplicada. Esse valor deve ser definido se a compressão estiver definida como ligado ou forçar. O valor padrão é 2.048 bytes.

Agentes do usuário sem compressão

Uma lista separada por vírgulas de expressões regulares que coincidem com agentes do usuário de clientes HTTP, para os quais a compressão não deve ser utilizada. Por padrão esse valor é uma sequência vazia.

Agente do usuário restringido

Uma lista de agentes do usuário restringido no qual a compressão HTTP é aplicada. Se nenhum agente do usuário é especificado, a compressão HTTP é aplicada em todos os agentes do usuário. Por padrão, nenhum agente do usuário é especificado.

Tempo de resposta padrão

Uma sequência que especifica o tempo de resposta padrão. O formato é uma sequência delimitada por dois pontos, consistindo no tipo de conteúdo, codificação, idioma e conjunto de caracteres. O valor padrão é text/html; charset=iso-8859-1.

Tempo de resposta forçado

Uma sequência que especifica o tipo de solicitação usado, caso nenhum mapeamento MIME esteja disponível que coincida com a extensão do arquivo. O formato é uma sequência delimitada por dois pontos, consistindo no tipo de conteúdo, codificação, idioma e conjunto de caracteres. O valor padrão é text/html; charset=iso-8859-1.

Adaptador

O nome da classe do adaptador de recursos estáticos. O valor padrão é com.sun.grizzly.tcp.StaticResourcesAdapter.

Suporte Comet

Se essa opção for selecionada, o suporte Comet será ativado para o protocolo. Por padrão, a opção está desativada.

Pesquisa DNS

Se essa opção for selecionada, a pesquisa DNS será ativada. Por padrão, a opção está desativada.

Suporte RCM

Se esta opção for selecionada, o Gerenciamento de configuração de recursos (RCM) será ativado. Por padrão, a opção está desativada.

Rastrear

Se essa opção for selecionada, a operação RASTREAR será ativada. Se essa opção for desativada, o Servidor GlassFish é menos suscetível ao ataques de scripts de site cruzado. Por padrão, a opção está ativada.

Auth Pass Through

Se essa opção for selecionada, indica que a escuta de rede que utiliza esse protocolo recebe tráfego de um proxy de servidor SSL com terminação. Por padrão, a opção está desativada.

Amostragem

Se essa opção for selecionada, a resposta de amostragem HTTP será ativada. Por padrão, a opção está ativada.

XPowered por

Se essa opção for selecionada, o Servidor GlassFish utiliza os cabeçalhos X-Powered-By, de acordo com as especificações do Java Servlet 3.0 e do Java Server Pages. Por padrão, a opção está ativa.
