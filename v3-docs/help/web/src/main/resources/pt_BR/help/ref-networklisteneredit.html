Editar escuta de rede

Utilize a página Editar escuta de rede para modificar uma escuta de rede existente.

Quando você edita uma escuta de rede, as abas SSL, HTTP e Arquivo de cache para o protocolo associado, também estão disponíveis para edição.

A página Editar escuta de rede contém essas opções.

Carregar padrões

Botão para restaurar os valores padrão de todas as configurações.

Nome da configuração

Nome da configuração ao qual as configurações desta página serão enviadas. Este campo é de somente leitura.

Nome

O nome único da escuta. O campo Nome é de somente leitura. Você pode especificar o nome somente quando cria uma nova escuta de rede.

Protocolo

O protocolo associado à escuta de rede. O campo Protocolo é de somente leitura. Você pode criar ou especificar um protocolo somente ao criar uma nova escuta de rede.

Status

O status da escuta de rede. A escuta pode ser ativada ou desativada. Por padrão, a opção está ativa. Se o interpretador estiver desativado, qualquer tentativa de se conectar à escuta resulta em uma exceção de soquete (java.net.ConnectException).

Nas versões do Servidor GlassFish anteriores a 9.1, uma escuta cujo atributo ativo estava definido como falso retornava um código de resposta 404, para quaisquer solicitações enviadas ao mesmo. Para obter esse comportamento na versão atual do Servidor GlassFish, defina o status da escuta como ativo e cada estado de servidor virtual associado como DESATIVADO. Um servidor virtual lista suas escutas associadas na sua lista suspensa Escutas de rede.

Segurança

Se essa opção for selecionada, ela será ativada para o protocolo utilizado para a escuta de rede. Por padrão, a opção está desativada.

Escuta JK

Se essa opção estiver selecionada, a escuta suporta o protocolo mod-jk do Apache. O protocolo mod-jk é um protocolo proprietário de comunicação/rede entre o httpd (servidor da Web) da Apache e uma instância back-end do Servidor GlassFish. Se a escuta oferece suporte a esse protocolo, o httpd deveria manipular todo o conteúdo estático, enquanto quaisquer solicitações por recursos dinâmicos (ou seja, Servlets e JSPs) são roteadas para a escuta mod-jk ativado para a rede do Servidor GlassFish, utilizando o protocolo mod-jk. Por padrão, a opção está desativada.

Porta

O número da porta em que a escuta será efetuada. Os valores válidos são 1 a 65.535. Em sistemas UNIX, a criação de soquetes que escutam nas portas 1 a 1024 requer privilégios de superusuário.

Endereço

O endereço IP em que a escuta de rede atuará. O endereço pode estar na notação de par de pontos ou IPv6. Ele pode ser qualquer (para INADDR_ANY) para escutar todos s endereços de IP. Ele pode ser um nome de host.

Conjunto de segmentos

O conjunto de segmentos associado à escuta de rede.

Transporte

O tipo de transporte da escuta de rede. Por padrão, o único transporte é o tcp.
