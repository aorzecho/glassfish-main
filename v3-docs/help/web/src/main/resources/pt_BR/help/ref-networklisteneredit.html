Editar Listener de Rede

Utilize a página Editar Listener de Rede para modificar um listener de rede existente.

Quando você edita um listener de rede, as guias SSL, HTTP e Arquivo de Cache para o protocolo associado, também estão disponíveis para edição.

A página Editar Listener de Rede contém as opções a seguir.

Carregar Defaults

Botão para restaurar com os valores default as definições que apresentam valores default. As definições que não apresentam valores default não são alteradas.

Nome da Configuração

O nome da configuração à qual as definições nesta página se aplicam. Este campo é somente para leitura.

Nome

O nome único do listener. O campo Nome é somente para leitura. Você só pode especificar o nome quando cria uma novo listener de rede.

Protocolo

O protocolo associado ao listener de rede. O campo Protocolo é somente para leitura. Você só pode criar ou especificar um protocolo criar um novo listener de rede.

Status

O status do listener de rede. O listener pode ser ativado ou desativado. Por default, a opção está ativada. Se o interpretador estiver desativado, qualquer tentativa de conectar ao listener resulta em uma exceção de soquete (java.net.ConnectException).

Nas versões do GlassFish Server anteriores a 9.1, um listener cujo atributo ativado estava definido como falso retornava um código de resposta 404, para quaisquer solicitações enviadas ao mesmo. Para obter esse comportamento na versão atual do GlassFish Server, defina o status do listener como ativado e cada estado de servidor virtual associado como DESLIGADO. Um servidor virtual lista suas listeners associadas na sua lista drop-down Listeners de rede.

Segurança

Se esta opção for selecionada, ela será ativada para o protocolo utilizado para o listener de rede. Por default, essa opção está desativada.

Interpretador JK

Se esta opção estiver selecionada, o listener suportará o protocolo mod-jk do Apache. O protocolo mod-jk é um protocolo proprietário de communication/network entre o httpd do Apache (servidor da web) e uma instância de back-end do GlassFish Server. Se o listener oferecer suporte a esse protocolo, o httpd deverá tratar todo o conteúdo estático, enquanto as solicitações por recursos dinâmicos (ou seja, Servlets e JSPs) serão roteadas para o interpretador mod-jk ativado para a rede do GlassFish Server, utilizando o protocolo mod-jk. Por default, essa opção está desativada.

Porta

O número da porta do listener. Os valores válidos são 1 a 65.535. Em sistemas UNIX, criar soquetes que fazem listening nas portas 1 - 1024 exige privilégios de superusuário.

Endereço

O endereço IP do listener de rede. O endereço pode estar na notação de par de pontos ou IPv6. Pode ser qualquer (para INADDR_ANY) para fazer listening em todos os endereços IP. Ele pode ser um nome de host.

Pool de Threads

O pool de threads associado ao listener de rede.

Transporte

O tipo de transporte do listener de rede. Por default, o único transporte é o tcp.
