Modification d’un pool de threads

  1. Dans l’arborescence de navigation, développez le nœud Configuration.
  2. Sous le nœud Configuration, sélectionnez le nœud Pools de threads.

    La page Pools de threads s’affiche.

  3. Dans la page Pools de threads, cliquez sur le nom du pool de threads à modifier.

    La page Modification du pool de threads s’affiche.

  4. Sur la page Modification d’un pool de threads, dans le champ Nom de classe, saisissez le nom qualifié complet de la classe qui implémente le pool de thread.

    La valeur par défaut est com.sun.grizzly.http.StatsThreadPool.

  5. Dans le champ Taille Maximum de file d’attente, saisissez le nombre maximal de messages pouvant être mis en attente, jusqu’à ce que des threads soient disponibles pour les traiter.

    Une valeur de -1 signifie que la taille de la file d’attente est illimitée. La valeur par défaut est 4096.

  6. Dans le champ Taille max. du pool de threads, saisissez le plus grand nombre de threads pouvant exister dans le pool de threads.

    La valeur par défaut est 5. Il s’agit de la valeur recommandée pour les pools de threads du listener réseau.

  7. Dans le champ Taille min. du pool de threads, saisissez le plus petit nombre de threads pouvant exister dans le pool de threads.

    Ces threads sont créés lorsque le pool de thread est instancié. La valeur par défaut est 2.

  8. Dans le champ Délai d’expiration de thread inactif, saisissez le délai maximal, en secondes, durant lequel un thread peut demeurer inactif dans le pool.

    Une fois ce délai expiré, le thread est supprimé du pool. La valeur par défaut est 900.

  9. Cliquez sur Enregistrer.

    Astuce - Pour revenir aux paramètres par défaut, cliquez sur le bouton Charger valeurs par défaut, puis sur Enregistrer.
