Modification d’un listener réseau

  1. Dans l’arborescence de navigation, développez le nœud Configuration.
  2. Sous le nœud Configuration, développez le nœud Config réseau.
  3. Sous le nœud Configuration de réseau, sélectionnez le nœud Auditeurs de réseau.

    La page Listeners réseau s’affiche.

  4. Sur la page Auditeurs de réseau, cliquez sur le nom de l’auditeur de réseau que vous modifiez.

    La page Modifier le listener réseau s’affiche.

  5. Cochez ou décochez la case État activé pour modifier l’état du listener réseau.

    Le listener peut être activé ou désactivé. Cette option est activée par défaut. Si l’auditeur est désactivé, toute tentative de connexion à l’auditeur a pour résultat une exception de socket (java.net.ConnectException).

  6. Sélectionnez la case Activé de Sécurité pour activer la sécurité du protocole utilisé par l’auditeur de réseau.

    Cette option est désactivée par défaut.

  7. Sélectionnez la case Activé de Listener JK pour indiquer que le module d’écoute prend en charge le protocole mod-jk Apache.

    Le protocole mod-jk est un protocole communication/réseau propriétaire entre httpd Apache (serveur Web) et une instance d’arrière-plan de GlassFish Server. Si l’auditeur prend en charge ce protocole, httpd est supposé gérer tout le contenu statique, tandis qu’une demande de ressources dynamiques (c’est-à-dire, de servlets et de JSP) est acheminée vers l’auditeur réseau activé mod-jk de GlassFish Server, utilisant le protocole mod-jk. Cette option est désactivée par défaut.

  8. Dans le champ Port, saisissez le numéro de port sur lequel le listener réseau va écouter.

    Les valeurs valides sont comprises entre 1 et 65535. Sur un système UNIX, la création de sockets écoutant sur les ports 1 à 1024 requiert des privilèges de superutilisateur.

  9. Dans le champ Adresse, saisissez l’adresse IP sur laquelle le listener réseau va écouter.

    L’adresse peut présenter une notation IPv6 ou des paires de chiffres séparées par des points. Elle peut être définie sur any (pour INADDR_ANY) pour écouter sur toutes les adresses IP. Il peut s’agir d’un nom d’hôte.

  10. Dans la liste déroulante Pool de thread, sélectionnez le pool de thread à associer à l’auditeur de réseau.
  11. Cliquez sur Enregistrer.

    Astuce - Pour revenir aux paramètres par défaut, cliquez sur le bouton Charger valeurs par défaut, puis sur Enregistrer.

Étapes suivantes

Vous pouvez utiliser les onglets SSL, HTTP et Cache de fichiers pour modifier les paramètres pour le protocole associé à l’auditeur de réseau. L’onglet SSL n’est utile que si vous avez activé la sécurité pour l’auditeur de réseau.
