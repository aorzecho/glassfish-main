Modification d’un pool de threads

La page Modification du pool de threads vous permet de modifier un pool de threads existant.

La page Modification du pool de threads contient les options suivantes :

Charger les valeurs par défaut

Bouton permettant de restaurer les valeurs par défaut de tous les paramètres.

Nom de configuration

Le nom de la configuration à laquelle les paramètres de cette page s'appliquent. Ce champ est en lecture seule.

Nom

Nom du pool de threads. Le champ Nom est en lecture seule. Vous pouvez spécifier le nom uniquement lorsque vous créez un nouveau pool de thread.

Nom de classe

Nom complet de la classe qui implémente le pool de threads. La valeur par défaut est com.sun.grizzly.http.StatsThreadPool.

Taille max. de la file d’attente

Le nombre maximal de messages pouvant être mis en attente jusqu’à ce que des threads soient disponibles pour les traiter. Une valeur de -1 signifie que la taille de la file d’attente est illimitée. La valeur par défaut est 4096.

Taille max. du pool de threads

Nombre maximum de threads dans le pool de threads. La valeur par défaut est 5.

Taille min. du pool de threads

Nombre minimum de threads dans le pool de threads. La valeur par défaut est 2.

Délai d’expiration de thread inactif

Temps maximum en secondes durant lequel un thread peut rester inactif dans le pool. Une fois ce délai expiré, le thread est supprimé du pool. La valeur par défaut est 900.
