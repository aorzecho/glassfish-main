Propriétés spécifiques au service HTTP

Les propriétés supplémentaires suivantes sont disponibles pour le service HTTP. Si vous spécifiez ces propriétés, elles s’appliquent à tous les Listeners réseau.

ssl-cache-entriese

Le nombre de session SSL à placer dans la mémoire cache. La valeur par défaut est 10000.

ssl3-session-timeout

L’intervalle auquel les sessions SSL3 sont placées dans la mémoire cache. La valeur par défaut est 86400.

ssl-session-timeout

L’intervalle selon lequel les sessions SSL2 sont placées dans la mémoire cache. La valeur par défaut est 100.

proxyHandler

Le nom de classe qualifié complet d’une implémentation personnalisée de la classe abstraite service com.sun.appserv.ProxyHandler, permettant une instance GlassFish Server d’arrière-plan pour récupérer des informations sur la requête client d’origine qui a été interceptée par un serveur proxy à terminaison SSL (un équilibreur de charge, par exemple). Une implémentation de cette classe abstraite inspecte une requête donnée pour la méthode par laquelle en-têtes de la demande du serveur proxy transmet l’information sur les clients initiaux demande pour le serveur d’entreprise instance, et renvoie ces informations à le programme appelant.

L’implémentation par défaut lit l’adresse IP du client à partir d’un en-tête de requête HTTP appelé Proxy-ip, la valeur keysize SSL à partir d’un en-tête de requête HTTP appelé Proxy-keysize et la chaîne de certificat client SSL à partir d’un en-tête de requête HTTP appelé Proxy-auth-cert. La valeur Proxy-auth-cert doit contenir la chaîne de certificat client en encodage BASE-64 sans les limites BEGIN CERTIFICATE et END CERTIFICATE et avec \n remplacé par % d% a.

Ce paramètre est utilisé uniquement si la case Auth. d’intercommunication pour la page HTTP du protocole d’auditeur de réseau est définie sur true.

connectionTimeout

Spécifie le nombre de secondes que patientent les auditeurs de réseau, après avoir accepté une connexion, pour la ligne URI de requête à présenter. La valeur par défaut est 30.
