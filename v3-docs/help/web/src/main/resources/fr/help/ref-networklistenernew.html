Nouveau processus d'écoute réseau

La page Nouveau processus d'écoute réseau permet de créer un processus d'écoute réseau.

Lorsque vous créez un processus d'écoute réseau, vous pouvez créer simultanément un protocole que le processus utilisera ou employer un protocole que vous avez d’abord créé. En règle générale, il existe une relation 1 à 1 entre un processus d'écoute réseau et les protocoles associés, mais ce n'est pas obligatoire.

La page Nouveau processus d'écoute réseau contient les options suivantes :

Nom de configuration

Nom de la configuration à laquelle s'appliquent les paramètres de cette page. Ce champ est en lecture seule.

Nom

Nom de processus d'écoute unique. Le nom d'un processus d'écoute réseau ne peut pas commencer par un chiffre.

Protocole

Protocole associé au processus d'écoute réseau. Les options suivantes indiquent s'il faut créer un protocole ou utiliser un protocole existant.

Créer un protocole

Un protocole avec des paramètres par défaut est créé pour le processus d'écoute réseau. Le nom par défaut du protocole est celui du processus d'écoute réseau avec la mention -protocol ajoutée à la fin, mais vous pouvez modifier cette valeur.

Serveur virtuel

Pour un nouveau protocole, serveur virtuel à associer à ce protocole.

Utiliser un protocole existant

Le protocole sélectionné est utilisé.

Statut

Statut du processus d'écoute réseau. Le processus d'écoute peut être activé ou désactivé. Cette option est activée par défaut. Si le processus d'écoute est désactivé, toute tentative de connexion au processus d'écoute a pour résultat une exception de socket (java.net.ConnectException).

Dans les versions antérieures à la version 9.1 de GlassFish Server, un processus d'écoute dont la valeur d'attribut activé était False renvoyait un code de réponse 404 pour toute demande qui lui était envoyée. Pour obtenir ce comportement dans la version en cours de GlassFish Server, activez le processus d'écoute et définissez l’état de chaque serveur virtuel associé sur OFF. Un serveur virtuel répertorie les processus d'écoute qui lui sont associés dans la liste déroulante Processus d'écoute réseau correspondante.

Sécurité

Si cette option est sélectionnée, la sécurité est activée pour le protocole utilisé par le processus d'écoute réseau. Cette option est désactivée par défaut.

Processus d'écoute JK

Si cette option est sélectionnée, le processus d'écoute prend en charge le protocolemod-jk d'Apache. Le protocole mod-jk est un protocole communication/réseau propriétaire entre le serveur Web httpd Apache et une instance back-end de GlassFish Server. Si le processus d'écoute prend en charge ce protocole, httpd est censé gérer tout le contenu statique, tandis que les demandes de ressources dynamiques (c'est-à-dire de servlets et de JSP) sont acheminées vers le processus d'écoute réseau activé mod-jk de GlassFish Server, à l'aide du protocole mod-jk. Cette option est désactivée par défaut.

Port

Numéro de port sur lequel le processus d'écoute écoutera. Les valeurs autorisées sont comprises entre 1 et 65535. Sur un système UNIX, la création de sockets écoutant sur les ports 1 à 1024 requiert des privilèges de superutilisateur.

Adresse

Adresse IP sur laquelle le processus d'écoute réseau écoute. L'adresse peut présenter une notation IPv6 ou des paires de chiffres séparées par des points. Elle peut être définie sur any (pour INADDR_ANY) pour écouter sur toutes les adresses IP. Il peut s'agir d'un nom d'hôte.

Pool de threads

Pool de threads associé au processus d'écoute réseau. Normalement, vous devez sélectionner l’un des deux pools de threads qui sont configurés lorsque vous installez GlassFish Server.

Transport

Type de transport du processus d'écoute réseau. Par défaut, le seul choix possible est tcp.
