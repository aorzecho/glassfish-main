Modifier un processus d'écoute HTTP

  1. Dans l'arborescence de navigation, développez le noeud Configuration.

  2. Sous le noeud Configuration, développez le noeud Service HTTP.

  3. Sous le noeud Configuration réseau, sélectionnez le noeud Processus d'écoute réseau.

    La page Processus d'écoute HTTP apparaît.

  4. Sur la page Processus d'écoute HTTP, cliquez sur le nom du processus d'écoute HTTP que vous modifiez.

    La page Modifier le processus d'écoute HTTP apparaît.

  5. Dans le champ Port, saisissez le numéro de port sur lequel le processus d'écoute HTTP va écouter.

    Les valeurs autorisées sont comprises entre 1 et 65535. Sur un système UNIX, la création de sockets écoutant sur les ports 1 à 1024 requiert des privilèges de superutilisateur.

  6. Cochez ou désélectionnez la case Activé de l'option Statut pour modifier le statut du processus d'écoute HTTP.

    Le processus d'écoute peut être activé ou désactivé. Cette option est activée par défaut. Si le processus d'écoute est désactivé, toute tentative de connexion au processus d'écoute a pour résultat une exception de socket (java.net.ConnectException).

  7. Cochez la case Sécurité activée pour activer la sécurité pour le processus d'écoute HTTP.

    Cette option est désactivée par défaut.

  8. Cochez la case Activé de Processus d'écoute JK pour indiquer que le processus d'écoute prend en charge le protocole mod-jk Apache.

    Le protocole mod-jk est un protocole communication/réseau propriétaire entre le serveur Web httpd Apache et une instance back-end de GlassFish Server. Si le processus d'écoute prend en charge ce protocole, httpd est censé gérer tout le contenu statique, tandis que les demandes de ressources dynamiques (c'est-à-dire de servlets et de JSP) sont acheminées vers le processus d'écoute HTTP activé mod-jk de GlassFish Server, à l'aide du protocole mod-jk. Cette option est désactivée par défaut.

  9. Dans le champ Adresse, saisissez l'adresse IP sur laquelle le processus d'écoute HTTP va écouter.

    L'adresse peut présenter une notation IPv6 ou des paires de chiffres séparées par des points. Elle peut être définie sur any (pour INADDR_ANY) pour écouter sur toutes les adresses IP. Il peut s'agir d'un nom d'hôte.

  10. Dans la liste déroulante Serveur virtuel par défaut, sélectionnez le serveur virtuel à associer avec ce processus d'écoute HTTP.

    La page Serveurs virtuels permet de définir des serveurs virtuels.

  11. Dans la liste déroulante Pool de threads, sélectionnez le pool de threads à associer au processus d'écoute HTTP.

    Normalement, vous devez sélectionner l’un des deux pools de threads qui sont configurés lorsque vous installez GlassFish Server.

  12. Dans le champ Nom du serveur, saisissez le nom d'hôte à utiliser dans les URL envoyées au client par le serveur.

    Ce nom correspond à l'alias si votre serveur en utilise un. Si votre serveur n'utilise aucun alias, laissez ce champ vide.

    Cette valeur a une influence sur les URL que le serveur génère automatiquement ; elle n'a aucun impact sur les URL des répertoires et des fichiers stockés sur le serveur Ce nom correspond au nom de l'alias si votre serveur en utilise un. Si vous ajoutez le signe deux-points et un numéro de port à la fin, ce port sera utilisé dans les URL que le serveur envoie au client.

  13. Cliquez sur Enregistrer.

    Conseil:

    Pour revenir aux paramètres par défaut, cliquez sur le bouton Charger les valeurs par défaut, puis sur Enregistrer.
Etapes suivantes

Si vous avez activé la sécurité pour le processus d'écoute HTTP, vous pouvez utiliser l'onglet SSL pour modifier les paramètres SSL du processus d'écoute HTTP.

Un processus d'écoute HTTP est un type de processus d'écoute réseau. Pour accéder aux fonctionnalités avancées, vous pouvez modifier un processus d'écoute HTTP à l'aide des pages Processus d'écoute réseau.
