Création d’un listener réseau

Avant de commencer

Lorsque vous créez un auditeur de réseau, vous pouvez créer simultanément un protocole que l’auditeur de réseau utilisera ou utiliser un protocole que vous avez d’abord créé. En règle générale, il existe une correspondance un-un entre un auditeur de réseau et ses protocoles associés, mais ce n’est pas obligatoire.

  1. Dans l’arborescence de navigation, développez le nœud Configuration.
  2. Sous le nœud Configuration, développez le nœud Config réseau.
  3. Sous le nœud Configuration de réseau, sélectionnez le nœud Auditeurs de réseau.

    La page Listeners réseau s’affiche.

  4. Dans la page Listeners réseau, cliquez sur Nouveau.

    La page Nouveau listener réseau s’affiche.

  5. Dans le champ Nom, saisissez un nom pour l’auditeur de réseau.

    Le nom d’un auditeur de réseau ne peut pas commencer par un chiffre.

  6. Dans le champ Protocole, spécifiez le protocole à associer à l’auditeur de réseau.
    • Sélectionnez Créer un nouveau protocole pour créer un nouveau protocole avec les paramètres par défaut. De plus, dans la liste déroulante Serveur virtuel, sélectionnez le serveur virtuel à associer au protocole.

      Le nom par défaut du protocole est le nom de l’auditeur de réseau avec la mention -protocol ajoutée, mais vous pouvez modifier cette valeur.

    • Sélectionnez Utiliser un protocole existant, puis sélectionnez un protocole dans la liste déroulante.

      Si vous sélectionnez cette option, vous aurez généralement créé le protocole pour ce module d’écoute au préalable.

    Vous pouvez spécifier le protocole uniquement lorsque vous créez l’auditeur de réseau. Vous ne pouvez pas modifier le protocole associé ultérieurement.

  7. Cochez la case État activé pour activer le listener réseau.

    Cette option est activée par défaut. Si l’auditeur est désactivé, toute tentative de connexion à l’auditeur a pour résultat une exception de socket (java.net.ConnectException).

    Dans les versions antérieures à la version 9.1 de GlassFish Server, un auditeur dont l’attribut activé a été défini sur false renvoyait un code de réponse 404 pour toute requête qui lui était envoyée. Pour atteindre ce comportement dans la version actuelle de GlassFish Server, réglez le statut de l’auditeur sur activé et définissez l’état de chaque serveur virtuel associé sur OFF. Un serveur virtuel répertorie ses auditeurs associés dans sa liste déroulante Auditeurs de réseau.

  8. Sélectionnez la case Activé de Sécurité pour activer la sécurité du protocole utilisé par l’auditeur de réseau.

    Cette option est désactivée par défaut.

  9. Sélectionnez la case Activé de Listener JK pour indiquer que le module d’écoute prend en charge le protocole mod-jk Apache.

    Le protocole mod-jk est un protocole communication/réseau propriétaire entre httpd Apache (serveur Web) et une instance d’arrière-plan de GlassFish Server. Si l’auditeur prend en charge ce protocole, httpd est supposé gérer tout le contenu statique, tandis qu’une demande de ressources dynamiques (c’est-à-dire, de servlets et de JSP) est acheminée vers l’auditeur réseau activé mod-jk de GlassFish Server, utilisant le protocole mod-jk. Cette option est désactivée par défaut.

  10. Dans le champ Port, saisissez le numéro de port sur lequel le listener réseau va écouter.

    Les valeurs valides sont comprises entre 1 et 65535. Sur un système UNIX, la création de sockets écoutant sur les ports 1 à 1024 requiert des privilèges de superutilisateur.

  11. Dans le champ Adresse, saisissez l’adresse IP sur laquelle le listener réseau va écouter.

    L’adresse peut présenter une notation IPv6 ou des paires de chiffres séparées par des points. Elle peut être définie sur any (pour INADDR_ANY) pour écouter sur toutes les adresses IP. Il peut s’agir d’un nom d’hôte.

  12. Dans la liste déroulante Pool de thread, sélectionnez le pool de thread à associer à l’auditeur de réseau.

    Normalement, vous sélectionnez l’un des deux pools de thread qui sont configurées lorsque vous installez GlassFish Server.

  13. Dans la liste déroulante Transport, sélectionnez le type de transport à utiliser par le listener réseau.

    Par défaut, le seul choix possible est tcp.

  14. Cliquez sur OK.
Étapes suivantes

Si vous avez activé la sécurité pour l’auditeur de réseau, vous pouvez utiliser l’onglet SSL pour modifier les paramètres SSL du protocole utilisé par l’auditeur réseau.
