Éditer le listener réseau

La page Modification du listener réseau vous permet de modifier un listener réseau existant.

Lorsque vous modifiez un auditeur de réseau, les onglets SSL, HTTP et Cache de fichiers pour le protocole associé sont disponibles pour toute modification de votre part.

La page Modification du listener réseau contient les options suivantes :

Charger les valeurs par défaut

Bouton permettant de restaurer les valeurs par défaut de tous les paramètres.

Nom de configuration

Le nom de la configuration à laquelle les paramètres de cette page s'appliquent. Ce champ est en lecture seule.

Nom

Nom de l’auditeur unique. Le champ Nom est en lecture seule. Vous pouvez uniquement spécifier le nom lorsque vous créez un nouvel auditeur de réseau.

Protocole

Protocole associé au listener réseau. Le champ Protocole est un champ en lecture seule. Vous pouvez créer ou spécifier un protocole uniquement lorsque vous créez un nouvel auditeur de réseau.

Statut

Statut du listener de réseau. Le listener peut être activé ou désactivé. Cette option est activée par défaut. Si l’auditeur est désactivé, toute tentative de connexion à l’auditeur a pour résultat une exception de socket (java.net.ConnectException).

Dans les versions antérieures à la version 9.1 de GlassFish Server, un auditeur dont l’attribut activé a été défini sur false renvoyait un code de réponse 404 pour toute requête qui lui était envoyée. Pour atteindre ce comportement dans la version actuelle de GlassFish Server, réglez le statut de l’auditeur sur activé et définissez l’état de chaque serveur virtuel associé sur OFF. Un serveur virtuel répertorie ses auditeurs associés dans sa liste déroulante Auditeurs de réseau.

Sécurité

Si cette option est sélectionnée, la sécurité est activée pour le protocole utilisé par l’auditeur de réseau. Cette option est désactivée par défaut.

Listener JK

Si cette option est sélectionnée, l’auditeur prend en charge le protocolemod-jk d’Apache. Le protocole mod-jk est un protocole communication/réseau propriétaire entre httpd Apache (serveur Web) et une instance d’arrière-plan de GlassFish Server. Si l’auditeur prend en charge ce protocole, httpd est supposé gérer tout le contenu statique, tandis qu’une demande de ressources dynamiques (c’est-à-dire, de servlets et de JSP) est acheminée vers l’auditeur réseau activé mod-jk de GlassFish Server, utilisant le protocole mod-jk. Cette option est désactivée par défaut.

Port

Le numéro de port sur lequel l’auditeur écoutera. Les valeurs valides sont comprises entre 1 et 65535. Sur un système UNIX, la création de sockets écoutant sur les ports 1 à 1024 requiert des privilèges de superutilisateur.

Adresse

Adresse IP sur laquelle le listener réseau écoute. L’adresse peut présenter une notation IPv6 ou des paires de chiffres séparées par des points. Elle peut être définie sur any (pour INADDR_ANY) pour écouter sur toutes les adresses IP. Il peut s’agir d’un nom d’hôte.

Pool de threads

Pool de threads associé au listener réseau.

Transport

Type de transport pour le listener réseau. Par défaut, le seul choix possible est tcp.
