Nouveau serveur virtuel

La page Nouveau serveur virtuel vous permet de créer un serveur virtuel.

La page Nouveau serveur virtuel contient les options suivantes :

Nom de configuration

Le nom de la configuration à laquelle les paramètres de cette page s'appliquent. Ce champ est en lecture seule.

ID

Identificateur de serveur virtuel visible en interne. Elle n’est pas visible des clients HTTP. Les noms d’hôtes visibles des clients HTTP doivent être spécifiés dans le champ Hôtes.

Hôtes

Nom(s) d’hôte de la machine sur laquelle le serveur est exécuté. Utilisez des noms d’hôte réels ou virtuels enregistrés dans le serveur DNS pour votre réseau (et, sous UNIX, dans votre fichier /etc/hosts). La valeur par défaut est la valeur de propriété système ${com.sun.aas.hostName}.

État

État souhaité pour le serveur virtuel. La valeur peut être l’une des suivantes :

Activé

Le serveur virtuel est actif. Il s’agit de la valeur par défaut.

Désactivé

Le serveur virtuel est inactif. Les tentatives d’accès au serveur renvoient le code d’erreur 404 (ressource non disponible).

Désactivé

Le serveur virtuel est désactivé. Les tentatives d’accès au serveur renvoient le code d’erreur 403 (refus d’exécuter la requête).

SSO

Spécifie si un comportement de connexion unique est hérité du service HTTP, activé ou désactivé. Par défaut, elle est héritée du service HTTP. Si cette option est activée, la connexion unique est activée pour les applications Web sur ce serveur virtuel qui sont configurées pour le même domaine. Si cette option est désactivée, la connexion unique est désactivée pour ce serveur virtuel et les utilisateurs doivent s’authentifier séparément pour chaque application sur le serveur virtuel.

Network Listeners

L’auditeur ou les auditeurs de réseau associés à ce serveur, le cas échéant.

Module Web par défaut

Module Web déployé (le cas échéant) qui doit répondre à toutes les requêtes ne pouvant pas être mappées à d’autres modules Web déployés sur le serveur virtuel. Si aucun module Web par défaut n’est spécifié, le module Web qui possède une racine contexte vide est utilisé. S’il n’existe aucun module Web avec une racine contexte vide, un module Web par défaut du système est créé et utilisé.

Fichier journal

Nom de chemin du fichier où les messages de journalisation provenant de ce serveur virtuel doivent s’afficher. Par défaut, les messages de journalisation apparaîtront dans domain-dir/logs/server.log.

Docroot

Chemin d’accès absolu au répertoire de documents racine pour le serveur. La valeur par défaut est domain-dir/docroot.

Journalisation des accès

Spécifie si la journalisation d’accès est héritée du service HTTP, activée ou désactivée. Par défaut, elle est héritée du service HTTP.

Répertoire

Chemin d’accès absolu aux journaux d’accès du serveur. La valeur par défaut est domain-dir/logs/access.

Autres propriétés

Propriétés supplémentaires pour le serveur virtuel.

Pour obtenir des informations sur les propriétés disponibles, consultez Propriétés spécifiques aux serveurs virtuels.
