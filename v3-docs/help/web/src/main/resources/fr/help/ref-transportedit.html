Modifiez le transport

Utilisez la page Modifiez le transport pour modifier les paramètres d’un transport existant.

La page Modifiez le transport contient les options suivantes.

Restaurer les valeurs par défaut

Bouton de restauration des valeurs par défaut des paramètres qui ont des valeurs par défaut. Les paramètres qui n'ont pas de valeurs par défaut ne sont pas modifiés.

Nom de la configuration

Nom de la configuration à laquelle les paramètres de cette page s'appliquent. Ce champ est en lecture seule.

Nom

Le nom du transport. Le champ Nom est en lecture seule. Vous pouvez spécifier le nom uniquement lorsque vous créez un nouveau transport.

Nom de classe

Le nom de classe de l’implémentation du transport. La valeur par défaut est com.sun.grizzly.TCPSelectorHandler.

Gestionnaires de clé de sélection

Le nom de classe du gestionnaire clé de sélection. Le gestionnaire de clé de sélection est une classe abstraite qui gère le cycle de vie des touches de sélection. Si vous ne spécifiez pas une valeur, une implémentation par défaut est utilisée.

Type de tampon de byte

Le type de tampon d’octet à utiliser. Les choix disponibles sont HEAP et DIRECT. La valeur par défaut est HEAP.

Threads accepteur

Le nombre de processeurs dans la machine. Une valeur de -1 indique que Grizzly évaluera le nombre de threads accepteur lui-même, en fonction du nombre de processeurs de la machine. La valeur par défaut est 1.

Pour définir le nombre de threads de traitement des requêtes, définissez la valeur Taille max. du pool de threads du pool de threads utilisé par un auditeur de réseau qui utilise ce transport.

Nombre max. de connexions

Le nombre maximal de connexions en attente sur un auditeur de réseau qui utilise ce transport. La valeur par défaut est 4096.

Taille du tampon

La taille (en octets) du tampon à fournir pour les flux d’entrée créés par un auditeur de réseau qui utilise ce transport. La valeur par défaut est 8192.

Délai d’expiration de la clé d’inactivité

Le délai (en secondes) après lequel la clé d’inactivité sera annulée et le canal fermé. La valeur par défaut est 30.

Délai d’expiration de lecture

Le délai (en millisecondes) pendant lequel GlassFish Server patiente lors de la phase d’analyse de l’en-tête et du corps d’une opération de lecture. La valeur par défaut est 30000.

Délai d’expiration de l’interrogation du sélecteur

Le délai (en millisecondes) pendant lequel un sélecteur NIO bloque l’attente d’événements (requêtes utilisateur). La valeur par défaut est 1000.

Délai d’expiration d’écriture

Le délai (en millisecondes) pendant lequel GlassFish Server patiente avant de considérer le client distant déconnecté lors de l’écriture de la réponse. La valeur par défaut est 30000.

Configuration de l’affichage

Si cette option est sélectionnée, la configuration interne de Grizzly est purgée vers les journaux du serveur. Cette option peut fournir des informations intéressantes pour le débogage. Cette option est désactivée par défaut.

Snoop

Si cette option est sélectionnée, les informations requête/réponse sont purgées vers le journal du serveur. Cette option peut fournir des informations intéressantes pour le débogage, mais peut réduire considérablement les performances. Cette option est désactivée par défaut.

Aucun délai TCP

Si cette option est sélectionnée, TCP_NODELAY (également appelé l’algorithme de Nagle) est activé. Cette option est activée par défaut.
