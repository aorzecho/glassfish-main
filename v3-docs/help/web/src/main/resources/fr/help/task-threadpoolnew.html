Création d’un pool de threads

Les pools de thread sont utilisés par les auditeurs de réseau ainsi que par l’ORB. Généralement, un auditeur de réseau utilise l’un des deux pools de thread créés lors de l’installation de GlassFish Server, et ORB utilise l’autre. Vous pouvez modifier l’un ou l’autre des pools de thread par défaut afin de spécifier des paramètres propres à vos applications ou à vos systèmes matériels, ou vous pouvez créer et utiliser un pool de thread avec ces paramètres.

  1. Dans l’arborescence de navigation, développez le nœud Configuration.
  2. Sous le nœud Configuration, sélectionnez le nœud Pools de threads.

    La page Pools de threads s’affiche.

  3. Cliquez sur Nouveau dans la page Pools de threads.

    La page Nouveau pool de threads s’affiche.

  4. Dans le champ Nom, saisissez un nom pour le pool de threads.
  5. Dans le champ Nom de classe, saisissez le nom qualifié complet de la classe qui implémente le pool de thread.

    La valeur par défaut est com.sun.grizzly.http.StatsThreadPool.

  6. Dans le champ Taille Maximum de file d’attente, saisissez le nombre maximal de messages pouvant être mis en attente, jusqu’à ce que des threads soient disponibles pour les traiter.

    Une valeur de -1 signifie que la taille de la file d’attente est illimitée. La valeur par défaut est 4096.

  7. Dans le champ Taille max. du pool de threads, saisissez le plus grand nombre de threads pouvant exister dans le pool de threads.

    La valeur par défaut est 5. Il s’agit de la valeur recommandée pour les pools de threads du listener réseau.

  8. Dans le champ Taille min. du pool de threads, saisissez le plus petit nombre de threads pouvant exister dans le pool de threads.

    Ces threads sont créés lorsque le pool de thread est instancié. La valeur par défaut est 2.

  9. Dans le champ Délai d’expiration de thread inactif, saisissez le délai maximal, en secondes, durant lequel un thread peut demeurer inactif dans le pool.

    Une fois ce délai expiré, le thread est supprimé du pool. La valeur par défaut est 900.

  10. Cliquez sur OK.
