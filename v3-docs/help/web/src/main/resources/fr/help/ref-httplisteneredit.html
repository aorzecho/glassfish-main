Modifier le processus d'écoute HTTP

La page Modifier le processus d'écoute HTTP permet de modifier un processus d'écoute HTTP existant.

Lorsque vous modifiez un processus d'écoute HTTP, vous pouvez également modifier l'onglet SSL.

La page Modifier le processus d'écoute HTTP contient les options suivantes :

Charger les valeurs par défaut

Bouton de restauration des valeurs par défaut des paramètres qui ont des valeurs par défaut. Les paramètres qui n'ont pas de valeurs par défaut ne sont pas modifiés.

Nom de configuration

Nom de la configuration à laquelle s'appliquent les paramètres de cette page. Ce champ est en lecture seule.

Nom

Nom unique du processus d'écoute. Le champ Nom est en lecture seule. Vous pouvez uniquement spécifier le nom lorsque vous créez un processus d'écoute HTTP.

Port

Numéro de port sur lequel le processus d'écoute écoutera. Les valeurs autorisées sont comprises entre 1 et 65535. Sur un système UNIX, la création de sockets écoutant sur les ports 1 à 1024 requiert des privilèges de superutilisateur.

Statut

Statut du processus d'écoute HTTP. Le processus d'écoute peut être activé ou désactivé. Cette option est activée par défaut. Si le processus d'écoute est désactivé, toute tentative de connexion au processus d'écoute a pour résultat une exception de socket (java.net.ConnectException).

Dans les versions antérieures à la version 9.1 de GlassFish Server, un processus d'écoute dont la valeur d'attribut activé était False renvoyait un code de réponse 404 pour toute demande qui lui était envoyée. Pour obtenir ce comportement dans la version en cours de GlassFish Server, activez le processus d'écoute et définissez l’état de chaque serveur virtuel associé sur OFF. Un serveur virtuel répertorie les processus d'écoute qui lui sont associés dans la liste déroulante Processus d'écoute réseau correspondante.

Sécurité

Si cette option est sélectionnée, la sécurité est activée pour le processus d'écoute HTTP. Cette option est désactivée par défaut.

Processus d'écoute JK

Si cette option est sélectionnée, le processus d'écoute prend en charge le protocolemod-jk d'Apache. Le protocole mod-jk est un protocole communication/réseau propriétaire entre le serveur Web httpd Apache et une instance back-end de GlassFish Server. Si le processus d'écoute prend en charge ce protocole, httpd est censé gérer tout le contenu statique, tandis que les demandes de ressources dynamiques (c'est-à-dire de servlets et de JSP) sont acheminées vers le processus d'écoute HTTP activé mod-jk de GlassFish Server, à l'aide du protocole mod-jk. Cette option est désactivée par défaut.

Adresse

Adresse IP sur laquelle le processus d'écoute réseau écoute. L'adresse peut présenter une notation IPv6 ou des paires de chiffres séparées par des points. Elle peut être définie sur any (pour INADDR_ANY) pour écouter sur toutes les adresses IP. Il peut s'agir d'un nom d'hôte.

Serveur virtuel par défaut

Serveur virtuel à associer à ce processus d'écoute HTTP. La page Serveurs virtuels permet de définir des serveurs virtuels.

Pool de threads

Pool de threads associé au processus d'écoute HTTP.

Nom de serveur

Nom d'hôte à utiliser dans les URL envoyées par le serveur au client. Ce nom correspond à l'alias si votre serveur en utilise un. Si votre serveur n'utilise aucun alias, laissez ce champ vide.

Cette valeur a une influence sur les URL que le serveur génère automatiquement ; elle n'a aucun impact sur les URL des répertoires et des fichiers stockés sur le serveur Ce nom correspond au nom de l'alias si votre serveur en utilise un. Si vous ajoutez le signe deux-points et un numéro de port à la fin, ce port sera utilisé dans les URL que le serveur envoie au client.
