Nouveau protocole

Utilisez la page Nouveau protocole pour créer un nouveau protocole réseau.

Une fois que vous avez créé un protocole, en règle générale, vous créez un auditeur de réseau du même nom, et vous associez l’auditeur réseau au protocole. Vous pouvez également créer un auditeur de réseau et avoir un protocole avec des paramètres par défaut créé pour vous en même temps.

La page Nouveau protocole contient les options suivantes :

Nom de configuration

Le nom de la configuration à laquelle les paramètres de cette page s'appliquent. Ce champ est en lecture seule.

Nom

Le nom unique du protocole. Un nom de protocole ne peut pas commencer par un chiffre.

Sécurité

Si l’option est sélectionnée, la sécurité est activée pour le protocole. Cette option est désactivée par défaut. Si vous activez la sécurité, vous pouvez utiliser l’onglet SSL de la page Modification de protocole pour modifier les paramètres de sécurité.

Statut

Si cette option est sélectionnée, le cache de fichiers est activé. Cette option est activée par défaut.

Âge max.

Âge maximum, en secondes, d’une entrée de cache valide.

Ce paramètre détermine comment des informations en cache depuis un certain temps sont utilisées après la mise en cache d’un fichier. Une entrée antérieure à l’âge maximal est remplacée par une nouvelle entrée pour le même fichier. Si le contenu change rarement, augmentez cette valeur pour améliorer les performances.

L’âge maximal optimal dépend de la fréquence à laquelle les fichiers existants sont modifiés. Par exemple, si les fichiers sont modifiés quatre fois par jour à intervalles réguliers, vous pouvez définir l’âge maximal sur 21600 secondes (6 heures). Sinon, pensez à définir l’âge maximal sur la période la plus longue durant laquelle vous acceptez d’utiliser la version précédente d’un fichier après sa modification.

La valeur par défaut est 30.

Taille max. du cache

Taille totale maximale (en octets) du cache de fichiers sur le disque. La valeur par défaut est 10485760.

Nombre max. de fichiers

Nombre maximal de fichiers pouvant être stockés dans le cache.

Si la valeur est trop élevée, le serveur met en cache les fichiers peu utiles qui consomment de la mémoire. Si elle est trop faible, vous perdez l’avantage de la mise en cache. Testez différentes valeurs de cet attribut pour rechercher la solution optimale pour des applications spécifiques.

La valeur par défaut est 1024.

Nom de serveur

Nom d’hôte à utiliser dans les URL envoyées par le serveur au client. Ce nom est l’alias si votre serveur en utilise un. Si votre serveur n’utilise aucun alias, laissez ce champ vide.

Cette valeur affecte les URL que le serveur génère automatiquement ; elle n’affecte pas les URL des répertoires et des fichiers stockés dans le serveur. Si votre serveur utilise un alias, le nom du serveur doit être le nom de l’alias. Si un signe deux-points et un numéro de port sont ajoutés, ce port est utilisé dans les URL envoyés au client par le serveur.

Serveur virtuel par défaut

Le serveur virtuel à associer à ce protocole. La page Serveurs virtuels vous permet de définir des serveurs virtuels.

Port de redirection

Valeur de port qui redirige une requête vers un autre port. GlassFish Server réachemine automatiquement la requête si les deux conditions suivantes sont remplies :

  • L’auditeur de réseau qui référencie ce protocole prend en charge les requêtes non SSL.

  • Une requête pour laquelle une contrainte de sécurité correspondante requiert le transport SSL est reçue.

Si aucun port de redirection n’est spécifié, le GlassFish Server utilise le numéro de port spécifié dans la requête d’origine. Par défaut, aucun port n’est spécifié.

Nb max. de connexions

Spécifie le nombre maximum de requêtes qui peuvent être envoyées dans le pipeline jusquà ce que la connexion soit fermée par le serveur. Le sous-système Keep-Alive interroge périodiquement les connexions inactives. La valeur par défaut est 256.

Définissez cette propriété sur 1 pour désactiver la connexion Keep-Alive HTTP/1.0, ainsi que la connexion Keep-Alive HTTP/1.1 et le pipelining. Une valeur de 0 signifie que les requêtes sont toujours rejetées. Une valeur de -1 indique que le nombre de connexions persistantes est illimité.

Délai d’attente

Temps maximum en secondes pendant lequel une connexion peut être considérée comme inactive et conservée à l’état de persistance. Une valeur de 0 ou moins signifie que les connexions persistantes sont conservées ouvertes indéfiniment. La valeur par défaut est 30 secondes.

Délai d’expiration du chargement

Si cette option est sélectionnée, la connexion pour un servlet qui lit les octets lentement est fermée lorsque la limite de Délai d’expiration du chargement de la connexion est atteinte. Si cette option est désactivée, les connexions servlet n’ont pas de délai d’expiration. Cette option est désactivée par défaut.

Délai d’expiration du chargement de la connexion

Le délai d’expiration pour les téléchargements, en millisecondes. Ce champ n’est applicable que si la case Activé de Délai d’expiration du chargement est sélectionnée. La valeur par défaut est 300000 millisecondes.

Délai d’expiration de la requête

Nombre de secondes avant l’expiration de la requête. Si la requête n’est pas traitée avant d’atteindre la valeur de délai d’expiration, la requête est ignorée. La valeur par défaut est 30 secondes.

Taille du tampon d’envoi

La taille (en octets) du tampon d’envoi. La valeur par défaut est 8192 octets.

Longueur du tampon d’en-tête

La taille (en octets) du tampon utilisé par la requête traitant les threads pour lire les données de la requête. La valeur par défaut est 8192 octets.

Taille maximum des billets

La taille maximale en octets des actions POST. La valeur par défaut est 2097152 octets.

Codage URI

Le nom du jeu de caractères utilisé pour décoder les URI des requêtes reçues. La valeur doit être un nom de jeu de caractères IANA valide. La valeur par défaut est UTF-8.

Version

La version du protocole HTTP utilisée. La valeur par défaut est HTTP/1.1.

Compression

Spécifie l'utilisation de la compression HTTP/1.1 GZIP pour enregistrer la bande passante du serveur. Les choix disponibles sont les suivants :

on

Compresse les données.

off

Désactive la compression.

force

Force la compression des données dans tous les cas.

La valeur par défaut est off.

Types MIME compressibles

Une liste séparée par des virgules de types MIME pour lesquels la compression HTTP est utilisée. La valeur par défaut est text/html,text/xml,text/plain.

Taille minimum de compression

La taille minimale d’un fichier lorsque la compression est appliquée. Cette valeur doit être configurée si la compression est définie sur on ou force. La valeur par défaut est 2048 octets.

Agents utilisateur sans compression

Une liste séparée par des virgules d’expressions régulières correspondant à des agents utilisateur de clients HTTP pour lesquels la compression ne devrait pas être utilisée. Par défaut, cette valeur est une chaîne vide.

Agent utilisateur limité

Une liste des agents utilisateur limités sur lesquels la compression HTTP est appliquée. Si aucun utilisateur agent n’est spécifié, la compression HTTP est appliquée à tous les agents utilisateur. Par défaut, aucun utilisateur agent n’est spécifié.

Type de réponse par défaut

Une chaîne qui spécifie le type de réponse par défaut. Le format est une chaîne séparée par des points-virgule présentant le type de contenu (content-type), l’encodage, la langue et le jeu de caractères. La valeur par défaut est text/html; charset=iso-8859-1.

Type de réponse forcée

Une chaîne qui spécifie le type de requête utilisé si aucun mappage MIME n’est disponible correspondant à l’extension du fichier. Le format est une chaîne séparée par des points-virgule présentant le type de contenu (content-type), l’encodage, la langue et le jeu de caractères. La valeur par défaut est text/html; charset=iso-8859-1.

Adaptateur

Le nom de classe de l’adaptateur de ressources statiques. La valeur par défaut est com.sun.grizzly.tcp.StaticResourcesAdapter.

Support Comet

Si cette option est sélectionnée, la prise en charge de Comet est activée pour l’auditeur de réseau qui référencie ce protocole. Par défaut, cette option est désactivée. Si votre servlet ou votre page JSP utilise la technologie Comet, assurez-vous qu’il ou elle est initialisé(e) lorsque GlassFish Server démarre en ajoutant l’élément load-on-startup à votre fichier web.xml. Exemple :

<servlet>
<servlet-name>CheckIn</servlet-name>
<servlet-class>CheckInServlet</servlet-class>
<load-on-startup>0</load-on-startup>
</servlet>
Recherche DNS

Si cette option est sélectionnée, la recherche DNS est activée. Cette option est désactivée par défaut.

Recherche DNS

Si cette option est sélectionnée, la recherche DNS est activée. Cette option est désactivée par défaut.

Prise en charge de RCM

Si cette option est sélectionnée, RCM est activé. Cette option est désactivée par défaut.

Suivi

Si cette option est sélectionnée, l’opération Suivi est activée. Si cette option est désactivée, GlassFish Server est moins exposé à des attaques de scripts entre sites. Cette option est activée par défaut.

Auth. d’intercommunication

Si cette option est sélectionnée, l’auditeur de réseau qui utilise ce protocole reçoit le trafic d’un serveur proxy à terminaison SSL. Cette option est désactivée par défaut.

Segmentation

Si cette option est sélectionnée, le découpage des réponses HTTP est activé. Cette option est activée par défaut.

XPowered By

Si cette option est sélectionnée, GlassFish Server utilise les en-têtes X-Powered-By? en fonction des spécifications Java Servlet 3.0 et Java Server Pages 2.0. Cette option est activée par défaut.
