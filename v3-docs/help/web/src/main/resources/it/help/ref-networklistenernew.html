New Network Listener

Nella pagina New Network Listener è possibile creare un nuovo listener di rete.

Quando si crea un listener di rete, è possibile creare simultaneamente un protocollo per il listener di rete, oppure utilizzare un protocollo creato in precedenza. In genere tra un listener di rete e il protocollo associato viene stabilita una relazione uno-a-uno, ma non si tratta di un requisito obbligatorio.

La pagina New Network Listener contiene le seguenti opzioni.

Configuration Name

Nome della configurazione a cui vengono applicate le impostazioni in questa pagina. Il campo è di sola lettura.

Name

Nome univoco del listener. I nomi di listener di rete non possono iniziare con un numero.

Protocol

Protocollo associato al listener di rete. Le seguenti opzioni specificano se creare un protocollo o utilizzarne uno esistente.

Create a New Protocol

Viene creato un nuovo protocollo con impostazioni predefinite per il listener di rete. Il nome predefinito del protocollo è il nome del listener di rete con l'aggiunta del suffisso -protocol, ma il valore è modificabile.

Virtual Server

Server virtuale da associare a un nuovo protocollo.

Use an Existing Protocol

Viene utilizzato il protocollo selezionato.

Status

Stato del listener di rete. Il listener può essere abilitato o disabilitato. Questa opzione è abilitata per impostazione predefinita. Se il listener è disabilitato, ogni tentativo di connessione al listener genera un'eccezione nel socket (java.net.ConnectException).

Nelle versioni di GlassFish Server anteriori alla 9.1, in caso di listener con attributo Enabled impostato su False, veniva restituito un codice di risposta 404 per ogni richiesta inviata. Per ottenere questo funzionamento nella versione corrente di GlassFish Server, impostare lo stato del listener su Enabled e lo stato di ciascun server virtuale associato su OFF. I listener associati a un server virtuale vengono riportati nell'elenco a discesa Network Listeners disponibile nel server.

Security

Se l'opzione è selezionata viene abilitata la protezione per il protocollo utilizzato dal listener di rete. Questa opzione è disabilitata per impostazione predefinita.

JK Listener

Se l'opzione è selezionata, il listener supporta il protocollo mod-jk di Apache. Il protocollo mod-jk è un protocollo di comunicazione/rete proprietario tra httpd (server Web) di Apache e un'istanza backend di GlassFish Server. Se il listener supporta il protocollo, tutto il contenuto statico dovrebbe essere gestito mediante httpd, mentre le richieste di risorse dinamiche (ad esempio servlet e pagine JSP) vengono reindirizzate al listener di rete abilitato mod-jk di GlassFish Server tramite il protocollo mod-jk. Questa opzione è disabilitata per impostazione predefinita.

Port

Numero della porta di attesa del listener. Sono ammessi i valori compresi tra 1 e 65535. In un sistema UNIX, per creare socket in attesa su porte comprese tra 1 e 1024 sono necessari privilegi di superutente.

Address

Indirizzo IP di attesa per il listener di rete. L'indirizzo può essere specificato utilizzando la notazione per indirizzi IPv6 o con separazione mediante coppie di punti. È possibile specificare any (per INADDR_ANY) per impostare tutti gli indirizzi IP per l'attesa. Può essere costituito da un nome host.

Thread Pool

Pool di thread associato al listener di rete. In genere si seleziona uno dei due pool di thread configurati durante l'installazione di GlassFish Server.

Transport

Tipo di trasporto per il listener di rete. Per impostazione predefinita, l'unica opzione è tcp.
