New Virtual Server

Nella pagina New Virtual Server è possibile creare un server virtuale.

La pagina New Virtual Server contiene le seguenti opzioni.

Configuration Name

Nome della configurazione a cui vengono applicate le impostazioni in questa pagina. Il campo è di sola lettura.

Id

Identificatore del server virtuale visibile internamente. Non viene visualizzato ai client HTTP. I nomi host visibili ai client HTTP devono essere specificati nel campo Hosts.

Hosts

Nome o nomi host per la macchina in cui è in esecuzione il server. Utilizzare i nomi reali o i nomi virtuali registrati con il server DNS per la rete (e, in un sistema UNIX, nel file /etc/hosts). Il valore predefinito è il valore della proprietà di sistema ${com.sun.aas.hostName}.

State

Stato desiderato per il server virtuale. Il valore può essere uno qualsiasi dei seguenti:

On

Il server virtuale è attivo. Questo è il valore predefinito.

Off

Il server virtuale è inattivo. Ogni tentativo di accedere al server restituirà il codice errore 404 (risorsa non disponibile).

Disabled

Il server virtuale è disabilitato. Ogni tentativo di accedere al server restituirà il codice errore 403 (richiesta non accettata).

SSO

Specifica se il comportamento del sistema SSO viene ereditato dal servizio HTTP, se viene abilitato o disabilitato. Per impostazione predefinita, viene ereditato dal servizio HTTP. Se l'opzione è selezionata, il sistema SSO viene abilitato per le applicazioni Web di questo server virtuale configurate per lo stesso settore. Se l'opzione non è selezionata, il sistema SSO viene disabilitato per questo server virtuale e gli utenti devono eseguire l'autenticazione per ogni applicazione nel server virtuale.

Network Listeners

Listener di rete associati al server, se disponibili.

Default Web Module

Modulo Web distribuito (se disponibile) che deve rispondere a tutte le richieste che non possono essere mappate su altri moduli Web distribuiti nel server virtuale. Se non viene specificato un modulo Web predefinito, viene utilizzato il modulo Web con una root di contesto vuota. Se non sono disponibili moduli Web con una root di contesto vuota, viene creato e utilizzato un modulo Web predefinito per il sistema.

Log File

Nome del percorso del file in cui visualizzare i messaggi di log provenienti da questo server virtuale. Per impostazione predefinita, i messaggi di log verranno visualizzati nel file domain-dir/logs/server.log.

Docroot

Percorso completo della directory di documenti root per il server. Il valore predefinito è domain-dir/docroot.

Access Logging

Specifica se la registrazione degli accessi viene ereditata dal servizio HTTP, se viene abilitata o disabilitata. Per impostazione predefinita, viene ereditata dal servizio HTTP.

Directory

Percorso completo della directory dei log di accesso del server. Il valore predefinito è domain-dir/logs/access.

Additional Properties

Proprietà aggiuntive per il server virtuale.

Per informazioni sulle proprietà disponibili, vedere Proprietà specifiche dei server virtuali.
