Configurare il servizio HTTP

  1. Nell'albero di navigazione espandere il nodo Configurazione.

  2. Select the configuration that you are changing.

  3. Nel nodo Configurazione, selezionare il nodo Servizio HTTP.

    Viene visualizzata la pagina Servizio HTTP.

  4. Selezionare la casella di controllo SSO abilitato per abilitare il sistema SSO per impostazione predefinita per tutte le applicazioni Web in tutti i server virtuali di questo server configurate per lo stesso realm.

    È possibile sovrascrivere questa impostazione per un singolo server virtuale. Questa opzione è disabilitata per impostazione predefinita.

  5. Selezionare la casella di controllo Log degli accessi abilitato per abilitare la registrazione degli accessi per tutti i sottoelementi del server virtuale in cui non è specificata tale proprietà.

    Se la casella di controllo Log degli accessi abilitato è deselezionata, viene disabilitata la registrazione degli accessi per tutti i sottoelementi del server virtuale in cui non è specificata tale proprietà. Questa opzione è disabilitata per impostazione predefinita.

  6. Selezionare la casella di controllo Rotazione abilitata per abilitare la rotazione dei file di log.

    Questa opzione è abilitata per impostazione predefinita.

  7. Dall'elenco a discesa Criteri di rotazione, selezionare un criterio di rotazione.

    L'unico criterio di rotazione disponibile è time.

  8. Nel campo Intervallo di rotazione, digitare il numero di minuti che intercorrono tra le rotazioni del log di accesso.

    Questo campo è valido solo se il criterio di rotazione è time. Il valore predefinito è 1.440 minuti (24 ore).

  9. Nel campo Suffisso rotazione, digitare il valore di stringa da aggiungere alla fine del nome file di log in seguito alla rotazione.

    Il valore predefinito è yyyy-MM-dd.

    Verificare che il suffisso di rotazione contenga valori sufficienti per la creazione di un suffisso univoco per ogni rotazione di log. Ad esempio, se il log di accesso viene ruotato più di una volta al giorno, includere l'ora nel suffisso di rotazione.

    Il suffisso univoco consente di assicurare che venga eseguita la rotazione del file di log di accesso a ogni scadenza dell'intervallo di rotazione. Se il suffisso di rotazione del nuovo file di log di accesso è identico al suffisso di rotazione del file di log di accesso esistente, la rotazione del file non viene eseguita.

  10. Nel campo Numero massimo di file, digitare il numero massimo di file di log di accesso ruotati da salvare.

    Un valore negativo indica che il numero dei file è illimitato. Il valore predefinito è -1.

  11. Nel campo Dimensione buffer, digitare le dimensioni in byte del buffer del log di accesso.

    Un valore inferiore o pari a 0 consente di disabilitare il buffering del log di accesso. Il valore predefinito è 32768.

  12. Nel campo Intervallo di scrittura, digitare il numero di secondi al termine dei quali il log viene scritto sul disco.

    Il log di accesso viene scritto quando il buffer è pieno o allo scadere dell'intervallo. Se il valore è inferiore o pari a 0, il buffer viene sempre scritto, anche se non è pieno. Ogni volta che si accede al server, i messaggi del log vengono pertanto memorizzati direttamente nel file. Il valore predefinito è 300.

  13. Nel campo Formato, digitare il valore di stringa per specificare le seguenti proprietà del log di accesso:

    • Informazioni da catturare nel log di accesso

    • Ordine in base al quale le informazioni vengono catturate

    Il valore predefinito è il seguente:

    %client.name% %auth-user-name% %datetime% %request% %status% %response.length%

    I seguenti valori possono essere inclusi nella stringa di formato:

    %attribute.name%

    Registra il valore (mediante la chiamata al rispettivo metodo toString) dell'attributo della richiesta con il valore name specificato.

    %auth-user-name%

    Nome dell'utente autorizzato

    %client.name%

    Nome host client

    %client.dns%

    DNS client

    %cookie.name%

    Valore del cookie di richiesta con il valore name specificato

    %cookie.value%

    Valore del primo cookie trovato nella richiesta

    %datetime%

    Data di sistema

    %request%

    Riga di richiesta HTTP completa

    %status%

    Stato

    %response.length%

    Lunghezza contenuto risposta

    %header.referer%

    Intestazione Referer

    %header.user-agent%

    Intestazione User-Agent

    %http-method%

    Metodo HTTP

    %http-uri%

    URI HTTP

    %query-str%

    Stringa query HTTP

    %http-version%

    Versione protocollo HTTP

    %header.accept%

    Intestazione Accept

    %header.date%

    Intestazione Date

    %header.if-mod-since%

    Intestazione If-Modified-Since

    %header.auth%

    Intestazione Authorization

    %header.any%

    Qualsiasi valore di intestazione HTTP valido definito nella RFC 2616 (anche any è un valore di intestazione valido; qui è specificato come variabile)

    %session.name%

    Valore dell'attributo di sessione con nome name o NULL-SESSION-ATTRIBUTE-name se l'attributo denominato non esiste nella sessione o NULL-SESSION se la sessione non esiste

    %time-taken%

    Tempo (in millisecondi) necessario per eseguire ogni richiesta

    %vs.id%

    ID server virtuale

  14. Nella sezione Proprietà aggiuntive, specificare le proprietà aggiuntive.

    Per una descrizione delle proprietà disponibili, vedere Proprietà specifiche del servizio HTTP.

    • Per aggiungere una proprietà, fare clic sul pulsante Aggiungi proprietà. Nella riga vuota visualizzata, digitare il nome della proprietà nel campo Nome e il valore della proprietà nel campo Valore.

    • Per modificare una proprietà, è necessario modificare il valore del campo Valore della proprietà.

    • Per eliminare una proprietà, selezionare la casella di controllo a sinistra del campo Nome relativo alla proprietà da eliminare, quindi fare clic sul pulsante Elimina proprietà.

  15. Fare clic su Salva.

    Suggerimento:

    Per tornare alle impostazioni predefinite, fare clic sul pulsante Carica impostazioni predefinite, quindi su Salva.
