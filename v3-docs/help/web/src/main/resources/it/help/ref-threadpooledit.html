Edit Thread Pool

Nella pagina Edit Thread Pool è possibile modificare un pool di thread esistenti.

La pagina Edit Thread Pool contiene le seguenti opzioni.

Load Defaults

Pulsante che consente di ripristinare i valori predefiniti delle impostazioni che dispongono di tali valori. Le impostazioni prive di valori predefiniti restano invariate.

Configuration Name

Nome della configurazione a cui vengono applicate le impostazioni in questa pagina. Il campo è di sola lettura.

Name

Nome del pool di thread. Il campo Name è di sola lettura. È possibile specificare il nome solo quando si crea un nuovo pool di thread.

Class Name

Nome completo della classe che implementa il pool di thread. Il valore predefinito è com.sun.grizzly.http.StatsThreadPool.

Max Queue Size

Numero massimo di messaggi che è possibile accodare fino a quando non sono disponibili i thread per elaborarli. Un valore pari a -1 indica una dimensione illimitata della coda. Il valore predefinito è 4096.

Max Thread Pool Size

Numero massimo di thread nel pool di thread. Il valore predefinito è 5.

Min Thread Pool Size

Numero minimo di thread nel pool di thread. Il valore predefinito è 2.

Idle Thread Timeout

Numero massimo di secondi durante i quali un thread può rimanere inattivo nel pool. Una volta trascorso tale numero di secondi, il thread viene rimosso dal pool. Il valore predefinito è 900.
