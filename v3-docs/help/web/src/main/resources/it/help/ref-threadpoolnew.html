Nuovo pool di thread

Nella pagina Nuovo pool di thread è possibile creare un pool di thread.

La pagina Nuovo pool di thread contiene le opzioni riportate di seguito.

Nome configurazione

Nome della configurazione a cui vengono applicate le impostazioni definite in questa pagina. Questo campo è di sola lettura.

Nome

Nome del pool di thread.

Nome classe

Nome completamente qualificato della classe che implementa il pool di thread. Il valore predefinito è com.sun.grizzly.http.StatsThreadPool.

Dimensione massima coda

Numero massimo di messaggi che è possibile accodare fino a quando non sono disponibili i thread per elaborarli. Un valore pari a -1 che non sono impostati limiti alla dimensione della coda. Il valore predefinito è 4096.

Dimensione massima pool di thread

Numero massimo di thread nel pool di thread. Il valore predefinito è 5. Questo è il valore consigliato per i pool di thread dei listener di rete.

Dimensione minima pool di thread

Numero minimo di thread nel pool di thread. Il valore predefinito è 2.

Timeout thread inattivo

Quantità massima di tempo, in secondi, durante la quale un thread può rimanere inattivo nel pool. Allo scadere di questo periodo, il thread viene rimosso dal pool. Il valore predefinito è 900.
