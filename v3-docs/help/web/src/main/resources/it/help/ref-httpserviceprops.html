Proprietà specifiche del servizio HTTP

Per il servizio HTTP sono disponibili le seguenti proprietà aggiuntive. Se specificate, queste proprietà vengono applicate a tutti i Network Listeners.

ssl-cache-entriese

Numero di sessioni SSL da memorizzare nella cache. Il valore predefinito è 10.000.

ssl3-session-timeout

Intervallo durante il quale le sessioni SSL3 vengono memorizzate nella cache. Il valore predefinito è 86.400.

ssl-session-timeout

Intervallo durante il quale le sessioni SSL2 vengono memorizzate nella cache. Il valore predefinito è 100.

proxyHandler

Nome completo di un'implementazione personalizzata della classe astratta com.sun.appserv.ProxyHandler, che consente di recuperare informazioni relative alla richiesta client originale intercettata da un server proxy di terminazione SSL (ad esempio, un servizio di bilanciamento del carico) tramite un'istanza backend di GlassFish Server. Con l'implementazione di questa classe astratta, una richiesta specifica viene sottoposta ad esame allo scopo di individuare le intestazioni di richiesta personalizzate tramite cui il server proxy comunica le informazioni sulla richiesta client originale all'istanza di Enterprise Server, quindi tali informazioni vengono restituite al rispettivo chiamante.

L'implementazione predefinita legge l'indirizzo IP del client da un'intestazione di richiesta HTTP denominata Proxy-ip, il valore keysize di SSL da un'intestazione di richiesta HTTP denominata Proxy-keysize e la catena di certificati client SSL da un'intestazione di richiesta HTTP denominata Proxy-auth-cert . Il valore Proxy-auth-cert deve contenere la catena di certificati client con codifica BASE-64 senza i limiti BEGIN CERTIFICATE e END CERTIFICATE e con \n sostituito da % d% a.

Questa impostazione viene utilizzata soltanto se la casella di controllo Auth Pass Through relativa alla pagina HTTP del protocollo dei listener di rete è impostata su true.

connectionTimeout

Specifica il numero di secondi di attesa dei listener di rete per la visualizzazione della riga URI di richiesta in seguito all'accettazione di una connessione. Il valore predefinito è 30.
