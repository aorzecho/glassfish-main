Servizio HTTP

Nella pagina Servizio HTTP è possibile specificare i criteri generali del log degli accessi per le applicazioni Web.

Il servizio HTTP fornisce le funzioni per la distribuzione delle applicazioni Web e per consentire l'accesso alle applicazioni Web distribuite da parte di client HTTP, server virtuali, pool di thread e funzionalità di configurazione di rete.

GlassFish Server usa il log degli accessi al servizio HTTP per abilitare e configurare la rotazione dei log degli accessi dei server virtuali. Questi log si trovano nella directory domain-dir/logs/access e sono denominati come riportato di seguito: nome_server_virtualelog_degli_accessi.aaaaMMgg-HHhmmmsss.txt

La pagina Servizio HTTP contiene le opzioni riportate di seguito.

Carica impostazioni predefinite

Pulsante che consente di ripristinare i valori predefiniti delle impostazioni che dispongono di tali valori. Le impostazioni prive di valori predefiniti restano invariate.

Nome configurazione

Nome della configurazione a cui vengono applicate le impostazioni definite in questa pagina. Questo campo è di sola lettura.

SSO

Indica se, per impostazione predefinita, il Single Sign-On è abilitato per tutte le applicazioni Web in tutti i server virtuali di questo server configurate per lo stesso realm. Se questa opzione non è abilitata, per impostazione predefinita il Single Sign-On è disabilitato per tutti i server virtuali e gli utenti devono eseguire l'autenticazione per ogni applicazione in ciascun server virtuale. È possibile sostituire questa impostazione per un singolo server virtuale. Il Single Sign-On può essere abilitato o disabilitato. Questa opzione è disabilitata per impostazione predefinita.

Log degli accessi

Se questa opzione è selezionata, il log degli accessi è abilitato per tutti i sottoelementi di server virtuali in cui non è specificata questa proprietà. Se non è selezionata, questa opzione disabilita il log degli accessi per tutti i sottoelementi di server virtuali in cui non è specificata questa proprietà. Questa opzione è disabilitata per impostazione predefinita.

Rotazione

Se questa opzione è selezionata, la rotazione dei file di log è abilitata. Questa opzione è abilitata per impostazione predefinita.

Criteri di rotazione

L'unico criterio di rotazione disponibile è time.

Intervallo di rotazione

Numero di minuti che intercorrono tra le rotazioni del log degli accessi. Questo campo è valido solo se il criterio di rotazione è time. Il valore predefinito è 1.440 minuti (24 ore).

Suffisso rotazione

Valore di stringa da aggiungere alla fine del nome del file di log dopo la rotazione. Il valore predefinito è aaaa-MM-gg.

Assicurarsi che il suffisso di rotazione contenga valori sufficienti per la creazione di un suffisso univoco per ogni rotazione dei log. Ad esempio, se il log degli accessi viene ruotato più di una volta al giorno, includere l'ora nel suffisso di rotazione.

Un suffisso univoco è necessario per assicurare che venga eseguita la rotazione del file del log degli accessi a ogni scadenza dell'intervallo di rotazione. Se il suffisso di rotazione del nuovo file di log degli accessi è identico al suffisso di rotazione del file di log degli accessi esistente, la rotazione del file non viene eseguita.

Numero massimo di file

Numero massimo di file di log degli accessi ruotati da conservare. Un valore negativo indica che il numero di file è illimitato. Il valore predefinito è -1.

Dimensione buffer

Dimensione, in byte, del buffer del log degli accessi oppure un valore inferiore o uguale a 0 per i log degli accessi non inseriti in un buffer. Il valore predefinito è 32768.

Intervallo di scrittura

Numero di secondi prima della scrittura del log sul disco. Il log degli accessi viene scritto quando il buffer è pieno o allo scadere dell'intervallo. Se il valore è inferiore o uguale a 0, il buffer viene sempre scritto, anche se non è pieno. Ciò significa che ogni volta che si accede al server, i messaggi del log vengono memorizzati direttamente nel file. Il valore predefinito è 300.

Formato

Valore di stringa che specifica le informazioni da acquisire nel log degli accessi e l'ordine in cui vengono acquisite. Il valore predefinito è il seguente:

%client.name% %auth-user-name% %datetime% %request% %status% %response.length%

I seguenti valori possono essere inclusi nella stringa di formato:

%attribute.name%

Registra il valore (mediante la chiamata al metodo toString corrispondente) dell'attributo della richiesta con il name specificato

%auth-user-name%

Nome dell'utente autorizzato

%client.name%

Nome host del client

%client.dns%

DNS del client

%cookie.name%

Valore del cookie di richiesta con il valore nome specificato

%cookie.value%

Valore del primo cookie trovato nella richiesta

%datetime%

Data di sistema

%request%

Riga di richiesta HTTP completa

%status%

Stato

%response.length%

Lunghezza contenuto risposta

%header.referer%

Intestazione provenienza

%header.user-agent%

Intestazione User Agent

%http-method%

Metodo HTTP

%http-uri%

URI HTTP

%query-str%

Stringa query HTTP

%http-version%

Versione protocollo HTTP

%header.accept%

Intestazione accettazione

%header.date%

Intestazione data

%header.if-mod-since%

Intestazione If-Modified-Since

%header.auth%

Intestazione autorizzazione

%header.any%

Qualsiasi valore di intestazione HTTP valido definito nella specifica RFC 2616 (anche any è un valore di intestazione valido; qui è specificato come variabile)

%session.name%

Valore dell'attributo di sessione con il nome nameo NULL-SESSION-ATTRIBUTE-name se l'attributo denominato non esiste nella sessione oppure NULL-SESSION se non esiste alcuna sessione

%time-taken%

Tempo, in millisecondi, necessario per eseguire ciascuna richiesta

%vs.id%

ID server virtuale
