New Protocol

Nella pagina New Protocol è possibile creare un nuovo protocollo di rete.

Dopo aver creato un protocollo, in genere si procede alla creazione di un listener di rete con lo stesso nome e si associa il listener di rete al protocollo. In alternativa è possibile creare un listener di rete: contemporaneamente viene creato un protocollo con le impostazioni predefinite.

La pagina New Protocol contiene le seguenti opzioni.

Configuration Name

Nome della configurazione a cui vengono applicate le impostazioni in questa pagina. Il campo è di sola lettura.

Name

Nome univoco del protocollo. I nomi di protocolli non possono iniziare con un numero.

Security

Se l'opzione è selezionata, per il protocollo viene abilitata la protezione. Questa opzione è disabilitata per impostazione predefinita. Se si abilita la protezione è possibile utilizzare la scheda SSL della pagina Edit Protocol per modificare le impostazioni di protezione.

Status

Se l'opzione è selezionata viene abilitata la cache di file. Questa opzione è abilitata per impostazione predefinita.

Max Age

Durata massima in secondi per una voce della cache valida.

Questo parametro definisce il tempo di utilizzo dei dati memorizzati nella cache dopo la creazione di un file. Le voci di durata superiore a quella massima vengono sostituite con nuove voci per lo stesso file. Se il contenuto cambia sporadicamente, impostare un valore più elevato per ottimizzare le prestazioni.

La durata massima ottimale dipende dalla frequenza con cui si modificano i file esistenti. Ad esempio, se i file vengono modificati quattro volte al giorno a intervalli regolari, si consiglia di impostare una durata massima di 21.600 secondi (6 ore). È altrimenti possibile impostare una durata massima corrispondente al periodo più lungo durante il quale si desidera utilizzare la versione precedente di un file dopo che il file è stato modificato.

Il valore predefinito è 30.

Max Cache Size

Dimensioni massime totali (in byte) della cache di file sul disco. Il valore predefinito è 10485760.

Max File Count

Numero massimo di file che è possibile memorizzare nella cache di file.

Se il valore è eccessivamente alto, il server memorizza nella cache i file utilizzati meno di recente al fine di evitare sprechi di memoria. Se il valore è eccessivamente basso, l'utilizzo della cache non presenta alcun vantaggio. Provare a specificare valori differenti per l'attributo fino a quando non si ottiene la soluzione ottimale per le specifiche applicazioni.

Il valore predefinito è 1024.

Server Name

Nome host da utilizzare negli URL che il server invia al client. Se il server utilizza un alias, questo nome corrisponde al nome alias. Se il server non utilizza un alias, lasciare vuoto questo campo.

Questo valore ha effetto sugli URL generati automaticamente dal server ma non sugli URL relativi a directory e file memorizzati nel server. Se il server utilizza un alias, il nome del server deve corrispondere al nome alias. Se vengono aggiunti i due punti e il numero della porta, tale porta viene utilizzata negli URL che il server invia al client.

Default Virtual Server

Server virtuale da associare a questo protocollo. Nella pagina Virtual Servers è possibile definire server virtuali.

Redirect Port

Valore di porta che reindirizza una richiesta a un'altra porta. La richiesta viene reindirizzata automaticamente se si verificano le due condizioni seguenti:

  • Il listener di rete che fa riferimento a questo protocollo supporta richieste non SSL.

  • Viene ricevuta una richiesta per la quale un vincolo di protezione corrispondente richiede trasporto SSL.

Se non si specifica una porta di reindirizzamento, viene utilizzato il numero di porta specificato nella richiesta originale. Per impostazione predefinita non è specificata alcuna porta.

Max Connections

Specifica il numero massimo di richieste che è possibile trasmettere fino a quando la connessione non viene chiusa dal server. Il sottosistema Keep-Alive esegue regolarmente il polling delle connessioni inattive. Il valore predefinito è 256.

Impostare questa priorità su 1 per disabilitare le attività di keep-alive HTTP/1.0, keep-alive HTTP/1.1 e pipelining. Un valore pari a 0 indica che le richieste verranno sempre rifiutate. Un valore pari a -1 indica un numero illimitato di connessioni keep-alive.

Timeout

Numero massimo di secondi durante i quali una connessione può essere considerata inattiva e mantenuta nello stato keep-alive. Un valore inferiore o pari a 0 indica che le connessioni keep-alive sono mantenute aperte a tempo indeterminato. Il valore predefinito è 30 secondi.

Upload Timeout

Se l'opzione è selezionata, la connessione per un servlet che legge byte lentamente viene chiusa una volta raggiunto il limite Connection Upload Timeout. Se l'opzione non è selezionata, non si verifica alcun timeout per le connessioni del servlet. Questa opzione è disabilitata per impostazione predefinita.

Connection Upload Timeout

Timeout per i caricamenti, in millisecondi. Questo campo è valido solo se è selezionata la casella Upload Timeout Enabled. Il valore predefinito è 300000 millisecondi.

Request Timeout

Numero di secondi precedenti al timeout di una richiesta. Se la richiesta non viene elaborata prima che sia raggiunto il valore di timeout, la richiesta viene ignorata. Il valore predefinito è 30 secondi.

Send Buffer Size

Dimensioni in byte del buffer di invio. Il valore predefinito è 8192 byte.

Header Buffer Length

Dimensioni in byte del buffer utilizzato dai thread di elaborazione della richiesta per leggere i dati della richiesta. Il valore predefinito è 8192 byte.

Max Post Size

Dimensioni massime in byte di azioni POST. Il valore predefinito è 2097152 byte.

URI Encoding

Nome del set di caratteri utilizzato per decodificare gli URI della richiesta ricevuti. Il valore deve essere un nome del set di caratteri IANA valido. Il valore predefinito è UTF-8.

Version

Versione del protocollo HTTP utilizzato. Il valore predefinito è HTTP/1.1.

Compression

Specifica la compressione HTTP/1.1 GZIP da utilizzare per salvare la larghezza di banda del server. Sono disponibili le seguenti opzioni:

on

Comprime i dati.

off

Disabilita la compressione.

force

Forza la compressione dei dati in tutti i casi.

Il valore predefinito è off.

Compressible Mime Types

Elenco separato da virgole di tipi MIME per cui viene utilizzata la compressione HTTP. Il valore predefinito è text/html,text/xml,text/plain.

Compression Minimum Size

Dimensioni minime di un file quando viene applicata la compressione. È necessario specificare questo valore se la compressione è impostata su on o force. Il valore predefinito è 2048 byte.

No-Compression User Agents

Elenco separato da virgole di espressioni regolari corrispondenti ad agenti utente di client HTTP per cui non si desidera utilizzare la compressione. Per impostazione predefinita questo valore è una stringa vuota.

Restricted User Agent

Elenco di agenti utenti con restrizioni a cui viene applicata la compressione HTTP. Se non vengono specificati agenti utente, la compressione HTTP viene applicata a tutti gli agenti utente. Per impostazione predefinita non vengono specificati agenti utente.

Default Response Type

Stringa che specifica il tipo di risposta predefinita. Il formato è una stringa delimitata da punti e virgola composta da tipo di contenuto, codifica, lingua e insieme di caratteri. Il valore predefinito è text/html; charset=iso-8859-1.

Forced Response Type

Stringa che specifica il tipo di richiesta utilizzato se non è disponibile un mapping MIME con la stessa estensione del file. Il formato è una stringa delimitata da punti e virgola composta da tipo di contenuto, codifica, lingua e insieme di caratteri. Il valore predefinito è text/html; charset=iso-8859-1.

Adapter

Nome della classe dell'adattatore di risorse statiche. Il valore predefinito è com.sun.grizzly.tcp.StaticResourcesAdapter.

Comet Support

Se l'opzione è selezionata, il supporto Comet viene abilitato per il listener di rete che fa riferimento a questo protocollo. Questa opzione è disabilitata per impostazione predefinita. Se il servlet o la pagina JSP utilizza la tecnologia Comet, verificare che ne venga eseguita l'inizializzazione durante l'avvio di GlassFish Server mediante l'aggiunta dell'elemento load-on-startup al file web.xml. Ad esempio:

<servlet>
<servlet-name>CheckIn</servlet-name>
<servlet-class>CheckInServlet</servlet-class>
<load-on-startup>0</load-on-startup>
</servlet>
DNS Lookup

Se l'opzione è selezionata viene abilitata la ricerca DNS. Questa opzione è disabilitata per impostazione predefinita.

DNS Lookup

Se l'opzione è selezionata viene abilitata la ricerca DNS. Questa opzione è disabilitata per impostazione predefinita.

RCM Support

Se l'opzione è selezionata viene abilitato RCM (Resource Configuration Management). Questa opzione è disabilitata per impostazione predefinita.

Trace

Se l'opzione è selezionata viene abilitata l'operazione TRACE. Se l'opzione è disabilitata, GlassFish Server è meno soggetto ad attacchi con script da altri siti. Questa opzione è abilitata per impostazione predefinita.

Auth Pass Through

Se l'opzione è selezionata, indica che il listener di rete che utilizza questo protocollo riceve il traffico da un server proxy con terminazione SSL. Questa opzione è disabilitata per impostazione predefinita.

Chunking

Se l'opzione è selezionata viene abilitata la suddivisione in blocchi della risposta HTTP. Questa opzione è abilitata per impostazione predefinita.

XPowered By

Se l'opzione è selezionata, GlassFish Server utilizza le intestazioni X-Powered-By conformemente alle specifiche Java Servlet 3.0 e Java Server Pages 2.0. Questa opzione è abilitata per impostazione predefinita.
