Nuovo protocollo

Nella pagina Nuovo protocollo è possibile creare un nuovo protocollo di rete.

Dopo aver creato un protocollo, in genere si procede alla creazione di un listener di rete con lo stesso nome e si associa il listener di rete al protocollo. In alternativa, è possibile creare un listener di rete e contemporaneamente fare in modo che venga creato in modo automatico un protocollo con le impostazioni predefinite.

La pagina Nuovo protocollo contiene le opzioni riportate di seguito.

Nome configurazione

Nome della configurazione a cui vengono applicate le impostazioni definite in questa pagina. Questo campo è di sola lettura.

Nome

Nome univoco del protocollo. Un nome di protocollo non può iniziare con un numero.

Sicurezza

Se questa opzione è selezionata, la sicurezza viene abilitata per il protocollo. Questa opzione è disabilitata per impostazione predefinita. Se si abilita la sicurezza, è possibile utilizzare la scheda SSL della pagina Modifica protocollo per modificare le impostazioni di sicurezza.

Stato

Se questa opzione è selezionata, viene abilitata la cache di file. Questa opzione è abilitata per impostazione predefinita.

Durata massima

Durata massima, in secondi, per una voce della cache valida.

Questo parametro controlla il tempo di utilizzo delle informazioni inserite nella cache dopo l'inserimento di un file nella cache. Le voci di durata superiore a quella massima vengono sostituite con nuove voci per lo stesso file. Se il contenuto cambia sporadicamente, impostare un valore più elevato per migliorare le prestazioni.

La durata massima ottimale dipende dalla frequenza con cui si modificano i file esistenti. Ad esempio, se i file vengono modificati quattro volte al giorno a intervalli regolari, si consiglia di impostare una durata massima di 21.600 secondi (6 ore). È altrimenti possibile impostare una durata massima corrispondente al periodo più lungo durante il quale si desidera utilizzare la versione precedente di un file dopo che il file è stato modificato.

Il valore predefinito è 30.

Dimensione massima cache

Dimensione massima totale (in byte) della cache di file sul disco. Il valore predefinito è 10485760.

Numero massimo di file

Numero massimo di file che è possibile memorizzare nella cache di file.

Se il valore è eccessivamente alto, il server inserisce nella cache i file utilizzati meno di recente al fine di evitare sprechi di memoria. Se il valore è eccessivamente basso, l'inserimento nella cache non presenta alcun vantaggio. Provare a specificare valori differenti per l'attributo fino a quando non si ottiene la soluzione ottimale per le specifiche applicazioni.

Il valore predefinito è 1024.

Nome server

Nome host da utilizzare negli URL che il server invia al client. Se il server utilizza un alias, questo nome corrisponde al nome alias. Se il server non utilizza un alias, lasciare vuoto questo campo.

Questo valore ha effetto sugli URL generati automaticamente dal server, ma non sugli URL relativi a directory e file memorizzati nel server. Se il server utilizza un alias, il nome del server deve corrispondere al nome alias. Se vengono aggiunti due punti e il numero di porta, tale porta viene utilizzata negli URL che il server invia al client.

Server virtuale predefinito

Server virtuale da associare a questo protocollo. Nella pagina Server virtuali è possibile definire server virtuali.

Porta di reindirizzamento

Valore della porta che reindirizza una richiesta a un'altra porta. GlassFish Server reindirizza automaticamente la richiesta se si verificano le due condizioni riportate di seguito.

  • Il listener di rete che fa riferimento a questo protocollo supporta richieste non SSL.

  • Viene ricevuta una richiesta per la quale un vincolo di sicurezza corrispondente richiede il trasporto SSL.

Se non si specifica una porta di reindirizzamento, GlassFish Server utilizza il numero di porta specificato nella richiesta originale. Per impostazione predefinita, non è specificata alcuna porta.

Numero massimo di connessioni

Specifica il numero massimo di richieste che è possibile trasmettere fino a quando la connessione non viene chiusa dal server. Il sottosistema di controllo attività esegue periodicamente il polling delle connessioni inattive. Il valore predefinito è 256.

Impostare questa proprietà su 1 per disabilitare il controllo attività HTTP/1.0, nonché il pipeline e il controllo attività HTTP/1.1. Un valore pari a 0 indica che le richieste verranno sempre rifiutate. Un valore pari a -1 imposta un numero illimitato di connessioni di controllo attività.

Timeout

Numero massimo di secondi durante i quali una connessione può essere considerata inattiva e mantenuta nello stato controllo attività. Un valore inferiore o pari a 0 indica che le connessioni di controllo attività sono mantenute aperte a tempo indeterminato. Il valore predefinito è 30 secondi.

Timeout di caricamento

Se questa opzione è selezionata, la connessione per un servlet che legge byte lentamente viene chiusa dopo il raggiungimento del timeout di caricamento della connessione. Se questa opzione è disabilitata, non si verifica il timeout per le connessioni del servlet. Questa opzione è disabilitata per impostazione predefinita.

Timeout di caricamento connessione

Timeout relativo ai caricamenti, espresso in millisecondi. Questo campo è valido solo se la casella di controllo Timeout di caricamento abilitato è selezionata. Il valore predefinito è 300000 millisecondi.

Timeout richieste

Numero di secondi prima del timeout di una richiesta. Se la richiesta non viene elaborata prima del raggiungimento del valore di timeout, la richiesta viene ignorata. Il valore predefinito è 30 secondi.

Dimensione buffer di invio

Dimensione in byte del buffer di invio. Il valore predefinito è 8192 byte.

Lunghezza buffer intestazione

Dimensione in byte del buffer utilizzato dai thread di elaborazione delle richieste per la lettura dei dati delle richieste. Il valore predefinito è 8192 byte.

Dimensione massima POST

Dimensione massima in byte delle azioni POST. Il valore predefinito è 2097152 byte.

Codifica URI

Nome del set di caratteri utilizzato per decodificare gli URI delle richieste ricevuti. Il valore deve essere un nome di set di caratteri IANA valido. Il valore predefinito è UTF-8.

Versione

Versione del protocollo HTTP in uso. Il valore predefinito è HTTP/1.1.

Compressione

Specifica l'uso della compressione GZIP HTTP/ 1.1 per risparmiare l'ampiezza di banda del server. Sono disponibili le opzioni riportate di seguito.

on

Comprime i dati.

off

Disabilita la compressione.

force

Forza la compressione dei dati in tutti i casi.

Il valore predefinito è off.

Tipi MIME comprimibili

Lista di valori separati da virgole di tipi MIME per i quali si utilizza la compressione HTTP. Il valore predefinito è text/html,text/xml,text/plain.

Dimensione minima di compressione

Dimensione minima di un file quando vi si applica la compressione. È necessario impostare questo valore se la compressione è impostata su on o su force . Il valore predefinito è 2048 byte.

User Agent senza compressione

Lista di valori separati da virgole di espressioni regolari corrispondenti agli User Agent dei client HTTP per i quali non è consigliabile usare la compressione. Per impostazione predefinita, questo valore è una stringa vuota.

User Agent con limitazioni

Lista di User Agent con limitazioni su cui è applicata la compressione HTTP. Se non vengono specificati User Agent, la compressione HTTP viene applicata a tutti gli User Agent. Per impostazione predefinita, non vengono specificati User Agent.

Tipo di risposta predefinito

Stringa che specifica il tipo di risposta predefinito. Il formato è una stringa delimitata da punto e virgola e costituita da content-type, codifica, lingua e set di caratteri. Il valore predefinito è text/html; charset=iso-8859-1.

Tipo di risposta forzata

Stringa che specifica il tipo di richiesta utilizzato se non è disponibile un mapping MIME con la stessa estensione del file. Il formato è una stringa delimitata da punto e virgola e costituita da content-type, codifica, lingua e set di caratteri. Il valore predefinito è text/html; charset=iso-8859-1.

Adattatore

Nome classe del Resource Adapter statico. Il valore predefinito è com.sun.grizzly.tcp.StaticResourcesAdapter.

Supporto Comet

Se questa opzione è selezionata, il supporto Comet viene abilitato per il listener di rete che fa riferimento a questo protocollo. Questa opzione è disabilitata per impostazione predefinita. Se il servlet o la pagina JSP utilizza la tecnologia Comet, assicurarsi che ne venga eseguita l'inizializzazione durante l'avvio di GlassFish Server mediante l'aggiunta dell'elemento load-on-startup al file web.xml. Ad esempio:

<servlet>
<servlet-name>CheckIn</servlet-name>
<servlet-class>CheckInServlet</servlet-class>
<load-on-startup>0</load-on-startup>
</servlet>
Ricerca DNS

Se questa opzione è selezionata, viene abilitata la ricerca DNS. Questa opzione è disabilitata per impostazione predefinita.

Ricerca DNS

Se questa opzione è selezionata, viene abilitata la ricerca DNS. Questa opzione è disabilitata per impostazione predefinita.

Supporto RCM

Se questa opzione è selezionata, viene abilitato il supporto RCM (Resource Configuration Management). Questa opzione è disabilitata per impostazione predefinita.

Trace

Se questa opzione è selezionata, viene abilitata l'operazione TRACE. Se questa opzione è disabilitata, GlassFish Server è meno soggetto ad attacchi di tipo XSS (Cross-Site Scripting). Questa opzione è abilitata per impostazione predefinita.

Autenticazione pass-through

Se questa opzione è selezionata, indica che il listener di rete che utilizza questo protocollo riceve il traffico da un server proxy con terminazione SSL. Questa opzione è disabilitata per impostazione predefinita.

Creazione chunk

Se questa opzione è selezionata, viene abilitata la creazione di chunk di risposta HTTP. Questa opzione è abilitata per impostazione predefinita.

XPowered By

Se questa opzione è selezionata, vengono utilizzate le intestazioni X-Powered-By in base alle specifiche Java Servlet 3.0 e Java Server Pages 2.0. Questa opzione è abilitata per impostazione predefinita.

Barra codificata

Se questa opzione è selezionata, gli URL possono contenere barre codificate. Questa opzione è disabilitata per impostazione predefinita.

Supporto socket Web

Se questa opzione è selezionata, il protocollo WebSockets è supportato. Questa opzione è disabilitata per impostazione predefinita.
