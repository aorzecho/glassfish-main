Modificare un server virtuale

  1. Nella struttura ad albero, espandere il nodo Configuration.
  2. Nel nodo Configuration, selezionare il nodo Virtual Servers.

    Viene visualizzata la pagina Virtual Servers.

  3. Nella pagina Virtual Servers, fare clic sul nome del server virtuale da modificare.

    Viene visualizzata la pagina Edit Virtual Server.

  4. Nel campo Hosts, immettere i nomi host o gli indirizzi IP per il computer in cui è in esecuzione il server.

    Utilizzare i nomi reali o i nomi virtuali registrati con il server DNS per la rete (e in un sistema UNIX nel file /etc/hosts). Il valore predefinito è il valore della proprietà di sistema ${com.sun.aas.hostName}.

  5. Selezionare un'opzione State per il server virtuale.

    Il valore può essere uno qualsiasi dei seguenti:

    On

    Il server virtuale è attivo. Questo è il valore predefinito.

    Off

    Il server virtuale è inattivo. Ogni tentativo di accedere al server restituirà il codice errore 404 (risorsa non disponibile).

    Disabled

    Il server virtuale è disabilitato. Ogni tentativo di accedere al server restituirà il codice errore 403 (richiesta non accettata).

  6. Selezionare un'opzione SSO per il nuovo server virtuale.

    Per impostazione predefinita, un server virtuale utilizza le impostazioni SSO del servizio HTTP. Selezionare Enabled o Disabled per sovrascrivere l'impostazione del servizio HTTP attivando o disattivando l'autorizzazione SSO per questo server virtuale.

    L'autorizzazione SSO nelle applicazioni di GlassFish Server è supportata da servlet e file JSP. Questa funzionalità consente la condivisione di informazioni SSO utente tra più applicazioni che le richiedono, anziché l'accesso separato dei singoli utenti per ogni applicazione.

  7. Dall'elenco a discesa Network Listeners, selezionare uno o più listener di rete o lasciare il campo vuoto.
  8. Dall'elenco a discesa Default Web Module, selezionare il modulo da utilizzare per rispondere a tutte le richieste di cui non è possibile eseguire il mapping agli altri moduli Web distribuiti nel server virtuale.

    Se non viene specificato un modulo Web predefinito, viene utilizzato il modulo Web con una root di contesto vuota. Se non sono disponibili moduli Web con una root di contesto vuota, viene creato e utilizzato un modulo Web predefinito per il sistema.

  9. Nel campo Log File, digitare il nome del percorso del file in cui visualizzare i messaggi di log provenienti da questo server virtuale.

    Per impostazione predefinita i messaggi di log vengono inviati al log del server predefinito domain-dir/logs/server.log.

  10. Nel campo Docroot, immettere il percorso completo della directory di documenti root per il server.

    Il valore predefinito è domain-dir/docroot.

  11. Selezionare un'opzione Access Logging per abilitare o disabilitare la registrazione degli accessi.

    Per impostazione predefinita, un server virtuale eredita le impostazioni di registrazione degli accessi del servizio HTTP. Selezionare Enabled o Disabled per sovrascrivere l'impostazione del servizio HTTP attivando o disattivando la registrazione degli accessi per questo server virtuale.

  12. Nel campo Directory, immettere il percorso completo della directory dei log di accesso del server.

    Il valore predefinito è domain-dir/logs/access.

  13. (Opzionale) Nella sezione Additional Properties, specificare le proprietà aggiuntive per il server virtuale.

    Per informazioni sulle proprietà disponibili, vedere Proprietà specifiche dei server virtuali.

    • Per aggiungere una proprietà, fare clic sul pulsante Add Property. Nella riga vuota visualizzata, digitare il nome della proprietà nel campo Name e il valore della proprietà nel campo Value.
    • Per modificare una proprietà, è necessario cambiare il valore del campo Value della proprietà.
    • Per eliminare una proprietà, selezionare la casella di controllo a sinistra del campo Name relativo alla proprietà da eliminare, quindi fare clic sul pulsante Delete Properties.
  14. Fare clic su Save.

    Suggerimento - Per tornare alle impostazioni predefinite, fare clic sul pulsante Load Defaults, quindi su Save.
