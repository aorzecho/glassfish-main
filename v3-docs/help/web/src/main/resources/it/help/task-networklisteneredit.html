Modificare un listener di rete

  1. Nella struttura ad albero, espandere il nodo Configuration.
  2. Nel nodo Configuration, espandere il nodo Network Config.
  3. Nel nodo Network Config, selezionare il nodo Network Listeners.

    Viene visualizzata la pagina Network Listeners.

  4. Nella pagina Network Listeners, fare clic sul nome del listener di rete da modificare.

    Viene visualizzata la pagina Edit Network Listener.

  5. Selezionare o deselezionare la casella di controllo Status Enabled per modificare lo stato del listener di rete.

    Il listener può essere abilitato o disabilitato. Questa opzione è abilitata per impostazione predefinita. Se il listener è disabilitato, ogni tentativo di connessione al listener genera un'eccezione nel socket (java.net.ConnectException).

  6. Selezionare la casella di controllo Security Enabled per abilitare la protezione per il protocollo utilizzato dal listener di rete.

    Questa opzione è disabilitata per impostazione predefinita.

  7. Selezionare la casella di controllo JK Listener Enabled per impostare nel listener il supporto del protocollo mod-jk di Apache.

    Il protocollo mod-jk è un protocollo di comunicazione/rete proprietario tra httpd (server Web) di Apache e un'istanza backend di GlassFish Server. Se il listener supporta il protocollo, tutto il contenuto statico dovrebbe essere gestito mediante httpd, mentre le richieste di risorse dinamiche (ad esempio servlet e pagine JSP) vengono reindirizzate al listener di rete abilitato mod-jk di GlassFish Server tramite il protocollo mod-jk. Questa opzione è disabilitata per impostazione predefinita.

  8. Nel campo Port, digitare il numero della porta di attesa desiderato per il listener di rete.

    Sono ammessi i valori compresi tra 1 e 65535. In un sistema UNIX, per creare socket in attesa su porte comprese tra 1 e 1024 sono necessari privilegi di superutente.

  9. Nel campo Address, digitare l'indirizzo IP di attesa desiderato per il listener di rete.

    L'indirizzo può essere specificato utilizzando la notazione per indirizzi IPv6 o con separazione mediante coppie di punti. È possibile specificare any (per INADDR_ANY) per impostare tutti gli indirizzi IP per l'attesa. Può essere costituito da un nome host.

  10. Dall'elenco a discesa Thread Pool, selezionare il pool di thread da associare al listener di rete.
  11. Fare clic su Save.

    Suggerimento - Per tornare alle impostazioni predefinite, fare clic sul pulsante Load Defaults, quindi su Save.

Procedura successiva

È possibile utilizzare le schede SSL, HTTP e File Cache per modificare le impostazioni del protocollo associato al listener di rete. La scheda SSL risulta fondamentale solo se per il listener di rete è stata abilitata la protezione.
