Modificare un trasporto

  1. Nell'albero di navigazione espandere il nodo Configurazione.

  2. Nel nodo Configurazione, espandere il nodo Configurazione di rete.

  3. Nel nodo Configurazione di rete, selezionare il nodo Trasporti.

    Viene visualizzata la pagina Trasporti.

  4. Nella pagina Trasporti, fare clic sul nome del trasporto da modificare.

    Viene visualizzata la pagina Modifica trasporto.

  5. Nel campo Nome classe, digitare il nome della classe dell'implementazione del trasporto.

    Il valore predefinito è com.sun.grizzly.TCPSelectorHandler.

  6. Nel campo Handler chiavi di selezione, digitare il nome della classe del gestore dei tasti di selezione.

    Nome della classe del gestore dei tasti di selezione. Per gestore di tasti di selezione si intende una classe astratta che gestisce il ciclo di vita dei tasti di selezione. Se non si specifica un valore viene utilizzata un'implementazione predefinita.

  7. Dall'elenco a discesa Tipo di buffer di byte, selezionare il tipo di buffer di byte da utilizzare.

    Le opzioni disponibili sono HEAP e DIRECT. Il valore predefinito è HEAP.

  8. Nel campo Thread processo di accettazione, digitare il numero di processori presenti nel computer.

    Un valore pari a -1 indica che il calcolo dei thread di processo di accettazione viene eseguito tramite Grizzly, sulla base del numero di processori presenti nel computer. Il valore predefinito è 1.

    Per impostare il numero dei thread di elaborazione della richiesta, specificare il valore Dimensione massima pool di thread del pool di thread utilizzato da un listener di rete che utilizza questo trasporto.

  9. Nel campo Numero massimo connessioni, digitare il numero massimo di connessioni in sospeso su un listener di rete che utilizza questo trasporto.

    Il valore predefinito è 4096.

  10. Nel campo Dimensione buffer, digitare le dimensioni, in byte, del buffer da fornire per i flussi di input creati da un listener di rete che utilizza questo trasporto.

    Il valore predefinito è 8192.

  11. Nel campo Timeout chiave inattiva, digitare il numero di secondi al termine dei quali un tasto inattivo viene annullato e il canale viene chiuso.

    Il valore predefinito è 30.

  12. Nel campo Timeout di lettura, digitare il numero di millisecondi di attesa di GlassFish Server durante la fase di analisi di intestazione e corpo del testo di un'operazione di lettura.

    Il valore predefinito è 30.000.

  13. Nel campo Timeout di polling del selettore, digitare il numero di millisecondi al termine dei quali viene bloccata l'attesa di eventi (richieste utenti) mediante un selettore NIO.

    Il valore predefinito è 1000.

  14. Nel campo Timeout di scrittura, digitare il numero di millisecondi di attesa di GlassFish Server prima che venga considerata terminata la connessione del client remoto durante la scrittura della risposta.

    Il valore predefinito è 30.000.

  15. Selezionare la casella di controllo Visualizza configurazione abilitato per scaricare la configurazione interna di Grizzly nei log del server.

    Questa opzione potrebbe fornire informazioni utili per il debugging. Questa opzione è disabilitata per impostazione predefinita.

  16. Selezionare la casella di controllo Snooping abilitato per eseguire il dump delle informazioni di richiesta/risposta nel log del server.

    Questa opzione potrebbe fornire informazioni utili per il debugging, tuttavia causa un peggioramento significativo delle prestazioni. Questa opzione è disabilitata per impostazione predefinita.

  17. Selezionare la casella di controllo TCP senza ritardo per abilitare TCP_NODELAY (noto anche come algoritmo di Nagle).

    Questa opzione è abilitata per impostazione predefinita.

  18. Fare clic su Salva.

    Suggerimento:

    Per tornare alle impostazioni predefinite, fare clic sul pulsante Carica impostazioni predefinite, quindi su Salva.
