Modifica server virtuale

Nella pagina Modifica server virtuale è possibile modificare le impostazioni di un server virtuale.

La pagina Modifica server virtuale contiene le opzioni riportate di seguito.

Carica impostazioni predefinite

Pulsante che consente di ripristinare i valori predefiniti delle impostazioni che dispongono di tali valori. Le impostazioni prive di valori predefiniti restano invariate.

Nome configurazione

Nome della configurazione a cui vengono applicate le impostazioni definite in questa pagina. Questo campo è di sola lettura.

ID

Identificativo del server virtuale visibile internamente. Non viene visualizzato ai client HTTP. I nomi host visibili ai client HTTP devono essere specificati nel campo Host.

Host

Nome o nomi host per il computer su cui è in esecuzione il server. Utilizzare nomi effettivi o virtuali registrati con il server DNS per la rete e, in un sistema UNIX, nel file /etc/hosts. Il valore predefinito è il valore della proprietà di sistema ${com.sun.aas.hostName}.

Stato

Stato desiderato per il server. Il valore può essere uno dei seguenti:

On

Server virtuale attivo. Questo è il valore predefinito.

Off

Server virtuale non attivo. I tentativi di accesso al server restituiranno il codice errore 404 (risorsa non disponibile).

Disabled

Server virtuale disabilitato. I tentativi di accesso al server restituiranno il codice di errore 403 (elaborazione richiesta rifiutata).

SSO

Specifica se il funzionamento del Single Sign-On viene ereditato dal servizio HTTP, se è abilitato o disabilitato. Per impostazione predefinita, viene ereditato dal servizio HTTP. Se questa opzione è selezionata, il Single Sign-On è abilitato per le applicazioni Web di questo server virtuale configurate per lo stesso realm. Se questa opzione non è selezionata, il Single Sign-On è disabilitato per questo server virtuale e gli utenti devono eseguire l'autenticazione per ogni applicazione nel server virtuale.

Listener di rete

Listener o listener di rete associati a questo server.

Modulo Web predefinito

Modulo Web distribuito (se disponibile) che deve rispondere a tutte le richieste che non possono essere mappate ad altri moduli Web distribuiti sul server virtuale. Se non viene specificato un modulo Web predefinito, viene utilizzato il modulo Web con una radice contesto vuota. Se non sono disponibili moduli Web con una radice contesto vuota, viene creato e utilizzato un modulo Web predefinito per il sistema.

File di log

Nome del percorso del file in cui verranno visualizzati i messaggi di log provenienti da questo server virtuale. Per impostazione predefinita, i messaggi di log verranno visualizzati nel file domain-dir/logs/server.log..

Docroot

Percorso assoluto della directory di documenti radice per il server. Il valore predefinito è domain-dir/docroot.

Log degli accessi

Specifica se il log degli accessi viene ereditato dal servizio HTTP, se è abilitato o disabilitato. Per impostazione predefinita, viene ereditato dal servizio HTTP.

Directory

Percorso assoluto della directory dei log degli accessi del server. Il valore predefinito è domain-dir/logs/access.

Proprietà aggiuntive

Proprietà aggiuntive per il server virtuale.

Per informazioni sulle proprietà disponibili, vedere Proprietà specifiche dei server virtuali.
