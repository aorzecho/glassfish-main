Modificare un listener HTTP

  1. Nell'albero di navigazione espandere il nodo Configurazione.

  2. Nel nodo Configurazione, espandere il nodo Servizio HTTP.

  3. Nel nodo Configurazione di rete, selezionare il nodo Listener HTTP.

    Viene visualizzata la pagina Listener HTTP.

  4. Nella pagina Listener HTTP, fare clic sul nome del listener da modificare.

    Viene visualizzata la pagina Modifica listener HTTP.

  5. Nel campo Porta digitare il numero della porta di ascolto desiderato per il listener HTTP.

    Sono consentiti valori compresi tra 1 e 65535. In un sistema UNIX, per creare socket in ascolto sulle porte comprese tra 1 e 1024 sono necessari i privilegi di utente privilegiato.

  6. Selezionare o deselezionare la casella di controllo Stato abilitato per modificare lo stato del listener HTTP.

    Il listener può essere abilitato o disabilitato. Questa opzione è abilitata per impostazione predefinita. Se il listener è disabilitato, ogni tentativo di connessione al listener genera un'eccezione nel socket (java.net.ConnectException).

  7. Selezionare la casella di controllo Sicurezza abilitata per abilitare la sicurezza per il listener HTTP.

    Questa opzione è disabilitata per impostazione predefinita.

  8. Selezionare la casella di controllo Listener JK abilitato per impostare nel listener il supporto del protocollo mod-jk di Apache.

    Il protocollo mod-jk è un protocollo di comunicazione/rete proprietario tra httpd (server Web) di Apache e un'istanza backend di &ProductName. Se il listener supporta il protocollo, tutto il contenuto statico dovrebbe essere gestito mediante httpd, mentre le richieste di risorse dinamiche (ad esempio servlet e pagine JSP) vengono reindirizzate al listener HTTP abilitato mod-jk di GlassFish Server tramite il protocollo mod-jk. Questa opzione è disabilitata per impostazione predefinita.

  9. Nel campo Indirizzo, digitare l'indirizzo IP di ascolto desiderato per il listener HTTP.

    L'indirizzo può essere specificato utilizzando la notazione per indirizzi IPv6 o dotted-pair. È possibile specificare any (per INADDR_ANY) per impostare tutti gli indirizzi IP per l'ascolto. Può essere costituito da un nome host.

  10. Dall'elenco a discesa Server virtuale predefinito, selezionare il server virtuale da associare a questo listener HTTP.

    Nella pagina Server virtuali è possibile definire server virtuali.

  11. Dall'elenco a discesa Pool di thread, selezionare il pool di thread da associare al listener HTTP.

    In genere si seleziona uno dei due pool di thread configurati durante l'installazione di GlassFish Server.

  12. Nel campo Nome server, digitare il nome host da utilizzare negli URL che il server invia al client.

    Se il server utilizza un alias, questo nome corrisponde al nome alias. Se il server non utilizza un alias, lasciare vuoto questo campo.

    Questo valore ha effetto sugli URL generati automaticamente dal server ma non sugli URL relativi a directory e file memorizzati nel server. Se il server utilizza un alias, il nome del server deve corrispondere al nome alias. Se vengono aggiunti i due punti e il numero della porta, tale porta viene utilizzata negli URL che il server invia al client.

  13. Fare clic su Salva.

    Suggerimento:

    Per tornare alle impostazioni predefinite, fare clic sul pulsante Carica impostazioni predefinite, quindi su Salva.
Passi successivi

Se per il listener HTTP è stata abilitata la sicurezza è possibile utilizzare la scheda SSL per modificare le impostazioni SSL del listener HTTP.

Un listener HTTP è un tipo di listener di rete. Per accedere alle funzioni avanzate, è possibile modificare un listener HTTP utilizzando le pagine Listener di rete.
