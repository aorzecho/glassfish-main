Nuovo trasporto

Nella pagina Nuovo trasporto è possibile creare un nuovo trasporto.

In genere, non è necessario utilizzare altri trasporti in aggiunta al trasporto tcp predefinito.

La pagina Nuovo trasporto contiene le opzioni riportate di seguito.

Nome configurazione

Nome della configurazione a cui vengono applicate le impostazioni definite in questa pagina. Questo campo è di sola lettura.

Nome trasporto

Nome univoco per il trasporto.

Nome classe

Nome della classe di implementazione del trasporto. Il valore predefinito è com.sun.grizzly.TCPSelectorHandler.

Handler chiavi di selezione

Nome della classe dell'handler delle chiavi di selezione. Un handler delle chiavi di selezione è una classe astratta che gestisce il ciclo di vita delle chiavi di selezione. Se non si specifica un valore, viene utilizzata un'implementazione predefinita.

Tipo di buffer di byte

Tipo di ByteBuffer da utilizzare. Le opzioni disponibili sono HEAP e DIRECT. Il valore predefinito è HEAP.

Thread processo di accettazione

Numero di processori nel computer. Un valore pari a -1 indica che Grizzly calcolerà direttamente il numero di thread del processo di accettazione in base al numero di processori nel computer. Il valore predefinito è 1.

Per impostare il numero dei thread di elaborazione delle richieste, impostare il valore di Dimensione massima pool di thread del pool di thread utilizzato da un listener di rete che utilizza questo trasporto.

Numero massimo connessioni

Numero massimo di connessioni in sospeso in un listener di rete che utilizza questo trasporto. Il valore predefinito è 4096.

Dimensione buffer

Dimensione del buffer, in byte, da fornire per i flussi di input creati da un listener di rete che utilizza questo trasporto. Il valore predefinito è 8192.

Timeout chiave inattiva

Numero di secondi allo scadere dei quali una chiave inattiva viene annullata e il canale chiuso. Il valore predefinito è 30.

Timeout di lettura

Numero di millisecondi di attesa di GlassFish Server durante la fase di analisi dell'intestazione e del corpo di un'operazione di lettura. Il valore predefinito è 30.000.

Timeout di polling del selettore

Numero di millisecondi durante i quali viene bloccato un selettore NIO in attesa di eventi (richieste di utenti). Il valore predefinito è 1000.

Timeout di scrittura

Numero di millisecondi di attesa di GlassFish Server prima che venga considerata terminata la connessione del client remoto durante la scrittura della risposta. Il valore predefinito è 30.000.

Visualizza configurazione

Se questa opzione è selezionata, viene eseguito il flush della configurazione interna di Grizzly nei log del server. Questa opzione potrebbe fornire informazioni utili per il debug. Questa opzione è disabilitata per impostazione predefinita.

Snooping

Se questa opzione è selezionata, viene eseguito il dump delle informazioni di richiesta/risposta nel log del server. Questa opzione potrebbe fornire informazioni utili per il debug, ma determina notevoli riduzioni di prestazioni. Questa opzione è disabilitata per impostazione predefinita.

TCP senza ritardo

Se questa opzione è selezionata, viene abilitato TCP_NODELAY (anche noto come algoritmo di Nagle). Questa opzione è abilitata per impostazione predefinita.
