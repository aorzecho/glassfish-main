Nuevo Servidor Virtual

Utilice la página Nuevo Servidor Virtual para crear un servidor virtual.

La página Nuevo Servidor Virtual contiene las siguientes opciones.

Nombre de Configuración

Nombre de la configuración a la que se aplican los valores de esta página. Este campo es de sólo lectura.

ID

Identificador de servidor virtual visible internamente. No se muestra a los clientes HTTP. Los nombres de host que se muestran a los clientes HTTP se deben especificar en el campo Hosts.

Hosts

Nombre o nombres de host para el equipo en el que se ejecuta el servidor. Utilice nombres de host reales o virtuales que estén registrados en el servidor DNS de la red (y, en un sistema UNIX, en el archivo /etc/hosts). El valor por defecto es el valor de la propiedad del sistema ${com.sun.aas.hostName}.

Estado

El estado deseado para el servidor virtual. El valor puede ser cualquiera de los siguientes:

Activado

El servidor virtual está activo. Éste es el valor por defecto.

Desactivado

El servidor virtual está inactivo. Los intentos de acceder al servidor generarán el código de error 404 (recurso no disponible).

Desactivado

El servidor virtual está desactivado. Los intentos de acceder al servidor generarán el código de error 403 (denegación de cumplimiento de la solicitud).

SSO

Especifica si el comportamiento de single sign-on se hereda del Servicio HTTP, activado o desactivado. Por defecto, se hereda del Servicio HTTP. Si esta opción se encuentra activada, el single sign-on está activado para aplicaciones web que se han configurado para el mismo dominio en este servidor virtual. Si esta opción no se encuentra activada, el single sign-on está desactivado para este servidor virtual y los usuarios deben autenticarse por separado para cada aplicación del servidor virtual.

Listeners de Red

El listener o listeners de red asociados a este servidor, si los hubiera.

Módulo Web por Defecto

El módulo web desplegado (si lo hay) que debe responder a todas las solicitudes que no pueden asignarse a otros módulos web desplegados en el servidor virtual. Si no se especifica un módulo web por defecto, se utiliza un módulo web con una raíz de contexto vacía. Si no hay ningún módulo web con una raíz de contexto vacía, se crea y utiliza un módulo web por defecto del sistema.

Archivo Log

El nombre de ruta de acceso del archivo en el que aparecerán los mensajes de registro de este servidor virtual. Por defecto, aparecerán mensajes de registro en domain-dir/logs/server.log.

Raíz de Documento

La ruta de acceso absoluta al directorio raíz de documentos del servidor. El valor por defecto es domain-dir/docroot.

Registro de Acceso

Especifica si el registro de acceso se hereda del servicio HTTP, activado o desactivado. Por defecto, se hereda del Servicio HTTP.

Directorio

La ruta de acceso de directorio absoluta a los logs de acceso del servidor. El valor por defecto es domain-dir/logs/access.

Propiedades Adicionales

Propiedades adicionales para el servidor virtual.

Para obtener información sobre las propiedades disponibles, consulte Propiedades Específicas de los Servidores Virtuales.
