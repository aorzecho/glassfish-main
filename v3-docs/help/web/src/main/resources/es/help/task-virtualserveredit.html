Editar un Servidor Virtual

  1. En el árbol de navegación, amplíe el nodo Configuración.

  2. Select the configuration that you are changing.

  3. En el nodo Configuración, seleccione el nodo Servidores Virtuales.

    Se abrirá la página Servidores Virtuales.

  4. En la página Servidores Virtuales, haga clic en el nombre del servidor virtual que está editando.

    Se abrirá la página Editar Servidor Virtual.

  5. En el campo Hosts, introduzca los nombres de host o las direcciones IP del equipo en el que se está ejecutando el servidor.

    Utilice nombres de host reales o virtuales que estén registrados en el servidor DNS de la red (y, en un sistema UNIX, en el archivo /etc/hosts). El valor por defecto es el valor de la propiedad del sistema ${com.sun.aas.hostName}.

  6. Seleccione una opción de estado para el servidor virtual.

    El valor puede ser cualquiera de los siguientes:

    Activado

    El servidor virtual está activo. Éste es el valor por defecto.

    Desactivado

    El servidor virtual está inactivo. Los intentos de acceder al servidor generarán el código de error 404 (recurso no disponible).

    Desactivado

    El servidor virtual está desactivado. Los intentos de acceder al servidor generarán el código de error 403 (denegación de cumplimiento de la solicitud).

  7. Seleccione una opción SSO para el nuevo servidor virtual.

    Por defecto, un servidor virtual hereda la configuración de single sign-on (SSO) del servicio HTTP. Seleccione Activado o Desactivado para sustituir la configuración del servicio HTTP activando o desactivando el single sign-on para este servidor virtual.

    Los servlets y los archivos JSP admiten el single sign-on en las aplicaciones de Servidor GlassFish. Esta función permite a varias aplicaciones que requieren la misma información de inicio de sesión compartir esta información de modo que el usuario no tenga que iniciar una sesión por separado para cada aplicación.

  8. Desde la lista desplegable Listeners de Red, seleccione uno o varios listeners de red o deje el campo en blanco.

  9. En la lista desplegable Módulo Web por Defecto, seleccione el módulo que va a responder a todas las solicitudes que no se pueden asignar a otros módulos web desplegados en el servidor virtual.

    Si no se especifica un módulo web por defecto, se utiliza un módulo web con una raíz de contexto vacía. Si no hay ningún módulo web con una raíz de contexto vacía, se crea y utiliza un módulo web por defecto del sistema.

  10. En el campo Archivo Log, introduzca el nombre de ruta de acceso del archivo en el que aparecerán los mensajes de registro de este servidor virtual.

    Por defecto, los mensajes de registro se enviarán al log del servidor por defecto, domain-dir/logs/server.log.

  11. En el campo Raíz de Documento, introduzca la ruta de acceso absoluta al directorio raíz de documentos del servidor.

    El valor por defecto es domain-dir/docroot.

  12. Seleccione la opción Registro de Acceso para activar o desactivar el registro de acceso.

    Por defecto, un servidor virtual hereda la configuración de registro de acceso del servicio HTTP. Seleccione Activado o Desactivado para sustituir la configuración del servicio HTTP activando o desactivando el registro de acceso para este servidor virtual.

  13. En el campo Directorio, introduzca la ruta de acceso de directorio absoluta en los logs de acceso del servidor.

    El valor por defecto es domain-dir/logs/access.

  14. En la sección Propiedades Adicionales, especifique propiedades adicionales del servidor virtual.

    Para obtener información sobre las propiedades disponibles, consulte Propiedades Específicas de los Servidores Virtuales.

    • Para agregar una propiedad, haga clic en el botón Agregar Propiedad. En la fila en blanco que aparece, escriba el nombre de propiedad en el campo Nombre y escriba el valor de propiedad en el campo Valor.

    • Para modificar una propiedad, edite su campo Valor.

    • Para suprimir una propiedad, seleccione la casilla de verificación a la izquierda del campo Nombre de la propiedad que está suprimiendo y, a continuación, haga clic en el botón Suprimir Propiedades.

  15. Haga clic en Guardar.

    Sugerencia:

    Para volver a la configuración por defecto, haga clic en el botón Cargar Valores por Defecto y, a continuación, en Guardar.
