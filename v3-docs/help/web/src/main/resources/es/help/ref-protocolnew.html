Nuevo protocolo

Utilice la página Nuevo protocolo para crear un nuevo protocolo de red.

Después de crear un protocolo, debe crear normalmente una escucha de red con el mismo nombre y asociarla con el protocolo. Si lo desea, también puede crear una escucha de red y hacer que se cree un protocolo con configuración predeterminada al mismo tiempo.

La página Nuevo protocolo contiene las siguientes opciones.

Configuration Name

El nombre de la configuración a la que aplicar los valores de esta página. Este campo es de sólo lectura.

Nombre

El nombre único del protocolo. Un nombre de protocolo no puede empezar con un número.

Security

Si se selecciona esta opción, se habilita la seguridad para el protocolo. Esta opción se deshabilita por defecto. Si habilita la seguridad, puede utilizar la ficha SSL de la página Editar protocolo para modificar la configuración de seguridad.

Estado

Si se selecciona esta opción, se habilita la antememoria de archivo. Esta opción está habilitada por defecto.

Edad máxima

La edad máxima, en segundos, de una entrada en caché válida.

Este parámetro controla cómo se utiliza la información almacenada en caché después de que un archivo se haya almacenado en caché. Una nueva entrada sustituye a una entrada más antigua que la edad máxima para el mismo archivo. Si el contenido no cambia con mucha frecuencia, aumente este valor para un mejor rendimiento.

La edad máxima óptima depende de si se modifican regularmente los archivos existentes. Por ejemplo, si los archivos se han modificado cuatro veces al día a intervalos regulares, considere establecer la edad máxima en 21.600 segundos (6 horas). De lo contrario, considere definir la edad máxima en el tiempo más largo que desea servir la versión anterior de un archivo después de que dicho archivo se haya modificado.

El valor predeterminado es 30.

Tamaño de caché máximo

El tamaño máximo total (en bytes) de la caché de archivo en disco. El valor predeterminado es 10485760.

Recuento máximo de archivos

El número máximo de archivos que se pueden almacenar en la caché de archivo.

Si el valor es demasiado grande, el servidor almacena en caché los archivos menos necesarios, lo que supone un gasto de memoria. Si el valor es demasiado pequeño, se pierden las ventajas del almacenamiento en caché. Pruebe distintos valores de este atributo para encontrar la solución óptima para aplicaciones específicas.

El valor predeterminado es 1024.

Nombre de servidor

El nombre de host que utilizar en las URL que el servidor envía al cliente. Este nombre es el nombre de alias si el servidor utiliza un alias. Si el servidor no utiliza un alias, deje vacío este campo.

Este valor afecta a las URL que genera automáticamente el servidor; no afecta a las URL de los directorios y archivos almacenados en el servidor. Si el servidor utiliza un alias, el nombre-servidor debería ser el nombre de alias. Si se añaden dos puntos y un número de puerto, dicho puerto se usará en las URL que el servidor envía al cliente.

Servidor virtual predeterminado

El servidor virtual que se va a asociar con este protocolo. Utilice la página de Servidores virtuales para definir dichos servidores.

Redirigir puerto

Valor de puerto que redirige una solicitud a otro puerto. Servidor GlassFish redirige automáticamente la solicitud si se dan estas dos condiciones:

  • La escucha de red que hace referencia a este protocolo admite otras solicitudes que no son SSL.

  • Se recibe una solicitud para la que una restricción de seguridad coincidente requiere transporte SSL.

Si no se especifica un puerto de redirección, Servidor GlassFish utiliza el número de puerto especificado en la solicitud original. De forma predeterminada, no se especifica ningún puerto.

Conexiones máximas

Especifica el número máximo de solicitudes que se pueden canalizar hasta que el servidor cierre la conexión. El subsistema de mantenimiento de conexiones sondea periódicamente las conexiones inactivas. El valor predeterminado es 256.

Establezca esta propiedad en 1 para deshabilitar el mantenimiento HTTP/1.0, así como la canalización y el mantenimiento HTTP/1.1. Un valor de 0 significa que las solicitudes serán siempre rechazadas. Un valor de -1 no establece ningún límite en cuanto al número de conexiones de mantenimiento.

Tiempo de espera

El tiempo máximo en segundos que una conexión puede considerarse como inactiva y conservarse en el estado de mantenimiento. Un valor de 0 o menos significa que las conexiones de mantenimiento se mantienen abiertas de forma indefinida. El valor predeterminado es 30 segundos.

Tiempo de espera de cargar

Si esta opción se encuentra seleccionada, la conexión a un servlet que lee bytes lentamente se cerrará tras alcanzarse el límite del Tiempo de carga de espera de conexión. Si esta opción se encuentra deshabilitada, las conexiones del servlet no agotan el tiempo de espera. Esta opción se deshabilita por defecto.

Tiempo de carga de espera

El tiempo de carga de espera, en milisegundos. Este campo únicamente está disponible si se ha seleccionado la casilla de verificación destinada a habilitar el tiempo de carga de espera. El valor predeterminado es 300000 milisegundos.

Tiempo de espera de solicitud

Número de segundos antes de que se agote el tiempo de espera de una solicitud. Si la solicitud no se procesa antes de alcanzarse el valor de tiempo de espera, se ignorará la solicitud. El valor predeterminado es 30 segundos.

Tamaño del búfer de envío

El tamaño en bytes del búfer de envío. El valor predeterminado es 8192 bytes.

Longitud del búfer de encabezado

Tamaño, en bytes, del búfer utilizado por los subprocesos de procesamiento de solicitudes para leer los datos de las solicitudes. El valor predeterminado es 8192 bytes.

Tamaño máximo de publicación

El tamaño máximo en bytes de las acciones POST. El valor predeterminado es 2097152 bytes.

Codificación de URI

El nombre del conjunto de caracteres que se utiliza para decodificar los URI de solicitud recibidos. El valor debe ser un nombre de conjunto de caracteres IANA válido. El valor predeterminado es UTF-8.

Versión

La versión del protocolo HTTP utilizado. El valor predeterminado es HTTP/1.1.

Compresión

Especifica el uso de compresión GZIP HTTP/1.1 para ahorrar ancho de banda de servidor. Las opciones disponibles son:

on

Comprime datos.

off

Inhabilita la compresión.

force

Fuerza la compresión de datos en todos los casos.

El valor predeterminado es off.

Tipos MIME comprimibles

Una lista separada por comas de los tipos MIME para los que se haya utilizado compresión HTTP. El valor predeterminado es text/html,text/xml,text/plain.

Tamaño mínimo de compresión

El tamaño mínimo de un archivo cuando se le aplica compresión. Este valor debe configurarse si Compresión se establece en on o force. El valor predeterminado es 2048 bytes.

Agentes de usuario sin compresión

Una lista separada por comas de expresiones regulares que coinciden con agentes de usuario de clientes HTTP para los que no se debe usar compresión. De forma predeterminada, este valor es una cadena vacía.

Agente de usuario restringido

Una lista de agentes de usuario restringidos a los que se les aplica compresión HTTP. Si no se ha especificado ningún agente de usuario, se aplica la compresión HTTP a todos los agentes de usuario. De forma predeterminada, no hay agentes de usuario especificados.

Tipo de respuesta predeterminada

Una cadena que especifica el tipo de respuesta predeterminada. El formato es una cadena separada por punto y coma que incluye el tipo de contenido, la codificación, el idioma y el conjunto de caracteres. El valor predeterminado es text/html; charset=iso-8859-1.

Tipo de respuesta forzada

Una cadena que especifica el tipo de solicitud utilizado en caso de que no haya ninguna asignación MIME disponible que coincida con la extensión del archivo. El formato es una cadena separada por punto y coma que incluye el tipo de contenido, la codificación, el idioma y el conjunto de caracteres. El valor predeterminado es text/html; charset=iso-8859-1.

Adaptador

El nombre de clase del adaptador de recursos estático. El valor predeterminado es com.sun.grizzly.tcp.StaticResourcesAdapter.

Compatibilidad con Comet

Si esta opción se encuentra seleccionada, la compatibilidad Comet estará habilitada para la escucha de red que hace referencia a este protocolo. De forma predeterminada, esta opción se encuentra deshabilitada. Si el servlet o la página JSP utiliza tecnología Comet, asegúrese de que se ha inicializado cuando se inicia Servidor GlassFish al añadir el elemento load-on-startup al archivo web.xml. Por ejemplo:

<servlet>
<servlet-name>CheckIn</servlet-name>
<servlet-class>CheckInServlet</servlet-class>
<load-on-startup>0</load-on-startup>
</servlet>
Búsqueda de DNS

Si está opción se encuentra seleccionada, la búsqueda de DNS estará habilitada. Esta opción se encuentra deshabilitada de forma predeterminada.

Búsqueda de DNS

Si está opción se encuentra seleccionada, la búsqueda de DNS estará habilitada. Esta opción se encuentra deshabilitada de forma predeterminada.

Compatibilidad de RCM

Si esta opción se encuentra seleccionada, la administración de configuración de recursos (RCM) está habilitada. Esta opción se deshabilita por defecto.

Seguimiento

Si esta opción se encuentra seleccionada, la operación TRACE está habilitada. Si esta opción se encuentra deshabilitada, Servidor GlassFish es menos susceptible de sufrir ataques de secuencias de comandos entre sitios. Esta opción se habilita por defecto.

Pase de autenticación

Si esta opción se encuentra seleccionada, eso indica que la escucha de red que utiliza este protocolo recibe tráfico desde un servidor proxy con terminación SSL. Esta opción se deshabilita por defecto.

Fragmentando

Si esta opción se encuentra seleccionada, la fragmentación de respuesta HTTP está habilitada. Esta opción se habilita por defecto.

Funciona gracias a

Si está seleccionada esta opción, Servidor GlassFish utiliza encabezados de X-Powered-By conforme a las especificaciones de Java Servlet 3.0 y Java Server Pages 2.0. Esta opción se habilita por defecto.
