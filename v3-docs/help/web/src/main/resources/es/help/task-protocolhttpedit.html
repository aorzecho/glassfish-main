Cómo editar la configuración HTTP para un protocolo

  1. En el árbol de navegación, amplíe el nodo Configuración.
  2. En el nodo Configuración, expanda el nodo Configuración de red.
  3. En el nodo Configuración de red, seleccione el nodo Protocolos.

    Se abrirá la página Protocolos.

  4. En la página Protocolos, haga clic en el nombre del protocolo que está editando.

    Se abrirá la página Editar protocolo.

  5. En la página Editar protocolo, haga clic en la ficha HTTP.

    Se abrirá la página HTTP.

  6. En el campo Nombre de servidor, introduzca el nombre del host que se va a utilizar en las URL que el servidor envía al cliente.

    Este nombre es el alias si el servidor hace uso de un nombre de alias. Si el servidor no utiliza un alias, deje vacío este campo.

  7. En la lista desplegable Servidor virtual predeterminado, seleccione el servidor virtual asociado al protocolo.

    Utilice la página Servidores virtuales para definir los servidores virtuales.

  8. En el campo Redireccionar puerto, introduzca el valor del puerto que se utilizará para redireccionar una solicitud a otro puerto.

    GlassFish Server redirecciona automáticamente la solicitud si se dan las dos condiciones siguientes:

    • Este protocolo admite otras solicitudes que no son SSL.

    • Se recibe una solicitud para la que una restricción de seguridad coincidente requiere transporte SSL.

    Si no se especifica un puerto de redirección, GlassFish Server utiliza el número de puerto especificado en la solicitud original.

  9. En el campo Número máximo de conexiones, introduzca el número máximo de conexiones persistentes que se van a mantener en el modo de mantenimiento.

    El subsistema de mantenimiento de conexiones sondea periódicamente las conexiones inactivas.

  10. En el campo Tiempo de espera, introduzca el tiempo máximo en segundos que una conexión puede considerarse inactiva y conservarse en estado de mantenimiento.

    Un valor de 0 o menos significa que las conexiones de mantenimiento se mantienen abiertas de forma indefinida. El valor predeterminado es 30 segundos.

  11. Seleccione la casilla de verificación de habilitación del tiempo de espera de carga para que la conexión de un servlet que lea bytes lentamente se cierre tras alcanzarse el límite indicado en Tiempo de espera de carga de conexión.

    Si esta opción se encuentra deshabilitada, las conexiones del servlet no agotan el tiempo de espera. Esta opción se deshabilita por defecto.

  12. En el campo Tiempo de espera de cara de conexión, introduzca el tiempo de carga, en milisegundos.

    Este campo únicamente está disponible si se ha seleccionado la casilla de verificación destinada a deshabilitar el tiempo de carga de espera. El valor predeterminado es 300000 milisegundos.

  13. En el campo Tiempo de espera de solicitud, introduzca el número de segundos que deben transcurrir antes de que se agote el tiempo de espera de una solicitud.

    Si la solicitud no se procesa antes de alcanzarse el valor de tiempo de espera, se ignorará la solicitud. El valor predeterminado es 30 segundos.

  14. En el campo Tamaño de búfer de envío, introduzca el tamaño, en bytes, del búfer de envío.

    El valor predeterminado es 8192 bytes.

  15. En el campo Longitud del búfer de encabezado, introduzca el tamaño, en bytes, del búfer que utilizan los subprocesos de procesamiento de solicitudes para leer los datos de las solicitudes.

    El valor predeterminado es 8192 bytes.

  16. En el campo Tamaño máximo de publicación, especifique el tamaño máximo en bytes de las acciones POST.

    El valor predeterminado es 2097152 bytes.

  17. En el campo Codificación de URI, escriba el nombre del conjunto de caracteres utilizados para descodificar los URI de solicitudes recibidos.

    El valor debe ser un nombre de conjunto de caracteres IANA válido. El valor predeterminado es UTF-8.

  18. En el campo Versión, introduzca la versión del protocolo HTTP utilizado.

    El valor predeterminado es HTTP/1.1.

  19. En la lista desplegable Compresión, seleccione la compresión GZIP HTTP/1.1 para ahorrar ancho de banda de servidor.

    Las opciones disponibles son:

    on

    Comprime datos.

    off

    Inhabilita la compresión.

    force

    Fuerza la compresión de datos en todos los casos.

    El valor predeterminado es off.

  20. En el campo Tipos MIME comprimibles, escriba una lista separada por comas de los tipos MIME para los que se utiliza la compresión HTTP.

    El valor predeterminado es text/html,text/xml,text/plain.

  21. En el campo Tamaño mínimo de compresión, escriba el tamaño mínimo de un archivo cuando se le aplica compresión.

    Este valor debe configurarse si Compresión se establece en on o force. El valor predeterminado es 2048 bytes.

  22. En el campo Agentes de usuario sin compresión, escriba una lista separada por comas de expresiones regulares que coinciden con agentes de usuario de clientes HTTP para los que no se debe usar compresión.

    De forma predeterminada, este valor es una cadena vacía.

  23. En el campo Agente de usuario restringido, escriba una lista de agentes de usuario restringidos a los que se les aplica compresión HTTP.

    Si no se ha especificado ningún agente de usuario, se aplica la compresión HTTP a todos los agentes de usuario. De forma predeterminada, no hay agentes de usuario especificados.

  24. En el campo Tipo de respuesta predeterminada, escriba una cadena que especifique el tipo de respuesta predeterminada.

    El formato es una cadena separada por punto y coma que incluye el tipo de contenido, la codificación, el idioma y el conjunto de caracteres. El valor predeterminado es text/html; charset=iso-8859-1.

  25. En el campo Tipo de respuesta forzada, escriba una cadena que especifica el tipo de solicitud utilizado en caso de que no haya ninguna asignación MIME disponible que coincida con la extensión del archivo.

    El formato es una cadena separada por punto y coma que incluye el tipo de contenido, la codificación, el idioma y el conjunto de caracteres. El valor predeterminado es text/html; charset=iso-8859-1.

  26. En el campo Adaptador, escriba el nombre de clase del adaptador de recursos estático.

    El valor predeterminado es com.sun.grizzly.tcp.StaticResourcesAdapter.

  27. Seleccione la casilla de verificación Asistencia Comet habilitada para habilitar la Asistencia Comet del protocolo.

    Esta opción se deshabilita por defecto.

  28. Seleccione la casilla de verificación de habilitación de búsqueda de DNS para habilitar la búsqueda en el sistema de nombres de dominio (DNS).

    Esta opción se deshabilita por defecto.

  29. Seleccione la casilla de verificación de habilitación de la compatibilidad RCM para habilitar la administración de configuración de recursos (RCM).

    Esta opción se deshabilita por defecto.

  30. Seleccione la casilla de verificación de habilitación de seguimiento para habilitar la operación TRACE.

    Si esta opción se encuentra deshabilitada, GlassFish Server es menos susceptible de sufrir ataques de secuencias de comandos entre sitios. Esta opción se habilita por defecto.

  31. Seleccione la casilla de verificación Pase de autenticación para indicar que la escucha de red que utiliza este protocolo recibe tráfico desde un servidor proxy con terminación SSL.

    Esta opción se deshabilita por defecto.

  32. Seleccione la casilla de verificación de habilitación de la fragmentación para habilitar la fragmentación de la respuesta HTTP.

    Esta opción se habilita por defecto.

  33. Seleccione la casilla de verificación XPowered By habilitado para indicar a GlassFish Server que utilice los encabezados de X-Powered-By conforme a las especificaciones de Java Servlet 3.0 y Java Server Pages 2.0.

    Esta opción se habilita por defecto.

  34. Haga clic en Guardar.

    Consejo - Para volver a los valores de configuración predeterminados, haga clic en el botón Cargar predeterminados y, a continuación, en Guardar.
