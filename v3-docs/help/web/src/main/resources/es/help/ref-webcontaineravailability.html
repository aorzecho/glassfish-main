Disponibilidad de contenedor Web

Utilice la página Disponibilidad de contenedor Web para configurar las opciones de disponibilidad de contenedor web.

La página Disponibilidad de contenedor Web contiene las siguientes opciones.

Cargar valores predeterminados

Botón para restablecer las opciones que tienen valores por defecto a estos. Las opciones que no tienen valores por defecto no se modifican.

Configuration Name

El nombre de la configuración a la que aplicar los valores de esta página. Este campo es de sólo lectura.

Servicio de disponibilidad

Si se selecciona la casilla de verificación Habilitada, se ha habilitado la disponibilidad para el contenedor web.

Tipo de persistencia

El mecanismo de persistencia utilizado.

  • Si el Servicio de disponibilidad Habilitado no está activado, seleccione una de las siguientes opciones:

    • memory para ninguna persistencia de sesión

    • file para la persistencia de sesión en función del sistema de archivos que no es de alta disponibilidad

  • Si el Servicio de disponibilidad Habilitado está activado, seleccione replicated para la persistencia de sesión en función de la conmutación por error del clúster que es de alta disponibilidad.

Frecuencia de persistencia

La frecuencia con la que se almacena la sesión HTTP:

  • web-method — El estado de la sesión se almacena al final de cada solicitud web antes de enviar una respuesta al cliente. Este modo ofrece la mejor garantía de que el estado de la sesión se actualizará completamente en caso de fallo. Éste es el valor predeterminado.

Ámbito de persistencia

El ámbito de los cambios de la sesión HTTP necesarios para que se realice el almacenamiento:

  • session — Se almacena cada vez el estado de la sesión completo. Este modo ofrece la mejor garantía de que sus datos de sesión se almacenan correctamente para las aplicaciones web distribuibles.

  • modified-session — El estado de la sesión completo se almacena si se ha modificado. Se considera que se ha modificado una sesión si se ha realizado una llamada a HttpSession.setAttribute() o HttpSession.removeAttribute(). Debe garantizar que se llama a setAttribute() cada vez que se cambia un atributo. Esto no es un requisito de especificación de Java EE, pero es necesario para que este modo funcione correctamente.

  • modified-attribute — Sólo se almacenan los atributos de sesión modificados. Para que este modo funcione correctamente, debe seguir estas directrices:

    • Llame a setAttribute cada vez que se modifique el estado de la sesión.

    • Asegúrese de que no existen referencias cruzadas entre atributos. El gráfico de objetos debajo de cada clave de atributo distinto se serializa y almacena por separado. Si hay referencias cruzadas de objetos entre los objetos de cada clave diferente, no están serializados ni deserializados correctamente.

    • Distribuya el estado de la sesión en varios atributos o por lo menos entre un atributo de sólo lectura y uno modificable.

Estado de inicio de sesión único

Si se selecciona la casilla de verificación Habilitado, el estado de inicio de sesión único estará disponible para la conmutación por error.

Propiedades adicionales

Propiedades adicionales de la Disponibilidad de contenedor web.

Servidor GlassFish no define propiedades adicionales para la disponibilidad de contenedor web.
