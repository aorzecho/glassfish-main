Editar servidor virtual

Utilice la página Editar servidor virtual para modificar la configuración de un servidor virtual.

La página Editar servidor virtual contiene las siguientes opciones.

Cargar valores predeterminados

Botón para restablecer las opciones que tienen valores por defecto a estos. Las opciones que no tienen valores por defecto no se modifican.

Configuration Name

El nombre de la configuración a la que aplicar los valores de esta página. Este campo es de sólo lectura.

Id

Identificador de servidor virtual visible internamente. No se muestra a los clientes HTTP. Los nombres de host que se muestran a los clientes HTTP se deben especificar en el campo Hosts.

Hosts

Nombre o nombres de host para el equipo en el que se ejecuta el servidor. Utilice nombres de host reales o virtuales que estén registrados en el servidor DNS de la red (y, en un sistema UNIX, en el archivo /etc/hosts). El valor predeterminado es el valor de la propiedad del sistema ${com.sun.aas.hostName}.

Estado

El estado deseado para el servidor. El valor puede ser cualquiera de los siguientes:

Activado

El servidor virtual está activo. Éste es el valor predeterminado.

Desactivado

El servidor virtual está inactivo. Los intentos de acceder al servidor generarán el código de error 404 (recurso no disponible).

Deshabilitado

El servidor virtual está deshabilitado. Los intentos de acceder al servidor generarán el código de error 403 (denegación de cumplir la solicitud).

SSO

Especifica si el comportamiento de inicio de sesión único se hereda del servicio HTTP, habilitado o deshabilitado. De forma predeterminada, se hereda del servicio HTTP. Si esta opción se encuentra habilitada, el inicio de sesión único está habilitado para aplicaciones web que se han configurado para el mismo dominio en este servidor virtual. Si esta opción no se encuentra habilitada, el inicio de sesión único está deshabilitado para este servidor virtual y los usuarios deben autenticarse por separado para cada aplicación del servidor virtual.

Network Listeners

La escucha o las escuchas de red asociadas a este servidor.

Módulo web predeterminado

El módulo web implementado (si lo hay) que debe responder a todas las solicitudes que no pueden asignarse a otros módulos web implementados en el servidor virtual. Si no se especifica un módulo web predeterminado, se utiliza un módulo web con una raíz de contexto vacía. Si no hay ningún módulo web con una raíz de contexto vacía, se crea y utiliza un módulo web predeterminado del sistema.

Archivo de registro

El nombre de ruta del archivo en el que se guardarán los mensajes de registro de este servidor virtual. De forma predeterminada, los mensajes de registro aparecerán en dir_dominio/logs/server.log .

Docroot

La ruta absoluta al directorio raíz de documentos del servidor. El valor predeterminado es dir_dominio/docroot.

Registro de acceso

Especifica si el registro de acceso se hereda del servicio HTTP, habilitado o deshabilitado. De forma predeterminada, se hereda del servicio HTTP.

Directorio

La ruta de directorio absoluta a los registros de acceso del servidor. El valor predeterminado es dir_dominio/logs/access.

Propiedades adicionales

Propiedades adicionales para el servidor virtual.

Para obtener información sobre las propiedades disponibles, consulte Propiedades específicas de los servidores virtuales.
