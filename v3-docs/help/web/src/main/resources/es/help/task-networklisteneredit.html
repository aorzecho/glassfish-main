Cómo editar la escucha de red

  1. En el árbol de navegación, amplíe el nodo Configuración.
  2. En el nodo Configuración, expanda el nodo Configuración de red.
  3. En el nodo Configuración de red, seleccione el nodo Escuchas de red.

    Se abrirá la página Escuchas de red.

  4. En la página Escuchas de red, haga clic en el nombre de la escucha que está editando.

    Se abrirá la página Editar escucha de red.

  5. Seleccione o anule la selección de la casilla de verificación Estado habilitado para cambiar el estado de la escucha de red.

    La escucha puede estar habilitada o deshabilitada. Esta opción se habilita por defecto. Si la escucha se encuentra deshabilitada, cualquier intento de establecer una conexión con ella tiene como resultado una excepción de socket (java.net.ConnectException).

  6. Seleccione la casilla de verificación Seguridad habilitada para habilitar la seguridad para el protocolo que utiliza la escucha de red.

    Esta opción se deshabilita por defecto.

  7. Seleccione la casilla de verificación de habilitación de la escucha JK para especificar que la escucha admite el protocolo mod-jk de Apache.

    El protocolo mod-jk es un protocolo de red/comunicación de propiedad entre httpd de Apache (servidor web) y una instancia back-end de GlassFish Server. Si la escucha admite este protocolo, se supone que httpd maneja todo el contenido estático, mientras que cualquier solicitud de recursos dinámicos (es decir, servlets y JSP) se direccionan a la escucha de red con el protocolo mod jk habilitado de GlassFish Server a través del protocolo mod-jk. Esta opción se deshabilita por defecto.

  8. En el campo Puerto, introduzca el número de puerto en el que se efectuará la escucha de red.

    Los valores válidos se encuentran entre 1 y 65535. En un sistema UNIX, la creación de sockets de escucha para puertos 1-1024 requiere privilegios de superusuario.

  9. En el campo Dirección, escriba la dirección IP en la que se efectuará la escucha de red.

    La dirección se puede especificar en par punteado o notación IPv6. Puede ser cualquiera (para INADDR_ANY) para recibir escuchas en todas las direcciones IP. Puede ser un nombre de host.

  10. En la lista desplegable Conjunto de subprocesos, seleccione el conjunto de subprocesos que se debe asociar a la escucha de red.
  11. Haga clic en Guardar.

    Consejo - Para volver a los valores de configuración predeterminados, haga clic en el botón Cargar predeterminados y, a continuación, en Guardar.

Pasos siguientes

Puede utilizar las fichas SSL, HTTP y Caché de archivos para editar la configuración del protocolo asociado a la escucha de red. La ficha SSL únicamente tiene sentido si se ha habilitado la seguridad para la escucha de red.
