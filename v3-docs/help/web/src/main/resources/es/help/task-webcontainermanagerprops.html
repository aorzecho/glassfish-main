Configurar Propiedades de Gestor para el Contenedor Web

El gestor de sesiones proporciona los medios para configurar el modo de crear y destruir las sesiones, el lugar en el que se ubica el estado de sesión y el número máximo de sesiones disponibles.

  1. En el árbol de navegación, amplíe el nodo Configuración.

  2. Seleccione la configuración que esté cambiando.

  3. En el nodo de configuración, seleccione el nodo Contenedor Web.

    Se abre la página Propiedades Generales.

  4. En la página Propiedades Generales, haga clic en el separador Propiedades de Gestor.

    Se abrirá la página Propiedades de Gestor.

  5. En el campo Intervalo de Recolección, introduzca el número de segundos que deben transcurrir antes de que los datos de sesión inactiva se supriman del almacén.

    El valor por defecto es 60.

    Establezca este valor de forma que sea menor que la frecuencia con la que cambian los datos de la sesión. Por ejemplo, este valor debe ser lo más bajo posible (1 segundo) para un servlet con contador de visitas en un sitio web con visitas frecuentes; de lo contrario, podría perder las últimas visitas cada vez que reinicie el servidor.

  6. En el campo Número Máximo de Sesiones, escriba el número máximo de sesiones que puede existir en la caché.

    Un valor de-1 indica que no hay un límite para el número de sesiones. El valor por defecto es -1.

    Una vez alcanzado el límite, un intento de crear una nueva sesión provoca la emisión de una excepción IllegalStateException.

  7. En el campo Nombre de Archivo de Sesión, escriba la ruta de acceso absoluta o relativa al directorio en el que se conserva el estado de la sesión entre reinicios de la aplicación, en caso de que sea posible conservar el estado.

    Si no se define esta ruta de acceso, el estado de la sesión no se conserva. Por defecto, esta ruta de acceso no se encuentra definida.

    Una ruta de acceso relativa se relaciona con el directorio temporal para esta aplicación web.

    Este valor únicamente se aplica si el Tipo de Persistencia es memory. Para definir el Tipo de Persistencia, consulte Configurar Opciones de Disponibilidad para el Contenedor Web.

  8. En la sección Propiedades Adicionales, especifique las propiedades adicionales.

    Servidor GlassFish no define ninguna propiedad adicional para el gestor de sesiones.

    • Para agregar una propiedad, haga clic en el botón Agregar Propiedad. En la fila en blanco que aparece, escriba el nombre de propiedad en el campo Nombre y escriba el valor de propiedad en el campo Valor.

    • Para modificar una propiedad, edite su campo Valor.

    • Para suprimir una propiedad, seleccione la casilla de verificación a la izquierda del campo Nombre de la propiedad que está suprimiendo y, a continuación, haga clic en el botón Suprimir Propiedades.

  9. Haga clic en Guardar.

    Sugerencia:

    Para volver a la configuración por defecto, haga clic en el botón Cargar Valores por Defecto y, a continuación, en Guardar.
