Cómo configurar propiedades de administrador para el contenedor web

El administrador de sesiones proporciona los medios para configurar el modo de crear y destruir las sesiones, dónde se ubica el estado de sesión y el número máximo de sesiones disponibles.

  1. En el árbol de navegación, amplíe el nodo Configuración.
  2. Seleccione la configuración que esté cambiando.
  3. En el nodo de configuración, seleccione el nodo Contenedor web.

    Se abre la página Propiedades generales.

  4. En la página Propiedades generales, haga clic en la ficha Propiedades del administrador.

    Se abrirá la página Propiedades del administrador.

  5. En el campo Intervalo de cierre, introduzca el número de segundos que deben transcurrir antes de que los datos de sesión inactiva se eliminen del almacén.

    El valor predeterminado es 60.

    Establezca este valor de forma que sea menor que la frecuencia con la que cambian los datos de la sesión. Por ejemplo, este valor debe ser lo más bajo posible (1 segundo) para un servlet con contador de visitas en un sitio web con visitas frecuentes; de lo contrario, podría perder las últimas visitas cada vez que reinicie el servidor.

  6. En el campo Número máximo de sesiones, escriba el número máximo de sesiones que puede existir en la caché.

    Un valor de -1 indica que no hay un límite para el número de sesiones. El valor predeterminado es -1.

    Una vez alcanzado el límite, un intento de crear una nueva sesión provoca la emisión de una excepción IllegalStateException.

  7. En el campo Nombre de archivo de sesión, escriba la ruta absoluta o relativa al directorio en el que se conserva el estado de la sesión entre reinicios de la aplicación, en caso de que sea posible conservar el estado.

    Si no se define esta ruta, el estado de la sesión no se conserva. De forma predeterminada, esta ruta no se encuentra definida.

    Una ruta relativa se relaciona con el directorio temporal para esta aplicación web.

    Este valor sólo se aplica si el Tipo de persistencia es memory. Para definir el Tipo de persistencia, consulte Cómo configurar las opciones de disponibilidad para el contenedor web.

  8. (Opcional) En la sección Propiedades adicionales, especifique las propiedades adicionales.

    GlassFish Server no define ninguna propiedad adicional para el administrador de sesiones.

    • Para añadir una propiedad, haga clic en el botón Añadir propiedad. En la fila en blanco que aparece, escriba el nombre de propiedad en el campo Nombre y escriba el valor de propiedad en el campo Valor.
    • Para modificar una propiedad, edite su campo Valor.
    • Para eliminar una propiedad, seleccione la casilla de verificación a la izquierda del campo Nombre de la propiedad que está eliminando y, a continuación, haga clic en el botón Eliminar propiedades.
  9. Haga clic en Guardar.

    Consejo - Para volver a los valores de configuración predeterminados, haga clic en el botón Cargar predeterminados y, a continuación, en Guardar.
