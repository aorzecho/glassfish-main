Editar configuración de SSL para un protocolo

Antes de Empezar

Esta tarea únicamente tiene sentido si se ha activado la seguridad para el protocolo.

  1. En el árbol de navegación, amplíe el nodo Configuración.

  2. En el nodo Configuración, amplíe el nodo Configuración de Red.

  3. En el nodo Configuración de Red, seleccione el nodo Protocolos.

    Se abrirá la página Protocolos.

  4. En la página Protocolos, haga clic en el nombre del protocolo que está editando.

    Se abrirá la página Editar Protocolo.

  5. En la página Editar Protocolo, haga clic en el separador SSL.

    Se abrirá la página SSL.

  6. Seleccione la casilla de verificación de activación de SSL3 para activar la opción SSL3.

    Esta opción se encuentra activada por defecto.

  7. Seleccione la casilla de verificación de activación de TLS para activar la seguridad a nivel de transporte (TLS).

    Esta opción se encuentra activada por defecto.

  8. Seleccione la casilla de verificación de activación de Autenticación de Cliente para solicitar a los clientes que se identifiquen en el servidor con cada solicitud.

    Esta opción se encuentra desactivada por defecto.

  9. En el campo Apodo de Certificado, introduzca el apodo del certificado del servidor en la base de datos de certificados o token PKCS#11.

    En el certificado, el formato del nombre es tokenname:nickname. Incluyendo el tokenname: parte del nombre de este atributo es opcional.

  10. En el campo Almacén de Claves, introduzca el nombre del archivo de almacén de claves (por ejemplo, keystore.jks).

  11. En el campo Algoritmo de Confianza, introduzca el nombre del algoritmo de gestión de confianza (por ejemplo, PKIX) que se utilizará para la validación de rutas de acceso de certificación.

  12. En el campo Longitud de Certificado Máxima, introduzca el número máximo de certificados intermedios no emitidos automáticamente que puede haber en una ruta de acceso de certificación.

    Este campo se utiliza solamente en caso de que el campo Algoritmo de Confianza esté configurado en PKIX. Un valor de 0 implica que la ruta de acceso únicamente puede contener un certificado. Un valor de -1 implica que la longitud de la ruta de acceso no está restringida (no hay un máximo). La configuración en un valor inferior a -1 inicia una excepción.

  13. En el campo Almacén de Confianza, introduzca el nombre de un archivo del almacén de confianza (por ejemplo, cacerts.jks).

  14. En el área Conjuntos de Cifrado, especifique los conjuntos de cifrado que se utilizarán.

    Si no añade ningún conjunto de cifrado, se utilizarán todos los conjuntos de cifrado.

  15. Haga clic en Aceptar.
