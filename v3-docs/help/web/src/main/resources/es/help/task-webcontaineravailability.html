Cómo configurar las opciones de disponibilidad para el contenedor web

  1. En el árbol de navegación, amplíe el nodo Configuración.
  2. Seleccione la configuración que esté cambiando.
  3. En el nodo de configuración, seleccione el nodo Servicio de disponibilidad.

    Se abrirá la página Servicio de disponibilidad.

  4. En la página Servicio de disponibilidad, haga clic en la ficha Disponibilidad de contenedor Web.

    Se abre la página Disponibilidad de contenedor Web.

  5. En el campo Servicio de disponibilidad, active la casilla Habilitado para habilitar la disponibilidad del contenedor web.

    Para deshabilitar la disponibilidad, quite la marca de selección de la casilla Servicio de disponibilidad.

  6. En la lista desplegable Tipo de persistencia, seleccione el mecanismo de persistencia.
    • Si el Servicio de disponibilidad Habilitado no está activado, seleccione una de las opciones siguientes:

      • memory para ninguna persistencia de sesión

      • file para la persistencia de sesión en función del sistema de archivos que no es de alta disponibilidad

    • Si el Servicio de disponibilidad Habilitado está activado, seleccione replicated para la persistencia de sesión en función de la conmutación por error del clúster que es de alta disponibilidad.
  7. De la lista desplegable Frecuencia de persistencia, seleccione la frecuencia con la que se almacena la sesión HTTP:

    • web-method — El estado de la sesión se almacena al final de cada solicitud web antes de enviar una respuesta al cliente. Este modo ofrece la mejor garantía de que el estado de la sesión se actualizará completamente en caso de fallo. Éste es el valor predeterminado.

    • time-based — El estado de la sesión se almacena en segundo plano con la frecuencia establecida en la configuración de Intervalo de cierre. Para obtener más información sobre esta opción, consulte Cómo configurar propiedades de administrador para el contenedor web.

      Este modo no garantiza que se actualice por completo el estado de la sesión. Sin embargo, puede ofrecer una mejora de rendimiento significativa porque el estado no se almacena tras cada solicitud.

    La Frecuencia de persistencia sólo es aplicable si el Tipo de persistencia es replicated.

  8. En la lista desplegable Ámbito de persistencia, seleccione el ámbito de los cambios de la sesión HTTP necesarios para que se realice el almacenamiento:

    • session — Se almacena cada vez el estado de la sesión completo. Este modo ofrece la mejor garantía de que sus datos de sesión se almacenan correctamente para las aplicaciones web distribuibles.

    • modified-session — El estado de la sesión completo se almacena si se ha modificado. Se considera que se ha modificado una sesión si se ha realizado una llamada a HttpSession.setAttribute() o HttpSession.removeAttribute(). Debe garantizar que se llama a setAttribute() cada vez que se cambia un atributo. Esto no es un requisito de especificación de Java EE, pero es necesario para que este modo funcione correctamente.

    • modified-attribute — Sólo se almacenan los atributos de sesión modificados. Para que este modo funcione correctamente, debe seguir estas directrices:

      • Llame a setAttribute() cada vez que se modifique el estado de la sesión.

      • Asegúrese de que no existen referencias cruzadas entre atributos. El gráfico de objetos debajo de cada clave de atributo distinto se serializa y almacena por separado. Si hay referencias cruzadas de objetos entre los objetos de cada clave diferente, no están serializados ni deserializados correctamente.

      • Distribuya el estado de la sesión en varios atributos o por lo menos entre un atributo de sólo lectura y uno modificable.

    El Ámbito de persistencia sólo es aplicable si el Tipo de persistencia es replicated.

  9. En el campo Estado de inicio de sesión único, active la casilla Habilitado para que el estado de inicio de sesión único esté disponible para la conmutación por error.

    Para que el estado de inicio de sesión único esté disponible para la conmutación por error, desactive la casilla Estado de inicio de sesión único habilitado.

  10. Para restablecer los valores predeterminados de todos los parámetros de configuración, haga clic en el botón Cargar valores predeterminados.
  11. (Opcional) En la sección Propiedades adicionales, especifique las propiedades adicionales.

    GlassFish Server no define ninguna propiedad adicional para la disponibilidad de contenedor web.

    • Para añadir una propiedad, haga clic en el botón Añadir propiedad. En la fila en blanco que aparece, escriba el nombre de propiedad en el campo Nombre y escriba el valor de propiedad en el campo Valor.
    • Para modificar una propiedad, edite su campo Valor.
    • Para eliminar una propiedad, seleccione la casilla de verificación a la izquierda del campo Nombre de la propiedad que está eliminando y, a continuación, haga clic en el botón Eliminar propiedades.
  12. Haga clic en Guardar.
