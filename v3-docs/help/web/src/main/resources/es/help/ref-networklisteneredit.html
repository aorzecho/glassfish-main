Editar Listener de Red

Utilice la página Editar Listener de Red para modificar un listener de red existente.

Cuando se edita un listener de red, también estarán disponibles los separadores SSL, HTTP y Caché de Archivos del protocolo asociado para que sea posible editarlas.

La página Editar Listener de Red contiene las siguientes opciones.

Cargar Valores por Defecto

Botón para restaurar la configuración que tiene valores por defecto a sus valores por defecto. La configuración que no tiene valores por defecto no se modifica.

Nombre de Configuración

Nombre de la configuración a la que se aplican los valores de esta página. Este campo es de sólo lectura.

Nombre

El nombre único del listener. El campo Nombre es un campo de sólo lectura. Únicamente puede especificar el nombre al crear un nuevo listener de red.

Protocolo

El protocolo asociado al listener de red. El campo Protocolo es un campo de sólo lectura. Puede crear o especificar un protocolo únicamente al crear un nuevo listener de red.

Estado

El estado del listener de red. El listener puede activarse o desactivarse. Esta opción se encuentra activada por defecto. Si el listener se encuentra desactivado, cualquier intento de establecer una conexión con él tiene como resultado una excepción de socket (java.net.ConnectException).

En las versiones anteriores a la 9.1 de Servidor GlassFish, un listener cuyo atributo activado se hubiera definido como false devolvía un código de respuesta 404 para cualquier solicitud que se le enviaba. Para lograr este comportamiento en la versión actual de Servidor GlassFish, establezca el estado del listener en activado y defina cada estado de servidor virtual asociado como desactivado. Un servidor virtual incluye sus listeners asociados en la lista desplegable Listeners de Red.

Seguridad

Si se selecciona esta opción, se activa para el protocolo utilizado por el listener de red. Esta opción se encuentra desactivada por defecto.

Listener JK

Si esta opción se encuentra seleccionada, el listener admite el protocolo mod-jk de Apache. El protocolo mod-jk es un protocolo de red/comunicación de propiedad entre httpd de Apache (servidor web) y una instancia back-end de Servidor GlassFish. Si el listener admite este protocolo, se supone que httpd maneja todo el contenido estático, mientras que cualquier solicitud de recursos dinámicos (es decir, servlets y JSP) se direccionan al listener de red con el protocolo mod jk activado de Servidor GlassFish a través del protocolo mod-jk. Esta opción se encuentra desactivada por defecto.

Puerto

El número de puerto en el que actuará el listener. Los valores válidos se encuentran entre 1 y 65535. En un sistema UNIX, la creación de sockets de escucha para los puertos 1-1024 requiere privilegios de superusuario.

Dirección

La dirección IP en la que actuará el listener de red. La dirección se puede especificar en par punteado o notación IPv6. Puede ser cualquiera (para INADDR/ANY) para recibir escuchas en todas las direcciones IP. Puede ser un nombre de host.

Pool de Threads

El pool de threads asociado al listener de red.

Transporte

El tipo de transporte del listener de red. Por defecto, el único transporte es tcp.
