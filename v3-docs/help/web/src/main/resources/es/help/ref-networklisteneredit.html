Editar escucha de red

Utilice la página Editar escucha de red para modificar una escucha de red existente.

Cuando se edita una escucha de red, también estarán disponibles las fichas SSL, HTTP y Caché de archivos del protocolo asociado para que sea posible editarlas.

La página Editar escucha de red contiene las siguientes opciones.

Cargar valores predeterminados

Botón para restablecer las opciones que tienen valores por defecto a estos. Las opciones que no tienen valores por defecto no se modifican.

Configuration Name

El nombre de la configuración a la que aplicar los valores de esta página. Este campo es de sólo lectura.

Nombre

El nombre único de la escucha. El campo Nombre es un campo de sólo lectura. Únicamente puede especificar el nombre al crear una nueva escucha de red.

Protocolo

El protocolo asociado a la escucha de red. El campo Protocolo es un campo de sólo lectura. Puede crear o especificar un protocolo únicamente al crear una nueva escucha de red.

Estado

El estado de la escucha de red. La escucha puede habilitarse o deshabilitarse. Esta opción se habilita por defecto. Si la escucha se encuentra deshabilitada, cualquier intento de establecer una conexión con ella tiene como resultado una excepción de socket (java.net.ConnectException).

En las versiones anteriores a la 9.1 de Servidor GlassFish, una escucha cuyo atributo habilitado se hubiera definido como false devolvía un código de respuesta 404 para cualquier solicitud que se le enviaba. Para lograr este comportamiento en la versión actual de Servidor GlassFish, establezca el estado de la escucha en habilitado y defina cada estado de servidor virtual asociado como desactivado. Un servidor virtual incluye sus escuchas asociadas en la lista desplegable Escuchas de red.

Security

Si se selecciona esta opción, se habilita para el protocolo utilizado por la escucha de red. Esta opción se deshabilita por defecto.

Escucha JK

Si se encuentra seleccionada esta opción, la escucha admite el protocolo mod-jk de Apache. El protocolo mod-jk es un protocolo de red/comunicación de propiedad entre httpd de Apache (servidor web) y una instancia back-end de Servidor GlassFish. Si la escucha admite este protocolo, se supone que httpd maneja todo el contenido estático, mientras que cualquier solicitud de recursos dinámicos (es decir, servlets y JSP) se direccionan a la escucha de red con el protocolo mod jk habilitado de Servidor GlassFish a través del protocolo mod-jk. Esta opción se deshabilita por defecto.

Puerto

El número de puerto en el que se efectuará la escucha. Los valores válidos se encuentran entre 1 y 65535. En un sistema UNIX, la creación de sockets de escucha para puertos 1-1024 requiere privilegios de superusuario.

Dirección

La dirección IP en la que efectuará la escucha de red. La dirección se puede especificar en par punteado o notación IPv6. Puede ser cualquiera (para INADDR_ANY) para recibir escuchas en todas las direcciones IP. Puede ser un nombre de host.

Conjunto de subprocesos

El conjunto de subprocesos asociado a la escucha de red.

Transporte

El tipo de transporte de la escucha de red. De forma predeterminada, el único transporte es tcp.
