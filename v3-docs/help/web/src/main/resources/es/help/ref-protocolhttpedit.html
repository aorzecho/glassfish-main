HTTP

Utilice la página HTTP para modificar la configuración HTTP para un protocolo de red.

La página HTTP contiene las opciones siguientes.

Cargar valores predeterminados

Botón para restablecer los valores predeterminados de todos los parámetros de configuración.

Nombre de la configuración

El nombre de la configuración a la que se aplican los valores de esta página. Este campo es de sólo lectura.

Nombre del protocolo

El nombre del protocolo. El campo Nombre del protocolo es un campo de sólo lectura. Únicamente puede especificar el nombre al crear un nuevo protocolo.

Nombre de servidor

El nombre de host que utilizar en las URL que el servidor envía al cliente. Este nombre es el nombre de alias si el servidor utiliza un alias. Si el servidor no utiliza un alias, deje el campo en blanco.

Servidor virtual predeterminado

El servidor virtual que se va a asociar al protocolo. Utilice la página de Servidores virtuales para definir dichos servidores.

Redirigir puerto

Valor de puerto que redirige una solicitud a otro puerto. GlassFish Server redirige automáticamente la solicitud si se dan estas dos condiciones:

  • Este protocolo admite solicitudes que no sean SSL.

  • Se recibe una solicitud para la que una restricción de seguridad coincidente requiere transporte SSL.

Si no se especifica un puerto de redirección, GlassFish Server utiliza el número de puerto especificado en la solicitud original.

Conexiones máximas

El número máximo de conexiones persistentes que mantener en el modo de mantenimiento de conexiones. El subsistema de mantenimiento de conexiones sondea periódicamente las conexiones inactivas. El valor predeterminado es 256.

Tiempo de espera

El tiempo máximo en segundos que una conexión puede considerarse como inactiva y conservarse en el estado de mantenimiento. Un valor de 0 o menos significa que las conexiones de mantenimiento se mantienen abiertas de forma indefinida. El valor predeterminado es 30 segundos.

Tiempo de espera de cargar

Si esta opción se encuentra seleccionada, la conexión a un servlet que lee bytes lentamente se cerrará tras alcanzarse el límite del Tiempo de carga de espera de conexión. Si esta opción se encuentra deshabilitada, las conexiones del servlet no agotan el tiempo de espera. Esta opción se deshabilita por defecto.

Tiempo de carga de espera

El tiempo de carga de espera, en milisegundos. Este campo únicamente está disponible si se ha seleccionado la casilla de verificación destinada a habilitar el tiempo de carga de espera. El valor predeterminado es 300000 milisegundos.

Tiempo de espera de solicitud

Número de segundos antes de que se agote el tiempo de espera de una solicitud. Si la solicitud no se procesa antes de alcanzarse el valor de tiempo de espera, se ignorará la solicitud. El valor predeterminado es 30 segundos.

Tamaño del búfer de envío

El tamaño en bytes del búfer de envío. El valor predeterminado es 8192 bytes.

Longitud del búfer de encabezado

Tamaño, en bytes, del búfer utilizado por los subprocesos de procesamiento de solicitudes para leer los datos de las solicitudes. El valor predeterminado es 8192 bytes.

Tamaño máximo de publicación

El tamaño máximo en bytes de las acciones POST. El valor predeterminado es 2097152 bytes.

Codificación de URI

El nombre del conjunto de caracteres que se utiliza para decodificar los URI de solicitud recibidos. El valor debe ser un nombre de conjunto de caracteres IANA válido. El valor predeterminado es UTF-8.

Versión

La versión del protocolo HTTP utilizado. El valor predeterminado es HTTP/1.1.

Compresión

Especifica el uso de compresión GZIP HTTP/1.1 para ahorrar ancho de banda de servidor. Las opciones disponibles son:

on

Comprime datos.

off

Inhabilita la compresión.

force

Fuerza la compresión de datos en todos los casos.

El valor predeterminado es off.

Tipos MIME comprimibles

Una lista separada por comas de los tipos MIME para los que se haya utilizado compresión HTTP. El valor predeterminado es text/html,text/xml,text/plain.

Tamaño mínimo de compresión

El tamaño mínimo de un archivo cuando se le aplica compresión. Este valor debe configurarse si Compresión se establece en on o force. El valor predeterminado es 2048 bytes.

Agentes de usuario sin compresión

Una lista separada por comas de expresiones regulares que coinciden con agentes de usuario de clientes HTTP para los que no se debe usar compresión. De forma predeterminada, este valor es una cadena vacía.

Agente de usuario restringido

Una lista de agentes de usuario restringidos a los que se les aplica compresión HTTP. Si no se ha especificado ningún agente de usuario, se aplica la compresión HTTP a todos los agentes de usuario. De forma predeterminada, no hay agentes de usuario especificados.

Tipo de respuesta predeterminada

Una cadena que especifica el tipo de respuesta predeterminada. El formato es una cadena separada por punto y coma que incluye el tipo de contenido, la codificación, el idioma y el conjunto de caracteres. El valor predeterminado es text/html; charset=iso-8859-1.

Tipo de respuesta forzada

Una cadena que especifica el tipo de solicitud utilizado en caso de que no haya ninguna asignación MIME disponible que coincida con la extensión del archivo. El formato es una cadena separada por punto y coma que incluye el tipo de contenido, la codificación, el idioma y el conjunto de caracteres. El valor predeterminado es text/html; charset=iso-8859-1.

Adaptador

El nombre de clase del adaptador de recursos estático. El valor predeterminado es com.sun.grizzly.tcp.StaticResourcesAdapter.

Compatibilidad con Comet

Si esta opción está seleccionada, la asistencia de Comet se habilita para el protocolo. Esta opción se deshabilita por defecto.

Búsqueda de DNS

Si esta opción se encuentra seleccionada, la búsqueda en el sistema de nombres de dominio (DNS) está habilitada. Esta opción se deshabilita por defecto.

Compatibilidad de RCM

Si esta opción se encuentra seleccionada, la administración de configuración de recursos (RCM) está habilitada. Esta opción se deshabilita por defecto.

Seguimiento

Si esta opción se encuentra seleccionada, la operación TRACE está habilitada. Si esta opción se encuentra deshabilitada, GlassFish Server es menos susceptible de sufrir ataques de secuencias de comandos entre sitios. Esta opción se habilita por defecto.

Pase de autenticación

Si esta opción se encuentra seleccionada, eso indica que la escucha de red que utiliza este protocolo recibe tráfico desde un servidor proxy con terminación SSL. Esta opción se deshabilita por defecto.

Fragmentando

Si esta opción se encuentra seleccionada, la fragmentación de respuesta HTTP está habilitada. Esta opción se habilita por defecto.

Funciona gracias a

Si está seleccionada esta opción, GlassFish Server utiliza encabezados de X-Powered-By conforme a las especificaciones de Java Servlet 3.0 y Java Server Pages 2.0. Esta opción se habilita por defecto.
