Para crear un servidor virtual

  1. En el árbol de navegación, amplíe el nodo Configuración.
  2. En el nodo Configuración, seleccione el nodo Servidores virtuales.

    Se abrirá la página Servidores virtuales.

  3. En la página Servidores virtuales, haga clic en Nuevo.

    Se abrirá la página Nuevo servidor virtual.

  4. En el campo Id., introduzca un valor que identificará el servidor virtual de forma interna.

    Este valor no se mostrará a los clientes HTTP. Los nombres de host que se muestran a los clientes HTTP se deben especificar en el campo Hosts.

  5. En el campo Hosts, introduzca los nombres de host o las direcciones IP del equipo en el que se está ejecutando el servidor.

    Utilice nombres de host reales o virtuales que estén registrados en el servidor DNS de la red (y, en un sistema UNIX, en el archivo /etc/hosts ).

    El valor predeterminado es el valor de la propiedad del sistema ${com.sun.aas.hostName}.

  6. Seleccione una opción de estado para el nuevo servidor virtual.

    El servidor virtual puede estar activado (es decir, active), desactivado (Off) o deshabilitado (Disabled). El valor predeterminado es activado (On).

  7. Seleccione una opción SSO para el nuevo servidor virtual.

    De forma predeterminada, un servidor virtual hereda la configuración de inicio de sesión único (SSO) del servicio HTTP. Seleccione Habilitado o Deshabilitado para anular la configuración del servicio HTTP habilitando o deshabilitando el inicio de sesión único para este servidor virtual.

    Los servlets y los archivos JSP admiten el inicio de sesión único en las aplicaciones de GlassFish Server. Esta función permite a varias aplicaciones que requieren la misma información de inicio de sesión compartir esta información de modo que el usuario no tenga que iniciar una sesión por separado para cada aplicación.

  8. Desde la lista desplegable Escuchas de red, seleccione una o varias escuchas de red o deje el campo en blanco.
  9. En la lista desplegable Módulo web predeterminado, seleccione el módulo que va a responder a todas las solicitudes que no se pueden asignar a otros módulos web implementados en el servidor virtual.

    Si no se especifica un módulo web predeterminado, se utiliza un módulo web con una raíz de contexto vacía. Si no hay ningún módulo web con una raíz de contexto vacía, se crea y utiliza un módulo web predeterminado del sistema.

  10. En el campo Archivo de registro, introduzca el nombre de ruta del archivo en el que se guardarán los mensajes de registro de este servidor virtual.

    De forma predeterminada, los mensajes de registro se enviarán al registro del servidor predeterminado, domain-dir/logs/server.log .

  11. En el campo Raíz de documentos, introduzca la ruta absoluta al directorio raíz de documentos del servidor.

    Si el campo se deja en blanco, se utilizará domain-dir/docroot.

  12. Seleccione la opción Registro de acceso para habilitar o deshabilitar el registro de acceso.

    Como opción alternativa, puede especificar si el registro de acceso debe heredarse del servicio HTTP. De forma predeterminada, se hereda del servicio HTTP.

  13. En el campo Directorio, introduzca la ruta de directorio absoluta en los registros de acceso del servidor.

    El valor predeterminado es domain-dir/logs/access .

  14. (Opcional) En la sección Propiedades adicionales, especifique propiedades adicionales del servidor virtual.

    Para obtener información sobre las propiedades disponibles, consulte Propiedades específicas de los servidores virtuales.

    Para añadir una propiedad, haga clic en el botón Agregar propiedad. En la fila vacía escriba el nombre de la propiedad en el campo Nombre y el valor de la propiedad en el campo Valor.

  15. Haga clic en Aceptar.
