Nuevo conjunto de subprocesos

Utilice la página Nuevo conjunto de subprocesos para crear un conjunto de subprocesos.

La página Nuevo conjunto de subprocesos contiene las siguientes opciones.

Configuration Name

El nombre de la configuración a la que aplicar los valores de esta página. Este campo es de sólo lectura.

Nombre

El nombre del conjunto de subprocesos.

Nombre de clase

El nombre completo de la clase que implementa el conjunto de subprocesos. El valor predeterminado es com.sun.grizzly.http.StatsThreadPool .

Tamaño de cola máximo

El número máximo de mensajes que pueden ponerse en cola hasta que haya subprocesos disponibles para procesarlos. Un valor de -1 indica que no hay límite en el tamaño de la cola. El valor predeterminado es 4096.

Tamaño máximo del conjunto de subprocesos

El número máximo de subprocesos en el conjunto de subprocesos. El valor predeterminado es 5. Éste es el valor recomendado de los conjuntos de subprocesos de escucha de red.

Tamaño mínimo del conjunto de subprocesos

El número mínimo de subprocesos en el conjunto de subprocesos. El valor predeterminado es 2.

Tiempo de espera de subproceso inactivo

La cantidad de tiempo máxima, en segundos, que un subproceso pueden mantenerse inactivo en el conjunto. Una vez caducado este tiempo, el subproceso se elimina del conjunto. El valor predeterminado es 900.
