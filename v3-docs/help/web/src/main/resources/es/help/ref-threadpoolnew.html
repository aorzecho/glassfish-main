Nuevo Pool de Threads

Utilice la página Nuevo Pool de Threads para crear un pool de threads.

La página Nuevo Pool de Threads contiene las siguientes opciones.

Nombre de Configuración

Nombre de la configuración a la que se aplican los valores de esta página. Este campo es de sólo lectura.

Nombre

El nombre del pool de threads.

Nombre de Clase

El nombre de clase completo que implementa el pool de threads. El valor por defecto es com.sun.grizzly.http.StatsThreadPool .

Tamaño de Cola Máximo

El número máximo de mensajes que pueden ponerse en cola hasta que haya threads disponibles para procesarlos. Un valor de -1 indica que no hay límite en el tamaño de la cola. El valor por defecto es 4096.

Tamaño Máximo del Pool de Threads

El número máximo de threads en el pool de threads. El valor por defecto es 5. Éste es el valor recomendado de los pools de threads de listener de red.

Tamaño Mínimo de Pool de Threads

El número mínimo de threads en el pool de threads. El valor por defecto es 2.

Timeout de Thread Inactivo

La cantidad de tiempo máxima, en segundos, que un thread puede mantenerse inactivo en el pool. Una vez transcurrido este tiempo, el thread se elimina del pool. El valor por defecto es 900.
