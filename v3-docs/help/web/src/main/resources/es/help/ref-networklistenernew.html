Nueva escucha de red

Utilice la página Nueva escucha de red para crear una nueva escucha de red.

Cuando se crea una escucha de red, es posible crear simultáneamente un protocolo que utilizará la escucha de red o utilizar un protocolo que cree previamente. Normalmente, existe una relación de uno a uno entre una escucha de red y su protocolo asociado, pero no es necesario que sea así.

La página Nueva escucha de red contiene las siguientes opciones.

Nombre de la configuración

El nombre de la configuración a la que se aplican los valores de esta página. Este campo es de sólo lectura.

Nombre

Un nombre de escucha único. Un nombre de escucha de red no puede empezar con un número.

Protocolo

El protocolo asociado a la escucha de red. Las siguientes opciones especifican si se debe crear un protocolo o utilizar uno existente.

Crear un protocolo

Se crea un nuevo protocolo con valores de configuración predeterminados para la escucha de red. El nombre predeterminado del protocolo es el nombre de la escucha de red con la terminación -protocol añadida, pero es posible editar este valor.

Servidor virtual

En el caso de un nuevo protocolo, el servidor virtual que se asociará con el protocolo.

Utilizar un protocolo existente

Se utiliza el protocolo seleccionado.

Estado

El estado de la escucha de red. La escucha puede habilitarse o deshabilitarse. Esta opción se habilita por defecto. Si la escucha se encuentra deshabilitada, cualquier intento de establecer una conexión con ella tiene como resultado una excepción de socket (java.net.ConnectException).

En las versiones anteriores a la 9.1 de GlassFish Server, una escucha cuyo atributo habilitado se hubiera definido como false devolvía un código de respuesta 404 para cualquier solicitud que se le enviaba. Para lograr este comportamiento en la versión actual de GlassFish Server, establezca el estado de la escucha en habilitado y defina cada estado de servidor virtual asociado como desactivado. Un servidor virtual incluye sus escuchas asociadas en la lista desplegable Escuchas de red.

Security

Cuando esta opción está seleccionada, la seguridad se encuentra habilitada para el protocolo utilizado por la escucha de red. Esta opción se deshabilita por defecto.

Escucha JK

Si se encuentra seleccionada esta opción, la escucha admite el protocolo mod-jk de Apache. El protocolo mod-jk es un protocolo de red/comunicación de propiedad entre httpd de Apache (servidor web) y una instancia back-end de GlassFish Server. Si la escucha admite este protocolo, se supone que httpd maneja todo el contenido estático, mientras que cualquier solicitud de recursos dinámicos (es decir, servlets y JSP) se direccionan a la escucha de red con el protocolo mod jk habilitado de GlassFish Server a través del protocolo mod-jk. Esta opción se deshabilita por defecto.

Puerto

El número de puerto en el que se efectuará la escucha. Los valores válidos se encuentran entre 1 y 65535. En un sistema UNIX, la creación de sockets de escucha para puertos 1-1024 requiere privilegios de superusuario.

Dirección

La dirección IP en la que efectuará la escucha de red. La dirección se puede especificar en par punteado o notación IPv6. Puede ser cualquiera (para INADDR_ANY) para recibir escuchas en todas las direcciones IP. Puede ser un nombre de host.

Conjunto de subprocesos

El conjunto de subprocesos asociado a la escucha de red. Normalmente, puede seleccionar uno de las dos conjuntos de subprocesos que están configurados cuando se instala GlassFish Server.

Transporte

El tipo de transporte de la escucha de red. De forma predeterminada, la única alternativa es tcp.
