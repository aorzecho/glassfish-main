Nuevo Transporte

Utilice la página Nuevo Transporte para crear un nuevo transporte.

No es frecuente necesitar otro transporte además del transporte por defecto tcp.

La página Nuevo Transporte contiene las siguientes opciones.

Nombre de Configuración

Nombre de la configuración a la que se aplican los valores de esta página. Este campo es de sólo lectura.

Nombre de Transporte

Un nombre único para el transporte.

Nombre de Clase

El nombre de clase de la implementación del transporte. El valor por defecto es com.sun.grizzly.TCPSelectorHandler.

Manejador de Clave de Selección

El nombre de clase del manejador de clave de selección. Un manejador de clave de selección es una clase abstracta que administra el ciclo de vida de las claves de selección. Si no se especifica ningún valor, se utiliza una implementación por defecto.

Tipo de Buffer de Bytes

El tipo de buffer de bytes que se va a utilizar. Las opciones disponibles son HEAP y DIRECT. El valor por defecto es HEAP.

Threads de Responsable de Aceptación

El número de procesadores del equipo. Un valor de -1 indica que Grizzly calculará por su cuenta el número de threads del dispositivo de aceptación según el número de procesadores del equipo. El valor por defecto es 1.

Para definir el número de threads de procesamiento de solicitudes, establezca el valor del Tamaño Máximo del Pool de Threads del pool de threads utilizado por un listener de red que emplea este transporte.

Número Máximo de Conexiones

El número máximo de conexiones pendientes en un listener de red que utiliza este transporte. El valor por defecto es 4096.

Tamaño de Buffer

El tamaño (en bytes) del buffer que se proporciona para las secuencias de entrada creadas por un listener de red que utiliza este transporte. El valor por defecto es 8192.

Timeout de Clave Inactiva

El número de segundos después de los cuales se cancelará una clave inactiva y se cerrará el canal. El valor por defecto es 30.

Timeout de Lectura

El número de milisegundos que Servidor GlassFish esperará durante la fase de análisis de cabecera y cuerpo de una operación de lectura. El valor por defecto es 30000.

Timeout de Sondeo de Selector

El número de milisegundos tras los cuales un selector NIO se bloqueará a la espera de eventos (solicitudes de usuarios). El valor por defecto es 1.000.

Timeout de Escritura

El número de milisegundos que Servidor GlassFish espera antes de considerar que el cliente remoto está desconectado al escribir la respuesta. El valor por defecto es 30000.

Configuración de Pantalla

Si esta opción se encuentra seleccionada, la configuración interna de Grizzly se descarga en los logs del servidor. Esta opción puede proporcionar información útil para la depuración. Esta opción se encuentra desactivada por defecto.

Investigar

Si esta opción se encuentra seleccionada, la información de la respuesta/solicitud se vuelca en el log del servidor. Puede que esta opción proporcione información útil para realizar la depuración, pero reduce significativamente el rendimiento. Esta opción se encuentra desactivada por defecto.

TCP sin Retraso

Si esta opción se encuentra seleccionada, la opción TCP_NODELAY (también llamada algoritmo de Nagle) está activada. Esta opción se encuentra activada por defecto.
