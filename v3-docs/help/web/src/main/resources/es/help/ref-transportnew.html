Nuevo transporte

Utilice la página Nuevo transporte para crear un nuevo transporte.

No es frecuente necesitar otro transporte además del transporte predeterminado tcp.

La página Nuevo transporte contiene las siguientes opciones.

Nombre de la configuración

El nombre de la configuración a la que se aplican los valores de esta página. Este campo es de sólo lectura.

Nombre de transporte

Un nombre único para el transporte.

Nombre de clase

El nombre de clase de la implementación del transporte. El valor predeterminado es com.sun.grizzly.TCPSelectorHandler.

Controlador de clave de selección

El nombre de clase del controlador de clave de selección. Un controlador de clave de selección es una clase abstracta que administra el ciclo de vida de las claves de selección. Si no se especifica ningún valor, se utiliza una implementación predeterminada.

Tipo de búfer en byte

El tipo de búfer en byte que se va a utilizar. Las opciones disponibles son HEAP y DIRECT. El valor predeterminado es HEAP.

Subprocesos del aceptador

El número de procesadores del equipo. Un valor de -1 indica que Grizzly calculará por su cuenta el número de subprocesos del aceptador según el número de procesadores de la máquina. El valor predeterminado es 1.

Para definir el número de subprocesos de procesamiento de solicitudes, establezca el valor del Tamaño máximo del conjunto de subprocesos del conjunto de subprocesos utilizado por una escucha de red que emplea este transporte.

Número máx. de conexiones

El número máximo de conexiones pendientes en una escucha de red que utiliza este transporte. El valor predeterminado es 4096.

Tamaño de búfer

El tamaño (en bytes) del búfer que se proporciona para las secuencias de entrada creadas por una escucha de red que utiliza este transporte. El valor predeterminado es 8192.

Tiempo de espera clave inactiva

El número de segundos después de los cuales se cancelará una clave inactiva y se cerrará el canal. El valor predeterminado es 30.

Tiempo de espera de lectura

El número de milisegundos que GlassFish Server esperará durante la fase de análisis de encabezado y cuerpo de una operación de lectura. El valor predeterminado es 30000.

Tiempo de espera de sondeo de selector

El número de milisegundos tras los cuales un selector NIO se bloqueará a la espera de eventos (solicitudes de usuarios). El valor predeterminado es 1.000.

Tiempo de espera de escritura

El número de milisegundos que GlassFish Server espera antes de considerar que el cliente remoto está desconectado al escribir la respuesta. El valor predeterminado es 30000.

Configuración de pantalla

Si esta opción se encuentra seleccionada, la configuración interna de Grizzly se descarga en los registros del servidor. Esta opción puede proporcionar información útil para la depuración. Esta opción se deshabilita por defecto.

Snoop

Si esta opción se encuentra seleccionada, la información de la respuesta/solicitud se vuelca en el registro del servidor. Puede que esta opción proporcione información útil para realizar la depuración, pero reduce significativamente el rendimiento. Esta opción se deshabilita por defecto.

TCP sin retraso

Si esta opción se encuentra seleccionada, la opción TCP_NODELAY (también llamada algoritmo de Nagle) está habilitada. Esta opción se habilita por defecto.
