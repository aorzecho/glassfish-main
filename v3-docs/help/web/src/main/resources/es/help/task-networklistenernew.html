Crear un Listener de Red

Antes de Empezar

Cuando se crea un listener de red, es posible crear simultáneamente un protocolo que utilizará el listener de red o utilizar un protocolo que cree previamente. Normalmente, existe una relación de uno a uno entre un listener de red y su protocolo asociado, pero no es necesario que sea así.

  1. En el árbol de navegación, amplíe el nodo Configuración.

  2. En el nodo Configuración, amplíe el nodo Configuración de Red.

  3. En el nodo Configuración de Red, seleccione el nodo Listeners de Red.

    Se abrirá la página Listeners de Red.

  4. En la página Listeners de Red, haga clic en Nuevo.

    Se abrirá la página Nuevo Listener de Red.

  5. En el campo Nombre, escriba un nombre para el listener de red.

    Un nombre del listener de red no puede empezar con un número.

  6. En el campo Protocolo, especifique el protocolo que se va a asociar al listener de red.

    • Seleccione Crear un Protocolo Nuevo para crear un nuevo protocolo con una configuración por defecto. Además, en la lista desplegable Servidor Virtual por Defecto, seleccione el servidor virtual que se va a asociar al protocolo.

      El nombre por defecto del protocolo es el nombre del listener de red con la terminación -protocol añadida, pero es posible editar este valor.

    • Seleccione Usar un Protocolo Existente y, a continuación, seleccione un protocolo de la lista desplegable.

      Si selecciona esta opción, por lo general habrá creado previamente el protocolo de este listener.

    Podrá especificar el protocolo únicamente si ha creado el listener de red. No es posible cambiar el protocolo asociado más adelante.

  7. Seleccione la casilla de verificación de activación de Estado para activar el listener de red.

    Esta opción se encuentra activada por defecto. Si el listener se encuentra desactivado, cualquier intento de establecer una conexión con él tiene como resultado una excepción de socket (java.net.ConnectException).

    En las versiones anteriores a la 9.1 de Servidor GlassFish, un listener cuyo atributo activado se hubiera definido como false devolvía un código de respuesta 404 para cualquier solicitud que se le enviaba. Para lograr este comportamiento en la versión actual de Servidor GlassFish, establezca el estado del listener en activado y defina cada estado de servidor virtual asociado como desactivado. Un servidor virtual incluye sus listeners asociados en la lista desplegable Listeners de Red.

  8. Seleccione la casilla de verificación Seguridad Activada para activar la seguridad para el protocolo que utiliza el listener de red.

    Esta opción se encuentra desactivada por defecto.

  9. Seleccione la casilla de verificación de activación de Listener JK para especificar que el listener admite el protocolo mod-jk de Apache.

    El protocolo mod-jk es un protocolo de red/comunicación de propiedad entre httpd de Apache (servidor web) y una instancia back-end de Servidor GlassFish. Si el listener admite este protocolo, se supone que httpd maneja todo el contenido estático, mientras que cualquier solicitud de recursos dinámicos (es decir, servlets y JSP) se direccionan al listener de red con el protocolo mod jk activado de Servidor GlassFish a través del protocolo mod-jk. Esta opción se encuentra desactivada por defecto.

  10. En el campo Puerto, introduzca el número de puerto en el que actuará el listener de red.

    Los valores válidos se encuentran entre 1 y 65535. En un sistema UNIX, la creación de sockets de escucha para los puertos 1-1024 requiere privilegios de superusuario.

  11. En el campo Dirección, escriba la dirección IP en la que actuará el listener de red.

    La dirección se puede especificar en par punteado o notación IPv6. Puede ser cualquiera (para INADDR/ANY) para recibir escuchas en todas las direcciones IP. Puede ser un nombre de host.

  12. En la lista desplegable Pool de Threads, seleccione el pool de threads que se debe asociar al listener de red.

    Normalmente, puede seleccionar uno de los dos pools de threads que están configurados cuando se instala Servidor GlassFish.

  13. En la lista desplegable Transporte, seleccione el tipo de transporte que se debe utilizar en el listener de red.

    Por defecto, la única alternativa es tcp.

  14. Haga clic en Aceptar.

Pasos Siguientes

Si ha activado la seguridad del listener de red, puede utilizar el separador SSL para editar la configuración de SSL del protocolo utilizado por el listener de red.
