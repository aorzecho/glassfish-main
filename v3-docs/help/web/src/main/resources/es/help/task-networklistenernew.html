Cómo crear una escucha de red

Antes de empezar

Cuando se crea una escucha de red, es posible crear simultáneamente un protocolo que utilizará la escucha de red o utilizar un protocolo que cree previamente. Normalmente, existe una relación de uno a uno entre una escucha de red y su protocolo asociado, pero no es necesario que sea así.

  1. En el árbol de navegación, amplíe el nodo Configuración.
  2. En el nodo Configuración, expanda el nodo Configuración de red.
  3. En el nodo Configuración de red, seleccione el nodo Escuchas de red.

    Se abrirá la página Escuchas de red.

  4. En la página Escuchas de red, haga clic en Nuevo.

    Se abrirá la página Nueva escucha de red.

  5. En el campo Nombre, escriba un nombre para la escucha de red.

    Un nombre de escucha de red no puede empezar con un número.

  6. En el campo Protocolo, especifique el protocolo que se va a asociar a la escucha de red.
    • Seleccione Crear un protocolo nuevo para crear un nuevo protocolo con una configuración predeterminada. Además, en la lista desplegable Servidor virtual predeterminado, seleccione el servidor virtual que se va a asociar al protocolo.

      El nombre predeterminado del protocolo es el nombre de la escucha de red con la terminación -protocol añadida, pero es posible editar este valor.

    • Seleccione Utilizar un protocolo existente y, a continuación, seleccione un protocolo de la lista desplegable.

      Si selecciona esta opción, por lo general habrá creado previamente el protocolo de esta escucha.

    Podrá especificar el protocolo únicamente si ha creado la escucha de red. No es posible cambiar el protocolo asociado más adelante.

  7. Seleccione la casilla de verificación Estado habilitado para habilitar la escucha de red.

    Esta opción se habilita por defecto. Si la escucha se encuentra deshabilitada, cualquier intento de establecer una conexión con ella tiene como resultado una excepción de socket (java.net.ConnectException).

    En las versiones anteriores a la 9.1 de Servidor GlassFish, una escucha cuyo atributo habilitado se hubiera definido como false devolvía un código de respuesta 404 para cualquier solicitud que se le enviaba. Para lograr este comportamiento en la versión actual de Servidor GlassFish, establezca el estado de la escucha en habilitado y defina cada estado de servidor virtual asociado como desactivado. Un servidor virtual incluye sus escuchas asociadas en la lista desplegable Escuchas de red.

  8. Seleccione la casilla de verificación Seguridad habilitada para habilitar la seguridad para el protocolo que utiliza la escucha de red.

    Esta opción se deshabilita por defecto.

  9. Seleccione la casilla de verificación de habilitación de la escucha JK para especificar que la escucha admite el protocolo mod-jk de Apache.

    El protocolo mod-jk es un protocolo de red/comunicación de propiedad entre httpd de Apache (servidor web) y una instancia back-end de Servidor GlassFish. Si la escucha admite este protocolo, se supone que httpd maneja todo el contenido estático, mientras que cualquier solicitud de recursos dinámicos (es decir, servlets y JSP) se direccionan a la escucha de red con el protocolo mod jk habilitado de Servidor GlassFish a través del protocolo mod-jk. Esta opción se deshabilita por defecto.

  10. En el campo Puerto, introduzca el número de puerto en el que se efectuará la escucha de red.

    Los valores válidos se encuentran entre 1 y 65535. En un sistema UNIX, la creación de sockets de escucha para puertos 1-1024 requiere privilegios de superusuario.

  11. En el campo Dirección, escriba la dirección IP en la que se efectuará la escucha de red.

    La dirección se puede especificar en par punteado o notación IPv6. Puede ser cualquiera (para INADDR_ANY) para recibir escuchas en todas las direcciones IP. Puede ser un nombre de host.

  12. En la lista desplegable Conjunto de subprocesos, seleccione el conjunto de subprocesos que se debe asociar a la escucha de red.

    Normalmente, puede seleccionar uno de las dos conjuntos de subprocesos que están configurados cuando se instala Servidor GlassFish.

  13. En la lista desplegable Transporte, seleccione el tipo de transporte que se debe utilizar en la escucha de red.

    De forma predeterminada, la única alternativa es tcp.

  14. Haga clic en Aceptar.

Pasos siguientes

Si ha habilitado la seguridad de la escucha de red, puede utilizar la ficha SSL para editar la configuración de SSL del protocolo utilizado por la escucha de red.
