執行緒池

使用 [執行緒池] 頁面可配置執行緒池。

使用執行緒池可將服務限制在特定數目的同步運作執行緒。

當您第一次安裝 GlassFish Server 時，預設已存在兩個執行緒池。名稱為 http-thread-pool 的執行緒池是為供網路偵聽程式使用而配置，而 thread-pool-1 是為供 ORB 針對 RMI/IIOP 請求使用而配置。您可能需要建立具有不同設定的執行緒池，以供特定種類應用程式和特定硬體系統使用，或視您的需要調校預設執行緒池。

Java 虛擬機器 (JVM) 可以支援一次執行多個執行緒。為了提昇效能，GlassFish Server 可維護一個或多個執行緒池。您可以將特定的執行緒池指定給特定的服務。

一個執行緒池可供多個服務使用。請求執行緒會處理使用者請求。伺服器收到請求時，它會將該請求指定給執行緒池中的空閑執行緒。執行緒會執行用戶端請求，並傳回結果。例如，如果請求需要使用目前被佔用的系統資源，則此執行緒將等待，直至此資源可用時，才允許請求使用此資源。

指定保留給請求的最大執行緒數和最小執行緒數。可以在這兩個值之間，動態調整執行緒池。指定的最小執行緒池大小，將要求伺服器配置至少這個數量的保留執行緒給請求。該數目可以增加到所指定的最大執行緒池大小。

增加程序可用的執行緒數目，可讓程序同時回應更多的請求。

對於每個執行緒池，均提供以下資訊。

執行緒池 ID

執行緒池的名稱。

最大執行緒池大小

執行緒池中執行緒的最大數目。

最小執行緒池大小

執行緒池中執行緒的最小數目。這些執行緒是在實例化執行緒池時建立的。

佇列最大大小

在有執行緒可處理訊息之前，可排入佇列的最大訊息數目。

閒置執行緒逾時

從池中移除閒置執行緒之前經過的時間 (以秒為單位)。

執行緒池表也包含下列選項。

新建

建立新執行緒池的按鈕。

刪除

刪除一或多個選取執行緒池的按鈕。
