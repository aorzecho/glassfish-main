建立傳輸

  1. 在瀏覽樹狀結構中，展開 「組態」 節點。

  2. 在 「組態」 節點下，展開 「網路組態」 節點。

  3. 在 「網路組態」 節點下，選取 「傳輸」 節點。

    此時將會開啟 「傳輸」 頁面。

  4. 在 「傳輸」 頁面上按一下 「新建」。

    「新建傳輸」 頁面會隨即開啟。

  5. 在 「名稱」 欄位中，為傳輸鍵入唯一的名稱。

  6. 在 「類別名稱」 欄位中，鍵入傳輸實作的類別名稱。

    預設值為 com.sun.grizzly.TCPSelectorHandler。

  7. 在 「選取金鑰處理程式」 欄位中，鍵入選取金鑰處理程式的類別名稱。

    選取金鑰處理程式是管理選取金鑰生命週期的抽象類別。如果您沒有指定值，則會使用預設的實作。

  8. 從 「ByteBuffer 類型」 下拉式清單中，選取要使用的 ByteBuffer 類型。

    可用的選擇有 HEAP 和 DIRECT。預設值為 HEAP。

  9. 在 「接收器執行緒」 欄位中，鍵入機器中的處理器數目。

    -1 值表示 Grizzly 將會根據機器中的處理器數目，計算接收器執行緒本身的數目。預設值為 1。

    若要設定請求處理執行緒的數目，請設定使用此傳輸之網路偵聽程式所用執行緒集區的 「最大執行緒集區大小」 值。

  10. 在 「最大連線數」 欄位中，鍵入使用此傳輸之網路偵聽程式上的最大擱置連線數目。

    預設值為 4096。

  11. 在 「緩衝區大小」 欄位中，鍵入針對使用此傳輸之網路偵聽程式所建立的輸入串流提供的緩衝區大小 (以位元組為單位)。

    預設值為 8192。

  12. 在 「閒置金鑰逾時」 欄位中，鍵入在閒置金鑰將取消且通道將關閉之前經過的秒數。

    預設值為 30。

  13. 在 「讀取逾時」 欄位中，鍵入 GlassFish Server 在讀取作業的標頭和內文剖析階段期間等待的毫秒數。

    預設值為 30,000。

  14. 在 「選擇器輪詢逾時」 欄位中，鍵入 NIO 選擇器將會停止等待事件 (使用者請求) 的毫秒數。

    預設值為 1000。

  15. 在 「寫入逾時」 欄位中，鍵入 GlassFish Server 在寫入回應時將遠端用戶端視為中斷連線前等待的毫秒數。

    預設值為 30,000。

  16. 選取「顯示組態」的「已啟用」核取方塊可將 Grizzly 的內部組態寫回伺服器記錄。

    此選項可提供有用的資訊以供除錯。此選項預設為停用。

  17. 選取「窺探」的「已啟用」核取方塊可將請求/回應資訊傾印到伺服器記錄。

    此選項可提供有用的資訊以供除錯，但會大幅降低效能。此選項預設為停用。

  18. 選取「TCP 不延遲」核取方塊可啟用 TCP_NODELAY (也稱為 Nagle 演算法)。

    此選項預設為啟用。

  19. 按一下 「確定」。
