編輯網路偵聽程式

使用 [編輯網路偵聽程式] 頁面可修改現有的網路偵聽程式。

當您編輯網路偵聽程式時，也可以編輯相關聯通訊協定的 [SSL]、[HTTP] 和 [檔案快取] 標籤。

[編輯網路偵聽程式] 頁面包含下列選項。

載入預設值

將所有設定復原為預設值的按鈕。

配置名稱

套用此頁面上之設定的配置名稱。此欄位是唯讀欄位。

名稱

唯一的偵聽程式名稱。[名稱] 欄位是唯讀欄位。您僅能在建立新的網路偵聽程式時指定名稱。

通訊協定

與網路偵聽程式關聯的通訊協定。[通訊協定] 欄位是唯讀欄位。您僅能在建立新的網路偵聽程式時建立或指定通訊協定。

狀態

網路偵聽程式的狀態。您可啟用或停用此偵聽程式。此選項預設為啟用。如果停用偵聽程式，嘗試連線至偵聽程式時皆會發生通訊端異常 (java.net.ConnectException)。

在 GlassFish Server 9.1 之前的版本中，若將偵聽程式已啟用的屬性設為 False，偵聽程式會針對任何傳送給它的請求傳回 404 回應碼。若要在目前的 GlassFish Server 版本中實現此運作方式，請將偵聽程式的狀態設為已啟用，並將每個關聯的虛擬伺服器狀態設為關閉。虛擬伺服器會在其 [網路偵聽程式] 下拉式清單中列出其關聯的偵聽程式。

Security

若選取此選項，則會針對網路偵聽程式所使用的通訊協定啟用安全性。此選項預設為停用。

JK 偵聽程式

若選取此選項，偵聽程式會支援 Apache 的 mod-jk 通訊協定。mod-jk 通訊協定是 Apache 的 httpd (Web 伺服器) 和 GlassFish Server 後端實例之間的專用通訊/網路通訊協定。如果偵聽程式支援此通訊協定，httpd 應會處理所有靜態內容，而任何對動態資源 (亦即 Servlet 和 JSP) 的請求都會使用 mod-jk 通訊協定路由至 GlassFish Server 已啟用 mod-jk 的網路偵聽程式。此選項預設為停用。

連接埠

偵聽程式將偵聽的連接埠號碼。合法值為 1 - 65535。在 UNIX 系統上，建立偵聽連接埠 1 - 1024 的通訊端需要超級使用者權限。

位址

網路偵聽程式將偵聽的 IP 位址。此位址可以用「點對」或 IPv6 表示法。它可以表示為 any (用於 INADDR_ANY) 以偵聽所有 IP 位址。它可以是主機名稱。

執行緒池

與網路偵聽程式關聯的執行緒池。

傳輸

網路偵聽程式的傳輸類型。依預設，唯一的傳輸類型是 tcp。
