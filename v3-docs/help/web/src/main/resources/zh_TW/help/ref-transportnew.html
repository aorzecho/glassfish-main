新建傳輸

使用 [新建傳輸] 頁面可建立新的傳輸。

除預設的 tcp 傳輸之外，需要其他傳輸的情形並不常見。

[新建傳輸] 頁面包含下列選項。

配置名稱

套用此頁面上之設定的配置名稱。此欄位是唯讀欄位。

傳輸名稱

傳輸的唯一名稱。

類別名稱

傳輸實作的類別名稱。預設值為 com.sun.grizzly.TCPSelectorHandler。

選取金鑰處理程式

選取金鑰處理程式的類別名稱。選取金鑰處理程式是管理選取金鑰生命週期的抽象類別。 如果您沒有指定值，則會使用預設的實作。

ByteBuffer 類型

要使用的 ByteBuffer 類型。可用的選擇有 HEAP 和 DIRECT。預設值為 HEAP。

接收器執行緒

機器中的處理器數目。–1 值表示 Grizzly 將會根據機器中的處理器數目，計算接收器執行緒本身的數目。預設值為 1。

若要設定請求處理執行緒的數目，請設定使用此傳輸之網路偵聽程式所用執行緒池的 [最大執行緒池大小] 值。

最大連線數

使用此傳輸之網路偵聽程式上的最大擱置連線數目。預設值為 4096。

緩衝區大小

針對使用此傳輸之網路偵聽程式所建立的輸入串流提供的緩衝區大小 (以位元組為單位)。預設值為 8192。

閒置金鑰逾時

閒置金鑰將取消且通道將關閉之前經過的秒數。預設值為 30。

讀取逾時

GlassFish Server 在讀取作業的標頭和內文剖析階段期間等待的毫秒數。預設值為 30,000。

選擇器輪詢逾時

NIO 選擇器將會封鎖以等待事件 (使用者請求) 的毫秒數。預設值為 1000。

寫入逾時

GlassFish Server 在寫入回應時將遠端用戶端視為中斷連線前等待的毫秒數。預設值為 30,000。

顯示配置

若選取此選項，Grizzly 的內部配置會寫回伺服器記錄。此選項可提供有用的資訊以供除錯。此選項預設為停用。

探測

若選取此選項，請求/回應資訊會傾印到伺服器記錄。此選項可提供有用的資訊以供除錯，但會大幅降低效能。此選項預設為停用。

TCP 無延遲

若選取此選項，會啟用 TCP_NODELAY (也稱為 Nagle 演算法)。此選項預設為啟用。
