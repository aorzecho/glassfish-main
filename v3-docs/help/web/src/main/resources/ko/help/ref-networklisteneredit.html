네트워크 수신기 편집

네트워크 수신기 편집 페이지에서는 기존 네트워크 수신기를 수정할 수 있습니다.

네트워크 수신기를 편집할 때, 연결된 프로토콜에 대한 SSL, HTTP 및 파일 캐시 탭을 함께 사용하여 편집할 수 있습니다.

네트워크 수신기 편집 페이지에는 다음 옵션이 포함되어 있습니다.

기본값 로드

기본값이 있는 설정을 해당 기본값으로 복원하는 버튼입니다. 기본값이 없는 설정은 변경되지 않습니다.

구성 이름

이 페이지의 설정을 적용할 구성의 이름입니다. 이 필드는 읽기 전용입니다.

이름

고유한 수신기 이름입니다. 이름 필드는 읽기 전용 필드입니다. 새 네트워크 수신기를 만들 때만 이름을 지정할 수 있습니다.

프로토콜

네트워크 수신기에 연관된 프로토콜입니다. 프로토콜 필드는 읽기 전용 필드입니다. 새 네트워크 수신기를 만들 때만 프로토콜을 만들거나 지정할 수 있습니다.

상태

네트워크 수신기의 상태입니다. 수신기는 사용 가능하거나 사용 가능하지 않을 수 있습니다. 이 옵션은 기본적으로 활성화됩니다. 수신기가 비활성화된 상태에서 수신기에 연결하려고 하면 소켓 예외(java.net.ConnectException)가 발생합니다.

GlassFish Server 9.1 이전 버전에서는 활성화된 속성이 false로 설정된 수신기는 해당 수신기로 전송된 모든 요청에 대해 404 응답 코드를 반환했습니다. 현재 GlassFish Server 버전에서 이 동작을 수행하려면 수신기의 상태를 사용 가능으로 설정하고 연결된 모든 가상 서버의 상태를 OFF로 설정합니다. 가상 서버의 네트워크 수신기 드롭다운 목록에는 해당 가상 서버와 연결된 수신기가 나열됩니다.

Security

이 옵션을 선택하면 네트워크 수신기에서 사용하는 프로토콜에 대해 보안이 활성화됩니다. 이 옵션은 기본적으로 비활성화됩니다.

JK 수신기

이 옵션을 선택하면 수신기에서 Apache의 mod-jk 프로토콜을 지원합니다. mod-jk 프로토콜은 Apache의 httpd(웹 서버)와 GlassFish Server 백엔드 인스턴스 사이의 소유 통신/네트워크 프로토콜입니다. 수신기에서 이 프로토콜을 지원하는 경우 httpd에서 모든 정적 내용을 처리하는 것으로 간주되지만, 동적 자원(즉, 서블릿 및 JSP)에 대한 요청은 mod-jk 프로토콜을 통해 GlassFish Server의 mod-jk 사용 네트워크 수신기에 라우팅됩니다. 이 옵션은 기본적으로 비활성화됩니다.

포트

수신기에서 수신할 포트 번호입니다. 유효한 값은 1 - 65535입니다. UNIX 시스템에서 포트 1 - 1024를 수신하는 소켓을 만들려면 수퍼유저 권한이 필요합니다.

주소

네트워크 수신기에서 수신할 IP 주소입니다. 주소는 점으로 된 쌍이거나 IPv6 표기 형식일 수 있습니다. 모든 IP 주소를 수신하려면 any(INADDR_ANY의 경우)일 수 있습니다. 호스트 이름일 수 있습니다.

스레드 풀

네트워크 수신기에 연관된 스레드 풀입니다.

전송

네트워크 수신기의 전송 유형입니다. 기본적으로 유일한 전송은 tcp입니다.
