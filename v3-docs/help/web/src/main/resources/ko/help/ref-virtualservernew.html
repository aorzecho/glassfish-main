새 가상 서버

새 가상 서버 페이지에서는 가상 서버를 만들 수 있습니다.

새 가상 서버 페이지에는 다음 옵션이 포함되어 있습니다.

구성 이름

이 페이지의 설정이 적용되는 구성의 이름입니다. 이 필드는 읽기 전용입니다.

아이디

내부적으로 표시되는 가상 서버 식별자입니다. HTTP 클라이언트에는 노출되지 않습니다. HTTP 클라이언트에 노출되는 호스트 이름은 호스트 필드에서 지정해야 합니다.

호스트

서버가 실행되고 있는 시스템의 호스트 이름입니다. 네트워크의 DNS 서버(UNIX 시스템에서는 /etc/hosts 파일)에 등록된 실제 또는 가상 호스트 이름을 사용합니다. 기본값은 시스템 등록 정보 값 ${com.sun.aas.hostName}입니다.

상태

가상 서버에 대해 원하는 상태입니다. 다음과 같은 값을 사용할 수 있습니다.

켜기

가상 서버가 활성화됩니다. 이 값이 기본값입니다.

끄기

가상 서버가 비활성화됩니다. 서버에 액세스하려 하면 오류 코드 404(사용 가능한 자원 없음)를 반환합니다.

사용 불가

가상 서버를 사용하지 않습니다. 서버에 액세스하려 하면 오류 코드 403(요청 수행 거부)을 반환합니다.

SSO

SSO(단일 사인 온) 동작이 HTTP 서비스에서 상속되는지 여부(사용/사용 안 함)를 지정합니다. 기본적으로 로깅은 HTTP 서비스에서 상속됩니다. 이 옵션을 선택하면 동일한 영역에 대해 구성된 이 가상 서버의 웹 응용 프로그램에 대해 SSO(단일 사인 온)가 활성화됩니다. 이 옵션을 비활성화하면 이 가상 서버에 대해 SSO(단일 사인 온)가 비활성화되고 사용자는 가상 서버의 모든 응용 프로그램에 개별적으로 인증해야 합니다.

네트워크 수신기

이 서버와 연결된 네트워크 수신기(있는 경우)입니다.

기본 웹 모듈

가상 서버에 배포된 다른 웹 모듈에 매핑될 수 없는 모든 요청에 응답할 배포된 웹 모듈(있는 경우)입니다. 기본 웹 모듈이 지정되지 않은 경우 빈 컨텍스트 루트가 있는 웹 모듈이 사용됩니다. 빈 컨텍스트 루트가 있는 웹 모듈이 없을 때는 시스템 기본 웹 모듈이 만들어져 사용됩니다.

로그 파일

이 가상 서버로부터의 로깅 메시지가 나타날 파일의 경로 이름입니다. 기본적으로 로깅 메시지는 domain-dir/logs/server.log에 나타납니다.

Docroot

서버에 대한 루트 문서 디렉토리의 절대 경로입니다. 기본값은 domain-dir/docroot입니다.

액세스 로깅

액세스 로깅이 HTTP 서비스에서 상속되는지 여부(사용/사용 안 함)를 지정합니다. 기본적으로 로깅은 HTTP 서비스에서 상속됩니다.

디렉토리

서버 액세스 로그의 절대 디렉토리 경로입니다. 기본값은 domain-dir/logs/access입니다.

추가 등록 정보

가상 서버에 대한 추가 등록 정보입니다.

사용할 수 있는 등록 정보에 대한 설명은 가상 서버와 관련된 등록 정보를 참조하십시오.
