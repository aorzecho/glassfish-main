새 프로토콜

[새 프로토콜] 페이지에서는 새 네트워크 프로토콜을 생성할 수 있습니다.

프로토콜을 생성한 후 일반적으로 동일한 이름으로 네트워크 리스너를 생성하고 네트워크 리스너와 프로토콜을 연결합니다. 또는 네트워크 리스너를 생성하면 동시에 기본 설정을 갖는 프로토콜이 자동으로 생성됩니다.

[새 프로토콜] 페이지에는 다음 옵션이 포함되어 있습니다.

구성 이름

이 페이지의 설정이 적용되는 구성의 이름입니다. 이 필드는 읽기 전용입니다.

이름

프로토콜의 고유한 이름입니다. 프로토콜 이름은 숫자로 시작할 수 없습니다.

보안

이 옵션을 선택하면 프로토콜에 대해 보안이 사용으로 설정됩니다. 이 옵션은 기본적으로 사용 안함으로 설정되어 있습니다. 보안을 사용으로 설정하면 [프로토콜 편집] 페이지의 SSL 탭을 사용하여 보안 설정을 수정할 수 있습니다.

상태

이 옵션을 선택하면 파일 캐시가 사용으로 설정됩니다. 이 옵션은 기본적으로 사용으로 설정되어 있습니다.

최대 사용 기간

적합한 캐시 항목에 대한 최대 사용 기간(초)입니다.

이 매개변수는 파일이 캐시된 후 캐시된 정보가 사용되는 기간을 제어합니다. 최대 사용 기간보다 오래된 항목은 같은 파일에 대한 새 항목으로 대체됩니다. 내용이 자주 변경되지 않는 경우에는 성능 향상을 위해 이 값을 늘리십시오.

최적의 최대 사용 기간은 기존 파일이 정기적으로 수정되는지 여부에 따라 다릅니다. 예를 들어 파일이 하루에 네 번 정기적인 간격으로 수정되는 경우 최대 사용 기간을 21600초(6시간)로 설정하는 것이 좋습니다. 그렇지 않은 경우에는 파일이 수정된 후 파일의 이전 버전을 서비스하려고 하는 가장 긴 시간으로 최대 사용 기간을 설정하는 것을 고려해 보십시오.

기본값은 30입니다.

최대 캐시 크기

디스크에 있는 파일 캐시의 최대 전체 크기(바이트)입니다. 기본값은 10485760입니다.

최대 파일 수

파일 캐시에 저장할 수 있는 파일의 최대 수입니다.

값이 너무 크면 서버가 자주 사용되지 않은 파일까지 캐시하므로 메모리가 낭비됩니다. 값이 너무 작으면 캐싱의 장점이 사라질 수 있습니다. 이 속성에 여러 값을 테스트하여 특정 응용 프로그램에 맞는 최적의 솔루션을 찾으십시오.

기본값은 1024입니다.

서버 이름

서버가 클라이언트에 전송하는 URL에 사용될 호스트 이름입니다. 서버가 별칭을 사용하는 경우 이 이름은 별칭 이름입니다. 서버가 별칭을 사용하지 않는 경우에는 이 필드를 비워 두십시오.

이 값은 서버에서 자동으로 생성하는 URL에 영향을 주지만, 서버에 저장된 디렉토리 및 파일의 URL에는 영향을 주지 않습니다. 서버에서 별칭을 사용하는 경우 서버 이름이 별칭 이름이어야 합니다. 콜론과 포트 번호를 추가하면 서버에서 클라이언트에 전송하는 URL에 해당 포트가 사용됩니다.

기본 가상 서버

이 프로토콜과 연관된 가상 서버입니다. [가상 서버] 페이지에서는 가상 서버를 정의할 수 있습니다.

포트 재지정

요청을 다른 포트로 재지정하는 포트 값입니다. 다음 두 조건이 존재하면 GlassFish Server에서 자동으로 요청을 재지정합니다.

  • 이 프로토콜을 참조하는 네트워크 리스너는 비SSL 요청을 지원합니다.

  • SSL 전송이 필요한 일치하는 보안 제약 조건에 대한 요청이 수신된 경우

재지정 포트가 지정되지 않은 경우에는 GlassFish Server에서 원래 요청에 지정된 포트 번호를 사용합니다. 기본적으로 포트가 지정되지 않습니다.

최대 접속 수

서버에서 접속을 닫을 때까지 파이프라인될 수 있는 최대 요청 수를 지정합니다. 접속 유지 하위 시스템은 정기적으로 유휴 접속을 폴링합니다. 기본값은 256입니다.

HTTP/1.0 접속 유지와 HTTP/1.1 접속 유지 및 파이프라인 처리를 사용 안함으로 설정하려면 이 속성을 1로 설정합니다. 값이 0이면 요청이 항상 거부됩니다. 값이 –1이면 접속 유지 접속 수에 제한이 없습니다.

시간 초과

접속이 유휴 상태로 간주되고 접속 유지 상태로 유지되는 최대 시간(초)입니다. 값이 0보다 작거나 같으면 접속 유지 접속이 무기한 열린 상태로 유지됩니다. 기본값은 30초입니다.

업로드 시간 초과

이 옵션을 선택하면 접속 업로드 시간 초과 제한에 도달한 후에 바이트를 느리게 읽는 서블릿에 대한 접속이 닫힙니다. 이 옵션을 사용 안함으로 설정하면 서블릿 접속이 시간 초과되지 않습니다. 이 옵션은 기본적으로 사용 안함으로 설정되어 있습니다.

접속 업로드 시간 초과

업로드에 대한 시간 초과(밀리초)입니다. 이 필드는 업로드 시간 초과 사용 체크 박스를 선택한 경우에만 적용될 수 있습니다. 기본값은 300000밀리초입니다.

요청 시간 초과

요청이 시간 초과되기 전의 시간(초)입니다. 시간 초과 값에 도달하기 전에 요청이 처리되지 않으면 요청이 무시됩니다. 기본값은 30초입니다.

전송 버퍼 크기

전송 버퍼의 크기(바이트)입니다. 기본값은 8192바이트입니다.

헤더 버퍼 길이

요청 처리 스레드가 요청 데이터를 읽기 위해 사용하는 버퍼의 크기(바이트)입니다. 기본값은 8192바이트입니다.

최대 게시 크기

POST 작업의 최대 크기(바이트)입니다. 기본값은 2097152바이트입니다.

URI 인코딩

수신된 요청 URI를 디코딩하는 데 사용되는 문자 집합의 이름입니다. 값은 적합한 IANA 문자 집합 이름이어야 합니다. 기본값은 UTF-8입니다.

버전

사용되는 HTTP 프로토콜의 버전입니다. 기본값은 HTTP/1.1입니다.

압축

HTTP/1.1 GZIP 압축 사용을 지정하여 서버 대역폭을 저장합니다. 사용 가능한 선택 항목은 다음과 같습니다.

on

데이터를 압축합니다.

off

압축을 사용하지 않습니다.

force

모든 경우 데이터 압축을 적용합니다.

기본값은 off입니다.

압축 가능한 MIME 유형

HTTP 압축이 사용되는 MIME 유형의 콤마로 구분된 목록입니다. 기본값은 text/html,text/xml,text/plain입니다.

압축 최소 크기

압축이 적용될 때 파일의 최대 크기입니다. 압축이 on 또는 force로 설정되어 있으면 이 값을 설정해야 합니다. 기본값은 2048바이트입니다.

비압축 사용자 에이전트

압축을 사용해서는 안되는 HTTP 클라이언트의 사용자 에이전트와 일치하는 정규 표현식의 콤마로 구분된 목록입니다. 기본적으로 이 값은 빈 문자열입니다.

제한된 사용자 에이전트

HTTP 압축이 적용되는 제한된 사용자 에이전트 목록입니다. 사용자 에이전트를 지정하지 않으면 모든 사용자 에이전트에 HTTP 압축이 적용됩니다. 기본적으로 사용자 에이전트가 지정되지 않습니다.

기본 응답 유형

기본 응답 유형을 지정하는 문자열입니다. 형식은 내용 유형, 인코딩, 언어 및 문자 집합이 세미콜론으로 구분된 문자열입니다. 기본값은 text/html; charset=iso-8859-1입니다.

강제 응답 유형

파일 확장명과 일치하는 MIME 매핑을 사용할 수 없는 경우 사용되는 요청 유형을 지정하는 문자열입니다. 형식은 내용 유형, 인코딩, 언어 및 문자 집합이 세미콜론으로 구분된 문자열입니다. 기본값은 text/html; charset=iso-8859-1입니다.

어댑터

정적 리소스 어댑터의 클래스 이름입니다. 기본값은 com.sun.grizzly.tcp.StaticResourcesAdapter입니다.

Comet 지원

이 옵션을 선택하면 이 프로토콜을 참조하는 네트워크 리스너에 대해 Comet 지원이 사용으로 설정됩니다. 기본적으로 이 옵션은 사용 안함으로 설정됩니다 서블릿 또는 JSP 페이지에서 Comet 기술을 사용하는 경우 web.xml 파일에 load-on-startup 요소를 추가하여 GlassFish Server이(가) 시작될 때 Comet 기술이 초기화되도록 합니다. 예를 들면 다음과 같습니다.

<servlet>
<servlet-name>CheckIn</servlet-name>
<servlet-class>CheckInServlet</servlet-class>
<load-on-startup>0</load-on-startup>
</servlet>
DNS 조회

이 옵션을 선택하면 DNS 조회가 사용으로 설정됩니다. 이 옵션은 기본적으로 사용 안함으로 설정되어 있습니다.

DNS 조회

이 옵션을 선택하면 DNS 조회가 사용으로 설정됩니다. 이 옵션은 기본적으로 사용 안함으로 설정되어 있습니다.

RCM 지원

이 옵션을 선택하면 RCM(Resource Configuration Management)이 사용으로 설정됩니다. 이 옵션은 기본적으로 사용 안함으로 설정되어 있습니다.

추적

이 옵션을 선택하면 TRACE 작업이 사용으로 설정됩니다. 이 옵션이 사용 안함으로 설정된 경우 GlassFish Server에서 사이트 간 스크립팅 공격을 받을 가능성이 낮아집니다. 이 옵션은 기본적으로 사용으로 설정되어 있습니다.

인증 통과

이 옵션을 선택하면 이 프로토콜을 사용하는 네트워크 리스너가 SSL 종료 프록시 서버에서 트래픽을 수신합니다. 이 옵션은 기본적으로 사용 안함으로 설정되어 있습니다.

청크

이 옵션을 선택하면 HTTP 응답 청크가 사용으로 설정됩니다. 이 옵션은 기본적으로 사용으로 설정되어 있습니다.

XPowered By

이 옵션을 선택하면 Java Servlet 3.0 및 Java Server Pages 2.0 사양에 따라 X-Powered-By 헤더가 사용됩니다. 이 옵션은 기본적으로 사용으로 설정되어 있습니다.

인코딩 슬래시

이 옵션을 선택하면 URI에 인코딩 슬래시가 포함될 수 있습니다. 이 옵션은 기본적으로 사용 안함으로 설정되어 있습니다.

웹 소켓 지원

이 옵션을 선택하면 웹 소켓 프로토콜이 지원됩니다. 이 옵션은 기본적으로 사용 안함으로 설정되어 있습니다.
