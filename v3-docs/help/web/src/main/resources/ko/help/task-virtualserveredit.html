가상 서버를 편집하는 방법

  1. 탐색 트리에서 구성 노드를 선택합니다.
  2. 구성 노드에서 가상 서버 노드를 선택합니다.

    가상 서버 페이지가 열립니다.

  3. 가상 서버 페이지에서 편집할 가상 서버의 이름을 누릅니다.

    가상 서버 편집 페이지가 열립니다.

  4. 호스트 필드에 서버가 실행되고 있는 시스템의 호스트 이름 또는 IP 주소를 입력합니다.

    네트워크의 DNS 서버(UNIX 시스템에서는 /etc/hosts 파일)에 등록된 실제 또는 가상 호스트 이름을 사용합니다. 기본값은 시스템 등록 정보 값 ${com.sun.aas.hostName}입니다.

  5. 가상 서버에 대한 상태 옵션을 선택합니다.

    다음과 같은 값을 사용할 수 있습니다.

    켜기

    가상 서버가 활성화됩니다. 이 값이 기본값입니다.

    끄기

    가상 서버가 비활성화됩니다. 서버에 액세스하려 하면 오류 코드 404(사용 가능한 자원 없음)를 반환합니다.

    사용 불가

    가상 서버를 사용하지 않습니다. 서버에 액세스하려 하면 오류 코드 403(요청 수행 거부)을 반환합니다.

  6. 새 가상 서버에 대한 SSO 옵션을 선택합니다.

    기본적으로 가상 서버는 HTTP 서비스의 SSO(단일 사인 온) 설정을 상속합니다. 사용 또는 사용 안 함을 선택하여 이 가상 서버에 대한 SSO(단일 사인 온)를 활성화하거나 비활성화함으로써 HTTP 서비스 설정을 대체합니다.

    GlassFish 서버의 응용 프로그램에 대한 단일 사인 온은 서블릿 및 JSP 파일에서 지원됩니다. 이 기능을 통해 사용자가 응용 프로그램마다 따로 사인 온하는 것이 아니라 같은 사용자 사인 온 정보가 필요한 여러 응용 프로그램이 이 정보를 공유할 수 있습니다.

  7. 네트워크 수신기 드롭다운 목록에서 하나 이상의 네트워크 수신기를 선택하거나 필드를 비워 둡니다.
  8. 기본 웹 모듈 드롭다운 목록에서 가상 서버에 배포된 다른 웹 모듈에 매핑할 수 없는 모든 요청에 응답할 모듈을 선택합니다.

    기본 웹 모듈이 지정되지 않은 경우 빈 컨텍스트 루트가 있는 웹 모듈이 사용됩니다. 빈 컨텍스트 루트가 있는 웹 모듈이 없을 때는 시스템 기본 웹 모듈이 만들어져 사용됩니다.

  9. 로그 파일 필드에 이 가상 서버로부터의 로깅 메시지가 나타날 파일의 경로 이름을 입력합니다.

    기본적으로 로깅 메시지는 기본 서버 로그 domain-dir/logs/server.log로 전송됩니다.

  10. Docroot 필드에 서버의 루트 문서 디렉토리에 대한 절대 경로를 입력합니다.

    기본값은 domain-dir/docroot입니다.

  11. 액세스 로깅을 활성화하거나 비활성화하려면 액세스 로깅 옵션을 선택합니다.

    기본적으로 가상 서버는 HTTP 서비스의 액세스 로깅 설정을 상속합니다. 사용 또는 사용 안 함을 선택하여 이 가상 서버에 대한 액세스 로깅을 활성화하거나 비활성화함으로써 HTTP 서비스 설정을 대체합니다.

  12. 디렉토리 필드에 서버 액세스 로그의 절대 디렉토리 경로를 입력합니다.

    기본값은 domain-dir/logs/access입니다.

  13. (옵션) 추가 등록 정보 섹션에 가상 서버에 대한 추가 등록 정보를 지정합니다.

    사용할 수 있는 등록 정보에 대한 설명은 가상 서버와 관련된 등록 정보를 참조하십시오.

    • 등록 정보를 추가하려면 등록 정보 추가 버튼을 누릅니다. 빈 행이 표시되면 이름 필드에 등록 정보 이름을 입력하고 값 필드에 등록 정보 값을 입력합니다.
    • 등록 정보를 수정하려면 등록 정보의 값 필드를 편집합니다.
    • 등록 정보를 삭제하려면 삭제할 등록 정보의 이름 필드 왼쪽에 있는 확인란을 선택한 다음 등록 정보 삭제 버튼을 누릅니다.
  14. 저장을 누릅니다.

    참고 - 기본 설정으로 돌아가려면 기본값 로드 버튼을 누른 다음 저장을 누릅니다.
