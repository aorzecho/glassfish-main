네트워크 수신기를 편집하는 방법

  1. 탐색 트리에서 구성 노드를 선택합니다.
  2. 구성 노드에서 네트워크 구성 노드를 확장합니다.
  3. 네트워크 구성 노드에서 네트워크 수신기 노드를 선택합니다.

    네트워크 수신기 페이지가 열립니다.

  4. 네트워크 수신기 페이지에서 편집할 네트워크 수신기의 이름을 누릅니다.

    네트워크 수신기 편집 페이지가 열립니다.

  5. 네트워크 수신기의 상태를 변경하려면 상태 사용 가능 확인란을 선택 또는 선택 해제합니다.

    수신기는 사용 가능하거나 사용 가능하지 않을 수 있습니다. 이 옵션은 기본적으로 활성화됩니다. 수신기가 비활성화된 상태에서 수신기에 연결하려고 하면 소켓 예외(java.net.ConnectException)가 발생합니다.

  6. 보안 활성화 확인란을 선택하여 네트워크 수신기에서 사용하는 프로토콜에 대한 보안을 활성화합니다.

    이 옵션은 기본적으로 비활성화됩니다.

  7. JK 수신기 사용 확인란을 선택하여 수신기가 Apache의 mod-jk 프로토콜을 지원하도록 지정합니다.

    mod-jk 프로토콜은 Apache의 httpd(웹 서버)와 GlassFish 서버 백엔드 인스턴스 사이의 소유 통신/네트워크 프로토콜입니다. 수신기에서 이 프로토콜을 지원하는 경우 httpd에서 모든 정적 내용을 처리하는 것으로 간주되지만, 동적 자원(즉, 서블릿 및 JSP)에 대한 요청은 mod-jk 프로토콜을 통해 GlassFish 서버의 mod-jk 사용 네트워크 수신기에 라우팅됩니다. 이 옵션은 기본적으로 비활성화됩니다.

  8. 포트 필드에 네트워크 수신기에서 수신할 포트 번호를 입력합니다.

    유효한 값은 1 - 65535입니다. UNIX 시스템에서 포트 1 - 1024를 수신하는 소켓을 만들려면 수퍼유저 권한이 필요합니다.

  9. 주소 필드에 네트워크 수신기에서 수신할 IP 주소를 입력합니다.

    주소는 점으로 된 쌍이거나 IPv6 표기 형식일 수 있습니다. 모든 IP 주소를 수신하려면 any(INADDR_ANY의 경우)일 수 있습니다. 호스트 이름일 수 있습니다.

  10. 스레드 풀 드롭다운 목록에서 네트워크 수신기와 연결할 스레드 풀을 선택합니다.
  11. 저장을 누릅니다.

    참고 - 기본 설정으로 돌아가려면 기본값 로드 버튼을 누른 다음 저장을 누릅니다.

다음 순서

SSL, HTTP 및 파일 캐시 탭을 사용하여 네트워크 수신기와 연결된 프로토콜에 대한 설정을 편집할 수 있습니다. SSL 탭은 네트워크 수신기에 대한 보안을 활성화한 경우에만 사용할 수 있습니다.
