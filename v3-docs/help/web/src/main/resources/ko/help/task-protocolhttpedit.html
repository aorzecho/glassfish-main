프로토콜에 대한 HTTP 설정을 편집하는 방법

  1. 탐색 트리에서 구성 노드를 선택합니다.
  2. 구성 노드에서 네트워크 구성 노드를 확장합니다.
  3. 네트워크 구성 노드에서 프로토콜 노드를 선택합니다.

    프로토콜 페이지가 열립니다.

  4. 프로토콜 페이지에서 편집할 프로토콜의 이름을 누릅니다.

    프로토콜 편집 페이지가 열립니다.

  5. 프로토콜 편집 페이지에서 HTTP 탭을 누릅니다.

    HTTP 페이지가 열립니다.

  6. 서버가 클라이언트에 보내는 URL에 사용할 호스트 이름을 서버 이름 필드에 입력합니다.

    서버가 별칭을 사용하는 경우 이 이름은 별칭 이름입니다. 서버가 별칭을 사용하지 않는 경우에는 이 필드를 비워 두십시오.

  7. 기본 가상 서버 드롭다운 목록에서 프로토콜과 연관될 가상 서버를 선택합니다.

    가상 서버 페이지에서는 가상 서버를 정의할 수 있습니다.

  8. 요청을 다른 포트로 리디렉션할 때 사용할 포트 값을 포트 리디렉션 필드에 입력합니다.

    다음 두 조건이 존재하면 GlassFish 서버는 자동으로 요청을 리디렉션합니다.

    • 이 프로토콜이 비-SSL 요청을 지원하는 경우

    • SSL 전송이 필요한 일치하는 보안 제약 조건에 대한 요청이 수신된 경우

    리디렉션 포트가 지정되지 않은 경우에는 GlassFish 서버에서 원래 요청에 지정된 포트 번호를 사용합니다.

  9. 최대 연결 수 필드에 연결 유지 모드에서 유지해야 할 영구 연결의 최대 수를 입력합니다.

    연결 유지 하위 시스템은 정기적으로 유휴 연결을 폴링합니다.

  10. 시간 초과 필드에 연결이 유휴 상태로 간주되고 연결 유지 상태로 유지되는 최대 시간(초)을 입력합니다.

    값이 0보다 작거나 같으면 연결 유지 연결이 무기한 열린 상태로 유지됩니다. 기본값은 30초입니다.

  11. 업로드 시간 초과 사용 확인란을 선택하여 연결 업로드 시간 초과 제한에 도달한 후에 바이트를 느리게 읽는 서블릿에 대한 연결이 닫히도록 합니다.

    이 옵션을 비활성화하면 서블릿 연결이 시간 초과되지 않습니다. 이 옵션은 기본적으로 비활성화됩니다.

  12. 연결 업로드 시간 초과 필드에 업로드 시간 초과(밀리초)를 입력합니다.

    이 필드는 업로드 시간 초과 사용 안 함 확인란을 선택한 경우에만 적용될 수 있습니다. 기본값은 300000밀리초입니다.

  13. 요청 시간 초과 필드에 요청 시간이 초과되기 전까지의 시간(초)을 입력합니다.

    시간 초과 값에 도달하기 전에 요청이 처리되지 않으면 요청이 무시됩니다. 기본값은 30초입니다.

  14. 전송 버퍼 크기 필드에 전송 버퍼의 크기(바이트)를 입력합니다.

    기본값은 8192바이트입니다.

  15. 헤더 버퍼 길이 필드에 요청 처리 스레드에서 요청 데이터를 읽을 때 사용할 버퍼의 크기(바이트)를 입력합니다.

    기본값은 8192바이트입니다.

  16. 최대 게시 크기 필드에 POST 작업의 최대 크기(바이트)를 입력합니다.

    기본값은 2097152바이트입니다.

  17. URI 인코딩 필드에 수신된 요청 URI를 디코딩하는 데 사용되는 문자 집합의 이름을 입력합니다.

    값은 유효한 IANA 문자 집합 이름이어야 합니다. 기본값은 UTF-8입니다.

  18. 버전 필드에 사용할 HTTP 프로토콜의 버전을 입력합니다.

    기본값은 HTTP/1.1입니다.

  19. 압축 드롭다운 목록에서 HTTP/1.1 GZIP 압축 사용을 선택하여 서버 대역폭을 저장합니다.

    사용 가능한 선택 항목은 다음과 같습니다.

    on

    데이터를 압축합니다.

    off

    압축을 사용하지 않습니다.

    force

    모든 경우 데이터 압축을 적용합니다.

    기본값은 off입니다.

  20. 압축 가능한 MIME 유형 필드에 HTTP 압축이 사용되는 MIME 유형의 쉼표로 구분된 목록을 입력합니다.

    기본값은 text/html,text/xml,text/plain입니다.

  21. 압축 최소 크기 필드에 압축이 적용될 때 파일의 최소 크기를 입력합니다.

    압축이 on 또는 force 로 설정되어 있으면 이 값을 설정해야 합니다. 기본값은 2048바이트입니다.

  22. 비압축 사용자 에이전트 필드에 압축을 사용해서는 안 되는 HTTP 클라이언트의 사용자 에이전트와 일치하는 정규 표현식의 쉼표로 구분된 목록을 입력합니다.

    기본적으로 이 값은 빈 문자열입니다.

  23. 제한된 사용자 에이전트 필드에 HTTP 압축이 적용되는 제한된 사용자 에이전트의 목록을 입력합니다.

    사용자 에이전트를 지정하지 않으면 모든 사용자 에이전트에 HTTP 압축이 적용됩니다. 기본적으로 사용자 에이전트가 지정되지 않습니다.

  24. 기본 응답 유형 필드에 기본 응답 유형을 지정하는 문자열을 입력합니다.

    형식은 내용 유형, 인코딩, 언어 및 문자 집합이 세미콜론으로 구분된 문자열입니다. 기본값은 text/html; charset=iso-8859-1입니다.

  25. 강제 응답 유형 필드에 파일 확장명과 일치하는 MIME 매핑을 사용할 수 없는 경우 사용되는 요청 유형을 지정하는 문자열을 입력합니다.

    형식은 내용 유형, 인코딩, 언어 및 문자 집합이 세미콜론으로 구분된 문자열입니다. 기본값은 text/html; charset=iso-8859-1입니다.

  26. 어댑터 필드에 정적 자원 어댑터의 클래스 이름을 입력합니다.

    기본값은 com.sun.grizzly.tcp.StaticResourcesAdapter입니다.

  27. 프로토콜에 대해 Comet 지원을 사용하려면 Comet 지원 사용 가능 확인란을 선택합니다.

    이 옵션은 기본적으로 비활성화됩니다.

  28. DNS 조회 사용 확인란을 선택하여 DNS(Domain Name System) 조회를 활성화합니다.

    이 옵션은 기본적으로 비활성화됩니다.

  29. RCM 지원 사용 확인란을 선택하여 RCM(Resource Configuration Management)을 활성화합니다.

    이 옵션은 기본적으로 비활성화됩니다.

  30. 추적 사용 확인란을 선택하여 TRACE 작업을 활성화합니다.

    이 옵션을 비활성화하면 GlassFish 서버에서 크로스 사이트 스크립팅 공격을 받을 가능성이 낮아집니다. 이 옵션은 기본적으로 활성화됩니다.

  31. 인증 통과 확인란을 선택하여 이 프로토콜을 사용하는 네트워크 수신기가 SSL 종료 프록시 서버에서 트래픽을 수신하도록 지정합니다.

    이 옵션은 기본적으로 비활성화됩니다.

  32. 청크 사용 확인란을 선택하여 HTTP 응답 청크를 활성화합니다.

    이 옵션은 기본적으로 활성화됩니다.

  33. XPowered By 사용 확인란을 선택하여 GlassFish 서버에서 Java Servlet 3.0 및 Java Server Pages 2.0 사양에 따라 X-Powered-By 헤더를 사용하도록 지정합니다.

    이 옵션은 기본적으로 활성화됩니다.

  34. 저장을 누릅니다.

    참고 - 기본 설정으로 돌아가려면 기본값 로드 버튼을 누른 다음 저장을 누릅니다.
