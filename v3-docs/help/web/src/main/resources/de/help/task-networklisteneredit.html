So bearbeiten Sie ein Netzwerk-Zielgerät

  1. Erweitern Sie in der Navigationsstruktur den Konfigurationsknoten.
  2. Erweitern Sie unter dem Konfigurationsknoten den Knoten "Netzwerk konfigurieren".
  3. Wählen Sie unter dem Knoten "Netzwerkkonfiguration" den Knoten "Netzwerk-Zielgeräte".

    Die Seite "Netzwerk-Zielgeräte" wird geöffnet.

  4. Klicken Sie auf der Seite "Netzwerk-Zielgeräte" auf den Namen des zu bearbeitenden Netzwerk-Zielgeräts.

    Die Seite "Netzwerk-Zielgerät bearbeiten" wird geöffnet.

  5. Markieren Sie das Kontrollkästchen "Status aktiviert" oder entfernen Sie die Markierung, um den Status des Netzwerkzielgeräts zu ändern.

    Das Zielgerät kann aktiviert oder deaktiviert werden. Diese Option ist standardmäßig aktiviert. Wenn das Zielgerät deaktiviert ist, führen alle Versuche, eine Verbindung zum Zielgerät herzustellen, zu einer Socket-Ausnahme (java.net.ConnectException).

  6. Aktivieren Sie das Kontrollkästchen "Sicherheit", um Sicherheit für das vom Netzwerk-Zielgerät verwendete Protokoll zu aktivieren.

    Diese Option ist standardmäßig deaktiviert.

  7. Aktivieren Sie das Kontrollkästchen "JK-Zielgerät", um anzugeben, dass das Zielgerät das Apache-Protokoll mod-jk unterstützt.

    Das Protokoll mod-jk ist ein proprietäres Kommunikations-/Netzwerkprotokoll zwischen dem Apache-Webserver httpd und einer Backend-Instanz von GlassFish-Server. Wenn das Zielgerät dieses Protokoll unterstützt, wird davon ausgegangen, dass httpd den gesamten statischen Inhalt verarbeitet, während alle Anforderungen von dynamischen Ressourcen (d. h. Servlets und JSPs) an das mod-jk-fähige Zielgerät von GlassFish-Server weitergeleitet werden, wobei das Protokoll mod-jk verwendet wird. Diese Option ist standardmäßig deaktiviert.

  8. Geben Sie im Feld "Port" die Nummer des Ports ein, den das Netzwerk-Zielgerät überwachen soll.

    Gültige Werte sind 1 bis 65535. Bei einem UNIX-System sind zum Erstellen von Sockets, die die Ports 1 bis 1024 überwachen, Superuser-Rechte erforderlich.

  9. Geben Sie im Feld "Adresse" die IP-Adresse ein, über die die Verbindung zum Netzwerkzielgerät hergestellt wird.

    Die Adresse kann als Punktepaar oder in IPv6-Notation angegeben werden. Beim Wert any (für INADDR_ANY) werden alle IP-Adressen überwacht. Auch die Angabe eines Hostnamens ist möglich.

  10. Wählen Sie aus der Dropdown-Liste "Threadpool" den Threadpool aus, der mit dem Netzwerk-Zielgerät verbunden werden soll.
  11. Klicken Sie auf "Speichern".

    Tipp - Um zu den Standardeinstellungen zurückzukehren, klicken Sie auf die Schaltfläche "Standardwerte laden" und anschließend auf "Speichern".

Nächste Schritte

Auf den Registerkarten "SSL", "HTTP" und "Dateicache" können Sie die Einstellungen für das mit dem Netzwerk-Zielgerät verbundene Protokoll bearbeiten. Die Registerkarte "SSL" ist nur dann von Bedeutung, wenn Sie Sicherheit für das Netzwerk-Zielgerät aktiviert haben.
