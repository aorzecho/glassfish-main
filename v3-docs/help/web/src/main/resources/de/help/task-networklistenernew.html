So erstellen Sie ein Netzwerk-Zielgerät

Bevor Sie beginnen

Wenn Sie ein Netzwerk-Zielgerät erstellen, können Sie gleichzeitig auch ein vom Netzwerk-Zielgerät zu verwendendes Protokoll erstellen. Alternativ können Sie ein Protokoll verwenden, das Sie zuvor erstellt haben. In der Regel liegt eine 1:1-Beziehung zwischen einem Netzwerk-Zielgerät und dem verbundenen Protokoll vor, dies ist jedoch nicht obligatorisch.

  1. Erweitern Sie in der Navigationsstruktur den Konfigurationsknoten.
  2. Erweitern Sie unter dem Konfigurationsknoten den Knoten "Netzwerk konfigurieren".
  3. Wählen Sie unter dem Knoten "Netzwerkkonfiguration" den Knoten "Netzwerk-Zielgeräte".

    Die Seite "Netzwerk-Zielgeräte" wird geöffnet.

  4. Klicken Sie auf der Seite "Netzwerkzielgeräte" auf "Neu".

    Die Seite "Neues Netzwerk-Zielgerät" wird geöffnet.

  5. Geben Sie im Feld "Name" einen Namen für das Netzwerk-Zielgerät ein.

    Der Name eines Netzwerk-Zielgeräts darf nicht mit einer Ziffer beginnen.

  6. Geben Sie im Feld "Protokoll" das Protokoll an, das mit dem Netzwerk-Zielgerät verbunden werden soll.
    • Wählen Sie "Neues Protokoll erstellen", um ein neues Protokoll mit Standardeinstellungen zu erstellen. Wählen Sie außerdem aus der Dropdown-Liste "Virtueller Server" den virtuellen Server aus, der mit dem Protokoll verbunden werden soll.

      Als Protokollname wird standardmäßig der Name des Netzwerk-Zielgeräts mit dem Zusatz -protocol verwendet, dieser Name kann jedoch geändert werden.

    • Wählen Sie "Vorhandenes Protokoll verwenden", und wählen Sie dann ein Protokoll aus der Dropdown-Liste.

      Wenn Sie diese Option wählen, haben Sie im Normalfall zuvor bereits das Protokoll für dieses Zielgerät erstellt.

    Das Protokoll kann nur bei der Erstellung des Netzwerk-Zielgeräts angegeben werden. Sie können das verbundene Protokoll später nicht ändern.

  7. Aktivieren Sie das Kontrollkästchen "Status", um das Netzwerk-Zielgerät zu aktivieren.

    Diese Option ist standardmäßig aktiviert. Wenn das Zielgerät deaktiviert ist, führen alle Versuche, eine Verbindung zum Zielgerät herzustellen, zu einer Socket-Ausnahme (java.net.ConnectException).

    In Versionen vor GlassFish-Server 9.1 wurde bei einem Zielgerät, dessen Attribut "Aktiviert" auf "false" gesetzt war, für alle an das Zielgerät gesendeten Anforderungen der Antwortcode 404 zurückgegeben. Um dieses Verhalten in der aktuellen Version von GlassFish-Server zu erzielen, setzen Sie den Status des Zielgeräts auf "aktiviert" und den Status aller verbundenen virtuellen Server auf "Deaktiviert". Die mit einem virtuellen Server verbundenen Zielgeräte werden in der Dropdown-Liste "Netzwerk-Zielgeräte" des Servers aufgeführt.

  8. Aktivieren Sie das Kontrollkästchen "Sicherheit", um Sicherheit für das vom Netzwerk-Zielgerät verwendete Protokoll zu aktivieren.

    Diese Option ist standardmäßig deaktiviert.

  9. Aktivieren Sie das Kontrollkästchen "JK-Zielgerät", um anzugeben, dass das Zielgerät das Apache-Protokoll mod-jk unterstützt.

    Das Protokoll mod-jk ist ein proprietäres Kommunikations-/Netzwerkprotokoll zwischen dem Apache-Webserver httpd und einer Backend-Instanz von GlassFish-Server. Wenn das Zielgerät dieses Protokoll unterstützt, wird davon ausgegangen, dass httpd den gesamten statischen Inhalt verarbeitet, während alle Anforderungen von dynamischen Ressourcen (d. h. Servlets und JSPs) an das mod-jk-fähige Zielgerät von GlassFish-Server weitergeleitet werden, wobei das Protokoll mod-jk verwendet wird. Diese Option ist standardmäßig deaktiviert.

  10. Geben Sie im Feld "Port" die Nummer des Ports ein, den das Netzwerk-Zielgerät überwachen soll.

    Gültige Werte sind 1 bis 65535. Bei einem UNIX-System sind zum Erstellen von Sockets, die die Ports 1 bis 1024 überwachen, Superuser-Rechte erforderlich.

  11. Geben Sie im Feld "Adresse" die IP-Adresse ein, über die die Verbindung zum Netzwerkzielgerät hergestellt wird.

    Die Adresse kann als Punktepaar oder in IPv6-Notation angegeben werden. Beim Wert any (für INADDR_ANY) werden alle IP-Adressen überwacht. Auch die Angabe eines Hostnamens ist möglich.

  12. Wählen Sie aus der Dropdown-Liste "Threadpool" den Threadpool aus, der mit dem Netzwerk-Zielgerät verbunden werden soll.

    Normalerweise wählen Sie einen der beiden Threadpools aus, die bei der Installation von GlassFish-Server konfiguriert werden.

  13. Wählen Sie aus der Dropdown-Liste "Transport" den Transporttyp aus, den das Netzwerkzielgerät verwenden soll.

    Standardmäßig steht tcp als einzige Option zur Auswahl.

  14. Klicken Sie auf OK.

Nächste Schritte

Wenn Sie Sicherheit für das Netzwerk-Zielgerät aktiviert haben, können Sie auf der Registerkarte "SSL" die SSL-Einstellungen des vom Netzwerk-Zielgerät verwendeten Protokolls bearbeiten.
