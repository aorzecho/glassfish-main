Threadpool bearbeiten

Sie verwenden die Seite "Edit Thread Pool" (Threadpool bearbeiten), um einen vorhandenen Threadpool zu ändern.

Die Seite "Edit Thread Pool" (Threadpool bearbeiten) bearbeiten umfasst die folgenden Optionen.

Standardeinstellungen laden

Schaltfläche zum Wiederherstellen der Standardwerte für Einstellungen, für die es Standardwerte gibt. Einstellungen, die nicht über Standardwerte verfügen, können nicht geändert werden.

Name der Konfiguration

Der Name der Konfiguration, für die die Einstellungen auf dieser Seite gelten. Dieses Feld ist schreibgeschützt.

Name

Der Name des Threadpools. Das Feld Name ist ein schreibgeschütztes Feld. Sie können den Namen nur festlegen, wenn Sie einen neuen Threadpool erstellen.

Klassenname

Der vollständig qualifizierte Name der Klasse, die den Threadpool implementiert. Der Standardwert ist com.sun.grizzly.http.StatsThreadPool .

Max Queue Size (Maximale Warteschlangengröße)

Die maximale Anzahl von Meldungen, die in die Warteschlange gestellt werden können, bis Threads für ihre Verarbeitung zur Verfügung stehen. Der Wert –1 zeigt an, dass die Warteschlangengröße unbegrenzt ist. Der Standardwert ist 4096.

Max Thread Pool Size (Maximale Threadpool-Größe)

Die maximale Anzahl an Threads im Threadpool. Der Standardwert beträgt 5.

Min Thread Pool Size (Minimale Threadpool-Größe)

Die minimale Anzahl an Threads im Threadpool. Der Standardwert beträgt 2.

Idle Thread Timeout (Zeitüberschreitung von Leerlauf-Threads)

Die maximale Zeit in Sekunden, die sich ein Thread im Pool im Leerlauf befinden kann. Nach Ablauf dieser Zeitspanne wird der Thread aus dem Pool entfernt. Der Standardwert beträgt 900.
