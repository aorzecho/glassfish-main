Neuer Threadpool

Sie verwenden die Seite "Neuer Threadpool" zum Erstellen eines Threadpools.

Die Seite "Neuer Threadpool" bietet die folgenden Optionen.

Name der Konfiguration

Der Name der Konfiguration, für die die Einstellungen auf dieser Seite angewendet werden. Dieses Feld ist schreibgeschützt.

Name

Der Name des Threadpools.

Class Name (Klassenname)

Der vollständig angegebene Name der Klasse, die den Threadpool implementiert. Der Standardwert ist com.sun.grizzly.http.StatsThreadPool.

Max. Queuegröße

Die maximale Anzahl von Meldungen, die in die Queue gestellt werden können, bis Threads für ihre Verarbeitung zur Verfügung stehen. Der Wert –1 zeigt an, dass die Queuegröße unbegrenzt ist. Der Standardwert ist 4096.

Max. Threadpoolgröße

Die maximale Anzahl an Threads im Threadpool. Der Standardwert ist 5. Dies ist der empfohlene Wert für Threadpools von Netzwerk-Listenern.

Min Thread Pool Size (Minimale Threadpool-Größe)

Die minimale Anzahl an Threads im Threadpool. Der Standardwert beträgt 2.

Threadleerlauf-Timeout

Die maximale Zeit in Sekunden, die sich ein Thread im Pool im Leerlauf befinden kann. Nach Ablauf dieser Zeitspanne wird der Thread aus dem Pool entfernt. Der Standardwert beträgt 900.

Verwandte Aufgaben

  • So wird ein Threadpool erstellt

  • So wird ein Threadpool bearbeitet

  • So wird ein Threadpool gelöscht
