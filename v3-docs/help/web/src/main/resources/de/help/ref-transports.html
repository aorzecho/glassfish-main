Transporte

Auf der Seite "Transporte" können Sie Transporte konfigurieren. Jedes Netzwerk-Zielgerät ist mit einem Transport verbunden.

Wenn Sie GlassFish-Server erstmalig installieren, steht standardmäßig ein TCP-Transport namens tcp zur Verfügung. Sie können einen TCP- oder UDP-Transport ändern oder erstellen oder aber eine benutzerdefinierte Transportimplementierung bereitstellen.

Für jeden Transport stehen die folgenden Informationen zur Verfügung.

Transportname

Der Name des Transports.

Klassenname

Der Klassenname der Transportimplementierung.

Bytepuffertyp

Der ByteBuffer-Typ, der für den Transport verwendet werden soll.

Die Tabelle "Transporte" enthält darüber hinaus die folgenden Optionen.

Neu

Schaltfläche zum Erstellen eines neuen Transports.

Löschen

Schaltfläche zum Löschen eines oder mehrerer ausgewählter Transporte.

Verwandte Aufgaben

  • So erstellen Sie einen Transport

  • So bearbeiten Sie einen Transport

  • So löschen Sie einen Transport

Verwandte asadmin-Befehle

  • create-transport(1)

  • list-transports(1)

  • delete-transport(1)
