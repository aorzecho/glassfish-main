HTTPサービスを構成する

  1. ナビゲーション・ツリーで、「構成」ノードを展開します。

  2. Select the configuration that you are changing.

  3. 「構成」ノードの下で、「HTTPサービス」ノードを選択します。

    「HTTPサービス」ページが開きます。

  4. 「SSO」の「有効」チェックボックスを選択して、このサーバーのすべての仮想サーバー上で、同じレルムで構成されたすべてのWebアプリケーションに対して、デフォルトでシングル・サインオンを有効にします。

    この設定は各仮想サーバーでオーバーライドできます。このオプションは、デフォルトでは無効になっています。

  5. 「アクセス・ロギング」の「有効」チェックボックスを選択して、このプロパティを指定していないすべての仮想サーバーのサブ要素に対するアクセス・ロギングを有効にします。

    「アクセス・ロギング」の「有効」チェックボックスの選択を解除すると、このプロパティを指定していないすべての仮想サーバーのサブ要素に対するアクセス・ロギングは無効になります。このオプションは、デフォルトでは無効になっています。

  6. 「ローテーション」の「有効」チェックボックスを選択して、ログ・ファイルのローテーションを有効にします。

    このオプションはデフォルトで有効になっています。

  7. 「ローテーション・ポリシー」ドロップダウン・リストから、ローテーション・ポリシーを選択します。

    使用可能なローテーション・ポリシーはtimeのみです。

  8. 「ローテーション間隔」フィールドに、アクセス・ログのローテーション間隔を分単位で入力します。

    このフィールドは、「ローテーション・ポリシー」が「time」の場合にのみ有効です。デフォルト値は1,440分(24時間)です。

  9. 「ローテーション接尾辞」フィールドに、ローテーション後にログ・ファイル名の最後に追加される文字列値を入力します。

    デフォルト値はyyyy-MM-ddです。

    ログ・ローテーションごとに一意の接尾辞を作成するために必要な値がローテーション接尾辞に含まれていることを確認してください。たとえば、アクセス・ログを毎日複数回の頻度でローテーションする場合には、ローテーション接尾辞に時間を含めます。

    ローテーション間隔が経過するたびにアクセス・ログ・ファイルがローテーションされるようにするには、一意の接尾辞が必要です。新しいアクセス・ログ・ファイルのローテーション接尾辞が既存のアクセス・ログ・ファイルのローテーション接尾辞と同じ場合、ファイルはローテーションされません。

  10. 「最大ファイル数」フィールドに、ローテーションで維持するアクセス・ログ・ファイルの最大数を入力します。

    負の値を指定すると、ファイル数は制限されません。デフォルト値は-1です。

  11. 「バッファ・サイズ」フィールドに、アクセス・ログ・バッファのサイズをバイト単位で入力します。

    0以下の値を指定すると、アクセス・ログのバッファリングが無効になります。デフォルト値は32768です。

  12. 「書込み間隔」フィールドに、ログをディスクに書き込むまでの時間を秒単位で入力します。

    アクセス・ログは、バッファがフルになるか、指定した時間が経過すると書き込まれます。0以下の値を指定すると、バッファがフルでなくても常に書き込まれます。つまり、サーバー・アクセスがあるたびに、ログ・メッセージはファイルに直接格納されます。デフォルト値は300です。

  13. 「形式」フィールドに、文字列値を入力して次のアクセス・ログのプロパティを指定します。

    • アクセス・ログに取り込まれる情報

    • 情報が取り込まれる順序

    デフォルト値は次のとおりです。

    %client.name% %auth-user-name% %datetime% %request% %status% %response.length%

    形式文字列に含めることができる値は次のとおりです。

    %attribute.name%

    指定したnameを持つリクエスト属性の値を(toStringメソッドを呼び出して)記録します

    %auth-user-name%

    承認ユーザーの名前

    %client.name%

    クライアント・ホスト名

    %client.dns%

    クライアントDNS

    %cookie.name%

    指定したnameを持つリクエストのクッキーの値

    %cookie.value%

    リクエストにある最初のクッキーの値

    %datetime%

    システム日付

    %request%

    完全なHTTPリクエスト行

    %status%

    ステータス

    %response.length%

    レスポンス・コンテンツ長

    %header.referer%

    Refererヘッダー

    %header.user-agent%

    ユーザー・エージェント・ヘッダー

    %http-method%

    HTTPメソッド

    %http-uri%

    HTTP URI

    %query-str%

    HTTPクエリー文字列

    %http-version%

    HTTPプロトコル・バージョン

    %header.accept%

    受入れヘッダー

    %header.date%

    日付ヘッダー

    %header.if-mod-since%

    If-Modified-Sinceヘッダー

    %header.auth%

    認証ヘッダー

    %header.any%

    RFC 2616で定義された任意の有効なHTTPヘッダー値(anyも有効なヘッダー値。ここでは変数として指定される)

    %session.name%

    nameという名前を持つセッション属性の値。指定した属性がセッションに存在しない場合はNULL-SESSION-ATTRIBUTE-name、セッションが存在しない場合はNULL-SESSIONになります。

    %time-taken%

    各リクエストの処理にかかった時間(ミリ秒単位)

    %vs.id%

    仮想サーバーID

  14. 「追加プロパティ」セクションで、追加プロパティを指定します。

    使用可能なプロパティについては、HTTPサービスに固有のプロパティを参照してください。

    • プロパティを追加するには、「プロパティを追加」ボタンをクリックします。表示される空白行で、「名前」フィールドにプロパティ名を入力し、「値」フィールドにプロパティ値を入力します。

    • プロパティを変更するには、そのプロパティの「値」フィールドを編集します。

    • プロパティを削除するには、削除するプロパティの「名前」フィールドの左側にあるチェックボックスを選択し、「プロパティを削除」ボタンをクリックします。

  15. 「保存」をクリックします。

    ヒント:

    デフォルト設定に戻すには、「デフォルトをロード」ボタンをクリックし、「保存」をクリックします。
