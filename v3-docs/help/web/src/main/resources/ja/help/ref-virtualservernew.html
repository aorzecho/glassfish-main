新しい仮想サーバー

仮想サーバーを作成するには、「新しい仮想サーバー」ページを使用します。

「新しい仮想サーバー」ページには、次のオプションが含まれます。

設定名

このページの設定が適用される設定の名前。このフィールドは読み取り専用です。

Id

内部的に参照可能な仮想サーバーの識別子。これが HTTP クライアント側に表示されることはありません。HTTP クライアント側に表示するホスト名は、「ホスト」フィールドに指定する必要があります。

ホスト

サーバーが稼動するマシンのホスト名 (1 つまたは複数)。ネットワークの DNS サーバ (UNIX システムでは /etc/hosts ファイル) に登録されている、実際のホスト名または仮想ホスト名を使用します。デフォルト値は、システムプロパティーの値の ${com.sun.aas.hostName} です。

状態

仮想サーバーに求められる状態。値は次のいずれかになります。

オン

仮想サーバーはアクティブです。この値はデフォルトです。

オフ

仮想サーバーは非アクティブです。サーバーへのアクセスを試行すると、エラーコード 404 (リソースが使用できません) が返されます。

使用不可

仮想サーバーは使用不可です。サーバーへのアクセスを試行すると、エラーコード 403 (要求の実行拒否) が返されます。

SSO

シングルサインオンの動作を、HTTP サービスから継承、有効、または無効のいずれかに指定します。デフォルトでは、HTTP サービスから継承されます。このオプションが有効の場合、この仮想サーバー上の同じレルムで設定された Web アプリケーションに対してシングルサインオンが有効になります。このオプションが無効の場合、この仮想サーバーではシングルサインオンが無効になり、ユーザーは仮想サーバー上のアプリケーションごとに個別に認証を行う必要があります。

Network Listeners

このサーバーに関連付けられた 1 つまたは複数のネットワークリスナー (存在する場合)。

デフォルト Web モジュール

仮想サーバーに配備されているその他の Web モジュールにマッピングできないすべての要求に対応する配備済み Web モジュール (配備されている場合)。デフォルトの Web モジュールが指定されない場合は、コンテキストルートが空の Web モジュールが使用されます。コンテキストルートが空の Web モジュールが存在しない場合は、システムのデフォルトの Web モジュールが作成され、これが使用されます。

ログファイル

この仮想サーバーからのログメッセージが表示されるファイルのパス名。デフォルトでは、ログメッセージは domain-dir/logs/server.log に表示されます。

docroot

サーバーのルートドキュメントディレクトリへの絶対パス。デフォルト値は domain-dir/docroot です。

アクセスログ

アクセスログを、HTTP サービスから継承、有効、または無効のいずれかに指定します。デフォルトでは、HTTP サービスから継承されます。

ディレクトリ

サーバーのアクセスログへの絶対ディレクトリパス。デフォルト値は domain-dir/logs/access です。

追加プロパティー

仮想サーバーの追加プロパティー。

使用可能なプロパティーの詳細については、「仮想サーバーに固有のプロパティー」を参照してください。
