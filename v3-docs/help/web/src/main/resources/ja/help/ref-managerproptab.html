マネージャ・プロパティ

セッション・マネージャを使用して、セッションを作成および破棄する方法、セッション状態を格納する場所、および使用可能なセッションの最大数を構成できます。

「マネージャ・プロパティ」ページを使用して、Webコンテナのセッション・マネージャ・プロパティを追加または変更します。

「マネージャ・プロパティ」ページには、次のオプションが含まれます。

デフォルトをロード

デフォルト値を持つ設定をデフォルト値に戻すボタン。デフォルト値を持たない設定は変更されません。

構成名

このページの設定が適用される構成の名前。このフィールドは読取り専用です。

取得間隔

アクティブでないセッション・データがストアから削除されるまでの秒数。デフォルト値は60です。

この値は、セッション・データが変更される頻度より小さい値に設定してください。たとえば、ヒット頻度の高いWebサイトのヒット・カウンタ・サーブレットでは、この値をできるだけ小さい値(1秒)にしてください。値が大きい場合、サーバーを再起動するたびに、最後の数回のヒットを取得できない場合があります。

最大セッション数

キャッシュに維持できるセッションの最大数。-1を指定すると、セッションの数は制限されません。デフォルト値は-1です。

制限に達した後に新しいセッションを作成しようとすると、IllegalStateExceptionがスローされます。

セッション・ファイル名

状態の保存が可能な場合に、アプリケーションの再起動間でセッション状態を保存するディレクトリの絶対パスまたは相対パス。このパスが設定されていない場合、セッション状態は保存されません。デフォルトでは、このパスは設定されていません。

相対パスは、このWebアプリケーションの一時ディレクトリからの相対で指定します。

追加プロパティ

セッション・マネージャの追加プロパティ。GlassFish Serverはセッション・マネージャの追加プロパティを定義しません。
