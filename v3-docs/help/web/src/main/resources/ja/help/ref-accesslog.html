HTTPサービス

Webアプリケーションの一般的なアクセス・ログ・ポリシーを指定するには、「HTTPサービス」ページを使用します。

HTTPサービスは、Webアプリケーションをデプロイする機能、デプロイされているWebアプリケーションをHTTPクライアントでアクセス可能にする機能、および仮想サーバー、スレッド・プール、ネットワーク構成機能を提供します。

GlassFish ServerはHTTPサービスのアクセス・ログを使用して、仮想サーバーのアクセス・ログのローテーションを有効にし、構成します。これらのログは、 domain-dir/logs/accessディレクトリにあり、名前の形式はvirtual_server_name_access_log.yyyyMMdd-HHhmmmsss.txtです。

「HTTPサービス」ページには、次のオプションが含まれます。

デフォルトをロード

デフォルト値を持つ設定をデフォルト値に戻すボタン。デフォルト値を持たない設定は変更されません。

構成名

このページの設定が適用される構成の名前。このフィールドは読取り専用です。

SSO

このサーバーのすべての仮想サーバー上で、同じレルムで構成されたすべてのWebアプリケーションに対して、デフォルトでシングル・サインオンを有効にするかどうかを識別します。このオプションが有効でない場合は、すべての仮想サーバーでシングル・サインオンがデフォルトで無効になります。ユーザーは、各仮想サーバー上のアプリケーションごとに個別に認証を行う必要があります。この設定は各仮想サーバーでオーバーライドできます。SSOは有効または無効にできます。このオプションは、デフォルトでは無効になっています。

アクセス・ロギング

このオプションを選択する場合、このプロパティが指定されていないすべての仮想サーバー・サブ要素のアクセス・ロギングが有効になります。選択しない場合、このオプションにより、このプロパティが指定されていないすべての仮想サーバー・サブ要素のアクセス・ロギングが無効になります。このオプションは、デフォルトでは無効になっています。

ローテーション

このオプションを選択すると、ログ・ファイルのローテーションが有効になります。このオプションはデフォルトで有効になっています。

ローテーション・ポリシー

使用可能なローテーション・ポリシーはtimeのみです。

ローテーション間隔

アクセス・ログのローテーション間隔(分)です。このフィールドは、「ローテーション・ポリシー」が「time」の場合にのみ有効です。デフォルト値は1,440分(24時間)です。

ローテーション接尾辞

ローテーション後にログ・ファイルの最後に追加される文字列値。デフォルト値はyyyy-MM-ddです。

ログ・ローテーションごとに一意の接尾辞を作成するために必要な値がローテーション接尾辞に含まれていることを確認してください。たとえば、アクセス・ログを毎日複数回の頻度でローテーションする場合には、ローテーション接尾辞に時間を含めます。

ローテーション間隔が経過するたびにアクセス・ログ・ファイルがローテーションされるようにするには、一意の接尾辞が必要です。新しいアクセス・ログ・ファイルのローテーション接尾辞が既存のアクセス・ログ・ファイルのローテーション接尾辞と同じ場合、ファイルはローテーションされません。

最大ファイル数

ローテーションで維持するアクセス・ログ・ファイルの最大数。負の値を指定すると、ファイル数は制限されません。デフォルト値は-1です。

バッファ・サイズ

アクセス・ログ・バッファのサイズ(バイト単位)。バッファされていないアクセス・ログでは、0以下の値を指定します。デフォルト値は32768です。

書込み間隔

ログがディスクに書き込まれるまでの秒数。アクセス・ログは、バッファがフルになるか、指定した時間が経過すると書き込まれます。0以下の値を指定すると、バッファがフルでなくても常に書き込まれます。つまり、サーバー・アクセスがあるたびに、ログ・メッセージはファイルに直接格納されます。デフォルト値は300です。

形式

アクセス・ログに取り込まれる情報と取り込まれる順序を指定する文字列値。デフォルト値は次のとおりです。

%client.name% %auth-user-name% %datetime% %request% %status% %response.length%

形式文字列に含めることができる値は次のとおりです。

%attribute.name%

指定したnameを持つリクエスト属性の値を(toStringメソッドを呼び出して)記録します

%auth-user-name%

承認ユーザーの名前

%client.name%

クライアント・ホスト名

%client.dns%

クライアントDNS

%cookie.name%

指定したnameを持つリクエストのクッキーの値

%cookie.value%

リクエストにある最初のクッキーの値

%datetime%

システム日付

%request%

完全なHTTPリクエスト行

%status%

ステータス

%response.length%

レスポンス・コンテンツ長

%header.referer%

Refererヘッダー

%header.user-agent%

ユーザー・エージェント・ヘッダー

%http-method%

HTTPメソッド

%http-uri%

HTTP URI

%query-str%

HTTPクエリー文字列

%http-version%

HTTPプロトコル・バージョン

%header.accept%

受入れヘッダー

%header.date%

日付ヘッダー

%header.if-mod-since%

If-Modified-Sinceヘッダー

%header.auth%

認証ヘッダー

%header.any%

RFC 2616で定義された任意の有効なHTTPヘッダー値(anyも有効なヘッダー値。ここでは変数として指定される)

%session.name%

nameという名前を持つセッション属性の値。指定した属性がセッションに存在しない場合はNULL-SESSION-ATTRIBUTE-name、セッションが存在しない場合はNULL-SESSIONになります。

%time-taken%

各リクエストの処理にかかった時間(ミリ秒単位)

%vs.id%

仮想サーバーID
