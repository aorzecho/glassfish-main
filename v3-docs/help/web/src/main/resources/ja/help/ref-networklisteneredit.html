ネットワーク・リスナーを編集

既存のネットワーク・リスナーを変更するには、「ネットワーク・リスナーを編集」ページを使用します。

ネットワーク・リスナーを編集するときに、関連付けられたプロトコルの「SSL」、「HTTP」、および「ファイル・キャッシュ」タブを使用して編集することもできます。

「ネットワーク・リスナーを編集」ページには、次のオプションが含まれます。

デフォルトをロード

デフォルト値を持つ設定をデフォルト値に戻すボタン。デフォルト値を持たない設定は変更されません。

構成名

このページの設定が適用される構成の名前。このフィールドは読取り専用です。

名前

リスナーの一意の名前。「名前」フィールドは読取り専用です。名前を指定できるのは、新しいネットワーク・リスナーの作成時のみです。

プロトコル

ネットワーク・リスナーに関連付けられているプロトコル。「プロトコル」フィールドは読取り専用です。プロトコルを作成または指定できるのは、新しいネットワーク・リスナーの作成時のみです。

ステータス

ネットワーク・リスナーのステータス。リスナーは、有効または無効の状態になります。このオプションはデフォルトで有効になっています。リスナーが無効の場合は、リスナーに接続しようとするとソケット例外(java.net.ConnectException)が発生します。

バージョン9.1より前のGlassFish Serverでは、有効化された属性がfalseに設定されたリスナーは、送信されたリクエストに対して404レスポンス・コードを返します。現在のバージョンのGlassFish Serverでこの動作を行うには、リスナーのステータスを有効に設定し、関連する各仮想サーバーの状態をオフに設定します。仮想サーバーは、関連付けられたリスナーを「ネットワーク・リスナー」ドロップダウン・リストに一覧表示します。

セキュリティ

このオプションを選択すると、ネットワーク・リスナーが使用するプロトコルのセキュリティが有効になります。このオプションは、デフォルトでは無効になっています。

JKリスナー

このオプションを選択すると、リスナーはApacheのmod-jkプロトコルをサポートします。mod-jkプロトコルは、Apacheのhttpd(Webサーバー)とGlassFish Serverのバックエンド・インスタンス間で使用される、専用の通信およびネットワークのプロトコルです。リスナーがこのプロトコルをサポートする場合、httpdはすべての静的なコンテンツを処理すると想定され、動的なリソース(サーブレットおよびJSP)のリクエストは、mod-jkに対応するGlassFish Serverのネットワーク・リスナーにmod-jkプロトコルを使用してルーティングされます。このオプションは、デフォルトでは無効になっています。

ポート

リスナーがリスニングするポート番号。適切な値は1から65535です。UNIXシステムで、ポート1から1024でリスニングするソケットを作成するには、スーパーユーザー権限が必要です。

アドレス

ネットワーク・リスナーがリスニングするIPアドレス。アドレスはドット区切りかIPv6の表記法で指定します。すべてのIPアドレスでリスニングする場合は、any(INADDR_ANY)を指定できます。ホスト名を指定できます。

スレッド・プール

ネットワーク・リスナーに関連付けられているスレッド・プール。

トランスポート

ネットワーク・リスナーのトランスポートのタイプ。デフォルトでは、トランスポートはtcpのみです。
