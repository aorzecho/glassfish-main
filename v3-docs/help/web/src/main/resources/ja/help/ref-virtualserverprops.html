仮想サーバーに固有のプロパティー

仮想サーバーに使用できる追加のプロパティーは次のとおりです。

sso-max-inactive-seconds

クライアントが活動を停止後、何秒後にユーザーのシングルサインオンの記録を削除可能にするかを指定します。シングルサインオンは同一仮想サーバー上の複数のアプリケーションに適用されるので、これらのアプリケーションのいずれかにアクセスすることでシングルサインオンの記録は有効なまま確保されます。

デフォルト値は 300 秒 (5 分) です。値を大きくするとユーザーのシングルサインオンの持続時間は長くなりますが、サーバー上のメモリー消費量も増加します。

sso-reap-interval-seconds

有効期限が切れたシングルサインオンの記録の削除を行う間隔を秒単位で指定します。

デフォルト値は 60 です。

ssoCookieSecure

この仮想サーバーに配備された Web アプリケーションに関連付けられた、JSESSIONIDSSO クッキーの Secure 属性を設定します。このプロパティーは、シングルサインオンが有効な場合にのみ適用されます。使用できる値は次のとおりです。

true

Secure を true に設定します。

false

Secure を false に設定します。

dynamic

JSESSIONIDSSO クッキーは、SSO の最初のセッションの Secure 設定を継承します。

JSESSIONID クッキーの Secure 属性を設定するには、sun-web.xml ファイルの cookieSecure cookie-properties プロパティーを使用します。詳細については、『Oracle GlassFish Server 3.1 Application Deployment Guide』の「cookie-properties」を参照してください。

setCacheControl

Cache-Control 応答指令のコンマ区切りのリストを指定します。有効な指令のリストについては、http://www.ietf.org/rfc/rfc2616.txt の文書のセクション 14.9 を参照してください。

accessLogBufferSize

アクセスログの呼び出しが格納されるバッファーのサイズをバイト単位で指定します値が 5120 未満の場合は警告メッセージが表示され、値が 5120 に設定されます。デフォルト値は 32768 です。

accessLogWriterInterval

ログがディスクに書き込まれるまでの時間 (秒単位) を指定します。アクセスログは、バッファーがいっぱいになるか、指定した時間が経過すると書き込まれます。0 を指定すると、バッファーがいっぱいでなくても常に書き込まれます。つまり、サーバーアクセスがあるたびに、ログメッセージはファイルに直接格納されます。デフォルト値は 300 です。

allowRemoteAddress

リモートクライアントの IP アドレスを比較する、正規表現パターンのコンマ区切りのリストを指定します。このプロパティーが指定されている場合、要求を受け入れるためにはリモートアドレスが「一致している」必要があります。このプロパティーが指定されていない場合、リモートアドレスが denyRemoteAddress パターンと「一致していない限り」、すべての要求が受け入れられます。

denyRemoteAddress

リモートクライアントの IP アドレスを比較する、正規表現パターンのコンマ区切りのリストを指定します。このプロパティーが指定されている場合、要求を受け入れるためにはリモートアドレスが「一致していない」必要があります。このプロパティーが指定されていない場合、要求の受け入れは allowRemoteAddress プロパティーのみで制御されます。

allowRemoteHost

リモートクライアントのホスト名 ([java.net.]Socket.getInetAddress().getHostName() によって返される) を比較する正規表現パターンのコンマ区切りのリストを指定します。このプロパティーが指定されている場合、要求を受け入れるためにはリモートホスト名が「一致している」必要があります。このプロパティーが指定されていない場合、リモートホスト名が denyRemoteHost パターンと「一致していない限り」、すべての要求が受け入れられます。

denyRemoteHost

リモートクライアントのホスト名 ([java.net.]Socket.getInetAddress().getHostName() によって返される) を比較する正規表現パターンのコンマ区切りのリストを指定します。このプロパティーが指定されている場合、要求を受け入れるためにはリモートホスト名が「一致していない」必要があります。このプロパティーが指定されていない場合、要求の受け入れは allowRemoteHost プロパティーのみで制御されます。

GlassFish サーバー ドメインが、「JK リスナー」オプションが有効なネットワークリスナーを通してアクセスされる場合、このプロパティーを設定しても影響はありません。

authRealm

認証レルムの名前を指定します。この値は、この仮想サーバーに配備されているスタンドアロン Web アプリケーションの、サーバーインスタンスのデフォルトレルムをオーバーライドします。スタンドアロンの Web アプリケーションの web.xml ファイルに定義されているレルムは、仮想サーバーのレルムをオーバーライドします。詳細については、レルムを参照してください。

securePagesWithPragma

このプロパティーを false に設定すると、この仮想サーバー上のすべての Web アプリケーションで、SSL を使用したファイルのダウンロードが Internet Explorer で必ず正しく機能します。

個々の Web アプリケーションでこの設定をオーバーライドできます。それには、sun-web.xml ファイルの sun-web-app 要素を使用します。詳細については、『Oracle GlassFish Server 3.1 Application Deployment Guide』の「glassfish-web-app」を参照してください。

デフォルト値は true です。

alternatedocroot_n

代替ドキュメントルート (docroot) を指定します。ここで、n は正の整数で複数の指定が可能になります。代替 docroot により、Web アプリケーションは特定のリソースに対するそれぞれの docroot 以外からの要求に対応できます。これは、それらの要求が Web アプリケーションの代替 docroot の URI パターンの 1 つ (または複数) に一致するかどうかに基づきます。

要求が代替 docroot の URI パターンと一致する場合、要求の URI (Web アプリケーションのコンテキストルートをマイナスしたもの) を代替 docroot の物理的な位置 (ディレクトリ) に付加することによって、要求を代替 docroot にマップします。要求が複数の URI パターンに一致する場合、代替 docroot は次の優先順位で判断されます。

  • 完全一致

  • 最長パス一致

  • 拡張一致

たとえば、次のプロパティーは 3 つの代替 docroot を指定します。最初の代替 docroot の URI パターンは完全一致を使用し、2 番目と 3 番目の代替 docroot の URI パターンは拡張および最長パスプレフィックス一致をそれぞれ使用しています。

<property name="alternatedocroot_1" 
   value="from=/my.jpg dir=/srv/images/jpg"/>
<property name="alternatedocroot_2" 
   value="from=*.jpg dir=/srv/images/jpg"/>
<property name="alternatedocroot_3" 
   value="from=/jpg/* dir=/src/images"/>

各代替 docroot の value には 2 つの構成要素があります。最初の構成要素の from は代替 docroot の URI パターンを指定し、2 番目の構成要素の dir は代替 docroot の物理的な位置 (ディレクトリ) を指定します。 dir 構成要素にはスペースが許容されています。

個々の Web アプリケーションでこの設定をオーバーライドできます。それには、sun-web.xml ファイルの sun-web-app 要素を使用します。詳細については、『Oracle GlassFish Server 3.1 Application Deployment Guide』の「glassfish-web-app」を参照してください。

contextXmlDefault

使用されている場合は、この仮想サーバーに対して、context.xml ファイルの domain-dir に相対的な位置を指定します。context.xml ファイルの詳細については、コンテキストルートコンテナ を参照してください。

allowLinking

true の場合、この仮想サーバー上に配備されているすべての Web アプリケーションに対し、シンボリックリンクのリソースが提供されます。個々の Web アプリケーションでこの設定をオーバーライドできます。それには、sun-web.xml ファイル内で sun-web-app のプロパティー allowLinking を次のように指定します。

<sun-web-app>
   <property name="allowLinking" 
      value="{true|false}"/>
</sun-web-app>

詳細については、『Oracle GlassFish Server 3.1 Application Deployment Guide』の「glassfish-web-app」を参照してください。

デフォルト値は false です。

注意 - Windows システムでこのプロパティーを true に設定すると、JSP ソースコードが公開されます。

send-error_n

仮想サーバー上に配備されているすべての Web アプリケーションで継承する、仮想サーバーのカスタムエラーページのマッピングを指定します。Web アプリケーションは、web.xml 配備記述子のこれらカスタムエラーページのマッピングをオーバーライドできます。各 send-error_ n プロパティーの値には 3 つの構成要素があり、任意の順番で指定できます。

  • 最初の構成要素の code は、応答でカスタムエラーページを返す必要がある 3 桁の HTTP 応答状態コードを指定します。

  • 2 番目の構成要素の path は、カスタムエラーページの絶対、または相対ファイルシステムパスを指定します。相対ファイルシステムパスは、domain-dir/config ディレクトリに相対的なものとして解釈されます。

  • 3 番目の構成要素の reason はオプションで、返される理由文字列のテキスト (Unauthorized または Forbidden) を指定します。

次に例を示します。

<property name="send-error_1"
   value="code=401 path=/myhost/401.html reason=MY-401-REASON"/>

この例のプロパティー定義では、/myhost/401.html の内容が 401 応答で返されるとともに、次の応答が返されます。

HTTP/1.1 401 MY-401-REASON
redirect_n

古い URL に対する要求を新しい URL に対する要求として処理するように指定します。これらのプロパティーは、仮想サーバー上に配備されているすべての Web アプリケーションで継承されます。各 redirect_n プロパティーの値には 2 つの構成要素があり、任意の順番で指定できます。

  • 最初の構成要素の from は、照合する要求された URI のプレフィックスを指定します。

  • 2 番目の構成要素の url-prefix はクライアントに返す新しい URL プレフィックスを指定します。 from プレフィックスは、この URL プレフィックスで単純に置き換えられます。

次に例を示します。

<property name="redirect_1" 
  value="from=/dummy url-prefix=http://etude"/>
valve_n

カスタム値の完全修飾クラス名を指定します。n は正の整数で、複数の指定が可能です。値クラスは、Tomcat または GlassFish サーバー の以前のリリースの org.apache.catalina.Valve インタフェースを実装するか、GlassFish サーバー の現在のリリースの org.glassfish.web.valve.GlassFishValve インタフェースを実装する必要があります。次に例を示します。

<property name="valve_1"
value="org.glassfish.extension.Valve"/>

このプロパティーは、特定の Web アプリケーションに対して設定できます。詳細については、『Oracle GlassFish Server 3.1 Application Deployment Guide』の「glassfish-web-app」を参照してください。

listener_n

カスタム Catalina リスナーの完全修飾クラス名を指定します。n は正の整数で、複数の指定が可能です。リスナークラスは、 org.apache.catalina.ContainerListener または org.apache.catalina.LifecycleListener インタフェースを実装する必要があります。次に例を示します。

<property name="listener_1"
value="org.glassfish.extension.MyLifecycleListener"/>

このプロパティーは、特定の Web アプリケーションに対して設定できます。詳細については、『Oracle GlassFish Server 3.1 Application Deployment Guide』の「glassfish-web-app」を参照してください。

errorReportValve

この仮想サーバー上で、アプリケーションのデフォルトのエラーページを生成するカスタム値の完全修飾クラス名を指定します。この仮想サーバーのデフォルトのエラーページメカニズムを無効にするには、空の文字列を指定します。
