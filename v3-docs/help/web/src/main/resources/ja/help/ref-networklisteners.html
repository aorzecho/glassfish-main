ネットワークリスナー

ネットワークリスナーを構成するには、「ネットワークリスナー」ページを使用します。

GlassFish Server をはじめてインストールする場合は、デフォルトで admin-listener、http-listener1、および http-listener2 という名前の 3 つのネットワークリスナーが存在します。SSL で設定される http-listener2 は、デフォルトでは無効になっています。

ネットワークリスナーごとに、次の情報が表示されます。

名前

ネットワークリスナーの名前。

ポート #

ネットワークリスナーが待機しているポート番号。

プロトコル

ネットワークリスナーによって使用されるプロトコル。

スレッドプール

ネットワークリスナーによって使用されるスレッドプール。

使用可能

ネットワークリスナーが有効な場合は true、ネットワークリスナーが有効でない場合は false です。

「ネットワークリスナー」の表には、次のオプションも表示されます。

新規

新しいネットワークリスナーを作成するボタン。

削除

選択した 1 つまたは複数のネットワークリスナーを削除するボタン。

関連操作

  • 「ネットワークリスナーを作成する」

  • 「ネットワークリスナーを編集する」

  • 「ネットワークリスナーを削除する」

  • 「仮想サーバーを作成する」

関連する asadmin コマンド

  • create-network-listener(1)

  • list-network-listeners(1)

  • delete-network-listener(1)
