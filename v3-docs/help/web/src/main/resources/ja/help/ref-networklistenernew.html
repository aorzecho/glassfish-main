新しいネットワークリスナー

新しいネットワークリスナーを作成するには、「新しいネットワークリスナー」ページを使用します。

ネットワークリスナーを作成するときに、同時にネットワークリスナーが使用するプロトコルを作成できます。または、先に作成したプロトコルを使用することもできます。通常、ネットワークリスナーと関連付けられるプロトコルは 1 対 1 の関係になりますが、これは必須ではありません。

「新しいネットワークリスナー」ページには、次のオプションが含まれます。

設定名

このページの設定を適用する設定の名前。 このフィールドは読み取り専用です。

名前

リスナーの一意の名前。ネットワークリスナー名の先頭に数字を使用することはできません。

プロトコル

ネットワークリスナーに関連付けられているプロトコル。次のオプションは、新しいプロトコルを作成するか、既存のプロトコルを使用するかを指定します。

新しいプロトコルを作成

ネットワークリスナーに対して、新しいプロトコルをデフォルトの設定で作成します。プロトコルのデフォルト名は、ネットワークリスナーの名前に -protocol を付加した文字列になりますが、この値は編集できます。

Virtual Server

新しいプロトコルを作成する場合に、プロトコルに関連付けられる仮想サーバー。

既存のプロトコルを使用

選択したプロトコルを使用します。

ステータス

ネットワークリスナーのステータス。リスナーは、有効または無効の状態になります。このオプションはデフォルトで有効になっています。リスナーが無効の場合は、リスナーに接続しようとするとソケット例外 (java.net.ConnectException) が発生します。

バージョン 9.1 より前の GlassFish Server では、有効化された属性が false に設定されたリスナーは、送信された要求に対して 404 応答コードを返します。現在のバージョンの GlassFish Server でこの動作を行うには、リスナーのステータスを有効に設定し、関連する各仮想サーバーのステータスをオフに設定します。仮想サーバーは、関連付けられたリスナーを「ネットワークリスナー」ドロップダウンリストに表示します。

Security

このオプションを選択すると、ネットワークリスナーが使用するプロトコルのセキュリティーが有効になります。このオプションは、デフォルトでは無効になっています。

JK リスナー

このオプションを選択すると、リスナーは Apache の mod-jk プロトコルをサポートします。mod-jk プロトコルは、Apache の httpd (Web サーバー) と GlassFish Server のバックエンドインスタンス間で使用される、専用の通信およびネットワークのプロトコルです。リスナーがこのプロトコルをサポートする場合、httpd はすべての静的なコンテンツを処理すると想定され、動的なリソース (サーブレットおよび JSP) の要求は、mod-jk に対応する GlassFish Server のネットワークリスナーに mod-jk プロトコルを使用してルーティングされます。このオプションは、デフォルトでは無効になっています。

ポート

リスナーが待機するポート番号。適切な値は 1 ～ 65535 です。UNIX システムでは、ポート 1 ～ 1024 で待機するソケットを作成する場合、スーパーユーザーの権限が必要です。

アドレス

ネットワークリスナーが待機する IP アドレス。アドレスはドット区切りか IPv6 の表記法で指定します。すべての IP アドレスで待機する場合は、any (INADDR_ANY) を指定できます。ホスト名を指定できます。

スレッドプール

ネットワークリスナーに関連付けられているスレッドプール。通常は、GlassFish Server のインストール時に設定される 2 つのスレッドプールのいずれかを選択します。

トランスポート

ネットワークリスナーのトランスポートのタイプ。デフォルトでは、選択できるのは tcp のみです。
