编辑虚拟服务器

  1. 在导航树中，展开“配置”节点。
  2. 在“配置”节点下，选择“虚拟服务器”节点。

    此时将打开“虚拟服务器”页面。

  3. 在“虚拟服务器”页面上，单击要编辑的虚拟服务器的名称。

    此时将打开“编辑虚拟服务器”页面。

  4. 在“主机”字段中，键入运行服务器的计算机的主机名或 IP 地址。

    请使用已向网络的 DNS 服务器注册（而且在 UNIX 系统上，位于 /etc/hosts 文件中）的实际主机名或虚拟主机名。默认值为系统属性值 ${com.sun.aas.hostName}。

  5. 选择虚拟服务器的“状态”选项。

    值可为以下任意一种：

    打开

    虚拟服务器处于活动状态。该值为默认值。

    关闭

    虚拟服务器处于非活动状态。如果尝试访问服务器，则将会返回错误代码 404（资源不可用）。

    已禁用

    虚拟服务器已禁用。如果尝试访问服务器，则将会返回错误代码 403（拒绝完成请求）。

  6. 为新的虚拟服务器选择 SSO 选项。

    默认情况下，虚拟服务器继承 HTTP 服务的单点登录 (SSO) 设置。选择“已启用”或“已禁用”，以启用或禁用此虚拟服务器的单点登录，从而覆盖 HTTP 服务设置。

    Servlet 和 JSP 文件支持在 GlassFish Server 上跨应用程序的单点登录。此功能允许需要同一用户登录信息的多个应用程序共享登录信息，用户不必在使用每个应用程序时都分别进行登录。

  7. 从“网络侦听器”下拉式列表中，选择一个或多个网络侦听器，或将该字段留空。
  8. 从“默认 Web 模块”下拉式列表中，选择一个模块，此模块将响应所有无法映射到已部署到虚拟服务器中的其他 Web 模块的请求。

    如果未指定默认 Web 模块，系统将使用具有空上下文根目录的 Web 模块。如果不存在包含空上下文根目录的 Web 模块，系统将创建并使用默认 Web 模块。

  9. 在“日志文件”字段中，键入将记录此虚拟服务器的日志信息的文件的路径名。

    默认情况下，日志信息将发送至默认服务器日志 domain-dir/logs/server.log 。

  10. 在“文档根目录”字段中，键入服务器的文档根目录的绝对路径。

    默认值为 domain-dir/docroot。

  11. 选择“访问日志记录”选项，以启用或禁用访问日志记录。

    默认情况下，虚拟服务器继承 HTTP 服务的访问日志记录设置。选择“已启用”或“已禁用”，以启用或禁用此虚拟服务器的访问日志记录，从而覆盖 HTTP 服务设置。

  12. 在“目录”字段中，键入服务器访问日志的绝对目录路径。

    默认值为 domain-dir/logs/access。

  13. （可选的）在“其他属性”部分中，指定虚拟服务器的其他属性。

    有关可用属性的信息，请参见特定于虚拟服务器的属性。

    • 要添加属性，请单击“添加属性”按钮。在显示的空白行中，在“名称”字段中键入属性名称，在“值”字段中键入属性值。
    • 要修改属性，请编辑该属性的“值”字段。
    • 要删除属性，请选中位于您要删除属性的“名称”字段左边的复选框，然后单击“删除属性”按钮。
  14. 单击“保存”。

    提示 - 要返回到默认设置，请单击“装入默认值”按钮，然后单击“保存”。
