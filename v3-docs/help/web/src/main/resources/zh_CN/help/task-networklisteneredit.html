编辑网络监听程序

  1. 在导航树中，展开“配置”节点。

  2. 在“配置”节点下，展开“网络配置”节点。

  3. 在“网络配置”节点下，选择“网络监听程序”节点。

    此时将打开“网络监听程序”页。

  4. 在“网络监听程序”页上，单击要编辑的网络监听程序的名称。

    此时将打开“编辑网络监听程序”页。

  5. 选择或取消选择“启用状态”复选框，以更改网络监听程序的状态。

    监听程序可能已启用或已禁用。默认情况下启用此选项。如果禁用了监听程序，则只要尝试连接到监听程序，便会导致套接字异常 (java.net.ConnectException)。

  6. 选中“已启用安全性”复选框，为网络监听程序使用的协议启用安全性。

    默认情况下禁用此选项。

  7. 选中“启用 JK 监听程序”复选框，指定监听程序支持 Apache 的 mod-jk 协议。

    mod-jk 协议是 Apache 的 httpd (Web 服务器) 与 GlassFish Server 后端实例之间的专有通信/网络协议。如果监听程序支持此协议，则假定 httpd 处理所有静态内容，而针对动态资源 (即 Servlet 和 JSP) 的任何请求，都将使用 mod-jk 协议路由到 GlassFish Server 的启用 mod-jk 的网络监听程序。默认情况下禁用此选项。

  8. 在“端口”字段中，键入网络监听程序将监听的端口号。

    合法值为 1 到 65535。在 UNIX 系统上，创建用于监听 1 至 1024 端口的套接字需要超级用户权限。

  9. 在“地址”字段中，键入网络监听程序将监听的 IP 地址。

    该地址可以采用点对形式或 IPv6 表示法。它可以为 any (对于 INADDR_ANY)，监听所有 IP 地址。它可以为主机名。

  10. 从“线程池”下拉列表中，选择要与网络监听程序关联的线程池。

  11. 单击“保存”。

    提示:

    要返回到默认设置，请单击“加载默认值”按钮，然后单击“保存”。
后续步骤

可以使用 SSL、HTTP 和“文件高速缓存”选项卡来编辑与网络监听程序关联的协议的设置。只有在为网络监听程序启用了安全性时，SSL 选项卡才有意义。
