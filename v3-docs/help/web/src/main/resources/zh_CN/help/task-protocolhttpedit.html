编辑协议的 HTTP 设置

  1. 在导航树中，展开“配置”节点。
  2. 在“配置”节点下，展开“网络配置”节点。
  3. 在“网络配置”节点下，选择“协议”节点。

    此时将打开“协议”页面。

  4. 在“协议”页面上，单击要编辑的协议的名称。

    此时将打开“编辑协议”页面。

  5. 在“编辑协议”页面上，单击 "HTTP" 选项卡。

    此时将打开 "HTTP" 页面。

  6. 在“服务器名称”字段中，键入要在服务器发送给客户机的 URL 中使用的主机名。

    如果服务器使用别名，则此名称为别名。如果服务器未使用别名，请将此字段保留为空。

  7. 从“默认虚拟服务器”下拉式列表中，选择要与此协议关联的虚拟服务器。

    使用“虚拟服务器”页面可以定义虚拟服务器。

  8. 在“重定向端口”字段中，键入要用于将请求重定向至另一端口的端口值。

    如果满足以下两个条件，GlassFish Server 将自动重定向请求：

    • 此协议支持非 SSL 请求。

    • 接收匹配安全性约束需要 SSL 传输的请求。

    如果未指定重定向端口，则 GlassFish Server 会使用原始请求中指定的端口号。

  9. 在“最大连接数”字段中，键入要维持在保持活动模式的持久性连接最大数目。

    保持活动子系统会定期轮询空闲连接。

  10. 在“超时”字段中，键入可以认为连接处于空闲状态并维持在保持活动状态的最长时间（以秒为单位）。

    如果值为 0 或小于 0，则表示保持活动连接将无限期地保持打开状态。默认值为 30 秒。

  11. 选中“启用上载超时”复选框，可以在到达“连接上载超时”限制之后，关闭缓慢读取字节的 Servlet 连接。

    如果禁用此选项，则 Servlet 连接将不会超时。默认情况下禁用此选项。

  12. 在“连接上载超时”字段中，键入上载的超时，以毫秒为单位。

    只有在选中“启用上载超时”复选框时，此字段才可用。默认值为 300000 毫秒。

  13. 在“请求超时”字段中，键入请求超时之前经过的秒数。

    如果在到达超时值之前未处理请求，则请求将被忽略。默认值为 30 秒。

  14. 在“发送缓冲区大小”字段中，键入发送缓冲区的大小（以字节为单位）。

    默认值为 8192 字节。

  15. 在“标头缓冲区长度”字段中，键入请求处理线程用于读取请求数据的缓冲区的大小（以字节为单位）。

    默认值为 8192 字节。

  16. 在“最大 POST 大小”字段中，键入 POST 操作的最大大小（以字节为单位）。

    默认值为 2097152 字节。

  17. 在“URI 编码”字段中，键入用于对接收的请求 URI 进行解码的字符集的名称。

    该值必须是有效的 IANA 字符集名称。默认值为 UTF-8。

  18. 在“版本”字段中，键入使用的 HTTP 协议的版本。

    默认值为 HTTP/1.1。

  19. 从“压缩”下拉式列表中，选择使用 HTTP/1.1 GZIP 压缩来节约服务器带宽。

    可用选项为：

    on

    压缩数据。

    off

    禁用压缩。

    force

    在所有情况下强制执行数据压缩。

    默认值为 off。

  20. 在“可压缩 Mime 类型”字段中，键入使用 HTTP 压缩的 MIME 类型的逗号分隔列表。

    默认值为 text/html,text/xml,text/plain。

  21. 在“最小压缩大小”字段中，键入应用压缩时，文件的最小大小。

    如果“压缩”设置为 on 或 force，则必须设置此值。默认值为 2048 字节。

  22. 在“无压缩用户代理”字段中，键入匹配不应使用压缩的 HTTP 客户机的用户代理的正则表达式列表，以逗号分隔。

    默认情况下，此值为空字符串。

  23. 在“受限用户代理”字段中，键入应用了 HTTP 压缩的受限用户代理的列表。

    如果未指定用户代理，则 HTTP 压缩将应用到所有用户代理。默认情况下，未指定用户代理。

  24. 在“默认响应类型”字段中，键入指定默认响应类型的字符串。

    此格式是以分号分隔的字符串，由内容类型、编码、语言和字符集组成。默认值为 text/html; charset=iso-8859-1。

  25. 在“强制响应类型”字段中，键入一个字符串，该字符串指定在没有与文件扩展名匹配的 MIME 映射时使用的请求类型。

    此格式是以分号分隔的字符串，由内容类型、编码、语言和字符集组成。默认值为 text/html; charset=iso-8859-1。

  26. 在“适配器”字段中，键入静态资源适配器的类名。

    默认值为 com.sun.grizzly.tcp.StaticResourcesAdapter。

  27. 选中“启用 Comet 支持”复选框以为协议启用 Comet 支持。

    默认情况下禁用此选项。

  28. 选中“启用 DNS 查找”复选框以启用域名系统 (DNS) 查找。

    默认情况下禁用此选项。

  29. 选中“启用 RCM 支持”复选框以启用资源配置管理 (RCM)。

    默认情况下禁用此选项。

  30. 选中“启用跟踪”复选框以启用跟踪操作。

    如果禁用此选项，则 GlassFish Server 将不易受到跨站点脚本攻击。默认情况下启用此选项。

  31. 选中“验证传递”复选框，以指示使用此协议的网络侦听器接收来自 SSL 终端代理服务器的流量。

    默认情况下禁用此选项。

  32. 选中“启用分块”复选框以启用 HTTP 响应分块。

    默认情况下启用此选项。

  33. 选中“启用 XPowered By”复选框，以指示 GlassFish Server 根据 Java Servlet 3.0 和 Java Server Pages 2.0 规范使用 X-Powered-By 标头。

    默认情况下启用此选项。

  34. 单击“保存”。

    提示 - 要返回到默认设置，请单击“装入默认值”按钮，然后单击“保存”。
