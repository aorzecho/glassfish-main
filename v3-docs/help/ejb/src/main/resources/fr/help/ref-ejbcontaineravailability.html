Disponibilité du conteneur EJB

La page Disponibilité du conteneur EJB permet de configurer les paramètres du conteneur EJB.

Elle contient les options suivantes.

Charger les valeurs par défaut

Bouton de restauration des valeurs par défaut des paramètres qui ont des valeurs par défaut. Les paramètres qui n'ont pas de valeurs par défaut ne sont pas modifiés.

Nom de configuration

Nom de la configuration à laquelle s'appliquent les paramètres de cette page. Ce champ est en lecture seule.

Service de disponibilité

Si la case Activé est cochée, la disponibilité du conteneur EJB est activée.

Type de persistance HA

Mécanisme de persistance utilisé lorsque le service de disponibilité est activé. La valeur par défaut est replicated, ce qui fournit une persistance basée sur le basculement en cas d'incident dans un cluster. Ne modifiez pas cette valeur.

Type de persistance SFSB

Mécanisme de persistance utilisé lorsque le service de disponibilité n'est pas activé. La valeur par défaut est file, ce qui fournit une persistance basée sur le système de fichiers. Ne modifiez pas cette valeur.

Le champ Emplacement de stockage des sessions détermine l'endroit où les états de point de reprise SFSB sont conservés. Pour plus d'informations sur la définition de l'emplacement de stockage des sessions, reportez-vous à Configurer les paramètres EJB.

Autres propriétés

Propriétés supplémentaires de la disponibilité du conteneur EJB.

GlassFish Server ne définit aucune autre propriété de disponibilité du conteneur EJB.
