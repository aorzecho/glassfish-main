Pour configurer le service d’horloge EJB

Le service d’horloge est un service de notification persistant et transactionnel fourni par le conteneur de beans entreprise qui sert à programmer des notifications ou des événements utilisés par les beans entreprise. Tous les beans entreprise, à l’exception des beans de session avec état, peuvent recevoir des notifications du service d’horloge. Les horloges persistantes définies par le service ne sont pas détruites lors de l’arrêt ou du redémarrage du serveur.

  1. Dans l’arborescence de navigation, développez le nœud Configuration.
  2. Sélectionnez le nœud Conteneur EJB.

    La page Enterprise Java Beans (EJB) s’ouvre.

  3. Cliquez sur l’onglet Service d’horloge EJB.

    La page Service d’horloge EJB s’ouvre.

  4. Dans la champ Intervalle de distribution minimal, indiquez le nombre minimum de millisecondes autorisées avant la prochaine expiration d’une horloge particulière.

    La définition de cet intervalle sur une valeur trop basse peut provoquer une surcharge du serveur. La valeur par défaut est 1000.

  5. Dans la champ Nombre maximal de redistributions, indiquez le nombre maximal de tentatives effectuées par le service d’horloge pour transmettre une expiration d’horloge en raison d’une exception ou d’une annulation.

    La valeur par défaut est 1.

  6. Dans le champ Intervalle de redistribution, indiquez l’ntervalle, exprimé en millisecondes, entre deux tentatives de redistribution. .

    La valeur par défaut est 5000.

  7. Dans le champ Source de données de l’horloge, tapez le nom JNDI de la ressource JDBC à utiliser comme source de données de l’horloge.

    Si elle est spécifiée, cette valeur remplace la valeur par défaut de jdbc/__TimerPool de l’application du système de service d’horloge.

  8. Cliquez sur Enregistrer.

    Astuce - Pour revenir aux paramètres par défaut, cliquez sur le bouton Charger valeurs par défaut, puis sur Enregistrer.

  9. Redémarrez GlassFish Server.
