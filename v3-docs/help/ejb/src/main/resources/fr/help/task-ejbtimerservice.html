Configurer le service d'horloge EJB

Le service d'horloge est un service de notification persistant et transactionnel fourni par le conteneur d'Enterprise Beans qui sert à programmer des notifications ou des événements utilisés par les Enterprise Beans. Tous les Enterprise Beans, à l'exception des beans Session avec conservation de statut, peuvent recevoir des notifications du service d'horloge. Les horloges persistantes définies par le service ne sont pas détruites lors de l'arrêt ou du redémarrage du serveur.

  1. Dans l'arborescence de navigation, développez le noeud Configuration.

  2. Sélectionnez le noeud Conteneur EJB.

    La page Enterprise Java Beans (EJB) apparaît.

  3. Cliquez sur l'onglet Service d'horloge EJB.

    La page Service d'horloge EJB apparaît.

  4. Dans le champ Intervalle de distribution minimal, indiquez le délai minimal, en millisecondes, au-delà duquel l'expiration suivante d'une horloge donnée survient.

    La définition de cet intervalle sur une valeur trop basse peut provoquer une surcharge du serveur. La valeur par défaut est 1 000.

  5. Dans le champ Nombre maximal de redistributions, indiquez le nombre maximal de tentatives effectuées par le service d'horloge pour transmettre une expiration d'horloge en raison d'une exception ou d'une annulation.

    La valeur par défaut est 1.

  6. Dans le champ Intervalle de redistribution, indiquez le délai, en millisecondes, entre deux tentatives de redistribution.

    La valeur par défaut est 5 000.

  7. Dans le champ Source de données de l'horloge, indiquez le nom JNDI de la ressource JDBC à utiliser comme source de données de l'horloge.

    Si elle est spécifiée, cette valeur remplace la valeur par défaut de jdbc/__TimerPool de l'application du système de service d'horloge.

  8. Cliquez sur Enregistrer.

    Conseil:

    Pour revenir aux paramètres par défaut, cliquez sur le bouton Charger les valeurs par défaut, puis sur Enregistrer.

  9. Redémarrez GlassFish Server.
