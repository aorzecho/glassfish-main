Para configurar el servicio de temporizador EJB

El servicio de temporizador consiste en un servicio de notificaciones transaccional y continuo proporcionado por el contenedor de Enterprise Bean. Se utiliza para planificar notificaciones o eventos utilizados por Enterprise Beans. All enterprise beans except stateful session beans can receive notifications from the timer service. Los temporizadores continuos establecidos por el servicio no se destruyen si el servidor se apaga o se reinicia.

  1. En el árbol de navegación, amplíe el nodo Configuración.
  2. Seleccione el nodo Contenedor EJB.

    Se abre la página de Enterprise Java Beans (EJB).

  3. Haga clic en la ficha Servicio de temporizador EJB.

    Se abre la página de Servicio de temporizador EJB.

  4. En el campo Intervalo mínimo de entrega, especifique la cantidad mínima de milisegundos permitidos antes de que expire el temporizador de un determinado temporizador.

    Si se establece un intervalo demasiado bajo, el servidor se puede sobrecargar. El valor predeterminado es 1.000.

  5. En el campo Cantidad máxima de intentos, especifique el número máximo de intentos que el servicio de temporizador efectúa para entregar una expiración de temporizador que vence por excepción o por una operación de deshacer.

    El valor predeterminado es 1.

  6. En el campo Intervalo de reintento de entrega, escriba el intervalo, en milisegundos, entre reintentos de entrega.

    El valor predeterminado es 5.000.

  7. En el campo Fuente de datos de temporizador, escriba el nombre JNDI del recurso de JDBC que se usará como fuente de datos de temporizador.

    Si se especifica, este valor, se anula el valor predeterminado de jdbc/__TimerPool para la aplicación del sistema de servicio de temporizador.

  8. Haga clic en Guardar.

    Consejo - Para volver a los valores de configuración predeterminados, haga clic en el botón Cargar predeterminados y, a continuación, en Guardar.

  9. Reinicie Servidor GlassFish.
