Servicio de Termporizador EJB

Utilice la página de Servicio de temporizador EJB para configurar el servicio de temporizador EJB.

El servicio de temporizador consiste en un servicio de notificaciones transaccional y continuo proporcionado por el contenedor de Enterprise Bean. Se utiliza para planificar notificaciones o eventos utilizados por Enterprise Beans. El servicio de temporizador puede enviar notificaciones a todos los Enterprise Beans menos los beans de sesión con estado. Los temporizadores continuos establecidos por el servicio no se destruyen si el servidor se apaga o se reinicia.

La página de Servicio de temporizador EJB contiene las opciones siguientes.

Cargar valores predeterminados

Botón para restaurar la configuración que tiene valores por defecto a estos valores. La configuración que no tiene valores por defecto no se modifica.

Nombre de Configuración

Nombre de la configuración a la que se aplican los valores de esta página. Este campo es de sólo lectura.

Intervalo mínimo de entrega

Cantidad mínima de minisegundos antes de que suceda la próxima expiración de temporizador de un determinado temporizador. Si se establece un intervalo demasiado bajo, el servidor se puede sobrecargar. El valor predeterminado es 1.000.

Número máximo de reintentos de entrega

Cantidad máxima de intentos que el servicio de temporizador efectúa para entregar una expiración de temporizador que vence por excepción o por una operación de deshacer. El valor predeterminado es 1.

Intervalo de reintento de entrega

Intervalo en milisegundos entre reintentos de entrega. El valor predeterminado es 5.000.

Fuente de datos de temporizador

El nombre JNDI del recurso de JDBC que se usará como fuente de datos de temporizador. Si se especifica, este valor anula el valor por defecto de jdbc/__TimerPool para la aplicación del sistema de servicio de temporizador.
