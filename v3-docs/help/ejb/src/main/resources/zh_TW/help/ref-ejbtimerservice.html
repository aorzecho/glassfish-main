EJB 計時器服務

使用 「EJB 計時器服務」 頁面可組態 EJB 計時器服務。

計時器服務是由企業 Bean 容器提供，用於排程企業 Bean 使用的通知或事件的永久性作業事件通知服務。所有企業 Bean (有狀態階段作業 Bean 除外) 均可從計時器服務接收通知。當伺服器關閉或重新啟動時，不會銷毀服務設定的永久性計時器。

「EJB 計時器服務」 頁面包含下列選項。

載入預設值

用於將具有預設值的設定復原成其預設值的按鈕。沒有預設值的設定保持不變。

組態名稱

此頁面上的設定套用的組態名稱。本欄位是唯讀的。

最小傳送間隔

特定計時器的下一個計時器到期之前允許的最小毫秒數。如果此間隔時間設定得過小，可能會導致伺服器超載。預設值為 1000。

最大重新傳送次數

計時器服務傳送異常或回復所造成之計時器到期的最大嘗試次數。預設值為 1。

重新傳送間隔

兩次重新傳送嘗試之間的間隔毫秒數。預設值為 5000。

計時器資料來源

要用做計時器資料來源之 JDBC 資源的 JNDI 名稱。若指定，會以此值置換計時器服務系統應用程式的 jdbc/__TimerPool 預設值。
