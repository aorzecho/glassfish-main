为 EJB 容器配置可用性设置

  1. 在导航树中，展开“配置”节点。

  2. 选择您要更改的配置。

  3. 在配置的节点下，选择“可用性服务”节点。

    此时将打开“可用性服务”页。

  4. 在“可用性服务”页上，单击“EJB 容器可用性”选项卡。

    此时将打开“EJB 容器可用性”页。

  5. 选中“已启用可用性服务”复选框，以启用 EJB 容器的可用性。

    要禁用可用性，请清除“可用性服务”的“已启用”框中的复选标记。

  6. “HA 持久类型”为 replicated。

    这是选中“已启用可用性服务”时所使用的持久性机制。持久性基于集群中的故障转移。请勿更改此值。

  7. “SFSB 持久类型”为 file。

    这是未选中“已启用可用性服务”时所使用的持久性机制。持久性基于文件系统。请勿更改此值。

    “会话存储位置”字段确定持久保存 SFSB 检查点状态的位置。要设置“会话存储位置”，请参阅配置 EJB 设置。

  8. 要还原默认值，请单击“加载默认值”按钮。

  9. 在“其他属性”部分中，指定其他属性。

    GlassFish Server 不定义 EJB 容器可用性的任何其他属性。

    • 要添加属性，请单击“添加属性”按钮。在显示的空白行中，在“名称”字段中键入属性名称，在“值”字段中键入属性值。

    • 要修改属性，请编辑该属性的“值”字段。

    • 要删除属性，请选中位于您要删除属性的“名称”字段左边的复选框，然后单击“删除属性”按钮。

  10. 单击“保存”。
