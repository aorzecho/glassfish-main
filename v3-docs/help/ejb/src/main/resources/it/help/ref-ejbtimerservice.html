Servizio timer EJB

Nella pagina Servizio timer EJB è possibile configurare il servizio timer EJB.

Il servizio timer è un servizio di notifica persistente e transazionale che viene fornito dal contenitore di bean enterprise e viene utilizzato per pianificare notifiche o eventi utilizzati dai bean enterprise. Tutti i bean enterprise possono ricevere notifiche dal servizio timer, a eccezione dei bean di sessione con conservazione dello stato. I timer persistenti impostati dal servizio non vengono eliminati quando il server viene chiuso o riavviato.

La pagina Servizio timer EJB contiene le opzioni riportate di seguito.

Carica impostazioni predefinite

Pulsante che consente di ripristinare i valori predefiniti delle impostazioni che dispongono di tali valori. Le impostazioni prive di valori predefiniti restano invariate.

Nome configurazione

Nome della configurazione a cui vengono applicate le impostazioni definite in questa pagina. Questo campo è di sola lettura.

Intervallo minimo di consegna

Numero minimo di millisecondi consentito prima della scadenza del timer successivo per un determinato timer. L'impostazione di tale intervallo su valori eccessivamente bassi può causare il sovraccarico del server. Il valore predefinito è 1000.

N. massimo ripetizioni consegna

Numero massimo di tentativi eseguiti dal servizio timer per riattivare la scadenza di un timer a causa di un'eccezione o del rollback. Il valore predefinito è 1.

Intervallo tra tentativi di consegna

Intervallo, in millisecondi, tra i tentativi di consegna. Il valore predefinito è 5000.

Origine dati timer

Nome JNDI della risorsa JDBC che verrà utilizzata come origine dati del timer. Se specificato, questo valore sostituisce il valore predefinito di jdbc/__TimerPool per l'applicazione di sistema del servizio timer.
