EJB-Timerservice

Auf der Seite "EJB-Timerservice" können Sie den EJB-Timerservice konfigurieren.

Der Timerservice ist ein vom Enterprise Bean-Container bereitgestellter, persistenter Transaktionsbenachrichtigungsservice, mit dem von Enterprise Beans verwendete Benachrichtigungen oder Ereignisse geplant werden. Alle Enterprise Beans akzeptieren Stateful Session-Beans und können Benachrichtigungen vom Timerservice empfangen. Durch den Service gesetzte persistente Timer werden nicht zerstört, wenn der Server heruntergefahren oder neu gestartet wird.

Die Seite "EJB-Timerservice" enthält die folgenden Optionen.

Standardeinstellungen laden

Schaltfläche zum Wiederherstellen der Standardwerte für Einstellungen, für die es Standardwerte gibt. Einstellungen, die nicht über Standardwerte verfügen, können nicht geändert werden.

Name der Konfiguration

Der Name der Konfiguration, für die die Einstellungen auf dieser Seite angewendet werden. Dieses Feld ist schreibgeschützt.

Minimales Lieferintervall

Die Mindestanzahl Millisekunden, die verstreichen dürfen, bis der nächste Timerablauf für einen bestimmten Timer auftreten kann. Ein zu niedrig gewählter Wert für diese Einstellung kann zu einer Serverüberlastung führen. Der Standardwert ist 1000.

Maximale erneute Sendungen

Die maximale Anzahl an Versuchen, die der Timerservice unternimmt, um einen Timerablauf aufgrund einer Ausnahme oder einer Zurücksetzung zu senden. Der Standardwert beträgt 1.

Intervall für erneutes Senden

Die Zeit zwischen erneuten Sendeversuchen in Millisekunden ein. Der Standardwert lautet 5000.

Timerdatenquelle

Der JNDI-Name der JDBC-Ressource, die als Timerdatenquelle verwendet werden soll. Wenn dieser Wert angegeben ist, überschreibt er den Standardwert von jdbc/__TimerPool für die Timerservice-Systemanwendung.

Verwandte Aufgaben

  • So wird der EJB-Timerservice konfiguriert

  • So werden EJB-Einstellungen konfiguriert

  • So werden MDB-Pooleinstellungen konfiguriert

Zugehörige asadmin-Befehle

list-timers(1)
