Para configurar o serviço de temporizador EJB

O serviço de temporizador é um serviço de notificação persistente e transacional que é fornecido pelo recipiente do bean corporativo e utilizado para programar notificações ou eventos utilizados por beans corporativos. Todos os beans corporativos, exceto os beans de sessão com estado, podem receber notificações do serviço de temporizador. Os temporizadores persistentes definidos pelo serviço não são destruídos quando o servidor é desligado ou reiniciado.

  1. Na árvore de navegação, expanda o nó Configuração.
  2. Selecione o nó Recipiente EJB.

    A página Enterprise Java Beans (EJB) é aberta.

  3. Clique na guia Serviço de temporizador EJB.

    A página Serviço de temporizador EJB é aberta.

  4. No campo Intervalo mínimo de entrega, digite o número mínimo de milissegundos permitidos antes que o próxima vencimento do temporizador para um determinado temporizador possa ocorrer.

    A definição deste intervalo com um valor muito baixo pode causar a sobrecarga do servidor. O valor padrão é 1000.

  5. No campo Máximo de reentregas, digite o número máximo de tentativas feitas pelo serviço de temporizador para entregar um vencimento do temporizador devido à exceção ou ao retorno.

    O valor padrão é 1.

  6. No campo Intervalo de reentrega, digite o intervalo, em segundos, entre a as tentativas de reentrega.

    O valor padrão é 5.000.

  7. No campo Origem dos dados do temporizador, digite o nome JNDI do recurso JDBC que será utilizado como a origem dos dados do temporizador.

    Se especificado, este valor sobrepõe o valor padrão de jdbc/__TimerPool para o aplicativo do sistema do serviço de temporizador.

  8. Clique em Salvar.

    Dica - Para restaurar as configurações padrão, clique no botão Carregar padrões e, em seguida, em Salvar.

  9. Reinicie o Servidor GlassFish.
