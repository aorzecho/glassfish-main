Serviço de temporizador EJB

Use a página Serviços de temporizador EJB para configurar o serviço de temporizador EJB.

O serviço de temporizador é um serviço de notificação persistente e transacional que é fornecido pelo recipiente do bean corporativo, e é usado para programar notificações ou eventos usados por beans corporativos. Todos os beans corporativos, exceto os beans de sessão com estado, podem receber notificações do serviço de temporizador. Os temporizadores persistentes definidos pelo serviço não são destruídos quando o servidor é desligado ou reiniciado.

A página Serviço de temporizador EJB contém a seguintes opções:

Carregar padrões

Botão para restaurar com os valores padrão as configurações que apresentam valores padrão. As configurações que não apresentam valores padrão não são alteradas.

Configuration Name

O nome da configuração para a qual as definições nesta página se aplicam. Este campo é de somente leitura.

Intervalo mínimo de entrega

O número mínimo de milissegundos permitidos antes que o próximo vencimento do temporizador para um determinado temporizador possa ocorrer. Definir esse intervalo com um valor muito baixo pode causar a sobrecarga do servidor. O valor padrão é 1000.

Máximo de reentregas

O número máximo de tentativas feitas pelo serviço de temporizador para entregar um vencimento do temporizador devido à exceção ou ao retorno. O valor padrão é 1.

Intervalo de reentrega

O intervalo em milissegundos entre as tentativas de reentrega. O valor padrão é 5.000.

Origem dos dados do temporizador

O nome JNDI do recurso JDBC que será usado como a origem dos dados do temporizador. Se especificado, este valor sobrepõe o valor padrão de jdbc/__TimerPool para o aplicativo do sistema do serviço de temporizador.
