Nueva Fábrica de Conexiones JMS

Utilice la página de Nueva Fábrica de Conexiones JMS para crear una fábrica de conexiones JMS nueva.

La página Nueva Fábrica de Conexiones JMS contiene las siguientes opciones.

Nombre de Pool

Un nombre único que identifica la fábrica de conexiones. Servidor GlassFish establece el nombre JNDI de la fábrica de conexiones en este valor.

Tipo de Recurso

El tipo de la fábrica de conexiones. Las opciones disponibles son javax.jms.ConnectionFactory, javax.jms.QueueConnectionFactory y javax.jms.TopicConnectionFactory.

Descripción

Una descripción de la fábrica de conexiones.

Estado

El estado de la fábrica de conexiones. La fábrica de conexiones se puede activar o desactivar. Esta opción se encuentra activada por defecto.

Tamaño de Pool Inicial y Mínimo

El número mínimo de conexiones del pool para la fábrica de conexiones. El valor por defecto es 8.

Tamaño de Pool Máximo

El número máximo de conexiones del pool para la fábrica de conexiones. El valor por defecto es 32.

Cantidad de Cambio de Tamaño del Pool

El número de conexiones que se pueden eliminar cuando se agota el timeout de inactividad del pool. El valor por defecto es 2.

Timeout de Inactividad

El tiempo máximo en segundos que una conexión puede permanecer inactiva en el pool. El valor por defecto es 300.

Tiempo de Espera Máximo

La cantidad de tiempo que la aplicación que solicita una conexión puede esperar para obtener un mensaje de timeout de conexión. El valor por defecto es 60000.

Ante Cualquier Fallo

Si se selecciona la casilla de verificación Cerrar Todas las Conexiones, Servidor GlassFish cerrará todas las conexiones del pool y las restablecerá aunque tan sólo falle una. Esta opción se encuentra desactivada por defecto.

Soporte de Transacción

El soporte de transacción seleccionado sustituye el atributo de soporte de la transacción en el adaptador de recursos asociado con el pool de conexiones de forma descendente compatible. Dicho de otro modo, puede admitir un nivel de transacción inferior o idéntico al que se especifica en el adaptador de recursos, pero no puede indicar un nivel superior.

El valor puede ser cualquiera de los siguientes:

XATransaction

El recurso se puede utilizar para transacciones que suponen el uso de más de un recurso dentro de un ámbito de transacción. Éste es el valor por defecto de una fábrica de conexiones JMS.

Por ejemplo, en las transacciones puede participar este recurso más un recurso JDBC, un recurso de conector u otro recurso de fábrica de conexiones JMS. Este valor ofrece la máxima flexibilidad. Un recurso que se configure como XATransaction participará en operaciones de confirmación de dos fases.

LocalTransaction

Este recurso se puede utilizar bien para transacciones que impliquen solamente un recurso dentro del ámbito de la transacción o bien como el último agente en una transacción distribuida que involucre a más de un recurso XA.

Este valor ofrece un rendimiento mucho más óptimo. Un recurso que se configura como LocalTransaction no se utilizará en operaciones de confirmación de dos fases.

NoTransaction

Este recurso no puede participar nunca en transacciones. Esta configuración es de uso limitado en aplicaciones JMS.

Validación de Conexión

Si la casilla de verificación Necesaria está seleccionada, las conexiones se validarán antes de proporcionárselas a la aplicación. Si falla la validación de un recurso, éste se destruye, y se crea y se devuelve un recurso nuevo. Esta opción se encuentra desactivada por defecto.

Propiedades Adicionales

Son las propiedades adicionales de la fábrica de conexiones.

Para obtener información sobre las propiedades disponibles, consulte Propiedades Específicas de las Fábricas de Conexiones JMS.

Destinos

Clústeres e instancias independientes para la fábrica de conexiones. Mueva los destinos seleccionados a la columna Destinos Seleccionados mediante los botones Agregar y Agregar Todos. Mueva los destinos que no necesite a la columna Destinos Disponibles mediante los botones Eliminar y Eliminar Todos.

Esta opción sólo se muestra si en el dominio se han creado instancias de cluster o independientes.
