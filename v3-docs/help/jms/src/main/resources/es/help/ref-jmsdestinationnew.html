Nuevo recurso de destino JMS

Utilice la página Nuevo recurso de destino JMS para crear un nuevo recurso de destino JMS.

La página Nuevo recurso de destino JMS contiene las siguientes opciones.

Nombre JNDI

Un nombre exclusivo que identifica el recurso de destino.

Es una práctica recomendada utilizar el prefijo de subcontexto de nomenclatura jms/ para recursos JMS. Por ejemplo: jms/Queue.

Nombre de destino físico

El nombre del destino físico que se asocia a este recurso.

GlassFish Server creará el destino físico automáticamente cuando sea necesario y lo eliminará cuando se elimine el recurso de destino.

Tipo de recurso

El tipo de recurso de destino. Las opciones disponibles son javax.jms.Topic y javax.jms.Queue.

Descripción

Una descripción del recurso de destino.

Estado

El estado del recurso de destino. El recurso de destino se puede habilitar o deshabilitar. Esta opción se habilita por defecto.

Propiedades adicionales

Propiedades adicionales para el recurso de destino. ProductName; no define ninguna propiedad adicional para los recursos de destino JMS.

Destinos

Instancias de clúster e independientes para el recurso de destino. Mueva los destinos seleccionados a la columna Destinos seleccionados mediante los botones Agregar y Agregar todos. Mueva cualquier destino innecesario a la columna Destinos disponibles mediante los botones Eliminar y Eliminar todos.

Esta opción sólo se muestra si se han creado en el dominio las instancias de clúster o independientes.
