Disponibilidad de JMS

Utilice la página Disponibilidad de JMS para configurar las opciones de disponibilidad para el Java Message Service (JMS).

La página Disponibilidad JMS contiene las siguientes opciones.

Cargar valores predeterminados

Botón para restablecer las opciones que tienen valores por defecto a estos. Las opciones que no tienen valores por defecto no se modifican.

Configuration Name

El nombre de la configuración a la que aplicar los valores de esta página. Este campo es de sólo lectura.

Servicio de disponibilidad

El tipo de clúster de Message Queue usado para proporcionar servicios JMS a un clúster de GlassFish. Si la casilla de verificación Habilitado aparece seleccionada, se utilizará un clúster de Cola de mensajes mejorado. De lo contrario, se utilizará un clúster de Cola de mensajes convencional.

Si la casilla de verificación Habilitado está seleccionada, se ignorarán las opciones Tipo de almacén de configuración y Tipo de almacén de mensajes.

Tipo de almacén de configuración

El tipo de almacén de datos para los datos de configuración de un clúster de Cola de mensajes convencional. El valor masterbroker especifica el uso de un agente maestro para almacenar y gestionar los datos de configuración. El valor shareddb especifica el uso de una base de datos compartida para almacenar los datos de configuración.

Esta opción se ignora si la opción Servicio de disponibilidad está habilitada.

Tipo de almacén de mensajes

El tipo de almacén de datos para datos de los mensajes en los agentes de un clúster convencional. El valor file especifica un almacén de archivos. El valor JDBC especifica un almacén JDBC.

Esta opción se ignora si la opción Servicio de disponibilidad está habilitada.

Proveedor de la base de datos
Nombre de usuario de la base de datos
Contraseña de la base de datos
URL de acceso a la base de datos

El proveedor de la base de datos, el usuario, la contraseña y la URL de acceso de la base de datos JDBC que debe utilizarse en cualquiera de estas situaciones:

  • Cuando la opción Servicio de disponibilidad está habilitada

  • Cuando la opción Tipo de almacén de configuración está definida en shareddb

  • Cuando la opción Tipo de almacén de mensajes está definida en jdbc

Para obtener información sobre los proveedores admitidos y los formatos de las URL de acceso para cada proveedor, consulte JDBC-Based Persistence de Oracle GlassFish Server Message Queue 4.5 Administration Guide.

Propiedades adicionales

Propiedades adicionales de la disponibilidad de JMS.

Servidor GlassFish no predefine ninguna propiedad de disponibilidad de JMS adicional.
