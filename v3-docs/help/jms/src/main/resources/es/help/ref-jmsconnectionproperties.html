Propiedades específicas de las fábricas de conexión JMS

Las siguientes propiedades adicionales están disponibles para las fábricas de conexión JMS.

ClientID

Especifica un ID de cliente para una fábrica de conexión que utilizará una suscripción duradera.

AddressList

Indique el nombre (y, de forma opcional, número de puerto) de una instancia o instancias del agente de mensajes con el que se comunicarán las aplicaciones. Cada una de las direcciones de la lista incluye el nombre del host (y, de forma opcional, el puerto del host y el servicio de conexión) para efectuar la conexión. Por ejemplo, el valor podría ser earth o earth:7677. Especifique el número de puerto si el agente de mensajes se está ejecutando en un puerto distinto del predeterminado (7676). Si la configuración de propiedades indica varios hosts y puertos en un entorno de clústeres, se utiliza el primer host disponible de la lista, a menos que la propiedad AddressListBehavior se defina en RANDOM.

Para obtener más detalles, consulte la Guía del desarrollador de Message Queue para clientes Java.

El valor predeterminado es el host local y el número de puerto predeterminado (7676). El cliente intentará una conexión a un agente en el puerto 7676 del host local.

UserName

El nombre de usuario para la fábrica de conexión.

El valor predeterminado es guest.

Contraseña

La contraseña para la fábrica de conexión.

El valor predeterminado es guest.

ReconnectEnabled

Si está establecida como true, indica que el tiempo de ejecución del cliente intenta volver a conectar con un servidor de mensajes (o la lista de direcciones de AddressList) cuando se pierde una conexión.

El valor predeterminado es verdadero.

ReconnectAttempts

Especifica el número de intentos de conectar (o volver a conectar) para cada dirección de la AddressList antes de que el tiempo de ejecución del cliente continúe con la siguiente dirección de la lista. El valor -1 indica que el número de intentos de reconexión es ilimitado (el tiempo d ejecución del cliente intenta conectar con la primera dirección hasta que lo consigue).

El valor predeterminado es 3.

ReconnectInterval

Especifica en milisegundos el intervalo entre dos intentos de reconexión. Se refiere a intentos en cada dirección de la AddressList y direcciones sucesivas de la lista. Si el intervalo es demasiado corto, el agente no tiene tiempo de recuperar. Si es demasiado largo, la reconexión puede suponer un retraso inaceptable.

El valor predeterminado es 30000.

AddressListBehavior

Especifica si los intentos de conexión se realizan en el orden de las direcciones del atributo AddressList (PRIORITY) o de forma aleatoria (RANDOM).

RANDOM significa que la reconexión selecciona una dirección al azar de la AddressList. Si es probable que muchos clientes intenten una conexión con la misma fábrica de conexión, este valor evita que se conecten todos a la misma dirección.

PRIORITY significa que la reconexión siempre intenta conectar con la primera dirección del servidor en la AddressList y utiliza otra dirección sólo si el primer agente no está disponible.

El valor predeterminado es RANDOM.

AddressListIterations

Especifica el número de veces que el tiempo de ejecución del cliente se repite a través de la AddressList en un esfuerzo por establecer (o restablecer) una conexión. El valor -1 indica que el número de intentos es ilimitado.

El valor predeterminado es 3.
