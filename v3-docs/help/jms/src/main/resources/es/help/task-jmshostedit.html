Editar un Host JMS

  1. En el árbol de navegación, amplíe el nodo Configuración.

  2. Select the instance to configure:

    • To configure a particular instance, select the instance's config node. For example, for the default instance, server, select the server-config node.

    • To configure the default settings for future instances that use a copy of default-config, select the default-config node.

  3. En el nodo Configuración, amplíe el nodo Java Message Service.

  4. En el nodo Java Message Service, seleccione el nodo Hosts JMS.

    Se abrirá la página Hosts JMS.

  5. En la página Hosts JMS, seleccione el host que va a editar.

    Se abrirá la página Editar Host JMS.

  6. En el campo Host, introduzca el nombre o la dirección IP (Internet Protocol) del sistema donde se ejecutará el host JMS (localhost o el nombre del sistema local o remoto).

  7. En el campo Puerto, introduzca el número de puerto del servicio JMS.

    Modifique este campo sólo si el servicio JMS que va a utilizar se ejecuta en un puerto que no es el que se utiliza por defecto. El puerto por defecto es 7676.

  8. En los campos Nombre de Usuario Administrador y Contraseña de Administrador, introduzca el nombre de usuario y la contraseña del broker de Message Queue.

    Éstos son diferentes del nombre de usuario y la contraseña de Servidor GlassFish. Edite estos campos solamente si los valores del broker de Message Queue se han modificado mediante el comando as-install-parent/mq/bin/imqusermgr. Los valores por defecto son admin y admin.

  9. Haga clic en Guardar.

    Sugerencia:

    Para volver a la configuración por defecto, haga clic en el botón Cargar Valores por Defecto y, a continuación, en Guardar.
