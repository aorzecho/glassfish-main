Para crear un recurso de destino JMS

Cuando se crea un recurso de destino JMS, Servidor GlassFish crea un recurso de objeto de administrador para el recurso de destino. Puede visualizar este recurso bajo el nodo Conectores.

  1. En el árbol de navegación, amplie el nodo Recursos.
  2. En el nodo Recursos, amplíe el nodo Recursos JMS.
  3. En el nodo Recursos JMS, seleccione el nodo Recursos de destino JMS.

    Se abrirá la página Recursos de destino JMS.

  4. En la página Recursos de destino JMS, haga clic en Nuevo.

    Se abrirá la página Nuevo recurso de destino JMS.

  5. En el campo Nombre JNDI, introduzca el nombre JNDI del recurso.

    Es una práctica recomendada utilizar el prefijo de subcontexto de nomenclatura jms/ para recursos JMS. Por ejemplo: jms/Queue.

  6. En el campo Nombre de destino físico, introduzca el nombre del destino físico que se asociará a este recurso.

    Servidor GlassFish creará el destino físico automáticamente cuando sea necesario y lo eliminará cuando se elimine el recurso de destino.

  7. En la lista desplegable Tipo de recurso, seleccione javax.jms.Topic o javax.jms.Queue.
  8. En el campo Descripción, escriba una descripción del recurso de destino.
  9. Seleccione la casilla de verificación Estado habilitado para habilitar el recurso en tiempo de ejecución.

    Esta opción se habilita por defecto.

  10. (Opcional) En el área de Propiedades adicionales, indique las propiedades adicionales para el recurso de destino.

    ProductName; no define ninguna propiedad adicional para los recursos de destino JMS.

    Para añadir una propiedad, haga clic en el botón Añadir propiedad. En la fila vacía escriba el nombre de la propiedad en el campo Nombre y el valor de la propiedad en el campo Valor.

  11. Haga clic en Aceptar.
