Crear un Destino Físico JMS

Normalmente, no es necesario crear destinos físicos, ya que se especifica un nombre de destino físico al crear un recurso de destino JMS y Servidor GlassFish crea automáticamente el destino físico cuando hace falta.

  1. En el árbol de navegación, amplíe el nodo Configuración.

  2. Select the instance to configure:

    • To configure a particular instance, select the instance's config node. For example, for the default instance, server, select the server-config node.

    • To configure the default settings for future instances that use a copy of default-config, select the default-config node.

  3. En el nodo Configuración, amplíe el nodo Java Message Service.

  4. En el nodo Java Message Service, seleccione el nodo Destinos Físicos.

    Se abrirá la página Destinos Físicos.

  5. En la página Destinos Físicos, haga clic en Nuevo.

    Se abrirá la página Nuevo Destino Físico.

  6. En el campo Nombre, introduzca el nombre del destino físico.

  7. En la lista desplegable Tipo, seleccione javax.jms.Topic o javax.jms.Queue.

  8. En el campo Número Máximo de Mensajes no Consumidos, escriba el número máximo de mensajes no consumidos permitido para el destino.

    Un valor de -1 indica que no hay límite para el número de mensajes no consumidos. El valor por defecto es -1.

  9. En el campo Tamaño Máximo de Mensaje, especifique el tamaño máximo en bytes que puede tener un mensaje.

    Un valor de -1 indica un tamaño de mensaje ilimitado. El valor por defecto es -1.

  10. En el campo Memoria Máxima Total, escriba la cantidad total de memoria destinada a los mensajes no consumidos, en bytes.

    Un valor de -1 indica que no hay límite para el tamaño de memoria destinado a los mensajes no consumidos. El valor por defecto es -1.

  11. En la lista desplegable Comportamiento Límite de Umbral, seleccione el comportamiento que ha de observar el broker intermediario cuando se alcance el umbral del límite de la memoria.

    • Rechazar los mensajes más recientes le indica al broker que debe rechazar los mensajes más recientes y notificárselo al cliente que los produzca mediante una excepción, esto último solamente si el mensaje es persistente.

      Rechazar los mensajes más recientes es el valor por defecto.

    • Ralentizar productores indica al broker que reduzca la velocidad y la frecuencia con que envían mensajes los productores.

    • Rechazar los más antiguos indica al broker que descarte los mensajes más antiguos.

    • Rechazar los mensajes con prioridad más baja indica al broker que descarte los mensajes con la prioridad más baja.

    Si el valor esRechazar los más antiguos o Rechazar los mensajes con prioridad más baja y Usar Cola de Mensajes Inoperativos se establecen en "True", los mensajes que superen el límite se mueven a la cola de mensajes inoperativos.

  12. En el campo Número Máximo de Productores, escriba el número máximo de productores de mensajes que se pueden asociar a este destino físico.

    Cuando se alcanza este límite, no se pueden crear nuevos productores. El valor -1 indica un número ilimitado de productores. El valor por defecto es 100.

  13. En el campo Recuento Máximo de Mensajes de Entrega de Lotes, escriba el número máximo de mensajes que se pueden entregar a los usuarios en un solo lote.

    Un valor de -1 indica un número ilimitado de mensajes. El valor por defecto es 1.000.

  14. Desde la lista desplegable Usar Cola de Mensajes Inoperativos, seleccione "True" para colocar los mensajes inoperativos en la cola de mensajes inoperativos.

    Si el valor se establece en "False", los mensajes inoperativos se descartarán. Por defecto, esta opción está establecida en "True".

  15. Desde la lista desplegable de activación de Validar Esquema XML, seleccione "True" para activar la validación del esquema XML para el destino.

    Si el valor se establece en "True", el tiempo de ejecución del cliente de Message Queue intentará validar un mensaje de XML en comparación con las definiciones del esquema XML especificadas (o comparándolo con el DTD, si es que no se especifica XSD) antes de enviarlo al broker. Si el esquema especificado no se puede localizar o no es posible validar el mensaje, éste no se enviará y aparecerá una excepción. Por defecto, esta opción está establecida en "False".

  16. En el campo Lista URI de Esquema XML, escriba una lista separada por espacios de cadenas URI de definición de esquema XML (XSD).

    Si está activada la validación del esquema XML, las URI indican la ubicación de uno o más XSD que se emplearán para la validación del esquema XML.
