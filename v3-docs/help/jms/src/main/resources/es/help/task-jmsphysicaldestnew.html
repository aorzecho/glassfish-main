Para crear un destino físico JMS

Normalmente, no es necesario crear destinos físicos, ya que se especifica un nombre de destino físico al crear un recurso de destino JMS y GlassFish Server crea automáticamente el destino físico cuando hace falta.

  1. En el árbol de navegación, amplie el nodo Configuración.
  2. En el nodo Configuración, seleccione y expanda el nodo Java Message Service.
  3. En el nodo Java Message Service, seleccione el nodo Destinos físicos.

    Se abrirá la página Destinos físicos.

  4. En la página Destinos físicos, haga clic en Nuevo.

    Se abrirá la página Nuevo destino físico.

  5. En el campo Nombre, introduzca el nombre del destino físico.
  6. En la lista desplegable Tipo, seleccione javax.jms.Topic o javax.jms.Queue.
  7. En el campo Número máximo de mensajes no consumidos, escriba el número máximo de mensajes no consumidos permitido para el destino.

    Un valor de -1 indica que no hay límite para el número de mensajes no consumidos. El valor predeterminado es -1.

  8. En el campo Tamaño máximo de mensaje, especifique el tamaño máximo en bytes que puede tener un mensaje.

    Un valor de -1 indica un tamaño de mensaje ilimitado. El valor predeterminado es -1.

  9. En el campo Total máximo de memoria, escriba la cantidad total de memoria destinada a los mensajes no consumidos, en bytes.

    Un valor de -1 indica que no hay límite para el tamaño de memoria destinado a los mensajes no consumidos. El valor predeterminado es -1.

  10. En la lista desplegable Comportamiento de umbral límite, seleccione el comportamiento que ha de observar el agente intermediario cuando se alcance el umbral del límite de la memoria.
    • Rechazar los mensajes más recientes le indica al agente intermediario que debe rechazar los mensajes más recientes y notificárselo al cliente que los produzca mediante una excepción, esto último solamente si el mensaje es persistente.

      Rechazar los mensajes más recientes es el valor predeterminado.

    • Ralentizar productores indica al agente intermediario que reduzca la velocidad y la frecuencia con que envían mensajes los productores.
    • Eliminar los más antiguos indica al agente intermediario que descarte los mensajes más antiguos.
    • Eliminar los mensajes con prioridad más baja indica al agente intermediario que descarte los mensajes con la prioridad más baja.

    Si el valor esEliminar los más antiguos o Eliminar los mensajes con la prioridad más baja y Utilizar la cola de mensajes inactivos se establecen en "True", los mensajes que superen el límite se mueven a la cola de mensajes inactivos.

  11. En el campo Número máximo de productores, escriba el número máximo de productores de mensajes que se pueden asociar a este destino físico.

    Cuando se alcanza este límite, no se pueden crear nuevos productores. Un valor de -1 indica un número ilimitado de productores. El valor predeterminado es 100.

  12. En el campo Recuento máx. de mensajes de entrega por lotes, escriba el número máximo de mensajes que se pueden entregar a los usuarios en un sólo lote.

    Un valor de -1 indica un número ilimitado de mensajes. El valor predeterminado es 1.000.

  13. Desde la lista desplegable Utilizar la cola de mensajes inactivos, seleccione "True" para colocar los mensajes inactivos en la cola de mensajes inactivos.

    Si el valor se establece en "False", los mensajes inactivos se descartarán. De manera predeterminada, esta opción está establecida en "True".

  14. Desde la lista desplegable Validar el esquema XML habilitado, seleccione "True" para habilitar la validación del esquema XML para el destino.

    Si el valor se establece en "True", el tiempo de ejecución del cliente de Message Queue intentará validar un mensaje de XML en comparación con las definiciones del esquema XML especificadas (o comparándolo con el DTD, si es que no se especifica XSD) antes de enviarlo al agente intermediario. Si el esquema especificado no se puede localizar o no es posible validar el mensaje, éste último no se enviará y aparecerá una excepción. De forma predeterminada, esta opción está establecida en "False".

  15. En el campo Lista URI del esquema XML, escriba una lista separada por espacios de cadenas URI de definición de esquema XML (XSD).

    Si está habilitada la validación del esquema XML, las URI indican la ubicación de uno o más XSD que se emplearán para la validación del esquema XML.
