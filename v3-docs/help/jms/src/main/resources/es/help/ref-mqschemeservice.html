Sintaxis de las direcciones de esquema MQ y servicio MQ

La sintaxis completa para una dirección del servicio de mensajes es

scheme://address-syntax

donde scheme y address-syntax se describen en la tabla siguiente.

Esquema MQ y Servicio MQ son los valores mostrados en las dos primeras columnas de la siguiente tabla.

Nombre de esquema

Servicio de conexión

Descripción

Sintaxis de dirección

mq

jms y ssljms

El tiempo de ejecución del cliente Message Queue realiza una conexión al asignador de puertos MQ en el host y puerto especificados. El asignador de puertos suministra una lista de los puertos de servicio de conexión establecidos de forma dinámica y, a continuación, el tiempo de ejecución del cliente Message Queue realiza una conexión al puerto que aloja el servicio de conexión especificado.

[hostName][: port][/serviceName ]

Valores predeterminados: hostName = localhost, port = 7676, serviceName = jms

Los valores predeterminados sólo se aplican al servicio de conexión jms. Para la conexión de servicio ssljms, es necesario especificar todas las variables.

Ejemplo: mq:MyHost:7677/ssljms

mqtcp

jms

El tiempo de ejecución del cliente Message Queue realiza una conexión TCP al host y puerto especificados (ignorando el asignador de puertos MQ) para establecer una conexión.

hostName:port /jms

Ejemplo: mqtcp:localhost:7676/jms

mqssl

ssljms

El tiempo de ejecución del cliente Message Queue realiza una conexión SSL segura al host y puerto especificados (ignorando el asignador de puertos MQ) para establecer una conexión.

hostName:port /ssljms

Ejemplo: mqssl:localhost:7676/ssljms

http

httpjms

El tiempo de ejecución del cliente Message Queue realiza una conexión HTTP a un servlet de túnel Message Queue en la URL especificada. (El agente se debe configurar para acceder al servlet de túnel de HTTP, según se describe en la Guía del administrador de Message Queue.)

hostName:port /contextRoot/tunnel

Si varias instancias del agente utilizan el mismo servlet de túnel, la sintaxis para conectar a una instancia de agente específica (en vez de hacerlo a una seleccionada de forma aleatoria) es: http://nombrehost :puerto/contextRoot /tunnel?nombreservidor=nombrehost: nombreinstancia

https

httpsjms

Message Queue realiza una conexión HTTPS segura a la dirección URL especificada del servlet de túnel de Message Queue. (El agente se debe configurar para acceder al servlet de túnel de HTTPS, según se describe en la Guía del administrador de Message Queue.)

hostName:port /contextRoot/tunnel

Si varias instancias del agente utilizan el mismo servlet de túnel, la sintaxis para conectar a una instancia de agente específica (en vez de hacerlo a una seleccionada de forma aleatoria) es: http://nombrehost :puerto/contextRoot /tunnel?nombreservidor=nombrehost: nombreinstancia
