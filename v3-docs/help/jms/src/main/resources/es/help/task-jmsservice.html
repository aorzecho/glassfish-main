Configurar el Servicio JMS

Utilice la página Servicio JMS para ajustar la configuración que se va a utilizar en todas las conexiones JMS.

Para obtener más información sobre cómo configurar el servicio JMS, consulte el Capítulo 19 sobre cómo utilizar el Java Message Service, en la Guía de Desarrollo de la Aplicación de Oracle GlassFish Server.

  1. En el árbol de navegación, amplíe el nodo Configuración.

  2. Select the instance to configure:

    • To configure a particular instance, select the instance's config node. For example, for the default instance, server, select the server-config node.

    • To configure the default settings for future instances that use a copy of default-config, select the default-config node.

  3. En el nodo Configuración, seleccione el nodo Java Message Service.

    Se abrirá la página del Servicio JMS.

  4. En la lista desplegable Tipo:

    • Seleccione EMBEDDED para acceder al servicio JMS en el host local.

      El servicio JMS se inicia en el mismo equipo JVM que Servidor GlassFish. La inicialización mínima inicia el broker por defecto incrustado al acceder por primera vez a los servicios JMS, en lugar de al iniciar Servidor GlassFish. Éste es el tipo por defecto de servicio JMS.

    • Seleccione LOCAL para acceder al servicio JMS en el host local.

      El servicio JMS se inicia junto con Servidor GlassFish.

    • Seleccione REMOTE para acceder al servicio JMS en otro sistema or on a cluster.

      Si selecciona REMOTE, Servidor GlassFish no iniciará el servicio JMS la siguiente vez que se inicie el servidor. En su lugar, el servicio JMS se inicia y se gestiona a través de Message Queue; por tanto, debe iniciar el broker Message Queue por separado. Para obtener información sobre cómo iniciar el broker, consulte el apartado sobre cómo iniciar el broker de Oracle GlassFish Server Message Queue Administration Guide. Si selecciona este valor y está utilizando un host remoto, siga las instrucciones de la sección Editar un Host JMS para especificar el nombre del host remoto.

  5. En el campo Timeout de Inicio, escriba el número de segundos que Servidor GlassFish esperará para que se inicie el servicio JMS antes de anular el inicio.

    Para un sistema lento o sobrecargado, aumente el valor por defecto. El valor por defecto es 60.

  6. En el campo Argumentos de Inicio, introduzca los argumentos para personalizar el inicio del servicio JMS.

    Utilice los argumentos disponibles mediante el comando as-install-parent/mq/bin/imqbrokerd.

  7. Seleccione la casilla de verificación de activación de Volver a Conectar para que el servicio JMS intente volver a conectar con un servidor de mensajes (o la lista de direcciones de la propiedad AddressList) cuando se pierda una conexión.

    Esta opción se encuentra activada por defecto.

  8. En el campo Intervalo de Reconexión, introduzca los segundos que habrá entre intentos de reconexión.

    Este intervalo se aplica a intentos con cada dirección de la AddressList y direcciones sucesivas de la lista. Si es demasiado corto, este intervalo de tiempo no dará tiempo al broker para la recuperación. Si es demasiado largo, la reconexión puede suponer un retraso inaceptable. El valor por defecto es de 5 segundos.

  9. En el campo Intentos de Reconexión, introduzca el número de intentos de conexión (o reconexión) para cada dirección de la lista de direcciones antes de que el tiempo de ejecución del cliente continúe con la siguiente dirección de la lista.

    El valor -1 indica que el número de intentos de reconexión es ilimitado (el tiempo de ejecución del cliente intenta conectar con la primera dirección hasta que lo consigue). El valor por defecto es 3.

  10. En la lista desplegable Host JMS por Defecto, seleccione el nombre del host JMS por defecto.

    El valor por defecto es default_JMS_host.

  11. De la lista desplegable Comportamiento de Lista de Direcciones, seleccione el orden de intentos de conexión.

    • random significa que se selecciona una dirección de la lista de direcciones AddressList de forma aleatoria.

      Si hay muchos clientes que intentan una conexión utilizando la misma fábrica de conexiones, especifique RANDOM para evitar que todos se conecten a la misma dirección. Ésta es la opción por defecto.

    • PRIORITY significa que la reconexión siempre intenta conectar con la primera dirección del servidor existente en la lista de direcciones y utiliza otra dirección sólo si el primer broker no está disponible.

  12. En el campo Iteraciones de Lista de Direcciones, introduzca el número de veces que el servicio JMS se repite en la lista de direcciones en un esfuerzo por establecer (o restablecer) una conexión.

    El valor -1 indica que el número de intentos es ilimitado. El valor por defecto es 3.

  13. En los campos Esquema MQ y Servicio MQ, introduzca el nombre de esquema de la dirección Message Queue y el nombre de servicio de la conexión Message Queue si se va a utilizar un esquema o un servicio que no es el que se usa por defecto.

    Consulte Sintaxis de las Direcciones de Esquema MQ y Servicio MQ para ver información sobre la sintaxis.

  14. En el área de Propiedades Adicionales, indique las propiedades adicionales del servicio JMS.

    Para obtener información sobre las propiedades disponibles, consulte Propiedades Específicas del Servicio JMS.

    • Para agregar una propiedad, haga clic en el botón Agregar Propiedad. En la fila en blanco que aparece, escriba el nombre de propiedad en el campo Nombre y escriba el valor de propiedad en el campo Valor.

    • Para modificar una propiedad, edite su campo Valor.

    • Para suprimir una propiedad, seleccione la casilla de verificación a la izquierda del campo Nombre de la propiedad que está suprimiendo y, a continuación, haga clic en el botón Suprimir Propiedades.

  15. Haga clic en Ping para comprobar que el servicio JMS se está ejecutando en el host JMS por defecto.

    Si el servicio JMS está iniciado y en ejecución, un mensaje informa que el ping se ha realizado correctamente.

  16. Haga clic en Guardar.

    Sugerencia:

    Para volver a la configuración por defecto, haga clic en el botón Cargar Valores por Defecto y, a continuación, en Guardar.
Acceso a Servidores Remotos

Cambiar el proveedor y el host a un sistema remoto hace que todas las aplicaciones JMS se ejecuten en el servidor remoto. Para utilizar el servidor local y uno o varios servidores remotos, cree un recurso de fábrica de conexiones con la propiedad AddressList para crear conexiones que accedan a servidores remotos. Consulte Crear un Recurso de Fábricas de Conexiones JMS.

Comando equivalente a asadmin

jms-ping
