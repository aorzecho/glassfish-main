Cómo configurar el servicio JMS

Utilice la página Servicio JMS para ajustar los parámetros de configuración que se van a utilizar en todas las conexiones JMS.

Para obtener más información sobre cómo configurar el servicio JMS , consulte el Capítulo 17, Using the Java Message Service de Oracle GlassFish Server 3.1 Application Development Guide.

  1. En el árbol de navegación, expanda el nodo Configuración.
  2. En el nodo Configuración, seleccione el nodo Java Message Service.

    Se abrirá la página del servicio JMS.

  3. En la lista desplegable Tipo:
    • Seleccione EMBEDDED para acceder al servicio JMS en el host local.

      El servicio JMS se inicia en la misma máquina JVM que GlassFish Server. La inicialización mínima inicia el agente intermediario predeterminado incrustado al acceder por primera vez a los servicios JMS, en lugar de al iniciar GlassFish Server. Éste es el tipo predeterminado de servicio JMS.

    • Seleccione LOCAL para acceder al servicio JMS en el host local.

      El servicio JMS se inicia junto con GlassFish Server.

    • Seleccione REMOTE para acceder al servicio JMS en otro sistema.

      Si selecciona REMOTE, GlassFish Server no iniciará el servicio JMS la siguiente vez que se inicie el servidor. En su lugar, el servicio JMS se inicia y se administra a través de Message Queue; por tanto, debe iniciar el agente Message Queue por separado. Para obtener información sobre cómo iniciar el agente, consulte Starting Brokers de Oracle GlassFish Message Queue 4.5 Administration Guide. Si selecciona este valor y está utilizando un host remoto, siga las instrucciones de la sección Para editar un host JMS para especificar el nombre del host remoto.

  4. En el campo Tiempo de espera de inicio, escriba el número de segundos que GlassFish Server esperará para que se inicie el servicio JMS antes de anular el inicio.

    Para un sistema lento o sobrecargado, aumente el valor predeterminado. El valor predeterminado es 60.

  5. En el campo Argumentos de inicio, introduzca los argumentos para personalizar el inicio del servicio JMS.

    Puede utilizar todos los argumentos disponibles a través del comando as-install-parent/mq/bin/imqbrokerd.

  6. Seleccione la casilla de verificación Volver a conectar habilitado para que el servicio JMS intente volver a conectar con un servidor de mensajes (o la lista de direcciones de la propiedad AddressList) cuando se pierda una conexión.

    Esta opción se habilita por defecto.

  7. En el campo Intervalo de reconexión, introduzca los segundos que habrá entre intentos de reconexión.

    Este intervalo se refiere a intentos con cada dirección de la AddressList y direcciones sucesivas de la lista. Si es demasiado corto, este intervalo de tiempo no dará tiempo al agente para recuperar. Si es demasiado largo, la reconexión puede suponer un retraso inaceptable. El valor predeterminado es de 5 segundos.

  8. En el campo Intentos de reconexión, introduzca el número de intentos de conexión (o reconexión) para cada dirección de la lista de direcciones antes de que el tiempo de ejecución del cliente continúe con la siguiente dirección de la lista.

    El valor -1 indica que el número de intentos de reconexión es ilimitado (el tiempo d ejecución del cliente intenta conectar con la primera dirección hasta que lo consigue). El valor predeterminado es 3.

  9. En la lista desplegable Host JMS predeterminado, seleccione el nombre del host JMS predeterminado.

    El valor predeterminado es default_JMS_host.

  10. De la lista desplegable Comportamiento de la lista de direcciones, seleccione el orden de intentos de conexión.
    • random significa que se selecciona una dirección de la lista de direcciones AddressList de forma aleatoria.

      Si hay muchos clientes que intentan una conexión utilizando la misma fábrica de conexión, especifique RANDOM para evitar que todos se conecten a la misma dirección. Ésta es la opción predeterminada.

    • PRIORITY significa que la reconexión siempre intenta conectar con la primera dirección del servidor existente en la lista de direcciones y utiliza otra dirección sólo si el primer agente no está disponible.
  11. En el campo Interacciones de lista de direcciones, introduzca el número de veces que el servicio JMS se repite en la lista de direcciones en un esfuerzo por establecer (o restablecer) una conexión.

    El valor -1 indica que el número de intentos es ilimitado. El valor predeterminado es 3.

  12. En los campos Esquema MQ y Servicio MQ, introduzca el nombre de esquema de la dirección Message Queue y el nombre de servicio de la conexión Message Queue si se va a utilizar un esquema o un servicio que no es el predeterminado.

    Consulte Sintaxis de las direcciones de esquema MQ y servicio MQ para obtener información sobre la sintaxis.

  13. (Opcional) En el área de Propiedades adicionales, indique las propiedades adicionales del servicio JMS.

    Para obtener información sobre las propiedades disponibles, consulte Propiedades específicas del servicio JMS.

    • Para añadir una propiedad, haga clic en el botón Añadir propiedad. En la fila vacía escriba el nombre de la propiedad en el campo Nombre y el valor de la propiedad en el campo Valor.
    • Para modificar una propiedad, edite su campo Valor.
    • Para eliminar una propiedad, seleccione la casilla de verificación a la izquierda del campo Nombre de la propiedad que está eliminando y, a continuación, haga clic en el botón Eliminar propiedades.
  14. Haga clic en Ping para comprobar que el servicio JMS se está ejecutando en el host JMS predeterminado.

    Si el servicio JMS está iniciado y en ejecución, un mensaje informa que el ping se ha realizado correctamente.

  15. Haga clic en Guardar.

    Consejo - Para volver a los valores de configuración predeterminados, haga clic en el botón Cargar predeterminados y, a continuación, en Guardar.

Acceso a servidores remotos

Cambiar el proveedor y el host a un sistema remoto hace que todas las aplicaciones JMS se ejecuten en el servidor remoto. Para utilizar el servidor local y uno o varios servidores remotos, cree un recurso de fábrica de conexión con la propiedad AddressList para crear conexiones que accedan a servidores remotos. Consulte Para crear un recurso de fábrica de conexión JMS.

Comando equivalente a asadmin

jms-ping
