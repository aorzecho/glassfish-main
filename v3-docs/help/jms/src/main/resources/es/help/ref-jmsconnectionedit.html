Editar fábrica de conexión JMS

Utilice la página Editar fábrica de conexión JMS para modificar los parámetros de configuración de una fábrica de conexión JMS.

La página Editar fábrica de conexión JMS contiene las siguientes opciones.

Cargar valores predeterminados

Botón para restablecer las opciones que tienen valores por defecto a estos. Las opciones que no tienen valores por defecto no se modifican.

Nombre de conjunto

El nombre exclusivo del conjunto de conexiones del conector creado para la fábrica de conexión. Este nombre también es el nombre JNDI de la fábrica de conexión. El nombre del conjunto es un campo sólo de lectura.

Tipo de recurso

El tipo de la fábrica de conexión. El tipo de recurso es un campo de sólo lectura. Sólo se puede especificar el tipo de recurso al crear una nueva fábrica de conexión JMS.

Descripción

Una descripción de la fábrica de conexión.

Estado

El estado de la fábrica de conexión. La fábrica de conexión se puede habilitar o deshabilitar. Esta opción se habilita por defecto.

Tamaño inicial y mínimo del conjunto

El número mínimo de conexiones del conjunto para la fábrica de conexión. El valor predeterminado es 8.

Tamaño de conjunto máximo

El número máximo de conexiones del conjunto. El valor predeterminado es 32.

Cantidad de cambio de tamaño del conjunto

El número de conexiones que se eliminarán cuando se llegue al límite el tiempo de espera del conjunto. El valor predeterminado es 2.

Tiempo de espera inactivo

El tiempo máximo en segundos que una conexión puede permanecer inactiva en el conjunto. El valor predeterminado es 300.

Tiempo de espera máximo

La cantidad de tiempo que la aplicación que solicita una conexión puede esperar para obtener la conexión. El valor predeterminado es 60000.

Con cualquier fallo

Si se selecciona la casilla de verificación Cerrar todas las conexiones, Servidor GlassFish cerrará todas las conexiones del conjunto y las restablecerá con que tan sólo falle una. Esta opción se deshabilita por defecto.

Compatibilidad de transacción

El valor puede ser cualquiera de los siguientes:

XATransaction

El recurso se puede utilizar para transacciones que suponen el uso de más de un recurso dentro de un ámbito de transacción. Éste es el valor predeterminado de una fábrica de conexión JMS.

Por ejemplo, en las transacciones pueden involucrarse este recurso más un recurso JDBC, un recurso de conector u otro recurso de fábrica de conexión JMS. Este valor ofrece la máxima flexibilidad. Un recurso que se configure como XATransaction participará en operaciones de confirmación de dos fases.

LocalTransaction

Este recurso se puede utilizar bien para transacciones que impliquen solamente un recurso dentro del ámbito de la transacción o bien como el último agente en una transacción distribuida que involucre a más de un recurso XA.

Este valor ofrece un rendimiento mucho más óptimo. Un recurso que se configura como LocalTransaction no se utilizará en operaciones de confirmación de dos fases.

NoTransaction

Este recurso no puede participar nunca en transacciones. Esta configuración es de uso limitado en aplicaciones JMS.

Validación de la conexión

Si la casilla de verificación Necesaria está seleccionada, las conexiones se validarán antes de proporcionárselas a la aplicación. Si falla la validación de un recurso, éste se destruye, para a continuación crear y devolver un recurso nuevo. Esta opción se deshabilita por defecto.

Propiedades adicionales

Son las propiedades adicionales de la fábrica de conexión.

Para obtener información sobre las propiedades disponibles, consulte Propiedades específicas de las fábricas de conexión JMS.
