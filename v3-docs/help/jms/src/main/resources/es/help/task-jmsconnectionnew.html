Crear un Recurso de Fábricas de Conexiones JMS

Cuando se crea una fábrica de conexiones JMS, Servidor GlassFish crea un pool de conexiones de conector y un recurso de conector para la fábrica de conexiones. Puede ver estos objetos en el nodo Conectores.

  1. En el árbol de navegación, amplíe el nodo Recursos.

  2. En el nodo Recursos, amplíe el nodo Recursos de JMS.

  3. En el nodo de Recursos de JMS, seleccione el nodo Fábricas de Conexiones JMS.

    Se abrirá la página Fábricas de Conexiones JMS.

  4. En la página Fábricas de Conexiones JMS, haga clic en Nuevo.

    Se muestra la página Nueva Fábrica de Conexiones JMS.

  5. En el campo Nombre de Pool, introduzca el nombre único de la fábrica de conexiones.

    Servidor GlassFish establece el nombre JNDI de la fábrica de conexiones en este valor.

  6. En la lista desplegable Tipo de Recurso, seleccione javax.jms.ConnectionFactory, javax.jms.QueueConnectionFactory o javax.jms.TopicConnectionFactory.

  7. En el campo Descripción, introduzca una descripción de la fábrica de conexiones.

  8. Seleccione la casilla de verificación de activación de Estado para activar el recurso en tiempo de ejecución.

    Esta opción se encuentra activada por defecto.

  9. En el campo Tamaño de Pool Inicial y Mínimo, introduzca el número mínimo de conexiones del pool.

    El valor por defecto es 8.

  10. En el campo Tamaño de Pool Máximo, introduzca el número máximo de conexiones de este pool.

    El valor por defecto es 32.

  11. En el campo Cantidad de Cambio de Tamaño del Pool, especifique el número de conexiones que se eliminarán cuando se supere el timeout límite de inactividad del pool.

    El valor por defecto es 2.

  12. En el campo Timeout de Inactividad, indique el tiempo máximo en segundos que una conexión puede permanecer inactiva en el pool.

    El valor por defecto es 300.

  13. En el campo Tiempo de Espera Máximo, indique la cantidad de tiempo que la aplicación que solicita una conexión puede esperar para obtener un mensaje de timeout de conexión.

    El valor por defecto es 60000.

  14. Seleccione la casilla de verificación Ante Cualquier Fallo Cerrar Todas las Conexiones, para que Servidor GlassFish cierre todas las conexiones del pool y las restablezca aunque tan sólo falle una.

    Si no selecciona esta casilla de verificación, las conexiones individuales se restablecerán sólo cuando vaya a utilizarlas. Esta opción se encuentra desactivada por defecto.

  15. Para un recurso de fábrica de conexiones JMS, seleccione el nivel de soporte de transacción.

    El valor seleccionado sobrescribe el atributo de soporte de transacción en el Adaptador de Recursos, de manera que es compatible en sentido descendente.

    • Seleccione XATransaction (el valor por defecto para el adaptador de recursos) para un recurso que se pueda utilizar para transacciones que supongan el uso de más de un recurso dentro de un ámbito de transacción.

      Por ejemplo, en las transacciones puede participar este recurso más un recurso JDBC, un recurso de conector u otro recurso de fábrica de conexiones JMS. Este valor ofrece la máxima flexibilidad. Un recurso que se configure como XATransaction participará en operaciones de confirmación de dos fases.

    • Seleccione LocalTransaction para un recurso que se puede utilizar para transacciones que impliquen sólo un recurso dentro del ámbito de la transacción o como el último agente en una transacción distribuida que implique más de un recurso XA.

      Este valor ofrece un rendimiento mucho más óptimo. Un recurso que se configure como LocalTransaction no se utilizará en operaciones de confirmación de dos fases.

    • Seleccione NoTransaction para un recurso que nunca puede participar en transacciones.

      Esta configuración es de uso limitado en aplicaciones JMS.

  16. Marque la casilla de verificación Validación de Conexión Necesaria para activar la validación de la conexión.

    Si se selecciona esta opción, las conexiones se validan antes de proporcionárselas a la aplicación. Si falla la validación de un recurso, éste se destruye, y se crea y se devuelve un recurso nuevo. Esta opción se encuentra desactivada por defecto.

  17. En la sección Propiedades Adicionales, especifique las propiedades adicionales de la fábrica de conexiones.

    Para obtener información sobre las propiedades disponibles, consulte Propiedades Específicas de las Fábricas de Conexiones JMS.

    Para agregar una propiedad, haga clic en el botón Agregar Propiedad. En la fila vacía, escriba el nombre de la propiedad en el campo Nombre y el valor de la propiedad en el campo Valor.

  18. In the Targets area, do the following:

    1. From the Available column, select the target or targets to which applications that use the resource will be deployed.

      The available targets include the available clusters and server instances as well as the default server instance, server.

    2. Click Add to move the target to the Selected column.

  19. Haga clic en Aceptar.
