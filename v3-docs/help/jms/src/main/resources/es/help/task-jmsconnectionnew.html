Para crear un recurso de fábrica de conexión JMS

Cuando se crea una fábrica de conexión JMS, GlassFish Server crea un conjunto de conexiones de conector y un recurso de conector para la fábrica de conexión. Puede ver estos objetos en el nodo Conectores.

  1. En el árbol de navegación, expanda el nodo Recursos.
  2. En el nodo Recursos, amplíe el nodo Recursos JMS.
  3. En el nodo de Recursos JMS, seleccione el nodo Fábricas de conexión JMS.

    Se abrirá la página Fábricas de conexión JMS.

  4. En la página Fábricas de conexión JMS, haga clic en Nuevo.

    Se muestra la página Nueva fábrica de conexión JMS.

  5. En el campo Nombre de conjunto, introduzca el nombre exclusivo de la fábrica de conexión.

    GlassFish Server establece el nombre JNDI de la fábrica de conexión en este valor.

  6. En la lista desplegable Tipo de recurso, seleccione javax.jms.ConnectionFactory, javax.jms.QueueConnectionFactory o javax.jms.TopicConnectionFactory.
  7. En el campo Descripción, introduzca una descripción de la fábrica de conexión.
  8. Seleccione la casilla de verificación Estado habilitado para habilitar el recurso en tiempo de ejecución.

    Esta opción se habilita por defecto.

  9. En el campo Tamaño de conjunto inicial y mínimo, introduzca el número mínimo de conexiones del conjunto.

    El valor predeterminado es 8.

  10. En el campo Tamaño de conjunto máximo, especifique el número máximo de conexiones del conjunto.

    El valor predeterminado es 32.

  11. En el campo Cantidad de redimensionamiento del conjunto, especifique el número de conexiones que se eliminarán cuando se upere el tiempo límite de espera de inactividad del conjunto.

    El valor predeterminado es 2.

  12. En el campo Tiempo de espera de inactividad, indique el tiempo máximo que una conexión puede permanecer inactiva en el conjunto, en segundos.

    El valor predeterminado es 300.

  13. En el campo Tiempo de espera máximo, indique la cantidad de tiempo que la aplicación que solicita una conexión puede esperar antes de superar el límite de espera para establecer la conexión.

    El valor predeterminado es 60000.

  14. Seleccione la casilla de verificación Cerrar todas las conexiones en caso de fallo, de manera que GlassFish Server cerrará todas las conexiones del conjunto y las restablecerá con que tan sólo se produzca el fallo de una conexión.

    Si no selecciona esta casilla de verificación, las conexiones individuales se restablecerán sólo cuando vaya a utilizarlas. Esta opción se deshabilita por defecto.

  15. Para un recurso de fábrica de conexión JMS, seleccione el nivel de compatibilidad de transacción.

    El valor seleccionado sobrescribe el atributo de compatibilidad de transacción en el adaptador de recursos, de manera que es compatible en sentido descendente.

    • Seleccione XATransaction (el valor predeterminado para el adaptador de recursos) para un recurso que se pueda utilizar para transacciones que supongan el uso de más de un recurso dentro de un ámbito de transacción.

      Por ejemplo, en las transacciones pueden involucrarse este recurso más un recurso JDBC, un recurso de conector u otro recurso de fábrica de conexión JMS. Este valor ofrece la máxima flexibilidad. Un recurso que se configure como XATransaction participará en operaciones de confirmación de dos fases.

    • Seleccione LocalTransaction para un recurso que se puede utilizar para transacciones que impliquen sólo un recurso dentro del ámbito de la transacción o como el último agente en una transacción distribuida que implique más de un recurso XA.

      Este valor ofrece un rendimiento mucho más óptimo. Un recurso que se configure como LocalTransaction no se utilizará en operaciones de confirmación de dos fases.

    • Seleccione NoTransaction para un recurso que nunca puede participar en transacciones.

      Esta configuración es de uso limitado en aplicaciones JMS.

  16. Marque la casilla de verificación Validación de conexión necesaria para habilitar la validación de la conexión.

    Si se selecciona esta opción, las conexiones se validan antes de proporcionárselas a la aplicación. Si falla la validación de un recurso, éste se destruye, para a continuación crear y devolver un recurso nuevo. Esta opción se deshabilita por defecto.

  17. (Opcional) En la sección Propiedades adicionales, especifique propiedades adicionales de la fábrica de conexión.

    Para obtener información sobre las propiedades disponibles, consulte Propiedades específicas de las fábricas de conexión JMS.

    Para añadir una propiedad, haga clic en el botón Añadir propiedad. En la fila vacía escriba el nombre de la propiedad en el campo Nombre y el valor de la propiedad en el campo Valor.

  18. Haga clic en Aceptar.
