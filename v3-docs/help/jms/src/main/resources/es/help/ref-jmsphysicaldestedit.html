Editar destino físico JMS

Utilice la página Editar destino físico JMS para modificar los parámetros de configuración de un destino físico JMS.

La página Editar destino físico JMS contiene las siguientes opciones.

Nombre

El nombre del destino físico. Éste es un campo de solo lectura. Solamente es posible especificar el nombre cuando se crea un recurso de destino JMS o un destino físico.

Tipo

El tipo del destino físico. Éste es un campo de solo lectura. Solamente es posible especificar el tipo cuando se crea un recurso de destino JMS o un destino físico.

Número máximo de mensajes no consumidos

El número máximo de mensajes no consumidos permitido para el destino. Un valor de -1 indica que no hay límite para el número de mensajes no consumidos. El valor predeterminado es -1.

Tamaño máximo de mensaje

El tamaño máximo que un mensaje puede tener, en bytes. Un valor de -1 indica un tamaño de mensaje ilimitado. El valor predeterminado es -1.

Memoria total máxima

El total máximo de memoria para mensajes no consumidos, en bytes. Un valor de -1 indica que no hay límite para el tamaño de memoria destinado a los mensajes no consumidos. El valor predeterminado es -1.

Comportamiento de umbral límite

El comportamiento que debe seguir el agente cuando se alcanza el umbral del límite de memoria. Las opciones disponibles son:

Rechazar los mensajes más recientes

Rechazar los mensajes más recientes y notificar al cliente que los produce con una excepción sólo si el mensaje es persistente. Éste es el valor predeterminado.

Ralentizar productores

Ralentizar la frecuencia con que los productores de mensajes los envían.

Eliminar los más antiguos

Descartar los mensajes más antiguos.

Eliminar los mensajes con prioridad más baja

Descartar los mensajes con la prioridad más baja.

Si el valor Eliminar los más antiguos o Eliminar los mensajes con prioridad más baja y la opción Utilizar la cola de mensajes inactivos está establecida como "True", los mensajes que superen el límite se moverán a la cola de mensajes inactivos.

Número máximo de productores

El número máximo de productores de mensajes que se pueden asociar a este destino físico. Cuando se alcanza este límite, no se pueden crear nuevos productores. El valor -1 indica un número ilimitado de productores. El valor predeterminado es 100.

Recuento máx. de mensajes de entrega por lotes

El número máximo de mensajes que pueden entregarse a los consumidores en un solo lote. Un valor de -1 indica un número ilimitado de mensajes. El valor predeterminado es 1.000.

Utilizar la cola de mensajes inactivos

Si esta opción está habilitada, los mensajes inactivos se colocan en la cola de mensajes inactivos. Si esta opción está deshabilitada, los mensajes inactivos se descartan. Esta opción se habilita por defecto.

Validar esquema XML

Si esta opción está habilitada, el tiempo de ejecución del cliente de Message Queue intentará validar un mensaje XML comparándolo con las definiciones de esquema XML especificadas (o bien comparándolo con DTD, si no hay ningún XSD especificado) antes de enviarlo al agente intermediario. Si el esquema especificado no se puede localizar o no es posible validar el mensaje, éste último no se enviará y aparecerá una excepción. Esta opción se deshabilita por defecto.

Lista URI de esquema XML

Una lista separada por espacios de cadenas URI (XSD) de definición de esquema XML. Si está habilitada la validación del esquema XML, las URI indican la ubicación de uno o más XSD que se emplearán para la validación del esquema XML.
