Editar Destino Físico JMS

Utilice la página Editar Destino Físico JMS para modificar la configuración de un destino físico JMS.

La página Editar Destino Físico JMS contiene las siguientes opciones.

Nombre

El nombre del destino físico. Éste es un campo de sólo lectura. Solamente es posible especificar el nombre cuando se crea un recurso de destino JMS o un destino físico.

Tipo

El tipo del destino físico. Éste es un campo de sólo lectura. Solamente es posible especificar el tipo cuando se crea un recurso de destino JMS o un destino físico.

Número Máximo de Mensajes no Consumidos

El número máximo de mensajes no consumidos permitido para el destino. Un valor de -1 indica que no hay límite para el número de mensajes no consumidos. El valor por defecto es -1.

Tamaño Máximo de Mensaje

El tamaño máximo que un mensaje puede tener, en bytes. Un valor de -1 indica un tamaño de mensaje ilimitado. El valor por defecto es -1.

Memoria Máxima Total

El total máximo de memoria para mensajes no consumidos, en bytes. Un valor de -1 indica que no hay límite para el tamaño de memoria destinado a los mensajes no consumidos. El valor por defecto es -1.

Comportamiento Límite de Umbral

El comportamiento que debe seguir el broker cuando se alcanza el umbral del límite de memoria. Las opciones disponibles son:

Rechazar los mensajes más recientes

Rechazar los mensajes más recientes y notificar al cliente que los produce con una excepción sólo si el mensaje es persistente. Éste es el valor por defecto.

Ralentizar productores

Ralentizar la frecuencia con que los productores de mensajes los envían.

Rechazar los más antiguos

Descartar los mensajes más antiguos.

Rechazar los mensajes con prioridad más baja

Descartar los mensajes con la prioridad más baja.

Si el valor Rechazar los más antiguos o Rechazar los mensajes con prioridad más baja y la opción Usar Cola de Mensajes Inoperativos está establecida como "True", los mensajes que superen el límite se moverán a la cola de mensajes inoperativos.

Número Máximo de Productores

El número máximo de productores de mensajes que se pueden asociar a este destino físico. Cuando se alcanza este límite, no se pueden crear nuevos productores. El valor -1 indica un número ilimitado de productores. El valor por defecto es 100.

Recuento Máximo de Mensajes de Entrega de Lotes

El número máximo de mensajes que pueden entregarse a los consumidores en un solo lote. Un valor de -1 indica un número ilimitado de mensajes. El valor por defecto es 1.000.

Usar Cola de Mensajes Inoperativos

Si esta opción está activada, los mensajes inoperativos se colocan en la cola de mensajes inoperativos. Si esta opción está desactivada, los mensajes inoperativos se descartan. Esta opción se encuentra activada por defecto.

Validar Esquema XML

Si esta opción está activada, el tiempo de ejecución del cliente de Message Queue intentará validar un mensaje XML comparándolo con las definiciones de esquema XML especificadas (o bien comparándolo con DTD, si no hay ningún XSD especificado) antes de enviarlo al broker. Si el esquema especificado no se puede localizar o no es posible validar el mensaje, éste no se enviará y aparecerá una excepción. Esta opción se encuentra desactivada por defecto.

Lista URI de Esquema XML

Una lista separada por espacios de cadenas URI (XSD) de definición de esquema XML. Si está activada la validación del esquema XML, las URI indican la ubicación de uno o más XSD que se emplearán para la validación del esquema XML.
