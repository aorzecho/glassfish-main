Servicio JMS

Utilice la página Servicio JMS para ajustar la configuración que se va a utilizar en todas las conexiones JMS.

Para obtener más información sobre cómo configurar el servicio JMS, consulte el Capítulo 19 sobre cómo utilizar el Java Message Service, en la Guía de Desarrollo de la Aplicación de Oracle GlassFish Server.

La página Servicio JMS contiene las siguientes opciones.

Cargar Valores por Defecto

Botón para restaurar la configuración que tiene valores por defecto a sus valores por defecto. La configuración que no tiene valores por defecto no se modifica.

Ping

Botón para comprobar que el servicio JMS se está ejecutando en el host JMS por defecto. Si el servicio JMS está iniciado y en ejecución, un mensaje informa que el ping se ha realizado correctamente.

Nombre de Configuración

Nombre de la configuración a la que se aplican los valores de esta página. Este campo es de sólo lectura.

Tipo

El tipo de acceso para el servicio JMS. Las opciones disponibles son:

EMBEDDED

Acceso al servicio JMS en el host local. El servicio JMS se inicia en el mismo equipo JVM que Servidor GlassFish. La inicialización mínima inicia el broker intermediario por defecto incrustado al acceder por primera vez a los servicios JMS, en lugar de al iniciar Servidor GlassFish. Éste es el tipo por defecto de servicio JMS.

LOCAL

Acceso al servicio JMS en el host local. El proveedor JMS se inicia junto con Servidor GlassFish.

REMOTE

Acceso al servicio JMS en otro sistema. Si selecciona REMOTE, Servidor GlassFish no iniciará el servicio JMS la siguiente vez que se inicie el servidor. En su lugar, el servicio JMS se inicia y se gestiona a través de Message Queue; por tanto, debe iniciar el broker Message Queue por separado. Para obtener información sobre cómo iniciar el broker, consulte el apartado sobre cómo iniciar el broker de Oracle GlassFish Server Message Queue Administration Guide. Si selecciona este valor y está utilizando un host remoto, siga las instrucciones de la sección Editar un Host JMS para especificar el nombre del host remoto.

Timeout de Inicio

El número de segundos que Servidor GlassFish esperará para que se inicie el servicio JMS antes de anular el inicio. Para un sistema lento o sobrecargado, aumente el valor por defecto. El valor por defecto es 60.

Argumentos de Inicio

Los argumentos para personalizar el inicio del servicio JMS. Utilice los argumentos disponibles mediante el comando as-install-parent/mq/bin/imqbrokerd.

Volver a Conectar Activada

Si se selecciona la casilla de verificación de activación de Volver a Conectar, el servicio JMS intentará volver a conectarse a un servidor de mensajes (o a la lista de direcciones de AddressList) cuando se pierda una conexión. Esta opción se encuentra activada por defecto.

Intervalo de Reconexión

El número de segundos entre los intentos de reconexión. Este intervalo se refiere a intentos con cada dirección de la AddressList y direcciones sucesivas de la lista. Si es demasiado corto, este intervalo de tiempo no permitirá al broker la recuperación. Si es demasiado largo, la reconexión puede suponer un retraso inaceptable. El valor por defecto es de 5 segundos.

Intentos de Reconexión

Especifica el número de intentos de conexión (o reconexión) para cada dirección de la AddressList antes de que el tiempo de ejecución del cliente continúe con la siguiente dirección de la lista. El valor -1 indica que el número de intentos de reconexión es ilimitado (el tiempo de ejecución del cliente intenta conectar con la primera dirección hasta que lo consigue). El valor por defecto es 3.

Host JMS por Defecto

El nombre del host JMS por defecto. El valor por defecto es default_JMS_host.

Comportamiento de Lista de Direcciones

El orden de intentos de conexión. Las opciones disponibles son:

random

Seleccione una dirección de lista de direcciones de forma aleatoria. Si hay muchos clientes que intentan una conexión utilizando la misma fábrica de conexiones, especifique RANDOM para evitar que todos se conecten a la misma dirección. Ésta es la opción por defecto.

priority

La reconexión siempre intenta volver a conectarse a la primera dirección de servidor de AddressList y utiliza otra solamente cuando el primer broker no está disponible.

Iteraciones de Lista de Direcciones

El número de veces que servicio JMS repite el recorrido por la lista AddressList en un esfuerzo por establecer (o restablecer) una conexión. El valor -1 indica que el número de intentos es ilimitado. El valor por defecto es 3.

Esquema MQ y Servicio MQ

El nombre de esquema de la dirección de Message Queue y el nombre del servicio de conexión de Message Queue si no se van a utilizar esquemas o servicios por defecto. Consulte Sintaxis de las Direcciones de Esquema MQ y Servicio MQ para ver información sobre la sintaxis.

Propiedades Adicionales

Propiedades adicionales del servicio JMS.

Para obtener información sobre las propiedades disponibles, consulte Propiedades Específicas del Servicio JMS.
