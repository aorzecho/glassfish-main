Servicio JMS

Utilice la página Servicio JMS para ajustar los parámetros de configuración que se van a utilizar en todas las conexiones JMS.

Para obtener más información sobre cómo configurar el servicio JMS, consulte el Capítulo 16, Using the Java Message Service de Oracle GlassFish Server 3.1 Application Development Guide.

La página Servicio JMS contiene las siguientes opciones.

Cargar valores predeterminados

Botón para restablecer las opciones que tienen valores por defecto a estos. Las opciones que no tienen valores por defecto no se modifican.

Ping

Botón para comprobar que el servicio JMS se está ejecutando en el host JMS predeterminado. Si el servicio JMS está iniciado y en ejecución, un mensaje informa que el ping se ha realizado correctamente.

Configuration Name

El nombre de la configuración a la que aplicar los valores de esta página. Este campo es de sólo lectura.

Tipo

El tipo de acceso para el servicio JMS. Las opciones disponibles son:

EMBEDDED

Acceso al servicio JMS en el host local. El servicio JMS se inicia en la misma máquina JVM que Servidor GlassFish. La inicialización mínima inicia el agente intermediario predeterminado incrustado al acceder por primera vez a los servicios JMS, en lugar de al iniciar Servidor GlassFish. Éste es el tipo predeterminado de servicio JMS.

LOCAL

Acceso al servicio JMS en el host local. El proveedor JMS se inicia junto con Servidor GlassFish.

REMOTE

Acceso al servicio JMS en otro sistema. Si selecciona REMOTE, Servidor GlassFish no iniciará el servicio JMS la siguiente vez que se inicie el servidor. En su lugar, el servicio JMS se inicia y se administra a través de Message Queue; por tanto, debe iniciar el agente Message Queue por separado. Para obtener más información sobre cómo iniciar el agente, consulte la sección Starting Brokers de Oracle GlassFish Server Message Queue 4.5 Administration Guide. Si selecciona este valor y está utilizando un host remoto, siga las instrucciones de la sección Para editar un host JMS para especificar el nombre del host remoto.

Tiempo de espera de inicio

El número de segundos que Servidor GlassFish esperará para que se inicie el servicio JMS antes de anular el inicio. Para un sistema lento o sobrecargado, aumente el valor predeterminado. El valor predeterminado es 60.

Argumentos de inicio

Los argumentos para personalizar el inicio del servicio JMS. Puede utilizar todos los argumentos disponibles a través del comando as-install-parent/mq/bin/imqbrokerd.

Volver a conectar habilitada

Si la se selecciona la casilla de verificación Volver a conectar habilitada, el servicio JMS intentará volver a conectarse a un servidor de mensajes (o a la lista de direcciones de AddressList) cuando se pierda una conexión. Esta opción se habilita por defecto.

Intervalo de reconexión

El número de segundos entre los intentos de reconexión. Este intervalo se refiere a intentos con cada dirección de la AddressList y direcciones sucesivas de la lista. Si es demasiado corto, este intervalo de tiempo no dará tiempo al agente para recuperar. Si es demasiado largo, la reconexión puede suponer un retraso inaceptable. El valor predeterminado es de 5 segundos.

Intentos de reconexión

Especifica el número de intentos de conexión (o reconexión) para cada dirección de la AddressList antes de que el tiempo de ejecución del cliente continúe con la siguiente dirección de la lista. El valor -1 indica que el número de intentos de reconexión es ilimitado (el tiempo d ejecución del cliente intenta conectar con la primera dirección hasta que lo consigue). El valor predeterminado es 3.

Host JMS predeterminado

El nombre del host JMS predeterminado. El valor predeterminado es default_JMS_host.

Comportamiento de la lista de direcciones

El orden de intentos de conexión. Las opciones disponibles son:

random

Selecciona una dirección de lista de direcciones de forma aleatoria. Si hay muchos clientes que intentan una conexión utilizando la misma fábrica de conexión, especifique RANDOM para evitar que todos se conecten a la misma dirección. Ésta es la opción predeterminada.

priority

La reconexión siempre intenta volver a conectarse a la primera dirección de servidor de AddressList y utiliza otra solamente cuando el primer agente no está disponible.

Iteraciones de lista de direcciones

El número de veces que servicio JMS repite el recorrido por la lista AddressList en un esfuerzo por establecer (o restablecer) una conexión. El valor -1 indica que el número de intentos es ilimitado. El valor predeterminado es 3.

Esquema MQ y Servicio MQ

El nombre de esquema de la dirección de Message Queue y el nombre del servicio de conexión de Message Queue si no se van a utilizar esquemas o servicios no predeterminados. Consulte Sintaxis de las direcciones de esquema MQ y servicio MQ para obtener información sobre la sintaxis.

Propiedades adicionales

Propiedades adicionales del servicio JMS.

Para obtener información sobre las propiedades disponibles, consulte Propiedades específicas del servicio JMS.
