JMS 物理送信先を作成する

通常は、JMS 送信先リソースの作成時に物理送信先名を指定すると、GlassFish Server が必要なときに物理送信先を自動的に作成するため、物理送信先を作成する必要はありません。

  1. ナビゲーションツリーで、「設定」ノードを展開します。
  2. 「設定」ノードで、「Java メッセージサービス」ノードを展開します。
  3. 「Java メッセージサービス」ノードで、「物理送信先」ノードを選択します。

    「物理送信先」ページが開きます。

  4. 「物理送信先」ページで、「新規」をクリックします。

    「新しい物理送信先」ページが開きます。

  5. 「名前」フィールドに、物理送信先の名前を入力します。
  6. 「タイプ」ドロップダウンリストから、javax.jms.Topic または javax.jms.Queue のいずれかを選択します。
  7. 「消費していないメッセージの最大数」フィールドに、送信先で許可されている未消費メッセージの最大数を入力します。

    -1 を指定すると、未消費メッセージの数は制限されません。デフォルト値は –1 です。

  8. 「最大メッセージサイズ」フィールドに、単一メッセージの最大サイズをバイト単位で入力します。

    -1 を指定すると、メッセージのサイズは制限されません。デフォルト値は –1 です。

  9. 「最大総メモリー」フィールドに、未消費メッセージの最大合計メモリーをバイト単位で入力します。

    -1 を指定すると、未消費メッセージのメモリーサイズは制限されません。デフォルト値は –1 です。

  10. 「しきい値制限の動作」ドロップダウンリストから、メモリー制限のしきい値に達したときのブローカの動作を選択します。
    • Reject newest messages は、最新のメッセージを拒否し、メッセージが継続する場合のみ、メッセージを生成しているクライアントに例外を通知するようにブローカに指示します。

      Reject newest messages はデフォルト値です。

    • Slow down producers は、メッセージプロデューサがメッセージを送信する頻度を下げるようにブローカに指示します。
    • Throw out oldest は、もっとも古いメッセージを破棄するようにブローカに指示します。
    • Throw out lowest-priority messages は、優先度のもっとも低いメッセージを破棄するようにブローカに指示します。

    値が Throw out oldest または Throw out lowest-priority messages で、「デッドメッセージキューを使用」が true に設定されている場合、超過したメッセージはデッドメッセージキューに送られます。

  11. 「プロデューサの最大数」フィールドに、この物理送信先に関連付けることができるメッセージプロデューサの最大数を入力します。

    この制限に達すると、新しいプロデューサを作成できなくなります。-1 を指定すると、プロデューサの数は制限されません。デフォルト値は 100 です。

  12. 「バッチ配信の最大メッセージ数」フィールドに、単一のバッチでコンシューマに配送可能なメッセージの最大数を入力します。

    -1 を指定すると、メッセージの数は制限されません。デフォルト値は 1000 です。

  13. 「デッドメッセージキューを使用」ドロップダウンリストから true を選択して、デッドメッセージをデッドメッセージキューに送ります。

    値を false に設定すると、デッドメッセージは破棄されます。デフォルトでは、このオプションは true に設定されます。

  14. 「XML スキーマの検証」の「有効」ドロップダウンから true を選択して、送信先での XML スキーマの妥当性検査を有効にします。

    値を true に設定すると、Message Queue クライアントランタイムは XML メッセージをブローカに送信する前に、指定した XML Schema Definition (XSD が指定されていない場合は DTD) を使用してメッセージの検証を行います。指定したスキーマが見つからないかメッセージを検証できない場合、メッセージは送信されず例外がスローされます。デフォルトでは、このオプションは false に設定されます。

  15. 「XML スキーマ URI リスト」フィールドに、XML Schema Definition (XSD) URI 文字列を空白区切りのリストで入力します。

    XML スキーマ検証が有効な場合、URI は XML スキーマ検証に使用する 1 つまたは複数の XSD の場所を指します。
