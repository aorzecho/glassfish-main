新しいJMS宛先リソース

新しい宛先リソースを作成するには、「新しいJMS宛先リソース」ページを使用します。

「新しいJMS宛先リソース」ページには、次のオプションが含まれます。

JNDI名

宛先リソースを識別する一意の名前。

JMSリソースのネーミング・サブコンテキスト接頭辞jms/を使用することをお薦めします。例: jms/Queue。

物理宛先名

このリソースに関連付ける物理宛先の名前。

GlassFish Serverは、物理宛先を必要なときに自動的に作成し、宛先リソースを削除するときに自動的に削除します。

リソース・タイプ

宛先リソースのタイプ。選択可能な値は、javax.jms.Topicとjavax.jms.Queueです。

説明

宛先リソースの説明。

ステータス

宛先リソースのステータス。宛先リソースは、有効または無効の状態になります。このオプションはデフォルトで有効になっています。

追加プロパティ

宛先リソースの追加プロパティ。GlassFish ServerはJMS宛先リソースの追加プロパティを定義しません。

ターゲット

宛先リソースのクラスタおよびスタンドアロン・インスタンス。必要なターゲットを「選択したターゲット」列に移動するには、「追加」ボタンまたは「すべて追加」ボタンを使用します。不要なターゲットを「使用可能なターゲット」列に移動するには、「削除」ボタンまたは「すべて削除」ボタンを使用します。

このオプションは、ドメイン内にクラスタまたはスタンドアロン・インスタンスが作成されている場合にのみ表示されます。
