新しい JMS 送信先リソース

新しい送信先リソースを作成するには、「新しい JMS 送信先リソース」ページを使用します。

「新しい JMS 送信先リソース」ページには、次のオプションが含まれます。

JNDI 名

送信先リソースを識別する一意の名前。

JMS リソースのネーミングサブコンテキストプレフィックス jms/ を使用することをお勧めします。次に例を示します。 jms/Queue

物理送信先名

このリソースに関連付ける物理送信先の名前。

GlassFish サーバー は、物理送信先を必要なときに自動的に作成し、送信先リソースを削除するときに自動的に削除します。

リソースタイプ

送信先リソースのタイプ。指定できるタイプは、javax.jms.Topic と javax.jms.Queue です。

説明

送信先リソースの説明。

ステータス

送信先リソースのステータス。送信先リソースは、有効または無効の状態になります。このオプションはデフォルトで有効になっています。

追加プロパティー

送信先リソースの追加プロパティー。GlassFish サーバー は JMS 送信先リソースの追加プロパティーを定義しません。

ターゲット

送信先リソースのクラスタおよびスタンドアロンインスタンス。必要なターゲットを「選択したターゲット」列に移動するには、「追加」ボタンまたは「すべて追加」ボタンを使用します。不要なターゲットを「利用可能なターゲット」列に移動するには、「削除」ボタンまたは「すべて削除」ボタンを使用します。

このオプションは、ドメイン内にクラスタまたはスタンドアロンインスタンスが作成されている場合にのみ表示されます。
