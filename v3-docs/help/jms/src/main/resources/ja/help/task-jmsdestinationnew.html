JMS宛先リソースを作成する

JMS宛先リソースを作成するときに、GlassFish Serverは宛先リソースの管理オブジェクト・リソースを作成します。このリソースは、「コネクタ」ノードに表示されます。

  1. ナビゲーション・ツリーで、「リソース」ノードを展開します。

  2. 「リソース」ノードで、「JMSリソース」ノードを展開します。

  3. 「JMSリソース」ノードで、「JMS宛先リソース」ノードを選択します。

    「JMS宛先リソース」ページが開きます。

  4. 「JMS宛先リソース」ページで、「新規」をクリックします。

    「新しいJMS宛先リソース」ページが開きます。

  5. 「JNDI名」フィールドに、リソースのJNDI名を入力します。

    JMSリソースのネーミング・サブコンテキストの接頭辞jms/を使用することをお薦めします。例: jms/Queue。

  6. 「物理宛先名」フィールドに、このリソースに関連付ける物理宛先の名前を入力します。

    GlassFish Serverは、物理宛先を必要なときに自動的に作成し、宛先リソースを削除するときに自動的に削除します。

  7. 「リソース・タイプ」ドロップダウン・リストから、javax.jms.Topicまたはjavax.jms.Queueのいずれかを選択します。

  8. 「説明」フィールドに、宛先リソースの説明を入力します。

  9. 実行時にリソースを有効にするには、「ステータス」の「有効」チェックボックスを選択します。

    このオプションはデフォルトで有効になっています。

  10. 「追加プロパティ」領域で、宛先リソースの追加プロパティを指定します。

    GlassFish ServerはJMS宛先リソースの追加プロパティを定義しません。

    プロパティを追加するには、「プロパティを追加」ボタンをクリックします。表示される空白行で、「名前」フィールドにプロパティ名を入力し、「値」フィールドにプロパティ値を入力します。

  11. In the Targets area, do the following:

    1. From the Available column, select the target or targets to which applications that use the resource will be deployed.

      The available targets include the available clusters and server instances, as well as the default server instance, server.

    2. Click Add to move the target to the Selected column.

  12. 「OK」をクリックします。
