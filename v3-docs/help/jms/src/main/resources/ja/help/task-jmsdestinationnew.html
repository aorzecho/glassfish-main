JMS 送信先リソースを作成する

JMS 送信先リソースを作成するときに、GlassFish Server は送信先リソースの管理オブジェクトリソースを作成します。このリソースは、「コネクタ」ノードに表示されます。

  1. ナビゲーションツリーで、「リソース」ノードを展開します。
  2. 「リソース」ノードで、「JMS リソース」ノードを展開します。
  3. 「JMS リソース」ノードで、「JMS 送信先リソース」ノードを選択します。

    「JMS 送信先リソース」ページが開きます。

  4. 「JMS 送信先リソース」ページで、「新規」をクリックします。

    「新しい JMS 送信先リソース」ページが開きます。

  5. 「JNDI 名」フィールドに、リソースの JNDI 名を入力します。

    JMS リソースのネーミングサブコンテキストプレフィックス jms/ を使用することをお勧めします。次に例を示します。 jms/Queue

  6. 「物理送信先名」フィールドに、このリソースに関連付ける物理送信先の名前を入力します。

    GlassFish Server は、物理送信先を必要なときに自動的に作成し、送信先リソースを削除するときに自動的に削除します。

  7. 「リソースタイプ」ドロップダウンリストから、javax.jms.Topic または javax.jms.Queue のいずれかを選択します。
  8. 「説明」フィールドに、送信先リソースの説明を入力します。
  9. 実行時にリソースを有効にするには、「状態を有効」チェックボックスを選択します。

    このオプションはデフォルトで有効になっています。

  10. (省略可能) 「追加プロパティー」領域で、送信先リソースの追加プロパティーを指定します。

    GlassFish Server は JMS 送信先リソースの追加プロパティーを定義しません。

    プロパティーを追加するには、「プロパティーを追加」ボタンをクリックします。表示される空白行で、「名前」フィールドにプロパティー名を入力し、「値」フィールドにプロパティー値を入力します。

  11. 「了解」をクリックします。
