JMS の可用性

「JMS の可用性」ページを使用し、JMS (Java メッセージサービス) の可用性を設定します。

「JMS の可用性」ページには、次のオプションが含まれます。

デフォルトを読込み

デフォルト値を持つ設定をデフォルト値に戻すボタン。デフォルト値を持たない設定は変更されません。

設定名

このページの設定を適用する設定の名前。 このフィールドは読み取り専用です。

可用性サービス

GlassFish クラスタに JMS サービスを提供するために使用される、メッセージキュークラスタの種類。「有効」チェックボックスを選択すると、拡張 Message Queue クラスタが使用されます。それ以外の場合、従来の Message Queue クラスタが使用されます。

「有効」チェックボックスを選択すると、設定のストアタイプおよびメッセージのストアタイプは無視されます。

設定のストアタイプ

従来の Message Queue クラスタで使用する、設定データ用データストアの種類。値 masterbroker を指定すると、設定データの格納と管理にマスターブローカが使用されます。値 shareddb を指定すると、設定データの格納に共有データベースが使用されます。

可用性サービスが有効になっている場合、このオプションは無視されます。

メッセージのストアタイプ

従来のクラスタで使用する、ブローカ内のメッセージデータ用データストアの種類。値 file はファイルストアを表します。値 jdbc は JDBC ストアを表します。

可用性サービスが有効になっている場合、このオプションは無視されます。

データベースベンダー
データベースユーザー名
データベースパスワード
データベースアクセス URL

次の場合に使用する JDBC データベースのベンダー、ユーザー、パスワード、およびアクセス URL。

  • 可用性サービスが有効の場合

  • 設定のストアタイプが shareddb に設定されている場合

  • メッセージのストアタイプが jdbc に設定されている場合

サポートされているベンダーと、各ベンダーのアクセス URL の形式については、『Oracle GlassFish Server Message Queue 4.5 Administration Guide』の「JDBC-Based Persistence」を参照してください。

追加プロパティー

JMS の可用性の追加プロパティー。

GlassFish Server では、JMS の可用性の追加プロパティーは事前定義されません。
