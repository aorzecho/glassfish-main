JMSの可用性

JMS(Java Message Service)の可用性設定を構成するには、「JMSの可用性」ページを使用します。

「JMSの可用性」ページには、次のオプションが含まれます。

デフォルトをロード

デフォルト値を持つ設定をデフォルト値に戻すボタン。デフォルト値を持たない設定は変更されません。

構成名

このページの設定が適用される構成の名前。このフィールドは読取り専用です。

可用性サービス

JMSサービスをGlassFishクラスタに提供するのに使用するMessage Queueクラスタのタイプ。「有効」チェックボックスを選択すると、拡張されたMessage Queueクラスタが使用されます。それ以外の場合は、従来のMessage Queueクラスタが使用されます。

「有効」チェックボックスを選択すると、「構成ストア・タイプ」オプションと「メッセージ・ストア・タイプ」オプションは無視されます。

構成ストア・タイプ

従来のMessage Queueクラスタで使用する構成データ用データ・ストアのタイプ。値masterbrokerを指定すると、構成データの格納と管理にマスター・ブローカが使用されます。値shareddbを指定すると、構成データの格納に共有データベースが使用されます。

「可用性サービス」が有効な場合、このオプションは無視されます。

メッセージ・ストア・タイプ

従来のクラスタで使用する、ブローカ内のメッセージ・データ用データ・ストアのタイプ。値fileはファイル・ストアを表します。値jdbcはJDBCストアを表します。

「可用性サービス」が有効な場合、このオプションは無視されます。

データベース・ベンダー
データベース・ユーザー名
データベース・パスワード
データベース・アクセスURL

次の場合に使用するJDBCデータベースのベンダー、ユーザー、パスワード、およびアクセスURL。

  • 「可用性サービス」が有効な場合

  • 「構成ストア・タイプ」がshareddbに設定された場合

  • 「メッセージ・ストア・タイプ」がjdbcに設定された場合

サポート対象のベンダー、および各ベンダーのアクセスURLの形式については、Oracle GlassFish Server Message Queue管理ガイドのJDBC-Based Persistenceを参照してください。

追加プロパティ

JMSの可用性の追加プロパティ。

GlassFish ServerはJMSの可用性の追加プロパティを事前に定義しません。
