JMS 物理送信先の編集

JMS 物理送信先の設定を変更するには、「JMS 物理送信先の編集」ページを使用します。

「JMS 物理送信先の編集」ページには、次のオプションが含まれます。

名前

物理送信先の名前。このフィールドは読み取り専用です。名前を指定できるのは、JMS 送信先リソースまたは物理送信先の作成時のみです。

タイプ

物理送信先のタイプ。このフィールドは読み取り専用です。タイプを指定できるのは、JMS 送信先リソースまたは物理送信先の作成時のみです。

消費していないメッセージの最大数

送信先で許可されている未消費メッセージの最大数。-1 を指定すると、未消費メッセージの数は制限されません。デフォルト値は –1 です。

最大メッセージサイズ

単一メッセージの最大サイズ (バイト単位)。-1 を指定すると、メッセージのサイズは制限されません。デフォルト値は –1 です。

最大総メモリー

未消費メッセージの最大合計メモリー (バイト単位)。-1 を指定すると、未消費メッセージのメモリーサイズは制限されません。デフォルト値は –1 です。

しきい値制限の動作

メモリー制限のしきい値に達したときのブローカの動作。次のタイプを指定できます。

Reject newest messages

最新のメッセージを拒否し、メッセージが継続する場合のみ、メッセージを生成しているクライアントに例外を通知します。この値がデフォルト値です。

Slow down producers

メッセージプロデューサがメッセージを送信する頻度を下げます。

Throw out oldest

もっとも古いメッセージを破棄します。

Throw out lowest-priority essages

優先度のもっとも低いメッセージを破棄します。

値が Throw out oldest または Throw out lowest-priority messages で、「デッドメッセージキューを使用」が true に設定されている場合、超過したメッセージはデッドメッセージキューに送られます。

プロデューサの最大数

この物理送信先に関連付けることができるメッセージプロデューサの最大数。この制限に達すると、新しいプロデューサを作成できなくなります。-1 を指定すると、プロデューサの数は制限されません。デフォルト値は 100 です。

バッチ配信の最大メッセージ数

単一のバッチでコンシューマに配送可能なメッセージの最大数。-1 を指定すると、メッセージの数は制限されません。デフォルト値は 1000 です。

デッドメッセージキューを使用

このオプションを有効にすると、デッドメッセージがデッドメッセージキューに送られます。このオプションが無効の場合、デッドメッセージは破棄されます。このオプションはデフォルトで有効になっています。

XML スキーマの妥当性検査

このオプションを有効にすると、Message Queue クライアントランタイムは XML メッセージをブローカに送信する前に、指定した XML Schema Definition (XSD が指定されていない場合は DTD) を使用してメッセージの検証を行います。指定したスキーマが見つからないかメッセージを検証できない場合、メッセージは送信されず例外がスローされます。このオプションは、デフォルトでは無効になっています。

XML スキーマ URI リスト

XML Schema Definition (XSD) URI 文字列の空白区切りのリスト。XML スキーマ検証が有効な場合、URI は XML スキーマ検証に使用する 1 つまたは複数の XSD の場所を指します。
