Para configurar o serviço JMS

Utilize a página Serviço JMS para definir as configurações a serem utilizadas por todas as conexões do JMS.

Para obter mais informações sobre a configuração do serviço JMS, consulte o Capítulo 16, Using the Java Message Service, no Oracle GlassFish Server 3.1 Application Development Guide.

  1. Na árvore de navegação, expanda o nó Configuração.
  2. No nó Configuração, selecione o nó Java Message Service.

    A página Serviço JMS é aberta.

  3. Na lista suspensa Tipo:
    • Selecione INCORPORADO para acessar o serviço JMS no host local.

      O serviço JMS é iniciado na mesma máquina que o Servidor GlassFish. A inicialização lenta inicia o broker padrão incorporado no primeiro acesso aos serviços JMS, em vez de na inicialização do Servidor GlassFish. Esse é o tipo padrão do serviço JMS.

    • Selecione LOCAL para acessar o serviço JMS no host local.

      O serviço JMS é iniciado juntamente com o Servidor GlassFish.

    • Selecione REMOTO para acessar o serviço JMS em outro sistema.

      Se escolher REMOTO, o serviço JMS não será iniciado pelo Servidor GlassFish na próxima vez que o servidor for iniciado. Em vez disso, o serviço JMS é iniciado e gerenciado pela Fila de mensagens, de forma que será necessário iniciar o broker da Fila de mensagens separadamente. Para obter informações sobre como iniciar o broker, consulte Starting Brokers no Oracle GlassFish Server Message Queue 4.5 Administration Guide . Se você escolher esse valor e estiver utilizando um host remoto, siga as instruções em Para editar um host do JMS para especificar o nome do host remoto.

  4. No campo Tempo limite para inicialização, digite o número de segundos que o Servidor GlassFish aguarda o serviço JMS ser iniciado antes de anular a inicialização.

    Em um sistema lento e sobrecarregado, aumente o valor a partir do padrão. O valor padrão é 60.

  5. No campo Argumentos de inicialização, digite argumentos para personalizar a inicialização do serviço JMS.

    Utilize qualquer argumento disponível através do comando as-install-parent/mq/bin/imqbrokerd.

  6. Marque a caixa de seleção Reconexão ativa para que o serviço JMS tente reconectar-se a um servidor de mensagens (ou a lista de endereços na AddressList) quando uma conexão é perdida.

    Por padrão, a opção está ativa.

  7. No campo Intervalo para reconectar, digite o número de segundos entre as tentativas de reconexão.

    Esse intervalo aplica-se a tentativas em cada endereço na lista AddressList e a endereços sucessivos na lista. Se for muito curto, esse intervalo de tempo não dá ao broker tempo para ser recuperar. Se for muito longo, a reconexão pode representar um atraso inaceitável. O valor padrão é 5 segundos.

  8. No campo Tentativas de reconexão, digite o número de tentativas de conexão (ou reconexão) para cada endereço na AddressList antes de o tempo de execução do cliente tentar o próximo endereço na lista.

    Um valor de -1 indica que o número de tentativas de reconexão é ilimitado (o tempo de execução do cliente tenta conectar-se ao primeiro endereço até conseguir). O valor padrão é 3.

  9. Na lista suspensa Host padrão do JMS, selecione o nome do host padrão do JMS.

    O valor padrão é default_JMS_host.

  10. Na lista suspensa Comportamento da lista de endereços, selecione a ordem de tentativas de conexão.
    • random significa selecionar um endereço na AddressList aleatoriamente.

      Se houver muitos clientes tentando uma conexão usando a mesma fábrica de conexões, especifique random para impedi-los de serem conectados ao mesmo endereço. Essa é a opção padrão.

    • priority significa que a reconexão sempre tenta conectar-se ao primeiro endereço do servidor na AddressList e utiliza outro somente se o primeiro broker não estiver disponível.
  11. No campo Iterações da lista de endereços, digite o número de vezes que o serviço JMS itera através da AddressList em um esforço para estabelecer (ou restabelecer) uma conexão.

    Um valor de -1 indica que o número de tentativas é ilimitado. O valor padrão é 3.

  12. Nos campos Esquema MQ e Serviço MQ, digite o nome do esquema de endereço e do serviço de conexão da Fila de mensagens, se um esquema ou serviço não padrão for utilizado.

    Para obter informações sobre sintaxe, consulte Sintaxe de endereços do esquema MQ e serviço MQ.

  13. (Opcional) Na seção Propriedades adicionais, especifique propriedades adicionais do serviço JMS.

    Para obter informações sobre as propriedades disponíveis, consulte Propriedades específicas do serviço JMS.

    • Para adicionar uma propriedade, clique no botão Adicionar propriedade. Na linha em branco que é exibida, digite o nome da propriedade no campo Nome e digite o valor da propriedade no campo Valor.
    • Para modificar uma propriedade, edite o campo Valor da propriedade.
    • Para excluir uma propriedade, marque a caixa de seleção à esquerda do campo Nome da propriedade a ser excluída. Clique no botão Excluir propriedades.
  14. Clique em Ping para verificar se o serviço JMS está sendo executado no host padrão do JMS.

    Se o serviço JMS estiver em execução, uma mensagem informa que o ping foi bem-sucedido.

  15. Clique em Salvar.

    Dica - Para restaurar as configurações padrão, clique no botão Carregar padrões e depois em Salvar.

Acessando servidores remotos

A mudança de um provedor e um host para um sistema remoto faz com que todos os aplicativos do JMS sejam executados no servidor remoto. Para utilizar o servidor local e um ou mais servidores remotos, crie um recurso de fábrica de conexões com a propriedade AddressList para criar conexões que acessem servidores remotos. Consulte Para criar um recurso de fábrica de conexões do JMS.

Comando asadmin equivalente

jms-ping
