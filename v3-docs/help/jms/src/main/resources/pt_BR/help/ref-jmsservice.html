Serviço JMS

Utilize a página Serviço JMS para definir as configurações a serem utilizadas por todas as conexões do JMS.

Para obter mais informações sobre a configuração do serviço JMS, consulte o Capítulo 16, Using the Java Message Service, no Oracle GlassFish Server 3.1 Application Development Guide.

A página Serviço JMS contém as opções a seguir.

Carregar padrões

Botão para restaurar com os valores padrão as configurações que apresentam valores padrão. As configurações que não apresentam valores padrão não são alteradas.

Ping

Botão utilizado para verificar que o serviço JMS está sendo executado no host padrão do JMS. Se o serviço JMS estiver em execução, uma mensagem informa que o ping foi bem-sucedido.

Configuration Name

O nome da configuração para a qual as definições nesta página se aplicam. Este campo é de somente leitura.

Tipo

O tipo de acesso para o serviço JMS. As opções disponíveis são:

INCORPORADO

Acessa o serviço JMS no host local. O serviço JMS é iniciado na mesma máquina que o Servidor GlassFish. A inicialização lenta inicia o broker padrão incorporado no primeiro acesso aos serviços JMS, em vez de na inicialização do Servidor GlassFish. Esse é o tipo padrão do serviço JMS.

LOCAL

Acessa o serviço JMS no host local. O provedor do JMS é iniciado juntamente com o Servidor GlassFish.

REMOTO

Acessa o serviço JMS em outro sistema. Se escolher REMOTO, o serviço JMS não será iniciado pelo Servidor GlassFish na próxima vez que o servidor for iniciado. Em vez disso, o serviço JMS é iniciado e gerenciado pela Fila de mensagens, de forma que será necessário iniciar o broker da Fila de mensagens separadamente. Para obter informações sobre como iniciar o broker, consulte Starting Brokers no Oracle GlassFish Server Message Queue 4.5 Administration Guide . Se você escolher esse valor e estiver utilizando um host remoto, siga as instruções em Para editar um host do JMS para especificar o nome do host remoto.

Tempo limite de inicialização

O número de segundos que o Servidor GlassFish aguarda o serviço JMS ser iniciado antes de anular a inicialização. Em um sistema lento e sobrecarregado, aumente o valor a partir do padrão. O valor padrão é 60.

Argumentos de inicialização

Argumentos para personalizar a inicialização do serviço JMS. Utilize qualquer argumento disponível através do comando as-install-parent/mq/bin/imqbrokerd.

Reconexão ativa

Se a caixa de seleção Reconexão ativa estiver marcada, o serviço JMS tentará reconectar-se a um servidor de mensagens (ou à lista de endereços na AddressList) quando uma conexão se perder. Por padrão, a opção está ativada.

Intervalo de reconexão

O número de segundos entre tentativas de reconexão. Esse intervalo aplica-se a tentativas em cada endereço na lista AddressList e a endereços sucessivos na lista. Se for muito curto, esse intervalo de tempo não dá ao broker tempo para ser recuperar. Se for muito longo, a reconexão pode representar um atraso inaceitável. O valor padrão é 5 segundos.

Tentativas de reconexão

O número de tentativas de conexão (ou reconexão) para cada endereço na AddressList antes de o tempo de execução do cliente tentar o próximo endereço na lista. Um valor de -1 indica que o número de tentativas de reconexão é ilimitado (o tempo de execução do cliente tenta conectar-se ao primeiro endereço até conseguir). O valor padrão é 3.

Host JMS padrão

O nome do host JMS padrão. O valor padrão é default_JMS_host.

Comportamento da lista de endereços

A ordem de tentativas de conexão. As opções disponíveis são:

aleatório

Selecione um endereço na AddressList aleatoriamente. Se houver muitos clientes tentando conectar-se usando a mesma fábrica de conexões, especifique random para impedi-los de conectarem-se ao mesmo endereço. Essa é a opção padrão.

prioridade

A reconexão sempre tenta conectar-se ao primeiro endereço do servidor na AddressList e usa outro somente se o primeiro broker não estiver disponível.

Iterações da lista de endereços

O número de vezes que o serviço JMS itera através da AddressList em um esforço de estabelecer (ou restabelecer) uma conexão. Um valor de -1 indica que o número de tentativas é ilimitado. O valor padrão é 3.

Esquema MQ e Serviço MQ

O nome do esquema de endereços e do nome do serviço da Fila de mensagens, se um esquema ou serviço não padrão for utilizado. Para obter informações sobre sintaxe, consulte Sintaxe de endereços do esquema MQ e serviço MQ.
