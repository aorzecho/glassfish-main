Novo destino físico do JMS

Utilize a página Novo destino físico do JMS para criar um novo destino físico do JMS.

Normalmente não é preciso criar destinos físicos, porque você especifica um nome de destino físico quando cria um recurso de destino do JMS e o Servidor GlassFish cria o destino físico automaticamente quando é necessário.

A página Novo destino físico do JMS contém as opções a seguir.

Nome

O nome do destino físico.

Tipo

O tipo do destino físico. As escolhas disponíveis são javax.jms.Topic e javax.jms.Queue.

Número máximo de mensagens não consumidas

O número máximo de mensagens não consumidas permitido para o destino. Um valor de –1 indica que não há limite para o número de mensagens não consumidas. O valor padrão é -1.

Tamanho máximo de mensagem

O tamanho máximo, em bytes, de uma única mensagem. Um valor de –1 denota um tamanho de mensagem sem limite. O valor padrão é -1.

Total máximo de memória

O total máximo de memória, em bytes, para mensagens não consumidas. Um valor de –1 indica que não há limite para o tamanho de memória para mensagens não consumidas. O valor padrão é -1.

Comportamento do limite limiar

O comportamento que o broker deve seguir quando o limiar do limite de memória é atingido. As opções disponíveis são:

Rejeitar mensagens mais recentes

Rejeita as mensagens mais recentes e notifica o cliente de produção com uma exceção somente se a mensagem é persistente. Esse é o valor padrão.

Diminuir a taxa de produtores

Diminui a taxa com que produtores de mensagem enviam mensagens.

Rejeitar mais antiga

Descarta as mensagens mais antigas.

Rejeitar mensagens de prioridade mais baixa

Descarta as mensagens com a prioridade mais baixa.

Se o valor for Rejeitar mais antiga ou Rejeitar mensagens de prioridade mais baixa, e Utilizar a Fila de mensagens inativas estiver definido como verdadeiro, mensagens em excesso são movidas para a Fila de mensagens inativas.

Número máximo de produtores

O número máximo de produtores de mensagens que pode ser associado a este destino físico. Quando esse limite é atingido, nenhum novo produtor pode ser criado. Um valor de –1 denota um número ilimitado de produtores. O valor padrão é 100.

Contagem de mensagens máx. da entrega em lotes

O número máximo de mensagens que podem ser entregues para consumidores em um único lote. Um valor de –1 denota um número ilimitado de mensagens. O valor padrão é 1000.

Utilizar a Fila de mensagens inativas

Se o valor for definido como Verdadeiro, mensagens inativas são colocadas na fila de mensagens inativas. Se o valor for definido como Falso, mensagens inativas são descartadas. Por padrão, essa opção é definida como Verdadeiro.

Validar esquema XML

Se o valor for definido como Verdadeiro, o tempo de execução do cliente da Fila de mensagens tentará validar uma mensagem XML contra as Definições de esquema XML especificadas (ou contra o DTD, se nenhuma XSD estiver especificado) antes de enviá-la para o broker. Se o esquema especificado não pode ser localizado ou a mensagem não pode ser validada, a mensagem não é enviada e uma exceção é lançada. Por padrão, essa opção é definida como Falso.

Lista de URIs de esquema XML

Uma lista separada por vírgulas de sequências de URI de definição de esquema XML (XSD) URI. Se a validação de esquema XML estiver ativada, as URIs apontam para o local de uma ou mais XSDs a serem usadas para a validação do esquema XML.
