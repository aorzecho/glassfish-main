Sintaxe de endereços do esquema MQ e serviço MQ

A sintaxe inteira para um endereço de serviço de mensagem é

scheme://address-syntax

onde a sintaxe do esquema e do serviço são descritas no quadro abaixo.

O esquema MQ e o serviço MQ são os valores mostrados nas primeiras duas colunas na tabela abaixo.

Nome do esquema

Serviço de conexão

Descrição

Sintaxe de endereço

mq

jms e ssljms

O tempo de execução do cliente da Fila de mensagens estabelece uma conexão com o Mapeador de porta MQ no host e na porta especificados. O Mapeador de porta retorna uma lista das portas de serviço de conexão estabelecida dinamicamente e o tempo de execução do cliente da Fila de mensagens estabelece então uma conexão com a porta que serve de host do serviço de conexão especificado.

[hostName][:porta][/serviceName]

Padrões: hostName = localhost, port = 7676(644), serviceName = jms

Padrões somente aplicam-se ao serviço de conexão jms. Para o serviço de conexão ssljms, todas as variáveis precisam ser especificadas.

Exemplo: mq:MyHost:7677/ssljms

mqtcp

jms

O tempo de execução do cliente da Fila de mensagens estabelece uma conexão TCP para o host e a porta especificados (ignorando o Mapeador de porta MQ) para estabelecer uma conexão.

hostName:port/jms

Exemplo: mqtcp:localhost:7676/jms

mqssl

ssljms

O tempo de execução do cliente da Fila de mensagens estabelece uma conexão SSL segura para o host e a porta especificados (ignorando o Mapeador de porta MQ) para estabelecer uma conexão.

hostName:porta/ssljms

Exemplo: mqssl:localhost:7676/ssljms

http

httpjms

O tempo de execução do cliente da Fila de mensagens estabelece uma conexão HTTP com um servlet de túnel a uma Fila de mensagens no URL especificado. (O broker deve ser configurado para acessar o servlet de túnel HTTP, como descrito no Guia do administrador da Fila de mensagens.)

hostName:port/contextRoot/tunnel

Se várias instâncias de broker estiverem utilizando o mesmo servlet de túnel, a sintaxe para a conexão com uma instância de broker específica (em vez de uma selecionada aleatoriamente) é: http://hostNameport/contextRoot/tunnel?serverName=hostNameinstanceName

https

httpsjms

O tempo de execução do cliente da Fila de mensagens estabelece uma conexão HTTPS segura com o URL do servelet de túnel especificado da Fila de mensagens. (O broker deve ser configurado para acessar o servlet de túnel HTTPS, como descrito no Guia do administrador da Fila de mensagens.

hostName:port/contextRoot/tunnel

Se várias instâncias de broker estiverem usando o mesmo servlet de túnel, a sintaxe para a conexão com uma instância de broker específica (em vez de uma selecionada aleatoriamente) é: http://hostNameport/contextRoot/tunnel?serverName=hostNameinstanceName
