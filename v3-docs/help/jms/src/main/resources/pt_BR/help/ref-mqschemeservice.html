Sintaxe de endereços do esquema MQ e serviço MQ

A sintaxe inteira para um endereço de serviço de mensagem é

scheme://address-syntax

onde a sintaxe do esquema e do serviço são descritas no quadro abaixo.

O esquema MQ e o serviço MQ são os valores mostrados nas primeiras duas colunas na tabela abaixo.

Nome do esquema Serviço de conexão Descrição Sintaxe de endereço
mq jms e ssljms O runtime do cliente do Message Queue estabelece uma conexão com o Mapeador de Portas MQ no host e na porta especificados. O Mapeador de porta retorna uma lista das portas de serviço de conexão estabelecida dinamicamente e o runtime do cliente do Message Queue estabelece conexão com a porta que serve de host do serviço de conexão especificado. [hostName][:port][/serviceName]

Defaults: hostName = localhost, porta = 7676, serviceName = jms

Defaults somente se aplicam ao serviço de conexão jms. Para o serviço de conexão ssljms, todas as variáveis precisam ser especificadas.

Exemplo: mq:MyHost:7677/ssljms
mqtcp jms O runtime do cliente do Message Queue estabelece uma conexão TCP para o host e a porta especificados (ignorando o Mapeador de Portas MQ) para estabelecer uma conexão. hostName:porta/jms

Exemplo: mqtcp:localhost:7676/jms
mqssl ssljms O runtime do cliente do Message Queue estabelece uma conexão SSL segura para o host e a porta especificados (ignorando o Mapeador de Porta MQ) para estabelecer uma conexão. hostName:porta/ssljms

Exemplo: mqssl:localhost:7676/ssljms
http httpjms O runtime do cliente do Message Queue estabelece uma conexão HTTP com um servlet de túnel do Message Queue no URL especificado. (O controlador deve estar configurado para acessar o servlet do túnel HTTP, conforme descrito no Message Queue Administrator's Guide.) hostName:port/contextRoot/tunnel

Se várias instâncias do controlador estiverem usando o mesmo servlet do túnel, então a sintaxe para conectar a uma instância do controlador (em vez de uma selecionada aleatoriamente) é: http://hostName:port/contextRoot/tunnel?serverName=hostName:instanceName
https httpsjms O runtime do cliente do Message Queue estabelece uma conexão HTTPS segura com o URL do servelet de túnel especificado do Message Queue. (O controlador deve estar configurado para acessar o servlet do túnel HTTPS, conforme descrito no Message Queue Administrator's Guide.) hostName:port/contextRoot/tunnel

Se várias instâncias do controlador estiverem usando o mesmo servlet do túnel, então a sintaxe para conectar a uma instância do controlador (em vez de uma selecionada aleatoriamente) é: http://hostName:port/contextRoot/tunnel?serverName=hostName:instanceName
