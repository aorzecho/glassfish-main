Para criar um destino físico do JMS

Normalmente não é preciso criar destinos físicos, porque você especifica um nome de destino físico quando cria um recurso de destino do JMS e o Servidor GlassFish cria o destino físico automaticamente quando é necessário.

  1. Na árvore de navegação, expanda o nó Configuração.
  2. No nó Configuração, expanda o nó do Java Message Service.
  3. No nó Java Message Service, selecione o nó Destinos físicos.

    A página Destinos físicos é aberta.

  4. Na página Destinos físicos, clique em Novo.

    A página Novo destino físico é aberta.

  5. No campo Nome, digite o nome do destino físico.
  6. Na lista suspensa Tipo, selecione javax.jms.Topic ou javax.jms.Queue.
  7. No campo Número máximo de mensagens não consumidas, digite o número máximo de mensagens não consumidas permitido para o destino.

    Um valor de –1 indica que não há limite para o número de mensagens não consumidas. O valor padrão é -1.

  8. No campo Tamanho máximo de mensagem, digite o tamanho máximo, em bytes, de uma única mensagem.

    Um valor de –1 denota um tamanho de mensagem ilimitado. O valor padrão é -1.

  9. No campo Total máximo de memória, digite o total máximo de memória, em bytes, para mensagens não consumidas.

    Um valor de –1 indica que não há limite para o tamanho de memória para mensagens não consumidas. O valor padrão é -1.

  10. Na lista suspensa Comportamento do limite, selecione o comportamento que o broker deve seguir quando o limiar do limite de memória é atingido.
    • Rejeitar mensagens recentes informa ao broker que rejeite mensagens recentes e notifique o cliente produtor com uma exceção somente se a mensagem for persistente.

      Rejeitar mensagens recentes é o valor padrão.

    • Diminuir taxa de produtores informa ao broker que diminua a taxa na qual produtores de mensagem enviam mensagens.
    • Rejeitar mais antiga informa ao broker que descarte as mensagens mais antigas.
    • Rejeitar mensagens de prioridade mais baixa informa ao broker que descarte as mensagens com a prioridade mais baixa.

    Se o valor for Rejeitar mais antiga ou Rejeitar mensagens de prioridade mais baixa e Utilizar a Fila de mensagens inativas estiver definido como Verdadeiro, mensagens em excesso são movidas para a fila de mensagens inativas.

  11. No campo Número máximo de produtores, digite o número máximo de produtores de mensagem que pode ser associados a esse destino físico.

    Quando esse limite é atingido, nenhum novo produtor pode ser criado. Um valor de –1 denota um número ilimitado de produtores. O valor padrão é 100.

  12. No campo Contagem de mensagens máx. da entrega em lotes, digite o número máximo de mensagens que podem ser entregues a consumidores em um único lote.

    Um valor de –1 denota um número ilimitado de mensagens. O valor padrão é 1000.

  13. Na lista suspensa Utilizar a Fila de mensagens inativas, selecione Verdadeiro para colocar mensagens inativas na filha de mensagens inativas.

    Se o valor for definido como Falso, mensagens inativas são descartadas. Por padrão, essa opção é definida como Verdadeiro.

  14. Na lista suspensa Validar esquema XML, selecione Verdadeiro para ativar a validação do esquema XML para o destino.

    Se o valor for definido como Verdadeiro, o tempo de execução do cliente da Fila de mensagens tentará validar uma mensagem XML contra as Definições de esquema XML especificadas (ou contra o DTD, se nenhuma XSD estiver especificado) antes de enviá-la para o broker. Se o esquema especificado não pode ser localizado ou a mensagem não pode ser validada, a mensagem não é enviada e uma exceção é lançada. Por padrão, essa opção é definida como Falso.

  15. No campo Lista de URIs de esquema XML, digite uma lista separada por vírgulas de sequências URI de Definição de esquema XML (XSD).

    Se a validação de esquema XML estiver ativa, as URIs apontam para o local de uma ou mais XSDs a serem utilizadas para a validação do esquema XML.
