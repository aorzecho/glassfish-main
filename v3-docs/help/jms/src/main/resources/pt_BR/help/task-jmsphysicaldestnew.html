Para Criar uma Destinação Física de JMS

Normalmente não é preciso criar destinos físicos, porque você especifica um nome de destino físico quando cria um recurso de destino JMS e o GlassFish Server cria o destino físico automaticamente quando é necessário.

  1. Na árvore de navegação, expanda o nó Configuração.

  2. Select the instance to configure:

    • To configure a particular instance, select the instance's config node. For example, for the default instance, server, select the server-config node.

    • To configure the default settings for future instances that use a copy of default-config, select the default-config node.

  3. No nó Configuração, expanda o nó do Java Message Service.

  4. No nó Java Message Service, selecione o nó Destinos físicos.

    A página Destinos físicos é aberta.

  5. Na página Destinos físicos, clique em Novo.

    A página Novo destino físico é aberta.

  6. No campo Nome, digite o nome do destino físico.

  7. Na lista drop-down Tipo, selecione javax.jms.Topic ou javax.jms.Queue.

  8. No campo Número máximo de mensagens não consumidas, digite o número máximo de mensagens não consumidas permitido para o destino.

    Um valor de -1 indica que não há limite para o número de mensagens não-consumidas. O valor default é -1.

  9. No campo Tamanho máximo de mensagem, digite o tamanho máximo, em bytes, de uma única mensagem.

    Um valor de -1 significa um tamanho de mensagem ilimitado. O valor default é -1.

  10. No campo Total máximo de memória, digite o total máximo de memória, em bytes, para mensagens não consumidas.

    Um valor de -1 indica que não há limite para o tamanho da memória de mensagens não-consumidas. O valor default é -1.

  11. Na lista drop-down Comportamento do limite, selecione o comportamento que o broker deve seguir quando o limiar do limite de memória é atingido.

    • Rejeitar mensagens recentes informa ao broker que rejeite mensagens recentes e notifique o cliente produtor com uma exceção somente se a mensagem for persistente.

      Rejeitar mensagens recentes é o valor default.

    • Diminuir taxa de produtores informa ao broker que diminua a taxa na qual produtores de mensagem enviam mensagens.

    • Rejeitar mais antiga informa ao broker que descarte as mensagens mais antigas.

    • Rejeitar mensagens de prioridade mais baixa informa ao broker que descarte as mensagens com a prioridade mais baixa.

    Se o valor for Rejeitar mais antiga ou Rejeitar mensagens de prioridade mais baixa e se a opção Usar Fila de Mensagens Inativas estiver definida como Verdadeira, as mensagens em excesso serão movidas para a fila de mensagens inativas.

  12. No campo Número máximo de produtores, digite o número máximo de produtores de mensagem que pode ser associados a esse destino físico.

    Quando esse limite é atingido, nenhum novo produtor pode ser criado. Um valor de -1 denota um número ilimitado de produtores. O valor default é 100.

  13. No campo Contagem Máx. de Mensagens de Entrega em batches, digite o número máximo de mensagens que podem ser entregues a consumidores em um único batch.

    Um valor de -1 denota um número ilimitado de mensagens. O valor default é 1000.

  14. Na lista drop-down Usar Fila de Mensagens Inativas, selecione Verdadeiro para colocar as mensagens inativas na filha de mensagens inativas.

    Se o valor for definido como Falso, mensagens inativas são descartadas. Por default, essa opção é definida como Verdadeiro.

  15. Na lista drop-down Validar esquema XML, selecione Verdadeiro para ativar a validação do esquema XML para o destino.

    Se o valor for definido como Verdadeiro, o runtime do cliente do Message Queue tentará validar uma mensagem XML contra as Definições de esquema XML especificadas (ou contra o DTD, se nenhuma XSD estiver especificado) antes de enviá-la para o broker. Se o esquema especificado não pode ser localizado ou a mensagem não pode ser validada, a mensagem não é enviada e uma exceção é lançada. Por default, essa opção é definida como Falso.

  16. No campo Lista de URIs de esquema XML, digite uma lista separada por espaços de strings URI de Definição de esquema XML (XSD).

    Se a validação de esquema XML estiver ativa, as URIs apontam para o local de uma ou mais XSDs a serem utilizadas para a validação do esquema XML.
