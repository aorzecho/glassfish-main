Editar Destino Físico de JMS

Utilize a página Editar destino físico JMS para modificar as definições de um destino físico JMS.

A página Editar destino físico JMS contém as opções a seguir.

Nome

O nome do destino físico. Esse é um campo de somente leitura. Você pode especificar o nome somente quando cria um recurso de destino JMS ou um destino físico.

Tipo

O tipo do destino físico. Esse é um campo de somente leitura. Você pode especificar o tipo somente quando cria um recurso de destino JMS ou um destino físico.

Número máximo de mensagens não consumidas

O número máximo de mensagens não consumidas permitido para o destino. Um valor de -1 indica que não há limite para o número de mensagens não-consumidas. O valor default é -1.

Tamanho máximo de mensagem

O tamanho máximo, em bytes, de uma única mensagem. Um valor de -1 significa um tamanho de mensagem ilimitado. O valor default é -1.

Total máximo de memória

O total máximo de memória, em bytes, para mensagens não consumidas. Um valor de -1 indica que não há limite para o tamanho da memória de mensagens não-consumidas. O valor default é -1.

Comportamento do limite limiar

O comportamento que o broker deve seguir quando o limiar do limite de memória é atingido. As opções disponíveis são:

Rejeitar mensagens mais novas

Rejeita as mensagens mais recentes e notifica o cliente de produção com uma exceção somente se a mensagem é persistente. Este é o valor default.

Tornar os produtores mais lentos

Diminui a taxa com que produtores de mensagem enviam mensagens.

Eliminar o mais antigo

Descarta as mensagens mais antigas.

Eliminar as mensagens de menor prioridade

Descarta as mensagens com a prioridade mais baixa.

Se o valor for Rejeitar mais antigas ou Rejeitar mensagens com prioridade mais baixa, e se a opção Usar Fila de Mensagens Inativas for definida como Verdadeira, as mensagens em excesso serão movidas para a fila de mensagens inativas.

Número máximo de produtores

O número máximo de produtores de mensagens que pode ser associado a este destino físico. Quando esse limite é atingido, nenhum novo produtor pode ser criado. Um valor de -1 denota um número ilimitado de produtores. O valor default é 100.

Contagem de mensagens máx. da entrega batch

O número máximo de mensagens que podem ser entregues para consumidores em um único batch. Um valor de -1 denota um número ilimitado de mensagens. O valor default é 1000.

Usar Fila de Mensagens Inativas

Se essa opção estiver ativada, as mensagens inativas serão colocadas na fila de mensagens inativas. Se essa opção estiver desativada, mensagens inativas serão descartadas. Por default, a opção está ativada.

Validar esquema XML

Se essa opção estiver ativa, o runtime do cliente do Message Queue tentará validar uma mensagem XML contra as Definições de esquema XML especificadas (ou contra o DTD, se nenhuma XSD for especificada) antes de enviá-la para o broker. Se o esquema especificado não pode ser localizado ou a mensagem não pode ser validada, a mensagem não é enviada e uma exceção é lançada. Por default, essa opção está desativada.

Lista de URIs de esquema XML

Uma lista separada por espaços de strings de URI de definição de esquema XML (XSD) URI. Se a validação de esquema XML estiver ativada, as URIs apontam para o local de uma ou mais XSDs a serem usadas para a validação do esquema XML.
