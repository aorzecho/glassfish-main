Novo Recurso de Destinação do JMS

Utilize a página Novo recurso de destino JMS para criar um novo recurso de destino JMS.

A página Novo recurso de destino JMS contém as opções a seguir.

Nome JNDI

Um nome exclusivo que identifica o recurso de destino.

Recomenda-se usar o prefixo de subcontexto de nomeação jms/ para recursos do JMS. Por exemplo: jms/Queue.

Nome de destino físico

O nome do destino físico a ser associado a esse recurso.

O GlassFish Server cria um destino físico automaticamente quando necessário e o deleta quando você deleta o recurso de destino.

Tipo de recurso

O tipo do recurso de destino. As escolhas disponíveis são javax.jms.Topic e javax.jms.Queue.

Descrição

Uma descrição do recurso de destino.

Status

O status do recurso de destino. O recurso de destino pode ser ativado ou desativado. Por default, a opção está ativada.

Propriedades adicionais

Propriedades adicionais para o recurso de destino. O GlassFish Server não define quaisquer propriedades adicionais para recursos de destino JMS.

Destinos

Clusters e instâncias standalone do recurso de destino. Mova os destinos desejados para a coluna Destinos selecionados utilizando os botões Adicionar e Adicionar tudo. Mova os destinos desnecessários para a coluna Destinos disponíveis utilizando os botões Remover e Remover tudo.

Esta opção é exibida somente se os clusters ou instâncias standalone tiverem sido criados no domínio.
