Disponibilidade de JMS

Utilize a página Disponibilidade de JMS para definir as configurações de disponibilidade do Java Message Service (JMS).

A página Disponibilidade de JMS contém as opções a seguir.

Carregar padrões

Botão para restaurar com os valores padrão as configurações que apresentam valores padrão. As configurações que não apresentam valores padrão não são alteradas.

Configuration Name

O nome da configuração para a qual as definições nesta página se aplicam. Este campo é de somente leitura.

Serviço de disponibilidade

Tipo de cluster da Fila de mensagens utilizado para fornecer serviços JMS a um cluster GlassFish. Se a caixa de seleção Habilitado estiver marcada, é utilizado um cluster Message Queue aprimorado. Do contrário, é utilizado um cluster Message Queue convencional.

Se a caixa de seleção Habilitado estiver marcada, as opções Tipo de armazenamento da configuração e Tipo de armazenamento da mensagem são ignoradas.

Tipo de armazenamento da configuração

O tipo de armazenador de dados para os dados de configuração em um cluster Message Queue convencional. O valor masterbroker especifica o uso de um broker master para armazenar e gerenciar os dados da configuração. O valor shareddb especifica o uso de um banco de dados compartilhado para armazenar os dados de configuração.

Esta opção é ignorada se o Serviço de disponibilidade estiver habilitado.

Tipo de armazenamento da mensagem

O tipo de armazenador de dados para os dados de mensagens em um cluster convencional. O valor file especifica um armazenador de arquivos. O valor jdbc especifica um armazenador JDBC.

Esta opção é ignorada se o Serviço de disponibilidade estiver habilitado.

Fornecedor do banco de dados
Nome de usuário do banco de dados
Senha do banco de dados
URL de acesso do banco de dados

O fornecedor do banco de dados, o usuário, a senha e o URL de acesso do banco de dados JDBC a ser usado em qualquer uma dessas situações:

  • Quando o Serviço de disponibilidade estiver habilitado

  • Quando o Tipo de armazenamento da configuração estiver definido como shareddb

  • Quando o Tipo de armazenamento da mensagem estiver definido como jdbc

Para obter informações sobre os fornecedores suportados e os formatos de URL de acesso para cada fornecedor, consulte JDBC-Based Persistence no Oracle GlassFish Server Message Queue 4.5 Administration Guide .

Propriedades Adicionais

Propriedades adicionais da disponibilidade do JMS.

O Servidor GlassFish não define propriedades adicionais da disponibilidade do JMS.
