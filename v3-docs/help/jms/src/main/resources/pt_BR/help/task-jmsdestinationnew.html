Para criar um recurso de destino do JMS

Quando você cria um recurso de destino do JMS, o Servidor GlassFish cria um recurso de objeto de admin para o recurso de destino. Você pode visualizar esse recurso no nó Conectores.

  1. Na árvore de navegação, expanda o nó Recursos.
  2. No nó Recursos, expanda o nó Recursos do JMS.
  3. No nó Recursos do JMS, selecione o nó Recursos de destino do JMS.

    A página Recursos de destino do JMS é aberta.

  4. Na página Recursos de destino do JMS, clique em Novo.

    A página Novos recursos de destino do JMS é aberta.

  5. No campo Nome JNDI, digite o nome JNDI do recurso.

    Recomenda-se utilizar o prefixo de subcontexto de nomeação jms/ para recursos do JMS. Por exemplo: jms/Queue.

  6. No campo Nome do destino físico, digite o nome do destino físico associado a este recurso.

    O Servidor GlassFish cria um destino físico automaticamente quando necessário e o exclui quando você exclui o recurso de destino.

  7. Na lista suspensa Tipo de recurso, selecione javax.jms.Topic ou javax.jms.Queue.
  8. No campo Descrição, digite uma descrição do recurso de destino.
  9. Marque a caixa de seleção Status ativo para ativar o recurso no tempo de execução.

    Por padrão, a opção está ativada.

  10. (Opcional) Na seção Propriedades adicionais, especifique propriedades adicionais do servidor de destino.

    O Servidor GlassFish não define propriedades adicionais para recursos de destino do JMS.

    Para adicionar uma propriedade, clique no botão Adicionar propriedade. Na linha em branco que é exibida, digite o nome da propriedade no campo Nome e digite o valor da propriedade no campo Valor.

  11. Clique em OK.
