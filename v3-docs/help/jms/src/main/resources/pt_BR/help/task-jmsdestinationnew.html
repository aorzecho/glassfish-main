Para Criar um Recurso de Destinação do JMS

Quando você cria um recurso de destino JMS, o GlassFish Server cria um recurso de objeto de admin para o recurso de destino. Você pode visualizar esse recurso no nó Conectores.

  1. Na árvore de navegação, expanda o nó Recursos.

  2. No nó Recursos, expanda o nó Recursos JMS.

  3. No nó Recursos JMS, selecione o nó Recursos de destino JMS.

    A página Recursos de destino JMS é aberta.

  4. Na página Recursos de destino JMS, clique em Novo.

    A página Novos recursos de destino JMS é aberta.

  5. No campo Nome JNDI, digite o nome JNDI do recurso.

    Recomenda-se usar o prefixo de subcontexto de nomeação jms/ para recursos do JMS. Por exemplo: jms/Queue.

  6. No campo Nome do destino físico, digite o nome do destino físico associado a este recurso.

    O GlassFish Server cria um destino físico automaticamente quando necessário e o deleta quando você deleta o recurso de destino.

  7. Na lista drop-down Tipo de recurso, selecione javax.jms.Topic ou javax.jms.Queue.

  8. No campo Descrição, digite uma descrição do recurso de destino.

  9. Marque a caixa de seleção Status ativo para ativar o recurso durante o runtime.

    Por default, a opção está ativada.

  10. Na seção Propriedades adicionais, especifique propriedades adicionais do servidor de destino.

    O GlassFish Server não define propriedades adicionais para recursos de destino do JMS.

    Para adicionar uma propriedade, clique no botão Adicionar propriedade. Na linha em branco que é exibida, digite o nome da propriedade no campo Nome e digite o valor da propriedade no campo Valor.

  11. In the Targets area, do the following:

    1. From the Available column, select the target or targets to which applications that use the resource will be deployed.

      The available targets include the available clusters and server instances, as well as the default server instance, server.

    2. Click Add to move the target to the Selected column.

  12. Clique em OK.
