New JMS Physical Destination

Nella pagina New JMS Physical Destination è possibile creare una nuova destinazione fisica.

Di norma non è necessario creare destinazioni fisiche in quanto al momento della creazione di una risorsa di destinazione JMS è necessario specificare il nome di una destinazione fisica. Se necessaria, la destinazione fisica viene creata automaticamente.

La pagina New JMS Physical Destination contiene le opzioni seguenti.

Name

Nome della destinazione fisica.

Type

Tipo della destinazione fisica. Le opzioni disponibili sono javax.jms.Topic e javax.jms.Queue.

Maximum Number of Unconsumed Messages

Numero massimo di messaggi non utilizzati consentiti per la destinazione. Un valore pari a -1 indica un numero illimitato di messaggi non utilizzati. Il valore predefinito è -1.

Maximum Message Size

Dimensione massima di ogni singolo messaggio, in byte. Un valore pari a -1 indica una dimensione illimitata del messaggio. Il valore predefinito è -1.

Maximum Total Memory

Memoria totale massima, in byte, per messaggi non utilizzati. Un valore pari a -1 indica una dimensione illimitata di memoria per messaggi non utilizzati. Il valore predefinito è -1.

Threshold Limit Behavior

Comportamento desiderato per il broker quando viene raggiunta la soglia del limite di memoria. Sono disponibili le seguenti opzioni:

Reject newest messages

Consente di rifiutare i messaggi più recenti e notificare al client di produzione un'eccezione solo se il messaggio è persistente. È il valore predefinito.

Slow down producers

Consente di ridurre la velocità alla quale i produttori inviano messaggi.

Throw out oldest

Consente di ignorare i messaggi meno recenti.

Throw out lowest-priority messages

Consente di ignorare i messaggi con la priorità minima.

Se il valore è Throw out oldest o Throw out lowest-priority messages e Use Dead Message Queue è impostato su True, i messaggi in eccesso vengono spostati nella coda dei messaggi bloccati.

Maximum Number of Producers

Numero massimo di produttori di messaggi che è possibile associare alla destinazione fisica. Una volta raggiunto il limite, non è possibile creare nuovi produttori. Un valore pari a -1 indica un numero illimitato di produttori. Il valore predefinito è 100.

Batch Delivery Max Message Count

Numero massimo di messaggi che è possibile consegnare ai consumatori in un solo batch. Un valore pari a -1 indica un numero illimitato di messaggi. Il valore predefinito è 1000.

Use Dead Message Queue

Se il valore è impostato su True, i messaggi bloccati vengono posizionati nella coda di messaggi bloccati. Se il valore è impostato su False, i messaggi bloccati vengono ignorati. L'impostazione predefinita per questa opzione è True.

Validate XML Schema

Se il valore è impostato su True, prima dell'invio di un messaggio XML al broker, il runtime del client Message Queue tenterà di convalidarlo in base alle XSD (XML Schema Definition) specificate o, se non è specificata nessuna XSD, in base al DTD. Se non è possibile individuare lo schema specificato o convalidare il messaggio, quest'ultimo non viene inviato e viene generata un'eccezione. L'impostazione predefinita per questa opzione è False.

XML Schema URI List

Elenco separato da spazi di stringhe URI per XSD (XML Schema Definition). Se è abilitata la convalida dello schema XML, gli URI fanno riferimento alla posizione di una o più XSD da utilizzare per tale convalida.
