Disponibilità JMS

Nella pagina Disponibilità JMS è possibile configurare le impostazioni di disponibilità per Java Message Service (JMS).

La pagina Disponibilità JMS contiene le opzioni riportate di seguito.

Carica impostazioni predefinite

Pulsante che consente di ripristinare i valori predefiniti delle impostazioni che dispongono di tali valori. Le impostazioni prive di valori predefiniti restano invariate.

Nome configurazione

Nome della configurazione a cui vengono applicate le impostazioni definite in questa pagina. Questo campo è di sola lettura.

Servizio disponibilità

Tipo di cluster Message Queue utilizzato per fornire i servizi JMS a un cluster GlassFish. Se la casella di controllo Abilitato è selezionata, viene utilizzato un cluster Message Queue avanzato. In caso contrario, viene utilizzato un cluster Message Queue convenzionale.

Se la casella di controllo Abilitato è selezionata, le opzioni Tipo di area di memorizzazione dei messaggi configurazione e Tipo di area di memorizzazione messaggi vengono ignorate.

Tipo di area di memorizzazione configurazione

Tipo di datastore e per i dati di configurazione in un cluster Message Queue convenzionale. Il valore masterbroker specifica l'uso di un broker principale per memorizzare e gestire i dati di configurazione. Il valore shareddb specifica l'uso di un database condiviso per memorizzare i dati di configurazione.

Questa opzione viene ignorata se il servizio disponibilità è abilitato.

Tipo di area di memorizzazione messaggi

Tipo di datastore per i dati dei messaggi nei broker in un cluster tradizionale. Il valore file specifica un'area di memorizzazione file. Il valore jdbc specifica un'area di memorizzazione JDBC.

Questa opzione viene ignorata se il servizio disponibilità è abilitato.

Fornitore database
Nome utente database

URL di accesso al database

Fornitore, utente, password e URL di accesso del database JDBC da utilizzare in una delle situazioni riportate di seguito.

  • Quando il servizio disponibilità è abilitato.

  • Quando il tipo di area di memorizzazione di configurazione è impostato su shareddb.

  • Quando il tipo di area di memorizzazione messaggi è impostato su jdbc.

Per informazioni sui fornitori supportati e sui formati degli url di accesso per ciascun fornitore, vedere JDBC-Based Persistence in Oracle GlassFish Server Message Queue Administration Guide.
