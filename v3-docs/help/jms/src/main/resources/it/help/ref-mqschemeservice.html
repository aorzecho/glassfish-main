Sintassi degli indirizzi dello schema MQ e del servizio MQ

La sintassi completa per l'indirizzo di un servizio di messaggi è

scheme://address-syntax

dove lo schema e la sintassi-indirizzo vengono descritti nella tabella riportata di seguito.

I valori Schema MQ e Servizio MQ sono quelli riportati nelle prime due colonne della tabella seguente.

Nome schema Servizio di connessione Descrizione Sintassi indirizzo
mq jms e ssljms La fase di esecuzione del client Message Queue stabilisce una connessione al port mapper MQ sull'host e sulla porta specificati. Port mapper restituisce una lista di porte di servizio di connessione definite dinamicamente, quindi la fase di esecuzione del client Message Queue esegue una connessione alla porta host del servizio di connessione specificato. [Nomehost][:porta][/Nomeservizio]

Le impostazioni predefinite sono: Nomehost = localhost, porta = 7676, Nomeservizio = jms

Le impostazioni predefinite vengono applicate solo al servizio di connessione jms. Per il servizio di connessione ssljms è necessario specificare tutte le variabili.

Esempio: mq:MyHost:7677/ssljms
mqtcp jms Per stabilire una connessione, la fase di esecuzione del client Message Queue esegue una connessione TCP all'host e alla porta specificati ignorando il port mapper MQ. Nomehost:porta/jms

Esempio: mqtcp:localhost:7676/jms
mqssl ssljms Per stabilire una connessione, la fase di esecuzione del client Message Queue esegue una connessione SSL sicura all'host e alla porta specificati evitando il port mapper MQ. Nomehost:porta/ssljms

Esempio: mqssl:localhost:7676/ssljms
http httpjms La fase di esecuzione del client Message Queue esegue una connessione HTTP a un servlet di tunnel Message Queue nell'URL specificato. Il broker deve essere configurato per l'accesso al servlet di tunnel HTTP, come descritto in Message Queue Administrator's Guide. Nomehost:porta/Radicecontesto/tunnel

Se più istanze di broker utilizzano lo stesso servlet di tunnel, la sintassi per la connessione a un'istanza di broker specifica (anziché a una selezionata in modo casuale) è: http://Nomehost:porta/radicecontesto/tunnel?Nomeserver=Namehost:Nomestanza
https httpsjms La fase di esecuzione del client Message Queue esegue una connessione sicura HTTPS all'URL del servlet di tunnel Message Queue specificato. Il broker deve essere configurato per l'accesso al servlet di tunnel HTTPS, come descritto in Administrator's Guide di Message Queue. Nomehost:porta/Radicecontesto/tunnel

Se più istanze di broker utilizzano lo stesso servlet di tunnel, la sintassi per la connessione a un'istanza di broker specifica (anziché a una selezionata in modo casuale) è: http://Nomehost:porta/radicecontesto/tunnel?Nomeserver=Namehost:Nomestanza
