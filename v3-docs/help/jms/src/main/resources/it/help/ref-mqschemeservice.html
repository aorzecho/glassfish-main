Sintassi di indirizzi MQ Scheme e MQ Service

Per l'indirizzo di un servizio di messaggi viene utilizzata la sintassi

scheme://address-syntax

dove scheme e address-syntax indicano quanto descritto nella tabella sottostante.

I valori MQ Scheme e MQ Service sono quelli riportati nelle prime due colonne della tabella seguente.

Nome schema

Servizio di connessione

Descrizione

Sintassi indirizzo

mq

jms e ssljms

Il runtime del client Message Queue stabilisce una connessione con MQ Port Mapper sull'host e sulla porta specificati. Port Mapper restituisce un elenco di porte di servizio di connessione definite dinamicamente, quindi il runtime del client di Message Queue esegue la connessione alla porta host del servizio di connessione specificato.

[hostName][: port][/serviceName ]

Impostazioni predefinite: hostName = localhost, port = 7676, serviceName = jms

Le impostazioni predefinite vengono applicate solo al servizio di connessione jms. Per il servizio di connessione ssljms, è necessario specificare tutte le variabili.

Esempio: mq:MyHost:7677/ssljms

mqtcp

jms

Il runtime del client Message Queue esegue una connessione TCP all'host e alla porta specificati (ignorando MQ Port Mapper) per stabilire una connessione.

hostName:port /jms

Esempio: mqtcp:localhost:7676/jms

mqssl

ssljms

Il runtime del client Message Queue esegue una connessione SSL sicura all'host e alla porta specificati (evitando MQ Port Mapper) per stabilire una connessione.

hostName:port /ssljms

Esempio: mqssl:localhost:7676/ssljms

http

httpjms

Il runtime del client Message Queue esegue una connessione HTTP a un servlet di tunnel Message Queue nell'URL specificato. Il broker deve essere configurato per l'accesso al servlet di tunnel HTTP, come descritto nel manuale dell'amministratore di Message Queue.

hostName:port /contextRoot/tunnel

Se viene utilizzato lo stesso servlet di tunnel per più istanze del broker, la sintassi per la connessione a un'istanza specifica del broker (anziché a una selezionata a caso) è: http://hostName :port/contextRoot /tunnel?serverName =hostName: instanceName

https

httpsjms

Il runtime del client Message Queue esegue una connessione sicura HTTPS all'URL del servlet di tunnel Message Queue specificato. Il broker deve essere configurato per l'accesso al servlet di tunnel HTTPS, come descritto nel manuale dell'amministratore di Message Queue.

hostName:port /contextRoot/tunnel

Se viene utilizzato lo stesso servlet di tunnel per più istanze del broker, la sintassi per la connessione a un'istanza specifica del broker (anziché a una selezionata a caso) è: http://hostName :port/contextRoot /tunnel?serverName =hostName: instanceName
