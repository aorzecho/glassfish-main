JMS Service

Nella pagina JMS Service è possibile configurare le impostazioni da utilizzare con tutte le connessioni JMS.

Per ulteriori informazioni sulla configurazione del servizio JMS, vedere il Capitolo 17, Using the Java Message Service in Oracle GlassFish Server 3.1 Application Development Guide.

La pagina JMS Service contiene le seguenti opzioni.

Load Defaults

Pulsante che consente di ripristinare i valori predefiniti di tutte le impostazioni.

Ping

Pulsante che consente di verificare che il servizio JMS sia in esecuzione sull'host JMS predefinito. Il servizio JMS è attivo e in funzione; un messaggio notifica che il ping è stato eseguito correttamente.

Configuration Name

Nome della configurazione a cui si applicano le impostazioni visualizzate nella pagina. Campo di sola lettura.

Type

Tipo di accesso per il servizio JMS. Sono disponibili le seguenti opzioni:

EMBEDDED

Consente l'accesso al servizio JMS sull'host locale. Il servizio JMS viene avviato nello stesso computer JVM in cui è installato GlassFish Server. Il broker integrato predefinito viene avviato con un'inizializzazione lenta al primo accesso dei servizi JMS anziché all'avvio di GlassFish Server. Rappresenta il tipo predefinito per il servizio JMS.

LOCAL

Consente l'accesso al servizio JMS sull'host locale. Il provider JMS viene avviato insieme a GlassFish Server.

REMOTE

Consente l'accesso al servizio JMS in un altro sistema. Se si sceglie REMOTE, il servizio JMS non viene avviato all'avvio successivo del server; viene bensì avviato e gestito in Message Queue, pertanto è necessario avviare il broker Message Queue separatamente. Per informazioni sull'avvio del broker, vedere Starting Brokers in Oracle GlassFish Message Queue 4.5 Administration Guide. Se si sceglie questo valore e si utilizza un host remoto, per specificarne il nome seguire le istruzioni in Modificare un host JMS.

Startup Timeout

Numero di secondi di attesa per l'avvio del servizio JMS in GlassFish Server prima dell'interruzione. In caso di sistemi lenti o sovraccarichi, specificare un valore superiore rispetto a quello predefinito. Il valore predefinito è 60.

Start Arguments

Argomenti per la personalizzazione dell'avvio del servizio JMS. È possibile utilizzare tutti gli argomenti disponibili tramite il comando as-install-parent/mq/bin/imqbrokerd.

Reconnect Enabled

Se la casella di controllo Reconnect Enabled è selezionata, il servizio JMS tenta di ricollegarsi a un server di messaggi (o all'elenco di indirizzi di AddressList) quando si perde la connessione. Questa opzione è abilitata per impostazione predefinita.

Reconnect Interval

Numero di secondi tra i tentativi di riconnessione. L'intervallo viene applicato ai tentativi per ciascun indirizzo in AddressList e per gli indirizzi successivi nell'elenco. Se l'intervallo è troppo breve, il broker non riesce a eseguire il ripristino. Se l'intervallo è troppo lungo, si potrebbe verificare un ritardo eccessivo per la riconnessione. Il valore predefinito è 5 secondi.

Reconnect Attempts

Numero di tentativi di connessione (o riconnessione) per ogni indirizzo di AddressList prima che il runtime del client esegua un tentativo con il successivo indirizzo dell'elenco. Un valore pari a -1 indica un numero di tentativi di connessione illimitato (il runtime del client tenta di connettersi al primo indirizzo fino a quando il tentativo non riesce). Il valore predefinito è 3.

Default JMS Host

Nome host JMS predefinito. Il valore predefinito è default_JMS_host.

Address List Behavior

Ordine dei tentativi di connessione. Sono disponibili le seguenti opzioni:

random

Selezionare un indirizzo da AddressList a caso. Se molti client tentano di connettersi utilizzando la stessa factory di connessione, specificare random per evitarne la connessione allo stesso indirizzo. Questa è l'opzione predefinita.

priority

Il tentativo di riconnessione viene sempre eseguito con il primo indirizzo server disponibile in AddressList quindi, se il primo broker non è disponibile, con un altro indirizzo.

Address List Iterations

Numero di ripetizioni del servizio JMS in AddressList nel tentativo di stabilire o ristabilire una connessione. Un valore pari a -1 indica un numero di tentativi illimitato. Il valore predefinito è 3.

MQ Scheme e MQ Service

Nome dello schema di indirizzo Message Queue e nome del servizio di connessione Message Queue, nel caso sia necessario utilizzare uno schema o un servizio non predefinito. Per informazioni sulla sintassi, vedere Sintassi di indirizzi MQ Scheme e MQ Service.

Additional Properties

Proprietà aggiuntive per il servizio JMS.

Per informazioni sulle proprietà disponibili, vedere Proprietà specifiche del servizio JMS.
