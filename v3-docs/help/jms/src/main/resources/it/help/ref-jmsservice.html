Servizio JMS

Nella pagina Servizio JMS è possibile configurare le impostazioni da utilizzare con tutte le connessioni JMS.

Per ulteriori informazioni sulla configurazione del servizio JMS, vedere il Chapter 19, Using the Java Message Service, in Oracle GlassFish Server Application Development Guide.

La pagina Servizio JMS contiene le opzioni riportate di seguito.

Carica impostazioni predefinite

Pulsante che consente di ripristinare i valori predefiniti delle impostazioni che dispongono di tali valori. Le impostazioni prive di valori predefiniti restano invariate.

Ping

Pulsante che consente di verificare se il servizio JMS è in esecuzione sull'host JMS predefinito. Se il servizio JMS è attivo e in esecuzione, un messaggio notifica che il ping è stato eseguito correttamente.

Nome configurazione

Nome della configurazione a cui vengono applicate le impostazioni definite in questa pagina. Questo campo è di sola lettura.

Tipo

Tipo di accesso per il servizio JMS. Sono disponibili le opzioni riportate di seguito.

EMBEDDED

Consente l'accesso al servizio JMS sull'host locale. Il servizio JMS viene avviato nello stesso computer JVM su cui è installato GlassFish Server. Il broker incorporato predefinito viene avviato con un'inizializzazione ritardata al primo accesso dei servizi JMS anziché all'avvio di GlassFish Server. Questo è il tipo predefinito per il servizio JMS.

LOCAL

Consente l'accesso al servizio JMS sull'host locale. Il provider JMS viene avviato insieme a GlassFish Server.

REMOTE

Consente l'accesso al servizio JMS su un altro sistema. Se si sceglie REMOTE, il servizio JMS non viene avviato da GlassFish Server all'avvio successivo del server; viene bensì avviato e gestito da Message Queue; pertanto, è necessario avviare il broker Message Queue separatamente. Per informazioni sull'avvio del broker, vedere Starting Brokers in Oracle GlassFish Server Message Queue Administration Guide. Se si sceglie questo valore e si utilizza un host remoto, seguire le istruzioni riportate in Modificare un host JMS per specificare il nome dell'host remoto.

Timeout avvio

Numero di secondi che GlassFish Server deve attendere per l'avvio del servizio JMS prima dell'interruzione. In caso di sistemi lenti o sovraccarichi, specificare un valore superiore rispetto a quello predefinito. Il valore predefinito è 60.

Argomenti di avvio

Argomenti per la personalizzazione dell'avvio del servizio JMS. Utilizzare qualsiasi argomento disponibile per il comando as-install-parent/mq/bin/imqbrokerd.

Riconnessione abilitata

Se la casella di controllo Riconnessione abilitata è selezionata, il servizio JMS tenta di riconnettersi a un server di messaggi o alla lista di indirizzi in AddressList in caso di connessione persa. Questa opzione è abilitata per impostazione predefinita.

Intervallo di riconnessione

Numero di secondi tra i tentativi di riconnessione. L'intervallo viene applicato ai tentativi per ciascun indirizzo in AddressList e per gli indirizzi successivi nella lista. Se l'intervallo è troppo breve, il broker non riesce a eseguire il recupero. Se l'intervallo è troppo lungo, si potrebbe verificare un ritardo eccessivo per la riconnessione. Il valore predefinito è 5 secondi.

Tentativi di riconnessione

Numero di tentativi di connessione o riconnessione per ogni indirizzo disponibile in AddressList prima che la fase di esecuzione del client esegua un tentativo con il successivo indirizzo della lista. Un valore pari a -1 indica un numero di tentativi di riconnessione illimitato; la fase di esecuzione del client tenta di connettersi al primo indirizzo fino a quando il tentativo non riesce. Il valore predefinito è 3.

Host JMS predefinito

Nome dell'host JMS predefinito. Il valore predefinito è default_JMS_host.

Funzionamento lista di indirizzi

Ordine dei tentativi di connessione. Sono disponibili le opzioni riportate di seguito.

random

Selezionare un indirizzo da AddressList in modo casuale. Se molti client tentano di connettersi utilizzando la stessa Connection Factory, specificare random per evitarne la connessione allo stesso indirizzo. Questa è l'opzione predefinita.

priority

Il tentativo di riconnessione viene sempre eseguito con il primo indirizzo server disponibile in AddressList quindi, se il primo broker non è disponibile, con un altro indirizzo.

Iterazioni lista di indirizzi

Numero di volte in cui il servizio JMS scorre AddressList nel tentativo di stabilire o ristabilire una connessione. Un valore pari a -1 indica un numero di tentativi illimitato. Il valore predefinito è 3.

Schema MQ e Servizio MQ

Nome dello schema di indirizzi Message Queue e nome del servizio di connessione Message Queue, nel caso sia necessario utilizzare uno schema o un servizio non predefinito. Per informazioni sulla sintassi, vedere Sintassi degli indirizzi dello schema MQ e del servizio MQ.

Proprietà aggiuntive

Proprietà aggiuntive per il servizio JMS.

Per informazioni sulle proprietà disponibili, vedere Proprietà specifiche del servizio JMS.
