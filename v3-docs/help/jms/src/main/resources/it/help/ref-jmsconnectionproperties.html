Proprietà specifiche delle factory di connessione JMS

Per la factory di connessione sono disponibili le seguenti proprietà aggiuntive.

ClientId

Specifica l'ID client per una factory di connessione da utilizzare in una sottoscrizione durable.

AddressList

Specifica i nomi (e, facoltativamente, i numeri di porta) di una o più istanze del broker di messaggi con cui le applicazioni comunicano. Ciascun indirizzo dell'elenco è costituito dal nome host (e facoltativamente dalla porta host e dal servizio di connessione) per la connessione. Ad esempio il valore può essere earth o earth:7677. Il numero di porta va specificato se il broker di messaggi è in esecuzione su una porta diversa da quella predefinita (7676). Se la proprietà è stata impostata per specificare più host e porte in un ambiente con cluster, viene utilizzato il primo host disponibile nell'elenco, a meno che la proprietà AddressListBehavior non sia impostata su RANDOM.

Per dettagli, vedere il manuale per lo sviluppatore di Message Queue per client Java.

I valori predefiniti sono l'host locale e il numero di porta predefinito (7676). Il client tenterà una connessione a un broker sulla porta 7676 dell'host locale.

UserName

Nome utente per la factory di connessione.

Il valore predefinito è guest.

Password

Password per la factory di connessione.

Il valore predefinito è guest.

ReconnectEnabled

Se impostato su true, specifica che il runtime del client tenta di ricollegarsi al server di messaggi (o all'elenco di indirizzi di AddressList) quando si perde la connessione.

Il valore predefinito è true.

ReconnectAttempts

Specifica il numero di tentativi di connessione (o riconnessione) per ogni indirizzo di AddressList prima che il runtime del client tenti di collegarsi al successivo indirizzo dell'elenco. Un valore pari a -1 indica un numero di tentativi di connessione illimitato (il runtime del client tenta di connettersi al primo indirizzo fino a quando il tentativo non riesce).

Il valore predefinito è 3.

ReconnectInterval

Specifica l'intervallo in millisecondi tra i tentativi di riconnessione. Il valore viene applicato ai tentativi per ciascun indirizzo in AddressList e per gli indirizzi successivi nell'elenco. Se l'intervallo è troppo breve, il broker non riesce a eseguire il ripristino. Se l'intervallo è troppo lungo, si potrebbe verificare un ritardo eccessivo per la riconnessione.

Il valore predefinito è 30000.

AddressListBehavior

Specifica se i tentativi di connessione vengono eseguiti nell'ordine degli indirizzi nell'attributo AddressList (PRIORITY) o in ordine casuale (RANDOM).

RANDOM indica che per la riconnessione viene scelto un indirizzo casuale da AddressList. Se è probabile che molti client provino a stabilire una connessione mediante la stessa factory di connessione, impostando questo valore si evita che la connessione venga eseguita con lo stesso indirizzo.

PRIORITY indica che il tentativo di riconnessione viene sempre eseguito con il primo indirizzo server disponibile in AddressList quindi, se il primo broker non è disponibile, con un altro indirizzo.

Il valore predefinito è RANDOM.

AddressListIterations

Specifica il numero di runtime del client ripetuti in AddressList nel tentativo di stabilire o ristabilire una connessione. Un valore pari a -1 indica un numero di tentativi illimitato.

Il valore predefinito è 3.
