Creare una destinazione fisica JMS

Di norma non è necessario creare destinazioni fisiche in quanto al momento della creazione di una risorsa di destinazione JMS è necessario specificare il nome di una destinazione fisica. Se necessaria, la destinazione fisica viene creata automaticamente.

  1. Nella struttura ad albero, espandere il nodo Configuration.
  2. Nel nodo Configuration, espandere il nodo Java Message Service.
  3. Nel nodo Java Message Service, selezionare il nodo Physical Destinations.

    Viene visualizzata la pagina Physical Destinations.

  4. Nella pagina Physical Destinations, fare clic su New.

    Viene visualizzata la pagina New Physical Destinations.

  5. Nel campo Name digitare il nome della destinazione fisica.
  6. Nell'elenco a discesa Type, selezionare javax.jms.Topic o javax.jms.Queue.
  7. Nel campo Maximum Number of Unconsumed Messages, digitare il numero massimo di messaggi non utilizzati consentiti per la destinazione.

    Un valore pari a -1 indica un numero illimitato di messaggi non utilizzati. Il valore predefinito è -1.

  8. Nel campo Maximum Message Size, digitare la dimensione massima, in byte, di ogni singolo messaggio.

    Un valore pari a -1 indica una dimensione illimitata del messaggio. Il valore predefinito è -1.

  9. Nel campo Maximum Total Memory, digitare la memoria totale massima, in byte, per messaggi non utilizzati.

    Un valore pari a -1 indica una dimensione illimitata di memoria per messaggi non utilizzati. Il valore predefinito è -1.

  10. Nell'elenco a discesa Threshold Limit Behavior, selezionare il comportamento desiderato per il broker quando viene raggiunta la soglia del limite di memoria.
    • Reject newest messages indica che il broker rifiuterà i messaggi più recenti e notifica un'eccezione al client di produzione solo se il messaggio è persistente.

      Reject newest messages è il valore predefinito.

    • Slow down producers indica che il broker ridurrà la velocità con cui i produttori inviano i messaggi.
    • Throw out oldest indica che il broker deve ignorare i messaggi più vecchi.
    • Throw out lowest-priority messages indica che il broker deve ignorare i messaggi con la priorità più bassa.

    Se il valore è Throw out oldest o Throw out lowest-priority messages e Use Dead Message Queue è impostato su True, i messaggi in eccesso vengono spostati nella coda dei messaggi bloccati.

  11. Nel campo Maximum Number of Producers, digitare il numero massimo di produttori di messaggi che possono essere associati alla destinazione fisica.

    Una volta raggiunto il limite, non è possibile creare nuovi produttori. Un valore pari a -1 indica un numero illimitato di produttori. Il valore predefinito è 100.

  12. Nel campo Batch Delivery Max Message Count, digitare il numero massimo di messaggi che è possibile inviare ai consumatori in un solo batch.

    Un valore pari a -1 indica un numero illimitato di messaggi. Il valore predefinito è 1000.

  13. Nell'elenco Use Dead Message Queue, selezionare True per posizionare i messaggi bloccati nella coda di messaggi bloccati.

    Se il valore è impostato su False, i messaggi bloccati vengono ignorati. L'impostazione predefinita per questa opzione è True.

  14. Nell'elenco a discesa Validate XML Schema Enabled, selezionare True per abilitare la convalida dello schema XML per la destinazione.

    Se il valore è impostato su True, prima dell'invio di un messaggio XML al broker, il runtime del client Message Queue tenterà di convalidarlo in base alle XSD (XML Schema Definition) specificate o, se non è specificata nessuna XSD, in base al DTD. Se non è possibile individuare lo schema specificato o convalidare il messaggio, quest'ultimo non viene inviato e viene generata un'eccezione. L'impostazione predefinita per questa opzione è False.

  15. Nel campo XML Schema URI List, digitare un elenco separato da spazi di stringhe URI per XSD (XML Schema Definition).

    Se è abilitata la convalida dello schema XML, gli URI fanno riferimento alla posizione di una o più XSD da utilizzare per tale convalida.
