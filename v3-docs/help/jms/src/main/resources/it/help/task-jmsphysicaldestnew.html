Creare una destinazione fisica JMS

Di norma non è necessario creare destinazioni fisiche in quanto al momento della creazione di una risorsa di destinazione JMS è necessario specificare il nome di una destinazione fisica. Se necessaria, la destinazione fisica viene creata automaticamente.

  1. Nell'albero di navigazione espandere il nodo Configurazione.

  2. Select the instance to configure:

    • To configure a particular instance, select the instance's config node. For example, for the default instance, server, select the server-config node.

    • To configure the default settings for future instances that use a copy of default-config, select the default-config node.

  3. Nel nodo Configurazione, espandere il nodo Java Message Service.

  4. Nel nodo Java Message Service, selezionare il nodo Destinazioni fisiche.

    Viene visualizzata la pagina Destinazioni fisiche.

  5. Nella pagina Destinazioni fisiche, fare clic su Nuovo.

    Viene visualizzata la pagina Nuova destinazione fisica.

  6. Nel campo Nome digitare il nome della destinazione fisica.

  7. Nell'elenco a discesa Tipo, selezionare javax.jms.Topic o javax.jms.Queue.

  8. Nel campo Numero massimo di messaggi non utilizzati, digitare il numero massimo di messaggi non utilizzati consentiti per la destinazione.

    Un valore pari a -1 indica un numero illimitato di messaggi non utilizzati. Il valore predefinito è -1.

  9. Nel campo Dimensione massima messaggio, digitare la dimensione massima, in byte, di ogni singolo messaggio.

    Un valore pari a -1 indica una dimensione illimitata del messaggio. Il valore predefinito è -1.

  10. Nel campo Memoria totale massima, digitare la memoria totale massima, in byte, per i messaggi non utilizzati.

    Un valore pari a -1 indica una dimensione illimitata di memoria per messaggi non utilizzati. Il valore predefinito è -1.

  11. Nell'elenco a discesa Funzionamento limite soglia, selezionare il comportamento desiderato per il broker quando viene raggiunta la soglia del limite di memoria.

    • Rifiuta i messaggi più recenti indica che il broker rifiuterà i messaggi più recenti e notifica un'eccezione al client di produzione solo se il messaggio è persistente.

      Rifiuta i messaggi più recenti è il valore predefinito.

    • Rallenta producer indica che il broker ridurrà la velocità con cui i produttori inviano i messaggi.

    • Elimina i meno recenti indica che il broker deve ignorare i messaggi più vecchi.

    • Elimina i messaggi con la priorità più bassa indica che il broker deve ignorare i messaggi con la priorità più bassa.

    Se il valore è Elimina i meno recenti o Elimina i messaggi con la priorità più bassa e Usa coda dei messaggi bloccati è impostato su True, i messaggi in eccesso vengono spostati nella coda dei messaggi bloccati.

  12. Nel campo Numero massimo di producer, digitare il numero massimo di produttori di messaggi che possono essere associati alla destinazione fisica.

    Una volta raggiunto il limite, non è possibile creare nuovi produttori. Un valore pari a -1 indica un numero illimitato di produttori. Il valore predefinito è 100.

  13. Nel campo Numero massimo messaggi per consegna batch, digitare il numero massimo di messaggi che è possibile inviare ai consumatori in un solo batch.

    Un valore pari a -1 indica un numero illimitato di messaggi. Il valore predefinito è 1000.

  14. Nell'elenco a discesa Usa coda dei messaggi bloccati, selezionare True per posizionare i messaggi bloccati nella coda di messaggi bloccati.

    Se il valore è impostato su False, i messaggi bloccati vengono ignorati. L'impostazione predefinita per questa opzione è True.

  15. Nell'elenco a discesa Convalida schema XML abilitata, selezionare True per abilitare la convalida dello schema XML per la destinazione.

    Se il valore è impostato su True, prima dell'invio di un messaggio XML al broker, la fase di esecuzione del client Message Queue tenterà di convalidarlo in base alle XSD (XML Schema Definition) specificate o, se non è specificata nessuna XSD, in base al DTD. Se non è possibile individuare lo schema specificato o convalidare il messaggio, quest'ultimo non viene inviato e viene generata un'eccezione. L'impostazione predefinita per questa opzione è False.

  16. Nel campo Lista URI schema XML, digitare una lista separata da spazi di stringhe URI per XSD (XML Schema Definition).

    Se è abilitata la convalida dello schema XML, gli URI fanno riferimento alla posizione di una o più XSD da utilizzare per tale convalida.
