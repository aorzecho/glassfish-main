Configurare il servizio JMS

Nella pagina JMS Service è possibile configurare le impostazioni da utilizzare con tutte le connessioni JMS.

Per ulteriori informazioni sulla configurazione del servizio JMS, vedere il Capitolo 16, Using the Java Message Service in Oracle GlassFish Server 3.1 Application Development Guide.

  1. Nella struttura ad albero, espandere il nodo Configuration.
  2. Nel nodo Configuration, selezionare il nodo Java Message Service.

    Viene visualizzata la pagina JMS Service.

  3. Nell'elenco a discesa Type:
    • Selezionare EMBEDDED per accedere al servizio JMS sull'host locale.

      Il servizio JMS viene avviato nello stesso computer JVM in cui è installato GlassFish Server. Il broker integrato predefinito viene avviato con un'inizializzazione lenta al primo accesso dei servizi JMS anziché all'avvio di GlassFish Server. Rappresenta il tipo predefinito per il servizio JMS.

    • Selezionare LOCAL per accedere al servizio JMS sull'host locale.

      Il servizio JMS viene avviato insieme a GlassFish Server.

    • Selezionare REMOTE per accedere al servizio JMS su un altro sistema.

      Se si sceglie REMOTE, il servizio JMS non viene avviato all'avvio successivo del server; viene bensì avviato e gestito in Message Queue, pertanto è necessario avviare il broker Message Queue separatamente. Per informazioni sull'avvio del broker, vedere Starting Brokers in Oracle GlassFish Server Message Queue 4.5 Administration Guide. Se si sceglie questo valore e si utilizza un host remoto, per specificarne il nome seguire le istruzioni in Modificare un host JMS.

  4. Nel campo Startup Timeout immettere il numero di secondi di attesa di attesa per l'avvio del servizio JMS in GlassFish Server prima dell'interruzione.

    In caso di sistemi lenti o sovraccarichi, specificare un valore superiore rispetto a quello predefinito. Il valore predefinito è 60.

  5. Nel campo Start Arguments digitare argomenti per la personalizzazione dell'avvio del servizio di JMS.

    È possibile utilizzare tutti gli argomenti disponibili tramite il comando as-install-parent/mq/bin/imqbrokerd.

  6. Selezionare la casella di controllo Reconnect Enabled per impostare nel servizio JMS l'esecuzione di nuovi tentativi di collegamento a un server di messaggi (o all'elenco di indirizzi di AddressList) quando si perde la connessione.

    Questa opzione è abilitata per impostazione predefinita.

  7. Nel campo Reconnect Interval, digitare il numero di secondi tra i tentativi di riconnessione.

    L'intervallo viene applicato ai tentativi per ciascun indirizzo in AddressList e per gli indirizzi successivi nell'elenco. Se l'intervallo è troppo breve, il broker non riesce a eseguire il ripristino. Se l'intervallo è troppo lungo, si potrebbe verificare un ritardo eccessivo per la riconnessione. Il valore predefinito è 5 secondi.

  8. Nel campo Reconnect Attempts, digitare il numero di tentativi di connessione (o riconnessione) per ogni indirizzo di AddressList prima che il runtime del client esegua un tentativo con il successivo indirizzo dell'elenco.

    Un valore pari a -1 indica un numero di tentativi di connessione illimitato (il runtime del client tenta di connettersi al primo indirizzo fino a quando il tentativo non riesce). Il valore predefinito è 3.

  9. Nell'elenco a discesa Default JMS Host, selezionare il nome dell'host JMS predefinito.

    Il valore predefinito è default_JMS_host.

  10. Nell'elenco a discesa Address List Behavior, selezionare l'ordine dei tentativi di connessione.
    • random indica la seleziona casuale di un indirizzo da AddressList.

      Se molti client tentano di connettersi utilizzando la stessa factory di connessione, specificare random per evitarne la connessione allo stesso indirizzo. Questa è l'opzione predefinita.

    • priority indica che il tentativo di riconnessione viene sempre eseguito con il primo indirizzo server disponibile in AddressList quindi, se il primo broker non è disponibile, con un altro indirizzo.
  11. Nel campo Address List Iterations, digitare il numero di ripetizioni del servizio JMS in AddressList nel tentativo di stabilire o ristabilire una connessione.

    Un valore pari a -1 indica un numero di tentativi illimitato. Il valore predefinito è 3.

  12. Nei campi MQ Scheme e MQ Service digitare il nome dello schema di indirizzo Message Queue e il nome del servizio di connessione Message Queue, nel caso sia necessario utilizzare uno schema o un servizio non predefinito.

    Per informazioni sulla sintassi, vedere Sintassi di indirizzi MQ Scheme e MQ Service.

  13. (Opzionale) Nell'area Additional Properties, specificare le proprietà aggiuntive per il servizio JMS.

    Per informazioni sulle proprietà disponibili, vedere Proprietà specifiche del servizio JMS.

    • Per aggiungere una proprietà, fare clic sul pulsante Add Property. Nella riga vuota visualizzata, digitare il nome della proprietà nel campo Name e il valore della proprietà nel campo Value.
    • Per modificare una proprietà, è necessario cambiare il valore del campo Value della proprietà.
    • Per eliminare una proprietà, selezionare la casella di controllo a sinistra del campo Name relativo alla proprietà da eliminare, quindi fare clic sul pulsante Delete Properties.
  14. Fare clic su Ping per verificare che il servizio JMS sia in esecuzione sull'host JMS predefinito.

    Il servizio JMS è attivo e in funzione; un messaggio notifica che il ping è stato eseguito correttamente.

  15. Fare clic su Save.

    Suggerimento - Per tornare alle impostazioni predefinite, fare clic sul pulsante Load Defaults, quindi su Save.

Altre informazioni
Accesso a server remoti

Se vengono modificati il provider e l'host in un sistema remoto, tutte le applicazioni JMS vengono eseguite nel server remoto. Per utilizzare il server locale e uno o più server remoti, creare una risorsa factory di connessione con la proprietà AddressList per creare connessioni con accesso ai server remoti. Vedere Creare una risorsa factory di connessione JMS .

Comando asadmin equivalente

jms-ping
