Configurare il servizio JMS

Nella pagina Servizio JMS è possibile configurare le impostazioni da utilizzare con tutte le connessioni JMS.

Per ulteriori informazioni sulla configurazione del servizio JMS, vedere il Chapter 19, Using the Java Message Service, in Oracle GlassFish Server Application Development Guide.

  1. Nell'albero di navigazione espandere il nodo Configurazione.

  2. Select the instance to configure:

    • To configure a particular instance, select the instance's config node. For example, for the default instance, server, select the server-config node.

    • To configure the default settings for future instances that use a copy of default-config, select the default-config node.

  3. Nel nodo Configurazione, selezionare il nodo Java Message Service.

    Viene visualizzata la pagina Servizio JMS.

  4. Nell'elenco a discesa Tipo:

    • Selezionare EMBEDDED per accedere al servizio JMS sull'host locale.

      Il servizio JMS viene avviato nello stesso computer JVM in cui è installato GlassFish Server. Il broker integrato predefinito viene avviato con un'inizializzazione lenta al primo accesso dei servizi JMS anziché all'avvio di GlassFish Server. Rappresenta il tipo predefinito per il servizio JMS.

    • Selezionare LOCAL per accedere al servizio JMS sull'host locale.

      Il servizio JMS viene avviato insieme a GlassFish Server.

    • Selezionare REMOTE per accedere al servizio JMS su un altro sistema or on a cluster.

      Se si sceglie REMOTE, il servizio JMS non viene avviato all'avvio successivo del server; viene bensì avviato e gestito in Message Queue, pertanto è necessario avviare il broker Message Queue separatamente. Per informazioni sull'avvio del broker, vedere Starting Brokers in Oracle GlassFish Server Message Queue Administration Guide. Se si sceglie questo valore e si utilizza un host remoto, per specificarne il nome seguire le istruzioni in Modificare un host JMS.

  5. Nel campo Timeout avvio immettere il numero di secondi di attesa di attesa per l'avvio del servizio JMS in GlassFish Server prima dell'interruzione.

    In caso di sistemi lenti o sovraccarichi, specificare un valore superiore rispetto a quello predefinito. Il valore predefinito è 60.

  6. Nel campo Argomenti di avvio digitare argomenti per la personalizzazione dell'avvio del servizio di JMS.

    Utilizzare qualsiasi argomento disponibile mediante il comando as-install-parent/mq/bin/imqbrokerd.

  7. Selezionare la casella di controllo Riconnessione abilitata per impostare nel servizio JMS l'esecuzione di nuovi tentativi di collegamento a un server di messaggi (o alla lista di indirizzi di AddressList) quando si perde la connessione.

    Questa opzione è abilitata per impostazione predefinita.

  8. Nel campo Intervallo di riconnessione, digitare il numero di secondi tra i tentativi di riconnessione.

    L'intervallo viene applicato ai tentativi per ciascun indirizzo in AddressList e per gli indirizzi successivi nella lista. Se l'intervallo è troppo breve, il broker non riesce a eseguire il recupero. Se l'intervallo è troppo lungo, si potrebbe verificare un ritardo eccessivo per la riconnessione. Il valore predefinito è 5 secondi.

  9. Nel campo Tentativi di riconnessione, digitare il numero di tentativi di connessione (o riconnessione) per ogni indirizzo di AddressList prima che la fase di esecuzione del client esegua un tentativo con il successivo indirizzo della lista.

    Un valore pari a -1 indica un numero di tentativi di connessione illimitato (la fase di esecuzione del client tenta di connettersi al primo indirizzo fino a quando il tentativo non riesce). Il valore predefinito è 3.

  10. Nell'elenco a discesa Host JMS predefinito, selezionare il nome dell'host JMS predefinito.

    Il valore predefinito è default_JMS_host.

  11. Nell'elenco a discesa Funzionamento lista di indirizzi, selezionare l'ordine dei tentativi di connessione.

    • random indica la seleziona casuale di un indirizzo da AddressList.

      Se molti client tentano di connettersi utilizzando la stessa Connection Factory, specificare random per evitarne la connessione allo stesso indirizzo. Questa è l'opzione predefinita.

    • priority indica che il tentativo di riconnessione viene sempre eseguito con il primo indirizzo server disponibile in AddressList quindi, se il primo broker non è disponibile, con un altro indirizzo.

  12. Nel campo Iterazioni lista di indirizzi, digitare il numero di ripetizioni del servizio JMS in AddressList nel tentativo di stabilire o ristabilire una connessione.

    Un valore pari a -1 indica un numero di tentativi illimitato. Il valore predefinito è 3.

  13. Nei campi Schema MQ e Servizio MQ digitare il nome dello schema di indirizzo Message Queue e il nome del servizio di connessione Message Queue, nel caso sia necessario utilizzare uno schema o un servizio non predefinito.

    Per informazioni sulla sintassi, vedere Sintassi degli indirizzi dello schema MQ e del servizio MQ.

  14. Nell'area Proprietà aggiuntive, specificare le proprietà aggiuntive per il servizio JMS.

    Per informazioni sulle proprietà disponibili, vedere Proprietà specifiche del servizio JMS.

    • Per aggiungere una proprietà, fare clic sul pulsante Aggiungi proprietà. Nella riga vuota visualizzata, digitare il nome della proprietà nel campo Nome e il valore della proprietà nel campo Valore.

    • Per modificare una proprietà, è necessario modificare il valore del campo Valore della proprietà.

    • Per eliminare una proprietà, selezionare la casella di controllo a sinistra del campo Nome relativo alla proprietà da eliminare, quindi fare clic sul pulsante Elimina proprietà.

  15. Fare clic su Ping per verificare che il servizio JMS sia in esecuzione sull'host JMS predefinito.

    Il servizio JMS è attivo e in funzione; un messaggio notifica che il ping è stato eseguito correttamente.

  16. Fare clic su Salva.

    Suggerimento:

    Per tornare alle impostazioni predefinite, fare clic sul pulsante Carica impostazioni predefinite, quindi su Salva.
Accesso a server remoti

Se vengono modificati il provider e l'host in un sistema remoto, tutte le applicazioni JMS vengono eseguite nel server remoto. Per utilizzare il server locale e uno o più server remoti, creare una risorsa Connection Factory con la proprietà AddressList per creare connessioni con accesso ai server remoti. Vedere Creare una risorsa Connection Factory JMS.

Comando asadmin equivalente

jms-ping
