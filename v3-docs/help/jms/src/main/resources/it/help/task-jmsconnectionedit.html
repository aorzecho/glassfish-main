Modificare una risorsa Connection Factory JMS

  1. Nell'albero di navigazione espandere il nodo Risorse.

  2. Nel nodo Risorse espandere il nodo Risorse JMS.

  3. Nel nodo Risorse JMS selezionare il nodo Connection Factory JMS.

    Viene visualizzata la pagina Connection Factory JMS.

  4. Nella pagina Connection Factory JMS, fare clic sul nome della Connection Factory da modificare.

    Viene visualizzata la pagina Modifica Connection Factory JMS.

  5. Nel campo Descrizione, digitare una descrizione della Connection Factory.

  6. Selezionare la casella di controllo Stato abilitato per abilitare la risorsa in fase di esecuzione.

    Questa opzione è abilitata per impostazione predefinita.

  7. Nel campo Dimensione pool iniziale e minima, specificare il numero minimo di connessioni per il pool.

    Il valore predefinito è 8.

  8. Nel campo Dimensione massima pool, specificare il numero massimo di connessioni per il pool.

    Il valore predefinito è 32.

  9. Nel campo Quantità di ridimensionamento pool, specificare il numero di connessioni da rimuovere allo scadere del timeout di inattività del pool.

    Il valore predefinito è 2.

  10. Nel campo Timeout inattività, specificare il numero massimo di secondi durante i quali una connessione può rimanere inattiva nel pool.

    Il valore predefinito è 300.

  11. Nel campo Tempo massimo di attesa, specificare il tempo di attesa per la richiesta di connessione di un'applicazione, prima che si verifichi il timeout della connessione.

    Il valore predefinito è 60000.

  12. Se la casella di controllo In caso di qualsiasi errore: Chiudi tutte le connessioni è selezionata, in GlassFish Server tutte le connessioni del pool vengono chiuse e quindi ristabilite in caso di errore di una singola connessione.

    Se non si seleziona la casella di controllo, le singole connessioni vengono ristabilite solo quando vengono utilizzate.

  13. Per una risorsa Connection Factory JMS, specificare il valore Supporto transazioni come segue:

    • Specificare XATransaction (valore predefinito) per una risorsa utilizzabile in transazioni che implicano l'uso di più risorse nell'ambito della transazione,

      ad esempio la presente risorsa in abbinamento a una risorsa JDBC, a una risorsa del connettore o a un'altra risorsa della Connection Factory JMS. Questo valore offre la massima flessibilità. Una risorsa configurata come XATransaction verrà utilizzata in operazioni di commit a due fasi.

    • Specificare LocalTransaction per una risorsa utilizzabile in transazioni che prevedono l'uso di una sola risorsa nell'ambito della transazione o come ultimo agente in una transazione distribuita che prevede l'uso di più risorse XA.

      Il valore offre prestazioni decisamente migliori. Una risorsa configurata come LocalTransaction non verrà utilizzata in operazioni di commit a due fasi.

    • Specificare NoTransaction per una risorsa che non può mai essere utilizzata in una transazione.

      L'impostazione è di uso limitato in applicazioni JMS.

  14. Selezionare la casella di controllo Supporto transazioni per abilitare la convalida della connessione.

    Questa opzione è disabilitata per impostazione predefinita.

  15. Nella tabella Proprietà aggiuntive, specificare le proprietà aggiuntive per la Connection Factory.

    Per informazioni sulle proprietà disponibili, vedere Proprietà specifiche delle Connection Factory JMS.

    • Per aggiungere una proprietà, fare clic sul pulsante Aggiungi proprietà. Nella riga vuota visualizzata, digitare il nome della proprietà nel campo Nome e il valore della proprietà nel campo Valore.

    • Per modificare una proprietà, è necessario modificare il valore del campo Valore della proprietà.

    • Per eliminare una proprietà, selezionare la casella di controllo a sinistra del campo Nome relativo alla proprietà da eliminare, quindi fare clic sul pulsante Elimina proprietà.

  16. Optionally, click the Targets tab to go to the JMS Connection Factory Resource Targets page. On this page, do the following:

    1. Click Manage Targets to open the Manage Resource Targets page.

      On this page, move targets between the Available column and the Selected column. Make sure to place in the Selected column the target or targets to which applications that use the resource will be deployed. The available targets include the available clusters and server instances, as well as the default server instance, server. Click OK to save the changes.

    2. Select the checkbox for a target, then click Enable or Disable to enable or disable the resource for that target.

  17. Fare clic su Salva.

    Suggerimento:

    Per tornare alle impostazioni predefinite, fare clic sul pulsante Carica impostazioni predefinite, quindi su Salva.
