Modificare una risorsa factory di connessione JMS

  1. Nella struttura ad albero, espandere il nodo Resources.
  2. Nel nodo Resources espandere il nodo JMS Resources.
  3. Nel nodo JMS Resources selezionare il nodo JMS Connection Factories.

    Viene visualizzata la pagina JMS Connection Factories.

  4. Nella pagina JMS Connection Factories, fare clic sul nome della factory di connessione da modificare.

    Viene visualizzata la pagina Edit JMS Connection Factory.

  5. Nel campo Description, digitare una descrizione della factory di connessione.
  6. Selezionare la casella di controllo Status Enabled per abilitare la risorsa in fase di runtime.

    Questa opzione è abilitata per impostazione predefinita.

  7. Nel campo Initial and Minimum Pool Size, specificare il numero minimo di connessioni per il pool.

    Il valore predefinito è 8.

  8. Nel campo Maximum Pool Size, specificare il numero massimo di connessioni per il pool.

    Il valore predefinito è 32.

  9. Nel campo Pool Resize Quantity, specificare il numero di connessioni da rimuovere allo scadere del timeout di inattività del pool.

    Il valore predefinito è 2.

  10. Nel campo Idle Timeout, specificare il numero massimo di secondi durante i quali una connessione può rimanere inattiva nel pool.

    Il valore predefinito è 300.

  11. Nel campo Max Wait Time, specificare il tempo di attesa per la richiesta di connessione di un'applicazione, prima che si verifichi il timeout della connessione.

    Il valore predefinito è 60000.

  12. Se la casella di controllo On Any Failure Close All Connections è selezionata, in GlassFish Server tutte le connessioni del pool vengono chiuse e quindi ristabilite in caso di errore di una singola connessione.

    Se non si seleziona la casella di controllo, le singole connessioni vengono ristabilite solo quando vengono utilizzate.

  13. Per una risorsa factory di connessione JMS, specificare il valore Transaction Support come segue:
    • Specificare XATransaction (valore predefinito) per una risorsa utilizzabile in transazioni che implicano l'uso di più risorse nell'ambito della transazione,

      ad esempio la presente risorsa in abbinamento a una risorsa JDBC, a una risorsa del connettore o a un'altra risorsa della factory di connessione JMS. Questo valore offre la massima flessibilità. Una risorsa configurata come XATransaction verrà utilizzata in operazioni di commit a due fasi.

    • Specificare LocalTransaction per una risorsa utilizzabile in transazioni che prevedono l'uso di una sola risorsa o come ultimo agente in una transazione distribuita che prevede l'uso di più risorse XA.

      Il valore offre prestazioni decisamente migliori. Una risorsa configurata come LocalTransaction non verrà utilizzata in operazioni di commit a due fasi.

    • Specificare NoTransaction per una risorsa che non può mai essere utilizzata in una transazione.

      L'impostazione è di uso limitato in applicazioni JMS.

  14. Selezionare la casella di controllo Connection Validation per abilitare la convalida della connessione.

    Questa opzione è disabilitata per impostazione predefinita.

  15. (Opzionale) Nella tabella Additional Properties, specificare le proprietà aggiuntive per la factory di connessione.

    Per informazioni sulle proprietà disponibili, vedere Proprietà specifiche delle factory di connessione JMS.

    • Per aggiungere una proprietà, fare clic sul pulsante Add Property. Nella riga vuota visualizzata, digitare il nome della proprietà nel campo Name e il valore della proprietà nel campo Value.
    • Per modificare una proprietà, è necessario cambiare il valore del campo Value della proprietà.
    • Per eliminare una proprietà, selezionare la casella di controllo a sinistra del campo Name relativo alla proprietà da eliminare, quindi fare clic sul pulsante Delete Properties.
  16. Fare clic su Save.

    Suggerimento - Per tornare alle impostazioni predefinite, fare clic sul pulsante Load Defaults, quindi su Save.
