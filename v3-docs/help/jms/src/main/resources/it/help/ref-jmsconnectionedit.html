Modifica Connection Factory JMS

Nella pagina Modifica Connection Factory JMS è possibile modificare le impostazioni per una Connection Factory JMS.

La pagina Modifica Connection Factory JMS include le opzioni riportate di seguito.

Carica impostazioni predefinite

Pulsante che consente di ripristinare i valori predefiniti delle impostazioni che dispongono di tali valori. Le impostazioni prive di valori predefiniti restano invariate.

Nome pool

Nome univoco del connection pool del connettore creato per la Connection Factory. Questo nome corrisponde inoltre al nome JNDI della Connection Factory. Il campo relativo al nome del pool è di sola lettura.

Tipo di risorsa

Tipo di Connection Factory. Il campo Tipo di risorsa è di sola lettura. È possibile specificare il tipo di risorsa solo quando si crea una nuova Connection Factory JMS.

Descrizione

Descrizione della Connection Factory.

Stato

Stato della Connection Factory. La Connection Factory può essere abilitata o disabilitata. Questa opzione è abilitata per impostazione predefinita.

Dimensione pool iniziale e minima

Numero minimo di connessioni nel pool per la Connection Factory. Il valore predefinito è 8.

Dimensione massima pool

Numero massimo di connessioni nel pool. Il valore predefinito è 32.

Quantità di ridimensionamento pool

Numero di connessioni da rimuovere allo scadere del timeout di inattività del pool. Il valore predefinito è 2.

Timeout inattività

Tempo massimo, in secondi, durante il quale una connessione può rimanere inattiva nel pool. Il valore predefinito è 300.

Tempo massimo di attesa

Quantità di tempo che l'applicazione che richiede la connessione attende prima che si verifichi il timeout della connessione. Il valore predefinito è 60000.

In caso di qualsiasi errore

Se la casella di controllo Chiudi tutte le connessioni è selezionata, GlassFish Server chiude tutte le connessioni nel pool e le ristabilisce se una singola connessione non riesce. Questa opzione è disabilitata per impostazione predefinita.

Supporto transazioni

Il valore può essere uno dei seguenti:

XATransaction

È possibile utilizzare la risorsa per le transazioni che implicano l'uso di più risorse nell'ambito della transazione. Questo è il valore predefinito per una Connection Factory JMS.

Ad esempio, è possibile che le transazioni implichino l'uso di questa risorsa insieme a una risorsa JDBC, una risorsa connettore o un'altra risorsa Connection Factory JMS. Questo valore offre la massima flessibilità. Una risorsa configurata come XATransaction verrà utilizzata in operazioni di commit a due fasi.

LocalTransaction

È possibile utilizzare la risorsa per le transazioni che implicano l'uso di una sola risorsa nell'ambito della transazione o come ultimo agente in una transazione distribuita che prevede l'uso di più risorse XA.

Questo valore consente prestazioni decisamente migliori. Una risorsa configurata come LocalTransaction non verrà utilizzata in operazioni di commit a due fasi.

NoTransaction

La risorsa non può mai essere utilizzata nelle transazioni. Questa impostazione è di uso limitato nelle applicazioni JMS.

Convalida connessione

Se la casella di controllo Richiesta è selezionata, le connessioni vengono convalidate prima di essere fornite all'applicazione. Se la convalida di una risorsa non riesce, la risorsa viene eliminata e viene creata e restituita una nuova risorsa. Questa opzione è disabilitata per impostazione predefinita.
