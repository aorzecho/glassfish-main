Edit JMS Connection Factory

Nella pagina Edit JMS Connection Factory è possibile modificare le impostazioni per una factory di JMS.

La pagina Edit JMS Connection Factory include le seguenti opzioni.

Load Defaults

Pulsante che consente di ripristinare i valori predefiniti delle impostazioni che dispongono di tali valori. Le impostazioni prive di valori predefiniti restano invariate.

Pool Name

Nome univoco del pool di connessione del connettore creato per la factory di connessione. Tale nome corrisponde inoltre al nome JNDI della factory di connessione. Il campo relativo al nome del pool è di sola lettura.

Resource Type

Tipo della factory di connessione. Il campo Resource Type è di sola lettura. È possibile specificare il tipo di risorsa solo quando si crea una nuova factory JMS di connessione.

Description

Descrizione della factory di connettore.

Status

Stato della factory di connessione. La factory di connessione può essere abilitata o disabilitata. Questa opzione è abilitata per impostazione predefinita.

Initial and Minimum Pool Size

Numero minimo di connessioni nel pool per la factory di connessione. Il valore predefinito è 8.

Maximum Pool Size

Numero massimo di connessioni per il pool. Il valore predefinito è 32.

Pool Resize Quantity

Numero di connessioni da rimuovere allo scadere del timeout di inattività del pool. Il valore predefinito è 2.

Idle Timeout

Numero massimo di secondi durante i quali una connessione può rimanere inattiva nel pool. Il valore predefinito è 300.

Max Wait Time

Tempo di attesa per una richiesta di connessione da parte dell'applicazione prima che si verifichi di un timeout. Il valore predefinito è 60000.

On Any Failure

Se la casella di controllo Close All Connections è selezionata, in GlassFish Server tutte le connessioni del pool vengono chiuse e quindi ristabilite in caso di errore di una singola connessione. Questa opzione è disabilitata per impostazione predefinita.

Transaction Support

Il valore può essere uno qualsiasi dei seguenti:

XATransaction

La risorsa può essere utilizzata per transazioni che implicano l'uso di più risorse nell'ambito della transazione. È il valore predefinito per una factory di connessione JMS.

Ad esempio è possibile che in una transazione sia necessario utilizzare questa risorsa in abbinamento a una risorsa JDBC, una risorsa del connettore o un'altra risorsa della factory di connessione JMS. Questo valore offre la massima flessibilità. Una risorsa configurata come XATransaction verrà utilizzata in operazioni di commit a due fasi.

LocalTransaction

È possibile utilizzare la risorsa in transazioni che prevedono l'uso di una sola risorsa o come ultimo agente in una transazione distribuita che prevede l'uso di più risorse XA.

Il valore offre prestazioni decisamente migliori. Le risorse configurate come LocalTransaction non vengono utilizzate in operazioni di commit a due fasi.

NoTransaction

La risorsa non può mai partecipare alle transazioni. L'impostazione è di uso limitato in applicazioni JMS.

Connection Validation

Se la casella di controllo Required è selezionata, le connessioni vengono convalidate prima di essere fornite all'applicazione. Se la convalida non viene eseguita, la risorsa viene eliminata. Viene creata e restituita una risorsa nuova. Questa opzione è disabilitata per impostazione predefinita.

Additional Properties

Proprietà aggiuntive per la factory di connessione.

Per informazioni sulle proprietà disponibili, vedere Proprietà specifiche delle factory di connessione JMS.
