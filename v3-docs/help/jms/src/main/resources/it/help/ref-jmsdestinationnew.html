Nuova risorsa di destinazione JMS

Nella pagina Nuova risorsa di destinazione JMS è possibile creare una nuova risorsa di destinazione JMS.

La pagina Nuova risorsa di destinazione JMS contiene le opzioni riportate di seguito.

Nome JNDI

Nome univoco che identifica la risorsa di destinazione.

Si consiglia di utilizzare il prefisso di contesto secondario di denominazione jms/ per le risorse JMS. Ad esempio: jms/Queue.

Nome destinazione fisica

Nome della destinazione fisica da associare a questa risorsa.

In caso di necessità, GlassFish Server crea la destinazione fisica automaticamente e la elimina al momento dell'eliminazione della risorsa di destinazione.

Tipo di risorsa

Tipo di risorsa di destinazione. Le opzioni disponibili sono javax.jms.Topic e javax.jms.Queue.

Descrizione

Descrizione della risorsa di destinazione.

Stato

Stato della risorsa di destinazione. È possibile abilitare o disabilitare la risorsa di destinazione. Questa opzione è abilitata per impostazione predefinita.

Proprietà aggiuntive

Proprietà aggiuntive per la risorsa di destinazione. In GlassFish Server non è definita alcuna proprietà aggiuntiva per le risorse di destinazione JMS.

Destinazioni

Cluster e istanze standalone per la risorsa di destinazione. Per spostare le destinazioni desiderate nella colonna Destinazioni selezionate, utilizzare i pulsanti Aggiungi e Aggiungi tutto. Per spostare le destinazioni non necessarie nella colonna Destinazioni disponibili, utilizzare i pulsanti Rimuovi e Rimuovi tutto.

Questa opzione viene visualizzata solo se nel dominio sono stati creati cluster o istanze standalone.
