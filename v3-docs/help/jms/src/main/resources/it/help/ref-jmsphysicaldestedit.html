Modifica destinazione fisica JMS

Nella pagina Modifica destinazione fisica JMS è possibile modificare le impostazioni per una destinazione fisica JMS.

La pagina Modifica destinazione fisica JMS contiene le opzioni riportate di seguito.

Nome

Nome della destinazione fisica. Questo campo è di sola lettura. È possibile specificare il nome solo quando si crea una risorsa di destinazione JMS o una destinazione fisica.

Tipo

Tipo di destinazione fisica. Questo campo è di sola lettura. È possibile specificare il tipo solo quando si crea una risorsa di destinazione JMS o una destinazione fisica.

Numero massimo di messaggi non utilizzati

Numero massimo di messaggi non utilizzati consentiti per la destinazione. Un valore pari a -1 indica che non sono impostati limiti per il numero di messaggi non utilizzati. Il valore predefinito è -1.

Dimensione massima messaggio

Dimensione massima, in byte, di un singolo messaggio. Un valore pari a -1 indica una dimensione illimitata del messaggio. Il valore predefinito è -1.

Memoria totale massima

Memoria totale massima, in byte, per i messaggi non utilizzati. Un valore pari a -1 indica una dimensione illimitata di memoria per i messaggi non utilizzati. Il valore predefinito è -1.

Funzionamento limite soglia

Funzionamento del broker quando viene raggiunta la soglia del limite di memoria. Sono disponibili le opzioni riportate di seguito.

Rifiuta i messaggi più recenti

Consente di rifiutare i messaggi più recenti e notificare al client di produzione un'eccezione solo se il messaggio è persistente. Questo è il valore predefinito.

Rallenta producer

Consente di ridurre la velocità alla quale i producer inviano messaggi.

Elimina i meno recenti

Consente di eliminare i messaggi meno recenti.

Elimina i messaggi con la priorità più bassa

Consente di eliminare i messaggi con la priorità più bassa.

Se il valore è Elimina i meno recenti o Elimina i messaggi con la priorità più bassa e Usa coda dei messaggi bloccati è impostato su True, i messaggi in eccesso vengono spostati nella coda dei messaggi bloccati.

Numero massimo di producer

Numero massimo di producer di messaggi che è possibile associare a questa destinazione fisica. Una volta raggiunto il limite, non è possibile creare nuovi producer. Un valore pari a -1 indica un numero illimitato di producer. Il valore predefinito è 100.

Numero massimo messaggi per consegna batch

Numero massimo di messaggi che è possibile consegnare ai consumer in un singolo batch. Un valore pari a -1 indica un numero illimitato di messaggi. Il valore predefinito è 1000.

Usa coda dei messaggi bloccati

Se questa opzione è abilitata, i messaggi bloccati vengono posizionati nella coda di messaggi bloccati. Se questa opzione è disabilitata, i messaggi bloccati vengono eliminati. Questa opzione è abilitata per impostazione predefinita.

Convalida schema XML

Se questa opzione è abilitata, la fase di esecuzione del client Message Queue tenterà di convalidare un messaggio XML in base alle XSD (XML Schema Definition) specificate o in base al DTD se non è specificata nessuna XSD prima di inviare il messaggio al broker. Se non è possibile individuare lo schema specificato o convalidare il messaggio, quest'ultimo non viene inviato e viene restituita un'eccezione. Questa opzione è disabilitata per impostazione predefinita.

Lista URI schema XML

Lista di valori separati da spazi di stringhe URI per XSD (XML Schema Definition). Se è abilitata la convalida dello schema XML, gli URI puntano alla posizione di una o più XSD da utilizzare per tale convalida.
