JMS 연결 팩토리와 관련된 등록 정보

JMS 연결 팩토리에 대해 다음과 같은 추가 등록 정보를 사용할 수 있습니다.

ClientId

영구 가입자가 사용할 연결 팩토리에 대한 클라이언트 아이디를 지정합니다.

AddressList

응용 프로그램이 통신할 메시지 브로거 인스턴스의 이름(및 선택적으로 포트 번호)을 지정합니다. 목록의 각 주소는 연결에 대한 호스트 이름(그리고 선택적으로 호스트 포트 및 연결 서비스)을 지정합니다. 예를 들어 값은 earth 또는 earth:7677일 수 있습니다. 메시지 브로커가 기본값(7676) 이외의 포트에서 실행 중인 경우 포트 번호를 지정합니다. 등록 정보 설정이 클러스터링된 환경에서 여러 호스트 및 포트를 지정하는 경우, AddressListBehavior 등록 정보가 RANDOM으로 설정되지 않은 한 목록에서 사용 가능한 첫 번째 호스트가 사용됩니다.

자세한 내용은 Message Queue Developer’s Guide for Java Clients를 참조하십시오.

기본값은 로컬 호스트 및 기본 포트 번호(7676)입니다. 클라이언트는 로컬 호스트의 포트 7676에서 브로커에 연결을 시도합니다.

UserName

연결 팩토리에 대한 사용자 이름입니다.

기본값은 guest입니다.

Password

연결 팩토리에 대한 비밀번호입니다.

기본값은 guest입니다.

ReconnectEnabled

true로 설정된 경우 연결이 끊기면 클라이언트 런타임에서 메시지 서버(또는 AddressList의 주소 목록)로의 재연결을 시도하도록 지정합니다.

기본값은 true입니다.

ReconnectAttempts

클라이언트 런타임에서 목록에 있는 다음 주소를 시도하기 전에 AddressList의 각 주소에 대해 시도할 연결(또는 재연결) 횟수를 지정합니다. -1 값은 재연결 시도 횟수에 제한이 없음을 나타냅니다. 클라이언트 런타임은 성공할 때까지 첫 번째 주소에 연결을 시도합니다.

기본값은 3입니다.

ReconnectInterval

재연결 시도 간의 간격(밀리초 단위)을 지정합니다. AddressList의 각 주소에 대한 재연결 시도와 목록의 연속된 주소에 이 값이 적용됩니다. 간격이 너무 짧으면 브로커가 복구할 시간이 없습니다. 너무 길 경우에는 지연이 지나치게 길게 느껴질 수 있습니다.

기본값은 30000입니다.

AddressListBehavior

연결 시도를 AddressList 속성의 주소 순서(PRIORITY)로 수행할지 임의 순서(RANDOM)로 수행할지 지정합니다.

RANDOM은 재연결할 때 AddressList에서 임의의 주소를 선택한다는 의미입니다. 여러 클라이언트가 같은 연결 팩토리를 사용하여 연결을 시도할 가능성이 높은 경우, 이 값은 모두가 같은 주소에 연결되는 것을 방지합니다.

PRIORITY는 재연결할 때 항상 AddressList에 있는 첫 번째 서버 주소에 연결하고 첫 번째 브로커를 사용할 수 없는 경우에만 다른 주소를 사용한다는 의미입니다.

기본값은 RANDOM입니다.

AddressListIterations

클라이언트 런타임에서 연결을 설정(또는 재설정)하기 위해 AddressList를 반복하는 횟수를 지정합니다. -1 값은 횟수에 제한이 없음을 나타냅니다.

기본값은 3입니다.
