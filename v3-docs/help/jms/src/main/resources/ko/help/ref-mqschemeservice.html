MQ 체계 및 MQ 서비스 주소 구문

메시지 서비스 주소에 대한 전체 구문은 다음과 같습니다.

scheme://address-syntax

여기서 scheme 및 address-syntax는 아래 표에서 설명합니다.

MQ 방법 및 MQ 서비스는 다음 표의 처음 두 열에 표시된 값입니다.

스키마 이름 접속 서비스 설명
 주소 구문
mq jms 및 ssljms Message Queue 클라이언트 런타임은 지정된 호스트 및 포트에서 MQ 포트 매퍼로 접속을 만듭니다. 포트 매퍼는 동적으로 설정된 접속 서비스 포트의 목록을 반환하고, Message Queue 클라이언트 런타임은 지정된 접속 서비스를 호스트하는 포트에 접속합니다. [hostName][:port][/serviceName]

기본값: hostName = localhost, port = 7676, serviceName = jms

기본값은 jms 접속 서비스에만 적용됩니다. ssljms 접속 서비스의 경우 모든 변수를 지정해야 합니다.

예: mq:MyHost:7677/ssljms
mqtcp jms Message Queue 클라이언트 런타임은 지정된 호스트 및 포트로 TCP 접속을 만들어서(MQ 포트 매퍼 우회) 접속을 설정합니다. hostName:port/jms

예: mqtcp:localhost:7676/jms
mqssl ssljms Message Queue 클라이언트 런타임은 지정된 호스트 및 포트로 보안 SSL 접속을 만들어서(MQ 포트 매퍼 우회) 접속을 설정합니다. hostName:port/ssljms

예: mqssl:localhost:7676/ssljms
http httpjms Message Queue 클라이언트 런타임은 지정된 URL의 Message Queue 터널 서블릿에 대한 HTTP 접속을 만듭니다. 브로커는 Message Queue 관리자 설명서에 설명된 바와 같이, HTTP 터널 서블릿에 액세스하도록 구성되어야 합니다. hostName:port/contextRoot/tunnel

여러 브로커 인스턴스가 같은 터널 서블릿을 사용하는 경우 임의로 선택되지 않은 특정 브로커 인스턴스에 접속하기 위한 구문은 다음과 같습니다. http://hostName:port/contextRoot/tunnel?serverName=hostName:instanceName
https httpsjms Message Queue 클라이언트 런타임은 지정된 Message Queue 터널 서블릿 URL로 보안 HTTPS 접속을 만듭니다. 브로커는 Message Queue 관리자 설명서에 설명된 바와 같이, HTTPS 터널 서블릿에 액세스하도록 구성되어야 합니다. hostName:port/contextRoot/tunnel

여러 브로커 인스턴스가 같은 터널 서블릿을 사용하는 경우 임의로 선택되지 않은 특정 브로커 인스턴스에 접속하기 위한 구문은 다음과 같습니다. http://hostName:port/contextRoot/tunnel?serverName=hostName:instanceName
