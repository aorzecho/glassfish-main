JMS 서비스

[JMS 서비스] 페이지에서는 모든 JMS 접속에서 사용할 설정을 구성할 수 있습니다.

JMS 서비스 구성에 대한 자세한 내용은 Oracle GlassFish Server Application Development Guide의 19장, Using the Java Message Service를 참조하십시오.

[JMS 서비스] 페이지에는 다음 옵션이 포함되어 있습니다.

기본값 로드

기본값이 있는 설정을 해당 기본값으로 복원하는 단추입니다. 기본값이 없는 설정은 변경되지 않습니다.

핑

JMS 서비스가 기본 JMS 호스트에서 실행 중인지 확인하는 단추입니다. JMS 서비스가 실행 중이면 핑이 성공했다는 메시지가 보고됩니다.

구성 이름

이 페이지의 설정이 적용되는 구성의 이름입니다. 이 필드는 읽기 전용입니다.

유형

JMS 서비스에 대한 액세스 유형입니다. 사용 가능한 선택 항목은 다음과 같습니다.

EMBEDDED

로컬 호스트의 JMS 서비스에 액세스합니다. JMS 서비스는 GlassFish Server과(와) 동일한 JVM 시스템에서 시작됩니다. 초기화 지연은 GlassFish Server이(가) 시작될 때가 아닌 JMS 서비스에 처음에 액세스할 때 기본 포함된 브로커를 시작합니다. JMS 서비스의 기본 유형입니다.

LOCAL

로컬 호스트의 JMS 서비스에 액세스합니다. JMS 제공자는 GlassFish Server과(와) 함께 시작됩니다.

REMOTE

다른 시스템의 JMS 서비스에 액세스합니다. REMOTE를 선택하면 다음에 서버가 시작될 때 JMS 서비스가 GlassFish Server에 의해 시작되지 않습니다. 대신 JMS 서비스는 Message Queue에 의해 시작 및 관리되므로 Message Queue 브로커를 별도로 시작해야 합니다. 브로커 시작에 대한 자세한 내용은 Oracle GlassFish Server Message Queue Administration Guide의 Starting Brokers를 참조하십시오. 이 값을 선택하고 원격 호스트를 사용 중인 경우에는 JMS 호스트를 편집하는 방법의 지침에 따라 원격 호스트의 이름을 지정합니다.

시작 시간 초과

GlassFish Server이(가) 시작을 중지하기 전에 JMS 서비스가 시작되기를 기다리는 시간(초)입니다. 느리거나 부하가 많은 시스템에서는 값을 기본값보다 크게 늘리십시오. 기본값은 60입니다.

시작 인수

JMS 서비스 시작을 사용자 정의하는 인수입니다. as-install-parent /mq/bin/imqbrokerd 명령을 통해 사용 가능한 인수를 사용합니다.

재접속 사용

재접속 사용 체크 박스가 선택된 경우 JMS 서비스는 접속이 끊어졌을 때 메시지 서버(또는 AddressList의 주소 목록)에 재접속을 시도합니다. 이 옵션은 기본적으로 사용으로 설정되어 있습니다.

재접속 간격

재접속 시도 간격(초)입니다. AddressList의 각 주소에 대한 시도와 목록의 연속된 주소에 이 간격이 적용됩니다. 이 시간 간격이 너무 짧을 경우 브로커에게 복구할 시간이 없습니다. 너무 길 경우에는 지연이 지나치게 길게 느껴질 수 있습니다. 기본값은 5초입니다.

재접속 시도

클라이언트 런타임에서 목록에 있는 다음 주소를 시도하기 전에 AddressList의 각 주소에 대해 시도할 접속(또는 재접속) 횟수입니다. -1 값은 재접속 시도 횟수에 제한이 없음을 나타냅니다. 클라이언트 런타임은 성공할 때까지 첫번째 주소에 접속을 시도합니다. 기본값은 3입니다.

기본 JMS 호스트

기본 JMS 호스트의 이름입니다. 기본값은 default_JMS_host입니다.

주소 목록 동작

접속 시도 순서입니다. 사용 가능한 선택 항목은 다음과 같습니다.

random

AddressList에서 주소를 임의로 선택합니다. 동일한 접속 팩토리를 사용하여 접속을 시도하는 클라이언트가 많을 경우 random을 지정하여 클라이언트가 모두 같은 주소에 접속되지 못하도록 합니다. 이 옵션은 기본값입니다.

priority

재접속할 때에는 항상 AddressList에 있는 첫번째 서버 주소에 접속하고 첫번째 브로커를 사용할 수 없는 경우에만 다른 주소를 사용합니다.

주소 목록 반복

JMS 서비스에서 접속을 설정(또는 재설정)하기 위해 AddressList를 반복하는 횟수입니다. -1 값은 횟수에 제한이 없음을 나타냅니다. 기본값은 3입니다.

MQ 체계 및 MQ 서비스

기본값이 아닌 체계 또는 서비스를 사용할 경우의 Message Queue 주소 체계 이름 및 Message Queue 접속 서비스 이름입니다. 구문 정보는 MQ 체계 및 MQ 서비스 주소 구문을(를) 참조하십시오.

추가 속성

JMS 서비스에 대한 추가 속성입니다.

사용 가능한 속성에 대한 자세한 내용은 JMS 서비스 관련 속성을(를) 참조하십시오.

관련 작업

  • JMS 호스트를 생성하는 방법

  • JMS 호스트를 편집하는 방법

  • JMS 호스트를 삭제하는 방법
