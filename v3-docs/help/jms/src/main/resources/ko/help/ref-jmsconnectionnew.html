새 JMS 연결 팩토리

새 JMS 연결 팩토리 페이지에서는 새 JMS 연결 팩토리를 만들 수 있습니다.

새 JMS 연결 팩토리 페이지에는 다음 옵션이 포함되어 있습니다.

풀 이름

연결 팩토리를 식별하는 고유한 이름입니다. GlassFish 서버에서는 연결 팩토리의 JNDI 이름을 이 값으로 설정합니다.

자원 유형

연결 팩토리의 유형입니다. 사용 가능한 선택 항목은 javax.jms.ConnectionFactory, javax.jms.QueueConnectionFactory 및 javax.jms.TopicConnectionFactory입니다.

설명

연결 팩토리에 대한 설명입니다.

상태

연결 팩토리의 상태입니다. 연결 팩토리는 사용 가능하거나 사용 가능하지 않을 수 있습니다. 이 옵션은 기본적으로 활성화됩니다.

초기 및 최소 풀 크기

연결 팩토리에 대한 풀의 최소 연결 수입니다. 기본값은 8입니다.

최대 풀 크기

연결 팩토리에 대한 풀의 최대 연결 수입니다. 기본값은 32입니다.

풀 크기 조정 개수

풀 유휴 시간 초과가 만료될 때 제거할 연결 수입니다. 기본값은 2입니다.

유휴 시간 초과

풀에서 연결이 유휴 상태로 있을 수 있는 최대 시간(초)입니다. 기본값은 300입니다.

최대 대기 시간

연결을 요청한 응용 프로그램이 연결 시간 초과까지 대기하는 시간입니다. 기본값은 60000입니다.

실패 시

모든 연결 닫기 확인란이 선택된 경우 GlassFish 서버는 연결이 하나라도 실패하면 풀의 모든 연결을 닫은 후 다시 설정합니다. 이 옵션은 기본적으로 비활성화됩니다.

트랜잭션 지원

선택한 트랜잭션 지원은 이 연결 풀과 연관된 자원 어댑터의 트랜잭션 지원 속성을 역호환 방식으로 대체합니다. 즉, 자원 어댑터에 지정한 수준보다 낮은 트랜잭션 수준을 지원하거나 자원 어댑터에서 지정한 것과 같은 트랜잭션 수준을 지원할 수 있지만 더 높은 수준은 지정할 수 없습니다.

다음과 같은 값을 사용할 수 있습니다.

XATransaction

트랜잭션 범위 내에서 둘 이상의 자원 사용과 관련된 트랜잭션에 자원을 사용할 수 있습니다. 이 값은 JMS 연결 팩토리의 기본값입니다.

예를 들어 트랜잭션에는 이 자원과 JDBC 자원, 커넥터 자원 또는 다른 JMS 연결 팩토리 자원이 포함될 수 있습니다. 이 값은 가장 높은 유연성을 제공합니다. XATransaction으로 구성된 자원은 2단계 완결 작업에 참여합니다.

LocalTransaction

자원은 트랜잭션 범위 내에서 하나의 자원만 관여하는 트랜잭션에 사용되거나 둘 이상의 XA 자원이 관여하는 분산된 트랜잭션에서 마지막 에이전트로 사용될 수 있습니다.

이 값은 훨씬 높은 성능을 제공합니다. LocalTransaction으로 구성된 자원은 2단계 완결 작업에 사용되지 않습니다.

NoTransaction

자원은 트랜잭션에 참여할 수 없습니다. 이 설정은 JMS 응용 프로그램에서 제한적으로 사용됩니다.

연결 검증

필요 확인란이 선택된 경우 연결을 응용 프로그램에 전달하기 전에 먼저 검증합니다. 자원의 검증이 실패하면 자원이 삭제되고, 새 자원이 만들어진 후 반환됩니다. 이 옵션은 기본적으로 비활성화됩니다.

추가 등록 정보

연결 팩토리에 대한 추가 등록 정보입니다.

사용할 수 있는 등록 정보에 대한 설명은 JMS 연결 팩토리와 관련된 등록 정보를 참조하십시오.

대상

연결 팩토리에 대한 클러스터 및 독립 실행형 인스턴스입니다. 추가 및 모두 추가 버튼을 사용하여 원하는 대상을 선택한 대상 열로 이동합니다. 제거 및 모두 제거 버튼을 사용하여 필요하지 않은 대상을 사용 가능한 대상 열로 이동합니다.

이 옵션은 클러스터나 독립 실행형 인스턴스가 도메인에 만들어진 경우에만 표시됩니다.
