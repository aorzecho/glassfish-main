새 JMS 물리적 대상

새 JMS 물리적 대상 페이지에서는 새 JMS 물리적 대상을 만들 수 있습니다.

JMS 대상 자원을 만들 때 물리적 대상 이름을 지정하고, GlassFish 서버에서는 필요할 때 자동으로 물리적 대상을 만들기 때문에 대개는 물리적 대상을 만들 필요가 없습니다.

새 JMS 물리적 대상 페이지에는 다음 옵션이 포함되어 있습니다.

이름

물리적 대상의 이름입니다.

유형

물리적 대상의 유형입니다. 사용 가능한 선택 항목은 javax.jms.Topic 및 javax.jms.Queue입니다.

사용되지 않은 최대 메시지 수

대상에 허용된 사용되지 않은 최대 메시지 수입니다. 값이 –1이면 사용되지 않은 메시지 수에 제한이 없음을 나타냅니다. 기본값은 –1입니다.

최대 메시지 크기

단일 메시지의 최대 크기(바이트)입니다. 값이 –1이면 메시지 크기에 제한이 없음을 나타냅니다. 기본값은 –1입니다.

최대 전체 메모리

사용되지 않은 메시지의 최대 전체 메모리(바이트)입니다. 값이 –1이면 사용되지 않은 메시지의 메모리 크기에 제한이 없음을 나타냅니다. 기본값은 –1입니다.

임계값 제한 동작

메모리 제한 임계값에 도달했을 때 브로커가 수행할 동작입니다. 사용 가능한 선택 항목은 다음과 같습니다.

Reject newest messages

가장 최신 메시지를 거부하고 메시지가 지속적일 경우에만 생성 클라이언트에 예외를 알립니다. 이 값은 기본값입니다.

Slow down producers

메시지 생성자가 메시지를 보내는 속도를 늦춥니다.

Throw out oldest

가장 오래된 메시지를 삭제합니다.

Throw out lowest-priority messages

우선 순위가 가장 낮은 메시지를 삭제합니다.

값이 Throw out oldest 또는 Throw out lowest-priority messages이고 사용 불능 메시지 대기열 사용을 True로 설정하면 범위 밖 메시지가 사용 불능 메시지 대기열로 이동됩니다.

최대 생성자 수

이 물리적 대상과 연결할 수 있는 최대 메시지 생성자 수입니다. 이 제한에 도달하면 새 생성자를 만들 수 없습니다. 값이 –1이면 생성자 수에 제한이 없음을 나타냅니다. 기본값은 100입니다.

배치 전달 최대 메시지 수

단일 배치로 사용자에게 전달할 수 있는 최대 메시지 수입니다. 값이 -1이면 메시지 수에 제한이 없음을 나타냅니다. 기본값은 1000입니다.

사용 불능 메시지 대기열 사용

이 값을 True로 설정하면 사용 불능 메시지가 사용 불능 메시지 대기열에 배치됩니다. 이 값을 False로 설정하면 사용 불능 메시지는 삭제됩니다. 이 옵션은 기본적으로 True로 설정됩니다.

XML 스키마 검증

이 값을 True로 설정하면 Message Queue 클라이언트 런타임은 XML 메시지를 브로커로 보내기 전에 지정된 XML 스키마 정의에 대해(또는 XSD가 지정되지 않은 경우 DTD에 대해) XML 메시지의 검증을 시도합니다. 지정된 스키마를 찾을 수 없거나 메시지를 검증할 수 없으면 메시지가 전송되지 않고 예외가 발생합니다. 이 옵션은 기본적으로 False로 설정됩니다.

XML 스키마 URI 목록

XSD(XML 스키마 정의) URI 문자열의 공백으로 구분된 목록입니다. XML 스키마 검증이 활성화된 경우 URI는 XML 스키마 검증에 사용할 하나 이상의 XSD 위치를 가리킵니다.
