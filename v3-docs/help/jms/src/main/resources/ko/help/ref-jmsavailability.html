JMS 가용성

JMS 가용성 페이지에서는 JMS(Java Message Service)에 대한 가용성 설정을 구성할 수 있습니다.

JMS 가용성 페이지에는 다음 옵션이 포함되어 있습니다.

기본값 로드

기본값이 있는 설정을 해당 기본값으로 복원하는 단추입니다. 기본값이 없는 설정은 변경되지 않습니다.

구성 이름

이 페이지의 설정이 적용되는 구성의 이름입니다. 이 필드는 읽기 전용입니다.

가용성 서비스

GlassFish 클러스터에 JMS 서비스를 제공하는 데 사용되는 Message Queue 클러스터의 한 유형입니다. 사용 체크 박스를 선택한 경우 고급 Message Queue 클러스터가 사용됩니다. 그렇지 않은 경우 기존 Message Queue 클러스터가 사용됩니다.

사용 체크 박스를 선택한 경우 구성 저장소 유형 및 메시지 저장소 유형 옵션이 무시됩니다.

구성 저장소 유형

기존 Message Queue 클러스터에 있는 구성 데이터에 대한 데이터 저장소 유형입니다. masterbroker 값은 구성 데이터를 저장 및 관리할 마스터 브로커의 용도를 지정합니다. shareddb 값은 구성 데이터를 저장할 공유 데이터베이스의 용도를 지정합니다.

가용성 서비스가 사용으로 설정된 경우에는 이 옵션이 무시됩니다.

메시지 저장소 유형

기존 클러스터의 브로커에 있는 메시지 데이터에 대한 데이터 저장소 유형입니다. file 값은 파일 저장소를 지정합니다. jdbc 값은 JDBC 저장소를 지정합니다.

가용성 서비스가 사용으로 설정된 경우에는 이 옵션이 무시됩니다.

데이터베이스 공급업체
데이터베이스 사용자 이름
데이터베이스 비밀번호
데이터베이스 액세스 URL

다음 경우에 사용할 JDBC 데이터베이스의 데이터베이스 공급업체, 사용자, 비밀번호 및 액세스 URL입니다.

  • 가용성 서비스가 사용으로 설정된 경우:

  • 구성 저장소 유형이 shareddb로 설정된 경우

  • 구성 저장소 유형이 jdbc로 설정된 경우

지원되는 공급업체 및 각 공급업체의 액세스 URL 형식에 대한 자세한 내용은 Oracle GlassFish Server Message Queue Administration Guide의 JDBC-Based Persistence를 참조하십시오.

추가 속성

JMS 가용성에 대한 추가 속성입니다.

GlassFish Server에서는 추가 JMS 가용성 속성을 미리 정의하지 않습니다.
