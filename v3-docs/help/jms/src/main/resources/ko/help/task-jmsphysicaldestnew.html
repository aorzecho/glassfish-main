JMS 물리적 대상을 만드는 방법

JMS 대상 자원을 만들 때 물리적 대상 이름을 지정하고, GlassFish Server에서는 필요할 때 자동으로 물리적 대상을 만들기 때문에 대개는 물리적 대상을 만들 필요가 없습니다.

  1. 탐색 트리에서 구성 노드를 선택합니다.
  2. 구성 노드에서 Java Message Service 노드를 확장합니다.
  3. Java Message Service 노드에서 물리적 대상 노드를 선택합니다.

    물리적 대상 페이지가 열립니다.

  4. 물리적 대상 페이지에서 새로 만들기를 누릅니다.

    새 물리적 대상 페이지가 열립니다.

  5. 이름 필드에 물리적 대상의 이름을 입력합니다.
  6. 유형 드롭다운 목록에서 javax.jms.Topic 또는 javax.jms.Queue를 선택합니다.
  7. 사용되지 않은 최대 메시지 수 필드에 대상에 허용된 사용되지 않은 최대 메시지 수를 입력합니다.

    값이 –1이면 사용되지 않은 메시지 수에 제한이 없음을 나타냅니다. 기본값은 –1입니다.

  8. 최대 메시지 크기 필드에 단일 메시지의 최대 크기(바이트)를 입력합니다.

    값이 –1이면 메시지 크기에 제한이 없음을 나타냅니다. 기본값은 –1입니다.

  9. 최대 전체 메모리 필드에 사용되지 않은 메시지의 최대 전체 메모리(바이트)를 입력합니다.

    값이 –1이면 사용되지 않은 메시지의 메모리 크기에 제한이 없음을 나타냅니다. 기본값은 –1입니다.

  10. 임계값 제한 동작 드롭다운 목록에서 메모리 제한 임계값에 도달했을 때 브로커가 수행할 동작을 선택합니다.
    • Reject newest messages는 브로커가 가장 최신 메시지를 거부하고 메시지가 지속적일 경우에만 생성 클라이언트에 예외를 알리도록 합니다.

      Reject newest messages는 기본값입니다.

    • Slow down producers는 브로커가 메시지 생성자가 메시지를 보내는 속도를 늦추도록 합니다.
    • Throw out oldest는 브로커가 가장 오래된 메시지를 삭제하도록 합니다.
    • Throw out lowest-priority messages는 브로커가 우선 순위가 가장 낮은 메시지를 삭제하도록 합니다.

    값이 Throw out oldest 또는 Throw out lowest-priority messages이고 사용 불능 메시지 대기열 사용을 True로 설정하면 범위 밖 메시지가 사용 불능 메시지 대기열로 이동됩니다.

  11. 최대 생성자 수 필드에 이 물리적 대상과 연결할 수 있는 최대 메시지 생성자 수를 입력합니다.

    이 제한에 도달하면 새 생성자를 만들 수 없습니다. 값이 –1이면 생성자 수에 제한이 없음을 나타냅니다. 기본값은 100입니다.

  12. 배치 전달 최대 메시지 수 필드에 단일 배치로 사용자에게 전달할 수 있는 최대 메시지 수를 입력합니다.

    값이 -1이면 메시지 수에 제한이 없음을 나타냅니다. 기본값은 1000입니다.

  13. 사용 불능 메시지 대기열 사용 드롭다운 목록에서 True를 선택하여 사용 불능 메시지를 사용 불능 메시지 대기열에 배치합니다.

    이 값을 False로 설정하면 사용 불능 메시지는 삭제됩니다. 이 옵션은 기본적으로 True로 설정됩니다.

  14. XML 스키마 검증 사용 드롭다운 목록에서 True를 선택하여 대상에 대한 XML 스키마 검증을 활성화합니다.

    이 값을 True로 설정하면 Message Queue 클라이언트 런타임은 XML 메시지를 브로커로 보내기 전에 지정된 XML 스키마 정의에 대해(또는 XSD가 지정되지 않은 경우 DTD에 대해) XML 메시지의 검증을 시도합니다. 지정된 스키마를 찾을 수 없거나 메시지를 검증할 수 없으면 메시지가 전송되지 않고 예외가 발생합니다. 이 옵션은 기본적으로 False로 설정됩니다.

  15. XML 스키마 URI 목록 필드에 XSD(XML 스키마 정의) URI 문자열의 공백으로 구분된 목록을 입력합니다.

    XML 스키마 검증이 활성화된 경우 URI는 XML 스키마 검증에 사용할 하나 이상의 XSD 위치를 가리킵니다.
