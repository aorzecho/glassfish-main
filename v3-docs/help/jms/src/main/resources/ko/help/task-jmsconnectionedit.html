JMS 연결 팩토리 자원을 편집하는 방법

  1. 탐색 트리에서 자원 노드를 확장합니다.
  2. 자원 노드에서 JMS 자원 노드를 확장합니다.
  3. JMS 자원 노드에서 JMS 연결 팩토리 노드를 선택합니다.

    JMS 연결 팩토리 페이지가 열립니다.

  4. JMS 연결 팩토리 페이지에서 편집할 연결 팩토리의 이름을 누릅니다.

    JMS 연결 팩토리 편집 페이지가 열립니다.

  5. 설명 필드에 연결 팩토리에 대한 설명을 입력합니다.
  6. 런타임에 자원을 활성화하려면 상태 사용 가능 확인란을 선택합니다.

    이 옵션은 기본적으로 활성화됩니다.

  7. 초기 및 최소 풀 크기 필드에 풀의 최소 연결 수를 지정합니다.

    기본값은 8입니다.

  8. 최대 풀 크기 필드에 풀의 최대 연결 수를 지정합니다.

    기본값은 32입니다.

  9. 풀 크기 조정 개수 필드에 풀 유휴 시간 초과가 만료될 때 제거할 연결 수를 지정합니다.

    기본값은 2입니다.

  10. 유휴 시간 초과 필드에 풀에서 연결이 유휴 상태로 유지될 수 있는 최대 시간(초)을 지정합니다.

    기본값은 300입니다.

  11. 최대 대기 시간 필드에 연결을 요청하는 응용 프로그램이 연결 시간 초과까지 대기하는 시간을 지정합니다.

    기본값은 60000입니다.

  12. 실패 시 모든 연결 닫기 확인란을 선택하여 단일 연결이 실패할 경우 GlassFish 서버에서 풀의 모든 연결을 닫은 후 다시 설정하도록 합니다.

    이 확인란을 선택하지 않으면 개별 연결을 사용할 경우에만 연결이 다시 설정됩니다.

  13. JMS 연결 팩토리 자원에 대해 트랜잭션 지원 값을 다음과 같이 지정합니다.
    • 트랜잭션 범위 내에서 둘 이상의 자원 사용과 관련되는 트랜잭션에 사용할 수 있는 자원에 대해 XATransaction(기본값)을 지정합니다.

      예를 들어 이 자원과 JDBC 자원, 커넥터 자원 또는 다른 JMS 연결 팩토리 자원입니다. 이 값은 가장 높은 유연성을 제공합니다. XATransaction으로 구성된 자원은 2단계 완결 작업에 참여합니다.

    • 트랜잭션 범위 내에서 하나의 자원만 관여하는 트랜잭션에 사용되거나 둘 이상의 XA 자원이 관여하는 분산된 트랜잭션에서 마지막 에이전트로 사용될 수 있는 자원에 대해 LocalTransaction을 지정합니다.

      이 값은 훨씬 높은 성능을 제공합니다. LocalTransaction으로 구성된 자원은 2단계 완결 작업에 사용되지 않습니다.

    • 트랜잭션에 절대로 참여하지 않을 자원에 대해 NoTransaction을 지정합니다.

      이 설정은 JMS 응용 프로그램에서 제한적으로 사용됩니다.

  14. 연결 검증 확인란을 선택하여 연결 검증을 사용하도록 설정합니다.

    이 옵션은 기본적으로 비활성화됩니다.

  15. (옵션) 추가 등록 정보 표에서 연결 팩토리에 대한 추가 등록 정보를 지정합니다.

    사용할 수 있는 등록 정보에 대한 설명은 JMS 연결 팩토리와 관련된 등록 정보를 참조하십시오.

    • 등록 정보를 추가하려면 등록 정보 추가 버튼을 누릅니다. 빈 행이 표시되면 이름 필드에 등록 정보 이름을 입력하고 값 필드에 등록 정보 값을 입력합니다.
    • 등록 정보를 수정하려면 등록 정보의 값 필드를 편집합니다.
    • 등록 정보를 삭제하려면 삭제할 등록 정보의 이름 필드 왼쪽에 있는 확인란을 선택한 다음 등록 정보 삭제 버튼을 누릅니다.
  16. 저장을 누릅니다.

    참고 - 기본 설정으로 돌아가려면 기본값 로드 버튼을 누른 다음 저장을 누릅니다.
