JMS 대상 리소스를 생성하는 방법

JMS 대상 리소스를 생성할 때 GlassFish Server은(는) 대상 리소스에 대한 관리 객체 리소스를 생성합니다. [커넥터] 노드 아래에서 이 리소스를 볼 수 있습니다.

  1. 탐색 트리에서 [리소스] 노드를 확장합니다.

  2. [리소스] 노드에서 [JMS 리소스] 노드를 확장합니다.

  3. [JMS 리소스] 노드에서 [JMS 대상 리소스] 노드를 선택합니다.

    [JMS 대상 리소스] 페이지가 열립니다.

  4. [JMS 대상 리소스] 페이지에서 [새로 만들기]를 누릅니다.

    [새 JMS 대상 리소스] 페이지가 열립니다.

  5. [JNDI 이름] 필드에 리소스의 JNDI 이름을 입력합니다.

    JMS 리소스의 경우 이름 지정 하위 컨텍스트 접두어 jms/를 사용하는 것이 좋습니다. 예를 들면 jms/Queue와 같이 사용합니다.

  6. [물리적 대상 이름] 필드에 이 리소스와 연결할 물리적 대상의 이름을 입력합니다.

    GlassFish Server은(는) 물리적 대상이 필요할 때 자동으로 물리적 대상을 생성하고 사용자가 대상 리소스를 삭제할 때 물리적 대상을 삭제합니다.

  7. 리소스 유형 드롭다운 목록에서 javax.jms.Topic 또는 javax.jms.Queue를 선택합니다.

  8. [설명] 필드에 대상 리소스에 대한 설명을 입력합니다.

  9. 런타임에 리소스를 사용으로 설정하려면 [상태 사용] 체크 박스를 선택합니다.

    이 옵션은 기본적으로 사용으로 설정되어 있습니다.

  10. [추가 속성] 영역에서 대상 리소스에 대한 추가 속성을 지정합니다.

    GlassFish Server에서는 JMS 대상 리소스에 대한 추가 속성을 정의하지 않습니다.

    속성을 추가하려면 [속성 추가] 단추를 누릅니다. 빈 행이 표시되면 [이름] 필드에 속성 이름을 입력하고 [값] 필드에 속성값을 입력합니다.

  11. In the Targets area, do the following:

    1. From the Available column, select the target or targets to which applications that use the resource will be deployed.

      The available targets include the available clusters and server instances, as well as the default server instance, server.

    2. Click Add to move the target to the Selected column.

  12. [확인]을 누릅니다.
