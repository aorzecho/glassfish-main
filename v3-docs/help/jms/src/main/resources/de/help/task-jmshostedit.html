So wird ein JMS-Host bearbeitet

  1. Blenden Sie in der Navigationsbaumstruktur den Knoten "Konfiguration" ein.

  2. Select the instance to configure:

    • To configure a particular instance, select the instance's config node. For example, for the default instance, server, select the server-config node.

    • To configure the default settings for future instances that use a copy of default-config, select the default-config node.

  3. Blenden Sie unter dem Konfigurationsknoten den Knoten "Java Message Service" ein.

  4. Wählen Sie unter dem Knoten "Java Message Service" den Knoten "JMS-Hosts".

    Die Seite "JMS-Hosts" wird geöffnet.

  5. Wählen Sie auf der Seite "JMS-Hosts" den zu bearbeitenden Host aus.

    Die Seite "JMS-Host bearbeiten" wird geöffnet.

  6. Geben Sie im Feld "Host" den Namen oder die IP-Adresse (Internet Protocol) des Systems ein, auf dem der JMS-Host ausgeführt wird (localhost oder der Name des lokalen oder Remotesystems).

  7. Geben Sie im Feld "Port" die Portnummer des JMS-Service ein.

    Ändern Sie dieses Feld nur, wenn der JMS-Service nicht über den Standardport ausgeführt werden soll. Der Standardport lautet 7676.

  8. Geben Sie in den Feldern "Admin-Benutzername" und "Admin-Kennwort" den Benutzernamen und das Kennwort für den Message Queue-Broker ein.

    Diese Eingaben sind nicht mit dem GlassFish Server-Benutzernamen und -Kennwort identisch. Bearbeiten Sie diese Felder nur, wenn die Werte des Message Queue Brokers mit dem folgenden Befehl geändert wurden. as-install-parent/mq/bin/imqusermgr-Befehl. Die Standardwerte sind admin und admin.

  9. Klicken Sie auf "Speichern".

    Tipp:

    Um zu den Standardeinstellungen zurückzukehren, klicken Sie auf die Schaltfläche "Standardwerte laden" und anschließend auf "Speichern".
