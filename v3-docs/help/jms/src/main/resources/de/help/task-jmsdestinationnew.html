So wird eine JMS-Zielressource erstellt

Wenn Sie eine JMS-Zielressource erstellen, erstellt GlassFish Server eine Admin-Objektressource für die Zielressource. Diese Ressource wird unter dem Knoten "Connector" angezeigt.

  1. Blenden Sie in der Navigationsbaumstruktur den Knoten "Ressourcen" ein.

  2. Blenden Sie unter dem Ressourcenknoten den Knoten "JMS-Ressourcen" ein.

  3. Wählen Sie unter dem Knoten "JMS-Ressourcen" den Knoten "JMS-Zielressourcen" aus.

    Die Seite "JMS-Zielressourcen" wird geöffnet.

  4. Klicken Sie auf der Seite "JMS-Zielressourcen" auf "Neu".

    Die Seite "Neue JMS-Zielressource" wird geöffnet.

  5. Geben Sie im Feld "JNDI-Name" den JNDI-Namen der Ressource ein.

    Es wird empfohlen, für JMS-Ressourcen das Benennungssubkontext-Präfix jms/ zu verwenden. Beispiel: jms/Queue.

  6. Geben Sie im Feld "Name des physikalischen Ziels" den Namen des mit dieser Ressource zu verknüpfenden physischen Ziels ein.

    Das physische Ziel wird von GlassFish Server bei Bedarf automatisch erstellt und wieder gelöscht, wenn Sie die Zielressource löschen.

  7. Wählen Sie in der Dropdown-Liste "Ressourcentyp" entweder javax.jms.Topic oder javax.jms.Queue aus.

  8. Geben Sie im Feld "Beschreibung" eine Beschreibung der Zielressource ein.

  9. Aktivieren Sie das Kontrollkästchen "Status", um die Ressource zur Laufzeit zu aktivieren.

    Diese Option ist standardmäßig aktiviert.

  10. Geben Sie im Abschnitt "Weitere Eigenschaften" zusätzliche Eigenschaften für die Verbindungsressource an.

    In GlassFish Server werden keine weiteren Eigenschaften für JMS-Zielressourcen definiert.

    Um eine Eigenschaft hinzuzufügen, klicken Sie auf die Schaltfläche "Eigenschaft hinzufügen". Geben Sie in die daraufhin angezeigte leere Zeile den Eigenschaftsnamen in das Feld "Name" und den Eigenschaftswert in das Feld "Wert" ein.

  11. In the Targets area, do the following:

    1. From the Available column, select the target or targets to which applications that use the resource will be deployed.

      The available targets include the available clusters and server instances, as well as the default server instance, server.

    2. Click Add to move the target to the Selected column.

  12. Klicken Sie auf "OK".
