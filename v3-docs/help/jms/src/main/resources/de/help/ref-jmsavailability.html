JMS-Verfügbarkeit

Auf der Seite "JMS-Verfügbarkeit" können Sie Verfügbarkeitseinstellungen für den Java Message Service (JMS) konfigurieren.

Die Seite "JMS-Verfügbarkeit" enthält die folgenden Optionen.

Standardeinstellungen laden

Schaltfläche zum Wiederherstellen der Standardwerte für Einstellungen, für die es Standardwerte gibt. Einstellungen, die nicht über Standardwerte verfügen, können nicht geändert werden.

Name der Konfiguration

Der Name der Konfiguration, für die die Einstellungen auf dieser Seite gelten. Dieses Feld ist schreibgeschützt.

Verfügbarkeitsdienst

Der Typ von Nachrichtenwarteschlangen-Cluster, der zur Bereitstellung von JMS-Diensten für einen GlassFish-Cluster verwendet wird. Ist das Kontrollkästchen der Option "Aktiviert" aktiviert, wird ein erweiterter Message Queue-Cluster verwendet. Anderenfalls wird ein konventioneller Message Queue-Cluster verwendet.

Ist das Kontrollkästchen der Option "Aktiviert" aktiviert, werden die Optionen "Konfigurationsspeichertyp" und "Nachrichtenspeichertyp" ignoriert.

Konfigurationsspeichertyp

Der Datenspeichertyp für Konfigurationsdaten in einem konventionellen Message Queue-Cluster. Der Wert masterbroker gibt die Verwendung eines Master-Brokers für die Speicherung und Verwaltung der Konfigurationsdaten an. Der Wert shareddb gibt die Verwendung einer gemeinsamen Datenbank für die Speicherung der Konfigurationsdaten an.

Diese Option wird ignoriert, wenn der Verfügbarkeitsdienst aktiviert ist.

Nachrichtenspeichertyp

Der Datenspeichertyp für Nachrichtendaten in Brokern eines konventionellen Clusters. Der Wert file gibt einen Datenspeicher an. Der Wert jdbc gibt einen JDBC-Speicher an.

Diese Option wird ignoriert, wenn der Verfügbarkeitsdienst aktiviert ist.

Datenbankzugriffs-URL

Hersteller, Benutzer, Passwort und Zugriffs-URL der JDBC-Datenbank für die Verwendung in folgenden Fällen:

  • Wenn der Verfügbarkeitsdienst aktiviert ist

  • Wenn der Konfigurationsspeichertyp auf shareddb eingestellt ist

  • Wenn der Nachrichtenspeichertyp auf jdbc eingestellt ist

Weitere Informationen zu unterstützten Herstellern und den Zugriffs-URL-Formaten der jeweiligen Hersteller finden Sie unter JDBC-Based Persistence in Oracle GlassFish Server Message Queue 4.5 Administration Guide.
