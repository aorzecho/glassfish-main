JMS-Verfügbarkeit

Auf der Seite "JMS-Verfügbarkeit" können Sie Verfügbarkeitseinstellungen für den Java Message Service (JMS) konfigurieren.

Die Seite "JMS-Verfügbarkeit" enthält die folgenden Optionen.

Standardeinstellungen laden

Schaltfläche zum Wiederherstellen der Standardwerte für Einstellungen, für die es Standardwerte gibt. Einstellungen, die nicht über Standardwerte verfügen, können nicht geändert werden.

Name der Konfiguration

Der Name der Konfiguration, für die die Einstellungen auf dieser Seite angewendet werden. Dieses Feld ist schreibgeschützt.

Verfügbarkeitsservice

Der Typ des Nachrichtenqueueclusters zur Bereitstellung von JMS-Services für ein GlassFish-Cluster. Wenn das Kontrollkästchen "Aktiviert" gewählt ist, wird ein erweitertes Message Queue-Cluster verwendet. Sonst wird ein konventionelles Message Queue-Cluster verwendet.

Wenn das Kontrollkästchen "Aktiviert" gewählt ist, werden die Optionen "Konfigurationsspeichertyp" und "Nachrichtenspeichertyp" ignoriert.

Konfigurationsspeichertyp

Der Datenspeichertyp für Konfigurationsdaten in einem konventionellen Message Queue-Cluster. Der Wert masterbroker gibt die Verwendung eines Master-Brokers für die Speicherung und Verwaltung der Konfigurationsdaten an. Der Wert shareddb gibt die Verwendung einer gemeinsamen Datenbank für die Speicherung der Konfigurationsdaten an.

Diese Option wird ignoriert, wenn "Verfügbarkeitsservice" aktiviert ist.

Nachrichtenspeichertyp

Der Datenspeichertyp für Nachrichtendaten in Brokern eines konventionellen Clusters. Der Wert file gibt einen Datenspeicher an. Der Wert jdbc gibt einen JDBC-Speicher an.

Diese Option wird ignoriert, wenn "Verfügbarkeitsservice" aktiviert ist.

Datenbankzugriffs-URL

Hersteller, Benutzer und Zugriffs-URL der JDBC-Datenbank für die Verwendung in folgenden Fällen:

  • Wenn "Verfügbarkeitsservice" aktiviert ist

  • Wenn "Konfigurationsspeichertyp" auf shareddb festgelegt ist

  • Wenn "Nachrichtenspeichertyp" auf jdbc festgelegt ist

Weitere Informationen über unterstützte Hersteller und die Formate der Zugriffs-URLs für jeden Hersteller finden Sie in JDBC-Based Persistence im Oracle GlassFish Server Message Queue Administration Guide.
