JMS-Connection Factory bearbeiten

Auf der Seite "JMS-Connection Factory bearbeiten" können Sie die Einstellungen für eine JMS-Connection Factory ändern.

Die Seite "JMS-Connection Factory bearbeiten" enthält die folgenden Optionen.

Standardeinstellungen laden

Schaltfläche zum Wiederherstellen der Standardwerte für Einstellungen, für die es Standardwerte gibt. Einstellungen, die nicht über Standardwerte verfügen, können nicht geändert werden.

Pool Name (Poolname)

Der eindeutige Name des Connector Connection Pools, der für die Connection Factory erstellt wird. Dieser Name ist auch der JNDI-Name der Connection Factory. Der Poolname ist ein schreibgeschütztes Feld.

Resource Type (Ressourcen-Typ)

Der Ressourcentyp der Connection Factory. Der Ressourcentyp ist ein schreibgeschütztes Feld. Sie können den Ressourcentyp nur festlegen, wenn Sie eine neue JMS-Connection Factory erstellen.

Beschreibung

Eine Beschreibung der Connection Factory.

Status

Der Status der Connection Factory. Die Connection Factory kann aktiviert oder deaktiviert sein. Diese Option ist standardmäßig aktiviert.

Initial and Minimum Pool Size (Ursprüngliche und minimale Poolgröße)

Die Mindestanzahl Verbindungen im Pool für die Connection Factory. Der Standardwert ist 8.

Maximum Pool Size (Maximale Poolgröße)

Die maximale Anzahl an Verbindungen im Pool. Der Standardwert lautet 32.

Pool Resize Quantity (Anzahl der Größenänderungen des Pools)

Die Anzahl an Verbindungen, die nach Ablauf der Pool-Leerlaufzeit entfernt werden sollen. Der Standardwert ist 2.

Leerlauftimeout

Die maximale Zeit in Sekunden, die sich eine Verbindung im Pool im Leerlauf befinden kann. Der Standardwert beträgt 300.

Max. Wartezeit

Die Zeitspanne, die eine Anwendung nach dem Anfordern einer Verbindung abwartet, bevor ein Verbindungstimeout ausgegeben wird. Der Standardwert lautet 60000.

On Any Failure (Bei sämtlichen Fehlern)

Wenn das Kontrollkästchen "Sämtliche Verbindungen trennen" aktiviert ist, schließt GlassFish Server bei einem einzigen Verbindungsabbruch alle Verbindungen im Pool und stellt sie wieder her. Diese Option ist standardmäßig deaktiviert.

Transaktionsunterstützung

Folgende Werte sind möglich:

XATransaction

Die Ressource kann für Transaktionen eingesetzt werden, für die mehrere Ressourcen innerhalb eines Transaktionsbereichs verwendet werden. Dies ist der Standardwert für eine JMS-Connection Factory.

Beispiel: Transaktionen können diese Ressource sowie eine JDBC-Ressource, eine Verbindungsressource oder eine andere JMS-Connection Factory-Ressource verwenden. Dieser Wert bietet die meiste Flexibilität. Eine Ressource, die als XATransaction konfiguriert ist, wird für Two-Phase Commit-Vorgänge verwendet.

LocalTransaction

Die Ressource kann entweder für Transaktionen mit nur einer Ressource innerhalb des Transaktionsbereichs oder als letzter Agent in einer verteilten Transaktion mit mehreren XA-Ressourcen verwendet werden.

Dieser Wert bietet eine geringfügig bessere Performance. Eine als LocalTransaction konfigurierte Ressource wird nicht in Two-Phase Commit-Vorgängen verwendet.

NoTransaction

Die Ressource kann nie in Transaktionen verwendet werden. Diese Einstellung wird für JMS-Anwendungen nur eingeschränkt verwendet.

Connection Validation (Verbindungsüberprüfung)

Wenn das Kontrollkästchen "Erforderlich" aktiviert ist, werden Verbindungen vor der Übergabe an die Anwendung validiert. Schlägt die Überprüfung einer Ressource fehl, so wird die Ressource zerstört, und eine neue Ressource wird erstellt und zurückgegeben. Diese Option ist standardmäßig deaktiviert.
