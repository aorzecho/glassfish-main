Physisches JMS-Ziel bearbeiten

Auf der Seite "Physische JMS-Ziele bearbeiten" können Sie die Einstellungen für ein physisches JMS-Ziel ändern.

Die Seite "Physische JMS-Ziele bearbeiten" enthält die folgenden Optionen.

Name

Der Name des physischen Ziels. Dies ist ein schreibgeschütztes Feld. Sie können den Namen nur festlegen, wenn Sie eine JMS-Zielressource oder ein physisches Ziel erstellen.

Typ

Der Typ des physischen Ziels. Dies ist ein schreibgeschütztes Feld. Sie können den Typ nur festlegen, wenn Sie eine JMS-Zielressource oder ein physisches Ziel erstellen.

Höchstzahl nicht konsumierter Meldungen

Die für das Ziel maximal zulässige Anzahl nicht konsumierter Meldungen. Der Wert –1 bedeutet, dass die Anzahl der unkonsumierten Meldungen nicht begrenzt ist. Der Standardwert ist -1.

Maximalgröße von Meldungen

Die maximal zulässige Größe einer einzelnen Meldung in Byte. Der Wert –1 bedeutet, dass die Meldungsgröße unbegrenzt ist. Der Standardwert ist -1.

Maximaler Gesamtspeicher

Der maximale Gesamtspeicher für unkonsumierte Meldungen in Byte. Der Wert –1 bedeutet, dass die Speichergröße für unkonsumierte Meldungen nicht begrenzt ist. Der Standardwert ist -1.

Schwellenwertverhalten

Das Verhalten, das der Broker bei Erreichen des Speicher-Schwellenwerts anwenden soll. Zur Auswahl stehen:

Neueste Nachrichten ablehnen

Die neuesten Meldungen werden zurückgewiesen, und der produzierende Client wird nur dann mit einer Ausnahme benachrichtigt, wenn die Meldung persistent ist. Dies ist der Standardwert.

Produzenten verlangsamen

Die Geschwindigkeit, mit der Meldungsproduzenten Meldungen senden, wird verringert.

Älteste verwerfen

Die ältesten Meldungen werden verworfen.

Nachrichten mit niedrigster Priorität verwerfen

Die Meldungen mit der niedrigsten Priorität werden verworfen.

Wenn der Wert "Throw out oldest" oder "Throw out lowest-priority messages" gewählt und "Queue für nicht zugestellte Meldungen verwenden" auf "true" gesetzt ist, werden überschüssige Meldungen in die Queue für nicht zugestellte Meldungen verschoben.

Höchstzahl von Produzenten

Die maximale Anzahl an Meldungsproduzenten, die mit diesem physischen Ziel verknüpft werden können. Bei Erreichen dieses Grenzwerts können keine neuen Produzenten mehr erstellt werden. Der Wert –1 bedeutet, dass die Anzahl der Produzenten unbegrenzt ist. Der Standardwert beträgt 100.

Höchstzahl der Meldungen bei Batch-Zustellung

Die maximale Anzahl an Meldungen, die Konsumenten in einem einzigen Batch zugestellt werden können. Der Wert –1 bedeutet, dass die Anzahl der Meldungen unbegrenzt ist. Der Standardwert ist 1000.

Queue für nicht zugestellte Meldungen verwenden

Bei aktivierter Option werden nicht zugestellte Meldungen in die Queue für nicht zugestellte Meldungen gestellt. Bei deaktivierter Option werden nicht zugestellte Meldungen verworfen. Diese Option ist standardmäßig aktiviert.

XML-Schema überprüfen

Bei aktivierter Option versucht die Message Queue-Clientlaufzeit, eine XML-Meldung anhand der angegebenen XML-Schemadefinitionen (oder anhand der DTD, falls keine XSD angegeben ist) zu überprüfen, bevor sie an den Broker gesendet wird. Kann das angegebene Schema nicht gefunden oder die Meldung nicht validiert werden, wird die Meldung nicht gesendet, und eine Ausnahme wird ausgelöst. Diese Option ist standardmäßig deaktiviert.

XML-Schema - URI-Liste

Eine mit Leerzeichen getrennte Liste von URI-Zeichenfolgen einer XML-Schemadefinition (XSD). Wenn die XML-Schemaüberprüfung aktiviert ist, verweisen die URIs auf den Speicherort einer oder mehrerer XSDs, die für die XML-Schemaüberprüfung zu verwenden sind.
