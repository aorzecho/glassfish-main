JMS-Service

Auf der Seite "JMS-Service" können Sie Einstellungen konfigurieren, die von allen JMS-Verbindungen verwendet werden sollen.

Weitere Informationen zum Konfigurieren des JMS-Service finden Sie in Chapter 19, Using the Java Message Service, im Oracle GlassFish Server Application Development Guide.

Die Seite "JMS-Service" enthält die folgenden Optionen.

Standardeinstellungen laden

Schaltfläche zum Wiederherstellen der Standardwerte für Einstellungen, für die es Standardwerte gibt. Einstellungen, die nicht über Standardwerte verfügen, können nicht geändert werden.

Ping

Schaltfläche zur Überprüfung, ob der JMS-Service auf dem standardmäßigen JMS-Host ausgeführt wird. Wenn der JMS-Service gegenwärtig ausgeführt wird, wird die Meldung ausgegeben, dass der Ping-Test erfolgreich war.

Name der Konfiguration

Der Name der Konfiguration, für die die Einstellungen auf dieser Seite angewendet werden. Dieses Feld ist schreibgeschützt.

Typ

Der Zugriffstyp für den JMS-Service. Zur Auswahl stehen:

EMBEDDED

Es wird auf den JMS-Service auf dem lokalen Host zugegriffen. Der JMS-Service wird in derselben JVM gestartet wie GlassFish Server. Bei verzögerter Initialisierung wird der eingebettete Standard-Broker nicht beim Start von GlassFish Server, sondern beim ersten Zugriff auf JMS-Services gestartet. Dies ist der standardmäßige JMS-Servicetyp.

LOCAL

Es wird auf den JMS-Service auf dem lokalen Host zugegriffen. Der JMS-Provider wird gleichzeitig mit GlassFish Server gestartet.

REMOTE

Es wird auf einen JMS-Service auf einem anderen System zugegriffen. Bei Auswahl von REMOTE wird der JMS-Service beim nächsten Serverstart nicht durch GlassFish Server gestartet. Stattdessen wird der JMS-Service über Message Queue gestartet und verwaltet, sodass der Message Queue-Broker separat gestartet werden muss. Informationen über das Starten des Brokers finden Sie in Starting Brokers im Oracle GlassFish Server Message Queue Administration Guide. Wenn Sie diesen Wert wählen und einen Remote-Host verwenden, befolgen Sie die Anweisungen unter So wird ein JMS-Host bearbeitet, um den Namen des Remote-Hosts anzugeben.

Timeout beim Starten

Die Anzahl Sekunden, die GlassFish Server auf das Starten des JMS-Service wartet, bevor der Startvorgang abgebrochen wird. Auf einem langsamen oder überlasteten System sollten Sie den Standardwert erhöhen. Der Standardwert ist 60.

Startargumente

Argumente zum Anpassen des Startvorgangs für den JMS-Service. Verwenden Sie verfügbare Argumente über as-install-parent/mq/bin/imqbrokerd-Befehl.

Neu verbinden

Wenn das Kontrollkästchen "Neu verbinden" aktiviert ist, versucht der JMS-Dienst, die Verbindung mit einem Message Server (oder der Liste von Adressen im Attribut "AddressList") wiederherzustellen, wenn eine Verbindung unterbrochen wird. Diese Option ist standardmäßig aktiviert.

Intervall für erneute Verbindung

Die Anzahl Sekunden zwischen zwei Versuchen zur Verbindungswiederherstellung. Dieses Intervall gilt für Versuche mit den einzelnen Adressen im Attribut "AddressList" sowie für nachfolgende Adressen in der Liste. Falls dieses Intervall zu kurz ist, verfügt ein Broker nicht über genügend Zeit zur Wiederherstellung. Wenn das Intervall zu lang ist, erfährt die Verbindungsherstellung möglicherweise eine nicht akzeptable Verzögerung. Der Standardwert beträgt 5 Sekunden.

Versuche für erneute Verbindung

Die Anzahl an Versuchen für die (erneute) Verbindungsherstellung für jede Adresse im Attribut "AddressList", bevor die Clientlaufzeit mit der nächsten Adresse der Liste fortfährt. Der Wert -1 zeigt an, dass die Anzahl von Versuchen zum erneuten Herstellen einer Verbindung unbegrenzt ist (die Clientlaufzeit versucht, eine Verbindung mit der ersten Adresse herzustellen, bis dieser Vorgang erfolgreich ausgeführt werden kann). Der Standardwert ist 3.

Standard-JMS-Host

Der Name des Standard-JMS-Hosts. Der Standardwert ist default_JMS_host.

Adresslistenverhalten

Die Reihenfolge der Verbindungsversuche. Zur Auswahl stehen:

random

Es wird nach dem Zufallsprinzip eine Adresse aus "AddressList" ausgewählt. Wenn eine Vielzahl von Clients versuchen, eine Verbindung über dieselbe Verbindungsfactory herzustellen, geben Sie random an, um zu verhindern, dass diese Verbindungen alle über dieselbe Adresse erfolgen. Dies ist die Standardeinstellung.

priority

Der Versuch zur Verbindungswiederherstellung wird immer für die erste Serveradresse in "AddressList" durchgeführt; eine andere Adresse wird nur dann verwendet, wenn der erste Broker nicht verfügbar ist.

Adresslisteniterationen

Die Anzahl von AddressList-Durchläufen, die der JMS-Service durchführt, um (erneut) eine Verbindung herzustellen. Der Wert -1 gibt an, dass die Anzahl der Versuche unbegrenzt ist. Der Standardwert ist 3.

MQ-Schema und MQ-Service

Der Name des Message Queue-Adressenschemas und des Message Queue-Verbindungsservice, wenn nicht das Standardschema bzw. der Standardservice verwendet werden soll. Syntaxinformationen finden Sie in Syntax von MQ-Schema und MQ-Serviceadresse.

Weitere Eigenschaften

Weitere Eigenschaften für den JMS-Service.

Information zu verfügbaren Eigenschaften finden Sie unter Für JMS-Service spezifische Eigenschaften.
