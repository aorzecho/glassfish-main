JMS-Dienst

Auf der Seite "JMS-Dienst" können Sie Einstellungen konfigurieren, die von allen JMS-Verbindungen verwendet werden sollen.

Weitere Informationen zum Konfigurieren des JMS-Dienstes finden Sie in Kapitel 17, Using the Java Message Service in Oracle GlassFish Server 3.1 Application Development Guide.

Die Seite "JMS-Dienst" enthält die folgenden Optionen.

Standardeinstellungen laden

Schaltfläche zum Wiederherstellen der Standardwerte für alle Einstellungen.

Ping

Schaltfläche zur Überprüfung, ob der JMS-Dienst auf dem standardmäßigen JMS-Host ausgeführt wird. Wenn der JMS-Dienst gegenwärtig ausgeführt wird, wird die Meldung ausgegeben, dass der Ping-Test erfolgreich war.

Konfigurationsname

Der Name der Konfiguration, für die die Einstellungen auf dieser Seite gelten. Dieses Feld ist schreibgeschützt.

Typ

Der Zugriffstyp für den JMS-Dienst. Zur Auswahl stehen:

EMBEDDED

Es wird auf den JMS-Dienst auf dem lokalen Host zugegriffen. Der JMS-Dienst wird in derselben JVM gestartet wie GlassFish-Server. Bei träger Initialisierung wird der eingebettete Standard-Broker nicht beim Start von GlassFish-Server, sondern beim ersten Zugriff auf JMS-Dienste gestartet. Dies ist der standardmäßige JMS-Diensttyp.

LOCAL

Es wird auf den JMS-Dienst auf dem lokalen Host zugegriffen. Der JMS-Provider wird gleichzeitig mit GlassFish-Server gestartet.

REMOTE

Es wird auf einen JMS-Dienst auf einem anderen System zugegriffen. Bei Auswahl von REMOTE wird der JMS-Dienst beim nächsten Serverstart nicht durch GlassFish-Server gestartet. Stattdessen wird der JMS-Dienst über Message Queue gestartet und verwaltet, sodass der Message Queue-Broker separat gestartet werden muss. Weitere Informationen zum Starten des Brokers finden Sie unter Starting Brokers in Oracle GlassFish Message Queue 4.5 Administration Guide. Wenn Sie diesen Wert wählen und einen Remotehost verwenden, befolgen Sie die Anweisungen unter So bearbeiten Sie einen JMS-Host, um den Namen des Remotehosts anzugeben.

Zeitüberschreitung beim Starten

Die Anzahl Sekunden, die GlassFish-Server auf das Starten des JMS-Diensts wartet, bevor der Startvorgang abgebrochen wird. Auf einem langsamen oder überlasteten System sollten Sie den Standardwert erhöhen. Der Standardwert ist 60.

Startargumente

Argumente zum Anpassen des Startvorgangs für den JMS-Dienst. Verwenden Sie beliebige Argumente, die über den Befehl as-install-parent/mq/bin/imqbrokerd verfügbar sind.

Neu verbinden

Wenn das Kontrollkästchen "Neu verbinden" aktiviert ist, versucht der JMS-Dienst, die Verbindung mit einem Nachrichtenserver (oder der Liste von Adressen im Attribut "AddressList") wiederherzustellen, wenn eine Verbindung unterbrochen wird. Diese Option ist standardmäßig aktiviert.

Intervall für erneute Verbindung

Die Anzahl Sekunden zwischen zwei Versuchen zur Verbindungswiederherstellung. Dieses Intervall gilt für Versuche mit den einzelnen Adressen im Attribut "AddressList" sowie für nachfolgende Adressen in der Liste. Falls dieses Intervall zu kurz ist, verfügt ein Broker nicht über genügend Zeit zur Wiederherstellung. Wenn das Intervall zu lang ist, erfährt die Verbindungsherstellung möglicherweise eine nicht akzeptable Verzögerung. Der Standardwert beträgt 5 Sekunden.

Versuche für erneute Verbindung

Die Anzahl an Versuchen für die (erneute) Verbindungsherstellung für jede Adresse im Attribut "AddressList", bevor die Clientlaufzeit mit der nächsten Adresse der Liste fortfährt. Der Wert -1 zeigt an, dass die Anzahl von Versuchen zum erneuten Herstellen einer Verbindung unbegrenzt ist (die Clientlaufzeit versucht, eine Verbindung mit der ersten Adresse herzustellen, bis dieser Vorgang erfolgreich ausgeführt werden kann). Der Standardwert ist 3.

Standardmäßiger JMS-Host

Der Name des standardmäßigen JMS-Hosts. Der Standardwert lautet default_JMS_host.

Adresslistenverhalten

Die Reihenfolge der Verbindungsversuche. Zur Auswahl stehen:

random

Es wird nach dem Zufallsprinzip eine Adresse aus "AddressList" ausgewählt. Wenn eine Vielzahl von Clients versuchen, eine Verbindung über dieselbe Verbindungsfactory herzustellen, geben Sie random an, um zu verhindern, dass diese Verbindungen alle über dieselbe Adresse erfolgen. Dies ist die Standardeinstellung.

priority

Der Versuch zur Verbindungswiederherstellung wird immer für die erste Serveradresse in "AddressList" durchgeführt; eine andere Adresse wird nur dann verwendet, wenn der erste Broker nicht verfügbar ist.

Adresslisteniterationen

Die Anzahl von AddressList-Durchläufen, die der JMS-Dienst durchführt, um (erneut) eine Verbindung herzustellen. Der Wert -1 gibt an, dass die Anzahl der Versuche unbegrenzt ist. Der Standardwert ist 3.

MQ-Schema und MQ-Dienst

Der Name des Message Queue-Adressenschemas und des Message Queue-Verbindungsdiensts, wenn nicht das Standardschema bzw. der Standarddienst verwendet werden soll. Informationen zur Syntax finden Sie unter Syntax von MQ-Schema und MQ-Dienstadresse.

Weitere Eigenschaften

Weitere Eigenschaften für den JMS-Dienst.

Information zu verfügbaren Eigenschaften finden Sie unter Spezifische Eigenschaften des JMS-Diensts.
