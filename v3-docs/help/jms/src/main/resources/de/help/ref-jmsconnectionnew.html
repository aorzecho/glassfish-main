Neue JMS-Verbindungs-Factory

Auf der Seite "Neue JMS-Verbindungs-Factory" können Sie eine neue JMS-Verbindungsfactory erstellen.

Die Seite "Neue JMS-Verbindungs-Factory" enthält die folgenden Optionen.

Pool Name (Poolname)

Ein eindeutiger Name, der die Verbindungsfactory identifiziert. GlassFish-Server legt den JNDI-Namen der Verbindungsfactory für diesen Wert fest.

Resource Type (Ressourcen-Typ)

Der Ressourcentyp der Verbindungsfactory. Zur Auswahl stehen javax.jms.ConnectionFactory, javax.jms.QueueConnectionFactory und javax.jms.TopicConnectionFactory.

Beschreibung

Eine Beschreibung der Verbindungsfactory.

Status

Der Status der Verbindungsfactory. Die Verbindungsfactory kann aktiviert oder deaktiviert sein. Diese Option ist standardmäßig aktiviert.

Initial and Minimum Pool Size (Ursprüngliche und minimale Poolgröße)

Die minimale Anzahl an Verbindungen im Pool für die Verbindungsfactory. Der Standardwert ist 8.

Maximum Pool Size (Maximale Poolgröße)

Die maximale Anzahl an Verbindungen im Pool für die Verbindungsfactory. Der Standardwert lautet 32.

Pool Resize Quantity (Anzahl der Größenänderungen des Pools)

Die Anzahl an Verbindungen, die nach Ablauf der Pool-Leerlaufzeit entfernt werden sollen. Der Standardwert ist 2.

Idle Timeout (Leerlaufzeitlimit)

Die maximale Zeit in Sekunden, die sich eine Verbindung im Pool im Leerlauf befinden kann. Der Standardwert beträgt 300.

Max Wait Time (Maximale Wartezeit)

Die Zeitspanne, die eine Anwendung nach dem Anfordern einer Verbindung abwartet, bevor eine Verbindungszeitüberschreitung ausgegeben wird. Der Standardwert lautet 60000.

On Any Failure (Bei sämtlichen Fehlern)

Wenn das Kontrollkästchen "Sämtliche Verbindungen trennen" aktiviert ist, schließt GlassFish-Server bei einem einzigen Verbindungsabbruch alle Verbindungen im Pool und stellt sie wieder her. Diese Option ist standardmäßig deaktiviert.

Transaktionsunterstützung

Die gewählte Transaktionsunterstützung überschreibt das Transaktionsunterstützungsattribut des mit diesem Verbindungspool verknüpften Ressourcenadapters in einer Weise, mit der Abwärtskompatibilität gewährleistet wird. Anders ausgedrückt: Es kann dieselbe oder eine niedrigere Transaktionsebene unterstützt werden als die im Ressourcenadapter angegebene, jedoch keine höhere Transaktionsebene.

Folgende Werte sind möglich:

XATransaction

Die Ressource kann für Transaktionen eingesetzt werden, für die mehrere Ressourcen innerhalb eines Transaktionsbereichs verwendet werden. Dies ist der Standardwert für eine JMS-Verbindungsfactory.

Beispiel: Transaktionen können diese Ressource sowie eine JDBC-Ressource, eine Verbindungsressource oder eine andere JMS-Verbindungsfactory-Ressource verwenden. Dieser Wert bietet die meiste Flexibilität. Eine als XATransaction konfigurierte Ressource wird für zweiphasige Durchführungsoperationen verwendet.

LocalTransaction

Die Ressource kann entweder für Transaktionen mit nur einer Ressource innerhalb des Transaktionsbereichs oder als letzter Agent in einer verteilten Transaktion mit mehreren XA-Ressourcen verwendet werden.

Dieser Wert bietet eine geringfügig bessere Leistung. Eine als LocalTransaction konfigurierte Ressource wird nicht in zweiphasigen Durchführungsoperationen verwendet.

NoTransaction

Die Ressource kann nie in Transaktionen verwendet werden. Diese Einstellung wird für JMS-Anwendungen nur eingeschränkt verwendet.

Connection Validation (Verbindungsüberprüfung)

Wenn das Kontrollkästchen "Erforderlich" aktiviert ist, werden Verbindungen vor der Übergabe an die Anwendung überprüft. Schlägt die Überprüfung einer Ressource fehl, so wird die Ressource zerstört, und eine neue Ressource wird erstellt und zurückgegeben. Diese Option ist standardmäßig deaktiviert.

Weitere Eigenschaften

Weitere Eigenschaften für die Verbindungsfactory.

Information zu verfügbaren Eigenschaften finden Sie unter Spezifische Eigenschaften von JMS-Verbindungsfactories.

Ziele

Cluster und eigenständige Instanzen für die Verbindungsfactory. Verschieben Sie die benötigten Ziele in die Spalte "Ausgewählte Ziele", indem Sie die Schaltflächen "Hinzufügen" und "Alle hinzufügen" verwenden. Verschieben Sie die nicht benötigten Ziele in die Spalte "Verfügbare Ziele", indem Sie die Schaltflächen "Entfernen" und "Alle entfernen" verwenden.

Diese Option wird nur dann angezeigt, wenn in der Domäne Cluster oder eigenständige Instanzen erstellt wurden.
