So erstellen Sie ein physisches JMS-Ziel

Normalerweise ist die Erstellung physischer Ziele nicht erforderlich, da Sie beim Erstellen einer JMS-Zielressource einen Namen für ein physisches Ziel angeben und GlassFish-Server das physische Ziel bei Bedarf automatisch erstellt.

  1. Erweitern Sie in der Navigationsstruktur den Konfigurationsknoten.
  2. Erweitern Sie unter dem Konfigurationsknoten den Knoten "Java Message Service".
  3. Wählen Sie unter dem Knoten "Java Message Service" den Knoten "Physikalische Ziele" aus.

    Die Seite "Physikalische Ziele" wird geöffnet.

  4. Klicken Sie auf der Seite "Physikalische Ziele" auf "Neu".

    Die Seite "Neue physische JMS-Ziele" wird geöffnet.

  5. Geben Sie im Feld "Name" den Namen des physischen Ziels ein.
  6. Wählen Sie in der Dropdown-Liste "Typ" entweder javax.jms.Topic oder javax.jms.Queue aus.
  7. Geben Sie im Feld "Höchstzahl nicht konsumierter Meldungen" die für das Ziel maximal zulässige Anzahl unkonsumierter Meldungen ein.

    Der Wert –1 bedeutet, dass die Anzahl der unkonsumierten Meldungen nicht begrenzt ist. Der Standardwert ist –1.

  8. Geben Sie im Feld "Maximalgröße von Meldungen" die maximal zulässige Größe einer einzelnen Meldung in Byte ein.

    Der Wert –1 bedeutet, dass die Meldungsgröße unbegrenzt ist. Der Standardwert ist –1.

  9. Geben Sie im Feld "Maximaler Gesamtspeicher" den maximalen Gesamtspeicher für unkonsumierte Meldungen in Byte ein.

    Der Wert –1 bedeutet, dass die Speichergröße für unkonsumierte Meldungen nicht begrenzt ist. Der Standardwert ist –1.

  10. Wählen Sie in der Dropdown-Liste "Schwellenwertverhalten" das Verhalten aus, das der Broker bei Erreichen des Speicher-Schwellenwerts anwenden soll.
    • Reject newest messages weist den Broker an, die neuesten Meldungen zurückzuweisen, und den produzierenden Client nur dann mit einer Ausnahme zu benachrichtigen, wenn die Meldung persistent ist.

      Reject newest messages ist der Standardwert.

    • Slow down producers weist den Broker an, die Geschwindigkeit zu verringern, mit der Meldungsproduzenten Meldungen senden.
    • Throw out oldest weist den Broker an, die ältesten Meldungen zu verwerfen.
    • Throw out lowest-priority messages weist den Broker an, die Meldungen mit der niedrigsten Priorität zu verwerfen.

    Wenn der Wert Throw out oldest oder Throw out lowest-priority messages gewählt und "Warteschlange für nicht zugestellte Meldungen verwenden" auf "true" gesetzt ist, werden überschüssige Meldungen in die Warteschlange für nicht zugestellte Meldungen verschoben.

  11. Geben Sie im Feld "Höchstzahl von Produzenten" die maximale Anzahl an Meldungsproduzenten ein, die mit diesem physischen Ziel verknüpft werden können.

    Bei Erreichen dieses Grenzwerts können keine neuen Produzenten mehr erstellt werden. Der Wert –1 bedeutet, dass die Anzahl der Produzenten unbegrenzt ist. Der Standardwert beträgt 100.

  12. Geben Sie im Feld "Höchstzahl der Meldungen bei Sammelzustellung" die maximale Anzahl an Meldungen ein, die Konsumenten in einem einzigen Batch zugestellt werden können.

    Der Wert –1 bedeutet, dass die Anzahl der Meldungen unbegrenzt ist. Der Standardwert ist 1000.

  13. Wählen Sie in der Dropdown-Liste "Warteschlange für nicht zugestellte Meldungen verwenden" den Wert "true" aus, um nicht zugestellte Meldungen in die Warteschlange für nicht zugestellte Meldungen zu stellen.

    Wenn der Wert auf "false" gesetzt wird, werden nicht zugestellte Meldungen verworfen. Die Standardeinstellung für diese Option ist "true".

  14. Wählen Sie in der Dropdown-Liste "XML-Schema überprüfen" den Wert "true" aus, um die XML-Schemaüberprüfung für das Ziel zu aktivieren.

    Wenn der Wert auf "true" gesetzt ist, versucht die Message Queue-Clientlaufzeit, eine XML-Meldung anhand der angegebenen XML-Schemadefinitionen (oder anhand der DTD, falls keine XSD angegeben ist) zu überprüfen, bevor sie an den Broker gesendet wird. Kann das angegebene Schema nicht gefunden oder die Meldung nicht überprüft werden, wird die Meldung nicht gesendet, und eine Ausnahme wird ausgelöst. Die Standardeinstellung für diese Option ist "false".

  15. Geben Sie im Feld "XML-Schema - URI-Liste" eine mit Leerzeichen getrennte Liste von URI-Zeichenfolgen einer XML-Schemadefinition (XSD) ein.

    Wenn die XML-Schemaüberprüfung aktiviert ist, verweisen die URIs auf den Speicherort einer oder mehrerer XSDs, die für die XML-Schemaüberprüfung zu verwenden sind.
