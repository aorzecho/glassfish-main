Syntax von MQ-Schema und MQ-Dienstadresse

Die vollständige Syntax für die Adresse eines Nachrichtendiensts lautet 

scheme://address-syntax

dabei sind Schema und Adress-Syntax in der Tabelle unten beschrieben.

Die Werte für MQ-Schema und MQ-Dienst sind in den ersten beiden Spalten der folgenden Tabelle aufgeführt.

Schema

Verbindungs- dienst

Beschreibung

Adress-Syntax

mq

jms und ssljms

Die Message Queue-Clientlaufzeit stellt eine Verbindung zum MQ-Port-Mapper auf dem angegebenen Host und Port her. Der Port-Mapper gibt eine Liste der dynamisch eingerichteten Verbindungsdienstports zurück, und die Message Queue-Clientlaufzeit stellt anschließend eine Verbindung zu dem Port her, über den der angegebene Verbindungsdienst gehostet wird.

[Hostname][: Port][/ Dienstname]

Standardeinstellungen: hostName = localhost, port = 7676, serviceName = jms

Die Standardeinstellungen gelten ausschließlich für den jms-Verbindungsdienst. Für den Verbindungsdienst ssljms müssen alle Variablen angegeben werden.

Beispiel: mq:MyHost:7677/ssljms

mqtcp

jms

Die Message Queue-Clientlaufzeit verbindet sich über TCP mit dem angegebenen Host und Port (der MQ-Port-Mapper wird umgangen), um eine Verbindung herzustellen.

hostname:anschlus /jms

Beispiel: mqtcp:localhost:7676/jms

mqssl

ssljms

Die Message Queue-Clientlaufzeit verbindet sich über SSL mit den angegebenen Host und Port (der MQ-Port-Mapper wird umgangen), um eine Verbindung herzustellen.

hostname:anschluss /ssljms

Beispiel: mqssl:localhost:7676/ssljms

http

httpjms

Die Message Queue-Clientlaufzeit stellt über HTTP eine Verbindung zu einem Message Queue-Tunnel-Servlet mit der angegebenen URL her. (Der Broker muss für den Zugriff auf das HTTP-Tunnel-Servlet konfiguriert sein, wie im Message Queue Administrator’s Guide beschrieben.)

Hostname: Port/Kontext-Root /tunnel

Wenn mehrere Brokerinstanzen das gleiche Tunnel-Servlet verwenden, dann ist die Syntax für eine Verbindung mit einer bestimmten Brokerinstanz (eher als eine zufällig Ausgewählte): http://hostName :port/contextRoot /tunnel?serverName =hostName: instanceName

https

httpsjms

Die Message Queue-Clientlaufzeit stellt eine sichere HTTPS-Verbindung zur angegebenen Message Queue-Tunnel-Servlet-URL her. (Der Broker muss für den Zugriff auf das HTTPS-Tunnel-Servlet konfiguriert sein, wie im Message Queue Administrator’s Guide).

Hostname: Port/Kontext-Root /tunnel

Wenn mehrere Brokerinstanzen das gleiche Tunnel-Servlet verwenden, dann ist die Syntax für eine Verbindung mit einer bestimmten Brokerinstanz (eher als eine zufällig Ausgewählte): http://hostName :port/contextRoot /tunnel?serverName =hostName: instanceName
