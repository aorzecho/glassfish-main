Neue JMS-Zielressource

Auf der Seite "Neue JMS-Zielressource" können Sie eine neue JMS-Zielressource erstellen.

Die Seite "Neue JMS-Zielressource" enthält die folgenden Optionen.

JNDI Name (JNDI-Name)

Ein eindeutiger Name, der die Zielressource identifiziert.

Es wird empfohlen, für JMS-Ressourcen das Benennungssubkontext-Präfix jms/ zu verwenden. Beispiel: jms/Queue.

Name des physikalischen Ziels

Der Name des mit dieser Ressource zu verknüpfenden physischen Ziels.

Das physische Ziel wird von GlassFish Server bei Bedarf automatisch erstellt und wieder gelöscht, wenn Sie die Zielressource löschen.

Resource Type (Ressourcen-Typ)

Der Typ der Zielressource. Zur Auswahl stehen javax.jms.Topic und javax.jms.Queue.

Description (Beschreibung)

Eine Beschreibung der Zielressource.

Status

Der Status der Zielressource. Die Zielressource kann aktiviert oder deaktiviert sein. Diese Option ist standardmäßig aktiviert.

Weitere Eigenschaften

Weitere Eigenschaften für die Zielressource. In GlassFish Server werden keine weiteren Eigenschaften für JMS-Zielressourcen definiert.

Ziele

Cluster und eigenständige Instanzen für die Zielressource. Mithilfe der Schaltflächen "Hinzufügen" und "Alle hinzufügen" verschieben Sie die benötigten Ziele in die Spalte "Ausgewählte Ziele". Verschieben Sie die nicht benötigten Ziele in die Spalte "Verfügbare Ziele" mithilfe der Schaltflächen "Entfernen" und "Alle entfernen".

Diese Option wird nur dann angezeigt, wenn in der Domain Cluster oder Standalone-Instanzen erstellt wurden.

Verwandte Aufgaben

  • So wird eine JMS-Zielressource erstellt

  • So wird eine JMS-Zielressource bearbeitet

  • So wird eine JMS-Zielressource gelöscht
