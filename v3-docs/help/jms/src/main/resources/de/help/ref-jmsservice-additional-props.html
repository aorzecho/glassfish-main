Für JMS-Service spezifische Eigenschaften

Für den JMS-Service stehen die folgenden Oracle Message Queue-Broker-Konfigurationseigenschaften zur Verfügung.

instance-name

Gibt den vollständigen Namen der Message Queue-Brokerinstanz an. Der Standardname lautet imqbroker.

instance-name-suffix

Gibt ein Suffix an, das zum vollständigen Namen der Message Queue-Brokerinstanz hinzugefügt wird. Das Suffix und der Instanzname werden durch einen Unterstrich (_) getrennt. Beispiel: Wenn der Instanzname imqbroker ist, ändert sich dieser Name durch das Hinzufügen des Suffixes xyz in imqbroker_xyz.

append-version

Wenn dieser Wert auf True gesetzt ist, werden die Haupt- und Nebenversionsnummern mit vorangestellten Unterstrichen (_) an den vollständen Instanznamen des Message Queue Brokers angehängt. Beispiel: Wenn der Instanzname imqbroker ist, ändert sich der Name durch das Hinzufügen der Versionsnummern in imqbroker_8_0. Der Standardwert ist false.

user-name

Der Benutzername zum Herstellen der JMS-Verbindung. Wird nur benötigt, wenn der Standardbenutzername/das Standardkennwort von guest/guest im Broker nicht verfügbar ist. Der Standardwert ist guest.

Kennwort

Das Kennwort zum Herstellen der JMS-Verbindung. Wird nur benötigt, wenn der Standardbenutzername/das Standardkennwort von guest/guest im Broker nicht verfügbar ist. Der Standardwert ist guest.
