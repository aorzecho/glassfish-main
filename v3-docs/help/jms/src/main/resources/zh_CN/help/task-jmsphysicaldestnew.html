创建 JMS 物理目的地

通常无需创建物理目的地，因为在创建 JMS 目的地资源时，您会指定一个物理目的地名称，而 GlassFish Server 会在需要时自动创建物理目的地。

  1. 在导航树中，展开“配置”节点。
  2. 在“配置”节点下，展开“Java 消息服务”节点。
  3. 在“Java 消息服务”节点下，选择“物理目的地”节点。

    此时将打开“物理目的地”页面。

  4. 在“物理目的地”页面中，单击“新建”。

    此时将打开“新建物理目的地”页面。

  5. 在“名称”字段中，键入物理目的地的名称。
  6. 从“类型”下拉式列表中，选择 javax.jms.Topic 或 javax.jms.Queue。
  7. 在“未使用消息的最大数量”字段中，键入目的地允许的未使用消息的最大数量。

    如果值为 -1，则表示未使用消息的数量不受限制。默认值为 –1。

  8. 在“最大消息长度”字段中，键入任一消息的最大长度（以字节为单位）。

    如果值为 -1，则表示消息长度不受限制。默认值为 –1。

  9. 在“最大总内存”字段中，键入未使用消息的最大总内存（以字节为单位）。

    如果值为 -1，则表示未使用消息的内存大小不受限制。默认值为 –1。

  10. 从“阈值限制行为”下拉式列表中，选择达到内存限制阈值时，代理将要遵循的行为。
    • Reject newest messages 告诉代理拒绝最新消息，并通知生成消息的客户机（消息为持久性消息的情况除外）。

      Reject newest messages 为默认值。

    • Slow down producers 告诉代理降低消息生成方发送消息的速率。
    • Throw out oldest 告诉代理废弃最旧的消息。
    • Throw out lowest-priority messages 告诉代理废弃优先级最低的消息。

    如果该值为 Throw out oldest 或 Throw out lowest-priority messages，且“使用停用消息队列”设置为 True，则将过量的消息移到停用消息队列。

  11. 在“生成方最大数量”字段中，键入可以与此物理目的地关联的消息生成方的最大数量。

    达到此限制后，将无法创建新的生成方。如果值为 -1，则表示生成方数量不受限制。默认值为 100。

  12. 在“批量传送最大消息数量”字段中，键入可以向使用方一次性传送的最大消息数量。

    如果值为 -1，则表示消息数量不受限制。默认值是 1000。

  13. 从“使用停用消息队列”下拉式列表中，选择 "True" 以将停用消息放入停用消息队列。

    如果该值设置为 False，则废弃停用消息。默认情况下，此选项设置为 True。

  14. 从“启用验证 XML 模式”下拉式列表中，选择 "True"，为目的地启用 XML 模式验证。

    如果该值设置为 True，则 Message Queue 客户机运行时在将 XML 消息发送给代理前，会尝试根据指定的 XML 模式定义（如果未指定 XSD，则根据 DTD）验证 XML 消息。如果无法找到指定的模式或无法验证消息，则不发送消息并抛出异常。默认情况下，此选项设置为 False。

  15. 在“XML 模式 URI 列表”字段中，键入一个以空格分隔的 XML 模式定义 (XML Schema Definition, XSD) URI 字符串列表。

    如果启用 XML 模式验证，则 URI 将指向要用于 XML 模式验证的一个或多个 XSD 的位置。
