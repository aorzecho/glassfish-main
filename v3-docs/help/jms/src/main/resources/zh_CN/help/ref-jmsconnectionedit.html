编辑 JMS 连接工厂

使用“编辑 JMS 连接工厂”页面可以修改 JMS 连接工厂的设置。

“编辑 JMS 连接工厂”页面包含以下选项。

装入默认值

用于恢复所有设置的默认值的按钮。

池名称

为连接工厂创建的连接器连接池的唯一名称。该名称也可以是连接工厂的 JNDI 名称。池名称是只读字段。

资源类型

连接工厂的类型。资源类型是只读字段。只能在创建新的 JMS 连接工厂时指定资源类型。

说明

连接工厂的说明。

状态

连接工厂的状态。可以启用或禁用连接工厂。默认情况下启用此选项。

初始和最小池大小

连接工厂池中的最小连接数。默认值为 8。

最大池大小

池中连接的最大数目。默认值为 32。

池大小调整数量

池空闲超时到期后要删除的连接数。默认值为 2。

空闲超时

连接在池中保持空闲的最长时间（以秒为单位）。默认值为 300。

最长等待时间

在达到连接超时之前，请求连接的应用程序所等待的时间。默认值为 60000。

一旦失败

如果选中“关闭所有连接”复选框，则 GlassFish Server 将在某个连接失败时关闭池中的所有连接并重新建立这些连接。默认情况下禁用此选项。

事务支持

值可为以下任意一种：

XATransaction

资源可用于在事务范围内使用多个资源的事务。此值为 JMS 连接工厂的默认值。

例如，事务可能涉及此资源和 JDBC 资源、连接器资源或其他 JMS 连接工厂资源。此值提供最大的灵活性。配置为 XATransaction 的资源将参与分为两个阶段的提交操作。

LocalTransaction

资源可用于在事务范围内仅使用一个资源的事务，也可用作使用多个 XA 资源的分布式事务中的最后一个代理。

此值提供更好的性能。配置为 LocalTransaction 的资源将不会参与分为两个阶段的提交操作。

NoTransaction

资源永远不参与事务。JMS 应用程序中限用此设置。

连接验证

如果选中“需要”复选框，则先验证连接，然后才将其提供给应用程序。如果资源验证失败，则销毁此资源，然后创建并返回一个新的资源。默认情况下禁用此选项。

其他属性

连接工厂的其他属性。

有关可用属性的信息，请参见特定于 JMS 连接工厂的属性。
