JMS 可用性

使用“JMS 可用性”页面可配置 Java 消息服务 (Java Message Service, JMS) 的可用性设置。

“JMS 可用性”页面包含以下选项。

装入默认值

使具有默认值的设置恢复其默认值的按钮。不具有默认值的设置不会发生更改。

配置名称

要应用该页面上设置的配置名称。此字段为只读。

可用性服务

用来向 GlassFish 群集提供 JMS 服务的 Message Queue 群集的类型。如果选中“启用”复选框，将使用增强的 Message Queue 群集。否则，将使用常规 Message Queue 群集。

如果选中“启用”复选框，将忽略“配置存储库类型”和“消息存储库类型”选项。

配置存储库类型

常规 Message Queue 群集中配置数据的数据存储库类型。值 masterbroker 指定使用主代理存储和管理配置数据。值 shareddb 指定使用共享数据库存储配置数据。

如果启用“可用性服务”，则忽略该选项。

消息存储库类型

常规群集的代理中消息数据的数据存储库类型。值 file 指定文件存储库。值 jdbc 指定 JDBC 存储库。

如果启用“可用性服务”，则忽略该选项。

数据库供应商
数据库用户名
数据库密码
数据库访问 URL

要在以下任一情况中使用的 JDBC 数据库的数据库供应商、用户、密码和访问 URL：

  • 当启用“可用性服务”时

  • 当“配置存储库类型”设置为 shareddb 时

  • 当“消息存储库类型”设置为 jdbc 时

有关受支持的供应商以及每个供应商的访问 URL 格式的信息，请参见《Oracle GlassFish Server Message Queue 4.5 Administration Guide》中的"JDBC-Based Persistence"。

其他属性

JMS 可用性的其他属性。

GlassFish Server 不预定义任何其他 JMS 可用性属性。
