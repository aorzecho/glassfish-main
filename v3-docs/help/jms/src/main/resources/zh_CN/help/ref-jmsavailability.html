JMS 可用性

使用“JMS 可用性”页可为 Java 消息服务 (JMS) 配置可用性设置。

“JMS 可用性”页包含以下选项：

加载默认值

使具有默认值的设置还原到其默认值的按钮。不具有默认值的设置不会发生更改。

配置名称

此页上的设置应用到的配置的名称。此字段为只读。

可用性服务

用于向 GlassFish 集群提供 JMS 服务的消息队列集群的类型。如果选中“已启用”复选框，则使用增强型 消息队列 集群，否则使用常规 消息队列 集群。

如果选中“已启用”复选框，将忽略“配置存储类型”和“消息存储类型”选项。

配置存储类型

常规 消息队列 集群中配置数据的数据存储库类型。值 masterbroker 指定使用主中介存储和管理配置数据。值 shareddb 指定使用共享数据库存储配置数据。

如果启用了可用性服务，则忽略此选项。

消息存储类型

常规集群内中介的消息数据的数据存储类型。值 file 指定文件存储。值 jdbc 指定 JDBC 存储。

如果启用了可用性服务，则忽略此选项。

数据库供应商
数据库用户名
数据库口令
数据库访问 URL

要在以下任一情况下使用的 JDBC 数据库的数据库供应商、用户、口令和访问 URL：

  • 已启用“可用性服务”时

  • 已将“配置存储类型”设置为 shareddb 时

  • 已将“消息存储类型”设置为 jdbc 时

有关受支持的提供方以及每个提供方的访问 URL 格式的信息，请参阅 Oracle GlassFish Server Message Queue Administration Guide 中的 JDBC-Based Persistence。

其他属性

JMS 可用性的其他属性。

GlassFish Server 未预定义任何其他 JMS 可用性属性。
