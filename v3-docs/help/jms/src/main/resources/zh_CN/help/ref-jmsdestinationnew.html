新建 JMS 目的地资源

使用“新建 JMS 目的地资源”页面可以创建新的 JMS 目的地资源。

“新建 JMS 目的地资源”页面包含以下选项。

JNDI 名称

标识目的地资源的唯一名称。

建议的做法是使用 JMS 资源的命名子上下文前缀 jms/。例如：jms/Queue。

物理目的地名称

要与此资源关联的物理目的地的名称。

GlassFish Server 在需要时自动创建物理目的地，并在您删除目的地资源时删除物理目的地。

资源类型

目的地资源的类型。可用选项为 javax.jms.Topic 和 javax.jms.Queue。

说明

目的地资源的说明。

状态

目的地资源的状态。可以启用或禁用目的地资源。默认情况下启用此选项。

其他属性

目的地资源的其他属性。GlassFish Server 不定义 JMS 目的地资源的任何其他属性。

目标

目标资源的群集和独立实例。使用“添加”和“全部添加”按钮将所需目标移动到“选定目标”列。使用“删除”和“全部删除”按钮将任何不需要的目标移动到“可用目标”列。

仅当已在域中创建群集或独立实例时才会显示此选项。
