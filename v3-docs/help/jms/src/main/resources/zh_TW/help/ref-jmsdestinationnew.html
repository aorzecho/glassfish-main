新建 JMS 目標資源

使用 [新建 JMS 目標資源] 頁面可建立新的 JMS 目標資源。

[新建 JMS 目標資源] 頁面包含下列選項。

JNDI 名稱

可識別目標資源的唯一名稱。

建議的作業是使用 JMS 資源的命名子環境前綴 jms/。例如︰ jms/Queue。

實體目標名稱

要與此資源關聯的實體目標的名稱。

GlassFish Server 會在需要時自動建立實體目標，並在您刪除目標資源時將其刪除。

資源類型

目標資源的類型。可用的選項包括 javax.jms.Topic 和 javax.jms.Queue。

說明

目標資源的說明。

狀態

目標資源的狀態。您可啟用或停用目標資源。此選項預設為啟用。

其他特性

目標資源的其他特性。GlassFish Server 未定義 JMS 目標資源的任何其他特性。

目標

目標資源的叢集與獨立實例。可使用 [增加] 與 [全部加入] 按鈕，將所需的目標移至 [選取的目標] 欄。可使用 [移除] 與 [全部移除] 按鈕，將任何不需要的目標移至 [可用的目標] 欄。

唯有網域中已建立叢集或獨立實例時，才會顯示此選項。
