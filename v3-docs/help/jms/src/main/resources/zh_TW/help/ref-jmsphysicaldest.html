JMS 實體目標

Java 訊息服務 (JMS) 實體目標物件由 Message Queue 代理程式維護。名為 mq.sys.dmq 的佇列是系統目標，過期的和無法傳送的訊息都會重新導向至該目標。

使用 [JMS 實體目標] 頁面可配置 Message Queue 實體目標。

對於每個實體目標，都將提供以下資訊。

名稱

實體目標名稱。

類型

實體目標類型。

統計

實體目標的統計。

實體目標表也包含下列選項。

新建

建立新實體目標的按鈕。

刪除

刪除一或多個所選取實體目標的按鈕。

清除

從一或多個所選取實體目標移除訊息的按鈕。
