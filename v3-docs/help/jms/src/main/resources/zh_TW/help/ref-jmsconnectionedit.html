編輯 JMS 連線工廠

使用 [編輯 JMS 連線工廠] 頁面可修改 JMS 連線工廠的設定。

[編輯 JMS 連線工廠] 頁面包含下列選項。

載入預設值

用於將具有預設值的設定復原成其預設值的按鈕。沒有預設值的設定保持不變。

池名稱

針對連線工廠所建立之連接器連線池的唯一名稱。此名稱也是連線工廠的 JNDI 名稱。[池名稱] 是唯讀欄位。

資源類型

連線工廠的類型。[資源類型] 是唯讀欄位。您僅能在建立新的 JMS 連線工廠時指定資源類型。

說明

連線工廠的說明。

狀態

連線工廠的狀態。您可啟用或停用連線工廠。此選項預設為啟用。

池的初始大小和最小大小

連線工廠的池中最小的連線數目。預設值為 8。

池的最大大小

池中連線的最大數目。預設值為 32。

池設定大小數量

池的閒置逾時過期時要移除的連線數目。預設值為 2。

閒置逾時

連線可以在儲存區中閒置的最大時間 (以秒表示)。預設值為 300。

最大等待時間

請求連線的應用程式在達到連線逾時之前等待的時間。預設值為 60000。

一旦失敗

若選取了 [關閉所有連線] 核取方塊，GlassFish Server 將會在任何一個連線失敗時，關閉池中的所有連線並重新建立連線。此選項預設為停用。

作業事件支援

值可能為下列其中之一：

XATransaction

可供涉及在作業事件範圍內使用多個資源的作業事件使用的資源。對於 JMS 連線工廠而言，此值為預設值。

例如，作業事件可能涉及此資源加上 JDBC 資源、連接器資源或其他 JMS 連線工廠資源。該值具有最大的靈活性。配置為 XATransaction 的資源將參與兩階段的確定作業。

LocalTransaction

可用於僅涉及使用作業事件範圍內一個資源的作業事件的資源，或用做涉及多個 XA 資源的分散式作業事件中最後一個代理程式的資源。

使用該值可以獲得更好的效能。配置為 LocalTransaction 的資源不會用於兩階段的確定作業。

NoTransaction

此資源永遠不能參與作業事件。該設定在 JMS 應用程式中的使用有限。

連線驗證

若選取了 [需要連線驗證] 核取方塊，指定連線給應用程式之前會先驗證。若資源的驗證失敗，則會銷毀該資源，然後建立並傳回新的資源。此選項預設為停用。

其他特性

連線工廠的其他特性。

如需可用特性的相關資訊，請參閱JMS 連線工廠特有的特性。
