建立 JMS 目標資源

當您建立 JMS 目標資源時，GlassFish Server 會針對目標資源建立管理物件資源。您可以在 「連接器」 節點下檢視此資源。

  1. 在瀏覽樹狀結構中，展開 「資源」 節點。

  2. 在 「資源」 節點下，展開 「JMS 資源」 節點。

  3. 在 「JMS 資源」 節點下，選取 「JMS 目標資源」 節點。

    將開啟 「JMS 目標資源」 頁面。

  4. 在 「JMS 目標資源」 頁面上，按一下 「新建」。

    將開啟 「新建 JMS 目標資源」 頁面。

  5. 在 「JNDI 名稱」 欄位中，鍵入資源的 JNDI 名稱。

    建議的作業是使用 JMS 資源的命名子環境前綴 jms/。例如︰ jms/Queue。

  6. 在 「實體目標名稱」 欄位中，鍵入要與此資源關聯的實體目標名稱。

    GlassFish Server 會在需要時自動建立實體目標，並在您刪除目標資源時將其刪除。

  7. 從 「資源類型」 下拉式清單中，選取 「javax.jms.Topic」 或 「javax.jms.Queue」。

  8. 在 「說明」 欄位中，鍵入目標資源的說明。

  9. 選取「狀態」的「已啟用」核取方塊以在執行階段啟用該資源。

    此選項預設為啟用。

  10. 在 「其他特性」 區域中，指定目標資源的其他特性。

    GlassFish Server 未定義 JMS 目標資源的任何其他特性。

    若要增加特性，請按一下 「增加特性」 按鈕。在出現的空白列中，於 「名稱」 欄位中鍵入特性名稱，然後在 「值」 欄位中鍵入特性值。

  11. In the Targets area, do the following:

    1. From the Available column, select the target or targets to which applications that use the resource will be deployed.

      The available targets include the available clusters and server instances, as well as the default server instance, server.

    2. Click Add to move the target to the Selected column.

  12. 按一下 「確定」。
