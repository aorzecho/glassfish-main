JMS 可用性

使用 [JMS 可用性] 頁面可配置 Java 訊息服務 (JMS) 的可用性設定。

[JMS 可用性] 頁面包含下列選項。

載入預設值

用於將具有預設值的設定復原成其預設值的按鈕。沒有預設值的設定保持不變。

配置名稱

此頁面上的設定要套用到的配置名稱。此為唯讀欄位。

可用性服務

用於向 GlassFish 叢集提供 JMS 服務之 Message Queue 叢集的類型。若選取了 [已啟用] 核取方塊，則會使用增強的 Message Queue 叢集。否則便會使用傳統的 Message Queue 叢集。

若選取了 [已啟用] 核取方塊，便會忽略 [配置存放區類型] 和 [訊息存放區類型] 選項。

配置存放區類型

傳統的 Message Queue 叢集中配置資料的資料存放區類型。值 masterbroker 會指定使用主代理程式來儲存及管理配置資料。值 shareddb 會指定使用共用資料庫來儲存配置資料。

若已啟用 [可用性服務]，便會忽略此選項。

訊息存放區類型

傳統叢集中代理程式之訊息資料的資料存放區類型。值 file 會指定檔案存放區。值 jdbc 會指定 JDBC 存放區。

若已啟用 [可用性服務]，便會忽略此選項。

資料庫供應商
資料庫使用者名稱
資料庫密碼
資料庫存取 URL

要在以下任一情形使用之 JDBC 資料庫的資料庫供應商、使用者、密碼及存取 URL：

  • [可用性服務] 已啟用時

  • [配置存放區類型] 設定為 shareddb 時

  • [訊息存放區類型] 設定為 jdbc 時

如需有關支援的供應商及各供應商存取 URL 格式的相關資訊，請參閱「Oracle GlassFish Server Message Queue 4.5 Administration Guide」中的「JDBC-Based Persistence」。

其他特性

JMS 可用性的其他特性。

GlassFish Server 未預先定義任何其他的 JMS 可用性特性。
