JMS 可用性

使用「JMS 可用性」頁面可組態 Java 訊息服務 (JMS) 的可用性設定。

「JMS 可用性」頁面包含下列選項。

載入預設值

用於將具有預設值的設定復原成其預設值的按鈕。沒有預設值的設定保持不變。

組態名稱

此頁面上的設定套用的組態名稱。本欄位是唯讀的。

可用性服務

用於向 GlassFish 叢集提供 JMS 服務的 Message Queue 叢集類型。如果選取「已啟用」核取方塊，則會使用增強的 Message Queue 叢集。否則會使用傳統的 Message Queue 叢集。

如果選取「已啟用」核取方塊，則會忽略「配置存放區類型」與「訊息存放區類型」選項。

配置存放區類型

傳統 Message Queue 叢集中組態資料的資料存放區類型。值 masterbroker 會指定使用主代理程式來儲存及管理組態資料。值 shareddb 會指定使用共用資料庫來儲存組態資料。

如果啟用「可用性服務」，則會忽略此選項。

訊息存放區類型

傳統叢集中代理程式之訊息資料的資料存放區類型。值 file 會指定檔案存放區。值 jdbc 會指定 JDBC 存放區。

如果啟用「可用性服務」，則會忽略此選項。

資料庫供應商
資料庫使用者名稱
資料庫密碼
資料庫存取 URL

要在以下任一情形使用的資料庫供應商、使用者、密碼及 JDBC 資料庫的存取 URL：

  • 當可用性服務啟用時

  • 當「配置存放區類型」設為 shareddb 時

  • 當「訊息存放區類型」設為 jdbc 時

如需有關支援供應商及各供應商存取 URL 格式的相關資訊，請參閱 Oracle GlassFish Server Message Queue Administration Guide 中的 JDBC-Based Persistence。

其他特性

JMS 可用性的其他特性。

GlassFish Server 未預先定義任何其他 JMS 可用性特性。
