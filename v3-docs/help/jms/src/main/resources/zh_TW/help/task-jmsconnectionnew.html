建立 JMS 連線工廠資源

當您建立 JMS 連線工廠時，GlassFish Server 會針對連線工廠建立連接器連線集區和連接器資源。您可以在 「連接器」 節點下檢視這些物件。

  1. 在瀏覽樹狀結構中，展開 「資源」 節點。

  2. 在 「資源」 節點下，展開 「JMS 資源」 節點。

  3. 在 「JMS 資源」 節點下，選取 「JMS 連線工廠」 節點。

    將開啟 「JMS 連線工廠」 頁面。

  4. 在 「JMS 連線工廠」 頁面上，按一下 「新建」。

    將顯示 「新建 JMS 連線工廠」 頁面。

  5. 在 「集區名稱」 欄位中，鍵入連線工廠的唯一名稱。

    GlassFish Server 會將連線工廠的 JNDI 名稱設為此值。

  6. 從 「資源類型」 下拉式清單中，選擇 「javax.jms.ConnectionFactory」、「javax.jms.QueueConnectionFactory」 或 「javax.jms.TopicConnectionFactory」。

  7. 在 「說明」 欄位中，鍵入連線工廠的說明。

  8. 選取「狀態」的「已啟用」核取方塊以在執行階段啟用該資源。

    此選項預設為啟用。

  9. 在 「集區的初始大小和最小大小」 欄位中，指定集區中的最小連線數目。

    預設值為 8。

  10. 在 「集區的最大大小」 欄位中，指定集區中的最大連線數目。

    預設值為 32。

  11. 在 「集區設定大小數量」 欄位中，指定集區的閒置逾時過期時要移除的連線數目。

    預設值為 2。

  12. 在 「閒置逾時」 欄位中，指定連線可以在集區中閒置的最長時間 (以秒為單位)。

    預設值為 300。

  13. 在 「最大等待時間」 欄位中，指定請求連線的應用程式在連線逾時前等待的時間。

    預設值為 60000。

  14. 選取 「一旦失敗便關閉所有連線」 核取方塊，可讓 GlassFish Server 在任何一個連線失敗時，關閉集區中的所有連線並重新建立連線。

    如果未選取此核取方塊，則僅在使用個別連線時才會重新建立這些連線。此選項預設為停用。

  15. 針對 JMS 連線工廠資源，選取 「作業事件支援」 的層級。

    選取的值會以向下相容的形式覆寫資源配接卡中的作業事件支援屬性。

    • 為可用於涉及使用作業事件範圍內多個資源的作業事件的資源選取 「XATransaction」 (JMS 資源配接卡的預設值)。

      例如，作業事件可能涉及此資源加上 JDBC 資源、連接器資源或其他 JMS 連線工廠資源。該值具有最大的靈活性。組態為 XATransaction 的資源將參與兩階段的確定作業。

    • 為可用於僅涉及使用作業事件範圍內一個資源的作業事件的資源，或用做涉及多個 XA 資源的分散式作業事件中最後一個代理程式的資源選取 「LocalTransaction」。

      使用該值可以獲得更好的效能。組態為 LocalTransaction 的資源不會用於兩階段的確定作業。

    • 為永不參與作業事件的資源選取 「NoTransaction」。

      該設定在 JMS 應用程式中的使用有限。

  16. 選取 「需要連線驗證」 核取方塊以啟用連線驗證。

    若選取了此選項，指定連線給應用程式之前會先驗證。若資源的驗證失敗，則會銷毀該資源，然後建立並傳回新的資源。此選項預設為停用。

  17. 在 「其他特性」 區段中，指定連線工廠的其他特性。

    如需可用特性的相關資訊，請參閱JMS 連線工廠特定的特性。

    若要增加特性，請按一下 「增加特性」 按鈕。在出現的空白列中，於 「名稱」 欄位中鍵入特性名稱，然後在 「值」 欄位中鍵入特性值。

  18. In the Targets area, do the following:

    1. From the Available column, select the target or targets to which applications that use the resource will be deployed.

      The available targets include the available clusters and server instances as well as the default server instance, server.

    2. Click Add to move the target to the Selected column.

  19. 按一下 「確定」。
