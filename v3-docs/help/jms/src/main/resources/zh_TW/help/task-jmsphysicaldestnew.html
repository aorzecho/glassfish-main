建立 JMS 實體目標

您通常不需要建立實體目標，因為您可以在建立 JMS 目標資源時指定實體目標名稱，而 GlassFish Server 會在需要時自動建立實體目標。

  1. 在瀏覽樹狀結構中，展開 [配置] 節點。
  2. 在 [配置] 節點下，展開 [Java 訊息服務] 節點。
  3. 在 [Java 訊息服務] 節點下，選取 [實體目標] 節點。

    將開啟 [實體目標] 頁面。

  4. 在 [實體目標] 頁面上，按一下 [新建]。

    將開啟 [新建實體目標] 頁面。

  5. 在 [名稱] 欄位中，鍵入實體目標的名稱。
  6. 從 [類型] 下拉式清單中，選取 [javax.jms.Topic] 或 [javax.jms.Queue]。
  7. 在 [最大未使用訊息數目] 欄位中，鍵入目標所允許的最大未使用訊息數目。

    值 -1 表示未使用訊息的數目沒有限制。預設值為 –1。

  8. 在 [最大訊息大小] 欄位中，鍵入任何單一訊息的最大大小 (以位元組為單位)。

    值 -1 表示訊息大小沒有限制。預設值為 –1。

  9. 在 [最大記憶體總容量] 欄位中，鍵入未使用訊息的最大記憶體總容量 (以位元組為單位)。

    值 -1 表示未使用訊息的記憶體大小沒有限制。預設值為 –1。

  10. 從 [臨界值限制運作方式] 下拉式清單中，選取到達記憶體限制臨界值時代理程式需遵循的運作方式。
    • Reject newest messages 會要求代理程式僅在訊息持續出現時，拒絕最新的訊息並丟出異常以通知產生訊息的用戶端。

      Reject newest messages 為預設值。

    • Slow down producers 會要求代理程式減慢訊息產生器傳送訊息的速度。
    • Throw out oldest 會要求代理程式捨棄最舊的訊息。
    • Throw out lowest-priority messages 會要求代理程式捨棄優先權最低的訊息。

    如果值為 Throw out oldest 或丟出優先權最低的訊息，且 [使用停用的訊息佇列] 設為 True，則多出的訊息會移至停用的訊息佇列。

  11. 在 [最大產生器數目] 欄位中，鍵入可以與此實體目標關聯的訊息產生器最大數目。

    到達此限制時，則無法再建立新的產生器。值 -1 表示產生器的數目沒有限制。預設值為 100。

  12. 在 [批次傳送最大訊息計數] 欄位中，鍵入可在單一批次中傳送給用戶的最大訊息數目。

    值 -1 表示訊息的數目沒有限制。預設值為 1000。

  13. 從 [使用停用的訊息佇列] 下拉式清單中，選取 [True]，停用的訊息會置於停用的訊息佇列。

    如果值設為 False，則會捨棄停用的訊息。此選項預設會設為 True。

  14. 從 [啟用驗證 XML 模式] 下拉式清單中，選取 [True]，針對目標啟用 XML 模式驗證。

    如果值設為 True，Message Queue 用戶端執行階段會在將 XML 訊息傳送給代理程式之前，依據指定的 XML 模式定義 (若沒有指定 XSD，則依據 DTD) 嘗試驗證該訊息。如果找不到指定的模式或無法驗證訊息，則不會傳送訊息，並會丟出異常。此選項預設會設為 False。

  15. 在 [XML 模式 URI 清單] 欄位中，鍵入以空格分隔的 XML 模式定義 (XSD) URI 字串清單。

    如果啟用了 XML 模式驗證，URI 會指向用於 XML 模式驗證的一或多個 XSD 的位置。
