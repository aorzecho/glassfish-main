編輯 JMS 實體目標

使用 「編輯 JMS 實體目標」 頁面可修改 JMS 實體目標的設定。

「編輯 JMS 實體目標」 頁面包含下列選項。

名稱

實體目標名稱。此為唯讀欄位。您僅能在建立 JMS 目標資源或實體目標時指定名稱。

類型

實體目標類型。此為唯讀欄位。您僅能在建立 JMS 目標資源或實體目標時指定類型。

最大未使用訊息數目

目標所允許的最大未使用訊息數目。值 -1 表示未使用訊息的數目沒有限制。預設值為 -1。

最大訊息大小

任何單一訊息的最大大小 (以位元組為單位)。值 -1 表示訊息大小沒有限制。預設值為 -1。

最大記憶體總容量

未使用訊息的最大記憶體總容量 (以位元組為單位)。值 -1 表示未使用訊息的記憶體大小沒有限制。預設值為 -1。

臨界值限制運作方式

到達記憶體限制臨界值時代理程式需遵循的運作方式。可用的選項包括：

拒絕最新訊息

僅在訊息持續出現時，拒絕最新的訊息並丟出異常以通知產生訊息的用戶端。此為預設值。

減緩產生器

減慢訊息產生器傳送訊息的速度。

丟出最舊

捨棄最舊的訊息。

丟出最低優先順序的訊息

捨棄優先權最低的訊息。

如果值為「丟出最舊的訊息」或「丟出優先權最低的訊息」，且「使用停用的訊息佇列」設為 True，則多出的訊息會移至停用的訊息佇列。

最大產生器數目

可以與此實體目標關聯的訊息產生器最大數目。到達此限制時，則無法再建立新的產生器。值 -1 表示產生器的數目沒有限制。預設值為 100。

批次傳送最大訊息計數

可在單一批次中傳送給用戶的最大訊息數目。值 -1 表示訊息的數目沒有限制。預設值為 1000。

使用停用的訊息佇列

若啟用此選項，則停用的訊息會置於停用的訊息佇列。若停用此選項，則會捨棄停用的訊息。此選項預設為啟用。

驗證 XML 模式

若啟用此選項，Message Queue 用戶端執行階段會在將 XML 訊息傳送給代理程式之前，依據指定的 XML 模式定義 (若沒有指定 XSD，則依據 DTD) 嘗試驗證該訊息。如果找不到指定的模式或無法驗證訊息，則不會傳送訊息，並會丟出異常。此選項預設為停用。

XML 模式 URI 清單

以空格分隔的 XML 模式定義 (XSD) URI 字串清單。如果啟用了 XML 模式驗證，URI 會指向用於 XML 模式驗證的一或多個 XSD 的位置。
