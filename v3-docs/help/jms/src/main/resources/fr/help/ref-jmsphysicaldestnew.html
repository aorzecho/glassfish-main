Nouvelle destination physique JMS

Utilisez la page Nouvelle destination physique JMS pour créer une nouvelle destination physique JMS.

Vous n’avez généralement pas besoin de créer de destinations physiques, car vous spécifiez un nom de destination physique lorsque vous créez une ressource de destination JMS , et GlassFish Server crée automatiquement la destination physique si nécessaire.

La page Nouvelle destination physique JMS contient les options suivantes.

Nom

Le nom de la destination physique.

Type

Le type de la destination physique. Les choix disponibles sont javax.jms.Topic et javax.jms.Queue.

Nombre maximum de messages non consommés

Le nombre maximal de messages non consommés autorisé pour la destination. Une valeur de -1 signifie que le nombre de messages non consommés est illimité. La valeur par défaut est –1.

Taille maximum d’un message

La taille maximum, en octets, d’un message. Une valeur de –1 fait référence à une taille de message illimitée. La valeur par défaut est –1.

Mémoire totale maximum

La mémoire totale maximale, en octets, pour les messages non consommés. Une valeur de -1 signifie que la taille de la mémoire pour les messages non consommés est illimitée. La valeur par défaut est –1.

Comportement aux limites de seuil

Le comportement que le courtier doit suivre lorsque la limite de mémoire seuil est atteinte. Les choix disponibles sont les suivants :

Reject newest messages

Refusez les messages les plus récents et informez le client producteur à l’aide d’une exception uniquement si le message est persistant. Ceci est la valeur par défaut.

Slow down producers

Ralentissez la vitesse à laquelle les producteurs de messages envoient des messages.

Throw out oldest

Effacez les messages les plus anciens.

Throw out lowest-priority messages

Effacez les messages dont le niveau de priorité est le plus bas.

Si la valeur est Throw out oldest ou Throw out lowest-priority messages et que le paramètre Utiliser la file d’attente des messages bloqués est défini sur la valeur True, les messages en trop sont transférés vers la file d’attente des messages bloqués.

Nombre maximum de producteurs

Le nombre maximal de producteurs de messages qui peuvent être associés à cette destination physique. Lorsque cette limite est atteinte, aucun nouveau producteur ne peut être créé. Une valeur de -1 indique un nombre illimité de producteurs. La valeur par défaut est 100.

Nombre maximum de messages lors de la distribution par lots

Le nombre maximal de messages qui peuvent être transmis aux utilisateurs dans un seul lot. Une valeur de -1 indique un nombre illimité de messages. La valeur par défaut est 1000.

Utiliser la file d’attente des messages bloqués

Si la valeur est définie sur True, les messages bloqués sont placés dans la file d’attente des messages bloqués. Si cette valeur est définie sur False, les messages bloqués sont ignorés. Cette option est définie sur True par défaut.

Valider le schéma XML

Si la valeur est définie sur True, l’exécution du client Message Queue tentera de valider un message XML par rapport aux définitions de schéma XML (ou par rapport au DTD, si aucun XSD n’est spécifié) avant de l’envoyer au courtier. Si le schéma spécifié est introuvable ou le message ne peut pas être validé, le message n’est pas envoyé et une exception est générée. Cette option est définie sur False par défaut.

Liste URI du schéma XML

Une liste séparée par des espaces de chaînes d’URI de définition de schéma XML (XSD). Si la validation de schéma XML est activée, les URI pointent vers l’emplacement d’un ou plusieurs XSDs à utiliser pour la validation de schéma XML.
