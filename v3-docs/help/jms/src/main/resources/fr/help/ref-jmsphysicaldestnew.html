Nouvelle destination physique JMS

La page Nouvelle destination physique JMS permet de créer une destination physique JMS.

Vous n'avez généralement pas besoin de créer de destinations physiques, car vous spécifiez un nom de destination physique lorsque vous créez une ressource de destination JMS, et GlassFish Server crée automatiquement la destination physique si nécessaire.

La page Nouvelle destination physique JMS contient les options suivantes.

Nom

Nom de la destination physique.

Type

Type de la destination physique. Les options disponibles sont javax.jms.Topic et javax.jms.Queue.

Nombre maximal de messages non distribués à un destinataire

Nombre maximal autorisé de messages non distribués pour la destination. La valeur -1 signifie que le nombre de messages non distribués est illimité. La valeur par défaut est -1.

Taille maximale d'un message

Taille maximale, en octets, d'un message. La valeur -1 indique une taille de message illimitée. La valeur par défaut est -1.

Mémoire totale maximale

Mémoire totale maximale, en octets, pour les messages non distribués. La valeur -1 signifie que la taille de la mémoire pour les messages non distribués est illimitée. La valeur par défaut est -1.

Comportement aux limites de seuil

Comportement que le broker doit suivre lorsque le seuil limite de la mémoire est atteint. Les choix disponibles sont les suivants :

Rejeter les derniers messages en date

Rejetez les messages les plus récents et informez-en le client émetteur à l'aide d'une exception, uniquement si le message est persistant. Il s'agit de la valeur par défaut.

Ralentir les émetteurs

Réduisez la fréquence à laquelle les émetteurs de messages envoient des messages.

Rejeter les éléments les plus anciens

Supprimez les messages les plus anciens.

Rejeter les messages dont la priorité est la plus basse

Supprimez les messages dont le niveau de priorité est le plus faible.

Si la valeur est Rejeter les éléments les plus anciens ou Rejeter les messages dont la priorité est la plus basse, et que le paramètre Utiliser la file d'attente des messages bloqués a la valeur True, les messages en trop sont transférés vers la file d'attente des messages bloqués.

Nombre maximal d'émetteurs

Nombre maximal d'émetteurs de messages pouvant être associés à cette destination physique. Lorsque cette limite est atteinte, aucun émetteur supplémentaire ne peut être créé. La valeur -1 indique un nombre illimité d'émetteurs. La valeur par défaut est 100.

Nombre maximal de messages lors de la distribution par batch

Nombre maximal de messages pouvant être transmis aux destinataires en un seul batch. La valeur -1 indique un nombre illimité de messages. La valeur par défaut est 1 000.

Utiliser la file d'attente des messages bloqués

Si la valeur est True, les messages bloqués sont placés dans la file d'attente des messages bloqués. Si cette valeur est False, les messages bloqués sont supprimés. Cette option a la valeur True par défaut.

Valider le schéma XML

Si la valeur est True, le programme client Message Queue tentera de valider un message XML par rapport aux définitions de schéma XML spécifiées (ou par rapport à la DTD si aucun fichier XSD n'est spécifié) avant de l'envoyer au broker. Si le schéma spécifié est introuvable ou que le message ne peut pas être validé, le message n'est pas envoyé et une exception est générée. Cette option a la valeur False par défaut.

Liste des URI du schéma XML

Liste de chaînes d'URI de définition de schéma XML (XSD), séparées par des espaces. Si la validation de schéma XML est activée, les URI pointent vers l'emplacement des fichiers XSD à utiliser pour la validation de schéma XML.
