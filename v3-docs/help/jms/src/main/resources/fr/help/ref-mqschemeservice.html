Syntaxe d’adresse du schéma MQ et du service MQ

Syntaxe d’une adresse de service de messagerie :

scheme://address-syntax

où scheme et syntaxe-adresse sont décrits dans le tableau ci-dessous.

Dans le tableau suivant, les deux premières colonnes indiquent les valeurs du Plan MQ et du Service MQ.

Nom du plan

Service de connexion

Description

Syntaxe de l’adresse

mq

jms et ssljms

Le programme client Message Queue établit une connexion avec le journal de correspondance des points de connexion MQ sur l’hôte et le port spécifiés. Le journal de correspondance des points de connexion envoie la liste des ports de service de connexion établis dynamiquement, puis le programme client Message Queue établit une connexion avec le port hébergeant le service de connexion spécifié.

[hostName][: port][/serviceName ]

Valeurs par défaut : hostName = localhost, port = 7676, serviceName = jms

Les valeurs par défaut ne s’appliquent qu’au service de connexion jms. Pour le service de connexion ssljms, toutes les variables doivent être spécifiées.

Exemple : mq:MyHost:7677/ssljms

mqtcp

jms

Le programme client Message Queue établit une connexion TCP avec l’hôte et le port spécifiés (contournant le journal de correspondance des points de connexion MQ) pour se connecter.

hostName:port /jms

Exemple : mqtcp:localhost:7676/jms

mqssl

ssljms

Le programme client Message Queue établit une connexion SSL sécurisée avec l’hôte et le port spécifiés (contournant le journal de correspondance des points de connexion MQ) pour se connecter.

hostName:port /ssljms

Exemple : mqssl:localhost:7676/ssljms

http

httpjms

Le programme client Message Queue établit une connexion HTTP avec un servlet de tunnel Message Queue à l’URL spécifié. (Le courtier doit être configuré pour accéder au servlet de tunnel HTTP, comme décrit dans le Guide de l’administrateur Message Queue.)

hostName:port /contextRoot/tunnel

Si plusieurs instances de courtier utilisent le même servlet de tunnel, la syntaxe de connexion à une instance de courtier spécifique (plutôt qu’à un courtier sélectionné au hasard) est : http://hostName :port/contextRoot /tunnel?serverName =hostName:serverName

https

httpsjms

Le programme client Message Queue établit une connexion HTTPS sécurisée avec l’URL du servlet de tunnel Message Queue spécifié. (Le courtier doit être configuré pour accéder au servlet de tunnel HTTPS, comme décrit dans le Guide de l’administrateur Message Queue.)

hostName:port /contextRoot/tunnel

Si plusieurs instances de courtier utilisent le même servlet de tunnel, la syntaxe de connexion à une instance de courtier spécifique (plutôt qu’à un courtier sélectionné au hasard) est : http://hostName :port/contextRoot /tunnel?serverName =hostName:serverName
