Syntaxe d'adresse du schéma et du service MQ

La syntaxe complète d'une adresse de service de messagerie se présente comme suit :

scheme://address-syntax

Les éléments scheme et address-syntax sont décrits dans le tableau ci-dessous.

Dans le tableau suivant, les deux premières colonnes indiquent les valeurs du schéma et du service MQ.

Nom du schéma Service de connexion Description Syntaxe de l'adresse
mq jms et ssljms Le programme client Message Queue établit une connexion avec le service portmapper MQ sur l'hôte et le port spécifiés. Le service portmapper renvoie la liste des ports de service de connexion établis de façon dynamique, puis le programme client Message Queue établit une connexion avec le port hébergeant le service de connexion spécifié. [hostName][:port][/serviceName]

Valeurs par défaut : hostName = localhost, port = 7676, serviceName = jms

Les valeurs par défaut ne s'appliquent qu'au service de connexion jms. Pour le service de connexion ssljms, toutes les variables doivent être spécifiées.

Exemple : mq:MyHost:7677/ssljms
mqtcp jms Le programme client Message Queue établit une connexion TCP avec l'hôte et le port spécifiés (ignorant le service portmapper MQ). hostName:port/jms

Exemple : mqtcp:localhost:7676/jms
mqssl ssljms Le programme client Message Queue établit une connexion SSL sécurisée avec l'hôte et le port spécifiés (ignorant le service portmapper MQ). hostName:port/ssljms

Exemple : mqssl:localhost:7676/ssljms
http httpjms Le programme client Message Queue établit une connexion HTTP avec un servlet de tunnel Message Queue à l'URL spécifié. (Le broker doit être configuré pour accéder au servlet de tunnel HTTP, comme décrit dans le manuel Administrator's Guide de Message Queue.) hostName:port/contextRoot/tunnel

Si plusieurs instances de broker utilisent le même servlet de tunnel, la syntaxe de connexion à une instance de broker spécifique (plutôt qu'à un broker sélectionné au hasard) est : http://hostName:port/contextRoot/tunnel?serverName=hostName:instanceName
https httpsjms Le programme client Message Queue établit une connexion HTTPS sécurisée avec l'URL du servlet de tunnel Message Queue spécifié. (Le broker doit être configuré pour accéder au servlet de tunnel HTTPS, comme décrit dans le manuel Administrator's Guide de Message Queue.) hostName:port/contextRoot/tunnel

Si plusieurs instances de broker utilisent le même servlet de tunnel, la syntaxe de connexion à une instance de broker spécifique (plutôt qu'à un broker sélectionné au hasard) est : http://hostName:port/contextRoot/tunnel?serverName=hostName:instanceName
