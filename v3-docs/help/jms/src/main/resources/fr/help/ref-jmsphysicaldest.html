Destinations physiques JMS

Les objets de destination physique JMS (Java Message Service) sont gérés par les brokers Message Queue. La file d'attente nommée mq.sys.dmq est la destination système vers laquelle les messages parvenus à expiration et non livrables sont réacheminés.

La page Destinations physiques JMS permet de configurer les destinations physiques Message Queue.

Pour chaque destination physique, les informations suivantes sont fournies.

Nom

Nom de la destination physique.

Type

Type de la destination physique.

Statistiques

Statistiques relatives à la destination physique.

La table Destinations contient également les options suivantes.

Nouveau

Bouton permettant de créer une destination physique.

Supprimer

Bouton permettant de supprimer les destinations physiques sélectionnées.

Supprimer tous les messages

Bouton permettant d'enlever les messages des destinations physiques sélectionnées.
