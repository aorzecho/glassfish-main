Créer une ressource de destination JMS

Lorsque vous créez une ressource de destination JMS, GlassFish Server crée une ressource d’objet d’administration pour la ressource de destination. Vous pouvez visualiser cette ressource sous le noeud Connecteurs.

  1. Dans l’arborescence de navigation, développez le nœud Ressources.
  2. Sous le noeud Connecteurs, développez le noeud Ressources JMS.
  3. Sous le noeud Ressources JMS, sélectionnez le noeud Ressources de destination JMS.

    La page Ressources de destination JMS s’ouvre.

  4. Sur la page Ressources destination JMS, cliquez sur Nouveau.

    La page Nouvelle ressource de destination JMS s’ouvre.

  5. Dans le champ Nom JNDI, saisissez le nom JNDI de la ressource.

    Il est recommandé d’utiliser le préfixe jms/ pour les ressources JMS. Par exemple : jms/Queue.

  6. Dans le champ Nom de la destination physique, saisissez le nom de la destination physique à associer à cette ressource.

    GlassFish Server crée la destination physique automatiquement lorsque c’est nécessaire et la supprime lorsque vous supprimez la ressource de destination.

  7. Dans la liste déroulante Type de ressource, sélectionnez javax.jms.Topic ou javax.jms.Queue.
  8. Dans le champ Description, saisissez une description de la ressource de destination.
  9. Sélectionnez la case Activé de Statut pour activer la ressource au moment de l’exécution.

    Cette option est activée par défaut.

  10. (Facultatif) Dans la zone Autres propriétés, spécifiez les autres propriétés pour la ressource de destination.

    GlassFish Server ne définit pas d’autres propriétés pour les ressources de destination JMS.

    Pour ajouter une propriété, cliquez sur le bouton Ajouter propriété. Sur la ligne vide qui apparaît, saisissez le nom de la propriété dans le champ Nom et la valeur de la propriété dans le champ Valeur.

  11. Cliquez sur OK.
