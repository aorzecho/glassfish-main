Service JMS

Utilisez la page Service JMS pour configurer les paramètres à utiliser par toutes les connexions JMS.

Pour de plus amples informations sur le service JMS, reportez-vous au Chapitre 17, Using the Java Message Service du Oracle GlassFish Server 3.1 Application Development Guide.

La page Service JMS contient les options suivantes.

Charger les valeurs par défaut

Bouton pour restaurer les valeurs par défaut de tous les paramètres.

Ping

Bouton pour vérifier que le service JMS est en cours d’exécution sur l’hôte JMS par défaut. Si le service JMS est activé et en cours d’exécution, un message signale que la commande ping a réussi.

Nom de configuration

Le nom de la configuration à laquelle les paramètres de cette page s'appliquent. Ce champ est en lecture seule.

Type

Le type d’accès pour le service JMS. Les choix disponibles sont les suivants :

EMBEDDED

Accédez au service JMS sur l’hôte local. Le service JMS est démarré sur la même machine JVM que GlassFish Server. L’initialisation paresseuse démarre le courtier imbriqué par défaut sur le premier accès des services JMS et non au démarrage de GlassFish Server. C’est le type de service JMS par défaut.

LOCAL

Accédez au service JMS sur l’hôte local. Le fournisseur JMS est démarré en même temps que GlassFish Server.

REMOTE

Accédez au service JMS sur un autre système. Si vous choisissez REMOTE, le service JMS n’est pas démarré par GlassFish Server au prochain démarrage du serveur. Dans le cas contraire, le service JMS est démarré et géré par Message Queue, vous devez donc démarrer le courtier Message Queue séparément. Pour de plus amples informations sur le démarrage du courtier, reportez-vous à la section Starting Brokers du Oracle GlassFish Message Queue 4.5 Administration Guide. Si vous choisissez cette valeur et si vous utilisez un hôte distant, suivez les instructions dans Modification d'un hôte JMS pour spécifier le nom de l’hôte distant.

Délai d’attente de démarrage

Le nombre de secondes que GlassFish Server patiente pour que le service JMS tente de démarrer avant d’abandonner. Sur un système lent ou surchargé, accroissez la valeur par défaut. La valeur par défaut est 60.

Arguments de démarrage

Arguments pour personnaliser le démarrage du service JMS. Utilisez tous les arguments disponibles via la commande as-install-parent/mq/bin/imqbrokerd.

Reconnecter Activé

Si la case Activé de Reconnecter est sélectionnée, le service JMS Cette option est activée par défaut.

Intervalle de reconnexion

Le nombre de secondes entre chaque tentative de reconnexion. Cet intervalle s’applique aux tentatives sur chaque adresse dans AddressList, ainsi qu’aux adressses successives dans la liste. Si l’intervalle est trop court , le courtier n’aura pas le temps de récupérer. S’il est trop long, le délai de reconnexion peut être inacceptable. La valeur par défaut est 5 secondes.

Tentatives de reconnexion

Le nombre de tentatives de connexion (ou de reconnexion) pour chaque adresse dans AddressList avant que l’exécution du client tente l’adresse suivante dans la liste. La valeur -1 indique que le nombre de tentatives de reconnexion est illimité (le programme client tentera de se connecter à la première adresse jusqu’à ce qu’il y arrive). La valeur par défaut est 3.

Hôte JMS par défaut

Le nom de l’hôte JMS par défaut. La valeur par défaut est default_JMS_host.

Comportement de la liste d’adresses

L’ordre des tentatives de connexion. Les choix disponibles sont les suivants :

random

Sélectionnez une adresse au hasard dans AddressList. Si un grand nombre de clients tente de se connecter en utilisant la même fabrique de connexion, choisissez random pour éviter qu’ils ne soient tous connectés à la même adresse. Cette option est l’option par défaut.

priority

Le module de reconnexion tente toujours de se connecter à la première adresse de serveur dans AddressList et en utilise une autre uniquement si le premier courtier n’est pas disponible.

Itérations de la liste d’adresses

Le nombre de fois que le service JMS parcourt AddressList afin d’établir (ou de réablir) une connexion. La valeur -1 indique que le nombre de tentatives est illimité. La valeur par défaut est 3.

Schéma MQ et Service MQ

Le nom du schéma d’adresses Message Queue et du service de connexion Message Queue si un schéma ou un service non défini par défaut doit être utilisé. Reportez-vous à Syntaxe d'adresse du schéma MQ et du service MQ pour plus d’informations sur la syntaxe.

Autres propriétés

Autres propriétés pour le service JMS.

Pour plus d’informations sur les propriétés disponibles, reportez-vous à Propriétés spécifiques au service JMS.
