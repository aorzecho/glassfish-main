Nouvelle ressource de destination JMS

Utilisez la page Nouvelle ressource de destination JMS pour créer une nouvelle ressource de destination JMS .

La page Nouvelle ressource de destination JMS contient les options suivantes.

Nom JNDI

Un nom unique qui identifie la ressource de destination.

Il est recommandé d’utiliser le préfixe jms/ pour les ressources JMS. Par exemple : jms/Queue.

Nom de la destination physique

Le nom de la destination physique à associer à cette ressource.

GlassFish Server crée la destination physique automatiquement lorsque c’est nécessaire et la supprime lorsque vous supprimez la ressource de destination.

Type de ressource

Le type de la ressource de destination. Les choix disponibles sont javax.jms.Topic et javax.jms.Queue.

Description

Une description de la ressource de destination.

Statut

Le statut de la ressource de destination. La ressource de destination peut être activée ou désactivée. Cette option est activée par défaut.

Autres propriétés

Autres propriétés pour la ressource de destination. GlassFish Server ne définit aucune autre propriété pour les ressources de destination JMS.

Cibles

Clusters et instances autonomes de la ressource de destination. Déplacez les cibles voulues dans la colonne Cibles sélectionnées à l'aide des boutons Ajouter et Tout Ajouter. Déplacez toutes les cibles inutiles dans la colonne Cibles disponibles à l'aide des boutons Supprimer et Tout Supprimer.

Cette option s'affiche uniquement si les clusters ou les instances autonomes ont été créés dans le domaine.
