Propriétés spécifiques aux usines de connexion JMS

Les propriétés supplémentaires suivantes sont disponibles pour une usine de connexion JMS.

ClientId

Spécifie un ID client si la fabrique de connexion est destinée à être utilisée par un abonné de manière durable.

AddressList

Indique les noms (et parfois les numéros de port) de la ou des instances d’un courtier de message avec lesquelles les applications vont communiquer. Chaque adresse de la liste indique le nom d’hôte (et parfois le port de l’hôte et le service de connexion) pour la connexion. Par exemple, la valeur peut être earth ou earth:7677. Spécifiez le numéro de port si le courtier de message utilise un autre port que le port par défaut (7676). Si les paramètres de propriété spécifient plusieurs hôtes et plusieurs ports dans un environnement clustérisé, le premier hôte disponible dans la liste est utilisé, à moins que la valeur de la propriété AddressListBehavior ne soit RANDOM.

Pour plus d’informations, reportez-vous au Guide du développeur Message Queue pour clients Java.

La valeur par défaut est l’hôte local et numéro de port par défaut (7676). Le client tentera de se connecter à un courtier sur le port 7676 de l’hôte local.

UserName

Nom d’utilisateur pour la fabrique de connexion.

La valeur par défaut est guest.

Password

Mot de passe pour la fabrique de connexion.

La valeur par défaut est guest.

ReconnectEnabled

Si elle est définie sur true, cela spécifie que l’exécution du client tente de se reconnecter à un serveur de messages (ou la liste des adresses dans AddressList) lorsqu’une connexion est perdue.

La valeur par défaut est true.

ReconnectAttempts

Définit le nombre de tentatives de connexion (ou de reconnexion) pour chaque adresse dans AddressList avant que le programme client ne passe à l’adresse suivante dans la liste. La valeur -1 indique que le nombre de tentatives de reconnexion est illimité (le programme client tentera de se connecter à la première adresse jusqu’à ce qu’il y arrive).

La valeur par défaut est 3.

ReconnectInterval

Spécifie l’intervalle en millisecondes entre les tentatives de reconnexion. Cela s’applique aux tentatives sur chaque adresse dans AddressList ainsi qu’aux adresses successives dans la liste. Si l’intervalle est trop court, le courtier n’aura pas le temps de récupérer. S’il est trop long, le délai de reconnexion peut être inacceptable.

La valeur par défaut est 30 000.

AddressListBehavior

Indique si les tentatives de connexion doivent suivre l’ordre des adresses dans l’attribut AddressList (PRIORITY) ou un ordre aléatoire (RANDOM).

RANDOM signifie que le processus de reconnexion choisit une adresse au hasard dans AddressList. Si un grand nombre de clients est susceptible de se connecter en utilisant la même fabrique de connexion, cette valeur les empêche d’être tous connectés à la même adresse.

PRIORITY signifie que le processus de reconnexion tentera toujours de se connecter à la première adresse de serveur dans AddressList et n’utilisera une autre adresse que si le premier courtier n’est pas disponible.

La valeur par défaut est RANDOM.

AddressListIterations

Indique le nombre d’itérations du programme client dans AddressList en vue d’établir (ou de rétablir) une connexion. La valeur -1 indique que le nombre de tentatives est illimité.

La valeur par défaut est 3.
