Créer un hôte JMS

  1. Dans l'arborescence de navigation, développez le noeud Configuration.

  2. Select the instance to configure:

    • To configure a particular instance, select the instance's config node. For example, for the default instance, server, select the server-config node.

    • To configure the default settings for future instances that use a copy of default-config, select the default-config node.

  3. Sous le noeud Configuration, développez le noeud Java Message Service.

  4. Sous le noeud Java Message Service, sélectionnez le noeud Hôtes JMS.

    La page Hôtes JMS apparaît.

  5. Sur la page Hôtes JMS, cliquez sur Nouveau.

    La page Nouvel hôte JMS apparaît.

  6. Dans le champ Nom, saisissez le nom de l'hôte.

    Exemple :

    NewJmsHost

  7. Dans le champ Hôte, saisissez le nom ou l'adresse IP du système sur lequel l'hôte JMS sera exécuté (localhost ou le nom du système local ou distant).

  8. Dans le champ Port, indiquez le numéro de port du service JMS.

    Ne modifiez ce champ que si le service JMS à utiliser n'est pas exécuté sur le port par défaut. Le port par défaut est 7676.

  9. Dans les champs Nom utilisateur de l'administrateur et Mot de passe de l'administrateur, saisissez le nom utilisateur et le mot de passe du broker Message Queue.

    Ceux-ci diffèrent du nom utilisateur et du mot de passe GlassFish Server. Ne modifiez ces champs que si les valeurs du broker Message Queue ont été modifiées à l'aide de la commande as-install-parent/mq/bin/imqusermgr. Les valeurs par défaut sont admin et admin.

  10. Cliquez sur OK.
