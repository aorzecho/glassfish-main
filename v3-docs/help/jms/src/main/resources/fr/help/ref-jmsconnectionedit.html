Modifier l’usine de connexion JMS

Utilisez la page Modifier l’usine de connexion JMS pour modifier les paramètres d’une usine de connexion JMS.

La page Modifier l’usine de connexion JMS contient les options suivantes.

Restaurer les valeurs par défaut

Bouton de restauration des valeurs par défaut des paramètres qui ont des valeurs par défaut. Les paramètres qui n'ont pas de valeurs par défaut ne sont pas modifiés.

Nom du pool

Nom unique du pool de connexions de connecteur créé pour la fabrique de connexion. Ce nom est également le nom JNDI de la fabrique de connexion. Le nom du pool est un champ en lecture seule.

Type de ressource

Le type de l’usine de connexion. Le type de ressource est un champ en lecture seule. Vous pouvez spécifier le type de ressource uniquement lorsque vous créez une nouvelle usine de connexion JMS.

Description

Une description de l’usine de connexion.

Statut

Le statut de l’usine de connexion. L’usine de connexion peut être activée ou désactivée. Cette option est activée par défaut.

Taille initiale et minimale du pool

Nombre minimum de connexions dans le pool pour l’usine de connexion. La valeur par défaut est 8.

Taille maximale du pool

Nombre maximum de connexions dans le pool. La valeur par défaut est 32.

Valeur de redimensionnement du pool

Nombre de connexions à supprimer lorsque le délai d’inactivité du pool expire. La valeur par défaut est 2.

Délai d’inactivité

Temps maximum en secondes durant lequel une connexion demeure inactive dans le pool. La valeur par défaut est 300.

Attente max.

Durée pendant laquelle l’application demandant une connexion doit patienter avant d’être notifiée de l’expiration de la connexion. La valeur par défaut est 60 000.

Sur tout échec

Si la case Fermer toutes les connexions est cochée, GlassFish Server interrompt toutes les connexions du pool et les rétablit si une seule connexion échoue. Cette option est désactivée par défaut.

Prise en charge des transactions

La valeur peut être l’une des suivantes :

XATransaction

La ressource peut être utilisée pour des transactions impliquant l’utilisation de plusieurs ressources dans une étendue de transaction. Il s’agit de la valeur par défaut pour une usine de connexion JMS.

Par exemple, les transactions peuvent impliquer cette ressource ainsi qu’une ressource JDBC, une ressource de connecteur ou une autre ressource d’usine de connexion JMS. Cette valeur offre un maximum de flexibilité. Une ressource qui est configurée comme XATransaction participera aux opérations de validation à deux phases.

LocalTransaction

La ressource peut être utilisée soit pour les transactions qui impliquent uniquement une ressource dans l’étendue de la transaction soit en tant que dernier agent dans une transaction distribuée qui implique plus d’une ressource XA.

Cette valeur permet de bien meilleures performances. Une ressource qui est configurée en tant que LocalTransaction ne sera pas utilisée dans les opérations de validation à deux phases.

NoTransaction

La ressource ne peut participer à aucune transaction. Dans les applications JMS, l’utilisation de cette configuration est limitée.

Validation des connexions

Si la case Requis est sélectionnée, les connexions sont validées avant d’être données à l’application. Si la validation d’une ressource échoue, elle est détruite et une nouvelle ressource est créée et renvoyée. Cette option est désactivée par défaut.

Autres propriétés

Autres propriétés pour l’usine de connexion.

Pour plus d’informations sur les propriétés disponibles, reportez-vous à la rubrique Propriétés spécifiques aux usines de connexion JMS.
