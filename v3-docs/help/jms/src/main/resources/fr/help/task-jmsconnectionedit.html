Modifier une ressource d’usine de connexion JMS

  1. Dans l’arborescence de navigation, développez le nœud Ressources.
  2. Sous le noeud Connecteurs, développez le noeud Ressources JMS.
  3. Sous le noeud Ressources JMS, sélectionnez le noeud Usines de connexion JMS.

    La page Usines de connexion JMS s’ouvre.

  4. Sur la page Usines de connexion JMS, cliquez sur le nom de l’usine de connexion que vous modifiez.

    La page Modifier l’usine de connexion JMS s’ouvre.

  5. Dans le champ Description, saisissez une description de l’usine de connexion.
  6. Sélectionnez la case Activé de Statut pour activer la ressource au moment de l’exécution.

    Cette option est activée par défaut.

  7. Dans le champ Taille initiale et minimale du pool, spécifiez le nombre minimum de connexions dans le pool.

    La valeur par défaut est 8.

  8. Dans le champ Taille maximale du pool, spécifiez le nombre maximum de connexions dans le pool.

    La valeur par défaut est 32.

  9. Dans le champ Valeur de redimensionnement du pool, spécifiez le nombre de connexions à supprimer lorsque le délai d’inactivité du pool expire.

    La valeur par défaut est 2.

  10. Dans le champ Délai d’inactivité, spécifiez le temps maximum (en secondes) pendant lequel une connexion peut rester inactive dans le pool.

    La valeur par défaut est 300.

  11. Dans le champ Attente max., spécifiez le temps pendant lequel l’application demandant une connexion patientera avant d’obtenir une expiration de la connexion.

    La valeur par défaut est 60 000.

  12. Si la case Échec sur tout Fermer toutes les connexions est cochée, GlassFish Server interrompt toutes les connexions du pool et les rétablit si une seule connexion échoue.

    Si vous ne sélectionnez pas cette case, les connexions individuelles ne seront rétablies que lorsqu’elles seront utilisées.

  13. Pour une ressource d’usine de connexion JMS, spécifiez la valeur de Prise en charge des transactions comme suit :
    • Choisissez XATransaction (valeur par défaut) si la ressource peut être utilisée pour des transactions impliquant l’utilisation de plusieurs ressources dans le cadre d’une transaction.

      Par exemple, cette ressource plus une ressource JDBC, une ressource de connecteur ou une autre ressource de fabrique de connexion JMS. Cette valeur offre un maximum de flexibilité. Les ressources configurées XATransaction participent à des opérations de validation à deux phases.

    • Choisissez LocalTransaction si la ressource peut être utilisée soit pour des transactions impliquant une seule ressource dans le cadre de la transaction, soit comme dernier agent dans une transaction distribuée impliquant plusieurs ressources XA.

      Cette valeur permet de bien meilleures performances. Les ressources configurées LocalTransaction ne sont pas utilisées dans les opérations de validation à deux phases.

    • Choisissez NoTransaction si la ressource ne peut participer à aucune transaction.

      Dans les applications JMS, l’utilisation de cette configuration est limitée.

  14. Sélectionnez la case Validation de la connexion pour activer la validation de la connexion.

    Cette option est désactivée par défaut.

  15. (Facultatif) Dans le tableau Autres propriétés, spécifiez les autres propriétés pour l’usine de connexion.

    Pour plus d’informations sur les propriétés disponibles, reportez-vous à la rubrique Propriétés spécifiques aux usines de connexion JMS.

    • Pour ajouter une propriété, cliquez sur le bouton Ajouter propriété. Sur la ligne vide qui apparaît, saisissez le nom de la propriété dans le champ Nom et la valeur de la propriété dans le champ Valeur.
    • Pour modifier une propriété, modifiez le champ Valeur de cette propriété.
    • Pour supprimer une propriété, cochez la case située à gauche du champ Nom de la propriété que vous supprimez, puis cliquez sur le bouton Supprimer propriétés.
  16. Cliquez sur Enregistrer.

    Astuce - Pour revenir aux paramètres par défaut, cliquez sur le bouton Charger valeurs par défaut, puis sur Enregistrer.
