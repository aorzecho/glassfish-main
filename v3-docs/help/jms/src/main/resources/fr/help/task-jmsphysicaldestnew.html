Création d’une destination physique JMS

Vous n’avez généralement pas besoin de créer de destinations physiques, car vous spécifiez un nom de destination physique lorsque vous créez une ressource de destination JMS , et GlassFish Server crée automatiquement la destination physique si nécessaire.

  1. Dans l’arborescence de navigation, développez le nœud Configuration.
  2. Sous le nœud Configuration, développez le nœud Java Message Service.
  3. Sous le noeud Java Message Service, sélectionnez le noeud Destinations physiques.

    La page Destinations physiques s’ouvre.

  4. Sur la page Destinations physiques, cliquez sur Nouveau.

    La page Nouvelle destination physique s’ouvre.

  5. Dans le champ Nom, saisissez le nom de la destination physique.
  6. Dans la liste déroulante Type, sélectionnez javax.jms.Topic ou javax.jms.Queue.
  7. Dans le champ Nombre maximal de messages non consommés, saisissez le nombre maximal de messages non consommés autorisés pour la destination.

    Une valeur de -1 signifie que le nombre de messages non consommés est illimité. La valeur par défaut est –1.

  8. Dans le champ Taille maximale d’un message, saisissez la taille maximale, en octets, d’un message.

    Une valeur de –1 signifie que la taille des messages est illimitée. La valeur par défaut est –1.

  9. Dans le champ Mémoire totale maximale, saisissez la mémoire totale maximale, en octets, des messages non consommés.

    Une valeur de -1 signifie que la taille de la mémoire pour les messages non consommés est illimitée. La valeur par défaut est –1.

  10. Dans la liste déroulante Comportement de limite de seuil, sélectionnez le comportement que le courtier doit suivre lorsque la limite de mémoire est atteinte.
    • Reject newest messages indique au courtier de refuser les messages les plus récents et de notifier le client producteur à l’aide d’une exception uniquement si le message est persistant.

      Reject newest messages est la valeur par défaut.

    • Slow down producers indique au courtier de ralentir la vitesse à laquelle les producteurs de messages envoient des messages.
    • Throw out oldest indique au courtier d’éliminer les messages les plus anciens.
    • Throw out lowest-priority messages indique au courtier d’éliminer les messages dont le niveau de priorité est le plus bas.

    Si la valeur est Throw out oldest ou Throw out lowest-priority messages et que le paramètre Utiliser la file d’attente des messages bloqués est défini sur la valeur True, les messages en trop sont transférés vers la file d’attente des messages bloqués.

  11. Dans le champ Nombre maximal de producteurs, saisissez le nombre maximal de producteurs de messages qui peuvent être associés à cette destination physique.

    Lorsque cette limite est atteinte, aucun nouveau producteur ne peut être créé. Une valeur de -1 indique un nombre illimité de producteurs. La valeur par défaut est 100.

  12. Dans le champ Nombre maximal de messages lors de la distribution par lots, saisissez le nombre maximal de messages qui peuvent être transmis aux consommateurs en un seul lot.

    Une valeur de -1 indique un nombre illimité de messages. La valeur par défaut est 1000.

  13. Dans la liste déroulante Utiliser la file d’attente des messages bloqués, sélectionnez True pour placer les messages bloqués dans la file d’attente des messages bloqués.

    Si cette valeur est définie sur False, les messages bloqués sont ignorés. Cette option est définie sur True par défaut.

  14. Dans la liste déroulante Validation de schéma XML, sélectionnez True pour activer la validation de schéma XML pour la destination.

    Si la valeur est définie sur True, l’exécution du client Message Queue tentera de valider un message XML par rapport aux définitions de schéma XML (ou par rapport au DTD, si aucun XSD n’est spécifié) avant de l’envoyer au courtier. Si le schéma spécifié est introuvable ou le message ne peut pas être validé, le message n’est pas envoyé et une exception est générée. Cette option est définie sur False par défaut.

  15. Dans le champ Liste URI du schéma XML, tapez une liste séparée par des espaces de chaînes d’URI de définitions de schéma XML (XSD).

    Si la validation de schéma XML est activée, les URI pointent vers l’emplacement d’un ou plusieurs XSDs à utiliser pour la validation de schéma XML.
