Disponibilité JMS

Utilisez la page Disponibilité JMS pour configurer les paramètres de disponibilité du service JMS (Java Message Service).

La page Disponibilité JMS contient les options suivantes.

Restaurer les valeurs par défaut

Bouton de restauration des valeurs par défaut des paramètres qui ont des valeurs par défaut. Les paramètres qui n'ont pas de valeurs par défaut ne sont pas modifiés.

Nom de la configuration

Nom de la configuration à laquelle les paramètres de cette page s'appliquent. Ce champ est en lecture seule.

Service de disponibilité

Type de cluster Message Queue fournissant des services JMS à un cluster GlassFish. Si la case Activé est cochée, un cluster Message Queue amélioré est utilisé. Dans le cas contraire, un cluster Message Queue conventionnel est utilisé.

Si la case Activé est cochée, les options Type de magasin de configurations et Type de mémoire de messages ne sont pas prises en compte.

Type de magasin de configurations

Type de magasin de données pour les données de configuration dans un cluster Message Queue conventionnel. La valeur masterbroker indique l'utilisation d'un courtier principal permettant de stocker et de gérer les données de configuration. La valeur shareddb indique l'utilisation d'une base de données partagée permettant de stocker les données de configuration.

Cette option est ignorée si Service de disponibilité est activé.

Type de mémoire de messages

Type de magasin de données pour des données de message de courtiers d'un cluster conventionnel. La valeur file indique un magasin de fichiers. La valeur jdbc indique un magasin JDBC.

Cette option est ignorée si Service de disponibilité est activé.

Fournisseur de bases de données
Nom d'utilisateur de base de données
Mot de passe de la base de données
URL d'accès de base de données

Le fournisseur, l'utilisateur, le mot de passe et l'URL d'accès de la base de données JDBC à utiliser dans les situations suivantes :

  • Lorsque le service de disponibilité est activé.

  • Lorsque Type de magasin de configurations est défini sur shareddb.

  • Lorsque Type de mémoire de messages est défini sur jdbc.

Pour de plus amples informations sur les fournisseurs pris en charge et sur les formats des URL d'accès de chaque fournisseur, reportez-vous à la section JDBC-Based Persistence du Oracle GlassFish Server Message Queue 4.5 Administration Guide.

Autres propriétés

Propriétés supplémentaires de disponibilité JMS.

GlassFish Server ne prédéfinit aucune autre propriété de disponibilité JMS.
