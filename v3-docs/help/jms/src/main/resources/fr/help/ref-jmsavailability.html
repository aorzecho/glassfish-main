Disponibilité JMS

La page Disponibilité JMS permet de configurer les paramètres de disponibilité JMS (Java Message Service).

Elle contient les options suivantes.

Charger les valeurs par défaut

Bouton de restauration des valeurs par défaut des paramètres qui ont des valeurs par défaut. Les paramètres qui n'ont pas de valeurs par défaut ne sont pas modifiés.

Nom de configuration

Nom de la configuration à laquelle s'appliquent les paramètres de cette page. Ce champ est en lecture seule.

Service de disponibilité

Type de cluster Message Queue permettant de fournir des services JMS à un cluster GlassFish. Si la case Activé est cochée, un cluster Message Queue amélioré est utilisé. Sinon, un cluster Message Queue classique est employé.

Si la case Activé est cochée, les options Type d'emplacement de stockage de la configuration et Type d'emplacement de stockage des messages sont ignorées.

Type d'emplacement de stockage de la configuration

Type de banque de données pour les données de configuration dans un cluster Message Queue conventionnel. La valeur masterbroker indique l'utilisation d'un broker principal permettant de stocker et de gérer les données de configuration. La valeur shareddb indique l'utilisation d'une base de données partagée permettant de stocker les données de configuration.

Cette option est ignorée si l'option Service de disponibilité est activée.

Type d'emplacement de stockage des messages

Type de banque de données pour les données de message des brokers dans un cluster conventionnel. La valeur file indique un emplacement de stockage de fichiers. La valeur jdbc indique un emplacement de stockage JDBC.

Cette option est ignorée si l'option Service de disponibilité est activée.

Fournisseur de base de données
Nom utilisateur de la base de données
Mot de passe de la base de données
URL d'accès à la base de données

Fournisseur de base de données, utilisateur, mot de passe et URL d'accès de la base de données JDBC à utiliser dans les situations suivantes :

  • L'option Service de disponibilité est activée.

  • L'option Type d'emplacement de stockage de la configuration a la valeur shareddb.

  • L'option Type d'emplacement de stockage des messages a la valeur jdbc.

Pour plus d'informations sur les fournisseurs pris en charge et sur les formats des URL d'accès de chaque fournisseur, reportez-vous à JDBC-Based Persistence dans le manuel Oracle GlassFish Server Message Queue Administration Guide.

Autres propriétés

Propriétés supplémentaires de la disponibilité JMS.

GlassFish Server ne prédéfinit aucune autre propriété de disponibilité JMS.
