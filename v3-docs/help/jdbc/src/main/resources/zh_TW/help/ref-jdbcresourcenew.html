新建 JDBC 資源

使用 「新增 JDBC 資源」 頁面可建立新的 JDBC 資源。

「新增 JDBC 資源」 頁面包含下列選項。

JNDI 名稱

可識別 JDBC 資源的唯一名稱。JNDI 名稱可用來為分散式運算環境中的元件加以組織與定位，就如同圖書館裡的卡片目錄，可用來將書籍分門別類並表示擺放位置。因此，JNDI 名稱便成為存取 JDBC 資源的重要方法。依慣例會是以 jdbc/ 字串開始的名稱，例如 jdbc/payrolldb。請勿遺漏正斜線。名稱必須僅包含字母數字、底線、破折號或點字元。

集區名稱

與 JDBC 資源相關聯的連線集區。

說明

JDBC 資源的簡短說明。

狀態

若啟用此選項，則資源可在執行階段使用。此選項預設為啟用。

其他特性

JDBC 資源的其他特性。GlassFish Server 未定義 JDBC 資源的任何其他特性。

目標

JDBC 資源的叢集與獨立實例。可使用 「加入」 與 「全部加入」 按鈕，將所需的目標移至 「選取的目標」 欄。可使用 「移除」 與 「全部移除」 按鈕，將任何不需要的目標移至 「可用的目標」 欄。

唯有網域中已建立叢集或獨立實例時，才會顯示此選項。
