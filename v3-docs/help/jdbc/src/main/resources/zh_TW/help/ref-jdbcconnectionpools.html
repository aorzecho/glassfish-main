連線集區

JDBC 連線集區包含一組在註冊該連線集區時建立的 JDBC 連線。連線集區使用 JDBC 驅動程式來建立實體資料庫連線。您的應用程式可以向連線集區借用連線、使用該連線，然後關閉該連線以便還給連線集區。若要建立 JDBC 資源，請指定關聯的連線集區。多個 JDBC 資源可指定單一連線集區。

JDBC 連線集區是針對特定資料庫的一組可重複使用的連線。由於建立每個新的實體連線都會耗時許久，因此伺服器會維護一組可用連線集區以提高效能。當應用程式請求連線時，它可以從集區中取得一個連線。應用程式關閉連線時，連線將傳回集區中。

使用管理主控台建立連線集區時，即表示您在定義特定資料庫連線的各項設定。

使用 「JDBC 連線集區」 頁面可組態 JDBC 連線集區。

對於每個連線集區，均提供以下資訊。

集區名稱

連線集區的名稱。

資源類型

連線集區的資源類型。

類別名稱

實作 javax.sql.XADataSource、java.sql.ConnectionPoolDataSource、javax.sql.DataSource 或 java.sql.Driver API 的供應商專屬類別名稱。

說明

連線集區的簡短說明。

連線集區表也包含下列選項。

新建

建立新連線集區的按鈕。

刪除

刪除一或多個選取連線集區的按鈕。
