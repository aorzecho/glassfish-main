連線池

JDBC 連線池包含一組在註冊該連線池時建立的 JDBC 連線。連線池使用 JDBC 驅動程式來建立實體資料庫連線。您的應用程式可以向連線池借連線、使用該連線，然後關閉該連線以便還給連線池。若要建立 JDBC 資源，請指定關聯的連線池。多個 JDBC 資源可指定單一連線池。

JDBC 連線池是針對特定資料庫的一組可重複使用的連線。由於建立每個新的實體連線都會耗時許久，因此伺服器會維護一組可用連線池以提高效能。當應用程式請求連線時，它可以從池中取得一個連線。應用程式關閉連線時，連線將傳回池中。

使用管理主控台建立連線池時，即表示您在定義特定資料庫連線的各項設定。

使用 [JDBC 連線池] 頁面可配置 JDBC 連線池。

對於每個連線池，均提供以下資訊。

池名稱

連線池的名稱。

資源類型

連線池的資源類型。

類別名稱

實作 javax.sql.XADataSource、java.sql.ConnectionPoolDataSource、javax.sql.DataSource 或 java.sql.Driver API 的供應商專屬類別名稱。

說明

連線池的簡短說明。

連線池表也包含下列選項。

新建

建立新連線池的按鈕。

刪除

刪除一或多個選取連線池的按鈕。
