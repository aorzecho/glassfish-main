Pools de connexions

Un pool de connexions JDBC contient un groupe de connexions JDBC créées lors de son inscription. Les pools de connexions utilisent un pilote JDBC pour créer des connexions physiques à la base de données. L'application emprunte une connexion au pool, l'utilise, puis la renvoie vers ce dernier en la fermant. Pour créer une ressource JDBC, indiquez le pool de connexions associé. Plusieurs ressources JDBC peuvent spécifier un seul pool de connexions.

Un pool de connexions JDBC est un groupe de connexions réutilisables pour une base de données particulière. La création de chaque connexion physique exigeant du temps, le serveur conserve un pool de connexions disponibles pour accroître les performances. Lorsqu'une application demande une connexion, elle en obtient une du pool. Lorsqu'une application met fin à une connexion, cette dernière est replacée dans le pool.

Lors de la création d'un pool de connexions à l'aide de la Console d'administration , vous définissez les aspects d'une connexion à une base de données spécifique.

La page Pools de connexions JDBC permet de configurer les pools de connexions JDBC.

Pour chaque pool de connexions, les informations suivantes sont fournies.

Nom du pool

Nom du pool de connexions.

Type de ressource

Type de ressource du pool de connexions.

Nom de la classe

Nom de classe propre au fournisseur qui implémente l'API javax.sql.XADataSource, java.sql.ConnectionPoolDataSource, javax.sql.DataSource ou java.sql.Driver.

Description

Brève description du pool de connexions.

La table Pools contient également les options suivantes.

Nouveau

Bouton permettant de créer un pool de connexions.

Supprimer

Bouton permettant de supprimer les pools de connexions sélectionnés.
