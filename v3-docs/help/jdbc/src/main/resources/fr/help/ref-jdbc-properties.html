Propriétés spécifiques aux pools de connexions JDBC

La plupart des pilotes JDBC permettent d’utiliser des listes de propriétés standard pour spécifier l’utilisateur, le mot de passe et d’autres informations de configuration des ressources. Bien que les propriétés soient facultatives par rapport au GlassFish Server, certaines propriétés peuvent être nécessaires pour la plupart des bases de données. Pour plus de détails, reportez-vous à JDBC 4.0 Standard Extension API.

La modification des propriétés de pilote JDBC nécessite un redémarrage du serveur.

Les propriétés standard suivantes sont communes à plusieurs fournisseurs JDBC. Pour obtenir des informations sur les propriétés disponibles pour votre pilote JDBC, consultez la documentation du fournisseur.

user

Spécifie le nom d’utilisateur pour la connexion à la base de données.

password

Spécifie le mot de passe pour la connexion à la base de données.

databaseName

Spécifie la base de données pour ce pool de connexions.

serverName

Spécifie le serveur de base de données pour ce pool de connexions.

port

Spécifie le port sur lequel le serveur de base de données écoute les requêtes.

networkProtocol

Spécifie le protocole de communication.

roleName

Spécifie le nom du rôle SQL initial.

datasourceName

Spécifie un XADataSource sous-jacent ou un ConnectionPoolDataSource si le pool de connexions est effectué.

description

Spécifie une description de texte.

url

Spécifie l’URL pour ce pool de connexions. Bien qu’il ne s’agisse pas d’une propriété standard, elle est couramment utilisée.
