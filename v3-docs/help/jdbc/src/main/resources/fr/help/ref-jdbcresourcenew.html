Nouvelle ressource JDBC

La page Nouvelle ressource JDBC permet de créer une ressource JDBC.

Elle contient les options suivantes.

Nom JNDI

Nom unique identifiant la ressource JDBC. Le nom JNDI organise et localise les composants dans un environnement informatique distribué de la même manière que des catalogues sur fiches organisent et représentent des emplacements de livres dans une bibliothèque. Par conséquent, le nom JNDI est une méthode importante d'accès à la ressource JDBC. Par convention, le nom commence par la chaîne jdbc/, par exemple jdbc/payrolldb. N'oubliez pas la barre oblique. Le nom ne peut contenir que des caractères alphanumériques, des traits de soulignement, des tirets ou des points.

Nom du pool

Pool de connexions associé à la ressource JDBC.

Description

Brève description de la ressource JDBC.

Statut

Si cette option est activée, la ressource est disponible lors de l'exécution. Cette option est activée par défaut.

Autres propriétés

Propriétés supplémentaires de la ressource JDBC. GlassFish Server ne définit aucune autre propriété pour les ressources JDBC.

Cibles

Clusters et instances autonomes de la ressource JDBC. Déplacez les cibles souhaitées dans la colonne Cibles sélectionnées à l'aide des boutons Ajouter et Tout ajouter. Déplacez toutes les cibles inutiles dans la colonne Cibles disponibles à l'aide des boutons Enlever et Tout enlever.

Cette option apparaît uniquement si des clusters ou des instances autonomes ont été créés dans le domaine.
