Nouvelle ressource\~JDBC

La page Nouvelle ressource JDBC vous permet de créer une nouvelle ressource JDBC.

La page Nouvelle ressource JDBC contient les options suivantes :

Nom JNDI

Nom unique identifiant la ressource JDBC. Le nom JNDI organise et recherche des composants dans un environnement informatique distribué de la même manière que des catalogues sur fiches organisent et représentent des emplacements de livres dans une bibliothèque. Par conséquent, le nom JNDI devient une méthode importante d’accès à la ressource JDBC. Par convention, le nom commence par la chaîne jdbc/, comme par exemple dans jdbc/payrolldb. N’oubliez pas la barre oblique. Le nom ne peut contenir que des caractères alphanumériques, des caractères de soulignement, des tirets ou des points.

Nom du pool

Pool de connexions associé à la ressource JDBC.

Description

Brève description de la ressource JDBC.

Statut

Si cette option est activée, la ressource est disponible au moment de l’exécution. Cette option est activée par défaut.

Autres propriétés

Propriétés supplémentaires de la ressource JDBC. GlassFish Server ne définit aucune autre propriété pour les ressources JDBC.

Cibles

Clusters et instances autonomes pour la ressource JDBC. Déplacez les cibles voulues dans la colonne Cibles sélectionnées à l'aide des boutons Ajouter et Tout Ajouter. Déplacez toutes les cibles inutiles dans la colonne Cibles disponibles à l'aide des boutons Supprimer et Tout Supprimer.

Cette option s'affiche uniquement si les clusters ou les instances autonomes ont été créés dans le domaine.
