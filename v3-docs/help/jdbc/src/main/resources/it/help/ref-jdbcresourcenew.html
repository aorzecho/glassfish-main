Nuova risorsa JDBC

Nella pagina Nuova risorsa JDBC è possibile creare una nuova risorsa JDBC.

La pagina Nuova risorsa JDBC contiene le opzioni riportate di seguito.

Nome JNDI

Nome univoco che identifica la risorsa JDBC. Il nome JNDI consente di organizzare e individuare i componenti all'interno di un ambiente di elaborazione distribuito in modo analogo agli schedari in una biblioteca, con cui viene organizzata e rappresentata la posizione dei libri. Di conseguenza, il nome JNDI rappresenta un metodo importante per accedere alla risorsa JDBC. Per convenzione, il nome inizia con la stringa jdbc/, ad esempio jdbc/payrolldb. Ricordare di includere la barra. Il nome può contenere solo caratteri alfanumerici, caratteri di sottolineatura, trattini o punti.

Nome pool

Connection pool associato alla risorsa JDBC.

Descrizione

Breve descrizione della risorsa JDBC.

Stato

Se questa opzione è abilitata, la risorsa è disponibile in fase di esecuzione. Questa opzione è abilitata per impostazione predefinita.

Proprietà aggiuntive

Proprietà aggiuntive per la risorsa JDBC. GlassFish Server non definisce alcuna proprietà aggiuntiva per le risorse JDBC.

Destinazioni

Cluster e istanze standalone per la risorsa JDBC. Per spostare le destinazioni desiderate nella colonna Destinazioni selezionate, utilizzare i pulsanti Aggiungi e Aggiungi tutto. Per spostare le destinazioni non necessarie nella colonna Destinazioni disponibili, utilizzare i pulsanti Rimuovi e Rimuovi tutto.

Questa opzione viene visualizzata solo se nel dominio sono stati creati cluster o istanze standalone.
