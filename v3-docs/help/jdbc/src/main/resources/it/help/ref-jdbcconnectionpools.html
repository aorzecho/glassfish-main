Pool di connessione

Un pool di connessione JDBC contiene un gruppo di connessioni JDBC create al momento della registrazione del pool di connessione. I pool di connessione si servono di un driver JDBC per creare connessioni database fisiche. L'applicazione prende in prestito una connessione dal pool, la utilizza e quindi la chiude restituendola al pool. Per creare una risorsa JDBC, è necessario specificare il pool di connessione associato. È possibile specificare un unico pool di connessione per più risorse JDBC.

Un pool di connessione JDBC è un gruppo di connessioni riutilizzabili per un determinato database. Poiché la creazione di ogni nuova connessione fisica richiede tempo, il server gestisce un pool di connessioni disponibili per incrementare le prestazioni. Quando un'applicazione richiede una connessione, può ottenerne una dal pool. Quando un'applicazione chiude una connessione, la connessione viene restituita al pool.

Quando si crea un pool di connessione con Administration Console, si definiscono gli aspetti di una connessione a un database specifico.

Nella pagina JDBC Connection Pools è possibile configurare i pool di connessione JDBC.

Per ogni pool di connessione vengono fornite le seguenti informazioni.

Pool Name

Nome del pool di connessione.

Resource Type

Tipo di risorsa del pool di connessione.

Classname

Nome della classe specifica del produttore che implementa l'API javax.sql.XADataSource, java.sql.ConnectionPoolDataSource, javax.sql.DataSource o java.sql.Driver.

Description

Breve descrizione del pool di connessione.

La tabella Pools contiene inoltre le seguenti opzioni.

New

Pulsante che consente di creare un nuovo pool di connessione.

Delete

Pulsante che consente di eliminare uno o più pool di connessione selezionati.
