Proprietà specifiche dei connection pool JDBC

La maggior parte dei driver JDBC consente l'utilizzo di liste di proprietà standard per specificare l'utente, la password e altre informazioni di configurazione delle risorse. Sebbene le proprietà siano facoltative per GlassFish Server, alcune risultano necessarie per la maggior parte dei database. Per informazioni dettagliate, vedere l'API JDBC 4.0 Standard Extension.

Per modificare le proprietà del driver JDBC, è necessario riavviare il server.

Le proprietà standard riportate di seguito sono comuni a molti fornitori di driver JDBC. Per informazioni sulle proprietà disponibili per il driver JDBC in uso, consultare la documentazione del fornitore.

user

Specifica il nome utente per la connessione al database.

password

Specifica la password per la connessione al database.

databaseName

Specifica il database per il connection pool corrente.

serverName

Specifica il database server per il connection pool corrente.

port

Specifica la porta sulla quale il database server ascolta le richieste.

networkProtocol

Specifica il protocollo di comunicazione.

roleName

Specifica il nome del ruolo SQL iniziale.

datasourceName

Specifica una classe XADataSource di base oppure una classe ConnectionPoolDataSource se viene eseguito il connection pooling.

description

Specifica una descrizione di testo.

url

Specifica l'URL per il connection pool corrente. Questa proprietà non è standard, tuttavia è utilizzata di frequente.
