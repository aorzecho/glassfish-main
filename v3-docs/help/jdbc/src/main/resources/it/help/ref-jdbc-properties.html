Proprietà specifiche per i pool di connessione JDBC

La maggior parte dei driver JDBC consente l'utilizzo di elenchi delle proprietà standard per specificare l'utente, la password e altre informazioni di configurazione delle risorse. Sebbene le proprietà siano facoltative per GlassFish Server, alcune risultano necessarie per la maggior parte dei database. Per informazioni dettagliate, vedere l'API JDBC 4.0 Standard Extension.

Per modificare le proprietà del driver JDBC, è necessario riavviare il server.

Le seguenti proprietà standard sono comuni a molti produttori JDBC. Per informazioni sulle proprietà disponibili per il driver JDBC in uso, consultare la documentazione del produttore.

user

Specifica il nome utente per la connessione al database.

password

Specifica la password per la connessione al database.

databaseName

Specifica il database per il pool di connessione corrente.

serverName

Specifica il server database per il pool di connessione corrente.

port

Specifica la porta in cui le richieste vengono rilevate dal server database.

networkProtocol

Specifica il protocollo di comunicazione.

roleName

Specifica il nome del ruolo SQL iniziale.

datasourceName

Specifica una classe XADataSource sottostante oppure una classe ConnectionPoolDataSource se viene implementato il pool di connessione.

description

Specifica una descrizione di testo.

url

Specifica l'URL per il pool di connessione corrente. Questa proprietà non è standard, tuttavia è utilizzata di frequente.
