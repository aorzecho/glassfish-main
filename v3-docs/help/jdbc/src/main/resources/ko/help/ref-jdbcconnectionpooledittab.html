접속 풀 편집

[접속 풀 편집] 페이지에서는 JDBC 접속 풀의 일반 속성을 편집할 수 있습니다.

[접속 풀 편집] 페이지에는 다음 옵션이 포함되어 있습니다.

기본값 로드

기본값이 있는 설정을 해당 기본값으로 복원하는 단추입니다. 기본값이 없는 설정은 변경되지 않습니다.

비우기

접속 풀의 접속을 다시 초기화하는 단추입니다. 이 단추를 누르면 기존의 모든 접속이 삭제되고 풀에 설정된 접속이 다시 생성된 후 풀이 초기 및 최소 풀 크기로 복원됩니다.

핑

접속 풀을 사용할 수 있는지 확인하는 단추입니다. 이↓단추를 누를 때 오류 메시지가 나타나면 데이터베이스 서버가 시작되었는지 ↓확인합니다.

풀 이름

접속 풀의 이름입니다. 읽기 전용 필드입니다. JDBC 접속 풀을 생성할 경우 풀 이름만 지정할 수 있습니다.

리소스 유형

선택 항목에는 javax.sql.XADataSource(전역 트랜잭션), java.sql.ConnectionPoolDataSource(로컬 트랜잭션, 성능 향상 가능), javax.sql.DataSource(로컬 트랜잭션만) 및 java.sql.Driver가 있습니다.

데이터 소스 클래스 이름

데이터 소스를 구현하는 공급업체별 클래스 이름입니다.

드라이버 클래스 이름

공급업체에서 제공한 JDBC 드라이버 클래스 이름입니다. java.sql.Driver 리소스 유형을 선택한 경우 일반적으로 이 필드에는 선택한 드라이버 및 데이터베이스 공급업체와 연관된 클래스 이름이 자동으로 입력됩니다. 데이터 소스 리소스 유형 중 하나를 선택한 경우 이 필드는 사용 안함으로 설정됩니다.

핑

이 옵션이 선택된 경우, 풀 생성 중이나 재구성 중 풀이 자동으로 핑되어 오류가 발생한 속성값을 식별하고 경고합니다. 이 옵션은 기본적으로 사용 안함으로 설정되어 있습니다.

설명

접속 풀에 대한 간단한 설명입니다.

초기 및 최소 풀 크기

풀의 최소 접속 수입니다. 풀 값에 따라 풀을 먼저 생성하거나 GlassFish Server을(를) 시작할 때 풀에 있는 접속 수도 결정합니다. 기본값은 8입니다.

최대 풀 크기

풀의 최대 접속 수입니다. 기본값은 32입니다.

풀 크기 조정 개수

풀이 각각 최대 및 최소 풀 크기로 확장되거나 축소되는 경우 크기가 일괄적으로 조정됩니다. 이 값은 일괄적으로 처리할 접속 수를 지정합니다. 이 값을 너무 크게 하면 접속 생성 및 재순환이 지체되며, 너무 작게 하면 효율성이 떨어집니다. 기본값은 2입니다.

유휴 시간 초과

풀에서 접속이 유휴 상태로 있을 수 있는 최대 시간(초)입니다. 이 시간이 만료되면 접속이 풀에서 제거됩니다. 기본값은 300입니다.

최대 대기 시간

접속을 요청한 응용 프로그램이 접속 시간이 초과되기 전까지 대기하는 최대 시간(밀리초)입니다. 기본값은 60,000입니다.

트랜잭션이 아닌 접속

이 옵션을 사용하도록 설정된 경우 트랜잭션이 아닌 접속이 반환됩니다. 기본적으로 이 옵션은 지정되어 있지 않습니다.

트랜잭션 격리

트랜잭션 격리 레벨을 지정하는 경우 이 풀의 접속이 지정된 레벨로 실행됩니다. 그렇지 않으면 접속은 JDBC 드라이버에서 제공하는 기본 격리 레벨로 실행됩니다. 기본적으로 이 옵션은 지정되어 있지 않습니다.

격리 레벨

보장 체크 박스를 선택한 경우 풀에서 가져온 모든 접속은 동일한 격리 레벨을 갖게 됩니다. 예를 들어, 마지막으로 사용할 때 접속의 격리 레벨을 프로그래밍 방식으로 변경한 경우(예: con.setTransactionIsolation 사용) 이 방식은 상태를 지정한 격리 레벨로 다시 변경합니다.

이 필드는 트랜잭션 격리 레벨이 지정된 경우에만 적용 가능합니다.
