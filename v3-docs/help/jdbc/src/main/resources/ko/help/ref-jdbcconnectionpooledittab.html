연결 풀 편집

연결 풀 편집 페이지에서는 JDBC 연결 풀의 일반 등록 정보를 편집할 수 있습니다.

연결 풀 편집 페이지에는 다음 옵션이 포함되어 있습니다.

기본값 로드

모든 설정의 기본값을 복원하는 버튼입니다.

플러시

연결 풀의 연결을 다시 초기화하는 버튼입니다. 이 버튼을 누르면 기존의 모든 연결이 삭제되고 풀에 설정된 연결이 다시 만들어진 후 풀이 초기 및 최소 풀 크기로 복원됩니다.

Ping

연결 풀을 사용할 수 있는지 확인하는 버튼입니다. 이 버튼을 누를 때 오류 메시지가 나타나면 데이터베이스 서버가 시작되었는지 확인합니다.

풀 이름

연결 풀의 이름입니다. 읽기 전용 필드입니다. JDBC 연결 풀을 만들 경우 풀 이름만 지정할 수 있습니다.

자원 유형

선택 항목에는 javax.sql.XADataSource(전역 트랜잭션), java.sql.ConnectionPoolDataSource(로컬 트랜잭션, 성능 향상 가능), javax.sql.DataSource(로컬 트랜잭션만) 및 java.sql.Driver가 있습니다.

데이터 소스 클래스 이름

데이터 소스를 구현하는 공급업체별 클래스 이름입니다.

드라이버 클래스 이름

공급업체에서 제공한 JDBC 드라이버 클래스 이름입니다. java.sql.Driver 자원 유형을 선택한 경우 일반적으로 이 필드에는 선택한 드라이버 및 데이터베이스 공급업체에 연관된 클래스 이름이 자동으로 입력됩니다. 데이터 소스 자원 유형 중 하나를 선택한 경우 이 필드는 비활성화됩니다.

Ping

이 옵션이 선택된 경우,풀 생성 중이나 재구성 중 풀이 자동으로 ping되어 오류가 발생한 속성 값을 식별하고 경고합니다. 이 옵션은 기본적으로 비활성화됩니다.

설명

연결 풀에 대한 간단한 설명입니다.

초기 및 최소 풀 크기

풀의 최소 연결 수입니다. 풀 값에 따라 풀을 먼저 만들거나 GlassFish 서버를 시작할 때 풀에 있는 연결 수도 결정합니다. 기본값은 8입니다.

최대 풀 크기

풀의 최대 연결 수입니다. 기본값은 32입니다.

풀 크기 조정 개수

풀이 각각 최대 및 최소 풀 크기로 확장되거나 축소되는 경우 크기 일괄적으로 조정됩니다. 이 값은 일괄적으로 처리할 연결 수를 지정합니다. 이 값을 너무 크게 하면 연결 생성 및 재순환이 지체되며, 너무 작게 하면 효율성이 떨어집니다. 기본값은 2입니다.

유휴 시간 초과

풀에서 연결이 유휴 상태로 있을 수 있는 최대 시간(초)입니다. 이 시간이 만료되면 연결이 풀에서 제거됩니다. 기본값은 300입니다.

최대 대기 시간

연결을 요청한 응용 프로그램이 연결 시간이 초과되기 전까지 대기하는 최대 시간(밀리초)입니다. 기본값은 60,000입니다.

트랜잭션이 아닌 연결

이 옵션을 사용하도록 설정된 경우 트랜잭션이 아닌 연결이 반환됩니다. 기본적으로 이 옵션은 지정되어 있지 않습니다.

트랜잭션 격리

트랜잭션 격리 수준을 지정하는 경우 이 풀의 연결이 지정된 수준으로 실행됩니다. 그렇지 않으면 연결은 JDBC 드라이버에서 제공하는 기본 격리 수준으로 실행됩니다. 기본적으로 이 옵션은 지정되어 있지 않습니다.

격리 수준

보장 확인란을 선택한 경우 풀에서 가져온 모든 연결은 동일한 격리 수준을 갖게 됩니다. 예를 들어, 마지막으로 사용할 때 연결의 격리 수준을 프로그래밍 방식으로 변경한 경우(예: con.setTransactionIsolation 사용) 이 메커니즘은 상태를 지정한 격리 수준으로 다시 변경합니다.

이 필드는 트랜잭션 격리 수준이 지정된 경우에만 적용 가능합니다.
