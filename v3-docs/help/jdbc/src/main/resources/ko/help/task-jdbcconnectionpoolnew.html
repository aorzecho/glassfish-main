JDBC 연결 풀을 만드는 방법

관리 콘솔에서 연결 풀을 만들 때 특정 데이터베이스에 대한 연결 부분을 정의합니다.

시작하기 전에

연결 풀을 만들려면 먼저 JDBC API를 지원하는 드라이버(JDBC 드라이버)를 설치 및 통합해야 합니다. JDBC 드라이버를 통합하는 방법을 참조하십시오.

JDBC 연결 풀을 만들 때 필요한 정보는 다음과 같습니다. 계속하기 전에 이 정보를 수집합니다:

  • 데이터베이스 공급업체 이름

  • 자원 유형. 예: javax.sql.DataSource(로컬 트랜잭션만), java.sql.ConnectionPoolDataSource(로컬 트랜잭션, 성능 향상 가능), javax.sql.XADataSource(전역 트랜잭션) 및 java.sql.Driver

  • 데이터 소스 클래스 이름

  • 필수 등록 정보: 데이터베이스 이름(URL), 사용자 이름 및 암호 등

  1. 탐색 트리에서 일반 작업 노드를 선택합니다.

    일반 작업 페이지가 열립니다.

  2. 일반 작업 페이지에서 새 JDBC 연결 풀 만들기 버튼을 누릅니다.

    새 JDBC 연결 풀 만들기(단계 1/2) 페이지가 열립니다.

    주 - 자원 노드와 JDBC 노드를 확장하고 연결 풀 노드를 선택하여 새 JDBC 연결 풀 페이지에 액세스할 수도 있습니다. 그런 다음 연결 풀 페이지에서 새로 만들기를 눌러 새 연결 풀을 만듭니다.

  3. 풀 이름 필드에 새 연결 풀의 이름을 입력합니다.

    필수 필드입니다.

  4. 자원 유형 드롭다운 목록에서 자원 유형을 선택합니다.

    선택 항목에는 javax.sql.XADataSource(전역 트랜잭션), java.sql.ConnectionPoolDataSource(로컬 트랜잭션, 성능 향상 가능), javax.sql.DataSource(로컬 트랜잭션만) 및 java.sql.Driver가 있습니다.

  5. 다음 작업 중 하나를 수행하여 데이터베이스 공급업체를 지정합니다.
    • 데이터베이스 공급업체 드롭다운 목록에서 데이터베이스 공급업체를 선택합니다.
    • 데이터베이스 공급업체가 목록에 없으면 데이터베이스 공급업체 텍스트 필드에 공급업체 이름을 입력합니다.
  6. 다음을 누릅니다.

    새 JDBC 연결 풀 만들기(단계 2/2) 페이지가 열립니다.

  7. 데이터 소스 클래스 이름 필드에 DataSource, ConnectionPoolDataSource 또는 XADataSource API를 구현하는 공급업체별 클래스 이름을 입력합니다.

    데이터 소스 자원 유형 중 하나를 선택한 경우 일반적으로 이 필드에는 선택한 자원 유형 및 데이터베이스 공급업체에 연관된 클래스 이름이 자동으로 입력됩니다. java.sql.Driver 자원 유형을 선택한 경우 이 필드는 비활성화됩니다.

  8. 드라이버 클래스 이름 필드에 java.sql.Driver API를 구현하는 공급업체별 클래스 이름을 입력합니다.

    java.sql.Driver 자원 유형을 선택한 경우 일반적으로 이 필드에는 선택한 드라이버 및 데이터베이스 공급업체에 연관된 클래스 이름이 자동으로 입력됩니다. 데이터 소스 자원 유형 중 하나를 선택한 경우 이 필드는 비활성화됩니다.

  9. 풀 생성 중 또는 재구성 중 풀을 자동으로 ping할 수 있도록 Ping 사용 가능 확인란을 선택하여 오류가 있는 속성 값을 확인하고 경고합니다.

    이 옵션은 기본적으로 비활성화됩니다.

  10. 설명 필드에 연결 풀에 대한 간단한 설명을 입력합니다.
  11. 초기 및 최소 풀 크기 필드에 이 풀의 최소 연결 수를 입력합니다.

    풀 값에 따라 풀을 먼저 만들거나 GlassFish Server를 시작할 때 풀에 있는 연결 수도 결정합니다. 기본값은 8입니다.

  12. 최대 풀 크기 필드에 이 풀의 최대 연결 수를 입력합니다.

    기본값은 32입니다.

  13. 풀 크기 조정 개수 필드에 풀 크기 조정 일괄 작업당 연결 수를 입력합니다.

    풀이 각각 최대 및 최소 풀 크기로 확장되거나 축소되는 경우 크기 일괄적으로 조정됩니다. 이 값은 일괄적으로 처리할 연결 수를 지정합니다. 이 값을 너무 크게 하면 연결 생성 및 재순환이 지체되며, 너무 작게 하면 효율성이 떨어집니다. 기본값은 2입니다.

  14. 유휴 시간 초과 필드에 풀에서 연결이 유휴 상태로 유지될 수 있는 최대 시간(초)을 입력합니다.

    이 시간이 만료되면 연결이 풀에서 제거됩니다. 기본값은 300입니다.

  15. 최대 대기 시간 필드에 연결을 요청하는 응용 프로그램이 연결 시간이 초과될 때까지 대기할 최대 시간(밀리초)을 입력합니다.

    기본값은 60,000입니다.

  16. 트랜잭션이 아닌 연결로 돌아가려면 트랜잭션이 아닌 연결 사용 가능 확인란을 선택합니다.

    이 옵션은 기본적으로 비활성화됩니다.

  17. 트랜잭션 격리 수준 드롭다운 목록에서 이 연결 풀의 트랜잭션 격리 수준을 선택합니다.

    지정하지 않는 경우 연결은 JDBC 드라이버에서 제공하는 기본 격리 수준으로 실행됩니다. 기본적으로 이 옵션은 지정되어 있지 않습니다.

  18. 격리 수준 보장 확인란을 선택하여 풀에서 가져온 모든 연결의 격리 수준이 동일한지 확인합니다.

    예를 들어, 마지막으로 사용 시 연결의 격리 수준을 프로그래밍 방식으로 변경한 경우(예: con.setTransactionIsolation 사용) 이 메커니즘은 상태를 지정한 격리 수준으로 다시 변경합니다. 이 옵션은 기본적으로 활성화됩니다.

    이 설정은 트랜잭션 격리 수준을 지정한 경우에만 적용됩니다.

  19. (옵션) 추가 등록 정보 섹션에서 추가 등록 정보를 지정합니다.

    필요한 등록 정보의 예로는 데이터베이스 이름(URL), 사용자 이름, 암호 등이 있습니다. 일반적으로 테이블에는 데이터베이스 공급업체에서 사용하는 등록 정보가 포함되어 있습니다. 사용 가능한 일부 등록 정보에 대한 내용은 JDBC 연결 풀과 관련된 등록 정보를 참조하십시오.

    등록 정보 값을 지정하지 않으면 연결 풀 만들기를 완료할 경우 이 값이 테이블에서 제거됩니다. 현재는 등록 정보 값을 설정하지 않지만 나중에 설정할 수 있도록 등록 정보를 테이블에 유지하려면 값 필드에 일련의 빈 괄호를 입력합니다.

    ()
    • 등록 정보를 추가하려면 등록 정보 추가 버튼을 누릅니다. 빈 행이 표시되면 이름 필드에 등록 정보 이름을 입력하고 값 필드에 등록 정보 값을 입력합니다.
    • 등록 정보를 수정하려면 등록 정보의 값 필드를 편집합니다.
    • 등록 정보를 삭제하려면 삭제할 등록 정보의 이름 필드 왼쪽에 있는 확인란을 선택한 다음 등록 정보 삭제 버튼을 누릅니다.
  20. 마침을 누릅니다.
