새 JDBC 연결 풀(단계 2/2)

새 JDBC 연결 풀(단계 2/2) 페이지에서는 새 JDBC 연결 풀을 만들 수 있습니다.

새 JDBC 연결 풀(단계 2/2) 페이지에는 다음 옵션이 포함되어 있습니다.

데이터 소스 클래스 이름

DataSource, ConnectionPoolDataSource 또는 XADataSource API를 구현하는 공급업체별 클래스 이름입니다. 데이터 소스 자원 유형 중 하나를 선택한 경우 일반적으로 이 필드에는 선택한 자원 유형 및 데이터베이스 공급업체에 연관된 클래스 이름이 자동으로 입력됩니다. java.sql.Driver 자원 유형을 선택한 경우 이 필드는 비활성화됩니다.

드라이버 클래스 이름

공급업체에서 제공한 JDBC 드라이버 클래스 이름입니다. java.sql.Driver 자원 유형을 선택한 경우 일반적으로 이 필드에는 선택한 드라이버 및 데이터베이스 공급업체에 연관된 클래스 이름이 자동으로 입력됩니다. 데이터 소스 자원 유형 중 하나를 선택한 경우 이 필드는 비활성화됩니다.

Ping

이 옵션이 선택된 경우,풀 생성 중이나 재구성 중 풀이 자동으로 ping되어 오류가 발생한 속성 값을 식별하고 경고합니다. 이 옵션은 기본적으로 비활성화됩니다.

설명

연결 풀에 대한 간단한 설명입니다.

초기 및 최소 풀 크기

풀의 최소 연결 수입니다. 풀 값에 따라 풀을 먼저 만들거나 GlassFish 서버를 시작할 때 풀에 있는 연결 수도 결정합니다. 기본값은 8입니다.

최대 풀 크기

풀의 최대 연결 수입니다. 기본값은 32입니다.

풀 크기 조정 개수

풀이 각각 최대 및 최소 풀 크기로 확장되거나 축소되는 경우 크기 일괄적으로 조정됩니다. 이 값은 일괄적으로 처리할 연결 수를 지정합니다. 이 값을 너무 크게 하면 연결 생성 및 재순환이 지체되며, 너무 작게 하면 효율성이 떨어집니다. 기본값은 2입니다.

유휴 시간 초과

풀에서 연결이 유휴 상태로 있을 수 있는 최대 시간(초)입니다. 이 시간이 만료되면 연결이 풀에서 제거됩니다. 기본값은 300입니다.

최대 대기 시간

연결을 요청한 응용 프로그램이 연결 시간이 초과하기 전까지 대기하는 최대 시간(밀리초)입니다. 기본값은 60,000입니다.

트랜잭션이 아닌 연결

이 옵션을 사용하도록 설정된 경우 트랜잭션이 아닌 연결이 반환됩니다.

트랜잭션 격리

트랜잭션 격리 수준을 지정하는 경우 이 풀의 연결이 지정된 수준으로 실행됩니다. 그렇지 않으면 연결은 JDBC 드라이버에서 제공하는 기본 격리 수준으로 실행됩니다. 기본적으로 이 옵션은 지정되어 있지 않습니다.

격리 수준

보장 확인란이 선택된 경우 풀에서 가져온 모든 연결은 동일한 격리 수준을 갖게 됩니다. 예를 들어, 마지막으로 사용할 때 연결의 격리 수준을 프로그래밍 방식으로 변경한 경우(예: con.setTransactionIsolation 사용) 이 메커니즘은 상태를 지정한 격리 수준으로 다시 변경합니다. 이 옵션은 기본적으로 활성화됩니다.

이 설정은 트랜잭션 격리 수준을 지정한 경우에만 적용됩니다.

추가 등록 정보

JDBC 연결 풀의 추가 등록 정보입니다. 이 등록 정보에는 데이터베이스 이름(URL), 사용자 이름 및 암호가 포함될 수 있습니다. 일반적으로 테이블에는 데이터베이스 공급업체에서 사용하는 등록 정보가 포함되어 있습니다.

사용 가능한 일부 등록 정보에 대한 설명은 JDBC 연결 풀과 관련된 등록 정보를 참조하십시오.

등록 정보 값을 지정하지 않으면 연결 풀 만들기를 완료할 경우 이 값이 테이블에서 제거됩니다. 현재는 등록 정보 값을 설정하지 않지만 나중에 설정할 수 있도록 등록 정보를 테이블에 유지하려면 값 필드에 일련의 빈 괄호를 입력합니다.
