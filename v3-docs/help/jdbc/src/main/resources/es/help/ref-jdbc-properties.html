Propiedades específicas de los conjuntos de conexiones JDBC

La mayoría de los controladores JDBC permiten utilizar listas de propiedades estándar para especificar el usuario, la contraseña y otra información de configuración de recursos. Aunque las propiedades son opcionales con respecto a GlassFish Server, es posible que algunas propiedades sean necesarias para la mayoría de bases de datos. Para obtener detalles, consulte la API de extensión estándar JDBC 4.0.

Se necesita reiniciar el servidor para cambiar las propiedades del controlador JDBC.

Las siguientes propiedades estándar son comunes a muchos proveedores JDBC. Para obtener información sobre las propiedades disponibles para su controlador JDBC, consulte la documentación del proveedor.

user

Especifica el nombre de usuario para conectarse a la base de datos.

password

Especifica la contraseña para conectarse a la base de datos.

databaseName

Especifica la base de datos de este conjunto de conexiones.

serverName

Especifica el servidor de la base de datos de este conjunto de conexiones.

port

Especifica el puerto en el que el servidor de la base de datos escucha las solicitudes.

networkProtocol

Especifica el protocolo de comunicación.

roleName

Especifica el nombre del función SQL inicial.

datasourceName

Especifica un XADataSource subyacente, o una ConnectionPoolDataSource si se ha realizado el conjunto de conexión.

description

Especifica una descripción de texto.

url

Especifica la URL para este conjunto de conexiones. Aunque no es una propiedad estándar, se utiliza habitualmente.
