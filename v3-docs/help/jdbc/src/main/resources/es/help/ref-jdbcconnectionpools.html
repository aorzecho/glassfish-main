Conjuntos de conexiones

Un conjunto de conexiones JDBC contiene un grupo de conexiones JDBC que se crean una vez registrado el conjunto de conexiones. Los conjuntos de conexiones utilizan un controlador JDBC para crear conexiones de bases de datos físicas. Su aplicación pide prestada una conexión al conjunto, la utiliza y, a continuación, se la devuelve al conjunto cerrándola. Para crear un recurso JDBC, especifique el conjunto de conexiones asociado. Varios recursos JDBC pueden especificar un único conjunto de conexiones.

Un conjunto de conexiones JDBC es un grupo de conexiones reutilizables para una determinada base de datos. Como la creación de cada nueva conexión física lleva mucho tiempo, el servidor mantiene un conjunto de conexiones disponibles para mejorar el rendimiento. Cuando una aplicación solicita una conexión, obtiene una del conjunto. Cuando una aplicación cierra una conexión, ésta se devuelve al conjunto.

Cuando se crea un conjunto de conexiones con Consola de administración, está definiendo los aspectos de una conexión con una base de datos específica.

Utilice la página Conjuntos de conexiones JDBC para configurar conjuntos de conexiones JDBC.

Para cada conjunto de conexiones se proporciona la siguiente información.

Nombre de conjunto

El nombre del conjunto de conexiones.

Tipo de recurso

El tipo de recurso del conjunto de conexiones.

Nombre de clase

El nombre de clase específico del proveedor que implementa las API javax.sql.XADataSource, java.sql.ConnectionPoolDataSource, javax.sql.DataSource o java.sql.Driver.

Descripción

Una breve descripción del conjunto de conexiones.

La tabla Conjuntos también contiene las siguientes opciones.

Nuevo

Botón para crear un nuevo conjunto de conexiones.

Eliminar

Botón para eliminar uno o más conjuntos de conexiones seleccionados.
