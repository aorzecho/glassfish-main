JDBC リソース

Java DataBase Connectivity (JDBC) リソース (データソース) は、データベースへの接続手段をアプリケーションに提供します。通常、管理者はドメイン内に配備されたアプリケーションがアクセスするデータベースごとに 1 つの JDBC リソースを作成します。ただし、1 つのデータベースに対して複数の JDBC リソースを作成できます。

アプリケーションは、Java Naming and Directory Interface (JNDI) API ツリーのデータソースを検索して接続プールからデータベース接続を取得してから、接続を要求します。データソースに関連付けられている接続プールが、アプリケーションへの接続を提供します。

「JDBC リソース」ページを使用して、JDBC リソースを設定します。

リソースごとに、次の情報が表示されます。

JNDI 名

JDBC リソースを識別する一意の名前。

使用可能

リソースが使用可能な場合は true、使用不可の場合は false です。

接続プール

リソースに関連付けられた JDBC 接続プール。

説明

リソースの簡単な説明。

「リソース」の表には、次のオプションも表示されます。

新規

新しいリソースを作成するボタン。

削除

選択した 1 つまたは複数のリソースを削除するボタン。

有効

選択した 1 つまたは複数のリソースを有効にするボタン。

無効

選択した 1 つまたは複数のリソースを無効にするボタン。
