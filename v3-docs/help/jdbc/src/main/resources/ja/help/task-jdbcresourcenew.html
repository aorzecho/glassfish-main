JDBC リソースを作成する

Java DataBase Connectivity (JDBC) リソース (データソース) は、データベースへの接続手段をアプリケーションに提供します。

始める前に

JDBC リソースを作成するには、まず JDBC 接続プールを作成する必要があります。「JDBC 接続プールを作成する」を参照してください。

  1. ナビゲーションツリーで、「リソース」ノードを展開します。
  2. 「リソース」ノードで、「JDBC」ノードを展開します。
  3. 「JDBC」ノードで、「JDBC リソース」ノードを選択します。

    「JDBC リソース」ページが開きます。

  4. 「JDBC リソース」ページで、「新規」をクリックします。

    「新しい JDBC リソース」ページが開きます。

  5. 「JNDI 名」フィールドに、一意のリソース名を入力します。

    カード式の目録によって図書館の蔵書の分類と保管場所の管理を行う場合と同様の方法で、分散コンピューティング環境に配置されたコンポーネントの編成と場所の管理が JNDI 名によって行われます。そのため、JNDI 名は JDBC リソースへの重要なアクセス方法となります。規則により、名前は jdbc/ 文字列で始まります。たとえば、jdbc/payrolldb などです。スラッシュを忘れないでください。

    名前に含めることができるのは、英数字、アンダースコア、ダッシュ、またはドットの各文字だけです。

  6. 「プール名」ドロップダウンリストから、新しい JDBC リソースに関連付ける接続プールを選択します。
  7. 「説明」フィールドで、新しい JDBC リソースの簡単な説明を入力します。
  8. 「状態を有効」チェックボックスを選択して、リソースを実行時に利用できるようにします。

    このオプションはデフォルトで有効になっています。

  9. (省略可能) 「追加プロパティー」セクションで、追加プロパティーを指定します。

    GlassFish サーバー は JDBC リソースの追加プロパティーを定義しません。

    プロパティーを追加するには、「プロパティーを追加」ボタンをクリックします。表示される空白行で、「名前」フィールドにプロパティー名を入力し、「値」フィールドにプロパティー値を入力します。

  10. 利用可能なターゲットを選択してターゲットを追加するか、「選択したターゲット」リストからターゲットを削除します。
  11. 「了解」をクリックします。
