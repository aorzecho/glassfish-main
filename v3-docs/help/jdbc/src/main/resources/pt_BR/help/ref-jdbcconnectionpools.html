Conjuntos de conexões

Um conjunto de conexões do JDBC contém um grupo de conexões JDBC; essas conexões são criadas quando o conjunto de conexões é registrado. Os conjuntos de conexões usam um driver JDBC para criar conexões de bancos de dados físicas. O aplicativo empresta uma conexão do conjunto, faz uso dela e a devolve ao conjunto, fechando-a. Para criar um recurso do JDBC, especifique o conjunto de conexões associado. Vários recursos do JDBC podem especificar um único conjunto de conexões.

Um conjunto de conexões do JDBC é um grupo de conexões reutilizáveis para um determinado banco de dados. A criação de cada nova conexão física consome muito tempo, por isso, o servidor mantém um conjunto de conexões disponíveis para melhorar o desempenho. Quando um aplicativo solicita uma conexão, ele a obtém do conjunto. Quando um aplicativo fecha uma conexão, ele a devolve ao conjunto.

Ao criar um conjunto de conexões com o Console de Administração, você define os aspectos de uma conexão para um banco de dados específico.

Use a página Conjuntos de conexões JDBC para configurar conjuntos de conexões JDBC.

Para cada conjunto de conexões, as seguintes informações são fornecidas.

Nome do conjunto

O nome do conjunto de conexões.

Tipo de recurso

O tipo de recurso do conjunto de conexões.

Nome da classe

O nome da classe específica do fornecedor que implementa a API javax.sql.XADataSource, java.sql.ConnectionPoolDataSource, javax.sql.DataSource ou java.sql.Driver.

Descrição

Uma breve descrição do conjunto de conexões.

A tabela Conjuntos também contém as seguintes opções.

Novo

Botão para criar um novo conjunto de conexões.

Excluir

Botão para excluir um ou mais conjuntos de conexões selecionados.
