Pools de Conexões

Um pool de conexões JDBC contém um grupo de conexões JDBC que são criadas quando o pool de conexões é registrado. Os pools de conexões usam um driver JDBC para criar conexões de bancos de dados físicas. A aplicação obtém uma conexão do pool, faz uso dela e a devolve ao pool, fechando-a. Para criar um recurso JDBC, especifique o pool de conexões associado. Vários recursos JDBC podem especificar um único pool de conexões.

Um pool de conexões JDBC é um grupo de conexões reutilizáveis para determinado banco de dados. A criação de cada nova conexão física consome muito tempo, por isso, o servidor mantém um pool de conexões disponíveis para melhorar o desempenho. Quando uma aplicação solicita uma conexão, ele a obtém do pool. Quando uma aplicação fecha uma conexão, ele a devolve ao pool.

Ao criar um pool de conexões com a Console de Administração, você define os aspectos de uma conexão para um banco de dados específico.

Use a página Pools de Conexões JDBC para configurar pools de conexões JDBC.

Para cada pool de conexões, as informações a seguir são apresentadas.

Nome do Pool

O nome do pool de conexões.

Tipo de recurso

O tipo de recurso do pool de conexões.

Nome da classe

O nome da classe específica do fornecedor que implementa a API javax.sql.XADataSource, java.sql.ConnectionPoolDataSource, javax.sql.DataSource ou java.sql.Driver.

Descrição

Uma breve descrição do pool de conexões.

A tabela Pools também contém as seguintes opções.

Novo

Botão para criar um novo pool de conexões.

Deletar

Botão para deletar um ou mais pools de conexões selecionados.
