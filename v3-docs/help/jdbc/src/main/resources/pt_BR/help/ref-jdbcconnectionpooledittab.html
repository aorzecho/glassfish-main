Editar conjuntos de conexões

Use a página Editar conjunto de conexões para editar as propriedades gerais de um conjunto de conexões do JDBC.

A página Editar conjunto de conexões contém essas opções.

Carregar padrões

Botão para restaurar os valores padrão de todas as configurações.

Descarregar

Botão para reinicializar as conexões no conjunto de conexões. Clicar nesse botão destrói quaisquer conexões existentes, recria as conexões estabelecidas para o conjunto e restaura o conjunto para seu tamanho inicial e mínimo.

Ping

Botão para verificar que o conjunto de conexões é utilizável. Se uma mensagem de erro for exibida quando você clica nesse botão, verifique se o servidor de banco de dados foi iniciado.

Nome do conjunto

O nome do conjunto de conexões. Esse é um campo somente leitura. Você só poderá especificar o nome do conjunto quando criar um conjunto de conexões do JDBC.

Tipo de recurso

As opções incluem javax.sql.XADataSource (transações globais), java.sql.ConnectionPoolDataSource (transações locais, possíveis melhorias de desempenho) e javax.sql.DataSource (somente transações locais)java.sql.Driver.

Nome de Classe da Fonte de Dados

O nome de classe específico do fornecedor que implementa a fonte de dados.

Nome de classe do driver

O nome da classe do driver JDBC suprido pelo fornecedor. Se você selecionou o tipo de recurso java.sql.Driver, normalmente esse campo é automaticamente preenchido com o nome da classe associado ao driver do fornecedor de banco de dados de sua escolha. Se você selecionou um dos tipos de recursos de origem de dados, esse campo é desativado.

Ping

Se essa opção for selecionada, será efetuado o ping automático no conjunto durante a criação ou reconfiguração, para identificar e avisar sobre valores de atributos com erros. Por padrão, a opção está desativada.

Descrição

Uma breve descrição do conjunto de conexões.

Tamanho inicial e mínimo do conjunto

O número mínimo de conexões no conjunto. Esse valor também determina o número de conexões colocadas no conjunto quando o conjunto é criado ou quando o Servidor GlassFish é iniciado. O valor padrão é 8.

Tamanho máximo do conjunto

O número máximo de conexões no conjunto. O valor padrão é 32.

Quantidade de redimensionamento do conjunto

Quando o conjunto aumente e diminui em direção ao tamanho máximo e mínimo, respectivamente, o conjunto é redimensionado em lotes. Esse valor determina o número de conexões do lote. A definição de um valor muito alto torna demorado o processo de criação e reciclagem de conexões; um valor muito baixo compromete a eficiência. O valor padrão é 2.

Tempo limite de inatividade

A quantidade máxima de tempo, em segundos, que uma conexão pode permanecer inativa no conjunto. Após esse período, a conexão será removida do conjunto. O valor padrão é 300.

Tempo máx. de espera

A quantidade máxima de tempo, em milissegundos, que o aplicativo solicitante da conexão aguardará até que o tempo limite da conexão seja atingido. O valor padrão é 60.000.

Conexões não transacionais

Se essa opção for ativada, as conexões não transacionais serão retornadas. Por padrão essa opção não é especificada.

Isolamento de transação

Se um nível de isolamento de transação for especificado, as conexões desse conjunto funcionarão no nível especificado. Caso contrário, as conexões funcionarão com os níveis de isolamento padrão, fornecidos pelo driver JDBC. Por padrão essa opção não é especificada.

Nível de isolamento

Se a caixa de seleção Garantido for marcada, todas as conexões obtidas no conjunto terão o mesmo nível de isolamento. Por exemplo, se o nível de isolamento da conexão for alterado via programação (com con.setTransactionIsolation), durante a última utilização, esse mecanismo alterará o status para o nível de isolamento especificado.

Esse campo somente é aplicável se um nível de isolamento de transação for especificado.
