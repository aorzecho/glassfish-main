Novo recurso do JDBC

Use a página Novo recurso do JDBC para criá-lo.

A página Novo recurso do JDBC contém essas opções.

Nome JNDI

Um nome único que identifica o recurso JDBC. O nome JNDI organiza e localiza os componentes em um ambiente de computação distribuído, de maneira semelhante ao catálogo de cartão que organiza e representa os locais de livros em uma biblioteca. Consequentemente, o nome JNDI se torna um método importante de acesso ao recurso JDBC. Por convenção, o nome começa com a sequência jdbc/; por exemplo, jdbc/payrolldb. Não se esqueça da barra. O nome deve conter somente caracteres alfanuméricos, sublinhado, traço ou ponto.

Nome do conjunto

O conjunto de conexões associado ao recurso do JDBC.

Descrição

Uma breve descrição do recurso do JDBC.

Status

Se essa opção for ativada, o recurso estará disponível no momento da execução. Por padrão, a opção está ativada.

Propriedades adicionais

Propriedades adicionais para o recurso JDBC. O Servidor GlassFish não define nenhuma propriedade adicional para os recursos JDBC.

Destinos

Clusters e instâncias independentes do recurso JDBC. Mova os destinos desejados para a coluna Destinos selecionados utilizando os botões Adicionar e Adicionar tudo. Mova os destinos desnecessários para a coluna Destinos disponíveis utilizando os botões Remover e Remover tudo.

Esta opção é exibida somente se os clusters ou instâncias independentes tiverem sido criados no domínio.
