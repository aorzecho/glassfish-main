Novo Recurso do JBDC

Use a página Novo Recurso JDBC para criá-lo.

A página Novo Recurso JDBC contém estas opções.

Nome JNDI

Um nome único que identifica o recurso JDBC. O nome JNDI organiza e localiza os componentes em um ambiente de computação distribuído, de maneira semelhante ao catálogo de cartão que organiza e representa os locais de livros em uma biblioteca. Consequentemente, o nome JNDI se torna um método importante de acesso ao recurso JDBC. Por convenção, o nome começa com a string jdbc/, por exemplo, jdbc/payrolldb. Não se esqueça de usar a barra para a direita. O nome deve conter somente caracteres alfanuméricos, sublinhado, traço ou ponto.

Nome do conjunto

O pool de conexões associado ao recurso JDBC.

Descrição

Uma breve descrição do recurso JDBC.

Status

Se essa opção for ativada, o recurso estará disponível no momento da execução. Por default, a opção está ativada.

Propriedades adicionais

Propriedades adicionais para o recurso JDBC. O GlassFish Server não define nenhuma propriedade adicional para os recursos JDBC.

Destinos

Clusters e instâncias standalone do recurso JDBC. Mova os destinos desejados para a coluna Destinos selecionados utilizando os botões Adicionar e Adicionar tudo. Mova os destinos desnecessários para a coluna Destinos disponíveis utilizando os botões Remover e Remover tudo.

Esta opção é exibida somente se os clusters ou instâncias standalone tiverem sido criados no domínio.
