JDBC-Ressourcen

Eine Java DataBase Connectivity (JDBC)-Ressource (Datenquelle) bietet Anwendungen über die Verbindung zu einer Datenbank. Normalerweise erstellt der Administrator eine JDBC-Ressource für jede Datenbank, auf die über eine in einer Domäne bereitgestellte Anwendung zugegriffen wird. Allerdings können mehrere JDBC-Ressourcen für eine Datenbank erstellt werden.

Anwendungen erhalten eine Datenbankverbindung aus einem Verbindungspool, indem sie in der Java Naming and Directory Interface (JNDI) API-Struktur nach einer Datenquelle suchen und dann eine Verbindung anfordern. Der mit der Datenquelle verknüpfte Verbindungspool stellt der Anwendung eine Verbindung bereit.

Auf der Seite "JDBC-Ressourcen" können Sie JDBC-Ressourcen konfigurieren.

Für jede Ressourcen werden die folgenden Informationen zur Verfügung gestellt.

JNDI Name (JNDI-Name)

Ein eindeutiger Name, der die JDBC-Ressource identifiziert.

Enabled (Aktiviert)

Wahr, wenn die Ressource aktiviert ist. Falsch, wenn sie deaktiviert ist.

Connection Pool (Verbindungspool)

Der mit der Ressource verbundene JDBC-Verbindungspool.

Description (Beschreibung)

Eine kurze Beschreibung der Ressource.

Die Tabelle "Ressourcen" enthält darüber hinaus die folgenden Optionen.

Neu

Schaltfläche zum Erstellen einer neuen Ressource.

Löschen

Schaltfläche zum Löschen einer oder mehrerer ausgewählter Ressourcen.

Enable (Aktivieren)

Schaltfläche zum Aktivieren einer oder mehrerer ausgewählter Ressourcen.

Disable (Deaktivieren)

Schaltfläche zum Deaktivieren einer oder mehrerer ausgewählter Ressourcen.

Verwandte Aufgaben
