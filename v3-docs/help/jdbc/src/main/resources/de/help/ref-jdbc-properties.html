Für JDBC-Connection Pools spezifische Eigenschaften

Die meisten JDBC-Treiber gestatten die Verwendung standardmäßiger Eigenschaftslisten, um den Benutzer, das Kennwort und andere Informationen zur Ressourcenkonfiguration anzugeben. Obwohl Eigenschaften in Bezug auf den GlassFish Server optional sind, sind einige Eigenschaften für die meisten Datenbanken möglicherweise erforderlich. Weitere Informationen finden Sie im Abschnitt zur JDBC 4.0 Standard Extension API.

Das Ändern der JDBC-Treibereigenschaften macht einen Server-Neustart erforderlich.

Die folgenden Standardeigenschaften sind bei vielen bekannten JDBC-Providern gleich. Informationen zu den für Ihren JDBC-Treiber verfügbaren Eigenschaften erhalten Sie in der Dokumentation des entsprechenden Providers.

user (Benutzer)

Der Benutzername für die Verbindung mit der Datenbank.

Kennwort

Das Kennwort für die Verbindung mit der Datenbank.

databaseName (Datenbankname)

Die Datenbank für diesen Connection Pool.

serverName (Servername)

Der Datenbankserver für diesen Connection Pool.

port (Port)

Der Port, auf dem der Datenbankserver auf Anforderungen horcht.

networkProtocol (Netzwerkprotokoll)

Das Kommunikationsprotokoll.

roleName (Rollenname)

Gibt den ursprünglichen SQL-Rollennamen an.

datasourceName (Datenquellenname)

Gibt eine zugrunde liegendes XADataSource oder ein ConnectionPoolDataSource an, wenn Connection Pools verwendet werden.

Description (Beschreibung)

Gibt eine Textbeschreibung an.

url

Die URL für diesen Connection Pool. Obwohl dies keine Standardeigenschaft ist, wird sie allgemein verwendet.
