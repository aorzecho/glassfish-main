Spezifische Eigenschaften von JDBC-Verbindungspools

Die meisten JDBC-Treiber gestatten die Verwendung standardmäßiger Eigenschaftslisten, um den Benutzer, das Passwort und andere Informationen zur Ressourcenkonfiguration anzugeben. Obwohl Eigenschaften in Bezug auf den GlassFish-Server optional sind, sind einige Eigenschaften für die meisten Datenbanken möglicherweise erforderlich. Weitere Informationen finden Sie im Abschnitt zur JDBC 4.0 Standard Extension API.

Das Ändern der JDBC-Treibereigenschaften macht einen Server-Neustart erforderlich.

Die folgenden Standardeigenschaften sind bei vielen bekannten JDBC-Anbietern gleich. Informationen zu den für Ihren Treiber verfügbaren Eigenschaften erhalten Sie in der Dokumentation des entsprechenden Anbieters.

user (Benutzer)

Der Benutzername für die Verbindung mit der Datenbank.

password (Passwort)

Das Passwort für die Verbindung mit der Datenbank.

databaseName (Datenbankname)

Die Datenbank für diesen Verbindungspool.

serverName (Servername)

Der Datenbankserver für diesen Verbindungspool.

port (Port)

Der Port, den der Datenbankserver auf Anforderungen überwacht.

networkProtocol (Netzwerkprotokoll)

Das Kommunikationsprotokoll.

roleName (Rollenname)

Gibt den ursprünglichen SQL-Rollennamen an.

datasourceName (Datenquellenname)

Gibt eine zugrunde liegendes XADataSource oder ein ConnectionPoolDataSource an, wenn Verbindungspools verwendet werden.

Description (Beschreibung)

Gibt eine Textbeschreibung an.

url

Die URL für diesen Verbindungspool. Obwohl dies keine Standardeigenschaft ist, wird sie allgemein verwendet.
