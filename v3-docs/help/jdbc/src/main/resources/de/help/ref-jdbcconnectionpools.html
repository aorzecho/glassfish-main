Connection Pools

Ein JDBC-Connection Pool enthält eine Gruppe von JDBC-Verbindungen, die erstellt werden, wenn der Connection Pool registriert wird. Connection Pools verwenden einen JDBC-Treiber, um physische Datenbankverbindungen zu erstellen. Ihre Anwendung "leiht" eine Verbindung aus dem Pool, verwendet sie und gibt sie anschließend wieder an den Pool zurück, indem die Verbindung geschlossen wird. Um eine JDBC-Ressource zu erstellen, geben Sie den verknüpften Connection Pool an. Mehrere JDBC-Ressourcen können hierbei auf denselben Connection Pool verweisen.

Ein JDBC-Connection Pool ist eine Gruppe wiederverwendbarer Verbindungen für eine bestimmte Datenbank. Da die Erstellung jeder neuen physischen Verbindung Zeit kostet, verwaltet der Server einen Pool der verfügbaren Verbindungen, um die Performance zu steigern. Wenn eine Anwendung eine Verbindung anfordert, wird eine Verbindung aus dem Pool verwendet. Wenn eine Anwendung eine Verbindung schließt, wird die Verbindung wieder an den Pool zurückgegeben.

Wenn Sie den Pool mit der Administrationskonsole erstellen, definieren Sie die Eigenschaften einer Verbindung mit einer spezifischen Datenbank.

Auf der Seite "JDBC-Connection Pools" können Sie JDBC-Connection Pools konfigurieren.

Für jeden Connection Pool werden die folgenden Informationen zur Verfügung gestellt.

Pool Name (Poolname)

Der Name des Connection Pools.

Resource Type (Ressourcen-Typ)

Der Ressourcentyp des Connection Pools.

Klassenname

Der Name der herstellerspezifischen Klasse, die die API javax.sql.XADataSource, java.sql.ConnectionPoolDataSource, javax.sql.DataSource oder java.sql.Driver implementiert.

Beschreibung

Eine kurze Beschreibung des Connection Pools.

Die Tabelle "Pools" enthält darüber hinaus die folgenden Optionen.

Neu

Schaltfläche zum Erstellen eines neuen Connection Pools.

Löschen

Schaltfläche zum Löschen von mindestens einem ausgewählten Connection Pool.
