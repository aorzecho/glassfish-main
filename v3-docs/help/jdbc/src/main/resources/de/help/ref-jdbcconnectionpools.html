Verbindungspools

Ein JDBC-Verbindungspool enthält eine Gruppe von JDBC-Verbindungen, die erstellt werden, wenn der Verbindungspool registriert wird. Verbindungspools verwenden einen JDBC-Treiber, um physische Datenbankverbindungen zu erstellen. Ihre Anwendung "leiht" eine Verbindung aus dem Pool, verwendet sie und gibt sie anschließend wieder an den Pool zurück, indem die Verbindung geschlossen wird. Um eine JDBC-Ressource zu erstellen, geben Sie den verknüpften Verbindungspool an. Mehrere JDBC-Ressourcen können hierbei auf denselben Verbindungspool verweisen.

Ein JDBC-Verbindungspool ist eine Gruppe wiederverwendbarer Verbindungen für eine bestimmte Datenbank. Da die Erstellung jeder neuen physischen Verbindung Zeit kostet, verwaltet der Server einen Pool der verfügbaren Verbindungen, um die Leistung zu steigern. Wenn eine Anwendung eine Verbindung anfordert, wird eine Verbindung aus dem Pool verwendet. Wenn eine Anwendung eine Verbindung schließt, wird die Verbindung wieder an den Pool zurückgegeben.

Wenn Sie den Pool mit der Administration Console erstellen, definieren Sie die Eigenschaften einer Verbindung mit einer spezifischen Datenbank.

Auf der Seite "JDBC-Verbindungspools" können Sie JDBC-Verbindungspools konfigurieren.

Für jeden Verbindungspool werden die folgenden Informationen zur Verfügung gestellt.

Pool Name (Poolname)

Der Name des Verbindungspools.

Resource Type (Ressourcen-Typ)

Der Ressourcentyp des Verbindungspools.

Klassenname

Der Name der herstellerspezifischen Klasse, die die API javax.sql.XADataSource, java.sql.ConnectionPoolDataSource, javax.sql.DataSource oder java.sql.Driver implementiert.

Beschreibung

Eine kurze Beschreibung des Verbindungspools.

Die Tabelle "Pools" enthält darüber hinaus die folgenden Optionen.

Neu

Schaltfläche zum Erstellen eines neuen Verbindungspools.

Löschen

Schaltfläche zum Löschen eines oder mehrerer ausgewählter Verbindungspools.
